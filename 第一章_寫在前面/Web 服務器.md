# Web 服務器

> 所屬章節：第一章｜寫在前面
> 關鍵字：Web 伺服器、Web 服務器、伺服器、服務端、主機、HTTP、HTTPS、靜態資源、動態內容、後端應用、本地開發伺服器、遠端伺服器、部署、反向代理
> 建議回查情境：想理解網站為什麼需要部署到伺服器上、分不清 Web 伺服器是機器還是程式、想分清本地開發伺服器和遠端伺服器、想知道 Web 伺服器和後端應用有什麼關係

## 本節導讀

這一節要釐清 Web 入門裡很容易混在一起的幾件事：

- 網站為什麼需要伺服器
- `Web 伺服器` 到底是在指機器、程式，還是提供服務的角色
- Web 伺服器和後端應用有什麼關係
- 本地開發伺服器與遠端伺服器差在哪
- 靜態資源和動態內容在伺服器端如何被提供

閱讀時先抓住一句話：

**Web 伺服器的核心任務，是接收瀏覽器或其他客戶端送來的 HTTP/HTTPS 請求，並回傳對應的 Web 資源或處理結果。**

但要注意：

**「Web 伺服器」在日常語境裡可能指服務程式，也可能指承載服務的主機或雲端環境。**

## 你會在這篇學到什麼

- 網站為什麼通常需要部署到伺服器上
- `Web 伺服器` 的狹義與廣義意思
- Web 伺服器、後端應用、資料庫之間的關係
- 靜態資源與動態內容的差別
- 本地開發伺服器與遠端伺服器的用途差異
- 為什麼一般電腦也可以在某個情境下扮演伺服器
- 初學時怎麼判斷文章裡的「伺服器」到底指什麼

## 這篇在解決什麼問題？

初學時常見混亂有幾種：

- 以為伺服器一定是一台很特殊、很大的電腦。
- 以為 Web 伺服器一定只指硬體主機。
- 以為 Web 伺服器和後端程式是同一件事。
- 以為網站只要有 HTML 檔案就能讓所有人穩定訪問。
- 以為本地開發伺服器和正式上線伺服器只是位置不同，其他都一樣。
- 以為資料庫、後端、Web 伺服器可以直接混用。

這篇要做的是把幾個層次拆開：

- 角色：誰接收請求、回傳結果
- 程式：哪個服務程式在處理 Web 請求
- 主機 / 環境：服務跑在哪裡
- 部署位置：本地、內網、遠端、雲端
- 內容類型：靜態資源、動態內容、API 資料

可以先用這張表抓住核心：

| 名詞 | 先這樣理解 | 重點 |

| Web 伺服器程式 | 專門提供 Web 服務的軟體 | 例如 Nginx、Apache、Caddy，或開發伺服器 |
| 伺服器主機 | 承載服務的機器或環境 | 實體主機、雲端主機、容器、本機 |
| 後端應用 | 處理業務邏輯的應用程式 | 登入、訂單、資料查詢、權限控制 |
| 資料庫 | 儲存與管理資料的系統 | 使用者、商品、訂單、文章資料 |
| 本地開發伺服器 | 開發時在自己電腦上啟動的服務 | 預覽、測試、開發用 |
| 遠端伺服器 | 部署在其他主機或雲端的服務環境 | 對內或對外提供穩定服務 |

一句話版本：

**Web 伺服器負責接收 Web 請求並回傳內容；它可能是一個程式，也可能被口語地用來指承載這個程式的主機或環境。**

## 為什麼網站需要伺服器？

如果只是自己學 HTML，你可以直接在電腦上打開 `.html` 檔案。

但如果希望其他人透過網址穩定訪問，就通常需要一個能提供 Web 服務的環境。

原因是：

- 其他人的瀏覽器需要知道去哪裡請求資源。
- 服務需要能長時間運作。
- 服務需要能接收網路請求。
- 服務需要能回傳 HTML、CSS、JavaScript、圖片或 API 資料。
- 正式網站通常還需要網域、HTTPS、權限、安全、日誌、更新與維護。

所以實務上，網站不會只停在開發者自己的資料夾裡。
它通常會被部署到一個能持續提供服務的環境中。

可以用最簡化流程理解：

```text
使用者瀏覽器
  -> 透過 URL 找到目標
  -> 發出 HTTP/HTTPS 請求
  -> Web 伺服器接收請求
  -> 回傳 HTML / CSS / JavaScript / 圖片 / API 資料
  -> 瀏覽器解析並顯示
```

## 什麼是 Web 伺服器？

`Web 伺服器` 有狹義和廣義兩種常見用法。

### 狹義：提供 Web 服務的程式

狹義來說，Web 伺服器指的是一種服務程式。

它負責：

- 監聽 HTTP/HTTPS 請求
- 接收客戶端請求
- 根據請求路徑找到資源或轉交請求
- 回傳 HTML、CSS、JavaScript、圖片、影片、字型等資源
- 回傳狀態碼、標頭與內容
- 處理 HTTPS、壓縮、快取、日誌等 Web 服務相關工作

常見 Web 伺服器程式包含：

- Nginx
- Apache HTTP Server
- Caddy
- 開發階段的 Vite dev server、webpack dev server 等
- 某些後端框架內建的開發伺服器

入門時不需要先深入每一種工具。
先知道它們都能在某種情境下接收 Web 請求並回傳內容即可。

### 廣義：承載 Web 服務的主機或環境

日常對話裡，人們也常把承載 Web 服務的機器或環境叫做 Web 伺服器。

例如：

- 一台實體主機
- 一台雲端虛擬機
- 一個容器
- 一個平台提供的部署環境
- 你自己電腦上跑起來的本地服務

所以當你看到「把網站放到伺服器上」時，它通常不是只指安裝某個 Web 伺服器軟體，而是指：

**把網站部署到一個能被目標使用者連線訪問、並能持續提供 Web 服務的環境。**

## Web 伺服器、後端應用、資料庫的關係

這三個概念很容易混，需要分開看。

| 名詞 | 負責什麼 | 例子 |
|---|---|---|
| Web 伺服器 | 接收 HTTP/HTTPS 請求，回傳資源或轉發請求 | Nginx、Apache、Caddy、開發伺服器 |
| 後端應用 | 處理業務邏輯、權限、資料整理、API 回應 | Node.js、Java、Python、PHP、Go 寫的應用 |
| 資料庫 | 儲存、查詢、更新資料 | MySQL、PostgreSQL、SQLite、MongoDB |

常見流程可能是：

```text
瀏覽器
  -> Web 伺服器
  -> 後端應用
  -> 資料庫
  -> 後端應用
  -> Web 伺服器
  -> 瀏覽器
```

但不是每個網站都一定這麼完整。

例如：

- 純靜態網站可能只有 Web 伺服器回傳 HTML、CSS、JS、圖片。
- 動態網站通常還有後端應用處理資料和邏輯。
- 後端應用可能自己就能接收 HTTP 請求，不一定前面都放一層 Nginx。
- 大型系統可能有多層代理、負載平衡、快取和多個後端服務。

入門階段先記：

**Web 伺服器不等於後端應用，也不等於資料庫；它們可能合作，但責任不同。**

## 靜態資源和動態內容

理解 Web 伺服器時，還要分清靜態資源和動態內容。

### 靜態資源

靜態資源通常指已經存在的檔案，伺服器可以直接回傳。

例如：

- `index.html`
- `style.css`
- `main.js`
- 圖片
- 字型
- PDF
- 影片

流程通常比較像：

```text
瀏覽器請求 /style.css
  -> Web 伺服器找到 style.css
  -> 回傳檔案
```

### 動態內容

動態內容通常需要根據請求、資料庫、使用者狀態或業務邏輯產生。

例如：

- 使用者登入結果
- 商品搜尋結果
- 訂單列表
- 個人資料頁
- API 回傳的 JSON

流程可能比較像：

```text
瀏覽器請求 /api/orders
  -> Web 伺服器收到請求
  -> 後端應用查詢資料庫
  -> 後端應用產生 JSON
  -> 回傳給瀏覽器
```

所以：

**Web 伺服器可以直接提供靜態資源，也可以把需要邏輯處理的請求交給後端應用。**

## Web 伺服器一定是很特別的電腦嗎？

不一定。

伺服器本質上可以先理解成：

**在某個互動中提供服務、接收請求並回傳結果的一端。**

它可以是：

- 高效能實體主機
- 雲端虛擬機
- 容器化環境
- 平台代管環境
- 公司內網主機
- 你自己電腦上啟動的本地開發服務

一般個人電腦也可以在某個情境下扮演伺服器。
例如你在自己電腦啟動本地開發伺服器，再用瀏覽器打開：

```text
http://localhost:5173
```

這時：

- 你的瀏覽器扮演客戶端。
- 你電腦上跑起來的開發服務扮演服務端。
- 同一台電腦上同時存在客戶端和服務端角色。

所以不要用「外觀是不是機房裡的大機器」判斷伺服器。
要看它在那次互動中是否提供服務。

## 本地開發伺服器是什麼？

本地開發伺服器通常指：

**開發時在自己電腦上啟動、主要用於預覽、測試和除錯的 Web 服務。**

例如：

- `npm run dev`
- `vite --host`
- `python -m http.server`
- 後端框架啟動的本地服務
- 編輯器或工具提供的 live server

它的用途通常是：

- 預覽頁面
- 測試路由
- 測試 API
- 檢查樣式和互動
- 看開發錯誤訊息
- 模擬接近正式 Web 請求的環境

本地開發伺服器通常重點不在「穩定對外服務」，而在「方便開發」。

常見特點：

- 跑在自己電腦上
- 通常只給自己或同一區域網路測試
- 可能有熱更新
- 可能輸出詳細錯誤
- 安全與效能設定不一定適合正式上線

## 遠端伺服器是什麼？

遠端伺服器通常指：

**部署在其他主機、雲端或平台環境中，用來讓目標使用者透過網路訪問的服務環境。**

例如：

- 公司正式上線的官網
- 雲端主機上的後台系統
- 部署平台上的靜態網站
- 內部網路中的企業系統
- 對外提供 API 的後端服務

遠端伺服器通常更重視：

- 穩定性
- 安全性
- 存取權限
- 日誌
- 監控
- HTTPS
- 備份
- 擴充能力
- 錯誤恢復

正式上線時，不能只考慮「能不能跑起來」，還要考慮服務是否能穩定、安全、持續地提供給使用者。

## 本地開發伺服器和遠端伺服器差在哪？

| 比較面向 | 本地開發伺服器 | 遠端伺服器 |
|---|---|---|
| 主要目的 | 開發、測試、除錯 | 對內或對外提供穩定服務 |
| 位置 | 自己電腦 | 雲端、主機、部署平台、內網環境 |
| 使用者 | 開發者本人或小範圍測試 | 真正使用者、團隊、外部客戶 |
| 穩定性要求 | 較低 | 較高 |
| 安全要求 | 通常較低，但仍需注意 | 較高 |
| 錯誤訊息 | 可顯示詳細錯誤方便除錯 | 不應暴露敏感錯誤 |
| 更新方式 | 開發中頻繁變動 | 需要部署流程和版本管理 |
| 常見例子 | Vite dev server、Live Server | Nginx、雲端主機、部署平台 |

核心差別不是只有「在哪裡」，而是：

**本地開發伺服器重點在開發效率；遠端伺服器重點在穩定提供服務。**

## Web 伺服器常做哪些事？

不同工具和架構下，Web 伺服器負責的事情會不同，但常見工作包含：

- 接收 HTTP/HTTPS 請求
- 回傳靜態檔案
- 將請求轉交給後端應用
- 設定 HTTPS
- 設定快取策略
- 壓縮回應內容
- 記錄存取日誌
- 限制某些路徑或來源
- 反向代理
- 負載平衡
- 錯誤頁處理

初學階段不需要一次掌握全部。
先知道 Web 伺服器不是只能「放檔案」，它也常參與請求轉發、安全、快取與部署。

## 反向代理是什麼？先有個印象

初學時可能會看到 `Nginx 反向代理到後端服務` 這類說法。

可以先這樣理解：

**反向代理是由 Web 伺服器站在使用者和後端應用之間，先接收外部請求，再把請求轉交給內部真正處理邏輯的服務。**

簡化流程：

```text
瀏覽器
  -> Nginx
  -> 後端應用
  -> Nginx
  -> 瀏覽器
```

這樣做常見原因包括：

- 對外只暴露一個入口
- 統一處理 HTTPS
- 統一處理靜態資源
- 把不同路徑轉給不同服務
- 做基本流量控制或日誌

這裡先不用深入配置，只要知道：

**Web 伺服器有時不直接產生內容，而是把請求轉交給後端應用。**

## 初學時怎麼判斷「伺服器」指什麼？

看到「伺服器」這個詞時，可以先問三個問題。

### 1. 它是在說角色嗎？

如果句子是：

- 伺服器接收請求。
- 伺服器回傳資料。
- 瀏覽器向伺服器請求資源。

這裡通常是在談服務端角色。

### 2. 它是在說程式嗎？

如果句子是：

- 安裝 Nginx。
- 啟動 Apache。
- 開發伺服器正在 5173 port 運行。

這裡通常是在談 Web 伺服器程式或服務。

### 3. 它是在說主機或環境嗎？

如果句子是：

- 把網站部署到伺服器上。
- 租一台雲端伺服器。
- 公司內網有一台伺服器。

這裡通常是在談承載服務的主機或環境。

## 常見混淆點

| 混淆說法 | 更清楚的理解 |
|---|---|
| Web 伺服器一定是一台硬體機器 | 它可以指服務程式，也可以口語指承載服務的主機或環境 |
| 伺服器一定很特殊 | 一般電腦在某個情境中也可以扮演伺服器 |
| Web 伺服器就是後端 | Web 伺服器偏接收 Web 請求與回傳/轉發，後端偏業務邏輯 |
| Web 伺服器就是資料庫 | 資料庫負責儲存和查詢資料，Web 伺服器負責 Web 請求入口 |
| 本地伺服器只是不能上網的遠端伺服器 | 本地重點是開發測試，遠端重點是穩定服務 |
| 只要 HTML 檔案存在就等於網站上線 | 還需要能被目標使用者透過網路請求和取得 |
| 開發伺服器可以直接當正式伺服器 | 開發伺服器通常不適合直接承擔正式上線需求 |
| Web 伺服器一定直接產生所有內容 | 它也可能只回傳靜態資源，或把請求轉交給後端應用 |

## 初學時最該記住的六句話

1. Web 伺服器的核心任務，是接收 Web 請求並回傳資源或結果。
2. Web 伺服器可能指服務程式，也可能口語指承載服務的主機或環境。
3. 伺服器不是特殊硬體的專屬名稱，一般電腦也可以在某個情境中扮演伺服器。
4. Web 伺服器、後端應用、資料庫責任不同，不能直接當成同義詞。
5. 本地開發伺服器重點是開發測試，遠端伺服器重點是穩定提供服務。
6. Web 伺服器可以直接提供靜態資源，也可以把動態請求轉交給後端應用。

## 一句話抓核心

Web 伺服器是接收 HTTP/HTTPS 請求並回傳 Web 資源或處理結果的服務；它可能指服務程式，也可能指承載服務的主機或環境。理解時要分清角色、程式、主機、後端應用、資料庫、本地開發與正式部署這幾個層次。
