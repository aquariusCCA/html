> 建議回查情境：想理解網站為什麼需要部署到伺服器上、分不清 Web 伺服器和一般電腦的差異、想快速確認本地與遠端伺服器的用途

## 本節導讀

這一節先說明網站為什麼需要放到伺服器上，再釐清「Web 伺服器」到底是在指機器、程式，還是提供服務的角色。  
閱讀時可以先抓住「誰在接收請求、誰在回應瀏覽器」這個核心。

## 正文筆記

### 這篇在解決什麼問題？

- 初學時常會把「伺服器是一台電腦」「Web 伺服器是一種程式」「網站放在遠端主機上」這幾件事混在一起。
- 這篇要先釐清：伺服器可以是角色，也可以指承載服務的機器；而 Web 伺服器的核心任務，是接收 Web 請求並回傳內容。

### 先講結論

- 網站如果只放在自己電腦裡、又沒有對外提供存取方式，通常只有自己能方便地開啟。
- 若希望其他使用者穩定地透過網路訪問網站，就需要把網站部署到能對外提供服務的系統上，這就是伺服器常見的用途。

![服務器示意圖](./images/1775985638808_YKITp9xzK6.png)

### 為甚麼網站需要伺服器？

- 你自己的電腦當然也能跑網站，但它通常不是為了長時間、穩定、對外提供服務而配置的。
- 如果網站要讓其他人透過網路持續訪問，就通常需要一個更穩定、可連線、可部署、可維護的運行環境。
- 這也是為什麼實務上會把網站部署到伺服器上，而不是只留在開發者自己的電腦裡。

### 甚麼是 Web 伺服器？

- 狹義來說，`Web 伺服器` 指的是接收 `HTTP` 請求，並回傳網頁、圖片、樣式、腳本或其他 Web 資源的程式或服務。
- 廣義來說，人們也常把承載這些服務的主機直接叫做 `Web 伺服器`。
- 所以在學習時，最好把兩個層次分開：
  - 一個是「提供 Web 服務的程式 / 系統」
  - 一個是「承載這些程式的機器或環境」

### 伺服器一定是很特別的電腦嗎？

- 伺服器本質上也是電腦。
- 它和一般個人電腦最大的差別，不一定是外觀，而是它扮演的角色：它要持續提供服務、接收請求、回傳結果。
- 因此，伺服器可以是高效能主機，也可以是雲端虛擬機，甚至在學習階段，也可以是你自己電腦上啟動的本地開發服務。

### 本地開發伺服器與遠端伺服器

#### 本地開發伺服器

- 指在自己電腦上啟動、主要用於開發與測試的伺服器。
- 它通常方便開發者自己存取，也可能只開放給同一台電腦或局域網內的裝置使用。
- 這類伺服器常用來驗證頁面、功能與開發流程是否正常。

#### 遠端伺服器

- 指部署在其他主機或雲端環境中的伺服器。
- 當網站或系統被部署到遠端伺服器後，其他使用者就能透過網路、網址或指定入口來訪問它。
- 這也是正式上線時最常見的做法。

### 常見混淆點

- `Web 伺服器` 有時指服務程式，有時指承載服務的主機；閱讀教材時要看上下文。
- 伺服器不等於後端程式，但後端程式通常會跑在伺服器上。
- 本地開發伺服器和遠端伺服器的差別，重點在部署位置與使用目的，不只是「能不能上網」。

## 一句話抓核心

- Web 伺服器的核心任務，是接收 Web 請求並回傳內容；而實務上，人們也常把承載這些服務的主機一起叫做伺服器。
