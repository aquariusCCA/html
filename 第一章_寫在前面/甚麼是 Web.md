# 甚麼是 Web

> 所屬章節：第一章｜寫在前面  
> 關鍵字：Web、World Wide Web、全球資訊網、網際網路、瀏覽器、HTTP、HTTPS、URL、HTML、資源、JavaWeb  
> 建議回查情境：剛開始學前端時想先釐清 Web 是什麼、分不清 Web 與網際網路的關係、想理解瀏覽器、HTTP 與資源之間的關係、想快速確認 JavaWeb 的基本意思

## 本節導讀

這一節要先回答一個最基礎的問題：`Web` 到底是什麼。  
很多初學者會把 `Web`、瀏覽器、網站、`HTTP`、網際網路混成同一件事，所以這篇會先拆開，再把它們接回同一個流程裡理解。

## 你會在這篇學到什麼

- Web 的基本定義
- Web、瀏覽器、HTTP、資源之間的關係
- Web 與網際網路的差別
- JavaWeb 的基本意思

## 正文筆記

### 這篇在解決什麼問題？

- 很多人知道自己在「上網看網頁」，但不清楚 `Web` 到底在指什麼。
- 也很常把「瀏覽器發請求」「HTTP」「網站資源」「網際網路」全部混在一起，最後只記得一堆名詞，卻抓不到主線。
- 這篇的目標，就是把這條主線講清楚。

### 先講結論

- `Web` 是 `World Wide Web`（全球資訊網）的簡稱。
- 它不是整個網際網路本身，而是建立在網際網路之上的一套資訊存取方式。
- 在最常見的情況下，使用者會透過瀏覽器，用 `URL` 找到資源所在位置，透過 `HTTP` 或 `HTTPS` 向伺服器請求資源，再由瀏覽器把結果顯示出來。

### 先分清楚幾個名詞

#### 1. 網際網路是底層連線基礎

- 網際網路負責把世界各地的裝置連接起來。
- 沒有這個連線基礎，裝置彼此就很難傳遞資料。
- 可以先把它想成道路系統。

#### 2. Web 是建立在網際網路上的資訊系統

- `Web` 的重點不是「有網路」而已，而是「可以透過連結、網址、協議與文件格式來存取資訊」。
- 它讓使用者能夠方便地查看網頁、圖片、影片、文件，或其他可被存取的內容。
- 可以把它想成建立在道路系統上的一種資訊服務。

#### 3. 瀏覽器是使用 Web 的工具

- 瀏覽器是客戶端軟體。
- 它負責向伺服器請求資源、接收回應、解析內容，最後把頁面顯示給使用者。
- 常見瀏覽器有 `Chrome`、`Safari`、`Firefox`、`Edge` 等。

#### 4. HTTP / HTTPS 是常見的傳輸規則

- `HTTP` 和 `HTTPS` 可以先理解成 Web 世界裡常見的通訊規則。
- 瀏覽器和伺服器會依照這套規則交換請求與回應。
- `HTTPS` 可以看成更安全的 `HTTP` 版本。

#### 5. 資源是被請求與回傳的內容

- 在 Web 裡，資源不只有網頁文字。
- 它也可以是 `HTML`、`CSS`、`JavaScript`、圖片、影片、API 回傳的資料，或其他能被存取的內容。

### Web 最常見的運作流程

可以先用最簡單的流程理解：

1. 使用者在瀏覽器輸入網址，或點擊一個連結。
2. 瀏覽器根據 `URL` 找到要請求的目標。
3. 瀏覽器透過 `HTTP` 或 `HTTPS` 向伺服器發出請求。
4. 伺服器收到請求後，把對應的資源回傳回來。
5. 瀏覽器拿到資源後，解析並呈現給使用者。

所以你剛剛的理解可以整理成一句更完整的話：

- 在最常見的 Web 情境裡，瀏覽器會透過 `HTTP/HTTPS` 向伺服器請求資源，再把這些資源顯示給使用者。

### 為甚麼說 Web 不等於瀏覽器？

- 瀏覽器只是使用 Web 的工具之一。
- `Web` 講的是整套資源存取方式，不是某一個特定軟體。
- 雖然入門時最常見的 Web 客戶端是瀏覽器，但其他工具也能發送 `HTTP` 請求去取得 Web 資源。

### 為甚麼說 Web 不等於整個網際網路？

- 網際網路是「連線基礎」。
- `Web` 是運作在這個基礎上的一種應用方式。
- 也就是說，有網際網路，不代表你就在使用 `Web`；你也可能在用電子郵件、即時通訊、檔案傳輸等其他網路服務。

### 甚麼是 JavaWeb？

- `JavaWeb` 通常是指使用 `Java` 來開發 Web 應用。
- 這類程式通常執行在伺服器端，負責接收請求、處理業務邏輯、存取資料，再把結果回傳給瀏覽器。
- 所以 `JavaWeb` 的重點，不是「只要是 Java 程式又能被瀏覽器碰到就算」，而是「用 Java 開發 Web 應用」。

### 常見混淆點

- `Web` 不等於整個網際網路；它只是網際網路上的一種常見應用。
- 瀏覽器不等於 `Web`；它只是最常見的 Web 客戶端。
- `HTTP` 不等於 `Web` 本身；它是 Web 中最常見的通訊規則之一。
- 資源不只是一個完整頁面，也可能是一張圖片、一份資料或一支腳本檔。

## 一句話抓核心

- `Web` 是建立在網際網路上的資源存取系統，而最常見的使用方式，就是由瀏覽器透過 `HTTP/HTTPS` 向伺服器請求資源，再把結果呈現出來。
