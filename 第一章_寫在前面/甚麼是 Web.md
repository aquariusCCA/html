# 甚麼是 Web

> 所屬章節：第一章｜寫在前面  
> 關鍵字：Web、World Wide Web、全球資訊網、網際網路、瀏覽器、HTTP、HTTPS、URL、HTML、資源、JavaWeb  
> 建議回查情境：剛開始學前端時想先釐清 Web 是什麼、分不清 Web 與網際網路的關係、想理解瀏覽器、HTTP 與資源之間的關係、想快速確認 JavaWeb 的基本意思

## 本節導讀

這一節要先回答一個最基礎的問題：`Web` 到底是什麼。  
很多初學者會把 `Web`、網站、瀏覽器、`HTTP`、網際網路混成同一件事，所以這篇會先把它們拆開，再用一條清楚的流程把它們接回來。

閱讀時請先抓住一個主線：  
**Web 不是整個網際網路，而是建立在網際網路之上的一套資源存取方式。**

## 你會在這篇學到什麼

- `Web` 的基本定義
- `Web`、網際網路、瀏覽器、`HTTP/HTTPS`、資源之間的關係
- 為什麼說 `Web` 不等於瀏覽器，也不等於整個網際網路
- `JavaWeb` 的基本意思

## 這篇在解決什麼問題？

- 很多人知道自己在「上網看網頁」，但其實不清楚 `Web` 到底在指什麼。
- 也常把「瀏覽器發請求」「網站回應內容」「HTTP 傳輸」「網際網路連線」全部混成一件事。
- 如果這些名詞沒有拆開，後面學前端、後端、伺服器時就很容易越學越亂。

所以這篇的目標不是背名詞，而是先把主線建立起來。

## 先講結論

- `Web` 是 `World Wide Web`（全球資訊網）的簡稱。
- 它不是整個網際網路本身，而是**建立在網際網路之上的一種資訊存取系統**。
- 在最常見的情況下，使用者會透過瀏覽器，利用 `URL` 指向某個資源位置，再透過 `HTTP` 或 `HTTPS` 向伺服器請求資源，最後由瀏覽器把結果解析並呈現出來。

你可以先把它粗略理解成：

- **網際網路**負責把裝置連起來。
- **Web**負責讓人用比較統一的方式存取資訊。
- **瀏覽器**是使用 Web 的常見工具。
- **HTTP/HTTPS**是瀏覽器和伺服器溝通時常見的規則。
- **資源**是被請求和回傳的內容。

## 先分清楚幾個名詞

### 1. 網際網路是底層的連線基礎

- 網際網路負責把世界各地的裝置連接起來。
- 沒有這個基礎，裝置彼此就很難交換資料。
- 可以先把它想成道路或交通網路。

重點是：

- 它解決的是「能不能連起來」。
- 但它本身不等於「怎麼存取網頁內容」。

### 2. Web 是建立在網際網路上的資訊系統

- `Web` 的重點不是「有網路」而已，而是「能透過網址、連結、協議與文件格式來存取資訊」。
- 它讓使用者可以方便地查看網頁、圖片、影片、文件，或其他可被存取的內容。
- 你每天打開網站、點連結、看頁面，通常都在使用 Web。

可以把它想成：

- 網際網路像道路系統。
- Web 像建立在道路系統上的一種資訊服務。

### 3. 瀏覽器是使用 Web 的工具

- 瀏覽器是客戶端軟體。
- 它負責向伺服器請求資源、接收回應、解析內容，最後把頁面顯示給使用者。
- 常見瀏覽器有 `Chrome`、`Safari`、`Firefox`、`Edge` 等。

所以瀏覽器不是 Web 本身，而是使用 Web 的常見入口。

### 4. HTTP / HTTPS 是常見的通訊規則

- `HTTP` 和 `HTTPS` 可以先理解成 Web 世界裡常見的通訊規則。
- 瀏覽器和伺服器會依照這套規則交換請求與回應。
- `HTTPS` 可以看成加上安全保護機制後的 `HTTP`。

入門時先記住一件事就夠了：

- **瀏覽器和伺服器不是亂傳資料，而是依照某種規則溝通。**
- 在 Web 裡，最常見的規則就是 `HTTP/HTTPS`。

### 5. 資源是被請求與回傳的內容

- 在 Web 裡，資源不只是一整個網頁。
- 它也可以是 `HTML`、`CSS`、`JavaScript`、圖片、影片、字型檔、API 回傳的資料，或其他可被存取的內容。

也就是說，當你打開一個網站時，瀏覽器通常不是只拿到一個檔案，而是會陸續請求很多不同類型的資源。

## Web 最常見的運作流程

可以先用最簡化的流程理解：

1. 使用者在瀏覽器輸入網址，或點擊一個連結。
2. 瀏覽器根據 `URL` 找到要請求的目標。
3. 瀏覽器透過 `HTTP` 或 `HTTPS` 向伺服器發出請求。
4. 伺服器收到請求後，把對應的資源回傳回來。
5. 瀏覽器拿到資源後，解析並呈現給使用者。

如果只想記一句最核心的描述，可以記成：

- **在最常見的 Web 情境裡，瀏覽器會透過 `HTTP/HTTPS` 向伺服器請求資源，再把這些資源顯示給使用者。**

## 為什麼說 Web 不等於瀏覽器？

- 瀏覽器只是使用 Web 的工具之一。
- `Web` 講的是整套資源存取方式，不是某一個特定軟體。
- 雖然入門時最常見的 Web 客戶端是瀏覽器，但其他工具也能發送 `HTTP` 請求去取得 Web 資源。

所以：

- **瀏覽器是 Web 的常見入口**
- 但 **Web 不等於瀏覽器**

## 為什麼說 Web 不等於整個網際網路？

- 網際網路是底層的連線基礎。
- `Web` 是建立在這個基礎上的一種應用方式。
- 有網際網路，不代表你就在使用 Web；你也可能在用電子郵件、即時通訊、檔案傳輸或其他網路服務。

所以：

- **網際網路解決的是連線**
- **Web 解決的是資訊存取**

## 什麼是 JavaWeb？

- `JavaWeb` 通常是指使用 `Java` 來開發 Web 應用。
- 這類程式通常執行在伺服器端，負責接收請求、處理業務邏輯、存取資料，再把結果回傳給瀏覽器。
- 所以 `JavaWeb` 的重點不是「只要是 Java 程式又能被瀏覽器碰到就算」，而是「用 Java 開發 Web 應用」。

## 常見混淆點

- `Web` 不等於整個網際網路；它只是網際網路上的一種常見應用。
- 瀏覽器不等於 `Web`；它只是最常見的 Web 客戶端。
- `HTTP` 不等於 `Web` 本身；它是 Web 中最常見的通訊規則之一。
- 資源不只是一個完整頁面，也可能是一張圖片、一份資料或一支腳本檔。
- 網站是 Web 的一種常見呈現形式，但 `Web` 這個概念比單一網站更大。

## 一句話抓核心

- `Web` 是建立在網際網路上的資源存取系統，而最常見的使用方式，就是由瀏覽器透過 `HTTP/HTTPS` 向伺服器請求資源，再把結果呈現出來。
