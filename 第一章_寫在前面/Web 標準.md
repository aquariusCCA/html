## 正文筆記

- Web 標準的構成可以先從三個部分理解：
  - 結構：`HTML`，負責頁面元素與內容結構。
  - 表現：`CSS`，負責頁面樣式與視覺呈現。
  - 行為：`JavaScript`，負責頁面互動與動態效果。

## 為甚麼需要了解 Web 標準？

- 不同瀏覽器的渲染引擎不同，因此對相同程式碼的解析效果可能會有差異。
- 如果使用者用不同瀏覽器開啟同一個網站，卻看到明顯不同的結果，整體使用體驗就會很差。

![為甚麼需要了解 Web 標準示意圖 1](./images/1775987332667_NnwBDqRUnD.png)

- Web 標準的目的，就是讓不同瀏覽器盡量依照相同規範顯示結果，讓最終展示效果更一致。

## Web 標準的構成

![Web 標準的構成](./images/1775987332672_A1ZXaM9J2f.png)

- Web 標準要求頁面實作時，盡量做到結構、表現、行為三層分離。
- 簡單來說，就是把：
  - 結構寫在 `HTML` 文件中
  - 表現寫在 `CSS` 文件中
  - 行為寫在 `JavaScript` 文件中
