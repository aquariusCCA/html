# Web 標準

> 所屬章節：第一章｜寫在前面  
> 關鍵字：Web 標準、HTML、CSS、JavaScript、結構、表現、行為、三層分離、瀏覽器相容性  
> 建議回查情境：想快速記住 Web 標準由哪些部分構成、想理解為什麼前端需要遵守統一標準、想確認 HTML、CSS、JavaScript 各自負責什麼

## 本節導讀

這一節先說明為什麼需要 Web 標準，再整理初學時最常見的三個核心分工：**結構、表現、行為**。

閱讀時先抓住兩個重點：

- Web 標準的存在，是為了讓不同瀏覽器對同類內容有更一致的理解方式
- 入門時常用 `HTML`、`CSS`、`JavaScript` 來理解 Web 的三層分工，但這是一種學習框架，不是把 Web 標準簡化成只有三個檔案類型

## 你會在這篇學到什麼

- 為什麼需要 Web 標準
- Web 標準和瀏覽器相容性的基本關係
- `HTML`、`CSS`、`JavaScript` 在頁面中的常見分工
- 為什麼會強調結構、表現、行為三層分離

## 這篇在解決什麼問題？

- 初學前端時，常會聽到「要符合 Web 標準」「要注意相容性」「HTML、CSS、JavaScript 要分離」這些說法。
- 但如果沒有先理解 Web 標準存在的原因，這些話就很容易只剩口號。
- 所以這篇不是只要你背三個名詞，而是要先回答：**為什麼需要標準？**

## 先講結論

- 不同瀏覽器的渲染引擎、實作方式與版本可能不同。
- 如果沒有一套共同規範，同樣的頁面可能在不同瀏覽器出現很大的差異。
- Web 標準的目的，就是讓不同瀏覽器盡量依照相同規範處理內容，讓頁面展示與互動更一致。
- 在入門學習中，最常用的理解方式是把頁面拆成三個面向：
  - 結構：`HTML`
  - 表現：`CSS`
  - 行為：`JavaScript`

## 為什麼需要了解 Web 標準？

![為甚麼需要了解 Web 標準示意圖 1](./images/1775987332667_NnwBDqRUnD.png)

![為甚麼需要了解 Web 標準示意圖 2](./images/1775987332672_Meoxt7ECsx.png)

如果不同瀏覽器對同一份程式碼的理解方式差很多，就會出現下面這些問題：

- 某個頁面在 A 瀏覽器正常，在 B 瀏覽器卻跑版
- 某個按鈕在這邊能點，在那邊卻互動異常
- 不同使用者看到的結果差很多，導致體驗不一致

所以 Web 標準的價值，不只是「看起來比較專業」，而是：

- 讓瀏覽器對內容的理解更一致
- 讓開發者有共同依循的規則
- 降低不同瀏覽器之間的差異造成的問題

## Web 標準的入門理解：三個核心分工

![Web 標準的構成](./images/1775987332672_A1ZXaM9J2f.png)

初學時，可以先從三個部分理解頁面的分工：

### 1. 結構：HTML

- `HTML` 負責描述頁面中有哪些內容與元素
- 例如標題、段落、圖片、列表、表單等
- 它關心的是「內容怎麼組織」

### 2. 表現：CSS

- `CSS` 負責控制頁面的外觀與視覺呈現
- 例如顏色、字體、大小、間距、排版、動畫等
- 它關心的是「看起來怎麼樣」

### 3. 行為：JavaScript

- `JavaScript` 負責頁面的互動與動態效果
- 例如點擊按鈕後展開內容、表單驗證、資料請求、畫面即時更新等
- 它關心的是「頁面怎麼動、怎麼回應操作」

## 為什麼會強調三層分離？

三層分離不是說一定要把所有東西機械式拆開，而是強調：

- 結構盡量由 `HTML` 負責
- 視覺呈現盡量由 `CSS` 負責
- 行為邏輯盡量由 `JavaScript` 負責

這樣做的好處通常有：

- 結構更清楚
- 維護更容易
- 分工更明確
- 調整樣式或互動時，不容易牽動整個頁面

所以你可以把「三層分離」理解成一種良好的實作方向，而不是死板的口號。

## Web 標準和相容性有什麼關係？

- 瀏覽器不是完全相同的程式。
- 它們的核心實作、版本與功能支援能力可能不同。
- Web 標準的存在，可以讓各家瀏覽器盡量往共同規範靠攏。

但也要注意：

- 有標準，不代表所有瀏覽器就一定完全一致
- 實務上仍然需要考慮瀏覽器版本差異與功能支援情況

所以學 Web 標準，不是保證「所有瀏覽器都一模一樣」，而是讓你更有依據地寫出較穩定、較一致的頁面。

## 初學時可以怎麼記？

你可以先用一句話記住：

- `HTML` 負責內容結構
- `CSS` 負責外觀呈現
- `JavaScript` 負責互動行為

再往上一層補一句：

- Web 標準的目的，是讓不同瀏覽器對這些內容的處理更一致。

## 常見混淆點

- Web 標準不是一句空話，它背後對應的是瀏覽器共同遵循的規範方向。
- `HTML`、`CSS`、`JavaScript` 的三層分工，是理解 Web 實作的常見入門框架，但不能把 Web 標準狹義理解成只剩這三樣。
- 有 Web 標準，不代表不同瀏覽器就完全沒有差異；相容性問題仍然可能存在。
- 三層分離強調的是分工清楚與維護性，不是要求任何情況下都絕對切得一乾二淨。

## 一句話抓核心

- Web 標準的目的，是讓不同瀏覽器盡量用一致的方式理解與呈現內容；而在入門學習中，最常見的理解框架就是結構、表現、行為三層分工。
