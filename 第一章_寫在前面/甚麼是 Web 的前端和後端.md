# 甚麼是 Web 的前端和後端

> 所屬章節：第一章｜寫在前面  
> 關鍵字：前端、後端、資料展示、資料提供、客戶端、服務端、伺服器、資料庫  
> 建議回查情境：剛開始學 Web 時分不清前端和後端、看到客戶端與服務端的說法想確認差異、想快速回想資料在 Web 中如何流動

## 本節導讀

這一節用最基本的角度區分 Web 的前端與後端。  
閱讀時可以先掌握前端與後端各自負責什麼，再理解資料庫、伺服器、客戶端與服務端在整個流程中的角色。

## 你會在這篇學到什麼

- 前端與後端最基本的分工
- 資料庫與伺服器在 Web 系統中的位置
- 客戶端與服務端的基本概念

## 正文筆記

- 前端主要負責展示資料，讓使用者可以看到畫面、操作介面。
- 後端主要負責提供資料，處理邏輯、回應請求，並把需要的內容交給前端呈現。

### 補充說明

- 資料庫：用來儲存資料。
- 伺服器：像是一個中轉站，負責保存資訊、接收請求、傳遞訊息。

### 甚麼是服務端與客戶端？

> 本質上兩者都是電腦，只是扮演的角色不同。

- 服務端是從功能角度來界定的名稱，重點在於它是否提供服務。
- 在電腦世界裡，服務端通常是一台效能較高、負責提供特定功能的機器，因此常被稱為伺服器。
- 例如網站是由許多網頁檔案組成，這些檔案會存放在一台高效能電腦上；這台電腦提供網站內容，因此可以稱為網站伺服器，也就是服務端。
- 當使用者透過自己的電腦或手機開啟網站、使用 App 時，使用者手上的裝置就屬於客戶端，也可以叫做用戶端。
- 以 App 為例，App 背後的檔案與服務通常也放在伺服器上；伺服器是服務端，而安裝 App 的手機則是客戶端。

### 前端／後端 與 客戶端／服務端 的對照

> 客戶端是相對於服務端來說的；前端是相對於後端來說的。  
> 粗略理解時，可以先把客戶端看成前端、把服務端看成後端，但實際上這兩組詞描述的角度並不完全相同。
