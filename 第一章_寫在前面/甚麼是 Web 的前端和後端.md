# 甚麼是 Web 的前端和後端

> 所屬章節：第一章｜寫在前面  
> 關鍵字：前端、後端、客戶端、服務端、伺服器、資料庫、使用者介面、業務邏輯  
> 建議回查情境：剛開始學 Web 時分不清前端和後端、看到客戶端與服務端的說法想確認差異、想快速回想資料在 Web 中如何流動

## 本節導讀

這一節要釐清兩組常被混在一起的概念：前端 / 後端，以及客戶端 / 服務端。  
閱讀時可以先抓住「前後端是在談職責分工，客戶端與服務端是在談互動角色」這個核心。

## 你會在這篇學到什麼

- 前端與後端最基本的分工
- 資料庫、伺服器與後端之間的關係
- 客戶端 / 服務端 和 前端 / 後端 的差別

## 正文筆記

### 這篇在解決什麼問題？

- 初學 Web 時，最容易把前端、後端、客戶端、服務端、伺服器、資料庫全部混成同一件事。
- 這篇先把它們拆開來看，再放回同一個請求流程中理解。

### 先講結論

- 前端主要負責直接面向使用者的畫面與互動。
- 後端主要負責處理業務邏輯、資料存取、權限控制與對外提供資料。
- 客戶端與服務端描述的是互動關係；前端與後端描述的是系統分工。這兩組詞常常有關聯，但不能直接畫上等號。

### 前端在做什麼？

- 負責使用者看得到、點得到、操作得到的部分，例如頁面、按鈕、表單、互動效果。
- 在 Web 場景裡，前端通常執行在瀏覽器中。
- 它會把後端提供的資料整理後顯示給使用者，並把使用者的操作轉成請求送出去。

### 後端在做什麼？

- 負責接收請求、處理業務邏輯、驗證資料、管理權限、讀寫資料庫，最後把結果回傳給前端。
- 後端通常執行在伺服器上。
- 使用者不直接看到後端程式本身，但使用者在頁面上的很多操作，背後都需要後端處理。

### 資料庫、伺服器、後端的關係

- 資料庫主要負責儲存與管理資料。
- 伺服器是提供運算與服務的環境或主機。
- 後端是執行在伺服器上的應用程式邏輯；它會依需求去存取資料庫，但後端本身不等於資料庫，也不等於伺服器。

### 一個簡化的資料流

1. 使用者在前端頁面上進行操作。
2. 前端把操作轉成請求，送到後端。
3. 後端根據需求處理邏輯，必要時讀寫資料庫。
4. 後端把結果回傳給前端。
5. 前端再把結果顯示給使用者。

### 甚麼是服務端與客戶端？

- 客戶端是發出請求、使用服務的一端。
- 服務端是接收請求、提供服務的一端。
- 在 Web 裡，瀏覽器通常扮演客戶端；提供網站資料與功能的系統，通常扮演服務端。
- 在 App 場景裡，手機上的 App 可以是客戶端，而提供資料與功能的後端系統則是服務端。

### 前端／後端 與 客戶端／服務端 的關係

- 前端 / 後端 是從「功能分工」來看系統。
- 客戶端 / 服務端 是從「誰發請求、誰提供服務」來看互動關係。
- 在很多 Web 系統裡，前端通常跑在客戶端，後端通常跑在服務端，所以兩組詞常一起出現。
- 但它們不是同一組概念，因此不能簡單地說「前端就是客戶端，後端就是服務端」。

### 常見混淆點

- 伺服器是環境或主機，不等於後端程式本身。
- 後端常會操作資料庫，但後端不等於資料庫。
- 客戶端 / 服務端 和 前端 / 後端 有關聯，但不是完全對應的同義詞。

## 一句話抓核心

- 前端 / 後端 是分工，客戶端 / 服務端 是互動角色；在 Web 中它們常重疊，但不能直接當成同一件事。
