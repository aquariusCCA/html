# 甚麼是 Web 的前端和後端

> 所屬章節：第一章｜寫在前面  
> 關鍵字：前端、後端、客戶端、服務端、伺服器、資料庫、使用者介面、業務邏輯  
> 建議回查情境：剛開始學 Web 時分不清前端和後端、看到客戶端與服務端的說法想確認差異、想快速回想資料在 Web 中如何流動

## 本節導讀

這一節要釐清兩組常被混在一起的概念：**前端 / 後端**，以及**客戶端 / 服務端**。  
閱讀時先抓住一句最重要的話：

- **前端 / 後端是在談分工**
- **客戶端 / 服務端是在談互動角色**

這兩組概念常常同時出現，但它們不是同一件事。

## 你會在這篇學到什麼

- 前端與後端最基本的分工
- 資料庫、伺服器與後端之間的關係
- 客戶端 / 服務端 和 前端 / 後端 的差別
- 一次請求在 Web 裡大致怎麼流動

## 這篇在解決什麼問題？

- 初學 Web 時，最容易把前端、後端、客戶端、服務端、伺服器、資料庫全部混成同一件事。
- 如果這些概念沒有先分清楚，之後學 API、資料庫、部署、框架時就會一直打架。
- 所以這篇的目標，是先把它們拆開，再放回同一個流程中理解。

## 先講結論

![前端和後端示意圖](./images/1775985638800_rWD8cWXIws.png)

- 前端主要負責直接面向使用者的畫面與互動。
- 後端主要負責處理業務邏輯、資料存取、權限控制，以及對外提供資料或功能。
- 客戶端與服務端描述的是**互動關係**；前端與後端描述的是**系統分工**。
- 在很多 Web 系統裡，前端通常跑在客戶端，後端通常跑在服務端，所以兩組詞常一起出現，但不能直接畫上等號。

## 前端在做什麼？

前端主要負責：

- 使用者看得到的畫面
- 使用者點得到、輸入得到、操作得到的互動
- 將資料整理成適合呈現的樣子
- 把使用者的操作轉成請求送出去

以網站為例，前端通常包含：

- 頁面結構
- 視覺樣式
- 按鈕、表單、互動效果
- 資料顯示與基本互動邏輯

在 Web 場景裡，前端通常執行在瀏覽器中。

## 後端在做什麼？

後端主要負責：

- 接收前端送來的請求
- 處理業務邏輯
- 驗證資料是否合法
- 控制權限與身分驗證
- 讀寫資料庫
- 把處理結果回傳出去

所以使用者雖然看不到後端程式本身，但很多在頁面上看到的資料、登入結果、送單結果、查詢內容，其實都要靠後端處理。

後端通常執行在伺服器上。

## 資料庫、伺服器、後端的關係

這三個概念非常容易混淆，所以一定要分開。

### 1. 資料庫是拿來存資料的

- 資料庫主要負責儲存、查詢、更新和管理資料。
- 它的重點是「資料本身」與「資料操作」。

### 2. 伺服器是提供服務的環境或主機

- 伺服器可以理解成提供運算與服務的機器或執行環境。
- 它負責讓某些程式可以持續運作、接收請求、回傳結果。

### 3. 後端是跑在伺服器上的應用程式邏輯

- 後端是應用層的程式邏輯。
- 它會依需求去存取資料庫，也會對外提供功能或資料。
- 所以後端不等於資料庫，也不等於伺服器。

可以用一句話整理：

- **資料庫管資料**
- **伺服器提供運行環境**
- **後端負責業務邏輯與資料處理**

## 一個簡化的資料流

可以先用最簡單的流程理解：

1. 使用者在前端頁面上進行操作。
2. 前端把操作轉成請求，送到後端。
3. 後端根據需求處理邏輯，必要時讀寫資料庫。
4. 後端把結果回傳給前端。
5. 前端再把結果顯示給使用者。

例如登入時可以這樣理解：

1. 使用者在頁面輸入帳號密碼。
2. 前端把資料送到後端。
3. 後端驗證帳號密碼是否正確。
4. 必要時向資料庫查詢使用者資料。
5. 後端把登入結果回傳給前端。
6. 前端再把成功或失敗的畫面顯示出來。

## 什麼是客戶端與服務端？

- 客戶端是發出請求、使用服務的一端。
- 服務端是接收請求、提供服務的一端。

在 Web 裡：

- 瀏覽器通常扮演客戶端。
- 提供網站資料與功能的系統通常扮演服務端。

在 App 場景裡：

- 手機上的 App 可以是客戶端。
- 提供資料與功能的後端系統則是服務端。

所以客戶端 / 服務端，重點在於**誰發請求、誰提供服務**。

## 前端 / 後端 與 客戶端 / 服務端 的關係

這裡是最容易搞混的地方。

### 前端 / 後端

- 是從**功能分工**來看系統。
- 重點是各自負責什麼工作。

### 客戶端 / 服務端

- 是從**互動關係**來看系統。
- 重點是誰在請求，誰在提供服務。

在很多 Web 系統裡：

- 前端通常跑在客戶端
- 後端通常跑在服務端

所以兩組詞常常會一起出現。  
但這只是常見情況，不代表它們是完全相同的概念。

## 常見混淆點

- 伺服器是環境或主機，不等於後端程式本身。
- 後端常會操作資料庫，但後端不等於資料庫。
- 客戶端 / 服務端 和 前端 / 後端 有關聯，但不是完全對應的同義詞。
- 前端不等於只有畫面；它也包含互動處理與資料呈現邏輯。
- 後端不等於「看不到的東西」；更精確地說，它負責的是應用邏輯與資料處理。

## 一句話抓核心

- 前端 / 後端 是分工，客戶端 / 服務端 是互動角色；在 Web 中它們常重疊，但不能直接當成同一件事。
