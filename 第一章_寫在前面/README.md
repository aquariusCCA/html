# 第一章｜寫在前面

> 章節定位：進入 HTML 與前端學習前的 Web 基礎概念地圖
>
> 建議閱讀方式：第一次學習時順讀；後面遇到名詞混淆時回來查定義、角色與分工

## 本章在學什麼

這一章不急著開始寫標籤，而是先把 Web 世界裡最常出現、也最容易混淆的基本概念放清楚。

你會先理解 `Web` 是什麼、瀏覽器在流程中扮演什麼角色、Web 標準為什麼存在，再往下分清前端 / 後端、客戶端 / 服務端、C/S / B/S、Web 伺服器等名詞。

如果後面章節出現 `HTML`、`CSS`、`JavaScript`、瀏覽器、URL、HTTP、伺服器、前端、後端、API、部署或 JavaWeb 這類詞，本章就是最適合先回來確認基本定義的地方。

## 本章和後續章節的關係

後面的 HTML 章節會開始進入標籤、屬性、路徑、表單、媒體、語意化與瀏覽器行為。

這些內容都建立在同一個前提上：你知道瀏覽器如何取得資源、如何解析頁面，以及前端程式在 Web 系統裡負責哪一段。

所以本章比較像「地圖」：

- 先建立 Web、瀏覽器、伺服器與資源請求的整體流程。
- 再理解 HTML / CSS / JavaScript 為什麼會被放在 Web 標準裡討論。
- 最後把前端、後端、客戶端、服務端、終端、C/S、B/S、Web 伺服器這些相近名詞拆開。

## 建議閱讀順序

1. [甚麼是 Web](./甚麼是%20Web.md)：先建立 Web、網際網路、URL、HTTP/HTTPS、HTML、瀏覽器與伺服器的整體關係。
2. [瀏覽器](./瀏覽器.md)：理解瀏覽器不只是打開網頁的工具，也負責請求資源、解析內容、執行 JavaScript 與渲染畫面。
3. [Web 標準](./Web%20標準.md)：理解為什麼 HTML、CSS、JavaScript 要遵守共同規範，以及結構、表現、行為三層分工的意義。
4. [甚麼是 Web 的前端和後端](./甚麼是%20Web%20的前端和後端.md)：把前端 / 後端、客戶端 / 服務端、伺服器、資料庫、API 放進同一條資料流裡理解。
5. [客戶端、終端、服務端是什麼意思](./客户端、终端、服务端是什么意思.md)：補清楚客戶端、服務端、伺服器、終端、Shell、命令列這幾組容易混用的詞。
6. [C/S 架構和 B/S 架構](./C%20S%20架構和%20B%20S%20架構.md)：比較專用客戶端與瀏覽器客戶端的差異，理解 C/S 與 B/S 不是完全對立的概念。
7. [Web 服務器](./Web%20服務器.md)：理解 Web 伺服器可能指程式、主機或服務角色，並分清本地開發伺服器、遠端伺服器、後端應用與資料庫。
