Web 基礎概念 Demo

這個示例把第一章幾個最重要的概念放進同一個互動流程：瀏覽器送出請求、伺服器回應資源、前端接手呈現， 然後再用一個迷你頁面對照 HTML、CSS、JavaScript 的分工。

瀏覽器 不是只顯示畫面，也會請求、解析、執行與渲染。
Web 伺服器 接收 HTTP/HTTPS 請求並回傳頁面與其他資源。
前端 / 後端 前端偏使用者看到與操作的部分，後端偏資料與服務邏輯。
Web 標準 入門上可先用結構、表現、行為三層分工理解。

這頁怎麼看

  • 再切換下方標籤，對照前端、後端與三層分工。
  • 最後用 C/S、B/S 切換，看客戶端形式怎麼改變。
第一章偏概念整理，所以示例重點不是單一 API，而是把零散名詞串成一個可視化流程。

1. 瀏覽器如何取得一個 Web 頁面

客戶端 / Browser

瀏覽器

等待使用者輸入網址，準備送出請求。

服務端 / Server

Web 伺服器

等待接收請求，尚未回傳資源。

前端呈現 / Render

頁面結果

瀏覽器還沒有拿到回應，因此畫面尚未渲染。

請求與回應

Request

尚未送出請求。

Response

尚未收到伺服器回應。

流程拆解

Step 1 瀏覽器取得網址後，向 Web 伺服器送出 HTTP 請求。
Step 2 伺服器回傳 HTML、CSS、JavaScript 等資源。
Step 3 瀏覽器解析結構、套用樣式、執行腳本，最後渲染畫面。

2. 前端 / 後端在做什麼

前端：使用者看得到、摸得到的部分

前端負責把服務呈現在瀏覽器裡，讓使用者可以閱讀內容、點按鈕、填表單並得到互動回饋。

  • 畫面結構與排版
  • 按鈕、表單、提示與互動體驗
  • 把後端資料轉成使用者能理解的畫面

4. C/S 和 B/S 差在哪裡

Browser / Client

客戶端是瀏覽器

使用者不必另外安裝專用程式，直接透過瀏覽器打開網址就能使用服務。

典型例子：官網、管理後台、線上表單、Web 系統。

Server

伺服器負責集中提供內容與功能

B/S 架構下，瀏覽器是固定客戶端形式，更新通常集中在伺服器端與前端部署內容。

重點差異：客戶端的形式，而不是有沒有客戶端。
