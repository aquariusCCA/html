## 本節導讀

這一節要釐清一個初學前端時很容易低估的角色：瀏覽器。

很多人會把瀏覽器理解成「打開網站的工具」。這個說法不算錯，但太簡化。
在 Web 裡，瀏覽器不只是顯示畫面的軟體，它同時也是：

- 發出 Web 請求的客戶端
- 解析 `HTML`、`CSS` 的工具
- 執行 `JavaScript` 的環境
- 把內容繪製成畫面的渲染系統
- 管理快取、Cookie、儲存、安全限制與開發工具的平台

閱讀時先抓住這句話：

**瀏覽器是使用者存取 Web 最常見的客戶端，它負責請求資源、解析資源、執行程式、渲染畫面，並提供使用者和網頁互動的環境。**

## 你會在這篇學到什麼

- 瀏覽器在 Web 流程中的基本角色
- 使用者輸入網址後，瀏覽器大致做了哪些事
- 常見瀏覽器有哪些
- 瀏覽器主要由哪些能力組成
- 渲染引擎、`JavaScript` 引擎、瀏覽器內核之間的差別
- 為什麼不同瀏覽器可能出現顯示或功能差異
- 前端開發為什麼需要關心瀏覽器相容性

## 這篇在解決什麼問題？

初學前端時，常見混亂有幾種：

- 以為瀏覽器只是「把畫面打開」。
- 分不清瀏覽器、Web、網站和伺服器。
- 聽到「瀏覽器內核」就以為它完全等於渲染引擎。
- 以為同一份 `HTML`、`CSS`、`JavaScript` 在所有瀏覽器都一定完全一樣。
- 不理解為什麼前端常要查瀏覽器支援度。

這篇的目標，是把瀏覽器放回 Web 請求與頁面呈現流程裡理解。

## 30 秒先講結論

瀏覽器是 Web 的常見客戶端。

當你輸入網址或點擊連結時，瀏覽器通常會：

1. 根據 `URL` 找到要請求的目標。
2. 透過 `HTTP/HTTPS` 向伺服器請求資源。
3. 接收伺服器回傳的 `HTML`、`CSS`、`JavaScript`、圖片等資源。
4. 解析這些資源，建立頁面結構與樣式。
5. 執行頁面中的 `JavaScript`。
6. 把最終結果渲染成使用者看到的畫面。
7. 持續處理使用者的點擊、輸入、滾動、跳轉與資料請求。

所以瀏覽器不是 Web 本身，而是使用 Web 的重要入口與執行環境。

常見的桌面或行動瀏覽器包含：

- `Chrome`

- `Edge`

- `IE`

但在現代前端學習中，通常會優先關注 `Chrome`、`Safari`、`Firefox`、`Edge` 這幾類主流環境。

## 瀏覽器在 Web 裡扮演什麼角色？

在 [甚麼是 Web](./甚麼是%20Web.md) 裡，已經先建立過一條主線：

```text
使用者操作
  -> 瀏覽器根據 URL 找目標
  -> 透過 HTTP/HTTPS 發出請求
  -> 伺服器回傳資源
  -> 瀏覽器解析並顯示
```

在這條流程裡，瀏覽器負責的是「使用者這一端」。

它主要做三件事：

| 角色 | 說明 | 例子 |
|---|---|---|
| 客戶端 | 代表使用者向伺服器請求資源 | 打開網址、送出表單、請求 API |
| 執行環境 | 解析與執行網頁需要的內容 | 解析 HTML/CSS、執行 JavaScript |
| 互動介面 | 讓使用者看到頁面並操作頁面 | 點按鈕、輸入文字、滾動頁面 |

所以瀏覽器不是單純的「閱讀器」。
它更像是一個專門用來運行 Web 頁面與 Web 應用的平台。

## 輸入網址後，瀏覽器大致做了什麼？

先用入門版流程理解即可：

1. 使用者在網址列輸入 `URL`，或點擊一個連結。
2. 瀏覽器判斷這個 `URL` 要使用什麼協議，例如 `https`。
3. 瀏覽器找到目標網站或服務所在的位置。
4. 瀏覽器向伺服器發出 `HTTP/HTTPS` 請求。
5. 伺服器回傳資源，最常見的第一份資源是 `HTML`。
6. 瀏覽器開始解析 `HTML`，理解頁面有哪些內容與元素。
7. 如果 `HTML` 引用了 `CSS`、`JavaScript`、圖片、字型等資源，瀏覽器會繼續請求它們。
8. 瀏覽器解析 `CSS`，計算元素應該如何顯示。
9. 瀏覽器執行 `JavaScript`，讓頁面可以互動或動態更新。
10. 瀏覽器把頁面繪製到螢幕上。

這裡先不用深入 DNS、TCP、TLS、程序模型、事件迴圈等細節。
入門階段最重要的是知道：

**瀏覽器打開一個網頁，不是只把一份檔案顯示出來，而是完成一連串請求、解析、執行與渲染工作。**

## 瀏覽器實際上負責哪些事？

可以把瀏覽器的工作分成幾類。

### 1. 請求資源

瀏覽器會向伺服器請求各種 Web 資源，例如：

- `HTML`
- `CSS`
- `JavaScript`
- 圖片
- 字型
- 音訊與影片
- API 回傳的資料

這些資源組合起來，才形成使用者看到和操作的頁面。

### 2. 解析 HTML，建立頁面結構

`HTML` 用來描述頁面內容與結構。

瀏覽器讀到 `HTML` 後，會理解：

- 哪裡是標題
- 哪裡是段落
- 哪裡是圖片
- 哪裡是連結
- 哪裡是表單
- 元素之間的父子、兄弟與嵌套關係

所以 `HTML` 不是直接等於畫面，而是讓瀏覽器知道頁面有哪些內容與結構。

### 3. 解析 CSS，決定外觀

`CSS` 用來描述頁面如何呈現。

瀏覽器會根據 CSS 決定：

- 顏色
- 字體
- 字級
- 間距
- 排版
- 背景
- 動畫
- 響應式變化

同一份 `HTML`，套用不同 CSS，視覺效果可能完全不同。

### 4. 執行 JavaScript，處理互動

`JavaScript` 讓頁面可以有互動與動態行為。

例如：

- 點擊按鈕後展開內容
- 表單輸入時即時檢查
- 向後端 API 請求資料
- 更新頁面上的清單或狀態
- 顯示彈窗或通知

這也是為什麼瀏覽器不只是顯示工具，它還是一個程式執行環境。

### 5. 渲染畫面

渲染可以先理解成：

**瀏覽器把解析後的結構、樣式、圖片與文字，轉成螢幕上看得到的畫面。**

瀏覽器會計算元素的位置、大小、樣式，再把結果繪製出來。
當使用者滾動、縮放、點擊，或 `JavaScript` 改變頁面內容時，瀏覽器也可能重新計算與更新畫面。

### 6. 管理瀏覽行為與安全限制

瀏覽器還會處理很多使用者平常不一定注意到的事情：

- 歷史紀錄
- 書籤
- 快取
- Cookie
- 本地儲存
- 下載
- 權限提示
- 網站安全限制
- 開發者工具

例如網站想讀取位置、使用攝影機或發送通知時，瀏覽器通常會跳出權限提示。
這些機制是瀏覽器保護使用者與管理網站能力的一部分。

## 瀏覽器內核是什麼？

![各家瀏覽器的內核](./images/1775986994339_AfGepjMeSx.png)

「瀏覽器內核」是入門教材裡很常見的說法，但它容易被講得太粗。

在很多中文教材中，「瀏覽器內核」常被拿來指：

- 負責解析網頁的核心部分
- 負責渲染頁面的核心部分
- 或者直接指「渲染引擎」

這種說法在入門階段方便理解，但要注意：

**瀏覽器內核不是一個非常精準、單一、固定邊界的術語。不同教材可能用它指不同層次。**

更清楚的拆法是：

| 名詞 | 先這樣理解 | 負責什麼 |
|---|---|---|
| 瀏覽器 | 完整的使用者軟體 | 網址列、分頁、網路請求、渲染、JS 執行、儲存、安全、開發者工具等 |
| 渲染引擎 | 瀏覽器中負責頁面解析與顯示的重要部分 | 解析 HTML/CSS、計算布局、繪製頁面 |
| JavaScript 引擎 | 執行 JavaScript 的核心部分 | 解析、編譯與執行 JS 程式 |
| 瀏覽器內核 | 教材中的概括說法 | 常被用來泛指渲染引擎，或瀏覽器中處理頁面的核心能力 |

所以，入門時可以先記：

- 「瀏覽器內核」常被教材當成「渲染頁面的核心」來講。
- 但更精準時，要分清「渲染引擎」和「JavaScript 引擎」。
- 完整瀏覽器還包含網路、儲存、安全、介面、多程序管理等其他模組。

## 渲染引擎和 JavaScript 引擎有什麼差別？

這是初學者很容易混淆的地方。

### 渲染引擎

渲染引擎主要負責把網頁內容變成畫面。

它會處理：

- `HTML` 結構
- `CSS` 樣式
- 元素布局
- 畫面繪製
- 部分頁面更新

你可以先把它想成「理解頁面結構與外觀，並把頁面畫出來」的核心。

### JavaScript 引擎

`JavaScript` 引擎主要負責執行 `JavaScript` 程式。

它會處理：

- 讀取 JS 程式碼
- 解析與編譯 JS
- 執行函式與邏輯
- 管理執行過程中的資料

你可以先把它想成「讓頁面中的 JavaScript 跑起來」的核心。

### 它們如何合作？

頁面互動常常需要兩者合作。

例如使用者點擊按鈕後，頁面新增一段文字：

1. 瀏覽器收到點擊事件。
2. `JavaScript` 引擎執行對應程式碼。
3. 程式碼改變頁面內容。
4. 渲染引擎重新計算需要更新的畫面。
5. 使用者看到新的內容。

所以：

**渲染引擎偏向把頁面畫出來，JavaScript 引擎偏向把程式跑起來。**

## 常見瀏覽器與引擎

下面是入門時常見的對照表。它的目的不是要求你背細節，而是幫你知道不同瀏覽器背後可能有不同實作。

| 瀏覽器 | 常見渲染引擎 | 常見 JavaScript 引擎 | 備註 |
|---|---|---|---|
| Chrome | Blink | V8 | 現代前端最常見的測試環境之一 |
| Edge | Blink | V8 | 現代 Edge 基於 Chromium |
| Safari | WebKit | JavaScriptCore | 在 iOS / macOS 上特別重要 |
| Firefox | Gecko | SpiderMonkey | 不是 Chromium 系瀏覽器 |
| IE | Trident | JScript / Chakra | 舊版相容性教材中常見 |

注意：這張表是入門理解用。實際瀏覽器版本、平台與內部實作會更複雜。
但它足以幫你理解一件事：

**不同瀏覽器不是同一套實作，因此同一份網頁可能在不同環境中出現差異。**

## 為什麼不同瀏覽器的顯示效果可能不同？

![瀏覽器內核示意圖](./images/1775986994340_Lcv5XvKnJc.png)

同一份網頁在不同瀏覽器中可能不完全一樣，原因通常不是只有一個。

常見原因包含：

- 渲染引擎不同
- `JavaScript` 引擎不同
- 瀏覽器版本不同
- 作業系統和字型環境不同
- 螢幕尺寸與裝置能力不同
- 新功能支援程度不同
- 某些 CSS 或 Web API 的實作細節不同
- 使用者瀏覽器設定不同

例如：

- 某個新的 CSS 屬性在 A 瀏覽器支援，在 B 瀏覽器還不支援。
- 同一個字型在不同作業系統上顯示效果略有不同。
- 某個 API 在桌面瀏覽器可用，但在手機瀏覽器受限制。
- 舊瀏覽器無法理解現代 JavaScript 語法。

所以前端開發不是只要「我這台電腦看起來正常」就結束。
如果使用者環境很多樣，就需要考慮瀏覽器支援與相容性。

## 為什麼前端開發要在意瀏覽器差異？

因為前端程式最後是在使用者的瀏覽器中執行與呈現。

如果只在某一個瀏覽器測試，可能會遇到：

- 在自己電腦正常，使用者手機跑版。
- 在 Chrome 正常，Safari 顯示不同。
- 在新版本瀏覽器正常，舊版本不支援某個功能。
- 在桌面正常，行動裝置操作體驗不好。

所以前端開發通常需要注意：

- HTML、CSS、JavaScript 是否符合標準用法
- 目標使用者可能使用哪些瀏覽器
- 新特性的支援程度
- 是否需要替代寫法或降級方案
- 頁面在不同螢幕尺寸下是否正常
- 互動是否能在滑鼠、觸控、鍵盤等不同操作方式下使用

這也是後面學 Web 標準、相容性、響應式設計與可存取性時會反覆遇到的主題。

## 開發者工具是什麼？

現代瀏覽器通常都有開發者工具。
它是前端開發非常重要的輔助工具。

開發者工具可以用來：

- 查看頁面的 `HTML` 結構
- 檢查元素套用了哪些 `CSS`
- 在 Console 執行或檢查 `JavaScript`
- 查看網路請求與回應
- 檢查圖片、字型、API 是否載入成功
- 模擬不同螢幕尺寸
- 查看錯誤訊息

所以當頁面出現問題時，瀏覽器本身也是重要的除錯工具。

初學階段先記住：

**瀏覽器不只負責顯示網頁，也提供觀察網頁如何運作的工具。**

## 瀏覽器不等於什麼？

### 瀏覽器不等於 Web

Web 是一套資源定位、請求、連結與呈現的系統。
瀏覽器是使用 Web 的常見工具。

除了瀏覽器之外，其他程式也可以發送 `HTTP/HTTPS` 請求取得 Web 資源。

### 瀏覽器不等於搜尋引擎

瀏覽器是打開網頁與執行 Web 內容的軟體，例如 `Chrome`、`Safari`、`Firefox`、`Edge`。

搜尋引擎是幫你搜尋網頁內容的服務，例如 Google、Bing 等。

你可以在瀏覽器裡使用搜尋引擎，但兩者不是同一件事。

### 瀏覽器不等於伺服器

瀏覽器通常位於使用者這一端，負責請求與呈現資源。
伺服器通常位於提供服務的一端，負責接收請求與回傳結果。

在 Web 流程中，它們常常是一組互動關係：

```text
瀏覽器發出請求 -> 伺服器回傳資源
```

### 瀏覽器不等於渲染引擎

渲染引擎是瀏覽器中的重要組成部分，但完整瀏覽器還包含很多其他功能。

把瀏覽器完全等同於渲染引擎，會漏掉網路請求、儲存、安全、使用者介面、開發者工具等重要能力。

## 常見混淆點

| 混淆說法 | 更清楚的理解 |
|---|---|
| 瀏覽器只是打開畫面的工具 | 瀏覽器還負責請求、解析、執行、渲染與管理互動 |
| 瀏覽器就是 Web | Web 是系統，瀏覽器是使用 Web 的客戶端 |
| 瀏覽器就是搜尋引擎 | 瀏覽器是軟體，搜尋引擎是搜尋服務 |
| 瀏覽器內核就是整個瀏覽器 | 內核通常是教材中的概括說法，完整瀏覽器還有更多模組 |
| 渲染引擎就是 JavaScript 引擎 | 前者偏頁面解析與顯示，後者偏 JS 執行 |
| 同一份程式碼在所有瀏覽器一定相同 | 不同瀏覽器、版本、平台與設定都可能造成差異 |
| 只要 Chrome 正常就代表所有使用者正常 | 實務上仍要看目標使用者的瀏覽器與裝置 |

## 初學時最該記住的六句話

1. 瀏覽器是存取 Web 的常見客戶端。
2. 瀏覽器不只是顯示畫面，還會請求、解析、執行與渲染資源。
3. `HTML` 負責結構，`CSS` 負責樣式，`JavaScript` 負責互動，瀏覽器負責理解並運行它們。
4. 「瀏覽器內核」是入門教材常用的概括說法，更精準時要分清渲染引擎與 `JavaScript` 引擎。
5. 不同瀏覽器的實作、版本和平台不同，所以頁面效果可能不完全一致。
6. 前端開發需要關心瀏覽器支援度、相容性、裝置差異與使用者操作方式。

## 一句話抓核心

瀏覽器是使用者存取 Web 的常見客戶端與執行環境；它會請求資源、解析 `HTML/CSS`、執行 `JavaScript`、渲染畫面，並在不同引擎、版本與裝置條件下影響頁面的實際呈現。
