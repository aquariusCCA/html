# 客户端、终端、服务端是什么意思

> 所屬章節：第一章｜寫在前面  
> 本文定位：補充 [甚麼是 Web 的前端和後端](./甚麼是%20Web%20的前端和後端.md) 中「客戶端 / 服務端」相關名詞  
> 關鍵字：客戶端、用戶端、服務端、伺服器、終端、命令列、CMD、Shell  
> 建議回查情境：看到客戶端、終端、服務端三個詞混在一起時、想補充理解「服務端 / 客戶端」與實際設備的對應、想知道終端在日常開發裡通常指什麼

## 本節導讀

這一節是 [甚麼是 Web 的前端和後端](./甚麼是%20Web%20的前端和後端.md) 的補充閱讀，主要把 `客戶端`、`終端`、`服務端` 這三個容易混用的詞拆開來看。

閱讀時請先記住：

- **客戶端 / 服務端** 主要是在描述互動關係
- **終端** 則更常是在描述操作入口或操作介面

## 你會在這篇學到什麼

- 客戶端與服務端最直觀的意思
- 這兩個詞和實際設備之間常見的對應方式
- 終端在日常使用與開發中的基本意思
- 為什麼這三個詞容易在不同情境下被混用

## 這篇在補什麼？

- 同一個概念在不同語境中，意思可能不完全一樣。
- `終端`、`客戶端`、`服務端` 這三個詞，在日常文章、教材、影片或工作對話裡常常會被混著用。
- 如果不先拆開，初學者很容易把「角色」「設備」「操作介面」當成同一件事。

所以這篇的目標，不是給最嚴格的學術定義，而是先幫你建立一個不容易搞混的理解框架。

## 1. 什麼是服務端和客戶端？

### 先抓最核心的判斷方式

- **客戶端**：發出請求、使用服務的一端
- **服務端**：接收請求、提供服務的一端

這裡的重點是「角色」，不是一定是哪一種硬體。

也就是說：

- 它們首先是在描述互動關係
- 不是先在描述「某一台固定類型的機器」

### 為什麼很多人會把服務端直接理解成伺服器？

因為在實務上，提供服務的程式通常會跑在伺服器上，所以人們很容易直接把「服務端」說成「伺服器」。

這種說法在很多情境下沒有大問題，但你心裡最好要知道：

- **服務端** 比較偏角色
- **伺服器** 比較偏承載服務的機器或環境

兩者經常一起出現，但不是完全同一個概念。

### 以網站為例

- 網站的內容與功能通常部署在某個提供服務的系統上。
- 當瀏覽器去請求這些內容時，瀏覽器所在的一端通常叫做客戶端。
- 接收請求並回應內容的那一端，通常叫做服務端。
- 如果服務端程式剛好跑在某台對外提供服務的主機上，人們也會把那台主機叫做伺服器。

### 以手機 App 為例

- 使用者手機上安裝並使用的 App，可以視為客戶端。
- App 背後提供資料、帳號、訂單、通知等功能的系統，通常是服務端。

所以你可以先這樣記：

- **客戶端 / 服務端** 是在看誰用服務、誰提供服務
- 而不是在看哪一邊比較大、比較強、比較遠

## 2. 什麼是終端？

`終端` 這個詞比 `客戶端`、`服務端` 更容易因情境不同而改變意思，所以初學時最容易被搞亂。

在你目前這套筆記的脈絡裡，可以先把終端理解成：

- **讓人和系統互動的入口或操作介面**

### 在日常開發裡，終端常常指什麼？

- 在 `Windows` 上，像 `cmd`、`PowerShell`、Windows Terminal 這類命令列視窗，常會被叫做終端。
- 在 `macOS` 或 `Linux` 上，命令列介面也常會被叫做終端。
- 在編輯器裡內建的命令列區塊，也常被稱為終端。

所以在很多開發語境裡，你聽到「打開終端」，通常不是在說某個遠端伺服器，而是在說：

- 打開一個可以輸入命令、操作系統或執行程式的介面。

### 為什麼叫終端？

你現在不用先追歷史細節，只要先抓住實用理解就好：

- 它是一個「人向系統下指令」的入口
- 所以在日常語境裡，可以先把它理解成命令列操作介面

## 3. 為什麼這三個詞容易被混用？

因為它們常常會在同一個情境裡一起出現。

例如你在電腦上打開終端，透過命令去連線一台遠端主機，或啟動一個本地服務時：

- 你眼前用來輸入命令的視窗，可能叫做**終端**
- 你自己的電腦在某些互動關係裡，可能扮演**客戶端**
- 那台提供服務的遠端系統，可能扮演**服務端**

所以這三個詞不是互斥關係，而是分別在不同角度描述同一個場景。

## 4. 初學時怎麼分比較不容易亂？

你可以先用這三句話快速判斷：

- **客戶端 / 服務端**：看的是誰發請求、誰提供服務
- **終端**：看的是你透過什麼入口操作系統或程式
- **伺服器**：看的是承載服務的機器或執行環境

只要這三層先分開，後面就比較不容易混掉。
