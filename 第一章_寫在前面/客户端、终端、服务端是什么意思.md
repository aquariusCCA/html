# 客戶端、終端、服務端是什麼意思

> 所屬章節：第一章｜寫在前面
> 本文定位：補充 [甚麼是 Web 的前端和後端](./甚麼是%20Web%20的前端和後端.md) 中「客戶端 / 服務端」相關名詞
> 關鍵字：客戶端、用戶端、Client、服務端、Server Side、伺服器、服務器、終端、Terminal、Shell、命令列、CMD、PowerShell、Bash、Zsh
> 建議回查情境：看到客戶端、用戶端、服務端、伺服器、終端、Shell、命令列混在一起時，想分清它們各自是在描述角色、設備、環境還是操作介面

## 本節導讀

這一節是前一篇 [甚麼是 Web 的前端和後端](./甚麼是%20Web%20的前端和後端.md) 的補充閱讀。

前一篇已經講過：

- **前端 / 後端** 是在談系統分工。
- **客戶端 / 服務端** 是在談互動角色。

這篇再往下補三個容易混用的詞：

- 客戶端
- 服務端
- 終端

閱讀時先抓住一句話：

**客戶端 / 服務端是在描述「誰使用服務、誰提供服務」；終端通常是在描述「人透過什麼入口操作系統或程式」。**

它們可能出現在同一個場景裡，但不是同一個概念。

## 用字先統一

這篇筆記採用繁體用語：

| 本文用語 | 也常看到的寫法 | 英文 |
|---|---|---|
| 客戶端 | 客户端、用戶端 | Client |
| 服務端 | 服务端 | Server Side |
| 伺服器 | 服務器、服务器 | Server |
| 終端 | 终端 | Terminal |
| 命令列 | 命令行 | Command Line |

注意：

- `客戶端` 和 `用戶端` 在多數入門語境裡可以先視為同類說法。
- `服務端` 偏向服務提供的一端或服務端程式邏輯。
- `伺服器` 常用來指承載服務的機器、環境或服務角色。
- `終端` 不是客戶端或服務端的同義詞，它更常指操作入口。

## 你會在這篇學到什麼

- 客戶端與服務端的核心判斷方式
- 服務端和伺服器為什麼常被混用
- 終端在日常開發裡通常指什麼
- 終端、Shell、命令列之間的差別
- 為什麼同一台電腦在不同情境下可能扮演不同角色
- 怎麼用具體場景判斷誰是客戶端、誰是服務端、哪個是終端

## 這篇在解決什麼問題？

初學時常見混亂有幾種：

- 以為客戶端就是自己的電腦。
- 以為服務端一定是一台很大的遠端機器。
- 以為伺服器和服務端永遠完全相同。
- 以為終端就是伺服器。
- 以為打開終端就代表自己正在操作服務端。
- 以為 Shell、終端、命令列是完全同義詞。

這篇的目標不是給最嚴格的歷史定義，而是建立一套足夠清楚、能用來學 Web 和開發工具的判斷框架。

## 30 秒先講結論

先用這張表抓住核心：

| 名詞 | 先這樣理解 | 判斷重點 |
|---|---|---|
| 客戶端 | 發出請求、使用服務的一端 | 誰在要求服務 |
| 服務端 | 接收請求、提供服務的一端 | 誰在回應和提供能力 |
| 伺服器 | 承載服務的機器、環境或服務角色 | 服務跑在哪裡、誰對外提供服務 |
| 終端 | 人和系統互動的文字操作入口 | 你透過哪裡輸入命令 |
| Shell | 解讀命令並執行程式的命令解譯器 | 誰理解你輸入的命令 |
| 命令列 | 用文字命令操作系統或程式的方式 | 你用什麼操作形式 |

最重要的判斷方式：

```text
客戶端 / 服務端：看誰請求、誰提供服務
終端 / Shell / 命令列：看人怎麼輸入命令、命令由誰解讀
伺服器：看誰承載並提供服務
```

## 什麼是客戶端？

客戶端是發出請求、使用服務的一端。

它不一定是某一種固定設備，而是一種互動角色。

常見客戶端包含：

- 瀏覽器
- 手機 App
- 桌面軟體
- 命令列工具
- 其他會向服務發請求的程式

例如：

- 你用瀏覽器打開網站，瀏覽器是客戶端。
- 你用手機 App 查訂單，App 是客戶端。
- 你用 `curl` 請求一個 API，`curl` 這個工具扮演客戶端。
- 你用資料庫管理工具連到資料庫，這個管理工具可以視為客戶端。

所以不要把客戶端死記成「自己的電腦」。

更精準地說：

**在某次互動中，誰主動發出請求、使用某個服務，誰就可以被稱為客戶端。**

## 什麼是服務端？

服務端是接收請求、提供服務的一端。

它也不是先看硬體大小，而是看角色。

服務端通常負責：

- 接收請求
- 判斷請求要做什麼
- 處理業務邏輯
- 查詢或更新資料
- 回傳結果
- 維持服務可用

例如：

- 網站後端收到瀏覽器請求並回傳資料。
- 訂單系統收到 App 請求並回傳訂單內容。
- API 服務收到 `curl` 請求並回傳 JSON。
- 資料庫服務收到查詢語句並回傳查詢結果。

所以：

**在某次互動中，誰接收請求並提供服務，誰就是服務端。**

## 客戶端和服務端是一組互動角色

客戶端和服務端通常要放在同一個互動裡看。

不要單獨問：

- 這台電腦是不是客戶端？
- 這台電腦是不是服務端？

更好的問法是：

**在這一次互動中，誰向誰發出請求？誰回應誰？**

例如你在自己的電腦上打開瀏覽器，請求一個網站：

```text
你的瀏覽器 -> 網站服務
```

這次互動中：

- 你的瀏覽器是客戶端。
- 網站服務所在的一端是服務端。

但如果你在自己的電腦上啟動一個本地開發服務，再用瀏覽器打開它：

```text
你的瀏覽器 -> 你自己電腦上的本地服務
```

這次互動中：

- 你的瀏覽器仍然是客戶端。
- 你電腦上跑起來的本地服務扮演服務端。
- 同一台電腦可以同時包含客戶端和服務端角色。

## 服務端和伺服器有什麼差別？

這兩個詞很常被混用，因為它們確實常一起出現。

但最好分清兩層：

| 名詞 | 偏向描述 | 例子 |
|---|---|---|
| 服務端 | 互動角色或服務端程式邏輯 | 接收 API 請求的後端程式 |
| 伺服器 | 承載服務的機器、環境或服務角色 | 雲端主機、實體主機、本地開發服務 |

在日常語境裡，人們可能說：

- 服務端回傳資料。
- 伺服器回傳資料。
- 後端回傳資料。

這些說法在很多場景下都能聽懂，但精確層次不同。

比較清楚的拆法是：

- **服務端**：偏「提供服務的一端」或「服務端程式邏輯」。
- **伺服器**：偏「承載服務的機器、環境或角色」。
- **後端**：偏「系統分工中的應用程式邏輯」。

例如：

```text
後端程式跑在伺服器上，並在這次互動中扮演服務端。
```

這句話把三個概念都放回正確位置。

## 什麼是終端？

`終端` 這個詞比 `客戶端`、`服務端` 更容易混，因為它在不同時代和語境中有不同用法。

在目前這套 Web 入門筆記裡，可以先把終端理解成：

**讓人用文字命令和系統或程式互動的操作入口。**

日常開發中，常見的終端例子包括：

- macOS 的 Terminal
- Windows Terminal
- VS Code 內建終端
- Linux 桌面環境中的 Terminal
- iTerm2
- 其他可以輸入命令的終端模擬器

你聽到「打開終端」，通常是在說：

**打開一個可以輸入命令、執行程式、查看輸出的操作介面。**

它不等於遠端伺服器，也不等於服務端。

## 終端、Shell、命令列差在哪？

這三個詞常一起出現，所以也要拆開。

| 名詞 | 先這樣理解 | 例子 |
|---|---|---|
| 終端 | 顯示文字、接收輸入、呈現輸出的操作介面 | Terminal、Windows Terminal、VS Code terminal |
| Shell | 解讀命令並啟動程式的命令解譯器 | Bash、Zsh、PowerShell、cmd |
| 命令列 | 用文字命令操作系統或程式的互動方式 | 輸入 `cd`、`ls`、`npm run dev` |

可以用一個比喻理解：

- 終端像「窗口」。
- Shell 像「聽懂命令並安排執行的人」。
- 命令列像「用文字下指令這種操作方式」。

例如你在 macOS 打開 Terminal，裡面跑的是 Zsh：

```text
Terminal 視窗 -> Zsh Shell -> 執行你輸入的命令
```

例如你在 Windows Terminal 裡打開 PowerShell：

```text
Windows Terminal 視窗 -> PowerShell -> 執行你輸入的命令
```

所以：

**終端不是 Shell；終端提供操作介面，Shell 負責解讀命令。**

只是日常對話裡，人們常把它們混著說。

## 終端和客戶端 / 服務端有什麼關係？

終端本身主要是操作入口，不是客戶端或服務端的同義詞。

但你可以透過終端啟動或使用客戶端、服務端程式。

### 情境 1：用終端發 API 請求

你在終端輸入：

```bash
curl https://example.com/api/products
```

這裡可以這樣看：

- 終端：你輸入命令的介面。
- Shell：解讀並執行 `curl`。
- `curl`：發出 HTTP 請求的客戶端工具。
- `example.com` 的 API：提供資料的服務端。

所以這個場景不是「終端是客戶端」。
更精準地說，是：

**你透過終端執行了一個客戶端工具。**

### 情境 2：用終端連遠端主機

你在終端輸入：

```bash
ssh user@example.com
```

這裡可以這樣看：

- 終端：你輸入命令的介面。
- `ssh`：發起連線的客戶端程式。
- 遠端主機上的 SSH 服務：服務端。
- 遠端主機：承載 SSH 服務的伺服器。

### 情境 3：用終端啟動本地開發伺服器

你在專案裡輸入：

```bash
npm run dev
```

可能會啟動一個本地開發服務，例如：

```text
http://localhost:5173
```

這時：

- 終端：顯示 dev server 的啟動與 log。
- 本地 dev server：在你電腦上提供網頁資源的服務端。
- 瀏覽器：打開 `localhost` 的客戶端。

所以同一台電腦上可能同時有：

- 終端操作入口
- 本地服務端程式
- 瀏覽器客戶端

這就是為什麼不能用「哪台機器」來死判斷客戶端或服務端。

## 具體場景對照表

| 場景 | 客戶端 | 服務端 | 終端是不是必要 |
|---|---|---|---|
| 用瀏覽器打開網站 | 瀏覽器 | 網站服務 | 不一定 |
| 用手機 App 查訂單 | App | 後端服務 | 不需要 |
| 用 `curl` 請求 API | `curl` | API 服務 | 通常透過終端執行 |
| 用 `ssh` 連遠端主機 | SSH client | SSH server | 通常透過終端執行 |
| 用瀏覽器打開本地 dev server | 瀏覽器 | 本地 dev server | 啟動服務時通常需要 |
| 用資料庫管理工具查資料 | 資料庫管理工具 | 資料庫服務 | 不一定 |

這張表的重點是：

**客戶端 / 服務端看互動角色；終端看操作入口。**

## 同一個東西在不同情境下角色會變

同一台電腦、同一個程式，角色可能因互動場景不同而變化。

例如你的筆電：

- 用瀏覽器打開網站時，它所在的一端通常是客戶端。
- 啟動本地網站服務並讓瀏覽器請求時，它也可以扮演服務端。
- 打開終端輸入命令時，它又提供了操作入口。

再例如一個後端服務：

- 對前端來說，它是服務端。
- 但如果它又去請求另一個第三方 API，它在那次互動中也可能扮演客戶端。

所以不要把「客戶端」和「服務端」理解成永遠固定的身份。
它們更像是在某次互動中的角色。

## 初學時怎麼判斷？

遇到名詞混亂時，可以照這個順序問：

### 1. 這裡是在談請求關係嗎？

如果是，先問：

- 誰發出請求？
- 誰接收請求？
- 誰提供結果？

發出請求的是客戶端，提供結果的是服務端。

### 2. 這裡是在談服務跑在哪裡嗎？

如果是，可能是在談伺服器。

再問：

- 是實體主機？
- 是雲端環境？
- 是本地開發服務？
- 是某個服務角色？

### 3. 這裡是在談輸入命令嗎？

如果是，可能是在談終端、Shell 或命令列。

再問：

- 哪個視窗或介面接收輸入？這通常是終端。
- 哪個程式解讀命令？這通常是 Shell。
- 是不是用文字命令操作？這是命令列操作方式。

## 常見混淆點

| 混淆說法 | 更清楚的理解 |
|---|---|
| 客戶端就是自己的電腦 | 客戶端是發出請求、使用服務的一端，不一定固定是哪台設備 |
| 服務端就是遠端大機器 | 服務端是提供服務的一端，也可以是本地開發服務 |
| 伺服器就是服務端 | 伺服器偏承載服務的機器或環境，服務端偏互動角色或服務端邏輯 |
| 終端就是伺服器 | 終端通常是操作入口，不是伺服器 |
| 打開終端就是在操作服務端 | 不一定；你可能只是操作本機，也可能透過命令連到遠端 |
| Shell 就是終端 | 終端是介面，Shell 是解讀命令的程式 |
| 命令列就是某一個軟體 | 命令列是一種文字命令操作方式，不是單一軟體 |
| 同一台電腦只能是客戶端或服務端其中一種 | 同一台電腦在不同互動中可以扮演不同角色 |

## 初學時最該記住的六句話

1. 客戶端 / 服務端是在談互動角色。
2. 客戶端是發出請求、使用服務的一端。
3. 服務端是接收請求、提供服務的一端。
4. 伺服器偏承載服務的機器、環境或角色。
5. 終端是人輸入命令、查看輸出的操作入口。
6. Shell 不是終端；Shell 負責解讀並執行命令。

## 自測問題

讀完這篇後，可以試著回答：

1. 為什麼不能把客戶端固定理解成「自己的電腦」？
2. 服務端和伺服器有什麼差別？
3. 終端、Shell、命令列三者各自是什麼？
4. 在 `curl https://example.com/api` 這個場景中，誰是客戶端？誰是服務端？終端扮演什麼角色？
5. 在本地執行 `npm run dev` 後用瀏覽器打開 `localhost`，客戶端和服務端分別是什麼？
6. 為什麼同一台電腦可能同時扮演客戶端和服務端？

如果這六題都能說清楚，客戶端、服務端、終端這組概念就基本穩了。

## 一句話抓核心

客戶端 / 服務端是在談一次互動中誰發出請求、誰提供服務；終端是在談人透過什麼入口輸入命令、操作系統或程式。它們常在同一個開發場景中一起出現，但不是同一個概念。
