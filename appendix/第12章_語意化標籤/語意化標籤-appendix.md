## 3. 名詞表

| 名詞 | 英文 | 簡要定義 | 用途 | 注意事項 | 來源 |
|---|---|---|---|---|---|
| 語意化 HTML | semantic HTML | 根據內容意義選擇合適 HTML 元素，而不是只根據畫面外觀選標籤。 | 讓 HTML 原始碼直接表達頁首、導覽、主要內容、文章、補充資訊與頁尾等角色。 | 語意化標籤不是為了讓畫面自動變漂亮，也不是取代 CSS。 | §2、§3.1、§3.2 |
| 語意 | semantics | HTML 元素所表達的內容角色與文件結構意義。 | 幫助人、瀏覽器、搜尋引擎與輔助工具理解頁面。 | 不應只看預設樣式決定使用哪個標籤。 | §2、§3.2、§6、§9 |
| 無語意容器 | generic container | 本身不表達具體內容角色的容器，例如 `div`。 | 在沒有更精準元素時包住一組內容。 | 有合適語意元素時，應優先使用語意元素。 | §3.4、§7、§9 |
| 頁面主要內容 | main content | 頁面中使用者最需要閱讀或操作的主要區域。 | 以 `main` 包住頁面主內容。 | 一頁通常只應有一個主要 `main`。 | §4.1、§4.2、§7、§9 |
| 可獨立內容 | self-contained content | 可以被獨立閱讀或重複發布的內容單元。 | 判斷是否適合使用 `article`。 | 一般主題區塊不一定適合使用 `article`。 | §4.1、§5.1、§7 |
| 主題區塊 | thematic section | 文件中有明確主題的內容區域。 | 判斷是否適合使用 `section`。 | `section` 通常應有能說明主題的標題。 | §4.1、§5.1、§7 |
| 可訪問性 | accessibility | 讓輔助工具與不同使用者能理解頁面區域與內容角色的設計方向。 | 語意化標籤有助於螢幕閱讀器理解與跳轉頁面區域。 | 原筆記未展開 ARIA 或輔助工具完整規則。 | §1、§6、§7、§8、§9 |
| SEO | Search Engine Optimization | 搜尋引擎理解頁面上下文與重要內容時相關的概念。 | 合理語意與標題結構有助於搜尋引擎理解內容。 | SEO 不只取決於標籤。 | §1、§6、§7、§8、§9 |
| CSS | CSS | 控制視覺呈現、排版與樣式的技術。 | 搭配語意化 HTML 控制外觀。 | HTML 負責內容與結構，CSS 負責視覺呈現。 | §3.2、§4.3、§8、§11.3 |
| DOM | Document Object Model | 瀏覽器解析 HTML 後建立的內容結構。 | 讓 CSS 與 JavaScript 可操作語意化元素。 | 原筆記只說明關聯，未教 DOM API。 | §6、§8 |

## 4. HTML 元素表

| 元素 | 用途 | 常用屬性 | 使用情境 | 最小範例 | 注意事項 | 來源 |
|---|---|---|---|---|---|---|
| `header` | 表示頁面或區塊的頭部內容。 | `class` | 網站頁首、文章頭部、區塊標題與簡介。 | `<header><h1>網站名稱</h1></header>` | 可以出現在整頁，也可以出現在 `article` 內部。 | §3.1、§3.3、§4.1、§5.2、§9 |
| `nav` | 表示主要或重要的導覽連結區。 | `aria-label`、`class` | 主導覽、頁內目錄、重要連結群組。 | `<nav><a href="/">首頁</a></nav>` | 不是所有連結清單都需要放進 `nav`。 | §3.1、§4.1、§4.2、§7、§9 |
| `main` | 表示頁面的主要內容。 | `class` | 包住目前頁面最核心的內容或工作區。 | `<main>主要內容</main>` | 一頁通常只應有一個主要 `main`。 | §4.1、§4.2、§6、§7、§9 |
| `article` | 表示可獨立閱讀或重複發布的內容。 | `class` | 文章、新聞、部落格貼文、評論、卡片型內容。 | `<article><h2>文章標題</h2></article>` | 不應當成一般排版區塊使用。 | §3.1、§4.1、§4.2、§5.1、§7、§9 |
| `section` | 表示文件中的主題區塊。 | `class` | 文章段落、頁面主題區、精選商品等有明確主題的區域。 | `<section><h2>主題</h2></section>` | 通常應搭配清楚標題；純排版容器不適合用 `section`。 | §3.1、§4.1、§4.2、§7、§9 |
| `aside` | 表示與主要內容相關但不是主線的補充內容。 | `class` | 側邊欄、延伸閱讀、相關連結、補充說明。 | `<aside><h2>延伸閱讀</h2></aside>` | 應與主要內容有關，但不應取代主內容。 | §3.1、§4.1、§4.2、§5.1、§9 |
| `footer` | 表示頁面或區塊的底部資訊。 | `class` | 版權、作者、聯絡資訊、相關連結、文章分類。 | `<footer><p>聯絡資訊</p></footer>` | 可以是整頁頁尾，也可以是 `article` 的底部資訊。 | §3.1、§3.3、§4.1、§5.2、§9 |
| `div` | 無語意通用容器。 | `class` | 沒有更精準語意元素時包住內容，或作為純排版容器。 | `<div class="wrapper">...</div>` | 不應在有明確內容角色時取代語意化標籤。 | §2、§3.4、§4.3、§7、§9 |

## 9. 常用片段

### 9.1 基本語意化頁面骨架

| 項目 | 說明 |
|---|---|
| 使用情境 | 建立含頁首、導覽、主內容與頁尾的基本頁面結構。 |
| 使用前提 | 各區塊確實對應頁面內容角色。 |
| 來源 | §3.2、§4.2 |

```html
<header>
  <h1>前端學習筆記</h1>
  <nav>
    <a href="/">首頁</a>
    <a href="/html">HTML</a>
  </nav>
</header>

<main>
  <article>
    <h2>HTML 語意化標籤</h2>
    <p>學習如何用正確元素描述頁面結構。</p>
  </article>
</main>

<footer>
  <p>&copy; 2026 Frontend Notes</p>
</footer>
```

### 注意事項

- 這段結構的價值在於角色清楚，不是因為標籤本身會產生特殊版面。
- CSS 仍負責實際排版與視覺呈現。

### 9.2 舊式 `div` 版面改成語意化標籤

| 項目 | 說明 |
|---|---|
| 使用情境 | 原本用 `div.header`、`div.nav`、`div.footer` 表示明確內容角色。 |
| 使用前提 | 區塊角色可由原生語意元素直接表達。 |
| 來源 | §3.4、§4.3 |

```html
<body>
  <header>頭部標籤</header>
  <nav>導航欄標籤</nav>
  <section>某個區域</section>
</body>
```

### 注意事項

- `div` 本身沒有語意，必須靠 `class` 或上下文猜用途。
- 若有原生 HTML 元素可表達語意，通常應優先使用原生元素。

### 9.3 `article` 內部的 `header` 與 `footer`

| 項目 | 說明 |
|---|---|
| 使用情境 | 一篇文章有自己的標題、作者、發布日期、分類或底部資訊。 |
| 使用前提 | `header` 與 `footer` 描述的是該篇 `article` 的範圍。 |
| 來源 | §3.3、§5.2 |

```html
<article>
  <header>
    <h2>語意化 HTML 的基本觀念</h2>
    <p>作者：Alex，發布日期：2026-05-22</p>
  </header>

  <p>語意化 HTML 的核心是根據內容意義選擇合適元素。</p>

  <footer>
    <p>分類：HTML 基礎</p>
  </footer>
</article>
```

### 注意事項

- 這裡的 `header` 與 `footer` 不代表整個頁面的頁首與頁尾。
- 語意化標籤是否能重複使用，取決於它描述的內容範圍。

### 9.4 使用 `section` 表示有主題的區塊

| 項目 | 說明 |
|---|---|
| 使用情境 | 區塊有明確主題，例如「精選商品」。 |
| 使用前提 | `section` 內應有能說明主題的標題。 |
| 來源 | §7 |

```html
<section class="featured-products">
  <h2>精選商品</h2>

  <article class="product-card">
    <h3>商品 A</h3>
    <p>適合日常使用的入門款。</p>
  </article>
</section>
```

### 注意事項

- `section` 不應只拿來當純排版盒子。
- 若外層只是為了排版且沒有主題，可使用 `div`。

### 9.5 使用 `nav aria-label` 標示主要導覽

| 項目 | 說明 |
|---|---|
| 使用情境 | 頁面有主要導覽連結，且需要清楚標示導覽用途。 |
| 使用前提 | 連結群組屬於主要或重要導覽。 |
| 來源 | §5.1、§7 |

```html
<nav aria-label="主要導覽">
  <a href="/">首頁</a>
  <a href="/courses">課程</a>
  <a href="/contact">聯絡我們</a>
</nav>
```

### 注意事項

- `nav` 直接表達連結群組的導覽角色。
- `aria-label` 在有多組導覽時有助於輔助工具區分用途。

### 9.6 舊環境相容性基礎 CSS

| 項目 | 說明 |
|---|---|
| 使用情境 | 專案需要支援舊版 IE 或特殊舊環境。 |
| 使用前提 | 已確認目標環境需要對 HTML5 語意化元素補上基礎顯示設定。 |
| 來源 | §11.1 |

```css
header,
nav,
main,
article,
section,
aside,
footer {
  display: block;
}
```

### 注意事項

- 現代瀏覽器通常不需要額外設定 `display: block` 才能正常使用這些元素。
- 是否需要此片段應依專案支援環境確認。

## 10. 錯誤速查表

| 錯誤現象 | 可能原因 | 修正方向 | 對應知識點 | 來源 |
|---|---|---|---|---|
| 所有區塊都用 `div`。 | `div` 沒有語意，頁面結構不清楚。 | 有明確內容角色時，優先使用語意化標籤。 | 語意化 HTML、無語意容器 | §3.4、§7 |
| 只因為想排版就使用 `section`。 | 把 `section` 當成通用容器。 | 純排版容器可用 `div`；有主題的區塊才用 `section`。 | `section`、主題區塊 | §4.1、§7 |
| 每個連結群組都放進 `nav`。 | 誤以為所有連結都屬於導覽。 | 只有主要導覽、頁內目錄等重要導航才使用 `nav`。 | `nav`、導覽連結 | §4.1、§7 |
| 把 `article` 當成一般區塊。 | 忽略 `article` 應可獨立閱讀或重複發布。 | 可獨立內容用 `article`；一般主題區塊用 `section`；通用容器用 `div`。 | `article`、`section`、`div` | §4.1、§7 |
| 一頁放多個主要 `main`。 | 混淆頁面主要內容與一般內容區域。 | 一個頁面通常只保留一個主要 `main`。 | `main`、頁面主要內容 | §4.1、§7 |
| 用語意化標籤後不寫標題。 | 區塊主題不清楚，讀者與輔助工具較難理解。 | `section`、`article` 等區塊通常應搭配清楚標題。 | 主題區塊、標題結構 | §4.1、§7 |
| 認為語意化只為了 SEO。 | 只看到搜尋引擎面向，忽略其他用途。 | 同時考慮可讀性、可訪問性、維護性與內容結構。 | SEO、可訪問性、維護性 | §6、§7、§9 |
| 使用語意化標籤後不再使用 `class`。 | 誤以為語意元素與 class 互斥。 | 語意元素表達角色，`class` 可提供 CSS 或 JavaScript 更精準的選取目標。 | 語意化標籤、CSS、`class` | §4.3、§11.3 |

## 11. 相似概念對照表

### 11.1 語意化標籤 vs `div`

| 比較項目 | 語意化標籤 | `div` |
|---|---|---|
| 定義 | 直接表達內容角色的 HTML 元素。 | 無語意的通用區塊容器。 |
| 使用時機 | 內容有明確角色，例如頁首、導覽、文章、主題區塊或頁尾。 | 沒有更精準元素，或只是需要通用排版容器。 |
| 常見寫法 | `<nav>...</nav>`、`<footer>...</footer>` | `<div class="nav">...</div>`、`<div class="footer">...</div>` |
| 注意事項 | 不應為了預設樣式而選標籤。 | 不應取代更有語意的原生元素。 |
| 常見錯誤 | 只因為想排版就使用 `section`。 | 所有區塊都用 `div`。 |

### 選用建議

- 有合適語意元素時，優先使用語意化標籤。
- 沒有合適元素或只是純排版容器時，再使用 `div`。

### 11.2 `article` vs `section`

| 比較項目 | `article` | `section` |
|---|---|---|
| 定義 | 可獨立閱讀或重複發布的內容。 | 文件中的主題區塊。 |
| 使用時機 | 文章、新聞、部落格貼文、評論、卡片型內容。 | 文章中的主題段落、頁面中的主題區。 |
| 常見寫法 | `<article><h2>文章標題</h2></article>` | `<section><h2>語意化的目的</h2></section>` |
| 注意事項 | 不應當成一般排版區塊。 | 通常應有能說明主題的標題。 |
| 常見錯誤 | 把 `article` 當成一般區塊。 | 只因為想排版就使用 `section`。 |

### 選用建議

- 內容能獨立被閱讀或重複發布時，優先考慮 `article`。
- 內容是文件中的主題段落時，優先考慮 `section`。

### 11.3 `header` / `footer` 的頁面範圍 vs 區塊範圍

| 比較項目 | 頁面範圍 | 區塊範圍 |
|---|---|---|
| 定義 | 整個網站或頁面的頭部與底部資訊。 | 某個 `article` 或區塊自己的頭部與底部資訊。 |
| 使用時機 | 網站名稱、主要導覽、版權、聯絡資訊。 | 文章標題、作者、發布日期、分類資訊。 |
| 常見寫法 | `<header><h1>網站名稱</h1></header>` | `<article><header>...</header><footer>...</footer></article>` |
| 注意事項 | 頁面頁首與頁尾描述的是整頁。 | 區塊內的 `header`、`footer` 描述的是該區塊。 |
| 常見錯誤 | 認為 `header`、`footer` 只能出現在整頁最上方與最下方。 | 未釐清它們所屬的內容範圍。 |

### 選用建議

- 先判斷 `header` 或 `footer` 是描述整頁，還是描述某個內容單元。
- 同一頁可以有頁面層級與區塊層級的 `header`、`footer`。

### 11.4 `nav` vs 一般連結群組

| 比較項目 | `nav` | 一般連結群組 |
|---|---|---|
| 定義 | 主要或重要的導覽連結區。 | 可能只是普通參考連結、延伸閱讀或內容中的連結。 |
| 使用時機 | 主要導覽、頁內目錄、重要導航。 | 一般文字連結、補充連結、非主要導覽清單。 |
| 常見寫法 | `<nav aria-label="主要導覽">...</nav>` | `<aside><a href="/html/div-span">延伸閱讀</a></aside>` |
| 注意事項 | 不是所有連結都要放進 `nav`。 | 若連結屬於重要導覽，應考慮 `nav`。 |
| 常見錯誤 | 每個連結群組都放進 `nav`。 | 主要導覽卻只用無語意 `div` 包住。 |

### 選用建議

- 判斷連結群組是否具有導覽用途與重要性。
- 主要導覽可搭配 `aria-label` 區分用途。

### 11.5 HTML 語意 vs CSS 外觀

| 比較項目 | HTML 語意 | CSS 外觀 |
|---|---|---|
| 定義 | 描述內容角色與文件結構。 | 控制版面、顏色、尺寸、間距等視覺呈現。 |
| 使用時機 | 決定內容應使用哪個 HTML 元素。 | 決定內容在畫面上如何呈現。 |
| 常見寫法 | `<main>主要內容</main>` | `header, nav { width: 800px; }` |
| 注意事項 | 不應只為了預設樣式選標籤。 | 不取代 HTML 的語意責任。 |
| 常見錯誤 | 認為語意化標籤會自動完成排版。 | 用外觀需求反推錯誤 HTML 結構。 |

### 選用建議

- HTML 先回答「這段內容是什麼」。
- CSS 再回答「這段內容看起來如何」。
