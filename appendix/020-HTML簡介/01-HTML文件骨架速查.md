# HTML 文件骨架速查

> 查表用。概念與用法說明見 notes/020-HTML簡介/03-HTML文件基本骨架.md。

## HTML5 基本骨架

```html
<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <title>我的網頁</title>
  </head>
  <body>
    <h1>歡迎來到我的網頁</h1>
  </body>
</html>
```

## 骨架層級

```text
<!DOCTYPE html>      ← 文件類型宣告，不是標籤，寫在最前面
<html>                ← 整份文件的根容器
├── <head>           ← 「設定區」：放不會直接顯示在畫面上的資訊
│   ├── <meta>
│   └── <title>
└── <body>           ← 「內容區」：放會顯示在畫面上的內容
    └── <h1>
```

## 骨架項目對照

| 骨架項目 | 位置 / 層級 | 角色 | 是否直接顯示在頁面上 | 備註 |
| --- | --- | --- | --- | --- |
| `<!DOCTYPE html>` | 整份文件最開頭 | 告訴瀏覽器用 HTML5 標準解析文件 | 否 | 不是 HTML 標籤 |
| `<html>` | 根容器 | 包住整份 HTML 文件內容 | 否 | 內含 `<head>` 與 `<body>` |
| `lang="zh-Hant"` | `<html>` 屬性 | 告訴瀏覽器、搜尋引擎、螢幕報讀器文件主要語言 | 否 | 範例中表示繁體中文 |
| `<head>` | `<html>` 的子元素 | 頁面的設定區，放描述文件的資訊 | 否 | 應位於 `<body>` 前 |
| `<meta charset="UTF-8">` | `<head>` 的子元素 | 宣告文件使用的文字編碼 | 否 | 可避免中文等字元顯示成亂碼 |
| `<title>` | `<head>` 的子元素 | 設定瀏覽器分頁標題列文字 | 否 | 分頁標題異常時可先檢查此標籤 |
| `<body>` | `<html>` 的子元素 | 頁面的內容區 | 是 | 之後學到的大部分標籤都會寫在這裡 |
| `<h1>` | 範例中位於 `<body>` 內 | 顯示在畫面上的標題內容 | 是 | 範例內容為「歡迎來到我的網頁」 |

## 常見錯誤與避免方法

| 錯誤 | 可能影響 | 避免方法 |
| --- | --- | --- |
| 漏寫或寫錯 `<!DOCTYPE html>` | 瀏覽器可能以相容模式解析，導致樣式、版面出現預期之外的差異 | 每份 HTML5 文件第一行固定寫 `<!DOCTYPE html>` |
| 把顯示內容寫到 `<head>` 裡 | 瀏覽器可能自動結束 `<head>` 或重新處理內容，DOM 結構不一定符合預期 | 所有要顯示在畫面上的內容一律寫在 `<body>` 內 |
| 把內容寫在 `<html>` 標籤外 | 瀏覽器多數情況會自動修正位置，但容易造成結構混亂 | 所有內容放入 `<html>` 內 |
| `<head>` 與 `<body>` 順序顛倒或巢狀錯誤 | 違反文件結構基本假設，也讓程式碼難以閱讀 | 固定按照 `<head>` 在前、`<body>` 在後的順序書寫 |
