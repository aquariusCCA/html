## 9. 常用片段

### 9.1 四組文本格式化標籤最小對照

| 項目 | 說明 |
|---|---|
| 使用情境 | 快速觀察四組外觀相近的標籤。 |
| 使用前提 | 需要同時比較視覺效果與語意差異。 |
| 來源 | §5.1 |

```html
<b>加粗</b>
<strong>重要</strong>

<u>底線標註</u>
<ins>新增內容</ins>

<i>術語或外文詞</i>
<em>語氣強調</em>

<s>不再適用</s>
<del>刪除內容</del>
```

### 注意事項

- 這些元素的預設外觀可能相近，但語意不同。
- 選擇標籤時應先判斷文字的內容意義。

### 9.2 重要提醒

| 項目 | 說明 |
|---|---|
| 使用情境 | 表示內容上真的需要特別注意。 |
| 使用前提 | 文字不是單純為了變粗，而是有重要提醒語意。 |
| 來源 | §5.2、§6 |

```html
<p>
  表單送出前，請務必確認
  <strong>必填欄位已完整填寫</strong>。
</p>
```

### 注意事項

- `<strong>` 負責重要語意。
- 文字顏色、大小或間距應交給 CSS 控制。

### 9.3 文件修訂內容

| 項目 | 說明 |
|---|---|
| 使用情境 | 表示原內容被刪除，並插入新的內容。 |
| 使用前提 | 內容具有修訂、改稿或版本差異的語意。 |
| 來源 | §5.3 |

```html
<p>
  報名截止日期：
  <del>2026-06-01</del>
  <ins>2026-06-08</ins>
</p>
```

### 注意事項

- `<del>` 表示原本內容已被刪除。
- `<ins>` 表示新的內容是後來插入的。

### 9.4 語氣強調

| 項目 | 說明 |
|---|---|
| 使用情境 | 強調句子中的某個字詞。 |
| 使用前提 | 需要表達語氣重點，而不是單純斜體外觀。 |
| 來源 | §5.4 |

```html
<p>
  這個屬性不是可選的，而是<em>必須</em>提供。
</p>
```

### 注意事項

- `<em>` 放在不同字詞上，句子強調重點也可能不同。
- 若只是術語、外文詞或作品名，可依筆記內容考慮 `<i>`。

### 9.5 純視覺粗體交給 CSS

| 項目 | 說明 |
|---|---|
| 使用情境 | 文字只是設計上需要變粗，內容本身沒有重要語意。 |
| 使用前提 | 需求屬於外觀控制，不是 HTML 語意。 |
| 來源 | §7 |

```html
<p class="lead-text">這段只是設計稿要求變粗，沒有特別重要。</p>
```

```css
.lead-text {
  font-weight: 700;
}
```

### 注意事項

- 這段片段來自 notes 的錯誤修正方向。
- HTML 保持一般段落語意，CSS 控制粗體外觀。

## 11. 相似概念對照表

### 11.1 `<b>` vs `<strong>`

| 比較項目 | `<b>` | `<strong>` |
|---|---|---|
| 定義 | 視覺上醒目的文字。 | 內容上重要、需要特別注意的文字。 |
| 使用時機 | 關鍵字、產品名稱、摘要中的醒目字詞。 | 警告、重點提醒、重要結論。 |
| 常見寫法 | `<b>醒目文字</b>` | `<strong>重要文字</strong>` |
| 注意事項 | 不表示內容更重要。 | 不應只為了粗體外觀使用。 |
| 常見錯誤 | 用 `<b>` 表示重要警告。 | 把所有粗體都寫成 `<strong>`。 |

### 選用建議

- 內容真的重要時，使用 `<strong>`。
- 只是視覺上要醒目但不表示重要時，依筆記可考慮 `<b>` 或 CSS。

### 11.2 `<u>` vs `<ins>`

| 比較項目 | `<u>` | `<ins>` |
|---|---|---|
| 定義 | 底線標註或特殊標記。 | 新增、插入的內容。 |
| 使用時機 | 標註文字、特殊註記。 | 文件修訂、版本差異、補充新增內容。 |
| 常見寫法 | `<u>標註文字</u>` | `<ins>新增內容</ins>` |
| 注意事項 | 不表示新增內容。 | 不應只是為了底線外觀而使用。 |
| 常見錯誤 | 用 `<u>` 表示新增內容。 | 把純底線樣式需求寫成 `<ins>`。 |

### 選用建議

- 要表達新增內容時，使用 `<ins>`。
- 只是需要底線外觀時，優先評估 CSS 或更合適的語意。

### 11.3 `<i>` vs `<em>`

| 比較項目 | `<i>` | `<em>` |
|---|---|---|
| 定義 | 與一般文字不同的語氣、術語、外文詞或作品名等。 | 語氣上的強調。 |
| 使用時機 | 技術詞、外文詞、另一種語氣。 | 句子中特別要強調的字詞。 |
| 常見寫法 | `<i>inline element</i>` | `<em>必須</em>` |
| 注意事項 | 不等於語氣強調。 | 不只是一般斜體工具。 |
| 常見錯誤 | 用 `<i>` 表示強調語氣。 | 把 `<em>` 當作斜體樣式工具。 |

### 選用建議

- 表示術語或外文詞等不同語氣時，可考慮 `<i>`。
- 需要改變句子強調重點時，使用 `<em>`。

### 11.4 `<s>` vs `<del>`

| 比較項目 | `<s>` | `<del>` |
|---|---|---|
| 定義 | 內容已不再正確或不再相關。 | 內容已被刪除。 |
| 使用時機 | 原價、過期資訊、不再適用的說法。 | 文件修訂、文章改稿、版本差異。 |
| 常見寫法 | `<s>NT$ 1,200</s>` | `<del>2026-06-01</del>` |
| 注意事項 | 不一定是正式刪除。 | 不應用於所有刪除線外觀。 |
| 常見錯誤 | 用 `<s>` 表示文件刪除。 | 用 `<del>` 表示單純原價不再適用。 |

### 選用建議

- 表示內容不再適用時，可考慮 `<s>`。
- 表示文件修訂中被刪除的內容時，使用 `<del>`。

### 11.5 HTML 語意 vs CSS 樣式

| 比較項目 | HTML 語意 | CSS 樣式 |
|---|---|---|
| 定義 | 描述內容本身的意思。 | 控制內容在畫面上的呈現。 |
| 使用時機 | 判斷文字是重要、強調、新增、刪除或不再適用。 | 調整文字顏色、粗細、底線、斜體等視覺效果。 |
| 常見寫法 | `<strong>注意</strong>` | `.notice { color: #b42318; }` |
| 注意事項 | 不應只為了預設樣式選標籤。 | 不取代 HTML 的語意責任。 |
| 常見錯誤 | 把外觀需求誤寫成語意標籤。 | 只靠顏色或外觀傳達重要資訊。 |

### 選用建議

- HTML 先回答「這段文字代表什麼意思」。
- CSS 再回答「這段文字看起來如何」。
