# HTML5 布局標籤速查

> 查表資料整理自 `notes/270-布局標籤/01-html5-語意化布局標籤.md`。

## 常見 HTML5 布局標籤

| 標籤 | 代表意義 | 常見使用情境 | 使用提醒 |
| --- | --- | --- | --- |
| `<header>` | 頭部、頁首或某個區塊的開頭 | 網站標題、Logo、頁面標題區 | 放在整個頁面最上方時，通常代表網站或頁面的頁首；放在 `<article>` 裡時，也可以代表文章自己的開頭。 |
| `<nav>` | 導覽區 | 主選單、頁尾連結、側邊欄分類導覽 | 適合用在一組具有導覽功能的連結；普通文字連結不需要硬包 `<nav>`。 |
| `<main>` | 頁面主要內容 | 每個頁面最核心、最直接相關的內容 | 一個頁面通常只會有一個主要的 `<main>`；頁首、全站導覽、側邊補充資訊與頁尾通常不要全部塞進 `<main>`。 |
| `<article>` | 可獨立理解的完整內容 | 一篇文章、一則商品介紹、一則新聞 | 如果這塊內容單獨拿出來，讀者仍然能理解它在講什麼，通常就適合用 `<article>`。 |
| `<section>` | 同一主題下的內容分段 | 文章中的章節、頁面中的主題區塊 | 適合有明確主題、通常可以加標題的內容區域。 |
| `<aside>` | 側邊或補充內容 | 側欄、相關連結、推薦內容 | 重點是「補充內容」，不一定只能放在畫面右側。 |
| `<footer>` | 尾部、頁尾或區塊結尾 | 版權資訊、友情連結、補充導覽 | 可以代表整頁頁尾，也可以放在某個文章或區塊的結尾。 |
| `<hgroup>` | 標題與副標題群組 | 較少使用 | 初學時了解即可。 |

## 頁面結構角色速查

| 頁面區域 | 適合標籤 | 角色說明 |
| --- | --- | --- |
| 頁面上方網站或頁面標題區 | `<header>` | 表示頁首或區塊開頭。 |
| 一組主要導覽連結 | `<nav>` | 表示導覽區。 |
| 頁面最核心、最直接相關的內容 | `<main>` | 表示主要內容。 |
| 一篇完整文章或可獨立理解的內容 | `<article>` | 表示內容本身具有獨立性。 |
| 文章或頁面中同一主題下的小節 | `<section>` | 表示有主題的內容分段。 |
| 相關連結、推薦內容或側欄補充資訊 | `<aside>` | 表示和主要內容相關，但不是主線內容。 |
| 頁尾、版權資訊、友情連結或區塊結尾 | `<footer>` | 表示尾部或結尾資訊。 |

## article / section / div 選用判斷

| 標籤 | 何時使用 | 判斷重點 |
| --- | --- | --- |
| `<article>` | 內容相對獨立、完整 | 單獨拿出來仍可理解。 |
| `<section>` | 同一主題下的有標題分段 | 是一個有主題的小節。 |
| `<div>` | 只是排版、分組或樣式容器 | 沒有特定語意。 |

可以依序判斷：

| 判斷順序 | 問題 | 建議 |
| --- | --- | --- |
| 1 | 這塊內容能不能獨立成一篇完整內容？ | 可以時，考慮 `<article>`。 |
| 2 | 這塊內容是不是同一主題下的一個小節，而且通常可以加標題？ | 可以時，考慮 `<section>`。 |
| 3 | 只是為了套 CSS、排版或包住幾個元素？ | 使用 `<div>`。 |

## 常見錯誤與修正

| 問題 | 不建議寫法 | 建議寫法 | 原因 |
| --- | --- | --- | --- |
| 整頁只用 `div` 分區 | `<div class="header">...</div>`、`<div class="nav">...</div>`、`<div class="main">...</div>`、`<div class="footer">...</div>` | `<header>...</header>`、`<nav>...</nav>`、`<main>...</main>`、`<footer>...</footer>` | 只靠 `class` 猜測結構時，HTML 本身沒有清楚語意。 |
| 把 `<section>` 當成萬用容器 | `<section class="button-group">...</section>` | `<div class="button-group">...</div>` | 按鈕群組只是排版或分組，沒有形成有主題的內容小節。 |
| 混淆 `<article>` 和 `<section>` | 整篇文章用 `<section>` 包住，或把文章裡每個小節都寫成 `<article>` | 整篇內容用 `<article>`，文章內不同小節用 `<section>` | `<article>` 強調完整、獨立；`<section>` 強調同一主題下的分段。 |
| 只看畫面位置決定標籤 | 因為看起來在旁邊就一定用 `<aside>` | 先判斷內容角色，再決定標籤 | 語意化標籤重點是內容角色，不是畫面位置。 |

## 實務使用建議

| 情境 | 建議 |
| --- | --- |
| 決定要用哪個布局標籤 | 先判斷內容角色，再決定標籤，不要只看畫面位置。 |
| 放置頁面主要內容 | 使用 `<main>`，但頁首、全站導覽、頁尾通常不要放進 `<main>`。 |
| 標記導覽連結 | 有導覽功能的一組連結用 `<nav>`，普通文字連結不需要硬包 `<nav>`。 |
| 標記完整、可獨立理解的內容 | 使用 `<article>`。 |
| 標記有主題、可加標題的分段 | 使用 `<section>`。 |
| 只是為了排版或樣式分組 | 使用 `<div>` 仍然是正確選擇。 |

語意化不是把所有 `<div>` 換掉，而是在有明確內容角色時使用更準確的標籤。
