## 9. 常用片段

### 9.1 `div` 基本語法

| 項目 | 說明 |
|---|---|
| 使用情境 | 建立一個通用區塊容器。 |
| 使用前提 | 內容沒有更精準的語意元素可用，或需要包住一組內容。 |
| 來源 | §3.2、§4.1 |

```html
<div>這是一個區塊容器。</div>
```

### 注意事項

- `div` 本身沒有語意。
- 若內容是段落、標題、導覽、文章等，應優先考慮更合適的元素。

### 9.2 `span` 基本語法

| 項目 | 說明 |
|---|---|
| 使用情境 | 標記一小段行內內容。 |
| 使用前提 | 內容只是文字中的局部範圍，不是整組區塊結構。 |
| 來源 | §3.3、§4.2 |

```html
<span>這是一小段行內內容。</span>
```

### 注意事項

- `span` 本身沒有語意。
- `span` 適合標記局部文字，不適合包住大型區塊。

### 9.3 用 `div` 包住卡片內容

| 項目 | 說明 |
|---|---|
| 使用情境 | 一組內容需要形成通用卡片或版面區塊。 |
| 使用前提 | 外層容器本身沒有比 `div` 更精準的語意需求。 |
| 來源 | §3.1、§4.1、§5、§6 |

```html
<div class="course-card">
  <h2>HTML 入門</h2>
  <p>學習網頁內容結構與常見標籤用法。</p>
</div>
```

### 注意事項

- 外層 `div` 只作為通用容器。
- 內部的標題與段落仍應使用 `h2`、`p` 等語意元素。

### 9.4 用 `span` 標記局部狀態文字

| 項目 | 說明 |
|---|---|
| 使用情境 | 只需要標記一句話中的狀態、價格或其他局部文字。 |
| 使用前提 | 文字仍屬於同一個段落或句子。 |
| 來源 | §3.1、§4.2、§5、§6 |

```html
<p>
  訂單狀態：
  <span class="status">已出貨</span>
</p>
```

### 注意事項

- `span` 只包住需要局部樣式或局部操作的文字。
- 整段文字仍使用 `p` 表示段落。

### 9.5 修正用 `div` 取代標題與段落

| 項目 | 說明 |
|---|---|
| 使用情境 | 原本全部使用 `div`，但內容其實有標題與段落。 |
| 使用前提 | 外層需要通用容器，內部內容仍有明確語意。 |
| 來源 | §7 |

```html
<div class="lesson-summary">
  <h2>HTML 入門</h2>
  <p>HTML 是用來描述網頁內容與結構的標記語言。</p>
</div>
```

### 注意事項

- `div` 可以作為外層容器。
- 標題與段落應使用 `h2` 與 `p`，不要只用 `div` 模擬外觀。

### 9.6 只更新 `span` 中的局部文字

| 項目 | 說明 |
|---|---|
| 使用情境 | JavaScript 只需要更新一句話中的某個數字或狀態。 |
| 使用前提 | 需要更新的文字被精準包在 `span` 中。 |
| 來源 | §11 |

```html
<p>目前人數：<span id="count">0</span></p>
```

```js
document.querySelector("#count").textContent = "10";
```

### 注意事項

- 此片段只保留 notes 中的最小示例。
- 本章未展開 DOM API 或事件處理，故不生成 API / 方法表。

## 10. 錯誤速查表

| 錯誤現象 | 可能原因 | 修正方向 | 對應知識點 | 來源 |
|---|---|---|---|---|
| 所有內容都用 `div` 包起來。 | 忽略 HTML 元素的語意，導致文件結構不清楚。 | 有標題、段落、導覽、文章等語意時，優先使用對應元素。 | 無語意容器、語意化 HTML | §3.1、§7 |
| 用 `span` 包住一大段區塊內容。 | 混淆行內容器與區塊容器。 | 改用 `div` 或更合適的語意元素。 | `span`、行內型容器 | §3.3、§7 |
| 把 `div` 當成段落使用。 | 一般文字段落應有段落語意。 | 一般文字段落使用 `p`。 | `div` vs `p` | §7 |
| 用 `div` 或 `span` 假裝按鈕。 | 無語意元素缺少原生互動行為與可訪問性支援。 | 使用 `button` 或適合的互動元素。 | 語意化 HTML、可訪問性 | §7 |
| 只根據預設樣式選標籤。 | 把 HTML 當成外觀控制工具。 | 先選語意正確的元素，再用 CSS 控制外觀。 | HTML 語意、CSS 分工 | §4.3、§7、§11 |
| 用 `div` 取代標題與段落。 | 標題與段落有明確語意，`div` 無法表達。 | 外層可用 `div`，內部改用 `h2`、`p` 等語意元素。 | 語意元素、通用容器 | §7 |
| 用 `span` 包住標題與段落。 | `span` 是行內容器，不適合承載整組區塊結構。 | 使用 `div` 作為通用區塊容器，或使用更合適的語意元素。 | `span` vs `div` | §7 |

## 11. 相似概念對照表

### 11.1 `div` vs `span`

| 比較項目 | `div` | `span` |
|---|---|---|
| 定義 | 無語意區塊容器。 | 無語意行內容器。 |
| 預設顯示方式 | 通常獨佔一行，佔滿父元素可用寬度。 | 通常不主動換行，寬度由內容決定。 |
| 使用時機 | 包住一組較大的區塊內容。 | 包住文字中的一小段內容。 |
| 常見寫法 | `<div class="card">...</div>` | `<span class="status">開放報名</span>` |
| 注意事項 | 不要取代更精準的語意元素。 | 不要包住整組區塊結構。 |
| 常見錯誤 | 所有內容都用 `div`。 | 用 `span` 包住標題、段落或卡片。 |

### 選用建議

- 若要包住一組區塊內容，先判斷是否有合適語意元素；沒有時可用 `div`。
- 若只要標記文字中的局部內容，且沒有更合適行內語意元素，可用 `span`。

### 11.2 無語意容器 vs 語意元素

| 比較項目 | 無語意容器 | 語意元素 |
|---|---|---|
| 本章代表 | `div`、`span` | `h1` 到 `h6`、`p`、`nav`、`article`、`header`、`footer` 等 |
| 定義 | 本身不表達具體內容角色。 | 直接表達內容角色或文件結構。 |
| 使用時機 | 沒有更精準語意元素，或需要包住內容供樣式與操作使用。 | 內容具有明確語意，例如標題、段落、導覽、文章、頁首或頁尾。 |
| 常見寫法 | `<div class="lesson-summary">...</div>` | `<h2>HTML 入門</h2>`、`<p>這是一段文字。</p>` |
| 注意事項 | 不應大量取代語意元素。 | 外觀仍交給 CSS 控制，不應只因預設樣式選元素。 |
| 常見錯誤 | 用 `div` 模擬標題與段落。 | 原筆記未列出語意元素本身的其他錯誤。 |

### 選用建議

- 先根據內容角色選擇語意元素。
- 只有在沒有更適合元素，或需要中性包裝時，再使用 `div` 或 `span`。

### 11.3 HTML 結構 vs CSS / JavaScript

| 比較項目 | HTML 結構 | CSS / JavaScript |
|---|---|---|
| 本章定位 | 描述內容與結構，建立 `div`、`span` 等元素節點。 | CSS 控制外觀；JavaScript 可透過 DOM 更新內容。 |
| 使用時機 | 決定內容應該使用哪個元素包住。 | 需要套樣式、排版或局部更新內容時。 |
| 常見寫法 | `<span id="count">0</span>` | `document.querySelector("#count").textContent = "10";` |
| 注意事項 | 不應為了外觀放棄正確語意。 | 不應用樣式或腳本彌補錯誤的 HTML 結構。 |
| 常見錯誤 | 只根據預設樣式選標籤。 | 原筆記未列出 CSS 或 JavaScript 本身的錯誤寫法。 |

### 選用建議

- HTML 先決定「內容是什麼」。
- CSS 再決定「看起來如何」。
- JavaScript 只更新需要操作的節點，例如只更新某個 `span` 中的數字。
