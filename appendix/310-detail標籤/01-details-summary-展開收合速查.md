# details summary 展開收合速查

> 查表用。概念與用法說明見 notes/310-detail標籤/01-details與summary標籤.md。

## 基本語意

| 項目 | 說明 |
| --- | --- |
| `<details>` | 表示一組可以展開或收合的詳細內容 |
| `<summary>` | 這組內容中永遠可見、可被點擊的摘要標題 |
| 詳細內容 | `<details>` 裡除了 `<summary>` 以外的其他內容，展開後才顯示 |
| 主要用途 | 建立簡單的原生展開與收合內容區塊 |
| JavaScript 需求 | 簡單展開收合情境不一定需要先寫 JavaScript |

## 基本結構

| 部分 | 寫法範例 | 用途 |
| --- | --- | --- |
| 外層容器 | `<details>...</details>` | 包住可展開與收合的內容 |
| 摘要標題 | `<summary>文章概要</summary>` | 使用者一開始看得到、可以點擊的標題 |
| 詳細內容 | `<div>這裡放文章的詳細概要內容。</div>` | 使用者展開後看到的內容 |

## 基本片段

```html
<details>
  <summary>文章概要</summary>
  <div>這裡放文章的詳細概要內容。</div>
</details>
```

## open 屬性

| 屬性 | 寫在哪裡 | 作用 | 備註 |
| --- | --- | --- | --- |
| `open` | `<details>` | 讓內容一開始就是展開狀態 | `open` 是布林屬性，只要出現在 `<details>` 上就代表預設展開 |

## 預設展開片段

```html
<details open>
  <summary>文章概要</summary>
  <div>文章內容、重點整理或補充說明可以放在這裡。</div>
</details>
```

## 內容放置

| 位置 | 可放內容 | 說明 |
| --- | --- | --- |
| `<summary>` | 摘要標題文字 | 定義可見標題，使用者點擊時切換展開狀態 |
| `<details>` 裡的其他內容 | 段落、列表、圖片或其他適合的 HTML 內容 | 放入真正的詳細內容 |

## 適合與不適合情境

| 情境 | 是否適合只用 `<details>` | 原因 |
| --- | --- | --- |
| 常見問題的答案展開 | 適合 | 是標題加上一段可選擇查看的詳細資訊 |
| 文章中的補充說明 | 適合 | 不需要一開始全部顯示，但又不能完全藏起來 |
| 表格或清單下方的詳細資訊 | 適合 | 可讓讀者自行展開查看 |
| 不想一開始佔用太多版面的次要內容 | 適合 | 適合簡單展開與收合 |
| 主要導覽選單 | 不適合 | 通常需要更完整的互動元件設計 |
| 多層級複雜互動 | 不適合 | `<details>` 不適合承擔所有互動需求 |
| 需要細緻控制動畫、焦點管理或資料狀態的元件 | 不適合 | 通常需要更完整的 HTML、CSS 與 JavaScript 配合 |

## 判斷準則

| 問題 | 判斷 | 建議 |
| --- | --- | --- |
| 這塊內容是不是「一個標題加上一段可選擇查看的詳細資訊」？ | 是 | 可優先考慮 `<details>` 與 `<summary>` |
| 只是要做簡單的點擊標題後展開內容嗎？ | 是 | 可優先使用原生標籤 |
| 是否要取代所有互動元件？ | 是 | 不適合只靠 `<details>` |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 只寫 `<details>`，不寫 `<summary>` | 瀏覽器可能顯示預設標題，例如 `Details`，不一定符合內容 | 明確提供 `<summary>` |
| 把 `<summary>` 放到 `<details>` 外面 | `<summary>` 不再是該組可展開內容的摘要標題，結構語意不清楚 | 將 `<summary>` 放在 `<details>` 裡，並作為第一個子元素 |
| 用 `<details>` 取代所有互動元件 | `<details>` 適合簡單展開與收合，不適合承擔所有互動需求 | 分頁切換、複雜選單、狀態操作面板等應使用更完整的 HTML、CSS 與 JavaScript 配合 |
