# 圖片格式與 Base64 選擇表

> 查表內容整理自 `notes/180-圖片標籤/02-常見圖片格式與Base64編碼.md`。

## 格式總覽

| 類型 | 主要特性 | 透明背景 | 動態圖 | 常見用途 | 選用提醒 |
| --- | --- | --- | --- | --- | --- |
| JPG / JPEG | 有損壓縮，適合豐富顏色 | 不支援 | 不支援 | 照片、產品宣傳圖 | 圖片是照片且不需要透明背景時，通常是常見選擇。 |
| PNG | 無損壓縮，品質較高 | 支援 | 不支援 | Logo、透明圖、重要配圖 | 需要透明背景或品質要求較高時適合；檔案通常比同類型 JPG 大。 |
| BMP | 通常檔案很大 | 不支援 | 不支援 | 網頁中極少使用 | 正式網頁通常會先轉成 JPG、PNG 或 WebP。 |
| GIF | 顏色少，可做簡單動畫 | 支援簡單透明 | 支援 | 簡單動態圖片 | 適合短小、簡單的動畫；動畫顏色很多或檔案很大時，可考慮影片格式或 WebP 動畫。 |
| WebP | 現代 Web 圖片格式，壓縮能力佳 | 支援 | 支援 | 網頁圖片、透明圖、動畫 | 通常能在相近品質下產生較小檔案；需要兼容舊瀏覽器時應準備 fallback。 |
| Base64 | 編碼方式，不是圖片格式 | 取決於原圖片 | 取決於原圖片 | 小圖或需要內嵌的圖片資料 | 不適合大量或大型圖片；大型圖片通常仍應作為獨立圖片檔案引用。 |

## Base64 使用判斷

| 情境 | 是否適合 Base64 | 原因 |
| --- | --- | --- |
| 很小的圖示 | 適合 | 可直接內嵌在 HTML 或 CSS 中。 |
| 需要和 HTML 或 CSS 一起打包的小型圖片 | 適合 | 不需要再另外透過路徑載入圖片檔案。 |
| 特定情境下想減少額外請求 | 可考慮 | Base64 可以把小圖片資料直接寫在 HTML 或 CSS 裡。 |
| 商品照片 | 不適合 | 圖片通常較大，轉成 Base64 會讓 HTML 或 CSS 變大。 |
| 大型背景圖 | 不適合 | 不利於大型圖片管理。 |
| 很多張圖片 | 不適合 | 會讓 HTML 或 CSS 變大，也不利於管理。 |
| 需要被瀏覽器獨立快取的圖片資源 | 不適合 | 大型圖片通常應作為獨立圖片檔案引用。 |

## 實務選擇準則

| 判斷順序 | 圖片需求 | 優先考慮 |
| --- | --- | --- |
| 1 | 照片或顏色豐富，且不需要透明背景 | JPG 或 WebP |
| 2 | 需要透明背景或高品質圖示 | PNG 或 WebP |
| 3 | 需要簡單動畫 | GIF 或 WebP |
| 4 | 網頁正式資源 | 通常避免直接使用 BMP |
| 5 | 非常小、需要內嵌的圖片資料 | 可以考慮 Base64 |
| 6 | 使用 WebP 且需要兼容舊瀏覽器 | 準備 JPG 或 PNG fallback |

## 常見誤解

| 誤解 | 正確理解 |
| --- | --- |
| Base64 是圖片格式 | Base64 是編碼方式，不是像 JPG、PNG 那樣的圖片格式。 |
| 所有圖片都用 PNG | PNG 支援透明且品質好，但大量照片如果都用 PNG，檔案可能過大。 |
| 所有圖片都轉成 Base64 | Base64 會讓 HTML 或 CSS 變大，也不利於大型圖片管理。 |
| 使用 WebP 一定不用準備其他格式 | 如果專案必須支援舊瀏覽器，可以使用 `<picture>` 提供備用格式。 |
