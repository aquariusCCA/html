# FAQ Index

## 用途

用常見問題快速找到本篇語意化標籤筆記中能回答疑問的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| 什麼是語意化？ | 語意化是讓 HTML 標籤本身能說明內容在頁面中的角色，而不是只靠 class 名稱猜測。 | [什麼是語意化](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#什麼是語意化) |
| 語意化標籤只是把 `div` 換成新標籤嗎？ | 不是。重點是內容角色是否清楚，有語意的地方使用語意化標籤，純包裝或排版仍可使用 `div`。 | [使用語意化標籤的判斷原則](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#使用語意化標籤的判斷原則) |
| 常見的語意化版面標籤有哪些？ | 本篇整理 `header`、`nav`、`main`、`article`、`section`、`aside`、`footer`。 | [常見的語意化版面標籤](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#常見的語意化版面標籤) |
| `main` 可以包住整個頁面嗎？ | 不適合。`main` 通常用來包住本頁最重要、最獨特的內容，不應把頁首、主導覽或頁尾一起放進去。 | [常見的語意化版面標籤](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#常見的語意化版面標籤) |
| 一個文章頁可以怎麼安排語意化結構？ | 可用 `header` 放頁首與導覽，`main` 包主要內容，`article` 放文章，`aside` 放相關內容，`footer` 放頁尾。 | [一個更完整的頁面結構](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#一個更完整的頁面結構) |
| 什麼時候適合使用 `section`？ | 當區塊有自己的主題，通常也有標題時，才適合用 `section`。 | [把 section 當成萬用 div](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#把-section-當成萬用-div) |
| 什麼時候保留 `div` 比較適合？ | 如果區塊只是為了排版、對齊或套 CSS，沒有明確內容角色，使用 `div` 會更自然。 | [使用語意化標籤的判斷原則](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#使用語意化標籤的判斷原則) |
| `nav` 可以用在所有連結區塊嗎？ | 不需要。`nav` 應該用於主要導覽區域，不是任何有連結的地方都要使用。 | [把 nav 用在所有連結區塊](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#把-nav-用在所有連結區塊) |
| 語意化是不是只為了 SEO？ | 不是。語意化也服務於使用者、輔助技術與開發維護，SEO 只是其中一個面向。 | [只為了 SEO 而語意化](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#只為了-seo-而語意化) |
| 語意化對螢幕閱讀器有什麼幫助？ | 螢幕閱讀器可以利用 `nav`、`main`、`article` 等結構，協助使用者理解與跳轉頁面區塊。 | [語意化標籤帶來的好處](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#語意化標籤帶來的好處) |
| 舊瀏覽器使用 HTML5 語意化標籤要注意什麼？ | 現代瀏覽器普遍支援；若要支援較舊瀏覽器，可能需要補上區塊顯示設定。 | [瀏覽器支援注意事項](../../../notes/第13章_語意化標籤/01-語意化標籤與版面結構.md#瀏覽器支援注意事項) |
