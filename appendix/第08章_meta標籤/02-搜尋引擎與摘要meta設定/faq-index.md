# FAQ Index

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `description` 是做什麼的？ | `description` 用來提供頁面摘要，搜尋引擎可能把它作為搜尋結果摘要的候選來源。 | [`description`：頁面摘要描述](../../../notes/第08章_meta標籤/02-搜尋引擎與摘要meta設定.md#description頁面摘要描述) |
| 搜尋結果一定會原封不動使用 `description` 嗎？ | 不一定。搜尋引擎可能依使用者查詢、頁面正文與搜尋情境自行改寫摘要。 | [`description`：頁面摘要描述](../../../notes/第08章_meta標籤/02-搜尋引擎與摘要meta設定.md#description頁面摘要描述) |
| `keywords` 可以讓頁面排名變好嗎？ | 不應把 `keywords` 視為現代搜尋排名保證；它比較適合理解成頁面相關詞記錄。 | [`keywords`：關鍵詞記錄](../../../notes/第08章_meta標籤/02-搜尋引擎與摘要meta設定.md#keywords關鍵詞記錄) |
| `robots` meta 是做什麼的？ | `robots` 用來對搜尋爬蟲提出索引與連結跟隨指示。 | [`robots`：搜尋爬蟲指示](../../../notes/第08章_meta標籤/02-搜尋引擎與摘要meta設定.md#robots搜尋爬蟲指示) |
| `noindex` 和 `nofollow` 差在哪裡？ | `noindex` 要求不要索引頁面，`nofollow` 要求不要跟隨頁面上的連結。 | [`robots`：搜尋爬蟲指示](../../../notes/第08章_meta標籤/02-搜尋引擎與摘要meta設定.md#robots搜尋爬蟲指示) |
| 公開頁面需要特別寫 `index, follow` 嗎？ | 通常不需要；公開頁面多半可以省略 `robots`，有索引或連結跟隨限制時再加入。 | [實務使用原則](../../../notes/第08章_meta標籤/02-搜尋引擎與摘要meta設定.md#實務使用原則) |
| 可以只靠 `robots` 保護私密內容嗎？ | 不可以。`robots` 是給爬蟲的指示，不是登入、權限或伺服器層級的安全控管。 | [用 `robots` 保護私密內容](../../../notes/第08章_meta標籤/02-搜尋引擎與摘要meta設定.md#用-robots-保護私密內容) |
| 好的 `description` 應該怎麼寫？ | 它應該準確描述本頁內容、避免關鍵字堆砌，並保持簡潔。 | [`description`：頁面摘要描述](../../../notes/第08章_meta標籤/02-搜尋引擎與摘要meta設定.md#description頁面摘要描述) |
