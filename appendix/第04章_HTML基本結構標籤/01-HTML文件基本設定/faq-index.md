# FAQ Index

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `<!DOCTYPE html>` 要放在哪裡？ | 必須放在整份 HTML 文件最前面，位於 `<html>` 標籤之前。 | [常見錯誤](../../../notes/第04章_HTML基本結構標籤/01-HTML文件基本設定.md#常見錯誤) |
| `<!DOCTYPE html>` 是 HTML 標籤嗎？ | 不是，它是文件類型宣告，用來宣告 HTML5。 | [重點整理](../../../notes/第04章_HTML基本結構標籤/01-HTML文件基本設定.md#重點整理) |
| `lang` 屬性做什麼？ | `lang` 用來標示元素內容使用的語言；寫在 `<html>` 上時，通常代表整份文件的主要語言。 | [實務使用原則](../../../notes/第04章_HTML基本結構標籤/01-HTML文件基本設定.md#實務使用原則) |
| 繁體中文頁面可以用哪個 `lang` 值？ | 筆記示範可使用 `zh-Hant` 標示繁體中文。 | [實務使用原則](../../../notes/第04章_HTML基本結構標籤/01-HTML文件基本設定.md#實務使用原則) |
| 為什麼需要標示文件語言？ | 文件語言會影響輔助工具朗讀、搜尋引擎理解，以及瀏覽器或翻譯工具判斷。 | [問題情境](../../../notes/第04章_HTML基本結構標籤/01-HTML文件基本設定.md#問題情境) |
| `<meta charset="UTF-8">` 放在哪裡？ | 通常放在 `<head>` 裡，用來指定文件使用 UTF-8 字元編碼。 | [忘記設定字元編碼](../../../notes/第04章_HTML基本結構標籤/01-HTML文件基本設定.md#忘記設定字元編碼) |
| 忘記設定 `charset` 可能發生什麼？ | 中文、符號或其他語言文字可能出現亂碼。 | [忘記設定字元編碼](../../../notes/第04章_HTML基本結構標籤/01-HTML文件基本設定.md#忘記設定字元編碼) |
| 新 HTML 文件可以先從哪個骨架開始？ | 可以先使用包含 `<!DOCTYPE html>`、`<html lang="...">`、`<head>`、`<meta charset="UTF-8">` 與 `<body>` 的基本骨架。 | [實務使用原則](../../../notes/第04章_HTML基本結構標籤/01-HTML文件基本設定.md#實務使用原則) |
