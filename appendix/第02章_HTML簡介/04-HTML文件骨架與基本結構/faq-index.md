# FAQ Index

## 用途

用常見問題快速找到本篇筆記中能回答疑問的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| HTML 文件骨架是什麼？ | HTML 文件骨架是每個網頁的基本外框，用來區分文件設定與實際顯示內容。 | [一句話理解](../../../notes/第02章_HTML簡介/04-HTML文件骨架與基本結構.md#一句話理解) |
| 基本 HTML5 骨架長什麼樣？ | 常見骨架包含 `<!DOCTYPE html>`、`html`、`head`、`meta charset`、`title` 與 `body`。 | [基本 HTML5 骨架](../../../notes/第02章_HTML簡介/04-HTML文件骨架與基本結構.md#基本-html5-骨架) |
| `<!DOCTYPE html>` 有什麼用？ | 它是文件類型宣告，讓瀏覽器以標準模式解析 HTML。 | [結構拆解](../../../notes/第02章_HTML簡介/04-HTML文件骨架與基本結構.md#結構拆解) |
| `<html>` 標籤做什麼？ | `<html>` 是整份 HTML 文件的根元素，所有 HTML 內容都放在裡面。 | [結構拆解](../../../notes/第02章_HTML簡介/04-HTML文件骨架與基本結構.md#結構拆解) |
| `<head>` 和 `<body>` 差在哪？ | `head` 放文件設定與描述資訊，`body` 放使用者會看到或操作的頁面主體內容。 | [head 與 body 的分工](../../../notes/第02章_HTML簡介/04-HTML文件骨架與基本結構.md#head-與-body-的分工) |
| 為什麼要寫 `<meta charset="UTF-8">`？ | 它指定文件編碼，可避免某些環境出現中文或特殊字元亂碼。 | [忘記設定字元編碼](../../../notes/第02章_HTML簡介/04-HTML文件骨架與基本結構.md#忘記設定字元編碼) |
| `<title>` 是頁面上的主標題嗎？ | 不是。`title` 顯示在瀏覽器分頁或搜尋結果中，頁面主標題通常放在 `body` 中。 | [把 title 當成頁面主標題](../../../notes/第02章_HTML簡介/04-HTML文件骨架與基本結構.md#把-title-當成頁面主標題) |
| 可見內容可以放在 `head` 裡嗎？ | 不理想。使用者要看到的標題或內容應放在 `body` 中。 | [把可見內容放進 head](../../../notes/第02章_HTML簡介/04-HTML文件骨架與基本結構.md#把可見內容放進-head) |
| HTML 頁面也是文件嗎？ | 是。HTML 頁面可稱為 HTML 文件，本地檔案通常使用 `.html` 或 `.htm` 副檔名。 | [HTML 頁面也是文件](../../../notes/第02章_HTML簡介/04-HTML文件骨架與基本結構.md#html-頁面也是文件) |
