# FAQ Index

## 用途

用常見問題快速找到本篇筆記中能回答的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| HTML 標籤通常由哪些部分組成？ | 常見 HTML 元素由起始標籤、內容與結束標籤組成；標籤本身由尖括號、斜線與標籤名等語法片段構成。 | [標籤的基本組成](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#標籤的基本組成) |
| 起始標籤和結束標籤差在哪？ | 起始標籤表示元素開始，結束標籤表示元素結束，結束標籤會在標籤名前加上 `/`。 | [標籤的基本組成](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#標籤的基本組成) |
| 什麼是雙標籤？ | 雙標籤是初學教材常用的說法，指有起始標籤與結束標籤的元素。 | [雙標籤與不寫結束標籤的元素](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#雙標籤與不寫結束標籤的元素) |
| 什麼是單標籤或 void elements？ | 有些特殊元素不寫結束標籤，初學教材常稱單標籤；在 HTML 中較精確的說法是 void elements。 | [雙標籤與不寫結束標籤的元素](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#雙標籤與不寫結束標籤的元素) |
| 標籤和元素有什麼差別？ | 標籤是語法片段，元素是由標籤與內容形成的完整節點。 | [元素與標籤的差別](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#元素與標籤的差別) |
| 什麼是嵌套關係？ | 當一個元素包住另一個元素時，就形成嵌套關係。 | [嵌套關係](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#嵌套關係) |
| 父元素與子元素怎麼判斷？ | 直接包住某個元素的外層元素是父元素，被直接包住的元素是子元素。 | [父元素與子元素](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#父元素與子元素) |
| 祖先元素和後代元素與父子關係差在哪？ | 祖先元素包含直接父元素與更外層的父元素，後代元素包含直接子元素與更深層的子元素。 | [祖先元素與後代元素](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#祖先元素與後代元素) |
| 什麼是兄弟元素？ | 同一個父元素底下的同層元素，稱為兄弟元素。 | [兄弟元素與並列關係](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#兄弟元素與並列關係) |
| 標籤交錯閉合為什麼是錯的？ | 巢狀標籤應該先開後關，內層元素要先結束，否則結構會不清楚。 | [標籤交錯閉合](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#標籤交錯閉合) |
| 忘記結束標籤可以嗎？ | 瀏覽器可能會自動修正某些錯誤，但正式撰寫時仍應把結構寫完整。 | [忘記結束標籤](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#忘記結束標籤) |
| 為什麼 `<img></img>` 不理想？ | `img` 是不寫結束標籤的元素，應寫成 `<img src="photo.jpg" alt="照片">` 這類形式。 | [把不該包內容的元素寫成雙標籤](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#把不該包內容的元素寫成雙標籤) |
| 清楚的元素關係對 CSS 和 JavaScript 有什麼幫助？ | HTML 結構層級清楚時，樣式選取和互動操作會更穩定。 | [實務使用場景](../../../notes/第02章_HTML簡介/05-HTML標籤結構與元素關係.md#實務使用場景) |
