| 問題 | 簡短回答 | 對應段落 |

| HTML5 是什麼？ | HTML5 是現代 HTML 的重要版本脈絡，在本教材中指新一代 HTML 標準。 | [HTML5 是什麼](../../../notes/第02章_HTML簡介/02-HTML5版本與兼容性.md#HTML5-是什麼) |
| 狹義 HTML5 和廣義 HTML5 有什麼差別？ | 狹義 HTML5 指 HTML 標準本身；廣義 HTML5 常泛稱現代 Web 技術集合。 | [狹義 HTML5 與廣義 HTML5](../../../notes/第02章_HTML簡介/02-HTML5版本與兼容性.md#狹義-HTML5-與廣義-HTML5) |
| 為什麼不能把所有現代前端技術都叫 HTML5？ | 因為排版多半屬於 CSS，互動多半屬於 JavaScript，瀏覽器能力也可能屬於 Web API。 | [把 HTML5 當成所有前端技術的總稱](../../../notes/第02章_HTML簡介/02-HTML5版本與兼容性.md#把-HTML5-當成所有前端技術的總稱) |
| HTML5 有哪些常見優勢？ | HTML5 強化語意、表單、多媒體支援，並對搜尋引擎、輔助技術與跨平台頁面更友好。 | [HTML5 的常見優勢](../../../notes/第02章_HTML簡介/02-HTML5版本與兼容性.md#HTML5-的常見優勢) |
| 為什麼學 HTML5 還要查兼容性？ | 不同 HTML5 或現代 Web 特性的瀏覽器支援程度不一定相同，正式開發時要確認目標環境。 | [兼容性為什麼重要](../../../notes/第02章_HTML簡介/02-HTML5版本與兼容性.md#兼容性為什麼重要) |
| 現代開發還要以 IE 當主要基準嗎？ | 不應該。IE 在舊資料中主要是歷史對照，不應作為現代開發的主要基準。 | [兼容性為什麼重要](../../../notes/第02章_HTML簡介/02-HTML5版本與兼容性.md#兼容性為什麼重要) |
| 入門階段該先學哪些 HTML5 內容？ | 可以先從基礎文件結構、常用內容標籤、語意化結構標籤、表單與多媒體開始。 | [學習時的取捨](../../../notes/第02章_HTML簡介/02-HTML5版本與兼容性.md#學習時的取捨) |
| 新特性一定要優先使用嗎？ | 不一定。新特性仍要同時考慮語意、兼容性、可維護性與團隊熟悉度。 | [只追求新特性](../../../notes/第02章_HTML簡介/02-HTML5版本與兼容性.md#只追求新特性) |
