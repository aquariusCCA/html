# FAQ Index

## 用途

用常見問題快速找到本篇筆記中能回答的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| 網站和網頁差在哪？ | 網站是一組互相關聯的網頁集合，網頁是其中的一個頁面。 | [網頁與網站是什麼](../../../notes/第02章_HTML簡介/01-網頁與HTML入門.md#網頁與網站是什麼) |
| 網頁通常是什麼檔案？ | 網頁常見形式是一份 HTML 文件，副檔名通常是 `.html` 或 `.htm`。 | [網頁與網站是什麼](../../../notes/第02章_HTML簡介/01-網頁與HTML入門.md#網頁與網站是什麼) |
| HTML 是什麼？ | HTML 是超文本標記語言，用標籤描述網頁內容與結構。 | [HTML 是什麼](../../../notes/第02章_HTML簡介/01-網頁與HTML入門.md#html-是什麼) |
| HyperText Markup Language 是什麼意思？ | 它是 HTML 的英文全稱，常譯為超文本標記語言。 | [HTML 是什麼](../../../notes/第02章_HTML簡介/01-網頁與HTML入門.md#html-是什麼) |
| HTML 為什麼不是程式語言？ | HTML 主要負責標記內容，不負責邏輯判斷、資料計算或流程控制。 | [HTML 不是程式語言](../../../notes/第02章_HTML簡介/01-網頁與HTML入門.md#html-不是程式語言) |
| HTML、CSS、JavaScript 各負責什麼？ | HTML 描述內容與結構，CSS 控制樣式，JavaScript 處理互動行為。 | [HTML 不是程式語言](../../../notes/第02章_HTML簡介/01-網頁與HTML入門.md#html-不是程式語言) |
| 瀏覽器如何把 HTML 變成網頁？ | 瀏覽器會讀取程式碼、解析 HTML 結構，再渲染成可閱讀、可操作的網頁。 | [瀏覽器如何形成網頁](../../../notes/第02章_HTML簡介/01-網頁與HTML入門.md#瀏覽器如何形成網頁) |
| 學 HTML 時為什麼要重視語意與結構？ | 清楚的語意與結構能讓瀏覽器、搜尋引擎、螢幕閱讀器與維護者更容易理解頁面。 | [實務理解方式](../../../notes/第02章_HTML簡介/01-網頁與HTML入門.md#實務理解方式) |
| 初學 HTML 常見錯誤有哪些？ | 常見錯誤包含混淆網站與網頁、只把 HTML 當成畫面效果，以及以為瀏覽器只是顯示文字。 | [常見錯誤](../../../notes/第02章_HTML簡介/01-網頁與HTML入門.md#常見錯誤) |
