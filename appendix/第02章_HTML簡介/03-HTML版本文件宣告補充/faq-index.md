# FAQ Index

## 用途

用常見問題快速找到本篇筆記中能回答的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| 文件宣告是做什麼的？ | 文件宣告用來告訴瀏覽器應該用哪種規則解析文件。 | [一句話理解](../../../notes/第02章_HTML簡介/03-HTML版本文件宣告補充.md#一句話理解) |
| 現代 HTML 頁面通常用哪個文件宣告？ | 現代 HTML 頁面通常使用簡潔的 `<!DOCTYPE html>`。 | [HTML5 文件宣告](../../../notes/第02章_HTML簡介/03-HTML版本文件宣告補充.md#HTML5-文件宣告) |
| `<!DOCTYPE html>` 是 HTML 標籤嗎？ | 不是。它是文件類型宣告，用來讓瀏覽器以標準模式解析頁面。 | [HTML5 文件宣告](../../../notes/第02章_HTML簡介/03-HTML版本文件宣告補充.md#HTML5-文件宣告) |
| HTML4 文件宣告為什麼比較長？ | HTML4 宣告會指定 DTD，因此寫法比 HTML5 宣告更長。 | [HTML4 文件宣告](../../../notes/第02章_HTML簡介/03-HTML版本文件宣告補充.md#HTML4-文件宣告) |
| XHTML 文件宣告有什麼特點？ | XHTML 宣告更嚴格，語法格式更接近 XML 規則。 | [XHTML 文件宣告](../../../notes/第02章_HTML簡介/03-HTML版本文件宣告補充.md#XHTML-文件宣告) |
| 新建頁面需要複製 HTML4 或 XHTML 的長宣告嗎？ | 一般不需要。新建 HTML 頁面通常使用 HTML5 的簡潔宣告即可。 | [把舊宣告複製到新頁面](../../../notes/第02章_HTML簡介/03-HTML版本文件宣告補充.md#把舊宣告複製到新頁面) |
| 可以省略文件宣告嗎？ | 不建議。現代 HTML 文件建議加上 `<!DOCTYPE html>`。 | [省略文件宣告](../../../notes/第02章_HTML簡介/03-HTML版本文件宣告補充.md#省略文件宣告) |
| HTML4 與 XHTML 宣告現在還要背嗎？ | 不需要當成新專案預設寫法，主要用於理解舊資料或維護舊系統。 | [對照表](../../../notes/第02章_HTML簡介/03-HTML版本文件宣告補充.md#對照表) |
