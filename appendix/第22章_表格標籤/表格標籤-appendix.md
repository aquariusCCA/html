## 3. 名詞表

| 名詞 | 英文 | 簡要定義 | 用途 | 注意事項 | 來源 |
|---|---|---|---|---|---|
| 表格 | table | 用於顯示、展示欄列資料的 HTML 結構。 | 呈現後台資料、成績、報表等規整資料。 | 不應用來做版面布局。 | §1 |
| 表格標題 | caption | 說明整張表格主題的內容。 | 放置表格標題，例如「段考成績」。 | 原始資料將其列為完整表格的一部分。 | §2 |
| 表格頭部 | table head / `thead` | 表格中放置欄位標題的區域。 | 分組表格表頭內容。 | 表格頭部中使用 `<th>`。 | §2 |
| 表格主體 | table body / `tbody` | 表格中放置主要資料列的區域。 | 分組主要資料內容。 | 表格主體中使用 `<td>`。 | §2 |
| 表格腳注 | table foot / `tfoot` | 表格中放置腳注、總計或彙總資料的區域。 | 放置總分、總計等資料。 | 原始範例中 `tfoot` 放「總分」。 | §2 |
| 表格列 | table row / `tr` | 表格中的一行。 | 包住同一列的表頭或資料儲存格。 | 必須嵌套在 `<table>` 或表格分區中。 | §1、§2 |
| 表頭儲存格 | table header / `th` | 表格中的表頭單元格。 | 放欄位名稱或表頭內容。 | 原始資料指出表頭單元格內容預設加粗居中顯示。 | §1、§2 |
| 資料儲存格 | table data / `td` | 表格中的一般資料單元格。 | 放主要資料或腳注資料。 | 必須嵌套在 `<tr>` 中。 | §1、§2 |
| 合併儲存格 | merged cell | 多個儲存格合併成一個儲存格。 | 特殊情況下跨行或跨列呈現資料。 | 合併後要刪除多餘儲存格。 | §4 |
| 跨行合併 | rowspan | 一個儲存格跨越多行。 | 垂直方向合併資料。 | 屬性值是合併單元格的個數。 | §4 |
| 跨列合併 | colspan | 一個儲存格跨越多列欄位。 | 水平方向合併資料。 | 屬性值是合併單元格的個數。 | §4 |

## 9. 常用片段

### 9.1 基本表格

| 項目 | 說明 |
|---|---|
| 使用情境 | 顯示一組欄列資料。 |
| 使用前提 | 資料具有列與欄的對應關係。 |
| 來源 | §1 |

```html
<table>
  <tr>
    <th>姓名</th>
    <th>性別</th>
    <th>年齡</th>
  </tr>
  <tr>
    <td>劉德華</td>
    <td>男</td>
    <td>56</td>
  </tr>
</table>
```

### 注意事項

- `<tr>` 定義表格列。
- `<th>` 放表頭內容。
- `<td>` 放一般資料。

### 9.2 完整表格區域

| 項目 | 說明 |
|---|---|
| 使用情境 | 需要表格標題、表頭、主體與腳注的完整表格。 |
| 使用前提 | 表格內容可分成標題、欄位標題、主要資料與總結資料。 |
| 來源 | §2 |

```html
<table border="1">
  <caption>段考成績</caption>
  <thead>
    <tr>
      <th>科目</th>
      <th>分數</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>語文</td>
      <td>99</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>總分</td>
      <td>159</td>
    </tr>
  </tfoot>
</table>
```

### 注意事項

- `<caption>` 是表格標題。
- `<thead>`、`<tbody>`、`<tfoot>` 分別表示表格頭部、主體與腳注。

### 9.3 table 常用屬性示例

| 項目 | 說明 |
|---|---|
| 使用情境 | 初學階段觀察表格邊框、間距與寬度。 |
| 使用前提 | 需要示範原始資料提到的表格屬性。 |
| 來源 | §3 |

```html
<table align="center" border="1" cellpadding="20" cellspacing="0" width="500">
  <tr>
    <td>資料</td>
  </tr>
</table>
```

### 注意事項

- 原始資料提醒這些屬性在實際開發中不常用，後續多半透過 CSS 設定。
- `border` 值控制表格最外邊框寬度，不控制單元格邊框高度。

### 9.4 合併儲存格

| 項目 | 說明 |
|---|---|
| 使用情境 | 特殊情況下把多個儲存格合併成一個。 |
| 使用前提 | 需要跨行或跨列呈現資料。 |
| 來源 | §4 |

```html
<table border="1" cellspacing="0">
  <tr>
    <td></td>
    <td colspan="2"></td>
  </tr>
  <tr>
    <td rowspan="2"></td>
    <td></td>
    <td></td>
  </tr>
  <tr>
    <td></td>
    <td></td>
  </tr>
</table>
```

### 注意事項

- `rowspan` 表示跨行合併。
- `colspan` 表示跨列合併。
- 合併後要刪除多餘的儲存格。

## 10. 錯誤速查表

| 錯誤現象 | 可能原因 | 修正方向 | 對應知識點 | 來源 |
|---|---|---|---|---|
| 用表格做頁面布局。 | 把表格當成排版工具。 | 表格只用於展示數據，布局交給其他方式處理。 | 表格用途 | §1 |
| `<td>` 沒有放在 `<tr>` 中。 | 忽略 `<td>` 必須嵌套在 `<tr>` 中。 | 先建立 `<tr>`，再放入 `<td>`。 | 表格基本結構 | §1 |
| 表頭資料都使用 `<td>`。 | 未區分表頭與資料單元格。 | 表頭單元格使用 `<th>`。 | `<th>` vs `<td>` | §1、§2 |
| 表格頭部中使用一般資料單元格。 | 未依原始資料區分表格頭部與主體。 | 表格頭部中用 `<th>`。 | `<thead>` | §2 |
| 表格主體或腳注中誤用表頭單元格。 | 未依原始資料區分資料區域。 | 表格主體、表格腳注中用 `<td>`。 | `<tbody>`、`<tfoot>` | §2 |
| `border` 設定後單元格高度不如預期。 | 誤以為 `border` 值控制單元格邊框高度。 | 依原始資料說明，該問題之後靠 CSS 控制。 | `border` 屬性 | §3 |
| 合併儲存格後表格欄列錯亂。 | 加了 `rowspan` 或 `colspan` 後沒有刪除多餘儲存格。 | 找到目標單元格設定合併方式後，刪除多餘單元格。 | 合併儲存格 | §4 |
| 跨行與跨列混淆。 | 不清楚 `rowspan` 與 `colspan` 的方向。 | 跨行用 `rowspan`，跨列用 `colspan`。 | 合併儲存格 | §4 |

## 11. 相似概念對照表

### 11.1 `<th>` vs `<td>`

| 比較項目 | `<th>` | `<td>` |
|---|---|---|
| 定義 | 表頭單元格。 | 資料單元格。 |
| 使用時機 | 表格頭部或欄位標題。 | 表格主體、表格腳注中的資料。 |
| 常見寫法 | `<th>姓名</th>` | `<td>劉德華</td>` |
| 注意事項 | 原始資料指出內容預設加粗居中。 | 必須嵌套在 `<tr>` 中。 |
| 常見錯誤 | 所有表頭都改用 `<td>`。 | 在表格頭部中誤用一般資料單元格。 |

### 選用建議

- 欄位名稱或表頭內容選 `<th>`。
- 一般資料內容選 `<td>`。

### 11.2 `<thead>` vs `<tbody>` vs `<tfoot>`

| 比較項目 | `<thead>` | `<tbody>` | `<tfoot>` |
|---|---|---|---|
| 定義 | 表格頭部。 | 表格主體。 | 表格腳注。 |
| 使用時機 | 放欄位標題列。 | 放主要資料列。 | 放腳注或總計資料。 |
| 常見寫法 | `<thead><tr><th>科目</th></tr></thead>` | `<tbody><tr><td>語文</td></tr></tbody>` | `<tfoot><tr><td>總分</td></tr></tfoot>` |
| 注意事項 | 表格頭部中用 `<th>`。 | 表格主體中用 `<td>`。 | 表格腳注中用 `<td>`。 |
| 常見錯誤 | 將主要資料放進表格頭部。 | 不區分主要資料與標題。 | 彙總資料與主體資料混在一起。 |

### 選用建議

- 欄位標題放 `<thead>`。
- 主要資料放 `<tbody>`。
- 總分、總計或腳注放 `<tfoot>`。

### 11.3 `rowspan` vs `colspan`

| 比較項目 | `rowspan` | `colspan` |
|---|---|---|
| 定義 | 跨行合併。 | 跨列合併。 |
| 使用時機 | 儲存格需要垂直合併多行。 | 儲存格需要水平合併多列欄位。 |
| 常見寫法 | `<td rowspan="2"></td>` | `<td colspan="2"></td>` |
| 注意事項 | 屬性值是合併單元格的個數。 | 屬性值是合併單元格的個數。 |
| 常見錯誤 | 沒有刪除被合併的多餘儲存格。 | 沒有刪除被合併的多餘儲存格。 |

### 選用建議

- 垂直方向跨多行，用 `rowspan`。
- 水平方向跨多列欄位，用 `colspan`。
- 設定合併屬性後，檢查並刪除多餘儲存格。

### 11.4 表格結構 vs 表格樣式

| 比較項目 | 表格結構 | 表格樣式 |
|---|---|---|
| 定義 | 使用 `<table>`、`<tr>`、`<th>`、`<td>` 等描述資料。 | 使用屬性或 CSS 控制邊框、間距、寬高等外觀。 |
| 使用時機 | 需要表達欄列資料關係。 | 需要調整視覺呈現。 |
| 常見寫法 | `<table><tr><td>資料</td></tr></table>` | `border="1"`、`cellpadding="20"`、`width="500"` |
| 注意事項 | 表格不是用來布局的。 | 原始資料提醒表格屬性實務不常用，後續通常交給 CSS。 |
| 常見錯誤 | 用表格排版。 | 誤以為 `border` 控制所有儲存格高度。 |

### 選用建議

- 先確認內容是不是表格資料。
- HTML 寫出表格結構，視覺效果後續再交給 CSS。
