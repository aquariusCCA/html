## 3. 名詞表

| 名詞 | 英文 | 簡要定義 | 用途 | 注意事項 | 來源 |
|---|---|---|---|---|---|
| 表格 | table | 用行與列呈現資料關係的 HTML 結構。 | 展示成績表、報表、商品規格、訂單明細、統計數據等。 | 不應用來做整頁版面布局。 | §2、§3、§9 |
| 表格資料 | tabular data | 具有欄列對應關係、可比較或可檢索的資料。 | 一筆資料有多個欄位，或多筆資料需要橫向、縱向比較時。 | 若內容不是資料表，通常不應使用 `<table>`。 | §2、§3.1 |
| 表格列 | table row / `<tr>` | 表格中的一列。 | 包住同一列中的表頭儲存格或資料儲存格。 | `<td>` 與 `<th>` 應放在 `<tr>` 內。 | §3、§4.1 |
| 表頭儲存格 | table header / `<th>` | 表示欄位名稱或列標題的儲存格。 | 放欄位標題、列標題或彙總列的標題。 | 不應把所有儲存格都寫成 `<td>` 而忽略標題語意。 | §3、§4.1、§5、§7 |
| 資料儲存格 | table data / `<td>` | 放置一般資料內容的儲存格。 | 放每筆資料的欄位值。 | 應放在 `<tr>` 中。 | §3、§4.1 |
| 表格標題 | caption / `<caption>` | 說明整張表格主題的元素。 | 讓讀者知道表格內容範圍或目的。 | 表格沒有標題或上下文時，使用者不容易理解用途。 | §3.3、§4.2、§7 |
| 表格區域分組 | table section grouping | 使用 `<thead>`、`<tbody>`、`<tfoot>` 區分表格區域。 | 讓欄位標題、主要資料、彙總資訊分開。 | 有助於結構清晰、CSS 選取、JavaScript 操作與輔助工具理解。 | §3.3、§4.2 |
| 儲存格合併 | cell spanning | 讓單一儲存格跨越多列或多欄。 | 課表、彙總表或需要合併欄列的表格。 | 合併後要刪除被合併位置上的多餘儲存格。 | §4.5、§5、§7 |
| 舊式表格屬性 | legacy table attributes | 早期用於控制表格外觀的 HTML 屬性。 | 初學展示表格邊框、間距、寬度與對齊。 | 正式專案多數樣式應交給 CSS。 | §4.3、§7、§9 |
| 可訪問性 | accessibility | 讓輔助工具能理解表格主題、表頭與資料關係。 | 透過 `<caption>`、`<th>` 與清楚結構提升理解。 | 複雜表格需要更仔細設計；本章先聚焦基礎做法。 | §7、§8、§11 |

## 4. HTML 元素表

| 元素 | 用途 | 常用屬性 | 使用情境 | 最小範例 | 注意事項 | 來源 |
|---|---|---|---|---|---|---|
| `<table>` | 定義整張表格。 | `align`、`border`、`cellpadding`、`cellspacing`、`width` | 需要呈現欄列資料關係時。 | `<table><tr><td>資料</td></tr></table>` | 不應用來做頁面布局；樣式建議交給 CSS。 | §2、§3、§4.1、§4.3、§7 |
| `<tr>` | 定義表格中的一列。 | 原筆記未列專屬屬性。 | 每一筆資料列或表頭列。 | `<tr><td>資料</td></tr>` | 應放在 `<table>`、`<thead>`、`<tbody>` 或 `<tfoot>` 內。 | §3、§4.1 |
| `<th>` | 定義表頭儲存格。 | `rowspan`、`colspan` | 欄位名稱、列標題或彙總列標題。 | `<th>姓名</th>` | 瀏覽器預設通常會加粗並置中；語意不同於 `<td>`。 | §3、§4.1、§4.5、§5、§7 |
| `<td>` | 定義一般資料儲存格。 | `rowspan`、`colspan` | 放實際資料值。 | `<td>劉德華</td>` | 應放在 `<tr>` 內；合併後需移除多餘儲存格。 | §3、§4.1、§4.5、§7 |
| `<caption>` | 定義表格標題。 | 原筆記未列專屬屬性。 | 說明整張表格的主題。 | `<caption>段考成績</caption>` | 可避免使用者不清楚表格用途。 | §3.3、§4.2、§7 |
| `<thead>` | 定義表格頭部。 | 原筆記未列專屬屬性。 | 放欄位標題列。 | `<thead><tr><th>科目</th></tr></thead>` | 讓表格結構比所有 `<tr>` 都直接放在 `<table>` 下更清楚。 | §3.3、§4.2 |
| `<tbody>` | 定義表格主體。 | 原筆記未列專屬屬性。 | 放主要資料列。 | `<tbody><tr><td>語文</td></tr></tbody>` | 適合承載主要資料內容。 | §3.3、§4.2 |
| `<tfoot>` | 定義表格腳注。 | 原筆記未列專屬屬性。 | 放總計、備註或彙總資料。 | `<tfoot><tr><th>平均</th><td>87.5</td></tr></tfoot>` | 彙總列中的標題可使用 `<th>` 表達語意。 | §3.3、§4.2、§5 |

> 注意：`<main>`、`<nav>`、`<section>` 等元素只在「不用表格做版面」的正確範例中出現，未擴寫成完整 HTML 元素表條目。

## 5. 屬性表

| 屬性 | 適用元素 | 常見值 | 用途 | 範例 | 注意事項 | 來源 |
|---|---|---|---|---|---|---|
| `align` | `<table>` | `center` | 設定表格相對周圍元素的對齊方式。 | `<table align="center">` | 現代開發通常改用 CSS。 | §4.3 |
| `border` | `<table>` | 數字，例如 `1` | 設定表格邊框寬度。 | `<table border="1">` | 可用於初學展示；正式樣式建議用 CSS。 | §4.3、§7 |
| `cellpadding` | `<table>` | 數字，例如 `20` | 設定儲存格內容與邊框之間的距離。 | `<table cellpadding="20">` | 現代開發通常用 CSS `padding`。 | §4.3 |
| `cellspacing` | `<table>` | 數字，例如 `0` | 設定儲存格之間的距離。 | `<table cellspacing="0">` | 現代開發通常用 CSS `border-spacing` 或 `border-collapse`。 | §4.3 |
| `width` | `<table>` | 數字，例如 `500` | 設定表格寬度。 | `<table width="500">` | 現代開發通常用 CSS `width`。 | §4.3 |
| `rowspan` | `<th>`、`<td>` | 正整數，例如 `2` | 讓一個儲存格跨越多列。 | `<td rowspan="2">HTML 基礎</td>` | 使用後要刪除被合併列中的多餘儲存格。 | §4.5、§5、§7 |
| `colspan` | `<th>`、`<td>` | 正整數，例如 `2` | 讓一個儲存格跨越多欄。 | `<td colspan="2">跨兩欄</td>` | 使用後要維持欄列數一致，避免表格錯亂。 | §4.5、§7 |

## 9. 常用片段

### 9.1 基本表格結構

| 項目 | 說明 |
|---|---|
| 使用情境 | 建立最基本的欄列資料。 |
| 使用前提 | 資料具有表頭與資料列。 |
| 來源 | §4.1 |

```html
<table>
  <tr>
    <th>姓名</th>
    <th>年齡</th>
  </tr>
  <tr>
    <td>劉德華</td>
    <td>56</td>
  </tr>
</table>
```

### 注意事項

- `<tr>` 表示一列。
- `<th>` 放欄位標題。
- `<td>` 放一般資料。

### 9.2 含標題與區域分組的表格

| 項目 | 說明 |
|---|---|
| 使用情境 | 表格需要清楚區分主題、欄位、資料與彙總資訊。 |
| 使用前提 | 表格有表格標題、表頭列、主要資料與總計或備註。 |
| 來源 | §4.2、§5 |

```html
<table>
  <caption>期末成績</caption>
  <thead>
    <tr>
      <th>科目</th>
      <th>分數</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>國文</td>
      <td>90</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>平均</th>
      <td>90</td>
    </tr>
  </tfoot>
</table>
```

### 注意事項

- `<caption>` 說明整張表格主題。
- `<thead>` 放欄位標題列。
- `<tbody>` 放主要資料列。
- `<tfoot>` 放總計、備註或彙總資料。

### 9.3 使用 CSS 控制表格樣式

| 項目 | 說明 |
|---|---|
| 使用情境 | 正式專案中需要控制表格寬度、邊框、間距與對齊。 |
| 使用前提 | HTML 已保留正確表格語意。 |
| 來源 | §4.4、§7、§9 |

```html
<table class="score-table">
  <caption>段考成績</caption>
  <thead>
    <tr>
      <th>科目</th>
      <th>分數</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>語文</td>
      <td>99</td>
    </tr>
  </tbody>
</table>
```

```css
.score-table {
  width: 100%;
  border-collapse: collapse;
}

.score-table th,
.score-table td {
  border: 1px solid #999;
  padding: 8px 12px;
  text-align: left;
}
```

### 注意事項

- HTML 負責資料結構。
- CSS 負責寬度、邊框、間距與對齊。
- 舊式表格屬性不適合作為正式專案的主要樣式管理方式。

### 9.4 使用 `rowspan` 合併列

| 項目 | 說明 |
|---|---|
| 使用情境 | 某個資料值跨越多個時間段或多列。 |
| 使用前提 | 找到要保留的儲存格，並移除被合併位置上的多餘儲存格。 |
| 來源 | §4.5、§5 |

```html
<table>
  <tr>
    <th>時間</th>
    <th>課程</th>
  </tr>
  <tr>
    <td>09:00 - 10:00</td>
    <td rowspan="2">HTML 基礎</td>
  </tr>
  <tr>
    <td>10:00 - 11:00</td>
  </tr>
</table>
```

### 注意事項

- `rowspan="2"` 表示跨兩列。
- 第二列不再重複寫已被合併的課程儲存格。

### 9.5 使用 `colspan` 合併欄

| 項目 | 說明 |
|---|---|
| 使用情境 | 某個儲存格需要跨越多個欄位。 |
| 使用前提 | 合併後仍需維持表格欄列結構一致。 |
| 來源 | §4.5 |

```html
<table>
  <tr>
    <td></td>
    <td colspan="2">跨兩欄</td>
  </tr>
  <tr>
    <td>項目</td>
    <td>資料 A</td>
    <td>資料 B</td>
  </tr>
</table>
```

### 注意事項

- `colspan="2"` 表示跨兩欄。
- 合併後不要留下會造成欄列數錯亂的多餘儲存格。

## 10. 錯誤速查表

| 錯誤現象 | 可能原因 | 修正方向 | 對應知識點 | 來源 |
|---|---|---|---|---|
| 用 `<table>` 做整頁排版。 | 表格語意是資料表，不是版面布局。 | 使用 CSS Flexbox、Grid 或一般區塊排版。 | 表格不是排版工具 | §3.2、§7 |
| 直接把 `<td>` 放在 `<table>` 裡。 | `<td>` 應該放在 `<tr>` 中。 | 先建立 `<tr>`，再放入 `<td>` 或 `<th>`。 | 表格基本結構 | §4.1、§7 |
| 所有儲存格都用 `<td>`。 | 欄位標題缺少表頭語意。 | 欄位名稱或列標題使用 `<th>`。 | `<th>` 與 `<td>` 語意 | §3、§4.1、§7 |
| 合併儲存格後表格欄列錯亂。 | 使用 `rowspan` 或 `colspan` 後沒有刪除多餘儲存格。 | 移除被合併位置上的多餘 `<td>`。 | 儲存格合併 | §4.5、§7 |
| 樣式與結構混在一起難維護。 | 使用舊屬性控制所有樣式。 | 用 HTML 寫結構，用 CSS 控制外觀。 | 舊式表格屬性與 CSS | §4.3、§4.4、§7 |
| 使用者不清楚表格用途。 | 表格沒有標題或上下文。 | 使用 `<caption>` 或在表格前提供清楚標題。 | 表格標題 | §3.3、§4.2、§7 |
| 誤以為 `border` 能控制所有儲存格高度。 | `border` 主要控制邊框，不負責資料列高度設計。 | 使用 CSS 的 `padding`、`line-height`、`height` 等控制視覺效果。 | 表格樣式控制 | §4.3、§4.4、§7 |

## 11. 相似概念對照表

### 11.1 表格資料 vs 版面布局

| 比較項目 | 表格資料 | 版面布局 |
|---|---|---|
| 定義 | 具有行與列關係、可比較或檢索的資料。 | 頁面區塊的視覺排列。 |
| 使用時機 | 成績表、報表、商品規格、訂單明細、統計數據。 | 側邊欄、主內容、導覽與頁面區塊排列。 |
| 常見寫法 | 使用 `<table>`、`<tr>`、`<th>`、`<td>`。 | 使用語意元素搭配 CSS Flexbox、Grid 或一般區塊排版。 |
| 注意事項 | 表格能表達資料的欄列對應關係。 | 不應只為了對齊畫面而使用 `<table>`。 |
| 常見錯誤 | 把非表格內容硬寫成表格。 | 用 `<table>` 取代 CSS 排版能力。 |

### 選用建議

- 內容是表格資料時使用 `<table>`。
- 內容只是頁面區塊排列時，使用適合的 HTML 結構並交給 CSS 排版。

### 11.2 `<th>` vs `<td>`

| 比較項目 | `<th>` | `<td>` |
|---|---|---|
| 定義 | 表頭儲存格。 | 一般資料儲存格。 |
| 使用時機 | 欄位名稱、列標題或彙總列標題。 | 實際資料內容。 |
| 常見寫法 | `<th>姓名</th>` | `<td>劉德華</td>` |
| 注意事項 | 瀏覽器預設通常會加粗並置中。 | 不具表頭語意。 |
| 常見錯誤 | 需要表頭語意時仍全部使用 `<td>`。 | 被誤用來表示欄位標題。 |

### 選用建議

- 欄位名稱或列標題使用 `<th>`。
- 一般資料值使用 `<td>`。

### 11.3 `<thead>` vs `<tbody>` vs `<tfoot>`

| 比較項目 | `<thead>` | `<tbody>` | `<tfoot>` |
|---|---|---|---|
| 定義 | 表格頭部。 | 表格主體。 | 表格腳注。 |
| 使用時機 | 放欄位標題列。 | 放主要資料列。 | 放總計、備註或彙總資料。 |
| 常見寫法 | `<thead><tr><th>科目</th></tr></thead>` | `<tbody><tr><td>語文</td></tr></tbody>` | `<tfoot><tr><td>總分</td></tr></tfoot>` |
| 注意事項 | 讓欄位標題與資料分開。 | 承載主要內容。 | 適合放彙總資訊。 |
| 常見錯誤 | 所有列都直接堆在 `<table>` 下而缺少結構分組。 | 原筆記未列其他常見錯誤。 | 原筆記未列其他常見錯誤。 |

### 選用建議

- 簡單表格可用基本結構。
- 較完整的表格建議使用 `<caption>`、`<thead>`、`<tbody>`、`<tfoot>` 組織內容。

### 11.4 `rowspan` vs `colspan`

| 比較項目 | `rowspan` | `colspan` |
|---|---|---|
| 定義 | 讓一個儲存格跨越多列。 | 讓一個儲存格跨越多欄。 |
| 使用時機 | 同一資料要覆蓋多個垂直列。 | 同一資料要覆蓋多個水平欄。 |
| 常見寫法 | `<td rowspan="2">跨兩列</td>` | `<td colspan="2">跨兩欄</td>` |
| 注意事項 | 刪除被合併列上的多餘儲存格。 | 刪除被合併欄位上的多餘儲存格。 |
| 常見錯誤 | 合併後仍保留多餘 `<td>`，造成欄列錯亂。 | 合併後仍保留多餘 `<td>`，造成欄列錯亂。 |

### 選用建議

- 跨垂直方向的列時使用 `rowspan`。
- 跨水平方向的欄時使用 `colspan`。

### 11.5 舊式表格屬性 vs CSS 樣式

| 比較項目 | 舊式表格屬性 | CSS 樣式 |
|---|---|---|
| 定義 | 直接寫在 HTML 表格元素上的外觀屬性。 | 透過 CSS 控制表格視覺呈現。 |
| 使用時機 | 初學展示表格外觀效果。 | 正式專案控制寬度、邊框、間距與對齊。 |
| 常見寫法 | `<table border="1" cellspacing="0">` | `.score-table { border-collapse: collapse; }` |
| 注意事項 | 容易讓 HTML 結構與樣式混在一起。 | 讓 HTML 專注資料結構，CSS 專注外觀。 |
| 常見錯誤 | 使用舊屬性控制所有樣式。 | 原筆記未列 CSS 表格樣式本身的錯誤。 |

### 選用建議

- 初學理解效果時可辨識舊屬性。
- 實務維護以 CSS 控制表格樣式。
