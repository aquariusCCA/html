# form 提交結構速查

> 查表內容整理自 `notes/240-表單標籤/01-表單的用途與提交結構.md`。

## 表單組成

| 組成 | 作用 | 例子 |
| --- | --- | --- |
| 表單容器 | 包住一組要提交的欄位 | `<form>` |
| 表單控件 | 讓使用者輸入或選擇資料 | `<input>`、`<select>`、`<textarea>` |
| 提示資訊 | 告訴使用者欄位要填什麼 | 文字標籤、說明文字、`placeholder` |
| 提交按鈕 | 觸發表單送出 | `<button type="submit">` |

## form 常用提交屬性

| 屬性 | 作用 | 判斷重點 |
| --- | --- | --- |
| `action` | 指定資料要送到哪個位址 | 決定送到哪裡 |
| `method` | 指定資料用哪種方式送出 | 常見為 `get` 或 `post`，決定怎麼送 |
| `name` | 表單本身的名稱 | 方便識別表單 |
| `target` | 指定提交結果如何開啟 | 依頁面需求決定 |

## 欄位提交條件

| 條件 | 說明 |
| --- | --- |
| 控件通常需要 `name` | 後端才知道這筆資料的欄位名稱 |
| 控件的值來自使用者輸入、選擇或元素本身設定的 `value` | 例如 `account=amy` 中，`account` 來自 `name`，`amy` 來自使用者輸入 |
| 被 `disabled` 禁用的控件通常不會提交 | 需要提交的資料不應只靠 `disabled` 設計 |

## 實務判斷準則

| 判斷問題 | 建議 |
| --- | --- |
| 這組欄位是否需要一起提交？ | 使用 `<form>` 包住相關控件 |
| 資料要送到哪裡？ | 設定清楚的 `action` |
| 資料要怎麼送？ | 設定合適的 `method` |
| 哪些欄位要被後端接收？ | 每個要提交的欄位都應有清楚的 `name` |
| 欄位被禁用但資料仍需要送出？ | 重新判斷資料來源與後端處理方式 |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 只放 `<input>`，沒有 `<form>` | 表單結構不完整，提交關係不清楚 | 用 `<form>` 包住要提交的一組欄位 |
| 忘記替欄位加上 `name` | 使用者可以輸入，但提交時缺少欄位名稱 | 為要提交的控件設定 `name` |
| 誤以為 `disabled` 欄位也會送出 | 被禁用的欄位通常不會包含在提交資料中 | 需要提交的資料應重新設計來源與後端處理 |
