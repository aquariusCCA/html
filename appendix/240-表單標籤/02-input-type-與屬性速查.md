# input type 與屬性速查

> 查表內容整理自 `notes/240-表單標籤/02-input基礎與文字密碼輸入.md` 與 `notes/240-表單標籤/05-HTML5輸入類型與表單屬性.md`。

## input 基本結構

| 部分 | 寫法 | 作用 |
| --- | --- | --- |
| 控件 | `<input>` | 建立輸入控件 |
| 輸入型態 | `type="text"` | 決定瀏覽器產生的控件與行為 |
| 欄位名稱 | `name="username"` | 提交時的欄位名稱 |
| 單標籤 | `<input ...>` | 不需要寫成 `<input></input>` |

## 常見 input type

| type | 用途 | 備註 |
| --- | --- | --- |
| `text` | 單行文字輸入 | 適合帳號、搜尋關鍵字、驗證碼等短文字 |
| `password` | 密碼輸入 | 會遮蔽畫面顯示，但不代表資料已加密 |
| `radio` | 單選 | 詳細選項規則見選項控件速查 |
| `checkbox` | 複選 | 詳細選項規則見選項控件速查 |
| `file` | 本機檔案選擇 | 實際接收、儲存與驗證仍需要後端 |
| `email` | 電子郵件 | 可讓瀏覽器用電子郵件格式檢查欄位 |
| `url` | 網址 | 讓欄位預期網址格式 |
| `date` | 日期 | 可能顯示日期選擇器，依瀏覽器而定 |
| `time` | 時間 | 不同瀏覽器呈現可能不同 |
| `number` | 數字 | 適合數字資料 |
| `tel` | 電話 | 主要提供輸入輔助，不一定自動驗證手機格式 |
| `search` | 搜尋文字 | 適合搜尋欄位 |
| `color` | 顏色選擇 | 不同瀏覽器呈現可能不同 |

## 常用欄位屬性

| 屬性 | 作用 | 實務提醒 |
| --- | --- | --- |
| `name` | 提交資料時的欄位名稱 | 要提交的欄位一定要有 `name` |
| `value` | 欄位的值 | 在文字框中可作為初始值，在按鈕或選項中常是提交值或顯示值 |
| `placeholder` | 輸入前顯示的提示文字 | 是提示，不是欄位真正的值，也不能取代 label |
| `maxlength` | 限制最多可輸入的字元數 | 不能取代完整驗證 |
| `checked` | 讓 radio 或 checkbox 預設被選中 | 用於選項控件 |
| `multiple` | 允許一次選擇多個檔案 | 可搭配 `type="file"` |
| `required` | 欄位必填 | 未填時瀏覽器會阻止提交並提示，但後端仍要驗證 |
| `autofocus` | 頁面載入後自動聚焦 | 過度使用會干擾使用者 |
| `autocomplete` | 控制瀏覽器是否提供自動完成建議 | 應依資料性質決定開啟或關閉 |

## value 與 placeholder

| 項目 | 意義 | 提醒 |
| --- | --- | --- |
| `value` | 欄位真正的值 | 使用者直接提交時可能被送出 |
| `placeholder` | 欄位空白時顯示的提示 | 使用者未輸入時不會當成欄位值 |
| 提示文字 | 應使用 `placeholder` | 不要用 `value` 假裝提示 |

## 實務判斷準則

| 情境 | 建議 |
| --- | --- |
| 短文字輸入 | 使用 `type="text"` |
| 密碼輸入 | 使用 `type="password"`，但安全處理仍需要 HTTPS、後端驗證與安全儲存 |
| 檔案選擇 | 使用 `type="file"`，多檔可加 `multiple` |
| 能描述資料格式 | 選擇比 `text` 更精確的 input type |
| 必填欄位 | 可使用 `required`，但後端仍要驗證 |
| 常見個人資料 | 可依需求開啟 `autocomplete` |
| 敏感或一次性資料 | 可依需求關閉 `autocomplete` |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 欄位沒有 `name` | 提交時缺少欄位名稱 | 加上清楚的 `name` |
| 密碼框預填固定密碼 | 使用體驗差，也容易造成安全風險 | 密碼欄位通常不應預填固定值 |
| 把 `maxlength` 當作完整驗證 | 只能限制字元數，不能保證格式正確 | 帳號規則、密碼強度等仍需完整驗證流程 |
| 以為 `tel` 會自動驗證手機格式 | 各地電話格式差異大，瀏覽器不一定驗證 | 把 `tel` 視為輸入輔助 |
| 把 `placeholder` 當成 label | 輸入後提示會消失，不能作為欄位名稱 | 搭配明確的 label |
| 只依賴前端驗證 | 使用者仍可能繞過前端限制 | 重要資料一定要在後端再次驗證 |
| 亂用 `autofocus` | 可能搶走頁面焦點 | 只在真正需要時使用 |
