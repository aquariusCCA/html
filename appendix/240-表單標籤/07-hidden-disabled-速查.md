# hidden disabled 速查

> 查表內容整理自 `notes/240-表單標籤/08-hidden與disabled表單狀態.md`。

## 一句話對照

| 狀態 | 核心觀念 |
| --- | --- |
| `hidden` | 看不見但會帶資料 |
| `disabled` | 不能操作且通常不會提交 |

## hidden 與 disabled 對照

| 特性 | hidden | disabled |
| --- | --- | --- |
| 使用者是否看得到 | 看不到 | 看得到但不可操作 |
| 是否常用來帶固定資料 | 是 | 否 |
| 是否通常會提交 | 會 | 不會 |
| 是否能當安全保護 | 不能 | 不能 |

## 使用情境

| 需求 | 可用方式 | 提醒 |
| --- | --- | --- |
| 提交固定資料且不需要使用者看到 | `type="hidden"` | 資料仍在前端，不能當可信來源 |
| 讓欄位暫時不可操作 | `disabled` | 通常不會提交 |
| 禁用一整組控件 | `fieldset disabled` | 適合整區暫時不可編輯 |
| 資料需要提交但不想讓使用者修改 | 重新設計資料來源與後端驗證 | 不能只靠 `disabled` |

## disabled 可用控件

| 控件 | 說明 |
| --- | --- |
| `input` | 可被禁用 |
| `textarea` | 可被禁用 |
| `button` | 可被禁用 |
| `select` | 可被禁用 |
| `option` | 可被禁用 |
| radio | 可被禁用 |
| checkbox | 可被禁用 |

## 安全提醒

| 狀態 | 不能做什麼 | 原因 |
| --- | --- | --- |
| hidden | 不能保存敏感或可信資料 | 使用者仍可能修改 hidden 值再送出 |
| disabled | 不能取代後端權限檢查 | 前端狀態可能被改動 |
| hidden 與 disabled | 都不能當安全邊界 | 重要資料與權限必須由後端驗證 |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 把 hidden 當成不能被修改 | 使用者仍可能修改 hidden 值再送出 | 價格、權限、使用者身分等重要資料應由後端查詢與驗證 |
| 期待 disabled 欄位被提交 | 被禁用欄位通常不會送出 | 需要提交時，重新設計資料來源，或使用 hidden 搭配後端驗證等方式 |
| 用 disabled 當權限控制 | disabled 只是前端狀態 | 權限必須由後端檢查 |
