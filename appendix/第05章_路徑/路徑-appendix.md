## 3. 名詞表

| 名詞 | 英文 | 簡要定義 | 用途 | 注意事項 | 來源 |
|---|---|---|---|---|---|
| 路徑 | path | 在 HTML 中用來表示檔案位置的資訊。 | 讓瀏覽器找到圖片、頁面、CSS、JavaScript 等資源。 | 路徑是否有效取決於檔案實際位置與引用基準。 | 3.1 什麼是路徑 |
| 絕對路徑 | absolute path | 從固定起點直接描述完整位置。 | 指向本機磁碟位置或完整網址資源。 | 本機絕對路徑不適合正式網頁；網路絕對路徑依賴外部伺服器。 | 3.2 路徑的兩大類型、4.1 絕對路徑 |
| 本機絕對路徑 | local absolute path | 從電腦磁碟位置開始描述檔案位置。 | 原筆記用來說明不建議的路徑寫法。 | 換設備、部署或交給他人開啟時容易失效。 | 4.1 絕對路徑、7. 常見錯誤與修正 |
| 網路絕對路徑 | URL absolute path | 從完整網址開始描述資源位置。 | 直接引用網路上的資源。 | 可能遇到檔案移除、網址改變、伺服器無法連線或防盜鏈。 | 4.1 絕對路徑 |
| 相對路徑 | relative path | 從目前引用資源的檔案位置出發描述目標檔案位置。 | 在同一專案資料夾中引用圖片、CSS、JavaScript 或頁面。 | HTML 檔案位置改變時，相對路徑可能需要同步調整。 | 3.2 路徑的兩大類型、4.2 相對路徑 |
| 目前資料夾 | current folder | 相對路徑中的目前所在資料夾。 | 使用 `./` 表示從目前資料夾開始尋找。 | 原筆記指出 `./` 通常可以省略。 | 4.3 同層、下一層與上一層 |
| 上一層資料夾 | parent folder | 目前資料夾的上一層。 | 使用 `../` 先回到上一層，再尋找目標檔案。 | 上兩層可連續使用 `../../`。 | 4.3 同層、下一層與上一層、5. 範例說明 |
| 專案資料夾結構 | project folder structure | 專案中 HTML、圖片、CSS、JavaScript 等檔案的擺放層級。 | 協助維持資源引用清楚、方便協作與部署。 | 路徑能否正確運作，取決於檔案位置與資料夾層級。 | 6.1 管理網站資源、8. 與其他概念的關係 |
| 防盜鏈 | hotlink protection | 外部網站阻止其他網站直接載入其圖片或資源的機制。 | 說明直接引用別人網站圖片可能失敗的原因之一。 | 不要未經允許直接引用別人網站圖片。 | 4.1 絕對路徑、7. 常見錯誤與修正 |
| URL | URL | 網路絕對路徑本質上使用的完整網址。 | 指向網路上的資源位置。 | 原筆記只在路徑關係中提到 URL，未展開 URL 結構。 | 8. 與其他概念的關係 |

## 4. 路徑寫法速查表

| 寫法 | 類型或位置關係 | 意義 | 最小範例 | 注意事項 | 來源 |
|---|---|---|---|---|---|
| `logo.png` | 同一層 | 目標檔案與目前 HTML 檔案在同一個資料夾。 | `<img src="logo.png" alt="網站標誌">` | 檔名需與實際檔案一致。 | 4.3 同層、下一層與上一層 |
| `images/logo.png` | 下一層 | 進入目前資料夾底下的 `images/`，再找 `logo.png`。 | `<img src="images/logo.png" alt="網站標誌">` | 適合 HTML 與 `images/` 位在同一層時。 | 4.2 相對路徑、4.3 同層、下一層與上一層 |
| `./images/logo.png` | 目前資料夾再進入下一層 | 從目前資料夾開始，進入 `images/` 找檔案。 | `<img src="./images/logo.png" alt="網站標誌">` | `./` 通常可以省略成 `images/logo.png`。 | 4.3 同層、下一層與上一層 |
| `../images/logo.png` | 上一層再進入下一層 | 先回到上一層，再進入 `images/` 找檔案。 | `<img src="../images/logo.png" alt="網站標誌">` | 常用於子資料夾頁面引用上層資源。 | 4.3 同層、下一層與上一層、5. 範例說明 |
| `../../logo.png` | 上兩層 | 連續回到上兩層資料夾，再找檔案。 | `<img src="../../logo.png" alt="網站標誌">` | 每一個 `../` 代表往上一層。 | 4.3 同層、下一層與上一層 |
| `https://example.com/images/logo.png` | 網路絕對路徑 | 使用完整網址指向網路資源。 | `<img src="https://example.com/images/logo.png" alt="Example 網站標誌">` | 依賴外部伺服器，且需注意授權與防盜鏈。 | 4.1 絕對路徑 |
| `D:\day01\images\1.jpg` | 本機絕對路徑 | 從特定電腦磁碟位置指向檔案。 | `<img src="D:\day01\images\1.jpg" alt="範例圖片">` | 不建議寫進正式網頁。 | 4.1 絕對路徑、7. 常見錯誤與修正 |

## 6. 常用片段

### 6.1 引用下一層圖片

| 項目 | 說明 |
|---|---|
| 使用情境 | HTML 檔案與 `images/` 資料夾位在同一層。 |
| 使用前提 | `images/logo.png` 實際存在於目前 HTML 檔案所在資料夾底下。 |
| 來源 | 3.1 什麼是路徑、4.2 相對路徑 |

```html
<img src="images/logo.png" alt="網站標誌">
```

### 6.2 子資料夾頁面引用上一層圖片

| 項目 | 說明 |
|---|---|
| 使用情境 | 目前 HTML 位於子資料夾，需要引用上一層底下的 `images/`。 |
| 使用前提 | 目前檔案位置類似 `pages/profile.html`，圖片位於 `images/avatar.png`。 |
| 來源 | 4.3 同層、下一層與上一層、5. 範例說明 |

```html
<img src="../images/avatar.png" alt="使用者頭像">
```

### 6.3 引用 CSS 與 JavaScript

| 項目 | 說明 |
|---|---|
| 使用情境 | `index.html` 需要載入同層資料夾中的 CSS 與 JavaScript。 |
| 使用前提 | `css/style.css` 與 `js/main.js` 實際存在。 |
| 來源 | 5. 範例說明 |

```html
<link rel="stylesheet" href="css/style.css">
<script src="js/main.js"></script>
```

### 6.4 多頁網站連結

| 項目 | 說明 |
|---|---|
| 使用情境 | 頁面之間使用相對路徑互相連結。 |
| 使用前提 | `pages/about.html` 或上一層的 `index.html` 實際存在。 |
| 來源 | 6.2 多頁網站連結 |

```html
<a href="pages/about.html">關於我們</a>
<a href="../index.html">回首頁</a>
```

## 7. 錯誤速查表

| 錯誤現象 | 可能原因 | 修正方向 | 對應知識點 | 來源 |
|---|---|---|---|---|
| 使用 `D:\...` 後，換設備或部署後資源失效。 | 本機絕對路徑只存在於特定電腦。 | 把資源放進專案資料夾，改用相對路徑。 | 本機絕對路徑、相對路徑 | 7. 常見錯誤與修正 |
| 使用 `./`、`../` 後仍找不到檔案。 | 混淆目前資料夾與上一層資料夾。 | 先畫出資料夾結構，再從目前 HTML 檔案位置開始找。 | 目前資料夾、上一層資料夾 | 7. 常見錯誤與修正 |
| HTML 檔案搬到其他資料夾後，圖片或資源失效。 | 相對路徑依賴目前檔案位置。 | 搬移檔案後同步檢查 `src`、`href` 等路徑。 | 相對路徑 | 7. 常見錯誤與修正 |
| 本機可顯示，上線後找不到檔案。 | 檔名大小寫與實際檔案不一致，部分伺服器會區分大小寫。 | 檔名與路徑大小寫保持完全一致。 | 檔名與路徑一致性 | 7. 常見錯誤與修正 |
| 直接引用別人網站圖片後無法顯示。 | 外部資源可能被移除、禁止引用，或涉及授權問題。 | 使用自己有權使用的圖片，或合法授權的 CDN / 圖片來源。 | 網路絕對路徑、防盜鏈 | 7. 常見錯誤與修正 |
| 路徑中有空白或特殊字元導致引用錯誤。 | 網址中可能需要編碼，也容易造成引用錯誤。 | 檔名建議使用英文小寫、數字與連字號。 | 檔名命名 | 7. 常見錯誤與修正 |

## 8. 相似概念對照表

### 8.1 絕對路徑 vs 相對路徑

| 比較項目 | 絕對路徑 | 相對路徑 |
|---|---|---|
| 定義 | 從固定起點直接描述完整位置。 | 從目前引用資源的檔案位置出發描述位置。 |
| 常見寫法 | `https://example.com/images/logo.png`、`D:\day01\images\1.jpg` | `images/logo.png`、`../images/logo.png` |
| 使用情境 | 指向完整網址資源，或用來說明本機固定位置。 | 專案內引用圖片、CSS、JavaScript 或頁面。 |
| 維護特性 | 網路絕對路徑依賴外部資源；本機絕對路徑不適合正式網頁。 | 只要專案資料夾結構一致，較適合搬移與協作。 |
| 常見錯誤 | 把本機磁碟路徑寫進正式網頁。 | 搬移 HTML 後沒有同步調整路徑。 |

### 選用建議

- 專案內資源優先使用相對路徑。
- 使用網路絕對路徑前，先確認授權、穩定性與是否可能被防盜鏈阻擋。

### 8.2 `./` vs `../`

| 比較項目 | `./` | `../` |
|---|---|---|
| 定義 | 目前資料夾。 | 上一層資料夾。 |
| 常見寫法 | `./images/logo.png` | `../images/logo.png` |
| 使用時機 | 從目前資料夾開始尋找目標檔案。 | 先回到上一層，再尋找目標檔案。 |
| 注意事項 | 原筆記指出通常可以省略。 | 可連續使用，例如 `../../logo.png` 表示上兩層。 |
| 常見錯誤 | 以為 `./` 會回到上一層。 | 沒有先從目前 HTML 檔案的位置開始計算。 |

### 選用建議

- 目標在目前資料夾底下時，可使用 `images/logo.png` 或 `./images/logo.png`。
- 目標在上一層或上一層的其他資料夾時，使用 `../`。

### 8.3 `src` vs `href`

| 比較項目 | `src` | `href` |
|---|---|---|
| 本章出現元素 | `<img>`、`<script>`、`<video>`、`<audio>` | `<a>`、`<link>` |
| 路徑用途 | 指定要載入的圖片、JavaScript 或影音資源位置。 | 指定連結目標，或指定 CSS 等外部資源位置。 |
| 常見寫法 | `<img src="images/banner.jpg" alt="網站橫幅">` | `<a href="pages/about.html">關於我們</a>` |
| 注意事項 | 本章不展開各元素完整語意與屬性規則。 | 本章不展開 `<a>`、`<link>` 的完整元素用法。 |

### 選用建議

- 查圖片、JavaScript、影音資源位置時，優先看 `src`。
- 查頁面連結或 CSS 檔案位置時，優先看 `href`。
