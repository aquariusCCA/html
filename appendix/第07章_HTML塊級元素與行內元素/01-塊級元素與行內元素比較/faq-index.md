# FAQ Index

## 用途

用常見問題快速找到本篇筆記中能回答疑問的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| 塊級元素和行內元素最大的差異是什麼？ | 塊級元素通常形成獨立區塊並佔據一行；行內元素通常跟文字一起流動，不會主動換行。 | [一句話理解](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#一句話理解) |
| 為什麼 `<div>`、`<p>`、`<h1>` 預設會換行？ | 它們是筆記列出的常見塊級元素，預設會從新的一行開始並形成區塊。 | [塊級元素是什麼](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#塊級元素是什麼) |
| 為什麼 `<span>`、`<a>`、`<em>` 不會主動換行？ | 它們是筆記列出的常見行內元素，通常會跟前後文字或其他行內元素排在同一行。 | [行內元素是什麼](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#行內元素是什麼) |
| `width`、`height` 對行內元素有效嗎？ | 一般非置換行內元素設定 `width`、`height` 通常不會產生預期效果；置換元素則是例外。 | [特性對照](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#特性對照) |
| `margin`、`padding` 在兩類元素上的影響一樣嗎？ | 不一樣。塊級元素上下左右通常都影響版面，行內元素左右較直覺，上下效果較有限。 | [特性對照](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#特性對照) |
| `<img>`、`<input>` 算行內脈絡時，為什麼還可以有寬高？ | 它們屬於置換元素，雖然常被放在行內排版脈絡中，但本身可以有可見寬高。 | [重要例外](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#重要例外) |
| `<br>` 可以拿來做版面排版嗎？ | 不應該。筆記指出 `<br>` 是明確插入換行的元素，不應拿來取代 CSS 做版面排版。 | [重要例外](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#重要例外) |
| CSS `display` 改變後，HTML 語意也會改變嗎？ | 不會。`display` 只改變排版行為，不代表元素原本的 HTML 語意也跟著改變。 | [重要例外](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#重要例外) |
| 塊級元素一定可以放任何內容嗎？ | 不是。筆記以 `<p>` 不能包含 `<div>` 這類區塊內容提醒，內容能否巢狀要另行判斷。 | [常見誤解](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#常見誤解) |
| 判斷巢狀規則只看塊級 / 行內夠嗎？ | 不夠。塊級與行內主要描述排版行為，元素能否互相包含還要看 HTML 語意與內容模型。 | [常見誤解](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#常見誤解) |
| 實作時該怎麼選塊級元素或行內元素？ | 先看內容是完整區塊、段落、容器或列表，還是文字中的小段標記、連結、強調或小型輸入控制。 | [實務判斷](../../../notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md#實務判斷) |
