# FAQ Index

## 用途

用常見問題快速找到本篇筆記中可以回答疑問的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `<h1>` 到 `<h6>` 是什麼？ | 它們是表示標題層級的 HTML 標籤，數字越小代表層級越高、重要性越大。 | [一句話理解](../../../notes/第10章_標題標籤/01-HTML標題標籤-h1-h6.md#一句話理解) |
| `<h1>` 通常用在哪裡？ | `<h1>` 通常用來表示頁面或文章的主要標題。 | [標籤語意](../../../notes/第10章_標題標籤/01-HTML標題標籤-h1-h6.md#標籤語意) |
| `<h2>` 和 `<h3>` 有什麼關係？ | `<h2>` 表示主要章節，`<h3>` 表示該章節底下的小節。 | [基本範例](../../../notes/第10章_標題標籤/01-HTML標題標籤-h1-h6.md#基本範例) |
| 標題標籤可以只拿來控制字體大小嗎？ | 不應該。HTML 標題標籤負責語意和結構，外觀大小應交給 CSS。 | [常見錯誤](../../../notes/第10章_標題標籤/01-HTML標題標籤-h1-h6.md#常見錯誤) |
| 可以從 `<h1>` 直接跳到 `<h4>` 嗎？ | 如果內容沒有多層巢狀關係，任意跳級會讓文件結構不清楚。 | [常見錯誤](../../../notes/第10章_標題標籤/01-HTML標題標籤-h1-h6.md#常見錯誤) |
| 一般說明文字適合包成標題標籤嗎？ | 不適合。標題應概括內容，正文才用來承載細節。 | [常見錯誤](../../../notes/第10章_標題標籤/01-HTML標題標籤-h1-h6.md#常見錯誤) |
| 實務上如何安排標題層級？ | 可以先把頁面想成有目錄的文件，再依內容層級轉成 `<h1>`、`<h2>`、`<h3>` 等標籤。 | [實務使用建議](../../../notes/第10章_標題標籤/01-HTML標題標籤-h1-h6.md#實務使用建議) |
| 瀏覽器預設會怎麼顯示標題？ | 標題通常會加粗、獨佔一行，並從 `<h1>` 到 `<h6>` 逐漸變小。 | [瀏覽器預設樣式](../../../notes/第10章_標題標籤/01-HTML標題標籤-h1-h6.md#瀏覽器預設樣式) |
| 為什麼清楚的標題結構重要？ | 它能幫助讀者閱讀，也有助於維護、SEO 與無障礙。 | [重點整理](../../../notes/第10章_標題標籤/01-HTML標題標籤-h1-h6.md#重點整理) |
