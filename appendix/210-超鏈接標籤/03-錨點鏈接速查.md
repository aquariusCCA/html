# 錨點鏈接速查

> 查表用。概念與用法說明見 notes/210-超鏈接標籤/03-錨點鏈接與頁內定位.md。

## 基本組成

| 角色 | 寫法 | 用途 |
| --- | --- | --- |
| 連結端 | `<a href="#two">第二季</a>` | 使用者點擊後跳到指定位置 |
| 目標端 | `<h3 id="two">第二季介绍</h3>` | 建立可被跳轉到的位置 |
| 片段識別符 | `#two` | 在 `href` 中表示頁面內的某個元素 |
| 目標名稱 | `two` | 在 `id` 中只寫名稱，不寫 `#` |

## 錨點類型

| 類型 | href 寫法 | 用途 |
| --- | --- | --- |
| 本頁錨點 | `#section` | 跳到目前頁面的某個元素 |
| 他頁錨點 | `about.html#team` | 前往其他頁面並定位到指定元素 |
| 一般內部連結 | `about.html` | 前往站內頁面，不指定段落 |
| 一般外部連結 | `https://example.com` | 前往其他網站 |

## 寫法對照

| 目的 | 連結端 | 目標端 |
| --- | --- | --- |
| 跳到本頁指定段落 | `<a href="#faq">FAQ</a>` | `<h2 id="faq">FAQ</h2>` |
| 跳到其他頁指定段落 | `<a href="about.html#team">團隊</a>` | 目標頁面中需要有 `id="team"` 的元素 |
| 回到明確頂部位置 | `<a href="#top">回到頂部</a>` | `<header id="top">...</header>` |

## id 命名建議

| 建議 | 說明 |
| --- | --- |
| 使用有意義名稱 | 例如 `features`、`faq`、`contact` |
| 避免過度抽象名稱 | 例如 `a1`、`x2` 容易失去意義 |
| 同一頁保持唯一 | 重複 `id` 會讓定位結果不可靠 |
| 連結文字描述目標段落 | 頁面目錄文字應讓使用者知道會跳到哪裡 |

## 固定導覽列情境

| 問題 | 可用做法 | 備註 |
| --- | --- | --- |
| 錨點跳轉後標題被固定導覽列遮住 | 對目標標題設定 `scroll-margin-top` | 這是 CSS 調整，不是錨點必要語法 |

```css
h2 {
  scroll-margin-top: 80px;
}
```

## 常見錯誤與修正

| 錯誤寫法 | 問題 | 正確方向 |
| --- | --- | --- |
| `<a href="#two">第二季</a><h3 id="second">第二季介绍</h3>` | `href` 與 `id` 名稱不一致 | 讓兩者名稱對上 |
| `<h3 id="#two">第二季介绍</h3>` | `id` 裡多寫了 `#` | `id="two"` |
| 同一頁多個 `id="intro"` | `id` 不唯一，定位不可靠 | 每個 `id` 保持唯一 |
| 把 `##` 當成回到頂部 | 不是可靠的頁面定位方式 | 使用 `#top` 搭配明確目標元素 |
