# 特殊 href 用法速查

> 查表用。概念與用法說明見 notes/210-超鏈接標籤/04-特殊-href-用法補充.md。

## 特殊 href 對照

| href 寫法 | 常見用途 | 實務提醒 |
| --- | --- | --- |
| `#` | 回到頂部、空片段示例 | 不要把 `##` 當成回頂部；正式功能建議使用明確錨點 |
| `""` | 指向目前頁面 | 意圖不清楚，容易造成重新載入 |
| `javascript:...` | 點擊後執行 JavaScript | 正式頁面通常避免，操作行為改用 `<button>` |
| `tel:10010` | 嘗試喚起撥號 | 依裝置與瀏覽器支援而定 |
| `mailto:name@example.com` | 開啟郵件應用 | 使用者需有可用郵件客戶端 |
| `line://...` | 嘗試喚起指定 app | 依 app、平台與安裝狀態而定 |

## 建議替代寫法

| 情境 | 不建議長期使用 | 較清楚的方向 |
| --- | --- | --- |
| 正式回到頂部 | `<a href="#">回到頂部</a>` | `<a href="#top">回到頂部</a>` 搭配 `id="top"` |
| 還沒有連結目標 | `<a href="">更多內容</a>` | 先不要輸出可點擊的 `<a>`，或清楚標記待補 |
| 執行 JavaScript 操作 | `<a href="javascript:submitForm()">送出</a>` | `<button type="submit">送出</button>` |
| 點擊後執行一般操作 | `<a href="javascript:...">` | `<button>` 搭配事件監聽 |
| 喚起 app | 只提供 `line://...` | 若是重要流程，提供一般網址或提示文字作為備用方案 |

## 特殊連結適用判斷

| 使用目的 | 可用寫法 | 注意 |
| --- | --- | --- |
| 前往頁面 | 一般 `<a href="...">` | 符合 `<a>` 的主要語意 |
| 前往檔案 | 一般 `<a href="...">` | 開啟或下載由瀏覽器與伺服器共同決定 |
| 前往錨點 | `#id` 或 `page.html#id` | 目標元素需要有對應 `id` |
| 撥打電話 | `tel:` | 裝置需支援撥號功能 |
| 開啟郵件 | `mailto:` | 使用者需有可用郵件客戶端 |
| 喚起 app | app scheme | 不應成為唯一完成任務的路徑 |
| 觸發操作 | `<button>` | 不應用特殊 `href` 假裝操作按鈕 |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 用 `javascript:` 做主要互動 | HTML 結構與 JS 行為混在一起，也容易誤用 `<a>` | 使用 `<button>` 與事件監聽 |
| 長期保留空 `href` | 意圖不明，可能造成重新載入 | 先不要輸出連結，或明確補上目標 |
| 認為 app scheme 一定成功 | 取決於 app、平台與安裝狀態 | 重要流程提供替代路徑 |
| 將 `tel:` 或 `mailto:` 視為所有環境都可用 | 桌面或未設定客戶端時不一定成功 | 作為簡單入口使用，並考慮使用者環境 |
