# 文件下載與元素連結速查

> 查表用。概念與用法說明見 notes/210-超鏈接標籤/02-文件下載與元素連結.md。

## 檔案連結行為

| 目標資源 | href 範例 | 常見瀏覽器行為 | 備註 |
| --- | --- | --- | --- |
| 圖片 | `./assets/images/hyperlink-tags-img-002-354aea.jpg` | 可能直接開啟圖片 | 實際行為由檔案類型、伺服器設定與瀏覽器支援決定 |
| 影片 | `./assets/files/hyperlink-tags-file-002-9a18a4.mp4` | 可能直接開啟或播放 | 實際行為由檔案類型、伺服器設定與瀏覽器支援決定 |
| 音訊 | `./assets/files/hyperlink-tags-file-003-278402.mp3` | 可能直接開啟或播放 | 實際行為由檔案類型、伺服器設定與瀏覽器支援決定 |
| PDF | `./assets/pdfs/hyperlink-tags-pdf-001-78621d.pdf` | 可能在瀏覽器中預覽 | 不一定會下載 |
| zip | `./assets/files/hyperlink-tags-file-004-28aaa1.zip` | 常見情況是觸發下載 | 不是 `<a>` 自己決定，而是瀏覽器與伺服器共同決定 |

## download 屬性

| 寫法 | 用途 | 限制 |
| --- | --- | --- |
| `<a href="file.pdf" download>` | 提示瀏覽器下載目標資源 | 不是絕對命令 |
| `<a href="file.pdf" download="一份PDF檔.pdf">` | 提示下載，並提供建議檔名 | 實際檔名與行為仍可能受環境影響 |

## download 可能受影響的因素

| 因素 | 影響 |
| --- | --- |
| 目標檔案是否同源 | 不同來源檔案可能不依照 `download` 下載 |
| HTTP 標頭 | 伺服器設定可能影響開啟或下載行為 |
| 瀏覽器安全策略 | 瀏覽器可基於安全策略調整行為 |
| 使用者瀏覽器設定 | 使用者設定可能改變檔案處理方式 |

## 圖片與非互動元素連結

| 寫法 | 適用情境 | 注意 |
| --- | --- | --- |
| `<a href="..."><img src="..." alt="前往百度" /></a>` | 讓圖片成為連結 | `alt` 應說明連結目的 |
| `<a href="product.html">商品卡片內容</a>` | 讓商品卡或文章卡整塊可點擊 | 內容應避免包含互動控制項 |
| `<a href="index.html"><img src="logo.png" alt="回到首頁" /></a>` | Logo 圖片連回首頁 | 圖片連結不能缺少有意義的替代文字 |

## 適合與不適合放進 a 的內容

| 類型 | 內容 | 判斷 |
| --- | --- | --- |
| 適合 | 文字 | 非互動內容，可作為連結文字 |
| 適合 | 圖片 | 可作為圖片連結，但需要合適 `alt` |
| 適合 | 圖示 | 可作為連結入口 |
| 適合 | 卡片內容 | 可讓整塊內容前往詳情頁 |
| 適合 | 一般區塊元素 | HTML5 後可包住多數非互動內容 |
| 不適合 | `<button>` | 本身已是互動元素 |
| 不適合 | `<input>` | 本身已是互動元素 |
| 不適合 | `<select>` | 本身已是互動元素 |
| 不適合 | 另一個 `<a>` | 連結巢狀會造成語意混亂 |
| 不適合 | 帶有 `controls` 的音訊或影片 | 本身已有互動控制 |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 以為 `download` 一定會下載 | `download` 只是提示 | 重要下載流程需考慮後端檔案回應設定 |
| `<a href="product.html"><button>查看商品</button></a>` | 互動元素包互動元素 | 前往頁面用 `<a>`，觸發操作用 `<button>` |
| `<a href="index.html"><img src="logo.png" /></a>` | 圖片連結沒有替代文字 | 補上描述連結目的的 `alt` |
