## 9. 常用片段

### 9.1 最小主題分隔

| 項目 | 說明 |
|---|---|
| 使用情境 | 前後內容有明確主題分隔。 |
| 使用前提 | 分隔線代表內容語意，而不是單純視覺裝飾。 |
| 來源 | `4. 語法與基本用法` |

```html
<hr>
```

### 注意事項

- `<hr>` 是空元素，不需要結束標籤。
- 在 HTML 文件中通常寫成 `<hr>` 即可。

### 9.2 分隔兩個內容主題

| 項目 | 說明 |
|---|---|
| 使用情境 | 文章或文件中從一個主題切換到另一個主題。 |
| 使用前提 | 兩段內容有語意上的分隔。 |
| 來源 | `5. 範例說明` |

```html
<section>
  <h2>HTML 的角色</h2>
  <p>HTML 負責描述網頁內容與結構。</p>
</section>

<hr>

<section>
  <h2>CSS 的角色</h2>
  <p>CSS 負責控制網頁的視覺樣式與版面配置。</p>
</section>
```

### 注意事項

- `<hr>` 放在兩個主題之間，表示內容轉換。
- 若只是區塊樣式分隔，應考慮 CSS。

### 9.3 用 CSS 調整 `<hr>` 外觀

| 項目 | 說明 |
|---|---|
| 使用情境 | 需要保留 `<hr>` 語意，同時調整線條樣式。 |
| 使用前提 | `<hr>` 本身代表主題分隔。 |
| 來源 | `5. 範例說明` |

```html
<hr class="topic-divider">
```

```css
.topic-divider {
  border: 0;
  border-top: 1px solid #999;
  margin: 24px 0;
}
```

### 注意事項

- `<hr>` 提供語意，CSS 控制顏色、粗細與間距。
- 不要依賴舊式 HTML 屬性控制外觀。

### 9.4 純裝飾線替代做法

| 項目 | 說明 |
|---|---|
| 使用情境 | 線條只是視覺樣式的一部分。 |
| 使用前提 | 前後內容沒有主題轉換。 |
| 來源 | `6. 實務應用場景` |

```html
<div class="card">
  <h2>帳戶資訊</h2>
  <p>這裡是帳戶內容。</p>
</div>
```

```css
.card {
  border-top: 1px solid #ddd;
  padding-top: 16px;
}
```

### 注意事項

- 此做法用 CSS 製造視覺線條，不賦予額外 HTML 主題分隔語意。

## 11. 相似概念對照表

### 11.1 `<hr>` vs CSS 裝飾線

| 比較項目 | `<hr>` | CSS 裝飾線 |
|---|---|---|
| 定義 | 表示主題分隔或內容轉換的 HTML 元素。 | 使用 `border`、偽元素等方式產生的視覺線條。 |
| 使用時機 | 前後內容有語意上的分隔。 | 只是畫面需要線條或間距效果。 |
| 常見寫法 | `<hr>` | `border-top: 1px solid #ddd;` |
| 注意事項 | 不應只為視覺裝飾而使用。 | 不會表達主題分隔語意。 |
| 常見錯誤 | 用多個 `<hr>` 控制間距。 | 原筆記未列出 CSS 裝飾線本身的錯誤情境。 |

### 選用建議

- 需求是「內容主題切換」時，使用 `<hr>`。
- 需求是「畫面上需要一條線」時，使用 CSS。

### 11.2 `<hr>` vs `<section>`

| 比較項目 | `<hr>` | `<section>` |
|---|---|---|
| 定義 | 表示段落層級的主題轉換或內容分隔。 | 表示有主題的章節。 |
| 使用時機 | 需要標示兩段內容之間的轉換。 | 需要包住一組有主題的內容。 |
| 常見寫法 | `<hr>` | `<section>...</section>` |
| 注意事項 | 不承載內容。 | 可以包含標題與段落等內容。 |
| 常見錯誤 | 把 `<hr>` 當成排版線。 | 原筆記未列出 `<section>` 的錯誤情境。 |

### 選用建議

- 要表示內容「區塊本身」時，使用 `<section>`。
- 要表示兩段內容「中間的主題轉換」時，使用 `<hr>`。

### 11.3 `<hr>` vs `<hr />` vs `<hr></hr>`

| 比較項目 | `<hr>` | `<hr />` | `<hr></hr>` |
|---|---|---|---|
| 定義 | HTML 中常見的 `<hr>` 寫法。 | XHTML 風格寫法。 | 不建議的結束標籤寫法。 |
| 使用時機 | 一般 HTML 文件。 | 原筆記指出一般 HTML 中通常也能被解析。 | 不應使用。 |
| 常見寫法 | `<hr>` | `<hr />` | `<hr></hr>` |
| 注意事項 | 簡潔且符合本章建議。 | 可見於部分程式碼風格。 | `<hr>` 是空元素，不需要結束標籤。 |
| 常見錯誤 | 原筆記未列出。 | 原筆記未列出。 | 誤以為 `<hr>` 需要包住內容。 |

### 選用建議

- 一般 HTML 文件優先使用 `<hr>`。
- 避免寫成 `<hr></hr>`。
