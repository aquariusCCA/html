# dl dt dd 速查

> 查表內容整理自 `notes/220-列表標籤/02-自定義列表-dl-dt-dd.md`。

## 標籤對照

| 標籤 | 語意 | 常見內容 |
| --- | --- | --- |
| `<dl>` | description list，描述列表 | 包住整組 `dt` 與 `dd` |
| `<dt>` | description term，被描述的名稱或項目 | 名詞、分類名稱、問題、規格名稱 |
| `<dd>` | description details，描述內容 | 定義、說明、答案、規格值、連結 |

## 結構關係

| 結構 | 說明 |
| --- | --- |
| `<dl>` 包住整組描述列表 | 描述列表的外層容器 |
| `<dt>` 表示要被說明的項目名稱 | 可是名詞、分類名稱、問題或規格名稱 |
| `<dd>` 表示對該項目的描述 | 可是定義、說明、答案、規格值或連結 |
| 一個 `<dt>` 可以對應一個 `<dd>` | 適合一般名詞與解釋 |
| 一個 `<dt>` 可以對應多個 `<dd>` | 適合一個分類名稱底下有多個描述項 |

## 適用情境

| 內容關係 | 建議標籤 | 原因 |
| --- | --- | --- |
| 名詞與解釋 | `<dl>`、`<dt>`、`<dd>` | 能表達名稱與描述的對應 |
| 問題與答案 | `<dl>`、`<dt>`、`<dd>` | 問題可作為被描述項，答案可作為描述內容 |
| 規格名稱與規格值 | `<dl>`、`<dt>`、`<dd>` | 能呈現名稱與值的關係 |
| 單純並列項目 | `<ul>` | 內容只是同類項目集合，不是名稱與描述 |
| 操作步驟 | `<ol>` | 步驟順序是內容的一部分 |

## 使用 dl 前的判斷

| 判斷問題 | 使用方向 |
| --- | --- |
| 內容是否有「名稱 + 描述」關係？ | 有，適合考慮 `<dl>` |
| 每個名稱是否需要一段或多段說明？ | 是，適合使用 `<dt>` 搭配 `<dd>` |
| 使用 `<ul>` 會不會讓名稱與說明的關係變模糊？ | 會，使用 `<dl>` 更清楚 |
| 只是想去掉項目符號嗎？ | 不應為了外觀改用 `<dl>` |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 在 `<dl>` 裡放 `<li>` | `<li>` 是 `<ul>` 與 `<ol>` 的列表項目 | 並列項目用 `<ul>`；名稱與描述用 `<dt>`、`<dd>` |
| 只用 `<p>` 模擬名稱與描述 | HTML 沒有表達兩段文字的對應關係 | 若內容本身是名稱與描述，改用 `<dl>` |
| 把 `<dl>` 當成沒有符號的 `<ul>` | 語意會變得不正確 | 依資料是否有描述關係決定是否使用 `<dl>` |
