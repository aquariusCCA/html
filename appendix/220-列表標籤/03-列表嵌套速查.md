# 列表嵌套速查

> 查表內容整理自 `notes/220-列表標籤/03-列表嵌套.md`。

## 核心規則

| 規則 | 說明 |
| --- | --- |
| 列表嵌套表示「列表項目裡面還有列表」 | 內層列表是某個外層項目的子內容 |
| 內層 `<ul>` 或 `<ol>` 應放在外層某個 `<li>` 裡 | 不能直接放成外層 `<ul>`、`<ol>` 的同級子元素 |
| 外層 `<ul>`、`<ol>` 的直接子元素仍應該是 `<li>` | 直接子元素不是 `<li>` 時，列表結構不清楚 |
| 父層 `<li>` 的結束標籤應在子列表之後 | 父層 `<li>` 要包住文字內容與子列表 |
| 縮排只是閱讀輔助 | 真正建立層級的是標籤位置 |

## 父子關係對照

| 角色 | 結構位置 | 用途 |
| --- | --- | --- |
| 外層列表 | 最外層 `<ul>` 或 `<ol>` | 負責主要項目清單 |
| 外層項目 | 外層列表中的 `<li>` | 每個主要項目 |
| 內層列表 | 放在某個外層 `<li>` 裡 | 表示該外層項目的子項目 |
| 內層項目 | 內層列表中的 `<li>` | 子列表中的每個項目 |

## 正確與錯誤放置

| 寫法狀況 | 判斷 | 原因 |
| --- | --- | --- |
| 內層列表放在外層某個 `<li>` 裡 | 正確 | 表示內層列表屬於該外層項目 |
| 內層 `<ul>` 直接放在外層 `<ul>` 裡 | 錯誤 | 外層 `<ul>` 的直接子元素應該是 `<li>` |
| `</li>` 在內層列表之前結束 | 錯誤 | 內層列表沒有被父層 `<li>` 包住 |
| 只靠縮排判斷層級 | 風險高 | 縮排不會建立 HTML 父子結構 |

## 實務檢查清單

| 檢查項目 | 通過條件 |
| --- | --- |
| 外層列表的每個項目都是 `<li>` | `<ul>`、`<ol>` 的直接子元素是 `<li>` |
| 子列表放置位置正確 | 子列表位於父層 `<li>` 內 |
| 父層 `<li>` 結束位置正確 | `</li>` 出現在子列表之後 |
| 縮排與標籤位置一致 | 縮排能輔助閱讀，但不取代標籤檢查 |
| 巢狀層級不過深 | 層級太深時，考慮拆成更清楚的章節或區塊 |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 把內層 `<ul>` 直接放在外層 `<ul>` 裡 | 內層列表沒有屬於某個外層 `<li>` | 把內層列表放進對應的父層 `<li>` |
| 只相信縮排，不檢查標籤閉合位置 | 縮排看似正確，但 `</li>` 可能已提前結束 | 確認父層 `<li>` 包住子列表 |
| 層級過深 | 讀者難以理解，也會影響 CSS、鍵盤操作與螢幕報讀器體驗 | 考慮拆成更清楚的章節或區塊 |
