| 名詞 | 英文 | 簡要定義 | 用途 | 注意事項 | 來源 |
|---|---|---|---|---|---|

| 項目 | 說明 |
|---|---|
| 使用情境 | 標記一句話中的局部文字、重要內容與連結。 |

```html

### 6.3 `div` 包住一組區塊內容

| 項目 | 說明 |
|---|---|
| 使用情境 | 建立沒有更精準語意時的通用內容區塊。 |
| 使用前提 | 若有 `main`、`section`、`article` 等更精準語意元素，應優先考慮。 |

<div class="card">
  <h2>會員方案</h2>
  <p>適合想要系統學習 HTML 的初學者。</p>

```

### 6.4 `span` 標記局部文字

| 項目 | 說明 |

## 8. 相似概念對照表

### 8.1 塊級元素 vs 行內元素

| 比較項目 | 塊級元素 | 行內元素 |
|---|---|---|
| 預設排列 | 通常從新的一行開始，獨立成區塊。 | 通常跟文字或其他行內內容排在同一行。 |
| 寬度 | 未設定時通常填滿父元素可用寬度。 | 通常由內容決定。 |
| 寬高控制 | 可以設定 `width`、`height`。 | 一般文字型行內元素設定 `width`、`height` 不會產生預期效果。 |
| 常見用途 | 建立頁面結構或包住一組內容。 | 標記文字中的局部內容。 |
| 本章例子 | `div`、`p`、`h1` 到 `h6`、`ul`、`form` | `span`、`a`、`strong`、`em`、`img`、`input` |

### 選用建議

- 建立內容區塊或頁面結構時，優先考慮語意化的塊級元素。
- 標記句子中的局部文字時，優先考慮行內元素。

### 8.2 `div` vs `span`

| 比較項目 | `div` | `span` |
|---|---|---|
| 語意 | 無語意區塊容器。 | 無語意行內容器。 |
| 預設顯示方式 | 塊級。 | 行內。 |
| 使用時機 | 包住一組區塊內容。 | 包住一句話中的部分文字。 |
| 常見寫法 | `<div class="card">...</div>` | `<span class="status">開放報名</span>` |
| 注意事項 | 不要取代更精準的語意元素。 | 不要拿來包住大型區塊結構。 |

### 選用建議

- 如果內容是一整塊，先考慮語意元素，其次才是 `div`。
- 如果內容只是文字中的一小段，且沒有更適合的行內語意元素，可以使用 `span`。

### 8.3 預設顯示方式 vs HTML 語意

| 比較項目 | 預設顯示方式 | HTML 語意 |
|---|---|---|
| 定義 | 瀏覽器預設如何排列元素。 | 元素表示的內容意義。 |
| 是否可用 CSS 改變 | 可以，例如 `display: inline`。 | 不會因 CSS 顯示方式而改變。 |
| 查找問題 | 這個元素預設會不會換行？ | 這個元素代表什麼內容？ |
| 常見誤解 | 看到字大就把 `h1` 當樣式工具。 | 忽略標題、段落、連結的語意。 |
| 本章提醒 | 排版行為可以交給 CSS 調整。 | HTML 結構與語意要在原始碼中寫正確。 |

### 選用建議

- 先根據內容意義選 HTML 元素。
- 再用 CSS 調整畫面排列與尺寸。

### 8.4 CSS `display` 相關值速查

| 顯示方式 | 本章中的關係 | 適合查找的問題 |
|---|---|---|
| `block` | 對應塊級顯示行為。 | 如何讓元素形成獨立區塊？ |
| `inline` | 對應行內顯示行為。 | 如何讓元素跟文字排在同一行？ |
| `inline-block` | 本章作為行內元素尺寸控制的修正方向。 | 如何保留同列排列又能控制寬高？ |
| `flex` | 本章在導覽列範例中用於排列連結。 | 如何用 CSS 改變一組元素的排列？ |

### 選用建議

- `display` 是 CSS 呈現層的工具，不應用來取代 HTML 語意判斷。
- 若只是想改變視覺排列，優先改 CSS；不要為了外觀換成語意不正確的 HTML 元素。
