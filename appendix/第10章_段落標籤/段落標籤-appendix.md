## 9. 常用片段

### 9.1 段落元素基本語法

| 項目 | 說明 |
|---|---|
| 使用情境 | 建立單一文字段落。 |
| 使用前提 | 內容是一段完整的文字說明。 |
| 來源 | §3、§4 |

```html
<p>這是一段文字。</p>
```

### 注意事項

- 段落內容放在開始標籤 `<p>` 與結束標籤 `</p>` 之間。
- `p` 的重點是文字段落語意，不是視覺上的空白。

### 9.2 多個獨立段落

| 項目 | 說明 |
|---|---|
| 使用情境 | 文字進入另一段意思時。 |
| 使用前提 | 每段文字都是獨立段落。 |
| 來源 | §4、§7 |

```html
<p>第一段文字，說明第一個重點。</p>
<p>第二段文字，說明另一個重點。</p>
```

### 注意事項

- 不建議把多個獨立段落塞進同一個 `p`。
- 段落結構應交給 HTML，視覺間距應交給 CSS。

### 9.3 文章中的段落結構

| 項目 | 說明 |
|---|---|
| 使用情境 | 文章或說明文件需要標題與正文段落。 |
| 使用前提 | 標題與段落語意要分開。 |
| 來源 | §3.1、§5 |

```html
<article>
  <h1>學習 HTML 的第一步</h1>
  <p>HTML 是建立網頁內容結構的基礎。</p>
  <p>段落標籤可以把文字切分成容易閱讀的段落。</p>
</article>
```

### 注意事項

- `article` 與 `h1` 在此只作為文章範例結構，本附錄不擴寫成元素表。
- `h1` 表示文章主標題，`p` 表示正文段落。

### 9.4 修正用 `<br>` 製造段落間距

| 項目 | 說明 |
|---|---|
| 使用情境 | 原本用連續 `<br>` 把兩段文字隔開。 |
| 使用前提 | 兩段文字其實是不同段落。 |
| 來源 | §7 |

```html
<p>HTML 是用來描述網頁內容與結構的標記語言。</p>
<p>CSS 則用來控制畫面樣式。</p>
```

### 注意事項

- `<br>` 表示換行，不是段落結構或段落間距工具。
- 每個獨立段落應使用自己的 `p` 元素。

### 9.5 用 CSS 控制段落外觀

| 項目 | 說明 |
|---|---|
| 使用情境 | 需要統一控制段落行高與段落間距。 |
| 使用前提 | HTML 已使用正確的 `p` 結構。 |
| 來源 | §3.2、§7、§11 |

```css
p {
  line-height: 1.7;
  margin-block: 0 1em;
}
```

### 注意事項

- `p` 負責表達段落語意。
- CSS 負責控制字體、行高、顏色、寬度與段落間距。

## 10. 錯誤速查表

| 錯誤現象 | 可能原因 | 修正方向 | 對應知識點 | 來源 |
|---|---|---|---|---|
| 把整篇文章放在同一個 `p` 裡。 | 內容缺少清楚分段，閱讀與維護較困難。 | 每個獨立段落使用一個 `p`。 | 段落語意、多段段落 | §4、§7 |
| 用多個 `<br>` 製造段落間距。 | 混淆換行與段落結構。 | 使用多個 `p` 表達段落，使用 CSS 控制間距。 | `<br>`、CSS 間距控制 | §3.2、§7 |
| 用 `div` 包一般文章段落。 | `div` 沒有段落語意。 | 一般文字段落優先使用 `p`。 | `p` vs `div` | §3.1、§7 |
| 用 `p` 包標題。 | 混淆標題與段落語意。 | 標題使用 `h1` 到 `h6`，段落內容使用 `p`。 | `p` vs 標題標籤 | §3.1、§7、§8 |
| 為了改變間距而亂用空白字元。 | 把 HTML 當成主要版面控制工具。 | 使用 CSS 的 `margin`、`line-height` 等屬性控制視覺效果。 | HTML 結構、CSS 樣式 | §7、§11 |
| 在同一段意思中手動逐行加入換行標籤。 | 不理解段落文字會依容器寬度自動換行。 | 同一段意思保留在同一個 `p` 中，由瀏覽器自然換行。 | 自動換行 | §3.2、§5 |

## 11. 相似概念對照表

### 11.1 `p` vs 標題標籤

| 比較項目 | `p` | 標題標籤 |
|---|---|---|
| 定義 | 表示一般文字段落。 | 表示頁面、文章或區塊標題。 |
| 使用時機 | 需要承載正文、敘述、說明文字時。 | 需要指出內容主題或層級時。 |
| 常見寫法 | `<p>本段介紹網站維護資訊。</p>` | `<h1>網站公告</h1>` |
| 注意事項 | 不應用 `p` 包標題。 | 不應把一般段落文字包成標題。 |
| 常見錯誤 | 用 `p` 包標題。 | 混淆標題文字與一般正文。 |

### 選用建議

- 章節或文章標題使用 `h1` 到 `h6`。
- 一般說明、敘述與正文內容使用 `p`。

### 11.2 `p` vs `div`

| 比較項目 | `p` | `div` |
|---|---|---|
| 定義 | 有段落語意的文字元素。 | 原筆記中作為沒有段落語意的一般容器。 |
| 使用時機 | 內容是一段一般文字。 | 需要一般容器時。 |
| 常見寫法 | `<p>這是一段商品說明。</p>` | `<div>這是一段商品說明。</div>` |
| 注意事項 | 可讓內容結構更清楚。 | 不應優先取代一般文字段落。 |
| 常見錯誤 | 原筆記未列出 `p` 在此比較中的其他錯誤。 | 用 `div` 包一般文章段落。 |

### 選用建議

- 一般文章、新聞或說明文字優先使用 `p`。
- `div` 不應作為一般段落文字的語意替代品。

### 11.3 `p` vs `<br>`

| 比較項目 | `p` | `<br>` |
|---|---|---|
| 定義 | 表示一段完整文字內容。 | 表示換行。 |
| 使用時機 | 文字進入另一段意思時建立新的 `p`。 | 原筆記只說明不應用多個 `<br>` 製造段落間距。 |
| 常見寫法 | `<p>第一段。</p><p>第二段。</p>` | `<br>` |
| 注意事項 | 段落之間的視覺距離應由 CSS 控制。 | 不代表段落結構或間距工具。 |
| 常見錯誤 | 把多段文字放進同一個 `p`。 | 用多個 `<br>` 假裝段落分隔。 |

### 選用建議

- 分段時使用多個 `p`。
- 不要用連續 `<br>` 取代段落結構。

### 11.4 HTML 段落結構 vs CSS 視覺控制

| 比較項目 | HTML 段落結構 | CSS 視覺控制 |
|---|---|---|
| 定義 | 用 `p` 表達文字是一段段落。 | 控制段落的字體、行高、顏色、寬度與間距。 |
| 使用時機 | 需要描述內容結構與段落語意時。 | 需要調整畫面呈現時。 |
| 常見寫法 | `<p>這是一段文字。</p>` | `p { line-height: 1.7; }` |
| 注意事項 | 不應用空白或空段落控制畫面。 | 不應取代 HTML 的內容語意。 |
| 常見錯誤 | 用 HTML 空白硬做版面。 | 原筆記未列出 CSS 本身的錯誤寫法。 |

### 選用建議

- 先用 HTML 建立清楚的段落結構。
- 再用 CSS 統一管理段落外觀與間距。
