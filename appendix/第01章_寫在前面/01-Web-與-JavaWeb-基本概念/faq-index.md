| Web 是什麼？ | Web 是 World Wide Web 的簡稱，可以理解成由許多資源彼此連結而成的資訊系統。 | [Web 是什麼](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#web-是什麼) |
| Web 裡的資源可能有哪些？ | Web 資源可能是 HTML 網頁、圖片、影片、音訊、下載檔案，或由後端程式產生的資料結果。 | [Web 是什麼](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#web-是什麼) |
| URL 在 Web 中做什麼？ | URL 描述資源的位置，也就是使用者在瀏覽器輸入的網址。 | [Web 如何把資源連起來](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#web-如何把資源連起來) |
| HTTP / HTTPS 在 Web 中做什麼？ | HTTP / HTTPS 是瀏覽器與服務器之間交換資料時常用的通訊規則。 | [Web 如何把資源連起來](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#web-如何把資源連起來) |
| 瀏覽器在 Web 中負責什麼？ | 瀏覽器會發送請求、接收資源、解析 HTML，最後把內容呈現成頁面。 | [Web 如何把資源連起來](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#web-如何把資源連起來) |
| 服務器在 Web 中負責什麼？ | 服務器保存或產生資源，並回應瀏覽器的請求。 | [Web 如何把資源連起來](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#web-如何把資源連起來) |
| HTML 在 Web 中主要負責什麼？ | HTML 的核心任務是描述文件結構，例如標題、段落與連結。 | [HTML 在 Web 中的位置](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#html-在-web-中的位置) |
| JavaWeb 是什麼？ | JavaWeb 通常指使用 Java 技術開發、部署在 Web 環境中，並可透過瀏覽器訪問的 Web 應用。 | [JavaWeb 是什麼](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#javaweb-是什麼) |
| Web 等於網際網路嗎？ | 不等於。網際網路是更底層的網路基礎設施，Web 是建立在其上的資訊服務。 | [Web 不等於網際網路](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#web-不等於網際網路) |
| Web 等於 HTML 嗎？ | 不等於。HTML 是 Web 的重要組成，但完整網站還可能包含 CSS、JavaScript、圖片、服務器、資料庫等內容。 | [Web 不等於 HTML](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#web-不等於-html) |
| JavaWeb 等於所有 Web 開發嗎？ | 不等於。JavaWeb 是 Web 開發的一種技術路線，Web 後端也可以用其他語言或平台完成。 | [JavaWeb 不等於所有 Web 開發](../../../notes/第01章_寫在前面/01-Web-與-JavaWeb-基本概念.md#javaweb-不等於所有-web-開發) |
