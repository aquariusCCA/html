# FAQ Index

## 用途

用常見問題快速找到本篇筆記中能回答的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| Web 服務器負責什麼？ | Web 服務器負責接收瀏覽器請求，並把網頁文件、圖片、資料或其他資源回傳給使用者。 | [一句話理解](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#一句話理解) |
| 服務器是什麼？ | 服務器是對外提供計算或網路服務的設備、程式或系統。 | [服務器是什麼](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#服務器是什麼) |
| Web 服務器可以提供哪些內容？ | 它可以存放 HTML、CSS、JavaScript、圖片等網站文件，也可以回傳資料、下載檔案或 API 資料。 | [Web 服務器做什麼](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#web-服務器做什麼) |
| 為什麼自己電腦能打開 HTML，不代表別人也能看？ | 本機檔案通常沒有穩定公開網址，也可能被網路、防火牆或路由器阻擋外部請求。 | [為什麼本機檔案不能直接讓所有人訪問](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#為什麼本機檔案不能直接讓所有人訪問) |
| 本地服務器是什麼？ | 本地服務器是在自己的電腦上啟動的服務器環境，常用於開發與測試。 | [本地服務器](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#本地服務器) |
| 本地服務器只能自己訪問嗎？ | 通常優先供自己的電腦訪問；如果網路、防火牆與服務綁定位址允許，同一局域網內的其他裝置也可能訪問。 | [本地服務器](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#本地服務器) |
| 遠程服務器是什麼？ | 遠程服務器通常部署在其他機房、雲端平台或主機服務商上，可讓使用者透過域名或公開網址訪問網站。 | [遠程服務器](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#遠程服務器) |
| 遠程服務器有什麼價值？ | 它能長時間提供服務、被網路上的使用者訪問、綁定域名，並配合安全憑證、資料庫或後端服務。 | [遠程服務器](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#遠程服務器) |
| 網站部署是什麼？ | 部署就是把本地開發完成的網站或應用放到可以提供服務的環境中。 | [網站部署是什麼](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#網站部署是什麼) |
| 靜態 HTML 網站部署通常要上傳什麼？ | 靜態 HTML 網站可能只需要上傳 `index.html`、`style.css`、`main.js` 與 `images/` 等檔案。 | [網站部署是什麼](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#網站部署是什麼) |
| 動態網站部署還可能包含什麼？ | 動態網站部署還可能包含啟動後端服務、設定資料庫、域名、HTTPS、環境變數與反向代理。 | [網站部署是什麼](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#網站部署是什麼) |
| 本機能開就代表網站已經上線嗎？ | 不代表。本機能開只表示自己的電腦能讀到檔案，是否能被其他人訪問取決於公開服務地址與可連線的服務器環境。 | [以為能在自己電腦打開就代表已經上線](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#以為能在自己電腦打開就代表已經上線) |
| 瀏覽器是直接看到服務器硬碟嗎？ | 不是。瀏覽器是透過請求取得資源，再由瀏覽器解析與呈現。 | [忽略瀏覽器與服務器的請求關係](../../../notes/第01章_寫在前面/03-Web-服務器與網站部署.md#忽略瀏覽器與服務器的請求關係) |
