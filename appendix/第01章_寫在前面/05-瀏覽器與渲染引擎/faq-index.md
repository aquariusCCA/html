# FAQ Index

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| 瀏覽器只是開網頁的工具嗎？ | 不是。瀏覽器會請求資源、解析 HTML、套用 CSS、執行 JavaScript，最後把結果繪製到畫面上。 | [問題情境](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#問題情境) |
| 瀏覽器在 Web 開發中負責哪些事？ | 瀏覽器會取得 HTML、CSS、JavaScript、圖片等資源，解析結構與樣式，執行互動邏輯並繪製畫面。 | [瀏覽器是什麼](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#瀏覽器是什麼) |
| 渲染引擎是什麼？ | 渲染引擎是瀏覽器中負責解析與呈現網頁的核心部分。 | [渲染引擎是什麼](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#渲染引擎是什麼) |
| 渲染引擎和瀏覽器內核是同一件事嗎？ | 筆記說明渲染引擎也常被稱為瀏覽器內核，重點是它負責解析與呈現網頁。 | [渲染引擎是什麼](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#渲染引擎是什麼) |
| 打開網頁時，瀏覽器大致做了哪些流程？ | 瀏覽器先向服務器請求資源，再解析 HTML 與 CSS，建立結構與樣式結果，可能執行 JavaScript，最後繪製畫面。 | [從請求到畫面的簡化流程](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#從請求到畫面的簡化流程) |
| 為什麼不同瀏覽器可能顯示不同？ | 不同瀏覽器在版本、渲染引擎、預設樣式、功能支援與實作細節上可能不同。 | [為什麼不同瀏覽器會有差異](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#為什麼不同瀏覽器會有差異) |
| 為什麼 Web 標準重要？ | 標準越清楚，瀏覽器越能依照共同規則實作，開發者也越能寫出穩定的頁面。 | [為什麼不同瀏覽器會有差異](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#為什麼不同瀏覽器會有差異) |
| 為什麼不能只在一個瀏覽器中測試？ | 初學可以先用一個瀏覽器練習，但正式頁面應至少考慮主要目標瀏覽器。 | [只在一個瀏覽器中測試](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#只在一個瀏覽器中測試) |
| 瀏覽器顯示正常是否代表 HTML 結構正確？ | 不代表。瀏覽器會嘗試修正錯誤標記，但畫面正常不等於 HTML 語意與結構合理。 | [把瀏覽器顯示正常當成 HTML 結構正確](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#把瀏覽器顯示正常當成-html-結構正確) |
| 忽略瀏覽器預設樣式會造成什麼問題？ | 標題、段落、列表、表單元素都有預設樣式；如果忽略它們，可能誤判頁面的實際呈現。 | [忽略瀏覽器預設樣式](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#忽略瀏覽器預設樣式) |
| 學 HTML 時為什麼要把瀏覽器視為第一個讀者？ | 因為瀏覽器會根據 HTML 語意理解標題、段落、順序與文件結構，進而影響頁面理解與呈現。 | [對 HTML 學習的意義](../../../notes/第01章_寫在前面/05-瀏覽器與渲染引擎.md#對-html-學習的意義) |
