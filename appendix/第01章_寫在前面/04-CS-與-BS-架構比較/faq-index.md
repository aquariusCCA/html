# FAQ Index

## 用途

用常見問題快速找到本篇筆記中能回答疑問的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| C/S 是什麼？ | C/S 是 Client/Server，也就是客戶端 / 服務器架構，通常需要安裝專用客戶端。 | [C/S 架構](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#cs-架構) |
| B/S 是什麼？ | B/S 是 Browser/Server，也就是瀏覽器 / 服務器架構，使用者通常只需要瀏覽器即可使用系統。 | [B/S 架構](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#bs-架構) |
| C/S 和 B/S 最大差異是什麼？ | C/S 偏向專用客戶端加服務器；B/S 偏向瀏覽器加服務器。 | [一句話理解](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#一句話理解) |
| B/S 架構的三層是哪三層？ | 常見三層是表現層、邏輯層與數據層。 | [B/S 架構的三層結構](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#bs-架構的三層結構) |
| 這裡的表現層是不是 CSS 負責的表現？ | 不是。這裡的表現層是系統架構中的前端層，不等於 Web 標準三層裡 CSS 負責的視覺樣式。 | [B/S 架構的三層結構](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#bs-架構的三層結構) |
| 為什麼學 HTML 要理解 B/S 架構？ | HTML 主要出現在 B/S 架構的表現層，用來定義瀏覽器要呈現的文件結構。 | [對 HTML 學習的意義](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#對-html-學習的意義) |
| B/S 一定比 C/S 好嗎？ | 不一定。兩者是不同架構選擇，要看使用場景、更新方式、使用者範圍、本機能力需求與維護成本。 | [以為 B/S 一定比 C/S 好](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#以為-bs-一定比-cs-好) |
| 瀏覽器能不能做複雜互動？ | 可以，現代瀏覽器能承擔大量互動與運算，但仍需要清楚劃分前端、後端與資料層責任。 | [以為瀏覽器不能做複雜事情](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#以為瀏覽器不能做複雜事情) |
| 前端可以直接操作資料庫嗎？ | 在 Web 系統中通常不應該，資料存取應經過後端服務處理權限、安全與業務規則。 | [讓瀏覽器直接接觸資料庫](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#讓瀏覽器直接接觸資料庫) |
| C/S 架構常見缺點有哪些？ | 常見限制包含需要安裝、客戶端升級、多平台適配成本，以及不適合大量未知使用者快速訪問。 | [C/S 架構的缺點](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#cs-架構的缺點) |
| B/S 架構常見限制有哪些？ | 常見限制包含瀏覽器差異、複雜互動需要更多前端工程投入，以及效能與安全性需要完整系統設計。 | [B/S 架構的缺點](../../../notes/第01章_寫在前面/04-CS-與-BS-架構比較.md#bs-架構的缺點) |
