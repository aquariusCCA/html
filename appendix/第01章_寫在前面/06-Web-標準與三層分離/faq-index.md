# FAQ Index

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| Web 標準主要解決什麼問題？ | Web 標準讓不同瀏覽器盡量依照共同規範解析與呈現內容，降低同一份程式碼在不同瀏覽器中的差異。 | [為什麼需要 Web 標準](../../../notes/第01章_寫在前面/06-Web-標準與三層分離.md#為什麼需要-web-標準) |
| 如果沒有共同標準，開發網頁會遇到什麼問題？ | HTML、CSS、JavaScript 可能在不同瀏覽器中被不同方式理解，開發者也可能需要寫大量特殊版本。 | [為什麼需要 Web 標準](../../../notes/第01章_寫在前面/06-Web-標準與三層分離.md#為什麼需要-web-標準) |
| 三層分離是什麼？ | 三層分離是讓 HTML、CSS、JavaScript 各自負責合適任務，而不是把結構、外觀與互動混在一起。 | [三層分離是什麼](../../../notes/第01章_寫在前面/06-Web-標準與三層分離.md#三層分離是什麼) |
| HTML、CSS、JavaScript 分別負責什麼？ | HTML 負責結構，CSS 負責樣式，JavaScript 負責互動。 | [Web 標準的三層構成](../../../notes/第01章_寫在前面/06-Web-標準與三層分離.md#web-標準的三層構成) |
| 為什麼 HTML 要重視語意？ | HTML 的責任是建立正確內容結構，語意清楚有助於瀏覽器、搜尋引擎、輔助技術與工程師理解頁面。 | [為什麼 HTML 要重視語意](../../../notes/第01章_寫在前面/06-Web-標準與三層分離.md#為什麼-html-要重視語意) |
| CSS 可以改變外觀，為什麼不能取代 HTML 語意？ | CSS 負責外觀，但 HTML 仍要承擔內容結構與語意責任。 | [為什麼 HTML 要重視語意](../../../notes/第01章_寫在前面/06-Web-標準與三層分離.md#為什麼-html-要重視語意) |
| 為什麼不建議把大量樣式直接寫在 HTML 裡？ | 這會讓 HTML 結構變得難讀，也不利於重複使用與維護。 | [把樣式直接塞進 HTML 結構](../../../notes/第01章_寫在前面/06-Web-標準與三層分離.md#把樣式直接塞進-html-結構) |
| HTML 結構混亂時，可以靠 JavaScript 補救嗎？ | 不建議。HTML 結構混亂會讓後續 JavaScript 操作變得更複雜，應先整理好結構。 | [用 JavaScript 補救不合理的 HTML 結構](../../../notes/第01章_寫在前面/06-Web-標準與三層分離.md#用-javascript-補救不合理的-html-結構) |
| 初學者只追求畫面看起來像，有什麼問題？ | 正式開發還要重視語意、層級、可讀性與可維護性，不能只用任何能顯示的方式完成頁面。 | [只追求畫面像，不在意結構](../../../notes/第01章_寫在前面/06-Web-標準與三層分離.md#只追求畫面像不在意結構) |
| 寫一個頁面時應該先想什麼？ | 可以先想頁面有哪些內容、語意與層級，再決定 HTML 標籤、CSS 樣式與 JavaScript 互動。 | [實務使用場景](../../../notes/第01章_寫在前面/06-Web-標準與三層分離.md#實務使用場景) |
