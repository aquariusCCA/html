# HTML 注釋速查

> 查表用。概念與用法說明見 notes/040-HTML基本結構標籤/03-HTML注釋.md。

## 基本語法

```html
<!-- 這是一段注釋 -->
```

```html
<!-- 網站主導覽 -->
<nav>
  <a href="/">首頁</a>
  <a href="/about.html">關於我們</a>
</nav>
```

## 注釋特性

| 項目 | 說明 | 範例 / 備註 |
| --- | --- | --- |
| 開始符號 | HTML 注釋以 `<!--` 開頭 | |
| 結束符號 | HTML 注釋以 `-->` 結束 | |
| 顯示行為 | 注釋不會顯示成頁面內容 | 畫面上會顯示導覽連結，但不會顯示 `網站主導覽` |
| 執行行為 | 注釋不會作為 HTML 標籤執行 | `<!-- <h1>這個標題不會顯示</h1> -->` |
| 暫時停用 | 可用於暫時停用一小段 HTML | 不要長期把大量不用的程式碼留在注釋中 |

## 好注釋與壞注釋

| 類型 | 範例 | 判斷 |
| --- | --- | --- |
| 較有幫助 | `<!-- 首頁主視覺區：包含標題、簡介與主要行動按鈕 -->` | 補充整個 `<section>` 的角色 |
| 較沒有幫助 | `<!-- 這是一個 h1 -->` | 只是重複 `<h1>` 本身已經清楚表示的資訊 |

## 適合與不適合的情境

| 類型 | 情境 |
| --- | --- |
| 適合加注釋 | 標記大型區塊的開始或用途 |
| 適合加注釋 | 解釋短時間內不容易從程式碼看出的意圖 |
| 適合加注釋 | 提醒後續維護者某段結構與其他檔案或功能有關 |
| 不太需要加注釋 | 注釋只是重複標籤名稱 |
| 不太需要加注釋 | 程式碼已經很短且語意清楚 |
| 不太需要加注釋 | 注釋內容會很快過期，卻沒有人維護 |

## 常見錯誤與建議

| 錯誤 | 影響 | 建議 |
| --- | --- | --- |
| 忘記結束注釋 | 後面的內容可能都被當成注釋，導致頁面顯示異常 | 補上結尾 `-->` |
| 把密碼、金鑰、內部帳號、未公開資料寫進注釋 | 注釋仍可能存在於瀏覽器可查看的原始碼中 | 不要把敏感資訊寫進 HTML 注釋 |
| 用注釋取代清楚的 HTML 結構 | 注釋只能輔助理解，不能取代合適標籤 | 若頁面區塊可用語意化標籤表達，就使用合適 HTML 標籤 |
| 長期保留大量不用的程式碼 | 檔案會變得難以維護 | 只暫時停用小段 HTML，不長期保留大量舊碼 |

## 注釋與語意化標籤

| 寫法 | 判斷 |
| --- | --- |
| `<!-- 導覽列 --><div><a href="/">首頁</a></div>` | 較差，依賴注釋說明區塊用途 |
| `<nav><a href="/">首頁</a></nav>` | 較好，使用合適標籤表達導覽用途 |
