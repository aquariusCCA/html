# 標籤屬性寫法速查

> 查表用。概念與用法說明見 notes/040-HTML基本結構標籤/02-標籤屬性.md。

## 基本寫法

```html
<標籤名 屬性名="屬性值">內容</標籤名>
```

```html
<p id="notice" class="highlight">
  尚硅谷，讓天下沒有難學的技術!!!
</p>
```

## 屬性寫法對照

| 類型 / 寫法 | 用途 | 範例 | 備註 |
| --- | --- | --- | --- |
| 一般屬性 | 替標籤提供附加資訊 | `id="notice"`、`class="highlight"` | 多數屬性寫成 `屬性名="屬性值"` |
| 連結目標屬性 | 指定連結前往位置 | `<a href="https://example.com">前往網站</a>` | `href` 常用在 `<a>` |
| 圖片相關屬性 | 指定圖片來源與替代文字 | `<img src="logo.png" alt="網站標誌">` | `src` 和 `alt` 常用在 `<img>` |
| 輸入框相關屬性 | 指定輸入類型與欄位名稱 | `<input type="text" name="username">` | `type` 常用在 `<input>` |
| 通用屬性 | 可寫在很多標籤上 | `id`、`class` | 常用於 CSS 或 JavaScript 選取 |
| 布林屬性 | 表示某個狀態是否啟用 | `<input disabled>` | 只要屬性存在，就代表狀態啟用 |
| 多個屬性 | 同一標籤同時有多個補充設定 | `<input type="text" name="username" id="username" class="form-control">` | 屬性之間用空格隔開 |

## 範例拆解

| 程式片段 | 說明 |
| --- | --- |
| `<p>` | 表示這是一個段落 |
| `id="notice"` | 替這個段落設定唯一識別名稱 |
| `class="highlight"` | 替這個段落設定分類名稱，常用於 CSS 選取 |
| `尚硅谷，讓天下沒有難學的技術!!!` | 實際顯示在頁面上的段落內容 |

## 多屬性範例拆解

| 屬性 | 用途 |
| --- | --- |
| `type="text"` | 設定輸入框類型 |
| `name="username"` | 設定表單送出時使用的欄位名稱 |
| `id="username"` | 設定頁面中的識別名稱 |
| `class="form-control"` | 設定分類名稱，通常給 CSS 或 JavaScript 使用 |

## 常見錯誤與正確寫法

| 錯誤 | 問題 | 正確寫法 |
| --- | --- | --- |
| `<pclass="intro">這是一段文字。</p>` | 標籤名和屬性之間少了必要空格，瀏覽器不會理解成 `<p>` 加上 `class` | `<p class="intro">這是一段文字。</p>` |
| `<input type="text"name="username">` | 屬性之間沒有空格，會讓程式碼難以閱讀，也可能造成解析問題 | `<input type="text" name="username">` |
| `<input type="text" type="password">` | 同一標籤出現同名屬性，設定互相衝突 | `<input type="password">` |
| `<div class=card primary></div>` | 省略引號後，遇到包含空格的屬性值會出問題 | `<div class="card primary"></div>` |
| `<p CLASS=intro>` | 可讀性與一致性較差，也容易在複雜屬性值中出錯 | `<p class="intro">` |

## 屬性檢查清單

| 檢查項目 | 判斷 |
| --- | --- |
| 屬性是否適合寫在目前標籤上 | 不同標籤可用屬性不同 |
| 屬性名與標籤名之間是否有空格 | 必須有空格 |
| 多個屬性之間是否有空格 | 必須用空格隔開 |
| 屬性值是否需要引號 | 若不確定，就使用雙引號 |
| 同一個標籤是否不小心寫了重複屬性 | 同一標籤不要出現同名屬性 |
