# 第20章_超鏈接標籤 - 附錄查表資料

## 1. 附錄定位

### 適用範圍

本附錄對應 `notes/第20章_超鏈接標籤/超鏈接標籤.md`，整理 HTML `<a>` 超鏈接標籤的用途、常見 `href` 類型、相關屬性、最小可用範例、常見錯誤與概念對照。

### 使用方式

- 查 `<a>` 標籤、`href`、`target`、`download`、錨點與 URL scheme 的用途。
- 查外部連結、內部連結、檔案連結、頁面錨點與應用喚起的最小寫法。
- 查 `target="_blank"`、空連結、`href="#"`、圖片連結與按鈕語意的常見問題。
- 透過來源索引回到正式筆記的小節深入閱讀。

## 3. 名詞表

| 名詞 | 英文 | 簡要定義 | 用途 | 注意事項 | 來源 |
|---|---|---|---|---|---|
| 超鏈接 | hyperlink | 從目前位置前往其他頁面、資源或頁內位置的連結。 | 建立網頁之間或頁內區塊之間的導航。 | 連結文字應讓使用者知道目的地。 | §2、§3、§6 |
| 錨 | anchor | `<a>` 的語意來源，代表可導航的目的地。 | 透過 `<a>` 建立連結。 | `<a>` 不只是讓文字可點擊，而是表示導航目的。 | §2、§3 |
| `href` | hypertext reference | 指定 `<a>` 連結目標的屬性。 | 指向外部網址、內部頁面、檔案、錨點或 URL scheme。 | 路徑、檔名與片段名稱需正確。 | §3、§4.1 |
| 外部連結 | external link | 指向其他網站的連結。 | 前往不同網域或外部網站。 | 新分頁開啟外部網站時建議搭配 `rel="noopener noreferrer"`。 | §4.1、§5 範例一 |
| 內部連結 | internal link | 指向同一網站其他頁面的連結。 | 網站導覽、公司簡介、列表詳情頁入口。 | 常使用相對路徑，例如 `./about.html`。 | §4.1、§5 範例二 |
| 檔案連結 | file link | 指向圖片、影片、音訊、PDF、ZIP 等資源的連結。 | 開啟或下載檔案。 | 是否直接開啟取決於檔案類型、瀏覽器能力與伺服器回應設定。 | §4.1、§5 範例三 |
| 錨點連結 | fragment link | 指向目前頁面或其他頁面中特定 `id` 的連結。 | 跳到章節、FAQ、目錄項目等指定位置。 | `href="#名稱"` 必須對應目標元素的 `id="名稱"`。 | §4.1、§5 範例四 |
| URL scheme | URL scheme | 以 `tel:`、`mailto:`、`line://` 等形式喚起特定應用的連結格式。 | 撥號、寄信或開啟指定應用。 | 是否成功取決於裝置、瀏覽器與應用支援。 | §5 範例七、§11 |
| 空連結 | empty link | `href=""` 的連結。 | 原筆記作為反例說明。 | 通常指向目前頁面，可能造成重新載入；不建議長期保留。 | §5 範例五、§7 |
| JavaScript 連結 | JavaScript URL | 使用 `href="javascript:..."` 執行 JavaScript 的寫法。 | 原筆記作為不建議的一般互動寫法。 | 若不是導航，通常應改用 `<button>`。 | §5 範例六、§7 |

## 5. 屬性表

| 屬性 | 適用元素 | 常見值 | 用途 | 範例 | 注意事項 | 來源 |
|---|---|---|---|---|---|---|
| `href` | `<a>` | `https://example.com`、`./about.html`、`#section-2`、`tel:10010`、`mailto:service@example.com` | 指定連結目標。 | `<a href="./about.html">公司簡介</a>` | 空值、錯誤路徑或找不到 `id` 都可能造成非預期結果。 | §3、§4.1、§7 |
| `target` | `<a>` | `_self`、`_blank` | 指定連結開啟位置。 | `<a href="https://example.com" target="_blank">Example</a>` | 未指定時預設為 `_self`。 | §4.2、§5 範例一 |
| `rel` | `<a>` | `noopener noreferrer` | 搭配 `target="_blank"` 降低新頁面影響原頁面的風險。 | `<a href="https://example.com" target="_blank" rel="noopener noreferrer">Example</a>` | 原筆記建議外部連結新分頁開啟時使用。 | §4.2、§5 範例一、§7 |
| `download` | `<a>` | `一份PDF檔.pdf` | 提示瀏覽器下載資源。 | `<a href="./assets/pdfs/test.pdf" download="一份PDF檔.pdf">下載 PDF 檔案</a>` | 正式拼字是 `download`，不是 `dowload`。 | §5 範例三 |
| `id` | 目標元素 | `season-2`、`intro` | 作為錨點連結的目標位置。 | `<h3 id="season-2">第二季介紹</h3>` | 需與 `href="#season-2"` 名稱一致。 | §5 範例四、§7 |
| `alt` | `<img>` | `前往百度` | 描述圖片連結的目的或圖片意義。 | `<img src="./assets/images/example.jpg" alt="前往百度">` | 不能只寫檔名；會影響輔助工具使用者理解。 | §5 範例八、§7 |

## 6. 常用片段

### 6.1 基本外部連結

| 項目 | 說明 |
|---|---|
| 使用情境 | 前往外部網站。 |
| 使用前提 | 已有明確 URL 與可辨識的連結文字。 |
| 來源 | §3、§4.1、§5 範例一 |

```html
<a href="https://example.com">前往 Example</a>
```

### 6.2 新分頁開啟外部連結

| 項目 | 說明 |
|---|---|
| 使用情境 | 外部網站需要在新分頁或新視窗開啟。 |
| 使用前提 | 需要使用 `target="_blank"`。 |
| 來源 | §4.2、§5 範例一、§7 |

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  前往 Example
</a>
```

### 6.3 內部頁面連結

| 項目 | 說明 |
|---|---|
| 使用情境 | 前往同一網站中的其他頁面。 |
| 使用前提 | 路徑、檔名與資料夾位置正確。 |
| 來源 | §4.1、§5 範例二 |

```html
<a href="./about.html">公司簡介</a>
```

### 6.4 檔案下載連結

| 項目 | 說明 |
|---|---|
| 使用情境 | 提示下載 PDF 或其他檔案。 |
| 使用前提 | 資源路徑正確，且需要下載提示。 |
| 來源 | §5 範例三 |

```html
<a href="./assets/pdfs/test.pdf" download="一份PDF檔.pdf">下載 PDF 檔案</a>
```

### 6.5 目前頁面錨點

| 項目 | 說明 |
|---|---|
| 使用情境 | 跳到目前頁面指定區塊。 |
| 使用前提 | 目標元素有對應且不重複的 `id`。 |
| 來源 | §5 範例四、§7 |

```html
<a href="#season-2">第二季</a>

<h3 id="season-2">第二季介紹</h3>
```

### 6.6 其他頁面錨點

| 項目 | 說明 |
|---|---|
| 使用情境 | 前往其他頁面的指定區塊。 |
| 使用前提 | 目標頁面與目標 `id` 都存在。 |
| 來源 | §4.1、§5 範例四 |

```html
<a href="./test.html#intro">跳轉到測試頁面的 intro 區塊</a>
```

### 6.7 電話與郵件連結

| 項目 | 說明 |
|---|---|
| 使用情境 | 手機網頁中喚起撥號或郵件應用。 |
| 使用前提 | 使用者裝置與瀏覽器支援對應 URL scheme。 |
| 來源 | §4.1、§5 範例七、§11 |

```html
<a href="tel:10010">電話聯繫</a>
<a href="mailto:10010@qq.com">郵件聯繫</a>
```

### 6.8 圖片連結

| 項目 | 說明 |
|---|---|
| 使用情境 | 讓圖片成為可點擊連結。 |
| 使用前提 | 圖片有合適的 `alt`。 |
| 來源 | §5 範例八、§7 |

```html
<a href="https://www.baidu.com">
  <img src="./assets/images/example.jpg" alt="前往百度">
</a>
```

### 6.9 動作按鈕

| 項目 | 說明 |
|---|---|
| 使用情境 | 觸發行為而不是前往 URL。 |
| 使用前提 | 互動本質不是導航。 |
| 來源 | §5 範例六、§7 |

```html
<button type="button" onclick="submitForm()">送出表單</button>
```

## 7. 錯誤速查表

| 錯誤現象 | 可能原因 | 修正方向 | 對應知識點 | 來源 |
|---|---|---|---|---|
| 連結文字只寫「點我」。 | 使用者與輔助工具無法從文字判斷目的。 | 改成「查看公司簡介」、「下載 PDF 講義」等具體文字。 | 連結文字、可訪問性 | §6、§7 |
| 點擊後找不到頁面或檔案。 | `href` 路徑、檔名或資料夾位置錯誤。 | 確認相對路徑、檔名與資料夾位置。 | `href`、URL 與路徑 | §4.1、§7 |
| 外部網站新分頁開啟但缺少 `rel`。 | 使用 `target="_blank"` 但沒有安全性補充。 | 加上 `rel="noopener noreferrer"`。 | `target`、`rel` | §4.2、§7 |
| 用 `<a>` 寫非導航動作。 | `<a>` 的語意是連結，不是一般按鈕。 | 不是前往某處時，優先使用 `<button type="button">`。 | `<a>` vs `<button>` | §5 範例六、§7 |
| 長期保留 `href=""`。 | 空連結可能重新載入目前頁面。 | 有明確目標再使用 `<a>`，暫時互動可改用 `button`。 | 空連結 | §5 範例五、§7 |
| 濫用 `href="#"`。 | 點擊後可能跳到頁首，或干擾歷史紀錄與捲動位置。 | 建立明確錨點，或改用適合的按鈕互動。 | 錨點、按鈕語意 | §5 範例四、§7 |
| `href="#xxx"` 無法跳到指定位置。 | 找不到對應的 `id="xxx"`。 | 確認片段名稱一致且不重複。 | 錨點連結、`id` | §5 範例四、§7 |
| `mailto:` 無法正常喚起郵件。 | URL scheme 格式錯誤，例如加入多餘空白。 | 使用 `mailto:user@example.com`。 | URL scheme | §5 範例七、§7 |
| `<a>` 中放另一個互動元素。 | 互動元素巢狀造成操作與可訪問性問題。 | 避免在 `<a>` 中放另一個 `<a>`、`button` 或表單控制項。 | 互動元素巢狀 | §4.3、§7、§11 |
| 圖片連結缺少合適 `alt`。 | 螢幕閱讀器無法理解連結目的。 | 讓 `alt` 描述連結目的或圖片內容。 | 圖片連結、可訪問性 | §5 範例八、§7 |

## 8. 相似概念對照表

### 8.1 `target="_self"` vs `target="_blank"`

| 比較項目 | `target="_self"` | `target="_blank"` |
|---|---|---|
| 定義 | 在目前頁面開啟。 | 在新分頁或新視窗開啟。 |
| 是否預設 | 是，未指定 `target` 時採用。 | 否，需要明確設定。 |
| 常見寫法 | `<a href="https://example.com">Example</a>` | `<a href="https://example.com" target="_blank" rel="noopener noreferrer">Example</a>` |
| 注意事項 | 會離開目前頁面。 | 連到外部網站時建議加上 `rel="noopener noreferrer"`。 |
| 來源 | §4.2、§5 範例一 | §4.2、§5 範例一、§7 |

### 8.2 `<a>` vs `<button>`

| 比較項目 | `<a>` | `<button>` |
|---|---|---|
| 定義 | 建立前往某個目的地的連結。 | 觸發動作的原生互動元素。 |
| 使用時機 | 前往頁面、檔案、錨點、電話、郵件或應用。 | 送出表單、顯示訊息等非導航動作。 |
| 常見寫法 | `<a href="./about.html">公司簡介</a>` | `<button type="button">送出表單</button>` |
| 注意事項 | 不應用來假裝按鈕。 | 不負責建立超鏈接。 |
| 常見錯誤 | `<a href="#" onclick="submitForm()">送出表單</a>` | 原筆記未列出 `button` 的錯誤寫法。 |
| 來源 | §5 範例六、§7、§8 | §5 範例六、§7、§8 |

### 8.3 `href=""` vs `href="#"` vs `href="#id"`

| 比較項目 | `href=""` | `href="#"` | `href="#id"` |
|---|---|---|---|
| 定義 | 空連結。 | 指向目前頁面的片段識別位置。 | 指向頁面中指定 `id` 的錨點。 |
| 常見效果 | 可能重新載入目前頁面。 | 通常跳到頁首。 | 捲動到對應 `id` 的元素。 |
| 適合情境 | 原筆記不建議長期保留。 | 回到頁首可見於範例，但不建議濫用。 | 章節目錄、FAQ、頁內導航。 |
| 注意事項 | 有明確目標再使用 `<a>`。 | 可能干擾歷史紀錄或捲動位置。 | `href` 名稱必須和目標 `id` 一致。 |
| 來源 | §5 範例五、§7 | §5 範例四、§7 | §5 範例四、§7 |

### 8.4 一般 URL vs URL scheme

| 比較項目 | 一般 URL / 路徑 | URL scheme |
|---|---|---|
| 定義 | 指向網站頁面、檔案或頁面片段。 | 以 `tel:`、`mailto:`、`line://` 等格式嘗試喚起應用。 |
| 使用時機 | 外部網站、內部頁面、PDF、圖片、錨點。 | 撥號、寄信或開啟特定應用。 |
| 常見寫法 | `<a href="./about.html">公司簡介</a>` | `<a href="tel:10010">電話聯繫</a>` |
| 注意事項 | 路徑與檔名需正確。 | 不保證所有環境都能成功喚起。 |
| 來源 | §4.1、§5 範例一至四 | §4.1、§5 範例七、§11 |

## 10. 維護備註

- 若 `notes/` 新增 `<a>` 的屬性、URL scheme、連結安全性說明或範例，需同步更新「屬性表」與「常用片段」。
- 若 `notes/` 後續明確教 DOM 方法、事件處理或 API，再新增「API / 方法表」；目前不從輔助 JavaScript 範例抽出 API 表。
- 若 `notes/` 新增 CLI、工具操作或設定檔內容，再補「指令表」或「設定檔範例」。
- 範例應維持最小可用，不把正式筆記改寫成另一份完整教學。
