# FAQ Index

## 用途

用常見問題快速找到本篇表單筆記中能回答的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| 表單在 HTML 裡主要做什麼？ | 表單用來收集使用者輸入，並把資料送出去。 | [一句話理解](../../../notes/測試/01-表單入門與form基本結構.md#一句話理解) |
| 什麼情境會需要表單？ | 註冊、登入、搜尋、留言、訂單等需要使用者輸入資料的頁面會需要表單。 | [問題情境](../../../notes/測試/01-表單入門與form基本結構.md#問題情境) |
| 一個完整表單通常有哪些部分？ | 通常包含表單域、表單控件與提示信息。 | [表單由哪幾個部分組成？](../../../notes/測試/01-表單入門與form基本結構.md#表單由哪幾個部分組成) |
| `<form>` 標籤的作用是什麼？ | `<form>` 定義哪些控件屬於同一份表單，提交時會送出範圍內資料。 | [form 標籤的作用](../../../notes/測試/01-表單入門與form基本結構.md#form-標籤的作用) |
| `action` 屬性做什麼？ | `action` 指定表單資料送出後要提交到哪個位址。 | [常用屬性](../../../notes/測試/01-表單入門與form基本結構.md#常用屬性) |
| `method` 屬性做什麼？ | `method` 指定提交方式，筆記列出 `get` 與 `post` 常見寫法。 | [常用屬性](../../../notes/測試/01-表單入門與form基本結構.md#常用屬性) |
| `name` 為什麼重要？ | `name` 能方便程式識別表單，也會影響提交資料時的欄位名稱。 | [重點整理](../../../notes/測試/01-表單入門與form基本結構.md#重點整理) |
| `target` 屬性做什麼？ | `target` 指定提交後結果在哪裡開啟。 | [常用屬性](../../../notes/測試/01-表單入門與form基本結構.md#常用屬性) |
| 輸入框放在 `<form>` 外面會怎樣？ | 它不會自然跟著這份表單一起送出。 | [把輸入框放在 form 外面](../../../notes/測試/01-表單入門與form基本結構.md#把輸入框放在-form-外面) |
| 初學表單應該先掌握哪些屬性？ | 可以先掌握 `action`、`method`、`name`，其他屬性再依需求補上。 | [實務判斷](../../../notes/測試/01-表單入門與form基本結構.md#實務判斷) |
