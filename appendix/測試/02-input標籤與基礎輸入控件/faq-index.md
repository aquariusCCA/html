# FAQ Index

## 用途

用常見問題快速找到本篇輸入控件筆記中能回答的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `<input>` 在表單裡做什麼？ | `<input>` 是表單中最常見的輸入控件，用來讓使用者輸入或提交資料。 | [一句話理解](../../../notes/測試/02-input標籤與基礎輸入控件.md#一句話理解) |
| `<input>` 需要結束標籤嗎？ | 不需要，`<input>` 是單標籤，不應寫成 `<input></input>`。 | [input 的基本語法](../../../notes/測試/02-input標籤與基礎輸入控件.md#input-的基本語法) |
| `type` 屬性做什麼？ | `type` 決定輸入控件類型，例如文字框、密碼框或隱藏欄位。 | [常用屬性](../../../notes/測試/02-input標籤與基礎輸入控件.md#常用屬性) |
| `name` 屬性為什麼重要？ | `name` 是提交資料時的欄位名稱，沒有它通常不會形成可辨識的提交資料。 | [常用屬性](../../../notes/測試/02-input標籤與基礎輸入控件.md#常用屬性) |
| `value` 和 `placeholder` 有什麼不同？ | `value` 是資料值或預設值；`placeholder` 是輸入前顯示的提示文字。 | [常用屬性](../../../notes/測試/02-input標籤與基礎輸入控件.md#常用屬性) |
| 可以用 `value` 當提示文字嗎？ | 不建議，因為 `value` 可能會被當成真正資料提交出去。 | [常見錯誤：把 value 當成提示文字](../../../notes/測試/02-input標籤與基礎輸入控件.md#常見錯誤把-value-當成提示文字) |
| `maxlength` 適合用在哪裡？ | `maxlength` 適合用來做簡單的最大字元數限制。 | [常用屬性](../../../notes/測試/02-input標籤與基礎輸入控件.md#常用屬性) |
| 什麼時候用 `type="text"`？ | 使用者需要輸入短文字時，例如帳號、姓名或搜尋關鍵字。 | [文本框](../../../notes/測試/02-input標籤與基礎輸入控件.md#文本框) |
| 什麼時候用 `type="password"`？ | 使用者需要輸入不希望直接顯示在畫面上的內容時使用。 | [密碼框](../../../notes/測試/02-input標籤與基礎輸入控件.md#密碼框) |
| 密碼框代表資料已經安全加密嗎？ | 不代表，密碼框只負責遮蔽畫面上的字元。 | [密碼框](../../../notes/測試/02-input標籤與基礎輸入控件.md#密碼框) |
| 什麼時候用 `type="hidden"`？ | 畫面不需要顯示，但提交時需要一起帶固定資料時使用。 | [hidden 隱藏域](../../../notes/測試/02-input標籤與基礎輸入控件.md#hidden-隱藏域) |
