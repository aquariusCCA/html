# mark 語意高亮速查

> 查表用。概念與用法說明見 notes/320-mark標籤/01-mark-標籤基本用法.md。

## 基本語意

| 項目 | 說明 |
| --- | --- |
| 標籤 | `<mark>` |
| 語意 | 標示目前上下文中需要讀者注意或參考的文字片段 |
| 標籤類型 | HTML5 新增的語意標籤 |
| 重點 | 不是單純把文字變成黃色，而是標記目前內容脈絡中的片段 |
| 預設呈現 | 大多數瀏覽器會顯示成黃色背景 |

## 常見使用情境

| 情境 | 是否適合 `<mark>` | 說明 |
| --- | --- | --- |
| 標示文章中的關鍵詞 | 適合 | 關鍵詞是在目前上下文中需要注意的片段 |
| 標示搜尋結果中命中的文字 | 適合 | 讓讀者更快看到搜尋命中的關鍵詞 |
| 標示目前正在討論的詞語 | 適合 | 這段文字在目前說明脈絡中被標記出來 |
| 標示需要讀者暫時注意的參考片段 | 適合 | 屬於目前上下文中的參考重點 |
| 只是想改變背景顏色 | 不適合 | 應使用 CSS 樣式 |
| 表示內容本身非常重要或有強烈重要性 | 不適合 | 通常更適合 `<strong>` |

## 基本片段

```html
<p>學習 HTML 時，先理解 <mark>標籤語意</mark> 會比只記語法更重要。</p>
```

## 樣式調整片段

```html
<p>這堂課會先介紹 <mark>HTML5 新增標籤</mark> 的用途。</p>
```

```css
mark {
  background-color: #ffe08a;
  color: #1f2933;
}
```

## mark strong CSS 差異

| 情境 | 建議標籤或做法 |
| --- | --- |
| 標示搜尋命中的文字 | `<mark>` |
| 標示目前句子中要讀者注意的關鍵詞 | `<mark>` |
| 表示內容本身非常重要或有強烈重要性 | `<strong>` |
| 只是想改變背景顏色，沒有語意上的標記需求 | CSS 樣式 |

## 無障礙注意

| 注意事項 | 說明 |
| --- | --- |
| 視覺標示 | `<mark>` 可以在視覺上標示重點 |
| 螢幕閱讀器 | 多數螢幕閱讀器在預設設定下不一定會特別宣告 `<mark>` 的開始與結束 |
| 重要資訊 | 重要資訊不應只靠黃色背景或 `<mark>` 本身傳達 |
| 文字脈絡 | 被標示內容對理解頁面非常關鍵時，文字本身應提供足夠脈絡 |

## 判斷準則

| 問題 | 判斷 | 建議 |
| --- | --- | --- |
| 這段文字是不是在目前上下文中被特別指出來？ | 是 | 可考慮 `<mark>` |
| 讀者看到標記後，是否能更快理解目前討論或搜尋命中的重點？ | 是 | 可考慮 `<mark>` |
| 如果拿掉背景顏色，這段標記在語意上是否仍然合理？ | 是 | 可考慮 `<mark>` |
| 只是想讓文字有顏色、背景或裝飾效果嗎？ | 是 | 應使用 CSS |
| 是要表示內容很重要嗎？ | 是 | 應考慮 `<strong>` |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 把 `<mark>` 當成黃底樣式工具 | 只看畫面效果，沒有確認語意是否成立 | 先看語意是否成立；只是促銷或視覺樣式時使用 CSS class |
| 用 `<mark>` 取代 `<strong>` | 重要警告的重點是內容很重要，不是目前上下文標記 | 使用 `<strong>` |
| 標記過多文字 | 讀者看不出真正需要注意的是哪裡 | 只標示關鍵片段 |
