## 3. 名詞表

| 名詞 | 英文 | 簡要定義 | 用途 | 注意事項 | 來源 |
|---|---|---|---|---|---|
| 布局標籤 | layout elements | 用來描述頁面不同區域角色的 HTML 元素。 | 表達頁首、導覽、主要內容、文章、側邊補充與頁尾等區域。 | 重點是語意，不是直接完成視覺排版。 | §1、§2、§3.1 |
| 語意化布局 | semantic layout | 根據內容角色選擇 HTML 元素的布局方式。 | 讓頁面結構更容易被人、瀏覽器、搜尋引擎與輔助工具理解。 | 不等於不用 CSS；視覺仍交給 CSS。 | §2、§3.1、§6 |
| 頁面主要內容 | main content | 目前頁面最核心、最需要閱讀或操作的內容。 | 使用 `main` 標示。 | 一個頁面通常只應有一個主要 `main`。 | §4.1、§4.4、§7 |
| 可獨立內容 | self-contained content | 離開目前頁面後仍能獨立理解的內容。 | 判斷是否適合使用 `article`。 | 不要把一般排版區塊誤用成 `article`。 | §3.3、§4.5、§7 |
| 主題區塊 | thematic section | 文件或文章中有明確主題的分段。 | 判斷是否適合使用 `section`。 | 通常應搭配清楚標題。 | §3.3、§4.6、§7 |
| 補充內容 | complementary content | 與主要內容相關，但不是主線內容的一部分。 | 使用 `aside` 放側邊欄、延伸閱讀、相關連結等。 | 不應取代主要內容。 | §4.7、§5 |
| 無語意容器 | generic container | 本身不表達具體內容角色的容器，例如 `div`。 | 沒有更合適元素時作為容器或純排版包裝。 | 有明確語意時，優先使用對應語意元素。 | §2、§4.6、§7 |
| DOM | Document Object Model | 瀏覽器根據 HTML 建立出的物件模型。 | CSS 與 JavaScript 多半基於 DOM 套用樣式或操作頁面。 | 原筆記只說明關聯，未教 DOM API。 | §8、§11 |

## 4. HTML 元素表

| 元素 | 用途 | 常用屬性 | 使用情境 | 最小範例 | 注意事項 | 來源 |
|---|---|---|---|---|---|---|
| `header` | 表示頁面或區塊的頭部內容。 | `class` | 網站頁首、文章頭部、區塊簡介。 | `<header><h1>網站名稱</h1></header>` | 可以用在整頁，也可以用在 `article` 或 `section` 內。 | §3.1、§4.1、§4.2、§5 |
| `nav` | 表示重要導覽連結區。 | `aria-label`、`class` | 主要導覽、活動導覽、頁尾導覽。 | `<nav><a href="/">首頁</a></nav>` | 不是所有連結都要放進 `nav`。 | §3.1、§4.1、§4.3、§5、§7 |
| `main` | 表示目前頁面的主要內容。 | `class` | 包住頁面核心內容。 | `<main>主要內容</main>` | 一個頁面通常只應有一個主要 `main`。 | §3.1、§4.1、§4.4、§5、§7 |
| `article` | 表示可獨立閱讀或發布的內容。 | `class` | 文章、新聞、評論、貼文、商品卡片。 | `<article><h2>文章標題</h2></article>` | 判斷重點是內容是否可獨立存在。 | §3.3、§4.1、§4.5、§5、§7 |
| `section` | 表示文件中的主題區塊。 | `class` | 文章小節、頁面功能區、章節段落。 | `<section><h2>主題</h2></section>` | 通常應有標題；純排版容器不適合用 `section`。 | §3.3、§4.1、§4.6、§5、§7 |
| `aside` | 表示與主要內容相關的補充內容。 | `class` | 側邊欄、延伸閱讀、相關連結、活動專區。 | `<aside><h2>延伸閱讀</h2></aside>` | 應與主要內容相關，但不是主線內容。 | §3.1、§4.1、§4.7、§5 |
| `footer` | 表示頁面或區塊的底部資訊。 | `class` | 版權、作者、聯絡方式、頁尾連結。 | `<footer><p>聯絡資訊</p></footer>` | 可以是整頁頁尾，也可以是 `article` 的底部資訊。 | §3.1、§4.1、§4.8、§5 |
| `hgroup` | 表示標題與相關副標題、替代標題或標語的群組。 | 無 | 標題與副標題需要組成一組時。 | `<hgroup><h1>標題</h1><p>副標題</p></hgroup>` | notes 標示為較少用，且規格歷史容易混淆；初學者可先掌握 `header` 搭配標題與段落。 | §4.1、§4.9、§11 |
| `div` | 無語意通用容器。 | `class` | 純排版容器，或沒有更精準語意元素時使用。 | `<div class="wrapper">...</div>` | 不應在有明確內容角色時取代語意化標籤。 | §2、§4.6、§7 |

## 9. 常用片段

### 9.1 基本頁面布局

| 項目 | 說明 |
|---|---|
| 使用情境 | 建立含頁首、導覽、主內容與頁尾的基本頁面。 |
| 使用前提 | 各區塊確實符合對應內容角色。 |
| 來源 | §3.2、§5 |

```html
<header>
  <h1>網站名稱</h1>
</header>

<nav aria-label="主要導覽">
  <a href="/">首頁</a>
  <a href="/products">商品</a>
</nav>

<main>
  <article>
    <h2>文章標題</h2>
    <p>文章主要內容。</p>
  </article>
</main>

<footer>
  <p>&copy; 2026 Example Site</p>
</footer>
```

### 注意事項

- `main` 包住頁面核心內容。
- 版面位置與視覺效果仍交給 CSS。

### 9.2 `article` 內包含多個 `section`

| 項目 | 說明 |
|---|---|
| 使用情境 | 一篇完整文章需要拆成多個主題分段。 |
| 使用前提 | 外層內容可獨立閱讀，內層分段有明確主題。 |
| 來源 | §3.3、§5 |

```html
<article>
  <h2>HTML 布局標籤介紹</h2>

  <section>
    <h3>頁首與導覽</h3>
    <p><code>header</code> 與 <code>nav</code> 可以描述網站上方的結構。</p>
  </section>

  <section>
    <h3>主要內容與側邊欄</h3>
    <p><code>main</code> 放主要內容，<code>aside</code> 放補充內容。</p>
  </section>
</article>
```

### 注意事項

- `article` 強調完整與獨立。
- `section` 強調文章中的主題分段。

### 9.3 側邊補充內容

| 項目 | 說明 |
|---|---|
| 使用情境 | 放置延伸閱讀、相關連結或活動專區。 |
| 使用前提 | 內容與主內容相關，但不是主線內容。 |
| 來源 | §4.7、§5 |

```html
<aside>
  <h2>活動專區</h2>
  <nav aria-label="活動導覽">
    <a href="/sale">秒殺專區</a>
    <a href="/coupon">領取優惠券</a>
  </nav>
</aside>
```

### 注意事項

- `aside` 可以包含補充導覽。
- 若這組連結屬於重要導覽，可使用 `nav` 並標示用途。

### 9.4 `hgroup` 標題群組

| 項目 | 說明 |
|---|---|
| 使用情境 | 標題與副標題、替代標題或標語需要被視為同一組。 |
| 使用前提 | 專案確實需要使用 `hgroup`，且已理解其規格歷史容易混淆。 |
| 來源 | §4.9、§11 |

```html
<hgroup>
  <h1>前端學習筆記</h1>
  <p>從 HTML 基礎到實務開發</p>
</hgroup>
```

### 注意事項

- notes 建議初學者通常先掌握 `header` 搭配標題與段落。
- 不要把較少用元素當成本章優先重點。

## 10. 錯誤速查表

| 錯誤現象 | 可能原因 | 修正方向 | 對應知識點 | 來源 |
|---|---|---|---|---|
| 所有區塊都寫成 `div`。 | 忽略 `div` 本身沒有語意。 | 有明確內容角色時，優先使用 `header`、`nav`、`main`、`article` 等元素。 | 語意化布局、無語意容器 | §2、§7 |
| 為了排版濫用 `section`。 | 把 `section` 當成一般容器。 | 沒有明確主題時使用 `div` 或調整結構。 | `section`、主題區塊 | §4.6、§7 |
| 混淆 `article` 與 `section`。 | 只看外觀分塊，沒有判斷內容是否可獨立。 | 可獨立閱讀用 `article`；文章內主題分段用 `section`。 | `article` vs `section` | §3.3、§7 |
| 每一組連結都使用 `nav`。 | 誤以為所有連結群組都是導覽。 | 只有主要導覽或重要連結群組才使用 `nav`。 | `nav`、導覽區 | §4.3、§7 |
| 忽略 `main`。 | 頁面主要內容沒有被明確標示。 | 使用 `main` 包住頁面核心內容。 | `main`、頁面主要內容 | §4.4、§7 |
| 同一頁放多個主要 `main`。 | 混淆主要內容與一般區塊。 | 一般頁面只保留一個主要 `main`。 | `main` | §4.1、§7 |
| 把標籤寫成 `< header >`。 | HTML 標籤名稱與角括號之間多了空白。 | 寫成 `<header>`、`<nav>`、`<footer>`。 | HTML 標籤基本寫法 | §7 |
| 認為布局標籤會自動完成排版。 | 混淆 HTML 與 CSS 的責任。 | HTML 描述語意，CSS 控制視覺呈現。 | HTML 與 CSS 分工 | §3.1、§8、§9 |

## 11. 相似概念對照表

### 11.1 `article` vs `section`

| 比較項目 | `article` | `section` |
|---|---|---|
| 定義 | 可獨立閱讀或重複發布的內容。 | 文件中的主題區塊。 |
| 使用時機 | 文章、新聞、評論、貼文、卡片型內容。 | 文章小節、頁面功能區、章節段落。 |
| 常見寫法 | `<article><h2>文章標題</h2></article>` | `<section><h2>主題</h2></section>` |
| 注意事項 | 重點是內容是否獨立完整。 | 通常應有清楚標題。 |
| 常見錯誤 | 把一般區塊都寫成 `article`。 | 把純排版容器寫成 `section`。 |

### 選用建議

- 內容離開目前頁面仍能獨立理解：優先考慮 `article`。
- 內容只是文章或頁面中的一個主題分段：優先考慮 `section`。
- 容器只是為了排版且沒有主題：使用 `div`。

### 11.2 語意化布局標籤 vs `div`

| 比較項目 | 語意化布局標籤 | `div` |
|---|---|---|
| 定義 | 直接描述內容角色的 HTML 元素。 | 無語意通用容器。 |
| 使用時機 | 內容角色明確，例如頁首、導覽、主要內容、文章、側邊欄、頁尾。 | 純排版容器，或沒有更精準語意元素時。 |
| 常見寫法 | `<header>...</header>`、`<main>...</main>` | `<div class="wrapper">...</div>` |
| 注意事項 | 不負責取代 CSS 排版。 | 需要靠 `class` 或上下文補充用途。 |
| 常見錯誤 | 把所有容器都硬改成語意標籤。 | 有明確語意時仍全部使用 `div`。 |

### 選用建議

- 有明確內容角色時，優先使用語意化布局標籤。
- 只有版面包裝需求時，`div` 仍然合適。

### 11.3 `nav` vs 一般連結群組

| 比較項目 | `nav` | 一般連結群組 |
|---|---|---|
| 定義 | 重要導覽連結區。 | 普通連結集合或內容中的參考連結。 |
| 使用時機 | 主要導覽、活動導覽、頁尾導覽等。 | 文章內參考連結、一般文字中的連結。 |
| 常見寫法 | `<nav aria-label="主要導覽">...</nav>` | `<p>參考 <a href="/docs">文件</a></p>` |
| 注意事項 | 多組導覽可用 `aria-label` 區分用途。 | 不必因為有連結就包成 `nav`。 |
| 常見錯誤 | 每一組連結都使用 `nav`。 | 重要導覽只用普通容器，角色不清楚。 |

### 選用建議

- 連結群組用於頁面或區域導覽：使用 `nav`。
- 只是內容中的普通連結：不需要使用 `nav`。

### 11.4 `main` vs `section`

| 比較項目 | `main` | `section` |
|---|---|---|
| 定義 | 頁面的主要內容。 | 文件中的主題區塊。 |
| 使用時機 | 包住目前頁面核心內容。 | 把頁面或文章分成幾個主題段落。 |
| 常見寫法 | `<main>主要內容</main>` | `<section><h2>主題</h2></section>` |
| 注意事項 | 一頁通常只應有一個主要 `main`。 | 一頁可有多個有主題的 `section`。 |
| 常見錯誤 | 忽略 `main` 或放多個主要 `main`。 | 把純排版容器寫成 `section`。 |

### 選用建議

- 需要標示整個頁面的核心內容：使用 `main`。
- 需要在核心內容中分出主題段落：使用 `section`。
