# progress 進度條速查

> 查表用。概念與用法說明見 notes/290-進度條標籤/01-progress-進度條標籤.md。

## 基本語意

| 項目 | 說明 |
| --- | --- |
| 標籤 | `<progress>` |
| 語意 | 表示一項任務目前完成到哪裡 |
| 適合情境 | 檔案上傳、檔案下載、表單送出、資料處理、大量資料載入 |
| 共同特徵 | 任務有目前進度與目標總量 |
| 不適合情境 | 視覺分隔線、裝飾線、評分條、普通數值條 |

## 基本結構

| 部分 | 寫法範例 | 用途 |
| --- | --- | --- |
| 文字說明 | `<span>當前進度: </span>` | 讓使用者知道進度條代表什麼 |
| 標籤 | `<progress>...</progress>` | 建立進度條 |
| 目前值 | `value="50"` | 表示目前完成的數值 |
| 目標值 | `max="100"` | 表示任務完成時的目標值 |
| 進度比例 | `value / max` | 瀏覽器依比例顯示進度 |

## 常用屬性

| 屬性 | 作用 | 規則 | 範例 |
| --- | --- | --- | --- |
| `max` | 指定進度完成時的目標值 | 必須是大於 `0` 的數值；未指定時最大值視為 `1` | `<progress value="3" max="10"></progress>` |
| `value` | 指定目前進度 | 必須是有效數值，且不能小於 `0`；有 `max` 時不應大於 `max` | `<progress value="75" max="100"></progress>` |

## 已知與未知進度

| 進度狀態 | 寫法 | 適用情境 |
| --- | --- | --- |
| 已知目前值與總目標 | `<progress value="40" max="100"></progress>` | 知道任務目前完成多少 |
| 不確定進度 | `<progress max="100"></progress>` | 只知道正在處理，但不知道完成比例 |

## 文字說明方式

| 寫法 | 範例 | 用途 |
| --- | --- | --- |
| 附近文字 | `<span>當前進度: </span>` | 讓使用者知道進度條代表的任務 |
| `label` | `<label for="upload-progress">上傳進度</label>` | 讓進度條用途更清楚 |
| ARIA 標籤 | ARIA 標籤 | 在正式介面中可用來說明進度條代表哪一項任務 |

## 適合與不適合情境

| 情境 | 是否適合 `<progress>` | 原因 |
| --- | --- | --- |
| 檔案上傳到一半 | 適合 | 表示任務完成進度 |
| 檔案下載 | 適合 | 表示已下載多少 |
| 表單送出或資料處理 | 適合 | 表示目前處理進度 |
| 載入大量資料 | 適合 | 表示已完成的比例 |
| 視覺分隔線 | 不適合 | 不是任務進度 |
| 裝飾線 | 不適合 | 只是視覺效果 |
| 評分條 | 不適合 | 不是任務完成進度 |
| 普通數值條 | 不適合 | 不是任務完成進度 |

## 判斷準則

| 問題 | 判斷 | 建議 |
| --- | --- | --- |
| 這個畫面是在表示一項任務的完成進度嗎？ | 是 | 可考慮 `<progress>` |
| 是否有目前值與目標值？ | 是 | 可使用 `value` 與 `max` |
| 是否只知道正在進行但比例未知？ | 是 | 可省略 `value` |
| 只是看起來像長條 UI 嗎？ | 是 | 不應只因外觀使用 `<progress>` |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 只放 `<progress value="80" max="100"></progress>`，沒有文字說明 | 使用者可能不知道進度條代表上傳、下載還是其他任務 | 搭配文字、`label` 或附近的說明文字 |
| `<progress value="120" max="100"></progress>` | `value` 大於 `max`，語意不合理 | 改成合理的目前值與目標值，例如 `<progress value="100" max="100"></progress>` |
| 用 `<progress>` 表示評分、溫度、庫存量等一般數值 | `<progress>` 適合表示任務進度，不適合表示一般數值 | 考慮其他更符合語意的做法 |
| 把所有長條狀畫面都寫成 `<progress>` | 外觀相似不代表語意相同 | 先確認是否真的是任務完成進度 |
