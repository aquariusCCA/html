# 屬性值與事件速查

> 查表內容依據 `notes/050-全局屬性/02-data-自訂資料屬性.md`、`notes/050-全局屬性/03-lang-dir-translate-語言方向與翻譯控制.md`、`notes/050-全局屬性/05-contenteditable-spellcheck-可編輯內容與拼字檢查.md`、`notes/050-全局屬性/06-tabindex-accesskey-鍵盤操作與導覽.md`、`notes/050-全局屬性/07-draggable-拖曳屬性與拖放事件.md`。

## data-* 與 dataset

| HTML 屬性 | JavaScript 讀取名稱 | 說明 |
| --- | --- | --- |
| `data-animal-type` | `dataset.animalType` | 去掉 `data-`，連字號後面的字母轉成大寫 |
| `data-food-type` | `dataset.foodType` | 自訂資料值本質上是字串 |
| `data-action` | `dataset.action` | 常用於按鈕操作類型 |
| `data-user-id` | `dataset.userId` | 常用於和元素直接相關的資料 ID |

| 規則 | 內容 |
| --- | --- |
| 固定前綴 | `data-` |
| 自訂名稱 | `animal-type`、`food-type` 等 |
| 命名限制 | `data-` 後面至少要有一個字元；屬性名稱不應包含大寫字母 |
| 常見讀取方式 | `element.dataset.name` 或對應的 camelCase 名稱 |
| 常見設定方式 | `setAttribute('data-food-type', 'fruit')` 或 `element.dataset.foodType = 'fruit'` |

## 語言、方向與翻譯

| 屬性 | 常見值/寫法 | 意義 | 實務建議 |
| --- | --- | --- | --- |
| `lang` | `zh-Hant` | 內容使用繁體中文 | 常放在 `<html>` 標示整份文件主要語言 |
| `lang` | `en` | 內容使用英文 | 局部外語內容可在該元素上另外設定 |
| `lang` | `ar` | 內容使用阿拉伯文 | 可搭配 `dir="rtl"` 標示文字方向 |
| `dir` | `ltr` | left to right，由左至右 | 適用由左至右文字方向 |
| `dir` | `rtl` | right to left，由右至左 | 適用內容語言本身由右至左的情境 |
| `dir` | `auto` | 由瀏覽器根據內容自動判斷 | 內容方向可能不固定時使用 |
| `translate` | `no` | 不翻譯元素內容 | 適合保護作品名、品牌名、程式碼術語或專有名詞 |

## contenteditable 與 spellcheck

| 屬性 | 值/寫法 | 意義 | 實務建議 |
| --- | --- | --- | --- |
| `contenteditable` | `true` | 使用者可以直接編輯元素內容 | 適合簡單備註區、可編輯標題等小範圍編輯 |
| `contenteditable` | `false` | 使用者不能直接編輯元素內容 | 用於明確關閉可編輯狀態 |
| `contenteditable` | `plaintext-only` | 只能編輯純文字內容，不啟用富文字格式 | 適合只需要純文字輸入的可編輯區塊 |
| `contenteditable` | 未設定或無效值 | 依父層可編輯狀態繼承 | 應避免靠無效值表達意圖 |
| `spellcheck` | `true` | 啟用瀏覽器拼字或語法檢查 | 適合自然語言輸入，例如留言 |
| `spellcheck` | `false` | 關閉瀏覽器拼字或語法檢查 | 適合帳號、產品代碼、程式碼片段等精確文字 |

## tabindex 與 accesskey

| 屬性 | 值/寫法 | 意義 | 實務建議 |
| --- | --- | --- | --- |
| `tabindex` | `-1` | 可被程式聚焦，但不會出現在一般 Tab 順序中 | 常用於跳轉焦點、對話框等情境 |
| `tabindex` | `0` | 加入一般 Tab 順序，順序依文件位置決定 | 非原生互動元素需要可聚焦時使用 |
| `tabindex` | 大於 `0` | 強制指定 Tab 順序 | 通常應避免 |
| `accesskey` | 單一快捷鍵字元，例如 `c` | 提供快速跳到元素或啟動元素的可能 | 只適合少量、明確、真的有必要的快捷操作 |

## draggable 與拖放事件

| 屬性/事件 | 值或觸發時機 | 說明 |
| --- | --- | --- |
| `draggable="true"` | 元素可拖動 | 一般元素若要拖曳，通常需要明確設定 |
| `draggable="false"` | 元素不可拖動 | 明確關閉拖曳 |
| `draggable="auto"` | 由瀏覽器決定 | 圖片與連結預設通常可拖動 |
| `dragstart` | 開始拖曳元素時 | 常用於設定拖曳開始狀態 |
| `drag` | 拖曳過程中 | 拖曳期間持續觸發 |
| `dragenter` | 拖曳元素進入某個目標時 | 拖放目標常會在此處處理接收狀態 |
| `dragover` | 拖曳元素停留在目標上方時 | 拖放目標通常需要在此呼叫 `preventDefault()` |
| `drop` | 在目標上放下元素時 | 放下後執行移動、複製或其他操作 |
| `dragend` | 拖曳結束時 | 常用於恢復拖曳元素狀態 |

## 常用程式處理

| 情境 | 必要處理 |
| --- | --- |
| 讀取 `data-animal-type` | 使用 `element.dataset.animalType` |
| 設定 `data-food-type` | 使用 `setAttribute('data-food-type', 'fruit')` 或 `element.dataset.foodType = 'fruit'` |
| 讓拖放目標可接收放下行為 | 在 `dragover` 中呼叫 `event.preventDefault()` |
| 儲存 `contenteditable` 修改結果 | 使用 JavaScript 讀取修改後文字，再送到後端或存到本地儲存 |
