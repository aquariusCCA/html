# 全局屬性用途速查

> 查表內容依據 `notes/050-全局屬性/01-id-class-style-元素識別分類與樣式.md`、`notes/050-全局屬性/02-data-自訂資料屬性.md`、`notes/050-全局屬性/03-lang-dir-translate-語言方向與翻譯控制.md`、`notes/050-全局屬性/04-hidden-title-顯示與補充資訊.md`、`notes/050-全局屬性/05-contenteditable-spellcheck-可編輯內容與拼字檢查.md`、`notes/050-全局屬性/06-tabindex-accesskey-鍵盤操作與導覽.md`、`notes/050-全局屬性/07-draggable-拖曳屬性與拖放事件.md`。

| 屬性 | 主要用途 | 適合情境 | 注意事項 |
| --- | --- | --- | --- |
| `id` | 標記文件中的唯一元素 | CSS、JavaScript、頁內錨點需要定位特定元素 | 同一份 HTML 文件中不應重複同一個 `id` |
| `class` | 標記同類元素 | 多個元素共用樣式或行為 | 一個元素可有多個 class，使用空格分隔 |
| `style` | 直接在元素上寫 CSS | 快速測試、少量一次性樣式、JavaScript 動態調整樣式 | 不適合大量取代樣式表，否則維護困難 |
| `data-*` | 在元素上存放少量自訂資料 | 商品 ID、操作類型、資料狀態、選單分類等給 JavaScript 讀取的資料 | 不應取代可見內容，也不適合塞入大量 JSON |
| `lang` | 說明元素內容使用的語言 | `<html>` 的主要語言、局部外語內容 | 會影響螢幕報讀器發音、搜尋引擎理解、翻譯與拼字檢查 |
| `dir` | 說明文字方向 | 阿拉伯文、希伯來文等由右至左內容，或需由瀏覽器自動判斷方向的內容 | `dir` 是文字方向，不是單純視覺靠右；排版對齊應交給 CSS |
| `translate` | 控制內容是否應被翻譯 | 作品名稱、品牌名稱、程式碼術語、專有名詞 | 少數專有名詞可用 `translate="no"`；整站禁止翻譯要謹慎 |
| `hidden` | 隱藏整個元素 | 尚未符合條件、之後可能由互動顯示的內容 | 重要資訊不應永遠隱藏；應有合理顯示時機或互動流程 |
| `title` | 補充元素額外資訊 | 縮寫全名、附加說明、簡短提示 | 重要警告或必要說明不應只放在 `title` |
| `contenteditable` | 讓元素內容可被使用者直接編輯 | 簡單備註區、可編輯標題、原型中的文字編輯區 | 只代表畫面可編輯，不會自動儲存資料 |
| `spellcheck` | 控制瀏覽器是否進行拼字或語法檢查 | 自然語言輸入、留言、可編輯內容 | 產品代碼、帳號、程式碼片段等精確文字通常不適合啟用 |
| `tabindex` | 控制元素能否被 Tab 聚焦與焦點順序 | 非原生互動元素需要聚焦，或程式需要控制焦點 | 正數 `tabindex` 容易破壞自然鍵盤導覽，通常應避免 |
| `accesskey` | 為元素提供鍵盤快捷鍵 | 少量、明確、真的有必要的快捷操作 | 觸發方式依瀏覽器與作業系統不同，且可能與既有快捷鍵衝突 |
| `draggable` | 控制元素是否可被拖起來 | 一般元素需要具備拖曳能力 | 只提供可拖曳能力；放到哪裡、放下後做什麼仍需 JavaScript 事件處理 |

## 常見決策

| 需求 | 優先考慮 |
| --- | --- |
| 唯一定位某個元素 | `id` |
| 多個元素共用樣式或行為 | `class` |
| 少量臨時樣式或程式動態樣式 | `style` |
| 把少量程式資料綁在元素上 | `data-*` |
| 標記整頁主要語言 | `<html lang="...">` |
| 標記局部外語或專有名詞 | `lang`、`translate="no"` |
| 暫時不顯示但之後可能顯示的內容 | `hidden` 搭配清楚的顯示流程 |
| 讓非原生互動元素加入自然 Tab 順序 | `tabindex="0"` |
| 讓元素可由程式聚焦但不進入一般 Tab 順序 | `tabindex="-1"` |
| 讓一般元素可被拖起來 | `draggable="true"` 並搭配拖放事件 |
