# 過時與不建議用法對照

> 查表內容依據 `notes/050-全局屬性/01-id-class-style-元素識別分類與樣式.md` 到 `notes/050-全局屬性/08-過時屬性-contextmenu-dropzone.md`。

## 常見誤用對照

| 情境 | 不建議寫法 | 建議做法 | 原因 |
| --- | --- | --- | --- |
| 多個元素共用同一分類 | 多個元素使用同一個 `id` | 改用 `class` | `id` 在同一份 HTML 文件中不應重複 |
| 多處使用同一套樣式 | 把所有 CSS 都寫在 `style` | 抽成 `class` 搭配 CSS | 大量行內樣式會分散在 HTML 中，後續維護困難 |
| 商品卡片、公告等可重複元件 | 用相同 `id` 表示可重複群組 | 用 `class="product-card"` 或類似類別 | 可重複分類應使用 `class` |
| 按鈕需要顯示文字 | 只寫 `data-label="刪除"`，元素內容留空 | 把「刪除」放在按鈕可見文字中，可另用 `data-action="delete"` | `data-*` 是給程式使用的資料，不應取代可見內容 |
| 複雜資料模型 | 把大量 JSON 塞進 `data-*` | 改由 JavaScript 狀態、API 回傳資料或資料模型管理 | `data-*` 適合少量、和元素直接相關的資料 |
| data 屬性命名 | 使用 `data-Animal-Type` | 使用 `data-animal-type` | 屬性名稱建議使用小寫與連字號 |
| 只想讓文字靠右 | 使用 `dir="rtl"` | 使用 CSS，例如 `text-align: right` | `dir` 是內容文字方向，不是單純排版對齊 |
| 專有名詞不應翻譯 | 忘記保護作品名、品牌名或產品名 | 在局部元素上使用 `translate="no"` | 避免翻譯工具改動不該翻譯的內容 |
| 必要說明或警告 | 永遠放在 `hidden` 中 | 直接可見，或提供清楚互動讓它出現 | 必要資訊不應無法取得 |
| 關鍵操作提示 | 只放在 `title` | 放在可見文字、確認對話框或鄰近說明 | 觸控、鍵盤或輔助工具使用者不一定能穩定取得 `title` |
| 重要主要內容 | 只放在 `title` | 放在頁面可見內容中 | `title` 適合簡短補充，不適合作為主要內容 |
| 可編輯內容 | 以為 `contenteditable` 會自動儲存 | 用 JavaScript 讀取內容並送到後端或本地儲存 | `contenteditable` 只控制畫面可否編輯 |
| 產品代碼或帳號輸入 | 啟用 `spellcheck="true"` | 考慮使用 `spellcheck="false"` | 拼字檢查可能干擾精確文字 |
| 完整文字編輯器 | 只依賴 `contenteditable` | 額外處理格式、貼上、快捷鍵、資料清理、儲存與安全性 | `contenteditable` 只是基本可編輯能力 |
| 鍵盤焦點順序 | 大量使用正數 `tabindex` | 讓 HTML 結構按合理順序排列，必要時用 `0` 或 `-1` | 正數容易讓焦點順序和視覺/文件順序不一致 |
| 普通文字段落 | 硬加 `tabindex="0"` | 不需要互動就不要加入 Tab 順序 | 無意義焦點會讓鍵盤使用者操作變慢 |
| 快捷鍵 | 假設 `accesskey="a"` 一定是 `Alt + A` | 認知觸發方式依瀏覽器與作業系統不同 | `accesskey` 的按法不是固定的 |
| 拖放功能 | 只加 `draggable="true"` | 搭配 Drag and Drop API 事件處理 | 可拖起來不代表能正確放置或移動資料 |
| 拖放目標 | 忘記在 `dragover` 中 `preventDefault()` | 在拖放目標的 `dragover` 中呼叫 `event.preventDefault()` | 很多情況下缺少此處理會讓 `drop` 不如預期 |
| 排序元件 | 把 `draggable` 當成完整排序功能 | 另外處理位置判斷、資料順序、鍵盤操作與儲存狀態 | `draggable` 只是底層拖曳能力 |

## 過時屬性對照

| 舊屬性 | 曾經用途 | 現代替代方向 | 注意事項 |
| --- | --- | --- | --- |
| `contextmenu` | 指定元素的上下文選單 | 監聽 JavaScript 的 `contextmenu` 事件 | 不應作為現代 HTML 的主要實作方式 |
| `dropzone` | 指定拖放資料的處理方式，例如複製、移動或建立連結 | 使用 Drag and Drop API 的 `dragenter`、`dragover`、`drop` 等事件 | 不應期待 `dropzone="copy"` 自動完成拖放功能 |

## contextmenu 事件使用提醒

| 做法 | 說明 |
| --- | --- |
| 監聽 `contextmenu` 事件 | 用於處理右鍵選單行為 |
| 呼叫 `event.preventDefault()` | 可阻止瀏覽器預設右鍵選單 |
| 提供替代操作 | 若關閉預設右鍵選單，應提供清楚的替代操作，避免使用者失去原本可用功能 |

## dropzone 替代方向

| 舊需求 | 現代處理 |
| --- | --- |
| 讓區域接收拖放 | 使用 Drag and Drop API 事件 |
| 拖曳元素進入目標區域 | 處理 `dragenter` |
| 拖曳元素停留在目標上方 | 處理 `dragover`，並視需要呼叫 `preventDefault()` |
| 在目標上放下元素 | 處理 `drop` |
| 支援不同使用者操作方式 | 除滑鼠拖放外，也思考鍵盤和輔助技術使用者的替代操作 |
