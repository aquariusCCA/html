# FAQ Index

## 用途

用具體問題快速找到本篇筆記中能回答的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `<div>` 和 `<span>` 是什麼？ | 它們都是沒有語意的盒子標籤；`<div>` 常包一大塊內容，`<span>` 常包一小段行內內容。 | [一句話理解](../../../notes/第12章_盒子標籤/01-div-與-span-盒子標籤.md#一句話理解) |
| `<div>` 和 `<span>` 有語意嗎？ | 沒有。它們本身不告訴瀏覽器或讀者這段內容是文章、導覽、強調或其他語意。 | [核心概念](../../../notes/第12章_盒子標籤/01-div-與-span-盒子標籤.md#核心概念) |
| 什麼時候適合使用 `<div>`？ | 當你需要包住一組相關內容，例如卡片、區塊、欄位群組或版面區域時，可以使用 `<div>`。 | [div：適合包住一塊內容](../../../notes/第12章_盒子標籤/01-div-與-span-盒子標籤.md#div適合包住一塊內容) |
| 什麼時候適合使用 `<span>`？ | 當你只需要包住文字中的一小段行內內容，而且不想打斷段落流動時，可以使用 `<span>`。 | [span：適合包住一小段行內內容](../../../notes/第12章_盒子標籤/01-div-與-span-盒子標籤.md#span適合包住一小段行內內容) |
| `<div>` 和 `<span>` 的預設顯示差在哪裡？ | `<div>` 預設是區塊顯示，通常另起一行；`<span>` 預設是行內顯示，可以和文字在同一行。 | [基本語法](../../../notes/第12章_盒子標籤/01-div-與-span-盒子標籤.md#基本語法) |
| CSS 可以改變 `<div>` 或 `<span>` 的顯示方式嗎？ | 可以。筆記提醒不要把「大盒子」與「小盒子」理解成不可改變的永久行為。 | [div 與 span 的比較](../../../notes/第12章_盒子標籤/01-div-與-span-盒子標籤.md#div-與-span-的比較) |
| 可以只為了文字換行就使用 `<div>` 嗎？ | 不建議。如果內容本質上是段落，應該優先使用能表達段落語意的標籤。 | [常見錯誤](../../../notes/第12章_盒子標籤/01-div-與-span-盒子標籤.md#常見錯誤) |
| `<span>` 可以包住標題和段落這種整塊結構嗎？ | 不適合。`<span>` 適合行內短內容，不適合包住標題、段落這類區塊結構。 | [常見錯誤](../../../notes/第12章_盒子標籤/01-div-與-span-盒子標籤.md#常見錯誤) |
| 有更明確的語意標籤時，還應該全部使用 `<div>` 嗎？ | 不應該。若內容有更明確的 HTML 語意標籤，應優先使用語意標籤。 | [實務判斷](../../../notes/第12章_盒子標籤/01-div-與-span-盒子標籤.md#實務判斷) |
