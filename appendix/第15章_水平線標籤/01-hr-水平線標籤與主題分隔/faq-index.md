# FAQ Index

## 用途

用常見問題快速找到本篇筆記中能回答疑問的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `<hr>` 的語意是什麼？ | `<hr>` 表示前後內容之間有主題上的分隔或轉換。 | [標籤語意](../../../notes/第15章_水平線標籤/01-hr-水平線標籤與主題分隔.md#標籤語意) |
| `<hr>` 需要寫 `</hr>` 嗎？ | 不需要。`<hr>` 是單標籤，不應寫成雙標籤，也不應包住其他內容。 | [基本語法](../../../notes/第15章_水平線標籤/01-hr-水平線標籤與主題分隔.md#基本語法) |
| `<hr />` 這種寫法可以理解嗎？ | 可以理解為 XHTML 風格或工具產生的寫法；在 HTML 中，`<hr>` 是最常見、最簡潔的寫法。 | [基本語法](../../../notes/第15章_水平線標籤/01-hr-水平線標籤與主題分隔.md#基本語法) |
| 只想畫一條水平線時，可以使用 `<hr>` 嗎？ | 不建議。若只是視覺裝飾或排版，應交給 CSS 處理。 | [常見錯誤](../../../notes/第15章_水平線標籤/01-hr-水平線標籤與主題分隔.md#常見錯誤) |
| CSS 改變 `<hr>` 外觀會改變它的語意嗎？ | 不會。CSS 可以調整寬度、顏色、粗細與間距，但 `<hr>` 在 HTML 中仍表示主題分隔。 | [瀏覽器預設樣式](../../../notes/第15章_水平線標籤/01-hr-水平線標籤與主題分隔.md#瀏覽器預設樣式) |
| 什麼情境適合使用 `<hr>`？ | 當文章、內容頁或表單前後區塊有明顯主題或資訊類型切換時，可以考慮使用。 | [實務使用建議](../../../notes/第15章_水平線標籤/01-hr-水平線標籤與主題分隔.md#實務使用建議) |
| 可以用大量 `<hr>` 取代標題或段落結構嗎？ | 不應該。清楚的小節應優先用標題、段落與合適的 HTML 結構建立。 | [常見錯誤](../../../notes/第15章_水平線標籤/01-hr-水平線標籤與主題分隔.md#常見錯誤) |
| 瀏覽器預設會怎麼顯示 `<hr>`？ | 通常會顯示成一條水平線，並在前後帶有一些間距。 | [瀏覽器預設樣式](../../../notes/第15章_水平線標籤/01-hr-水平線標籤與主題分隔.md#瀏覽器預設樣式) |
