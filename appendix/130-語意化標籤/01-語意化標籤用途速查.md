# 語意化標籤用途速查

> 查表用。概念與用法說明見 notes/130-語意化標籤/01-語意化標籤入門.md 與 notes/130-語意化標籤/02-div與語意化佈局比較.md。

## 常見語意化標籤

| 標籤 | 內容角色 | 適用情境 | 備註 |
| --- | --- | --- | --- |
| `<header>` | 頁面或區塊的開頭內容 | 頁面最上方的標題、Logo、介紹內容；頁面或文章開頭 | 可用於頁面或區塊 |
| `<nav>` | 導覽連結 | 主要選單、網站主要導覽 | 內容應是導覽連結 |
| `<main>` | 頁面的主要內容 | 文章或主要教學內容 | 通常一個頁面只會有一個主要的 `<main>` 區域 |
| `<article>` | 可獨立閱讀或分發的完整內容 | 文章、新聞、部落格貼文、單篇文章、卡片內容 | 一頁可以有多篇 `<article>` |
| `<section>` | 具有共同主題的一段內容 | 同一主題下的一段區塊、一段有標題的主題內容 | 通常會搭配標題 |
| `<aside>` | 與主要內容相關，但可以分離閱讀的補充內容 | 側邊推薦、補充說明、廣告 | 補充內容可和主要內容分離閱讀 |
| `<footer>` | 頁面或區塊的結尾資訊 | 版權資訊、聯絡方式、頁尾連結 | 可用於頁面或區塊 |
| `<div>` | 沒有語意的通用容器 | 排版容器、樣式包裝、沒有明確語意的分組；純粹為了套樣式的外層容器；包住一組按鈕方便排版 | 沒有明確語意需求時，繼續使用 `<div>` 是合理的 |

## div 與語意化佈局比較

| 面向 | `<div>` 佈局 | 語意化標籤佈局 |
| --- | --- | --- |
| 標籤本身是否有語意 | 沒有，主要靠 class 命名補充 | 有，標籤名稱直接表達內容角色 |
| 適合用途 | 排版容器、樣式包裝、沒有明確語意的分組 | 頁首、導覽、主要內容、文章、主題區塊、頁尾等 |
| 對維護者的幫助 | 需要看 class 與內容推測用途 | 看標籤就能先理解區塊角色 |
| 對輔助技術的幫助 | 較少 | 較容易提供導覽與內容結構 |
| 常見風險 | 整頁變成沒有語意的容器堆疊 | 誤把所有容器都換成語意標籤 |

## 標籤選擇情境

| 情境 | 建議標籤 | 原因 |
| --- | --- | --- |
| 網站主要導覽 | `<nav>` | 內容是導覽連結 |
| 頁面或文章開頭 | `<header>` | 內容是開頭資訊 |
| 一段有標題的主題內容 | `<section>` | 內容能形成一個主題區塊 |
| 單篇可獨立閱讀的文章 | `<article>` | 內容可獨立分發或重用 |
| 純粹為了套樣式的外層容器 | `<div>` | 沒有特別內容語意 |
| 包住一組按鈕方便排版 | `<div>` | 只是分組與排版需求 |
