# 第18章_音頻標籤 - 附錄查表資料

## 1. 附錄定位

### 適用範圍

本附錄對應 `notes/第18章_音頻標籤/音頻標籤.md`，範圍聚焦於 HTML `<audio>` 音訊元素、`<source>` 替代來源、`src`、`controls`、`autoplay`、`loop`、`preload`、`type` 等常見屬性、最小可用範例、常見錯誤，以及 `<audio>` 與 `<video>`、JavaScript、CSS、可訪問性和檔案路徑的關係。

## 9. 常用片段

### 9.1 基本音訊播放器

| 項目 | 說明 |
|---|---|
| 使用情境 | 在頁面中嵌入單一音訊檔案，並讓使用者操作播放。 |
| 使用前提 | 音訊路徑正確，且不需要提供多種格式。 |
| 來源 | §3、§5.1 |

```html
<audio src="./assets/files/music.mp3" controls></audio>
```

### 注意事項

- `src` 指向音訊檔案。
- `controls` 顯示瀏覽器內建控制列。
- 沒有自訂播放器時，不建議省略 `controls`。

### 9.2 多格式音訊來源

| 項目 | 說明 |
|---|---|
| 使用情境 | 需要提供多個候選格式，讓瀏覽器選擇可播放版本。 |
| 使用前提 | 已準備多個音訊格式檔案。 |
| 來源 | §4.5、§5.2 |

```html
<audio controls>
  <source src="./assets/files/music.mp3" type="audio/mpeg">
  <source src="./assets/files/music.ogg" type="audio/ogg">
  您的瀏覽器不支援 audio 標籤。
</audio>
```

### 注意事項

- `<source>` 放在 `<audio>` 內。
- 瀏覽器會從候選來源中選擇支援格式。
- 備用文字用於不支援 `<audio>` 的環境。

### 9.3 循環播放音訊

| 項目 | 說明 |
|---|---|
| 使用情境 | 音訊結束後需要自動從頭播放。 |
| 使用前提 | 使用者可以停止或關閉聲音。 |
| 來源 | §4.4、§5.3 |

```html
<audio src="./assets/files/background.mp3" controls loop></audio>
```

### 注意事項

- `loop` 會讓音訊播放結束後重新播放。
- 背景音或循環音效應避免打擾使用者。

### 9.4 避免依賴自動播放

| 項目 | 說明 |
|---|---|
| 使用情境 | 需要穩定提供音訊播放入口。 |
| 使用前提 | 音訊不應強制在頁面載入時播放。 |
| 來源 | §4.3、§5 |

```html
<audio src="./assets/files/intro.mp3" controls></audio>
```

### 注意事項

- 原筆記指出 `autoplay` 可能被瀏覽器限制。
- 讓使用者主動播放通常更穩定。

## 10. 錯誤速查表

| 錯誤現象 | 可能原因 | 修正方向 | 對應知識點 | 來源 |
|---|---|---|---|---|
| 音訊無法播放。 | `src` 路徑寫錯，瀏覽器找不到檔案。 | 確認 `src` 路徑與音訊檔案位置一致。 | `src`、檔案路徑 | §3、§7、§8 |
| 使用者看不到播放按鈕。 | 忘記加上 `controls`，也沒有自訂播放器。 | 在 `<audio>` 上加入 `controls`。 | `controls`、播放控制列 | §3、§4.2、§7 |
| 自動播放沒有成功。 | 瀏覽器可能阻擋有聲音的自動播放。 | 讓使用者主動播放，或設計明確播放按鈕。 | `autoplay`、使用者互動 | §4.3、§5、§7 |
| 某些瀏覽器或環境無法播放音訊。 | 只提供單一格式，該環境可能不支援。 | 使用多個 `<source>` 提供替代格式。 | `<source>`、音訊格式 | §4.5、§5.2、§7 |
| 背景音無法停止。 | 使用者缺少控制權。 | 提供 `controls` 或自訂停止按鈕。 | `controls`、`loop` | §4.4、§7 |
| 重要資訊只存在音訊中。 | 無法聽音訊或無法播放音訊的使用者會失去資訊。 | 提供文字稿、摘要或替代說明。 | 可訪問性、文字替代 | §6、§7、§9 |
| 瀏覽器無法正確判斷音訊格式。 | MIME type 可能填寫錯誤。 | 依檔案格式填寫正確的 `type`。 | `type`、MIME type | §4.5、§7、§11.1 |
| 把 `autoplay` 當成必要播放入口。 | 誤以為自動播放一定成功。 | 保留可操作的播放介面，避免只依賴自動播放。 | `autoplay`、`controls` | §5、§7 |

## 11. 相似概念對照表

### 11.1 `<audio src>` vs `<audio>` 內的 `<source>`

| 比較項目 | `<audio src>` | `<audio>` 內的 `<source>` |
|---|---|---|
| 定義 | 在 `<audio>` 上直接指定單一音訊檔案。 | 在 `<audio>` 內提供一個或多個候選音訊來源。 |
| 使用時機 | 只有一個音訊來源時。 | 需要提供多種格式提升播放成功機會時。 |
| 常見寫法 | `<audio src="./assets/files/music.mp3" controls></audio>` | `<source src="./assets/files/music.mp3" type="audio/mpeg">` |
| 注意事項 | 簡單直接，但缺少替代格式。 | 建議填寫 `type`，讓瀏覽器判斷格式。 |
| 常見錯誤 | 路徑寫錯導致無法播放。 | 只寫 `<source>` 但沒有放在 `<audio>` 中。 |

### 選用建議

- 只需要單一音訊檔時，可直接使用 `<audio src>`.
- 需要兼顧不同格式時，使用 `<audio>` 加多個 `<source>`。

### 11.2 `autoplay` vs 使用者主動播放

| 比較項目 | `autoplay` | 使用者主動播放 |
|---|---|---|
| 定義 | 希望頁面載入後自動播放音訊。 | 使用者透過控制列或按鈕啟動播放。 |
| 使用時機 | 原筆記只說明可嘗試自動播放。 | 一般內容頁或需要穩定播放入口時。 |
| 常見寫法 | `<audio src="./assets/files/music.mp3" controls autoplay></audio>` | `<audio src="./assets/files/music.mp3" controls></audio>` |
| 注意事項 | 可能被瀏覽器阻擋。 | 較符合使用者控制權。 |
| 常見錯誤 | 假設自動播放一定成功。 | 忘記提供明確播放入口。 |

### 選用建議

- 不要把 `autoplay` 當成唯一播放方式。
- 優先提供 `controls`，讓使用者主動播放。

### 11.3 `<audio>` vs `<video>`

| 比較項目 | `<audio>` | `<video>` |
|---|---|---|
| 定義 | 嵌入音訊內容。 | 嵌入同時包含影像與聲音的內容。 |
| 使用時機 | 背景音、課程錄音、播客、音效。 | 需要影像畫面與聲音時。 |
| 常見寫法 | `<audio src="./assets/files/music.mp3" controls></audio>` | 原筆記未提供 `<video>` 最小範例。 |
| 注意事項 | 只處理聲音。 | 常需要額外考慮尺寸、封面圖、字幕與畫面比例。 |
| 常見錯誤 | 忽略文字替代與控制權。 | 原筆記未展開 `<video>` 常見錯誤。 |

### 選用建議

- 只需要聲音內容時，使用 `<audio>`。
- 需要影像與聲音時，使用 `<video>`，並另行處理影片相關細節。

### 11.4 `controls` vs 自訂播放器

| 比較項目 | `controls` | 自訂播放器 |
|---|---|---|
| 定義 | 使用瀏覽器內建播放控制列。 | 用 JavaScript 自行控制播放介面。 |
| 使用時機 | 初學範例或不需要特殊介面時。 | 需要自訂播放、暫停、進度或音量互動時。 |
| 常見寫法 | `<audio src="./assets/files/music.mp3" controls></audio>` | 原筆記未提供自訂播放器程式碼。 |
| 注意事項 | 介面外觀由瀏覽器決定。 | 原筆記未明確教相關 API，不應在本附錄擴寫。 |
| 常見錯誤 | 忘記加 `controls` 導致使用者無法操作。 | 沒有完整處理播放控制與可訪問性。 |

### 選用建議

- 沒有自訂播放器時，使用 `controls`。
- 若要自訂播放器，應回到 JavaScript 與 DOM API 主題另行學習。
