# FAQ Index

## 用途

用常見問題快速定位本篇筆記中能回答拖放互動屬性疑問的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `draggable` 屬性做什麼？ | `draggable` 規定元素是否可被拖動。 | [`draggable`：指定元素是否可拖動](../../../notes/第05章_全局屬性/06-拖放互動屬性.md#draggable指定元素是否可拖動) |
| `draggable` 有哪些常見值？ | 常見值包含 `true`、`false` 與 `auto`，分別表示可拖動、不可拖動與由瀏覽器判斷。 | [`draggable`：指定元素是否可拖動](../../../notes/第05章_全局屬性/06-拖放互動屬性.md#draggable指定元素是否可拖動) |
| 哪些元素預設可拖動？ | 鏈接與圖像預設可拖動；其他元素通常需要明確設定 `draggable="true"`。 | [`draggable`：指定元素是否可拖動](../../../notes/第05章_全局屬性/06-拖放互動屬性.md#draggable指定元素是否可拖動) |
| 只加 `draggable="true"` 就會自動完成拖放功能嗎？ | 不會。它只表示元素可被拖動，放下後排序、移動或儲存仍需要 JavaScript。 | [只加 `draggable` 卻期待完整功能](../../../notes/第05章_全局屬性/06-拖放互動屬性.md#只加-draggable-卻期待完整功能) |
| 為什麼 `dragover` 裡要呼叫 `event.preventDefault()`？ | 範例中在 `dragover` 使用它，讓目標元素可以接收 `drop`。 | [範例拆解](../../../notes/第05章_全局屬性/06-拖放互動屬性.md#範例拆解) |
| `ondragstart` 和 `ondragend` 可以做什麼？ | 它們分別在拖曳開始與結束時觸發，可用來改變或恢復元素狀態。 | [範例拆解](../../../notes/第05章_全局屬性/06-拖放互動屬性.md#範例拆解) |
| `dropzone` 還建議在現代 HTML 使用嗎？ | 不建議。`dropzone` 已屬於 obsolete 屬性，現代拖放互動通常改用 Drag and Drop API 與 JavaScript 事件。 | [`dropzone`：已棄用的拖放屬性](../../../notes/第05章_全局屬性/06-拖放互動屬性.md#dropzone已棄用的拖放屬性) |
| 拖放互動需要考慮可及性嗎？ | 需要。不是每個使用者都能方便拖曳，應提供上移、下移、選單或鍵盤可操作的替代方式。 | [實務注意事項](../../../notes/第05章_全局屬性/06-拖放互動屬性.md#實務注意事項) |
