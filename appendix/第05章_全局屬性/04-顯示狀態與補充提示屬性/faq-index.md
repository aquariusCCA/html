# FAQ Index

## 用途

用具體問題快速找到本篇筆記中能回答的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `hidden` 屬性做什麼？ | 它表示元素目前不應被呈現，最常見用法是直接寫上 `hidden` 讓元素不顯示。 | [`hidden`：隱藏元素](../../../notes/第05章_全局屬性/04-顯示狀態與補充提示屬性.md#hidden隱藏元素) |
| `<p hidden>` 代表什麼？ | 這是 `hidden` 的常見簡寫方式，代表該段落處於一般隱藏狀態。 | [`hidden`：隱藏元素](../../../notes/第05章_全局屬性/04-顯示狀態與補充提示屬性.md#hidden隱藏元素) |
| `hidden="until-found"` 代表什麼？ | 內容平時隱藏，但仍可被頁面搜尋或片段連結揭示。 | [`hidden`：隱藏元素](../../../notes/第05章_全局屬性/04-顯示狀態與補充提示屬性.md#hidden隱藏元素) |
| 如何用 JavaScript 顯示 `hidden` 元素？ | 可以依條件把元素的 `hidden` 狀態設為 `false`，讓原本隱藏的元素顯示。 | [用 JavaScript 顯示 hidden 元素](../../../notes/第05章_全局屬性/04-顯示狀態與補充提示屬性.md#用-javascript-顯示-hidden-元素) |
| `title` 屬性做什麼？ | 它提供元素的額外資訊，通常會在滑鼠移到元素上時顯示為工具提示文字。 | [`title`：提供額外提示資訊](../../../notes/第05章_全局屬性/04-顯示狀態與補充提示屬性.md#title提供額外提示資訊) |
| `title` 適合放重要資訊嗎？ | 不適合。重要內容應直接顯示在畫面上，或使用更明確的說明文字。 | [`title` 的實務注意事項](../../../notes/第05章_全局屬性/04-顯示狀態與補充提示屬性.md#title-的實務注意事項) |
| 為什麼不能只依賴 `title` 提示？ | 觸控裝置、鍵盤操作、輔助科技與使用者認知都可能讓提示不穩定或看不到。 | [`title` 的實務注意事項](../../../notes/第05章_全局屬性/04-顯示狀態與補充提示屬性.md#title-的實務注意事項) |
| 刪除後無法復原這類警告可以只放在 `title` 嗎？ | 不建議。這類重要警告應直接顯示，讓使用者不用依賴提示文字。 | [把必要說明只放在 `title`](../../../notes/第05章_全局屬性/04-顯示狀態與補充提示屬性.md#把必要說明只放在-title) |
| `hidden` 適合只用來切換視覺裝飾嗎？ | 通常不適合。只切換視覺樣式時應考慮 CSS class，內容真的不應顯示時再用 `hidden`。 | [用 `hidden` 只做視覺裝飾](../../../notes/第05章_全局屬性/04-顯示狀態與補充提示屬性.md#用-hidden-只做視覺裝飾) |
| `hidden` 和 `title` 差在哪？ | `hidden` 處理內容是否顯示，`title` 處理元素是否有額外補充說明。 | [一句話理解](../../../notes/第05章_全局屬性/04-顯示狀態與補充提示屬性.md#一句話理解) |
