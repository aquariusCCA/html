# FAQ Index

## 用途

用常見問題快速找到本篇筆記中能回答鍵盤快捷鍵與焦點順序疑問的段落。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `accesskey` 屬性做什麼？ | `accesskey` 可以替元素設定一個或多個鍵盤快捷方式，讓使用者快速訪問常用元素。 | [`accesskey`：設定訪問快捷鍵](../../../notes/第05章_全局屬性/05-鍵盤操作與焦點順序屬性.md#accesskey設定訪問快捷鍵) |
| `accesskey` 的啟用方式在每個瀏覽器都一樣嗎？ | 不一定；它會依作業系統與瀏覽器而異。 | [`accesskey`：設定訪問快捷鍵](../../../notes/第05章_全局屬性/05-鍵盤操作與焦點順序屬性.md#accesskey設定訪問快捷鍵) |
| 為什麼不建議大量使用 `accesskey`？ | 快捷鍵太多難以記憶，也可能和瀏覽器、作業系統或輔助科技快捷鍵衝突。 | [`accesskey` 的使用風險](../../../notes/第05章_全局屬性/05-鍵盤操作與焦點順序屬性.md#accesskey-的使用風險) |
| `tabindex` 屬性做什麼？ | `tabindex` 規定使用 Tab 鍵導航時，元素如何進入焦點順序。 | [`tabindex`：控制 Tab 焦點順序](../../../notes/第05章_全局屬性/05-鍵盤操作與焦點順序屬性.md#tabindex控制-tab-焦點順序) |
| `tabindex="0"` 何時使用？ | 它會讓元素加入自然 Tab 順序，可用於需要聚焦的自訂互動元素。 | [常見 `tabindex` 值](../../../notes/第05章_全局屬性/05-鍵盤操作與焦點順序屬性.md#常見-tabindex-值) |
| `tabindex="-1"` 表示什麼？ | 它不讓元素進入 Tab 順序，但可用程式聚焦，常用於跳轉焦點或錯誤提示。 | [常見 `tabindex` 值](../../../notes/第05章_全局屬性/05-鍵盤操作與焦點順序屬性.md#常見-tabindex-值) |
| 為什麼通常要避免大於 `0` 的 `tabindex`？ | 正整數 `tabindex` 會讓鍵盤順序脫離 DOM 內容順序，維護成本高，也容易讓使用者迷失。 | [用正整數 `tabindex` 重排整個頁面](../../../notes/第05章_全局屬性/05-鍵盤操作與焦點順序屬性.md#用正整數-tabindex-重排整個頁面) |
| 表單順序不自然時，應該用正整數 `tabindex` 修補嗎？ | 不應優先這樣做；較好的做法是調整 HTML 順序，讓 DOM 順序符合操作順序。 | [較好的焦點順序寫法](../../../notes/第05章_全局屬性/05-鍵盤操作與焦點順序屬性.md#較好的焦點順序寫法) |
