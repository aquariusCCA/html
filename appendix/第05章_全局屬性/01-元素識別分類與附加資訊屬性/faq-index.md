# FAQ Index

## 用途

用常見問題快速找到本篇筆記中能回答的屬性用途、選用判斷與常見錯誤。

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `id`、`class`、`data-*` 和 `style` 最大差異是什麼？ | `id` 用於唯一識別，`class` 用於分類，`data-*` 用於自訂資料，`style` 用於行內 CSS。 | [一句話理解](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#一句話理解) |
| 同一份 HTML 文件中可以重複使用同一個 `id` 嗎？ | 不應該。同一份 HTML 文件中，某個 `id` 值應該只出現一次。 | [`id`：元素的唯一識別](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#id元素的唯一識別) |
| `id` 可以拿來做頁面內錨點嗎？ | 可以，像 `href="#pricing"` 會連到同頁中 `id="pricing"` 的位置。 | [`id`：元素的唯一識別](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#id元素的唯一識別) |
| 一個元素可以有多個 `class` 嗎？ | 可以，多個 `class` 使用空格分隔，代表同一元素同時屬於多個分類。 | [`class`：元素的分類名稱](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#class元素的分類名稱) |
| 多個元素可以使用同一個 `class` 嗎？ | 可以，`class` 適合重複用在多個元素上，讓它們共用樣式或被一起選取。 | [`class`：元素的分類名稱](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#class元素的分類名稱) |
| `data-*` 屬性會自動改變畫面或產生互動嗎？ | 不會。它的價值在於 JavaScript 可以從 DOM 中讀取這些資訊。 | [`data-*`：元素上的自訂資料](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#data-元素上的自訂資料) |
| `data-animal-type` 在 `dataset` 中會變成什麼？ | 在 JavaScript 的 `dataset` 中，`data-animal-type` 會變成 `animalType`。 | [`data-*`：元素上的自訂資料](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#data-元素上的自訂資料) |
| `data-*` 命名要注意什麼？ | 名稱必須以 `data-` 開頭，後面至少要有一個字元，且不應包含大寫字母。 | [`data-*`：元素上的自訂資料](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#data-元素上的自訂資料) |
| `style` 適合作為主要樣式管理方式嗎？ | 不適合。多個元素需要共用樣式時，通常應該用 `class` 搭配 CSS。 | [`style`：直接寫在元素上的 CSS](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#style直接寫在元素上的-css) |
| 行內 `style` 一定比所有樣式都優先嗎？ | 不一定。一般情況下行內樣式優先於普通作者樣式，但仍會受到 cascade、`!important`、使用者樣式與瀏覽器規則影響。 | [`style`：直接寫在元素上的 CSS](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#style直接寫在元素上的-css) |
| 如果資料要給 JavaScript 使用，應該放在 `class` 還是 `data-*`？ | 應使用 `data-*`，不要把商品編號或庫存狀態這類資料塞進 `class` 名稱。 | [把資料塞進 class 名稱](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#把資料塞進-class-名稱) |
| 多個提醒元素用了同一個 `id`，應該怎麼改？ | 如果它們是一組提醒，應改用同一個 `class`。 | [把多個元素設成同一個 `id`](../../../notes/第05章_全局屬性/01-元素識別分類與附加資訊屬性.md#把多個元素設成同一個-id) |
