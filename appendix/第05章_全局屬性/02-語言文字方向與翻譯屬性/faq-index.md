# FAQ Index

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `lang` 屬性做什麼？ | `lang` 用來標示元素內容使用哪種語言。 | [`lang`：標示元素內容語言](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#lang標示元素內容語言) |
| `lang` 應該放在哪裡？ | 最常見做法是先在 `<html>` 上設定整份文件語言，再針對局部不同語言的片段另外設定。 | [`lang`：標示元素內容語言](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#lang標示元素內容語言) |
| `lang` 的值應該怎麼寫？ | `lang` 的屬性值應使用有效的 BCP 47 語言標籤，例如 `en`、`en-US`、`zh-TW`。 | [`lang`：標示元素內容語言](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#lang標示元素內容語言) |
| `dir` 屬性做什麼？ | `dir` 用來規定元素內容的文字方向。 | [`dir`：控制文字方向](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#dir控制文字方向) |
| `dir` 有哪些常見值？ | 常見值包含 `ltr`、`rtl`、`auto`，分別代表由左到右、由右到左、由瀏覽器判斷方向。 | [`dir`：控制文字方向](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#dir控制文字方向) |
| 什麼時候適合使用 `dir="rtl"`？ | 內容需要從右到左書寫時可以使用，例如筆記中的阿拉伯文範例。 | [`dir`：控制文字方向](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#dir控制文字方向) |
| `translate` 屬性做什麼？ | `translate` 用來規定元素內容是否應該被翻譯。 | [`translate`：控制內容是否被翻譯](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#translate控制內容是否被翻譯) |
| 什麼內容適合使用 `translate="no"`？ | 品牌名稱、作品名稱、程式碼、專有名詞或不應被機器翻譯的文字，適合使用 `translate="no"`。 | [`translate`：控制內容是否被翻譯](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#translate控制內容是否被翻譯) |
| 可以把整份文件都設成 `translate="no"` 嗎？ | 可以放在 `<html>` 上，但筆記提醒實務上通常不會隨意禁止全站翻譯。 | [`translate`：控制內容是否被翻譯](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#translate控制內容是否被翻譯) |
| `lang`、`dir`、`translate` 可以一起使用嗎？ | 可以，筆記示範同時設定整份文件語言、英文書名不翻譯，以及阿拉伯文由右到左排列。 | [三者如何一起使用](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#三者如何一起使用) |
| 為什麼不應完全不設定文件語言？ | 文件語言可以幫助瀏覽器、搜尋引擎、翻譯工具與輔助科技更正確地處理內容。 | [完全不設定文件語言](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#完全不設定文件語言) |
| 可以只用 CSS 處理文字方向嗎？ | 如果問題是文字方向，不要只用 CSS 做表面排列，應該讓 HTML 也用 `dir` 表達方向。 | [用 CSS 取代語意屬性](../../../notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md#用-css-取代語意屬性) |
