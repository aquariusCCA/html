# 內容模型與巢狀規則速查

> 查表內容依據 `notes/070-HTML塊級元素與行內元素/02-HTML元素的內容模型與基本巢狀規則.md` 與 `notes/070-HTML塊級元素與行內元素/03-常見錯誤巢狀：標題p與a標籤.md`。

## 基本判斷

| 觀念 | 重點 |
| --- | --- |
| block / inline | 幫助理解預設排版，但不能取代合法巢狀判斷 |
| content model | 判斷元素可以放什麼內容的核心依據 |
| flow content | 可用 `div` 這類通用區塊容器建立直覺，常包含文字、行內元素與其他區塊元素 |
| phrasing content | 可用 `span` 這類行內容器建立直覺，通常是文字、`em`、`strong`、`a`、`input` 等 |
| 瀏覽器容錯 | 瀏覽器能顯示或自動修正錯誤 HTML，不代表原始結構正確 |

## div 與 span 容器直覺

| 元素 | 常見角色 | 可放內容的直覺 | 注意事項 |
| --- | --- | --- | --- |
| `div` | 通用區塊容器 | 可包含多數 flow content，例如文字、行內元素、其他區塊元素 | `div` 的自由度不能套用到所有塊級元素，`p`、`h1` 到 `h6` 等仍有自己的限制 |
| `span` | 通用行內容器 | 通常只應包含 phrasing content，例如文字、`em`、`strong`、`a`、`input` 等 | 不適合包住 `div` 這類區塊內容 |

## 巢狀判斷流程

| 步驟 | 問題 |
| --- | --- |
| 1 | 這個標籤的語意是否適合目前內容？ |
| 2 | 這個標籤的內容模型是否允許放入目前的子元素？ |
| 3 | 如果只是為了排版或分組，是否應改用 `div` 或 `span`，再用 CSS 控制外觀？ |
| 4 | 如果遇到 `p`、標題、連結、表單控制項等常見限制元素，是否已特別檢查巢狀是否合法？ |

## 常見錯誤巢狀

| 情境 | 錯誤寫法 | 建議判斷 | 原因/風險 |
| --- | --- | --- | --- |
| 大標題裡放小標題 | `h1` 包 `h2` | 標題應各自獨立，靠文件順序與層級表達關係 | `h1` 到 `h6` 都是標題元素，不應互相嵌套；瀏覽器可能自動結束前面的標題元素 |
| 段落裡放區塊 | `p` 包 `div` | 用 `div` 包 `p`，或拆成多個段落與區塊 | `p` 適合段落文字與 phrasing content，不適合包住 `div` 這類區塊內容；瀏覽器通常會自動提前結束 `p` |
| 連結裡再放連結 | `a` 包 `a` | 不要巢狀互動元素，重新拆分可點擊區域 | 互動目標會混亂，也不符合 HTML 規範 |
| 整張卡片都放進 `a`，內部還有按鈕或表單欄位 | `a` 包住 `button`、表單控制項或帶 `tabindex` 的後代元素 | 只讓標題或圖片成為連結，或把外層改成非連結容器 | `a` 雖可在某些情況包區塊內容，但不能無限制包含互動內容 |

## a 標籤限制

| 可行方向 | 不可行方向 |
| --- | --- |
| 在符合限制時，`a` 可以包住文字或一般區塊內容 | `a` 不能包含另一個 `a` |
| 可用於讓某段一般內容成為連結 | `a` 不能包含 `button` 等互動內容 |
| 若要做可點擊卡片，需檢查內部是否還有其他互動元素 | `a` 不能包含表單控制項等互動元素，也不能包含帶有 `tabindex` 的後代元素 |

## 瀏覽器自動修正的風險

| 風險 | 說明 |
| --- | --- |
| DOM 結構不同 | 實際 DOM 可能和原始 HTML 不一致 |
| CSS 選擇器失準 | 選擇器可能套不到預期元素 |
| JavaScript 節點結構錯誤 | 程式取得的節點可能不符合原始碼想像 |
| 輔助技術或搜尋引擎理解錯誤 | 錯誤結構可能影響語意理解 |

## 實務提醒

| 提醒 | 說明 |
| --- | --- |
| 不要只用 block/inline 判斷合法性 | 合法巢狀要回到個別元素的 content model |
| 不要把視覺層級寫成巢狀 | `h2` 是 `h1` 底下的次級標題，不代表要寫在 `h1` 裡 |
| 不要因為畫面正常就忽略錯誤 HTML | 瀏覽器容錯可能把錯誤藏到 DOM 結構裡 |
| 遇到錯誤巢狀要修原始 HTML | 不應依賴瀏覽器猜測與自動修正 |
