# 塊級與行內元素速查

> 查表內容依據 `notes/070-HTML塊級元素與行內元素/01-塊級元素與行內元素的排版差異.md`。

## 分類對照

| 分類 | 預設排版行為 | 尺寸與間距行為 | 常見用途 | 注意事項 |
| --- | --- | --- | --- | --- |
| 塊級元素 | 多數常見塊級元素會在正常文件流中另起一行，由上往下排列 | 可以設定 `width`、`height`、`margin`、`padding` 等盒模型屬性 | 建立文章段落、標題、清單、表單區塊或一整塊內容區域 | 常見 block non-replaced 元素若未設定 `width`，通常以 `width: auto` 填滿可用水平空間；這不完全等於 `width: 100%` |
| 行內元素 | 不會預設獨佔一整行，會和文字一起流動，直到空間不夠才換行 | 一般 non-replaced inline 元素寬度由內容決定；左右 `margin`、`padding` 會影響水平排列；上下 `padding`、`border` 會被繪製，但通常不會像 block 元素那樣撐高整個行盒 | 包住文字中的片段、插入小元件、標示強調或連結 | `img`、部分表單控制項，以及 `inline-block` 元素可以有可控尺寸，不要簡化成所有行內元素都不能設定寬高 |

## 常見元素

| 分類 | 常見元素 |
| --- | --- |
| 常見塊級元素 | `div`、`p`、`h1` 到 `h6`、`ul`、`ol`、`li`、`dl`、`dt`、`dd`、`form`、`table`、`hr`、`blockquote`、`address`、`menu`、`pre` |
| 常見行內或 phrasing 類元素 | `span`、`em`、`i`、`b`、`strong`、`a`、`img`、`input`、`br`、`select`、`textarea`、`q`、`bdo`、`sub`、`sup` |

## 實務選擇

| 使用情境 | 常見選擇 |
| --- | --- |
| 要建立文章段落、標題、清單、表單區塊 | 多半使用塊級元素 |
| 要包住一段文字中的某個片段 | 多半使用行內元素 |
| 要建立無語意的版面容器 | 常用 `div` |
| 要在文字中標示強調、連結或小段語意 | 常用 `strong`、`em`、`a`、`span` |
| 行內內容需要可控寬高 | 可考慮 replaced element、表單控制項，或用 CSS 改成 `inline-block` |

## 範例判斷

| 程式片段 | 預設行為 | 判斷重點 |
| --- | --- | --- |
| `<h1>標題</h1><p>段落文字</p><div>區塊內容</div>` | `h1`、`p`、`div` 通常上下排列 | 這些是常見塊級元素，偏向建立一整塊內容區域 |
| `<input type="text"><span>補充文字</span>` | `input` 和 `span` 通常會留在同一行 | 這些內容會和行內內容一起流動，直到空間不夠才換行 |

## 常見誤解

| 誤解 | 正確理解 |
| --- | --- |
| 塊級元素永遠等於 `width: 100%` | 常見 block non-replaced 元素未設定 `width` 時通常是 `width: auto`，和 `width: 100%` 在 margin、padding、border 或特殊盒模型下可能不同 |
| 所有行內元素都不能控制尺寸 | 一般 `span`、`em` 這類 non-replaced inline 元素不適合直接用 `width`、`height` 控制尺寸，但 `img`、`input`、replaced element、表單控制項與 `inline-block` 是例外 |
| 只靠畫面換不換行判斷標籤 | 塊級與行內描述預設排版行為；選 HTML 標籤仍應先考慮語意與內容模型 |
