## 9. 常用片段

### 9.1 HTML5 基本文件骨架

| 項目 | 說明 |
|---|---|
| 使用情境 | 建立一份現代 HTML5 文件。 |
| 使用前提 | 需要一份可被瀏覽器解析的基本 HTML 文件。 |
| 來源 | 4.1 基本 HTML5 文件骨架 |

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>我的第一個網頁</title>
</head>
<body>
  <h1>Hello HTML</h1>
  <p>這是一段網頁內容。</p>
</body>
</html>
```

### 注意事項

- `<!doctype html>` 用來宣告 HTML5 解析方式。
- `head` 放文件資訊，`body` 放主要可見內容。

### 9.2 HTML5 文件宣告

| 項目 | 說明 |
|---|---|
| 使用情境 | 需要在現代 HTML 文件最前方宣告解析方式。 |
| 使用前提 | 一般前端開發或初學 HTML。 |
| 來源 | 4.2 HTML4 與 XHTML 的文件宣告 |

```html
<!doctype html>
```

### 注意事項

- 現代初學與一般前端開發通常優先使用 HTML5 的簡潔宣告。

### 9.3 雙標籤元素

| 項目 | 說明 |
|---|---|
| 使用情境 | 需要用開始標籤與結束標籤包住內容。 |
| 使用前提 | 元素不是 notes 中提到的單標籤類型。 |
| 來源 | 4.3 HTML 標籤與元素、4.4 雙標籤與單標籤 |

```html
<p>這是一段文字。</p>
```

### 注意事項

- 一般雙標籤元素仍應寫出結束標籤。

### 9.4 常見單標籤元素

| 項目 | 說明 |
|---|---|
| 使用情境 | 使用不需要包住文字內容的元素。 |
| 使用前提 | 元素本身屬於 notes 中提到的單標籤例子。 |
| 來源 | 4.4 雙標籤與單標籤 |

```html
<meta charset="UTF-8">
<img src="photo.jpg" alt="教室中的 HTML 課程投影片">
<br>
<input type="text" name="username">
```

### 注意事項

- 單標籤不是「所有元素都可以省略結束標籤」。

### 9.5 元素巢狀結構

| 項目 | 說明 |
|---|---|
| 使用情境 | 表示 HTML 元素之間的父子、祖先後代與兄弟關係。 |
| 使用前提 | 需要建立有層級的頁面結構。 |
| 來源 | 4.5 HTML 元素之間的關係 |

```html
<body>
  <main>
    <h1>HTML 簡介</h1>
    <section>
      <h2>什麼是 HTML</h2>
      <p>HTML 是用來描述網頁內容與結構的標記語言。</p>
    </section>
  </main>
</body>
```

### 注意事項

- 縮排清楚有助於確認元素開始與結束位置一致。

### 9.6 簡單文章頁面結構

| 項目 | 說明 |
|---|---|
| 使用情境 | 建立包含頁首、主要內容、文章與連結的頁面結構。 |
| 使用前提 | 需要展示 HTML5 骨架與語意化元素。 |
| 來源 | 5. 範例說明 |

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>HTML 入門文章</title>
</head>
<body>
  <header>
    <h1>HTML 入門文章</h1>
  </header>

  <main>
    <article>
      <h2>什麼是 HTML？</h2>
      <p>HTML 是用來描述網頁內容與結構的標記語言。</p>
      <a href="https://developer.mozilla.org/">閱讀更多 HTML 文件</a>
    </article>
  </main>
</body>
</html>
```

### 注意事項

- 此片段保留 notes 範例中的核心結構，省略第二個小節以維持最小可用。

### 9.7 正確標題層級

| 沒有設定字元編碼 | 文件缺少編碼設定。 | 在 `head` 中加入 `<meta charset="UTF-8">`。 | `meta charset` | 7. 常見錯誤與修正 |

## 11. 相似概念對照表

### 11.1 網頁 vs 網站

| 比較項目 | 網頁 | 網站 |
|---|---|---|
| 定義 | 網站中的一個頁面，通常是一份 HTML 文件。 | 多個相關網頁與資源組成的集合。 |
| 常見形式 | `.html` 或 `.htm` 文件。 | 首頁、列表頁、文章頁、關於我們頁等組合。 |
| 使用時機 | 指單一頁面內容。 | 指整體站點。 |
| 常見錯誤 | 把單頁稱為整個網站。 | 把網站誤解成只有一份 HTML 文件。 |

### 選用建議

- 描述一份 HTML 文件時用「網頁」。
- 描述多個相關頁面與資源時用「網站」。

### 11.2 HTML vs CSS vs JavaScript

| 比較項目 | HTML | CSS | JavaScript |
|---|---|---|---|
| 核心責任 | 描述內容與結構。 | 套用樣式。 | 處理互動邏輯。 |
| 與 DOM 的關係 | 瀏覽器根據 HTML 建立 DOM。 | 多數情況基於 DOM 套用樣式。 | 常透過 DOM API 操作節點。 |
| 本章重點 | 文件骨架、標籤、元素關係、語意。 | 需要 HTML 提供清楚套用目標。 | 需要理解父子、兄弟、祖先與後代關係。 |
| 常見錯誤 | 把 HTML 當成程式語言。 | 原筆記未列出 CSS 錯誤。 | 原筆記未列出 JavaScript 錯誤。 |

### 選用建議

- 內容與結構先用 HTML 表達。
- 視覺樣式交給 CSS。
- 互動邏輯交給 JavaScript。

### 11.3 HTML 文件 vs HTML 元素

| 比較項目 | HTML 文件 | HTML 元素 |
|---|---|---|
| 定義 | 整份 `.html` 檔案。 | 文件中的一個結構單位。 |
| 範圍 | 包含 `doctype`、`html`、`head`、`body` 等完整結構。 | 例如 `<p>這是一段文字。</p>`。 |
| 使用時機 | 討論整份網頁檔案。 | 討論文件內的單一結構。 |
| 常見錯誤 | 把文件與其中某個元素混為一談。 | 把元素當成整份文件。 |

### 選用建議

- 指整份檔案時用「HTML 文件」。
- 指文件中的單一結構時用「HTML 元素」。

### 11.4 標籤 vs 元素

| 比較項目 | 標籤 | 元素 |
|---|---|---|
| 定義 | `<p>` 或 `</p>` 這種標記本身。 | 由開始標籤、內容與結束標籤形成的完整結構。 |
| 範例 | `<p>`、`</p>` | `<p>這是一段文字。</p>` |
| 使用時機 | 討論標記符號。 | 討論完整 HTML 結構單位。 |
| 常見錯誤 | 把單一標記稱為完整元素。 | 忽略元素包含內容與結束標籤。 |

### 選用建議

- 查符號本身時用「標籤」。
- 查完整結構時用「元素」。

### 11.5 雙標籤 vs 單標籤

| 比較項目 | 雙標籤 | 單標籤 |
|---|---|---|
| 定義 | 有開始標籤與結束標籤。 | 不需要包住文字內容，通常只用一個標籤表示。 |
| 常見寫法 | `<p>這是一段段落文字。</p>` | `<meta charset="UTF-8">` |
| notes 例子 | `h1`、`p` | `meta`、`img`、`br`、`input` |
| 注意事項 | 一般雙標籤元素仍應寫出結束標籤。 | 不是所有元素都可以省略結束標籤。 |

### 選用建議

- 需要包住內容時，確認是否應使用雙標籤。
- 只有特定 void elements 才使用單標籤形式。

### 11.6 `head` vs `body`

| 比較項目 | `head` | `body` |
|---|---|---|
| 定義 | 放置文件資訊。 | 放置使用者在頁面上主要看到的內容。 |
| 常見內容 | 編碼、標題、外部資源連結等。 | 標題、段落、連結、文章內容等。 |
| 是否直接顯示 | 通常不直接顯示在頁面主體。 | 主要可見內容所在位置。 |
| 注意事項 | 不應把頁面主體內容放入 `head`。 | 不應放置只屬於文件資訊的內容。 |

### 選用建議

- 文件資訊放 `head`。
- 主要內容放 `body`。

### 11.7 HTML5 文件宣告 vs HTML4 / XHTML 文件宣告

| 比較項目 | HTML5 | HTML4 | XHTML |
|---|---|---|---|
| 常見宣告 | `<!doctype html>` | 較長，會指定 DTD。 | 文件宣告與命名空間更嚴格。 |
| notes 範例特徵 | 簡潔宣告。 | `<!DOCTYPE HTML PUBLIC ...>` | `PUBLIC ... XHTML 1.0 Transitional ...`，並使用 `xmlns`。 |
| 使用建議 | 現代初學與一般前端開發通常優先使用。 | 早期宣告格式。 | 更接近 XML 的寫法。 |
| 注意事項 | 現代開發多數情況可直接使用 HTML5。 | 不代表所有舊寫法都值得繼續使用。 | notes 未要求初學階段深入使用。 |

### 選用建議

- 新建一般 HTML 文件時，優先使用 `<!doctype html>`。

### 11.8 HTML5 狹義 vs HTML5 廣義

| 比較項目 | HTML5 狹義 | HTML5 廣義 |
|---|---|---|
| 定義 | 指 HTML 標準本身。 | 有些教材指 HTML5、CSS3、JavaScript 以及相關 Web API。 |
| 使用時機 | 討論 HTML 文件結構與語法。 | 討論教材或技術集合用語。 |
| 注意事項 | HTML 本身負責文件結構。 | 不應混淆 HTML、CSS、JavaScript 的責任。 |

### 選用建議

- 學習時先確認對方說的 HTML5 是標準本身，還是廣義技術集。

### 11.9 父子關係 vs 祖先後代關係 vs 兄弟關係

| 比較項目 | 父子關係 | 祖先後代關係 | 兄弟關係 |
|---|---|---|---|
| 定義 | 直接上下層。 | 可跨多層的包含關係。 | 同一父元素下的並列關係。 |
| notes 範例 | `main` 是 `h1` 與 `section` 的父元素。 | `body` 是 `p` 的祖先元素。 | `h1` 與 `section` 同屬於 `main` 之下。 |
| 使用時機 | 判斷直接嵌套。 | 判斷某元素是否在另一元素內部。 | 判斷同層元素。 |
| 注意事項 | 子元素一定是後代元素，但後代元素不一定是子元素。 | 祖先後代範圍比父子更廣。 | 兄弟元素不互相包含。 |

### 選用建議

- 寫 CSS 選擇器、理解 DOM 操作與維護結構時，先確認元素關係。

### 11.10 `div` vs 語意化元素

| 比較項目 | `div` | 語意化元素 |
|---|---|---|
| 定義 | notes 提到 `div` 沒有語意。 | 根據內容意義選擇的元素。 |
| notes 例子 | 所有內容都用 `div` 包起來是常見錯誤。 | `main`、`article`、`nav`。 |
| 影響 | 過度使用會降低可讀性與可訪問性。 | 可以讓頁面結構更清楚。 |
| 注意事項 | 不要只用 `div` 建立所有頁面結構。 | 應根據內容意義選擇元素。 |

### 選用建議

- 有適合語意元素時，優先使用語意化元素。
