## 9. 常用片段

### 9.1 HTML5 基本文件骨架

| 項目 | 說明 |
|---|---|
| 使用情境 | 建立一份現代 HTML 文件。 |
| 使用前提 | 需要一份可被瀏覽器解析的基本 HTML 文件。 |
| 來源 | 4.1 建議的 HTML5 基本結構 |

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>HTML 基本結構</title>
</head>
<body>
  <h1>HTML 基本結構</h1>
  <p>這是一份使用 HTML5 結構撰寫的文件。</p>
</body>
</html>
```

### 注意事項

- `<!doctype html>` 放在第一行。
- `<html lang="zh-Hant">` 宣告頁面主要語言。
- `<meta charset="UTF-8">` 放在 `<head>` 中。

### 9.2 HTML5 文件類型宣告

| 項目 | 說明 |
|---|---|
| 使用情境 | 需要告訴瀏覽器使用 HTML5 方式解析文件。 |
| 使用前提 | 建立 HTML 文件時放在最前面。 |
| 來源 | 3.2 文件類型宣告 `<!doctype html>` |

```html
<!doctype html>
```

### 注意事項

- `<!doctype html>` 不是一般 HTML 元素。
- 它沒有開始標籤、結束標籤，也不會顯示在頁面中。

### 9.3 文件語言宣告

| 項目 | 說明 |
|---|---|
| 使用情境 | 宣告頁面主要語言。 |
| 使用前提 | 已建立 `<html>` 根元素。 |
| 來源 | 3.3 `lang` 文件語言 |

```html
<html lang="zh-Hant">
  <!-- 繁體中文頁面 -->
</html>
```

### 注意事項

- 繁體中文頁面 notes 建議使用 `zh-Hant`。
- 英文頁面可使用 `en`。

### 9.4 字元編碼設定

| 項目 | 說明 |
|---|---|
| 使用情境 | 避免中文、標點或特殊符號出現亂碼。 |
| 使用前提 | 可編輯 HTML 文件的 `<head>` 區塊。 |
| 來源 | 3.4 字元集 `charset` |

```html
<head>
  <meta charset="UTF-8">
</head>
```

### 注意事項

- `<meta charset="UTF-8">` 通常應放在 `<head>` 前段。
- 現代 HTML 文件通常使用 `UTF-8`。

### 9.5 屬性基本寫法

| 項目 | 說明 |
|---|---|
| 使用情境 | 在元素開始標籤中加入附加資訊。 |
| 使用前提 | 需要設定元素的類型、名稱、提示文字或其他資訊。 |
| 來源 | 3.5 HTML 標籤屬性、4.2 屬性的正確寫法 |

```html
<input type="text" name="username" placeholder="請輸入帳號">
```

### 注意事項

- 屬性寫在開始標籤內。
- 屬性與標籤名、多個屬性之間都要用空格隔開。
- 屬性值建議使用雙引號。

### 9.6 布林屬性

| 項目 | 說明 |
|---|---|
| 使用情境 | 表示元素的啟用狀態。 |
| 使用前提 | 屬性本身屬於 notes 提到的布林屬性。 |
| 來源 | 3.6 布林屬性 |

```html
<input type="text" name="username" disabled>
<input type="checkbox" checked>
```

### 注意事項

- 布林屬性只要出現在元素上，就表示啟用該狀態。
- 要取消 `disabled` 狀態時，應移除 `disabled` 屬性。

### 9.7 HTML 註解

| 項目 | 說明 |
|---|---|
| 使用情境 | 說明區塊用途或標記待處理事項。 |
| 使用前提 | 內容是給開發者閱讀，不是給頁面使用者顯示。 |
| 來源 | 3.7 HTML 註解 |

```html
<!-- 網站導覽列 -->
<nav>
  <a href="/">首頁</a>
  <a href="/courses">課程</a>
</nav>
```

### 注意事項

- 註解不會顯示在頁面可見內容中。
- 使用者仍可能透過檢視原始碼或開發者工具看到註解。
- 不要把敏感資料放進註解。

### 9.8 表單欄位與標籤關聯

| 項目 | 說明 |
|---|---|
| 使用情境 | 讓 `<label>` 對應指定輸入框。 |
| 使用前提 | 輸入框有可對應的 `id`。 |
| 來源 | 5. 範例說明 |

```html
<label for="email">Email</label>
<input id="email" type="email" name="email" required>
```

### 注意事項

- `for` 與 `id` 讓 `<label>` 對應到指定輸入框。
- notes 指出這有助於可訪問性。

## 11. 相似概念對照表

### 11.1 `<!doctype html>` vs HTML 元素

| 比較項目 | `<!doctype html>` | HTML 元素 |
|---|---|---|
| 定義 | 文件類型宣告。 | HTML 文件中的結構單位。 |
| 使用時機 | 放在 HTML 文件最前面。 | 依內容需求放在文件結構中。 |
| 常見寫法 | `<!doctype html>` | `<html lang="zh-Hant">...</html>` |
| 注意事項 | 不是一般 HTML 元素，沒有開始或結束標籤。 | 多數元素會有開始標籤、內容與結束標籤，部分元素例外。 |
| 常見錯誤 | 放在 `<html>` 後面或省略。 | 標籤、屬性與內容結構寫法不清楚。 |

### 選用建議

- 建立 HTML 文件時，先放 `<!doctype html>`，再撰寫 `<html>` 文件結構。

### 11.2 `lang` vs `charset`

| 比較項目 | `lang` | `charset` |
|---|---|---|
| 定義 | 宣告文件或元素內容使用的語言。 | 指定 HTML 文件使用的字元編碼。 |
| 使用時機 | 通常放在 `<html>` 上宣告主要語言。 | 通常放在 `<head>` 前段。 |
| 常見寫法 | `<html lang="zh-Hant">` | `<meta charset="UTF-8">` |
| 注意事項 | 影響瀏覽器、搜尋引擎與螢幕閱讀器理解內容語言。 | 設定不正確可能導致中文或特殊符號亂碼。 |
| 常見錯誤 | 繁體中文頁面使用不適合的語言宣告。 | 省略或放得太晚，使瀏覽器無法及早知道編碼。 |

### 選用建議

- `lang` 與 `charset` 解決的問題不同，基本 HTML 文件通常兩者都需要。

### 11.3 一般屬性 vs 布林屬性

| 比較項目 | 一般屬性 | 布林屬性 |
|---|---|---|
| 定義 | 透過屬性名與屬性值提供元素附加資訊。 | 只要屬性出現就表示啟用該狀態。 |
| 使用時機 | 需要指定值，例如類型、名稱、資源位置。 | 需要表示停用、勾選、必填等狀態。 |
| 常見寫法 | `<input type="text" name="username">` | `<input type="text" disabled>` |
| 注意事項 | 屬性值建議使用雙引號。 | `disabled` 本身就是屬性名，不是只有屬性值。 |
| 常見錯誤 | 屬性與標籤名沒有空格。 | 以為 `disabled="false"` 表示未停用。 |

### 選用建議

- 需要明確值時使用一般屬性；遇到 notes 提到的 `disabled`、`checked`、`required` 這類狀態時，依布林屬性規則判斷。

### 11.4 `head` vs `body`

| 比較項目 | `head` | `body` |
|---|---|---|
| 定義 | 放置文件資訊與解析相關設定。 | 放置使用者主要看到的頁面內容。 |
| 使用時機 | 設定字元編碼、分頁標題等。 | 放置標題、段落、表單等可見內容。 |
| 常見寫法 | `<head><meta charset="UTF-8"></head>` | `<body><h1>Hello HTML</h1></body>` |
| 注意事項 | `<meta charset="UTF-8">` 通常應放在前段。 | 頁面主要內容應放在這裡。 |
| 常見錯誤 | 省略重要編碼設定。 | 將文件資訊與頁面內容混淆。 |

### 選用建議

- 文件設定放 `head`，使用者主要看到的內容放 `body`。

### 11.5 HTML 註解 vs 頁面可見內容

| 比較項目 | HTML 註解 | 頁面可見內容 |
|---|---|---|
| 定義 | 寫給開發者閱讀的說明。 | 使用者在頁面中看到的內容。 |
| 使用時機 | 說明區塊用途、標記待處理事項。 | 放在 `body` 中供使用者閱讀或操作。 |
| 常見寫法 | `<!-- 網站導覽列 -->` | `<p>這是一段網頁內容。</p>` |
| 注意事項 | 不會顯示在頁面可見內容中，但仍可能被檢視原始碼看到。 | 會成為頁面呈現的一部分。 |
| 常見錯誤 | 放入密碼、API key 或內部機密。 | 原筆記未列出此類錯誤。 |

### 選用建議

- 註解只放維護需要的說明，不放敏感資料，也不長期堆放廢棄程式碼。
