# meter 度量衡速查

> 查表用。概念與用法說明見 notes/280-度量衡標籤/01-meter-度量衡標籤.md。

## 基本語意

| 項目 | 說明 |
| --- | --- |
| 標籤 | `<meter>` |
| 語意 | 表示已知範圍內的測量值 |
| 常見稱呼 | scalar measurement、gauge |
| 適合情境 | 電量、磁碟用量、評分等已有上下限的數值 |
| 不適合情境 | 下載、上傳、安裝、載入等任務進度 |

## 基本結構

| 部分 | 寫法範例 | 用途 |
| --- | --- | --- |
| 標籤 | `<meter>...</meter>` | 建立度量衡語意 |
| 目前值 | `value="50"` | 表示目前測量值 |
| 最小值 | `min="0"` | 設定測量範圍下限 |
| 最大值 | `max="100"` | 設定測量範圍上限 |
| 備援文字 | `50%` | 放在 `<meter>` 開始與結束標籤之間 |

## 常用屬性

| 屬性 | 作用 | 實務提醒 |
| --- | --- | --- |
| `value` | 規定目前的度量值 | 必要屬性，應落在合理的度量範圍中 |
| `min` | 規定度量範圍的最小值 | 實務上建議明確寫出 |
| `max` | 規定度量範圍的最大值 | 實務上建議明確寫出 |
| `low` | 規定被視為偏低的範圍界線 | 用來補充數值狀態語意 |
| `high` | 規定被視為偏高的範圍界線 | 用來補充數值狀態語意 |
| `optimum` | 規定最理想或最佳的值 | 意義會依情境改變 |

## low high optimum 判斷

| 屬性 | 例子 | 語意 |
| --- | --- | --- |
| `low="20"` | 20 以下可被視為偏低 | 定義低值區間界線 |
| `high="80"` | 80 以上可被視為偏高 | 定義高值區間界線 |
| `optimum="90"` | 接近 90 是較理想狀態 | 定義最理想或最佳值 |

`optimum` 不一定代表數值越高越好。手機電量接近 90 通常是好事，但磁碟用量接近 90 可能代表空間快滿了。

## meter 與 progress

| 情境 | 建議標籤 | 原因 |
| --- | --- | --- |
| 手機電量目前 50% | `<meter>` | 這是已知範圍內的測量值 |
| 磁碟已使用 80GB / 100GB | `<meter>` | 這是容量用量的度量 |
| 評分目前 4.5 / 5 | `<meter>` | 這是已知範圍內的評分數值 |
| 檔案下載到 60% | `<progress>` | 這是任務完成進度 |
| 表單送出處理中 | `<progress>` | 這是流程或任務進行狀態 |

## 判斷準則

| 問題 | 判斷 | 建議 |
| --- | --- | --- |
| 這個數值有明確最小值與最大值嗎？ | 有 | 可考慮 `<meter>` |
| 這個數值是在描述某個值的狀態嗎？ | 是 | 可考慮 `<meter>` |
| 這個值可以被判斷為偏低、正常、偏高或接近最佳值嗎？ | 是 | 可考慮 `<meter>` |
| 這個數字是在描述某件事完成到哪裡嗎？ | 是 | 改用 `<progress>` |
| 只是普通數字或沒有明確範圍嗎？ | 是 | 不適合使用 `<meter>` |
| 只是想做出類似進度條的視覺效果嗎？ | 是 | 不應只為外觀使用 `<meter>` |

## 常見錯誤與修正

| 錯誤 | 問題 | 建議 |
| --- | --- | --- |
| 用 `<meter>` 表示下載進度 | 下載進度是任務進行狀態，不是度量衡數值 | 改用 `<progress value="60" max="100">60%</progress>` |
| 只寫 `<meter value="50"></meter>` 表示 50 / 100 | 未指定 `min` 和 `max` 時，預設範圍是 0 到 1 | 寫成 `<meter value="50" min="0" max="100">50%</meter>` |
| 只有量表，沒有文字說明 | 使用者可能不知道量表代表電量、容量或評分 | 搭配文字標籤或上下文說明 |
| 把 `<meter>` 當成單純視覺進度條 | 標籤語意會被誤用 | 先確認數值是否真的是已知範圍內的測量值 |
