## 3. 名詞表

| 名詞 | 英文 | 簡要定義 | 用途 | 注意事項 | 來源 |
|---|---|---|---|---|---|
| 列表 | list | 一組相關內容的 HTML 結構。 | 表示新聞、排行榜、功能選單、步驟、術語解釋等項目集合。 | 重點是內容關係，不只是預設項目符號或數字。 | §2、§3、§9 |
| 無序列表 | unordered list / `<ul>` | 一組沒有先後順序的項目。 | 功能清單、選單、特色列表、新聞列表。 | 直接子元素應是 `<li>`。 | §3、§4.1、§9 |
| 有序列表 | ordered list / `<ol>` | 一組順序、步驟或排名有意義的項目。 | 步驟、排行榜、流程、排名。 | 順序會影響內容理解時才使用。 | §3、§4.2、§9 |
| 描述列表 | description list / `<dl>` | 一組項目名稱與項目說明的配對。 | 術語解釋、商品規格、聯絡資訊、設定項說明。 | 不適合只為了縮排而使用。 | §3、§4.3、§9 |
| 列表項目 | list item / `<li>` | `<ul>` 或 `<ol>` 中的單一項目。 | 包住列表中的每個項目內容。 | 可以包含文字、段落、圖片、連結、按鈕或巢狀列表。 | §3、§3.2、§4 |
| 描述項目名稱 | description term / `<dt>` | `<dl>` 中被描述的名稱或術語。 | 放置名詞、欄位名或項目名稱。 | 需搭配 `<dd>` 表達說明。 | §3、§4.3 |
| 描述內容 | description details / `<dd>` | `<dl>` 中對 `<dt>` 的說明。 | 放置定義、解釋或細節。 | 一個 `<dt>` 可對應多個 `<dd>`。 | §3、§4.3 |
| 列表巢狀 | nested list | 某個列表項目中包含另一個列表。 | 表達父子層級，例如城市與景點。 | 子列表應放在對應父層 `<li>` 裡。 | §4.4、§7、§9 |
| 列表語意 | list semantics | 透過列表元素表達內容之間的關係。 | 幫助瀏覽器、CSS、JavaScript 與輔助工具理解結構。 | 不應只根據預設樣式選標籤。 | §3.1、§8、§9 |
| `list-style: none` | CSS list style removal | 移除列表項目符號的 CSS 寫法。 | 常用於導覽列或自訂列表外觀。 | 只改變外觀，不改變 `<ul>` 與 `<li>` 的語意。 | §6 |

## 4. HTML 元素表

| 元素 | 用途 | 常用屬性 | 使用情境 | 最小範例 | 注意事項 | 來源 |
|---|---|---|---|---|---|---|
| `<ul>` | 建立無序列表。 | 原筆記未列專屬屬性。 | 沒有先後順序的一組項目，例如功能清單、選單、特色列表。 | `<ul><li>課程公告</li></ul>` | 直接子元素應是 `<li>`；不要只因移除項目符號就不用 `<ul>`。 | §3、§4.1、§7、§9 |
| `<ol>` | 建立有序列表。 | `start`、`reversed` | 步驟、流程、排行榜、排名。 | `<ol><li>建立 HTML 檔案</li></ol>` | 順序有意義時使用；編號樣式可交給 CSS 調整。 | §3、§4.2、§7、§9 |
| `<li>` | 表示 `<ul>` 或 `<ol>` 中的列表項目。 | `value` | 每個無序或有序列表項目。 | `<li>列表項目</li>` | `<li>` 裡可放其他元素；巢狀列表應放入對應 `<li>`。 | §3、§3.2、§4.4、§7、§9 |
| `<dl>` | 建立描述列表。 | 原筆記未列專屬屬性。 | 術語解釋、商品規格、聯絡方式、設定項說明。 | `<dl><dt>HTML</dt><dd>標記語言。</dd></dl>` | 適合名稱與說明的配對，不應作為排版縮排工具。 | §3、§4.3、§7、§9 |
| `<dt>` | 表示描述列表中的項目名稱。 | 原筆記未列專屬屬性。 | 術語、欄位名、項目名稱。 | `<dt>HTML</dt>` | 需放在 `<dl>` 的描述關係中使用。 | §3、§4.3 |
| `<dd>` | 表示描述列表中的項目說明。 | 原筆記未列專屬屬性。 | 定義、解釋、欄位內容、細節。 | `<dd>用來描述網頁內容與結構。</dd>` | 一個 `<dt>` 可對應多個 `<dd>`。 | §3、§4.3 |

> 注意：`<nav>`、`<a>`、`<section>`、`<h2>` 等元素在本章主要作為列表實務應用或範例載體出現，未擴寫成完整 HTML 元素表條目。

## 9. 常用片段

### 9.1 無序列表

| 項目 | 說明 |
|---|---|
| 使用情境 | 一組沒有順序差異的項目。 |
| 使用前提 | 項目彼此是並列關係，不是步驟或排名。 |
| 來源 | §4.1、§5.1 |

```html
<ul>
  <li>課程公告</li>
  <li>前端讀書會</li>
  <li>作業繳交提醒</li>
</ul>
```

### 注意事項

- `<ul>` 的直接子元素應該是 `<li>`。
- 不應直接把文字、`<p>`、`<a>` 放在 `<ul>` 第一層。

### 9.2 有序列表

| 項目 | 說明 |
|---|---|
| 使用情境 | 步驟、流程或排名。 |
| 使用前提 | 項目順序會影響內容解讀。 |
| 來源 | §4.2、§5.2 |

```html
<ol>
  <li>建立 HTML 檔案。</li>
  <li>撰寫基本文件結構。</li>
  <li>在瀏覽器中檢查結果。</li>
</ol>
```

### 注意事項

- 若項目沒有順序意義，通常改用 `<ul>`。
- 編號樣式可用 CSS 調整，但語意仍由 `<ol>` 表達。

### 9.3 描述列表

| 項目 | 說明 |
|---|---|
| 使用情境 | 名詞與說明、欄位與內容的配對。 |
| 使用前提 | 內容具有「被描述項目」與「描述內容」的關係。 |
| 來源 | §4.3、§5.3 |

```html
<dl>
  <dt>HTML</dt>
  <dd>用來描述網頁內容與結構的標記語言。</dd>

  <dt>CSS</dt>
  <dd>用來控制網頁外觀與排版的樣式語言。</dd>
</dl>
```

### 注意事項

- `<dt>` 表示名稱或術語。
- `<dd>` 表示對 `<dt>` 的說明。
- 一般項目清單不需要硬改成 `<dl>`。

### 9.4 巢狀列表

| 項目 | 說明 |
|---|---|
| 使用情境 | 某個列表項目底下還有子項目。 |
| 使用前提 | 子項目確實屬於某個父層項目。 |
| 來源 | §4.4、§7 |

```html
<ul>
  <li>
    上海
    <ul>
      <li>外灘</li>
      <li>迪士尼樂園</li>
    </ul>
  </li>
</ul>
```

### 注意事項

- 子列表放進對應的父層 `<li>`。
- 不要把子 `<ul>` 直接放成父層 `<ul>` 的直接子元素。

### 9.5 導覽列列表

| 項目 | 說明 |
|---|---|
| 使用情境 | 一組網站導覽連結。 |
| 使用前提 | 導覽連結是一組相關項目。 |
| 來源 | §6 |

```html
<nav aria-label="主要導覽">
  <ul>
    <li><a href="./index.html">首頁</a></li>
    <li><a href="./courses.html">課程</a></li>
    <li><a href="./about.html">關於我們</a></li>
    <li><a href="./contact.html">聯絡</a></li>
  </ul>
</nav>
```

```css
nav ul {
  list-style: none;
  padding: 0;
  margin: 0;
  display: flex;
  gap: 16px;
}
```

### 注意事項

- `list-style: none` 只移除項目符號，不改變列表語意。
- `<nav>` 與 `<a>` 在此為實務範例載體；本章主題仍是列表。

## 10. 錯誤速查表

| 錯誤現象 | 可能原因 | 修正方向 | 對應知識點 | 來源 |
|---|---|---|---|---|
| 列表容器中直接放文字。 | `<ul>` 或 `<ol>` 缺少 `<li>`。 | 把每個項目包在 `<li>` 中。 | `<ul>` / `<ol>` 的直接子元素 | §4.1、§7 |
| 步驟或排名看不出順序。 | 需要順序的內容使用了 `<ul>`。 | 改用 `<ol>`。 | 有序列表語意 | §4.2、§7 |
| 沒有順序意義的項目被誤解成排名。 | 一般並列項目使用了 `<ol>`。 | 改用 `<ul>`。 | 無序列表語意 | §4.1、§7 |
| 子項目與父項目關係不清楚。 | 子列表放在父層 `<ul>` 外面或成為直接子層。 | 把子列表放進對應父層 `<li>`。 | 巢狀列表 | §4.4、§7 |
| HTML 語意被拿來做縮排。 | 為了視覺縮排而濫用 `<dl>`。 | 若只是排版，使用適合的 HTML 結構並交給 CSS。 | 描述列表語意 | §4.3、§7 |
| 用表格排一般列表內容。 | 把 `<table>` 當成排版工具。 | 列表型內容使用列表標籤，排版交給 CSS。 | 列表 vs 表格 | §7、§8 |
| 為了移除項目符號而不用列表。 | 把列表預設樣式誤認為列表本質。 | 保留列表語意，用 CSS 控制外觀。 | 列表語意與 CSS | §3.1、§6、§9 |

## 11. 相似概念對照表

### 11.1 `<ul>` vs `<ol>` vs `<dl>`

| 比較項目 | `<ul>` | `<ol>` | `<dl>` |
|---|---|---|---|
| 定義 | 無序列表。 | 有序列表。 | 描述列表。 |
| 使用時機 | 一組沒有先後順序的項目。 | 步驟、流程、排名等順序有意義的項目。 | 名詞與說明、欄位與內容的配對。 |
| 常見寫法 | `<ul><li>項目</li></ul>` | `<ol><li>步驟</li></ol>` | `<dl><dt>名詞</dt><dd>說明</dd></dl>` |
| 注意事項 | 直接子元素應是 `<li>`。 | 順序會影響內容理解時使用。 | 不應作為縮排工具。 |
| 常見錯誤 | 用於有順序的步驟。 | 用於沒有順序意義的並列項目。 | 用於一般項目清單或排版。 |

### 選用建議

- 並列項目選 `<ul>`。
- 步驟、流程、排名選 `<ol>`。
- 名詞與說明、欄位與內容選 `<dl>`。

### 11.2 HTML 語意 vs CSS 外觀

| 比較項目 | HTML 語意 | CSS 外觀 |
|---|---|---|
| 定義 | 表達內容結構與關係。 | 控制畫面樣式、排列、間距與項目符號。 |
| 使用時機 | 決定內容應使用 `<ul>`、`<ol>` 或 `<dl>`。 | 決定列表如何顯示，例如橫排或移除項目符號。 |
| 常見寫法 | `<ul><li>首頁</li></ul>` | `list-style: none; display: flex;` |
| 注意事項 | 不應只根據預設樣式選標籤。 | 不會改變元素原本的 HTML 語意。 |
| 常見錯誤 | 為了畫面效果選錯元素。 | 誤以為移除項目符號後就不算列表。 |

### 選用建議

- 先根據內容關係決定 HTML 元素。
- 再用 CSS 調整視覺呈現。

### 11.3 列表 vs 表格

| 比較項目 | 列表 | 表格 |
|---|---|---|
| 定義 | 一組相關項目。 | 表格資料。 |
| 使用時機 | 功能清單、新聞列表、步驟、術語解釋。 | notes 只提到表格應用於表格資料。 |
| 常見寫法 | `<ul><li>項目</li></ul>` | 原筆記未提供表格最小範例。 |
| 注意事項 | 排版交給 CSS。 | 不應用表格排一般列表內容。 |
| 常見錯誤 | 為了排版改用表格。 | 把表格當成一般排版工具。 |

### 選用建議

- 一般項目集合、步驟或導覽使用列表。
- 表格資料才使用表格。
