## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| html5shiv 解決什麼問題？ | 它協助舊 IE 建立並樣式化部分 HTML5 新元素，讓這些元素比較能正常呈現。 | [為什麼需要 html5shiv](../../../notes/第03章_兼容性問題/02-IE-條件註解與-html5shiv-補充.md#為什麼需要-html5shiv) |
| html5shiv 主要服務哪些瀏覽器？ | 本篇脈絡中主要是 IE6、IE7、IE8。 | [重要限制](../../../notes/第03章_兼容性問題/02-IE-條件註解與-html5shiv-補充.md#重要限制) |
| 為什麼要用 IE 條件註解包住 html5shiv？ | 因為 html5shiv 對 IE6 到 IE8 之外的瀏覽器沒有意義，條件註解可以限制載入範圍。 | [使用條件註解載入 html5shiv](../../../notes/第03章_兼容性問題/02-IE-條件註解與-html5shiv-補充.md#使用條件註解載入-html5shiv) |
| `<!--[if lt IE 9]>` 是什麼意思？ | 它表示只有低於 IE9 的 IE 才符合條件並讀取其中內容。 | [範例拆解](../../../notes/第03章_兼容性問題/02-IE-條件註解與-html5shiv-補充.md#範例拆解) |
| `lt`、`lte`、`gt`、`gte`、`!` 分別代表什麼？ | 它們分別代表小於、小於等於、大於、大於等於與邏輯非。 | [條件判斷符號](../../../notes/第03章_兼容性問題/02-IE-條件註解與-html5shiv-補充.md#條件判斷符號) |
| html5shiv 會讓舊 IE 完整支援 HTML5 嗎？ | 不會；它主要處理部分 HTML5 元素的識別與樣式問題，不等於完整 HTML5 API 支援。 | [把 html5shiv 當成完整 HTML5 支援工具](../../../notes/第03章_兼容性問題/02-IE-條件註解與-html5shiv-補充.md#把-html5shiv-當成完整-html5-支援工具) |
| 可以對所有瀏覽器都載入 html5shiv 嗎？ | 不建議；它對非舊 IE 瀏覽器沒有必要。 | [對所有瀏覽器都載入 html5shiv](../../../notes/第03章_兼容性問題/02-IE-條件註解與-html5shiv-補充.md#對所有瀏覽器都載入-html5shiv) |
| IE10 可以用條件註解控制嗎？ | 不可以；本篇特別提醒 IE10 開始不再支援條件註解。 | [忽略 IE10 不支援條件註解](../../../notes/第03章_兼容性問題/02-IE-條件註解與-html5shiv-補充.md#忽略-ie10-不支援條件註解) |
| 現代新專案通常需要 html5shiv 嗎？ | 通常不需要；它更適合作為理解與維護舊專案的歷史兼容方案。 | [實務使用場景](../../../notes/第03章_兼容性問題/02-IE-條件註解與-html5shiv-補充.md#實務使用場景) |
