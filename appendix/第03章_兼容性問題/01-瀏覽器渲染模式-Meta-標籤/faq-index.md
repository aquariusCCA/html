# FAQ Index

## 常見問題索引

| 問題 | 簡短回答 | 對應段落 |
| --- | --- | --- |
| `X-UA-Compatible` 是做什麼的？ | 它主要用於舊版 IE，提示 IE 使用指定或最高可用的文檔模式來渲染頁面。 | [IE 文檔模式與 X-UA-Compatible](../../../notes/第03章_兼容性問題/01-瀏覽器渲染模式-Meta-標籤.md#ie-文檔模式與-x-ua-compatible) |
| `IE=Edge` 代表什麼？ | `IE=Edge` 表示請 IE 優先使用目前版本可支援的最高文檔模式。 | [IE 文檔模式與 X-UA-Compatible](../../../notes/第03章_兼容性問題/01-瀏覽器渲染模式-Meta-標籤.md#ie-文檔模式與-x-ua-compatible) |
| `renderer` 是做什麼的？ | 它是部分雙核瀏覽器支援的 meta 約定，用來提示優先使用 WebKit 類核心。 | [雙核瀏覽器與 renderer](../../../notes/第03章_兼容性問題/01-瀏覽器渲染模式-Meta-標籤.md#雙核瀏覽器與-renderer) |
| `renderer` 是標準 HTML 能力嗎？ | 不是。它是特定瀏覽器約定，不是所有瀏覽器都會遵守的標準寫法。 | [雙核瀏覽器與 renderer](../../../notes/第03章_兼容性問題/01-瀏覽器渲染模式-Meta-標籤.md#雙核瀏覽器與-renderer) |
| 為什麼同一份 HTML 與 CSS 在不同瀏覽器中可能呈現不同？ | 可能是不同瀏覽器、版本、渲染引擎、文件模式或兼容模式造成的差異。 | [問題情境](../../../notes/第03章_兼容性問題/01-瀏覽器渲染模式-Meta-標籤.md#問題情境) |
| 這些兼容性 meta 應該放在哪裡？ | 通常放在 `<head>` 內，而且應盡量靠前，避免瀏覽器已開始用錯誤模式解析後才讀到設定。 | [範例拆解](../../../notes/第03章_兼容性問題/01-瀏覽器渲染模式-Meta-標籤.md#範例拆解) |
| 加了兼容性 meta 就能解決所有兼容性問題嗎？ | 不能。它們只能影響特定瀏覽器的渲染選擇，仍需要查支援、替代寫法、降級方案或實際測試。 | [以為加了 meta 就能解決所有兼容性](../../../notes/第03章_兼容性問題/01-瀏覽器渲染模式-Meta-標籤.md#以為加了-meta-就能解決所有兼容性) |
| `DOCTYPE` 為什麼也會影響兼容性？ | 正確的 `<!DOCTYPE html>` 能讓瀏覽器進入標準模式，是避免舊式怪異模式的重要基礎。 | [忽略 DOCTYPE](../../../notes/第03章_兼容性問題/01-瀏覽器渲染模式-Meta-標籤.md#忽略-doctype) |
| 新專案一定要加 `X-UA-Compatible` 或 `renderer` 嗎？ | 不一定。如果不需要支援舊版 IE 或特定雙核瀏覽器，這些 meta 可能沒有實際意義。 | [把歷史寫法當成新專案必備](../../../notes/第03章_兼容性問題/01-瀏覽器渲染模式-Meta-標籤.md#把歷史寫法當成新專案必備) |
| 維護舊系統時可以直接刪除這些 meta 嗎？ | 不建議急著刪除，應先確認是否仍有舊 IE、企業內網或特定雙核瀏覽器需求。 | [實務使用建議](../../../notes/第03章_兼容性問題/01-瀏覽器渲染模式-Meta-標籤.md#實務使用建議) |
