這個範例把第 43 章筆記中的 inert 模板內容、template.content、DocumentFragment、cloneNode(true)、資料填入、批次插入與事件委派放在同一頁觀察。

<template> 先保存一段不直接渲染的 HTML 結構。
content 真正的模板內容在 template.content 裡。
cloneNode(true) 深層複製所有子節點，得到可修改的複製品。
appendChild 把複製品插入 DOM 後，畫面才會出現內容。
delegate 動態清單常用事件委派處理按鈕點擊。

不是隱藏元素

<template> 是未啟用的結構藍圖，不是單純把元素藏起來。

複製後再填資料

每次使用 template.content.cloneNode(true) 複製，修改複製品後再插入頁面。

避免重複 id

模板會被複製多次，內部應優先使用 class 與 data-*。

4. 對照：<template>、hidden、display: none

寫法 主要用途 關鍵差異
<template> 保存之後要複製使用的 HTML 結構。 內容在 template.content 中，尚未成為渲染內容。
hidden 讓已存在元素暫時不顯示。 元素已經是 DOM 的一部分，只是目前隱藏。
display: none 用 CSS 控制畫面顯示。 這是樣式控制，不是模板機制。

6. 回查清單

  • <template> 內容初始不渲染，屬於 inert 模板內容。
  • 真正要複製的是 template.content，不是 <template> 本身。
  • template.content 是 DocumentFragment。
  • 通常使用 cloneNode(true) 深層複製模板內容。
  • 不要直接 appendChild(template.content)，否則會把原始模板內容移走。
  • 填文字優先用 textContent，避免把不可信字串當 HTML 解析。
  • 動態清單可用事件委派，減少每張卡片都綁事件的負擔。
