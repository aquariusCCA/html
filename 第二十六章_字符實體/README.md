# 第二十六章｜字符實體

> - 章節定位：理解如何在 HTML 中安全地顯示特殊字符、展示 HTML 原始碼，並掌握常見空白處理方式  
> - 建議閱讀方式：先理解字符實體要解決的問題，再回看高頻實體、空格顯示規則與 HTML escaping 觀念

## 本章在學什麼

這一章整理 HTML 的字符實體，也就是 HTML character references。

重點不是把一大串實體名稱全部背起來，而是先理解：

- 什麼時候字符不能直接寫
- 為什麼 `<`、`>`、`&` 容易和 HTML 語法混淆
- 為什麼多個普通空格不會照原樣顯示
- `&nbsp;` 和普通空格有什麼差異
- 顯示 HTML 原始碼時應該怎麼轉義
- 什麼時候該用字符實體，什麼時候該交給 CSS 處理

## 本章與整體地圖的關係

- 前置知識：前面已學過各種 HTML 標籤，這一章補的是「當你想把某些符號當文字顯示時，該怎麼寫」。
- 這章的核心任務：建立字符實體的用途觀念，掌握高頻實體、空格處理、HTML escaping 與安全顯示文字的基本觀念。
- 讀完後通常接：之後在寫教學頁、展示 HTML 原始碼、處理特殊符號、顯示使用者輸入內容時，都可以回來查這一章。

## 適合快速回查的主題

- 什麼是字符實體
- 字符實體的常見寫法：`&名稱;`、`&#十進位;`、`&#x十六進位;`
- 為什麼 `<`、`>`、`&` 不能總是直接寫
- 為什麼多個普通空格不會照原樣顯示
- `&nbsp;` 和普通空格的差異
- `&nbsp;`、`&lt;`、`&gt;`、`&amp;`、`&copy;` 怎麼分
- 顯示 HTML 標籤本身時怎麼寫
- HTML 屬性中的引號怎麼處理
- URL 參數中的 `&` 為什麼常寫成 `&amp;`
- `textContent` 和 `innerHTML` 的差異
- 什麼時候該用 CSS，而不是大量使用 `&nbsp;`

## 易混淆主題

- 普通空格 vs `&nbsp;`：前者可能被合併顯示，後者是不換行空格。
- `&nbsp;` vs CSS 排版：`&nbsp;` 是文字字符，不是主要排版工具。
- 符號本身 vs HTML 語法：像 `<`、`>`、`&` 這類字符很容易和 HTML 語法混在一起。
- 名稱寫法 vs 數字寫法：例如 `&copy;`、`&#169;`、`&#xA9;` 都可以顯示 `©`。
- 直接輸入符號 vs 使用字符實體：在 UTF-8 網頁中，很多符號可以直接輸入，但遇到語法衝突或轉義需求時仍會使用字符實體。
- 顯示文字 vs 插入 HTML：`textContent` 會把內容當文字，`innerHTML` 會把內容當 HTML 解析。
- 高頻實體 vs 全部硬背：先記高頻幾個即可，少見符號需要時查表。

## 本章最小必背清單

```html
&lt;     <
&gt;     >
&amp;    &
&nbsp;   不換行空格
&copy;   ©
````

進階可再記：

```html
&quot;   "
&#39;    '
```
