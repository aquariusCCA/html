## 本節導讀

這篇整理 HTML 中的字符實體，也常被稱為 HTML 實體。

字符實體的核心用途是：  
**讓某些不方便直接寫、容易被瀏覽器誤解成 HTML 語法，或顯示行為比較特殊的字符，可以正確地出現在頁面上。**

例如：

- 想在頁面上顯示 `<div>` 這段文字，而不是讓瀏覽器把它當成真正的 HTML 標籤。
- 想顯示 `&`，但又不希望它被誤認為字符實體的開頭。
- 想顯示版權符號 `©`。
- 想理解為什麼原始碼中連續打了很多空格，頁面上通常只會顯示成一個空格。
- 想知道 `&nbsp;` 和普通空格到底差在哪裡。

這一章的重點不是背完所有字符實體，而是掌握三件事：

1. **什麼時候需要字符實體**
2. **常見字符實體怎麼寫**
3. **哪些情況應該用 CSS 或標籤處理，而不是硬塞一堆 `&nbsp;`**

- 字符實體是 HTML 用來表示特殊字符的一種寫法。
- 常見名稱寫法是：`&名稱;`，例如 `&lt;`、`&gt;`、`&amp;`、`&nbsp;`。
- `<` 容易被解析成標籤開始，所以想顯示 `<` 本身時，常寫成 `&lt;`。

- `&nbsp;` 是不換行空格，不只是普通空格。
- HTML 中連續多個普通空格、換行、Tab，在一般情況下會被合併成一個空格。
- 若只是為了控制排版距離，不建議大量使用 `&nbsp;`，通常應該改用 CSS。
- 少見符號不需要硬背，需要時查表即可。

## 速查區

### 核心概念

字符實體是 HTML 中用來表示特殊字符的語法。

它常用在這幾種情況：

1. **字符容易和 HTML 語法衝突**
   - 例如 `<`、`>`、`&`

2. **字符不容易直接輸入**
   - 例如 `©`、`®`、`€`

3. **字符有特殊顯示效果**
   - 例如 `&nbsp;` 是不換行空格

4. **想在頁面上展示 HTML 原始碼**
   - 例如想顯示 `<p>Hello</p>`，而不是讓它真的變成段落標籤

---

### 常見寫法

字符實體主要有三種寫法：

```html
&名稱;
&#十進位編號;
&#x十六進位編號;
```

例如版權符號 `©` 可以寫成：

```html
&copy;
&#169;
&#xA9;
```

初學階段先記名稱寫法即可，也就是：

```html
&名稱;
```

例如：

```html
&lt;
&gt;
&amp;
&nbsp;
&copy;
```

---

### 高頻字符實體表

| 想顯示的字符 | 字符實體 | 說明 |
|---|---|---|
| `<` | `&lt;` | less than，小於符號，常用來顯示 HTML 標籤本身 |
| `>` | `&gt;` | greater than，大於符號，常和 `&lt;` 搭配使用 |
| `&` | `&amp;` | ampersand，避免被當成字符實體的開頭 |
| `"` | `&quot;` | 雙引號，常用在 HTML 屬性值中 |
| `'` | `&apos;` 或 `&#39;` | 單引號，常用在 HTML 屬性值中 |
| 空格 | `&nbsp;` | 不換行空格，non-breaking space |
| `©` | `&copy;` | 版權符號 |
| `®` | `&reg;` | 註冊商標符號 |
| `™` | `&trade;` | 商標符號 |

---

### 最值得先記的 5 個

初學 HTML 時，最值得先記這幾個：

```html
&nbsp;  不換行空格
&lt;    <
&gt;    >
&amp;   &
&copy;  ©
```

如果再多記兩個，建議記：

```html
&quot;  "
&#39;   '
```

因為它們在處理 HTML 屬性值或輸出使用者輸入內容時很常見。

## 正文筆記

## 1. 什麼是字符實體？

在 HTML 中，有些字符直接寫進原始碼時，瀏覽器不一定會把它們當成普通文字。

例如：

```html
<p>我想顯示 <div> 這幾個字</p>
```

這段程式碼的問題是：  
瀏覽器看到 `<div>` 時，可能會把它理解成 HTML 標籤，而不是單純的文字。

如果你只是想讓畫面顯示：

```html
<div>
```

就應該寫成：

```html
&lt;div&gt;
```

完整範例：

```html
<p>我想顯示 &lt;div&gt; 這幾個字</p>
```

頁面顯示結果：

```text
我想顯示 <div> 這幾個字
```

---

## 2. 為什麼 `<`、`>`、`&` 常需要特殊寫法？

HTML 本身就是用 `<` 和 `>` 來包住標籤的。

例如：

```html
<p>Hello</p>
```

其中：

```html
<p>
```

代表段落標籤的開始。

所以當你想顯示 `<` 這個符號本身時，瀏覽器有可能會把它誤認成標籤語法的開始。

因此常見寫法是：

```html
&lt;
```

代表：

```text
<
```

---

### `>` 也一定要轉義嗎？

嚴格來說，`>` 在很多普通文字情境中可以直接顯示。

例如：

```html
<p>5 > 3</p>
```

通常可以正常顯示。

但是如果你在展示 HTML 程式碼，例如：

```text
<div>
```

通常會把 `<` 和 `>` 成對寫成：

```html
&lt;div&gt;
```

這樣可讀性較好，也比較不容易混淆。

---

### 為什麼 `&` 也要注意？

因為 `&` 是字符實體的開頭。

例如：

```html
&copy;
```

瀏覽器會把它解析成：

```text
©
```

所以如果你只是想顯示 `&` 本身，建議寫成：

```html
&amp;
```

例如：

```html
<p>Tom &amp; Jerry</p>
```

頁面顯示結果：

```text
Tom & Jerry
```

---

## 3. 字符實體的三種寫法

字符實體不只有名稱寫法，也有數字寫法。

以版權符號 `©` 為例：

```html
&copy;
&#169;
&#xA9;
```

它們顯示出來都是：

```text
©
```

三種寫法分別是：

| 寫法 | 說明 |
|---|---|
| `&copy;` | 名稱寫法，最容易閱讀 |
| `&#169;` | 十進位 Unicode 編號 |
| `&#xA9;` | 十六進位 Unicode 編號 |

初學階段不需要硬背數字寫法。  
只要先知道：「除了 `&名稱;`，字符實體也可以用數字表示」即可。

---

## 4. 為什麼空格也會用到字符實體？

在 HTML 中，連續多個普通空格通常不會照原樣顯示。

例如：

```html
<p>hello     world</p>
```

雖然 `hello` 和 `world` 中間有很多個空格，但頁面通常只會顯示成：

```text
hello world
```

原因是：  
在一般 HTML 顯示規則下，連續的普通空格、換行、Tab，通常會被瀏覽器合併成一個空格。

這個現象可以理解成：

```html
hello     world
```

瀏覽器通常會當成：

```html
hello world
```

---

## 5. `&nbsp;` 是什麼？

`&nbsp;` 的完整意思是：

```text
non-breaking space
```

也就是：

```text
不換行空格
```

它和普通空格不完全一樣。

普通空格：

```text
可能被合併，也可能成為自動換行的位置
```

`&nbsp;`：

```text
不容易被合併，也不允許瀏覽器在這個空格位置自動換行
```

例如：

```html
<p>hello&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;world</p>
```

頁面上會看到比較明顯的空白距離。

---

## 6. 什麼時候適合用 `&nbsp;`？

適合使用 `&nbsp;` 的情境：

### 情境 1：想讓兩段文字不要被拆開換行

例如：

```html
<p>HTML&nbsp;Entity</p>
```

這樣 `HTML` 和 `Entity` 中間的空格是不換行空格，瀏覽器比較不會把它們拆到兩行。

---

### 情境 2：教學範例中想展示多個空白

例如：

```html
hello&nbsp;&nbsp;&nbsp;world
```

可以用來讓初學者看到普通空格和 `&nbsp;` 的差異。

---

## 7. 什麼時候不建議用 `&nbsp;`？

如果你只是為了排版，例如想讓文字往右移、想控制欄位距離，不建議大量使用 `&nbsp;`。

不建議：

```html
姓名：&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;小明
```

比較建議用 CSS：

```html
<p class="user-name">
  <span>姓名：</span>
  <span>小明</span>
</p>
```

```css
.user-name {
  display: flex;
  gap: 16px;
}
```

原因是：

- `&nbsp;` 是文字字符，不是排版工具。
- 大量使用 `&nbsp;` 會讓 HTML 結構變得難維護。
- 排版應該交給 CSS，例如 `margin`、`padding`、`gap`、`text-indent`、`white-space`。

---

## 8. 如果真的要保留原始空白格式，該怎麼做？

如果你想保留原始碼中的空格、換行、縮排，通常可以考慮：

### 方法 1：使用 `<pre>`

```html
<pre>
hello     world
line 2
    line 3
</pre>
```

`<pre>` 會保留文字中的空格與換行，常用於顯示程式碼、範例文字或格式化內容。

---

### 方法 2：使用 CSS `white-space`

例如：

```html
<p class="text">
hello     world
line 2
</p>
```

```css
.text {
  white-space: pre-wrap;
}
```

常見值：

| CSS 寫法 | 效果 |
|---|---|
| `white-space: normal;` | 預設行為，連續空白會合併 |
| `white-space: pre;` | 保留空白與換行，不自動換行 |
| `white-space: pre-wrap;` | 保留空白與換行，也允許自動換行 |
| `white-space: nowrap;` | 不換行 |

如果需求是「排版」或「保留文字格式」，通常用 CSS 會比塞很多 `&nbsp;` 更好。

---

## 9. 顯示 HTML 程式碼時怎麼寫？

如果想在頁面上顯示：

```html
<div>Hello</div>
```

不能直接寫：

```html
<p><div>Hello</div></p>
```

因為瀏覽器會把 `<div>` 當成真正的 HTML 標籤。

應該寫成：

```html
<p>&lt;div&gt;Hello&lt;/div&gt;</p>
```

頁面顯示：

```text
<div>Hello</div>
```

如果是展示程式碼，更常搭配 `<pre>` 和 `<code>`：

```html
<pre><code>&lt;div&gt;Hello&lt;/div&gt;</code></pre>
```

## 10. 在 HTML 屬性中也會用到字符實體

如果屬性值本身包含引號，就可能需要轉義。

例如，使用雙引號包住屬性值：

```html
<input value="他說：&quot;Hello&quot;">
```

頁面或欄位值會顯示：

```text
他說："Hello"
```

如果屬性值使用單引號包住，而內容中又有單引號，也可以使用：

```html
<input value='It&#39;s OK'>
```

顯示結果：

```text
It's OK
```

常見規則：

- 屬性值外層用 `"`，內容中的 `"` 建議寫成 `&quot;`
- 屬性值外層用 `'`，內容中的 `'` 建議寫成 `&#39;` 或 `&apos;`
- `&` 建議寫成 `&amp;`

---

## 11. URL 裡面的 `&` 也要注意

例如你想寫一個連結：

```text
/search?keyword=html&page=1
```

在 HTML 屬性中，建議寫成：

```html
<a href="/search?keyword=html&amp;page=1">搜尋</a>
```

瀏覽器實際解析連結時，仍然會把它當成：

```text
/search?keyword=html&page=1
```

這裡的 `&amp;` 是 HTML 原始碼中的安全寫法，不代表網址真的變成 `amp;`。

## 12. 字符實體和 HTML escaping

字符實體也和 HTML escaping 有關。

HTML escaping 指的是：  
**把可能被瀏覽器當成 HTML 語法的字符，轉成安全的文字表示方式。**

常見轉換如下：

| 原字符 | 轉義後 |
|---|---|
| `<` | `&lt;` |
| `>` | `&gt;` |
| `&` | `&amp;` |
| `"` | `&quot;` |
| `'` | `&#39;` |

例如使用者輸入：

```html
<script>alert("hi")</script>
```

如果直接當成 HTML 插入頁面，就可能被瀏覽器當成真正的 `<script>` 標籤。

比較安全的文字顯示方式是：

```html
&lt;script&gt;alert(&quot;hi&quot;)&lt;/script&gt;
```

這樣頁面會顯示文字，而不是執行程式。

---

## 13. 前端實務補充：`textContent` 和 `innerHTML`

如果你用 JavaScript 把文字放到頁面上，要注意插入方式。

### 較安全：`textContent`

```js
const userInput = '<script>alert("hi")</script>';

document.querySelector('#app').textContent = userInput;
```

這樣瀏覽器會把內容當成普通文字顯示。

---

### 需要小心：`innerHTML`

```js
const userInput = '<script>alert("hi")</script>';

document.querySelector('#app').innerHTML = userInput;
```

`innerHTML` 會把字串當成 HTML 解析。  
如果內容來自使用者輸入，就要特別小心安全問題。

簡單判斷：

- 只是顯示文字：優先考慮 `textContent`
- 確定要插入 HTML：才使用 `innerHTML`
- 內容來自使用者：一定要做適當的轉義或過濾

## 15. 常見混淆點

### 混淆 1：`&nbsp;` 是不是普通空格？

不是完全一樣。

`&nbsp;` 是不換行空格。  
它除了能顯示空白，也會避免瀏覽器在這個空格位置自動換行。

---

### 混淆 2：多個普通空格為什麼不會照原樣顯示？

因為在一般 HTML 顯示規則下，連續空白會被合併。

例如：

```html
hello          world
```

通常會顯示成：

```text
hello world
```

---

### 混淆 3：是不是所有特殊符號都要用字符實體？

不是。

現在大多數網頁都使用 UTF-8，所以很多符號可以直接寫，例如：

```html
©
你好
😀
```

但是字符實體仍然常用於：

- 避免和 HTML 語法衝突
- 顯示 HTML 程式碼
- 表示不容易輸入的符號
- 處理 HTML escaping

---

### 混淆 4：`&copy;` 和 `©` 有什麼不同？

顯示結果一樣，都是：

```text
©
```

差別在於：

```html
&copy;
```

是字符實體寫法。

```html
©
```

是直接輸入字符。

在 UTF-8 網頁中，兩者通常都可以正常顯示。

---

### 混淆 5：分號 `;` 可以省略嗎？

有些瀏覽器在某些情況下可能容忍省略分號。

例如：

```html
&copy
```

有時也可能被解析成 `©`。

但是不建議這樣寫。

標準、清楚、穩定的寫法是：

```html
&copy;
```

結論：

```text
字符實體後面的分號建議一定要寫。
```

---

## 16. 一句話抓核心

字符實體是把「不方便直接寫、容易和 HTML 語法衝突，或具有特殊顯示行為的字符」，用安全且明確的方式顯示在 HTML 頁面上的方法。

## 17. 快速判斷：我到底需不需要用字符實體？

可以用下面這個判斷流程：

### 問題 1：你是不是想顯示 HTML 標籤本身？

例如想顯示：

```html
<div>
```

是的話，用：

```html
&lt;div&gt;
```

---

### 問題 2：內容裡面有 `<` 或 `&` 嗎？

如果有，通常建議轉義：

```html
&lt;
&amp;
```

---

### 問題 3：你是不是想控制排版距離？

如果只是排版，不建議用很多 `&nbsp;`。

建議用 CSS：

```css
margin
padding
gap
text-indent
white-space
```

---

### 問題 4：你是不是想保留文字原本的空格和換行？

可以考慮：

```html
<pre>
```

或：

```css
white-space: pre-wrap;
```

---

### 問題 5：你是不是只是想顯示少見符號？

例如：

```text
© ® ™ €
```

可以直接輸入，也可以查字符實體。  
不需要全部硬背。


頁面顯示：

```text
Tom & Jerry
```

---

### 範例 6：屬性中的雙引號

```html
<input value="他說：&quot;Hello&quot;">
```

顯示：

```text
他說："Hello"
```

---

### 範例 7：URL 參數中的 `&`

```html
<a href="/search?keyword=html&amp;page=1">搜尋</a>
```

實際連結意義仍然是：

```text
/search?keyword=html&page=1
```

## 19. 本章最小必背清單

如果只想先記最重要的，記這組就夠：

```html
&lt;     <
&gt;     >
&amp;    &
&nbsp;   不換行空格
&copy;   ©
```

再進階一點，加上：

```html
&quot;   "
&#39;    '
```

## 20. 常見問法

### Q1：為什麼 `<` 不能直接當普通文字？

因為 `<` 在 HTML 中通常代表標籤的開始。  
如果你只是想顯示 `<` 這個符號，建議寫成：

```html
&lt;
```

---

### Q2：為什麼我明明打了很多空格，頁面上卻只看到一個？

因為在一般 HTML 顯示規則下，連續多個普通空格、換行、Tab 會被合併成一個空格。

如果要保留格式，可以考慮：

```html
<pre>
```

或 CSS：

```css
white-space: pre-wrap;
```

如果只是想產生不換行空格，可以使用：

```html
&nbsp;
```

---

### Q3：`&nbsp;` 是不是就是空格？

不完全是。

`&nbsp;` 是不換行空格，意思是 non-breaking space。  
它會顯示空白，但也會避免文字在這個空格位置自動換行。

---

### Q4：字符實體是不是都要硬背？

不需要。

初學階段先記高頻幾個即可：

```html
&nbsp;
&lt;
&gt;
&amp;
&copy;
```

其他少見符號需要時查表即可。

---

### Q5：現在都用 UTF-8，還需要字符實體嗎？

需要，但用途比較集中。

很多符號現在可以直接輸入，例如：

```html
©
😀
中文
```

但是遇到 `<`、`&`、HTML 程式碼展示、HTML escaping、屬性中的引號時，字符實體仍然很重要。

---

### Q6：`&amp;` 顯示出來會是什麼？

```html
&amp;
```

顯示結果是：

```text
&
```

---

### Q7：我要顯示 `<h1>Hello</h1>`，該怎麼寫？

應該寫成：

```html
&lt;h1&gt;Hello&lt;/h1&gt;
```

顯示結果：

```html
<h1>Hello</h1>
```

## 22. 自測答案提示

### 1. 什麼是字符實體？

字符實體是 HTML 中用來表示特殊字符的寫法，常用來處理容易和 HTML 語法衝突、不容易直接輸入，或具有特殊顯示效果的字符。

---

### 2. `&lt;`、`&gt;`、`&amp;` 分別代表什麼？

```html
&lt;   <
&gt;   >
&amp;  &
```

---

### 3. 為什麼 `<` 和 `&` 特別容易需要轉義？

因為：

- `<` 容易被解析成 HTML 標籤開始。
- `&` 容易被解析成字符實體的開始。

---

### 4. 為什麼連續多個普通空格通常只會顯示成一個？

因為在一般 HTML 顯示規則下，連續空白、換行和 Tab 通常會被瀏覽器合併成一個空格。

---

### 5. `&nbsp;` 和普通空格有什麼差異？

`&nbsp;` 是不換行空格，不只是普通空格。  
它可以顯示空白，也會避免瀏覽器在該空格位置自動換行。

---

### 6. 如果想保留文字中的換行和縮排，可以用哪些方法？

可以使用：

```html
<pre>
```

或 CSS：

```css
white-space: pre-wrap;
```

---

### 7. 為什麼不建議大量使用 `&nbsp;` 做排版？

因為 `&nbsp;` 是文字字符，不是排版工具。  
大量使用會讓 HTML 難維護，也不符合結構與樣式分離的原則。  
排版應該優先使用 CSS。

---

### 8. `&copy;`、`&#169;`、`&#xA9;` 顯示結果是否相同？

相同，都會顯示：

```text
©
```

---

### 9. HTML 屬性中如果內容有雙引號，可以怎麼處理？

可以使用：

```html
&quot;
```

例如：

```html
<input value="他說：&quot;Hello&quot;">
```

---

### 10. `textContent` 和 `innerHTML` 有什麼差異？

- `textContent` 會把內容當成普通文字。
- `innerHTML` 會把內容當成 HTML 解析。

如果內容來自使用者輸入，使用 `innerHTML` 要特別小心安全問題。

---

## 23. 本章總結

字符實體不是冷門符號表，而是 HTML 中非常基礎的安全顯示機制。

本章最重要的觀念是：

1. `<` 容易被當成標籤開始，所以常寫成 `&lt;`。
2. `&` 容易被當成字符實體開始，所以常寫成 `&amp;`。
3. `&nbsp;` 是不換行空格，不只是普通空格。
4. 多個普通空格在一般 HTML 中通常會被合併。
5. 想展示 HTML 程式碼時，要把 `<`、`>` 等字符轉義。
6. 想做排版時，不要大量依賴 `&nbsp;`，應該優先使用 CSS。
7. 字符實體可以用名稱寫法，也可以用數字寫法。
8. 少見字符不用硬背，需要時查表即可。
