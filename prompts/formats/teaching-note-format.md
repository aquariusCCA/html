# teaching-note-format.md

## 格式定位

本檔案定義 `notes/` 中「教書型筆記」的固定輸出格式。

它只負責規範筆記長什麼樣子，不負責：

- 定義 AI 角色
- 定義 HTML 教學原則
- 定義 origin 到 notes 的轉換流程
- 生成練習題、複習卡或附錄資料

---

## 筆記整體要求

每份正式筆記應符合以下要求：

1. 主題單一明確。
2. 可以獨立閱讀。
3. 有清楚標題層級。
4. 有概念說明，也有範例。
5. 有實務注意事項。
6. 有常見錯誤與修正。
7. 適合日後回查、複習與延伸生成其他材料。

---

## 建議檔案結構

```md
---
title: ""
category: "HTML"
source: "origin/..."
status: "draft"
updated: "YYYY-MM-DD"
tags: []
---

# 章節標題

## 1. 本章學習目標

## 2. 本章定位

## 3. 核心概念

## 4. 語法與基本用法

## 5. 範例說明

## 6. 實務應用場景

## 7. 常見錯誤與修正

## 8. 與其他概念的關係

## 9. 重點整理

## 10. 自我檢查問題

## 11. 延伸補充
```

若某些章節與當前主題無關，可以省略或合併，但正式筆記應盡量維持一致結構。

---

## Front Matter 規則

每份正式筆記建議使用 YAML Front Matter：

```md
---
title: "HTML 文件基本結構"
category: "HTML"
source: "origin/html-basic.md"
status: "draft"
updated: "2026-05-17"
tags: ["html", "document-structure"]
---
```

欄位說明：

| 欄位 | 說明 |
|---|---|
| `title` | 筆記標題 |
| `category` | 所屬技術分類，例如 HTML |
| `source` | 來源資料路徑 |
| `status` | 狀態，例如 `draft`、`reviewed`、`final` |
| `updated` | 最後更新日期 |
| `tags` | 方便搜尋與分類的標籤 |

---

## 標題層級規則

### H1

每份筆記只能有一個 H1：

```md
# HTML 文件基本結構
```

### H2

H2 用於主要章節：

```md
## 1. 本章學習目標
## 2. 本章定位
## 3. 核心概念
```

### H3

H3 用於 H2 之下的子概念：

```md
### 3.1 `html` 元素
### 3.2 `head` 元素
### 3.3 `body` 元素
```

### H4

H4 只在必要時使用，避免層級過深。

---

## 章節格式規範

### 1. 本章學習目標

用條列方式說明讀完本章後應該會什麼。

格式：

```md
## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 HTML 文件的基本結構。
- 分辨 `head` 與 `body` 的用途。
- 理解瀏覽器如何解析基本 HTML 文件。
```

---

### 2. 本章定位

說明這個主題在整個 HTML 學習路線中的位置。

格式：

```md
## 2. 本章定位

本章是 HTML 的基礎主題，主要說明一份 HTML 文件最基本的骨架。

在學習 CSS、JavaScript 或 Vue 之前，必須先理解 HTML 如何描述頁面內容與結構。
```

---

### 3. 核心概念

用教學語氣解釋主要觀念。

要求：

1. 先定義。
2. 再說明用途。
3. 必要時補充實務情境。
4. 抽象概念要有例子。

格式：

```md
## 3. 核心概念

HTML 是用來描述網頁內容與結構的標記語言。

它不負責複雜邏輯，也不主要負責視覺排版；HTML 的核心任務是告訴瀏覽器：頁面中有哪些內容，以及這些內容具有什麼語意。
```

---

### 4. 語法與基本用法

說明基本語法、元素結構與常用屬性。

格式：

````md
## 4. 語法與基本用法

一個 HTML 元素通常由開始標籤、內容與結束標籤組成：

```html
<p>Hello HTML</p>
```

其中：

| 部分 | 說明 |
|---|---|
| `<p>` | 開始標籤 |
| `Hello HTML` | 元素內容 |
| `</p>` | 結束標籤 |
```

若需要在 Markdown 中放入巢狀程式碼區塊，可依實際工具需求調整外層 fences。

---

### 5. 範例說明

範例應包含「程式碼」與「解釋」。

格式：

````md
## 5. 範例說明

### 範例：基本 HTML 文件

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>我的第一個網頁</title>
</head>
<body>
  <h1>Hello HTML</h1>
  <p>這是一段文字。</p>
</body>
</html>
```

這個範例中：

- `<!doctype html>` 宣告文件使用 HTML5。
- `<html>` 是整份文件的根元素。
- `<head>` 放置給瀏覽器或搜尋引擎讀取的中繼資訊。
- `<body>` 放置使用者會在頁面上看到的主要內容。
````

---

### 6. 實務應用場景

說明這個知識在真實開發中的用途。

格式：

```md
## 6. 實務應用場景

在實務開發中，HTML 文件結構會影響：

- CSS 樣式套用
- JavaScript DOM 操作
- 搜尋引擎理解頁面
- 螢幕閱讀器讀取內容
- Vue 或其他框架最終渲染出的 DOM 結構
```

---

### 7. 常見錯誤與修正

常見錯誤建議使用表格。

格式：

```md
## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 把 `h1` 當成放大字體用 | `h1` 表示最高層級標題，不是純樣式工具 | 使用 CSS 控制字體大小 |
| 表單按鈕未指定 `type` | `button` 在表單中預設可能提交表單 | 明確寫 `type="button"` 或 `type="submit"` |
```

若錯誤需要程式碼示範，可使用以下格式：

````md
### 錯誤範例

```html
<div onclick="submitForm()">送出</div>
```

問題：`div` 沒有按鈕語意，鍵盤操作與輔助工具支援都較差。

### 正確範例

```html
<button type="button" onclick="submitForm()">送出</button>
```
````

### 8. 與其他概念的關係

說明本章概念與其他技術的連結。

格式：

```md
## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| CSS | CSS 透過選擇器選取 HTML 元素並套用樣式 |
| JavaScript | JavaScript 通常透過 DOM API 操作 HTML 解析後的節點 |
| Vue | Vue template 最後仍會渲染成瀏覽器中的 DOM |
| 可訪問性 | 正確 HTML 語意有助於輔助工具理解頁面 |
```

---

### 9. 重點整理

用短條列整理本章最重要的結論。

格式：

```md
## 9. 重點整理

- HTML 的核心任務是描述內容與結構。
- HTML 元素應根據語意選擇，而不是只根據預設樣式。
- 瀏覽器會將 HTML 解析成 DOM Tree。
- 良好的 HTML 結構會影響 CSS、JavaScript、SEO 與可訪問性。
```

---

### 10. 自我檢查問題

每份筆記建議附 3 到 7 題自我檢查問題。

格式：

```md
## 10. 自我檢查問題

1. HTML 的主要責任是什麼？
2. `head` 與 `body` 的差異是什麼？
3. 為什麼不建議只用 `div` 建立所有頁面結構？
4. HTML 與 DOM 有什麼關係？
```

不需要在正式筆記中提供完整答案；若要產生答案版，可另外放到 `review/`。

---

### 11. 延伸補充

延伸補充應短而精準。

適合放在此處的內容：

- 背景原理
- 實務注意事項
- 與其他標準或工具的關係
- 進一步學習方向

格式：

```md
## 11. 延伸補充

> 補充說明：瀏覽器不會直接以原始 HTML 字串作為操作對象，而是會先解析 HTML 並建立 DOM Tree。CSS 與 JavaScript 多數情況下都是基於 DOM 進行後續處理。
```

若延伸內容過長，應拆到 `supplements/`。

## 表格使用規則

適合使用表格的情境：

- 比較相似概念
- 說明元素屬性
- 整理常見錯誤
- 列出用途與差異

表格不適合承載過長段落。若說明超過兩行，改用小節文字。

---

## 程式碼區塊規則

1. HTML 範例使用 `html`。
2. CSS 範例使用 `css`。
3. JavaScript 範例使用 `js`。
4. Vue 範例使用 `vue`。
5. 錯誤範例必須標示原因。
6. 正確範例必須說明修正點。

範例：

````md
### 錯誤範例

```html
<button>取消</button>
```

如果這個按鈕位於表單中，可能會觸發預設提交行為。

### 正確範例

```html
<button type="button">取消</button>
```

明確指定 `type="button"`，可以避免誤送出表單。
````

## 注意、補充、待確認格式

### 注意

```md
> 注意：`button` 在表單內若未指定 `type`，預設可能是提交按鈕。
```

### 補充說明

```md
> 補充說明：ARIA 可以補強語意，但通常不應優先取代原生 HTML 語意元素。
```

### 待確認

```md
> 待確認：原始資料未提供此屬性的瀏覽器支援狀況，需要查證後再補充。
```

## 禁止事項

1. 不要在同一份筆記塞入太多不同主題。
2. 不要只有摘要，沒有教學說明。
3. 不要只有程式碼，沒有解釋。
4. 不要省略常見錯誤。
5. 不要把延伸補充寫得比主體內容還長。
6. 不要讓標題層級混亂。
7. 不要把格式規則寫成工作流程規則。
