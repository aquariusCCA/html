# Practice 輸出格式提示詞

> 檔案位置：`prompts/formats/practice-format.md`  
> 用途：規範從 `notes/` 生成 `practice/` 練習材料時的固定輸出格式。  
> 建議搭配：
>
> - `prompts/system/learning-coach.md`
> - `prompts/system/notes-practice-generator-rules.md`
> - `prompts/workflows/notes-to-practice.workflow.md`

---

## 一、格式定位

本檔案只負責規範練習材料的輸出格式。

它不負責：

- 定義教學角色
- 定義題目生成流程
- 決定某份 notes 要出哪些題
- 實際填入使用者的 notes 內容

---

## 二、輸出格式總原則

生成 practice 時，請輸出「可直接建立 Markdown 檔案」的內容。

每份練習材料都要讓學習者清楚知道：

1. 題目來自哪份 notes。
2. 每題要練習哪個知識點。
3. 題目類型與難度。
4. 作答時有哪些限制。
5. 如何判斷自己是否完成。
6. 需要時可以看哪些提示。
7. 參考答案與評分標準在哪裡。

預設輸出必須分成：

1. 題目區
2. 提示區
3. 參考答案區
4. 評分標準區

若使用者明確要求「不要給答案」，請省略參考答案區，並保留題目區、提示區與評分標準區。

---

## 三、題目編號規則

題目編號建議使用以下格式：

| 題型 | 編號格式 | 範例 |
|---|---|---|
| 概念理解題 | `C-{number}` | `C-01` |
| 練習題 | `E-{number}` | `E-01` |
| 實作任務 | `T-{number}` | `T-01` |
| 改錯題 | `B-{number}` | `B-01` |
| 重構題 | `R-{number}` | `R-01` |

同一份練習中，編號不可重複。

---

## 四、完整 Practice 輸出格式

請使用以下格式輸出一份完整練習材料。

````markdown
# `{notes-title}` 練習題

## 一、基本資訊

| 欄位 | 內容 |
|---|---|
| 對應 notes | `{notes-path}` |
| 章節名稱 | `{chapter-title}` |
| 建議輸出路徑 | `practice/{chapter-slug}/{note-slug}-practice.md` |
| 題目數量 | `{question-count}` |
| 難度範圍 | Lv1～Lv5 |
| 答案策略 | 提供提示、參考答案與評分標準 / 不提供參考答案 |

## 二、Notes 摘要

簡要整理這份 notes 的主題與核心知識點。

## 七、評分標準區

### C-01 評分標準

- [ ] 是否能說明核心概念
- [ ] 是否能指出關鍵差異或原因
- [ ] 是否沒有混淆 notes 中的概念

### E-01 評分標準

- [ ] 是否完成題目要求
- [ ] 是否符合限制條件
- [ ] 是否能說明使用的語法或概念

### T-01 評分標準

- [ ] 是否完成所有功能需求
- [ ] 是否符合驗收標準
- [ ] 程式碼是否清楚且容易維護

### B-01 評分標準

- [ ] 是否找出真正錯誤原因
- [ ] 是否正確修復問題
- [ ] 是否能說明錯誤與 notes 知識點的關係

### R-01 評分標準

- [ ] 是否保留原本外部行為
- [ ] 是否改善可讀性或可維護性
- [ ] 是否能說明重構取捨
````

## 七、格式檢查清單

輸出前請確認：

- [ ] 是否有基本資訊。
- [ ] 是否有題目清單。
- [ ] 是否每題都有題型、難度與對應 notes 知識點。
- [ ] 是否有題目區、提示區、參考答案區與評分標準區。
- [ ] 如果使用者要求不要給答案，是否已省略參考答案區。
- [ ] 改錯題是否提供有問題的程式碼。
- [ ] 重構題是否提供重構前程式碼與重構目標。
- [ ] 實作任務是否有功能需求、限制條件與驗收標準。
- [ ] notes 之外的內容是否標示為「補充延伸內容」。
