# Origin 本地資產 Alt 與連結文字整理提問模板

## 用途

用來請 AI 依照 `prompts/_drafts/origin-asset-alt-and-link-text-draft.md` 的完整規則，整理指定 `origin/<章節>/` 內 Markdown 文件中的本地圖片替代文字與本地附件連結文字。

這份模板只處理：

- Markdown 圖片語法中的 alt
- HTML `<img>` 標籤中的 `alt` 屬性
- Markdown 本地附件連結文字
- HTML 本地附件連結文字

這份模板不處理教材正文重寫、資產路徑標準化、實體檔案重新命名、檔案移動、外部連結改寫或程式碼範例內容改寫。

## 適用場景

當 `origin/<章節>/*.md` 中有低品質圖片 alt、缺少 `alt` 的 HTML 圖片、過於籠統的本地附件連結文字，或需要檢查圖片與附件描述是否符合教材上下文時使用。

建議在本地資產路徑與檔名已完成標準化後使用本模板。執行時建議先使用 dry-run，確認建議改寫清單、保留項目與需要人工確認項目後，再使用 apply。

## 不適用場景

不適合用來標準化資產引用路徑或重新命名實體資產檔案。若要處理本地資產路徑與檔名，應改用：

```text
prompts/requests/rewrite-origin-asset-paths.md
```

不適合用來重新切分、合併或產生 atomic notes。若要從 `origin/<章節>/` 產生 atomic notes，應改用：

```text
prompts/requests/rewrite-origin-to-atomic-notes.md
```

## 使用方式

1. 將下方「dry-run 預覽用提問」貼給具備檔案系統存取能力的 AI。
2. 將 `<章節路徑>` 替換成實際的 `origin/<章節>/` 目錄。
3. 檢查 dry-run 回報，確認建議改寫、保留原因與需要人工確認項目。
4. 確認安全後，再使用「apply 執行用提問」正式處理。
5. 執行後檢查目標 Markdown、回報清單與 git diff，確認沒有改到資產路徑、實體檔名或教材正文。

6. 無法根據圖片內容、檔案類型、鄰近標題、前後段落或教材上下文合理確認時，請列入「需要人工確認」，不要猜測。

* 執行模式：`dry-run`

## apply 執行用提問

```text
請依照 `prompts/_drafts/origin-asset-alt-and-link-text-draft.md` 的完整規則，正式整理以下章節中的本地圖片 alt 與本地附件連結文字：

## 本次處理範圍

* 專案根目錄：`html/`
* 目標章節路徑：`<章節路徑>`
* 掃描範圍：`<章節路徑>/*.md`
* 資產根目錄：`<章節路徑>/assets/`
* 是否處理整個章節中的所有 Markdown：是，只處理上述章節根目錄下的 `.md` 文件

如果目標章節路徑未填、仍是 placeholder，或不是明確的 `origin/<章節>/` 路徑，請停止並要求我補充，不要自行掃描整個 `origin/`。

請注意：

1. 只允許修改目標章節根目錄下 Markdown 文件中的圖片 alt、HTML `alt` 屬性或本地附件連結文字。
2. 不得建立、修改、刪除、移動或重新命名任何實體資產檔案。
3. 不得改寫資產路徑、實體檔名、hash、資料夾名稱、教材正文或程式碼範例內容。
4. 不得處理整個 `origin/`、其他章節、`origin/_raw/`、`origin/_manifests/` 或 `assets/` 內的實體資產檔案。
5. 請略過 fenced code block、行內程式碼、外部網址、錨點連結、特殊協議連結與空連結。

## 本次執行模式

* 執行模式：`apply`

## 執行前確認

請先根據規則檔檢查是否可以安全處理。

只有在以下條件都成立時，才可以實際修改檔案：

1. 目標章節目錄可以讀取。
2. 章節根目錄下可以找到 `.md` 文件。
3. 目標章節中的 `assets/` 目錄可以讀取。
4. 待處理項目屬於規則檔允許處理的圖片 alt、HTML `alt` 或本地附件連結文字。
5. 新文字可以由圖片內容、檔案類型、鄰近標題、前後段落或教材上下文合理支持。
6. 信心等級為高或中；信心等級為低的項目不得改寫，必須列入「需要人工確認」。
7. 修改後 Markdown 圖片語法、Markdown 連結語法、HTML 標籤、屬性與引號都不會被破壞。
8. 修改不會影響外部連結、錨點連結、特殊協議連結、空連結、程式碼區塊或行內程式碼。
9. 修改不會改到資產路徑、實體檔名、hash、資料夾名稱或教材正文。

如果有任何不確定，請不要修改該項目，並列入「需要人工確認」。如果無法安全處理整體任務，請停止並回報原因。

## 回報要求

請依照規則檔中的 apply 模式輸出格式回報處理摘要、已處理清單、已保留清單、需要人工確認清單與檢查結果。

如果沒有某一類資料，請填寫「無」。
```
