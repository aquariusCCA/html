# HTML 教學筆記重構提問模板

## 用途

用來請 AI 依照 `prompts/workflows/rewrite-origin-html-teaching-notes.md` 的規則，將指定的單一 `origin/<章節>/<檔名>.md` 原始資料，重構成高品質 HTML 教學筆記。

這份模板會要求 AI 同步處理：

- 原始資料核心內容保留與重組
- 教學脈絡補足
- 筆記類型判斷
- Markdown 教學結構安排
- 圖片與附件路徑修正
- 輸出到 `notes/<章節>/<原檔名>.md`

## 適用場景

當單一 `origin/<章節>/<檔名>.md` 已經放入原始學習資料，並準備轉換成正式教學筆記時使用。

建議在執行本模板之前，先確認該章節的本地資產已依照 `prompts/workflows/rewrite-origin-asset-paths.md` 完成檔名與引用路徑標準化。

## 使用方式

1. 將下方「可直接複製的提問」貼給具備檔案系統存取能力的 AI。
2. 將 `<原始筆記路徑>` 替換成實際 Markdown 檔案路徑。
3. 一次只處理一份筆記，避免跨章節或跨檔案誤改。
4. 執行後檢查 `notes/<章節>/<原檔名>.md` 與 git diff，確認內容與圖片、附件路徑正確。

## 可直接複製的提問

```text
請依照 prompts/workflows/rewrite-origin-html-teaching-notes.md 的完整規則，重構以下原始筆記：

<原始筆記路徑>

請務必先閱讀 workflow 文件，並嚴格遵守其中的角色、任務背景、任務要求、筆記類型判斷、重構原則、品質標準、圖片與附件路徑規則、輸出要求、建議格式、Few-Shot 風格範例、重構流程與品質檢查清單。

請執行以下工作：

1. 只讀取並處理上述指定的單一 Markdown 檔案。
2. 判斷這篇筆記屬於概念型、標籤型、結構型、實作型或比較型。
3. 找出原始資料中的核心概念、定義、範例、圖片、附件與重要連結。
4. 保留原始資料的重要內容，但不要逐字照抄。
5. 刪除重複、鬆散或低價值的內容。
6. 補足必要的教學脈絡，例如為什麼要學、解決什麼問題、什麼時候會用到、初學者容易誤解什麼，以及實務上如何判斷。
7. 依照主題選擇合適的教學結構，不要硬套固定標題模板。
8. 優先採用「問題情境 → 白話理解 → HTML 語法 → 程式碼拆解 → 常見錯誤 → 實務使用」的教學順序。
9. 如果主題涉及 HTML 標籤或結構，請提供完整、可理解的 HTML 範例與範例拆解。
10. 如果該主題容易寫錯，請加入錯誤寫法、錯誤原因與正確寫法。
11. 請使用繁體中文撰寫，內容要適合初學者理解，也要方便日後回查。
12. 如果原始筆記有圖片或附件，請保留引用，並將正式筆記中的路徑修正為 ../../origin/<章節>/assets/<資產分類>/<標準檔名>。
13. 不要新增原始資料中沒有、且與主題無關的圖片或附件。
14. 不要把簡單筆記擴寫成百科全書，補充內容必須服務於該篇主題。
15. 請直接建立或覆蓋目標檔案 notes/<章節>/<原檔名>.md。
16. 完成後回報目標路徑、判斷出的筆記類型、處理摘要與品質檢查結果。

請只處理上述指定筆記，不要處理其他章節、其他 Markdown 檔案或其他資料夾。
```

## 可替換欄位

| 欄位 | 說明 | 範例 |
| --- | --- | --- |
| `<原始筆記路徑>` | 要重構的單一 `origin/` Markdown 原始筆記 | `origin/HTML簡介/網頁網站.md` |
