# HTML 練習題生成提問模板

## 用途

用來請 AI 依照 `prompts/_drafts/notes-to-html-practice-draft.md` 的規則，將指定的 `notes/` 正式教學筆記轉換成 HTML 練習題。

這份模板支援兩種使用方式：

- 指定單一 `notes/<章節>/<檔名>.md`，產生該篇筆記的 Level 1 到 Level 4 練習題。
- 指定整個 `notes/<章節>/`，產生章節複習與 Level 5 實戰題。

## 適用場景

當正式教學筆記已經整理完成，並準備為學習者補上可實作、可除錯、可驗收的練習題時使用。

建議先從單篇筆記練習題開始，等同一章節的多篇筆記都完成後，再產生章節複習題。

## 使用方式

1. 將下方「可直接複製的提問」貼給具備檔案系統存取能力的 AI。
2. 將 `<輸入路徑>` 替換成實際的 `notes/<章節>/<檔名>.md` 或 `notes/<章節>/`。
3. 一次只處理一篇筆記或一個章節，避免跨章節誤改。
4. 執行後檢查 `practice/<章節>/` 與 git diff，確認題目、答案、難度分布與來源對應正確。

## 可直接複製的提問

```text
請依照 prompts/_drafts/notes-to-html-practice-draft.md 的完整規則，為以下正式筆記或章節產生 HTML 練習題：

<輸入路徑>

請務必先閱讀 workflow 文件，並嚴格遵守其中的角色、任務背景、產題模式、難度分級、題目設計規則、建議題量、輸出格式、檔案命名與輸出路徑、限制條件、品質標準與輸出回報格式。

請執行以下工作：

1. 判斷 `<輸入路徑>` 是單一 `notes/<章節>/<檔名>.md`，還是整個 `notes/<章節>/`。
2. 如果是單篇筆記模式，只讀取並處理指定的單一 Markdown 檔案。
3. 如果是章節複習模式，只讀取並處理指定章節資料夾中的 `*.md` 正式筆記。
4. 分析來源筆記的學習目標、核心概念、重要範例、常見錯誤與重點整理。
5. 題目必須以 HTML 實作為主，不要只產生名詞解釋或知識問答。
6. 單篇筆記模式優先產生 Level 1 到 Level 4，不要硬湊 Level 5。
7. 章節複習模式優先產生 Level 5 實戰題，並可加入少量 Level 3 到 Level 4 的整合題。
8. Level 3 必須改變條件、限制或情境，避免學習者直接照抄筆記範例。
9. Level 4 必須提供錯誤 HTML，並要求指出錯誤原因與修正方式。
10. Level 5 必須是完整頁面或完整區塊需求，並能整合多個觀念完成可用成果。
11. 每題都要包含難度、題型、對應觀念、來源筆記、任務說明、完成條件、提示、參考答案與檢查重點。
12. 參考答案要短、清楚、語法正確，適合初學者閱讀。
13. 不要引入來源筆記沒有教、且完成題目不必要的 CSS、JavaScript、外部 CDN 或框架。
14. 請使用繁體中文撰寫題目、提示、參考答案說明與檢查重點。
15. 單篇筆記模式請直接建立或覆蓋 `practice/<章節>/<筆記名>/README.md`。
16. 章節複習模式請直接建立或覆蓋 `practice/<章節>/chapter-review.md`。
17. 不要修改 `notes/`、`origin/`、`atomic/`、`demos/`、`review/`、`supplements/`、`appendix/` 或 `prompts/` 內的其他檔案。
18. 完成後回報使用模式、目標路徑、產生或覆蓋的檔案清單、題目數量、Level 分布與品質檢查結果。

請只根據上述指定路徑產生練習題，不要處理其他章節、其他 Markdown 檔案或其他資料夾。
```

## 可替換欄位

| 欄位 | 說明 | 範例 |
| --- | --- | --- |
| `<輸入路徑>` | 要產生練習題的單篇正式筆記或章節資料夾 | `notes/表單/01-表單入門與form基本結構.md` |
| `<輸入路徑>` | 要產生章節複習題的正式筆記章節資料夾 | `notes/表單/` |
