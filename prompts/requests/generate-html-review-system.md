# HTML 複習系統生成提問模板

## 用途

用來請 AI 依照 `prompts/_drafts/notes-to-review-system-draft.md` 的規則，將指定的 `notes/<章節>/` 正式教學筆記，轉換成可長期使用的 HTML 複習系統。

這份模板會要求 AI 同步處理：

- 章節知識地圖
- Level 1 記憶型複習題
- Level 2 理解型複習題
- Level 3 應用型複習題
- D0、D1、D3、D7、D14、D30 間隔複習排程
- 錯題記錄表與錯題回流規則
- 階段檢查點與章節完成標準
- 輸出到 `review/<章節>/`

## 適用場景

當某個 `notes/<章節>/` 已經整理成正式教學筆記，並準備為整個章節補上可回憶、可解釋、可應用、可追蹤錯題的複習材料時使用。

這份模板適合處理整個章節，不適合只處理單一筆記。如果只想為單篇筆記產生實作練習，應改用 `prompts/requests/generate-html-practice.md`。

## 使用方式

1. 將下方「可直接複製的提問」貼給具備檔案系統存取能力的 AI。
2. 將 `<notes章節路徑>` 替換成實際的 `notes/<章節>/`。
3. 一次只處理一個章節，避免跨章節誤改。
4. 執行後檢查 `review/<章節>/` 與 git diff，確認題目、答案、排程、錯題回流與來源對應正確。

## 可直接複製的提問

```text
請依照 prompts/_drafts/notes-to-review-system-draft.md 的完整規則，為以下正式筆記章節產生 HTML 複習系統：

<notes章節路徑>

請務必先閱讀 workflow 文件，並嚴格遵守其中的角色、任務、上下文、複習系統模型、來源分析規則、題目設計規則、建議題量、輸出格式、範例、限制條件、品質標準與輸出回報格式。

請執行以下工作：

1. 確認輸入路徑是 `notes/<章節>/` 章節資料夾；如果不是，請停止並要求改提供章節資料夾。
2. 只讀取並處理該章節資料夾中的 `*.md` 正式筆記。
3. 不要讀取或處理其他章節的筆記。
4. 不要把 `origin/` 或 `atomic/` 作為複習內容來源。
5. 依檔名前綴或自然章節順序閱讀正式筆記。
6. 分析每篇筆記的學習目標、核心概念、關鍵名詞、基本規則、重要範例、常見錯誤與可應用場景，並區分 `review_concepts` 與 `context_points`。
7. 建立章節知識地圖，整理本章需要記住、理解與能實作的重點。
8. `review_concepts` 只能收錄本篇主要教學、直接操作、明確展開的概念。
9. `context_points` 只能收錄順帶提到、比較、反例、注意事項或語意邊界判斷內容，不得作為獨立出題核心。
10. 依 Level 1 記憶型、Level 2 理解型、Level 3 應用型設計複習題。
11. Level 1 要檢查名詞定義、基本分類、基本語法與基本規則。
12. Level 2 要檢查原因解釋、概念差異、使用限制與情境判斷。
13. Level 3 要檢查 HTML 實作、結構設計、改需求、除錯或小型整合任務。
14. Level 1、Level 2、Level 3 的主要考點必須錨定在 `review_concepts`；`context_points` 只能作為比較、錯誤原因、情境補充或錯題回流輔助。
15. 每題都要包含題號、複習層次、題型、對應觀念、來源筆記、建議複習節點、題目、作答要求、參考答案或評分要點，以及錯題回流建議。
16. 請安排 D0、D1、D3、D7、D14、D30 的間隔複習排程，每個節點都要有目標、題目範圍、完成標準與錯題處理。
17. 請建立錯題記錄表、錯誤類型分類、高風險概念清單與錯題回流規則。
18. 請建立 Level 1、Level 2、Level 3 的階段檢查點與章節完成標準。
19. 不要假裝知道學習者實際錯題；錯題表應提供可填寫模板與回流規則。
20. 不要引入來源筆記沒有教、且完成題目不必要的 CSS、JavaScript、外部 CDN 或框架。
21. 請直接建立或覆蓋以下檔案：
    - `review/<章節>/README.md`
    - `review/<章節>/question-bank.md`
    - `review/<章節>/spaced-review-plan.md`
    - `review/<章節>/wrong-question-loop.md`
    - `review/<章節>/checkpoints.md`
22. 不要修改 `notes/`、`origin/`、`atomic/`、`appendix/`、`demos/`、`practice/`、`supplements/` 或 `prompts/` 內的內容。
23. 所有內容請使用繁體中文撰寫。
24. 完成後回報來源章節路徑、讀取的正式筆記清單、產生或覆蓋的檔案清單、題目總數、Level 分布、間隔複習節點摘要、錯題回流設計摘要、`review_concepts` / `context_points` 分流檢查結果與品質檢查結果。

請只根據上述指定章節產生複習系統，不要處理其他章節、其他 Markdown 檔案或其他資料夾。
```
