# HTML 複習系統生成提問模板

## 用途

用來請 AI 依照 `prompts/_drafts/notes-to-review-system-draft.md` 的規則，將指定的 `notes/<章節>/` 正式教學筆記，轉換成可長期使用的 HTML 複習系統。 

## 適用場景

當某個 `notes/<章節>/` 已經整理成正式教學筆記，並準備為整個章節補上可回憶、可解釋、可應用、可追蹤錯題的複習材料時使用。

這份模板適合處理整個章節，不適合只處理單一筆記。如果只想為單篇筆記產生實作練習，應改用 `prompts/requests/generate-html-practice.md`。

## 使用方式

1. 將下方「可直接複製的提問」貼給具備檔案系統存取能力的 AI。
2. 將 `<notes章節路徑>` 替換成實際的 `notes/<章節>/`。
3. 一次只處理一個章節，避免跨章節誤改。
4. 執行後檢查 `review/<章節>/` 與 git diff，確認題目、答案、排程、錯題回流與來源對應正確。

## 可直接複製的提問

```text
請依照 prompts/_drafts/notes-to-review-system-draft.md 的完整規則，為以下正式筆記章節產生 HTML 複習系統：

<notes章節路徑>

請只根據上述指定章節產生複習系統，不要處理其他章節、其他 Markdown 檔案或其他資料夾。
``` 

## 可替換欄位

| 欄位 | 說明 | 範例 |
| --- | --- | --- |
| `<notes章節路徑>` | 要產生複習系統的正式筆記章節資料夾 | `notes/表單/` |
