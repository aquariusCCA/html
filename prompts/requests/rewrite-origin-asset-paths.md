# 本地資產路徑標準化提問模板

## 用途

用來請 AI 依照 `prompts/_drafts/origin-asset-standardization-draft.md` 的規則，處理指定 `origin/<章節>/` 內 Markdown 文件中的本地資產引用。

這份模板會要求 AI 同步處理：

- 實體資產檔案重新命名
- Markdown 圖片與連結路徑
- HTML `src` / `href` 等本地資產引用
- 處理摘要與人工確認清單

## 適用場景

當指定章節的 `assets/` 內已經放入圖片、PDF、Excel、Word、影音、壓縮檔或其他輔助資源，但 Markdown 內的引用路徑或檔名尚未標準化時使用。

建議在整理圖片 alt 與附件連結文字之前先執行本模板。

## 使用方式

1. 將下方「可直接複製的提問」貼給具備檔案系統存取能力的 AI。
2. 將 `<章節路徑>` 替換成實際章節目錄。
3. 一次只處理一個章節，避免跨章節誤改。
4. 執行後檢查 git diff，確認資產檔名與 Markdown 引用已同步更新。

## 可直接複製的提問

```text
請依照 prompts/_drafts/origin-asset-standardization-draft.md 的完整規則，處理以下章節：

<章節路徑>

請務必先閱讀 workflow 文件，並嚴格遵守其中的角色、任務、上下文、命名規則、限制條件與輸出格式。

請執行以下工作：

1. 只掃描指定章節目錄下的 *.md 文件。
2. 找出 Markdown 與 HTML 中所有本地資產引用。
3. 確認引用是否指向該章節 assets/ 內的實體檔案。
4. 依照 workflow 規定，讀取實體資產檔案 bytes，計算 SHA-256 前 6 碼作為 hash6。
5. 依照標準檔名格式重新命名可確認的實體資產檔案。
6. 將 Markdown 或 HTML 中的引用路徑同步更新為 ./assets/<資產分類>/<標準檔名>。
7. 不要改寫教材文字內容、alt 文字、title 文字或外部連結。
8. 找不到檔案、無法計算 hash、重名衝突、多分類同名或任何不確定情況，都不要猜測，請列入對應清單。
9. 如果實體檔案無法安全改名，不要更新該引用路徑，避免造成斷鏈。
10. 完成後依照 workflow 的「輸出格式」回報處理摘要、已處理清單、無法匹配的資產、需要人工確認與檢查結果。

請只處理上述指定章節，不要處理其他章節或其他資料夾。
```

## 可替換欄位

| 欄位 | 說明 | 範例 |
| --- | --- | --- |
| `<章節路徑>` | 要處理的 `origin/` 章節目錄 | `origin/第17章_圖片標籤/` |
