# HTML 教學筆記重構提問模板

## 用途

用來請 AI 依照 `prompts/_drafts/atomic-to-html-teaching-notes-draft.md` 的規則，讀取指定章節的 `atomic/<章節>/*.md` 原子化資料，先建立正式筆記映射，再重構成一組高品質 HTML 教學筆記。

這份模板會要求 AI 同步處理：

- 整章 atomic notes 掃描與 `atomic -> notes` 映射判斷
- 短 atomic 合併、完整 atomic 單獨成篇、補充型內容分流
- 原子化資料核心內容保留與重組
- 教學脈絡補足
- 筆記類型判斷
- Markdown 教學結構安排
- 圖片與附件路徑修正
- atomic 來源區塊移除
- 輸出到 `notes/<章節>/*.md`

## 適用場景

當整個 `atomic/<章節>/` 已經完成原子化整理，並準備轉換成正式 `notes/<章節>/` 教學筆記時使用。

建議在執行本模板之前，先確認該章節的本地資產已依照 `prompts/_drafts/origin-asset-standardization-draft.md` 完成檔名與引用路徑標準化。

## 使用方式

1. 將下方「可直接複製的提問」貼給具備檔案系統存取能力的 AI。
2. 將 `<atomic章節路徑>` 替換成實際章節資料夾路徑。
3. 一次只處理一個章節，避免跨章節誤改。
4. 執行後檢查 `notes/<章節>/` 與 git diff，確認篇章映射、內容與圖片、附件路徑正確。

## 可直接複製的提問

```text
請依照 prompts/_drafts/atomic-to-html-teaching-notes-draft.md 的完整規則，重構以下章節的 atomic notes：

<atomic章節路徑>

請務必先閱讀 workflow 文件，並嚴格遵守其中的角色、任務背景、任務要求、正式筆記映射原則、筆記類型判斷、重構原則、品質標準、圖片與附件路徑規則、來源資訊處理、輸出要求、建議格式、Few-Shot 風格範例、重構流程與品質檢查清單。

請執行以下工作：

1. 只讀取並處理上述指定章節資料夾內的 `*.md` 檔案。
2. 依檔名前綴與學習順序掃描整章 atomic notes。
3. 先建立 `atomic -> notes` 映射，判斷哪些 atomic 應合併、哪些可單獨成篇、哪些應作為補充型筆記。
4. 不要機械地把每一篇 atomic note 都轉成同名 notes。
5. 對每篇正式 notes 判斷其屬於概念型、標籤型、結構型、實作型或比較型。
6. 找出對應 atomic 資料中的核心概念、定義、範例、圖片、附件與重要連結。
7. 保留原子化資料的重要內容，但不要逐字照抄。
8. 刪除重複、鬆散或低價值的內容。
9. 移除 atomic note 開頭的 `> 來源：...` 區塊，正式 notes 預設不要保留來源資訊。
10. 補足必要的教學脈絡，例如為什麼要學、解決什麼問題、什麼時候會用到、初學者容易誤解什麼，以及實務上如何判斷。
11. 依照主題選擇合適的教學結構，不要硬套固定標題模板。
12. 優先採用「問題情境 → 白話理解 → HTML 語法 → 程式碼拆解 → 常見錯誤 → 實務使用」的教學順序。
13. 如果主題涉及 HTML 標籤或結構，請提供完整、可理解的 HTML 範例與範例拆解。
14. 如果該主題容易寫錯，請加入錯誤寫法、錯誤原因與正確寫法。
15. 請使用繁體中文撰寫，內容要適合初學者理解，也要方便日後回查。
16. 如果 atomic notes 有圖片或附件，請保留引用；已是 `../../origin/<章節>/assets/...` 的路徑請直接保留，只有 `./assets/...` 這類路徑才修正為 `../../origin/<章節>/assets/<資產分類>/<標準檔名>`。
17. 不要新增原子化資料中沒有、且與主題無關的圖片或附件。
18. 不要把簡單筆記擴寫成百科全書，補充內容必須服務於該篇主題。
19. 請直接建立或覆蓋目標檔案到 `notes/<章節>/*.md`。
20. 完成後回報產生或更新的目標路徑、每篇 notes 的筆記類型、atomic 映射摘要、處理摘要與品質檢查結果。

請只處理上述指定章節，不要處理其他章節、其他 Markdown 檔案或其他資料夾。
```

| --- | --- | --- |
