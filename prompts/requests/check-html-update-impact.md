# HTML 筆記包更新影響判斷提問模板

## 用途

用來請 AI 依照 `meta/update-rules.md` 的規則，判斷指定章節、檔案或資料夾的改動是否需要同步其他層級。

這份模板只用於更新前判斷與規劃，適合確認：

1. 改動屬於哪一種更新類型。
2. 最新正確來源應該在哪一層。
3. 哪些上游或下游需要檢查。
4. 是否需要候選同步或候選重生成。
5. `meta/chapter-status.md` 應如何建議標記。

## 適用場景

當 `origin/`、`atomic/`、`notes/`、`appendix/`、`demos/`、`practice/`、`review/` 或資產檔案發生變動，且需要先判斷影響範圍時使用。

常見情境包括：

- 修改 `origin/<章節>/*.md` 後，判斷 `atomic/`、`notes/` 與下游是否過期。
- 修改 `atomic/<章節>/*.md` 後，判斷正式 `notes/` 是否需要同步。
- 修改 `notes/<章節>/*.md` 後，判斷 appendix、demo、practice 或 review 是否受影響。
- 圖片、附件、檔名、資產路徑、alt 或連結文字異動後，判斷引用位置是否需要檢查。
- 想更新 `meta/chapter-status.md` 前，確認狀態標記是否有依據。

## 不適用場景

不適合用來直接產生、重寫或審查正式 HTML 教學筆記。若要處理內容生成或審查，應改用對應的 `prompts/requests/` 模板。

不適合用來直接重生成全部下游內容。本模板只輸出候選同步、候選重生成與檢查建議；是否實際改檔應由使用者另行確認。

## 使用方式

1. 將下方「可直接複製的提問」貼給具備檔案系統存取能力的 AI。
2. 填入章節、改動檔案或資料夾、改動摘要與已知最新正確來源。
3. 第一輪只要求 AI 輸出更新影響判斷，不建立、修改、刪除或重生成任何檔案。
4. 若判斷結果需要實際同步，人工確認範圍後再另行要求 AI 執行。

## 可直接複製的提問

```text
請依照 `meta/update-rules.md` 的完整規則，幫我判斷以下更新是否需要同步其他層級。

章節：
改動檔案或資料夾：
改動摘要：
已知最新正確來源：
希望處理方式：只判斷
補充限制：

請先查證使用者指定的檔案、同章節上下游檔案、`meta/chapter-status.md` 與必要的章節 log。

請只輸出 `meta/update-rules.md` 中的「判斷結論格式」。
本輪不要建立、修改、刪除或重生成任何檔案。
```

## 實際更新確認用提問

如果第一輪判斷後確認要實際更新，可以另開第二輪使用：

```text
我確認依照剛才的更新影響判斷執行。

請只處理以下範圍：
<允許更新的檔案或資料夾>

請不要處理其他章節，不要重生成未列入範圍的下游內容，也不要修改 `meta/chapter-status.md` 的欄位結構。

完成後請回報：

1. 實際修改摘要
2. 修改檔案清單
3. 仍需人工確認的事項
4. `meta/chapter-status.md` 的狀態標記建議
```

## 可替換欄位

| 欄位 | 說明 | 範例 |
| --- | --- | --- |
| 章節 | 要判斷的章節名稱 | `第08章_meta標籤` |
| 改動檔案或資料夾 | 已改、想改或懷疑過期的位置 | `notes/第08章_meta標籤/` |
| 改動摘要 | 新增、修正、改名、搬移、刪除或重組的內容 | `修正一篇 notes 的 H2 標題，核心內容未改` |
| 已知最新正確來源 | 使用者已知的最新來源；不確定時填 `不確定` | `notes/第08章_meta標籤/` |
| 希望處理方式 | 第一輪建議固定填 `只判斷` | `只判斷` |
| 補充限制 | 額外限制或不應處理的範圍 | `不要重生成整章，不要實際改檔` |
