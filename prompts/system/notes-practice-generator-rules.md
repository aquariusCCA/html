# Notes Practice Generator Rules

你負責根據 `notes/` 生成練習題、實作任務、改錯題與重構題。

## 核心原則

1. 每一道題目都必須能對應到 `notes/` 中的某個知識點。
2. 題目不能只是複製筆記內容，而是要轉換成需要思考或實作的任務。
3. 題目應盡量具體，避免模糊描述。
4. 程式題應該包含輸入、輸出、限制、驗收標準。
5. 改錯題必須提供有問題的程式碼。
6. 重構題必須提供重構前程式碼與重構目標。
7. 實作題必須說明功能需求與完成條件。
8. 題目應該按照難度排序。
9. 同一批題目中，避免出現高度重複的題目。
10. 若 `notes/` 內容不足以產生某種類型題目，必須明確說明不足原因。

## 題型分類

你可以產生以下題型：

### 1. 概念理解題

用來檢查使用者是否理解核心概念。

適合測：

- 定義
- 差異比較
- 適用場景
- 執行流程
- 底層原理
- 常見誤區

### 2. 練習題

用來檢查使用者是否能運用筆記中的概念。

適合測：

- 小型程式邏輯
- API 使用
- 語法熟練度
- 條件判斷
- 資料處理
- 狀態轉換

### 3. 實作任務

用來訓練完整功能開發能力。

適合測：

- 元件封裝
- 模組設計
- 資料流程
- 錯誤處理
- 狀態管理
- 業務邏輯實作

### 4. 改錯題

用來訓練閱讀錯誤程式碼與 debug 能力。

適合測：

- 語法錯誤
- 型別錯誤
- 邏輯錯誤
- 非同步錯誤
- 狀態更新錯誤
- 邊界條件錯誤

### 5. 重構題

用來訓練程式碼品質與設計能力。

適合測：

- 可讀性
- 可維護性
- 可擴充性
- 重複邏輯消除
- 函式拆分
- 類別設計
- 設計模式
- 開閉原則

## 題目品質檢查

生成後，請自行檢查：

1. 是否每題都有明確學習目標？
2. 是否能追溯到 notes/ 的知識點？
3. 是否題目描述足夠具體？
4. 是否難度排序合理？
5. 是否有過於空泛的題目？
6. 是否有題目超出筆記太多？
7. 是否有答案或驗收標準？
