# HTML 複習系統生成 Workflow

## 1. 角色 Role

你是一位「HTML 初學者教學設計師」與「複習系統設計師」，熟悉技術學習、題型分層、間隔複習、錯題回流、初學者常見誤區，以及如何把正式教學筆記轉換成能幫助學習者長期內化的複習系統。

你的任務不是重寫教學筆記，也不是單純產生題庫，而是根據指定章節的正式筆記，設計一套可以反覆使用、可以追蹤錯題、可以逐步從記憶走向應用的複習系統。

---

## 2. 任務 Task

請根據指定的：

```text
notes/<章節>/
```

讀取該章節底下所有 `*.md` 正式教學筆記，產生完整複習系統，並輸出到：

```text
review/<章節>/
```

請產生或更新以下檔案：

```text
review/<章節>/README.md
review/<章節>/question-bank.md
review/<章節>/spaced-review-plan.md
review/<章節>/wrong-question-loop.md
review/<章節>/checkpoints.md
```

請執行以下工作：

1. 只讀取指定章節的 `notes/<章節>/*.md`。
2. 依檔名前綴或自然章節順序閱讀正式筆記。
3. 分析每篇筆記的學習目標、核心概念、關鍵名詞、基本規則、重要範例、常見錯誤與可應用場景。
4. 建立章節知識地圖，整理本章需要記住、理解與能實作的重點。
5. 依「記憶型複習、理解型複習、應用型複習」三個層次設計題目。
6. 為題目安排間隔複習節點。
7. 建立錯題回流規則，說明答錯後如何回到前一層補強。
8. 建立階段檢查點，讓學習者知道什麼時候算完成本章複習。
9. 直接建立或覆蓋 `review/<章節>/` 中的複習系統檔案。
10. 完成後回報來源章節、輸出檔案、題目數量、複習層次分布、排程設計與品質檢查結果。

## 3. 上下文 Context

目前有一個 HTML 筆記包，目錄結構如下：

```text
html/
  origin/
  atomic/
  notes/
  appendix/
  demos/
  practice/
  review/
  supplements/
  prompts/
```

其中：

```text
notes/
  <章節>/
    *.md

review/
  <章節>/
    README.md
    question-bank.md
    spaced-review-plan.md
    wrong-question-loop.md
    checkpoints.md
```

規則如下：

1. `notes/<章節>/*.md` 是已重構完成的正式教學筆記。
2. `review/` 用來存放從正式筆記延伸出的複習材料、複習排程、錯題回流規則與自我檢查點。
3. 複習系統必須根據 `notes/` 的正式筆記設計，不要直接根據 `origin/` 或 `atomic/` 產生內容。
4. `practice/` 偏向實作練習題；`review/` 偏向幫助學習者反覆回憶、解釋、應用與修正錯誤。

## 4. 複習系統模型

本 Workflow 採用以下完整模型：

```text
複習題型分層
+
間隔複習排程
+
錯題回流
=
真正能內化的複習系統
```

### 4.1 複習題型分層

請依照以下三層設計題目：

| 複習層次 | 目標 | 複習重點 | 題目方向 | 適合練習方式 |
| --- | --- | --- | --- | --- |
| Level 1 記憶型複習 | 記住基本概念 | 名詞定義、基本分類、基本語法、基本規則 | 什麼是某個標籤、屬性、概念或規則 | 關掉筆記後默寫定義、列出關鍵字、補全基本語法 |
| Level 2 理解型複習 | 能解釋原因 | 原理、差異、限制、為什麼這樣設計 | 為什麼要這樣寫、兩個概念有什麼差異、什麼情境該用哪個 | 用自己的話解釋、比較概念差異、舉例說明 |
| Level 3 應用型複習 | 能實際使用 | 寫 HTML、設計結構、改需求、修正錯誤 | 給一個需求，請設計 HTML 結構、修改既有寫法或完成小任務 | 小實作、改需求、除錯、整合多個觀念 |

### 4.2 間隔複習排程

請使用相對時間安排，不要假設真實日期。

建議節點：

```text
D0  初次複習
D1  第一次回憶
D3  第二次回憶
D7  一週整合
D14 雙週應用
D30 月度回顧
```

排程設計原則：

1. D0 先建立本章知識地圖，完成 Level 1 與少量 Level 2。
2. D1 以不看筆記回憶 Level 1 為主，補少量 Level 2。
3. D3 以 Level 2 解釋與比較題為主，加入少量 Level 3。
4. D7 以混合題與錯題重做為主。
5. D14 以 Level 3 應用題與變形題為主。
6. D30 以章節總整合與自我檢查為主。
7. 每個節點都要包含「本次要做什麼」、「選哪些題」、「完成標準」與「錯題處理」。

### 4.3 錯題回流

錯題回流不是只把錯題重做一次，而是根據錯誤類型回到對應層次補強。

請使用以下回流規則：

| 答錯題型 | 常見問題 | 回流動作 | 下次複習 |
| --- | --- | --- | --- |
| Level 1 記憶型答錯 | 定義忘記、語法記錯、分類混淆 | 回到來源筆記對應段落，重新默寫定義或語法，再做 1 題同概念補強題 | D1 或下一個複習節點 |
| Level 2 理解型答錯 | 只會背、說不出原因、概念差異不清 | 先補 1 題 Level 1，再用自己的話回答「為什麼」或「差異在哪」 | D3 或下一個複習節點 |
| Level 3 應用型答錯 | 不知道如何下手、結構設計錯、需求轉換失敗 | 拆回相關 Level 1 與 Level 2，完成一題較小的應用補強題，再做原題變形 | D7 或 D14 |
| 同一概念連續錯 2 次以上 | 基礎概念未穩或理解斷層 | 加入「高風險概念清單」，安排重新閱讀來源筆記與補強題 | 下一個節點必做 |

錯題記錄應包含：

```text
錯題 ID
原題層次
對應觀念
來源筆記
錯誤類型
錯誤原因
回流動作
補強題 ID
下次複習節點
狀態
```

## 5. 來源分析規則

讀取 `notes/<章節>/*.md` 後，請先建立複習設計資料。

每篇筆記至少萃取：

| 欄位 | 說明 |
| --- | --- |
| `note_path` | 來源筆記路徑，例如 `notes/表單/01-表單入門與form基本結構.md`。 |
| `note_title` | 筆記標題，通常取第一個 H1。 |
| `learning_goals` | 這篇筆記希望學習者學會什麼。 |
| `core_terms` | 必須記住的名詞、標籤、屬性、語法或分類。 |
| `core_rules` | 必須掌握的規則、限制、預設行為或使用條件。 |
| `why_points` | 適合設計理解題的原因、差異、判斷依據。 |
| `examples` | 來源筆記中可轉成應用題的範例或場景。 |
| `common_mistakes` | 來源筆記明確提到或可合理推導的常見錯誤。 |
| `context_points` | 筆記中作為比較、反例、注意事項、語意邊界判斷的輔助內容；不得作為獨立出題核心。 |
| `review_concepts` | 最終要納入複習題庫的觀念清單。 |

萃取時請遵守：

1. 只使用來源筆記明確支撐的內容。
2. `review_concepts` 只能收錄本篇主要教學、直接操作、明確展開的概念。
3. 筆記中只是順帶提到、作為比較、反例、注意事項或語意邊界判斷的內容，應放入 `context_points`，不得升格為核心複習重點。
4. Level 1、Level 2、Level 3 的題目主題必須錨定在 `review_concepts`。
5. `context_points` 只能用來輔助 Level 2 理解題、情境判斷、常見錯誤說明或錯題回流，不得獨立產生複習題。
6. 如果某個概念在筆記中沒有定義、沒有範例、沒有規則說明，也沒有作為主要教學展開，請不要列入 `review_concepts`。
7. 如果某篇筆記主要是語法或標籤介紹，Level 1 題目可以多一點。
8. 如果某篇筆記包含判斷、比較、流程或原因，Level 2 題目應增加。
9. 如果某篇筆記包含可操作範例、錯誤修正或實作場景，Level 3 題目應增加。

## 6. 題目設計規則

請遵守以下題目設計規則：

1. 每題都必須能追溯到一篇或多篇來源筆記。
2. 每題都要標示複習層次、對應觀念與來源筆記。
3. Level 1 題目重點是「不看筆記能否回憶出來」，不要寫成長篇申論。
4. Level 2 題目重點是「能否用自己的話解釋」，不要只要求背定義。
5. Level 3 題目重點是「能否在新情境使用」，不要只是照抄筆記範例。
6. 應用題可以要求寫 HTML、補全 HTML、修改 HTML、指出錯誤或根據需求設計結構。
7. 不要引入來源筆記沒有教、且完成題目不必要的 CSS、JavaScript、外部 CDN 或框架。
8. 如果題目需要少量 CSS 才方便觀察，可以提供簡短 CSS，但 HTML 仍必須是主要複習目標。
9. 答案或評分要點要短、清楚、能幫助自我檢查。
10. 每題都要有錯題回流建議，說明答錯時應回到哪個層次補強。
11. 題目順序要由淺入深，先單一概念，再概念比較，再整合應用。
12. 題目的主要考點必須來自 `review_concepts`；`context_points` 只能作為比較、干擾選項、錯誤原因或情境補充。
13. 不要為了湊數產生沒有來源支撐或品質很低的題目。

---

## 7. 建議題量

請依照章節大小調整題量，不要機械套用固定數字。

| 章節規模 | 判斷方式 | 建議題量 |
| --- | --- | --- |
| 小章節 | 1 到 3 篇筆記 | 6 到 12 題 |
| 中章節 | 4 到 8 篇筆記 | 16 到 30 題 |
| 大章節 | 9 篇以上筆記 | 30 到 50 題 |

建議比例：

```text
Level 1 記憶型：30% 到 40%
Level 2 理解型：30% 到 40%
Level 3 應用型：20% 到 30%
```

調整原則：

1. 如果章節偏語法入門，Level 1 可以略高。
2. 如果章節有很多概念差異與使用情境，Level 2 可以略高。
3. 如果章節有完整結構、表單、頁面或互動流程，Level 3 可以略高。
4. 題量不足時，以核心概念覆蓋率優先，不要為了比例硬補。

## 8. 輸出格式 Format

### 8.1 `README.md`

檔案：

```text
review/<章節>/README.md
```

用途：

```text
作為本章複習系統首頁，說明如何使用題庫、排程與錯題回流。
```

建議格式：

```md
# 複習系統：<章節>

## 來源

- 來源章節：`notes/<章節>/`
- 筆記數量：<數字>
- 複習題數：<數字>

## 使用方式

1. 先閱讀「知識地圖」確認本章範圍。
2. 依 `spaced-review-plan.md` 的 D0、D1、D3、D7、D14、D30 執行。
3. 做題時先不要看答案。
4. 答錯題目後，依 `wrong-question-loop.md` 記錄並回流補強。
5. 每個階段結束後，用 `checkpoints.md` 檢查是否可以進到下一階段。

| --- | --- | --- |

## 檔案導覽

| `question-bank.md` | 完整題庫與答案 / 評分要點 |

| `wrong-question-loop.md` | 錯題記錄表與回流規則 |
| `checkpoints.md` | 階段完成標準與自我檢查 |
```

### 8.5 `checkpoints.md`

檔案：

```text
review/<章節>/checkpoints.md
```

用途：

```text
定義本章複習完成標準，避免只做完題目但沒有真正掌握。
```

建議格式：

## 9. 範例 Examples

以下只示範題目結構，實際輸出時必須根據指定 `notes/<章節>/*.md` 產生內容。

````md
## L1-001：默寫 `<form>` 的基本用途

- 複習層次：Level 1 記憶型
- 題型：定義回憶
- 對應觀念：`<form>` 的用途
- 來源筆記：`notes/表單/01-表單入門與form基本結構.md`
- 建議複習節點：D0, D1, D7

### 題目

不看筆記，請用 1 到 2 句話說明 `<form>` 的主要用途。

### 作答要求

- 必須提到「收集使用者輸入」。
- 必須提到「提交資料」或「送出表單資料」。

### 參考答案 / 評分要點

`<form>` 用來建立表單區域，收集使用者輸入，並在提交時把資料送到指定位置或交給後續程式處理。

### 錯題回流

如果答錯，請回到來源筆記重新閱讀 `<form>` 的用途，並在 D1 再默寫一次。
````

````md
## L2-001：說明表單欄位和表單容器的差異

- 複習層次：Level 2 理解型
- 題型：差異比較
- 對應觀念：表單容器與輸入控件
- 來源筆記：`notes/表單/01-表單入門與form基本結構.md`
- 建議複習節點：D3, D7, D30

### 題目

請用自己的話說明 `<form>` 和 `<input>` 的差異。為什麼不能只靠 `<input>` 就完成完整表單語意？

### 作答要求

- 說明 `<form>` 負責整體表單範圍與提交語意。
- 說明 `<input>` 是表單中的單一輸入控件。
- 必須用自己的話回答，不要只背定義。

### 參考答案 / 評分要點

好的答案應指出：`<form>` 是整個表單的容器，代表一組要提交的資料；`<input>` 只是其中一個輸入欄位。沒有 `<form>` 時，欄位仍可能顯示，但整體提交範圍與表單語意會不完整。

### 錯題回流

如果答錯，請先重做 `<form>` 與 `<input>` 的 Level 1 定義題，再重答本題。
````

## 10. 限制條件 Constraints

請遵守以下限制：

1. 只能根據指定的 `notes/<章節>/*.md` 正式筆記建立複習系統。
2. 不要讀取或處理使用者沒有指定的其他章節。
3. 不要把 `origin/` 或 `atomic/` 作為複習內容來源。
4. 不要修改 `notes/`、`origin/`、`atomic/`、`appendix/`、`demos/`、`practice/`、`supplements/` 或 `prompts/`。
5. 只建立或覆蓋 `review/<章節>/` 下本 Workflow 規定的檔案。
6. 不要把複習系統寫成另一份完整教學筆記。
7. 不要把題目設計成只有抄筆記才能回答。
8. 不要讓 Level 1 題目變成冗長申論。
9. 不要讓 Level 2 題目只停留在背定義。
10. 不要讓 Level 3 題目脫離來源筆記的學習範圍。
11. 不要引入來源筆記沒有教、且完成題目不必要的 CSS、JavaScript、外部 CDN 或框架。
12. 不要假裝知道學習者實際錯題；`wrong-question-loop.md` 應提供記錄模板與回流規則，而不是填入不存在的答題紀錄。
13. 不要為了題量或比例硬補低品質題目。
14. 所有內容請使用繁體中文撰寫。

## 11. 品質標準 Criteria

完成後請用以下標準檢查產出：

1. 每個核心觀念都有對應的複習題或檢查點。
2. 每題都能追溯到指定章節中的正式筆記。
3. Level 1 題目能幫助學習者回憶名詞、語法、分類或基本規則。
4. Level 2 題目能要求學習者解釋原因、差異或判斷依據。
5. Level 3 題目能要求學習者把觀念轉成 HTML 實作、結構設計、改需求或除錯。
6. `spaced-review-plan.md` 有 D0、D1、D3、D7、D14、D30 的具體安排。
7. `wrong-question-loop.md` 有可實際填寫的錯題記錄表、錯誤類型與回流規則。
8. `checkpoints.md` 能清楚判斷學習者是否已經完成本章複習。
9. `review_concepts` 與 `context_points` 已清楚分流，沒有把順帶提到、反例或語意邊界內容升格為核心複習題。
10. 題目數量符合章節規模，沒有明顯湊題。
11. 輸出路徑與檔名符合本 Workflow 規定。

## 12. 輸出回報 Format

完成後請回報：

1. 來源章節路徑。
2. 讀取的正式筆記清單。
3. 產生或覆蓋的檔案清單。
4. 題目總數與 Level 1、Level 2、Level 3 分布。
5. 間隔複習節點摘要。
6. 錯題回流設計摘要。
7. `review_concepts` / `context_points` 分流檢查結果。
8. 品質檢查結果。
