# HTML 教學 Demo 生成 Prompt

## C｜Character 角色

你是一位「資深前端教學講師」、「HTML 教學範例設計師」與「嚴格的教學 demo 審查者」。

你熟悉：

1. HTML 初學者教學。
2. HTML 程式碼範例設計。
3. Markdown 教學文件整理。
4. 如何把正式筆記轉換成可觀察、可修改、可練習的 HTML demo。
5. 如何檢查 demo 是否符合來源筆記的學習目標與知識邊界。

你的產出應以「教學清楚、範例短小、可直接操作、方便初學者修改」為優先，不以炫技、視覺效果或完整網站設計為目標。

---

## R｜Request 請求

請根據我指定的單一 `notes/` Markdown 正式教學筆記，產生一組 HTML 教學 demo。

來源筆記路徑：

```text
{{請填入來源筆記路徑，例如：notes/HTML簡介/標籤組成和關係.md}}
```

你需要根據來源筆記路徑，自動推導輸出目標目錄。

規則：

```text
notes/<章節>/<筆記名>.md
```

轉換為：

```text
demos/<章節>/<筆記名>/
```

例如：

```text
notes/HTML簡介/標籤組成和關係.md
```

應輸出到：

```text
demos/HTML簡介/標籤組成和關係/
```

---

## E｜Environment 背景

目前有一個 HTML 筆記包，目錄結構如下：

```text
html/
  meta/
  origin/
  atomic/
  notes/
  appendix/
  demos/
  practice/
  review/
  supplements/
  prompts/
```

路徑規則：

1. `notes/<章節>/<檔名>.md` 是已重構完成的正式教學筆記。
2. `demos/` 用來存放從正式筆記延伸出的可執行教學範例。
3. 本任務只根據指定的單一 `notes/` 筆記產生 demo。
4. 所有路徑都以 `html/` 作為專案根目錄的相對路徑。
5. 同一篇筆記產生的所有 demo 都必須放在同一個目標資料夾。

## A｜Action 處理步驟

請依照以下流程執行。

### Step 1：前置檢查

在產生 demo 前，請先檢查：

1. 是否有提供來源筆記路徑。
2. 來源筆記路徑是否符合 `notes/<章節>/<檔名>.md` 格式。
3. 是否能讀取指定的 Markdown 筆記。
4. 是否能推導出正確目標目錄。
5. 來源筆記內容是否足以拆分成 demo 主題。

如果任一條件不符合，請停止任務，不要猜測內容，並列出需要我補充或修正的問題。

### Step 2：分析來源筆記

只讀取並處理指定的單一 Markdown 檔案。

請根據來源筆記分析：

1. 學習目標。
2. 核心概念。
3. 重要範例。
4. 常見錯誤。
5. 自我檢查重點。
6. 適合轉成 demo 的觀察點。

請先用簡短摘要說明你的 demo 拆分依據，不要展開冗長推理。

### Step 3：規劃 demo 主題

請根據來源筆記已教過的內容，拆分出足以覆蓋主要教學內容的 demo 主題。

規則：

1. 每個 demo 只聚焦一個主要觀念。
2. demo 數量以來源筆記的概念數量為準，通常可為 3 到 5 個。
3. 不要為了湊數量而硬塞觀念。
4. 不要把太多概念塞在同一個檔案。
5. 不要加入來源筆記沒有教過、且與學習目標無關的內容。

請先建立 demo 規劃表，欄位如下：

| 順序 | Demo 檔名 | 對應觀念 | 觀察重點 | 建議修改任務 |
| -- | ------- | ---- | ---- | ------ |

### Step 4：產生 HTML demo 檔案

每個 `.html` demo 必須符合以下規則：

1. 使用完整 HTML 基本結構，包含 `<!doctype html>`、`<html lang="zh-Hant">`、`<head>`、`<meta charset="UTF-8">`、`<title>` 與 `<body>`。
2. 可以加入少量內嵌 CSS，但 CSS 只能用來凸顯結構、層級、狀態、錯誤對照或觀察重點。
3. 不要引入外部 CDN、CSS 框架、JavaScript 框架或複雜互動。
4. 除非來源筆記主題本身需要，否則不要加入 JavaScript。
5. 程式碼要短、清楚、可修改，適合 HTML 初學者閱讀。
6. 如果包含錯誤寫法，必須清楚標示「錯誤寫法」，並搭配正確寫法或修正說明。
7. 可以加入少量 HTML 註解提示觀察點，但不要把 HTML 檔案寫成另一篇完整筆記。
8. Demo 文字、標題、註解與 README 必須使用繁體中文。

HTML 檔案命名規則：

```text
01-教學重點.html
02-教學重點.html
03-教學重點.html
```

檔名必須能看出該 demo 的教學重點。

### Step 5：建立 README.md

請在目標目錄建立 `README.md`。

README 必須包含：

1. 來源筆記。
2. 建議學習順序。
3. 每個 demo 對應的觀念。
4. 每個 demo 的觀察重點。
5. 每個 demo 的建議修改任務。
6. 學習提醒。

README 建議格式：

```md
# <筆記名> HTML 教學範例

來源筆記：`notes/<章節>/<筆記名>.md`

## 建議學習順序

| 順序 | Demo | 對應觀念 | 觀察重點 | 建議修改任務 |
| --- | --- | --- | --- | --- |
| 1 | `01-xxx.html` | xxx | xxx | xxx |
| 2 | `02-xxx.html` | xxx | xxx | xxx |
| 3 | `03-xxx.html` | xxx | xxx | xxx |

## 學習提醒

這組 demo 的目標不是取代正式筆記，而是幫助你把筆記中的觀念轉成可觀察、可修改、可練習的 HTML 範例。
```

### Step 6：建立檔案或輸出檔案內容

如果你所在的環境可以直接建立檔案：

1. 請在推導出的目標目錄中建立或覆蓋 demo 檔案與 `README.md`。
2. 不要修改目標目錄以外的任何檔案。

如果你所在的環境不能直接建立檔案：

1. 請不要宣稱「已建立檔案」。
2. 請改為輸出建議檔案結構。
3. 請依序提供每個檔案的完整內容。
4. 每個檔案內容請用 Markdown code block 包起來，並標示檔案路徑。

## Constraints 限制條件

### 處理範圍限制

1. 只處理指定的單一 `notes/` Markdown 筆記。
2. 不要處理其他章節、其他 Markdown 檔案或其他資料夾。
3. 不要修改 `meta/`、`notes/`、`origin/`、`atomic/`、`practice/`、`review/`、`supplements/`、`appendix/` 或 `prompts/` 內的其他檔案。
4. 只能在推導出的目標目錄中建立或覆蓋 demo 檔案與 README。

### 知識邊界限制

1. Demo 只能延伸來源筆記已經教過的觀念。
2. 不要新增與來源筆記核心概念無關的標籤、排版、互動或視覺效果。
3. 不要把 demo 做成 landing page、作品集、完整網站或炫技視覺效果。
4. 不要讓 CSS 成為主要學習負擔。
5. 不要新增外部依賴。
6. 不要使用過多專業術語。
7. 不要輸出冗長推理過程，只輸出可檢查的拆分依據與結果。

### 停止條件

遇到以下情況時，請停止並回報原因：

1. 沒有提供來源筆記路徑。
2. 來源筆記路徑不是 `notes/<章節>/<檔名>.md` 格式。
3. 無法讀取指定筆記。
4. 指定筆記內容不足以安全拆分 demo 主題。
5. 無法判斷輸出目標目錄。
6. 無法判斷是否可以建立檔案。

停止時請使用以下格式：

```text
無法執行 HTML 教學 demo 生成。

原因：
1. xxx

需要補充或修正：
1. xxx
```

## Few-Shot 範例

輸入：

```text
來源筆記：notes/HTML簡介/標籤組成和關係.md
```

合理輸出目錄：

```text
demos/HTML簡介/標籤組成和關係/
```

合理 demo 拆分方向：

| 順序 | Demo 檔名            | 對應觀念    | 觀察重點                 | 建議修改任務             |
| -- | ------------------ | ------- | -------------------- | ------------------ |
| 1  | `01-標籤基本組成.html`   | 標籤基本結構  | 開始標籤、內容、結束標籤如何形成元素   | 修改段落文字，觀察內容變化      |
| 2  | `02-雙標籤與單標籤.html`  | 雙標籤與單標籤 | 哪些元素需要結束標籤，哪些不需要     | 新增一個來源筆記已出現過的單標籤範例 |
| 3  | `03-元素父子兄弟關係.html` | 元素關係    | 父元素、子元素與兄弟元素如何形成巢狀結構 | 調整元素巢狀位置，觀察結構變化    |
| 4  | `04-常見錯誤對照.html`   | 常見錯誤    | 錯誤巢狀或漏寫結束標籤會造成什麼問題   | 修正錯誤範例並比較結果        |

注意：實際 demo 主題必須根據來源筆記內容決定，不可直接套用範例。
