# dropzone 屬性

> 所屬章節：第五章｜全局屬性
> 關鍵字：`dropzone`、拖放目標、drop target、drag and drop、`copy`、`move`、`link`、obsolete、`dragenter`、`dragover`、`drop`
> 建議回查情境：想知道 `dropzone` 原本是做什麼的、想分清它和 `draggable` 的角色差別、想確認為什麼現代拖放實作不再依賴 `dropzone`

---

## 本節導讀

`dropzone` 很容易被學成一個過度簡化的結論：

> 「把元素寫成 `dropzone="copy"`，它就會變成拖放區。」

這種理解方向不對。
`dropzone` 的真正定位，不是現代拖放 API 的主流解法，而是 HTML 曾經嘗試用**宣告式屬性**描述「放置目標」的一條舊路線。

更精確地說：

* `dropzone` 原本想處理的是「哪裡可以接收放下」
* `copy`、`move`、`link` 原本想描述的是「放下後偏向哪種操作語意」
* 但今天真正可靠的拖放做法，已經不是依賴這個屬性，而是回到 `dragenter`、`dragover`、`drop` 等事件流程

所以學這個屬性時，最重要的不是把它背成熟練語法，而是知道：

> **它比較適合用來理解歷史設計與舊教材，而不是當成現代實作答案。**

---

## 這篇在解決什麼問題

拖放互動通常包含兩端：

1. **誰可以被拖出去**
2. **哪裡可以接收放下**

`dropzone` 原本想處理的是第二件事，也就是：

* 這個元素是否應被視為放置目標
* 如果接收放下，這次操作比較像 `copy`、`move` 還是 `link`

但真正要釐清的是：

> 它今天更像「歷史上的設計方案」，
> 不應再被當成新專案的核心拖放解法。

---

## 先講結論 / 核心理解

先記住這幾點：

* `dropzone` 原本想描述某個元素是拖放目標。
* `copy`、`move`、`link` 描述的是放下時偏向的操作語意。
* 它不是拖曳來源屬性；拖曳來源通常看 `draggable`。
* 它不是現代拖放的主要實作方式。
* 今天要建立真正可用的 drop target，重點是處理 `dragover`、`drop` 等事件，而不是依賴 `dropzone` 這個屬性。

---

## 先把角色切開：`dropzone` 和 `draggable` 不一樣

這兩個很容易被一起記混，所以一定要先分清楚。

### `draggable`

處理的是：

> **哪個元素能被拖出去**

也就是拖曳來源（drag source）。

---

### `dropzone`

原本想處理的是：

> **哪個元素能接收放下**

也就是放置目標（drop target）。

---

### 一句話切開

* `draggable`：誰能開始拖
* `dropzone`：哪裡原本想被聲明成可放下區域

所以它們不是互斥，而是原本分別處理拖放互動的兩端。

---

## `dropzone` 原本在控制什麼

`dropzone` 原本想讓開發者直接用 HTML 寫出這種語意：

* 這個元素可以接收拖放
* 接收放下時，偏向哪種操作方式

也就是說，它原本不是在處理：

* 誰可以開始被拖
* 資料怎麼傳遞
* 放下後 DOM 怎麼更新

它比較像是在 HTML 層面提供一種：

> **「這裡是一個拖放區，而且它偏向某種放下語意」** 的宣告方式。

---

## 三個值原本怎麼理解

## 1. `copy`

表示放下後，偏向理解成「**建立副本**」。

重點是：

* 目標端取得的是一份拷貝
* 原資料仍保留在原位置

你可以把它理解成：

> 放下後比較像「複製一份過來」

---

## 2. `move`

表示放下後，偏向理解成「**移動原資料**」。

重點是：

* 原資料從舊位置轉移到新位置
* 不是保留原本再多一份副本

你可以把它理解成：

> 放下後比較像「搬過來」

---

## 3. `link`

表示放下後，偏向理解成「**建立連結或引用**」。

重點不是拿到獨立副本，而是保留和原資料之間的關聯。

你可以把它理解成：

> 放下後不是複製本體，而是建立一種指向原資料的關係

---

## 但最重要的一句是

這三個值描述的是：

> **操作語意**

不是：

> **只要寫上去，瀏覽器就會自動替你完成整套拖放邏輯**。

---

## 為什麼現代實作不再依賴 `dropzone`

這是整篇最重要的地方。

學 `dropzone`，不是為了把它背成熟練 API；而是要知道它今天**不再是現代拖放主線**。
你的學習重點應該放在：

* `dragenter`
* `dragover`
* `drop`
* `dataTransfer`

也就是事件式的拖放流程。

更具體地說，現代拖放要處理的事情包括：

* 目標區是否允許放下
* 拖曳過程中帶了什麼資料
* 放下時如何解析資料
* 要做複製、移動、排序、刪除，還是建立關聯

這些都不是單靠 `dropzone="copy"` 就能可靠完成的。

所以真正的結論是：

> `dropzone` 比較像歷史理解知識；
> 現代可用的 drop target，核心在事件，不在這個屬性。

---

## 如果看到教材在教 `dropzone`，該怎麼判斷

你可以用下面這個順序想：

### 1. 它是不是用 HTML 屬性直接聲明拖放區？

如果是，那它大多是在講舊的宣告式設計思路。

### 2. 它是不是把 `copy`、`move`、`link` 當成核心語法？

如果是，就要意識到：這通常偏向歷史知識，不是現代主線。

### 3. 它有沒有進一步談 `dragover`、`drop`、`dataTransfer`？

如果沒有，那它很可能只講到舊觀念，還沒進入真正可用的現代拖放實作。

### 4. 你的目標是讀懂舊資料，還是做新專案？

* 想讀懂舊教材：知道 `dropzone` 是什麼，有價值
* 想做新專案：應優先學事件式 drop target

## 例子 1：教材中的基本寫法

```html
<div dropzone="copy"></div>
```

這段想表達的是：

* 這個 `<div>` 原本希望被視為一個放置目標
* 而且放下時偏向 `copy` 這種操作語意

它的價值在於幫你理解：

> `dropzone` 曾試圖把「拖放目標 + 操作語意」做成 HTML 的宣告式屬性

但不要把它理解成：

> 今天只要這樣寫，瀏覽器就會替你完成可靠的拖放功能。

---

## 例子 2：今天更合理的最小做法

```html
<div id="target">Drop here</div>

<script>
  const target = document.getElementById("target");

  target.addEventListener("dragover", (event) => {
    event.preventDefault();
  });

  target.addEventListener("drop", (event) => {
    event.preventDefault();

    // 在這裡讀取 event.dataTransfer
    // 再自行決定要 copy、move，還是建立 link
  });
</script>
```

這段真正想傳達的是：

* 現代 drop target 不是靠 `dropzone="copy"` 建立
* 而是讓目標元素在 `dragover` 階段**允許放下**
* 再在 `drop` 階段**自行處理資料與行為**

也就是說，今天的核心不是舊屬性，而是：

> **允許放下 + 在 drop 時自己決定怎麼處理**。

## 常見混淆點 / 易錯點

### 1. `dropzone` 不是 `draggable`

* `draggable` 處理拖曳來源
* `dropzone` 原本想處理放置目標

兩者不是同一件事。

---

### 2. `dropzone="copy"` 不等於拖放完成

它只是在舊設計中表達某種放下語意，
不代表瀏覽器會自動幫你完成資料傳遞、目標接收與 UI 更新。

---

### 3. 畫面看起來像拖放區，不代表真的能接收 drop

真正能不能放下，現代通常還要看 `dragover`、`drop` 等事件處理。

---

### 4. 不要把 `dropzone` 當成新專案主解法

重點不是研究它在少數情況下能不能勉強用，而是：

> **新專案不應依賴它作為可靠的拖放方案。**

---

### 5. `copy / move / link` 是語意，不是整套功能

這三個值告訴你的是「偏向哪種操作理解」，
不是完整拖放系統的保證。

---

## 為什麼這個過時屬性還值得學

這個問題很重要。

既然它不是現代主線，為什麼還要學？

因為你還是可能在下面這些情況遇到它：

* 舊教材
* 舊筆記
* 老文章
* 舊專案原始碼
* 講 HTML 歷史設計的資料

所以學它的目的不是：

> 以後大量拿來寫新功能

而是：

> 當你看到它時，知道它在歷史上原本想解決什麼問題，並且知道現代該怎麼替代它

這樣你的知識結構才完整。

---

## 一句話抓核心

`dropzone` 的重點不在於背 `copy`、`move`、`link`，
而在於知道：

> **它代表的是一條已退場的 HTML 宣告式拖放方案；今天真正該學的是事件式 drop target。**
