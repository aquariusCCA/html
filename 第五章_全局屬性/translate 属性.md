# translate 屬性

> 所屬章節：第五章｜全局屬性  
> 關鍵字：`translate`、翻譯提示、機器翻譯、`translate="yes"`、`translate="no"`、內容是否翻譯、全局屬性  
> 建議回查情境：想知道 `translate` 屬性是做什麼的、想分清 `translate="yes"` 和 `translate="no"` 的差別、想知道怎麼標示某段內容不應被翻譯

## 本節導讀

這篇整理 HTML 全局屬性 `translate` 的基本用途。  
它的核心任務，是標示某段內容在翻譯流程中應不應該被翻譯。

這個屬性不是拿來指定語言，也不是直接把內容翻譯成另一種語言。  
它更像是一個提示，告訴瀏覽器或相關處理流程：這段內容該保留原文，還是可以一起進入翻譯。

## 你會在這篇學到什麼

- `translate` 屬性是在做什麼
- `translate="yes"` 和 `translate="no"` 的差別
- 怎麼禁止單一元素內容被翻譯
- 怎麼讓整份頁面都不進入翻譯

## 30 秒複習入口

- `translate` 用來標示元素內容是否應被翻譯。
- `translate="yes"` 代表允許翻譯，`translate="no"` 代表不要翻譯。
- 你可以只套在單一元素上，也可以套在 `<html>` 上影響整份文件。
- `translate` 處理的是「要不要翻譯」，不是「內容屬於哪種語言」；語言通常看 `lang`。

## 速查區

### 核心概念

- `translate` 是 HTML 全局屬性，可用來提示元素內容是否應被翻譯。
- 它主要在處理翻譯流程中的保留與排除，而不是內容本身的語言標記。

### 關鍵規則 / 判準

- 常見值是 `translate="yes"` 與 `translate="no"`。
- `yes` 代表該內容可以被翻譯，`no` 代表該內容不應被翻譯。
- `translate` 可以設在單一元素上，也可以設在 `<html>` 上影響整份文件。

### 常見使用方式

- 保留品牌名、作品名、專有名詞不被翻譯。
- 指定某一段英文標題或書名保持原文。
- 在 `<html>` 上標示整份頁面不應進入翻譯流程。

### 常見混淆點

- `translate` 不是翻譯工具，它只是標示是否應翻譯。
- `translate` 不是語言設定；如果要標示內容是中文還是英文，通常看 `lang`。
- 設了 `translate="no"` 不代表畫面外觀會改變，它主要影響的是翻譯相關處理邏輯。

### 一句話對比

- `lang` 回答的是「這段內容是什麼語言」，`translate` 回答的是「這段內容要不要被翻譯」。

## 正文筆記

### 這篇在解決什麼問題？

- 有些內容雖然出現在可翻譯的頁面裡，但不適合被翻成別的語言，例如品牌名、作品名、固定術語。
- `translate` 就是在處理這類需求：讓你能明確標示某段內容該不該被翻譯。

## 1. `translate` 的作用

> `translate` 屬性規定元素內容是否要翻譯。

- 可以把它理解成「翻譯流程中的保留開關」。
- 當你希望某段內容保持原文時，就可以透過這個屬性表達意圖。
- 這讓頁面在面對翻譯工具或相關流程時，能更清楚地區分哪些內容應保留原貌。

## 2. `translate` 的值怎麼看

### `translate="yes"`

- 代表這段內容可以被翻譯。
- 如果沒有特別要排除的理由，可以把它理解成允許進入一般翻譯流程。

### `translate="no"`

- 代表這段內容不應被翻譯。
- 常見於作品名、品牌名、專有名詞，或你想保留原文呈現的區塊。

## 3. 禁止單一元素翻譯

```html
<body>
  <p>
    The question in the
    <cite translate="no">How Far Can You Go?</cite> applies to...
  </p>
</body>
```

### 這個例子在說什麼？

- 這段文字裡，只有 `<cite>` 包住的書名或作品名被標記為不要翻譯。
- 也就是說，外面的句子仍可視情況翻譯，但 `How Far Can You Go?` 應保留原文。
- 這很適合用在頁面局部內容中，只排除某一小段專有名稱的情境。

## 4. 讓整份文件都不翻譯

```html
<!DOCTYPE html>
<html translate="no">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
<body>
  <p>
    The question in the
    <cite>How Far Can You Go?</cite> applies to...
  </p>
</body>
</html>
```

### 這個例子在說什麼？

- 這裡把 `translate="no"` 直接設在 `<html>` 上。
- 這表示整份文件的內容都被標記為不應翻譯，而不是只限某一小段。
- 當整頁內容本來就必須保持原文時，這種做法比一段一段標示更直接。

## 5. `translate` 和 `lang` 怎麼分

- `translate` 處理的是要不要翻譯。
- `lang` 處理的是內容本身屬於哪一種語言。
- 所以一段內容可以同時有 `lang` 和 `translate`：前者說明語言，後者說明是否應進入翻譯流程。

## 6. 使用 `translate` 時的注意點

- 不要把它理解成「翻譯功能按鈕」；它只是標示翻譯意圖，不負責真的執行翻譯。
- 如果某段內容對品牌識別、作品名稱或專有名詞很重要，通常可以考慮 `translate="no"`。
- 如果你只是想標示語言，而不是控制翻譯，應優先考慮 `lang`。
