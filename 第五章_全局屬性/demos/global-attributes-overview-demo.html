全局屬性總覽 Demo

這個示例把第五章裡高頻、容易混用的全局屬性集中成一頁：鍵盤訪問、元素識別、自定義資料、可編輯內容、語言與方向， 再補上 `style`、`contextmenu`、`dropzone` 的定位。拖曳、隱藏、拼寫檢查則保留在既有專項 demo。

鍵盤訪問 `accesskey` 是快捷訪問提示，`tabindex` 是焦點規則，不是畫面排序工具。
識別與分組 `id` 偏唯一識別，`class` 偏重複分組，`data-*` 偏附加資料。
內容與語言 `contenteditable` 控制能不能直接改內容；`lang`、`dir`、`translate` 補語言語意。
歷史與補充 `style` 是行內樣式；`contextmenu`、`dropzone` 更適合當歷史語法知識辨識。

這頁怎麼看

  • 先用第一區分清 `accesskey`、`tabindex`、`title` 的角色差異。
  • 再點選商品卡片，看 `id`、`class`、`data-*` 各自在解決什麼問題。
  • 接著切換 `contenteditable` 與語言設定，觀察 HTML 全局屬性如何改變元素語意。
  • 最後看附錄區，把 `style`、`contextmenu`、`dropzone` 放回正確定位。

1. 鍵盤訪問與提示：`accesskey`、`tabindex`、`title`

快捷訪問提示

accesskey：提供補充性的鍵盤訪問入口

它不是固定熱鍵系統，而是向瀏覽器提示：這個元素可以有一條更快的鍵盤訪問路徑。

<button accesskey="s">送出</button>
不同瀏覽器與作業系統的觸發組合可能不同，所以 `accesskey` 比較像補充入口，而不是主要操作方案。
目前示例聚焦在 `accesskey`。請觀察按鈕上的提示字元與說明。

2. `id`、`class`、`data-*` 各自在做什麼

唯一識別

id：告訴你「這個元素是誰」

同一頁面裡，`id` 應該盡量保持唯一，方便 CSS、JavaScript 或片段連結精準找到它。

<article id="product-101" class="product-card featured" data-product-id="101" data-category="book">
  Java 書籍
</article>
Java 書籍

id: product-101 / class: product-card featured

IDE 工具

id: product-102 / class: product-card

前端課程

id: product-103 / class: product-card new

目前示例聚焦在 `id`。這種情況適合精準指向某一個元素，而不是整批元素。
目前選中的卡片：`product-101`
對應的 class：`product-card featured`
對應的 data-*：`data-product-id="101"`、`data-category="book"`

3. 可編輯內容與語言方向：`contenteditable`、`lang`、`dir`、`translate`

可編輯內容

contenteditable：讓一般元素內容可直接編輯

這不是表單欄位本身，而是把一般元素變成可直接改內容的區域。

<div contenteditable="true">
  這裡可以直接修改內容。
</div>
`contenteditable="true"` 代表可編輯；`plaintext-only` 則偏向只保留純文字；`false` 則可用來排除某塊內容。
這裡可以直接修改內容。你可以試著輸入文字，觀察這和一般 div 的差別。
目前狀態：`contenteditable="true"`
lang="en"

標示主要語言是英文。

dir="ltr"

文字方向目前是由左到右。

translate="no"

適合品牌名、專有名詞這類不希望被翻譯的內容。

4. `style`、`contextmenu`、`dropzone` 的定位

style

style：直接寫在元素上的行內樣式

它能快速給單一元素加樣式，但通常不適合大量取代 `class` 與外部 CSS 規則。

<p style="color: tomato; font-weight: 700;">
  這段文字帶有行內樣式
</p>

這段文字帶有行內樣式。

歷史語法提醒

contextmenu / dropzone：更適合當辨識知識，而不是現代主流方案

contextmenu

偏舊式 HTML 寫法。現代自訂右鍵選單通常改用 JavaScript 的 `contextmenu` 事件與自訂介面。

dropzone

屬於較舊的拖放屬性語境。現代多半直接使用 `dragenter`、`dragover`、`drop` 事件建立放置目標。
