# lang 屬性

> 所屬章節：第五章｜全局屬性
> 關鍵字：`lang`、語言代碼、language code、文件語言、元素語言、`zh-TW`、`en`、全局屬性
> 建議回查情境：想知道 `lang` 是做什麼的、想知道整份文件的語言怎麼標示、想知道局部不同語言內容怎麼寫、想分清 `lang` 和 `dir` 的差別

## 本節導讀

`lang` 是 HTML 的全局屬性之一。
它的作用是：**標示這段內容使用的是哪一種語言。**

它不是翻譯工具，也不是拿來控制文字方向。
它比較像是在替瀏覽器、輔助技術、搜尋系統或其他處理流程補上一個語言提示，讓系統知道：

* 這整份文件主要是什麼語言
* 某一小段內容是不是用了不同語言
* 這段文字應該被當成哪種語言來理解

你可以先記一句最核心的話：

> `lang` 回答的是「這段內容是什麼語言」，不是「這段內容怎麼排版」或「要不要翻譯」。

* 這個資訊不一定會立刻讓畫面外觀改變，但對系統判讀內容很重要。

* `lang` 不是翻譯功能

## 這篇在解決什麼問題？

同一個頁面不一定只會出現一種語言。

例如：

* 整份頁面是繁體中文
* 其中某個按鈕文字是英文
* 某段引用內容是日文
* 某個產品名稱保留原文

如果你沒有明確標示語言，系統只能自己猜。
而 `lang` 就是在解決這件事：

> 讓 HTML 明確告訴系統，這段內容屬於哪一種語言。

---

# 1. `lang` 是什麼？

`lang` 屬性用來指定元素內容的語言。

你可以把它理解成：

> 幫這段內容貼上一個語言標籤。

例如：

```html
<p lang="en">This is a paragraph.</p>
```

這表示這個 `<p>` 裡的內容是英文。

重點不是它看起來一定會不一樣，
而是頁面已經明確指出：

> 這段內容的語言是英文。

---

# 2. `lang` 的值怎麼寫？

`lang` 的值通常是語言代碼。

常見例子：

* `en`：英文
* `zh-TW`：繁體中文
* `zh-CN`：簡體中文
* `ja`：日文
* `ko`：韓文

## 怎麼理解這些代碼？

可以先用最實用的方式記：

* 只有語言時，常寫：

  * `en`
  * `ja`
  * `fr`
* 如果還要更明確標出地區，常會寫：

  * `zh-TW`
  * `zh-CN`
  * `en-US`

所以你可以先記一個原則：

> `lang` 的值通常是「語言代碼」，有需要時再補上地區代碼。

---

# 3. 最常見的做法：先標整份文件的語言

這是 `lang` 最重要、也最常見的用法。

```html
<html lang="zh-TW">
```

### 這段怎麼理解？

* 這表示：整份 HTML 文件的主要語言是繁體中文。
* 如果頁面大部分內容都是繁體中文，這樣寫最合理。

所以在實務上，最常見的第一步不是先去標某個 `<p>`，
而是先在根元素 `<html>` 上設定整份文件的主要語言。

---

# 4. 局部內容語言不同時怎麼辦？

如果整頁主要是中文，但某一小段是英文，
你可以只在那一小段上單獨設定 `lang`。

## 例子

```html
<html lang="zh-TW">
  <body>
    <p>這是一段中文內容，<span lang="en">Hello</span> 是英文。</p>
  </body>
</html>
```

### 這段怎麼理解？

* 整份頁面主要語言是繁體中文
* 但 `Hello` 這一小段被明確標成英文
* 所以你不是把整頁改成英文，而是只標示局部不同語言的部分

這就是 `lang` 很實用的地方：

> 可以先標整體，再對局部不同語言內容做補充標示。

---

# 5. `lang` 最常出現在哪些地方？

## 用法一：標整份文件

```html
<html lang="zh-TW">
```

這表示整份文件主要是繁體中文。

---

## 用法二：標局部不同語言內容

```html
<p>請點擊 <span lang="en">Submit</span> 按鈕。</p>
```

這表示整句主要是中文，但 `Submit` 是英文。

---

## 用法三：多語內容切換時補充標示

```html
<article lang="ja">
  <p>これは日本語の段落です。</p>
</article>
```

這表示這整個區塊是日文內容。

---

# 6. `lang` 不是什麼？

這也是初學者最容易誤會的地方。

## 它不是翻譯功能

```html
<p lang="en">這是一段中文</p>
```

這樣不會把中文變成英文。
`lang` 只是在標示語言，不會自動翻譯內容。

---

## 它不是文字方向設定

如果你想處理的是：

* 左到右
* 右到左

那通常要看的是：

```html
dir="ltr"
dir="rtl"
dir="auto"
```

不是 `lang`。

---

## 它不是樣式工具

`lang` 的重點不是讓文字變顏色、變大小、靠左或靠右。
它的核心用途是補充「語言資訊」。

---

# 7. `lang` 和 `dir` 怎麼分？

這兩個都和文字有關，所以很容易混淆。

## `lang`

* 處理的是語言
* 回答的是：

  * 這段內容是中文嗎？
  * 是英文嗎？
  * 是日文嗎？

## `dir`

* 處理的是文字方向
* 回答的是：

  * 這段內容由左到右嗎？
  * 還是由右到左？

### 一句話記法

> `lang` 看「是哪種語言」
> `dir` 看「往哪個方向書寫」

所以：

* 中文 / 英文 / 日文 → 想 `lang`
* 左到右 / 右到左 → 想 `dir`

---

# 8. 基本示例

## 例子 1：單一元素標記語言

```html
<p lang="en">This is a paragraph.</p>
```

### 這段怎麼理解？

* 這個 `<p>` 的內容被標示為英文
* 重點不是畫面一定不同
* 而是 HTML 已經明確指出這段內容的語言

---

## 例子 2：整份文件是繁體中文

```html
<html lang="zh-TW">
  <head>
    <title>我的網站</title>
  </head>
  <body>
    <p>這是一段繁體中文內容。</p>
  </body>
</html>
```

### 這段怎麼理解？

* 整份頁面主要語言是繁體中文
* 所以內容若沒有另外指定，通常就會被當成繁體中文環境中的內容

---

## 例子 3：整頁中文、局部英文

```html
<html lang="zh-TW">
  <body>
    <p>請按下 <span lang="en">Start</span> 開始。</p>
  </body>
</html>
```

### 這段怎麼理解？

* 頁面主語言是中文
* 但 `Start` 這個字被單獨標成英文
* 這是局部語言切換最典型的寫法

# 9. HTML4 和 HTML5 怎麼看？

如果你正在學現代 HTML，先抓住這個重點就夠了：

> 在 HTML5 裡，`lang` 是全局屬性，可以廣泛使用在各種元素上。

你原本筆記提到 HTML4 中某些元素不能使用 `lang`，
這屬於版本差異補充，知道即可。
對現在的學習來說，真正重要的是：

* 把 `lang` 當成全局屬性理解
* 知道最常設在 `<html>`
* 知道局部內容也能單獨設

# 10. 常見混淆點 / 易錯點

## 錯誤一：以為 `lang` 會自動翻譯文字

不對。

`lang` 只是標示語言，不會替你翻譯內容。

---

## 錯誤二：把 `lang` 和 `dir` 當成同一件事

不對。

* `lang`：內容是什麼語言
* `dir`：內容往哪個方向寫

---

## 錯誤三：只知道可以標單一元素，不知道整頁通常先從 `<html>` 設起

不夠完整。

實務上最常見的第一步，通常是：

```html
<html lang="...">
```

---

## 錯誤四：以為設了 `lang`，畫面一定要有明顯外觀變化

不對。

它可能不會立刻讓你看到很大的外觀差異，
但它仍然是在提供很重要的語言資訊。

---

# 11. 一句話總結

`lang` 的本質是：

> 用 HTML 明確標示內容所屬的語言，可以標整份文件，也可以標局部元素；它處理的是語言，不是方向，也不是翻譯。
