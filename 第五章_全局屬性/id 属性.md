# id 屬性

> 所屬章節：第五章｜全局屬性  
> 關鍵字：`id`、唯一識別、unique identifier、CSS 選擇器、JavaScript 選取、片段連結、fragment  
> 建議回查情境：想確認 `id` 的作用、想分清 `id` 和 `class` 的差別、想知道為什麼同一頁面中的 `id` 應該唯一

## 本節導讀

這篇整理 HTML 全局屬性 `id` 的基本用途。  
`id` 最核心的作用，是替某個元素建立一個在目前 HTML 文件中應保持唯一的識別名稱。

這個屬性很常出現在兩類情境。  
第一類是用 CSS 或 JavaScript 精準選到某個元素；第二類是把頁面中的某個位置當成可跳轉的目標，例如錨點連結。

## 30 秒複習入口

- `id` 用來唯一識別一個元素。
- 同一份 HTML 文件中，`id` 應避免重複。
- `id` 常被 CSS、JavaScript 與頁面片段連結拿來定位元素。

## 速查區

### 核心概念

- `id` 是 HTML 全局屬性，可用在多數元素上。
- 它的任務是替元素提供一個可被精準指向的名稱。

### 關鍵規則 / 判準

- 同一份 HTML 文件中，`id` 應該唯一。
- 一個元素只能有一個 `id` 值。
- `id` 常被 CSS 的 `#id` 選擇器與 JavaScript 的 `getElementById()` 拿來選取元素。

### 常見使用方式

- 用 CSS 針對單一元素套樣式。
- 用 JavaScript 找到特定元素並綁定事件或更新內容。
- 用連結跳到頁面中的特定區塊。

### 常見混淆點

- `id` 和 `class` 不一樣：`id` 偏單一識別，`class` 偏分組與重複使用。
- `id` 不是拿來表示「這個元素是哪一類」，而是拿來表示「這個元素是誰」。
- 如果同一頁面重複使用相同 `id`，CSS 和 JavaScript 的行為會變得難以判讀與維護。

### 一句話對比

- `id` 像元素的身分證號，`class` 比較像元素所屬的群組標籤。

## 正文筆記

### 這篇在解決什麼問題？

- 當頁面上有很多元素時，需要一種方式能精準指到其中某一個。
- `id` 就是在解決「如何唯一識別這個元素」這件事。

## 1. `id` 是什麼

> `id` 屬性用來指定 HTML 元素的唯一識別名稱，而且在同一份 HTML 文件中應保持唯一。

- 可以把它理解成替元素命名，但這個名字的重點不是好看，而是方便精準定位。
- 也因為它是全局屬性，所以能套在多數元素上。

## 2. 為什麼要唯一

- 如果同一頁面有兩個元素都叫同一個 `id`，就失去「唯一定位」的意義。
- 這會讓 CSS 選取、JavaScript 操作與頁面跳轉都更容易產生混淆。
- 所以實務上會把 `id` 當成「單一元素專用名稱」，而不是可重複套用的分類名稱。

## 3. 基本示例

```html
<body>
  <p id="name">這是一個帶有 id 的段落</p>
</body>
```

### 這段怎麼理解？

- 這個 `<p>` 被指定了一個 `id="name"`。
- 之後如果你想精準選到這個段落，就可以透過這個 `id` 來定位。
- 這也是原始教材最想表達的重點：`id` 讓某個元素在文件中有一個可被明確指到的名稱。

## 4. `id` 常拿來做什麼

### CSS 選取

```css
#name {
  color: steelblue;
}
```

- `#name` 代表「選取 `id="name"` 的元素」。
- 這種寫法通常用來針對單一元素做樣式設定。

### JavaScript 選取

```html
<script>
  const paragraph = document.getElementById("name");
</script>
```

- 這段程式會取得 `id="name"` 的元素。
- 取得之後，就可以繼續改文字、綁事件或調整樣式。

## 5. `id` 和 `class` 怎麼分

- `id` 偏向唯一識別，用在「這一個元素就是我要找的目標」。
- `class` 偏向分組，用在「這些元素屬於同一類型或要共用同一組樣式」。
- 如果你想表示一組元素的共同特徵，通常比較適合用 `class`；如果你想精準點名某一個元素，通常比較適合用 `id`。
