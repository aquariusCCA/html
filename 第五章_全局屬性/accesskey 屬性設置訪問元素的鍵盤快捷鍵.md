# accesskey 屬性設置訪問元素的鍵盤快捷鍵

## 本節導讀

這篇整理 `accesskey` 這個全局屬性的基本用途與使用方式。  
它的核心目的是替頁面上的某些元素指定快捷鍵，讓熟悉網站操作的人可以更快地訪問常用功能。

不過，`accesskey` 真正使用時要特別注意瀏覽器與作業系統差異，因為快捷鍵組合不一定完全一致。

## 30 秒複習入口

- `accesskey` 可以替元素指定一組快捷鍵，方便快速訪問常用功能。
- 在 `HTML5` 中，`accesskey` 可用於任何 HTML 元素，但實際上不一定每種元素都適合設定。
- 在 `HTML4` 中，`accesskey` 的可用元素範圍比較有限。
- 實際觸發方式會因作業系統與瀏覽器不同而有差異，不能只背一種按法。

## 速查區

### 核心概念

- `accesskey` 是用來替元素設定鍵盤快捷鍵的屬性。
- 它的重點不是改變元素功能，而是提供一條更快的訪問路徑。

### 關鍵規則 / 判準

- 基本語法是 `<element accesskey="character">`。
- `accesskey` 在 `HTML5` 中屬於全局屬性，因此理論上可以寫在任何 HTML 元素上。
- 雖然幾乎任何元素都能寫 `accesskey`，但不是每種元素都適合設置，應看實際操作情境。

### 常見混淆點

- `accesskey` 不是只給連結用的，它可以用在更多元素上。
- `accesskey` 不是所有瀏覽器都用完全相同的按鍵組合觸發。
- 能設定 `accesskey` 不代表一定值得設定，還要考慮是否容易記、是否和系統或瀏覽器快捷鍵衝突。

### 一句話對比

- `accesskey` 解決的是「如何更快找到並觸發元素」，不是「元素本身做什麼」。

## 正文筆記

### 這篇在解決什麼問題？

- 當網站中有一些經常被使用的元素時，使用者可能希望透過鍵盤更快速地訪問它們。
- `accesskey` 就是在處理這類需求：替元素指定快捷鍵，讓熟悉頁面的人可以少一次滑鼠操作。

## 1. `accesskey` 的作用

> 使用 `accesskey` 可以為一個或多個頁面元素設定快捷方式。它的目的是讓熟悉網頁或網站操作的使用者，可以用快捷鍵快速訪問經常用到的元素。

- 在某些教材裡，會把 Windows 下的常見按法寫成 `Alt + accesskey 的屬性值`。
- 但這只能當作入門理解，不能視為所有環境都完全相同的固定規則。

### 注意點

- 不同作業系統、不同瀏覽器，觸發 `accesskey` 的按鍵組合可能不同。
- 因此實際使用時，應把它理解成「瀏覽器提供的元素快捷訪問機制」，而不是死背單一按鍵組合。

## 2. `accesskey` 的適用範圍

- 在 `HTML5` 中，`accesskey` 屬性可用於任何 HTML 元素。
- 不過，能驗證通過不代表一定有實際使用價值，還是要看元素是否真的需要快捷訪問。
- 在 `HTML4` 中，`accesskey` 可使用於：`<a>`、`<area>`、`<button>`、`<input>`、`<label>`、`<legend>`、`<textarea>`。

## 3. 語法

```html
<element accesskey="character">
```

### 這段怎麼理解？

- `element` 代表你要設定快捷鍵的 HTML 元素。
- `character` 代表你希望綁定的快捷鍵字符。
- 實際按下去後會發生什麼事，仍取決於元素本身的功能，例如連結會跳轉、按鈕可能會觸發操作。

## 4. 用法示例

```html
<!-- 某些環境下可透過對應的 accesskey 快速聚焦或觸發這些連結。 -->
<a href="http://www.baidu.com" accesskey="a">百度</a>
<a href="http://www.sina.com" accesskey="g">新浪</a>
```

### 這個例子在說什麼？

- 這裡分別為兩個連結設定了不同的 `accesskey` 值。
- 使用者在支援 `accesskey` 的環境中，可以透過對應快捷鍵更快訪問這些連結。
- 不過，是否直接跳轉、先聚焦還是需要再確認，仍可能受到瀏覽器行為影響。
