# translate 屬性

> 所屬章節：第五章｜全局屬性  
> 關鍵字：`translate`、翻譯提示、機器翻譯、在地化、`translate="yes"`、`translate="no"`、文字子節點、可翻譯屬性值、全局屬性  
> 建議回查情境：想知道 `translate` 屬性真正控制的是什麼、想分清 `translate="yes"` 和 `translate="no"` 的差別、想釐清它和 `lang` 的用途邊界

## 本節導讀

這篇重構的重點，是把 `translate` 從「要不要翻譯」的口語說法，整理成更穩定的技術理解。  
原始教材的方向沒有錯，但如果只停在那一句，讀者很容易把它學成像翻譯按鈕開關，卻不知道它實際上是在對翻譯流程提供提示。

更精確地說，`translate` 不是直接執行翻譯，也不是標示語言種類。  
它是在告訴在地化流程、自動翻譯工具或相關處理機制：這個元素裡可翻譯的內容，是否應該被翻譯。

## 這篇在解決什麼問題

- 有些頁面會進入翻譯或在地化流程，但其中某些內容其實應該保留原文，例如品牌名、作品名、產品型號、固定術語。
- `translate` 就是在處理這類需求：讓你能明確標示某段內容在翻譯流程中應不應該被改動。
- 但如果只把它理解成「yes / no 開關」，會漏掉兩個核心觀念：它是列舉屬性，而且它影響的不只是表面文字，還包括元素中可翻譯的屬性值與文字子節點。

## 先講結論 / 核心理解

先記住一件事：`translate` 是對翻譯流程的提示，不是翻譯功能本身。  
最穩定的理解方式是：

- `translate` 是列舉屬性。
- 空字串或 `yes` 表示這段內容在頁面被在地化時可翻譯。
- `no` 表示這段內容不應被翻譯。
- 它主要影響元素中的可翻譯屬性值與文字子節點。
- 它回答的是「這段內容要不要翻譯」，不是「這段內容屬於哪種語言」；後者通常看 `lang`。

## 核心定義 / 規則

## 1. `translate` 在控制什麼

`translate` 控制的是：  
當頁面進入翻譯或在地化流程時，某個元素中的可翻譯內容是否應被翻譯。

這裡的「內容」不要只理解成畫面上看到的句子。  
更完整的理解是：

- 元素中的文字子節點
- 以及該元素中屬於可翻譯範圍的屬性值

所以它影響的是翻譯處理邏輯，而不是畫面樣式或語言標記本身。

## 2. 它是列舉屬性，不只是口語上的 yes / no

### 空字串或 `translate="yes"`

- 表示這個元素中的可翻譯內容可以被翻譯。
- 實務理解上，可以把它視為允許進入一般翻譯流程。

### `translate="no"`

- 表示這個元素中的可翻譯內容不應被翻譯。
- 常見於品牌名、作品名、產品名、專有名詞或希望保留原文的固定片段。

## 3. 它不是語言標記

- `translate` 不負責說明這段內容是中文、英文還是日文。
- 如果你要標示語言，通常應使用 `lang`。
- `translate` 只是在說：這段內容進入翻譯流程時，應不應該被改動。

## 判斷條件或推理過程

如果你在判斷某段內容要不要加 `translate="no"`，可以先用這個順序想：

1. 這段內容是不是品牌名、作品名、產品型號或固定術語？
2. 如果被翻掉，會不會破壞識別、引用準確性或原本表意？
3. 如果答案是會，那它通常比一般正文更適合標成 `translate="no"`。
4. 如果你只是想標示這段文字是什麼語言，應優先考慮 `lang`，不是 `translate`。

## 例子說明

### 例子 1：只禁止局部內容翻譯

```html
<body>
  <p>
    The question in the
    <cite translate="no">How Far Can You Go?</cite> applies to...
  </p>
</body>
```

- 這裡只有 `<cite>` 中的作品名被標成不應翻譯。
- 外面的句子仍然可以進入一般翻譯流程。
- 這很適合拿來保留局部專有內容的原文。

### 例子 2：整份文件都不翻譯

```html
<!DOCTYPE html>
<html translate="no">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
<body>
  <p>
    The question in the
    <cite>How Far Can You Go?</cite> applies to...
  </p>
</body>
</html>
```

- 這裡把 `translate="no"` 直接設在 `<html>` 上。
- 這代表整份文件的可翻譯內容都應保留原文。
- 如果整個頁面本來就不希望被在地化，這種做法比逐段標記更直接。

## 常見混淆點 / 易錯點

- `translate` 不是翻譯工具；它不會自己把內容翻成別的語言。
- `translate` 不是語言設定；如果要標示語言，通常看 `lang`。
- `translate="no"` 不代表畫面外觀會改變；它主要影響的是翻譯相關流程。
- 不要把它學成單純對「看得見的文字」生效；更穩妥的理解是，它作用在元素中的可翻譯內容。

## 一句話抓核心

`translate` 的關鍵不在於背 `yes` 或 `no`，而在於理解它是在告訴翻譯流程「這段可翻譯內容該不該被改動」，而不是在標示語言或直接執行翻譯。
