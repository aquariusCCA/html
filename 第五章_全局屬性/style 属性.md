# style 屬性

> 所屬章節：第五章｜全局屬性  
> 關鍵字：`style`、行內樣式、inline style、CSS、樣式覆蓋、優先級  
> 建議回查情境：想確認 `style` 是做什麼的、想知道行內樣式和外部 CSS 的差別、想理解為什麼 `style` 常會覆蓋其他樣式設定

## 本節導讀

這篇整理 HTML 全局屬性 `style` 的基本用途。  
`style` 的核心作用，是直接在元素上寫 CSS 樣式，也就是常說的行內樣式。

它的好處是直觀、立即、容易看到效果。  
但也因為樣式直接綁在元素上，所以常會蓋過其他較一般的樣式來源，並讓樣式維護變得比較分散。

## 30 秒複習入口

- `style` 用來直接在元素上寫行內樣式。
- 行內樣式通常會覆蓋一般的外部樣式表或 `<style>` 中的普通規則。
- 它適合快速指定單一元素樣式，但不適合大量重複與長期維護。

## 速查區

### 核心概念

- `style` 是 HTML 全局屬性。
- 它的任務是讓某個元素直接帶有 CSS 樣式。

### 關鍵規則 / 判準

- `style` 寫的是 CSS 聲明，例如 `color: red;`。
- 寫在 `style` 屬性中的樣式，通常會比一般外部樣式或 `<style>` 規則更直接地作用在該元素上。
- `style` 只影響當前元素，不會像 class 一樣自然形成可重複共用的樣式群組。

### 常見使用方式

- 臨時指定單一元素的顏色、對齊方式、背景或尺寸。
- 教學示例中快速展示 CSS 效果。
- 用 JavaScript 動態改變單一元素的樣式。

### 常見混淆點

- `style` 不是 `<style>` 標籤；前者是屬性，後者是寫整段 CSS 規則的元素。
- `style` 不是外部樣式表；它只作用在當前元素上。
- `style` 雖然方便，但如果大量使用，會讓樣式分散在 HTML 裡，不容易統一管理。

### 一句話對比

- `style` 是把樣式直接寫在元素身上，`class` 則是先幫元素貼標籤，再由 CSS 統一決定樣式。

## 正文筆記

### 這篇在解決什麼問題？

- 有時你只想快速替某個元素加上樣式，而不想另外寫一條 CSS 規則。
- `style` 就是在解決「如何直接對這個元素指定樣式」這件事。

## 1. `style` 是什麼

> `style` 屬性用來規定元素的行內樣式（inline style）。

- 可以把它理解成「直接把 CSS 寫在這個元素上」。
- 也因為它是全局屬性，所以能用在多數 HTML 元素上。

## 2. 為什麼它常會蓋過其他樣式

- 原始教材提到，`style` 屬性通常會覆蓋一般的全局樣式設定。
- 實際上，這代表如果某個元素同時受到外部樣式表、`<style>` 規則和 `style` 屬性影響，行內樣式往往更直接。
- 所以當你發現某個元素「明明有 class 樣式卻沒生效」，就要檢查它是不是另外寫了 `style`。

## 3. 基本示例

```html
<body>
  <h1 style="color: blue; text-align: center;">This is a header</h1>
  <p style="color: red;">This is a paragraph.</p>
</body>
```

### 這段怎麼理解？

- 第一個 `<h1>` 直接把文字顏色設成藍色，並把文字置中。
- 第二個 `<p>` 直接把文字顏色設成紅色。
- 這裡最重要的不是背屬性名稱，而是看懂：樣式直接寫在元素上，不需要另外透過 class 或外部 CSS 才能生效。

## 4. 什麼時候適合用

- 在教學或示例中，想快速展示某個 CSS 效果時，常會直接用 `style`。
- 在只影響單一元素、且樣式非常簡短的情況下，也可能會用到。
- 但如果同一組樣式會重複出現，通常更適合改用 class 和外部 CSS 管理。

## 5. `style` 和 `class` 怎麼分

- `style` 偏直接指定樣式結果。
- `class` 偏替元素貼分類標記，再由 CSS 規則統一決定樣式。
- 如果你要的是「這個元素現在就長這樣」，`style` 很直觀；如果你要的是「一批元素共用同一套樣式」，通常 `class` 更適合。
