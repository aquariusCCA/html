# tabindex 屬性：控制元素是否可被聚焦，以及 Tab 鍵移動順序

> 所屬章節：第五章｜全局屬性  
> 關鍵字：`tabindex`、tab order、focus order、焦點順序、鍵盤導航、可聚焦、`tabindex="0"`、`tabindex="-1"`、全局屬性、無障礙  
> 建議回查情境：想知道 `tabindex` 是做什麼的、想分清 `tabindex="0"`、`tabindex="-1"` 和正整數的差別、想知道按 `Tab` 鍵時元素順序怎麼控制

---

## 本節導讀

學 `tabindex` 時，最容易搞混的地方不是語法，而是「焦點」到底是什麼。

你可以先把焦點理解成：

> **目前鍵盤操作停留在哪個元素上。**

當某個元素取得焦點後，接下來使用者按 `Tab`、`Enter`、`Space` 或其他鍵盤操作時，通常都會先影響那個元素。

而 `tabindex` 的主要工作，就是處理兩件事：

1. 某個元素能不能被鍵盤聚焦
2. 使用者按 `Tab` 鍵時，焦點大致會按照什麼順序移動

所以，`tabindex` 解決的是**鍵盤導航問題**，不是畫面排列問題，也不是元素功能問題。

## 先講結論

先記住一句話：

> `tabindex` 是拿來控制元素的鍵盤焦點行為，不是拿來隨便重排整個頁面的工具。

因此你要先抓住三個核心：

- 正整數：可以自訂 `Tab` 順序，但不建議大量使用
- `tabindex="0"`：讓元素進入自然的鍵盤焦點順序
- `tabindex="-1"`：讓元素不參與一般 `Tab` 導航，但仍可被程式聚焦

### 一句話總整理

- `tabindex` 管的是「焦點怎麼移動」
- 不是「元素長什麼樣」
- 也不是「元素放在哪裡」

- `tabindex` 不是改變畫面排列，而是改變鍵盤焦點順序

## 1. `tabindex` 在做什麼？

`tabindex` 是 HTML 的全局屬性，用來控制元素的焦點行為。

它主要處理的是：

- 元素能不能被鍵盤聚焦
- 使用者按 `Tab` 時，焦點移動的大致順序

例如，當頁面上有：

- 連結
- 按鈕
- 輸入框
- 下拉選單
- 自訂互動區塊

使用者可能會透過 `Tab` 鍵在這些元素之間移動。  
而 `tabindex` 就是在必要時調整這種移動方式。

---

## 2. 什麼叫「聚焦」？

如果你對「聚焦」這個詞還不熟，可以這樣理解：

> **聚焦 = 鍵盤目前停留的位置。**

例如你按 `Tab` 鍵後，某個按鈕外框亮起來，代表它取得焦點。  
這時候如果再按 `Enter` 或 `Space`，通常就會作用在這個按鈕上。

所以 `tabindex` 的本質，其實是在控制：

> 哪些元素可以成為鍵盤目前操作的目標，以及它們輪到的先後順序。

---

## 3. 基本語法

```html
<element tabindex="number">
```

### 怎麼理解？

* `element`：任意 HTML 元素
* `number`：你指定的焦點行為值

重點不是死背語法，而是理解：

> 不同數值，代表不同的焦點規則。

---

## 4. `tabindex` 的三種常見值

## 4-1. 正整數：自訂焦點順序

例如：

```html
<a href="http://www.google.com/" tabindex="1">Google</a>
<a href="http://www.w3school.com.cn/" tabindex="2">W3School</a>
<a href="http://www.microsoft.com/" tabindex="3">Microsoft</a>
```

這表示當使用者按 `Tab` 時，焦點通常會先到：

1. `tabindex="1"` 的 Google
2. `tabindex="2"` 的 W3School
3. `tabindex="3"` 的 Microsoft

### 怎麼記？

* 正整數 = 你手動指定順序
* 數值越小，通常越先被聚焦

### 為什麼不建議大量使用？

因為一旦頁面元素變多，就會出現幾個問題：

* 新增元素時，可能要重新調整很多數字
* 刪除元素時，順序可能變亂
* 焦點順序可能和畫面上的視覺順序不一致
* 維護成本很高，使用者也容易混亂

所以正整數比較適合：

* 教學示範
* 少量特殊控制

不適合拿來大規模重排整頁操作順序。

---

## 4-2. `tabindex="0"`：進入自然 Tab 順序

例如：

```html
<div tabindex="0">這是一個可以被 Tab 聚焦的區塊</div>
```

這代表：

* 這個元素可以被鍵盤聚焦
* 它會進入一般的 `Tab` 導航流程
* 它通常會按照文件原本的自然順序被走到，而不是插隊到最前面

### 這個值常用在哪裡？

常見於原本不是可聚焦元素，但你希望它能被鍵盤使用者走到的情境。

例如：

* 自訂互動區塊
* 可點擊卡片
* 自訂元件容器

### 怎麼記？

* `0` = 可聚焦，而且跟著自然順序走
* 它不是硬排順序，而是加入正常流程

---

## 4-3. `tabindex="-1"`：不參與一般 Tab 導航

例如：

```html
<div tabindex="-1">這個元素通常不會被 Tab 走到</div>
```

這表示：

* 使用者平常按 `Tab` 時，通常不會走到這個元素
* 但它仍然可以被 JavaScript 或其他方式聚焦

### 什麼意思？

它不是「永遠不能聚焦」，而是：

> **不參與一般 Tab 巡覽，但仍保留被程式指定焦點的可能。**

### 常見情境

例如打開對話框、提示區塊、錯誤訊息區時，你可能希望：

* 使用者平常不要 Tab 到它
* 但在特定時機，由程式主動把焦點移過去

像這樣：

```javascript
element.focus();
```

### 怎麼記？

* `-1` = 平常 Tab 不會走到
* 但程式仍然可以把焦點移過去

---

## 5. 例子：教材中的基本示範

```html
<body>
  <a href="http://www.w3school.com.cn/" tabindex="2">W3School</a>
  <a href="http://www.google.com/" tabindex="1">Google</a>
  <a href="http://www.microsoft.com/" tabindex="3">Microsoft</a>
</body>
```

### 這個例子在說什麼？

它的重點不是連結本身，而是示範：

* `tabindex` 可以直接影響 `Tab` 鍵時的焦點先後順序
* 焦點順序不一定跟原始碼出現順序一樣
* 正整數可以強制排序

### 讀這個例子時要注意什麼？

這是很典型的教學示範。
它方便你理解規則，但不代表實務上應該大量這樣做。

因為真正的頁面一旦變複雜，這種手動排號方式會越來越難維護。

---

## 6. `tabindex` 的適用範圍

### HTML5

在 HTML5 中，`tabindex` 是全局屬性，理論上可以寫在任何 HTML 元素上。

但要注意：

> 能寫，不代表每個元素都適合寫。

---

### HTML4

在 HTML4 中，`tabindex` 屬性可用於：

* `<a>`
* `<area>`
* `<button>`
* `<input>`
* `<object>`
* `<select>`
* `<textarea>`

---

### 學習時怎麼抓重點？

現在學習時先記：

> 在 HTML5 中，`tabindex` 可以視為全局屬性。

而 HTML4 的範圍差異，知道是歷史背景即可。

---

## 7. 使用 `tabindex` 時的注意點

### 1）優先相信原生元素的自然焦點順序

像：

* `<a>`
* `<button>`
* `<input>`
* `<select>`
* `<textarea>`

這些元素本來就有合理的鍵盤操作行為。
通常不要一開始就用正整數去重排它們。

---

### 2）如果只是想讓某個元素可以被 Tab 走到，通常先考慮 `0`

如果元素原本不能聚焦，但你希望它加入自然焦點流程，`tabindex="0"` 往往比硬排正整數更自然，也更容易維護。

---

### 3）如果只想在特定時機聚焦它，可以考慮 `-1`

像：

* 錯誤訊息區
* 跳出的對話框
* 動態顯示的提示區

這些都可能適合用 `tabindex="-1"` 再搭配 JavaScript 控制焦點。

---

### 4）不要只改焦點順序，卻不管視覺順序與操作語意

如果使用者看到畫面上的順序是 A → B → C，
但實際按 `Tab` 卻變成 B → C → A，體驗通常會很差。

所以鍵盤順序、視覺順序、操作邏輯，最好盡量一致。

---

## 8. 一句話抓核心

`tabindex` 解決的是「鍵盤焦點怎麼移動」，不是「畫面怎麼排列」；學會分清正整數、`0`、`-1` 的差別，才算真正理解它。

## 最後整理

| 值    | 作用             | 常見理解                   |
| ---- | -------------- | ---------------------- |
| 正整數  | 自訂 `Tab` 順序    | 數值越小越先，但不適合大量使用        |
| `0`  | 進入自然焦點流程       | 可被 `Tab` 聚焦，通常跟著文件順序走  |
| `-1` | 不參與一般 `Tab` 導航 | 平常不會被 `Tab` 走到，但程式仍可聚焦 |

4. 因為維護成本高，且容易讓焦點順序和畫面順序不一致。
5. 全局屬性。
