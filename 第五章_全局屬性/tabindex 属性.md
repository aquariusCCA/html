# tabindex 屬性

> 所屬章節：第五章｜全局屬性  
> 關鍵字：`tabindex`、tab order、focus order、焦點順序、鍵盤導航、可聚焦、`tabindex="0"`、`tabindex="-1"`、全局屬性、無障礙  
> 建議回查情境：想知道 `tabindex` 是做什麼的、想分清 `tabindex="0"`、`tabindex="-1"` 和正整數的差別、想知道按 `Tab` 鍵時元素順序怎麼控制

## 本節導讀

這篇整理 HTML 全局屬性 `tabindex` 的基本用途。  
它的核心任務，是影響元素是否能被鍵盤聚焦，以及使用者按 `Tab` 鍵時的移動順序。

這個屬性很常出現在鍵盤操作、表單流程與無障礙語境裡。  
但它不是拿來隨便重排整個頁面的工具，實際使用時要先理解不同數值代表的焦點行為。

## 你會在這篇學到什麼

- `tabindex` 屬性是在做什麼
- `tabindex` 的基本語法
- `tabindex` 正整數、`0`、`-1` 的差別

## 30 秒複習入口

- `tabindex` 用來控制元素能不能被聚焦，以及按 `Tab` 時的大致順序。
- 正整數會把元素放進自訂的 `Tab` 順序中，數值越小越先。
- `tabindex="0"` 代表元素可被 `Tab` 導航聚焦，通常會跟著文件順序走。
- `tabindex="-1"` 代表元素通常不會被 `Tab` 走到，但仍可能被程式或其他操作聚焦。

## 速查區

### 核心概念

- `tabindex` 是用來控制元素焦點行為的屬性。
- 它關心的是鍵盤導航與聚焦順序，不是元素本身的功能內容。

### 關鍵規則 / 判準

- 基本語法是 `<element tabindex="number">`。
- 在 `HTML5` 中，`tabindex` 屬性可用於任何 HTML 元素，但不代表每個元素都適合設定。
- 正整數代表自訂 `Tab` 順序，值越小越先被聚焦。
- `tabindex="0"` 通常代表此元素可被 `Tab` 鍵聚焦，順序多半跟著文件原本的排列。
- `tabindex="-1"` 通常代表此元素不參與一般 `Tab` 導航，但仍可由 JavaScript 或其他互動方式聚焦。

### 常見混淆點

- `tabindex` 不是在改變畫面排列，而是在改變焦點移動行為。
- `tabindex="0"` 和 `tabindex="-1"` 不一樣：前者通常能用 `Tab` 走到，後者通常不能。
- 正整數雖然能強制排序，但濫用後很容易讓鍵盤操作順序變得難以理解。

### 一句話對比

- `tabindex` 解決的是「鍵盤焦點怎麼移動」，不是「元素長什麼樣」或「元素做什麼事」。

## 正文筆記

### 這篇在解決什麼問題？

- 當頁面上有多個可互動元素時，使用者常會透過 `Tab` 鍵在它們之間移動。
- `tabindex` 就是在處理這類需求：決定某個元素能不能進入鍵盤焦點流程，以及它在焦點流程中的位置。

## 1. `tabindex` 的作用

> `tabindex` 屬性規定當使用 `Tab` 鍵進行導航時元素的順序。

- 你可以把它理解成「元素的鍵盤焦點設定」。
- 只要頁面上有連結、按鈕、輸入框或其他可互動元素，就可能會遇到焦點順序的問題。
- `tabindex` 的存在，是為了讓開發者在必要時能調整這種順序或讓某些元素可被聚焦。

## 2. `tabindex` 的常見值怎麼看

### 正整數

- 像 `1`、`2`、`3` 這類正整數，表示你想明確指定 `Tab` 導航順序。
- 當多個元素都有正整數時，通常會先依數值由小到大聚焦。
- 這種寫法能快速看出教學效果，但在實務上不宜大量使用，否則後續維護會很麻煩。

### `tabindex="0"`

- `0` 通常表示這個元素可以進入一般的鍵盤焦點流程。
- 它常見於原本不是可聚焦元素，但你希望使用者能用 `Tab` 走到它的情境。
- 相比手動排一串正整數，`0` 往往是更自然也更容易維護的做法。

### `tabindex="-1"`

- `-1` 通常表示元素不參與一般的 `Tab` 導航順序。
- 不過，這不等於永遠不能聚焦；例如 JavaScript 仍可把焦點移到這個元素上。
- 這常用在你想手動控制焦點移動，但不希望使用者平常按 `Tab` 就走到該元素的情境。

## 3. 語法

```html
<element tabindex="number">
```

### 這段怎麼理解？

- `element` 代表任意 HTML 元素。
- `number` 代表你指定的焦點順序值。
- 重點不是把數字背下來，而是理解不同數值會讓元素進入不同的鍵盤焦點模式。

## 4. 教材中的基本示例

```html
<body>
  <a href="http://www.w3school.com.cn/" tabindex="2">W3School</a>
  <a href="http://www.google.com/" tabindex="1">Google</a>
  <a href="http://www.microsoft.com/" tabindex="3">Microsoft</a>
</body>
```

### 這個例子在說什麼？

- 三個連結都被設定了不同的 `tabindex` 值。
- 如果依這組設定進行鍵盤導航，通常會先聚焦 `tabindex="1"` 的 `Google`，再到 `W3School`，最後到 `Microsoft`。
- 這個例子主要是示範：`tabindex` 可以直接影響 `Tab` 鍵移動時的先後順序。

## 5. `tabindex` 的適用範圍

- 在 `HTML5` 中，`tabindex` 屬性可用於任何 HTML 元素，它會驗證任何 HTML 元素，但不一定都有實際用途。
- 在 `HTML4` 中，`tabindex` 屬性可用於：`<a>`、`<area>`、`<button>`、`<input>`、`<object>`、`<select>`、`<textarea>`。
- 這也再次說明：能不能寫，和是否值得寫，是兩件不同的事。

## 6. 使用 `tabindex` 時的注意點

- 優先保留原生可聚焦元素本來就合理的鍵盤順序，不要一開始就用一串正整數硬排。
- 如果只是想讓非互動元素在特定情境下能被聚焦，常見做法通常是考慮 `tabindex="0"` 或 `tabindex="-1"`。
- 只改焦點順序、卻沒有一起考慮可見順序與操作語意，容易讓鍵盤使用者感到混亂。
