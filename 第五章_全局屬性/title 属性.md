# title 屬性

> 所屬章節：第五章｜全局屬性

> 建議回查情境：想知道 `title` 屬性是做什麼的、想確認滑鼠移到元素上時顯示的小提示怎麼寫、想分清 `title` 屬性和 `<title>` 標籤的差別、想確認 `title` 能不能拿來放主要說明

## 本節導讀

這篇整理 HTML 全局屬性 `title` 的基本用途。  
它最常見的作用，是替元素補上一段額外說明，讓使用者在某些情況下能看到提示文字。

很多初學者第一次接觸 `title`，會把它和 `<title>` 標籤混在一起。  
但兩者其實不是同一件事：`title` 屬性是寫在元素上的附加資訊，`<title>` 則是整個網頁的標題。

另外，雖然 `title` 常被理解成「滑鼠移上去會跳出來的小提示」，但這種呈現不是所有裝置、所有操作方式都一樣穩定可見。  
所以理解這個屬性時，重點不只是會不會出現 tooltip，還要知道它適合放補充資訊，不適合承擔主要內容。

## 30 秒複習入口

- `title` 用來補充元素的額外資訊。
- 這段資訊常會在滑鼠移到元素上時，以工具提示文字的形式出現。
- `title` 是寫在元素上的屬性，不是 `<head>` 裡的 `<title>` 標籤。
- `title` 常適合拿來放簡短提示，但不適合塞太長或太關鍵的主要內容。
- 如果某段資訊是操作、理解或填寫表單的必要條件，不該只放在 `title` 裡。

## 速查區

### 核心概念

- `title` 是 HTML 全局屬性，可用來替元素補上一段附加說明。
- 這段說明常被瀏覽器當成提示文字顯示，但實際呈現方式可能依裝置、瀏覽器與操作方式而異。

### 關鍵規則 / 判準

- 基本語法是 `<element title="text">`。
- `title` 屬於全局屬性，因此理論上可以寫在任何 HTML 元素上。
- `title` 的內容通常應保持簡短、補充性強，不要取代頁面本來就該直接看見的重要資訊。
- 如果資訊本身必須穩定被看見，就不應只依賴 `title`。

### 常見使用方式

- 對縮寫補上完整名稱。
- 對元素補充一小段提示說明。
- 在畫面文字較短時，額外提供背景資訊或解釋。
- 對圖示、縮寫或短標籤補一段輔助理解文字。

### 常見混淆點

- `title` 屬性不是 `<title>` 標籤；前者屬於元素本身，後者屬於整份 HTML 文件。
- `title` 常會顯示成工具提示，但不能把它當成所有裝置上都穩定可見的主要說明方式。
- `title` 提供的是補充資訊，不是拿來取代清楚的介面文案。
- `title` 可以幫助補充說明，但不等於按鈕文字、表單標籤或其他本來就該直接顯示的名稱。

### 一句話對比

- `title` 屬性解決的是「替元素補上一段附加說明」，不是「設定整個頁面的標題」。

## 正文筆記

### 這篇在解決什麼問題？

- 有些元素本身顯示的文字很短，或者只看畫面不容易立刻理解它的意思。
- `title` 就是在處理這類需求：讓元素可以帶上一段額外資訊，方便在需要時補充說明。

## 1. `title` 的作用

> `title` 屬性規定關於元素的額外資訊。這些資訊通常會在滑鼠移到元素上時顯示一段工具提示文字。

- 你可以把它理解成「附在元素上的說明小字」。
- 這段說明不一定會直接常駐在畫面上，但在許多桌面瀏覽器中，滑鼠移上去時常會以提示框的形式出現。
- 也因為它屬於補充資訊，所以適合放簡短說明，而不是整段主要內容。
- 如果某個資訊必須讓所有使用者立刻看到，就應直接寫在介面上，而不是藏在 `title` 裡。

## 2. 語法

```html
<element title="text">
```

### 這段怎麼理解？

- `element` 代表任意 HTML 元素。
- `text` 代表你想提供給這個元素的額外說明文字。
- 這段文字最常見的呈現方式，是在使用者將滑鼠移到元素上時顯示提示內容。
- 但要記得：這只是常見呈現方式，不等於所有使用者都一定會看到。

## 3. 教材中的基本示例

```html
<body>
  <abbr title="People's Republic of China">PRC</abbr> was founded in 1949.
  <p title="Free Web tutorials">W3School.com.cn</p>
</body>
```

### 這個例子在說什麼？

- 第一個例子把 `PRC` 這個縮寫的完整名稱放進 `title` 裡。
- 第二個例子則是替段落補上一段額外提示文字 `Free Web tutorials`。
- 這兩個例子都在說明同一件事：`title` 可以讓元素附帶一段平常不一定直接顯示、但需要時能補充理解的文字。
- 其中 `abbr` 是特別典型的用法，因為縮寫本來就常需要一個不打斷版面的全稱說明。

## 4. `title` 常見適用情境

### 縮寫說明

- 像 `abbr` 這類元素很常搭配 `title` 使用。
- 因為縮寫本身可能太短，讀者不一定知道完整意思，所以 `title` 很適合拿來補上全稱。

### 補充提示

- 當按鈕、連結或某些可互動區塊需要一點額外說明時，也可以使用 `title`。
- 但如果那段資訊本身是操作的必要條件，仍然應該優先讓它直接出現在畫面上。

### 圖示或短文字元素

- 當元素本身只有一個圖示、簡短縮寫或很短的文字時，`title` 可以補上背景資訊，幫助理解。
- 但這種補法只能當輔助，不能假設所有使用者都會靠懸停看到它。

## 5. `title` 和 `<title>` 怎麼分

- `title` 屬性是寫在某個元素上的附加說明。
- `<title>` 標籤則是整份網頁文件的標題，通常出現在瀏覽器分頁名稱或收藏標題中。
- 一個是元素級的補充資訊，一個是文件級的頁面名稱，兩者用途完全不同。
- 實務上可以這樣記：`title` 是「元素的說明」，`<title>` 是「頁面的名稱」。

## 6. 使用 `title` 時的注意點

- 不要把關鍵操作說明只放在 `title` 裡，否則某些使用情境下使用者可能根本看不到。
- `title` 比較適合當成補充資訊，而不是主要資訊載體。
- 如果只是想讓介面更清楚，優先考慮直接把重要文案寫在看得見的位置。
- 在觸控裝置、純鍵盤操作，或某些輔助技術情境下，`title` 的可見性與可用性都不一定可靠。
- 若控制項需要清楚名稱，應優先使用可見文字、正確標籤或其他更直接的方式表達，而不是只靠 `title`。

## 常見問法

### 為什麼我寫了 `title`，卻沒有明顯看到提示框？

- 因為 `title` 的呈現高度依賴瀏覽器、裝置與操作方式。
- 在桌面環境中，滑鼠懸停時較常看到工具提示；但在觸控裝置上，這種體驗通常不穩定，甚至根本不會出現。

### `title` 能不能拿來取代表單標籤或按鈕文字？

- 不建議。
- `title` 比較像補充說明，不是主要名稱；如果使用者必須先知道那段文字才能操作，就應直接把名稱或說明顯示出來。

### `title` 最常見、最穩妥的用法是什麼？

- 對縮寫補全稱，是最經典也最容易理解的用法之一。
- 另外也常用來替短文字、圖示或連結補一句簡短提示，但前提仍是畫面本身已經具備基本可理解性。
