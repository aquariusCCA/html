## 本節導讀

`dir` 是 HTML 的全局屬性之一。
它的作用是：**告訴瀏覽器這段內容的文字方向應該怎麼解讀與呈現。**

最常見的方向有兩種：

* 由左到右，例如英文
* 由右到左，例如阿拉伯文、希伯來文

這個屬性在多語系頁面中特別重要。
尤其當頁面中可能出現：

* 阿拉伯文或希伯來文
* 使用者輸入的未知語言內容
* 不同方向文字混在一起的情況

如果沒有正確標示方向，顯示結果可能會變亂，閱讀順序也可能讓人困惑。
所以學 `dir` 時，最重要的不是只背一個 `rtl` 範例，而是要分清：

1. `ltr`、`rtl`、`auto` 各自代表什麼
2. 為什麼 `auto` 特別適合方向未知的內容
3. 為什麼通常應該優先用 HTML 的 `dir`，而不是只靠 CSS `direction`

你可以先記一句最核心的話：

> `dir` 不是單純拿來調整外觀，而是用來描述文字內容本身的方向性。

## 速查區

### 核心概念

* `dir` 決定的是**文字內容的方向性**
* 它不只是畫面看起來靠左或靠右
* 它是在告訴瀏覽器：這段內容應該用哪種文字方向規則來處理

### 關鍵規則

* 常見值有三個：

  * `ltr`
  * `rtl`
  * `auto`
* `ltr` 適合由左到右的語言，例如英文
* `rtl` 適合由右到左的語言，例如阿拉伯文
* `auto` 會依內容中的第一個具有明確方向性的強字元來判斷整體方向
* 若未指定，通常會繼承父元素的 `dir`

### 常見混淆點

* `dir` 不是單純做版面翻轉
* `dir="auto"` 不是亂猜，而是依內容做基本判斷
* `dir` 和 CSS `direction` 不完全相同
* `dir` 比較像是在描述內容方向，`direction` 比較像是在做樣式層控制

### 一句話對比

* `dir` 解決的是「這段文字應該朝哪個方向讀」，不是「元素外觀怎麼排版」。

---

## 這篇在解決什麼問題？

當頁面中出現不同書寫方向的文字時，如果沒有明確標示方向，瀏覽器可能無法正確處理閱讀方向、對齊感受與混合文字顯示。

例如：

* 英文和阿拉伯文混在一起
* 使用者輸入內容，但你事先不知道會是哪種語言
* 某個區塊是整段 RTL 語言，但頁面本身是 LTR

這時候，`dir` 就是在處理：

> 讓瀏覽器知道這段內容應該用哪種方向規則來理解。

---

# 1. `dir` 的作用

`dir` 屬性用來指定元素內容的文字方向。

你可以把它理解成：

> 幫這段內容加上一個「閱讀方向標記」。

例如：

```html
<p dir="ltr">Hello, world!</p>
<p dir="rtl">مرحبا بالعالم</p>
```

這表示：

* 第一段文字用由左到右的方向處理
* 第二段文字用由右到左的方向處理

所以 `dir` 的重點不是讓畫面「看起來像」某種方向，
而是讓瀏覽器知道：

> 這段內容本來就應該用哪一種方向來讀。

---

# 2. `dir` 的三個值怎麼分？

## `ltr`

`ltr` 是 **left to right**，表示文字由左到右。

適合像這類語言：

* 英文
* 中文一般混排情境
* 日文多數橫排情境
* 大多數歐洲語言

### 例子

```html
<p dir="ltr">Hello, this is an English sentence.</p>
```

### 怎麼理解？

這段內容明確指定為左到右。
瀏覽器會用 LTR 的方向規則處理它。

---

## `rtl`

`rtl` 是 **right to left**，表示文字由右到左。

適合像這類語言：

* 阿拉伯文
* 希伯來文

### 例子

```html
<p dir="rtl">مرحبا بالعالم</p>
```

### 怎麼理解？

這段內容明確指定為右到左。
瀏覽器會用 RTL 的方向規則處理這段文字。

這比用一段「內容不是 RTL 語言，但只靠說明文字表示右到左」的例子更直觀，
因為它更貼近 `rtl` 真正的使用場景。

---

## `auto`

`auto` 代表：**交給瀏覽器依內容自動判斷方向**。

瀏覽器通常會看內容中第一個具有明確方向性的強字元，
再決定整段內容應該偏向 LTR 還是 RTL。

### 為什麼這很重要？

因為有些內容在寫程式時，你根本不知道它會是什麼語言。
例如：

* 使用者留言
* 使用者名稱
* 外部 API 回傳文字
* 後台資料來源

這時候，`auto` 就很有價值。

### 例子

```html
<div dir="auto">مرحبا Hello</div>
```

或：

```html
<div dir="auto">使用者輸入的留言內容</div>
```

### 怎麼理解？

你不是自己手動決定這段是 `ltr` 還是 `rtl`，
而是把判斷工作交給瀏覽器。

所以：

> `auto` 最適合方向未知、來源不固定、可能是多語系的內容。

---

# 3. 基本示例

## 最簡單的 RTL 例子

```html
<p dir="rtl">مرحبا بالعالم</p>
```

### 這段怎麼理解？

* `<p>` 明確指定 `dir="rtl"`
* 所以瀏覽器會用由右到左的方向規則處理這段文字
* 這不只是視覺上的左右差異，而是文字方向規則本身不同

---

## `auto` 的直觀例子

```html
<label>留言：</label>
<div dir="auto">這裡顯示使用者輸入的內容</div>
```

### 這段怎麼理解？

* 這個區塊的內容可能是中文、英文、阿拉伯文，事先不確定
* 所以用 `dir="auto"` 比硬寫 `ltr` 或 `rtl` 更合理
* 這是 `dir="auto"` 最常見、也最有實務價值的情境之一

---

# 4. 適用範圍與繼承

`dir` 是全局屬性，
所以它可以寫在很多 HTML 元素上。

例如：

```html
<div dir="rtl">
  <p>這段通常也會跟著用 RTL 規則處理。</p>
</div>
```

### 這段怎麼理解？

* 父元素 `<div>` 設了 `dir="rtl"`
* 子元素 `<p>` 沒有自己再寫 `dir`
* 所以它通常會繼承父元素的方向設定

這就是為什麼在頁面根層或某個大區塊設定方向後，
裡面的子元素往往會一起跟著變。

---

## 父子元素範例

```html
<div dir="rtl">
  <h2>عنوان</h2>
  <p>هذه فقرة داخل區塊。</p>
</div>
```

### 重點

* 這整個區塊都採用 RTL 方向
* 裡面的標題與段落如果沒另行指定，通常都會跟著父層方向走

你可以把它記成一句話：

> 沒有自己寫 `dir` 時，通常先看父元素。

---

# 5. `dir="auto"` 為什麼特別重要？

因為很多資料不是你手動寫死在 HTML 裡的。

例如：

* 使用者輸入內容
* 聊天訊息
* 留言板資料
* 後端回傳文字
* 國際化網站的動態內容

你事先無法保證它一定是英文，也無法保證一定是阿拉伯文。
這時候如果硬寫：

```html
<div dir="ltr"></div>
```

就可能在某些語言下表現不理想。

所以更合理的做法常常是：

```html
<div dir="auto"></div>
```

因為你把方向判斷交給瀏覽器，
而不是自己先亂假設。

---

# 6. `dir` 和 CSS `direction` 有什麼差別？

這是最容易混淆的地方之一。

## `dir`

* 是 HTML 屬性
* 比較偏內容語意
* 在描述：這段文字本來應該用什麼方向解讀

## CSS `direction`

* 是 CSS 樣式屬性
* 比較偏表現層控制
* 常用在進一步的樣式或版面需求中

### 實務上怎麼判斷？

如果你的目的是：

> 描述內容本身的書寫方向

優先考慮用：

```html
dir="ltr"
dir="rtl"
dir="auto"
```

如果你的目的是：

> 進一步控制樣式、排版或視覺效果

才去考慮 CSS 相關做法。

所以可以簡單記成：

> 內容方向先想 `dir`，版面控制再想 CSS。

---

# 7. 常見混淆點 / 易錯點

## 錯誤一：把 `dir` 當成單純的對齊工具

不對。

`dir` 不是在做：

* 文字靠左
* 文字靠右
* 版面左右翻轉

它真正處理的是：

* 文字方向規則
* 閱讀方向
* 內容語意

---

## 錯誤二：以為 `dir="auto"` 是隨便猜

不對。

它不是亂判斷，
而是依內容中的強方向字元做基本判定。

所以它不是「不確定就亂選一個」，
而是「不確定時交給瀏覽器按內容判斷」。

---

## 錯誤三：忽略繼承效果

如果父元素已經設了 `dir`，
子元素沒有自己指定時，通常會跟著父元素的方向。

所以判斷一個元素的方向時，不要只看它自己有沒有寫。

---

## 錯誤四：只用 CSS `direction`，不管 HTML 語意

如果你處理的是內容本身的方向，
通常應該先從 HTML 的 `dir` 思考，
因為這件事本質上和內容語意有關，不只是樣式問題。

---

# 8. 一個完整的小例子

## HTML

```html
<section dir="ltr">
  <h2>Comments</h2>

  <p dir="auto">مرحبا بكم</p>
  <p dir="auto">Hello everyone</p>
  <p dir="auto">這是一則留言</p>
</section>
```

## 這個例子在說什麼？

* 整個區塊本身預設是 `ltr`
* 但每一則留言因為語言可能不同，所以各自用 `dir="auto"`
* 這樣英文留言會依英文方向處理
* 阿拉伯文留言會依 RTL 方向處理
* 中文或其他內容則依瀏覽器判斷規則處理

這就是 `dir` 在多語系與動態內容場景中的實際價值。

---

# 9. 一句話總結

`dir` 的本質是：

> 用 HTML 明確標示文字內容的方向，讓瀏覽器用正確的方向規則處理它；其中 `auto` 特別適合方向未知的內容。
