# dir 屬性

> 所屬章節：第五章｜全局屬性

> 建議回查情境：想知道 HTML 文字方向怎麼設定、想分清 `ltr`、`rtl`、`auto` 的差別、想確認 `dir` 和 CSS `direction` 的定位差異

## 本節導讀

這篇整理 `dir` 這個 HTML 全局屬性的用途與基本規則。  
它的核心作用，是標示元素內容的文字書寫方向，例如從左到右或從右到左。

這個屬性在多語系頁面中特別重要，尤其是當內容可能包含阿拉伯文、希伯來文，或來自使用者輸入、方向未知的資料時。  
理解 `dir` 時，最重要的不是只背一個 `rtl` 範例，而是分清 `ltr`、`rtl`、`auto` 的差別，以及它和 CSS `direction` 的角色界線。

## 你會在這篇學到什麼

- `dir` 屬性是在做什麼
- `ltr`、`rtl`、`auto` 三個值的差別
- `dir` 的繼承行為怎麼理解
- 為什麼語意上通常優先用 HTML 的 `dir`，而不是只靠 CSS `direction`

## 30 秒複習入口

- `dir` 是用來標示元素文字方向的列舉型全局屬性。
- `ltr` 代表由左到右，`rtl` 代表由右到左。
- `auto` 代表交給瀏覽器依內容判斷方向，特別適合方向未知的資料。
- 若元素沒設定 `dir`，通常會繼承父元素的方向設定。

## 速查區

### 核心概念

- `dir` 決定的是元素文字內容的方向性，而不是單純的視覺排版技巧。
- 這個資訊和內容語意有關，因此通常應該直接寫在 HTML 上。

### 關鍵規則 / 判準

- `dir` 常見值有 `ltr`、`rtl`、`auto`。
- `ltr` 適合由左到右書寫的語言，例如英文。
- `rtl` 適合由右到左書寫的語言，例如阿拉伯文。
- `auto` 會讓瀏覽器在解析內容時，根據第一個具有明確方向性的強字元來判斷整體方向。
- 若未指定，通常會從父元素繼承 `dir` 值。

### 常見混淆點

- `dir` 不是單純拿來做版面翻轉；它是在描述文字方向的語意資訊。
- `dir="auto"` 不是亂猜，而是根據內容中的強方向字元做基本判斷。
- CSS 的 `direction` 也能影響方向，但 MDN 建議在可能時優先使用 HTML `dir`，因為方向性和內容語意有關。

### 一句話對比

- `dir` 解決的是「這段文字應該朝哪個方向讀」，不是「元素外觀怎麼排版」。

## 正文筆記

### 這篇在解決什麼問題？

- 當頁面中出現不同書寫方向的文字時，如果沒有明確標示方向，顯示結果可能會混亂。
- `dir` 正是在處理這類需求：讓瀏覽器知道這段文字該用哪種方向來呈現與解讀。

## 1. `dir` 的作用

> `dir` 屬性規定元素內容的文本方向。

- `dir` 是一個列舉型全局屬性，用來指出元素文字的方向性。
- 依 MDN 說明，它可以使用 `ltr`、`rtl`、`auto` 三種值。

## 2. 三個值怎麼分

### `ltr`

- `ltr` 是 left to right，表示文字由左到右。
- 適合英文等由左到右書寫的語言。

### `rtl`

- `rtl` 是 right to left，表示文字由右到左。
- 適合阿拉伯文等由右到左書寫的語言。

### `auto`

- `auto` 代表交由使用者代理，也就是瀏覽器，自動判斷方向。
- MDN 的說明是：瀏覽器會解析元素內容，直到找到具有強方向性的字元，再用它決定整體方向。
- 這個值特別適合方向未知的資料，例如使用者輸入或外部資料來源。

## 3. 基本示例

```html
<p dir="rtl">文本方向从右到左!</p>
```

### 這段怎麼理解？

- 這個 `<p>` 明確指定 `dir="rtl"`，所以文字會以由右到左的方向呈現。
- 即使內容本身很短，這個屬性仍是在告訴瀏覽器：請用 RTL 的方向規則處理這段文字。

## 4. 適用範圍與繼承

- 在 HTML5 中，`dir` 屬性可用於任何 HTML 元素，但不代表寫在所有元素上都一定有實際用途。
- 原始教材也提到，在 HTML4 中有一些元素不能使用 `dir`。
- 如果某個元素沒有自己設定 `dir`，它通常會繼承父元素的 `dir` 值，這也是為什麼在頁面根層設定方向時，子元素往往會跟著變。

## 5. `dir` 和 CSS `direction` 的差別

- MDN 建議：在可以的情況下，應優先使用 HTML 的 `dir` 屬性，而不是只依賴 CSS `direction`。
- 原因是文字方向和內容語意有關，不只是表現層問題。
- 這裡可以做一個實務判斷：如果你是在描述內容本身的書寫方向，用 `dir`；如果你是在做更進一步的版面控制，才再考慮相關 CSS。
