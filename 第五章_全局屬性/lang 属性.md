# lang 屬性

> 所屬章節：第五章｜全局屬性  
> 關鍵字：`lang`、語言代碼、language code、文件語言、元素語言、HTML5、HTML4  
> 建議回查情境：想確認 `lang` 是做什麼的、想知道元素內容的語言怎麼標示、想分清 `lang` 和文字方向 `dir` 的差別

## 本節導讀

這篇整理 HTML 全局屬性 `lang` 的基本用途。  
`lang` 的核心任務，是標示元素內容使用的是哪一種語言。

它不負責改變文字方向，也不是翻譯工具。  
它更像是在替瀏覽器、輔助技術與其他處理流程補上一個語言提示，讓系統知道這段內容應該被當成哪種語言來理解。

## 30 秒複習入口

- `lang` 用來標示元素內容的語言。
- 屬性值通常是語言代碼，例如 `en`、`zh-CN`、`zh-TW`。
- `lang` 處理的是語言，不是文字方向；文字方向通常看 `dir`。

## 速查區

### 核心概念

- `lang` 是 HTML 全局屬性，可用在多數元素上。
- 它的用途是標示文件或元素內容所使用的語言。

### 關鍵規則 / 判準

- `lang` 的值通常寫語言代碼。
- 你可以把 `lang` 設在整份文件，也可以設在個別元素上。
- 在 HTML5 中，`lang` 可用於任何 HTML 元素。

### 常見使用方式

- 在 `<html>` 上標記整份文件的主要語言。
- 在局部元素上標記和主文件不同的語言內容。
- 為多語內容提供更清楚的語言資訊。

### 常見混淆點

- `lang` 不是拿來控制文字方向；若是左右書寫方向問題，通常看 `dir`。
- `lang` 不是翻譯功能；它只負責標示語言。
- 設了 `lang` 不代表頁面外觀一定會明顯改變，但它仍然可能對語言判讀有幫助。

### 一句話對比

- `lang` 回答的是「這段內容是什麼語言」，`dir` 回答的是「這段內容從哪個方向書寫」。

## 正文筆記

### 這篇在解決什麼問題？

- 同一個頁面可能會出現不同語言的內容。
- `lang` 讓你能明確標示某份內容屬於哪一種語言，而不是只靠讀者自己猜。

## 1. `lang` 是什麼

> `lang` 屬性用來規定元素內容的語言。

- 可以把它理解成「替這段內容加上語言標籤」。
- 這個標籤可以標在整份文件，也可以標在個別元素上。

## 2. `lang` 的值怎麼寫

- `lang` 的值通常是語言代碼。
- 例如常見的 `en` 代表英文，`zh-CN` 常用來表示簡體中文，`zh-TW` 常用來表示繁體中文。
- 原始教材中的圖片，就是在列出常見語言代碼的參考表。

## 3. HTML4 和 HTML5 的位置差異

- 原始教材補充了一個版本差異：在 HTML5 中，`lang` 可用於任何 HTML 元素。
- 教材也提到，在 HTML4 中，`lang` 不能用於部分元素，例如 `<base>`、`<br>`、`<frame>`、`<frameset>`、`<hr>`、`<iframe>`、`<param>`、`<script>`。
- 如果你現在是在學現代 HTML，先把重點放在 HTML5 的理解即可：`lang` 是全局屬性，可以廣泛使用。

## 4. 基本示例

```html
<body>
  <p lang="en">This is a paragraph.</p>
</body>
```

### 這段怎麼理解？

- 這個 `<p>` 的內容被標示為英文。
- 重點不是這段字一定要長得不一樣，而是頁面已經明確指出「這段內容的語言是英文」。
- 如果頁面主要是中文，但其中某小段是英文，也可以只在那個元素上單獨加 `lang="en"`。

## 5. `lang` 和 `dir` 怎麼分

- `lang` 處理的是語言，例如中文、英文、日文。
- `dir` 處理的是文字方向，例如由左到右或由右到左。
- 所以它們雖然都和文字內容有關，但回答的是不同問題，不能互相取代。
