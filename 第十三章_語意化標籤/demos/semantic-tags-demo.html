HTML 語意化標籤 Demo

語意化標籤的重點不是「畫面會變不一樣」，而是讓 HTML 更清楚地表達內容角色。當頁面結構更清楚時，讀者、工具與搜尋系統都更容易理解這一頁在做什麼。

語意先於樣式 `header`、`nav`、`main`、`article`、`section`、`aside`、`footer` 的重點是角色，不是外觀。
不是 `div` 換名字 它們仍是容器，但比單純 `div` 更能說明內容用途。
提升可理解性 更清楚的結構通常有助於內容維護、頁面分析與資訊判讀。
SEO 與工具友善 語意更清楚時，搜尋系統與其他工具通常更容易理解頁面架構。

3. `article` 和 `section` 的差異

        

          

            `article`
            
偏向可獨立存在的內容

            

              如果一塊內容被單獨拿出去閱讀、分享、轉載，仍然能成立，通常更接近 `article`。
            

            

              
前端學習心得

              
這篇文章即使被抽離原頁面，仍然是一篇完整內容。

            

          


          

            `section`
            
偏向主題下的一個區段

            

              如果它主要是某個頁面裡的其中一段內容、其中一個主題區塊，通常更接近 `section`。
            

            

              
課程特色

              
這更像某個頁面裡的其中一個介紹區塊，而不是獨立文章。

            

          

        

      


      

        
4. 何時應優先用語意化標籤

        

          

            

              
              
            

          


          

            
優先語意

            
先找有語意的標籤，再退回泛用容器

            

              語意化標籤不是要完全取代 `div`，而是讓「本來就有明確角色」的區塊先用更恰當的標籤表達。
            

            

            

              原則很簡單：先問這塊內容是不是頁首、導覽、主內容、文章、區段、側欄或頁尾。
            

          

        

      


      

        
5. 常見實務提醒

        

          

            語意不等於樣式
            `header` 看起來不一定要像網站頂部橫條，重點是它扮演的角色。
          

          

            `div` 不是錯
            當內容只是純排版分組、沒有更明確角色時，`div` 仍然是合理選擇。
          

          

            先分清內容角色
            先搞清楚內容在頁面中的用途，再選語意標籤，會比先想畫面更穩定。
