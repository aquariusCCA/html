> h5 語義化標籤的推出很好的解決了滿屏的 div 佈局，語義化顧名思義就是能讓人一眼就看出來每一個標籤的作用和含義，使用語義恰當的標籤，使頁面有良好的結構，頁面元素有含義，能夠讓和搜索引擎都容易理解。

- 一張圖就可以清晰的展示語義化標籤的作用，確實挺好，感覺像搭積木一樣，一塊一塊的把整體搭建出來
    
    ![語意化標籤.png](./assets/images/semantic-tags-img-001-fe28a0.png)
    
    - `< header >` : 頭部標籤。
    - `< nav >`: 導航標籤。
    - `< article >`： 內容標籤。
    - `< section >` : 定義文檔某個區域。
    - `< aside >` : 側邊欄標籤。
    - `< footer >` : 尾部標籤。
- 總結語義化優點：
    - 易於用戶閱讀，樣式丟失的時候能讓頁面呈現清晰的結構。
    - 有利於 SEO，搜索引擎根據標籤來確定上下文和各個關鍵字的權重。
    - 方便其他設備解析，如盲人閱讀器根據語義渲染網頁。
    - 有利於開發和維護，語義化更具可讀性，代碼更好維護，與 CSS3 關係更和諧。
- 注意點 :
    - 這種語義化標準主要是針對`搜索引擎`的。
    - 這些新標籤頁面中可以使用多次。
    - 在`IE9`中，需要把這些元素轉換為`塊級元素`。
    - 其實，移動端更喜歡使用這些標籤。因為移動端沒有兼容性的問題。
- 舉例來說以前佈局，我們基本用 div 來做。div 對於搜索引擎來說，是沒有語義的。
    
    ```css
    .header, .nav {
      height: 120px;
      background-color: pink;
      border-radius: 15px;
      width: 800px;
      margin: 15px auto;
    }
    
    .section {
      width: 500px;
      height: 300px;
      background-color: skyblue;
    }
    ```
    
    ```html
    <body>
      <div class="header">頭部標籤</div>
      <div class="nav">導航欄標籤</div>
      <div class="section">某個區域</div>
    </body>
    ```
    
- 換成語意化標籤來佈局
    
    ```css
    header, nav {
      height: 120px;
      background-color: pink;
      border-radius: 15px;
      width: 800px;
      margin: 15px auto;
    }
    section {
      width: 500px;
      height: 300px;
      background-color: skyblue;
    }
    ```
    
    ```html
    <body>
        <header>頭部標籤</header>
        <nav>導航欄標籤</nav>
        <section>某個區域</section>
    </body>
    ```
