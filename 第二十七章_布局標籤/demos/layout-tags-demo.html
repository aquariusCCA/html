為什麼需要語意化布局？

它讓 HTML 結構能直接表達區塊角色，而不是只留下無語意的容器。

常見布局標籤有哪些？

2. 常見布局標籤速查

標籤 核心角色 使用提醒
header 頁面或區塊的開頭資訊。 可以是整頁頁首，也可以是文章或區段的開頭。
nav 重要導覽連結區。 主選單、頁內目錄、側邊選單適合用；普通段落連結不一定需要。
main 目前頁面最核心、不重複的主要內容。 通常一個頁面只有一個可見 main。
article 可以獨立閱讀或獨立分發的內容單元。 文章、新聞、貼文、商品卡片、留言都可能適合。
section 同一份內容中的主題分段。 通常應該有標題；純樣式容器不要用。
aside 補充資訊或旁支內容。 不一定要視覺上放在側邊，位置交給 CSS。
footer 頁面或區塊的結尾資訊。 可以放版權、作者、更新時間、相關連結。
div 無語意通用容器。 純排版、樣式包裝、JS 操作用它更合適。

3. article 和 section 怎麼分

可以獨立拿出去看：用 article

<article>
  <h2>HTML5 布局標籤</h2>
  <p>介紹常見 HTML5 結構標籤。</p>
</article>

整篇文章、單則新聞、商品卡片、留言，都可以先問「拿出去還像完整內容嗎？」

同一內容裡的主題分段：用 section

<article>
  <h2>JavaScript 學習路線</h2>

  <section>
    <h3>基礎語法</h3>
    <p>先學變數、流程控制與函式。</p>
  </section>
</article>

section 是分段，不是萬用容器。沒有清楚標題時要先停下來判斷。

4. 判斷流程

  1. 這是整頁最主要、最核心的內容嗎？是的話考慮 main。
  2. 這是一組重要導航連結嗎？是的話考慮 nav。
  3. 這塊內容拿出去後還能獨立閱讀嗎？是的話考慮 article。
  4. 這是同一份內容中的主題分段嗎？是的話考慮 section。
  5. 這是補充資訊、延伸閱讀或旁支內容嗎？是的話考慮 aside。
  6. 這是頁面或區塊的開頭 / 結尾資訊嗎？開頭用 header，結尾用 footer。
  7. 只是為了排版、樣式或 JS 操作包一層嗎？用 div。

5. 常見錯誤對照

把所有容器都改成 section

section 表示主題區段。純樣式包裝應該用 div。

看到連結就用 nav

nav 適合重要導航系統。普通文字裡的一兩個連結不一定需要。

把 aside 當成右側欄

aside 的重點是補充性，不是畫面位置。左右排列由 CSS 決定。

6. hgroup 與標題群組

適合：主標題 + 副標題

<hgroup>
  <h1>HTML5 布局標籤</h1>
  <p>理解頁面結構與語意化標籤</p>
</hgroup>

不建議：亂包多層標題

<hgroup>
  <h1>HTML5 布局標籤</h1>
  <h2>header</h2>
  <h2>nav</h2>
</hgroup>
