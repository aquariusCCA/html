# 第二十七章｜布局標籤

> 章節定位：理解 HTML5 常見布局標籤如何表達頁面區塊角色  
> 建議閱讀方式：先掌握各標籤用途，再回看 `article` / `section` 的差別與整頁結構範例

## 本章在學什麼

這一章整理 HTML5 常見的布局標籤。  
重點不是背標籤名字，而是建立一個穩定觀念：頁面中的頁首、導航、主要內容、補充資訊、文章區塊與頁尾，應該如何用更有語意的標籤表達。

## 本章與整體地圖的關係

- 前置知識：前面已學過基本結構標籤、盒子標籤與語意化標籤，這一章開始把它們放進更完整的頁面結構中理解。
- 這章的核心任務：分清各種布局標籤的角色，特別是 `main`、`article`、`section`、`aside` 的差別。
- 讀完後通常接：之後在寫完整頁面骨架、整理內容區塊，或重新閱讀語意化標籤時，都會回到這一章。

## 易混淆主題

- 布局標籤 vs CSS 布局：前者在表達語意角色，後者在控制視覺排版。
- `article` vs `section`：前者偏獨立內容，後者偏同主題下的分段。
- 語意標籤 vs `div`：有清楚角色就用語意標籤，純包裝容器通常用 `div`。
