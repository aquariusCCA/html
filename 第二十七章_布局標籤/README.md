# 第二十七章｜布局標籤

> - 章節定位：理解 HTML5 常見布局標籤如何表達頁面區塊角色  
> - 建議閱讀方式：先建立「語意化布局不是 CSS 排版」的觀念，再回查 `main`、`nav`、`article`、`section`、`aside`、`hgroup` 的使用邊界

## 本章在學什麼

這一章整理 HTML5 常見的布局標籤。

重點不是背標籤名稱，也不是用這些標籤直接排出畫面，而是建立一個穩定觀念：  
當頁面被拆成頁首、導航、主要內容、文章內容、主題分段、補充資訊與頁尾時，應該如何用更有語意的標籤描述每個區塊的角色。

簡單說：

```txt
HTML 布局標籤：描述這塊內容是什麼角色。
CSS 布局語法：控制這塊內容長什麼樣子、放在哪裡。
```

## 易混淆主題

### 布局標籤 vs CSS 布局

- 布局標籤：描述內容角色，例如頁首、導航、主要內容、文章、補充資訊。
- CSS 布局：控制視覺呈現，例如左右排列、兩欄版面、置中、間距、寬高、RWD。

### `main` vs 一般內容容器

- `main` 表示整個頁面的主要內容。
- 通常一個頁面只保留一個主要且可見的 `main`。
- 如果只是一般包裝容器，不要硬用 `main`。

### `nav` vs 普通連結

- `nav` 適合主要導航、頁內目錄、側邊選單、分類選單。
- 普通段落裡的一兩個連結，不一定需要放進 `nav`。

### `article` vs `section`

- `article`：重點是獨立性，內容拿出去後仍然可以單獨閱讀或分發。
- `section`：重點是分段性，是同一份內容中的一個主題區段。

### `aside` vs 視覺側欄

- `aside` 的重點是補充性、旁支性。
- 它常被做成側欄，但語意上不要求一定放在畫面左邊或右邊。

### `hgroup` vs 多層標題

- `hgroup` 適合把一個主標題和副標題、標語、替代標題分成一組。
- 不要把它理解成把所有 `h1`、`h2`、`h3` 隨便包在一起的容器。

### 語意標籤 vs `div`

- 有清楚內容角色時，優先考慮語意標籤。
- 只是為了 CSS 排版、樣式包裝、JS 操作時，通常用 `div` 更合適。
