# 布局標籤（HTML5 結構標籤）

> - 所屬章節：第二十七章｜布局標籤  
> - 關鍵字：布局標籤、語意化布局、結構標籤、semantic elements、`header`、`nav`、`main`、`article`、`section`、`aside`、`footer`、`hgroup`、`div`  
> - 建議回查情境：想建立 HTML5 頁面骨架、想分清 `article` / `section`、想確認 `main` / `nav` / `aside` 怎麼用、想知道何時該用語意標籤或 `div`

## 本節導讀

這一章整理 HTML5 常見的布局標籤。

這裡的「布局標籤」不是指用標籤直接控制畫面排版，而是指：  
當頁面被拆成頁首、導覽、主要內容、文章區塊、主題分段、補充資訊與頁尾時，可以使用更有語意的標籤來描述每個區塊的角色。

真正控制畫面位置、寬高、欄位、間距的是 CSS。  
HTML5 布局標籤的任務是讓頁面結構更容易被人、搜尋引擎、瀏覽器與輔助技術理解。

## 整理說明

- 本版定位：9.5 分版本。
- 本版修正重點：
  - 補強 `main` 的正確定位：它代表頁面主要內容，不應包含大量重複性導覽、頁尾、側欄內容。
  - 補強 `nav` 的使用邊界：不是所有連結群都需要 `nav`，通常用於主要導航區塊。
  - 補強 `article` / `section` 判斷規則：`article` 看獨立性，`section` 看主題分段。
  - 補強 `aside` 的語意：重點是補充性，不是畫面上一定要放在側邊。
  - 修正 `hgroup` 說明：適合把一個主標題和副標題、替代標題、標語分成一組，副標題通常用 `p` 表示。
  - 加入「判斷流程」、「錯誤示範」、「整頁結構範例」，讓筆記更適合回查與實作。

## 30 秒複習入口

- 布局標籤的核心用途是「表達內容角色」，不是取代 CSS 排版。
- `main` 表示頁面主要內容，通常一個頁面只有一個可見的 `main`。
- `nav` 用於主要導航區，不是所有連結都要包進 `nav`。
- `article` 強調內容可以獨立成立、獨立閱讀或獨立分發。
- `section` 強調同一份內容中的主題分段，通常應該有標題。
- `aside` 表示補充資訊、旁支資訊，不一定要真的放在畫面側邊。
- `header` / `footer` 可以是整頁的頁首 / 頁尾，也可以是某個 `article` 或 `section` 的開頭 / 結尾。
- `hgroup` 用於把一個標題和副標題、替代標題、標語分成一組，實務上不是高頻標籤。
- 如果只是為了 CSS 包一層容器，通常用 `div` 比硬套語意標籤更合適。

## 速查區

### 一句話抓核心

布局標籤是在替網頁區塊命名角色，讓頁面不只是「排得出來」，也能「看得懂結構」。

### 語意化布局 vs CSS 布局

| 類型 | 負責內容 | 例子 |
|---|---|---|
| HTML 語意化布局 | 描述內容角色 | 這是頁首、這是導航、這是主要內容、這是一篇文章 |
| CSS 視覺布局 | 控制畫面呈現 | 橫向排列、兩欄版面、置中、間距、寬高、RWD |

簡單說：

```txt
HTML：這塊內容是什麼？
CSS：這塊內容長什麼樣子、放在哪裡？
```

### 常見布局標籤與用途

| 標籤 | 核心語意 | 常見內容 | 使用提醒 |
|---|---|---|---|
| `header` | 頁面或區塊的開頭 | Logo、標題、導言、搜尋框、導覽前資訊 | 可以用在整頁，也可以用在 `article` / `section` 裡 |
| `nav` | 導航區 | 主選單、頁內目錄、側邊選單、分類選單 | 只適合重要導航，不是所有連結都要用 |
| `main` | 頁面主要內容 | 當前頁面的核心內容 | 通常只有一個可見 `main`，不要把重複性頁首、頁尾、主導覽放進去 |
| `article` | 可獨立成立的內容單元 | 文章、新聞、貼文、留言、商品卡片、評論 | 通常應該有自己的標題 |
| `section` | 同一主題下的區段 | 章節、功能區、內容分段 | 通常應該有標題；純樣式容器不要用 |
| `aside` | 補充或旁支資訊 | 相關文章、延伸閱讀、作者資訊、廣告、補充說明 | 重點是「補充性」，不是「視覺上在側邊」 |
| `footer` | 頁面或區塊的結尾資訊 | 版權、作者、相關連結、最後更新時間 | 可以是整頁頁尾，也可以是文章或區塊結尾 |
| `hgroup` | 標題群組 | 主標題 + 副標題 / 標語 / 替代標題 | 通常是一個 `h1`～`h6` 搭配一個或多個 `p` |
| `div` | 無語意容器 | 包版面、包樣式、包 JS 操作用區塊 | 沒有明確語意時，用 `div` 反而更正確 |

### 判斷流程

寫頁面時可以照這個順序判斷：

```txt
1. 這是整頁最主要、最核心的內容嗎？
   是：考慮 main。

2. 這是一組重要導航連結嗎？
   是：考慮 nav。

3. 這塊內容拿出去後，還能獨立閱讀或獨立分發嗎？
   是：考慮 article。

4. 這是同一份內容中的一個主題分段嗎？
   是：考慮 section。

5. 這是補充資訊、延伸閱讀、旁支內容嗎？
   是：考慮 aside。

6. 這是頁面或區塊的開頭 / 結尾資訊嗎？
   開頭：考慮 header。
   結尾：考慮 footer。

7. 只是為了排版、樣式或 JS 操作而包一層嗎？
   是：使用 div。
```

### 常見錯誤

| 錯誤寫法 | 問題 | 建議 |
|---|---|---|
| 把所有容器都改成 `section` | `section` 不是萬用容器 | 純包裝用 `div` |
| 看到連結就用 `nav` | `nav` 只適合主要導航區 | 普通連結直接放在文字或 `footer` 中即可 |
| 把 `main` 包在 `article` 裡 | `main` 表示整頁主要內容，不是文章內部區塊 | 通常是 `main` 裡面放 `article` |
| 在一個頁面放多個可見 `main` | 主要內容區會不清楚 | 通常只保留一個可見 `main` |
| 把 `article` 和 `section` 當成同義詞 | 兩者判斷角度不同 | `article` 看獨立性，`section` 看分段性 |
| `section` 沒有標題 | 區段主題不明確 | 通常加上 `h2`～`h6` |
| 以為 `aside` 一定要放在右邊 | `aside` 是語意，不是版面位置 | 位置交給 CSS 控制 |
| `hgroup` 裡塞一堆 `h1`、`h2` | 現代寫法通常不是這樣用 | 用一個 heading 搭配 `p` 副標題 |

## 正文筆記

### 1. 布局標籤在解決什麼問題？

早期寫 HTML 時，很多區塊都會用 `div` 包起來：

```html
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>
<div class="footer">...</div>
```

這樣當然可以配合 CSS 排出畫面，但光看 HTML 標籤本身，很難直接知道每個區塊的角色。

HTML5 提供了更有語意的結構標籤，例如：

```html
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>
```

這樣一看就能知道：

- `header` 是頁首或區塊開頭
- `nav` 是導覽
- `main` 是主要內容
- `footer` 是頁尾或區塊結尾

所以，布局標籤的重點不是「讓畫面變漂亮」，而是「讓結構變清楚」。

---

### 2. `header`

#### 核心用途

`header` 表示頁面或某個區塊的開頭內容。

常見內容包含：

- 網站 Logo
- 網站名稱
- 頁面標題
- 文章標題
- 導言
- 搜尋框
- 作者與日期資訊
- 導覽前的介紹內容

#### 整頁的 `header`

```html
<header>
  <h1>前端學習筆記</h1>
  <p>整理 HTML、CSS、JavaScript 的學習內容</p>
</header>
```

這裡的 `header` 是整個網站或頁面的開頭。

#### 文章裡的 `header`

```html
<article>
  <header>
    <h2>HTML5 語意化布局介紹</h2>
    <p>發布日期：2026-04-26</p>
  </header>

  <p>這篇文章會介紹常見 HTML5 結構標籤。</p>
</article>
```

這裡的 `header` 不是整頁頁首，而是這篇文章的開頭資訊。

#### 使用提醒

`header` 不會自動建立新的主題區段。  
它只是替某個頁面或區塊提供「開頭介紹」。

---

### 3. `nav`

#### 核心用途

`nav` 表示導航區，也就是一組用來前往其他頁面、其他區塊或其他功能的主要連結。

常見內容包含：

- 網站主選單
- 頁內目錄
- 側邊選單
- 分類選單
- 上一頁 / 下一頁導覽
- 後台功能選單

#### 基本範例

```html
<nav aria-label="主要導覽">
  <ul>
    <li><a href="/">首頁</a></li>
    <li><a href="/notes">學習筆記</a></li>
    <li><a href="/projects">作品集</a></li>
    <li><a href="/about">關於我</a></li>
  </ul>
</nav>
```

#### 頁內導覽範例

```html
<nav aria-label="本頁目錄">
  <ol>
    <li><a href="#header">header</a></li>
    <li><a href="#main">main</a></li>
    <li><a href="#article-section">article 和 section</a></li>
  </ol>
</nav>
```

#### 使用提醒：不是所有連結都要用 `nav`

例如頁尾只有幾個普通連結時，可以直接放在 `footer` 裡：

```html
<footer>
  <a href="/privacy">隱私權政策</a>
  <a href="/terms">服務條款</a>
</footer>
```

如果這組連結是重要的導覽系統，才需要考慮加上 `nav`。

#### 多個 `nav` 可以共存

一個頁面可以有多個 `nav`，例如：

- 主導覽
- 側邊導覽
- 頁內目錄
- 文章上下篇導覽

如果同一頁有多個 `nav`，可以用 `aria-label` 區分用途：

```html
<nav aria-label="主要導覽">...</nav>
<nav aria-label="文章目錄">...</nav>
<nav aria-label="分頁導覽">...</nav>
```

### 4. `main`

#### 核心用途

`main` 表示目前頁面的主要內容，也就是這個頁面最核心、最不重複的部分。

它通常不應該包含：

- 網站 Logo
- 全站主選單
- 頁尾版權
- 全站共用側欄
- 每個頁面都重複出現的內容

#### 基本範例

```html
<body>
  <header>
    <h1>前端學習筆記</h1>
  </header>

  <nav aria-label="主要導覽">
    <a href="/">首頁</a>
    <a href="/html">HTML</a>
    <a href="/css">CSS</a>
  </nav>

  <main>
    <h2>HTML5 布局標籤</h2>
    <p>這一頁主要介紹 HTML5 常見的結構標籤。</p>
  </main>

  <footer>
    <small>© 2026 Frontend Notes</small>
  </footer>
</body>
```

#### 使用提醒

一般情況下，一個頁面只需要一個可見的 `main`。

比較推薦的結構是：

```html
<main>
  <article>
    ...
  </article>
</main>
```

不推薦寫成：

```html
<article>
  <main>
    ...
  </main>
</article>
```

因為 `main` 是針對整份文件的主要內容，不是某篇文章內部的小區塊。

#### 補充：`main` 不是分段標籤

`main` 不像 `article`、`section` 那樣用來建立內容區段。  
它主要是在告訴瀏覽器與輔助技術：「這裡是本頁核心內容」。

### 5. `article`

#### 核心用途

`article` 表示一塊可以獨立成立、獨立閱讀、獨立分發或獨立重複使用的內容。

常見例子：

- 部落格文章
- 新聞文章
- 論壇貼文
- 留言
- 商品卡片
- 評論
- 社群貼文
- 可獨立存在的小工具

#### 判斷方式

可以問自己：

> 如果把這塊內容單獨拿出去，它還像一個完整內容嗎？

如果答案是「是」，就偏向使用 `article`。

#### 基本範例

```html
<article>
  <h2>HTML5 語意化布局介紹</h2>
  <p>這篇文章介紹常見的 HTML5 結構標籤。</p>
</article>
```

#### 商品卡片範例

```html
<article class="product-card">
  <h2>機械式鍵盤</h2>
  <p>適合程式開發與長時間打字。</p>
  <p>價格：NT$2,980</p>
</article>
```

如果每張商品卡片拿出去後仍然是一個完整商品資訊，就可以考慮 `article`。

#### 留言範例

```html
<article>
  <h3>小明的留言</h3>
  <p>這篇文章對我很有幫助。</p>
  <footer>
    <time datetime="2026-04-26">2026-04-26</time>
  </footer>
</article>
```

留言本身也可以是一個獨立內容單元，所以可以使用 `article`。

#### 使用提醒

每個 `article` 通常應該有自己的標題，方便讀者和輔助技術理解這篇內容的主題。

### 6. `section`

#### 核心用途

`section` 表示文件或應用程式中的一個主題區段。

它強調的是：

> 這是同一份內容底下的一個主題分段。

常見例子：

- 文章中的章節
- 首頁中的功能區塊
- 文件中的小節
- 表單中的一組欄位區塊
- 後台頁面的不同功能區

#### 基本範例

```html
<article>
  <h2>HTML5 語意化布局介紹</h2>

  <section>
    <h3>為什麼需要語意化標籤？</h3>
    <p>語意化標籤可以讓頁面結構更清楚。</p>
  </section>

  <section>
    <h3>常見布局標籤有哪些？</h3>
    <p>例如 header、nav、main、article、section、aside、footer。</p>
  </section>
</article>
```

這裡的兩個 `section` 都是同一篇文章裡面的主題分段。

#### 使用提醒：`section` 通常要有標題

如果一個區塊沒有明確標題，通常要先思考它是不是真的需要 `section`。

不太建議：

```html
<section class="container">
  <div class="card">...</div>
  <div class="card">...</div>
</section>
```

如果只是為了排版，應該改成：

```html
<div class="container">
  <div class="card">...</div>
  <div class="card">...</div>
</div>
```

#### 什麼時候適合用 `section`？

可以問自己：

> 如果產生一份文章大綱，這個區塊的標題是否應該出現在大綱裡？

如果答案是「是」，就比較適合 `section`。  
如果答案是「否，只是包版面」，那通常用 `div`。

### 7. `article` 和 `section` 怎麼分？

這是本章最容易混淆的地方。

#### 核心差異

| 標籤 | 判斷重點 | 常見場景 |
|---|---|---|
| `article` | 內容可以獨立成立 | 文章、貼文、新聞、留言、商品卡片 |
| `section` | 同一份內容中的主題分段 | 章節、功能區、內容分區 |

#### 快速判斷

```txt
可以獨立拿出去看嗎？
是 → article

只是某篇內容中的一個小節嗎？
是 → section
```

#### 例子 1：文章中的段落

```html
<article>
  <h2>JavaScript 學習路線</h2>

  <section>
    <h3>基礎語法</h3>
    <p>先學變數、流程控制、函式與陣列。</p>
  </section>

  <section>
    <h3>DOM 操作</h3>
    <p>再學事件處理、元素查找與畫面更新。</p>
  </section>
</article>
```

整篇「JavaScript 學習路線」可以獨立成文，所以用 `article`。  
裡面的「基礎語法」、「DOM 操作」只是文章內的小節，所以用 `section`。

#### 例子 2：文章列表

```html
<main>
  <h1>最新文章</h1>

  <article>
    <h2>HTML5 布局標籤</h2>
    <p>介紹常見 HTML5 結構標籤。</p>
  </article>

  <article>
    <h2>CSS Flexbox 入門</h2>
    <p>介紹 Flexbox 的主軸、交叉軸與對齊方式。</p>
  </article>
</main>
```

每一篇文章摘要都可以獨立代表一篇文章，所以每一則可以用 `article`。

#### 例子 3：section 裡面包含多個 article

```html
<section>
  <h2>最新消息</h2>

  <article>
    <h3>系統維護公告</h3>
    <p>本週六凌晨將進行系統維護。</p>
  </article>

  <article>
    <h3>新版功能上線</h3>
    <p>本次新增批次匯出功能。</p>
  </article>
</section>
```

「最新消息」是一個主題區塊，所以用 `section`。  
每一則消息可以獨立閱讀，所以用 `article`。

### 8. `aside`

#### 核心用途

`aside` 表示和周圍主要內容有關，但不是主線內容本身的補充資訊。

常見內容包含：

- 側邊欄
- 延伸閱讀
- 相關文章
- 作者資訊
- 廣告
- 補充說明
- 引用重點
- 文章目錄
- 相關連結

#### 基本範例

```html
<article>
  <h2>HTML5 語意化布局介紹</h2>
  <p>語意化標籤可以讓頁面結構更清楚。</p>

  <aside>
    <h3>補充說明</h3>
    <p>語意化標籤不會自動改變畫面排版，排版仍然要靠 CSS。</p>
  </aside>
</article>
```

這個 `aside` 是文章的補充說明。

#### 頁面側欄範例

```html
<aside aria-labelledby="related-title">
  <h2 id="related-title">延伸閱讀</h2>

  <nav aria-label="延伸閱讀">
    <ul>
      <li><a href="/html/semantic">語意化標籤</a></li>
      <li><a href="/html/headings">標題標籤</a></li>
      <li><a href="/css/layout">CSS 版面配置</a></li>
    </ul>
  </nav>
</aside>
```

這裡的 `aside` 表示延伸閱讀區，裡面剛好又有一組導覽連結，所以可以再放 `nav`。

#### 使用提醒

`aside` 的重點是「內容是否為補充或旁支」，不是它在畫面上位於左邊還是右邊。

畫面位置應該交給 CSS 決定，例如：

```css
.layout {
  display: grid;
  grid-template-columns: 1fr 280px;
}
```

### 9. `footer`

#### 核心用途

`footer` 表示頁面或區塊的結尾資訊。

常見內容包含：

- 版權資訊
- 作者資訊
- 聯絡方式
- 相關連結
- 最後更新時間
- 文章標籤
- 返回頂部連結

#### 整頁的 `footer`

```html
<footer>
  <small>© 2026 Frontend Notes. All rights reserved.</small>
</footer>
```

#### 文章裡的 `footer`

```html
<article>
  <h2>HTML5 布局標籤</h2>
  <p>這篇文章介紹常見結構標籤。</p>

  <footer>
    <p>作者：Siao</p>
    <p>最後更新：2026-04-26</p>
  </footer>
</article>
```

這裡的 `footer` 是文章的結尾資訊，不是整個網站的頁尾。

#### 使用提醒

`footer` 不會自動建立新的區段。  
它只是描述某個頁面或區塊的結尾資訊。

---

### 10. `hgroup`

#### 核心用途

`hgroup` 用來把一個標題和它的相關副標題、替代標題或標語分成一組。

比較適合的寫法是：

```html
<hgroup>
  <h1>HTML5 布局標籤</h1>
  <p>理解頁面結構與語意化標籤</p>
</hgroup>
```

其中：

- `h1` 是真正的主標題
- `p` 是副標題或標語

#### 不建議這樣理解

不要把 `hgroup` 簡單理解成「把很多標題全部包起來」：

```html
<hgroup>
  <h1>HTML5 布局標籤</h1>
  <h2>header</h2>
  <h2>nav</h2>
</hgroup>
```

這樣容易讓文件大綱變得不清楚。

如果 `h2` 是獨立小節標題，就應該寫成正常的小節：

```html
<h1>HTML5 布局標籤</h1>

<section>
  <h2>header</h2>
  <p>header 表示頁面或區塊的開頭。</p>
</section>

<section>
  <h2>nav</h2>
  <p>nav 表示導航區。</p>
</section>
```

#### 使用提醒

`hgroup` 實務上不是高頻標籤。  
你可以先知道它是「主標題 + 副標題 / 標語」的群組工具，不需要把它當成本章主角。

---

### 11. `div`

#### 核心用途

`div` 是沒有語意的通用容器。

它最適合用在：

- 純 CSS 排版
- 包一層容器
- 配合 JavaScript 操作
- 沒有明確語意的結構分組
- UI 組件內部包裝

#### 基本範例

```html
<div class="layout">
  <main>
    ...
  </main>

  <aside>
    ...
  </aside>
</div>
```

這裡的 `.layout` 只是為了做版面配置，所以用 `div` 很合適。

#### 使用提醒

不要以為語意標籤越多越好。  
如果一個區塊沒有明確角色，硬套 `section`、`article`、`aside` 反而會讓語意變混亂。

## 完整頁面結構範例

下面是一個比較清楚的頁面骨架：

```html
<body>
  <header class="site-header">
    <h1>前端學習筆記</h1>
    <p>整理 HTML、CSS、JavaScript 的學習內容</p>
  </header>

  <nav aria-label="主要導覽">
    <ul>
      <li><a href="/">首頁</a></li>
      <li><a href="/html">HTML</a></li>
      <li><a href="/css">CSS</a></li>
      <li><a href="/javascript">JavaScript</a></li>
    </ul>
  </nav>

  <div class="page-layout">
    <main id="main-content">
      <article>
        <header>
          <h2>HTML5 語意化布局介紹</h2>
          <p>理解常見結構標籤的使用場景</p>
        </header>

        <section>
          <h3>為什麼需要語意化布局？</h3>
          <p>
            語意化布局可以讓頁面不只看起來有結構，
            也能在 HTML 層級清楚表達內容角色。
          </p>
        </section>

        <section>
          <h3>常見布局標籤有哪些？</h3>
          <p>
            常見標籤包含 header、nav、main、article、
            section、aside、footer 與 hgroup。
          </p>
        </section>

        <footer>
          <p>最後更新：2026-04-26</p>
        </footer>
      </article>
    </main>

    <aside aria-labelledby="related-title">
      <h2 id="related-title">延伸閱讀</h2>

      <nav aria-label="延伸閱讀">
        <ul>
          <li><a href="/html/semantic">語意化標籤</a></li>
          <li><a href="/html/headings">標題標籤</a></li>
          <li><a href="/css/layout">CSS 版面配置</a></li>
        </ul>
      </nav>
    </aside>
  </div>

  <footer class="site-footer">
    <small>© 2026 Frontend Notes. All rights reserved.</small>
  </footer>
</body>
```

### 這個範例的角色分工

| 區塊 | 標籤 | 角色 |
|---|---|---|
| 網站名稱與導言 | `header` | 整頁頁首 |
| 主選單 | `nav` | 主要導覽 |
| `.page-layout` | `div` | 純版面容器 |
| 文章主體 | `main` | 本頁主要內容 |
| 一篇完整教學文章 | `article` | 可獨立閱讀的內容 |
| 文章開頭 | `article > header` | 文章標題與導言 |
| 文章小節 | `section` | 同一文章中的主題分段 |
| 文章結尾資訊 | `article > footer` | 文章更新資訊 |
| 延伸閱讀 | `aside` | 補充資訊 |
| 延伸閱讀連結 | `aside > nav` | 補充區內的導覽 |
| 網站版權 | `footer` | 整頁頁尾 |

## 常見問法

### `main` 真的很少用嗎？

不是。  
`main` 是很重要的結構標籤，用來標示頁面的主要內容。

一般頁面通常適合有一個 `main`，讓讀者、瀏覽器與輔助技術更快理解哪裡是本頁核心內容。

### `main` 裡面可以放 `article` 嗎？

可以，而且很常見。

```html
<main>
  <article>
    <h1>文章標題</h1>
    <p>文章內容</p>
  </article>
</main>
```

這表示：

- `main`：本頁主要內容區
- `article`：主要內容區中的一篇獨立文章

### `article` 裡面可以放 `section` 嗎？

可以。

```html
<article>
  <h1>文章標題</h1>

  <section>
    <h2>第一章</h2>
    <p>...</p>
  </section>

  <section>
    <h2>第二章</h2>
    <p>...</p>
  </section>
</article>
```

這是很合理的結構：  
整篇文章是 `article`，文章內的章節是 `section`。

### `section` 裡面可以放 `article` 嗎？

可以。

```html
<section>
  <h2>最新文章</h2>

  <article>
    <h3>HTML5 布局標籤</h3>
    <p>...</p>
  </article>

  <article>
    <h3>CSS Flexbox 入門</h3>
    <p>...</p>
  </article>
</section>
```

這表示：  
「最新文章」是一個主題區塊，裡面有多篇可獨立閱讀的文章。

### `section` 一定要有標題嗎？

實務上通常應該有。

因為 `section` 表示一個主題區段，如果沒有標題，讀者很難知道這個區段的主題是什麼。

如果只是為了包一層樣式，應該使用 `div`。

### `aside` 一定要放在 `main` 外面嗎？

不一定，要看它補充的是誰。

如果它是整個頁面的補充內容，例如全站側欄、廣告、推薦連結，常見寫法是放在 `main` 外面，和 `main` 並列。

```html
<div class="layout">
  <main>主要內容</main>
  <aside>頁面補充資訊</aside>
</div>
```

如果它是某篇文章內的補充說明，也可以放在 `article` 裡面。

```html
<article>
  <h1>文章標題</h1>
  <p>文章內容</p>

  <aside>
    <h2>補充說明</h2>
    <p>這是和本文有關的旁支資訊。</p>
  </aside>
</article>
```

### `nav` 裡面一定要用 `ul` / `ol` 嗎？

不一定，但常見且推薦。

如果是一組選單連結，用 `ul` 或 `ol` 可以讓結構更清楚：

```html
<nav aria-label="主要導覽">
  <ul>
    <li><a href="/">首頁</a></li>
    <li><a href="/about">關於我</a></li>
  </ul>
</nav>
```

如果只是簡單幾個連結，也可以直接放：

```html
<nav aria-label="分頁導覽">
  <a href="/page/1">上一頁</a>
  <a href="/page/3">下一頁</a>
</nav>
```

### `hgroup` 要不要特別背？

不用。

它不是日常最高頻的布局標籤。  
你先記得：`hgroup` 適合處理「一個主標題 + 副標題 / 標語」這種標題群組即可。
