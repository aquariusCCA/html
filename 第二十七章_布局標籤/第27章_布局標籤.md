## 30 秒複習入口

- 布局標籤的核心用途是表達區塊角色，不是取代 CSS 布局。
- `header`、`nav`、`main`、`article`、`section`、`aside`、`footer` 是最常見的結構標籤。
- `article` 強調內容可獨立成篇，`section` 強調同一主題下的區段。
- `section` 通常要有標題；如果只是樣式包裝，通常用 `div`。
- `main` 代表頁面主要內容，一般一個頁面只會有一個主要且可見的 `main`。
- `hgroup` 可用來把主標題和副標題包成一組，但實務上相對少見。

## 速查區

### 核心概念

- HTML5 布局標籤是在替頁面區塊補上「語意」。
- 它們幫助開發者、搜尋引擎與輔助技術理解每個區塊在頁面中的角色。
- 真正控制排版位置的是 CSS；標籤本身主要負責描述結構。

### 常見標籤與用途

- `header`：頁面或區塊的開頭，常放標題、Logo、導言。
- `nav`：主要導航區。
- `main`：頁面的主要內容區。
- `article`：可獨立閱讀、可獨立分發的內容單元。
- `section`：同一主題下的分段或分區。
- `aside`：補充、側欄、旁支資訊。
- `footer`：頁面或區塊的結尾資訊。
- `hgroup`：把標題和副標題、標語這類相關標題內容分成一組。

### 關鍵判準

- 想表達「這是一篇可以單獨存在的內容」時，用 `article`。
- 想把同一主題切成幾個段落或區塊時，用 `section`。
- 想放主要導航時，用 `nav`。
- 想放補充但不是主線內容的資訊時，用 `aside`。
- 如果只是為了 CSS 包一層容器，而沒有特定語意，通常用 `div`。

### 常見錯誤

- 把語意標籤當成純視覺容器，卻不管內容角色。
- 以為 `main` 幾乎不用；其實它是常見且重要的主要內容標記。
- 以為 `article` 和 `section` 可以完全互換。
- `section` 裡沒有標題，導致區塊語意不清楚。

### 一句話抓核心

- 布局標籤是在替網頁區塊命名角色，讓頁面不只是「排得出來」，也能「看得懂結構」。

## 正文筆記

### 1. 布局標籤在解決什麼問題？

在較早期的 HTML 寫法中，很多區塊都只用 `div` 包起來。  
這樣雖然也能配合 CSS 做出頁面，但光看標籤本身，很難立刻知道哪裡是頁首、哪裡是導航、哪裡是正文、哪裡是補充資訊。

HTML5 新增這些布局標籤，就是希望開發者在切頁面時，能直接把區塊角色表達出來。  
因此它們的重點是「語意化結構」，不是「視覺排版技巧」。

### 2. 常見布局標籤怎麼理解？

#### `header`

- 表示頁面或某個區塊的開頭。
- 常放網站名稱、標題、導言、導覽前的介紹資訊。

#### `nav`

- 表示導航區。
- 常見於網站主選單、章節切換、主要連結群。

#### `main`

- 表示頁面的主要內容。
- 一般一個頁面只會有一個主要且可見的 `main`。
- 它不是「很少用」，反而是描述正文主體時很有價值的標籤。

#### `article`

- 表示一塊相對獨立、完整，甚至可單獨轉載或抽出來閱讀的內容。
- 例如：文章、貼文、留言、商品卡片、新聞條目。

#### `section`

- 表示文件中的一個主題區段。
- 它通常有自己的標題，用來把同一主題拆成幾個部分。
- 如果有更具體的標籤可用，就優先用更具體的標籤，不要什麼都包成 `section`。

#### `aside`

- 表示和主內容相關，但不是主線本身的補充資訊。
- 常見於側欄、延伸閱讀、作者資訊、相關連結。

#### `footer`

- 表示頁面或區塊的結尾資訊。
- 常放版權、作者資訊、相關連結、返回頂部等內容。

#### `hgroup`

- 用來把一個標題和它的副標題或標語包成一組。
- 這個標籤不是日常最常用的布局工具，但可以理解它是在處理「一組標題內容」。

### 3. `article` 和 `section` 怎麼分？

這是最容易混淆的地方，可以用下面的方式判斷：

- `article`：強調內容本身可以獨立成立。
- `section`：強調這是同一份內容中的其中一段。

可先用這個問題判斷：

- 如果把這塊內容單獨拿出去，仍然像一篇完整內容，偏向 `article`。
- 如果它只是同一主題下的某個小節，偏向 `section`。

兩者可以嵌套：

- `article` 裡面可以再切成多個 `section`
- `section` 裡面也可能包含多個獨立 `article`

但不要把它們理解成「只是名字不同、功能完全一樣」。

### 4. 一個簡單的頁面結構範例

大致布局示意如下：

![布局介面.png](./images/image_1776154064289_310f9cf8.png)

下面這個例子把頁首、主導航、主要內容、側欄、文章內容與頁尾分開標示：

```html
<header>
  <h1>尚品匯</h1>
</header>

<nav>
  <a href="#">首頁</a>
  <a href="#">訂單</a>
  <a href="#">購物車</a>
  <a href="#">我的</a>
</nav>

<main>
  <aside>
    <nav>
      <ul>
        <li><a href="#">秒殺專區</a></li>
        <li><a href="#">會員專區</a></li>
        <li><a href="#">領取優惠券</a></li>
        <li><a href="#">品牌專區</a></li>
      </ul>
    </nav>
  </aside>

  <article>
    <h2>如何一夜暴富</h2>

    <section>
      <h3>第一種方式：通過作夢</h3>
      <p>你要這麼作夢：xxxxxxxxxxxxxx</p>
    </section>

    <section>
      <h3>第二種方式：通過買彩票</h3>
      <p>你要這麼買彩票：xxxxxxxxxxxx</p>
    </section>

    <section>
      <h3>第三種方式：通過學習前端</h3>
      <p>你要來到尚硅谷前端：xxxxxxxxxxxxx</p>
    </section>
  </article>
</main>

<footer>
  <nav>
    <a href="#">友情鏈接 1</a>
    <a href="#">友情鏈接 2</a>
    <a href="#">友情鏈接 3</a>
    <a href="#">友情鏈接 4</a>
  </nav>
</footer>
```

這個範例的重點不是版面漂不漂亮，而是角色分工是否清楚：

- `header` 放網站標題
- `nav` 放主導航
- `main` 包主要內容
- `aside` 放側欄導航或補充內容
- `article` 表示一塊可獨立閱讀的正文
- `section` 把正文再切成幾個小段
- `footer` 放頁尾資訊

### 5. 什麼時候不要硬用語意標籤？

如果你只是想多包一層來做 CSS 排版，卻沒有更明確的內容角色，直接用 `div` 就可以。  
語意標籤不是越多越好，而是要用在「真的能表達角色」的地方。

## 常見問法

### `main` 真的很少用嗎？

不是。  
它用來表示頁面主要內容，對結構理解和輔助技術都很有幫助。一般頁面通常都適合有一個 `main`。

### `article` 和 `section` 最大差別是什麼？

- `article` 強調獨立性。
- `section` 強調分段性。

如果拿出去能單獨成立，通常偏向 `article`；如果只是同主題下的一部分，通常偏向 `section`。

### `section` 一定要有標題嗎？

通常應該有。  
如果沒有標題，這個區段的主題會變得不清楚，也比較不利於理解與回查。

### `hgroup` 要不要特別背？

不用把它當高頻主角。  
先知道它是「把標題和副標題包成一組」的標籤即可；日常最常用的仍然是 `header`、`nav`、`main`、`article`、`section`、`aside`、`footer`。
