HTML 可存取性基礎 Demo

這個範例對應筆記中的核心原則：先使用正確的 HTML，再補 ARIA。頁面示範 landmark、標題層級、圖片替代文字、表單標籤、表格標題、鍵盤操作、`tabindex` 與 `aria-expanded` / `aria-live`。

手動測試

  • 按 `Tab` 檢查焦點順序是否合理。
  • 按 skip link 確認可跳到主要內容。
  • 用 Enter / Space 操作按鈕與選單。
  • 觀察錯誤訊息是否和欄位關聯。
semantic first

語意化 HTML 優先於 ARIA

不理想

<div class="title">課程總覽</div>
<div class="btn" onclick="save()">儲存</div>
<span onclick="location.href='./notes.html'">看更多</span>

視覺上像標題、按鈕、連結，不代表輔助技術會把它們理解成標題、按鈕、連結。

建議

<h2>課程總覽</h2>
<button type="button">儲存</button>
<a href="./notes.html">閱讀 HTML 可存取性筆記</a>

原生元素會自帶語意、鍵盤操作、焦點行為與可預期的瀏覽器互動。

項目 建議 常見問題
標題 依內容層級使用 `h1` 到 `h6`。 只用 CSS 做大字。

tabindex

`tabindex` 使用邊界

  • `tabindex="0"`：讓自訂元件進入自然 Tab 順序，但應少用。
  • `tabindex="-1"`：可用 JS 聚焦，但不進入 Tab 順序，常用於跳轉到主要內容或錯誤摘要。
  • 正整數 `tabindex` 會改寫焦點順序，容易讓鍵盤使用者迷路。
