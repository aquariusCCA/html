# 第三十八章｜HTML 可存取性基礎

> 章節定位：用 HTML 本身建立可被鍵盤、螢幕閱讀器與搜尋工具理解的頁面基礎  
> 建議閱讀方式：先理解語意化 HTML 的價值，再看 ARIA 何時才需要補上

## 本章在學什麼

這一章整理 HTML 層面的可存取性基礎。

重點不是一開始就背 ARIA，而是先掌握：

- 正確使用語意化標籤通常是可存取性的第一步。
- 標題階層、連結文字、圖片替代文字、表單標籤、表格標頭都會影響輔助技術理解頁面。
- 鍵盤焦點與操作順序會影響不使用滑鼠的使用者。
- ARIA 是補充語意，不是替代正確 HTML 的捷徑。

## 易混淆主題

- 看得見 vs 可理解：畫面上能看不代表輔助技術能正確理解。
- ARIA vs HTML：能用正確 HTML 表達時，通常先用 HTML。
- `alt` vs `title`：圖片替代文字不是滑鼠提示文字。
