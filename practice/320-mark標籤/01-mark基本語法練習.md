# mark 基本語法練習

> 動手練習。概念與用法說明見 notes/320-mark標籤/01-mark-標籤基本用法.md。

## 題目 1（基礎）· 應用練習題

把下面句子中的 `標籤語意` 標示為目前句子中需要注意的關鍵片段。

```html
<p>學習 HTML 時，先理解 標籤語意 會比只記語法更重要。</p>
```

完成標準：

- 只用 `<mark>` 包住 `標籤語意`。
- 不要把整句都包進 `<mark>`。
- 保留原本的段落結構。

<details>
<summary>參考解答</summary>

```html
<p>學習 HTML 時，先理解 <mark>標籤語意</mark> 會比只記語法更重要。</p>
```

解題說明：`<mark>` 用來標示目前上下文中需要讀者注意或參考的文字片段。這題的重點是「標籤語意」，所以只標示該片段即可。

</details>

## 題目 2（基礎）· 應用練習題

下面這段說明正在討論 `HTML5 新增標籤`。請用 `<mark>` 標示這個目前正在討論的詞語。

```html
<p>這堂課會先介紹 HTML5 新增標籤 的用途。</p>
```

完成標準：

- 使用 `<mark>` 標示 `HTML5 新增標籤`。
- 不加入 CSS 或其他標籤。
- 標示後，讀者能看出目前討論重點。

<details>
<summary>參考解答</summary>

```html
<p>這堂課會先介紹 <mark>HTML5 新增標籤</mark> 的用途。</p>
```

解題說明：notes 提到 `<mark>` 可用來標示一段說明裡目前正在討論的詞語。這裡標示的是目前脈絡中的重點，而不是整句話的重要性。

</details>

## 題目 3（進階）· 改錯題

下面的寫法把整段都標起來，讀者很難看出真正重點。請改成只標示關鍵片段。

```html
<p><mark>這一整段很長的文字全部都被標記，讀者很難判斷真正的重點在哪裡。</mark></p>
```

完成標準：

- 不要把整段都包進 `<mark>`。
- 只標示 `真正的重點` 這個關鍵片段。
- 段落仍然是有效的 HTML。

<details>
<summary>參考解答</summary>

```html
<p>這一整段很長的文字全部都被標記，讀者很難判斷 <mark>真正的重點</mark> 在哪裡。</p>
```

解題說明：notes 說明如果一整段文字幾乎都被 `<mark>` 包起來，讀者反而看不出真正需要注意的是哪裡。比較好的做法是只標示關鍵片段。

</details>
