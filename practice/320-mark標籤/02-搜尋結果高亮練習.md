# 搜尋結果高亮練習

> 動手練習。概念與用法說明見 notes/320-mark標籤/01-mark-標籤基本用法.md。

## 題目 1（基礎）· 應用練習題

搜尋關鍵字是 `HTML`。請在下面搜尋摘要中，把命中的 `HTML` 標示出來。

```html
<p>這篇文章介紹 HTML 的基本結構與常用標籤。</p>
```

完成標準：

- 使用 `<mark>` 包住命中的 `HTML`。
- 不標示其他沒有命中的文字。
- 不把整段摘要都包起來。

<details>
<summary>參考解答</summary>

```html
<p>這篇文章介紹 <mark>HTML</mark> 的基本結構與常用標籤。</p>
```

解題說明：搜尋結果中標示命中的文字，是 notes 提到的 `<mark>` 常見使用情境。這段文字在搜尋脈絡中被特別指出來，因此適合使用 `<mark>`。

</details>

## 題目 2（基礎）· 應用練習題

下面段落正在解釋「語意標籤」這個詞。請用 `<mark>` 標示目前討論的詞語。

```html
<p>語意標籤 可以讓 HTML 不只描述畫面，也描述內容關係。</p>
```

完成標準：

- 只標示 `語意標籤`。
- 標示目的要是目前上下文中的討論重點。
- 不使用 `<strong>` 取代 `<mark>`。

<details>
<summary>參考解答</summary>

```html
<p><mark>語意標籤</mark> 可以讓 HTML 不只描述畫面，也描述內容關係。</p>
```

解題說明：這裡不是在表示「語意標籤」本身具有強烈重要性，而是在目前句子脈絡中指出正在討論的詞語，所以 `<mark>` 比 `<strong>` 更貼近語意。

</details>

## 題目 3（進階）· 實作任務

建立一小段搜尋結果摘要。使用者搜尋的是 `CSS`，摘要文字是 `CSS 可以調整 mark 的背景色與文字顏色。`。請只標示命中的 `CSS`。

完成標準：

- 使用一個 `<p>` 放搜尋摘要。
- 只用 `<mark>` 標示 `CSS`。
- 不標示 `背景色`、`文字顏色` 或整句摘要。

<details>
<summary>參考解答</summary>

```html
<p><mark>CSS</mark> 可以調整 mark 的背景色與文字顏色。</p>
```

解題說明：搜尋結果高亮的重點是命中字詞，而不是摘要中的每個看起來重要的詞。只標示 `CSS`，讀者才能快速對應自己的搜尋字串。

</details>
