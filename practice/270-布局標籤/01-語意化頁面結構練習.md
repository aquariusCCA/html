# 語意化頁面結構練習

> 動手練習。概念與用法說明見 notes/270-布局標籤/01-html5-語意化布局標籤.md。

## 題目 1（基礎）· 實作任務

請建立一個簡單網站首頁結構。

完成標準：

- 使用 `<header>` 放網站名稱。
- 使用 `<nav>` 放一組導覽連結。
- 使用 `<main>` 放頁面主要內容。
- 使用 `<footer>` 放頁尾資訊。

<details>
<summary>參考解答</summary>

```html
<header>
  <h1>尚品匯</h1>
</header>

<nav>
  <a href="#">首頁</a>
  <a href="#">商品</a>
  <a href="#">會員中心</a>
</nav>

<main>
  <h2>今日推薦</h2>
  <p>這裡放本頁最核心的內容。</p>
</main>

<footer>
  <p>Copyright 2026</p>
</footer>
```

解題說明：這些標籤的重點不是讓畫面變漂亮，而是用標籤名稱表達區塊角色。頁首、導覽、主要內容與頁尾各自有不同語意。
</details>

## 題目 2（基礎）· 應用練習題

請把下面主要內容放進合適的語意化標籤中。

```html
<article>
  <h2>前端學習路線</h2>
  <p>這篇文章介紹初學者可以如何安排 HTML、CSS 與 JavaScript 的學習順序。</p>
</article>
```

完成標準：

- 使用 `<main>` 包住頁面最核心、最直接相關的內容。
- 不把頁首、導覽列或頁尾一起塞進 `<main>`。

<details>
<summary>參考解答</summary>

```html
<main>
  <article>
    <h2>前端學習路線</h2>
    <p>這篇文章介紹初學者可以如何安排 HTML、CSS 與 JavaScript 的學習順序。</p>
  </article>
</main>
```

解題說明：`<main>` 表示頁面主要內容。一個頁面通常只會有一個主要的 `<main>`，而頁首、導覽列、側邊補充資訊與頁尾通常不屬於主要內容。
</details>

## 題目 3（進階）· 改錯題

下面頁面只用 `div` 分區。請重構成更有語意的 HTML 結構。

```html
<div class="header">網站名稱</div>
<div class="nav">首頁 商品 會員中心</div>
<div class="main">主要內容</div>
<div class="footer">Copyright 2026</div>
```

完成標準：

- 頁首改用 `<header>`。
- 導覽區改用 `<nav>`。
- 主要內容改用 `<main>`。
- 頁尾改用 `<footer>`。

<details>
<summary>參考解答</summary>

```html
<header>網站名稱</header>
<nav>首頁 商品 會員中心</nav>
<main>主要內容</main>
<footer>Copyright 2026</footer>
```

解題說明：整頁只用 `div` 畫面可能正常，但 HTML 本身沒有說明各區塊角色。語意化布局標籤能讓瀏覽器、搜尋引擎、輔助技術與其他開發者更容易理解頁面結構。
</details>
