# 語意化標籤入門練習

> 動手練習。概念與用法說明見 notes/130-語意化標籤/01-語意化標籤入門.md。

## 題目 1（基礎）· 應用練習題

請依照下列頁面角色，寫出一段語意清楚的 HTML 結構：

- 頁面最上方有網站標題。
- 網站有主要導覽選單。
- 頁面主要內容是一篇可獨立閱讀的文章。
- 文章旁邊有補充資訊。
- 頁面最下方有版權資訊。

完成標準：

- 使用能表達內容角色的語意化標籤。
- 頁面主要內容放在合適的主要內容區域。
- 補充資訊不要放進主要文章裡。
- 不用 `div` 取代每一個有明確角色的區塊。

<details>
<summary>參考解答</summary>

```html
<header>
  <h1>HTML 學習站</h1>
</header>

<nav>
  <a href="#">首頁</a>
  <a href="#">文章</a>
</nav>

<main>
  <article>
    <h2>語意化標籤入門</h2>
    <p>語意化標籤能讓 HTML 結構更容易被理解。</p>
  </article>

  <aside>
    <p>補充閱讀：HTML5 常見語意化標籤。</p>
  </aside>
</main>

<footer>
  <p>Copyright 2026 HTML 學習站</p>
</footer>
```

解題說明：`<header>` 表示頁面開頭，`<nav>` 表示導覽連結，`<main>` 表示頁面主要內容，`<article>` 表示可獨立閱讀的文章，`<aside>` 表示與主要內容相關但可分離閱讀的補充資訊，`<footer>` 表示頁尾資訊。這些標籤的重點是表達內容角色，而不是改變外觀。

</details>

## 題目 2（基礎）· 改錯題

下面這段 HTML 整頁都只用 `div` 表示結構。請把它改成更語意化的寫法。

```html
<div class="nav">首頁｜課程｜聯絡我們</div>
<div class="main">這裡是網站主要內容。</div>
<div class="footer">Copyright 2026</div>
```

完成標準：

- 導覽區改用合適的語意化標籤。
- 主要內容區改用合適的語意化標籤。
- 頁尾資訊改用合適的語意化標籤。
- 不加入 notes 沒教過的額外屬性或技術。

<details>
<summary>參考解答</summary>

```html
<nav>首頁｜課程｜聯絡我們</nav>
<main>這裡是網站主要內容。</main>
<footer>Copyright 2026</footer>
```

解題說明：原本的 `class` 名稱雖然暗示了導覽、主要內容與頁尾，但標籤本身仍是沒有語意的 `<div>`。當內容角色已經明確時，應該優先使用對應的語意化標籤，讓 HTML 本身能說明哪裡是導覽、哪裡是主要內容、哪裡是頁尾。

</details>

## 題目 3（進階）· 重構題

下面這段文章卡片使用 `div` 包出所有區塊。請在不改變內容順序的前提下，把它重構成語意更清楚的 HTML。

```html
<div class="card">
  <div class="title">學習語意化標籤</div>
  <div class="content">語意化可以讓內容角色更清楚。</div>
  <div class="note">補充：語意化不是只為了 SEO。</div>
  <div class="info">作者：小明</div>
</div>
```

完成標準：

- 整張可獨立閱讀的內容使用合適的語意化標籤。
- 標題使用標題標籤。
- 補充資訊使用合適的補充內容標籤。
- 結尾資訊使用合適的結尾標籤。

<details>
<summary>參考解答</summary>

```html
<article>
  <header>
    <h2>學習語意化標籤</h2>
  </header>

  <p>語意化可以讓內容角色更清楚。</p>

  <aside>
    <p>補充：語意化不是只為了 SEO。</p>
  </aside>

  <footer>
    <p>作者：小明</p>
  </footer>
</article>
```

解題說明：這張卡片本身是一段可獨立閱讀的內容，因此可以用 `<article>`。標題應用標題標籤表示；補充說明可以用 `<aside>`；作者資訊屬於這篇內容的結尾資訊，因此可以放在 `<footer>`。語意化的目的不是完全不用 `<div>`，而是在內容有明確角色時選擇更合適的標籤。

</details>
