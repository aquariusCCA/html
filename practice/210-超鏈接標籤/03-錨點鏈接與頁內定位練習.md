# 錨點鏈接與頁內定位練習

> 動手練習。概念與用法說明見 notes/210-超鏈接標籤/03-錨點鏈接與頁內定位.md。

## 題目 1（基礎）· 應用練習題

請建立一個頁面目錄連結，點擊「FAQ」後可以跳到同一頁中的 FAQ 段落。

完成標準：

- 連結端使用 `<a href="#faq">`。
- 目標端使用 `id="faq"`。
- `#` 只寫在 `href` 中，不寫進 `id` 值。
- 連結文字清楚描述目標段落。

<details>
<summary>參考解答</summary>

```html
<a href="#faq">FAQ</a>

<h2 id="faq">FAQ</h2>
```

解題說明：

錨點鏈接由連結端和目標端組成。`href="#faq"` 會讓瀏覽器尋找同一頁中 `id="faq"` 的元素，並嘗試捲動到那個位置。

</details>

## 題目 2（基礎）· 應用練習題

請建立一個跨頁錨點連結。連結文字是「團隊介紹」，點擊後前往 `about.html`，並定位到該頁的 `team` 區塊。

完成標準：

- `href` 同時包含頁面路徑與片段識別符。
- 使用 `about.html#team`。
- 連結文字能描述目的地。

<details>
<summary>參考解答</summary>

```html
<a href="about.html#team">團隊介紹</a>
```

解題說明：

`about.html#team` 會先前往 `about.html`，再嘗試定位到該頁中 `id="team"` 的元素。目標頁面必須真的有對應的 `id`，定位才會生效。

</details>

## 題目 3（基礎）· 改錯題

以下錨點連結無法跳到預期段落，因為 `href` 與目標 `id` 不一致。請修正它。

```html
<a href="#two">第二季</a>
<h3 id="second">第二季介绍</h3>
```

完成標準：

- `href="#two"` 對應到 `id="two"`。
- 不改變使用者看到的連結文字與標題文字。

<details>
<summary>參考解答</summary>

```html
<a href="#two">第二季</a>
<h3 id="two">第二季介绍</h3>
```

解題說明：

`href` 裡的 `#two` 和目標元素的 `id="two"` 必須對上。若目標元素是 `id="second"`，瀏覽器找不到 `id="two"` 的元素，就無法跳到預期位置。

</details>

## 題目 4（進階）· 改錯題

以下程式碼有兩個錨點問題：第一段把 `#` 寫進 `id`，第二段重複使用相同 `id`。請修正它們。

```html
<a href="#two">第二季</a>
<h3 id="#two">第二季介绍</h3>

<h2 id="intro">第一段介紹</h2>
<h2 id="intro">第二段介紹</h2>
```

完成標準：

- `id` 的值不要包含 `#`。
- 同一頁中的 `id` 不重複。
- 修正後仍保留可理解的錨點名稱。

<details>
<summary>參考解答</summary>

```html
<a href="#two">第二季</a>
<h3 id="two">第二季介绍</h3>

<h2 id="intro-first">第一段介紹</h2>
<h2 id="intro-second">第二段介紹</h2>
```

解題說明：

`#` 是寫在 `href` 裡用來表示錨點的符號，`id` 的值本身不要寫 `#`。同一頁的 `id` 也應保持唯一，否則定位結果不可靠，並會影響 JavaScript 與無障礙工具辨識頁面。

</details>
