# 文件下載與元素連結練習

> 動手練習。概念與用法說明見 notes/210-超鏈接標籤/02-文件下載與元素連結.md。

## 題目 1（基礎）· 應用練習題

請建立兩個檔案連結：一個用來預覽 PDF，一個用來連到 zip 壓縮檔。PDF 路徑是 `./assets/pdfs/guide.pdf`，連結文字是「看 PDF 檔」；zip 路徑是 `./assets/files/resources.zip`，連結文字是「內部資源」。

完成標準：

- 兩個連結都使用 `<a>`。
- `href` 指向正確檔案路徑。
- PDF 連結文字表達「看」或「預覽」。
- zip 連結文字表達資源內容。

<details>
<summary>參考解答</summary>

```html
<a href="./assets/pdfs/guide.pdf">看 PDF 檔</a>
<a href="./assets/files/resources.zip">內部資源</a>
```

解題說明：

`<a>` 的 `href` 可以指向網頁，也可以指向檔案。瀏覽器是開啟、預覽還是下載，會受到檔案類型、伺服器設定與瀏覽器支援影響。

</details>

## 題目 2（基礎）· 應用練習題

請建立一個提示下載 PDF 的連結。檔案路徑是 `./assets/pdfs/guide.pdf`，建議下載檔名是 `課程講義.pdf`，連結文字是「下載課程講義」。

完成標準：

- 使用 `<a>`。
- `href` 指向 PDF。
- 使用 `download` 屬性提示瀏覽器下載。
- `download` 的屬性值作為建議下載檔名。

<details>
<summary>參考解答</summary>

```html
<a href="./assets/pdfs/guide.pdf" download="課程講義.pdf">下載課程講義</a>
```

解題說明：

`download` 可以提示瀏覽器下載目標資源，屬性值可作為建議下載檔名。不過它不是絕對命令，實際行為仍可能受到同源、伺服器標頭、瀏覽器安全策略與使用者設定影響。

</details>

## 題目 3（基礎）· 改錯題

以下 Logo 圖片被包在首頁連結中，但圖片缺少替代文字。請修正它，讓圖片無法載入或被螢幕報讀器讀取時，仍能知道連結目的。

```html
<a href="index.html">
  <img src="logo.png" />
</a>
```

完成標準：

- 保留圖片連結結構。
- 補上能描述連結目的的 `alt`。
- 不只寫「圖片」這類沒有目的資訊的替代文字。

<details>
<summary>參考解答</summary>

```html
<a href="index.html">
  <img src="logo.png" alt="回到首頁" />
</a>
```

解題說明：

圖片可以包在 `<a>` 裡，讓整張圖片變成可點擊範圍。當圖片本身是連結入口時，`alt` 應該說明這個圖片連結的目的，例如「回到首頁」。

</details>

## 題目 4（進階）· 改錯題

以下程式碼把 `<button>` 放進 `<a>`，造成互動語意混亂。請依照「前往頁面」與「觸發操作」分別改成合適寫法。

```html
<a href="product.html">
  <button>查看商品</button>
</a>
```

完成標準：

- 前往頁面時，直接使用 `<a>`。
- 觸發操作時，使用 `<button>`。
- 不把 `<button>` 放進 `<a>`。

<details>
<summary>參考解答</summary>

```html
<!-- 前往頁面：使用 a -->
<a href="product.html" class="button-link">查看商品</a>

<!-- 觸發操作：使用 button -->
<button type="button">加入收藏</button>
```

解題說明：

`<a>` 適合導航到某個位置，`<button>` 適合觸發操作。互動元素包互動元素會讓瀏覽器、鍵盤操作與輔助技術難以判斷使用者到底要操作哪一層。

</details>
