# HTML 標籤結構與元素關係練習

> 動手練習。概念與用法說明見 `notes/020-HTML簡介/04-HTML標籤與元素關係.md`。

---

## 題目 1（基礎）· 應用練習題

根據以下關係描述，寫出對應的 HTML 片段：

- 最外層是一個 `<div>` 元素。
- `<div>` 內有兩個 `<p>` 元素，這兩個 `<p>` 互為兄弟元素。
- 第一個 `<p>` 的文字內容為「一般文字」。
- 第二個 `<p>` 內含一個 `<strong>` 元素，`<strong>` 的文字內容為「重點文字」，`<strong>` 是 `<p>` 的子元素、`<div>` 的後代元素。

完成標準：
1. 巢狀層級與描述一致（`div` → `p`；第二個 `p` → `strong`）。
2. 所有雙標籤都有正確的起始標籤與結束標籤。
3. 兩個 `<p>` 位於同一層（同為 `<div>` 的子元素）。

<details>
<summary>參考解答</summary>

```html
<div>
  <p>一般文字</p>
  <p><strong>重點文字</strong></p>
</div>
```

解題說明：

- `<div>` 是最外層的父元素，兩個 `<p>` 都是它的直接子元素，彼此互為兄弟元素。
- 第二個 `<p>` 直接包含 `<strong>`，所以 `<p>` 是 `<strong>` 的父元素；`<div>` 則是 `<strong>` 的祖先元素（中間隔了一層 `<p>`，不是直接的父元素）。
- `<div>`、`<p>`、`<strong>` 都是雙標籤，每個起始標籤都需要對應的結束標籤；結束標籤比起始標籤多一個 `/`，例如 `</strong>`、`</p>`、`</div>`。

</details>

## 題目 2（基礎）· 改錯題

下面這段 HTML 的標籤巢狀順序有錯誤，請找出問題所在，說明原因，並寫出修正後的版本。

```html
<div>
  <p>第一段文字，<strong>重點內容</div></strong></p>
  <p>第二段文字。</p>
```

完成標準：
1. 找出結束標籤順序錯誤的位置並說明原因。
2. 寫出修正後的版本，確保所有標籤都正確配對且遵循「後開的先關」原則。
3. 修正後最外層的 `<div>` 也需要有對應的結束標籤。

<details>
<summary>參考解答</summary>

**錯誤說明：**

這段程式碼有兩處問題：

1. `<strong>` 在 `<p>` 的內部開啟，但結束標籤 `</strong>` 卻出現在 `</div>` 的後面，違反了「後開的先關」原則——`<strong>` 比 `<div>` 晚開啟，應該比 `<div>` 更早關閉。
2. 最外層的 `<div>` 缺少結束標籤 `</div>`（正確的 `</div>` 應出現在兩個 `<p>` 的後面，而非夾在 `<p>` 與 `<strong>` 的中間）。

瀏覽器雖然會嘗試自動修正這類錯誤，但修正後的 DOM 結構不一定符合原本的預期，容易導致樣式套用錯誤且難以察覺。

**修正後的版本：**

```html
<div>
  <p>第一段文字，<strong>重點內容</strong></p>
  <p>第二段文字。</p>
</div>
```

解題說明：正確的巢狀順序是：`<div>` 最先開啟、最後關閉；`<p>` 在 `<div>` 內開啟、在 `<div>` 內關閉；`<strong>` 在 `<p>` 內開啟、在 `<p>` 內關閉。這樣每一層的結束標籤都在自己的「開啟範圍」內，結構清晰、不會交叉。

</details>
