# 判斷塊級與行內元素的排版行為

> 動手練習。概念與用法說明見 `notes/070-HTML塊級元素與行內元素/01-塊級元素與行內元素的排版差異.md`。

---

## 題目 1（基礎）· 應用練習題

觀察以下 HTML，判斷每一組元素在瀏覽器預設排版中比較可能如何排列。

```html
<h1>課程標題</h1>
<p>第一段說明文字。</p>
<div>延伸補充區塊</div>

<input type="text">
<span>請輸入姓名</span>
```

請回答：

1. 哪些元素通常會上下排列？
2. 哪些元素通常會留在同一行，直到空間不夠才換行？
3. 為什麼不能只用「畫面上有沒有換行」來決定 HTML 標籤？

完成標準：

1. 能正確指出 `h1`、`p`、`div` 的預設區塊行為。
2. 能正確指出 `input`、`span` 的預設行內流動行為。
3. 說明選標籤時還要考慮語意，而不是只看排版。

<details>
<summary>參考解答</summary>

1. `h1`、`p`、`div` 是常見塊級元素，瀏覽器預設通常會讓它們一個接一個上下排列。
2. `input` 和 `span` 通常會和行內內容一起流動，若同一行還有空間，會排在同一行。
3. HTML 標籤不只影響預設排版，也帶有語意與內容模型。例如 `p` 代表段落，`h1` 代表最高層級標題，`span` 是沒有特殊語意的行內容器。實務上應先選語意正確的標籤，再用 CSS 調整排版。

解題說明：

- 塊級元素偏向建立一整塊內容區域，常見行為是另起一行。
- 行內元素偏向包住一行文字中的片段，常見行為是跟文字一起排在同一行。
- 「會不會換行」只是預設排版行為，不足以取代語意判斷。

</details>

## 題目 2（基礎）· 判斷題

請判斷下列說法是否正確，並簡短說明原因。

**(A)** 塊級元素預設常會填滿可用水平空間，所以 `width: auto` 可以直接當成 `width: 100%` 理解。

**(B)** 一般 `span`、`em` 這類 non-replaced inline 元素，不適合直接靠 `width`、`height` 改變主要尺寸。

**(C)** 只要是行內元素，就一定不能有可控寬高。

完成標準：

1. 能分辨 `width: auto` 與 `width: 100%` 不能直接畫上等號。
2. 能說明一般 non-replaced inline 元素的尺寸主要由內容決定。
3. 能指出 `img`、`input`、`inline-block` 是重要例外。

<details>
<summary>參考解答</summary>

**(A) 錯誤。**

塊級元素預設常會填滿可用水平空間，但這通常來自 `width: auto`，不完全等於 `width: 100%`。遇到 margin、padding、border 或盒模型計算時，兩者可能產生不同結果。

**(B) 正確。**

一般 `span`、`em` 這類 non-replaced inline 元素的寬度主要由內容決定，不適合直接用 `width`、`height` 控制主要尺寸。

**(C) 錯誤。**

不能把「行內元素」簡化成「一定不能有可控寬高」。`img`、部分表單控制項，以及被 CSS 設為 `inline-block` 的元素，都可能有可控尺寸。

</details>

## 題目 3（進階）· 情境選擇題

請根據情境選擇較合適的 HTML 標籤，並說明理由。

| 情境 | 候選標籤 |
| --- | --- |
| 一篇文章的主標題 | `h1` / `span` |
| 一段完整段落文字 | `p` / `div` |
| 包住句子中的幾個字以便套樣式 | `div` / `span` |
| 建立沒有特定語意的版面容器 | `div` / `em` |

完成標準：

1. 每個情境都選出較合理的標籤。
2. 理由要同時提到語意或常見用途。
3. 不把標籤選擇只歸因於「會不會換行」。

<details>
<summary>參考解答</summary>

| 情境 | 較合適選擇 | 理由 |
| --- | --- | --- |
| 一篇文章的主標題 | `h1` | `h1` 表示最高層級標題，語意符合主標題。 |
| 一段完整段落文字 | `p` | `p` 表示段落，比只為了排版使用 `div` 更有語意。 |
| 包住句子中的幾個字以便套樣式 | `span` | `span` 是通用行內容器，適合包住文字中的小片段。 |
| 建立沒有特定語意的版面容器 | `div` | `div` 是通用區塊容器，常用於分組或排版。 |

解題說明：選標籤時應先看內容語意，再看需要的結構與排版。預設排版可以用 CSS 調整，語意錯了則會影響文件結構的可理解性。

</details>
