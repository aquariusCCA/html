# 用內容模型檢查巢狀結構

> 動手練習。概念與用法說明見 `notes/070-HTML塊級元素與行內元素/02-HTML元素的內容模型與基本巢狀規則.md`、`notes/070-HTML塊級元素與行內元素/03-常見錯誤巢狀：標題p與a標籤.md`。

---

## 題目 1（基礎）· 應用練習題

請判斷下列 HTML 片段是否適合，並說明原因。

**(A)**

```html
<div>
  <span>尚硅谷1</span>
  <input type="text">
  <div>尚硅谷2</div>
</div>
```

**(B)**

```html
<span>
  <span>尚硅谷1</span>
  <input type="text">
  <span>尚硅谷2</span>
</span>
```

**(C)**

```html
<span>
  <div>這是一個區塊</div>
</span>
```

完成標準：

1. 能判斷 `div` 作為通用 flow 容器時可包含多數 flow content。
2. 能判斷 `span` 通常適合包含 phrasing content。
3. 能說明 `span` 包 `div` 的問題不只是畫面換行，而是內容模型不適合。

<details>
<summary>參考解答</summary>

**(A) 適合。**

外層 `div` 是通用區塊容器，通常可以包含文字、行內元素、表單控制項，以及其他區塊容器。這裡的 `span`、`input`、內層 `div` 都可以放在外層 `div` 中。

**(B) 通常適合。**

外層 `span` 是通用行內容器，裡面放的是 `span`、`input`、`span` 這類 phrasing content，通常是合理的。

**(C) 不適合。**

`span` 通常只應包含 phrasing content，不適合放入 `div` 這類區塊內容。問題不是 `div` 會不會換行，而是 `span` 的內容模型不適合包含這種內容。

</details>

## 題目 2（進階）· 改錯題

下面這段 HTML 混用了幾個常見錯誤觀念。請找出不適合的巢狀，說明原因，並改成較合理的結構。

```html
<main>
  <p>
    最新課程資訊
    <div>本週開放報名</div>
  </p>

  <span>
    <div>課程特色</div>
  </span>
</main>
```

完成標準：

1. 找出 `p` 包 `div` 的問題。
2. 找出 `span` 包 `div` 的問題。
3. 修正後要保留原本的文字內容。
4. 能說明修正依據是 content model，而不只是排版外觀。

<details>
<summary>參考解答</summary>

**錯誤說明：**

1. `p` 代表段落，適合放段落中的文字與 phrasing content，不適合包含 `div` 這類區塊內容。
2. `span` 是通用行內容器，通常只適合包含 phrasing content，不適合拿來包住 `div`。

**修正後的版本：**

```html
<main>
  <p>最新課程資訊</p>

  <div>本週開放報名</div>

  <div>課程特色</div>
</main>
```

解題說明：

- 如果「本週開放報名」是一整塊內容，可以把它放在 `p` 外，使用 `div` 作為區塊容器。
- 如果「課程特色」是區塊內容，也不應該先用 `span` 包住再放 `div`。
- 瀏覽器可能會修正錯誤巢狀，但原始 HTML 仍然不是良好結構。

</details>

## 題目 3（基礎）· 概念說明題

請用自己的話說明：為什麼「塊級元素裡可以放塊級元素與行內元素，行內元素裡只能放行內元素」只是初學直覺，而不是完整規則？

完成標準：

1. 回答中要提到 HTML 的 content model。
2. 能指出 `div`、`span` 可以建立基本直覺，但不能套用到所有元素。
3. 至少舉出 `p`、`h1` 到 `h6`、`a` 其中兩種特殊限制。

<details>
<summary>參考解答</summary>

HTML 巢狀是否合法，真正要看每個元素允許的 content model，而不是只看它在畫面上像 block 或 inline。

`div` 可以幫我們理解通用 flow 容器，`span` 可以幫我們理解通用 phrasing 容器，但不能因此認為所有塊級元素都像 `div` 一樣自由。例如 `p` 不應包含 `div` 這類區塊內容，`h1` 到 `h6` 不應互相嵌套，`a` 雖然有 transparent content model，但不能包含另一個 `a` 或互動內容。

所以 block/inline 是排版直覺，不是完整的合法性規則。

</details>
