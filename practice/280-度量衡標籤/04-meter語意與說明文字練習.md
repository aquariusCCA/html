# meter 語意與說明文字練習

> 動手練習。概念與用法說明見 notes/280-度量衡標籤/01-meter-度量衡標籤.md。

## 題目 1（基礎）· 應用練習題

請建立一個磁碟用量量表，表示目前用了 `80GB`，總容量是 `100GB`。

完成標準：

- 使用文字說明這是磁碟用量。
- 使用 `<meter>` 表示已知範圍內的測量值。
- `value="80"`、`min="0"`、`max="100"`。
- 量表內提供 `80GB / 100GB` 文字。

<details>
<summary>參考解答</summary>

```html
<span>磁碟用量：</span>
<meter value="80" min="0" max="100">80GB / 100GB</meter>
```

解題說明：磁碟用量有明確的最小值與最大值，且是在描述容量用量的狀態，因此適合用 `<meter>`。文字說明能讓使用者知道量表代表什麼。
</details>

## 題目 2（進階）· 改錯題

下面量表缺少上下文，使用者可能不知道它代表什麼。請補上文字說明。

```html
<meter value="70" min="0" max="100"></meter>
```

完成標準：

- 加上量表代表內容的文字說明。
- 保留 `<meter>` 的 `value`、`min`、`max`。
- 量表內提供可讀文字。

<details>
<summary>參考解答</summary>

```html
<span>手機電量：</span>
<meter value="70" min="0" max="100">70%</meter>
```

解題說明：畫面上只有量表時，使用者可能不知道它代表電量、容量、評分或其他數值。通常應該搭配文字標籤或上下文說明。
</details>

## 題目 3（進階）· 改錯題

下面程式碼只是想顯示普通數字 `50`，並沒有要表達已知範圍內的測量值。請改成更合適的寫法。

```html
<meter value="50" min="0" max="100">50</meter>
```

完成標準：

- 不使用 `<meter>`。
- 保留要顯示的數字。
- 說明為什麼這個情境不適合 `<meter>`。

<details>
<summary>參考解答</summary>

```html
50
```

解題說明：如果只是想印出數字 `50`，不需要 `<meter>`。只有當這個數字具有「已知範圍內的測量值」語意時，才適合使用 `<meter>`。
</details>
