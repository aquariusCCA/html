# 路徑練習

> 動手練習。概念與用法說明見 `notes/060-路徑/01-路徑：讓 HTML 找到圖片與資源.md`。

---

## 題目 1（基礎）· 應用練習題

以下是一個專案的資料夾結構：

```text
project/
  index.html
  about/
    contact.html
    contact-bg.jpg
  images/
    logo.png
    hero.jpg
```

請依各小題的需求，寫出正確的 `src` 屬性值：

**(A)** `index.html` 要引用 `images/hero.jpg`：

```html
<img src="___" alt="首頁橫幅">
```

**(B)** `contact.html` 要引用 `images/logo.png`：

```html
<img src="___" alt="網站標誌">
```

**(C)** `contact.html` 要引用與它位於同一資料夾的 `contact-bg.jpg`：

```html
<img src="___" alt="聯絡頁背景">
```

完成標準：三個路徑全部正確，能反映各 HTML 檔案的實際位置與目標檔案的相對關係。

<details>
<summary>參考解答</summary>

**(A)**
```html
<img src="images/hero.jpg" alt="首頁橫幅">
```

**(B)**
```html
<img src="../images/logo.png" alt="網站標誌">
```

**(C)**
```html
<img src="contact-bg.jpg" alt="聯絡頁背景">
```

解題說明：

- **(A)** `index.html` 位於 `project/`，`hero.jpg` 位於 `project/images/`，從 `index.html` 往下一層進入 `images/` 即可，路徑為 `images/hero.jpg`。
- **(B)** `contact.html` 位於 `project/about/`，`logo.png` 位於 `project/images/`。需要先用 `../` 回到上一層 `project/`，再進入 `images/`，完整路徑為 `../images/logo.png`。
- **(C)** `contact.html` 和 `contact-bg.jpg` 在同一個 `about/` 資料夾，直接寫檔名即可。也可以寫 `./contact-bg.jpg`，`./` 代表目前資料夾，兩種寫法等效。

</details>

## 題目 2（基礎）· 改錯題

以下是專案結構：

```text
project/
  pages/
    gallery.html
  images/
    photo.jpg
```

`gallery.html` 中有以下兩段程式碼，各有一個路徑錯誤，請找出並全部修正。

```html
<!-- 錯誤一 -->
<img src="D:\project\images\photo.jpg" alt="相片">

<!-- 錯誤二 -->
<img src="images/photo.jpg" alt="相片">
```

完成標準：兩處錯誤全部修正，且能說明每一處為什麼有問題。

<details>
<summary>參考解答</summary>

```html
<!-- 修正後 -->
<img src="../images/photo.jpg" alt="相片">

<img src="../images/photo.jpg" alt="相片">
```

兩個錯誤說明：

1. **`src="D:\project\images\photo.jpg"`：使用了本機磁碟絕對路徑** → 這種路徑只在有相同磁碟和資料夾結構的電腦上才有效。換一台電腦、或把專案上傳到伺服器後，路徑就失效，圖片無法顯示。正式專案應把圖片放進專案資料夾，改用相對路徑。

   `gallery.html` 在 `project/pages/`，`photo.jpg` 在 `project/images/`，用 `../` 回到上一層 `project/` 再進入 `images/`，正確寫法為 `../images/photo.jpg`。

2. **`src="images/photo.jpg"`：相對路徑沒有反映 `gallery.html` 的實際位置** → `images/photo.jpg` 這個路徑是從 `gallery.html` 所在的 `pages/` 資料夾出發往下找 `images/`，也就是瀏覽器會去找 `project/pages/images/photo.jpg`，但該路徑並不存在，圖片因此顯示失敗。

   `gallery.html` 移入 `pages/` 後，必須補上 `../` 才能正確跨層引用，應改為 `../images/photo.jpg`。

</details>
