# 已知與未知進度練習

> 動手練習。概念與用法說明見 notes/290-進度條標籤/01-progress-進度條標籤.md。

## 題目 1（基礎）· 應用練習題

請建立一個上傳進度條，表示目前已完成 `75 / 100`。

完成標準：

- 使用 `<progress>`。
- `value="75"`。
- `max="100"`。
- 搭配文字說明這是上傳進度。

<details>
<summary>參考解答</summary>

```html
<span>上傳進度: </span>
<progress value="75" max="100"></progress>
```

解題說明：已知任務目前值與總目標時，可以明確寫出 `value` 與 `max`。瀏覽器會根據 `value / max` 顯示比例。
</details>

## 題目 2（基礎）· 應用練習題

請建立一個資料載入進度條，情境是「只知道正在載入，但不知道完成比例」。

完成標準：

- 使用 `<progress>`。
- 省略 `value`。
- 可保留 `max="100"`。
- 搭配文字說明資料正在載入。

<details>
<summary>參考解答</summary>

```html
<span>資料載入中: </span>
<progress max="100"></progress>
```

解題說明：如果省略 `value`，進度條會變成「不確定進度」狀態，適合用在只知道正在處理、但不知道完成比例的情境。
</details>

## 題目 3（進階）· 情境判斷實作題

請根據情境選擇 progress 寫法：表單正在送出，但目前不知道完成比例。

完成標準：

- 使用 `<progress>`。
- 不設定 `value`。
- 使用文字說明這是表單送出處理狀態。

<details>
<summary>參考解答</summary>

```html
<span>表單送出處理中: </span>
<progress max="100"></progress>
```

解題說明：表單送出處理中屬於任務進行狀態，適合使用 `<progress>`。不知道比例時，應省略 `value`，而不是隨便填一個目前值。
</details>
