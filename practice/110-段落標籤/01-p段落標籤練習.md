# p 段落標籤練習

> 動手練習。概念與用法說明見 `notes/110-段落標籤/01-p-段落標籤語意與用法.md`。

---

## 題目 1（基礎）· 應用練習題

下面有三個自然段。請使用 `<p>` 標籤把它們標記成清楚的段落。

```text
HTML 是用來描述網頁內容結構的標記語言。

CSS 可以控制文字、顏色、間距與版面外觀。

JavaScript 可以讓網頁產生互動行為。
```

完成標準：

1. 每一個自然段都使用一組 `<p>`。
2. 不使用連續 `<br>` 製造段落。
3. 不使用空 `<p>` 製造段落間距。

<details>
<summary>參考解答</summary>

```html
<p>HTML 是用來描述網頁內容結構的標記語言。</p>

<p>CSS 可以控制文字、顏色、間距與版面外觀。</p>

<p>JavaScript 可以讓網頁產生互動行為。</p>
```

解題說明：

如果文章有三個自然段，就應使用三個 `<p>`。`<p>` 的重點是表達段落語意，而不只是讓畫面看起來分開。

</details>

## 題目 2（基礎）· 改錯題

下面這段 HTML 用連續 `<br>` 製造段落。請找出問題，並改成較好的段落結構。

```html
第一段文字。<br><br>
第二段文字。<br><br>
第三段文字。
```

完成標準：

1. 說明 `<br>` 表示換行，不表示段落。
2. 使用多個 `<p>` 表達多個段落。
3. 保留原本三段文字內容。

<details>
<summary>參考解答</summary>

**錯誤說明：**

`<br>` 表示換行，不表示段落。連續使用 `<br>` 只是讓畫面看起來有空隙，卻沒有把內容標記成不同段落。

**修正後：**

```html
<p>第一段文字。</p>
<p>第二段文字。</p>
<p>第三段文字。</p>
```

解題說明：

如果內容本來就是三個段落，應該用三個 `<p>` 表達。瀏覽器會依容器寬度自動處理一般文字換行，不需要為了段落文字換行手動插入很多 `<br>`。

</details>

## 題目 3（基礎）· 改錯題

下面這段 HTML 用空的 `<p>` 製造段落間距。請修正結構，並用 CSS 控制段落間距。

```html
<p>第一段文字。</p>
<p></p>
<p>第二段文字。</p>
```

完成標準：

1. 移除空的 `<p>`。
2. 保留兩個有內容的段落。
3. 使用 CSS 控制段落下方間距。

<details>
<summary>參考解答</summary>

```html
<p>第一段文字。</p>
<p>第二段文字。</p>
```

```css
p {
  margin-bottom: 1rem;
}
```

解題說明：

空的 `<p>` 沒有實際內容，只是拿語意標籤做排版用途。段落本身應由 `<p>` 表達，段落之間的視覺間距應交給 CSS 控制。

</details>

## 題目 4（進階）· 情境判斷題

請根據情境選擇較合適的做法：`<p>`、`<br>` 或 CSS。

| 情境 | 較合適做法 |
| --- | --- |
| 一篇文章有三個自然段 | ___ |
| 同一段地址文字中需要強制換到下一行 | ___ |
| 只是想讓兩段之間多一點空白 | ___ |
| 一段文字主題轉到新的重點 | ___ |

完成標準：

1. 能分辨完整段落應使用 `<p>`。
2. 能分辨同一段內的強制換行才考慮 `<br>`。
3. 能分辨純視覺間距應用 CSS 處理。

<details>
<summary>參考解答</summary>

| 情境 | 較合適做法 | 理由 |
| --- | --- | --- |
| 一篇文章有三個自然段 | 使用三個 `<p>` | 每個自然段都是完整文字段落，應用段落語意表示。 |
| 同一段地址文字中需要強制換到下一行 | 可考慮 `<br>` | 這是在同一段文字內強制換行，不是建立新段落。 |
| 只是想讓兩段之間多一點空白 | 使用 CSS | 視覺間距是 CSS 的責任，不應新增空 `<p>`。 |
| 一段文字主題轉到新的重點 | 開始新的 `<p>` | 主題或語氣明顯切換時，應拆成新的段落。 |

解題說明：

寫 HTML 時要先考慮內容語意，再處理視覺樣式。`<p>` 表示段落，`<br>` 表示換行，CSS 負責外觀與間距。

</details>
