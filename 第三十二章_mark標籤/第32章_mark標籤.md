## 本節導讀

這篇整理 HTML5 的 `<mark>` 標籤。  
它不是單純把文字塗成黃色，而是用來標示「在當前上下文中特別值得注意、值得參考、或和目前操作有關」的文本片段。

原始內容方向有一半是對的，但還不夠穩：它強調了瀏覽器常見的黃色背景，卻沒有把 `<mark>` 的語意說清楚，容易讓人誤以為它只是視覺高亮效果。  
這次先做輕度內容重構，再補上筆記整理結構，讓這篇更適合順讀與回查。

## 30 秒複習入口

- `<mark>` 用來標示在當前上下文中特別相關的文字。
- 常見情境有搜尋命中、引用中需要特別留意的片段、閱讀時想標示的重要參考段落。
- 它通常會顯示為黃色背景，但外觀不是它的核心。
- `<strong>` 強調的是重要性；`<mark>` 強調的是相關性或被標出來的參考價值。
- 如果你只是想改背景顏色，沒有語意需求，通常用 CSS 搭配 `<span>` 就夠了。

## 速查區

### 核心概念

- `<mark>` 代表被標記或高亮的文字。
- 被標記的原因，是它在目前上下文中特別值得注意或具有相關性。
- 重點不是「看起來黃黃的」，而是「這段文字被特別標出來」。

### 常見使用場景

- 搜尋結果中的命中關鍵字
- 引用或文章中需要特別注意的片段
- 使用者目前正在關注的重點內容

### 和其他標籤的差別

- `<mark>`：標示相關或被高亮參考的文字
- `<strong>`：標示重要、嚴重或緊急的文字
- `<span>`：中性容器，本身沒有特定語意

### 常見錯誤

- 把 `<mark>` 當成單純的黃色背景樣式工具。
- 以為 `<mark>` 和 `<strong>` 可以互換。
- 為了語法高亮或純視覺效果濫用 `<mark>`。

### 一句話抓核心

- `<mark>` 是在標示「這段文字在目前脈絡下值得被特別看到」，不是在表達重要性等級。

## 正文筆記

### 1. `<mark>` 在解決什麼問題？

有些文字不是因為它本身「很重要」而被強調，而是因為在目前閱讀情境中，它特別值得被看見。  
例如搜尋結果命中的關鍵字、引用中想特別標出的片段、或文件中當前要讀者注意的某段內容，這些都很適合用 `<mark>`。

因此，`<mark>` 的核心語意是「被標記出來、供參考或因上下文而特別相關的文字」。

### 2. 為什麼不能只把它理解成黃底文字？

很多瀏覽器預設會把 `<mark>` 顯示成黃色背景，這也是原始筆記最先提到的部分。  
但這只是瀏覽器常見的預設樣式，不是這個標籤真正的意義。

更穩定的理解方式是：

- 黃底只是常見外觀
- 語意才是這個標籤存在的原因

如果你只是想讓文字有黃底，卻沒有「被特別標示、在當前上下文中相關」這層語意，就不一定要用 `<mark>`。

### 3. `<mark>` 和 `<strong>` 差在哪？

這兩個都可能讓文字看起來比較突出，但語意不同：

- `<mark>`：這段文字在當前脈絡中特別值得注意
- `<strong>`：這段文字具有重要性、嚴重性或緊急性

可以這樣記：

- 搜尋命中詞：適合用 `<mark>`
- 警告、重要提醒：比較適合用 `<strong>`

所以它們不是單純「一個黃底、一個粗體」的差別，而是語意焦點不同。

### 4. 和 `<span>` 搭配 CSS 的差別是什麼？

`<span>` 本身沒有特定語意。  
如果你只是想為了設計效果把某段文字改背景色、加邊框、換顏色，通常用 `<span>` 配合 CSS 就可以。

只有當你真的想表達「這段文字被標記、被高亮、在這個脈絡中值得被特別看到」時，才更適合用 `<mark>`。

### 5. 範例怎麼看？

```html
<h2>Example of Mark Tag</h2>
<p>
  You can easily learn designing a website with our
  <mark>Web designing tutorial</mark>.
</p>
```

這段的重點是：

- 句子中有一小段文字被特別標出來
- 這段文字在目前句子脈絡中，被當成值得注意或特別參考的片段

如果只是想替這段文字做設計效果，也可以改用 `<span>` 加 CSS；  
但如果你想保留「這段被特別標記」的語意，`<mark>` 更合適。

### 6. 什麼情況最常用 `<mark>`？

- 搜尋結果命中的文字
- 引文中自己特別標出的片段
- 文件閱讀時需要強調「目前相關」而不是「永久重要」的文字

這些都是 `<mark>` 比較自然的使用場景。

## 常見問法

### `<mark>` 是不是只是讓文字變黃？

不是。  
黃色背景只是常見預設外觀，真正的重點是標示這段文字在目前上下文中特別相關。

### `<mark>` 和 `<strong>` 可以互換嗎？

不行。  
`<mark>` 強調相關性或被標示出來，`<strong>` 強調重要性或嚴重性。

### 如果我只想改背景色，該用什麼？

通常用 `<span>` 配合 CSS 即可。  
只有在你真的需要「被高亮標示」這層語意時，再用 `<mark>`。

### 搜尋結果命中詞適合用 `<mark>` 嗎？

很適合。  
這正是 `<mark>` 的典型使用場景之一。
