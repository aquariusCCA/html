> - 建議回查情境：想知道 `<mark>` 是做什麼的、搜尋命中詞要不要用 `<mark>`、想分清 `<mark>` / `<strong>` / `<em>` / `<span>`、想判斷「語意高亮」和「純 CSS 黃底」的差別時

## 本節導讀

`<mark>` 是 HTML5 的語意化標籤，用來標示「在目前上下文中特別相關、值得注意、或需要被讀者看到」的文字片段。

它常被瀏覽器預設顯示成黃色背景，但**黃色不是它的重點**。  
真正重要的是：這段文字被標出來，是因為它和目前的閱讀情境、搜尋結果、引用內容或操作脈絡有關。

簡單說：

- `<mark>` 不是「讓文字變黃」的工具。
- `<mark>` 是「把目前脈絡下值得注意的文字標示出來」的語意標籤。

## 30 秒複習入口

- `<mark>` 用來標示在「目前上下文」中特別相關或值得注意的文字。
- 它很適合用在搜尋結果命中詞、引用中被特別標出的片段、目前正在關注的內容。
- 瀏覽器常把 `<mark>` 顯示成黃色背景，但外觀不是它的核心。
- `<strong>` 表示內容本身重要、嚴重或緊急。
- `<em>` 表示語氣上的強調。
- `<span>` 沒有語意，通常用來搭配 CSS 或 JavaScript。
- 如果只是想做黃底視覺效果，沒有語意需求，用 `<span>` + CSS 通常更合適。

## 速查區

### 核心概念

`<mark>` 代表被標記或高亮的文字。  
被標記的原因不是「它永遠很重要」，而是「它在目前脈絡中特別相關」。

可以這樣理解：

> `<mark>` 標示的是「目前值得被看到的內容」。

### 常見使用場景

| 使用場景 | 是否適合用 `<mark>` | 原因 |
|---|---:|---|
| 搜尋結果中的命中關鍵字 | 適合 | 關鍵字和使用者目前搜尋行為有關 |
| 引文中被特別標出的片段 | 適合 | 該片段在目前引用脈絡中值得注意 |
| 文章中目前正在討論的詞 | 適合 | 該詞和目前段落重點有關 |
| 警告、危險、重要提醒 | 不一定適合 | 通常應該優先考慮 `<strong>` |
| 單純想讓文字變黃 | 不適合 | 這是樣式需求，不是語意需求 |
| 程式碼語法高亮 | 通常不適合 | 通常應使用專門的程式碼結構與 CSS class |

### 和其他標籤的差別

| 標籤 | 核心語意 | 適合情境 |
|---|---|---|
| `<mark>` | 目前上下文中特別相關、被標出來供參考 | 搜尋命中、引用高亮、目前關注片段 |
| `<strong>` | 內容本身重要、嚴重、緊急 | 重要提醒、警告、不可忽略的資訊 |
| `<em>` | 語氣上的強調 | 改變句子語氣、強調某個詞 |
| `<span>` | 沒有語意的行內容器 | 純樣式控制、JS 操作、包住文字片段 |

### 一句話抓核心

> `<mark>` 標示的是「這段文字在目前脈絡下值得被特別看到」，不是在表達它本身的重要性等級。

## 正文筆記

### 1. `<mark>` 在解決什麼問題？

在閱讀網頁時，有些文字會因為「目前情境」而特別值得注意。

例如：

- 使用者搜尋 `HTML`，搜尋結果中的 `HTML` 命中詞需要被標出來。
- 文章引用一段文字，作者想標出其中和討論主題最相關的片段。
- 文件正在說明某個概念，想把目前正在討論的詞標出來。

這些文字不一定是「最重要」或「最嚴重」的內容。  
它們只是因為和目前脈絡有關，所以需要被標記出來。

這就是 `<mark>` 的用途。

### 2. 為什麼不能只把 `<mark>` 理解成黃底文字？

很多瀏覽器預設會把 `<mark>` 顯示成黃色背景，所以初學時很容易把它記成「黃底標籤」。

但這樣記不夠準確。

更正確的理解是：

- 黃底只是瀏覽器預設樣式。
- `<mark>` 的核心是語意。
- 它代表這段文字在目前上下文中被標記、被高亮、值得讀者注意。

如果你只是想讓文字背景變黃，卻沒有「目前脈絡中特別相關」這層意思，就不一定要使用 `<mark>`。

### 3. 什麼是「目前上下文」？

「目前上下文」可以理解成：

> 使用者現在正在看的內容、正在做的操作、正在搜尋的關鍵字，或文章目前正在討論的主題。

例如使用者搜尋 `mark`：

```html
<p>
  搜尋結果：HTML 的 <mark>mark</mark> 標籤可以用來標示搜尋命中的文字。
</p>
```

這裡的 `<mark>mark</mark>` 很合理，因為它是使用者目前搜尋的詞。  
它被標記不是因為 `mark` 這個字永遠重要，而是因為它和「目前搜尋結果」這個上下文有關。

### 4. `<mark>` 和 `<strong>` 差在哪？

`<mark>` 和 `<strong>` 都可能讓文字看起來比較醒目，但語意不同。

```html
<p>
  搜尋結果中包含 <mark>信用卡</mark> 相關資料。
</p>

<p>
  <strong>請勿將密碼告訴他人。</strong>
</p>
```

第一段使用 `<mark>`，因為 `信用卡` 是目前搜尋或比對的關鍵字。  
第二段使用 `<strong>`，因為整句話本身具有重要性與警告性。

可以這樣記：

- `<mark>`：這段文字和目前脈絡有關，所以被標出來。
- `<strong>`：這段文字本身很重要，需要被重視。

所以它們不是「黃底」和「粗體」的差別，而是語意焦點不同。

### 5. `<mark>` 和 `<em>` 差在哪？

`<em>` 用來表達語氣上的強調，通常會影響一句話的語氣。

例如：

```html
<p>我 <em>真的</em> 不建議這樣做。</p>
```

這裡的 `真的` 是語氣強調，所以適合用 `<em>`。

但如果是搜尋命中：

```html
<p>搜尋結果：我真的不建議使用 <mark>innerHTML</mark> 處理不可信資料。</p>
```

這裡的 `innerHTML` 是因為搜尋或目前主題而被標示，所以適合用 `<mark>`。

簡單區分：

- `<em>`：強調語氣。
- `<mark>`：標示目前脈絡相關的文字。

### 6. `<mark>` 和 `<span>` 搭配 CSS 差在哪？

`<span>` 本身沒有特定語意。  
它只是用來包住一小段行內文字，方便加 CSS 或用 JavaScript 操作。

例如：

```html
<p>
  這是 <span class="highlight">純視覺效果</span>。
</p>
```

如果只是為了設計效果，例如讓某段文字有背景色、圓角、邊框，通常用 `<span>` 搭配 CSS 就可以。

但是，如果你想表達「這段文字在目前脈絡中被特別標記」，就比較適合用 `<mark>`。

### 7. 典型範例：搜尋結果命中詞

這是 `<mark>` 最典型的使用場景之一：

```html
<p>
  你搜尋了「Java」。
</p>

<p>
  搜尋結果：<mark>Java</mark> 是一種常見的後端開發語言。
</p>
```

這裡的 `Java` 因為符合使用者搜尋關鍵字，所以被標示出來。  
這正是 `<mark>` 的語意：目前上下文中值得被注意的文字。

### 8. 典型範例：引用中標示關鍵片段

```html
<blockquote>
  HTML 的語意化標籤可以幫助瀏覽器、搜尋引擎與輔助工具更好地理解內容。
</blockquote>

<p>
  在這段引用中，<mark>語意化標籤</mark> 是目前討論的重點。
</p>
```

這裡的 `<mark>` 表示：在目前討論脈絡中，`語意化標籤` 這段文字特別值得注意。

### 9. 如何客製化 `<mark>` 的外觀？

`<mark>` 有預設樣式，但你可以用 CSS 修改。

```css
mark {
  background-color: #fff3a3;
  padding: 0 0.15em;
  border-radius: 0.2em;
}
```

也可以針對不同用途加 class：

```html
<p>
  搜尋結果：<mark class="search-hit">HTML</mark> 是網頁內容的基礎。
</p>
```

```css
.search-hit {
  background-color: #ffe58f;
}
```

要注意的是：  
即使你改掉 `<mark>` 的外觀，它仍然保留 `<mark>` 的語意。

### 10. 可讀性與無障礙提醒

使用 `<mark>` 時，不要只依賴顏色來傳達重要資訊。

原因是：

- 有些使用者可能不容易分辨顏色差異。
- 有些輔助工具不一定會明確朗讀「這是被標記的文字」。
- 如果標示的內容很關鍵，最好在文字內容本身也說清楚。

例如，不要只寫：

```html
<p>請注意 <mark>這裡</mark>。</p>
```

比較好的寫法是：

```html
<p>
  請注意：<mark>申請截止日是 5 月 31 日</mark>。
</p>
```

這樣即使使用者沒有看到高亮效果，也能從文字本身理解重點。

## 使用判斷流程

可以用這個問題判斷：

> 這段文字是因為目前的搜尋、閱讀、引用、操作或討論脈絡，才需要被特別標出來嗎？

如果答案是「是」，可以考慮用 `<mark>`。  
如果答案是「不是」，就要再判斷：

- 是不是內容本身很重要？如果是，考慮 `<strong>`。
- 是不是語氣上要強調？如果是，考慮 `<em>`。
- 是不是只是想改樣式？如果是，使用 `<span>` + CSS。

## 常見錯誤

### 錯誤 1：把 `<mark>` 當成單純的黃色背景工具

錯誤理解：

```html
<p>這是 <mark>黃色文字效果</mark>。</p>
```

如果只是想要黃色背景，沒有語意需求，這樣不一定適合。

較合理的做法：

```html
<p>這是 <span class="highlight">黃色文字效果</span>。</p>
```

### 錯誤 2：把 `<mark>` 和 `<strong>` 當成可以互換

錯誤理解：

```html
<p><mark>請勿洩漏密碼。</mark></p>
```

如果這句話是重要提醒，通常更適合：

```html
<p><strong>請勿洩漏密碼。</strong></p>
```

### 錯誤 3：大量包住整段文字

不建議：

```html
<p>
  <mark>這一整段文字全部都被標示起來，導致讀者很難看出真正需要注意的片段。</mark>
</p>
```

`<mark>` 比較適合標示局部片段。  
如果整段都很重要，應該從段落結構、標題、提示框或 `<strong>` 等方式重新設計。

### 錯誤 4：拿 `<mark>` 做程式碼語法高亮

不建議把 `<mark>` 當成語法高亮工具。

例如：

```html
<code>
  const <mark>name</mark> = 'Tom';
</code>
```

如果是程式碼語法高亮，通常應該使用 `<code>`、`<pre>` 搭配 CSS class 或語法高亮工具，而不是靠 `<mark>` 表示語法分類。

## 常見問法

### `<mark>` 是不是只是讓文字變黃？

不是。  
黃色背景只是瀏覽器常見的預設外觀。  
真正的重點是：這段文字在目前上下文中特別相關，所以被標記出來。

### `<mark>` 和 `<strong>` 可以互換嗎？

不建議互換。  
`<mark>` 強調「目前脈絡相關」。  
`<strong>` 強調「內容本身重要、嚴重或緊急」。

### `<mark>` 和 `<em>` 有什麼不同？

`<em>` 是語氣強調。  
`<mark>` 是上下文相關的標示。

例如：

```html
<p>我 <em>真的</em> 不建議這樣做。</p>
<p>搜尋結果包含 <mark>JavaScript</mark>。</p>
```

### 如果我只想改背景色，該用什麼？

通常用 `<span>` 搭配 CSS 即可。

```html
<p>
  這是 <span class="highlight">純視覺高亮</span>。
</p>
```

```css
.highlight {
  background-color: yellow;
}
```

### 搜尋結果命中詞適合用 `<mark>` 嗎？

很適合。  
搜尋命中詞是 `<mark>` 最典型的使用情境之一。

### 可以改 `<mark>` 的顏色嗎？

可以。  
`<mark>` 的外觀可以用 CSS 修改。  
修改外觀不會改變它的語意。

## 自測問題

1. `<mark>` 的核心語意是什麼？
2. 為什麼 `<mark>` 不能只理解成黃色背景？
3. `<mark>` 和 `<strong>` 的最大差別是什麼？
4. `<mark>` 和 `<em>` 的差別是什麼？
5. 什麼情況下用 `<span>` 加 CSS 會比 `<mark>` 更合適？
6. 搜尋結果命中詞為什麼適合使用 `<mark>`？
7. 為什麼不建議用 `<mark>` 包住一整大段文字？

## 自測答案

1. `<mark>` 用來標示在目前上下文中特別相關、值得注意或被標記出來的文字。
2. 因為黃色背景只是瀏覽器預設樣式，`<mark>` 的核心是語意，不是外觀。
3. `<mark>` 強調目前脈絡中的相關性；`<strong>` 強調內容本身的重要性、嚴重性或緊急性。
4. `<em>` 是語氣強調；`<mark>` 是上下文相關的文字標示。
5. 如果只是想改背景色、做設計效果，而沒有語意需求，使用 `<span>` + CSS 更合適。
6. 因為搜尋命中詞和使用者目前的搜尋行為有關，屬於目前上下文中特別相關的文字。
7. 因為 `<mark>` 適合標示局部片段；如果整段都被標示，讀者反而不容易看出真正需要注意的重點。

## 最終記憶版

`<mark>` 是用來標示「目前上下文中特別相關的文字」。  
它常見外觀是黃底，但核心不是黃色，而是語意。

判斷時可以記這句：

> 搜尋命中、引用高亮、目前關注片段，用 `<mark>`；  
> 重要警告用 `<strong>`；  
> 語氣強調用 `<em>`；  
> 純樣式效果用 `<span>` + CSS。
