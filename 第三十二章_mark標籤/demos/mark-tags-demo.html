第三十二章：mark 標籤 Demo

這個示例展示 <mark> 如何標示目前上下文中特別相關的文字，並對照 strong、em、span 與純 CSS 高亮。

一句話核心

<mark> 的重點不是黃底，而是「這段文字在目前脈絡下值得被特別看到」。

搜尋命中

使用者目前搜尋的字詞，適合用 <mark> 標示。

引用焦點

引用內容中目前討論的片段，可以用 <mark>。

不是純樣式

如果只是想變黃，通常用 <span> + CSS。

3. mark、strong、em、span 怎麼分

標籤 示例 語意焦點
mark 搜尋結果包含 innerHTML。 目前脈絡中特別相關。
strong 請勿將密碼告訴他人。 內容本身重要、嚴重或緊急。
em 我 真的 不建議這樣做。 語氣或語意重音。
span 這是 純視覺高亮。 無語意，通常只為 CSS 或 JS 包住文字。

5. 適合使用 mark

  • 搜尋結果中的命中關鍵字
  • 引用中目前討論的片段
  • 文件中目前正在關注的詞
  • 操作或比對結果中需要被讀者看到的片段

6. 不適合硬用 mark

  • 只是想做黃色背景
  • 重要警告或不可忽略的提醒
  • 語氣上的強調
  • 整段文字全部高亮
  • 程式碼語法高亮

7. 常見錯誤

當成黃底工具

沒有上下文相關語意時，通常用 span + CSS 更合適。

和 strong 混用

重要警告應優先考慮 strong，不是因為想醒目就用 mark。

包住整大段

mark 適合局部片段；整段都標記會讓真正重點變模糊。

8. 判斷流程

1. 和目前脈絡相關？

是搜尋、引用、比對、目前討論的焦點，可以考慮 mark。

2. 內容本身很重要？

如果是警告、風險、緊急提醒，考慮 strong。

3. 只是改樣式？

沒有語意需求時，用 span 搭配 CSS。
