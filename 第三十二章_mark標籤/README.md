# 第三十二章｜`mark` 標籤

> 章節定位：理解 `<mark>` 如何標示在當前上下文中特別相關的文字  
> 建議閱讀方式：先掌握語意，再學會判斷什麼時候該用 `<mark>`，什麼時候只需要 CSS 或其他語意標籤

## 本章在學什麼

這一章整理 HTML5 的 `<mark>` 標籤。

重點不是記住它預設會顯示成黃色背景，而是理解：

- `<mark>` 用來標示「在目前上下文中特別相關、值得被注意或值得被參考」的文字。
- 它不是單純的視覺高亮工具。
- 它和 `<strong>`、`<em>`、`<span>` 的語意不同。
- 使用前應該先判斷自己要表達的是「目前相關」、「內容重要」、「語氣強調」，還是單純的樣式效果。

## 本章與整體地圖的關係

- 前置知識：前面已學過多種文字與語意標籤，這一章補的是「高亮標示但不等於重要性」這個語意。
- 這章的核心任務：理解 `<mark>` 的語意，並能判斷它和 `<strong>`、`<em>`、`<span>`、純 CSS 高亮的差別。
- 讀完後通常接：之後在做搜尋命中標示、文件高亮、引用重點標註或語意化標記時，都可以回查這一章。

## 易混淆主題

- `<mark>` vs `<strong>`：前者強調目前脈絡下的相關性，後者強調內容本身的重要性、嚴重性或緊急性。
- `<mark>` vs `<em>`：前者標示被特別標出的文字，後者改變語氣或語意重音。
- `<mark>` vs `<span>`：前者有語意，後者是沒有語意的中性容器。
- `<mark>` vs 黃底 CSS：前者是語意高亮，後者只是樣式效果。
- 被高亮 vs 很重要：兩者不是同一件事。
- 搜尋命中高亮 vs 程式碼語法高亮：搜尋命中詞適合用 `<mark>`；程式碼語法高亮通常不適合用 `<mark>`。

## 本章記憶句

`<mark>` 標示的是「這段文字在目前脈絡下特別相關」，不是單純讓文字變黃，也不是用來表示內容本身很重要。
