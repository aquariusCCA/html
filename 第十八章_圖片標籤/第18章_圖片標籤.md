# 圖片標籤

> 所屬章節：第十八章｜圖片標籤  
> 關鍵字：圖片標籤、`img`、`src`、`alt`、`title`、`width`、`height`、`jpg`、`png`、`gif`、`webp`、`bmp`、`data URL`、`base64`  
> 建議回查情境：想知道 HTML 圖片怎麼插入、想分清 `src` / `alt` / `title` 的差別、想知道圖片尺寸怎麼設、想快速選擇圖片格式

## 本節導讀

這篇整理 HTML 的圖片標籤 `<img>`。  
重點不只是把圖片放上頁面，而是一起搞清楚圖片資源從哪裡來、替代文字要怎麼想、顯示尺寸怎麼設，以及常見圖片格式該怎麼選。

原始內容能看出教學重點，但順序比較像課堂講稿，容易把 `base64` 直接當成一般圖片格式，也容易把 `alt` 和 `title` 都學成「提示文字」。  
這裡改成較穩定的學習順序：先認識 `<img>` 與屬性，再看尺寸，最後比較格式與 `data URL`。

## 你會在這篇學到什麼

- `<img>` 是什麼、基本語法怎麼寫
- `src`、`alt`、`title` 各自在處理什麼
- `width`、`height` 對顯示尺寸有什麼影響
- 常見圖片格式的差別
- 為什麼 `base64` / `data URL` 不應直接和 `jpg`、`png` 並列成同一類

## 30 秒複習入口

- `<img>` 是用來在網頁中嵌入圖片的空元素。
- `src` 指向圖片資源；`alt` 是替代文字；`title` 通常只是提示文字。
- 只設定 `width` 或 `height` 其中一個時，圖片通常會依原比例縮放；兩個都設錯時可能變形。
- `jpg` 適合照片，`png` 適合透明背景或需要較清楚邊緣的圖，`gif` 適合簡單動畫，`webp` 是現代網頁常見格式。
- `base64` 更接近資料表示或嵌入方式，不是一般意義上的圖片檔案格式。

## 速查區

### 核心概念

- `<img>` 的核心任務是把圖片資源放進頁面。
- 真正要一起學的是：資源路徑、替代文字、顯示尺寸、格式選擇。

### 關鍵規則 / 判準

- `src` 幾乎是 `<img>` 能正常顯示的前提。
- `alt` 不是滑鼠提示，而是圖片無法顯示或輔助技術需要時的替代文字。
- `title` 可以提供額外提示，但不應拿來取代 `alt`。
- `width` 和 `height` 改的是顯示尺寸，不是原始檔案本身。
- 選圖片格式時，要看照片、透明背景、動畫、檔案大小等需求。

### 常見使用場景

- 文章配圖、產品圖、圖示、Logo、教學截圖
- 需要透明背景的圖片
- 需要簡單動圖的情境
- 需要把極小圖片直接內嵌在頁面或樣式中的情境

### 常見混淆點

- `alt` 和 `title` 不是同一件事。
- `base64` / `data URL` 不是傳統意義上的圖片格式。
- 在頁面上把圖片縮小，不代表下載的檔案就變小。
- `webp` 不該再簡化理解成「優點很多，但相容性很差」；更準確的理解是：它已是現代網頁常見格式，但仍要按實際支援環境決定是否提供 fallback。

### 一句話抓核心

- 圖片標籤的關鍵不只是「把圖放上去」，而是同時處理資源、替代文字、尺寸與格式選擇。

## 正文筆記

### 這篇在解決什麼問題？

- 在 HTML 裡，如果你要把圖片放進頁面，最直接的方式就是用 `<img>`。
- 但只會寫 `<img src="...">` 還不夠；你還需要分清楚屬性各自的用途，以及不同圖片格式適合什麼場景。

## 1. `<img>` 標籤在做什麼？

- `<img>` 是 image 的縮寫，用來在網頁中嵌入圖片。
- 它是空元素，不包內容，主要靠屬性來描述圖片來源與顯示方式。

```html
<img
  src="圖片地址"
  alt="替代文字"
  title="提示文字"
  width="寬度"
  height="高度"
/>
```

![圖像標籤屬性介紹.png](./images/image_1776138504618_8385ec9e.png)

### 這些屬性各自在做什麼？

- `src`：指定圖片資源的位置。
- `alt`：提供替代文字，當圖片無法顯示或需要文字替代時會派上用場。
- `title`：提供額外提示文字，常見效果是滑鼠停留時顯示提示。
- `width` / `height`：控制圖片在頁面上的顯示尺寸。

## 2. `src`、`alt`、`title` 怎麼分？

### `src`

- `src` 是 source 的縮寫，用來告訴瀏覽器圖片資源在哪裡。
- 你可以放相對路徑、絕對路徑，或 `data URL`。

### `alt`

- `alt` 是 alternative text 的縮寫，也就是替代文字。
- 它的核心不是「滑鼠移上去的提示」，而是當圖片本身無法正常呈現，或需要用文字說明圖片內容時，提供可替代的資訊。

### `title`

- `title` 通常是補充提示文字。
- 初學時可以把它理解成「額外提示」，但不要把它當成 `alt` 的替代品。

## 3. `width` 和 `height` 要怎麼理解？

- 如果只設定其中一個維度，圖片通常會依原始比例縮放，因此較不容易變形。
- 如果同時設定 `width` 和 `height`，但比例和原圖不一致，圖片就可能被拉伸或壓扁。
- 這兩個屬性改的是顯示尺寸，不是原始圖片檔案的解析度或檔案大小。

## 4. 常見圖片格式怎麼選？

### `jpg` / `jpeg`

- 有損壓縮，通常適合照片類圖片。
- 優點是色彩表現好、檔案通常較小。
- 不支援透明背景，也不適合動畫。

### `png`

- 常見的無損格式，適合需要透明背景或較清楚邊緣的圖片。
- 常見於 Logo、圖示、介面配圖、截圖等。
- 通常比 `jpg` 更適合需要透明度或較穩定畫質的情境。

### `gif`

- 只支援有限色彩，畫質通常不如 `jpg` 或 `png`。
- 優點是可用於簡單動畫，也能處理基本透明效果。
- 常見於表情圖、簡單動圖。

### `webp`

- 現代網頁很常見的圖片格式。
- 可支援有損、無損、透明與動畫等能力，很多情況下檔案大小表現也不錯。
- 實務上常作為網頁圖片的優先選項之一，但仍應依目標環境決定是否提供其他格式作為備援。

### `bmp`

- 傳統點陣圖格式，通常檔案較大。
- 優點是資料保留較直接，但在網頁實務中並不常見。
- 若是一般網站開發，通常不會優先選它。

## 5. `base64` 為什麼不應和前面並列成同一類？

- `base64` 更接近「把圖片資料編成文字後，再嵌入頁面」的方式。
- 在 HTML 裡，它通常會以 `data URL` 的形式出現在 `src` 裡。

```html
<img src="data:image/png;base64,..." alt="內嵌圖片" />
```

- 這種方式可以讓圖片不依賴外部檔案路徑，但它不是 `jpg`、`png` 那種傳統圖片格式分類。
- 初學時比較穩定的理解是：`base64` / `data URL` 是圖片的表示與載入方式，不是一般圖片格式的同類比較項。
- 實務上較常用在非常小的資源或特殊嵌入情境。

## 6. 圖片標籤怎麼實際使用？

```html
<body>
  <h4>基本圖片顯示</h4>
  <img src="./images/image_1776138504618_7ad86ea6.jpg" />
  <hr>

  <h4>alt：圖片顯示不出來時的替代文字</h4>
  <img src="./images/missing-image.jpg" alt="課程示意圖片" />
  <hr>

  <h4>title：滑鼠停留時的提示文字</h4>
  <img src="./images/image_1776138504618_7ad86ea6.jpg" title="課程示意圖片" />
  <hr>

  <h4>width / height：控制顯示尺寸</h4>
  <img src="./images/image_1776138504618_7ad86ea6.jpg" width="200" height="200" />
</body>
```

### 這段例子在展示什麼？

- 第一張圖展示最基本的圖片插入方式。
- 第二張圖展示 `alt` 的用途。
- 第三張圖展示 `title` 的提示效果。
- 第四張圖展示尺寸控制；若原圖不是正方形，這種同時指定 `200 x 200` 的寫法就可能讓圖片變形。

## 常見問法

### `alt` 和 `title` 最大差別是什麼？

- `alt` 是替代文字，重點在「圖片無法直接看到時，還有什麼資訊能被理解」。
- `title` 是補充提示，重點不一樣。

### 只寫 `width` 不寫 `height` 可以嗎？

- 可以。
- 很多情況下，這樣反而更不容易把圖片比例拉壞。

### 照片、Logo、動圖通常各自優先考慮哪種格式？

- 照片常先考慮 `jpg` 或 `webp`。
- 需要透明背景的 Logo 或圖示常先考慮 `png` 或 `webp`。
- 簡單動圖可考慮 `gif` 或 `webp`。

### `src` 可以直接放 `base64` 嗎？

- 可以。
- 但要把它理解成 `data URL` 形式的內嵌資料，不是一般圖片格式本身。
