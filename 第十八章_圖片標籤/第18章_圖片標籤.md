# 圖片標籤

> 所屬章節：第十八章｜圖片標籤
> 關鍵字：圖片標籤、`img`、`src`、`alt`、`title`、`width`、`height`、`jpg`、`png`、`gif`、`webp`、`bmp`、`data URL`、`base64`
> 建議回查情境：想知道 HTML 圖片怎麼插入、想分清 `src` / `alt` / `title` 的差別、想知道圖片尺寸怎麼設、想快速判斷圖片格式怎麼選

## 本節導讀

在 HTML 中，插入圖片最常用的標籤是 `<img>`。
但學圖片標籤，不能只停留在「把圖片放上頁面」這件事。

更完整的理解應該包含四個面向：

* **圖片資源從哪裡來**
* **圖片看不到時，要用什麼文字替代**
* **圖片在頁面上要顯示多大**
* **不同圖片格式適合什麼情境**

初學時很容易把幾件事混在一起，例如：

* 把 `alt` 和 `title` 都記成提示文字
* 把 `base64` 當成和 `jpg`、`png` 同一類的圖片格式
* 以為把圖片在頁面上縮小，檔案就會跟著變小

這篇的重點，就是把這些常見混淆拆開，建立一個更穩定的理解方式。

* `src`、`alt`、`title` 各自在解決什麼問題
* `width`、`height` 怎麼影響圖片顯示
* 常見圖片格式的差別與選擇方式
* 為什麼 `base64` / `data URL` 不應直接和 `jpg`、`png` 並列成同一類

## 30 秒複習入口

* `<img>` 是用來在網頁中嵌入圖片的空元素。
* `src` 指向圖片資源位置。
* `alt` 是替代文字，不是滑鼠提示。
* `title` 通常是補充提示文字，但不能取代 `alt`。
* `width` / `height` 改的是顯示尺寸，不是原始檔案大小。
* `jpg` 常用於照片，`png` 常用於透明背景或邊緣清楚的圖片，`gif` 常用於簡單動畫，`webp` 是現代網頁常見格式。
* `base64` / `data URL` 更接近一種資料嵌入方式，不是一般意義上的圖片格式。

---

## 速查區

### 核心概念

學 `<img>` 不只是學一個標籤，而是在同時處理四件事：

1. **資源來源**：圖片放在哪裡，由誰提供
2. **替代資訊**：圖片無法直接被看到時，讀者還能知道什麼
3. **顯示尺寸**：圖片在頁面上看起來多大
4. **格式選擇**：哪種圖片格式比較適合目前需求

---

### 關鍵規則 / 判準

* `src` 幾乎是圖片能否顯示的前提。
* `alt` 的核心是「替代資訊」，不是「提示效果」。
* `title` 只能當補充資訊，不應拿來代替 `alt`。
* `width` 和 `height` 控制的是顯示大小，不會直接改變原始檔案內容。
* 只設定一個維度時，圖片通常較不容易變形。
* 選格式時，要根據照片、透明背景、動畫、檔案大小與相容需求來判斷。

---

### 圖片格式快速對照

| 格式           | 常見用途              | 特色                     |
| ------------ | ----------------- | ---------------------- |
| `jpg / jpeg` | 照片、寫實圖片           | 有損壓縮，檔案通常較小，不支援透明      |
| `png`        | Logo、圖示、截圖、透明背景圖片 | 畫質穩定，可支援透明             |
| `gif`        | 簡單動圖、表情圖          | 可做簡單動畫，但色彩有限           |
| `webp`       | 網頁常見圖片            | 可支援有損、無損、透明、動畫，整體表現常不錯 |
| `bmp`        | 傳統點陣圖             | 檔案通常較大，網頁實務較少優先使用      |

---

### 常見混淆點

* `alt` 和 `title` 不是同一件事。
* `base64` / `data URL` 不是傳統意義上的圖片格式。
* 把圖片顯示得比較小，不代表實際下載檔案就比較小。
* `webp` 不是「很新但不能用」的格式，而是現代網頁中很常見的格式之一，只是仍要看目標環境是否需要 fallback。

---

### 一句話抓核心

**圖片標籤的重點不只是把圖片放上去，而是同時處理圖片來源、替代資訊、顯示尺寸與格式選擇。**

# 正文筆記

## 1. `<img>` 標籤在做什麼？

`<img>` 是 **image** 的縮寫，用來在網頁中嵌入圖片。
它本身不包內容，屬於**空元素**，主要依靠屬性來描述圖片的來源與呈現方式。

基本寫法如下：

```html
<img
  src="圖片地址"
  alt="替代文字"
  title="補充提示"
  width="寬度"
  height="高度"
/>
```

這裡最重要的不是把屬性背起來，而是理解每個屬性各自負責什麼。

---

## 2. `src`、`alt`、`title` 各自在做什麼？

### `src`：圖片資源位置

`src` 是 **source** 的縮寫，用來告訴瀏覽器圖片資源在哪裡。

它可以指向：

* 相對路徑
* 絕對路徑
* `data URL`

例如：

```html
<img src="./images/cat.jpg" alt="貓咪照片" />
<img src="https://example.com/banner.png" alt="網站橫幅" />
```

沒有 `src`，瀏覽器就不知道該去哪裡取得圖片，因此圖片通常無法正常顯示。

---

### `alt`：替代文字

`alt` 是 **alternative text** 的縮寫，也就是替代文字。
它的核心不是提示，而是：**當圖片無法直接被看到時，還能留下什麼資訊給使用者。**

這裡要掌握一個重點：

**`alt` 寫的不是圖片外觀的所有細節，而是「看不到這張圖時，仍然應該知道的內容」。**

例如：

```html
<img src="./images/logo.png" alt="OpenAI 官方 Logo" />
<img src="./images/submit-button.png" alt="送出" />
```

如果圖片只是裝飾用途，沒有實質資訊，才可能使用空的 `alt`：

```html
<img src="./images/decorative-line.png" alt="" />
```

---

### `title`：補充提示

`title` 通常用來提供額外的補充資訊。
在很多瀏覽器中，滑鼠停留時可能會出現提示文字，但這不是它最本質的學習重點。

更穩定的理解方式是：

* `alt`：圖片本身的替代資訊
* `title`：額外補充說明

所以：

* `alt` 不能省略思考，只靠 `title` 代替
* `title` 也不應被理解成圖片說明的主要機制

例如：

```html
<img
  src="./images/course-cover.jpg"
  alt="HTML 入門課程封面"
  title="點擊可查看完整課程內容"
/>
```

---

## 3. `width` 和 `height` 要怎麼理解？

`width` 和 `height` 用來控制圖片在頁面上的**顯示尺寸**。

要特別分清楚：

* 它們改的是畫面上的大小
* 不是原始圖片檔案本身的解析度
* 也不代表下載檔案會因此變小

---

### 只設定一個維度

如果只設定 `width` 或只設定 `height`，圖片通常會依照原始比例縮放，因此比較不容易變形。

```html
<img src="./images/photo.jpg" alt="風景照片" width="300" />
```

這種寫法在初學時通常比較安全。

---

### 同時設定兩個維度

如果同時設定 `width` 和 `height`，但比例與原圖不一致，圖片就可能被拉伸或壓扁。

```html
<img src="./images/photo.jpg" alt="風景照片" width="200" height="200" />
```

若原圖本來不是正方形，這種寫法就可能造成變形。

---

### 錯誤示範 vs 較佳示範

**較不理想：**

```html
<img src="./images/banner.jpg" alt="活動橫幅" width="200" height="200" />
```

問題：如果原圖是長方形，可能會被硬壓成正方形。

**較佳：**

```html
<img src="./images/banner.jpg" alt="活動橫幅" width="200" />
```

這樣通常會依原始比例縮放，比較不容易失真。

---

## 4. 常見圖片格式怎麼選？

學圖片格式時，不要只背名稱，而要知道它們分別適合什麼情境。

---

### `jpg / jpeg`

* 常用於照片、寫實圖片
* 採用有損壓縮，通常能把檔案壓得比較小
* 色彩表現通常不錯
* 不支援透明背景
* 不適合動畫

**適合場景：**
照片、人物照、風景照、商品實拍圖

---

### `png`

* 常見的無損格式
* 支援透明背景
* 常用於邊緣清楚、需要保留細節的圖片
* 常見於 Logo、圖示、介面配圖、截圖

**適合場景：**
Logo、透明背景圖片、UI 圖示、教學截圖

---

### `gif`

* 可做簡單動畫
* 色彩數量有限
* 畫質通常不如 `jpg` 或 `png`
* 也能處理基本透明效果

**適合場景：**
簡單動圖、表情圖、短小循環動畫

---

### `webp`

* 現代網頁中很常見
* 可支援有損、無損、透明、動畫
* 在很多情況下，檔案大小與效果表現都不錯
* 實務上常作為網頁圖片的優先選項之一

但仍要注意：
是否需要提供其他格式作為備援，取決於你的目標環境與相容需求。

**適合場景：**
一般網站圖片、需要兼顧體積與畫質的情境、現代前端專案

---

### `bmp`

* 傳統點陣圖格式
* 通常檔案較大
* 在網站開發中較少作為優先選項

**適合場景：**
較少是一般網頁開發的首選，多作為理解格式時的補充知識

---

## 5. `base64` / `data URL` 為什麼不應和圖片格式並列？

這是最容易混淆的地方之一。

`jpg`、`png`、`gif`、`webp` 這些，討論的是**圖片檔案格式**。
但 `base64` 更接近一種**資料表示方式**，而 `data URL` 更接近一種**把資料直接寫進 `src` 的嵌入方式**。

例如：

```html
<img src="data:image/png;base64,..." alt="內嵌圖片" />
```

這代表：

* 圖片資料沒有從外部檔案載入
* 而是直接以文字形式嵌入在 HTML 裡

所以更精準的說法是：

* `jpg` / `png` / `gif` / `webp`：圖片格式
* `base64` / `data URL`：資料表示或嵌入方式

實務上，這種方式較常用在：

* 很小的圖示
* 特殊內嵌需求
* 不想額外請求外部小資源的情境

但它不是一般初學者在選圖片格式時的同類比較項。

---

## 6. 圖片標籤怎麼實際使用？

### 基本範例

```html
<body>
  <h4>基本圖片顯示</h4>
  <img src="./images/example.jpg" alt="課程示意圖片" />
  <hr>

  <h4>alt：圖片顯示不出來時的替代資訊</h4>
  <img src="./images/missing-image.jpg" alt="課程示意圖片" />
  <hr>

  <h4>title：補充提示文字</h4>
  <img src="./images/example.jpg" alt="課程示意圖片" title="點擊可查看詳細說明" />
  <hr>

  <h4>width：只指定一個維度</h4>
  <img src="./images/example.jpg" alt="課程示意圖片" width="200" />
  <hr>

  <h4>width / height：可能導致變形</h4>
  <img src="./images/example.jpg" alt="課程示意圖片" width="200" height="200" />
</body>
```

---

### 這段範例在展示什麼？

* 第一張圖：最基本的 `<img>` 插入方式
* 第二張圖：`alt` 不是提示，而是替代資訊
* 第三張圖：`title` 是補充說明，不取代 `alt`
* 第四張圖：只指定一個維度，通常比較不易失真
* 第五張圖：若原圖比例不同，強行設定寬高可能變形

---

## 7. 實務上怎麼快速判斷？

你可以用下面這個簡化版判斷：

* **照片類**：先考慮 `jpg` 或 `webp`
* **透明背景 / Logo / 圖示**：先考慮 `png` 或 `webp`
* **簡單動畫**：先考慮 `gif` 或 `webp`
* **很小的特殊內嵌資源**：才考慮 `data URL / base64`

這樣在初學階段，已經足夠應付大部分常見情境。

**`<img>` 不只是把圖片放上頁面，而是在同時決定圖片從哪來、看不到時怎麼替代、顯示多大，以及該用什麼格式更合適。**

## 常見問法

### `alt` 和 `title` 最大差別是什麼？

* `alt` 是替代文字，重點是圖片無法直接看到時，還能留下什麼資訊。
* `title` 是補充提示，重點不同，不能拿來取代 `alt`。

---

### 只寫 `width` 不寫 `height` 可以嗎？

可以。
而且很多時候這樣反而更安全，因為圖片通常會依原始比例縮放，比較不容易變形。

---

### 把圖片在頁面上縮小，檔案會不會跟著變小？

不會。
頁面上的顯示尺寸變小，不等於原始檔案大小變小。

---

### 照片、Logo、動圖通常各自優先考慮哪種格式？

* 照片：`jpg` 或 `webp`
* Logo、透明背景圖片：`png` 或 `webp`
* 簡單動圖：`gif` 或 `webp`

---

### `src` 可以直接放 `base64` 嗎？

可以。
但比較準確的理解是：那是 `data URL` 形式的內嵌資料，不是一般圖片格式本身。
