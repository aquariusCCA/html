# 第三十七章｜表單進階驗證

> 章節定位：補齊 HTML 表單欄位驗證、輸入限制與瀏覽器提示行為  
> 建議閱讀方式：先理解瀏覽器內建驗證能做什麼，再分清它不能取代後端驗證

## 本章在學什麼

這一章延續第二十四章表單內容，補上更完整的表單驗證與欄位限制。

重點包括：

- `required`、`minlength`、`maxlength`、`min`、`max`、`step` 如何限制輸入。
- `type="email"`、`type="url"`、`type="number"` 等輸入類型如何帶來內建驗證。
- `pattern` 如何用正則表達式描述格式。
- `readonly`、`disabled`、`placeholder`、`autocomplete` 的差異。
- 為什麼前端驗證不能取代後端驗證。

## 本章與整體地圖的關係

- 前置知識：第二十四章已學過表單域、表單控件、label、fieldset 與基本屬性。
- 這章的核心任務：讓表單從「可以輸入」進一步變成「能描述輸入規則」。
- 讀完後通常接：JavaScript 表單處理、使用者體驗與可存取性。

## 適合快速回查的主題

- `required` 怎麼判斷必填
- `min` / `max` / `step` 如何限制數值或日期
- `minlength` / `maxlength` 和 `pattern` 差在哪
- `readonly` 和 `disabled` 差在哪
- `placeholder` 為什麼不能取代 `<label>`
- `autocomplete` 是做什麼的
- `novalidate` 和 `formnovalidate` 如何關閉驗證

## 易混淆主題

- 前端驗證 vs 後端驗證：前者改善體驗，後者才是資料安全邊界。
- `readonly` vs `disabled`：前者不能改但通常會提交，後者失效且通常不提交。
- `placeholder` vs `<label>`：提示文字不能取代欄位名稱。
