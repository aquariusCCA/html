`rel` 與 `href`

`rel` 說明關係，`href` 指向資源位置。兩者通常要一起看才知道瀏覽器會做什麼。

1. 這個 demo 的 `` 實際寫法

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="HTML head 補充標籤 demo...">
  <title>Head 補充標籤 Demo</title>
  <link rel="icon" type="image/svg+xml" href="./assets/favicon.svg">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link rel="preload" href="./assets/card-preview.svg" as="image" type="image/svg+xml">
  <link rel="stylesheet" href="./css/base.css">
  <link rel="stylesheet" href="./css/page.css" media="screen">
  <link rel="stylesheet" href="./css/print.css" media="print">
</head>
  • stylesheet 把外部 CSS 檔接到 HTML。這個頁面先載入 `base.css`，再載入 `page.css`。
  • icon 指定瀏覽器分頁、書籤或歷史紀錄可能使用的網站圖示。
  • preconnect 提前和外部來源建立連線準備，適合字型 CDN、API 或重要第三方資源。
  • preload 提前抓取接下來很快會用到的資源，必須搭配正確的 `as` 類型。
