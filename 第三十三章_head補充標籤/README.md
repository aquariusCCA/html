# 第三十三章｜`head` 補充標籤

> 章節定位：補齊 HTML 文件頭部中用來連接外部資源與設定基準路徑的標籤  
> 建議閱讀方式：先分清 `<meta>`、`<title>`、`<link>`、`<base>` 各自在處理什麼，再回頭看完整 `head` 範例

## 本章在學什麼

這一章整理前面尚未獨立展開的 `head` 相關標籤，重點放在 `<link>` 和 `<base>`。

`<meta>` 和 `<title>` 前面已經學過，這一章補的是：

- `<link>` 如何把 CSS、favicon、預連線等外部資源和目前文件建立關係。
- `rel` 如何描述「目前文件」和「外部資源」的關係。
- `<base>` 如何改變整份文件中相對 URL 的解析基準。
- 為什麼 `<base>` 雖然有用，但實務上要非常小心。

## 本章與整體地圖的關係

- 前置知識：第二章已學過 HTML 骨架，第八章已學過 `meta`，第九章已學過 `title`。
- 這章的核心任務：補完整份 HTML 文件頭部的資源連接觀念。
- 讀完後通常接：`script` / `noscript`、響應式圖片、結構化資料等更完整的資源與資料描述章節。

## 適合快速回查的主題

- `<link>` 是做什麼的
- `rel="stylesheet"` 如何載入 CSS
- favicon 怎麼寫
- `rel` 和 `href` 怎麼分工
- `<base>` 會影響哪些相對路徑
- 為什麼 `<base>` 通常不建議隨便使用

## 易混淆主題

- `<link>` vs `<a>`：前者描述文件與外部資源的關係，後者建立使用者可點擊的超連結。
- `rel` vs `href`：前者描述關係，後者指定資源位置。
- `<base>` vs 一般相對路徑：`<base>` 會改變整份文件的相對 URL 解析方式。
