# 技術與細節文本標籤

> - 所屬章節：第四十章｜技術與細節文本標籤  
> - 關鍵字：`code`、`kbd`、`samp`、`var`、`sub`、`sup`、`small`、`time`、程式碼、鍵盤輸入、程式輸出、變數、時間  
> - 建議回查情境：想寫技術文件、鍵盤快捷鍵、終端輸出、變數、註腳、化學式、日期時間，或想分清 `<pre>` 和 `<code>` 時

## 本節導讀

有些文字不是普通段落內容，而是技術文件中的特殊角色：

- 這是一段程式碼。
- 這是使用者要按的按鍵。
- 這是程式輸出。
- 這是變數。
- 這是一個日期時間。
- 這是旁註或法律小字。

HTML 提供了一組細節文本標籤，用來把這些角色標示清楚。

## 整理說明

- 原始學習重點：第十六章已補過 `<pre>` 和 `<code>` 的關係。
- 本次整理方向：把技術文件與細節文字常用標籤集中整理。
- 補強內容：加入 `<kbd>`、`<samp>`、`<var>`、`<time>` 的判斷。
- 待查證：本章聚焦 HTML 語意，不處理完整程式碼高亮工具或日期格式化函式庫。

## 你會在這篇學到什麼

- `<code>` 適合什麼內容
- `<pre><code>` 什麼時候搭配
- `<kbd>`、`<samp>`、`<var>` 怎麼分
- `<sub>`、`<sup>` 什麼時候不是純樣式
- `<small>` 的語意不是單純小字
- `<time datetime>` 怎麼讓時間機器可讀

## 30 秒複習入口

- `<code>` 表示程式碼。
- `<pre>` 保留空白和換行，常搭配 `<code>` 顯示多行程式碼。
- `<kbd>` 表示使用者輸入，常見於鍵盤按鍵。
- `<samp>` 表示程式或系統輸出。
- `<var>` 表示變數。
- `<sub>` / `<sup>` 表示下標 / 上標語意。
- `<small>` 表示旁註、小字說明、法律聲明等。
- `<time>` 搭配 `datetime` 可以提供機器可讀時間。

## 速查區

### 技術文本四件組

| 標籤 | 語意 | 例子 |
|---|---|---|
| `<code>` | 程式碼 | `const name = "HTML";` |
| `<kbd>` | 使用者輸入 | `Ctrl` + `S` |
| `<samp>` | 程式或系統輸出 | `Error: file not found` |
| `<var>` | 變數 | `x`、`n`、`count` |

### 細節文本

| 標籤 | 語意 | 例子 |
|---|---|---|
| `<sub>` | 下標 | H₂O |
| `<sup>` | 上標 | x²、註腳標號 |
| `<small>` | 旁註、小字說明 | 版權聲明、補充條款 |
| `<time>` | 日期或時間 | 2026-04-26 |

## 正文筆記

### 1. `<code>`：程式碼

行內程式碼：

```html
<p>使用 <code>document.querySelector()</code> 可以選取元素。</p>
```

`<code>` 表示這段文字是程式碼。  
它不負責保留換行或縮排。

多行程式碼通常搭配 `<pre>`：

```html
<pre><code>const title = document.querySelector("h1");
title.textContent = "Hello HTML";</code></pre>
```

這裡：

- `<pre>` 保留格式。
- `<code>` 表示內容是程式碼。

### 2. `<kbd>`：使用者輸入

`<kbd>` 表示使用者要輸入的內容，常見於鍵盤快捷鍵。

```html
<p>按下 <kbd>Ctrl</kbd> + <kbd>S</kbd> 可以儲存檔案。</p>
```

也可以表示命令列輸入：

```html
<p>請在終端機輸入 <kbd>npm install</kbd>。</p>
```

`<kbd>` 的重點是「使用者輸入」，不是「鍵盤外觀」。

### 3. `<samp>`：程式輸出

`<samp>` 表示程式、系統或終端機輸出的範例。

```html
<p>如果檔案不存在，終端機可能顯示 <samp>No such file or directory</samp>。</p>
```

可以和 `<kbd>` 搭配：

```html
<p>輸入 <kbd>node -v</kbd> 後，終端機輸出 <samp>v22.0.0</samp>。</p>
```

一個是使用者輸入，一個是系統輸出。

### 4. `<var>`：變數

`<var>` 表示數學或程式中的變數。

```html
<p>設 <var>x</var> 為使用者輸入的數字。</p>
```

或：

```html
<p>函式會回傳 <var>count</var> 的目前值。</p>
```

如果是完整程式碼片段，通常用 `<code>`。  
如果是在說明文字中提到抽象變數，可以用 `<var>`。

### 5. `<sub>` 和 `<sup>`

`<sub>` 表示下標：

```html
<p>水的化學式是 H<sub>2</sub>O。</p>
```

`<sup>` 表示上標：

```html
<p>面積公式可以寫成 x<sup>2</sup>。</p>
```

也常見於註腳標號：

```html
<p>HTML 是網頁結構的基礎。<sup>1</sup></p>
```

不要只為了把文字移上移下就使用它們。  
如果只是視覺效果，應該交給 CSS。

### 6. `<small>`：旁註或小字說明

`<small>` 不是單純讓文字變小的標籤。  
它常用於旁註、補充說明、版權、法律文字等。

```html
<footer>
  <small>&copy; 2026 HTML Notes. All rights reserved.</small>
</footer>
```

如果只是想控制字體大小，應該使用 CSS。

### 7. `<time>`：日期與時間

`<time>` 用來標示日期或時間。  
搭配 `datetime` 可以提供機器可讀格式。

```html
<p>
  本文更新於
  <time datetime="2026-04-26">2026 年 4 月 26 日</time>。
</p>
```

畫面上可以顯示人類易讀格式，`datetime` 裡放標準格式，方便機器解析。

也可以表示時間：

```html
<time datetime="14:30">下午 2:30</time>
```

### 8. 什麼時候不用這些標籤？

如果只是要改外觀，不應該用語意標籤硬套。

例如只是讓字變小：

```html
<span class="text-sm">這只是視覺變小</span>
```

如果不是程式碼，不要用 `<code>`。  
如果不是使用者輸入，不要用 `<kbd>`。  
如果不是時間，不要用 `<time>`。

## 常見問法

### `<code>` 可以單獨用嗎？

可以。

行內程式碼常單獨用 `<code>`。  
多行程式碼才常搭配 `<pre>`。

### `<pre>` 和 `<code>` 誰負責保留縮排？

`<pre>` 負責保留空白、換行和縮排。  
`<code>` 負責表示程式碼語意。

### `<kbd>` 是不是只能表示鍵盤按鍵？

不只。

它表示使用者輸入，鍵盤按鍵是最常見情境，也可以表示命令列輸入。

### `<small>` 是不是就是小字？

不是只等於小字。

它有旁註、小字說明、法律文字等語意。  
單純樣式大小應交給 CSS。

### `<time>` 一定要寫 `datetime` 嗎？

不一定，但建議在顯示文字不是標準機器可讀格式時補上。  
這樣瀏覽器、搜尋工具或程式更容易理解。
