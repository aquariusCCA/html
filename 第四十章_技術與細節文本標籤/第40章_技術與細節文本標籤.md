# 技術與細節文本標籤

> - 所屬章節：第四十章｜技術與細節文本標籤  
> - 關鍵字：`code`、`kbd`、`samp`、`var`、`sub`、`sup`、`small`、`time`、程式碼、鍵盤輸入、程式輸出、變數、上下標、旁註、日期時間  
> - 建議回查情境：想寫技術文件、鍵盤快捷鍵、終端機輸入與輸出、變數、註腳、化學式、日期時間，或想分清 `<pre>`、`<code>`、`<kbd>`、`<samp>` 時

## 本節導讀

有些文字看起來只是「一小段文字」，但在語意上不是普通句子，而是有特定角色：

- 這是一段程式碼。
- 這是使用者要輸入的按鍵或命令。
- 這是程式、系統或終端機輸出的結果。
- 這是公式或說明文字中的變數。
- 這是化學式、次方或註腳編號。
- 這是旁註、補充說明、版權或法律小字。
- 這是一段日期或時間。

HTML 提供了一組細節文本標籤，用來把這些文字角色標示清楚。

本章的核心觀念是：

> 先判斷文字的「語意角色」，再決定要不要使用對應標籤。  
> 如果只是想改外觀，應該交給 CSS，而不是硬套語意標籤。

## 本章範圍

- 原始學習重點：第十六章已補過 `<pre>` 和 `<code>` 的關係。
- 本次整理方向：把技術文件與細節文字常用標籤集中整理。
- 補強內容：加入 `<kbd>`、`<samp>`、`<var>`、`<time>` 的判斷，並補上易混淆情境。
- 本章不處理：完整程式碼高亮工具、Markdown 程式碼區塊語法、日期格式化函式庫與 CSS 美化細節。

## 你會在這篇學到什麼

- `<code>` 適合什麼內容。
- `<pre><code>` 什麼時候搭配使用。
- `<kbd>`、`<samp>`、`<var>` 如何分辨。
- `<sub>`、`<sup>` 什麼時候是語意需求。
- `<small>` 為什麼不是單純的小字工具。
- `<time datetime>` 如何讓時間兼顧人類閱讀與機器解析。
- 遇到特殊文字時，如何用一套流程判斷該不該使用語意標籤。

## 30 秒複習入口

- `<code>`：表示程式碼。
- `<pre>`：保留空白、換行與縮排；多行程式碼常搭配 `<code>`。
- `<kbd>`：表示使用者輸入，例如鍵盤快捷鍵或命令列輸入。
- `<samp>`：表示程式、系統或終端機輸出。
- `<var>`：表示數學或程式說明中的變數。
- `<sub>` / `<sup>`：表示下標 / 上標語意，例如化學式、次方、註腳。
- `<small>`：表示旁註、小字說明、版權、法律聲明等。
- `<time>`：表示日期或時間；搭配 `datetime` 可以提供機器可讀格式。
- 只想改外觀時：不要用語意標籤，改用 CSS。

| `<kbd>` | 這是使用者要輸入的內容。 |
| `<samp>` | 這是系統或程式輸出的內容。 |
| `<var>` | 這是變數。 |
| `<sub>` | 這是下標。 |
| `<sup>` | 這是上標。 |
| `<small>` | 這是旁註或小字說明。 |
| `<time>` | 這是日期或時間。 |

## 判斷流程

看到一段特殊文字時，可以照下面流程判斷：

```text

├─ 程式碼？
│  ├─ 行內程式碼：<code>
│  └─ 多行且要保留縮排：<pre><code>
│
├─ 使用者輸入？
│  └─ <kbd>
│
├─ 程式或系統輸出？
│  └─ <samp>
│
├─ 變數？
│  └─ <var>
│
├─ 下標或上標語意？
│  ├─ 下標：<sub>
│  └─ 上標：<sup>
│
├─ 旁註、補充條款、版權或法律文字？
│  └─ <small>
│
├─ 日期或時間？
│  └─ <time>，必要時加 datetime
│
└─ 只是想改外觀？
   └─ 不用這些語意標籤，改用 CSS
```

## 速查區

### 技術文本四件組

| 標籤 | 語意 | 適合情境 | 不適合情境 |
|---|---|---|---|
| `<code>` | 程式碼 | API 名稱、函式名、變數名、短程式片段 | 只是想用等寬字體 |
| `<kbd>` | 使用者輸入 | 快捷鍵、使用者要輸入的命令 | 程式碼本身、系統輸出 |
| `<samp>` | 程式或系統輸出 | 終端機輸出、錯誤訊息、執行結果 | 使用者輸入、程式碼教學片段 |
| `<var>` | 變數 | 公式變數、說明文字中的抽象變數 | 完整程式碼片段 |

### 細節文本

| 標籤 | 語意 | 適合情境 | 不適合情境 |
|---|---|---|---|
| `<sub>` | 下標 | H<sub>2</sub>O、化學式、數學下標 | 單純想讓文字往下 |
| `<sup>` | 上標 | x<sup>2</sup>、註腳編號、次方 | 單純想讓文字往上 |
| `<small>` | 旁註、小字說明 | 版權聲明、補充條款、法律文字 | 單純想讓字變小 |
| `<time>` | 日期或時間 | 文章日期、活動時間、更新時間 | 普通數字或無時間語意的文字 |

## 正文筆記

### 1. `<code>`：程式碼

`<code>` 用來表示一段文字是程式碼。  
它可以用在行內文字中，例如函式名稱、變數名稱、CSS 屬性、HTML 標籤名稱或短程式片段。

```html
<p>使用 <code>document.querySelector()</code> 可以選取元素。</p>
```

這裡的 `document.querySelector()` 是程式碼，所以適合用 `<code>`。

#### `<code>` 不負責保留格式

`<code>` 只負責表示「這是程式碼」。  
它不負責保留換行、空白或縮排。

如果是多行程式碼，通常會搭配 `<pre>`：

```html
<pre><code>const title = document.querySelector("h1");
title.textContent = "Hello HTML";</code></pre>
```

這裡分工很清楚：

- `<pre>`：保留空白、換行與縮排。
- `<code>`：表示裡面的內容是程式碼。

#### 常見錯誤

不要因為想要等寬字體就使用 `<code>`。

```html
<!-- 不建議：這只是想讓文字變成等寬樣式 -->
<p>我的品牌名稱是 <code>Mono Shop</code>。</p>
```

如果只是樣式需求，應該使用 CSS：

```html
<span class="font-mono">Mono Shop</span>
```

### 2. `<kbd>`：使用者輸入

`<kbd>` 表示使用者要輸入的內容。  
最常見的情境是鍵盤快捷鍵。

```html
<p>按下 <kbd>Ctrl</kbd> + <kbd>S</kbd> 可以儲存檔案。</p>
```

也可以表示命令列輸入：

```html
<p>請在終端機輸入 <kbd>npm install</kbd>。</p>
```

`<kbd>` 的重點不是「看起來像鍵盤按鍵」，而是「這是使用者要輸入的內容」。

#### `<kbd>` 和 `<code>` 的差異

```html
<p>請輸入 <kbd>npm run dev</kbd> 啟動開發伺服器。</p>
<p><code>npm run dev</code> 是 package.json scripts 中的指令。</p>
```

第一句強調「使用者要輸入」。  
第二句強調「這是一段指令文字或程式碼內容」。

兩者內容可能長得一樣，但語意角度不同。

### 3. `<samp>`：程式或系統輸出

`<samp>` 表示程式、系統或終端機輸出的範例。

```html
<p>如果檔案不存在，終端機可能顯示 <samp>No such file or directory</samp>。</p>
```

可以和 `<kbd>` 搭配，清楚區分「輸入」與「輸出」：

```html
<p>輸入 <kbd>node -v</kbd> 後，終端機輸出 <samp>v22.0.0</samp>。</p>
```

這裡：

- `<kbd>node -v</kbd>`：使用者輸入的命令。
- `<samp>v22.0.0</samp>`：系統輸出的結果。

#### 常見錯誤

不要把所有終端機相關文字都用 `<code>`。  
如果你是在描述「畫面上顯示的結果」，`<samp>` 會比 `<code>` 更精準。

### 4. `<var>`：變數

`<var>` 表示數學或程式說明中的變數。

```html
<p>設 <var>x</var> 為使用者輸入的數字。</p>
```

也可以用在程式教學的抽象說明中：

```html
<p>函式會回傳 <var>count</var> 的目前值。</p>
```

如果是在完整程式碼片段裡，通常用 `<code>`。  
如果是在一般說明文字中提到抽象變數，才比較適合用 `<var>`。

```html
<p>令 <var>n</var> 代表陣列長度。</p>

<pre><code>const n = arr.length;</code></pre>
```

這裡：

- 第一個 `n` 是說明文字中的抽象變數，所以用 `<var>`。
- 第二個 `n` 是完整程式碼的一部分，所以放在 `<code>` 裡。

### 5. `<sub>` 和 `<sup>`：上下標語意

`<sub>` 表示下標，常見於化學式或數學表示法。

```html
<p>水的化學式是 H<sub>2</sub>O。</p>
```

`<sup>` 表示上標，常見於次方或註腳編號。

```html
<p>面積可以寫成 x<sup>2</sup>。</p>
```

```html
<p>HTML 是網頁結構的基礎。<sup>1</sup></p>
```

#### 使用重點

只有當「下標」或「上標」本身有語意時，才使用 `<sub>` 或 `<sup>`。

例如：

| 情境 | 適合標籤 | 原因 |
|---|---|---|
| H<sub>2</sub>O | `<sub>` | 2 是化學式下標。 |
| x<sup>2</sup> | `<sup>` | 2 表示平方。 |
| 註腳編號 | `<sup>` | 編號通常以上標呈現，且有註記語意。 |
| 裝飾性文字位移 | CSS | 只是視覺效果，不是語意。 |

### 6. `<small>`：旁註與小字說明

`<small>` 不是單純讓文字變小的標籤。  
它表示旁註、補充說明、版權、法律文字、免責聲明等「附屬性說明」。

```html
<footer>
  <small>&copy; 2026 HTML Notes. All rights reserved.</small>
</footer>
```

也可以用在表單或商品頁的補充條件：

```html
<p>
  免費試用 14 天
  <small>需綁定付款方式，試用期結束後自動續訂。</small>
</p>
```

#### 常見錯誤

不要只是為了讓文字變小就使用 `<small>`。

```html
<!-- 不建議：只是視覺尺寸需求 -->
<small>這段文字只是想變小，沒有旁註語意。</small>
```

如果只是樣式需求，應該使用 CSS：

```html
<span class="text-sm">這段文字只是視覺變小。</span>
```

### 7. `<time>`：日期與時間

`<time>` 用來標示日期或時間。  
它可以讓畫面顯示人類易讀格式，同時透過 `datetime` 提供機器可讀格式。

```html
<p>
  本文更新於
  <time datetime="2026-04-26">2026 年 4 月 26 日</time>。
</p>
```

畫面上使用「2026 年 4 月 26 日」這種人類易讀格式。  
`datetime="2026-04-26"` 則提供比較標準、容易解析的格式。

#### 常見 `datetime` 寫法

```html
<!-- 日期 -->
<time datetime="2026-04-26">2026 年 4 月 26 日</time>

<!-- 年月 -->
<time datetime="2026-04">2026 年 4 月</time>

<!-- 時間 -->
<time datetime="14:30">下午 2:30</time>

<!-- 日期 + 時間 + 時區 -->
<time datetime="2026-04-26T14:30:00+08:00">2026 年 4 月 26 日下午 2:30</time>
```

#### `<time>` 一定要寫 `datetime` 嗎？

不一定。  
但當顯示文字不是標準機器可讀格式時，建議加上 `datetime`。

例如：

```html
<time datetime="2026-04-26">昨天</time>
```

畫面顯示「昨天」對人類來說可能看得懂，但機器不一定知道實際日期。  
加上 `datetime` 後，就能明確提供實際時間資訊。

### 8. 什麼時候不用這些標籤？

如果只是要改外觀，不應該硬套語意標籤。

例如只是讓字變小：

```html
<span class="text-sm">這只是視覺變小</span>
```

例如只是讓字變成等寬字體：

```html
<span class="font-mono">這只是視覺等寬</span>
```

例如只是把字移到上方做裝飾：

```html
<span class="decorative-sup">NEW</span>
```

判斷原則很簡單：

> 文字有特殊角色，用語意標籤。  
> 文字只是要改外觀，用 CSS。

## 完整情境示範

假設你要寫一段教學文字：

```html
<p>
  請在終端機輸入 <kbd>node -v</kbd>，
  如果安裝成功，可能會看到 <samp>v22.0.0</samp>。
</p>
```

這裡用到兩個標籤：

- `<kbd>`：因為 `node -v` 是使用者要輸入的內容。
- `<samp>`：因為 `v22.0.0` 是終端機輸出的結果。

如果你要補充一段程式碼，則可以這樣寫：

```html
<pre><code>const version = process.version;
console.log(version);</code></pre>
```

這裡：

- `<pre>` 負責保留多行程式碼格式。
- `<code>` 負責表示內容是程式碼。

如果你要說明變數意義，則可以這樣寫：

```html
<p>其中 <var>version</var> 代表目前 Node.js 的版本字串。</p>
```

這裡的 `version` 不是完整程式碼片段，而是在說明文字中扮演變數角色，所以可以用 `<var>`。

## 易混淆對照表

| 情境 | 建議寫法 | 不建議寫法 | 原因 |
|---|---|---|---|
| 行內提到函式名稱 | `<code>querySelector()</code>` | `<span>querySelector()</span>` | 函式名稱有程式碼語意。 |
| 多行程式碼 | `<pre><code>...</code></pre>` | 只用 `<code>` | `<code>` 不保留換行與縮排。 |
| 快捷鍵 | `<kbd>Ctrl</kbd> + <kbd>S</kbd>` | `<code>Ctrl + S</code>` | 這是使用者輸入，不是程式碼。 |
| 終端機輸入 | `<kbd>npm install</kbd>` | `<samp>npm install</samp>` | 這是使用者輸入，不是輸出。 |
| 終端機輸出 | `<samp>Done</samp>` | `<kbd>Done</kbd>` | 這是系統輸出，不是使用者輸入。 |
| 說明公式中的 x | `<var>x</var>` | `<code>x</code>` | 這是抽象變數，不一定是程式碼。 |
| 化學式 H₂O | `H<sub>2</sub>O` | 用 CSS 往下移 | 2 有下標語意。 |
| 平方 x² | `x<sup>2</sup>` | 用 CSS 往上移 | 2 有次方語意。 |
| 版權文字 | `<small>&copy; 2026...</small>` | 只用 `<span>` | 這是附屬性小字說明。 |
| 文章日期 | `<time datetime="2026-04-26">2026 年 4 月 26 日</time>` | 純文字日期 | `<time>` 能表達日期語意。 |
| 單純縮小字 | `<span class="text-sm">...</span>` | `<small>...</small>` | 沒有旁註語意時，不應用 `<small>`。 |

## 常見問法

### `<code>` 可以單獨用嗎？

可以。  
行內程式碼常單獨使用 `<code>`。

```html
<p>可以使用 <code>addEventListener()</code> 綁定事件。</p>
```

多行程式碼才常搭配 `<pre>`。

### `<pre>` 和 `<code>` 誰負責保留縮排？

`<pre>` 負責保留空白、換行與縮排。  
`<code>` 負責表示內容是程式碼。

所以多行程式碼常寫成：

```html
<pre><code>function sayHi() {
  console.log("Hi");
}</code></pre>
```

### `<kbd>` 是不是只能表示鍵盤按鍵？

不是。  
`<kbd>` 表示使用者輸入，鍵盤按鍵只是最常見情境。

它也可以表示命令列輸入：

```html
<p>輸入 <kbd>git status</kbd> 查看目前狀態。</p>
```

### `<samp>` 和 `<code>` 差在哪？

`<code>` 表示程式碼。  
`<samp>` 表示程式、系統或終端機輸出的範例。

```html
<p>執行後會看到 <samp>Success</samp>。</p>
```

這裡的 `Success` 是輸出結果，所以用 `<samp>` 更精準。

### `<var>` 和 `<code>` 怎麼分？

看它是在「說明文字」還是在「程式碼片段」中。

```html
<p>令 <var>count</var> 表示目前數量。</p>
<pre><code>const count = items.length;</code></pre>
```

說明文字中的抽象變數用 `<var>`。  
完整程式碼片段用 `<code>`。

### `<small>` 是不是就是小字？

不是。  
`<small>` 表示旁註、小字說明、法律文字、版權聲明等附屬內容。

單純控制字體大小，應該使用 CSS。

### `<time>` 一定要寫 `datetime` 嗎？

不一定，但建議在顯示文字不是標準機器可讀格式時補上。

```html
<time datetime="2026-04-26">2026 年 4 月 26 日</time>
```

這樣人類看到的是易讀文字，程式或工具也能讀到明確日期。

## 自測問題

### 基礎題

1. `<code>` 和 `<pre>` 的分工是什麼？
2. `<kbd>` 和 `<samp>` 分別表示什麼？
3. `<var>` 適合用在哪些說明情境？
4. `<sub>` 和 `<sup>` 什麼時候是語意需求？
5. `<small>` 為什麼不是純樣式工具？
6. `<time datetime>` 有什麼好處？

### 判斷題

請判斷下列情境適合使用哪個標籤：

1. 文章中提到 `Array.prototype.map()`。
2. 教使用者按下 `Ctrl + C`。
3. 終端機顯示 `Permission denied`。
4. 公式中使用變數 `n` 表示資料筆數。
5. 化學式中的 H₂O。
6. 頁尾的版權聲明。
7. 顯示「2026 年 4 月 26 日」這個文章更新日期。
8. 只是想讓一段文字變小。

### 參考答案

1. `<code>`
2. `<kbd>`
3. `<samp>`
4. `<var>`
5. `<sub>`
6. `<small>`
7. `<time datetime="2026-04-26">2026 年 4 月 26 日</time>`
8. CSS，例如 `<span class="text-sm">...</span>`

## 本章重點回收

本章不是在背「哪個標籤預設長怎樣」，而是在訓練 HTML 語意判斷：

- 程式碼：`<code>`
- 多行格式保留：`<pre>`
- 使用者輸入：`<kbd>`
- 系統輸出：`<samp>`
- 變數：`<var>`
- 下標：`<sub>`
- 上標：`<sup>`
- 旁註小字：`<small>`
- 日期時間：`<time>`
- 純外觀調整：CSS

> HTML 負責語意，CSS 負責外觀。
