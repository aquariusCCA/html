# 第四十章｜技術與細節文本標籤

> 章節定位：整理程式碼、鍵盤輸入、程式輸出、變數、上下標、旁註與時間等細節語意  
> 建議閱讀方式：先分清技術文本四件組，再補上下標、旁註與時間標籤

## 本章在學什麼

這一章整理常出現在技術文件、教材、說明頁中的細節文本標籤：

- `<code>`、`<kbd>`、`<samp>`、`<var>`
- `<sub>`、`<sup>`
- `<small>`
- `<time>`

它們的重點不是外觀，而是讓文字角色更精準。

## 本章與整體地圖的關係

- 前置知識：第十六章已學過 `<pre>`，第十七章已學過基礎文本格式化。
- 這章的核心任務：補齊技術文件與細節資訊常用的行內語意標籤。
- 讀完後通常接：引用與註記文本、結構化資料、可存取性。

## 適合快速回查的主題

- `<code>` 和 `<pre>` 怎麼搭配
- `<kbd>`、`<samp>`、`<var>` 分別代表什麼
- `<sub>` 和 `<sup>` 什麼時候是語意需求
- `<small>` 不是單純縮小字
- `<time datetime>` 如何提供機器可讀時間

## 易混淆主題

- `<pre>` vs `<code>`：前者保留格式，後者表示程式碼語意。
- `<kbd>` vs `<code>`：前者是使用者輸入，後者是程式碼。
- `<small>` vs CSS 字體大小：前者表示旁註、小字說明等語意，不是純樣式工具。
