# 第四十章｜技術與細節文本標籤

> 章節定位：整理技術文件與細節文字常用的行內語意標籤。  
> 核心問題：當一段文字不是普通敘述，而是「程式碼、使用者輸入、程式輸出、變數、上下標、旁註或時間」時，應該用哪個 HTML 標籤標示？  
> 建議閱讀方式：先掌握「判斷流程」，再看各標籤範例，最後用「易混淆對照表」回查。

## 本章在學什麼

這一章整理常出現在技術文件、教材、說明頁、API 文件與表單說明中的細節文本標籤：

- 技術文本四件組：`<code>`、`<kbd>`、`<samp>`、`<var>`
- 上下標語意：`<sub>`、`<sup>`
- 旁註與小字說明：`<small>`
- 日期與時間：`<time>`

它們的重點不是「預設長什麼樣子」，而是讓瀏覽器、搜尋工具、輔助工具與維護者更清楚這段文字的角色。

## 為什麼要學這一章

很多初學者會把這些標籤當成「字體樣式」來記，例如：

- `<code>` 好像會變成等寬字。
- `<small>` 好像會讓字變小。
- `<sup>` 好像可以把文字移到上面。

但 HTML 的重點是語意。  
如果只是視覺效果，應該交給 CSS；如果文字本身具有特殊角色，才適合使用這些標籤。

## 本章與整體地圖的關係

- 前置知識：
  - 第十六章已學過 `<pre>`，知道它可以保留空白、換行與縮排。
  - 第十七章已學過基礎文本格式化，知道 HTML 標籤不應只從外觀判斷。
- 本章核心任務：
  - 補齊技術文件與細節資訊常用的行內語意標籤。
  - 建立「這段文字到底是什麼角色」的判斷能力。
- 讀完後通常接：
  - 引用與註記文本
  - 結構化資料
  - 可存取性
  - 技術文件撰寫

## 學完應該能做到

讀完本章後，你應該能判斷：

1. 什麼時候用 `<code>`，什麼時候需要 `<pre><code>`。
2. `<kbd>`、`<samp>`、`<var>` 分別對應哪種技術文字角色。
3. `<sub>`、`<sup>` 什麼時候是語意需求，什麼時候只是視覺效果。
4. `<small>` 為什麼不是單純的縮小文字工具。
5. `<time datetime>` 如何同時兼顧人類閱讀與機器解析。
6. 面對一段文字時，能先判斷「語意」再決定標籤。

## 快速判斷流程

看到一段特殊文字時，可以照這個順序判斷：

1. 這是程式碼嗎？  
   是：用 `<code>`；如果要保留多行格式，外層加 `<pre>`。
2. 這是使用者要輸入的內容嗎？  
   是：用 `<kbd>`。
3. 這是程式、系統或終端機輸出的內容嗎？  
   是：用 `<samp>`。
4. 這是數學或程式說明中的變數嗎？  
   是：用 `<var>`。
5. 這是化學式、數學次方或註腳編號嗎？  
   是：依需求使用 `<sub>` 或 `<sup>`。
6. 這是旁註、補充條款、版權或法律小字嗎？  
   是：用 `<small>`。
7. 這是日期、時間或可機器讀取的時間資訊嗎？  
   是：用 `<time>`，必要時補上 `datetime`。
8. 如果以上都不是，只是想改外觀：  
   不要硬套語意標籤，改用 CSS。

## 易混淆主題

| 容易混淆 | 差異 |
|---|---|
| `<pre>` vs `<code>` | `<pre>` 保留格式；`<code>` 表示程式碼語意。多行程式碼常用 `<pre><code>`。 |
| `<kbd>` vs `<code>` | `<kbd>` 是使用者輸入；`<code>` 是程式碼本身。 |
| `<kbd>` vs `<samp>` | `<kbd>` 是使用者打的；`<samp>` 是系統回傳的。 |
| `<var>` vs `<code>` | `<var>` 是說明文字中的變數；`<code>` 是程式碼片段。 |
| `<sub>` / `<sup>` vs CSS | 有下標、上標語意才用；只是視覺位移應用 CSS。 |
| `<small>` vs CSS 字體大小 | `<small>` 表示旁註、小字說明、法律聲明；單純縮小字應用 CSS。 |
| `<time>` vs 普通文字 | `<time>` 表示日期或時間；搭配 `datetime` 可提供機器可讀值。 |

## 本章不處理

本章聚焦 HTML 語意，不深入處理：

- 程式碼高亮工具，例如 Prism.js、Highlight.js。
- Markdown 程式碼區塊語法。
- JavaScript 日期格式化函式庫。
- CSS 如何美化 `<code>`、`<kbd>`、`<samp>` 等標籤。

這些可以放到後續「技術文件樣式」或「前端工具整合」章節再整理。
