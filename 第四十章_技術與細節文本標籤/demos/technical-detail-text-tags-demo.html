<code> <kbd> <samp> <var> <time>

技術與細節文本標籤 Demo

這個範例把第 40 章筆記中的技術文本四件組、上下標、旁註與時間標籤放在同一頁觀察。核心判斷是：先確認文字角色，再選擇 HTML 標籤；只想改外觀時交給 CSS。

<code> 程式碼、函式名、屬性名或短程式片段。
<kbd> 使用者要輸入的按鍵或命令。
<samp> 程式、系統或終端機輸出的結果。
<var> 數學或程式說明中的變數。
<time> 日期或時間，可搭配 datetime 給機器解析。

HTML 表達語意

技術標籤不是為了預設樣式，而是讓文件知道這段文字是程式碼、輸入、輸出、變數或時間。

<pre> 和 <code> 分工

<pre> 保留格式；<code> 表示程式碼語意。多行程式碼常兩者一起用。

CSS 處理外觀

只是想等寬、小字、上移或下移，不應硬套語意標籤，應使用 CSS。

1. 快速判斷器

判斷順序對應筆記：程式碼、輸入、輸出、變數、上下標、旁註、時間；以上都不是才回到 CSS。

2. 程式碼：<code> 與 <pre><code>

行內程式碼

使用 document.querySelector() 可以選取頁面中的第一個符合條件的元素。

<p>
  使用 <code>document.querySelector()</code> 可以選取元素。
</p>

多行程式碼

const title = document.querySelector("h1");
title.textContent = "Hello HTML";

<pre> 保留換行和縮排；<code> 表示內容是程式碼。

3. 輸入與輸出：<kbd>、<samp>

快捷鍵與命令列輸入

按下 Ctrl + S 可以儲存檔案。

請在終端機輸入 npm run dev 啟動開發伺服器。

系統或終端機輸出

如果檔案不存在，終端機可能顯示 No such file or directory。

$ node -v

4. 變數：<var>

說明文字中的抽象變數

令 n 代表資料筆數，total 代表加總結果。

平均值可寫成 total / n。

完整程式碼仍放在 <code>

const n = items.length;
const total = items.reduce((sum, item) => sum + item.price, 0);

這裡的 n 和 total 是程式碼片段的一部分，所以放在 <code> 裡。

7. 易混淆對照

情境 建議寫法 原因
快捷鍵 <kbd>Ctrl</kbd> + <kbd>S</kbd> 這是使用者輸入，不是程式碼。
終端機輸出 <samp>Permission denied</samp> 這是系統顯示的結果。
說明公式中的變數 <var>x</var> 這是抽象變數，不一定是程式碼。
單純縮小文字 <span class="text-sm">...</span> 沒有旁註語意時，不應使用 <small>。

8. 回查清單

  • 行內程式碼用 <code>；多行且要保留格式時用 <pre><code>。
  • 使用者輸入用 <kbd>，系統輸出用 <samp>。
  • 說明文字中的抽象變數用 <var>，完整程式碼片段仍放在 <code>。
  • 有下標或上標語意才用 <sub> / <sup>。
  • <small> 是旁註或附屬說明，不是單純縮小字工具。
  • <time datetime> 可同時提供人類可讀文字與機器可讀時間。
