# 兼容性問題

> 所屬章節：第三章｜兼容性問題  
> 關鍵字：瀏覽器兼容性、相容性、`X-UA-Compatible`、`IE=Edge`、`renderer`、雙核瀏覽器、`webkit`、`Trident`、`html5shiv`、HTML5 Shiv、條件註解、Conditional Comment、舊版 IE  
> 建議回查情境：想知道舊版 `IE` 為什麼需要額外指定文件模式、想確認雙核瀏覽器怎麼指定渲染核心、想處理 `HTML5` 語意化標籤在舊版瀏覽器中的兼容問題

## 本節導讀

這一節整理早期前端教材裡很常出現的幾種兼容性處理方式。  
主線是先理解兩件事：舊版 `IE` 可能因文件模式與支援差異造成頁面行為不一致，而某些國內雙核瀏覽器也可能切換不同渲染核心。  
如果你在舊專案、舊教材或歷史文章裡看到 `X-UA-Compatible`、`renderer` 或 `html5shiv`，可以先回這篇建立判讀框架。

## 你會在這篇學到什麼

- `X-UA-Compatible` 的作用與常見寫法
- `renderer` 這個 `meta` 標籤想解決什麼問題
- 為什麼舊版 `IE` 會需要 `html5shiv`
- `IE Conditional Comment` 的基本判讀方式

## 30 秒複習入口

- `X-UA-Compatible` 是給舊版 `IE` 用的文件模式控制標籤，常見寫法是 `IE=Edge`。
- `renderer` 主要出現在雙核瀏覽器語境，用來提示優先使用哪個渲染核心，例如 `webkit`。
- `html5shiv` 是一段用來幫舊版 `IE` 辨識 `HTML5` 語意化標籤的 JavaScript。
- `IE Conditional Comment` 是舊版 `IE` 專用語法，常用來只在特定 IE 版本載入兼容性程式碼。

## 速查區

### 核心概念

- 這篇的核心，是處理舊版瀏覽器對頁面解析方式不一致時的兼容手段。
- 常見場景包含：舊版 `IE` 的文件模式、雙核瀏覽器的渲染核心選擇，以及舊版 `IE` 對 `HTML5` 新標籤的辨識問題。

### 關鍵規則 / 判準

- 若舊版 `IE` 可能以非最新文件模式解析頁面，可用 `X-UA-Compatible` 提示使用 `IE=Edge`。
- 若教材或專案需要指定雙核瀏覽器優先走 `webkit` 內核，常會看到 `meta name="renderer" content="webkit"`。
- 若需要讓 `IE6`、`IE7`、`IE8` 認得 `HTML5` 新標籤，舊教材通常會搭配 `html5shiv` 與條件註解。

### 常見錯誤

- 把這些做法當成現代所有專案都必備，而忽略它們多半是舊瀏覽器相容性的歷史處理。
- 看到 `renderer` 就以為所有瀏覽器都支援這個標籤；它主要是特定瀏覽器環境下的兼容提示。
- 把 `html5shiv` 當成所有瀏覽器都需要的通用方案；它的重點其實是舊版 `IE`。

### 一句話對比

- `X-UA-Compatible` 管的是舊版 `IE` 的文件模式，`renderer` 管的是部分雙核瀏覽器的渲染核心傾向，`html5shiv` 則是在補舊版 `IE` 對 `HTML5` 新標籤的辨識能力。

## 正文筆記

### 這篇在解決什麼問題？

- 舊版瀏覽器之間，尤其是不同版本的 `IE`，對 `HTML`、`CSS` 和新標籤的支援並不一致。
- 同一份頁面在不同文件模式或不同渲染核心下，可能出現外觀與行為差異。
- 這篇整理的三個重點，就是早期常見的兼容性補救方式。

## 1. 添加元信息，讓瀏覽器處於較合適的渲染模式

### 1.1 強制 Internet Explorer 使用最新文件模式的 HTML Meta 標籤

- `X-UA-Compatible` 是一個寫在頁面中的 `meta` 標籤，用來告訴 `Internet Explorer` 應該採用哪種文件模式進行渲染。
- 由於不同版本的 `IE` 在解析 `HTML` 與 `CSS` 時可能存在差異，舊專案常會額外指定文件模式，以避免頁面在某些環境下退回較舊的相容模式。
- 常見寫法是 `IE=Edge`，意思是盡量讓 `IE` 使用它可用的最新文件模式，而不是用較舊的相容模式解析頁面。

```html
<meta http-equiv="X-UA-Compatible" content="IE=Edge">
```

### 1.2 控制雙核瀏覽器渲染行為的 HTML Meta 標籤：`renderer`

- 某些早期常見的國內瀏覽器採雙核設計，常見會同時具備 `WebKit` 與 `IE Trident` 兩套核心。
- 因此同一個頁面在這類瀏覽器中，可能會遇到「到底要用哪個核心渲染」的兼容性問題。
- 舊教材裡常見的做法，是在頁面中加入 `renderer` 這個 `meta` 標籤，提示瀏覽器優先使用哪個核心。
- 常見寫法 `content="webkit"`，表示希望優先使用 `WebKit` 內核渲染頁面。

```html
<meta name="renderer" content="webkit">
```

### 這一段要怎麼理解？

- `X-UA-Compatible` 的重點在 `IE` 文件模式。
- `renderer` 的重點在部分雙核瀏覽器的核心選擇。
- 它們都屬於「為了降低舊環境差異而額外補充的提示資訊」，不是 `HTML` 頁面本身的核心語法。

## 2. 使用 `html5shiv` 讓低版本瀏覽器認識 HTML5 的語意化標籤

### 為什麼需要 `html5shiv`？

> `html5shiv` 是一段讓 `IE6`、`IE7`、`IE8` 能辨識 `HTML5` 新標籤的 JavaScript。這類處理主要是舊版 `IE` 相容性的歷史方案，常會搭配條件註解只在特定 IE 版本中載入。

- `HTML5` 引入了許多語意化標籤，例如 `header`、`nav`、`section`、`article` 等。
- 舊版 `IE` 對這些新標籤的辨識能力有限，因此早期常需要額外載入 `html5shiv` 幫助瀏覽器理解這些元素。
- 由於這個需求主要發生在舊版 `IE`，因此通常不會讓所有瀏覽器都載入，而是搭配 `IE Conditional Comment` 使用。

### IE Conditional Comment 條件註解

- 這是舊版 `IE` 獨有的語法。
- 其他瀏覽器把它當成普通註解，`IE` 則會把它當成可判斷版本條件的程式碼區塊。
- 教材裡常見的寫法，是只在 `IE 9` 以下載入 `html5shiv`。
- `IE10` 之後不再支援這種條件註解語法，所以它本身也是歷史性方案。

```html
<!--[if lt IE 9]>
<script src="./js/html5shiv.min.js"></script>
<![endif]-->
```

### 條件註解裡常見的判斷詞

- `lt`：小於
- `lte`：小於等於
- `gt`：大於
- `gte`：大於等於
- `!`：邏輯非

```html
<!--[if IE 8]>僅 IE8 可見<![endif]-->
<!--[if gt IE 8]>僅 IE8 以上可見<![endif]-->
<!--[if lt IE 8]>僅 IE8 以下可見<![endif]-->
<!--[if gte IE 8]>IE8 以上可見<![endif]-->
<!--[if lte IE 8]>IE8 及以下可見<![endif]-->
<!--[if !IE 8]>非 IE8 的 IE 可見<![endif]-->
```

### 常見混淆點

- `html5shiv` 的目標不是讓所有瀏覽器都變快或變新，而是補舊版 `IE` 對新標籤的辨識問題。
- 條件註解不是一般 `HTML` 註解的進階版，而是舊版 `IE` 的特殊機制。
- 看到這些寫法時，通常代表你在讀的是偏舊版前端相容性語境，而不是現代前端的主流日常配置。
