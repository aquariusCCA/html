兼容性問題 Demo

這頁不是假裝你真的切進舊版 IE，而是幫你把第三章常見舊相容寫法和它們在補的問題對上： X-UA-Compatible、renderer、html5shiv，以及 IE 條件註解。

X-UA-Compatible 管的是舊版 IE 要用哪種文件模式解讀頁面。
renderer 管的是某些雙核瀏覽器傾向用哪個核心渲染頁面。
html5shiv 補的是舊版 IE 對 HTML5 新標籤的辨識能力。
條件註解 只讓特定版本的 IE 載入特定的舊相容程式碼。

這頁怎麼看

  • 先用第一區切換三種舊問題，確認每種手段補的是哪個洞。
  • 再看 `X-UA-Compatible` 和 `renderer` 的差別，不要把它們混在一起。
  • 接著看 `html5shiv` 對舊版 IE 的作用，理解它不是現代通用補丁。
  • 最後切換 IE 版本與條件註解，練習判讀哪段程式碼會載入。

1. 這段舊寫法到底在補哪一種相容性問題？

IE 文件模式

X-UA-Compatible：讓舊版 IE 盡量用較新的規則解析頁面

這個 `meta` 標籤主要不是給現代瀏覽器看的，而是在提示舊版 IE：盡量使用它能提供的較新文件模式。 

<meta http-equiv="X-UA-Compatible" content="IE=Edge">
這裡處理的是「IE 要怎麼解讀頁面」，不是「所有瀏覽器都一定會理會的標準功能」。
問題類型

舊版 IE 可能進入不預期的舊文件模式。

影響範圍

主要是舊版 IE，不是現代主流瀏覽器。

現代專案

通常不是現代專案的日常必備設定。

2. `X-UA-Compatible` 和 `renderer` 差在哪裡

X-UA-Compatible

偏向「IE 怎麼看」

處理的是舊版 IE 的文件模式問題，例如不要退回過舊的相容模式。

<meta http-equiv="X-UA-Compatible" content="IE=Edge">
renderer

偏向「瀏覽器怎麼畫」

處理的是部分雙核瀏覽器傾向用哪個核心渲染頁面，例如偏向 WebKit。

<meta name="renderer" content="webkit">
記法 1 X-UA-Compatible：偏向規則怎麼看。
記法 2 renderer：偏向頁面怎麼畫。

WebKit 核心

某些雙核瀏覽器會被提示優先用這一邊渲染。

IE Trident 核心

雙核環境下也可能存在這一套舊核心。

3. 為什麼舊版 IE 需要 html5shiv

<header>網站標頭</header>
<section>主內容區</section>
<footer>頁尾</footer>

<!--[if lt IE 9]>
<script src="./js/html5shiv.min.js"></script>
<![endif]-->

舊版 IE 看到 HTML5 新標籤時，可能不太知道怎麼處理

問題不只是名字陌生，還可能進一步影響樣式套用與版面參與。

<header>網站標頭</header>
<section>主內容區</section>
<footer>頁尾</footer>
