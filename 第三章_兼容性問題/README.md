# 第三章｜兼容性問題

> 章節定位：建立舊版瀏覽器相容性處理的基本判讀框架  
> 建議閱讀方式：先理解每種兼容性手段在處理哪個問題，再回頭辨認舊專案中的對應寫法

## 本章在學什麼

這一章先整理前端早期很常遇到的瀏覽器兼容性處理。  
重點不是把所有歷史瀏覽器細節背起來，而是先分清不同做法各自在補哪一塊相容性問題。

## 本章與整體地圖的關係

- 前置知識：第二章已先建立 `HTML` 骨架、版本差異與基本結構概念。
- 這章的核心任務：辨認舊版 `IE`、雙核瀏覽器與 `HTML5` 新標籤相關的兼容性處理。
- 讀完後通常接：更細的標籤、屬性、CSS 或實作內容時，遇到舊語法可以回來查。

## 適合快速回查的主題

- `X-UA-Compatible` 是在處理什麼
- `IE=Edge` 這種寫法代表什麼
- `renderer` 為什麼會和雙核瀏覽器一起出現
- `html5shiv` 的用途是什麼
- `IE Conditional Comment` 要怎麼判讀

## 高頻回查入口

- 想確認 `X-UA-Compatible`、`renderer`、`html5shiv` 分別在做什麼時，看：[兼容性問題](./第03章_兼容性問題.md)
- 想快速判讀 `lt`、`lte`、`gt`、`gte` 這些條件註解關鍵字時，看：[兼容性問題](./第03章_兼容性問題.md)

## 易混淆主題

- `X-UA-Compatible` vs `renderer`：前者偏舊版 `IE` 文件模式，後者偏部分雙核瀏覽器的核心選擇。
- `html5shiv` vs 一般 `script` 載入：前者是特定舊版瀏覽器的兼容性補丁，不是所有瀏覽器都需要。
