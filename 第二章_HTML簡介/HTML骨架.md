# HTML骨架

> 所屬章節：第二章｜HTML 簡介
> 關鍵字：HTML骨架、基本結構、`<!DOCTYPE html>`、標準模式、怪異模式、`<html>`、`lang`、`<head>`、`<meta charset>`、`viewport`、`<title>`、`<body>`、HTML文檔、`.html`、`.htm`
> 建議回查情境：想建立一份新的 `HTML` 檔案、想確認基本骨架怎麼寫、想分清 `head` 和 `body`、想理解 `DOCTYPE`、`charset`、`viewport`、`title` 各自負責什麼

## 本節導讀

知道 `HTML` 是用來描述網頁之後，下一步就要理解：**一個完整頁面不是把內容隨便堆上去，而是有固定的基本外框**。
這個外框，就是我們常說的 `HTML` 骨架。

第一次讀這篇時，先不要急著背每一行。你要先抓住三件事：

- `<!DOCTYPE html>` 告訴瀏覽器用現代標準方式解析。
- `<head>` 放頁面資訊、設定與資源連結。
- `<body>` 放使用者主要會看到或操作的內容。

## 你會在這篇學到什麼

- 為什麼頁面需要基本骨架
- 現代常見 `HTML` 基本模板怎麼寫
- `<!DOCTYPE html>`、`<html>`、`<head>`、`<body>` 各自的角色
- `lang`、`charset`、`viewport`、`title` 為什麼重要
- `<head>` 和頁面視覺上的「頁首」為什麼不是同一件事
- 瀏覽器容錯和「正確結構」之間的差別
- `HTML` 文檔副檔名與瀏覽器之間的基本關係

## 30 秒複習入口

- 現代 `HTML` 頁面常見骨架包含 `<!DOCTYPE html>`、`<html>`、`<head>`、`<meta charset>`、`<meta name="viewport">`、`<title>`、`<body>`。
- `<!DOCTYPE html>` 不是元素標籤，它用來讓瀏覽器以現代標準模式解析文件。
- `<head>` 主要放頁面設定、描述資訊與外部資源連結，不是畫面上的頁首。
- `<body>` 主要放真正顯示在頁面中的內容。
- 瀏覽器會容錯，但學習與實務上仍應明確寫出完整骨架。

## 先講結論

一份現代 `HTML` 頁面可以先想成四層：

1. 用 `<!DOCTYPE html>` 宣告文件類型。
2. 用 `<html>` 包住整份文件。
3. 在 `<head>` 放頁面設定與資訊。
4. 在 `<body>` 放頁面可見內容與主要互動內容。

最常用的基本模板如下：

```html
<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>網頁標題</title>
  </head>
  <body>
    頁面內容寫在這裡
  </body>
</html>
```

這份模板不是為了形式好看，而是讓瀏覽器、搜尋引擎、輔助技術與未來維護者都能穩定理解這份文件。

## 這篇在解決什麼問題？

初學 `HTML` 時，很多人會先注意到標題、段落、圖片、連結這些內容標籤，卻忽略：

- 一個完整頁面本身也有基本結構。
- 不是所有內容都應該寫在同一層。
- 有些資訊是給瀏覽器、搜尋引擎或輔助技術看的，不一定直接顯示在頁面上。
- 瀏覽器能顯示錯誤結構，不代表那份結構值得學。
- 手機畫面是否正常，常和 `viewport` 設定有關。

這篇的目標，就是建立「**頁面先有骨架，再把內容放進正確位置**」的基本觀念。

## 網頁為什麼需要骨架？

你可以把網頁類比成一篇文章。

- 文章通常會有標題、章節、內文、註解等結構。
- 網頁也一樣，需要基本外框來告訴瀏覽器：哪裡是整份文件、哪裡是頁面資訊、哪裡是實際顯示內容。

如果沒有這種基本結構，瀏覽器雖然有時仍會嘗試幫你修正，但你就很難穩定地理解與控制頁面。

更實務地說，清楚的骨架可以幫助：

- 瀏覽器用正確模式解析頁面。
- 中文、日文、表情符號等字元正常顯示。
- 手機瀏覽器用合理寬度顯示頁面。
- 搜尋引擎理解頁面標題與基本資訊。
- 螢幕閱讀器等輔助技術理解頁面語言與結構。
- 後續維護者快速看懂文件入口。

## HTML 用標籤把結構寫出來

`HTML` 不只是在描述「有什麼內容」，也在描述「這些內容位在什麼結構位置」。
因此，一個完整頁面通常會先有一組最基本的結構，再把實際內容放進去。

可以先把整體關係看成這樣：

```text
HTML 文檔
├── <!DOCTYPE html>
└── html
    ├── head
    │   ├── meta charset
    │   ├── meta viewport
    │   └── title
    └── body
        └── 頁面可見內容
```

## 現代最常見的基本骨架

```html
<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>網頁標題</title>
  </head>
  <body>
    <h1>我的第一個網頁</h1>
    <p>頁面內容寫在這裡。</p>
  </body>
</html>
```

這段不用死背，但建議你能默寫出大致順序。
後面建立新頁面時，這會是最常用的起點。

## 骨架各部分怎麼理解？

### `<!DOCTYPE html>`

`<!DOCTYPE html>` 用來宣告這是一份現代 `HTML` 文件。

它有幾個重點：

- 它通常放在文件最前面。
- 它不是一般元素標籤，不需要結束標籤。
- 它會影響瀏覽器使用標準模式還是接近舊時代的相容模式解析頁面。
- 入門時先記成：**請瀏覽器用現代標準方式理解這份 HTML**。

如果省略 `DOCTYPE`，某些情況下瀏覽器可能進入怪異模式，導致 CSS 排版與尺寸計算出現和現代標準不同的結果。
因此，新頁面應該明確寫上：

```html
<!DOCTYPE html>
```

### `<html>`

`<html>` 是整份頁面的根元素。

你可以把它理解成：

- 整個 `HTML` 文件最外層的外框。
- `<head>` 和 `<body>` 的共同父元素。
- 瀏覽器建立頁面結構時的主要根節點。

基本寫法：

```html
<html lang="zh-Hant">
  ...
</html>
```

### `lang` 屬性

`lang` 通常寫在 `<html>` 上，用來標示頁面的主要語言。

例如：

```html
<html lang="zh-Hant">
```

常見語言代碼可以先這樣理解：

| 寫法 | 大致意思 |
|---|---|
| `zh-Hant` | 繁體中文 |
| `zh-Hans` | 簡體中文 |
| `en` | 英文 |
| `ja` | 日文 |
| `ko` | 韓文 |

`lang` 的作用不只是給人看，它也有助於：

- 螢幕閱讀器選擇合適的朗讀語音。
- 瀏覽器與翻譯工具判斷頁面語言。
- 搜尋引擎理解頁面主要語言。
- 文字排版與語言相關處理更合理。

### `<head>`

`<head>` 主要放**頁面資訊、設定與外部資源連結**。

這裡的內容通常不會直接顯示成頁面主畫面上的文字區塊。
常見會放在 `<head>` 裡的內容包括：

- 字元編碼，例如 `<meta charset="UTF-8">`
- 行動裝置顯示設定，例如 `<meta name="viewport">`
- 分頁標題，例如 `<title>`
- 頁面描述，例如 `<meta name="description">`
- CSS 檔案連結，例如 `<link rel="stylesheet" href="style.css">`
- 網站小圖示，例如 `<link rel="icon" href="favicon.ico">`

初學時先抓一個核心觀念：

- `<head>` 不是主要可見內容區。
- 它更像是頁面的設定區、說明區與資源入口。

### `<meta charset="UTF-8">`

`<meta charset="UTF-8">` 用來宣告文件採用的文字編碼。

基本寫法：

```html
<meta charset="UTF-8">
```

它的作用是讓瀏覽器知道要用什麼編碼讀取這份文件。
如果編碼判斷錯誤，中文或其他非英文文字可能會顯示成亂碼。

入門時先記：

- 現代網頁通常使用 `UTF-8`。
- `charset` 建議放在 `<head>` 裡很前面的位置。
- 它不會直接顯示在頁面上，但會影響文字能不能正常顯示。

### `<meta name="viewport" content="width=device-width, initial-scale=1.0">`

`viewport` 用來控制行動裝置瀏覽器如何理解頁面的可視寬度與初始縮放。

常見寫法：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

可以先拆成兩部分理解：

- `width=device-width`：讓頁面寬度跟裝置螢幕寬度一致。
- `initial-scale=1.0`：頁面初始縮放比例為 `1`。

如果沒有這行，手機瀏覽器可能會用較寬的虛擬版面來排頁，再縮小顯示，導致文字看起來太小或響應式排版不如預期。

所以現在建立新頁面時，通常會把 `viewport` 放進基本骨架。

### `<title>`

`<title>` 用來定義頁面標題。

基本寫法：

```html
<title>網頁標題</title>
```

它通常會出現在：

- 瀏覽器分頁名稱
- 書籤名稱
- 搜尋結果標題的參考資訊
- 瀏覽器歷史紀錄中

注意：`<title>` 不等於頁面正文中的大標題。
頁面中真正顯示給使用者看的主標題，通常會寫在 `<body>` 裡，並使用 `<h1>`：

```html
<body>
  <h1>網頁主標題</h1>
</body>
```

### `<body>`

`<body>` 主要放實際顯示在頁面上的內容。

例如：

- 標題
- 段落
- 圖片
- 清單
- 連結
- 表格
- 表單
- 按鈕
- 頁首、導覽、主內容、頁尾等版面區塊

基本寫法：

```html
<body>
  <h1>我的第一個網頁</h1>
  <p>這是一段內容。</p>
</body>
```

這裡可以先簡單記成：

- `<head>` 偏設定與資訊。
- `<body>` 偏可見內容與互動入口。

## `<head>` 和 `<body>` 的分工表

| 內容 | 應該放哪裡 | 原因 |
|---|---|---|
| 字元編碼 | `<head>` | 這是瀏覽器讀文件時需要的設定 |
| 分頁標題 | `<head>` | 它是頁面資訊，不是正文內容 |
| CSS 連結 | `<head>` | 讓瀏覽器載入樣式資源 |
| 頁面主要標題 | `<body>` | 它是使用者會看到的正文內容 |
| 段落文字 | `<body>` | 它是頁面實際內容 |
| 圖片 | `<body>` | 通常是頁面可見內容 |
| 表單 | `<body>` | 使用者需要在頁面中操作 |

## 一個最小理解流程

可以先把整件事記成下面這條線：

1. 建立一份 `HTML` 文檔。
2. 用 `<!DOCTYPE html>` 宣告現代文件類型。
3. 用 `<html>` 建立整份文件的根元素。
4. 把頁面設定、描述與資源入口放進 `<head>`。
5. 把實際內容與互動入口放進 `<body>`。
6. 交給瀏覽器讀取、解析與顯示。

## HTML 頁面也是 HTML 文檔

`HTML` 頁面也常被稱為 `HTML` 文檔。

在本地建立與保存時，常見副檔名是：

- `.html`
- `.htm`

它們都可以代表 `HTML` 文件。
現代新檔案通常使用 `.html`。

你可以先這樣理解：

- 檔案本身是 `HTML` 文檔。
- 瀏覽器讀取這份文檔後，會把它顯示成網頁。
- 真實網站中，瀏覽器也可能透過 URL 向伺服器取得 HTML，而不是只打開本地檔案。

## 瀏覽器會容錯，但不要依賴容錯

瀏覽器很擅長修正不完整或不標準的 `HTML`。

例如，有時候即使你省略 `<html>`、`<head>` 或 `<body>`，瀏覽器仍可能自動補出類似結構，讓頁面看起來可以顯示。

但學習與實務上不建議依賴這種容錯，原因是：

- 你會比較難判斷自己的結構到底對不對。
- 不同瀏覽器或工具可能有不同處理細節。
- 後續學 CSS、JavaScript、SEO、可存取性時，結構不清楚會增加排查成本。
- 團隊合作時，明確骨架比依賴瀏覽器猜測更容易維護。

所以入門時要養成習慣：**新頁面先寫完整骨架，再放內容**。

## 常見錯誤示例

### 錯誤：把可見內容放進 `<head>`

```html
<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <title>錯誤示例</title>
    <h1>這不應該放在 head 裡</h1>
  </head>
  <body>
  </body>
</html>
```

`<h1>` 是頁面正文內容，應該放在 `<body>` 裡。

### 正確：把頁面資訊和正文內容分開

```html
<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>正確示例</title>
  </head>
  <body>
    <h1>這是頁面主標題</h1>
    <p>這是頁面正文內容。</p>
  </body>
</html>
```

這樣的結構更清楚，也更適合後續加入 CSS、JavaScript 與其他頁面內容。

## 常見混淆點

### 骨架標籤不是裝飾

它們不是可有可無的外觀配件，而是頁面成立與被理解的基本框架。

### `<title>` 不等於頁面內文標題

`<title>` 主要是分頁標題；真正顯示在頁面正文中的大標題，通常使用 `<h1>`。

### `<head>` 不等於「頁首畫面」

很多新手會把 `<head>` 誤解成畫面最上方那一塊。
其實 `<head>` 指的是文件資訊區，不是畫面版面上的頁首視覺區塊。

畫面上的頁首區塊，後面通常會用 `<header>` 這類語意化標籤表示，而且它通常放在 `<body>` 裡。

### `<body>` 不等於只能放文字

`<body>` 可以放標題、段落、圖片、連結、表單、按鈕、表格、語意化區塊與互動內容。
它代表的是頁面主要內容區，而不是只能放純文字。

### 不是只要能顯示就代表結構正確

瀏覽器會幫你容錯，但學習時還是要養成正確的骨架習慣，後面才不容易亂。

### `DOCTYPE` 不是 HTML 標籤

`<!DOCTYPE html>` 是文件類型宣告，不是一般元素標籤，不需要寫成成對標籤。

### `meta charset` 和 `viewport` 都不會直接顯示

它們放在 `<head>` 裡，主要影響瀏覽器如何讀取文字與如何在不同裝置上顯示頁面。

## 小練習

請判斷下面內容應該放在 `<head>` 還是 `<body>`：

| 內容 | 答案 |
|---|---|
| `<meta charset="UTF-8">` | `<head>` |
| `<title>我的網站</title>` | `<head>` |
| `<h1>首頁</h1>` | `<body>` |
| `<p>歡迎來到我的網站</p>` | `<body>` |
| `<meta name="viewport" content="width=device-width, initial-scale=1.0">` | `<head>` |
| `<img src="photo.jpg" alt="照片">` | `<body>` |

## 一句話抓核心

- **`HTML` 頁面先用 `DOCTYPE`、`html`、`head`、`body` 建立基本骨架，再把頁面設定放進 `<head>`、把可見內容放進 `<body>`，這樣瀏覽器和其他工具才能穩定理解與呈現頁面。**
