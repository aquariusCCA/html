# HTML骨架

> 所屬章節：第二章｜HTML 簡介  
> 關鍵字：HTML骨架、基本結構、`<!DOCTYPE html>`、`<html>`、`lang`、`<head>`、`<meta charset>`、`<title>`、`<body>`、HTML文檔、`.html`、`.htm`  
> 建議回查情境：想知道一個 `HTML` 頁面最基本要有哪些結構、想分清 `head`、`title`、`body` 的角色、想確認 `HTML` 檔案與瀏覽器的關係

## 本節導讀

知道 `HTML` 是用來描述網頁之後，下一步就要理解：**一個完整頁面不是把內容隨便堆上去，而是有固定的基本外框**。  
這個外框，就是我們常說的 `HTML` 骨架。

## 你會在這篇學到什麼

- 為什麼頁面需要固定結構
- 一個基本 `HTML` 文件通常包含哪些部分
- `<!DOCTYPE html>`、`<html>`、`<head>`、`<body>` 各自的角色
- `HTML` 文檔副檔名與瀏覽器之間的基本關係

## 30 秒複習入口

- 每個 `HTML` 頁面都有最基本的骨架。
- 現代常見骨架通常會看到 `<!DOCTYPE html>`、`<html>`、`<head>`、`<meta charset>`、`<title>`、`<body>`。
- `<head>` 主要放頁面設定與資訊，`<body>` 主要放頁面可見內容。
- `HTML` 文檔常見副檔名是 `.html` 或 `.htm`。

## 先講結論

你可以先把一份最基本的 `HTML` 頁面想成三層：

1. 先告訴瀏覽器這是什麼類型的文件。
2. 再用最外層標籤把整份頁面包起來。
3. 然後把「頁面資訊」和「頁面內容」分開放。

這樣一來，頁面才有清楚、可解析的結構。

## 這篇在解決什麼問題？

初學 `HTML` 時，很多人會先注意到標題、段落、圖片、連結這些內容標籤，卻忽略：

- 一個完整頁面本身也有固定結構。
- 不是所有東西都應該寫在同一層。
- 有些資訊是給瀏覽器、搜尋引擎或系統看的，不一定直接顯示在頁面上。

這篇的目標，就是先建立「**頁面先有骨架，再把內容放進去**」的基本觀念。

## 網頁為什麼需要骨架？

你可以把網頁類比成一篇文章。

- 文章通常會有標題、內文、章節等結構。
- 網頁也一樣，需要基本外框來告訴瀏覽器：哪裡是整份文件、哪裡是頁面資訊、哪裡是實際顯示內容。

如果沒有這種基本結構，瀏覽器雖然有時仍會嘗試幫你修正，但你就很難穩定地理解與控制頁面。

## HTML 用標籤把結構寫出來

`HTML` 不只是在描述「有什麼內容」，也在描述「這些內容位在什麼結構位置」。  
因此，一個完整頁面通常會先有一組最基本的結構標籤，再把實際內容放進去。

## 現代最常見的基本骨架

```html
<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <title>網頁標題</title>
  </head>
  <body>
    頁面內容寫在這裡
  </body>
</html>
```

這段先不用急著背，你只要先知道每個部分在做什麼。

## 骨架各部分怎麼理解？

### `<!DOCTYPE html>`

- 用來宣告這是一份 `HTML5` 文件。
- 它不是一般元素標籤，但它會影響瀏覽器用什麼模式解析文件。
- 入門時先記成：**這份文件是現代常見的 HTML 文件**。

### `<html>`

- 這是整份頁面的根元素。
- 可以把它理解成整個文件最外層的外框。

#### `lang` 屬性

- 通常寫在 `<html>` 上，例如 `lang="zh-Hant"`。
- 用來標示頁面主要語言。
- 這有助於瀏覽器、搜尋引擎、翻譯工具與輔助技術理解頁面。

### `<head>`

- 主要放**頁面資訊、設定與外部資源連結**。
- 這裡的內容不一定直接顯示在頁面主畫面上。
- 例如字元編碼、頁面標題、樣式表連結、描述資訊等，通常都放在這裡。

初學時先抓一個核心觀念：

- `<head>` 不是主要可見內容區。
- 它更像是頁面的設定區與說明區。

### `<meta charset="UTF-8">`

- 用來宣告文件採用的文字編碼。
- 常見寫法是 `UTF-8`。
- 這有助於避免中文或其他文字顯示成亂碼。

### `<title>`

- 用來定義頁面標題。
- 它通常會顯示在瀏覽器分頁名稱上。
- 它屬於頁面資訊的一部分，不等於頁面內文標題。

### `<body>`

- 主要放實際顯示在頁面上的內容。
- 例如標題、段落、圖片、清單、連結、表格等，通常都寫在這裡。

這裡可以先簡單記成：

- `<head>` 偏設定與資訊。
- `<body>` 偏可見內容。

## 一個最小理解流程

可以先把整件事記成下面這條線：

1. 建立一份 `HTML` 文檔。
2. 用骨架標籤建立頁面的基本外框。
3. 把頁面設定放進 `<head>`。
4. 把實際內容放進 `<body>`。
5. 交給瀏覽器讀取與顯示。

## HTML 頁面也是 HTML 文檔

- `HTML` 頁面也常被稱為 `HTML` 文檔。
- 在本地建立與保存時，常見副檔名是 `.html` 或 `.htm`。
- 瀏覽器會讀取這類文件，再把它顯示成網頁。

## 常見混淆點

### 骨架標籤不是裝飾

它們不是可有可無的外觀配件，而是頁面成立的基本框架。

### `<title>` 不等於頁面內文標題

`<title>` 主要是分頁標題；真正顯示在頁面正文中的大標題，後面通常會用像 `<h1>` 這類標籤來表示。

### `<head>` 不等於「頁首畫面」

很多新手會把 `<head>` 誤解成畫面最上方那一塊。  
其實 `<head>` 指的是文件資訊區，不是畫面版面上的頁首視覺區塊。

### 不是只要能顯示就代表結構正確

瀏覽器會幫你容錯，但學習時還是要養成正確的骨架習慣，後面才不容易亂。

## 一句話抓核心

- **`HTML` 頁面先有固定骨架，再把頁面資訊放進 `<head>`、把實際內容放進 `<body>`，這樣瀏覽器才能穩定地解析與顯示。**
