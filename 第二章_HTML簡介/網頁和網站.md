# 網頁和網站

> 所屬章節：第二章｜HTML 簡介  
> 關鍵字：網頁、網站、HTML、超文本標記語言、超文本、標記語言、瀏覽器、`.html`、`.htm`、Web page、Website  
> 建議回查情境：想分清網頁和網站的差別、想知道 HTML 到底是什麼、想理解瀏覽器如何把程式碼變成頁面

## 本節導讀

這一節是學 `HTML` 前最基礎的名詞整理。  
如果你常把「網頁」「網站」「HTML」當成同一件事，這篇就是先把它們拆開，再用一條簡單流程把它們接回來。

## 你會在這篇學到什麼

- 網頁和網站的基本差異
- HTML 的基本定義
- 為什麼 HTML 叫做超文本標記語言
- 網頁如何經過瀏覽器解析並呈現

## 正文筆記

### 這篇在解決什麼問題？

- 初學前端時，最容易混在一起的三個詞就是「網頁」「網站」和 `HTML`。
- 如果一開始沒有分清楚，之後看到瀏覽器、標籤、頁面結構時就容易越學越亂。
- 這篇的目標，是先建立後續學 `HTML` 的最小共同語言。

### 先講結論

- 網頁是網站中的單一頁面，通常以 `HTML` 文件為基礎。
- 網站是多個彼此相關的網頁和資源所組成的整體。
- `HTML` 是描述網頁內容與結構的標記語言，瀏覽器會把它解析後顯示成我們看到的頁面。

### 什麼是網頁？

- 網頁是網站中的一個頁面，通常需要透過瀏覽器閱讀。
- 它是構成網站的基本元素，常見內容包含文字、圖片、音頻、視頻與超連結。
- 實務上，很多網頁會對應到副檔名為 `.htm` 或 `.html` 的文件，因此入門教材常把它們直接稱為 `HTML` 文件。

### 什麼是網站？

- 網站不是單一一頁，而是一組互相關聯的網頁與資源。
- 這些頁面通常圍繞同一個主題、品牌、服務或功能組成，例如首頁、產品頁、文章頁、登入頁等。
- 可以把網站理解成一本書，而網頁就是書中的其中一頁。

### 網頁和網站差在哪裡？

- 網頁是單一頁面；網站是多個頁面的集合。
- 網頁偏向一個具體文件或畫面；網站偏向完整的內容或服務系統。
- 當你打開某個網站時，實際上通常是在瀏覽這個網站裡的一個或多個網頁。

### 什麼是 HTML？

- `HTML` 是 `HyperText Markup Language`，中文通常譯為「超文本標記語言」。
- 它的主要用途，是描述網頁中的內容與結構，例如標題、段落、圖片、連結、清單等。
- `HTML` 不是程式語言，而是標記語言；也就是說，它不是拿來寫計算邏輯，而是拿來標示內容該怎麼被組織與呈現。

### 為什麼叫做超文本標記語言？

可以拆成三個部分理解：

#### 1. HyperText：超文本

- 它不只包含純文字，也可以整合圖片、音頻、視頻、動畫等多媒體內容。
- 它還可以透過超連結，從一個文件跳到另一個文件，讓不同內容彼此連接。

#### 2. Markup：標記

- `HTML` 會使用一組標籤來標示內容的意義與位置。
- 例如某段文字是標題、某段內容是段落、某個元素是圖片或連結，都可以用標籤描述。

#### 3. Language：語言

- 它有自己的語法與規則，讓瀏覽器知道該如何解析內容。

### 網頁是怎麼形成的？

網頁可以先用下面這條流程理解：

1. 前端人員撰寫 `HTML` 等網頁程式碼。
2. 瀏覽器讀取這些程式碼。
3. 瀏覽器對內容進行解析與渲染。
4. 最終顯示成使用者看到的 Web 頁面。

所以可以把原本那句話整理成更完整的理解：

- 網頁是由各種網頁元素組成，而這些元素會先由 `HTML` 標籤描述，再交給瀏覽器解析後顯示出來。

### 常見混淆點

- 網頁不等於網站；網頁是一頁，網站是一組頁面。
- `HTML` 不等於瀏覽器；`HTML` 負責描述內容，瀏覽器負責解析與顯示。
- `HTML` 不等於程式語言；它主要負責結構標記，而不是邏輯運算。

## 一句話抓核心

- 網頁是網站中的單一頁面，網站是多個網頁的集合，而 `HTML` 則是用來描述這些網頁內容與結構的標記語言。
