# 網頁和網站

> 所屬章節：第二章｜HTML 簡介
> 關鍵字：網頁、網站、HTML、超文本標記語言、超文本、標記語言、瀏覽器、URL、`.html`、`.htm`、Web page、Website、Web App
> 建議回查情境：想分清網頁和網站的差別、想知道 `HTML` 到底是什麼、想理解瀏覽器如何把程式碼變成頁面、想釐清網頁檔案與網站之間的關係

## 本節導讀

這一節先處理學 `HTML` 前最容易混在一起的幾個名詞：**網頁、網站、HTML、瀏覽器**。
如果這四個角色一開始沒有分清楚，後面在學標籤、頁面結構、CSS、JavaScript 與網站部署時，就很容易沒有主線。

第一次讀這篇時，建議先抓住「誰負責什麼」：

- `HTML` 負責描述內容與結構。
- 瀏覽器負責讀取、解析與顯示。
- 網頁是使用者看到或操作的單一頁面。
- 網站是圍繞同一個主題、服務或功能組成的整體。

## 你會在這篇學到什麼

- 網頁和網站的基本差異
- `HTML` 的基本定義與責任邊界
- 為什麼 `HTML` 叫做超文本標記語言
- `.html`、`.htm`、URL、瀏覽器和網頁之間的關係
- 瀏覽器如何把 `HTML` 解析並呈現成頁面
- 單頁網站、Web App 這類現代情境要怎麼理解

## 30 秒複習入口

- **網頁**是使用者透過瀏覽器看到或操作的單一頁面，也可以理解成網站中的一個具體畫面或文件。
- **網站**是一組彼此相關的網頁、資源與功能所形成的整體。
- `HTML` 是用來描述網頁**內容、結構與語意**的標記語言。
- 瀏覽器會讀取 `HTML`，解析成可以操作與顯示的結構，再把它渲染成畫面。
- `.html` 和 `.htm` 都是常見的 `HTML` 文件副檔名；現代新檔案通常使用 `.html`。

## 先講結論

學這一節時，可以先記住這條主線：

1. 開發者用 `HTML` 描述網頁的內容與結構。
2. 瀏覽器取得 `HTML`，並把它解析成頁面結構。
3. 使用者在瀏覽器中看到或操作單一**網頁**。
4. 多個相關網頁、資源與功能組合起來，就形成**網站**。

只要這條主線沒有亂，後面學標籤、屬性、CSS、JavaScript、連結、圖片、表單與部署時，就不容易把概念混成一團。

## 這篇在解決什麼問題？

初學者常會把下面幾件事混在一起：

- 把「網站」和「網頁」當成完全相同的詞。
- 把 `HTML` 和瀏覽器當成同一種東西。
- 以為 `HTML` 是負責運算邏輯的程式語言。
- 以為網頁一定等於一個本地 `.html` 檔案。
- 看到單頁網站或 Web App 時，不知道它們和「多個網頁組成網站」的說法怎麼對上。

這篇的目的，就是先把這些概念拆開，再重新接成一條清楚流程。

## 核心角色總覽

| 名詞 | 可以先怎麼理解 | 主要責任 | 常見例子 |
|---|---|---|---|
| 網頁 | 單一頁面或單一畫面 | 承載某一頁的內容、結構與互動入口 | 首頁、文章頁、商品頁、登入頁 |
| 網站 | 一組相關網頁、資源與功能 | 提供完整內容、服務或產品體驗 | 部落格、公司官網、購物網站、課程平台 |
| `HTML` | 描述網頁結構的標記語言 | 標示標題、段落、圖片、連結、表單等內容角色 | `index.html`、文章頁 HTML |
| 瀏覽器 | 讀取並呈現網頁的工具 | 解析 HTML、套用 CSS、執行 JavaScript、顯示結果 | Chrome、Safari、Firefox、Edge |
| URL | 網路資源的位置 | 告訴瀏覽器要取得哪個資源 | `https://example.com/about` |

## 什麼是網頁？

網頁可以先理解成：**使用者透過瀏覽器看到或操作的單一頁面**。

一個網頁裡可以包含：

- 文字
- 圖片
- 音訊
- 影片
- 超連結
- 表單
- 按鈕與互動區塊

如果把網站想成一本書，那麼：

- **網站**像整本書。
- **網頁**像書裡的其中一頁。

如果把網站想成一間店，那麼：

- **網站**像整間店。
- **網頁**像店裡的不同區域，例如首頁、商品介紹、購物車、結帳頁。

### 網頁一定是一個 `.html` 檔案嗎？

入門時，常會把網頁和 `.html` / `.htm` 文件連在一起講，這樣理解沒有問題。
因為很多最基本的靜態網頁，確實就是由一份 `HTML` 文件作為基礎，再交給瀏覽器顯示。

但更精確地說：

- 在本地練習時，一個網頁常常對應一個 `.html` 檔案。
- 在真實網站中，一個網頁可能由伺服器動態產生。
- 在現代 Web App 中，畫面可能由 JavaScript 根據路由與資料動態切換，不一定每個畫面都有一個獨立的 `.html` 檔案。

所以入門階段可以先記：

- **學習 HTML 時，網頁常從 `.html` 檔案開始。**
- **實務網站中，網頁不一定永遠只是一個靜態檔案。**

## 什麼是網站？

網站不是單一頁面，而是**一組彼此相關的網頁、資源與功能**。

這些內容通常圍繞同一個：

- 主題
- 品牌
- 服務
- 產品
- 社群
- 功能系統

常見的網站頁面可能包括：

- 首頁
- 關於我們
- 產品頁
- 文章頁
- 登入頁
- 會員頁
- 購物車
- 聯絡頁

所以當你說「我在看某個網站」，實際上你通常是在看那個網站中的某一個或多個網頁。

## 網頁和網站差在哪裡？

可以先用最簡單的方式區分：

- 網頁：單一頁面。
- 網站：多個頁面、資源與功能形成的整體。

再精確一點說：

- 網頁偏向一個具體畫面、文件或路由。
- 網站偏向一整套內容、服務或功能系統。

### 對照例子

| 情境 | 比較像網頁 | 比較像網站 |
|---|---|---|
| 部落格 | 某一篇文章頁 | 整個部落格 |
| 公司官網 | 關於我們頁 | 公司完整官網 |
| 購物平台 | 某個商品頁 | 整個購物平台 |
| 課程平台 | 某一堂課的頁面 | 整個課程網站 |

### 單頁網站怎麼理解？

有些網站可能只有一個主要頁面，例如活動頁、作品集頁、產品介紹頁。
這種情況仍然可以叫網站，因為它代表的是一個完整的主題或服務入口。

也就是說：

- **網站不一定必須很多頁。**
- 但只要它作為一個完整主題、品牌、服務或功能入口，就可以被稱為網站。

前面說「網站由多個網頁組成」是入門時最常見、最容易理解的情況；遇到單頁網站時，可以把它理解成「網站也可能只有一個主要網頁」。

## 什麼是 HTML？

`HTML` 全名是 `HyperText Markup Language`。
中文通常譯為「超文本標記語言」。

它的主要用途，是描述網頁中的：

- **內容**：頁面有哪些文字、圖片、連結、清單、表格、表單等。
- **結構**：哪些內容是標題、段落、列表項目、表格列、表單欄位等。
- **語意**：這段內容在頁面中扮演什麼角色。

例如，`HTML` 可以標示：

- 哪裡是標題
- 哪裡是段落
- 哪裡是圖片
- 哪裡是連結
- 哪裡是清單
- 哪裡是表單輸入欄位

### 一個最小例子

```html
<h1>我的第一個網頁</h1>
<p>這是一段介紹文字。</p>
<a href="https://example.com">前往範例網站</a>
```

在這段裡：

- `<h1>` 表示這是主要標題。
- `<p>` 表示這是一個段落。
- `<a>` 表示這是一個超連結。

`HTML` 的價值不只是把文字放到頁面上，而是讓瀏覽器知道「這些內容各自是什麼」。

## HTML、CSS、JavaScript 的責任邊界

初學時可以先用這個分工理解：

| 技術 | 主要負責 | 例子 |
|---|---|---|
| `HTML` | 內容與結構 | 標題、段落、圖片、連結、表單 |
| `CSS` | 外觀與排版 | 顏色、字體、間距、版面、響應式 |
| `JavaScript` | 互動與邏輯 | 按鈕事件、資料更新、表單驗證、動態畫面 |

### 一個很重要的觀念

`HTML` **不是拿來做運算邏輯的主角**。
它的主要工作不是判斷、迴圈、資料處理，而是把內容標示清楚，讓瀏覽器知道這些內容的結構與意義。

所以入門時可以先把它理解成：

- `HTML` 負責**描述**。
- `CSS` 負責**呈現**。
- `JavaScript` 負責**互動與邏輯控制**。
- 瀏覽器負責**解析、執行與顯示**。

這裡先建立方向即可，細節後面再展開。

## 為什麼叫做「超文本標記語言」？

這個名字可以拆成三部分理解。

### 1. HyperText：超文本

「超文本」不是只有純文字，而是可以把不同類型的內容與跳轉關係整合在一起。

例如：

- 文字
- 圖片
- 音訊
- 影片
- 超連結
- 表單與其他可互動內容

其中**超連結**尤其重要。
它讓使用者可以從一個頁面跳到另一個頁面，也讓不同網頁彼此連成網狀結構，這是 Web 很核心的特性。

### 2. Markup：標記

`HTML` 會用一組標籤來標示內容的角色與結構。

例如：

- 某段文字是標題
- 某段內容是段落
- 某個元素是圖片
- 某個位置是一個連結
- 某一組內容是一個清單

也就是說，`HTML` 的重點不是只寫「內容」，而是**幫內容加上結構與語意上的標記**。

### 3. Language：語言

`HTML` 有自己的語法與規則。
瀏覽器會依照這套規則去讀取與解析內容，所以它被稱為一種語言。

但要注意：`HTML` 是**標記語言**，不是一般用來寫運算流程的程式語言。
它描述頁面內容是什麼，而不是主要負責計算程式要怎麼執行。

## `.html`、`.htm` 和瀏覽器的關係

`HTML` 文件常見副檔名有兩種：

- `.html`
- `.htm`

它們都可以代表 `HTML` 文件。
現代新檔案通常使用 `.html`，`.htm` 則常見於較早期系統或某些舊習慣。

入門時不用把這件事想得太複雜，只要記住：

- 副檔名可以幫助系統與編輯器辨識這是一份 `HTML` 文件。
- 瀏覽器可以讀取 `HTML` 文件，並把它顯示成網頁。
- 真實網站中，瀏覽器也可能透過 URL 向伺服器取得 HTML，而不是只打開本地檔案。

## URL、伺服器與網頁

當你在瀏覽器輸入網址時，例如：

```text
https://example.com/about
```

可以先粗略理解成：

1. 瀏覽器根據 URL 找到目標網站或服務。
2. 伺服器回傳對應的內容，常見起點是 `HTML`。
3. 瀏覽器讀取 `HTML`，並繼續載入 CSS、JavaScript、圖片等資源。
4. 最後顯示成使用者看到的網頁。

這個流程先抓大方向即可。
後面學到 HTTP、伺服器、前後端互動時，會再把細節補完整。

## 網頁是怎麼形成的？

可以先用這條最小流程理解：

1. 開發者撰寫 `HTML`、`CSS`、`JavaScript` 等網頁程式碼。
2. 使用者用瀏覽器打開本地檔案，或透過 URL 請求網站。
3. 瀏覽器取得 `HTML`。
4. 瀏覽器解析 `HTML`，建立頁面結構。
5. 瀏覽器套用 CSS、載入圖片等資源，必要時執行 JavaScript。
6. 最終渲染成使用者看到或操作的頁面。

這裡先抓大方向即可，不需要一開始就深入 DOM、CSSOM、Render Tree 等細節。

## 用一句話把它們串起來

- `HTML` 負責描述網頁內容、結構與語意。
- 瀏覽器負責讀取、解析、執行相關資源並顯示內容。
- 使用者看到或操作的是網頁。
- 多個相關網頁、資源與功能，或一個完整的單頁入口，就形成網站。

## 常見混淆點

### 網頁不等於網站

- 網頁偏單一頁面或單一畫面。
- 網站偏一整套內容、服務或功能。

### 網站不一定永遠有很多頁

入門時說「網站由多個網頁組成」很常見，也很有幫助。
但實務上也有單頁網站或單頁應用程式，所以更精確的理解是：網站是一個完整主題、品牌、服務或功能的入口與整體。

### HTML 不等於瀏覽器

- `HTML` 是描述內容的標記語言。
- 瀏覽器是讀取與顯示內容的工具。

### HTML 不等於一般意義上的程式語言

- `HTML` 主要負責結構與語意標記。
- 它不是用來處理複雜邏輯運算的主角。

### 網頁不一定只來自本地 `.html` 檔案

- 本地練習時，常會直接開啟 `.html` 檔案。
- 真實網站中，瀏覽器通常會透過 URL 向伺服器取得內容。
- 現代 Web App 中，有些畫面會由 JavaScript 動態產生或切換。

### 能顯示出來不代表結構就一定好

瀏覽器有很強的容錯能力，就算 `HTML` 寫得不完整，有時也能顯示畫面。
但學習時仍然要養成正確描述結構與語意的習慣，後面學 CSS、JavaScript、SEO、可存取性時才不容易卡住。

## 小練習

請判斷下面情境比較像「網頁」還是「網站」：

1. 某個購物平台中的一個商品介紹頁。
2. 整個購物平台。
3. 公司官網的聯絡我們頁。
4. 一個只有單頁的活動介紹頁。

參考答案：

1. 網頁。
2. 網站。
3. 網頁。
4. 可以是網頁，也可以作為一個單頁網站；判斷重點在於它是否承載完整主題或服務入口。
