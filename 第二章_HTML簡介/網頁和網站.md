# 網頁和網站

> 所屬章節：第二章｜HTML 簡介  
> 關鍵字：網頁、網站、HTML、超文本標記語言、超文本、標記語言、瀏覽器、`.html`、`.htm`、Web page、Website  
> 建議回查情境：想分清網頁和網站的差別、想知道 `HTML` 到底是什麼、想理解瀏覽器如何把程式碼變成頁面

## 本節導讀

這一節先處理學 `HTML` 前最容易混在一起的幾個名詞：**網頁、網站、HTML、瀏覽器**。  
如果這四個角色一開始沒有分清楚，後面在學標籤、頁面結構與 CSS 時，就很容易沒有主線。

## 你會在這篇學到什麼

- 網頁和網站的基本差異
- `HTML` 的基本定義
- 為什麼 `HTML` 叫做超文本標記語言
- 瀏覽器如何把 `HTML` 解析並呈現成頁面

## 30 秒複習入口

- **網頁**是網站中的單一頁面。
- **網站**是多個彼此相關的網頁與資源組成的整體。
- `HTML` 是用來描述網頁**內容與結構**的標記語言。
- 瀏覽器會讀取 `HTML`，再把它解析並顯示成我們看到的畫面。

## 先講結論

學這一節時，可以先記住一句話：

- **網站**由很多**網頁**組成；
- **網頁**通常以 `HTML` 為基礎；
- **瀏覽器**負責把 `HTML` 解析後顯示成畫面。

只要這條主線沒有亂，後面很多概念就不容易混淆。

## 這篇在解決什麼問題？

初學者常會把下面幾件事混成一團：

- 把「網站」和「網頁」當成同一個詞。
- 把 `HTML` 和瀏覽器當成同一種東西。
- 以為 `HTML` 是負責運算邏輯的程式語言。

這篇的目的，就是先把這幾個概念拆開，再重新接成一條清楚流程。

## 什麼是網頁？

- 網頁是網站中的**單一頁面**。
- 它通常需要透過瀏覽器閱讀。
- 一個網頁裡可以包含文字、圖片、音訊、影片、超連結等內容。

如果把網站想成一本書，那麼：

- **網站**像整本書。
- **網頁**像書裡的其中一頁。

### 補充理解

在入門教材裡，常會把網頁和 `.html` / `.htm` 文件連在一起講，這樣理解沒有問題。  
因為很多最基本的網頁，確實是由 `HTML` 文件作為基礎，再交給瀏覽器顯示。

## 什麼是網站？

- 網站不是單一頁面，而是**一組彼此相關的網頁與資源**。
- 這些頁面通常圍繞同一個主題、品牌、服務或功能。
- 常見的網站頁面可能包括首頁、產品頁、文章頁、登入頁、會員頁等。

所以當你說「我在看某個網站」，實際上你通常是在看那個網站中的某一個或多個網頁。

## 網頁和網站差在哪裡？

可以先用最簡單的方式區分：

- 網頁：單一頁面。
- 網站：多個頁面的集合。

再精確一點說：

- 網頁偏向一個具體畫面或文件。
- 網站偏向一整套內容、服務或功能系統。

## 什麼是 HTML？

- `HTML` 全名是 `HyperText Markup Language`。
- 中文通常譯為「超文本標記語言」。
- 它的主要用途，是描述網頁中的**內容**與**結構**。

例如，`HTML` 可以標示：

- 哪裡是標題
- 哪裡是段落
- 哪裡是圖片
- 哪裡是連結
- 哪裡是清單

### 一個很重要的觀念

`HTML` **不是拿來做運算邏輯的主角**。  
它的主要工作不是判斷、迴圈、資料處理，而是把內容標示清楚，讓瀏覽器知道這些內容的結構與意義。

所以入門時可以先把它理解成：

- `HTML` 負責**描述**。
- 瀏覽器負責**解析與顯示**。
- `JavaScript` 才比較偏向**互動與邏輯控制**。

這裡先建立方向即可，細節後面再展開。

## 為什麼叫做「超文本標記語言」？

這個名字可以拆成三部分理解。

### 1. HyperText：超文本

「超文本」不是只有純文字，而是可以把不同類型的內容與跳轉關係整合在一起。  
例如：

- 文字
- 圖片
- 音訊
- 影片
- 超連結

尤其是**超連結**這個特性，讓你可以從一個頁面跳到另一個頁面，這也是 Web 很重要的基礎。

### 2. Markup：標記

`HTML` 會用一組標籤來標示內容的角色與結構。  
例如：

- 某段文字是標題
- 某段內容是段落
- 某個元素是圖片
- 某個位置是一個連結

也就是說，`HTML` 的重點不是只寫「內容」，而是**幫內容加上結構與語意上的標記**。

### 3. Language：語言

`HTML` 有自己的語法與規則。  
瀏覽器會依照這套規則去讀取與解析內容，所以它被稱為一種語言。

## 網頁是怎麼形成的？

可以先用這條最小流程理解：

1. 開發者撰寫 `HTML` 等網頁程式碼。
2. 瀏覽器讀取這些內容。
3. 瀏覽器解析結構並進行渲染。
4. 最終顯示成使用者看到的頁面。

這裡先抓大方向即可，不需要一開始就深入 DOM、CSSOM、Render Tree 等細節。

## 用一句話把它們串起來

- `HTML` 負責描述網頁內容與結構。
- 瀏覽器負責讀取並顯示這些內容。
- 多個網頁再加上相關資源與功能，就形成網站。

## 常見混淆點

### 網頁不等於網站

- 網頁是一頁。
- 網站是一組頁面。

### HTML 不等於瀏覽器

- `HTML` 是描述內容的語言。
- 瀏覽器是讀取與顯示內容的工具。

### HTML 不等於一般意義上的程式語言

- `HTML` 主要負責結構標記。
- 它不是用來處理複雜邏輯運算的主角。

### `.html` 和 `.htm` 不用先想得太複雜

在入門階段，可以先把它們都理解成 `HTML` 文檔常見的副檔名。  
重點是知道：瀏覽器會讀取這類文件並把內容顯示出來。

## 一句話抓核心

- **網頁是網站中的單一頁面，網站是多個網頁的集合，而 `HTML` 是用來描述這些網頁內容與結構的標記語言。**
