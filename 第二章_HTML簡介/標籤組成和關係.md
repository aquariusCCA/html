# 標籤組成和關係

> 所屬章節：第二章｜HTML 簡介  
> 關鍵字：HTML標籤、標籤結構、起始標籤、結束標籤、雙標籤、單標籤、空元素、HTML元素、巢狀、嵌套、並列、父元素、子元素、祖先元素、後代元素、兄弟元素  
> 建議回查情境：想分清標籤和元素的基本結構、想理解雙標籤與單標籤差別、想看懂父子兄弟這些 `HTML` 關係名稱

## 本節導讀

學 `HTML` 時，你很快就會看到大量尖括號，也很快會碰到「父元素」「子元素」「兄弟元素」這些說法。  
這一節的目的，就是先把**標籤怎麼組成**、**元素之間怎麼形成關係**這兩件事講清楚。

## 你會在這篇學到什麼

- `HTML` 標籤最基本的組成方式
- 什麼是起始標籤與結束標籤
- 什麼是雙標籤與單標籤
- 什麼是嵌套關係與並列關係
- 父元素、子元素、祖先元素、後代元素、兄弟元素各自代表什麼

## 30 秒複習入口

- 多數 `HTML` 標籤是成對出現的。
- 成對標籤通常由起始標籤與結束標籤組成。
- 有些標籤不包內容，常被稱為單標籤；更嚴謹地說，也常稱為空元素。
- 當元素包住其他元素時，會形成嵌套關係。
- 當元素位在同一層且共享同一個父元素時，會形成兄弟關係。

## 先講結論

這一節最重要的不是背一堆術語，而是建立下面兩個觀念：

1. 標籤有基本開合結構。
2. 一旦標籤開始互相包住或並列，就會形成層級關係。

只要這兩點懂了，後面學 CSS 選擇器與 DOM 結構就會順很多。

## 這篇在解決什麼問題？

初學 `HTML` 時，常見困惑包括：

- 看得到很多標籤，卻不知道每一部分叫什麼。
- 看到巢狀結構時，分不清誰是父、誰是子、誰是兄弟。
- 把「標籤」和「元素」完全當成同一件事，之後在討論結構時容易模糊。

這篇的目標，就是先把這套描述結構的基本語言建立起來。

## HTML 標籤的基本結構

![標籤的結構圖.png](./images/image_1776049915818_798e28e8.png)

### 標籤名

被 `< >` 包起來的英文單詞或字母，通常就是標籤名。  
例如：

- `html`
- `body`
- `p`
- `div`

### 起始標籤與結束標籤

以這段為例：

```html
<p>這是一段文字</p>
```

可以這樣拆：

- `<p>`：起始標籤
- `</p>`：結束標籤
- 中間的文字：元素內容

### 雙標籤

由起始標籤和結束標籤成對組成的，通常可稱為雙標籤。  
像 `<p>...</p>`、`<div>...</div>`、`<title>...</title>` 都屬於這類。

### 單標籤 / 空元素

有些標籤本身不需要包住內容，因此不會再寫一個對應的結束標籤。  
入門教材常把它們叫做**單標籤**；如果想更精確一點，也常稱為**空元素**。

入門時你先知道一件事就夠了：

- 不是所有標籤都一定長成「開始 + 結束」兩段。

## 標籤和元素要怎麼區分？

這兩個詞在入門時常被混用，先這樣理解最實用：

- **標籤**：偏向語法外殼，例如 `<p>`、`</p>`。
- **元素**：偏向完整節點概念，包含標籤與其中的內容或結構。

例如：

```html
<p>這是一段文字</p>
```

- `<p>` 和 `</p>` 是標籤。
- 整個 `<p>這是一段文字</p>` 可以看成一個段落元素。

入門時不需要把這兩個詞卡得太死，但至少要知道它們不完全是同一個焦點。

## 元素之間的兩種基本關係

當頁面開始有多個元素時，最常見可以先分成兩種關係：

- 嵌套關係
- 並列關係

### 1. 嵌套關係

當一個元素包住另一個元素時，就形成嵌套關係。

例如：

```html
<body>
  <div>
    <p>文字</p>
  </div>
</body>
```

在這段裡：

- `body` 包住 `div`
- `div` 包住 `p`

所以這些元素形成了層級結構。

### 2. 並列關係

如果兩個元素位在同一層，並且同時被同一個父元素包住，它們就是並列關係。

例如：

```html
<ul>
  <li>第一項</li>
  <li>第二項</li>
</ul>
```

這裡兩個 `li` 位在同一層，也都被 `ul` 包住，所以它們彼此是並列的。

## 常見的元素關係名稱

### 父元素

直接包住其他元素的那一層，叫父元素。

在下面這段裡：

```html
<div>
  <p>文字</p>
</div>
```

- `div` 是 `p` 的父元素。

### 子元素

被父元素直接包含的那一層，叫子元素。

以上面的例子來看：

- `p` 是 `div` 的子元素。

### 祖先元素

一路往外包住目前元素的所有上層元素，都可以叫祖先元素。

例如：

```html
<body>
  <div>
    <p>文字</p>
  </div>
</body>
```

對 `p` 來說：

- `div` 是祖先元素。
- `body` 也是祖先元素。

其中 `div` 同時還是它的直接父元素。

### 後代元素

某個元素裡面所有更深層的子孫元素，都可以叫後代元素。

以上面的例子來看：

- `div` 和 `p` 都可以算是 `body` 的後代元素。

### 兄弟元素

位在同一層、而且共享同一個父元素的元素，叫兄弟元素。

例如：

```html
<ul>
  <li>第一項</li>
  <li>第二項</li>
</ul>
```

- 兩個 `li` 互為兄弟元素。

## 一張心智圖式的理解方式

你可以先用下面這套判斷順序：

1. 先看兩個元素是不是在同一層。
2. 如果在同一層，再看它們是不是共享同一個父元素。
3. 若是，就可能是兄弟元素。
4. 如果不在同一層，就看其中一個是否包住另一個。
5. 若是直接包住，談父子。
6. 若中間隔了不只一層，談祖先與後代。

## 常見混淆點

### 父元素和祖先元素不一樣

- 父元素：一定是直接上一層。
- 祖先元素：可以是更外層，不一定直接相鄰。

### 子元素和後代元素不一樣

- 子元素：一定是直接下一層。
- 後代元素：可以跨很多層。

### 兄弟元素一定要同層

不是只要都出現在同一份文件裡，就能叫兄弟元素。  
它們必須同層，而且共享同一個父元素。

### 標籤和元素不是完全同義

入門時混用問題不大，但只要開始討論結構，最好慢慢分清：

- 標籤偏語法形式
- 元素偏節點與結構概念

## 一句話抓核心

- **先理解標籤怎麼開和怎麼關，再理解元素之間誰包住誰、誰跟誰同層，你就能看懂大部分 `HTML` 的基本結構。**
