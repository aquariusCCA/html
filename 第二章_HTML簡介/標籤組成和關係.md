# 標籤組成和關係

> 所屬章節：第二章｜HTML 簡介
> 關鍵字：HTML標籤、標籤結構、標籤名、屬性、屬性值、起始標籤、結束標籤、雙標籤、單標籤、空元素、void element、HTML元素、DOM節點、巢狀、嵌套、並列、父元素、子元素、祖先元素、後代元素、兄弟元素、合法嵌套
> 建議回查情境：想分清標籤、元素與屬性的差別、想理解雙標籤與空元素、想判斷父子兄弟關係、想看懂 HTML 層級結構、想避免常見嵌套錯誤

## 本節導讀

學 `HTML` 時，你很快就會看到大量尖括號，也很快會碰到「父元素」「子元素」「兄弟元素」這些說法。
這一節的目的，就是先把**標籤怎麼組成**、**元素之間怎麼形成關係**這兩件事講清楚。

第一次讀這篇時，請先抓住兩條主線：

- 從語法角度看：`HTML` 由標籤、屬性、內容組成。
- 從結構角度看：元素彼此會形成父子、祖先後代與兄弟關係。

這些概念看起來很基礎，但它們會直接影響後面學 CSS 選擇器、JavaScript DOM 操作、語意化結構與可存取性。

## 你會在這篇學到什麼

- `HTML` 標籤最基本的組成方式
- 什麼是標籤名、屬性、屬性值
- 什麼是起始標籤與結束標籤
- 什麼是雙標籤與空元素
- 標籤、元素、節點這幾個詞要怎麼初步區分
- 什麼是巢狀關係與並列關係
- 父元素、子元素、祖先元素、後代元素、兄弟元素各自代表什麼
- 為什麼不是所有標籤都可以任意互相包住

## 30 秒複習入口

- **標籤**偏語法形式，例如 `<p>`、`</p>`。
- **元素**偏完整結構，例如 `<p>這是一段文字</p>`。
- 起始標籤可以帶屬性，例如 `<a href="https://example.com">`。
- 多數元素有起始標籤與結束標籤，但有些元素不能包內容，稱為空元素，例如 `<br>`、`<img>`、`<input>`。
- 元素包住元素時形成巢狀關係；同一層且共享同一個父元素時形成兄弟關係。
- HTML 不是只要能顯示就算結構正確，元素之間還有合法嵌套規則。

## 先講結論

這一節最重要的不是背一堆術語，而是建立下面四個觀念：

1. 標籤有基本語法：標籤名、屬性、起始標籤、結束標籤。
2. 元素是比標籤更完整的結構概念。
3. 元素一旦互相包住或並列，就會形成層級關係。
4. 巢狀不是隨便包，HTML 有自己的內容規則與語意限制。

只要這些觀念穩，後面看到更複雜的頁面結構時，就能知道自己在讀的是「語法外殼」還是「元素關係」。

## 這篇在解決什麼問題？

初學 `HTML` 時，常見困惑包括：

- 看得到很多 `< >`，卻不知道每一部分叫什麼。
- 把「標籤」和「元素」完全當成同一件事。
- 以為每個標籤都一定要成對出現。
- 看到巢狀結構時，分不清誰是父、誰是子、誰是兄弟。
- 以為只要瀏覽器能顯示，標籤怎麼包都可以。

這篇的目標，就是先把這套描述結構的基本語言建立起來。

## HTML 標籤的基本結構

![標籤的結構圖.png](./images/image_1776049915818_798e28e8.png)

先看一個最常見的例子：

```html
<p class="intro">這是一段文字</p>
```

可以拆成：

| 部分 | 名稱 | 說明 |
|---|---|---|
| `<p class="intro">` | 起始標籤 | 元素開始的位置，也可以放屬性 |
| `p` | 標籤名 | 說明這是哪一種 HTML 標籤 |
| `class="intro"` | 屬性 | 補充這個元素的設定或資訊 |
| `這是一段文字` | 元素內容 | 被標籤包住的內容 |
| `</p>` | 結束標籤 | 元素結束的位置 |

## 標籤名

被 `< >` 包起來的英文單詞或字母，通常就是標籤名。

例如：

- `html`
- `head`
- `body`
- `p`
- `a`
- `img`
- `div`

標籤名用來告訴瀏覽器這段內容扮演什麼角色。

例如：

- `<p>` 表示段落。
- `<a>` 表示超連結。
- `<img>` 表示圖片。
- `<ul>` 表示無序清單。
- `<li>` 表示清單項目。

## 起始標籤與結束標籤

以這段為例：

```html
<p>這是一段文字</p>
```

可以這樣拆：

- `<p>`：起始標籤
- `</p>`：結束標籤
- `這是一段文字`：元素內容

結束標籤通常會比起始標籤多一個 `/`：

```html
</p>
```

這個 `/` 表示這個元素到這裡結束。

## 屬性與屬性值

屬性通常寫在**起始標籤**裡，用來補充元素的資訊或設定。

例如：

```html
<a href="https://example.com" title="前往範例網站">範例網站</a>
```

可以拆成：

| 部分 | 說明 |
|---|---|
| `a` | 標籤名 |
| `href="https://example.com"` | 連結目標 |
| `title="前往範例網站"` | 額外提示資訊 |
| `範例網站` | 連結文字 |

再看一個圖片例子：

```html
<img src="photo.jpg" alt="一張課程照片">
```

可以拆成：

| 屬性 | 作用 |
|---|---|
| `src` | 圖片來源 |
| `alt` | 圖片替代文字 |

### 屬性要注意什麼？

- 屬性通常寫在起始標籤裡，不寫在結束標籤裡。
- 多數屬性會使用 `屬性名="屬性值"` 的形式。
- 屬性值建議用引號包起來，這樣最清楚也最穩定。
- 同一個元素可以有多個屬性。

錯誤示例：

```html
<p>文字</p class="intro">
```

`class` 不應該寫在結束標籤上。

正確示例：

```html
<p class="intro">文字</p>
```

### 布林屬性

有些屬性只要出現就代表啟用，這類常稱為布林屬性。

例如：

```html
<input type="text" disabled>
```

這裡的 `disabled` 表示這個輸入框被禁用。
入門時先知道有這類寫法即可，後面學表單時會更常遇到。

## 雙標籤

由起始標籤和結束標籤成對組成的，入門教材常稱為雙標籤。

例如：

```html
<p>這是一段文字</p>
<div>這是一個區塊</div>
<title>網頁標題</title>
```

這類元素通常可以包住文字或其他元素。

你可以先記：

- 起始標籤負責開始。
- 中間放內容。
- 結束標籤負責收尾。

## 單標籤 / 空元素

有些元素本身不能包住內容，因此不會再寫一個對應的結束標籤。

入門教材常把它們叫做**單標籤**；更精確地說，常稱為**空元素**，英文常見說法是 `void element`。

常見空元素包括：

| 元素 | 常見用途 |
|---|---|
| `<br>` | 換行 |
| `<hr>` | 主題分隔線 |
| `<img>` | 圖片 |
| `<input>` | 表單輸入控制 |
| `<meta>` | 文件資訊 |
| `<link>` | 外部資源連結 |

例如：

```html
<br>
<img src="photo.jpg" alt="照片">
<input type="text" name="username">
```

這些元素不能像下面這樣包內容：

```html
<img src="photo.jpg" alt="照片">圖片文字</img>
```

這種寫法是錯誤觀念。
如果要替圖片補說明，後面會學到可以用其他元素，例如 `<figure>` 和 `<figcaption>`。

### `<br>` 和 `<br />` 差在哪裡？

你可能會在不同教材看到：

```html
<br>
<br />
```

在現代 `HTML` 裡，空元素不需要用 `/` 結尾，寫成 `<br>` 就可以。
`<br />` 這種寫法常見於 `XHTML` 或受到 `XHTML` 影響的舊習慣。

入門時建議先使用現代 `HTML` 的簡潔寫法：

```html
<br>
```

但要注意：不是所有元素都能靠加 `/` 變成空元素。
例如 `<script>`、`<div>`、`<p>` 這些元素不能隨便寫成自我結束的形式。

## 標籤、元素、節點要怎麼區分？

這幾個詞在入門時常被混用，但最好先建立基本差異。

| 名詞 | 可以先怎麼理解 | 例子 |
|---|---|---|
| 標籤 | HTML 語法外殼 | `<p>`、`</p>` |
| 元素 | 由標籤與內容形成的完整結構 | `<p>文字</p>` |
| 節點 | DOM 樹中的單位，範圍比元素更廣 | 元素節點、文字節點、註解節點 |

例如：

```html
<p>這是一段文字</p>
```

- `<p>` 和 `</p>` 是標籤。
- 整個 `<p>這是一段文字</p>` 可以看成一個段落元素。
- 瀏覽器解析後，這個段落會成為 DOM 裡的一個元素節點。

入門時不用把 DOM 細節一次學完，但至少要知道：

- 討論語法時，常說標籤。
- 討論結構與關係時，常說元素。
- 討論 JavaScript 操作頁面時，常會碰到節點。

## 元素之間的兩種基本關係

當頁面開始有多個元素時，最常見可以先分成兩種關係：

- 巢狀關係
- 並列關係

## 巢狀關係

當一個元素包住另一個元素時，就形成巢狀關係，也常稱為嵌套關係。

例如：

```html
<body>
  <div>
    <p>文字</p>
  </div>
</body>
```

在這段裡：

- `body` 包住 `div`
- `div` 包住 `p`

所以這些元素形成了層級結構。

可以畫成：

```text
body
└── div
    └── p
```

### 巢狀要正確關閉

巢狀時，標籤開啟與關閉要像疊盒子一樣，後開的要先關。

正確：

```html
<div>
  <p>文字</p>
</div>
```

錯誤：

```html
<div>
  <p>文字</div>
</p>
```

上面錯誤的原因是：`<p>` 在 `<div>` 裡面開始，卻沒有先正常結束 `</p>`，就先結束了 `</div>`。

## 並列關係

如果兩個元素位在同一層，並且同時被同一個父元素包住，它們就是並列關係。

例如：

```html
<ul>
  <li>第一項</li>
  <li>第二項</li>
</ul>
```

這裡兩個 `li` 位在同一層，也都被 `ul` 包住，所以它們彼此是並列的。

可以畫成：

```text
ul
├── li
└── li
```

## 常見的元素關係名稱

### 父元素

直接包住其他元素的那一層，叫父元素。

```html
<div>
  <p>文字</p>
</div>
```

在這段裡：

- `div` 是 `p` 的父元素。

注意：父元素強調的是**直接上一層**。

### 子元素

被父元素直接包含的那一層，叫子元素。

以上面的例子來看：

- `p` 是 `div` 的子元素。

注意：子元素強調的是**直接下一層**。

### 祖先元素

一路往外包住目前元素的所有上層元素，都可以叫祖先元素。

例如：

```html
<body>
  <main>
    <section>
      <p>文字</p>
    </section>
  </main>
</body>
```

對 `p` 來說：

- `section` 是祖先元素。
- `main` 是祖先元素。
- `body` 也是祖先元素。

其中 `section` 同時還是 `p` 的直接父元素。

### 後代元素

某個元素裡面所有更深層的子孫元素，都可以叫後代元素。

以上面的例子來看：

- `main`、`section`、`p` 都可以算是 `body` 的後代元素。
- `p` 是 `section` 的後代元素，也是 `main` 的後代元素。

### 兄弟元素

位在同一層、而且共享同一個父元素的元素，叫兄弟元素。

例如：

```html
<nav>
  <a href="/">首頁</a>
  <a href="/about">關於我們</a>
  <a href="/contact">聯絡我們</a>
</nav>
```

在這段裡：

- 三個 `a` 都是 `nav` 的子元素。
- 三個 `a` 彼此互為兄弟元素。

## 用一棵樹看懂所有關係

看這段：

```html
<body>
  <header>
    <h1>網站名稱</h1>
  </header>
  <main>
    <section>
      <h2>最新文章</h2>
      <p>這是一段摘要。</p>
    </section>
  </main>
</body>
```

可以畫成：

```text
body
├── header
│   └── h1
└── main
    └── section
        ├── h2
        └── p
```

從這棵樹可以判斷：

- `body` 是 `header` 和 `main` 的父元素。
- `header` 和 `main` 是兄弟元素。
- `section` 是 `main` 的子元素。
- `h2` 和 `p` 是兄弟元素。
- `body` 是 `p` 的祖先元素。
- `p` 是 `body` 的後代元素。

## 一張心智圖式的判斷方式

你可以先用下面這套順序判斷：

1. 先看兩個元素是不是在同一層。
2. 如果在同一層，再看它們是不是共享同一個父元素。
3. 若是，就可以稱為兄弟元素。
4. 如果不在同一層，就看其中一個是否包住另一個。
5. 若是直接包住，談父子。
6. 若中間隔了一層或多層，談祖先與後代。

## 不是所有標籤都能任意嵌套

`HTML` 不是只要一層包一層就一定正確。
不同元素有自己的語意與允許內容，這常稱為內容模型或嵌套規則。

入門時先記幾個常見規則：

- `<ul>` 和 `<ol>` 裡面通常應該直接放 `<li>`。
- `<p>` 裡面不應該放 `<div>` 這類區塊容器。
- 空元素不能包內容。
- 互動元素不要隨意互相嵌套，例如不要把一個連結包在另一個連結裡。

### 例子：清單結構

正確：

```html
<ul>
  <li>第一項</li>
  <li>第二項</li>
</ul>
```

不建議：

```html
<ul>
  <p>第一項</p>
  <p>第二項</p>
</ul>
```

因為清單的直接項目應該用 `<li>` 表示。

### 例子：段落裡不要放區塊容器

不建議：

```html
<p>
  這是一段文字
  <div>這是一個區塊</div>
</p>
```

`<p>` 適合放文字與行內內容，不適合包住 `<div>` 這類區塊容器。

如果需要區塊結構，可以改成：

```html
<div>
  <p>這是一段文字。</p>
  <div>這是一個區塊。</div>
</div>
```

## 瀏覽器會容錯，但結構仍要寫正確

瀏覽器通常很努力讓頁面顯示出來。
有些錯誤的 `HTML`，瀏覽器會自動修補或重新解釋。

但學習時不能只用「有沒有顯示」判斷正不正確，原因是：

- 錯誤結構會讓 CSS 選擇器判斷變困難。
- JavaScript 操作 DOM 時可能找不到你以為的層級。
- 搜尋引擎和輔助技術可能無法正確理解內容。
- 不同瀏覽器或工具的修補方式可能不同。

所以寫 `HTML` 時，應該追求清楚、合法、可維護的結構。

## 常見混淆點

### 標籤和元素不是完全同義

入門時混用問題不大，但只要開始討論結構，最好慢慢分清：

- 標籤偏語法形式。
- 元素偏完整結構與節點概念。

### 屬性寫在起始標籤上

像 `class`、`id`、`href`、`src`、`alt` 這些屬性，通常都寫在起始標籤裡。

錯誤：

```html
<a>連結文字</a href="https://example.com">
```

正確：

```html
<a href="https://example.com">連結文字</a>
```

### 單標籤是入門說法，空元素更精確

「單標籤」方便理解，但更精確的說法是「空元素」。
重點是：它們不能包內容，也不需要結束標籤。

### 父元素和祖先元素不一樣

- 父元素：一定是直接上一層。
- 祖先元素：可以是更外層，不一定直接相鄰。

### 子元素和後代元素不一樣

- 子元素：一定是直接下一層。
- 後代元素：可以跨很多層。

### 兄弟元素一定要同層

不是只要都出現在同一份文件裡，就能叫兄弟元素。
它們必須同層，而且共享同一個父元素。

### 並列不等於視覺上並排

兄弟元素指的是 HTML 結構中的同層關係，不代表它們在畫面上一定左右並排。
畫面如何排列，主要由 CSS 決定。

### 文字也可能成為節點

在 DOM 裡，文字本身也可能成為文字節點。
不過這篇先聚焦在 HTML 元素之間的關係，之後學 JavaScript DOM 時再細分節點類型。

## 常見錯誤示例

### 忘記結束標籤

不建議：

```html
<p>第一段
<p>第二段
```

建議：

```html
<p>第一段</p>
<p>第二段</p>
```

有些元素的結束標籤在特定情況下可以被瀏覽器推斷，但初學時建議明確寫完整，避免結構判斷混亂。

### 結束順序錯誤

錯誤：

```html
<strong><em>重要文字</strong></em>
```

正確：

```html
<strong><em>重要文字</em></strong>
```

後開的元素要先關，結構才會正確。

### 把空元素寫成成對元素

錯誤：

```html
<br></br>
<input type="text"></input>
```

正確：

```html
<br>
<input type="text">
```

## 小練習

請看這段：

```html
<article>
  <h2>文章標題</h2>
  <p>第一段內容。</p>
  <p>第二段內容。</p>
</article>
```

試著回答：

1. `article` 是哪幾個元素的父元素？
2. 兩個 `p` 彼此是什麼關係？
3. `h2` 是 `article` 的子元素還是後代元素？
4. `article` 是兩個 `p` 的父元素還是祖先元素？

參考答案：

1. `article` 是 `h2` 和兩個 `p` 的父元素。
2. 兩個 `p` 互為兄弟元素。
3. `h2` 是 `article` 的子元素，也可以說是後代元素；但更精確是直接子元素。
4. `article` 是兩個 `p` 的父元素；父元素也是祖先元素的一種，但這裡說父元素更精確。
