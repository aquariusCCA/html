# HTML 版本的區別

> 所屬章節：第二章｜HTML 簡介  
> 關鍵字：HTML5、HTML4、XHTML、DOCTYPE、版本差異、向下相容、語意化標籤、表單、新特性、瀏覽器相容性  
> 建議回查情境：想快速分清 HTML5、HTML4、XHTML 的差別、想知道為什麼現在多半使用 HTML5、想確認不同版本的文件宣告長什麼樣

## 本節導讀

這一節整理 `HTML5`、`HTML4` 與 `XHTML` 這幾個常見版本名稱的差異。  
如果你在舊教材、舊專案或網路文章裡看到不同的 `DOCTYPE` 寫法，這篇可以幫你先建立一個清楚的比較框架。

## 你會在這篇學到什麼

- 為什麼入門時常以 `HTML5` 為主
- `HTML5`、`HTML4`、`XHTML` 的基本差異
- 不同版本常見的文件宣告寫法
- 學習與實務上應該怎麼理解這些版本名稱

## 30 秒複習入口

- 現在大多數新專案預設使用的是 `HTML5`。
- `HTML5` 的文件宣告最簡潔，並加入了更多語意化標籤、表單能力與多媒體相關特性。
- `HTML4` 和 `XHTML` 主要會出現在舊教材、舊頁面或歷史文件中；看到它們時，重點是能辨認和理解，不一定代表今天還要照舊寫。

## 速查區

### 核心定義

- `HTML4`：較早期的 `HTML` 標準版本。
- `XHTML`：可先理解成把 `HTML 4.01` 以更嚴格、符合 `XML` 規則的方式重新表達的一組規格。
- `HTML5`：現代 `HTML` 寫法最常見的起點，也是現在學習與開發最常見的主線。

### 一句話對比

- `HTML4` 是較早期標準，`XHTML` 強調 `XML` 式嚴格語法，`HTML5` 則是現在最常見的寫法與學習主線。

### 常見錯誤

- 把 `HTML5` 誤解成只有一組新標籤。
- 把廣義的「HTML5 前端技術集合」和狹義的 `HTML5` 標準混成同一層概念。
- 把 `XHTML` 當成只是 `HTML4` 和 `HTML5` 中間的一個普通版本號，而忽略它其實更強調 `XML` 式寫法。
- 看到 `HTML4` 或 `XHTML` 的 `DOCTYPE` 就以為現代專案也必須照抄。

## 正文筆記

### 這篇在解決什麼問題？

- 初學 `HTML` 時，常會看到 `HTML4`、`XHTML`、`HTML5` 這幾個名字，但不知道它們差在哪裡。
- 也很容易只記住一大段 `DOCTYPE`，卻不知道那段宣告背後到底代表什麼版本。
- 這篇的目標，是先建立版本比較的主線，讓你知道現在為什麼大多從 `HTML5` 開始學。

### 先講結論

- 現代瀏覽器對舊寫法有大量相容處理，所以你在今天的網頁或教材中，仍可能看到較早期就存在的標籤、屬性與 `DOCTYPE`。
- 不過，現在學習與開發時，預設通常以 `HTML5` 為主。
- `HTML4` 與 `XHTML` 比較像是理解歷史、閱讀舊文件與辨識舊專案時需要知道的背景知識。

### HTML5 版本

#### 簡介

- `HTML5` 是新一代的 `HTML` 標準，常被視為現代前端開發的基礎。
- 它補強了較早版本在語意、表單、多媒體與互動能力上的不足。
- 相較於舊版本，`HTML5` 最常被提到的特點包括：
  - 新的語意化標籤
  - 更完整的表單與表單屬性
  - 更方便的多媒體支援
  - 更適合現代網站與應用程式的能力擴充

#### 狹義與廣義的 HTML5

- 狹義的 `HTML5`，是指新一代的 `HTML` 標準本身。
- 廣義上，教材有時會把 `HTML5`、`CSS3`、`JavaScript` 與周邊 API 一起統稱為「HTML5」技術集合。
- 所以當你看到有人說「學 HTML5」，要先分清他是在指：
  - `HTML` 這個標準本身
  - 還是整套現代前端技術語境

#### 相容性怎麼理解？

- 舊教材常會特別提到 `HTML5` 新特性與早期瀏覽器的相容性問題，尤其會提到 `IE`。
- 這類說法有其歷史背景，重點是告訴你：`HTML5` 的新能力不是在所有舊瀏覽器裡都能完全一致地工作。
- 以現在的學習角度來看，比起背特定舊版本瀏覽器支援表，更重要的是知道：
  - 新特性需要考慮瀏覽器支援情況
  - 實務上應依專案需求檢查相容性，而不是假設所有環境都完全一致

#### HTML5 的常見優勢

- 增加了更多可供 `JavaScript` 操作的能力。
- 新增語意化標籤與全域屬性，讓頁面結構更清楚。
- 提供多媒體相關能力，降低對舊式外掛技術的依賴。
- 更重視語意化，對內容結構與搜尋引擎理解較友好。
- 更適合現代網站與行動裝置場景。

#### HTML5 的基本文件宣告

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Document</title>
  </head>
  <body>

  </body>
</html>
```

### HTML4 版本

- `HTML4` 是較早期的標準版本。
- 和 `HTML5` 相比，常見差異不是只有標籤多寡，而是整體語意化能力、表單能力與現代 Web 特性的支援程度。
- 在今天的學習情境中，`HTML4` 主要是用來幫助你辨認舊寫法與理解歷史演進。

#### HTML4 的文件宣告範例

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<html>
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
    <title></title>
  </head>
  <body>

  </body>
</html>
```

### XHTML 版本

- `XHTML` 可以先理解成「用 `XML` 規則來寫的 `HTML`」。
- 因此它和 `HTML4`、`HTML5` 的差異，不只是版本名稱，而是語法要求更嚴謹。
- 更精確地說，`XHTML 1.0` 和 `HTML 4.01` 在內容模型上很接近，但 `XHTML` 要求你遵守更嚴格的書寫規則。
- 因此在閱讀舊教材時，若看到特別強調標籤閉合、屬性寫法或文件格式更嚴格，常常就是在 `XHTML` 的語境下。

#### XHTML 的文件宣告範例

```html
<!DOCTYPE html
PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="zh">
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
    <title></title>
  </head>
  <body>

  </body>
</html>
```

### 三個版本可以怎麼記？

- 如果你的目標是現在開始學前端，先把 `HTML5` 當成主線。
- 如果你在舊文件中看到很長的 `DOCTYPE`，通常是在 `HTML4` 或 `XHTML` 的脈絡。
- 如果你看到教材特別強調更嚴格的標籤與語法規則，常常是在講 `XHTML`。
- 如果你想快速區分：`HTML4` 偏舊標準、`XHTML` 偏嚴格語法、`HTML5` 偏現代主流。

### 常見混淆點

- `HTML5` 不只是「多幾個標籤」，而是整體能力更完整的一代標準。
- 廣義 `HTML5` 不等於狹義 `HTML5`；前者常把 `CSS3`、`JavaScript` 與相關 API 一起算進去。
- `XHTML` 不是單純「夾在 `HTML4` 和 `HTML5` 中間的一版」；它更像是把 `HTML 4.x` 放到 `XML` 規則下處理的一條分支。
- `HTML4`、`XHTML` 的存在不代表今天不能用 `HTML5`；更多時候，它們是歷史脈絡與舊專案知識。
