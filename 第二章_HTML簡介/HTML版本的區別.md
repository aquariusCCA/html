# HTML 版本的區別

> 所屬章節：第二章｜HTML 簡介  
> 關鍵字：HTML5、HTML4、XHTML、DOCTYPE、版本差異、語意化標籤、表單、多媒體、瀏覽器相容性、舊專案辨識  
> 建議回查情境：想快速分清 `HTML5`、`HTML4`、`XHTML` 的差別、想知道為什麼現在多半使用 `HTML5`、想看懂不同版本常見的 `DOCTYPE`

## 本節導讀

這一節要處理的是一個很常見的困惑：  
為什麼有時候會看到 `HTML5`，有時候又看到 `HTML4`、`XHTML`，甚至 `DOCTYPE` 長得完全不一樣？

這一篇的重點不是要你背歷史細節，而是幫你建立一個**夠清楚、夠實用的辨認框架**。

## 你會在這篇學到什麼

- 為什麼現在入門與新專案通常以 `HTML5` 為主
- `HTML5`、`HTML4`、`XHTML` 的基本差異
- 為什麼舊教材會出現很長的 `DOCTYPE`
- 實務上該怎麼看待這些版本名稱

## 30 秒複習入口

- 現在大多數新專案預設使用 `HTML5`。
- `HTML5` 的文件宣告最簡潔，也更符合現代開發需求。
- `HTML4` 和 `XHTML` 主要常見於舊教材、舊頁面、歷史文件與舊專案。
- 學這一節的重點是**能辨認、能理解**，不是把舊版宣告整段背起來。

## 先講結論

如果你是現在開始學前端，可以先這樣記：

- **主線用 `HTML5`。**
- **`HTML4` 和 `XHTML` 當成背景知識。**
- 看到舊版 `DOCTYPE` 時，知道它大致屬於哪個時代、哪種寫法脈絡，就已經很夠用了。

## 這篇在解決什麼問題？

初學 `HTML` 時，常見問題有三個：

- 為什麼網路上有不同版本名稱？
- 為什麼有些教材的 `DOCTYPE` 很短，有些卻長得很誇張？
- 現在到底該學哪一種寫法？

這篇的目標，就是先把版本主線建立出來，避免一開始被歷史細節干擾。

## 先有一個總覽

### `HTML4`

- 較早期的重要 `HTML` 標準版本。
- 在今天主要用來幫助你辨認舊文件與理解歷史演進。

### `XHTML`

- 可以先理解成：把較早期的 `HTML` 用更接近 `XML` 規則的方式來表達。
- 重點不只是版本名稱，而是它對語法要求更嚴格。

### `HTML5`

- 現代最常見、也是現在學習主線通常採用的 `HTML` 寫法。
- 對語意、表單、多媒體與現代 Web 能力支援更完整。

## 一句話快速對比

- `HTML4`：較早期標準。
- `XHTML`：偏向更嚴格的 `XML` 式語法。
- `HTML5`：現代主流寫法與學習主線。

## 為什麼現在多半以 HTML5 為主？

因為 `HTML5` 更符合現代網站與應用的需求，常見優勢包括：

- 更完整的語意化標籤
- 更實用的表單能力
- 更方便的多媒體支援
- 更適合現代瀏覽器與行動裝置場景

所以現在學習時，通常不會把重心放在舊版寫法，而是以 `HTML5` 為主，再補足對舊版本的辨識能力。

## HTML5 要怎麼理解？

### 1. 它是現代主線

現在大多數新專案預設使用的是 `HTML5`。  
如果你在學習現代前端，通常就是以這條線為核心。

### 2. 它不只是「多幾個新標籤」

很多人會把 `HTML5` 簡化成「新增一些標籤」。  
這樣不算錯，但不夠完整。更好的理解是：

- `HTML5` 不只是增加部分標籤。
- 它也補強了語意、表單、多媒體與整體現代 Web 的能力。

### 3. 它的文件宣告很簡潔

```html
<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <title>Document</title>
  </head>
  <body>
  </body>
</html>
```

這種簡潔寫法，是你現在最常看到、也最值得優先熟悉的版本。

## 狹義與廣義的 HTML5

這裡很容易混淆，所以要特別分開。

### 狹義的 HTML5

指的是 `HTML` 這個標準本身。

### 廣義的 HTML5

有些教材或文章會把 `HTML5`、`CSS3`、`JavaScript`，以及一些周邊 API，一起放進「HTML5 技術」這個大語境裡。

所以當你看到有人說「學 HTML5」，要先分清他是在指：

- `HTML` 這個標準本身
- 還是整套現代前端技術脈絡

## HTML4 要怎麼看？

- `HTML4` 是較早期的重要標準。
- 它對今天的新手來說，重點不是拿來當主線，而是拿來辨認舊寫法。
- 當你在舊教材或舊專案裡看到較長的 `DOCTYPE`，很可能就在 `HTML4` 的脈絡附近。

### HTML4 的文件宣告範例

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<html>
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
    <title></title>
  </head>
  <body>
  </body>
</html>
```

你不用把這段背起來。  
你只要知道：**這種比較長的宣告，多半不是現代新專案的預設主線。**

## XHTML 要怎麼看？

- `XHTML` 可以先理解成「用更嚴格的 `XML` 規則來寫 `HTML`」。
- 它和 `HTML4` 的關係很近，但重點在於語法要求更嚴謹。
- 例如標籤閉合、屬性書寫等，通常都會被要求更一致、更規範。

### XHTML 的文件宣告範例

```html
<!DOCTYPE html
PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="zh">
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
    <title></title>
  </head>
  <body>
  </body>
</html>
```

同樣地，這段不需要硬背。  
你只要能辨認：

- 宣告通常比較長
- 語法要求通常比較嚴格
- 它不是現在入門主線

這樣就足夠了。

## 相容性要怎麼理解？

舊教材常會提到：某些 `HTML5` 新特性在早期瀏覽器，尤其是舊版 `IE`，可能支援不一致。  
這種說法有歷史背景，並不是完全沒有意義。

但以現在的學習角度來看，你更應該抓住的是：

- 新特性要考慮瀏覽器支援情況。
- 實務上應根據專案需求查相容性。
- 不需要一開始就去背一整張舊瀏覽器支援表。

## 三個版本怎麼記最省力？

你可以直接記成下面這樣：

- `HTML4`：偏舊標準。
- `XHTML`：偏嚴格語法。
- `HTML5`：偏現代主流。

如果是現在開始學前端：

- 把 `HTML5` 當主線。
- 把 `HTML4`、`XHTML` 當辨識舊文件的背景知識。

## 常見混淆點

### `HTML5` 不只是「多幾個標籤」

它代表的是更完整的一代現代標準，而不只是標籤數量變多。

### 廣義 HTML5 不等於狹義 HTML5

有時候別人說的 `HTML5`，其實是指整套現代前端技術脈絡，不一定只是在講 `HTML` 標準本身。

### `XHTML` 不是單純夾在 `HTML4` 和 `HTML5` 中間的一版

它更像是一條強調 `XML` 式嚴格語法的分支脈絡。

### 看到舊版 `DOCTYPE` 不代表今天要照抄

你需要的是理解與辨認，而不是把舊專案寫法原封不動搬到現代專案。

## 自測問題

1. `HTML5`、`HTML4`、`XHTML` 最適合怎麼快速區分？
2. 為什麼現在多數新專案會以 `HTML5` 為主？
3. 廣義與狹義的 `HTML5` 差別是什麼？
4. 為什麼看到很長的 `DOCTYPE` 時，通常會聯想到 `HTML4` 或 `XHTML`？

## 一句話抓核心

- **現在學 `HTML` 的主線通常是 `HTML5`；`HTML4` 和 `XHTML` 主要用來理解歷史脈絡、辨認舊教材與看懂舊專案。**
