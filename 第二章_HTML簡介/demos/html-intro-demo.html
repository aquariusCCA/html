1. 網頁、網站、HTML 之間的關係

Web Page

這是一個單一網頁

網頁是網站中的單一頁面。你現在看到的，可以理解成網站裡其中一個具體畫面。

畫面焦點

通常是一個具體頁面，例如首頁、文章頁、產品頁。

整體焦點

由多個頁面與資源組成，服務同一個主題或功能。

網站示意

同一個網站裡，常會有多個彼此相關的網頁。

首頁 入口頁面
文章頁 內容頁面
會員頁 功能頁面
HTML 的角色是描述單一網頁中的內容與結構，瀏覽器再把它解析成你看到的畫面。

2. HTML 骨架怎麼拆

<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <title>我的第一個頁面</title>
  </head>
  <body>
    <h1>Hello HTML</h1>
    <p>頁面內容寫在這裡。</p>
  </body>
</html>
我的第一個頁面
先有骨架，瀏覽器才知道這是一份什麼樣的文件，以及哪裡是設定、哪裡是內容。

Hello HTML

頁面內容寫在這裡。

3. 標籤的父子、祖先、後代、兄弟關係

<body>
  <div class="card">
    <h1>課程標題</h1>
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </div>
</body>
<body>
<div>
<h1>
<ul>
<li>HTML</li>
<li>CSS</li>
目前主角 與主角形成當前關係的元素
Parent / Child

直接包住的是父元素，被直接包住的是子元素

在這個例子裡，<div> 是 <ul> 的父元素，而 <ul> 是 <div> 的子元素。

  • 父元素一定是直接上一層。
  • 子元素一定是直接下一層。
  • 跨多層時，就不再只是父子，而會談祖先與後代。

4. HTML5、HTML4、XHTML 怎麼辨識

現代主流

HTML5：現在學習與新專案的主線

現在大多數新專案會以 HTML5 為主。它的 DOCTYPE 簡潔，且更符合現代網站需求。

定位 現代主流寫法
特徵 宣告短、語意與現代能力更完整
學習建議 當主線學習 
<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <title>Document</title>
  </head>
  <body>
  </body>
</html>
看到這種簡潔的 <!DOCTYPE html>，通常就是現代最常見的 HTML5 脈絡。
