# 第23章｜表格標籤

> - 關鍵字：表格、`table`、`tr`、`th`、`td`、`caption`、`thead`、`tbody`、`tfoot`、表頭、表格主體、表格腳注、合併單元格、`rowspan`、`colspan`
> - 建議回查情境：想知道表格的基本結構怎麼寫、想分清 `th` 和 `td`、想理解 `caption` / `thead` / `tbody` / `tfoot` 的作用、想做跨欄或跨列合併

## 本節導讀

HTML 的表格標籤，不是拿來把頁面硬排成一格一格。
它真正的用途是：**表達有欄位、列、對照關係的結構化資料**。

只要資料具有這種特性，例如：

* 成績表
* 報表
* 規格比較表
* 後台資料清單
* 統計資料

就很適合用表格。

學表格時，最重要的不是只記住 `table`、`tr`、`td` 這幾個標籤，而是理解：

* 表格適合拿來表達什麼資料
* 表頭和一般資料儲存格差在哪
* 表格可以怎麼分區
* 合併單元格時應該怎麼思考方向與結構

只要把這幾件事釐清，表格就不容易學亂。

* 為什麼教材裡常見的舊式表格屬性，實務上多半改由 CSS 處理

## 30 秒複習入口

* 表格是用來展示**有欄列關係的資料**，不是用來做整頁布局。
* `table` 是整張表，`tr` 是一列，`th` 是表頭儲存格，`td` 是資料儲存格。
* `caption` 是表格標題，`thead` / `tbody` / `tfoot` 分別是表頭區、主體區、腳注區。
* `colspan` 是**向右合併**，`rowspan` 是**向下合併**。
* `border`、`align`、`cellpadding` 這類舊式 HTML 屬性可以認識，但現代實務通常優先用 CSS 控制樣式。

---

## 速查區

### 核心概念

* 表格適合拿來表達**可對照、可比較、有欄列結構**的資料。
* 如果資料的重點在「某個欄位對應某筆資料」，表格就很合適。
* 如果只是想把畫面排整齊，通常不應優先想到表格。

### 關鍵規則

* `table`：整張表格
* `tr`：一列
* `th`：表頭儲存格
* `td`：一般資料儲存格
* `caption`：表格標題
* `thead`：表頭區塊
* `tbody`：表格主體區塊
* `tfoot`：表格腳注或彙總區塊
* `colspan`：向右合併幾個儲存格
* `rowspan`：向下合併幾個儲存格

### 一句話抓核心

* 表格標籤的重點，是把**有欄列關係的資料表達清楚**，不是把頁面硬切成格子。

# 正文筆記

## 1. 表格是什麼？為什麼要用表格？

當你要在頁面中呈現一組資料，而且這組資料具有以下特徵時，就適合用表格：

* 有欄位名稱
* 有多筆資料列
* 資料之間需要橫向或縱向比較
* 讀者需要快速看出對照關係

例如：

* 姓名 / 性別 / 年齡
* 科目 / 分數
* 商品 / 價格 / 庫存
* 月份 / 營收 / 成長率

這些資料如果只用段落或列表呈現，通常不夠直觀；
但用表格呈現時，讀者可以很快看出每一格資料的意義。

### 核心觀念

表格不是拿來做一般頁面布局的。
它最適合處理的是：**資料展示**。

---

## 2. 表格的基本結構

HTML 表格最基本的幾個標籤如下：

* `<table>`：整張表格
* `<tr>`：table row，表格中的一列
* `<th>`：table header cell，表頭儲存格
* `<td>`：table data cell，一般資料儲存格

範例：

```html
<table>
  <tr>
    <th>姓名</th>
    <th>性別</th>
    <th>年齡</th>
  </tr>
  <tr>
    <td>劉德華</td>
    <td>男</td>
    <td>56</td>
  </tr>
  <tr>
    <td>張學友</td>
    <td>男</td>
    <td>58</td>
  </tr>
  <tr>
    <td>郭富城</td>
    <td>男</td>
    <td>51</td>
  </tr>
</table>
```

### 怎麼理解這段結構？

* `<table>` 是最外層，代表整張表格
* 每個 `<tr>` 代表一列
* 第一列放的是欄位名稱，所以用 `<th>`
* 後面的列放的是實際資料，所以用 `<td>`

### `th` 和 `td` 最重要的差別

很多初學者會先注意到：

* `th` 預設是粗體
* `th` 預設常會置中

但這不是最重要的。
真正要記的是：

* `th` 代表**表頭**
* `td` 代表**資料**

也就是說，差別不只是外觀，而是語意角色。

---

## 3. 更完整的表格區塊

當表格內容較完整時，常會再細分成幾個區塊：

* `caption`：表格標題
* `thead`：表頭區
* `tbody`：主體區
* `tfoot`：腳注或彙總區

範例：

```html
<table border="1">
  <caption>段考成績</caption>

  <thead>
    <tr>
      <th>科目</th>
      <th>分數</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>語文</td>
      <td>99</td>
    </tr>
    <tr>
      <td>數學</td>
      <td>60</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <td>總分</td>
      <td>159</td>
    </tr>
  </tfoot>
</table>
```

### 各自的作用

#### `caption`

用來表示這張表格的標題。

```html
<caption>段考成績</caption>
```

它的作用不是多放一行字而已，而是讓讀者一眼知道：
**這張表是在講什麼。**

---

#### `thead`

用來放表格的表頭區。
通常會放欄位名稱。

```html
<thead>
  <tr>
    <th>科目</th>
    <th>分數</th>
  </tr>
</thead>
```

---

#### `tbody`

用來放表格的主要資料內容。
大部分的資料列通常都在這裡。

```html
<tbody>
  <tr>
    <td>語文</td>
    <td>99</td>
  </tr>
</tbody>
```

---

#### `tfoot`

用來放表格底部的補充資訊、統計結果或彙總資料。

```html
<tfoot>
  <tr>
    <td>總分</td>
    <td>159</td>
  </tr>
</tfoot>
```

### 為什麼要分這些區塊？

因為這樣做可以讓：

* 結構更清楚
* 閱讀更容易
* 維護更方便
* 後續套樣式、列印、操作資料時更有條理

所以這些標籤不只是「看起來比較專業」，而是真的有助於結構化。

---

## 4. 表格的舊式屬性怎麼理解？

很多教材會先介紹一些直接寫在 HTML 標籤上的表格屬性，例如：

* `align`
* `border`
* `cellpadding`
* `cellspacing`
* `width`

例如：

```html
<table align="center" border="1" cellpadding="20" cellspacing="0" width="500">
  <!-- 中間省略 -->
</table>
```

### 這些屬性各自在做什麼？

* `align`：表格相對周圍內容的對齊方式
* `border`：表格邊框粗略設定
* `cellpadding`：儲存格內容和邊框之間的距離
* `cellspacing`：儲存格和儲存格之間的距離
* `width`：表格整體寬度或布局參考寬度

### 初學時怎麼看待它們？

你可以先認識，因為很多舊教材、舊專案裡還是會看到。
但現代實務通常不靠這些 HTML 屬性處理樣式，而是交給 CSS。

### 所以這一段最重要的結論是

* **教材中要認得**
* **實務中多半不用它們主控外觀**
* **現代樣式控制通常交給 CSS**

---

## 5. 寬度和高度怎麼理解？

這一段很容易被講得太死，所以要用比較穩的方式理解。

### 寬度

當你對某個儲存格或表格設定寬度時，常會影響整個欄位或整體表格的寬度分配。

### 高度

當你對某個儲存格或某一列設定高度時，常會影響那一列的高度表現。

### 最好這樣記

不要把它想成「只有某一格自己變大」。
比較穩的理解是：

**表格的尺寸分配，通常會牽動整張表的欄列配置。**

---

## 6. 合併單元格

有時候表格不是每一列都要維持完全相同的格子數量。
某些情況下，你會想把相鄰的儲存格合併成一格。

這時就會用到：

* `colspan`
* `rowspan`

---

## 6.1 `colspan` 是什麼？

`colspan` 用來讓一個儲存格**向右合併**多個欄位。

```html
<td colspan="2"></td>
```

這表示這個儲存格會占用原本兩格的寬度。

### 最穩定的記法

* `col` 可以聯想到 column
* **`colspan` = 向右跨幾欄**

---

## 6.2 `rowspan` 是什麼？

`rowspan` 用來讓一個儲存格**向下合併**多個列。

```html
<td rowspan="2"></td>
```

這表示這個儲存格會占用原本兩列的高度。

### 最穩定的記法

* `row` 可以聯想到 row
* **`rowspan` = 向下跨幾列**

---

## 6.3 最不容易搞混的總結

不要硬背模糊的中文名詞，直接記方向最穩：

* `colspan`：**向右合併**
* `rowspan`：**向下合併**

---

## 6.4 合併單元格三步驟

這是最實用的記法。

### 第一步：先判斷方向

你要的是：

* 向右吃掉幾格？ → `colspan`
* 向下吃掉幾格？ → `rowspan`

### 第二步：把屬性寫在保留下來的那一格上

例如：

```html
<td colspan="2"></td>
```

或：

```html
<td rowspan="2"></td>
```

### 第三步：刪掉被合併掉的多餘儲存格

這一步很多初學者最容易漏掉。
因為你已經讓某一格延伸出去，就不能再把被覆蓋的位置保留成原本的 `td`。

---

## 6.5 合併範例

```html
<table width="500" height="249" border="1" cellspacing="0">
  <tr>
    <td></td>
    <td colspan="2"></td>
  </tr>

  <tr>
    <td rowspan="2"></td>
    <td></td>
    <td></td>
  </tr>

  <tr>
    <td></td>
    <td></td>
  </tr>
</table>
```

### 怎麼看這段？

* 第一列第二格使用 `colspan="2"`，表示這格向右吃掉兩欄
* 第二列第一格使用 `rowspan="2"`，表示這格向下吃掉兩列

### 真正要記住的是

合併不是魔法，而是：

* 某一格往右延伸，或往下延伸
* 被它吃掉的位置，就不要再額外保留原本的儲存格

---

## 7. 這一章最需要記住什麼？

* 表格是為了展示資料，不是為了做一般頁面布局
* `table` 是整張表，`tr` 是列，`th` 是表頭，`td` 是資料
* `caption`、`thead`、`tbody`、`tfoot` 讓表格結構更清楚
* 舊式 HTML 表格屬性可以認識，但現代實務多半交給 CSS
* `colspan` 是向右合併，`rowspan` 是向下合併

# 常見問法

## `th` 和 `td` 差在哪？

* `th` 是表頭儲存格
* `td` 是一般資料儲存格

差別不只在預設外觀，而在它們的語意角色不同。

---

## `caption` 一定要寫嗎？

不一定。
但如果你希望表格主題很明確，`caption` 會是很直接的做法。

---

## `tbody` 可以省略嗎？

某些情況下，瀏覽器會自動補出表格主體區塊。
但在學習與整理筆記時，明確寫出 `tbody` 會更容易看懂整個結構。

---

## `rowspan` 和 `colspan` 最容易搞混的是什麼？

最容易混的是中文說法。
所以最穩的記法不是死背「跨行 / 跨列 / 跨欄」，而是直接記方向：

* `colspan`：向右
* `rowspan`：向下

---

## 表格可以拿來做整頁布局嗎？

不建議這樣理解。
表格最適合處理的是資料展示，而不是整頁版面設計。
