# 標題標籤

> 所屬章節：第十章｜標題標籤
> 關鍵字：標題標籤、`<h1>`、`<h2>`、`<h3>`、`<h4>`、`<h5>`、`<h6>`、heading、標題層級、語意結構、內容大綱
> 建議回查情境：想知道 `h1` 到 `h6` 的差別、想確認標題標籤是不是只能拿來放大字、想快速理解 heading 的層級概念與使用方式

## 本節導讀

這篇整理 HTML 的標題標籤，也就是 `<h1>` 到 `<h6>`。

它們最核心的作用，不是單純把文字變大，而是用來描述**內容的標題層級與結構**。
換句話說，標題標籤不是主要拿來做視覺效果，而是拿來告訴瀏覽器、開發者與讀者：

* 這一段是不是標題
* 這個標題在整份內容中屬於第幾層
* 它和其他段落之間是什麼關係

在文章、新聞、教學文件、產品說明頁中，標題幾乎一定會出現。
學標題標籤時，真正要掌握的不是只背 `<h1>` 到 `<h6>` 六個名稱，而是理解它們在**語意**與**內容結構**上的用途。

## 30 秒複習入口

* `<h1>` 到 `<h6>` 是 HTML 的六級標題標籤。
* 它們用來表示內容的**標題層級與語意結構**，不只是改變字體大小。
* `h1` 通常是整頁或整篇內容的主要標題，`h2`、`h3` 則用來往下展開子主題。
* 瀏覽器預設通常會讓標題加粗、獨佔一行，且字級由 `h1` 到 `h6` 逐漸變小。
* 如果只是想改外觀，應交給 CSS，不要濫用標題標籤。

## 速查區

### 核心概念

* 標題標籤是用來替頁面建立內容結構的。
* 這組結構通常會形成文件的大綱。
* 重點不是「字看起來多大」，而是「這段內容在結構上屬於哪一層」。

### 關鍵規則 / 判準

* HTML 提供六級標題：`<h1>` 到 `<h6>`。
* 層級越前面，代表越高層；層級越後面，代表越細的子主題。
* 實務上常見做法是：

  * `h1`：整頁主標題 / 文章主題
  * `h2`：主題底下的大段落
  * `h3`：`h2` 底下的子段落
* 如果只是想讓字變大、變粗，應該使用 CSS，而不是把一般文字改成 heading。

### 常見特點

* 預設通常會加粗
* 預設通常會獨佔一行
* 預設字級通常從 `h1` 到 `h6` 逐漸變小

### 常見混淆點

* 標題標籤的重點是**語意與結構**，不是只有外觀。
* 「看起來像標題」和「語意上是標題」不是同一件事。
* CSS 可以改掉 heading 的外觀，但不會改掉它原本的語意角色。

### 一句話對比

* 標題標籤解決的是「這段內容在結構上是第幾層標題」。
* CSS 解決的是「這段文字要長什麼樣子」。

## 正文筆記

### 這篇在解決什麼問題？

在文章、教學頁面或說明文件中，內容通常不會只有一段文字，而是會分成：

* 主題
* 章節
* 子章節
* 更細的說明段落

如果整份內容都沒有清楚標示層級，讀者就很難快速看懂整體架構。
HTML 提供 `<h1>` 到 `<h6>` 六級標題標籤，就是為了把這種結構表達出來。

---

## 1. 標題標籤是什麼

標題標籤是一組用來描述**內容層級**的 HTML 標籤。

它的核心不是裝飾文字，而是標示：

* 哪些內容是標題
* 這個標題屬於第幾層
* 它和前後內容之間的關係

你可以把它想成文件的大綱工具。

例如：

* `h1`：整份內容最上層的主題
* `h2`：主題底下的大章節
* `h3`：章節底下的小節
* `h4 ~ h6`：更細的下層結構

也就是說，heading 不只是「重要性遞減」，更重要的是它在表達**結構上的上下層關係**。

---

## 2. 基本語法

```html
<h1>一级标题</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
<h4>四级标题</h4>
<h5>五级标题</h5>
<h6>六级标题</h6>
```

### 這段怎麼理解？

這六個標籤都屬於標題，只是層級不同。

* `h1` 是最高層級
* `h6` 是最低層級
* 使用時通常會從高層往下展開
* 不需要為了「把六個都用到」而硬塞層級

上面的寫法只是展示語法。
真正更重要的是：**在實際內容中怎麼安排它們的層級關係**。

---

## 3. 用真實內容理解標題層級

只看 `<h1>` 到 `<h6>` 的排列，容易記住標籤名稱，卻不容易真正理解結構。
下面這個例子比較接近真實文件：

```html
<h1>JavaScript 教學</h1>

<h2>變數</h2>
<h3>let 與 const</h3>
<h3>作用域</h3>

<h2>函式</h2>
<h3>函式宣告</h3>
<h3>參數與回傳值</h3>
```

### 這個例子在表達什麼？

* `h1`：整篇內容的主題是「JavaScript 教學」
* `h2`：主題底下分成兩個大章節，分別是「變數」與「函式」
* `h3`：每個章節下面再往下細分小主題

所以標題標籤本質上是在建立一棵內容樹。
你在寫網頁時，應該先思考「內容結構」，再決定用哪一級 heading，而不是先看想讓字多大。

---

## 4. 標題標籤的預設特點

瀏覽器通常會對 heading 套用一些預設樣式，例如：

* 文字加粗
* 獨佔一行
* 字級從 `h1` 到 `h6` 逐漸變小

### 這些特點要怎麼理解？

這些只是**預設顯示效果**，方便使用者一眼看出層級差異。
但真正重要的不是外觀，而是它背後的語意。

也就是說：

* 外觀可以被 CSS 改掉
* 語意角色仍然存在

例如你可以把 `h2` 改得比 `h1` 還小，但它在文件結構中仍然是第二層標題。

---

## 5. 常見使用場景

### 新聞或文章頁面

* 文章主標題通常會是 `h1`
* 文章中的大段主題可以用 `h2`
* 更細的子段落可以用 `h3`

### 教學文件或說明文件

* 課程主題、章節名稱、小節名稱，常會形成清楚的層級
* 使用 heading 可以讓內容更容易閱讀與回查

### 文件型頁面 / 部落格 / 技術筆記

* 標題標籤可以幫助讀者快速掃描重點
* 對內容整理與維護也比較清楚

---

## 6. 使用標題標籤時的注意點

### 不要把 heading 當成單純放大字體的工具

錯誤觀念是：

> 我只是想讓這行字變大，所以我用 `h2`

這樣做不對，因為 heading 的主要作用不是樣式，而是結構。
如果只是想改大、改粗、改間距，應該交給 CSS。

---

### 標題層級應該反映內容結構

標題不是隨便挑一個順眼的大小來用，而是應該根據內容的上下層關係決定。

例如：

* `h1` 是整體主題
* `h2` 是主題下面的大段落
* `h3` 是 `h2` 底下的小段落

所以你在決定要用哪個 heading 時，應該先問自己：

> 這段內容在整份文件中屬於第幾層？

---

### 不一定要用滿六級

很多初學者會以為：

> 既然有 `h1 ~ h6`，是不是一篇內容最好全部都用到？

不是。
重點不是把六級都用滿，而是你真的有幾層結構，就用到對應層級即可。

---

### 層級最好自然遞進

例如從 `h1` 接到 `h2`，再到 `h3`，這樣通常比較容易理解。
如果直接從 `h1` 跳到 `h3`，語法上不一定是完全禁止，但在結構理解上通常會比較不直觀。

所以更精確地說法是：

* 不是「絕對不能跳層」
* 而是「實務上通常不建議隨意跳層，因為會讓文件層級變得不清楚」

---

## 7. 標題標籤 vs CSS 放大字體

這兩者最容易混淆，但它們解決的是不同問題。

### 標題標籤

負責：

* 定義內容層級
* 建立語意結構
* 表示這段內容是標題

### CSS

負責：

* 控制字體大小
* 控制粗細、顏色、間距
* 控制畫面呈現效果

所以：

* heading 是**內容結構**
* CSS 是**外觀樣式**

這兩者可以一起使用，但不要互相混用角色。

---

## 常見問法

### `h1` 到 `h6` 一定要全部用到嗎？

不一定。
重點不是把六級都用滿，而是依照內容的實際層級來使用。

---

### 可以跳層嗎？例如直接從 `h1` 到 `h3`？

語法上不一定是完全不能，但通常不建議。
因為跳層會讓整份內容的大綱不夠自然，讀者在理解結構時也比較容易混亂。

---

### 標題標籤和 CSS 放大字體有什麼差別？

* 標題標籤：表示內容層級與語意
* CSS 放大字體：控制視覺外觀

一個偏結構，一個偏樣式。

---

### 標題的外觀可以改嗎？

可以。
你可以用 CSS 修改 heading 的字級、粗細、顏色、間距。
但即使外觀改了，它在 HTML 中仍然是標題。

---

## 常見迷思整理

### 迷思 1：標題標籤就是拿大字體用的

錯。
標題標籤是語意標籤，主要是用來表達內容結構。

### 迷思 2：標題看起來像大字，就一定要用 `h1` 或 `h2`

錯。
應該先看它是不是內容中的標題，再決定要不要用 heading。

### 迷思 3：只要 CSS 改了樣式，標題的意義就會消失

錯。
CSS 只會改外觀，不會改變標題標籤本身的語意角色。
