HTML 標題標籤 Demo

`h1` 到 `h6` 不只是字體變大變小而已。它們真正的角色是表達內容層級、建立文件結構，讓人和工具都能理解「這一段是在講什麼、屬於哪個主題之下」。

層級語意 `h1` 到 `h6` 代表標題層級，不只是預設字體大小。
結構關係 `h2` 應該屬於某個 `h1` 之下，`h3` 應該再屬於對應的 `h2`。
常見錯誤 不能只因為想要字大，就拿 heading 當裝飾；也不要隨便跳級。
CSS 的角色 CSS 負責樣式，heading 負責語意，兩者不是互相替代。

3. 標題跳級的問題

        

          

            較穩定
            
層級循序展開

            
<h1>JavaScript 教學</h1>
<h2>基礎語法</h2>
<h3>let 與 const</h3>

            

              `h3` 清楚地屬於「基礎語法」之下，結構連續且可推理。
            

          


          

            不建議
            
直接從 `h1` 跳到 `h3`

              少了中間層，讀者和工具都更難判斷這個子主題究竟掛在哪一層之下。
            

          

        

      


      

        
4. heading 與 CSS 大字不是同一件事

        

          

            

              
              
            

          


          

            
semantic

            
這是有結構意義的標題

            

              heading 不只是字大，它會告訴人和工具：這裡是一個章節或段落的標題。
            

            

            

            

              如果只是想改視覺大小，應該優先思考是否真的需要 heading 語意。
            

          

        

      


      

        
5. 常見實務提醒

        

          

            先想結構再選 heading
            不要先想字要多大，而是先想這段內容在整份文件裡屬於哪一層。
          

          

            不要把每個大字都做成 `h1`
            `h1` 應該用來代表整篇或整塊內容的最高層主題。
          

          

            樣式交給 CSS
            heading 負責語意，字重、大小、顏色、留白則應該交給 CSS 處理。
