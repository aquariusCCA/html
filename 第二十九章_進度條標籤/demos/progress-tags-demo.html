第二十九章：進度條標籤 Demo

這個示例用原生 <progress> 表示任務完成進度，並對照定進度、不定進度、value="0"、max 預設值、JavaScript 動態更新，以及和 <meter> 的差異。

核心語意

<progress> 表示「任務完成到哪裡」，不是表示電量、分數、磁碟用量這類狀態值。

任務進度

上傳、下載、安裝、載入、批次處理，才是 <progress> 的主場景。

value / max

value 是目前完成量，max 是總工作量，最小值固定為 0。

省略 value

沒有 value 是不定進度，不是 0%。

1. 常見進度任務範例

檔案上傳 35 / 100
35%

這是定進度，因為已知目前完成比例。

下載進度 8 / 10
8 / 10

max 不一定要是 100，只要表示總工作量即可。

資料載入中 未知比例

沒有 value 時是不定進度，代表正在處理但不知道完成比例。

程式內部比例 0.75 / 1
75%

如果程式用 0 到 1 表示比例，也可以直接搭配 max="1"。

2. 定進度、不定進度與 0%

狀態 寫法 顯示 語意
定進度 <progress value="50" max="100"> 50% 知道目前完成比例，這裡是 50%。
不定進度 <progress> 任務正在進行，但不知道完成比例。
進度為 0 <progress value="0" max="100"> 0% 這是定進度，目前完成 0%。
已完成 <progress value="100" max="100"> 100% 任務完成。

3. 互動觀察：動態更新 value

互動 progress 40%
40%
目前完成 40 / 100。
<label for="interactiveProgress">批次處理進度：</label>
<progress id="interactiveProgress" value="40" max="100">40%</progress>

JavaScript 更新重點

progress.value = current;
text.textContent = `${percent}%`;

原生 <progress> 已有進度語意，JavaScript 只需要更新 value 和輔助文字。

不要只靠條狀外觀

建議搭配 <label> 與文字百分比，讓使用者知道這條進度代表哪個任務。

4. max 預設值與沒有 min

max 預設是 1

<progress value="0.5">50%</progress>
50%

沒有寫 max 時，0.5 / 1 表示完成一半。

progress 不支援 min

<progress min="10" value="50" max="100"></progress>

任務進度最小值固定是 0。如果需要 min、low、high、optimum，通常該重新判斷是否要用 <meter>。

5. <progress> vs <meter>

下載 80%：任務進度

80% 
<progress value="80" max="100">80%</progress>

電量 80%：狀態值

80% 
<meter value="80" min="0" max="100">80%</meter>

6. 常見錯誤

把狀態值寫成進度

手機電量、磁碟用量、考試分數通常不是任務進度，應考慮 <meter>。

以為省略 value 是 0%

<progress> 是不定進度；value="0" 才是 0%。

只放進度條不放文字

建議用 <label>、百分比文字或狀態說明補足可讀性。

7. 判斷流程

1. 任務完成到哪裡？

是上傳、下載、安裝、載入、批次處理，通常用 <progress>。

2. 狀態落在哪裡？

是電量、分數、磁碟用量、溫度，通常用 <meter>。

3. 只是普通資訊？

如果不是任務進度，也不是範圍值，可能直接用文字或其他 UI 更合適。
