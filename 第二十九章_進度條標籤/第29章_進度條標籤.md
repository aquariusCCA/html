# 進度條標籤（`<progress>`）

> - 所屬章節：第二十九章｜進度條標籤  
> - 關鍵字：進度條標籤、`progress`、進度條、loading、上傳進度、下載進度、不定進度、`meter`、無障礙、動態更新  
> - 建議回查情境：想知道 `<progress>` 是做什麼的、想表示上傳或下載進度、想分清 `<progress>` 和 `<meter>` 的差別、想確認 `value` / `max` 怎麼寫、想用 JavaScript 動態更新進度時

## 本節導讀

這篇整理 HTML5 的 `<progress>` 標籤。

`<progress>` 用來表示「任務目前完成了多少」，常見於檔案上傳、檔案下載、安裝流程、資料載入、批次處理等情境。

這一章最重要的觀念是：

> `<progress>` 表示任務進度，不是單純表示某個數值大小。

所以它適合用在「事情正在進行，而且可以或不可以知道完成比例」的場景。

讀這章時，請特別注意三件事：

1. `max` 是總工作量。
2. `value` 是目前已完成的量。
3. 沒有 `value` 時，不代表 `0%`，而是「不定進度」。

## 你會在這篇學到什麼

- `<progress>` 在解決什麼問題
- `max` 和 `value` 分別代表什麼
- 定進度與不定進度有什麼差別
- 為什麼 `<progress>` 沒有 `min`
- `<progress>` 和 `<meter>` 要怎麼分

## 30 秒複習入口

- `<progress>` 用來表示任務完成進度。
- 常見場景：上傳、下載、安裝、載入、批次處理。
- `max` 表示總工作量，預設值是 `1`。
- `value` 表示目前完成量。
- `value` 可以是整數，也可以是小數。
- 若省略 `value`，就是不定進度，不是 `0%`。
- `<progress>` 沒有 `min` 屬性，最小值固定是 `0`。
- 如果你要表達電量、磁碟用量、分數、溫度這種狀態值，通常應該用 `<meter>`。
- 進度條最好搭配文字說明或 `<label>`，不要只靠外觀讓使用者猜意思。

## 速查區

### 核心概念

| 問題 | 答案 |
|---|---|
| `<progress>` 是什麼？ | HTML5 用來表示任務進度的標籤 |
| 它表示什麼？ | 一個任務完成到哪裡 |
| 它通常長什麼樣？ | 瀏覽器通常會顯示成進度條 |
| 它適合用在哪裡？ | 上傳、下載、安裝、資料載入、批次任務 |
| 它不適合用在哪裡？ | 電量、分數、磁碟使用量、溫度等狀態值 |

### 基本語法

```html
<progress value="50" max="100"></progress>
```

這段表示：

- 任務總量是 `100`
- 目前完成了 `50`
- 完成比例是 `50 / 100`
- 所以可理解為目前完成 `50%`

### 常用屬性

| 屬性 | 說明 | 補充 |
|---|---|---|
| `max` | 任務總量或目標值 | 預設是 `1`，必須大於 `0` |
| `value` | 目前已完成的量 | 必須介於 `0` 和 `max` 之間 |
| `min` | 不支援 | `<progress>` 的最小值固定是 `0` |

### 進度狀態

| 狀態 | 寫法 | 意義 |
|---|---|---|
| 定進度 | `<progress value="50" max="100"></progress>` | 知道目前完成比例 |
| 不定進度 | `<progress></progress>` | 任務正在進行，但不知道完成比例 |
| 進度為 0 | `<progress value="0" max="100"></progress>` | 任務尚未完成任何進度 |
| 進度完成 | `<progress value="100" max="100"></progress>` | 任務已完成 |

### 一句話抓核心

> `<progress>` 是在描述「任務完成比例」，不是在描述「某個數值落在哪個範圍」。

## 正文筆記

### 1. `<progress>` 在解決什麼問題？

有些介面需要讓使用者知道某個任務目前進行到哪裡，例如：

- 檔案已經上傳多少
- 檔案下載完成多少
- 安裝流程跑到哪一步
- 資料載入是否還在進行
- 系統批次處理完成多少

這些資訊不是單純顯示一個數字，而是要表達「任務進度」。

`<progress>` 就是為這類情境設計的。

它的語意是：

> 目前有一個任務正在進行，這個標籤用來表示它完成了多少。

---

### 2. 為什麼不要只用 `<div>` 做進度條？

你當然可以用 `<div>` 加 CSS 做出一條進度條，例如：

```html
<div class="progress-bar">
  <div class="progress-value"></div>
</div>
```

但這種寫法本身沒有 HTML 語意。

瀏覽器和輔助工具只會看到幾個普通的 `<div>`，不一定知道這是一個進度條。

如果使用 `<progress>`：

```html
<progress value="50" max="100"></progress>
```

HTML 本身就帶有「進度」語意。

所以在可以使用原生標籤的情況下，優先考慮 `<progress>`，再視覺需求決定要不要額外用 CSS 客製化。

---

### 3. `max` 怎麼理解？

`max` 表示任務的總工作量或目標值。

```html
<progress value="30" max="100"></progress>
```

這裡的意思是：

- 總進度是 `100`
- 目前完成 `30`
- 目前完成比例是 `30%`

`max` 沒有指定時，預設值是 `1`。

所以這兩種寫法的觀念類似：

```html
<progress value="0.5"></progress>
```

```html
<progress value="50" max="100"></progress>
```

兩者都可以理解為完成了一半。

不過實務上，如果你要用百分比概念，通常會明確寫成：

```html
<progress value="50" max="100"></progress>
```

這樣比較直覺，也比較方便自己和其他工程師閱讀。

---

### 4. `value` 怎麼理解？

`value` 表示目前已完成的量。

```html
<progress value="75" max="100"></progress>
```

這表示任務已完成 `75 / 100`。

也就是目前完成 75%。

`value` 需要注意：

- `value` 不能小於 `0`
- `value` 不應大於 `max`
- `value` 可以是整數
- `value` 也可以是小數

例如：

```html
<progress value="0.6" max="1"></progress>
```

這表示目前完成 `0.6 / 1`，也就是 60%。

---

### 5. 為什麼 `<progress>` 沒有 `min`？

`<progress>` 沒有 `min` 屬性。

它的最小值固定是 `0`。

因為「任務進度」通常是從 `0` 開始：

- `0`：尚未完成
- `max`：完全完成

所以你不需要，也不能像 `<meter>` 一樣寫：

```html
<!-- 錯誤觀念：progress 不支援 min -->
<progress min="10" value="50" max="100"></progress>
```

即使你寫了 `min`，也不應該把它當成有效語意來理解。

正確觀念是：

```html
<progress value="50" max="100"></progress>
```

這裡的進度範圍就是從 `0` 到 `100`。

---

### 6. 什麼是定進度？

「定進度」是指你知道任務目前完成多少比例。

例如檔案上傳時，瀏覽器或程式可能知道：

- 檔案總大小：100MB
- 已上傳大小：40MB

這時就可以顯示明確進度：

```html
<label for="upload">上傳進度：</label>
<progress id="upload" value="40" max="100"></progress>
```

這代表目前完成 40%。

定進度的特徵是：

- 有 `value`
- 有明確完成量
- 可以計算百分比
- 使用者知道目前進度到哪裡

---

### 7. 什麼是不定進度？

「不定進度」是指任務正在進行，但目前不知道完成比例。

例如：

- 頁面正在初始化
- 系統正在等待伺服器回應
- 背景正在載入資料，但不知道總資料量
- 某個流程正在處理，但無法估算還要多久

這時就不應該亂寫一個假的百分比。

可以省略 `value`：

```html
<label for="loading">資料載入中：</label>
<progress id="loading"></progress>
```

這表示：

> 任務正在進行，但目前沒有明確完成比例。

注意：

```html
<progress></progress>
```

不等於：

```html
<progress value="0" max="100"></progress>
```

兩者語意不同。

| 寫法 | 意義 |
|---|---|
| `<progress></progress>` | 不定進度，不知道完成比例 |
| `<progress value="0" max="100"></progress>` | 定進度，目前完成 0% |

---

### 8. `<progress>` 和 `<meter>` 差在哪？

這是本章最容易混淆的地方。

兩者外觀都可能像一條橫條，但語意完全不同。

| 標籤 | 表示什麼 | 適合情境 |
|---|---|---|
| `<progress>` | 任務完成進度 | 上傳、下載、安裝、載入 |
| `<meter>` | 數值在已知範圍中的位置 | 電量、磁碟用量、分數、溫度、評分 |

可以這樣判斷：

> 如果你想表達「事情完成到哪裡」，用 `<progress>`。  
> 如果你想表達「某個數值目前是多少」，用 `<meter>`。

### 適合用 `<progress>` 的例子

```html
<label for="download">下載進度：</label>
<progress id="download" value="80" max="100"></progress>
```

這表示下載任務完成 80%。

### 適合用 `<meter>` 的例子

```html
<label for="battery">電量：</label>
<meter id="battery" value="80" min="0" max="100">80%</meter>
```

這表示目前電量是 80%。

電量不是「任務完成進度」，而是「狀態值」，所以更適合用 `<meter>`。

## 實務範例

### 範例 1：固定進度

```html
<span>目前進度：</span>
<progress value="50" max="100"></progress>
```

這表示目前完成 50%。

---

### 範例 2：搭配 `<label>`

```html
<label for="file-upload">檔案上傳進度：</label>
<progress id="file-upload" value="30" max="100"></progress>
```

這種寫法比單純放一個進度條更清楚。

因為使用者可以知道這條進度條代表什麼，輔助工具也比較容易理解它的用途。

---

### 範例 3：不定進度

```html
<label for="loading">資料載入中：</label>
<progress id="loading"></progress>
```

這表示資料正在載入，但目前不知道完成比例。

適合用在「正在處理，但還不能計算百分比」的場景。

---

### 範例 4：用 JavaScript 動態更新進度

```html
<label for="upload-progress">上傳進度：</label>
<progress id="upload-progress" value="0" max="100"></progress>
<span id="upload-text">0%</span>

<script>
  const progress = document.querySelector('#upload-progress');
  const text = document.querySelector('#upload-text');

  let current = 0;

  const timer = setInterval(() => {
    current += 10;

    progress.value = current;
    text.textContent = `${current}%`;

    if (current >= 100) {
      clearInterval(timer);
      text.textContent = '上傳完成';
    }
  }, 500);
</script>
```

這個範例重點是：

- `<progress>` 負責表示進度語意
- `value` 會隨著任務進度更新
- 旁邊的文字讓使用者更清楚目前狀態
- 當進度到 `100` 時，代表任務完成

---

### 範例 5：用小數表示比例

```html
<label for="task-progress">任務進度：</label>
<progress id="task-progress" value="0.75" max="1"></progress>
```

這表示完成 75%。

因為 `max` 是 `1`，所以：

```text
0.75 / 1 = 75%
```

這種寫法在程式內部已經使用 `0 ~ 1` 表示比例時很常見。

## 使用注意事項

### 1. 不要把不定進度誤解成 0%

```html
<progress></progress>
```

這不是 0%。

它表示目前不知道完成比例。

如果要表示 0%，應該寫：

```html
<progress value="0" max="100"></progress>
```

---

### 2. 不要用 `<progress>` 表示狀態值

以下情境通常不適合用 `<progress>`：

- 手機電量
- 磁碟使用量
- 考試分數
- 溫度
- 評分
- 健康值
- 剩餘容量

這些比較像「目前數值是多少」，不是「任務完成到哪裡」。

因此通常應該使用 `<meter>`。

---

### 3. 不要只靠顏色或外觀傳達資訊

不建議只放一條進度條，讓使用者自己猜意思。

比較好的寫法是加上文字：

```html
<label for="install">安裝進度：</label>
<progress id="install" value="60" max="100"></progress>
<span>60%</span>
```

這樣即使使用者看不清楚進度條，也能從文字知道目前狀態。

---

### 4. `<progress>` 裡面的文字不是主要標籤

你可能會看到這種寫法：

```html
<progress value="50" max="100">50%</progress>
```

中間的 `50%` 可以當作舊瀏覽器不支援 `<progress>` 時的備援內容。

但實務上，不建議只依賴這段文字來說明進度條。

更清楚的寫法是：

```html
<label for="progress">處理進度：</label>
<progress id="progress" value="50" max="100">50%</progress>
<span>50%</span>
```

這樣畫面、語意與備援內容都比較完整。

---

### 5. 樣式可以調整，但語意不要丟掉

如果只是想改外觀，不一定要放棄 `<progress>`。

你可以保留 `<progress>` 的語意，再用 CSS 調整樣式。

如果真的需要完全客製化的進度條，才考慮用 `<div>` 搭配 ARIA 屬性，例如 `role="progressbar"`、`aria-valuenow`、`aria-valuemin`、`aria-valuemax`。

對初學者來說，先掌握原生 `<progress>` 的語意比較重要。

## 常見錯誤整理

### 錯誤 1：把 `<progress>` 當成所有橫條 UI

錯誤觀念：

```html
<progress value="80" max="100"></progress>
```

如果這個 `80` 表示手機電量，就不適合用 `<progress>`。

比較適合：

```html
<meter value="80" min="0" max="100">80%</meter>
```

---

### 錯誤 2：以為不寫 `value` 就是 0%

錯誤觀念：

```html
<progress max="100"></progress>
```

這不是 0%。

它是「不定進度」。

如果你要表示 0%，要寫：

```html
<progress value="0" max="100"></progress>
```

---

### 錯誤 3：幫 `<progress>` 寫 `min`

錯誤觀念：

```html
<progress min="0" value="30" max="100"></progress>
```

`<progress>` 沒有 `min` 屬性。

正確寫法：

```html
<progress value="30" max="100"></progress>
```

---

### 錯誤 4：`value` 大於 `max`

不建議：

```html
<progress value="120" max="100"></progress>
```

`value` 應該介於 `0` 和 `max` 之間。

比較合理：

```html
<progress value="100" max="100"></progress>
```

---

## 實務判斷流程

當你不確定要用 `<progress>` 還是 `<meter>` 時，可以照這個順序判斷：

1. 這個數值是在表示「某個任務完成到哪裡」嗎？
   - 是：用 `<progress>`
   - 否：繼續判斷

2. 這個數值是在表示「某個狀態落在已知範圍中的位置」嗎？
   - 是：用 `<meter>`
   - 否：可能只是普通文字或其他 UI

### 判斷例子

| 情境 | 建議標籤 | 原因 |
|---|---|---|
| 檔案上傳到 70% | `<progress>` | 任務進度 |
| 檔案下載到 30% | `<progress>` | 任務進度 |
| 安裝完成 90% | `<progress>` | 任務進度 |
| 手機電量 70% | `<meter>` | 狀態值 |
| 磁碟使用量 80% | `<meter>` | 狀態值 |
| 考試分數 85 分 | `<meter>` 或文字 | 狀態值，不是任務進度 |

## 常見問法

### `<progress>` 可以拿來當 loading 條嗎？

可以。

如果 loading 是某個任務的進行狀態，例如資料載入、檔案處理、安裝流程，就可以使用 `<progress>`。

如果可以知道完成比例，就寫 `value`：

```html
<progress value="40" max="100"></progress>
```

如果不知道完成比例，就不要寫 `value`：

```html
<progress></progress>
```

---

### `value` 沒寫是不是等於 `0`？

不是。

沒有 `value` 代表不定進度，也就是任務正在進行，但目前不知道完成比例。

`value="0"` 才代表目前完成 0%。

---

### `<progress>` 可以寫 `min` 嗎？

不可以。

`<progress>` 的最小值固定是 `0`，沒有 `min` 屬性。

如果你需要 `min`、`low`、`high`、`optimum` 這類範圍判斷，通常代表你應該思考是不是要用 `<meter>`。

---

### `max` 一定要寫 `100` 嗎？

不一定。

`max` 可以是任何大於 `0` 的數值。

例如：

```html
<progress value="3" max="10"></progress>
```

這表示完成 `3 / 10`。

也可以寫：

```html
<progress value="0.3" max="1"></progress>
```

這也表示完成 30%。

只是對初學者來說，`max="100"` 最直覺。

---

### `<progress>` 可以用 CSS 改樣式嗎？

可以，但不同瀏覽器對進度條的預設樣式與可客製化程度不同。

初學階段先記住：

- HTML 負責語意
- CSS 負責外觀
- JavaScript 負責更新進度

不要因為想改外觀，就直接放棄 `<progress>` 的語意。

## 本章重點整理

### 必記觀念

- `<progress>` 表示任務進度。
- `max` 表示總工作量，預設是 `1`。
- `value` 表示目前完成量。
- 沒有 `value` 是不定進度。
- `<progress>` 沒有 `min`，最小值固定是 `0`。
- `value` 應該介於 `0` 和 `max` 之間。
- `<progress>` 和 `<meter>` 的差別在語意，不在外觀。

### 最容易考的點

| 題目 | 答案 |
|---|---|
| `<progress value="50" max="100">` 是什麼意思？ | 完成 50% |
| `<progress>` 沒有 `value` 是什麼意思？ | 不定進度 |
| `<progress>` 可以寫 `min` 嗎？ | 不可以 |
| 電量 80% 適合用 `<progress>` 嗎？ | 不適合，通常用 `<meter>` |
| `max` 沒寫時是多少？ | 預設是 `1` |

## 自測問題

1. `<progress>` 最適合拿來表示哪一類資訊？
2. `max` 和 `value` 分別代表什麼？
3. 為什麼 `value` 缺省時不是 `0%`？
4. `<progress value="0" max="100">` 和 `<progress max="100">` 有什麼差別？
5. 為什麼 `<progress>` 沒有 `min`？
6. 手機電量 50% 為什麼通常不該用 `<progress>`？
7. 如果檔案上傳進度是 75%，你會怎麼寫？
8. 如果資料正在載入，但不知道完成比例，你會怎麼寫？
9. 如果要用 JavaScript 更新進度條，應該更新哪個屬性？
10. `<progress>` 和 `<meter>` 最大的差別是什麼？

1. 任務完成進度，例如上傳、下載、安裝、載入。
2. `max` 是總工作量，`value` 是目前完成量。
3. 因為省略 `value` 代表不定進度，表示不知道目前完成比例。
4. 前者是定進度且目前為 0%，後者是不定進度。
5. 因為任務進度的最小值固定是 `0`。
6. 因為電量是狀態值，不是任務完成進度。
7. `<progress value="75" max="100"></progress>`
8. `<progress></progress>`
9. 更新 `value` 屬性。
10. `<progress>` 表示任務進度，`<meter>` 表示數值在已知範圍中的位置。
