# 第二十九章｜進度條標籤

> 章節定位：理解 `<progress>` 如何表示任務完成進度  
> 建議閱讀方式：先掌握 `max` / `value` 的角色，再回看定進度與不定進度的差別

## 本章在學什麼

這一章整理 HTML5 的進度條標籤 `<progress>`。  
重點是建立一個穩定觀念：它表示的是任務完成到哪裡，而不是某個狀態值落在哪個區間。

## 本章與整體地圖的關係

- 前置知識：前一章已學過 `<meter>` 這類範圍值標籤，這一章剛好可以對照理解「進度」與「狀態值」的差別。
- 這章的核心任務：掌握 `max`、`value`、不定進度，以及 `<progress>` / `<meter>` 的分工。
- 讀完後通常接：之後在做上傳、下載、安裝或載入中的任務提示時，都可以回來查這一章。

## 易混淆主題

- `<progress>` vs `<meter>`：前者是任務進度，後者是狀態值。
- `value` 缺省 vs `0`：前者是不定進度，後者才是進度為零。
- 看起來像條狀元件 vs 用對語意：重點不在外觀，而在資料型態。
