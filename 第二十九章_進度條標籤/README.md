> - 章節定位：理解 `<progress>` 如何用 HTML 語意表示「任務完成進度」  
> - 建議閱讀方式：先掌握 `max` / `value`，再分清「定進度 / 不定進度」，最後對照 `<progress>` 與 `<meter>` 的使用情境

## 本章在學什麼

這一章整理 HTML5 的進度條標籤 `<progress>`。

`<progress>` 用來表示一個任務目前完成到哪裡，常見於檔案上傳、檔案下載、安裝流程、資料載入、批次處理等情境。

本章的核心觀念是：

> `<progress>` 表示任務進度，不是單純表示某個狀態值。

也就是說，當你想表達「事情完成到哪裡」時，才適合使用 `<progress>`；如果你想表達電量、分數、磁碟用量這類狀態值，通常應該改用 `<meter>`。

## 易混淆主題

- `<progress>` vs `<meter>`：前者是任務進度，後者是狀態值。
- 定進度 vs 不定進度：有 `value` 通常是定進度；沒有 `value` 是不定進度。
- `value` 缺省 vs `value="0"`：前者是不定進度，後者才是完成 0%。
- `max="100"` vs `max="1"`：兩者都可以表示比例，只是 `100` 較適合百分比閱讀，`1` 較適合程式內部比例。
- `<progress>` 沒有 `min`：它的最小值固定是 `0`。
- 看起來像條狀元件 vs 用對語意：重點不在外觀，而在資料型態與語意。
- 原生 `<progress>` vs 自製 `<div>` 進度條：前者自帶 HTML 語意，後者需要額外補 ARIA 才能接近相同語意。

## 建議練習

1. 寫一個固定進度條，顯示任務完成 50%。
2. 寫一個不定進度條，表示資料載入中但不知道完成比例。
3. 寫一個 `value="0"` 的進度條，觀察它和不定進度的差異。
4. 寫一個 JavaScript 範例，每 0.5 秒讓進度增加 10%。
5. 判斷以下情境該用 `<progress>` 還是 `<meter>`：
   - 檔案上傳 70%
   - 手機電量 80%
   - 安裝進度 90%
   - 磁碟使用量 60%
   - 資料載入中但不知道完成比例
