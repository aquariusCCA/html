核心判斷

`iframe` 適合載入獨立文件或第三方提供的嵌入內容；如果只是同一個網站內的共用 UI，通常應該用元件、模板或一般 HTML/CSS 結構處理。

`src`

指定 iframe 要載入哪一份 HTML、PDF 或外部可嵌入資源。

`title`

描述框架用途，讓閱讀者與輔助工具知道這個內嵌區塊是什麼。

`name`

提供目標名稱，讓 `` 或 `

` 可以把結果送進指定 iframe。

2. PDF 預覽

PDF 是 `iframe` 常見用途之一。這裡直接載入章節內的 `media/test.pdf`，並用 CSS 控制邊框與尺寸。

3. 用 CSS 取代 `frameborder`

舊教材可能會看到 `frameborder="0"`。現在寫新頁面時，優先用 CSS 管理外觀。

舊式可讀

<iframe
  src="./page.html"
  frameborder="0">
</iframe>

現代建議

<iframe
  src="./page.html"
  title="內嵌頁面"
  class="preview-frame">
</iframe>

.preview-frame {
  border: 1px solid #cad8d0;
}

5. 外部網站不一定能嵌入

語法正確不代表一定能顯示。若對方網站用安全標頭禁止嵌入，瀏覽器會阻擋 iframe 顯示。

看到的現象 常見判斷
本地 HTML 可以，外部網站空白 可能是對方網站禁止被 iframe 嵌入。
Console 出現 `X-Frame-Options` 對方明確限制頁面不能被放進框架。
Console 出現 `frame-ancestors` 對方用 CSP 指定允許嵌入它的來源。

6. 適合與不適合的情境

適合
  • PDF 預覽
  • 第三方提供的地圖或影片嵌入碼
  • 獨立 HTML 文件展示
  • 教學示範 `name` / `target`
不適合
  • 共用 header 或 footer
  • 一般元件化 UI
  • 需要頻繁同步狀態的畫面
  • 需要完整掌控 SEO 與路由的頁面

7. 常見錯誤對照

`target` 和 `name` 不一致

`target="contentFrame"` 卻寫成 `iframe name="container"`，瀏覽器找不到指定框架。

忘記 `title`

iframe 仍可能運作，但閱讀者與輔助工具不容易判斷框架用途。

把嵌入限制誤判成 CORS

iframe 被擋常見於 `X-Frame-Options` 或 `frame-ancestors`，不等於 JS 跨來源讀取問題。
