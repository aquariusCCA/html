# 框架標籤（iframe）

> - 所屬章節：第二十五章｜框架標籤  
> - 關鍵字：iframe、inline frame、內嵌框架、頁面嵌入、target、name、PDF 預覽、X-Frame-Options、CSP frame-ancestors、frameborder  
> - 建議回查情境：想把另一個頁面、PDF、影片、地圖或查詢結果嵌進目前頁面；想理解 `target` 為什麼能把內容顯示到指定框架；想判斷網站無法被嵌入是語法問題還是安全限制

## 本節導讀

HTML 的「框架標籤」在現代實務中主要指的是 `iframe`。

`iframe` 的作用是：**在目前頁面中建立一個獨立的內嵌顯示區塊，用來載入另一份文件或外部內容**。

你可以把它想成「頁面中的頁面」。例如：

- 在目前頁面中預覽 PDF
- 在固定區塊中顯示另一個 HTML 頁面
- 點擊連結後，把內容載入指定區塊
- 表單送出後，把回應結果顯示在指定區塊
- 嵌入地圖、影片播放器或第三方服務提供的可嵌入內容

學習這一章時，要掌握三條主線：

1. `iframe` 是用來「嵌入另一份內容」的。
2. `name` 可以配合 `<a>` 或 `<form>` 的 `target`，讓內容顯示到指定的 `iframe`。
3. 並不是所有網站都能被嵌入；有些網站會透過安全設定禁止自己出現在別人的 `iframe` 裡。

## 30 秒複習入口

- `iframe` 用來在目前頁面中嵌入另一份內容。
- `src` 指定要載入的內容來源。
- `title` 用來描述這個 `iframe` 的用途，對輔助工具與可讀性有幫助。

## 速查區

### 核心概念

`iframe` 是 inline frame，也就是「內嵌框架」。

它會在目前 HTML 文件中建立一個獨立的瀏覽區域，這個區域可以載入另一份內容。被載入的內容有自己的文件、自己的瀏覽上下文，也可能有自己的捲動、連結跳轉與表單行為。

簡單理解：

```text
目前頁面
├── 標題
├── 文章內容
└── iframe 區塊
    └── 另一份 HTML / PDF / 外部頁面
```

### 常用屬性速查

| 屬性 | 作用 | 備註 |
|---|---|---|
| `src` | 指定要載入的資源 URL | 可以是本地頁面、PDF、外部頁面等 |
| `title` | 描述 iframe 內容 | 建議加上，提升可讀性與無障礙體驗 |
| `name` | 設定 iframe 名稱 | 可讓 `<a>` 或 `<form>` 的 `target` 指向它 |
| `width` | 設定寬度 | 可用 HTML 屬性或 CSS 控制 |
| `height` | 設定高度 | 可用 HTML 屬性或 CSS 控制 |
| `style` / CSS | 控制外觀 | 現代寫法通常用 CSS 控制邊框、尺寸、排版 |
| `loading` | 控制載入時機 | 常見值：`lazy`，可延遲載入畫面外的 iframe |
| `sandbox` | 限制 iframe 內部能力 | 常用於提高安全性，但設定不當會影響表單、腳本或跳轉 |
| `allow` | 授權特定功能 | 例如全螢幕、相機、麥克風等能力 |
| `referrerpolicy` | 控制 Referer 傳送策略 | 和隱私與安全有關 |

> 初學階段最重要的是：`src`、`title`、`name`、`width`、`height`。  
> 進階實務再補：`loading`、`sandbox`、`allow`、`referrerpolicy`。

## 正文筆記

### 1. `iframe` 在做什麼？

`iframe` 可以在目前頁面中顯示另一份內容，而不需要整個頁面跳轉。

範例：在頁面中嵌入 PDF。

```html
<iframe
  src="./media/test.pdf"
  title="PDF 文件預覽"
  width="900"
  height="300"
  style="border: 1px solid #ccc;"
></iframe>
```

這段程式碼的意思是：

- `src="./media/test.pdf"`：載入 `test.pdf`
- `title="PDF 文件預覽"`：說明這個 iframe 的用途
- `width="900"`：寬度為 900
- `height="300"`：高度為 300
- `style="border: 1px solid #ccc;"`：用 CSS 設定邊框

### 2. 現在學框架標籤時，主角是誰？

現代 HTML 中，框架相關標籤的學習重點是 `iframe`。

舊式的 `frame`、`frameset` 已經不是現代新網站的主流寫法。它們過去常用來把整個瀏覽器視窗切成多個框架，但這種做法不利於現代網頁的排版、路由、狀態管理、SEO 與維護。

因此本章重點不是學舊式框架排版，而是理解：

```text
iframe = 在目前頁面中嵌入另一份內容
```

### 3. `src` 是什麼？

`src` 是 source 的意思，用來指定 `iframe` 要載入哪個資源。

```html
<iframe
  src="./pages/info.html"
  title="資訊頁面"
  width="600"
  height="300"
></iframe>
```

常見可放入 `src` 的內容包括：

- 本地 HTML 頁面
- PDF 文件
- 圖片或可被瀏覽器顯示的文件
- 第三方提供的嵌入網址
- 允許被嵌入的外部網站

但是要注意：**語法正確，不代表一定能顯示成功。**

如果對方網站禁止被嵌入，瀏覽器仍然會阻擋顯示。

### 4. `title` 為什麼建議加上？

`title` 用來描述這個 `iframe` 的內容或用途。

```html
<iframe
  src="./media/test.pdf"
  title="授信申請書 PDF 預覽"
  width="900"
  height="300"
></iframe>
```

不要只寫：

```html
<iframe src="./media/test.pdf"></iframe>
```

比較好的寫法是：

```html
<iframe
  src="./media/test.pdf"
  title="PDF 文件預覽"
></iframe>
```

原因是 `iframe` 本身可能只是一個空白框或外部內容，若沒有 `title`，讀者或輔助工具不容易判斷這個框架的用途。

### 5. `name` 是什麼？

`name` 是給 `iframe` 使用的名稱。

它的重點不是顯示給使用者看，而是讓其他元素可以透過 `target` 指向它。

```html
<iframe
  name="previewFrame"
  title="內容預覽區"
  width="900"
  height="300"
></iframe>
```

這裡的 `previewFrame` 就像是一個「目標區塊名稱」。

後續只要有連結或表單的 `target="previewFrame"`，瀏覽器就會把結果顯示到這個 `iframe` 裡。

### 6. 怎麼和超連結的 `target` 配合？

如果 `<a>` 的 `target` 值，和某個 `iframe` 的 `name` 一樣，點擊連結後，連結內容就會載入到該 `iframe` 裡。

```html
<a href="./media/test.pdf" target="previewFrame">查看 PDF</a>

<iframe
  name="previewFrame"
  title="PDF 預覽區"
  width="900"
  height="300"
  style="border: 1px solid #ccc;"
></iframe>
```

流程可以這樣理解：

```text
點擊 <a>
↓
href 指定要打開的內容
↓
target 指定要顯示的位置
↓
找到 name 相同的 iframe
↓
把內容載入該 iframe
```

所以這裡的關鍵配對是：

```html
<a target="previewFrame">
<iframe name="previewFrame"></iframe>
```

兩者名稱一致，才能形成對應關係。

### 7. 怎麼和表單的 `target` 配合？

表單送出時，`action` 決定資料要送到哪裡，`target` 決定伺服器回應結果要顯示在哪裡。

```html
<form action="https://example.com/search" target="resultFrame">
  <input type="text" name="keyword" />
  <input type="submit" value="搜尋" />
</form>

<iframe
  name="resultFrame"
  title="搜尋結果顯示區"
  width="900"
  height="300"
  style="border: 1px solid #ccc;"
></iframe>
```

流程可以這樣理解：

```text
填寫表單
↓
按下 submit
↓
表單資料送到 action 指定的位置
↓
伺服器回傳結果
↓
target 指定結果顯示到 resultFrame
↓
iframe name="resultFrame" 接收並顯示結果
```

因此表單和 `iframe` 的配合重點是：

```html
<form target="resultFrame">
<iframe name="resultFrame"></iframe>
```

注意：這種寫法很適合用來理解 `target` 的運作方式，但現代實務中，搜尋、查詢、表單提交結果通常也可能使用 AJAX、Fetch API、前端路由或元件狀態來處理。

### 8. 可以嵌入哪些內容？

常見可嵌入內容如下：

| 類型 | 是否常見 | 說明 |
|---|---|---|
| 本地 HTML | 常見 | 適合練習與簡單內嵌頁面 |
| PDF | 常見 | 常用於文件預覽 |
| 地圖 | 常見 | 通常由地圖服務提供嵌入碼 |
| 影片播放器 | 常見 | 例如影音平台提供的嵌入碼 |
| 外部網站 | 不一定 | 必須看對方是否允許被嵌入 |
| 表單回應頁 | 可用 | 可透過 `form target` 指向 iframe |

最重要的判斷是：

```text
不是你寫了 iframe，就代表對方網站一定允許被嵌入。
```

### 9. 為什麼有些網站不能被嵌入？

如果你寫了：

```html
<iframe
  src="https://example.com"
  title="外部網站"
></iframe>
```

但畫面沒有成功顯示，可能不是 HTML 寫錯，而是對方網站主動禁止自己被放進別人的 `iframe`。

常見安全限制包括：

- `X-Frame-Options`
- `Content-Security-Policy: frame-ancestors`

它們的目的通常是防止網站被惡意嵌入，進而造成點擊劫持等安全問題。

簡單判斷方式：

| 現象 | 可能原因 |
|---|---|
| 本地頁面可嵌入，外部網站不可嵌入 | 對方網站禁止 iframe |
| Console 出現 `X-Frame-Options` | 對方使用安全標頭限制嵌入 |
| Console 出現 `frame-ancestors` | 對方使用 CSP 限制誰能嵌入 |
| 語法正確但畫面空白 | 可能是安全限制、網路錯誤或資源不支援顯示 |

### 10. `iframe` 和 CORS 是同一件事嗎？

不是。

這裡很容易混淆。

- `iframe` 嵌入失敗，常見原因是對方網站限制「誰可以把我嵌入」。
- CORS 常見於 JavaScript 發出跨來源請求時，瀏覽器限制「誰可以讀取我的回應」。

簡化比較：

| 概念 | 主要在管什麼 |
|---|---|
| `X-Frame-Options` / `frame-ancestors` | 控制頁面能不能被 iframe 嵌入 |
| CORS | 控制跨來源請求後，前端 JS 能不能讀取回應 |

所以看到外部網站不能放進 `iframe`，不要第一時間就判斷成 CORS 問題。

### 11. 為什麼不建議把 `frameborder` 當成現代主力寫法？

以前常看到這種寫法：

```html
<iframe src="./media/test.pdf" frameborder="0"></iframe>
```

`frameborder` 的用途是控制框架邊框，但這是舊式寫法。

現代比較建議使用 CSS：

```html
<iframe
  src="./media/test.pdf"
  title="PDF 文件預覽"
  style="border: none;"
></iframe>
```

或：

```html
<iframe
  src="./media/test.pdf"
  title="PDF 文件預覽"
  class="pdf-preview"
></iframe>
```

```css
.pdf-preview {
  width: 900px;
  height: 300px;
  border: 1px solid #ccc;
}
```

學習時只要知道：

```text
看到 frameborder 看得懂即可；自己寫新程式時，優先用 CSS border。
```

### 12. 什麼時候適合用 `iframe`？

適合使用 `iframe` 的情境：

- 嵌入 PDF 預覽
- 嵌入第三方提供的地圖或影片
- 需要把外部頁面隔離在獨立區塊中
- 需要展示一份獨立 HTML 文件
- 教學時示範 `target` 和 `name` 的配合
- 嵌入一個和主頁面相對獨立的系統或工具

### 13. 什麼時候不適合用 `iframe`？

不適合使用 `iframe` 的情境：

- 只是想重複使用 header、footer 或側邊欄
- 只是想做一般元件化 UI
- 需要高度控制內部內容的樣式
- 需要和主頁面頻繁同步狀態
- 需要完整掌控 SEO、路由與使用者操作流程
- 可以用 Vue / React / 一般元件完成的畫面，卻硬用 iframe 拆開

簡單判斷：

```text
如果內容本身是一個獨立文件或第三方服務，可以考慮 iframe。
如果只是自己頁面中的一個 UI 區塊，通常不應該優先考慮 iframe。
```

### 14. `iframe` vs `frame` / `frameset`

| 項目 | `iframe` | `frame` / `frameset` |
|---|---|---|
| 現代實務使用 | 常見 | 幾乎不作為新網站主流寫法 |
| 用途 | 在頁面中嵌入另一份內容 | 把整個視窗切成多個框架 |
| 是否適合新手實作 | 適合 | 只需知道歷史概念即可 |
| 和現代前端整合 | 可以作為特定嵌入方案 | 不適合作為主要架構 |

本章重點放在 `iframe`，不是舊式 `frameset` 版面切割。

## 常見錯誤與查錯方式

### 錯誤 1：`target` 和 `name` 對不起來

錯誤寫法：

```html
<a href="./page.html" target="contentFrame">打開頁面</a>

<iframe name="container"></iframe>
```

問題：

```text
target="contentFrame"
name="container"
```

兩個名稱不同，所以連結不會載入到預期的 iframe。

正確寫法：

```html
<a href="./page.html" target="contentFrame">打開頁面</a>

<iframe
  name="contentFrame"
  title="內容顯示區"
></iframe>
```

### 錯誤 2：忘記加 `title`

可運作但不夠好的寫法：

```html
<iframe src="./page.html"></iframe>
```

較好的寫法：

```html
<iframe
  src="./page.html"
  title="內嵌頁面"
></iframe>
```

### 錯誤 3：以為外部網站都能嵌入

```html
<iframe
  src="https://some-website.com"
  title="外部網站"
></iframe>
```

如果失敗，請檢查瀏覽器 Console 是否出現：

- `X-Frame-Options`
- `Content-Security-Policy`
- `frame-ancestors`

這通常代表對方網站禁止被嵌入，不是你的 `iframe` 語法錯誤。

### 錯誤 4：把 `frameborder` 當成必背主力屬性

舊範例：

```html
<iframe src="./page.html" frameborder="0"></iframe>
```

現代建議：

```html
<iframe
  src="./page.html"
  title="內嵌頁面"
  style="border: none;"
></iframe>
```

或使用 class 統一管理樣式。

## 常見問法

### `iframe` 是什麼？

`iframe` 是內嵌框架，可以在目前頁面中載入另一份文件或外部內容。

### `src` 是什麼？

`src` 是 `iframe` 要載入的內容來源。

### `name` 是什麼？

`name` 是 `iframe` 的目標名稱，常用來讓 `<a>` 或 `<form>` 的 `target` 指向它。

### `target` 和 `name` 怎麼配合？

`target` 的值如果等於某個 `iframe` 的 `name`，內容就會載入那個 `iframe`。

```html
<a href="./page.html" target="mainFrame">打開頁面</a>

<iframe
  name="mainFrame"
  title="主內容區"
></iframe>
```

### 為什麼有些網站明明放進 `src` 卻顯示不出來？

常見原因是對方網站禁止被其他網站透過 `iframe` 嵌入。這通常和安全設定有關，不一定是 HTML 語法錯誤。

### `frameborder` 還需要背嗎？

不需要當成現代主力寫法背。看到舊教材中的 `frameborder` 時知道它是控制邊框即可；自己寫新程式時，通常用 CSS 的 `border`。

### `iframe` 可以取代 Vue 元件嗎？

通常不行。

`iframe` 適合嵌入獨立文件或第三方內容；Vue 元件適合拆分自己專案內的 UI 和邏輯。兩者用途不同。

## 自測答案

1. 在目前頁面中嵌入另一份內容。
2. `src` 指定內容來源；`title` 描述 iframe 用途；`name` 提供給 `target` 對應。
3. 因為瀏覽器會尋找 `name` 等於 `target` 的瀏覽上下文，若找到對應的 `iframe`，就會把內容載入其中。
4. `action` 決定表單送到哪裡；`target` 決定回應結果顯示在哪裡。
5. 因為對方網站可能透過安全標頭限制自己不能被其他網站嵌入。
6. 它們用來控制頁面是否允許被放進 `frame`、`iframe`、`object`、`embed` 等容器中。
7. 不一定。`iframe` 被擋通常更常見於嵌入限制；CORS 主要是限制前端 JavaScript 跨來源讀取回應。
8. 因為它是舊式寫法，現代控制邊框應優先使用 CSS。
9. 例如 PDF 預覽、地圖、影片、第三方服務、獨立 HTML 文件。
10. 例如一般 UI 元件、header/footer 共用、高度依賴主頁面狀態同步的功能。

## 一句話抓核心

`iframe` 的本質是「在目前頁面中開一個獨立顯示區塊載入另一份內容」，而 `name` 搭配 `target` 可以讓連結或表單結果顯示到指定的 `iframe`。
