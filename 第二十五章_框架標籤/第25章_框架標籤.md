## 本節導讀

這篇整理 HTML 中現在較常見的框架標籤：`iframe`。  
它的用途是在目前頁面中嵌入另一份內容，例如另一個 HTML 頁面、PDF 文件，或可被允許嵌入的外部頁面。

原始內容的方向正確，但有幾個明顯問題需要先修正：標題誤寫成第 26 章、主線只列屬性沒有先講清楚 `iframe` 在做什麼，且把 `frameborder` 當成主要屬性來記，這在現代寫法裡已不建議當作主流做法。  
這裡改成先建立整體觀，再看常見用法、`target` 配合方式，以及實務上容易踩到的限制。

## 30 秒複習入口

- `iframe` 用來在目前頁面中嵌入另一份內容。
- `src` 指定要載入的內容來源。
- `name` 可和超鏈接或表單的 `target` 配合使用。

## 速查區

### 核心概念

- `iframe` 是「頁面中的頁面」。
- 它能在目前文件裡顯示另一份文件內容。
- 常見情境包括嵌入本地頁面、PDF、地圖、影片播放器或其他可被允許嵌入的頁面。

### 關鍵規則 / 判準

- `src`：指定要顯示的資源。
- `name`：提供一個可被 `target` 指向的框架名稱。
- `width`、`height`：控制框架顯示尺寸。
- `frameborder`：屬於舊式屬性；若要控制邊框，現代通常改用 CSS `border`。
- 並不是所有外部網站都能被嵌入；有些網站會透過安全設定拒絕被放進 `iframe`。

### 常見使用場景

- 嵌入本地 HTML 頁面
- 嵌入 PDF 文件
- 點擊連結後在固定區塊顯示內容
- 表單送出後把結果顯示在指定區塊

### 常見混淆點

- `iframe` 不是舊式 `frame` / `frameset` 的直接照搬；現代網頁主要是使用 `iframe`。
- `name` 不是給使用者看的標題，而是給 `target` 對應的名稱。
- `frameborder="0"` 雖然還看得到舊教學使用，但不適合當成現代主要寫法。
- 外部網站不能嵌入時，問題常不在 HTML 拼錯，而在對方網站的安全限制。

### 一句話抓核心

- `iframe` 的重點是把另一份內容嵌進頁面，並可透過 `name` 與 `target` 形成固定顯示區塊。

## 正文筆記

### 1. `iframe` 在做什麼？

- `iframe` 是內嵌框架標籤。
- 它可以在當前頁面中顯示另一份文件內容，而不需要整個頁面跳轉。
- 如果你想保留目前頁面，同時在某個區塊展示另一份內容，就很可能會用到它。

### 2. 現在學框架標籤時，主角是誰？

- 在現代 HTML 中，實際最常見的是 `iframe`。
- 舊式的 `frame`、`frameset` 已不是新網站的主流做法。
- 因此這一章若要拿來實作或回查，重點應放在 `iframe`。

### 3. 常見屬性怎麼理解？

- `src`：要載入哪一份內容。
- `name`：這個框架的名稱，可讓超鏈接或表單用 `target` 指向它。
- `width`：框架寬度。
- `height`：框架高度。
- `frameborder`：舊式邊框控制寫法；若要控制邊框，現代通常用 CSS。

```html
<iframe src="./media/test.pdf" width="900" height="300" style="border: 1px solid #ccc;"></iframe>
```

### 4. 可以嵌入哪些內容？

- 本地 HTML 頁面
- PDF 文件
- 某些允許嵌入的外部頁面

要注意的是，不是所有網站都會接受被嵌入。  
有些網站會透過安全標頭限制自己不能出現在其他網站的 `iframe` 裡，因此就算語法正確，也可能無法顯示。

### 5. 怎麼和超鏈接的 `target` 配合？

- 如果超鏈接的 `target` 值，和某個 `iframe` 的 `name` 一致，連結內容就可以顯示在那個框架中。
- 這樣點擊連結時，就不一定要整頁跳轉。

```html
<a href="./media/test.pdf" target="container">查看 PDF</a>

<iframe
  name="container"
  width="900"
  height="300"
  style="border: 1px solid #ccc;"
></iframe>
```

### 6. 怎麼和表單的 `target` 配合？

- 表單送出後，結果預設通常會在目前頁面或新頁面中顯示。
- 若把 `<form>` 的 `target` 指向某個 `iframe` 的 `name`，送出結果就會顯示在該框架裡。
- 這種寫法在教學示例中常用來理解 `target`，但實務上是否採用，還要看整體介面與安全需求。

```html
<form action="https://so.toutiao.com/search" target="container2">
  <input type="text" name="keyword" />
  <input type="submit" value="搜索" />
</form>

<iframe
  name="container2"
  width="900"
  height="300"
  style="border: 1px solid #ccc;"
></iframe>
```

### 7. 為什麼不建議把 `frameborder` 當成現代主力寫法？

- 在舊教材或舊範例中，常會看到 `frameborder` 這個屬性。
- 但如果只是想控制 `iframe` 是否顯示邊框，現代通常會直接用 CSS `border`。
- 因此回查時知道這個屬性是什麼就夠了，不需要把它當成現在最主要的寫法來記。

## 常見問法

### `iframe` 和舊式 `frame` / `frameset` 差在哪？

- 現在實務上主要學的是 `iframe`。
- 舊式 `frame` / `frameset` 已不是新網站的主流方案。

### `name` 為什麼重要？

- 因為超鏈接或表單的 `target` 可以指向它。
- 有了 `name`，`iframe` 才能成為固定的顯示目標區塊。

### 為什麼有些網站明明寫進 `src` 卻顯示不出來？

- 常見原因是對方網站禁止被嵌入。
- 這通常和網站的安全設定有關，不一定是 HTML 語法寫錯。

### `frameborder` 還能不能看到？

- 在舊教材或舊範例裡還是可能看到。
- 但若是現代寫法，通常改用 CSS 控制邊框會比較合適。
