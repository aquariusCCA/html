第三十章：datalist 標籤 Demo

這個示例展示 <input list="..."> 如何綁定 <datalist id="...">，並對照 datalist、select、表單驗證與 autocomplete 的分工。

一句話核心

<datalist> 是幫 <input> 補上一份建議清單，不是建立強制只能選清單內值的下拉選單。

input 負責輸入

使用者真正輸入或送出的值都在 <input> 裡。

datalist 負責建議

<option value="..."> 定義候選建議值。

不是驗證工具

使用者通常仍可以輸入清單外的值。

1. 最小可用結構

<input id="browser" name="browser" type="text" list="browser-list">

<datalist id="browser-list">
  <option value="Chrome"></option>
  <option value="Firefox"></option>
</datalist>

送出資料預覽

欄位名稱
尚未送出
欄位值
尚未送出

2. 多種輸入建議場景

城市有常見選項，但仍可能允許輸入其他地區。

搜尋可以參考熱門字，但不應限制使用者只能搜尋這些字。

常用代碼可以先列出，降低輸入時拼錯的機率。

3. list、id、name 分工

屬性 放在哪裡 負責什麼
list <input> 指定要使用哪一份建議清單。
id <datalist> 提供給 input[list] 對應。
name <input> 表單送出時的欄位名稱，不是來自 datalist。
value <option> 真正會被填入輸入框的建議值。

4. datalist vs select

datalist：可輸入，也可參考建議

這裡可以輸入清單外的 Brave。

select：固定選項

如果需求是只能選固定值，select 通常更合適。

5. datalist 不是表單驗證

尚未檢查。
const allowed = ["台北市", "新北市", "桃園市"];
const isValid = allowed.includes(input.value);

6. 常見錯誤

以為 datalist 本身是輸入框

<datalist> 不會產生欄位；真正輸入的是 <input>。

list 和 id 對不上

<input list="browser-list">
<datalist id="browsers">...</datalist>

忘記 input name

沒有 name 時，表單送出不會有你期待的欄位名稱。

7. datalist vs autocomplete

datalist

建議值由開發者寫在 HTML 裡，例如城市、瀏覽器、商品代碼。

autocomplete

建議來源通常是瀏覽器、欄位語意或使用者過去輸入資料。

8. 判斷流程

1. 要讓使用者自由輸入嗎？

是的話，先用 input 思考。

2. 只是提供常見建議嗎？

是的話，datalist 很適合。

3. 必須只能選固定值嗎？

是的話，優先用 select 或額外驗證。
