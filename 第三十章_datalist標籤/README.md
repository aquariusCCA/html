# 第三十章｜`datalist` 標籤

> 章節定位：理解 `<datalist>` 如何替輸入框提供建議值  
> 建議閱讀方式：先掌握 `list` / `id` 綁定方式，再回看它和 `<select>` 的差別

## 本章在學什麼

這一章整理 HTML 的 `<datalist>` 標籤。  
重點是建立一個穩定觀念：它不是建立一個必選的下拉選單，而是替輸入框補上一份建議清單。

## 本章與整體地圖的關係

- 前置知識：前面已學過表單元素與選單類型，這一章補的是「帶建議值的輸入」這種互動模式。
- 這章的核心任務：理解 `<datalist>` 如何和 `<input>` 綁定，以及它和 `<select>` 的分工。
- 讀完後通常接：之後在做搜尋建議、常用城市輸入、瀏覽器名稱輸入等情境時，都可以回查這一章。

## 易混淆主題

- `<datalist>` vs `<select>`：前者是建議清單，後者是固定選項。
- `<datalist>` 本身 vs `<input>`：前者不是輸入框，後者才是。
- 顯示候選值 vs 強制限制值：前者不等於後者。
