# html5shiv 條件載入對照

## 1. Demo 目標

這個 demo 用來示範：

- 為什麼不應讓所有瀏覽器都載入 `html5shiv`。
- 如何用 IE 條件註解讓低於 IE9 的 IE 才載入相容腳本。
- `lt IE 9` 與 IE10 不支援條件註解的基本判斷。

## 2. 對應 notes

- 筆記路徑：`notes/第02章_兼容性問題/兼容性問題.md`
- 對應章節：`3.5 IE 條件註解`、`4.2 使用 html5shiv`、`7. 常見錯誤與修正`
- 對應知識點：
  - `html5shiv`
  - IE 條件註解
  - `lt IE 9`
  - 避免所有瀏覽器都載入舊版相容腳本

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 切換「現代瀏覽器」、「IE8」、「IE10」情境，觀察載入結果

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 直接寫 `<script src="./js/html5shiv.min.js"></script>` 會讓所有瀏覽器都載入。
2. `<!--[if lt IE 9]>` 代表低於 IE9 的 IE 才符合條件。
3. IE10 不支援條件註解，所以不能用這種方式讓 IE10 執行專用內容。

## 5. 程式碼重點

### HTML

- 左右兩欄分別呈現錯誤寫法與修正寫法。
- 使用 radio button 模擬不同瀏覽器情境。

### CSS

- 用不同顏色標示「會載入」與「不會載入」。

### JavaScript

- `renderResult()` 根據選取的瀏覽器情境更新載入結果。
- JavaScript 只是教學模擬，不是實際偵測瀏覽器。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把 `ie8` 的判斷改成 `selected === 'ie10'`，觀察為什麼這不符合 notes。
2. 在 HTML 中新增「IE7」選項，讓它也符合 `lt IE 9`。
3. 改寫狀態文字，用自己的話說明為什麼現代瀏覽器不需要 `html5shiv`。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 所有瀏覽器都載入 `html5shiv` | 對現代瀏覽器增加無意義負擔 | 使用 IE 條件註解限制載入範圍 |
| 以為 `html5shiv` 等於完整支援 HTML5 | 它主要處理 HTML5 元素識別，不補齊所有 API | 依需求另外處理 CSS、JavaScript 或 API 相容 |
| 忽略 IE10 不支援條件註解 | 錯判 IE10 會執行條件註解內容 | 把 IE10 視為不會執行這類條件註解 |
