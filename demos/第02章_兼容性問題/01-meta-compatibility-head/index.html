Demo 01

相容性 meta 放置位置

這個 demo 示範與瀏覽器解析、渲染模式相關的宣告，通常會放在 head 前段，讓瀏覽器盡早讀取。

本頁使用的 head 片段

<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="renderer" content="webkit">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>相容性 meta 放置位置</title>
</head>

每個設定的用途

寫法 主要用途 適用範圍
<!doctype html> 讓瀏覽器以 HTML5 標準模式解析文件。 所有 HTML5 文件。
X-UA-Compatible 提示舊版 IE 使用最新可用文件模式。 舊版 IE 相容處理。
renderer 提示部分雙核瀏覽器優先使用 WebKit 內核。 特定雙核瀏覽器環境。

觀察限制

在現代瀏覽器中，這些相容性設定通常不會產生明顯畫面差異。請用「檢視原始碼」或開發者工具觀察它們在文件中的位置。
