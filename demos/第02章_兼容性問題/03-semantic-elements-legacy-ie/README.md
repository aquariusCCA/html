# HTML5 語意化元素與舊版 IE

## 1. Demo 目標

這個 demo 用來示範：

- HTML5 語意化元素在現代瀏覽器中的正常結構。
- 舊版 IE 對 `header`、`nav`、`section`、`article`、`footer` 等新元素可能有識別問題。
- `html5shiv` 主要協助舊版 IE 建立並識別 HTML5 新元素，不是完整 HTML5 支援工具。

## 4. 觀察重點

1. 左側使用 HTML5 語意化元素呈現清楚的頁面結構。
2. 右側初始狀態刻意模擬舊版 IE 未正確識別新元素時的樣式問題。
3. 套用模擬狀態後，右側元素恢復成可被樣式化的區塊。

## 5. 程式碼重點

### HTML

- 範例使用 `header`、`nav`、`section`、`article`、`footer`。
- 兩側使用相同結構，方便觀察呈現差異。
- notes 也提到 `main`，但本 demo 不在同一頁中重複放置多個 `main`，避免示範出不理想的文件結構。

### CSS

- `.legacy-sample:not(.shiv-applied)` 用來模擬舊版環境中的不穩定呈現。
- 這是教學模擬，不是完整重現某一版 IE 的所有行為。

### JavaScript

- 按鈕只切換右側區塊的 class，讓學習者觀察差異。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 新增一個 `section` 到左右兩側，並補上對應 CSS。
2. 暫時刪除 `.legacy-sample:not(.shiv-applied)` 的規則，觀察兩側是否變得相同。
3. 修改按鈕文字，用自己的話描述 `html5shiv` 的用途。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 以為 `html5shiv` 能完整支援 HTML5 | 它不補齊所有新 API、表單功能或 CSS 能力 | 只把它視為特定舊版 IE 的元素識別補丁 |
| 使用 HTML5 語意化元素但忽略舊版 IE 支援範圍 | 舊系統中可能出現樣式或 DOM 行為問題 | 依專案支援範圍測試並決定是否使用 `html5shiv` |
| 新專案無條件加入 `html5shiv` | 增加不必要維護成本 | 若不支援 IE8 或更舊版本，通常可評估移除 |
