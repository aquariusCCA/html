# lang、dir 與 translate

## 1. Demo 目標

這個 demo 用來示範：

- `lang` 如何標示元素內容的語言。
- `dir` 如何影響文字呈現方向。
- `translate="no"` 如何提示專有名詞不建議被自動翻譯。
- 這些屬性和國際化、搜尋引擎、輔助工具的關係。

## 4. 觀察重點

1. `<html lang="zh-Hant">` 表示整份頁面的主要語言是繁體中文。
2. 英文段落可以用 `lang="en"` 標示局部語言。
3. `dir="rtl"` 會讓文字以從右到左的方向呈現。
4. `translate="no"` 適合用在術語、品牌名、書名或程式語法上。

## 5. 程式碼重點

### HTML

- 使用 `lang` 標示不同語言內容。
- 使用 `dir="ltr"`、`dir="rtl"`、`dir="auto"` 對照文字方向。
- 將英文術語標記為 `lang="en" translate="no"`。

### CSS

- CSS 只用來讓不同段落區塊更容易觀察。
- `[dir]` 選擇器用來標示有文字方向設定的段落。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 將 `dir="rtl"` 改成 `dir="ltr"`。
2. 將英文術語的 `translate="no"` 移除，思考自動翻譯工具可能如何處理它。
3. 新增一段英文或其他語言文字，並替它設定適合的 `lang`。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 整站沒有設定 `lang` | 輔助工具與搜尋引擎較難判斷主要語言 | 在 `<html>` 上設定主要語言 |
| 混合語言內容沒有局部標示 | 螢幕閱讀器可能用不適合的語音規則讀出 | 對局部語言片段加上 `lang` |
| 把 `translate` 當成安全保護 | 翻譯屬性不是資料保護機制 | 不要把敏感資料放進 HTML |
| 隨意設定 `dir` | 文字方向可能變得難以閱讀 | 只有在內容方向需要時才設定 |
