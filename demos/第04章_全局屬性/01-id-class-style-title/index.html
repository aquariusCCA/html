id、class、style 與 title 的用途差異

這個頁面用同一組文章卡片觀察：唯一識別用 `id`，重複分類用 `class`， 樣式管理優先使用 CSS class，重要說明不要只放在 `title`。

跳到唯一 notice 區塊 跳到 class 範例 跳到 style 範例

唯一區塊：`id` 適合識別單一元素

這個區塊使用 id="notice-panel"，上方頁內連結可以透過 href="#notice-panel" 跳到這裡。

樣式管理：優先用 class，少量才用 `style`

使用 class 管理樣式

樣式集中在 style.css，比較容易維護與重複使用。

使用行內 style

行內樣式會散落在 HTML 中，頁面變大時較難統一調整。

`title` 只適合補充，不適合承載重要資訊

將滑鼠停在 HTML 上，部分瀏覽器會顯示工具提示。

重要資訊應直接放在頁面可見文字中，而不是只放在 `title` 屬性。
