id、class、style 的用途差異

觀察唯一定位、分類套用與行內樣式的差異。這個 demo 不用大量文字摘要，而是讓屬性的作用直接出現在畫面上。

跳到唯一 id 元素

id：唯一定位

頁內連結或 JavaScript 通常會期待 `id` 指向唯一元素。

id="intro-target"

按上方連結或按鈕時，只有這個區塊會被定位。

<section id="intro-target">...</section>

class：重複分類

同一個 class 可以給多個元素共用，也能疊加多個 class。

class="notice"：第一則公告
class="notice"：第二則公告
class="notice important"：重要公告
.notice { border-left: 4px solid teal; }

style：直接設定這個元素

行內樣式適合少量測試，但大量使用會讓維護變困難。

這段文字可切換成行內樣式效果。 
<p style="color: red; font-weight: 700;">...</p>

常見錯誤對照

錯誤：重複使用同一個 id 
<article id="product-card">商品 A</article>
<article id="product-card">商品 B</article>
修正：可重複分類改用 class 
<article class="product-card">商品 A</article>
<article class="product-card">商品 B</article>
