# 常見錯誤：用 br 製造段落間距

## 1. Demo 目標

這個 demo 用來示範：

- `<br>` 表示換行，不應用來取代段落結構。
- 不同段落應使用不同的 `p` 元素。
- 段落間距應使用 CSS 控制。
- JavaScript 可以從 DOM 中觀察 `p` 與 `br` 的數量差異。

## 2. 對應 notes

- 筆記路徑：`notes/第10章_段落標籤/段落標籤.md`
- 對應章節：`4. 語法與基本用法`、`7. 常見錯誤與修正`、`11. 延伸補充`
- 對應知識點：
  - 多個段落應使用多個 `p`
  - 不應用連續 `<br>` 製造段落間距
  - HTML 表達結構，CSS 控制視覺呈現
  - DOM 中的段落節點

方式二：使用 VS Code Live Server

2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 錯誤寫法只有一個 `p`，但使用兩個 `<br>` 製造空隙。
2. 正確寫法使用兩個 `p` 表示兩段不同意思。
3. 表格中的統計結果來自 JavaScript 讀取 DOM。
4. 視覺上可以做出相似距離，但正確 HTML 結構更容易維護。

## 5. 程式碼重點

### HTML

- `.mistake-card` 展示同一個 `p` 中放入連續 `<br>` 的錯誤寫法。
- `.correct-card` 展示每段文字使用獨立 `p` 的正確寫法。
- `tbody[data-analysis-body]` 用來放入 JavaScript 產生的分析結果。

### CSS

- `.sample-content p` 設定段落行高，讓文字閱讀更清楚。
- `.mistake-card` 與 `.correct-card` 使用不同邊線顏色標示錯誤與正確範例。

### JavaScript

- `querySelectorAll("p")` 統計每組範例中的段落數量。
- `querySelectorAll("br")` 統計每組範例中的換行標籤數量。
- 依據統計結果將分析文字加入表格。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 在錯誤範例中再加入一個 `<br>`，觀察分析表的數量變化。
2. 把錯誤範例改成兩個 `p`，觀察判斷文字是否改變。
3. 調整 CSS 中 `.sample-content p` 的 `line-height`。
4. 在正確範例中新增第三個段落。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 用 `<br><br>` 代表新段落 | HTML 結構沒有真正分段 | 不同段落使用不同 `p` |
| 用空的 `p` 製造間距 | 產生沒有內容的段落 | 使用 CSS margin 控制距離 |
| 只看畫面距離是否接近 | 容易忽略語意與維護性 | 先確保 HTML 結構正確 |
