Demo 03

不要用連續 br 製造段落間距

這個 demo 比較錯誤寫法與正確寫法，並用 JavaScript 統計每組範例中的 <p> 與 <br> 數量。

錯誤與正確寫法對照

錯誤寫法

HTML 是用來描述網頁內容與結構的標記語言。

CSS 則用來控制畫面樣式。

兩段不同意思被塞在同一個 <p> 裡，並用連續 <br> 製造空隙。

正確寫法

HTML 是用來描述網頁內容與結構的標記語言。

CSS 則用來控制畫面樣式。

兩段不同意思各自使用一個 <p>，段落間距交給 CSS 控制。

DOM 結構觀察

範例 <p> 數量 <br> 數量 判斷

觀察重點：段落分段應該反映在 HTML 結構中，而不是只靠換行標籤模擬畫面。

修正方向

第一步：確認哪些文字屬於不同段落。

第二步：每個獨立段落使用一個 <p>。

第三步：需要調整段落距離時，修改 CSS 的 margin 或 line-height。
