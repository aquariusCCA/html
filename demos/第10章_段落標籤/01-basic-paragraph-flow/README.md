# 基本段落與自動換行

## 1. Demo 目標

這個 demo 用來示範：

- `p` 元素代表一段完整文字。
- 多個段落應使用多個 `p` 元素表示。
- 段落文字會依照容器寬度自動換行。
- 段落的視覺間距應交給 CSS 控制。

## 2. 對應 notes

- 筆記路徑：`notes/第10章_段落標籤/段落標籤.md`
- 對應章節：`3. 核心概念`、`3.2 段落的預設呈現`、`4. 語法與基本用法`
- 對應知識點：
  - `p` 段落元素
  - 多段文字分段
  - 段落自動換行
  - CSS 控制段落間距

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 觀察段落分段與自動換行

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 每個 `p` 會形成一段獨立文字。
2. 多個 `p` 之間會有段落間距。
3. 同一段文字在較窄容器中會自然換行。
4. 自動換行不需要在每一行手動加入 `<br>`。

## 5. 程式碼重點

### HTML

- 使用多個 `p` 建立文章正文。
- 使用兩個不同寬度的容器觀察同一類段落的換行效果。
- 使用表格整理段落元素的觀察重點。

### CSS

- `.article-sample p` 設定行高，讓正文更容易閱讀。
- `.compare-grid` 讓寬容器與窄容器並排。
- `.width-demo-narrow` 固定較窄欄位，方便觀察自動換行。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 在文章範例中新增第四個 `p`。
2. 將第二段文字合併到第一個 `p`，觀察分段是否變得不清楚。
3. 調整 `.width-demo-narrow` 的寬度，觀察文字換行變化。
4. 移除 `style.css` 連結，觀察瀏覽器預設段落樣式。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 把整篇文章放在同一個 `p` | 內容缺少清楚分段 | 每個獨立段落使用一個 `p` |
| 為每一行手動加 `<br>` | 把自動換行誤當成 HTML 結構 | 讓瀏覽器依容器寬度自動換行 |
| 用空白字元控制段落距離 | 樣式難以維護 | 使用 CSS 控制 `margin` 或 `line-height` |
