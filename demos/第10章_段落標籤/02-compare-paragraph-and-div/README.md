# 段落與 div 語意對照

## 1. Demo 目標

這個 demo 用來示範：

- `p` 適合表示正文段落。
- `h1` 到 `h6` 適合表示標題，不應用 `p` 假裝標題。
- `div` 是無語意容器，外觀相似不代表語意相同。
- HTML 應先表達內容結構，再用 CSS 控制外觀。

## 2. 對應 notes

- 筆記路徑：`notes/第10章_段落標籤/段落標籤.md`
- 對應章節：`3.1 段落元素的語意`、`7. 常見錯誤與修正`、`8. 與其他概念的關係`
- 對應知識點：
  - `p` 的段落語意
  - `div` 沒有段落語意
  - 標題與段落的差異
  - CSS 與 HTML 的責任分工

3. 比較左右兩側的 HTML 結構與外觀

方式二：使用 VS Code Live Server

2. 選擇 `Open with Live Server`

## 4. 觀察重點

1. 左側使用 `h3` 和多個 `p` 表達標題與段落。
2. 右側使用多個 `div` 模擬相似外觀，但沒有標題與段落語意。
3. CSS 可以讓不同元素看起來相似，但不能取代 HTML 語意。
4. 一般文章正文應優先使用 `p`，不是全部用 `div`。

## 5. 程式碼重點

### HTML

- 左側使用 `article`、`h3`、`p` 建立語意清楚的內容。
- 右側使用 `div` 展示常見的非語意寫法。
- 表格列出不同內容角色應選擇的元素。

### CSS

- `h3` 和 `.fake-heading` 被設定成相似外觀，讓學習者觀察外觀與語意的差異。
- `.semantic-sample` 與 `.nonsemantic-sample` 使用不同邊線顏色標示對照組。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把右側的 `.fake-paragraph` 改成真正的 `p`。
2. 把右側的 `.fake-heading` 改成 `h3`。
3. 移除 CSS 中 `h3, .fake-heading` 的共同樣式，觀察外觀不再相似後是否更容易看出差異。
4. 開啟瀏覽器開發者工具，檢查左右兩側實際 DOM 結構。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 所有文字都用 `div` | 內容角色不清楚 | 正文段落使用 `p` |
| 用 `p` 當標題 | 標題層級無法被清楚表達 | 標題使用 `h1` 到 `h6` |
| 只看畫面是否相同 | 忽略 HTML 結構與語意 | 先選正確元素，再用 CSS 控制樣式 |
