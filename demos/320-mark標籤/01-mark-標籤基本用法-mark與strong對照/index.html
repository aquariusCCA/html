mark 與 strong 對照、常見錯誤修正

<mark> 與 <strong> 都可能讓文字看起來醒目，但語意不同： <mark> 是「在目前上下文中被標記」，<strong> 是「內容本身很重要」。 選標籤時先看語意是否成立，再看畫面效果。

語意對照

搜尋命中片段 → <mark>

搜尋「HTML」後，把結果摘要中的 HTML 標出來，屬於上下文標記。

<p>...摘要中的 <mark>HTML</mark>...</p>

重要警告 → <strong>

請務必填寫真實姓名。這裡重點是內容很重要，而不是被標記。

<p><strong>請務必填寫真實姓名。</strong></p>

常見錯誤 → 修正

✗ 把 mark 當黃底樣式工具

今天特價：全部商品 8 折

<mark>全部商品 8 折</mark>

純粹想做促銷視覺，沒有「上下文標記」語意。

✓ 用 CSS class 表達樣式

今天特價：全部商品 8 折

<span class="sale-text">全部商品 8 折</span>

只是顏色/樣式需求，交給 CSS 更清楚。

✗ 用 mark 取代 strong

請勿關閉電源。

<mark>請勿關閉電源。</mark>

這是重要警告，重點是「內容很重要」。

✓ 改用 strong 表達重要性

請勿關閉電源。

<strong>請勿關閉電源。</strong>

語意正確：強調內容本身的重要性。

✗ 標記過多文字

這一整段很長的文字全部都被標記，讀者很難判斷真正的重點在哪裡。

幾乎整段都標，等於沒有重點。

✓ 只標關鍵片段

這一段主要說明 只標示關鍵片段 才能讓重點清楚。

聚焦在真正要讀者注意的詞。
