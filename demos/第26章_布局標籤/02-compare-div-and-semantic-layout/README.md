## 2. 學習目標

這個 demo 用來示範：

- 外觀相似不代表 HTML 語意相同。
- `div` 可以作為排版容器，但不會表達內容角色。
- 有明確內容角色時，可以優先使用 `header`、`nav`、`main`、`article`、`aside`、`footer`。

## 5. 觀察重點

1. 左右兩側視覺版面很接近。
2. 左側主要依靠 `class` 告訴人類這是什麼區塊。
3. 右側元素本身就描述了區塊角色。
4. `div` 不一定錯，但語意不如對應的原生布局標籤明確。

## 6. 程式碼重點

### HTML

- 左側使用 `div class="header-like"`、`div class="nav-like"` 模擬區塊。
- 右側使用 `header`、`nav`、`main`、`article`、`aside`、`footer`。
- 右側 `nav` 加上 `aria-label`，讓導覽用途更清楚。

### CSS

- 左右兩側共用同一組視覺樣式。
- CSS 的目的只是讓區塊容易比較，不代表 HTML 語意。

### JavaScript

- 此 demo 不需要 JavaScript。

## 7. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把左側的 `div.article-like` 改成 `article`，並觀察語意是否更清楚。
2. 把右側的 `main` 改成 `div`，思考主要內容的角色是否變得不明確。
3. 新增第二組導覽，並為每個 `nav` 設定不同的 `aria-label`。

## 8. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 只靠 class 表達語意 | HTML 元素本身仍沒有語意 | 有原生語意元素時優先使用 |
| 認為語意化標籤會自動完成排版 | 混淆 HTML 與 CSS 職責 | 使用 CSS 控制畫面，HTML 控制語意 |
| 把所有容器都改成語意標籤 | 沒有明確語意的容器反而變得混亂 | 單純排版容器仍可使用 `div` |
