## 2. 學習目標

這個 demo 用來示範：

- 缺少 `main` 會讓主要內容範圍不清楚。
- `section` 不應只是排版容器，通常應有明確主題與標題。
- 多組導覽時，`nav` 應有清楚用途標示。
- 可獨立閱讀的內容應考慮使用 `article`。

## 5. 觀察重點

1. 錯誤寫法看起來也能排版，但語意不清楚。
2. 修正寫法使用 `header`、`nav`、`main`、`article`、`section` 表達內容角色。
3. 檢查結果不是瀏覽器內建驗證，而是根據本章筆記整理出的觀察規則。
4. CSS 可以讓錯誤寫法看起來正常，但不能補足 HTML 元素本身缺少的語意。

## 6. 程式碼重點

### HTML

- 錯誤範例刻意缺少 `main`，並把 `section` 當成排版容器。
- 修正範例使用 `main` 包住主要內容，並讓 `section` 搭配標題。
- `nav aria-label="主要導覽"` 讓導覽用途更清楚。

### CSS

- CSS 只用來讓左右範例容易對照。
- `.bad` 與 `.good` 的色條只是提示，不代表語意本身。

### JavaScript

- `main.js` 只負責輸出檢查結果。
- 檢查內容根據本章常見錯誤整理，不加入額外框架或複雜邏輯。

## 7. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 在錯誤範例中補上 `main`，並修改檢查結果文字。
2. 替錯誤範例中的 `section` 加上標題，思考它是否仍只是排版容器。
3. 新增第二組 `nav`，並為兩組導覽設定不同的 `aria-label`。

## 8. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 缺少 `main` | 主要內容不容易被辨識 | 使用 `main` 包住頁面核心內容 |
| `section` 只為排版 | 產生沒有主題的語意區塊 | 純排版容器使用 `div` |
| `nav` 用途不明 | 多組導覽時難以區分 | 使用 `aria-label` 或清楚標題 |
| 可獨立內容只用 `div` | 內容角色不明確 | 考慮使用 `article` |
