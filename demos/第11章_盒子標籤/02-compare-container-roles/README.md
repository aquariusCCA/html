# div 與 span 使用情境對照

## 1. Demo 目標

這個 demo 用來示範：

- `div` 適合包住一組較大的區塊內容。
- `span` 適合包住一句話中的局部文字。
- `div` 與 `span` 都沒有語意，因此不應取代 `p`、`a`、`h1` 到 `h6` 等更精準的元素。

- 對應知識點：

  - `span` 作為行內局部容器

## 4. 觀察重點

1. 左側的 `div.course-card` 包住標題、段落與連結，是一整組內容。
2. 右側的 `span` 只包住狀態、價格與難度等局部文字。
3. 表格中的 `p` 與 `a` 代表：如果有更精準語意元素，不應只使用 `div` 或 `span`。

## 5. 程式碼重點

### HTML

- 使用 `div` 包住卡片內容。
- 使用 `span` 標記段落中的狀態、價格與標籤文字。
- 使用 `p` 表示段落，使用 `a` 表示連結。

### CSS

- `.course-card` 控制整張卡片的區塊外觀。
- `.status`、`.price`、`.tag` 控制文字中的局部標籤外觀。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把 `.course-card` 的外層 `div` 改成 `span`，觀察 HTML 結構與畫面是否變得不自然。
2. 把狀態文字外層的 `span` 移除，觀察 CSS 是否還能只選取那段文字。
3. 新增一個「課程時長」欄位，並用 `span` 包住時長數字。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 用 `span` 包住整張卡片 | 行內容器承載過大的區塊結構 | 改用 `div` 或更合適的語意容器 |
| 用 `div` 取代段落 | 缺少段落語意 | 段落文字使用 `p` |
| 用 `div` 取代連結 | 缺少連結語意與原生行為 | 可前往其他位置的內容使用 `a` |
