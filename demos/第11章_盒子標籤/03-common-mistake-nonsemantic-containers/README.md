# 盒子標籤常見錯誤

## 1. Demo 目標

這個 demo 用來示範：

- 不應把所有內容都寫成 `div`。
- 不應用 `div` 或 `span` 假裝有語意的元素。
- 原生 `button` 比 `div` 更適合表示按鈕。
- 外觀相似不代表 HTML 結構相同。

## 2. 對應 notes

- 筆記路徑：`notes/第11章_盒子標籤/盒子標籤.md`
- 對應章節：`7. 常見錯誤與修正`、`8. 與其他概念的關係`
- 對應知識點：
  - 所有內容都用 `div` 會讓結構不清楚
  - `div` 沒有標題或段落語意
  - 用 `div` 或 `span` 假裝按鈕會降低原生互動與可訪問性

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 比較錯誤寫法與正確寫法
4. 嘗試用 Tab 鍵移動焦點，再點擊真正的按鈕

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 錯誤文章區塊看起來像有標題與段落，但實際上全部是 `div`。
2. 正確文章區塊使用 `article`、`h3`、`p` 表示內容角色。
3. 假按鈕雖然外觀像按鈕，但不是原生 `button`。
4. 下方 DOM 結構觀察會列出兩組寫法的實際元素名稱。

## 5. 程式碼重點

### HTML

- `.mistake-box` 展示非語意寫法。
- `.correct-box` 展示較清楚的語意結構。
- 真正的操作按鈕使用 `button type="button"`。

### CSS

- `.fake-heading` 與 `h3` 被做成相似外觀，用來提醒「外觀不等於語意」。
- `.fake-button` 與 `button` 被做成相似外觀，用來比較元素行為。

### JavaScript

- 讀取 DOM 子元素名稱，讓學習者看到錯誤寫法與正確寫法的實際差異。
- 只替真正的 `button` 綁定 click 事件。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把錯誤文章區塊中的 `.fake-heading` 改成 `h3`。
2. 把錯誤文章區塊中的 `.fake-paragraph` 改成 `p`。
3. 把 `.fake-button` 改成真正的 `button type="button"`。
4. 移除 CSS 中讓假元素看起來像真元素的樣式，觀察語意錯誤是否更明顯。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 所有內容都用 `div` | 文件結構不清楚 | 根據內容角色選擇 `h1` 到 `h6`、`p`、`article` 等元素 |
| 用 `div` 假裝按鈕 | 缺少原生按鈕語意與鍵盤操作支援 | 使用 `button` |
| 只靠 CSS 做出相似外觀 | 會誤以為元素語意也相同 | 先選正確 HTML 元素，再用 CSS 控制外觀 |
