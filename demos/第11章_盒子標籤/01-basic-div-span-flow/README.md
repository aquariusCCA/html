# div 與 span 預設排列

## 1. Demo 目標

這個 demo 用來示範：

- `div` 預設是區塊型容器。
- `span` 預設是行內型容器。
- 相似內容使用不同元素時，預設排列方式會不同。

## 2. 對應 notes

- 筆記路徑：`notes/第11章_盒子標籤/盒子標籤.md`
- 對應章節：`3.2 div 是區塊型容器`、`3.3 span 是行內型容器`、`5. 範例說明`
- 對應知識點：
  - `div` 通常獨佔一行
  - `span` 通常可與其他內容同一行
  - 預設顯示方式不等於語意

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 觀察 `div` 與 `span` 的排列差異

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 多個 `div` 會由上到下排列。
2. 多個 `span` 會盡量排在同一行。
3. CSS 可以替不同元素加上相同外觀，但元素本身的預設排列仍不同。

## 5. 程式碼重點

### HTML

- 第一組直接放置多個 `div`。
- 第二組直接放置多個 `span`。
- 第三組使用相同文字比較不同元素造成的排列差異。

### CSS

- 使用邊框與背景色讓每個容器範圍更容易被看見。
- 沒有把 `div` 改成 `inline`，也沒有把 `span` 改成 `block`，以保留觀察重點。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 在 `inline-demo` 裡再加入幾個 `span`，觀察空間不足時如何換行。
2. 把其中一個 `span` 改成 `div`，觀察它是否另起一行。
3. 在 CSS 中加入 `.inline-demo span { display: block; }`，觀察外觀可以被 CSS 改變。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 只因為想換行就使用 `div` | 忽略內容本身是否需要語意元素 | 先判斷內容角色，再用 CSS 控制排版 |
| 只因為想同一行就使用 `span` | 可能把區塊內容塞進行內容器 | 大範圍內容使用 `div` 或更合適語意元素 |
| 把預設排列當成元素語意 | 混淆 HTML 與 CSS 的責任 | HTML 表達結構，CSS 控制外觀 |
