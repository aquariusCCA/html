## 1. Demo 目標

這個 demo 用來示範：

- `<table>` 是整張表格的容器。
- `<tr>` 表示表格中的一列。
- `<th>` 表示表頭儲存格。
- `<td>` 表示一般資料儲存格。

## 4. 觀察重點

1. 第一列使用 `<th>`，表示欄位名稱。
2. 後續資料列使用 `<td>`，表示每一筆實際資料。
3. 每一個 `<tr>` 中的儲存格數量一致時，表格比較容易閱讀。
4. CSS 的邊框與背景色只是輔助觀察，不是表格語意本身。

## 5. 程式碼重點

### HTML

- 表格外層使用 `<table>`。
- 每一列使用 `<tr>`。
- 欄位標題使用 `<th>`，資料內容使用 `<td>`。

### CSS

- 使用 `border-collapse: collapse;` 讓表格邊框合併。
- 使用不同背景色讓表頭更容易辨識。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 新增一列學生資料，確認每一列的欄位數是否一致。
2. 把第一列的 `<th>` 改成 `<td>`，思考語意有什麼差異。
3. 新增一個「分數」欄位，並替每一筆資料補上分數。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 直接把 `<td>` 放在 `<table>` 裡 | 表格結構不清楚，也不符合正確巢狀概念 | 先建立 `<tr>`，再放 `<td>` 或 `<th>` |
| 欄位名稱也使用 `<td>` | 缺少表頭語意 | 欄位名稱使用 `<th>` |
| 每一列欄位數不一致 | 表格可能變得難以閱讀 | 檢查每個 `<tr>` 中的儲存格數量 |
