## 1. Demo 目標

這個 demo 用來示範：

- `<table>` 應用於表格資料，不應用於頁面布局。
- 頁面布局應使用語意元素搭配 CSS。
- 資料表應盡量有清楚的 `<caption>`、`<th>` 與區域結構。

## 4. 觀察重點

1. 錯誤布局範例雖然畫面可以顯示，但內容不是資料表。
2. 正確布局範例使用 `nav` 與 `section` 表達頁面區域，再由 CSS 排列。
3. 不完整資料表缺少 `<caption>`、`<th>`、`<thead>` 等語意結構。
4. JavaScript 檢查結果可以幫助觀察表格語意是否完整。

## 5. 程式碼重點

### HTML

- 使用兩組對照範例呈現錯誤與修正。
- 完整資料表使用 `<caption>`、`<thead>`、`<tbody>`、`<th>`。

### CSS

- CSS Grid 用於正確的頁面布局。
- 紅色與綠色邊框只用來標示問題與修正。

### JavaScript

- `main.js` 檢查表格是否包含常見語意結構。
- 這段 JavaScript 只用於教學觀察，不是必要的表格功能。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 替不完整表格加上 `<caption>`，再按一次檢查按鈕。
2. 把不完整表格第一列的 `<td>` 改成 `<th>`。
3. 移除完整表格的 `<thead>`，觀察檢查結果如何改變。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 用表格排版頁面 | 語意錯誤，輔助工具可能誤判成資料表 | 使用語意元素與 CSS 排版 |
| 資料表沒有表頭 | 欄位意義不清楚 | 使用 `<th>` 表示欄位標題 |
| 表格沒有標題或上下文 | 使用者不容易知道表格用途 | 使用 `<caption>` 或明確標題 |
| 只靠 CSS 做出像表格的外觀 | HTML 語意不足 | 真正欄列資料應使用 `<table>` |
