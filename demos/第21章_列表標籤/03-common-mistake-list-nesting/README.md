# 常見錯誤：列表巢狀位置

## 1. Demo 目標

這個 demo 用來示範：

- 錯誤的巢狀列表寫法。
- 正確的巢狀列表寫法。
- 子列表應該放在對應的父層 `<li>` 裡。
- 父層 `<ul>` 或 `<ol>` 的直接子元素應該是 `<li>`。

## 2. 對應 notes

- 筆記路徑：`notes/第21章_列表標籤/列表標籤.md`

  - `<ul>` / `<ol>` 的直接子元素
  - 子列表應放在父層 `<li>` 內

2. 使用瀏覽器開啟

## 4. 觀察重點

1. 錯誤寫法中，子 `<ul>` 被放到父層 `<ul>` 的直接子層。
2. 正確寫法中，子 `<ul>` 放在「上海」這個 `<li>` 裡。
3. 頁面下方的黃色提示會列出父層列表的直接子元素。
4. 瀏覽器可能嘗試修正不合法結構，因此更應該主動寫出正確 HTML。

## 5. 程式碼重點

- 右側示範正確巢狀結構。

### CSS

- 紅色上框標示錯誤範例。
- 綠色上框標示正確範例。

### JavaScript

- `main.js` 讀取父層列表的直接子元素，幫助觀察 DOM 結構。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 在正確寫法的「西安」底下新增一個子列表。
2. 將錯誤寫法改成正確寫法，觀察黃色提示是否改變。
3. 把父層 `<ul>` 改成 `<ol>`，思考什麼情境下順序才有意義。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 子列表直接放在父層 `<ul>` 中 | 父子關係不清楚，結構不符合列表語意 | 把子列表放進對應的 `<li>` |
| 為了縮排亂加 `<ul>` | 語意錯誤且維護困難 | 縮排用 CSS，列表用來表達項目關係 |
| 只看畫面不看結構 | 畫面可能看起來正常，但 HTML 不正確 | 檢查 DOM 與標籤巢狀 |
