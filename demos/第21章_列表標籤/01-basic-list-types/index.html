Demo 01

基本列表類型：ul、ol、dl

這個 demo 用同一個頁面比較無序列表、有序列表與描述列表。 重點是觀察內容語意，而不是只看預設樣式。

三種列表的適合情境

無序列表 <ul>

項目彼此並列，沒有排名或步驟順序。

  • 課程公告
  • 前端讀書會
  • 作業繳交提醒

有序列表 <ol>

項目的順序會影響理解，例如步驟或排名。

  1. 建立 HTML 檔案
  2. 撰寫基本文件結構
  3. 加入列表內容
  4. 用瀏覽器檢查結果

描述列表 <dl>

適合名詞與說明、欄位與內容的配對。

HTML
描述網頁內容與結構的標記語言。
CSS
控制網頁外觀與排版的樣式語言。

如何選擇

問題 建議標籤
這些項目只是同一組內容，沒有順序差異嗎？ <ul>
這些項目是步驟、流程或排名嗎？ <ol>
這些內容是名詞與說明的配對嗎？ <dl>
