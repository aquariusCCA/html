# 基本列表類型：ul、ol、dl

## 1. Demo 目標

這個 demo 用來示範：

- `<ul>` 適合沒有順序差異的一組項目。
- `<ol>` 適合步驟、流程、排名等順序有意義的項目。
- `<dl>` 適合名詞與說明、欄位與內容的配對。

## 2. 對應 notes

- 筆記路徑：`notes/第21章_列表標籤/列表標籤.md`
- 對應章節：`3. 核心概念`、`4. 語法與基本用法`
- 對應知識點：
  - 無序列表 `<ul>`
  - 有序列表 `<ol>`
  - 描述列表 `<dl>`
  - 列表是語意，不只是樣式

## 4. 觀察重點

1. `<ul>` 預設顯示項目符號，但它的重點是「無順序的一組項目」。
2. `<ol>` 預設顯示數字，適合步驟與排名。
3. `<dl>` 使用 `<dt>` 與 `<dd>` 表達名稱與說明。
4. CSS 只改變外觀，不改變列表本身的語意。

## 5. 程式碼重點

### HTML

- 三個列表放在同一頁，方便比較語意差異。
- `<ul>` 與 `<ol>` 的直接子元素都是 `<li>`。
- `<dl>` 內使用 `<dt>` 與 `<dd>` 成對表達內容。

### CSS

- CSS 只用來排版與輔助觀察。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把「建立 HTML 檔案」那組步驟改成 `<ul>`，思考語意是否仍清楚。
2. 新增一個 `<dt>` 與兩個 `<dd>`，觀察描述列表能否表達多段說明。
3. 修改 CSS 的 `list-style-type`，觀察樣式改變是否影響 HTML 語意。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 只根據預設外觀選標籤 | 可能選錯語意 | 根據內容關係選擇 `<ul>`、`<ol>`、`<dl>` |
| `<ul>` 或 `<ol>` 直接放文字 | 結構不合法也不易維護 | 每個項目都使用 `<li>` |
| 把一般清單硬寫成 `<dl>` | 名詞與說明關係不清楚 | 一般並列項目使用 `<ul>` |
