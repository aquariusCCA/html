# 描述列表：dt 與 dd

## 1. Demo 目標

這個 demo 用來示範：

- `<dl>` 如何表示描述列表。
- `<dt>` 如何表示被描述的項目或名稱。
- `<dd>` 如何表示對 `<dt>` 的說明。
- 一個 `<dt>` 可以對應多個 `<dd>`。

## 4. 觀察重點

1. `<dt>` 是名稱或術語，`<dd>` 是說明。
2. `HTML` 與 `DOM` 各自有兩個 `<dd>`，表示同一個名詞可以有多段說明。
3. 聯絡資訊也可以用 `<dl>` 表達欄位與內容。
4. CSS 可以把描述列表排成兩欄，但 HTML 語意仍然是描述列表。

## 5. 程式碼重點

### HTML

- 使用 `<dl>` 包住整組描述內容。
- 使用 `<dt>` 表示被描述的名稱。
- 使用 `<dd>` 補充該名稱的說明。

### CSS

- `.info-list` 使用 grid 排版，方便觀察欄位與內容。
- CSS 只改變排版，不改變 `<dl>` 的語意。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 替 `CSS` 新增第二個 `<dd>`，補充「常與 HTML 一起使用」。
2. 新增一組聯絡資訊，例如「官方網站」與網址。
3. 把聯絡資訊改成 `<ul>`，比較哪一種語意更清楚。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 把 `<dl>` 當成縮排工具 | HTML 語意不清楚 | 只有名詞與說明關係時才使用 `<dl>` |
| 只寫 `<dd>` 不寫 `<dt>` | 讀者不知道說明對應哪個項目 | 補上對應的 `<dt>` |
| 一般項目清單使用 `<dl>` | 內容關係被誤解為名稱與描述 | 並列項目改用 `<ul>` |
