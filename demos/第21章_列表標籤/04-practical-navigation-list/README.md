# 用列表建立導覽列

## 1. Demo 目標

這個 demo 用來示範：

- 導覽列可以使用 `<nav>` 搭配 `<ul>`、`<li>`、`<a>` 建立。
- 導覽連結本質上是一組相關項目，適合用列表表示。
- 使用 CSS 移除項目符號與改成橫向排列，不會改變列表語意。

## 2. 對應 notes

- 筆記路徑：`notes/第21章_列表標籤/列表標籤.md`
- 對應章節：`6. 實務應用場景`、`8. 與其他概念的關係`
- 對應知識點：
  - 導覽列可以用列表表示
  - CSS 控制外觀
  - `list-style: none` 不改變列表語意

1. 找到 `index.html`

3. 點擊導覽列連結，觀察頁內跳轉

方式二：使用 VS Code Live Server

2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 導覽列 HTML 是 `<nav>` 裡面放一組 `<ul>`。
2. 每個導覽項目都是一個 `<li>`，每個 `<li>` 裡有一個 `<a>`。
3. CSS 將列表變成橫向排列，並移除項目符號。
4. 按 Tab 鍵可以觀察連結的 focus 狀態。

## 5. 程式碼重點

### HTML

- `<nav aria-label="主要導覽">` 說明這是一組主要導覽。
- `<ul>` 表示導覽連結是一組相關項目。
- `<a href="#section-id">` 連到頁面中的指定區塊。

### CSS

- `.site-nav ul` 使用 `display: flex` 讓導覽項目橫向排列。
- `list-style: none` 移除項目符號。
- `scroll-margin-top` 避免 sticky header 擋住跳轉目標。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 移除 `.site-nav ul` 的 `list-style: none`，觀察項目符號。
2. 將 `display: flex` 改掉，觀察導覽列如何回到直向列表。
3. 新增一個導覽項目，並在下方新增對應的 section。
4. 把導覽列中的 `<ul>` 改成一堆獨立 `<a>`，比較結構是否仍清楚。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 為了橫向排列而不用列表 | 少了「一組導覽項目」的語意 | 保留列表結構，使用 CSS 排版 |
| `<ul>` 裡直接放 `<a>` | 直接子元素不是 `<li>` | 使用 `<li><a href="...">文字</a></li>` |
| focus 樣式不清楚 | 鍵盤使用者難以判斷目前位置 | 加上清楚的 `:focus` 樣式 |
