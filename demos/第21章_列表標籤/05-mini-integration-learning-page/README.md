# 列表小型整合：課程學習頁

## 1. Demo 目標

這個 demo 用來示範：

- 如何在同一個頁面中選擇不同列表語意。
- 課程目標適合使用 `<ul>`。
- 操作步驟適合使用 `<ol>`。
- 術語解釋適合使用 `<dl>`。
- 章節目錄可以使用巢狀 `<ol>` 表示階層與順序。

## 2. 對應 notes

- 筆記路徑：`notes/第21章_列表標籤/列表標籤.md`
- 對應章節：`3. 核心概念`、`4. 語法與基本用法`、`6. 實務應用場景`
- 對應知識點：
  - 無序列表
  - 有序列表
  - 描述列表
  - 巢狀列表
  - 列表與 CSS 排版

## 3. 如何執行

1. 找到 `index.html`

方式二：使用 VS Code Live Server

## 4. 觀察重點

1. 左側章節目錄使用巢狀 `<ol>`，因為章節有順序。
2. 學習目標使用 `<ul>`，因為目標是並列項目。
3. 練習步驟使用 `<ol>`，因為執行順序有意義。
4. 本節術語使用 `<dl>`，因為內容是名詞與說明。

## 5. 程式碼重點

### HTML

- 根據內容關係選擇列表標籤。
- 巢狀章節放在父層 `<li>` 內。
- `<dl>` 使用 `<dt>` 與 `<dd>` 表達術語與解釋。

### CSS

- 使用 grid 建立側邊欄與主要內容。
- CSS 只處理排版，不改變列表語意。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 在章節目錄中新增「第 3 章 表單標籤」，並加入兩個子項目。
2. 把「建議練習步驟」改成 `<ul>`，思考語意是否合理。
3. 在「本節術語」中新增 `list item` 與說明。
4. 移除 CSS 中的 grid，觀察 HTML 結構是否仍然清楚。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 所有區塊都使用 `<ul>` | 無法表達步驟與術語關係 | 根據語意選擇 `<ol>` 或 `<dl>` |
| 巢狀目錄放錯位置 | 章節階層不清楚 | 子目錄放在對應的父層 `<li>` |
| 用 CSS 排版思維決定 HTML 標籤 | HTML 語意容易錯誤 | 先決定內容關係，再寫 CSS |
