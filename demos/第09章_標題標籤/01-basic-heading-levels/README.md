# 基本標題層級

## 1. Demo 目標

這個 demo 用來示範：

- `h1` 到 `h6` 是 HTML 的六種標題元素。
- 數字越小，標題層級越高。
- 瀏覽器的預設字體大小不是選擇標題標籤的主要理由。

## 2. 對應 notes

- 筆記路徑：`notes/第09章_標題標籤/標題標籤.md`
- 對應章節：`3. 核心概念`、`3.2 預設樣式不等於真正用途`、`4. 語法與基本用法`

  - `h1` 到 `h6`
  - 標題層級
  - 瀏覽器預設樣式

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 觀察六種標題元素的預設呈現

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. `h1` 通常最大，`h6` 通常最小。
2. 每個標題元素預設會獨佔一行。
3. 表格中的語意說明比視覺大小更重要。
4. 實務上應根據內容層級選擇標題，而不是根據想要的字體大小。

## 5. 程式碼重點

### HTML

- 使用 `h1` 到 `h6` 列出六種標題元素。
- 使用表格整理每個標題層級的語意與使用情境。

### CSS

- CSS 只用來讓範例區塊與表格容易閱讀。
- `.heading-stack` 的邊線用來協助觀察每一列標題。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把範例中的 `h3` 文字改成新的子段落標題。
2. 移除 `style.css` 連結，觀察瀏覽器原始預設樣式。
3. 把某一個 `h2` 改成 `h4`，思考內容層級是否仍然合理。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 只因為字大就使用 `h1` | 內容層級會被誤解 | 先判斷內容是否真的是最高層級標題 |
| 把普通段落包成標題 | 不重要的文字會被誤認為章節標題 | 一般文字使用 `p` |
| 只看瀏覽器預設樣式 | 容易混淆語意與外觀 | 用 CSS 控制外觀，用 HTML 表達結構 |
