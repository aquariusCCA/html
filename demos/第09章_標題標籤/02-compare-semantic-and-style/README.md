# 標題語意與外觀對照

## 1. Demo 目標

這個 demo 用來示範：

- 標題標籤的重點是語意與內容層級。
- CSS 負責控制字體大小、粗細、顏色與間距。
- 不應該只為了讓文字變大就使用 `h1`。

  - 避免把標題標籤當成字體放大工具

## 4. 觀察重點

1. 左側使用 `h1` 表示商品名稱，使用 `h2` 表示商品頁的主要區塊。
2. 左側的兩個 `h2` 可以透過 CSS 呈現不同大小。
3. 右側出現只為了視覺大小而使用的 `h1`。
4. 右側也出現用 `p` 假裝標題的情況，語意不清楚。

## 5. 程式碼重點

### HTML

- 左側示範正確分工：HTML 表達結構，CSS 控制外觀。
- 右側示範常見誤解：用標籤名稱追求視覺效果。

### CSS

- `.large-section-title` 與 `.small-section-title` 只改變外觀，不改變標題層級。
- `.fake-heading` 讓段落看起來像標題，用來示範語意錯誤。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把 `.large-section-title` 的 `font-size` 改小，確認它仍然是 `h2`。
2. 將右側的第二個 `h1` 改成 `h2`，讓它回到合理層級。
3. 將右側的 `p class="fake-heading"` 改成 `h2`，再用 CSS 控制外觀。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 用 `h1` 只是為了放大字 | 頁面最高層級標題變得混亂 | 使用符合內容層級的標題，再用 CSS 放大 |
| 用 `p` 假裝區塊標題 | 瀏覽器與輔助工具無法辨識它是標題 | 使用正確的 `h2` 或 `h3` |
| 因為想取消粗體而不用標題 | 混淆語意與樣式責任 | 保留標題元素，用 CSS 調整 `font-weight` |
