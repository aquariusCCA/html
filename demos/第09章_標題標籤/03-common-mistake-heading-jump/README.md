# 標題層級常見錯誤

## 1. Demo 目標

這個 demo 用來示範：

- `h1` 後直接接 `h4` 會讓層級關係不清楚。
- 普通促銷文字或提示文字不應該包成標題。
- JavaScript 可以從 DOM 中讀取標題節點，協助觀察文件結構。

## 2. 對應 notes

- 筆記路徑：`notes/第09章_標題標籤/標題標籤.md`
- 對應章節：`7. 常見錯誤與修正`、`8. 與其他概念的關係`
- 對應知識點：
  - 標題層級亂跳
  - 把普通段落文字包成標題
  - DOM 與 JavaScript 可以查找標題節點

## 4. 觀察重點

1. 左側錯誤範例從 `h1` 直接跳到 `h4`。
2. 左側把「限時優惠中」這種普通提示做成 `h3`。
3. 右側用 `h2` 表示 `h1` 底下的主要段落。
4. 右側用 `p` 呈現促銷提示，避免把它誤認為新章節。

## 5. 程式碼重點

### HTML

- `data-heading-sample` 用來標記要被 JavaScript 分析的範例區塊。
- 錯誤範例保留常見錯誤，方便和修正範例對照。

### CSS

- 左右範例使用不同上邊框顏色，協助分辨錯誤與修正。
- `.promo-text` 用 CSS 做出醒目提示，不需要改用標題標籤。

### JavaScript

- 使用 `querySelectorAll("h1, h2, h3, h4, h5, h6")` 取得標題節點。
- 檢查目前標題層級是否比上一個標題跳超過一級。
- 把檢查結果寫入表格。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把錯誤範例的 `h4` 改成 `h2`，重新整理後觀察分析表格。
2. 將 `h3 data-warning` 改成 `p class="promo-text"`，觀察表格少了哪一列。
3. 在修正範例的 `h2` 底下新增一個 `h3`，觀察分析表格是否仍顯示合理。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| `h1` 後直接接 `h4` | 中間層級不清楚 | 依內容關係使用 `h2` 或 `h3` |
| 把促銷文字做成標題 | 不重要內容被誤認為章節 | 使用 `p`，再用 CSS 控制樣式 |
| 只看畫面是否好看 | DOM 結構可能仍然混亂 | 用標題順序檢查文件層級 |
