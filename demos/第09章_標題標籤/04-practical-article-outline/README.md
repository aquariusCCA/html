# 文章頁標題結構

## 1. Demo 目標

這個 demo 用來示範：

- 在文章頁中使用 `h1`、`h2`、`h3` 建立清楚層級。
- 標題可以幫助使用者快速掃讀內容。
- JavaScript 可以從 DOM 中讀取標題，產生文章 outline。

## 2. 對應 notes

- 筆記路徑：`notes/第09章_標題標籤/標題標籤.md`
- 對應章節：`5. 範例說明`、`6. 實務應用場景`、`8. 與其他概念的關係`
- 對應知識點：
  - 文章頁標題結構
  - `h1`、`h2`、`h3` 的層級關係
  - DOM 與 JavaScript 操作標題節點
  - 可訪問性與內容掃讀

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 觀察右側 outline 與文章標題的對應關係

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 文章本身有一個主要 `h1`。
2. 每個主要段落使用 `h2`。
3. `h2` 底下更細的內容使用 `h3`。
4. 右側 outline 是 JavaScript 從文章 DOM 中讀取標題後產生的。

## 5. 程式碼重點

### HTML

- `article` 包住一篇可獨立閱讀的文章。
- `section` 分隔文章主要段落。
- `h1`、`h2`、`h3` 依照內容層級排列。

### CSS

- `.layout` 使用 grid 排出文章與 outline。
- `.outline-level-1`、`.outline-level-2`、`.outline-level-3` 用縮排呈現層級。
- `.active-heading` 用來標示使用者在 outline 中點選的標題。

### JavaScript

- 使用 `article.querySelectorAll("h1, h2, h3")` 取得文章標題。
- 依照標題標籤建立 outline 按鈕。
- 點選 outline 後捲動到對應標題並加上醒目樣式。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 在某個 `h2` 底下新增一個 `h3`，觀察 outline 是否自動新增項目。
2. 把一個 `h3` 改成 `h2`，觀察右側縮排與文章層級是否改變。
3. 把某段普通文字改成 `p` 或 `h3`，思考哪一種語意比較合理。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 文章沒有清楚 `h1` | 使用者難以判斷主題 | 為主要內容提供清楚的主標題 |
| `h2` 與 `h3` 隨意混用 | 章節關係不明確 | 依照父子層級安排標題 |
| 用 CSS 製造文字大小但不用標題 | DOM outline 無法反映內容結構 | 使用正確標題元素，再用 CSS 控制外觀 |
