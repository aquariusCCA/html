表單的用途與提交結構

<form> 定義「資料提交的範圍」，但不是畫面上每個欄位都會被送出——關鍵在 name 與欄位狀態。

對應筆記：notes/240-表單標籤/01-表單的用途與提交結構.md

A. 表單由哪幾部分組成

一個完整表單 = 表單域 + 表單控件 + 提示資訊。

  • <form> 表單域：可提交資料的範圍
  • <input>/<label> 控件與提示
<form action="/register" method="post">
  <label for="account">帳號</label>
  <input id="account" name="account" type="text">
  <button type="submit">註冊</button>
</form>

B. 互動：到底哪些欄位會被提交？

填好欄位後按「模擬提交」。表單不會真的送出，而是用 JavaScript 收集瀏覽器實際會提交的 name=value，讓你看清楚規則。

④ 上面還藏了一個 <input type="hidden" name="source" value="demo-page">，看不見但會提交。

瀏覽器實際會送出的資料（FormData）

  • 尚未提交，點「模擬提交」看看。

C. 常見錯誤：只放 input、沒有 form
