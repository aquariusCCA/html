表單按鈕與提交行為

按鈕的 type 決定它是提交、重置，還是普通操作；忘了寫 type 可能造成意外提交。

對應筆記：notes/240-表單標籤/04-表單按鈕與提交行為.md

A. 互動：三種按鈕，三種行為

先改一下下面的暱稱欄位，再分別點三顆按鈕，觀察右側的事件記錄與欄位變化。（提交已被攔截，不會真的離開頁面）

事件記錄

  • 尚未操作。
  • type="submit"：送出表單（這裡攔截後顯示「已觸發提交」）
  • type="reset"：恢復初始值（小明），不是清空
  • type="button"：不提交、不重置，只跑 JavaScript

B. 錯誤修正：表單內的 button 忘記寫 type

兩顆按鈕都叫「取得驗證碼」。點點看，左邊會意外提交表單，右邊不會。

建議：表單裡的每顆 <button> 都明確寫出 type。
