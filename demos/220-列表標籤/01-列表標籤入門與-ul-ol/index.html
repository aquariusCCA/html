HTML 列表標籤

ul、ol 與 li 的語意選擇

切換內容情境，觀察 HTML 結構如何對應無序列表、有序列表與正確的 li 容器。

內容情境

商品特色是一組並列項目，項目順序不影響內容意義。

建議 HTML

<ul> 
<ul>
  <li>價格清楚</li>
  <li>支援行動裝置</li>
  <li>容易維護</li>
</ul>

瀏覽器呈現

順序不重要

li 是列表項目的容器

列表中的文字、段落或連結應該放進 <li>，而不是直接放在 <ul> 或 <ol> 裡。

正確結構

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>
    <p>JavaScript</p>
    <a href="intro.html">查看介紹</a>
  </li>
</ul>
所有項目都是列表的 <li> 子元素。

不建議結構

<ul>
  HTML
  <p>CSS</p>
  <li>JavaScript</li>
</ul>
文字與 <p> 直接出現在列表裡，列表結構不清楚。
