# HTML 入門課程報名

## 1. Demo 目標

這個 demo 用來示範：

- 如何把 HTML 基本骨架套用到一個實務頁面。
- 如何使用 `class`、`id`、`for`、`name`、`placeholder`、`required` 等屬性。
- HTML 註解不會顯示在頁面上，但仍可被檢視原始碼看到。

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 嘗試在空白狀態按下「送出報名」，觀察必填欄位反應

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 頁面原始碼中有 `doctype`、`lang`、`charset`。
2. 每個 `label for` 都對應到輸入框的 `id`。
3. 註解不會顯示在畫面上，但可以透過檢視原始碼看到。

## 5. 程式碼重點

### HTML

- 以課程報名頁整合本章的基本結構與屬性。
- 使用 `required` 讓瀏覽器執行內建必填檢查。
- 註解只用來說明區塊目的，不放敏感資訊。

### CSS

- CSS 只負責清楚呈現表單與對照表，不改變 HTML 概念本身。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 修改 `form action="#"`，思考正式專案會提交到哪個位置。
2. 移除 Email 欄位的 `required`，觀察送出時的檢查差異。
3. 新增一段 HTML 註解，再確認它是否會顯示在畫面上。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 將註解當成安全的隱藏資料區 | 使用者仍可檢視原始碼看到內容 | 不把密碼、token 或內部資訊放進註解 |
| `label for` 與 `input id` 不一致 | 點擊標籤時無法聚焦對應欄位 | 讓 `for` 與 `id` 使用相同值 |
| 表單欄位缺少 `name` | 提交資料時缺少欄位名稱 | 每個需要提交的欄位都加上清楚的 `name` |
