HTML 屬性的正確寫法與常見錯誤

這個頁面比較屬性寫在開始標籤內、使用空格分隔、避免重複同名屬性的差異。

正確屬性寫法

<input type="text" name="username" placeholder="請輸入帳號">

屬性寫在開始標籤中，屬性與標籤名之間、不同屬性之間都要用空格隔開。

屬性順序通常不影響結果

寫法 A

<input type="email" name="email">

寫法 B

<input name="email" type="email">

兩個輸入框的屬性順序不同，但都清楚寫出 type 與 name，瀏覽器可以正常解析。

常見錯誤對照

錯誤寫法 問題 修正方式
<inputtype="text"name="username"> 標籤名與屬性沒有用空格隔開，瀏覽器無法按照預期解析。 <input type="text" name="username">
<input type="text" type="password"> 同一元素重複寫同名屬性，結果不清楚。 <input type="password" name="password">
<p CLASS="intro"> HTML 屬性名稱通常不區分大小寫，但團隊實務建議統一小寫。 <p class="intro">
<a href=https://example.com> 部分屬性值可以省略引號，但正式開發不利於一致維護。 <a href="https://example.com">
