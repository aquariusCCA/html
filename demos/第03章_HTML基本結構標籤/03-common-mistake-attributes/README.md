# HTML 屬性常見錯誤

## 1. Demo 目標

這個 demo 用來示範：

- 屬性應寫在元素開始標籤中。
- 標籤名、屬性名與不同屬性之間需要空格。
- 同一個元素不應重複寫同名屬性。

## 4. 觀察重點

1. 正確範例中的 `type`、`name`、`placeholder` 都寫在開始標籤內。
2. 屬性順序不同時，只要屬性本身清楚且不重複，瀏覽器仍能解析。
3. 錯誤寫法表格只用程式碼文字呈現，避免把錯誤結構真的放進頁面中。

## 5. 程式碼重點

### HTML

- 使用 `label` 與 `input` 示範常見屬性位置。
- 使用表格整理錯誤、問題與修正方式。

### CSS

- CSS 用來讓表單、程式碼片段與表格更容易比較。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 在正確範例中新增 `required` 屬性，觀察表單欄位狀態。
2. 將 `placeholder` 文字改成自己的提示文字。
3. 新增一列常見錯誤，例如重複 `id` 或屬性值未加引號。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 屬性與標籤名沒有空格 | 瀏覽器無法照預期解析元素 | 在標籤名後加入空格 |
| 同一元素重複寫同名屬性 | 結果不清楚，不應依賴瀏覽器採用哪個值 | 每個屬性只寫一次 |
| 屬性名稱大小寫混用 | 團隊維護時不一致 | 統一使用小寫屬性名 |
