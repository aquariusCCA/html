# 實務文章頁語意結構

## 1. Demo 目標

這個 demo 用來示範：

- 在實務文章頁中組合 `header`、`nav`、`main`、`article`、`section`、`aside`、`footer`。
- 一個頁面可以有網站層級的 `header` / `footer`，也可以有文章層級的 `header` / `footer`。
- 商品、景點、新聞摘要這類可獨立理解的項目可以使用 `article`。

## 2. 對應 notes

- 筆記路徑：`notes/第12章_語意化標籤/語意化標籤.md`
- 對應章節：`5. 範例說明`、`6. 實務應用場景`
- 對應知識點：
  - 網站首頁與文章頁的語意化結構
  - `article` 內部也可以使用 `header` 與 `footer`
  - `aside` 用於延伸閱讀、提醒或側邊補充資訊

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 觀察文章主內容與補充資訊的分工

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 網站名稱與主要導覽集中在最上方的 `header`。
2. 頁面只有一個主要 `main`，內含文章與補充資訊。
3. 主要文章使用 `article`，文章中用 `section` 拆分主題。
4. 每個景點卡片也使用 `article`，因為它可以獨立閱讀。
5. 旅遊提醒放在 `aside`，表示它是相關補充內容。

## 5. 程式碼重點

### HTML

- `site-header` 是網站層級頁首，`article-header` 是文章層級頁首。
- `section` 皆搭配明確標題，避免只有容器沒有主題。
- `aside` 使用 `aria-labelledby` 指向自己的標題。

### CSS

- 使用 class 控制不同層級的 `header` 與 `footer` 樣式。
- 響應式排版讓主內容與側邊補充在小螢幕改為上下排列。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 新增第三個景點卡片，判斷是否應使用 `article`。
2. 新增一個「費用預估」區塊，使用 `section` 並加上標題。
3. 把旅遊提醒移入文章內部，思考它和文章主線的關係是否改變。
4. 在頁面新增第二組導覽，並替兩組 `nav` 使用不同 `aria-label`。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 把文章頁所有區塊都寫成 `div` | 結構角色不清楚 | 依內容角色選擇語意化標籤 |
| `section` 沒有標題 | 區塊主題不明確 | 每個主題區塊加上能描述內容的標題 |
| 認為 `header` 只能出現在頁面最上方 | 無法表達文章內部頭部資訊 | 頁面與文章都可以有自己的 `header` |
| 把補充資訊放進主線段落 | 主內容與補充資訊混在一起 | 使用 `aside` 呈現相關但非主線的內容 |
