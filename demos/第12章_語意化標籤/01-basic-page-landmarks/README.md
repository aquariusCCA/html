# 基本頁面語意結構

## 1. Demo 目標

這個 demo 用來示範：

- 使用 `header`、`nav`、`main`、`article`、`section`、`aside`、`footer` 建立頁面骨架。
- `header` 與 `footer` 可以屬於整個頁面，也可以屬於單一 `article`。
- `aside` 適合放與主內容相關、但不是主線的補充資訊。

## 4. 觀察重點

1. 頁面最外層有網站用的 `header` 與 `footer`。
2. 主要內容集中在一個 `main` 中。
3. 文章本身使用 `article`，文章內部也有自己的 `header` 與 `footer`。
4. 文章段落以 `section` 分成主題區塊，並搭配標題。
5. 延伸閱讀放在 `aside`，表示它是補充內容。

## 5. 程式碼重點

### HTML

- `nav` 使用 `aria-label="主要導覽"` 表示這組連結的用途。
- `section` 搭配 `aria-labelledby` 與標題，讓主題更清楚。
- `aside` 與 `article` 並列在 `main` 中，形成主內容與補充內容的關係。

### CSS

- CSS 只用來讓區塊邊界更容易觀察。
- 使用 class 控制樣式，避免直接選取所有 `header` 或 `footer` 造成範圍過大。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 新增一個文章段落，並思考是否應使用新的 `section`。
2. 把 `aside` 移到 `article` 裡面，觀察它和文章的關係是否變得更緊密。
3. 移除 `nav` 的 `aria-label`，再思考如果頁面有多組導覽會不會難以區分。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 一頁放多個主要 `main` | 輔助工具難以判斷真正主內容 | 一頁通常只保留一個主要 `main` |
| `section` 沒有清楚標題 | 區塊主題不明確 | 為 `section` 加上能描述主題的標題 |
| 把所有補充內容都塞進 `article` | 主線與補充資訊混在一起 | 和主內容相關但不是主線時，可使用 `aside` |
