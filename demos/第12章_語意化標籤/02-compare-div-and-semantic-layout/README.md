# div 與語意化版面比較

## 1. Demo 目標

這個 demo 用來示範：

- `div class="nav"` 和 `nav` 在畫面上可以相似，但語意不同。
- 語意化標籤讓 HTML 元素直接表達內容角色。
- CSS 排版與 HTML 語意是不同責任。

## 2. 對應 notes

- 筆記路徑：`notes/第12章_語意化標籤/語意化標籤.md`
- 對應章節：`3.4 語意化標籤與 div 的差異`、`4.3 舊式 div 版面與語意化版面的比較`
- 對應知識點：
  - `div` 是無語意容器
  - 有原生 HTML 元素可以表達語意時，通常應優先使用原生元素
  - 語意化標籤的目的不是預設樣式

## 4. 觀察重點

1. 左右兩邊的版面示意很接近，但 HTML 寫法傳達的資訊不同。
2. `div` 寫法必須依賴 class 名稱才能知道區塊用途。
3. `header`、`nav`、`main`、`article`、`aside`、`footer` 直接描述內容角色。
4. CSS 仍然負責外觀；語意化標籤不會自動完成所有版面設計。

## 5. 程式碼重點

### HTML

- 左側用程式碼片段呈現舊式 `div` 寫法。
- 右側用程式碼片段呈現語意化寫法。
- 實際 demo 頁面本身仍維持合法結構，避免在同一頁插入多個主要 `main`。

### CSS

- `.visual-layout` 只是視覺示意，用來讓兩種寫法更容易比較。
- 左右色彩不同，只是輔助觀察，不代表語意化標籤有固定外觀。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把左側程式碼中的 `div class="nav"` 改成 `nav`。
2. 把左側程式碼中的 `div class="footer"` 改成 `footer`。
3. 在右側程式碼加入 `class="site-header"`，觀察語意標籤仍可搭配 class 控制樣式。
4. 修改 CSS 顏色，確認外觀變化不等於語意變化。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 認為 class 名稱等於語意 | `div` 本身仍是無語意容器 | 有合適元素時，優先使用語意化標籤 |
| 認為用了語意化標籤就不需要 CSS | 畫面仍缺少必要排版 | HTML 管結構，CSS 管視覺 |
| 為了套樣式而亂換標籤 | HTML 角色變得不準確 | 先根據內容意義選標籤，再用 CSS 排版 |
