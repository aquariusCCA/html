Demo 02

比較：div 版面與語意化版面

兩種寫法都能排出相似畫面，但 HTML 元素本身傳達的資訊不同。

同一個頁面骨架，兩種 HTML 寫法

只使用 `div` 與 class

<div class="header">網站頁首</div>
<div class="nav">主要導覽</div>
<div class="main">
  <div class="article">文章內容</div>
  <div class="aside">延伸閱讀</div>
</div>
<div class="footer">頁尾資訊</div>
div.header
div.nav
div.article
div.aside
div.footer

使用語意化標籤

<header>網站頁首</header>
<nav>主要導覽</nav>
<main>
  <article>文章內容</article>
  <aside>延伸閱讀</aside>
</main>
<footer>頁尾資訊</footer>
header
nav
article
aside
footer

觀察差異

比較項目 只用 `div` 語意化標籤
元素本身 都是無語意容器，需要看 class 才知道用途。 標籤名稱直接表達內容角色。
維護閱讀 後續維護者必須依賴命名與上下文判斷。 原始碼較像一份有結構的文件。
可訪問性與 SEO 機器較難直接理解區域角色。 輔助工具與搜尋引擎較容易判斷頁面區塊。
