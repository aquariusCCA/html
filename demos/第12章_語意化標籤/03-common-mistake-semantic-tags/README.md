# 語意化標籤常見錯誤

## 1. Demo 目標

這個 demo 用來示範：

- `section` 不應只當成排版容器。
- `nav` 應用在主要或重要導覽，不是所有連結都要包進 `nav`。
- 可獨立閱讀的內容適合使用 `article`。
- 一頁通常只應有一個主要 `main`。

## 2. 對應 notes

- 筆記路徑：`notes/第12章_語意化標籤/語意化標籤.md`
- 對應章節：`7. 常見錯誤與修正`
- 對應知識點：
  - 只因為想排版就使用 `section`
  - 每個連結群組都放進 `nav`
  - 把 `article` 當成一般區塊
  - 一頁放多個主要 `main`
  - 用語意化標籤後不寫標題

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 比較每組錯誤示意與修正示意
4. 查看下方「結構檢查結果」

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 錯誤的 `section` 沒有明確主題標題，只是包住商品卡片。
2. 修正後的 `section` 有「精選商品」標題，內部商品使用 `article` 表示可獨立理解的項目。
3. `nav` 應保留給主要導覽或重要導覽。
4. 多個主要 `main` 的錯誤以程式碼片段呈現，避免讓 demo 頁面本身真的變成多個主內容區。

## 5. 程式碼重點

### HTML

- `.mistake-box` 呈現常見誤用。
- `.correct-box` 呈現修正方向。
- 多個 `main` 的錯誤使用 `<pre><code>` 顯示，避免破壞此 demo 頁面本身的結構。

### CSS

- 橘色左框表示錯誤示意。
- 綠色左框表示修正示意。
- 樣式只用來輔助比較，不代表 HTML 元素的預設外觀。

### JavaScript

- 檢查特定區塊是否有標題。
- 顯示錯誤卡片與修正卡片的外層元素名稱。
- 顯示本頁實際主要 `main` 數量。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 為錯誤 `section` 加上標題，再觀察結構檢查結果。
2. 把錯誤卡片外層從 `div` 改成 `article`。
3. 把不重要的補充連結移出 `nav`。
4. 修改多個 `main` 的程式碼片段，練習改成單一 `main` 搭配 `section` 與 `aside`。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 用 `section` 當純排版盒子 | 產生沒有明確主題的文件區塊 | 純排版容器用 `div`，主題區塊才用 `section` |
| 所有連結都放進 `nav` | 導覽角色變得不精準 | 主要導覽、頁內目錄等重要連結才使用 `nav` |
| 可獨立內容不用 `article` | 列表項目語意較弱 | 新聞、文章、商品卡等可獨立理解內容可用 `article` |
| 一頁多個主要 `main` | 主內容判斷混亂 | 一頁通常只保留一個主要 `main` |
