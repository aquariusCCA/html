# 視覺效果與 HTML 語意比較

## 1. Demo 目標

這個 demo 用來示範：

- 外觀相近的標籤不一定語意相同。
- `<strong>`、`<em>`、`<ins>`、`<del>` 適合表達明確語意。
- `<b>`、`<i>`、`<u>`、`<s>` 仍有用途，但不應取代更準確的語意標籤。

## 4. 觀察重點

1. 左右兩欄可能有相近外觀，但使用情境不同。
2. `<strong>` 表示內容重要，不只是加粗。
3. `<ins>` 和 `<del>` 特別適合文件修訂或版本差異。
4. `<em>` 表示語氣強調，會影響句子重點。

## 5. 程式碼重點

### HTML

- 每一列使用兩個範例對照。
- 右欄示範更明確的語意標籤。
- 使用 `<pre><code>` 顯示關鍵程式碼。

### CSS

- CSS 使用邊框與左右欄排版輔助比較。
- 樣式不改變 HTML 元素的語意。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把右欄的 `<strong>` 改成 `<b>`，思考語意是否變弱。
2. 把文件修訂範例中的 `<del>` 改成 `<s>`，判斷是否仍能表達「刪除」。
3. 改寫一個自己的價格範例，使用 `<s>` 表示原價。
4. 改寫一個自己的提醒句，使用 `<strong>` 表示重要內容。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 看到粗體就一律用 `<strong>` | 會誤把一般醒目文字標成重要內容 | 只有重要內容才用 `<strong>` |
| 看到刪除線就一律用 `<del>` | 可能把原價或過期資訊誤標為修訂刪除 | 不再適用可用 `<s>` |
| 用 `<u>` 表示新增內容 | 語意不明確 | 使用 `<ins>` |
