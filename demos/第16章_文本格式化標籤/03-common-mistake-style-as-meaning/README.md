# 常見錯誤：把樣式當成語意

## 1. Demo 目標

這個 demo 用來示範：

- 不應只為了粗體而使用 `<strong>`。
- 不應用 `<u>` 表示新增內容。
- 不應用 `<s>` 表示正式刪除內容。
- 純視覺需求應交給 CSS，HTML 應保留正確語意。

## 4. 觀察重點

1. 錯誤與修正可能看起來相近，但 HTML 語意不同。
2. 純粹的字體粗細可以由 CSS 的 `font-weight` 控制。
3. 新增內容應使用 `<ins>`。
4. 文件中被刪除的內容應使用 `<del>`。

## 5. 程式碼重點

### HTML

- 左欄展示錯誤或不精準的語意。
- 右欄展示更合適的寫法。
- 使用 `<pre><code>` 顯示關鍵程式碼。

### CSS

- `.lead-text` 使用 CSS 控制字體變粗。
- `.wrong` 與 `.correct` 只用來輔助觀察錯誤與修正。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把 `.lead-text` 的 `font-weight` 改成 `400`，觀察視覺變化。
2. 把修正欄的 `<ins>` 改成 `<u>`，判斷語意是否變弱。
3. 新增一組錯誤：用 `<em>` 只做斜體外觀，再用 CSS 修正。
4. 新增一個真正重要的提醒句，使用 `<strong>`。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 把所有粗體文字都寫成 `<strong>` | 語意過度強調 | 純樣式用 CSS |
| 用 `<u>` 做文件新增標記 | 無法明確表達新增語意 | 使用 `<ins>` |
| 用 `<s>` 做版本刪除內容 | 語意不夠精準 | 使用 `<del>` |
