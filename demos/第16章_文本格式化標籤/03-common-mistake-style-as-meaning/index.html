Demo 03

常見錯誤：把樣式當成語意

HTML 應描述內容意義，CSS 負責控制外觀。 這個 demo 展示初學者常見的錯誤寫法與較合適的修正方式。

錯誤與修正對照

錯誤：只為了變粗使用 strong

這段只是設計稿要求放大加粗，沒有特別重要。

<strong>這段只是設計稿要求放大加粗</strong>

修正：一般段落加上 CSS class

這段只是設計稿要求放大加粗，沒有特別重要。

<p class="lead-text">這段只是設計稿要求放大加粗</p>

錯誤：用 u 表示新增內容

課程新增單元：文本格式化標籤

<u>文本格式化標籤</u>

修正：新增內容使用 ins

課程新增單元：文本格式化標籤

<ins>文本格式化標籤</ins>

錯誤：用 s 表示文件刪除

舊版說明：本欄位可以留空。

<s>本欄位可以留空。</s>

修正：正式刪除內容使用 del

舊版說明：本欄位可以留空。

<del>本欄位可以留空。</del>
