# 實務應用：商品與提醒文字

## 1. Demo 目標

這個 demo 用來示範：

- 商品原價適合用 `<s>` 表示不再適用。
- 重要提醒適合用 `<strong>` 表示需要注意。
- 文件修訂適合用 `<del>` 與 `<ins>` 表示刪除與新增。
- 技術術語與語氣強調可以分別使用 `<i>` 與 `<em>`。

## 4. 觀察重點

1. `<s>` 表示原價不再適用，不等於正式文件刪除。
2. `<strong>` 表示重要提醒，可以再用 CSS 強化視覺。
3. `<del>` 與 `<ins>` 可以表達時間更新的修訂關係。
4. `<i>` 用於術語，`<em>` 用於語氣強調。

## 5. 程式碼重點

### HTML

- 商品卡片使用 `<s>` 與 `<strong>`。
- 課程公告使用 `<del>` 與 `<ins>`。
- 技術說明使用 `<i>` 與 `<em>`。

### CSS

- `.price` 只控制顏色，不取代 `<strong>` 的重要語意。
- `del`、`s`、`ins` 的樣式只用於輔助觀察。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 將商品原價的 `<s>` 改成 `<del>`，思考語意是否正確。
2. 新增一個重要提醒句，使用 `<strong>`。
3. 新增一個課程時間修訂，使用 `<del>` 與 `<ins>`。
4. 將 `<em>` 移到句子中的不同詞語，觀察語氣重點改變。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 商品原價一律用 `<del>` | 原價不一定是文件修訂刪除 | 原價可使用 `<s>` |
| 重要提醒只靠紅色 | 只靠顏色不夠清楚 | 搭配 `<strong>` 與清楚文字 |
| 把術語和語氣強調都用 `<em>` | 語意可能混淆 | 術語可用 `<i>`，語氣強調用 `<em>` |
