# 01-塊級元素與行內元素比較 HTML 教學範例

來源筆記：`notes/第07章_HTML塊級元素與行內元素/01-塊級元素與行內元素比較.md`

## 建議學習順序

| 順序 | Demo | 對應觀念 | 觀察重點 | 建議修改任務 |
| --- | --- | --- | --- | --- |
| 1 | `01-預設排列比較.html` | 塊級元素與行內元素的預設排列 | `<h1>`、`<p>`、`<div>` 會由上往下排列；`<span>`、`<em>`、`<a>`、`<input>` 會跟文字同列 | 在塊級區加入一個 `<p>`，在行內區加入一個 `<span>`，觀察排列差異 |
| 2 | `02-寬高設定比較.html` | `width`、`height` 對兩類元素的影響 | `div` 設定寬高後形成明顯區塊；一般 `span` 設定寬高不會像區塊一樣撐開 | 修改兩個元素的 `width` 與 `height` 數值，觀察哪一個比較符合預期 |
| 3 | `03-間距效果比較.html` | `margin`、`padding` 對兩類元素的影響 | 塊級元素上下左右間距都明顯影響排列；行內元素左右較直覺，上下效果較有限 | 調大 `margin` 或 `padding`，比較 div 與 span 周圍空間的變化 |
| 4 | `04-例外與display改變.html` | 置換元素與 CSS `display` 改變排版行為 | `input` 可以有可見寬高；`display: block` 或 `display: inline` 會改變元素排列方式 | 把 `span` 的 `display: block` 移除，再把 `div` 的 `display: inline` 移除，觀察回到預設排列 |
| 5 | `05-巢狀判斷不是只看塊級行內.html` | 塊級 / 行內不是完整巢狀規則 | `<p>` 不能因為看起來像區塊就包住 `<div>`；`input` 這類 void elements 不能有子內容 | 將正確寫法中的 `<div>` 移到 `<p>` 外不同位置，確認結構仍然清楚 |

## 學習提醒

這組 demo 的目標不是取代正式筆記，而是幫助你把筆記中的觀念轉成可觀察、可修改、可練習的 HTML 範例。
