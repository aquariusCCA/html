# 01-p段落標籤與文章分段 HTML 教學範例

來源筆記：`notes/第11章_段落標籤/01-p段落標籤與文章分段.md`

## 建議學習順序

| 順序 | Demo | 對應觀念 | 觀察重點 | 建議修改任務 |
| --- | --- | --- | --- | --- |
| 1 | `01-p表示完整段落.html` | `<p>` 表示一段完整文字 | 一組 `<p>` 和 `</p>` 如何包住一個段落 | 把段落文字改成商品介紹或教學說明，保留完整 `<p>` 結構 |
| 2 | `02-多個p組成文章.html` | 多個段落使用多組 `<p>` | 文章主題與多個段落如何形成清楚結構 | 新增第四個段落，讓它提出結論或下一步 |
| 3 | `03-瀏覽器預設段落外觀.html` | 瀏覽器提供段落預設外觀 | 段落通常獨佔區塊、上下有空隙，文字會依容器寬度自動換行 | 加長段落文字或調整範例寬度，觀察自動換行 |
| 4 | `04-段落不是原始碼換行.html` | 段落語意不同於原始碼換行 | 在同一個 `<p>` 內換行不會產生新的段落語意 | 把三句文字先放在同一個 `<p>`，再拆成三個 `<p>` 比較 |
| 5 | `05-br不取代文章分段.html` | `<br>` 不應取代一般文章分段 | `<br>` 只表示換行，多段正文應使用多個 `<p>` | 把 `<br><br>` 寫法改成兩個 `<p>` |
| 6 | `06-p不要包住清單結構.html` | `<p>` 不應包住清單等結構 | 清單應使用 `<ul>` 和 `<li>`，不要硬塞進段落 | 新增一個清單項目，確認 `<ul>` 沒有被包在 `<p>` 裡 |
| 7 | `07-不要用空p控制間距.html` | 不要用空 `<p>` 控制間距 | HTML 負責語意，段落間距應交給 CSS | 刪除空 `<p>`，調整 CSS 的 `margin-bottom` 觀察間距 |

## 學習提醒

這組 demo 的目標不是取代正式筆記，而是幫助你把筆記中的觀念轉成可觀察、可修改、可練習的 HTML 範例。
