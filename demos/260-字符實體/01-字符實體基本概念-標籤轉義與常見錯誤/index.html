標籤轉義與常見錯誤

如果想在頁面上「顯示」一段 HTML 標籤文字（例如教別人 <p> 是什麼）， 必須把 < 寫成 &lt;、> 寫成 &gt;， 否則瀏覽器會把它當成真正的標籤解析。

互動：同一段文字，兩種對待方式

在輸入框輸入含有 <、> 的內容， 切換下方開關，觀察「當成 HTML 解析」與「當成文字顯示」的差別。

        「當成文字顯示」時，< 與 > 會被轉義成 &lt; 與 &gt;，
        所以標籤只會被當成文字，不會被執行。
      

    


    

      
常見錯誤對照


      

        
1. 漏掉分號

        

          

            錯誤
            
&copy

          

          

            正確
            
&copy;

          

        

        
命名字符實體通常要以分號 ; 結尾，漏掉可能導致解析結果不穩定，也降低可讀性。

      


      

        
2. 把 HTML 標籤直接寫成文字

        

          

            錯誤
            
<h1> 是標題標籤

            
瀏覽器把 <h1> 當成真正的標題開始標籤：

            
 是標題標籤

          

          

            正確
            
&lt;h1&gt; 是標題標籤

            
畫面正確顯示這段文字：

            
<h1> 是標題標籤

          

        

        
要顯示標籤文字時，把 <、> 轉義成 &lt;、&gt;。

      


      

        
3. 用大量 &nbsp; 做排版

        

          

            不建議
            
姓名：&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;小明

          

          

            建議
            
用 CSS：margin / padding / gap

          

        

        
&nbsp; 適合處理文字中的特殊空格，控制元素間距離應交給 CSS 排版。

      


      

        
4. 誤以為所有空白都會被保留

        

          一般文字排版中連續空白與換行會被合併，但
          <pre> 會保留空白與換行、<textarea> 會保留輸入內容、
          CSS white-space 也會改變空白顯示方式。看到結果不同時，先確認目前元素與 CSS 規則。
