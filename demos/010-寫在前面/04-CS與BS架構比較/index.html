架構比較

C/S 與 B/S 的差異

選擇比較項目，觀察需要安裝的客戶端程式與只需瀏覽器的 Web 應用，在使用與維護上有什麼不同。

C/S 架構

Client / Server

  • 使用前通常需要安裝專屬客戶端。
  • 客戶端可能承擔部分運算與業務邏輯。
  • 升級時常需要更新每一個客戶端。
  • 適合需要本機能力或受控安裝環境的場景。

B/S 架構

Browser / Server

  • 使用者有瀏覽器即可訪問。
  • 主要邏輯在伺服器端，瀏覽器端較輕量。
  • 更新伺服器即可讓使用者取得新版。
  • 表現層、邏輯層、數據層分工清楚。

安裝門檻

C/S 通常先下載安裝專屬客戶端；B/S 則只要打開瀏覽器輸入網址即可。
