限制與常見錯誤對照

錯誤寫法：把 html5shiv 當成完整 HTML5 支援工具

html5shiv = 完整支援所有 HTML5 功能

這會誤解 html5shiv 的範圍。

修正理解：只協助部分 HTML5 元素

html5shiv 主要協助舊 IE 識別與樣式化部分 HTML5 新元素，不會讓舊 IE 完整支援所有 HTML5 API。

錯誤寫法：對所有瀏覽器都載入

<script src="./js/html5shiv.min.js"></script>

非舊 IE 瀏覽器不需要這個相容性腳本。

修正方向：限制在低於 IE9 的 IE

<!--[if lt IE 9]>
<script src="./js/html5shiv.min.js"></script>
<![endif]-->

錯誤寫法：用條件註解控制 IE10

<!--[if IE 10]>IE10 專用內容<![endif]-->

IE10 開始不再支援條件註解。

修正理解：IE10 當成一般瀏覽器處理

這類條件註解主要是舊系統中照顧 IE6 到 IE8 的歷史兼容方案，不是現代新專案的常規寫法。
