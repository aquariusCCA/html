# 語意化文章結構與常見錯誤

## 1. Demo 目標

這個 demo 用來示範：

- 語意化元素如何讓文章頁結構更清楚。
- 過度使用 `div` 會造成哪些理解問題。
- 標題層級應表達文件結構，而不是單純控制字體大小。

## 4. 觀察重點

1. 左右兩邊畫面可能看起來相近，但 HTML 結構意義不同。
2. `header`、`section`、`article` 讓頁面區塊更容易被理解。
3. `h1`、`h2`、`h3` 是文件層級，不應只用來調整字體大小。

## 5. 修改練習

1. 將右側的 `fake-title` 改成合適的標題元素。
2. 將右側的 `fake-link` 改成真正的 `a` 元素。
3. 把右側錯誤標題層級調整成連續且合理的順序。

## 6. 常見錯誤

| 錯誤 | 問題 | 修正方式 |
|---|---|---|
| 所有區塊都用 `div` | 缺少語意，搜尋引擎與輔助工具較難理解 | 有明確意義時使用 `header`、`section`、`article` 等元素 |
| 用標題元素控制字體大小 | 文件大綱會混亂 | 先用標題表示結構，視覺大小交給 CSS |
| 用 `div` 假裝連結 | 無法真正導向其他頁面，也不利鍵盤操作 | 使用 `a href="..."` 建立連結 |
