# 語意化課程卡片列表

## 1. Demo 目標

這個 demo 用來示範：

- 使用語意元素建立頁面結構。
- 使用塊級元素組織卡片列表。
- 使用 `span` 標記卡片中的局部狀態文字。
- 用單一 `a` 包住卡片內容，避免巢狀連結。

## 4. 觀察重點

1. `header`、`nav`、`main`、`section`、`article` 都在描述內容角色。
2. 卡片使用 `article` 作為獨立內容項目。
3. 狀態文字使用 `span`，因為它只是標記標題前的一小段文字。
4. 每張卡片只有一個 `a`，沒有在連結裡再放另一個連結。

## 5. 程式碼重點

### HTML

- `nav aria-label="課程分類"` 表示這是一組課程分類導覽。
- `article` 表示每張課程卡片是一個獨立項目。
- `span class="status"` 標記卡片中的局部狀態。

### CSS

- `.course-grid` 使用 grid 排列卡片。
- `.course-card` 使用 `display: block` 讓整張卡片成為可點擊區域。
- CSS 只改變呈現方式，不改變 HTML 語意。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把 `span class="status"` 改成 `div`，觀察卡片內部排列是否改變。
2. 在卡片內新增另一個 `a`，再思考為什麼這是不建議的結構。
3. 把 `.course-card` 的 `display: block` 移除，觀察可點擊範圍與排版差異。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 所有外層結構都用 `div` | 頁面語意不清楚 | 使用 `header`、`main`、`section`、`article`、`nav` |
| 在卡片連結裡再放另一個連結 | 互動目標不清楚 | 每張卡片使用單一連結，或拆成不同互動區域 |
| 用 `span` 包住整張卡片 | 行內元素不適合承載大型區塊結構 | 使用 `article`、`section` 或 `div` 等區塊容器 |
| 用 CSS 外觀取代語意判斷 | 視覺正確但結構難以維護 | 先選正確 HTML 元素，再用 CSS 排版 |
