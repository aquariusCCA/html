# 常見巢狀錯誤與 DOM 解析

## 1. Demo 目標

這個 demo 用來示範：

- `p` 元素不能包含 `div` 這類區塊內容。
- `h1` 到 `h6` 不應互相嵌套。
- `a` 元素不能包含另一個 `a`。
- 瀏覽器可能自動修正不合法 HTML，導致 DOM 結構和原始碼不同。

## 4. 觀察重點

1. 錯誤區塊中的 `div` 不會真的留在 `p` 裡。
2. `h2` 不會成為 `h1` 的子元素。
3. 內層 `a` 不會成為外層 `a` 的子元素。
4. 下方 JavaScript 輸出的父元素資訊可以協助確認實際 DOM。

## 5. 程式碼重點

### HTML

- 第一欄刻意保留錯誤結構，用於觀察瀏覽器如何解析。
- 第二欄提供正式專案應採用的正確寫法。

### CSS

- `.mistake-box` 與 `.correct-box` 用不同底色協助對照。
- 樣式只用於觀察，不修正錯誤 HTML。

### JavaScript

- `main.js` 使用 DOM API 讀取元素的實際父元素。
- 觀察結果會輸出到 `#dom-results`。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 把錯誤的 `div` 改成 `span`，觀察 `p` 的子節點變化。
2. 移除巢狀 `h2`，改成平行的 `h1`、`h2`。
3. 把兩個連結拆開，確認 `#inner-link` 的父元素是否仍然合理。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 在 `p` 裡放 `div` | 瀏覽器可能自動結束 `p`，DOM 與原始碼不同 | 將 `div` 放在 `p` 外面 |
| 將 `h2` 寫在 `h1` 裡 | 文件標題層級混亂 | 讓標題依層級平行排列 |
| 在 `a` 裡再放 `a` | 互動目標不清楚，鍵盤與輔助工具難以處理 | 將連結拆成平行元素 |
| 只看原始碼不看 DOM | 無法發現瀏覽器自動修正後的結構 | 使用開發者工具檢查實際 DOM |
