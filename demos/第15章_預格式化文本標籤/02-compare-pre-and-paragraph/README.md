# pre 與 p 空白處理比較

## 1. Demo 目標

這個 demo 用來示範：

- 一般 HTML 文字會進行空白折疊。
- `<p>` 適合一般段落，不適合展示固定格式程式碼。
- `<pre>` 適合保留空格、縮排與換行。
- 視覺間距應由 CSS 控制。

## 2. 對應 notes

- 筆記路徑：`notes/第15章_預格式化文本標籤/預格式化文本標籤.md`
- 對應章節：`2. 本章定位`、`3. 核心概念`、`5. 範例說明`、`8. 與其他概念的關係`
- 對應知識點：
  - 一般 HTML 空白折疊
  - `<p>` 與 `<pre>` 的差異
  - 固定格式文字應使用 `<pre>`

## 4. 觀察重點

1. 左側 `<p>` 不會完整保留原始碼中的縮排與換行。
2. 右側 `<pre>` 會保留 CSS 片段的格式。
3. 同一段文字放在不同 HTML 元素中，會因元素語意與空白處理規則不同而呈現不同結果。
4. 如果內容是一般段落，不應為了排版改用 `<pre>`。

## 5. 程式碼重點

### HTML

- 左側用 `<p>` 放入 CSS 文字。
- 右側用 `<pre>` 放入同一類 CSS 文字。
- 表格整理選用判斷。

### CSS

- `.compare-grid` 讓兩種寫法並排比較。
- `.paragraph-sample` 與 `.pre-sample` 只用邊框和背景輔助觀察。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 在 `<p>` 裡增加更多空格，觀察畫面是否完整保留。
2. 在 `<pre>` 裡增加一層縮排，觀察畫面是否同步變化。
3. 把右側 `<pre>` 改成 `<p>`，比較結果。
4. 移除 `.pre-sample` 的 `font-family`，觀察等寬字體差異。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 用 `<p>` 顯示多行程式碼 | 程式碼縮排與換行不清楚 | 使用 `<pre>` |
| 用 `<pre>` 包一般段落 | 內容可能有多餘空白 | 普通正文使用 `<p>` |
| 用 `<pre>` 控制間距 | HTML 語意混亂 | 使用 CSS 的 `margin` 或 `padding` |
