# 常見錯誤：pre 中 HTML 未轉義

## 1. Demo 目標

這個 demo 用來示範：

- `<pre>` 會保留文字格式，但不會自動轉義 HTML 標籤。
- 直接在 `<pre>` 中放 HTML 標籤時，瀏覽器仍會解析標籤。
- 若要顯示 HTML 原始碼，需要使用 HTML entity。
- `&lt;` 與 `&gt;` 是顯示 HTML 標籤時常見的必要寫法。

  - 顯示 HTML 原始碼需要轉義 `<` 與 `>`

## 4. 觀察重點

1. 錯誤寫法中的 `<h1>` 與 `<p>` 不是以純文字方式顯示。
2. 正確寫法會顯示 `<h1>Hello HTML</h1>` 這類原始碼文字。
3. `<pre>` 負責保留格式，不負責轉義 HTML。
4. 顯示 HTML 原始碼時，特殊字元要先轉成 entity。

## 5. 程式碼重點

### HTML

- 錯誤區塊刻意直接放入 `<h1>` 與 `<p>`。
- 正確區塊使用 `&lt;` 與 `&gt;` 顯示標籤文字。
- 表格整理本 demo 用到的轉義字元。

### CSS

- `.mistake-source` 與 `.correct-source` 使用不同邊框色輔助比較。
- CSS 不修正 HTML 解析問題，只讓結果更容易觀察。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 在錯誤區塊加入 `<button>送出</button>`，觀察它是否被當成元素。
2. 在正確區塊加入 `&lt;button&gt;送出&lt;/button&gt;`，觀察是否顯示成原始碼。
3. 將正確區塊中的 `&lt;` 改回 `<`，觀察畫面變化。
4. 在 `<code>` 內新增一行 HTML 原始碼，練習手動轉義。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 以為 `<pre>` 會自動轉義 HTML | 標籤被瀏覽器解析，不會原樣顯示 | 使用 `&lt;` 與 `&gt;` |
| 只轉義 `<`，沒有轉義 `>` | 程式碼呈現不一致 | 兩個尖括號都轉義 |
| 為了顯示程式碼只使用 `<pre>` | 少了程式碼語意 | 程式碼內容建議使用 `<pre><code>...</code></pre>` |
