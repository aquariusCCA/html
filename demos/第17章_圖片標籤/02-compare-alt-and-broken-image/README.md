# 比較 alt 與圖片載入失敗

## 1. Demo 目標

這個 demo 用來示範：

- 圖片正常載入時的顯示結果。
- 圖片路徑錯誤時，有 `alt` 和沒有 `alt` 的差異。
- `alt` 不是裝飾文字，而是圖片失敗與輔助工具讀取時的重要資訊。

## 2. 對應 notes

- 筆記路徑：`notes/第17章_圖片標籤/圖片標籤.md`
- 對應章節：`4.3 alt：替代文字`、`8. 常見錯誤與修正`
- 對應知識點：
  - `alt` 是圖片替代文字
  - 圖片載入失敗時仍應讓使用者理解內容
  - 缺少 `alt` 會造成可訪問性問題

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 比較三張圖片區塊的顯示差異

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 第一張圖片路徑正確，可以正常顯示。
2. 第二張圖片路徑錯誤，但 `alt` 仍提供圖片用途。
3. 第三張圖片路徑錯誤且缺少 `alt`，使用者很難理解原本內容。
4. `alt` 應描述圖片在頁面中的用途，而不是只寫檔名。

## 5. 程式碼重點

### HTML

- 第二與第三個範例刻意使用不存在的圖片路徑。
- 第二個範例保留 `alt`，第三個範例故意省略 `alt`。

### CSS

- CSS 只用不同邊框色輔助比較，不修正圖片載入問題。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 替第三張圖片補上 `alt="使用者個人頭像"`。
2. 將第二張圖片的 `src` 改回正確路徑 `../assets/images/avatar-profile.svg`。
3. 將 `alt` 改成 `avatar-profile.svg`，比較它是否比「使用者個人頭像」更有意義。
4. 新增一張裝飾圖片，並練習使用 `alt=""`。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 圖片缺少 `alt` | 圖片失敗或輔助工具讀取時資訊不足 | 為有內容意義的圖片提供替代文字 |
| `alt` 只寫檔名 | 使用者仍不知道圖片用途 | 描述圖片在頁面中的意義 |
| 把所有圖片都寫很長的 `alt` | 裝飾圖片會造成讀取干擾 | 裝飾圖片可使用 `alt=""` |
