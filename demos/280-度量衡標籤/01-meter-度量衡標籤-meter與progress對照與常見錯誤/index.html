meter vs progress，與常見錯誤

<meter> 描述「某個值的狀態」，<progress> 描述「某件事完成到哪裡」。 下面用真實渲染的元素對照兩者，並示範三種常見錯誤的後果與修正。

一、語意對照

用 <meter>

手機電量目前 50%

手機電量： 50%

為什麼：這是「已知範圍內的測量值」，描述的是電量這個值的狀態。

用 <progress>

檔案下載到 60%

下載中： 60%

為什麼：這是「任務完成進度」，描述的是下載這件事做到哪裡。

情境判斷小卡

  • 手機電量目前 50%meter已知範圍內的測量值
  • 磁碟已使用 80GB / 100GBmeter容量用量的度量
  • 檔案下載到 60%progress任務完成進度
  • 表單送出處理中progress流程或任務進行狀態

判斷一句話：這個數字是在描述「某個值的狀態」，還是「某件事完成到哪裡」？

二、錯誤一：用 meter 表示下載進度

✗ 不建議

下載 60%，卻用了 meter

60% 
<meter value="60" min="0" max="100">60%</meter>

下載進度是任務進行狀態，不是度量衡數值，語意不對。

✓ 建議

改用 progress

60% 
<progress value="60" max="100">60%</progress>

progress 正確表達「下載這件事完成 60%」。

三、錯誤二：只給 value，沒有清楚範圍

✗ 錯誤

想表示 50/100，卻沒寫 min/max

50% 
<meter value="50">50%</meter>

未指定 min/max 時，預設範圍是 0~1，value=50 超出範圍，瀏覽器會當成最大值（量表全滿）。

✓ 正確

明確寫出 min 與 max

50% 
<meter value="50" min="0" max="100">50%</meter>

範圍清楚，value=50 才會正確顯示在一半的位置。

比較上面兩個量表的填滿程度，就能看出沒寫 min/max 的後果。

四、錯誤三：沒有文字說明量表代表什麼

✗ 語意不夠清楚

只有一個量表，看不出代表什麼

70% 
<meter value="70" min="0" max="100"></meter>

使用者不知道這是電量、容量還是評分。

✓ 正確

搭配文字標籤或上下文

手機電量： 70% 
<span>手機電量：</span>
<meter value="70" min="0" max="100">70%</meter>

加上文字後，量表的意義一目了然。
