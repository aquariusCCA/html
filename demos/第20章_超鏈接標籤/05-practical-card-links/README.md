# 實務應用：整張卡片都是連結

## 1. Demo 目標

這個 demo 用來示範：

- `<a>` 可以包裹一組區塊內容，讓整張卡片可點擊。
- 圖片放在連結內時，仍需要合適的 `alt`。
- 卡片連結不應巢狀其他互動元素。
- 如何用錨點連結模擬文章卡片的目標頁面。

## 2. 對應 notes

- 筆記路徑：`notes/第20章_超鏈接標籤/超鏈接標籤.md`
- 對應章節：`4. 語法與基本用法`、`5. 範例說明`、`7. 常見錯誤與修正`
- 對應知識點：
  - `<a>` 可以包裹其他元素
  - 網頁元素連結
  - 圖片連結的 `alt`
  - 避免互動元素巢狀

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 點擊任一文章卡片，觀察頁面跳轉

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 卡片圖片、標題、文字都在同一個 `<a>` 裡。
2. 點擊卡片任何位置都會前往同一個目標。
3. 圖片的 `alt` 描述了圖片與連結情境。
4. 卡片內沒有再放其他 `<a>` 或 `<button>`。

## 5. 程式碼重點

### HTML

- 使用 `<a class="article-card" href="#article-html-links">` 包裹圖片與文字。
- 使用 `img alt` 說明圖片內容或連結目的。

### CSS

- 使用 `display: block` 讓 `<a>` 像區塊卡片一樣呈現。
- 使用 hover 與 focus 樣式提示整張卡片可點擊。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 修改第一張卡片的 `href`，讓它跳到第二個目標區塊。
2. 移除圖片的 `alt`，思考對可訪問性的影響。
3. 新增第三張卡片與對應目標區塊。
4. 嘗試在卡片中加入一個 `<button>`，再思考為什麼這不是好結構。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 卡片連結內再放另一個連結 | 互動元素巢狀，操作不穩定 | 將多個操作拆成並列元素 |
| 圖片連結缺少 `alt` | 輔助工具無法理解連結目的 | 提供描述圖片或連結目的的 `alt` |
| 只靠 hover 表示可點擊 | 鍵盤使用者看不到 hover | 同時提供清楚的 focus 樣式 |
| 卡片連結文字過於模糊 | 使用者不知道會前往哪裡 | 使用具體標題與清楚的操作文字 |
