# 錨點連結：href="#id"

## 1. Demo 目標

這個 demo 用來示範：

- 如何使用 `href="#id"` 建立頁內錨點連結。
- 錨點連結如何對應目標元素的 `id`。
- 如何建立頁內目錄與「回到頁首」連結。

## 2. 對應 notes

- 筆記路徑：`notes/第20章_超鏈接標籤/超鏈接標籤.md`
- 對應章節：`5. 範例說明`、`7. 常見錯誤與修正`
- 對應知識點：
  - 頁面錨點
  - `href="#名稱"`
  - `id="名稱"`
  - `href="#"` 與明確錨點的差異

## 3. 如何執行

方式一：直接打開檔案

1. 找到 `index.html`
2. 使用瀏覽器開啟
3. 點擊目錄連結並觀察頁面捲動

方式二：使用 VS Code Live Server

1. 在 demo 資料夾上按右鍵
2. 選擇 `Open with Live Server`
3. 在瀏覽器中查看結果

## 4. 觀察重點

1. 目錄中的 `href="#intro"` 會跳到 `id="intro"` 的區塊。
2. 每個內容區塊都有明確且不重複的 `id`。
3. 「回到頁首」連結指向 `id="top"`。
4. 若 `href` 的名稱與 `id` 不一致，跳轉就不會成功。

## 5. 程式碼重點

### HTML

- 使用 `<nav aria-label="本頁目錄">` 表示本頁導覽。
- 使用 `href="#file-link"` 對應 `id="file-link"`。

### CSS

- `scroll-behavior: smooth` 讓跳轉較容易觀察。
- `scroll-margin-top` 讓跳轉後的區塊不會貼齊視窗頂端。

### JavaScript

- 此 demo 不需要 JavaScript。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 將 `href="#file-link"` 改成 `href="#file"`，觀察跳轉是否成功。
2. 新增一個內容區塊與對應目錄連結。
3. 移除 `scroll-behavior: smooth`，觀察跳轉感受。
4. 將兩個區塊設定成相同 `id`，觀察瀏覽器會跳到哪一個。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| `href="#xxx"` 找不到對應 `id` | 點擊後不會跳到預期位置 | 確認名稱完全一致 |
| 多個元素使用相同 `id` | 跳轉目標不明確，文件結構不合法 | 每個 `id` 應保持唯一 |
| 長期濫用 `href="#"` | 可能造成頁面跳到頂部 | 使用明確錨點或改用 `button` |
