# 常見錯誤：用連結假裝按鈕

## 1. Demo 目標

這個 demo 用來示範：

- `<a>` 的語意是導航到某個位置。
- `<button>` 的語意是執行頁面內動作。
- 為什麼不建議用 `<a href="#">` 或類似寫法假裝按鈕。
- 如何用小段 JavaScript 觀察按鈕動作。

## 2. 對應 notes

- 對應知識點：
  - 空連結
  - `href="#"`
  - `javascript:` 連結不適合作為一般互動寫法
  - `<a>` 與 `<button>` 的語意差異

## 4. 觀察重點

1. 錯誤範例看起來像按鈕，但本質是連結。
2. 點擊 `href="#top"` 會造成頁面跳到頁首。
3. 正確範例使用 `button`，點擊後只執行頁面內動作。
4. `aria-live="polite"` 讓狀態文字更新時更容易被輔助工具感知。

## 5. 程式碼重點

### HTML

- 用 `<a href="#top">` 示範錯誤語意。
- 用 `<button type="button">` 示範頁面內動作。

### CSS

- 讓錯誤範例與正確範例外觀相近，凸顯語意差異不是只看畫面。

### JavaScript

- 只監聽 `button` 的 click 事件，更新狀態文字。

## 6. 修改練習

請嘗試修改以下內容，觀察結果差異：

1. 將錯誤範例的 `href="#top"` 改成 `href="#"`。
2. 移除 `button` 的 `type="button"`，思考它在表單中可能造成什麼問題。
3. 將狀態文字改成自己的訊息。
4. 新增一個真正的導航連結，例如跳到 `#code-example`。

## 7. 常見錯誤

| 錯誤 | 可能造成的問題 | 修正方式 |
|---|---|---|
| 用 `<a href="#">` 觸發一般動作 | 可能造成頁面跳動或語意不清 | 使用 `<button type="button">` |
| 使用 `javascript:` 當互動入口 | HTML 與行為混在一起，不易維護 | 使用按鈕加事件監聽 |
| 只看外觀選元素 | 外觀看起來一樣，語意仍然不同 | 根據用途選 `<a>` 或 `<button>` |
