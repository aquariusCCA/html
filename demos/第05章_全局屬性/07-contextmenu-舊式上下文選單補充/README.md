# contextmenu 舊式上下文選單補充 HTML 教學範例

來源筆記：`notes/第05章_全局屬性/07-contextmenu-舊式上下文選單補充.md`

## 建議學習順序

| 順序 | Demo | 對應觀念 | 觀察重點 | 建議修改任務 |
| --- | --- | --- | --- | --- |
| 1 | `01-辨認舊式contextmenu屬性.html` | `contextmenu` 屬性是舊式寫法 | `contextmenu`、`menu type="context"` 與 `menuitem` 如何出現在舊資料中，以及為什麼新頁面不建議使用 | 刪除錯誤範例中的舊式屬性與元素，保留普通段落 |
| 2 | `02-用contextmenu事件處理單一元素.html` | 現代寫法使用 JavaScript 的 `contextmenu` 事件 | 只對指定元素監聽右鍵事件，並用 `event.preventDefault()` 改成自訂處理 | 把事件監聽目標改到另一個區塊，觀察被攔截的範圍如何改變 |
| 3 | `03-不要攔截整頁右鍵.html` | 不要只為了禁止右鍵而攔截整頁 | 錯誤寫法只展示不執行；實作範例只攔截真正需要自訂處理的元素 | 移除綠色區塊中的 `event.preventDefault()`，觀察瀏覽器預設右鍵選單是否恢復 |
| 4 | `04-保留替代操作路徑.html` | 自訂右鍵行為要保留其他操作方式 | 右鍵自訂處理與標準按鈕都能完成同一件事，避免使用者只能依賴右鍵 | 修改右鍵處理中的訊息文字，確認按鈕替代路徑仍然可用 |

## 學習提醒

這組 demo 的目標不是取代正式筆記，而是幫助你把筆記中的觀念轉成可觀察、可修改、可練習的 HTML 範例。
