# 語言文字方向與翻譯屬性 HTML 教學範例

來源筆記：`notes/第05章_全局屬性/02-語言文字方向與翻譯屬性.md`

## 建議學習順序

| 順序 | Demo | 對應觀念 | 觀察重點 | 建議修改任務 |
| --- | --- | --- | --- | --- |
| 1 | `01-lang整份文件與局部語言.html` | `lang` 標示內容語言 | `lang` 可以放在 `<html>` 標示整份文件，也可以放在局部元素標示外語片段 | 把其中一個 `lang="en"` 改成 `lang="en-US"`，觀察語言標示變得更精確 |
| 2 | `02-dir文字方向.html` | `dir` 控制文字方向 | 比較 `ltr`、`rtl` 與 `auto` 對文字排列方向的影響 | 把阿拉伯文段落的 `dir="rtl"` 改成 `dir="ltr"`，比較是否符合內容語言 |
| 3 | `03-translate局部禁止翻譯.html` | `translate="no"` 控制局部內容是否應被翻譯 | 作品名稱、品牌名稱與程式碼片段可以只在局部標示不翻譯 | 新增一個不應被翻譯的作品名稱，並只在該名稱上加入 `translate="no"` |
| 4 | `04-lang-dir-translate一起使用.html` | `lang`、`dir`、`translate` 搭配使用 | 同一頁可同時標示文件主要語言、局部英文書名、禁止翻譯與阿拉伯文方向 | 把英文書名換成另一個作品名稱，保留 `lang="en"` 與 `translate="no"` |
| 5 | `05-常見錯誤對照.html` | 避免漏寫語言、用 CSS 取代方向語意、過度禁止翻譯 | 比較錯誤寫法、不建議寫法與對應修正方式 | 說明三組修正分別解決「語言」、「文字方向」或「翻譯範圍」哪一種問題 |

## 學習提醒

這組 demo 的目標不是取代正式筆記，而是幫助你把筆記中的觀念轉成可觀察、可修改、可練習的 HTML 範例。
