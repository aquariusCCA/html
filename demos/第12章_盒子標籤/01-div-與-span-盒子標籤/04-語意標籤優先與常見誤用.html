語意標籤優先與常見誤用

判斷原則

先問內容有沒有更明確的 HTML 語意標籤；如果只是需要一個容器來分組、排版或套樣式，再使用 <div> 或 <span>。

錯誤與修正對照

文字段落

錯誤寫法：只為了換行而使用 div 
<div>這是一段文字。</div>
<div>這是另一段文字。</div>
正確寫法：內容是段落時使用 p 
<p>這是一段文字。</p>
<p>這是另一段文字。</p>

一整塊結構

錯誤寫法：用 span 包住標題和段落 
<span class="card">
  <h2>商品名稱</h2>
  <p>商品介紹文字</p>
</span>
正確寫法：一般區塊分組可使用 div 
<div class="card">
  <h2>商品名稱</h2>
  <p>商品介紹文字</p>
</div>

主要導覽連結

錯誤寫法：有導覽語意時仍只用 div 
<div class="nav">
  <a href="/">首頁</a>
  <a href="/about">關於我們</a>
</div>
正確寫法：主要導覽區可使用 nav 
<nav>
  <a href="/">首頁</a>
  <a href="/about">關於我們</a>
</nav>

修改練習

挑選一組錯誤寫法，先判斷內容要表達的意義，再把它改成右側的正確標籤。
