# 第十四章｜換行標籤

> 章節定位：建立 HTML 強制換行與內容分段差異的基本判讀框架  
> 建議閱讀方式：先分清 `<br>` 是在同一段內容內換行，不是建立新段落，再回頭看它的使用場景

## 本章在學什麼

這一章整理 HTML 的換行標籤 `<br>`。  
重點不是只記住它會讓文字跳到下一行，而是理解它和段落結構的差別，以及什麼時候應該用、什麼時候不該濫用。

## 本章與整體地圖的關係

- 前置知識：第十一章已建立段落標籤 `<p>` 的基本概念。
- 這章的核心任務：分清強制換行和新段落的差別，避免把內容分段和排版換行混在一起。
- 讀完後通常接：之後在整理文章、地址、歌詞或卡片型文字內容時，可以回來快速查 `<br>` 的使用時機。

- 強制換行和新段落怎麼分

- 為什麼不建議用很多個 `<br>` 來排版

## 易混淆主題

- 強制換行 vs 新段落：前者是同段內容內的斷行，後者是新的內容單位。
- 內容換行 vs 版面留白：前者和語意結構有關，後者通常應交給 CSS 處理。
