# 換行標籤

> 所屬章節：第十四章｜換行標籤
> 關鍵字：換行標籤、`<br>`、`<br />`、line break、強制換行、HTML 換行、段落標籤、文字分段
> 建議回查情境：想知道 `<br>` 是做什麼的、想分清強制換行和新段落的差別、想確認什麼情況適合手動換行、想知道為什麼不建議用很多個 `<br>` 排版

## 本節導讀

這篇整理 HTML 的換行標籤 `<br>`。

它的核心作用，是在**同一段內容中明確插入一個換行點**，讓後面的文字從下一行開始顯示。

初學時，很多人會把「換行」和「分段」混在一起。
但更準確的理解是：

* `<br>` 解決的是**同一段內容中的強制換行**
* `<p>` 解決的是**是否進入新的段落**

也就是說，`<br>` 改變的是**行的位置**，不是內容的段落層級。

## 30 秒複習入口

* `<br>` 用來在**同一段內容中強制換行**。
* 它適合處理地址、詩句、歌詞、聯絡資訊等需要明確斷行的內容。
* `<br>` 不是拿來建立新段落；如果內容已經換成新的意思，通常應改用新的 `<p>`。
* `<br>` 是空元素，本身不包內容，也不需要結束標籤。
* 如果只是想拉開間距，應優先用 CSS，而不是堆很多個 `<br>`。

## 速查區

### 核心概念

* `<br>` 代表 line break，也就是在目前文字流中插入一個換行點。
* 它不會建立新的段落，只是讓後面的內容從下一行開始。

### 關鍵規則 / 判準

* 常見寫法是 `<br>`；有些教材也會寫成 `<br />`。
* `<br>` 是空元素，本身不包內容，也不需要結束標籤。
* 如果只是想在同一段資訊內斷行，可以用 `<br>`。
* 如果內容已經變成另一個完整段落，應該用新的 `<p>`，而不是連續堆很多個 `<br>`。

### 常見使用場景

* 地址或聯絡資訊分行顯示
* 詩句、歌詞、固定格式文字
* 卡片中的多行短資訊
* 同一段說明中需要明確斷成兩行的內容

### 常見混淆點

* 強制換行不等於新段落
* `<br>` 是內容中的換行點，不是版面留白工具
* `<br>` 不適合拿來控制區塊間距
* 如果只是想拉開上下距離，通常應優先用 CSS，而不是濫用 `<br>`

### 一句話對比

* `<br>` 解決的是「同一段內容要不要換到下一行」
* `<p>` 解決的是「這裡是不是另一個新段落」

## 正文筆記

## 1. 這篇在解決什麼問題？

在 HTML 中，文字本來就會依照可用寬度**自動換行**。
但有些內容不是要讓瀏覽器自己決定，而是你要明確指定：

**這裡就要換到下一行。**

這時就會用到 `<br>`。

也就是說，`<br>` 不是處理「有沒有新段落」，而是處理：

**在同一段內容裡，這裡是否需要強制換行。**

---

## 2. `<br>` 的作用

> 標籤語意：強制換行。

這句話方向是對的，但可以再講得更完整：

`<br>` 的作用，是讓內容從下一行開始顯示。
它處理的是**同一段內容中的換行需求**，不是切出新的內容段落。

所以你可以把它理解成：

> 在目前文字流中插入一個明確的換行點。

重點是：

* 它改變的是行的位置
* 它不改變內容的段落結構

---

## 3. `<br>` 的名稱與寫法

* `br` 是 `break` 的縮寫，可以理解成 line break。
* 在 HTML 中，最常見的寫法是 `<br>`。
* 有些教材、舊範例，或受 XHTML 風格影響的寫法，會寫成 `<br />`。

在一般 HTML 學習情境中，你可以把它們理解為同一個換行標籤即可。

### 補充理解

`<br>` 是**空元素**，意思是：

* 它本身不包內容
* 它不需要像 `<p></p>` 那樣有成對的開始和結束標籤

所以通常看到它時，就是單獨出現：

```html
<br>
```

---

## 4. 什麼時候適合用 `<br>`

當你想保留**同一段資訊**，但希望它在視覺上固定分成兩行或多行時，就很適合使用 `<br>`。

常見情境包括：

* 地址
* 聯絡資訊
* 詩句
* 歌詞
* 卡片中的多行短文字
* 同一組資訊中的固定斷行

### 典型例子：聯絡資訊

```html
台北市中正區忠孝西路一段 100 號<br>
客服電話：02-1234-5678<br>
服務時間：09:00 - 18:00
```

這個例子裡，內容其實還是同一組聯絡資訊，
只是需要明確分行顯示，這時用 `<br>` 就很合理。

### 典型例子：詩句

```html
床前明月光<br>
疑是地上霜<br>
舉頭望明月<br>
低頭思故鄉
```

這種情況下，每一行都屬於同一段詩句內容，
但行與行之間需要固定換行，因此也很適合用 `<br>`。

---

## 5. 什麼情況不適合用 `<br>`

如果內容已經換成另一個完整意思，
那通常就不應該只靠 `<br>` 硬切行。

這時比較合理的做法，是改用新的 `<p>`，或其他更合適的結構標籤。

### 不適合的例子

```html
<p>今天我們先介紹 HTML 的基本概念。<br>接下來我們要進入 CSS 的盒模型。</p>
```

這樣雖然畫面上換行了，但語意上其實像是兩段不同內容。
如果它們已經是不同段落，就更適合寫成：

```html
<p>今天我們先介紹 HTML 的基本概念。</p>
<p>接下來我們要進入 CSS 的盒模型。</p>
```

### 最簡單的判準

你可以直接這樣記：

* **只是同一段內容要斷成多行：用 `<br>`**
* **內容已經換成新的完整意思：用新的 `<p>`**

---

## 6. 基本示例

```html
第一行文本<br>第二行文本
```

### 這個例子在說什麼？

* 這兩行內容仍然屬於同一段文字
* 中間插入了一個明確的換行點
* 瀏覽器讀到 `<br>` 後，後面的內容就會從下一行開始顯示

也就是說，它表達的是：

**同一段內容，這裡要換行。**

不是：

**這裡開始一個新的段落。**

---

## 7. `<br>` 和瀏覽器自動換行有什麼差別？

這也是初學者常混淆的地方。

### 瀏覽器自動換行

瀏覽器會依照容器寬度，自然決定什麼時候換行。
這種換行是版面排版的結果，不一定固定。

### `<br>` 指定換行

`<br>` 是你明確告訴瀏覽器：

**這裡一定要換行。**

所以兩者的差別是：

* 自動換行：瀏覽器自己依寬度安排
* `<br>`：開發者手動指定換行點

---

## 8. 使用 `<br>` 時的注意點

### 不要把 `<br>` 當成萬用排版工具

`<br>` 的用途是換行，不是排版留白。
如果你只是想讓畫面上下看起來有空隙，通常不應該靠 `<br>` 來撐。

例如這種寫法就不理想：

```html
<p>第一段內容</p>
<br><br><br>
<p>第二段內容</p>
```

因為這樣把**內容結構**和**視覺間距**混在一起了。

---

### 想控制間距時，優先考慮 CSS

如果你的目的是：

* 拉開段落距離
* 增加上下空白
* 控制區塊間隔

那通常應該用 CSS，例如：

* `margin`
* `padding`
* `line-height`

而不是連續寫很多個 `<br>`。

這樣做的好處是：

* 結構更清楚
* 樣式更集中
* 維護更方便
* 之後調整版面時更容易改

---

### 不要用很多個 `<br>` 模擬文章結構

如果一整篇文章需要分很多段，
應該用 `<p>` 來表達段落，而不是用一堆 `<br>` 假裝段落分隔。

因為：

* `<br>` 是行級斷行
* `<p>` 才是段落層級的結構

這兩者處理的層級不同，不能混用成同一件事。

---

## 常見問法

### `<br>` 和瀏覽器自動換行有什麼差別？

* 自動換行是瀏覽器依寬度自然排版的結果
* `<br>` 則是你明確指定「這裡一定要換行」

---

### `<br>` 和 `<p>` 可以互相取代嗎？

不建議。

因為兩者處理的層級不同：

* `<br>` 是換行
* `<p>` 是段落

如果內容只是同一段中的斷行，用 `<br>`。
如果內容已經是新的段落，用 `<p>`。

---

### 為什麼不建議用很多個 `<br>` 來排版？

因為這樣會把內容結構和視覺效果混在一起。
當畫面樣式需要調整時，這種寫法通常比較難維護，也不夠乾淨。

---

### `<br>` 和 CSS 的差別是什麼？

* `<br>` 處理的是內容中的換行點
* CSS 處理的是版面樣式與間距控制

所以不要拿 `<br>` 去做本來應該交給 CSS 的事。
