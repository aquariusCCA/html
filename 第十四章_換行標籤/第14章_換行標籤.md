# 換行標籤

> 所屬章節：第十四章｜換行標籤  
> 關鍵字：換行標籤、`<br>`、`<br />`、line break、強制換行、HTML 換行、段落標籤、文字分段  
> 建議回查情境：想知道 `<br>` 是做什麼的、想分清強制換行和新段落的差別、想確認什麼情況適合手動換行

## 本節導讀

這篇整理 HTML 的換行標籤 `<br>`。  
它的核心作用，是在同一段內容裡明確插入一個換行點，讓文字從下一行繼續顯示。

初學時，很多人會把「換行」和「分段」混在一起。  
但更準確的理解是：`<br>` 解決的是同一段內容裡的強制換行；如果內容已經變成新的段落，通常應該改用新的 `<p>`。

## 你會在這篇學到什麼

- `<br>` 的基本用途
- `<br>` 的常見寫法與特點
- 什麼情況適合用換行標籤
- `<br>` 和 `<p>` 在語意上的差別

## 30 秒複習入口

- `<br>` 用來在同一段內容中強制換行。
- 它適合處理地址、詩句、歌詞、聯絡資訊這類明確需要斷行的內容。
- `<br>` 不是拿來建立新段落；如果內容已經換成新的意思，應改用新的 `<p>`。
- `break` 是換行標籤名稱的來源，常見寫法有 `<br>` 和 `<br />`。

## 速查區

### 核心概念

- `<br>` 代表 line break，也就是在目前文字流中插入一個換行。
- 它不會產生新的段落，只是讓後面的內容從下一行開始。

### 關鍵規則 / 判準

- 常見寫法是 `<br>`；不少教材也會寫成 `<br />`。
- `<br>` 是空元素，本身不包內容，也不需要結束標籤。
- 如果只是想在同一段資訊內斷行，可以用 `<br>`。
- 如果內容已經是另一個完整段落，應該用新的 `<p>`，而不是連續堆很多個 `<br>`。

### 常見使用場景

- 地址或通訊資訊分行顯示
- 詩句、歌詞或固定格式文字
- 同一段說明中需要明確斷成兩行的內容

### 常見混淆點

- 強制換行不等於新段落。
- `<br>` 是內容中的換行點，不是版面留白工具。
- 如果只是想拉開上下距離，通常應優先用段落結構或 CSS，而不是濫用 `<br>`。

### 一句話對比

- `<br>` 解決的是「同一段內容要不要換到下一行」；`<p>` 解決的是「這裡是不是另一個新段落」。

## 正文筆記

### 這篇在解決什麼問題？

- 在 HTML 中，文字會依可用寬度自動換行。
- 但有些內容需要你明確指定「這裡就要換到下一行」，這時就會用到 `<br>`。

## 1. `<br>` 的作用

> 標籤語意：強制換行。

- 這句話的核心是對的：`<br>` 的用途就是讓內容從下一行開始顯示。
- 它處理的是同一段內容裡的換行需求，不是切出新的內容段落。

## 2. `<br>` 的名稱與寫法

- `br` 是 `break` 的縮寫，可以理解成 line break。
- 在 HTML 中，常見寫法是 `<br>`。
- 有些教材或舊例子會寫成 `<br />`，閱讀時也很常見；在這份筆記裡把它們視為同一個換行標籤即可。

## 3. 什麼時候適合用 `<br>`

- 當你想保留同一段資訊，但希望它在視覺上固定分成兩行或多行時。
- 例如地址、詩句、歌詞、聯絡資訊，或卡片上的多行短文字。

### 什麼情況不適合？

- 如果內容已經換成另一個完整意思，就不該只靠 `<br>` 硬切行。
- 這時比較合理的做法，是改用新的 `<p>` 或其他更合適的結構標籤。

## 4. 基本示例

```html
第一行文本<br>第二行文本
```

### 這個例子在說什麼？

- 這兩行內容仍然屬於同一段文字，只是中間插入了明確的換行點。
- 瀏覽器讀到 `<br>` 後，後面的文字就會從下一行開始顯示。

## 5. 使用 `<br>` 時的注意點

- 不要把 `<br>` 當成萬用排版工具。
- 如果只是想讓上下看起來有空隙，應優先考慮段落結構或 CSS。
- 如果一整篇文章需要分很多段，應該用 `<p>` 來表達段落，而不是用一堆 `<br>` 模擬文章結構。

## 常見問法

### `<br>` 和瀏覽器自動換行有什麼差別？

- 自動換行是瀏覽器依寬度自然排版的結果。
- `<br>` 則是你明確指定「這裡一定要換行」。

### `<br>` 和 `<p>` 可以互相取代嗎？

- 不建議。
- `<br>` 是換行，`<p>` 是段落，兩者處理的層級不同。

### 為什麼不建議用很多個 `<br>` 來排版？

- 因為這樣會把內容結構和視覺效果混在一起。
- 當畫面樣式需要調整時，這種寫法也比較難維護。
