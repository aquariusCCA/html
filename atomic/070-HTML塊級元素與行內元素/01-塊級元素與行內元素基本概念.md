# 塊級元素與行內元素基本概念

<aside>
👉

**塊級元素在預設排版中通常會產生區塊盒，常見行為如下：**

- 多數常見塊級元素會在正常文件流中另起一行顯示；
- 可以設定 width、height、margin、padding 等盒模型相關屬性；
- 若 width 未設定，常見 block non-replaced 元素的 `width: auto` 會填滿可用水平空間，但不完全等同於 `width: 100%`；
- 多個塊級元素在正常文件流中預設由上往下排列。
- 區塊元素常見包括 div、p、h1~h6、ul、ol、li、dl、dt、dd、form、table、hr、blockquote、address、menu、pre.....等等
- 元素能包含哪些內容要看個別 HTML content model；`div` 這類通用容器可包含多數 flow content，但 `p`、`h1`~`h6` 等元素仍有自己的內容限制。
</aside>

<aside>
👉

**行內元素在預設排版中通常與文字一起在同一行流動。一般 non-replaced inline 元素（如 `span`、`em`）不能用 `width`、`height` 改變盒尺寸；但 `img`、部分表單控制項或設為 `inline-block` 的元素可以有可控尺寸。行內元素有以下常見特點：**

- 不会独占一行，相邻的行内元素会排列在同一行里，直到一行排不下才会自动换行，其宽度随元素的内容而变化；
- 一般 non-replaced inline 元素的寬度由內容決定；若需要設定尺寸，可改用 block、inline-block，或針對 replaced element、表單控制項設定；
- 左右 margin、padding 會影響水平排版；上下 padding、border 會被繪製，但通常不撐高 line box；上下 margin 對行盒高度沒有一般 block 元素那樣的效果；
- `line-height` 會影響行內盒參與行盒高度計算，不等同於設定父元素行高；
- 多數 phrasing/行內類元素只能包含 phrasing content；不能簡化成所有行內元素都能或都不能放某一類元素。`a` 有 transparent content model，但不能包含互動內容、另一個 `a` 或帶 `tabindex` 的後代。
- 行內元素常見包括 span、em、i、b、strong、a、img、input、br、select、textarea、q、bdo、sub、sup...等等
</aside>
