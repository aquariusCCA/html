# 常見圖片格式

## jpg 格式

- 概述 : 擴展名為 `.jpg` 或 `.jpeg`，是一種有損的壓縮格式 ( 把肉眼不容易觀察出來的細節丟棄了 )。
- 主要特點 : **支持豐富顏色、佔用空間較小**，不支持透明背景、不支持動態圖。
- 使用場景 : 對圖片細節 **沒有極高要求** 的場景，例如 : 網站的產品宣傳圖等。

## png 格式

- 概述 : 擴展名為 `.png`，是一種無損的壓縮格式，能夠更高質量的保存圖片。
- 主要特點 : 支持的顏色豐富，佔用空間略大、支持透明背景、不支持動態圖。
- 使用場景 : 公司 logo 圖、重要配圖等。
    - 想讓圖片有透明背景。
    - 想更高質量的呈現圖片。

## bmp 格式

- 概述 : 擴展名為 `.bmp`，常見 BMP 多為未壓縮或壓縮效率不佳，通常檔案很大，網頁中極少使用。
- 主要特點 : 支持的顏色豐富、保留的細節更多，佔用空間極大，不支持背景透明，不支持動態圖。
- 使用場景 : 對圖片細節要求極高的場景，例如 : 一些大型遊戲中的圖片 ( 網頁中極少使用 )。

## gif 格式

- 概述 : 擴展名為 `.gif`，僅支持 256 種顏色，色彩呈現不是很完整。
- 主要特點 : 支持的顏色較少、支持簡單透明背景、支持動態圖。
- 使用場景 : 網頁中的動態圖片。

## webp 格式

- 概述 : 擴展名為 `.webp`，Google 推出的一種現代 Web 圖片格式，專門用來在網頁中呈現圖片。
- 主要特點 : 支持有損壓縮、無損壓縮、透明背景與動態圖；主流現代瀏覽器已支持，但若需要兼容舊瀏覽器，仍應提供 fallback。
- 使用場景 : 網頁中的各種圖片。

## base64 編碼

- 本質 : Base64 不是圖片格式，而是把圖片二進位資料轉成文字的編碼方式。
- 原理 : 把圖片進行 base64 編碼，形成一串文本。
- 如何生成 : 靠一些工具或網站。
- 如何使用 : 通常以 Data URL 形式作為 img 標籤的 `src` 屬性的值，例如 `data:image/png;base64,...`，並且不受到文件位置的影響。
- 使用場景 : 一些較小的圖片，或者需要和網頁一起加載的圖片。
