# 文本格式化標籤

> 來源：origin/第17章_文本格式化標籤/01-文本格式化標籤.md / 全文

> 這些標籤會影響文字的預設呈現，但不應只為了外觀而選用。若只是要加粗、底線、傾斜或刪除線，通常使用 CSS；若文字本身具有語意，再使用對應的 HTML 標籤。

- 常見語意：
  - `<strong>`：重要、嚴重或需要優先注意的內容，預設通常加粗。
  - `<em>`：語氣上的強調，預設通常傾斜。
  - `<ins>`：文件中新增的內容，預設通常有底線。
  - `<del>`：文件中移除的內容，預設通常有刪除線。
  - `<b>`、`<i>`、`<u>`、`<s>` 也有各自語意，不是單純取代 CSS 的樣式工具。

```html
<b>關鍵字</b>
<strong>警告：請先存檔</strong>

<u>拼字錯誤標記</u>
<ins>新增內容</ins>

<i>技術術語</i>
<em>真的</em>

<s>原價 100 元</s>
<del>已刪除的內容</del>
```

<aside>
👉

**不要只因視覺效果選標籤；`strong`、`em`、`ins`、`del` 應在語意分別符合重要、強調、新增、刪除時使用。**
