# 響應式圖片

> - 所屬章節：第三十五章｜響應式圖片  
> - 關鍵字：響應式圖片、responsive images、`img`、`srcset`、`sizes`、`picture`、`source`、DPR、WebP、AVIF、art direction  
> - 建議回查情境：想讓不同螢幕載入不同尺寸圖片、想支援 WebP / AVIF 備援、想分清 `srcset`、`sizes`、`picture` 時

## 本節導讀

第十八章已經學過 `<img>` 的基本寫法。  
這一章處理的是更實務的問題：

> 同一張圖片在手機、平板、桌機、高解析度螢幕上，應該載入同一個檔案嗎？

很多時候答案是否定的。  
響應式圖片就是讓瀏覽器根據螢幕條件、顯示尺寸、像素密度和格式支援，選擇更合適的圖片資源。

## 整理說明

- 原始學習重點：理解 `<img>` 的 `src` 與 `alt`。
- 本次整理方向：補上 `srcset`、`sizes`、`picture`、`source` 的使用判斷。
- 補強內容：加入像素密度、寬度描述符、格式備援和藝術方向切換。
- 待查證：實際選圖結果會受瀏覽器、網路狀態、快取與 CSS 影響，應以瀏覽器開發者工具驗證。

## 你會在這篇學到什麼

- 為什麼需要響應式圖片
- `srcset` 的 `x` 描述符和 `w` 描述符怎麼分
- `sizes` 為什麼要搭配 `w` 描述符
- `<picture>` 什麼時候比 `<img srcset>` 更適合
- `<source>` 在 `<picture>` 中的作用
- 為什麼 `<picture>` 裡一定要保留 `<img>`

## 30 秒複習入口

- `srcset` 提供多個候選圖片。
- `1x` / `2x` 適合處理不同像素密度的同尺寸圖片。
- `480w` / `800w` 這種寬度描述符通常要搭配 `sizes`。
- `sizes` 告訴瀏覽器圖片在不同 viewport 下大約顯示多寬。
- `<picture>` 適合做圖片格式備援或藝術方向切換。
- `<source>` 提供條件候選，最後仍由 `<img>` 負責顯示與替代文字。
- `alt` 仍然不能省略判斷。

## 速查區

### 先選哪一種寫法？

| 需求 | 建議 |
|---|---|
| 同一張圖，只是解析度不同 | `<img srcset="..." alt="...">` |
| 同一張圖，不同寬度版本 | `<img srcset="..." sizes="..." alt="...">` |
| 不同格式備援，例如 AVIF / WebP / JPG | `<picture>` |
| 不同螢幕要裁切成不同構圖 | `<picture>` |
| 只有一張普通圖片 | 一般 `<img>` 就夠 |

### 描述符對照

| 描述符 | 例子 | 重點 |
|---|---|---|
| 像素密度 | `logo.png 1x, logo@2x.png 2x` | 適合同尺寸高解析圖片 |
| 圖片寬度 | `photo-480.jpg 480w, photo-960.jpg 960w` | 搭配 `sizes` 讓瀏覽器按顯示寬度選圖 |

## 正文筆記

### 1. 為什麼需要響應式圖片？

如果手機只顯示一張寬 360px 的圖片，卻下載了 2400px 的大圖，就會浪費流量和載入時間。

反過來，如果高解析度螢幕只拿到低解析圖片，畫面又可能模糊。

響應式圖片的目標是：

- 小螢幕不要下載過大的圖片。
- 高解析度螢幕可以拿到足夠清晰的圖片。
- 支援新格式的瀏覽器可以使用更高效的格式。
- 不同版面需要不同構圖時，可以切換圖片。

### 2. 最基本的 `srcset`：處理像素密度

```html
<img
  src="./images/logo.png"
  srcset="./images/logo.png 1x, ./images/logo@2x.png 2x"
  alt="網站 Logo">
```

這裡提供兩個候選：

- `1x`：一般解析度螢幕使用。
- `2x`：高解析度螢幕可以使用。

如果圖片顯示尺寸固定，例如 Logo、圖示，這種寫法很好理解。

### 3. 寬度描述符：讓瀏覽器知道圖片本身多寬

```html
<img
  src="./images/banner-800.jpg"
  srcset="
    ./images/banner-480.jpg 480w,
    ./images/banner-800.jpg 800w,
    ./images/banner-1200.jpg 1200w
  "
  sizes="(max-width: 600px) 100vw, 800px"
  alt="前端課程封面">
```

這裡的 `480w`、`800w`、`1200w` 表示圖片檔案本身的寬度。  
瀏覽器會根據 `sizes`、螢幕條件與像素密度選擇合適的圖片。

### 4. `sizes` 在做什麼？

`sizes` 不是 CSS。  
它是告訴瀏覽器：這張圖片在不同條件下大概會顯示多寬。

```html
sizes="(max-width: 600px) 100vw, 800px"
```

可以理解成：

- 當 viewport 寬度小於等於 600px，圖片大約佔滿整個 viewport。
- 其他情況下，圖片大約顯示 800px 寬。

瀏覽器會在下載圖片前先根據這個資訊選圖。

如果使用 `w` 描述符卻沒有正確寫 `sizes`，瀏覽器可能選到不理想的圖片。

### 5. `<picture>` 在做什麼？

`<picture>` 提供多個 `<source>` 條件，最後搭配一個 `<img>` 顯示圖片。

格式備援範例：

```html
<picture>
  <source srcset="./images/photo.avif" type="image/avif">
  <source srcset="./images/photo.webp" type="image/webp">
  <img src="./images/photo.jpg" alt="山上的日出">
</picture>
```

瀏覽器會從上往下看：

- 如果支援 AVIF，就使用 AVIF。
- 不支援 AVIF 但支援 WebP，就使用 WebP。
- 都不支援，就使用 `<img src>` 的 JPG。

### 6. 藝術方向切換是什麼？

有時候不同螢幕不是只需要不同大小，而是需要不同構圖。

例如桌機顯示橫幅大圖，手機顯示裁切過的人物近照：

```html
<picture>
  <source media="(max-width: 600px)" srcset="./images/portrait-mobile.jpg">
  <source media="(min-width: 601px)" srcset="./images/hero-wide.jpg">
  <img src="./images/hero-wide.jpg" alt="講師正在講解 HTML">
</picture>
```

這種情況適合 `<picture>`，因為不同條件下其實不是同一張圖的不同尺寸，而是不同構圖版本。

### 7. 為什麼 `<picture>` 裡還要有 `<img>`？

`<img>` 是必要的 fallback，也是實際承載圖片語意與替代文字的元素。

```html
<picture>
  <source srcset="./images/chart.webp" type="image/webp">
  <img src="./images/chart.png" alt="HTML 學習路線圖">
</picture>
```

如果沒有任何 `<source>` 符合，瀏覽器會使用 `<img src>`。  
`alt` 也寫在 `<img>` 上，而不是寫在 `<source>` 上。

### 8. 響應式圖片和 CSS 背景圖怎麼分？

如果圖片是內容的一部分，例如文章配圖、商品照片、圖表，通常應該用 `<img>` 或 `<picture>`。

如果圖片只是裝飾背景，才比較可能放在 CSS background。

判斷方式：

- 使用者需要理解這張圖傳達的資訊：用 HTML 圖片。
- 圖片只是裝飾，不影響理解：可考慮 CSS 背景。

## 常見問法

### `srcset` 可以取代 `src` 嗎？

不建議這樣理解。

`src` 仍然是基本圖片來源與 fallback。  
`srcset` 是補充候選資源。

### 什麼時候需要 `sizes`？

當 `srcset` 使用 `480w`、`800w` 這類寬度描述符時，通常需要 `sizes`。  
如果只使用 `1x`、`2x`，通常不需要 `sizes`。

### `<picture>` 是不是永遠比 `<img>` 好？

不是。

如果只是普通圖片，`<img>` 更簡單。  
只有在需要格式備援、不同構圖或複雜條件選圖時，才考慮 `<picture>`。

### `alt` 要寫在哪？

寫在 `<img>` 上。

`<source>` 只提供候選資源，不承擔替代文字。
