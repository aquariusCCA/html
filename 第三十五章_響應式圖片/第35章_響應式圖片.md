# 響應式圖片

> - 所屬章節：第三十五章｜響應式圖片  
> - 關鍵字：響應式圖片、responsive images、`img`、`srcset`、`sizes`、`picture`、`source`、DPR、WebP、AVIF、art direction  
> - 建議回查情境：想讓不同螢幕載入不同尺寸圖片、想支援 AVIF / WebP 備援、想分清 `srcset`、`sizes`、`picture` 時

## 本節導讀

第十八章已經學過 `<img>` 的基本寫法，例如：

```html
<img src="./images/photo.jpg" alt="山上的日出">
```

這一章處理的是更實務的問題：

> 同一張圖片在手機、平板、桌機、高解析度螢幕上，應該載入同一個檔案嗎？

很多時候答案是否定的。

如果手機只顯示一張寬 360px 的圖片，卻下載了 2400px 的大圖，就會浪費流量和載入時間。  
反過來，如果高解析度螢幕只拿到低解析圖片，畫面又可能模糊。

**響應式圖片的核心目標不是讓圖片會縮放，而是讓瀏覽器能根據條件選擇更合適的圖片資源。**

---

## 本章核心觀念

響應式圖片主要處理四件事：

1. **螢幕寬度不同**：手機、平板、桌機需要不同尺寸的圖片。
2. **像素密度不同**：一般螢幕與 Retina / 高 DPR 螢幕需要不同清晰度的圖片。
3. **圖片格式支援不同**：支援 AVIF / WebP 的瀏覽器可以載入更高效的格式，不支援時退回 JPG / PNG。
4. **版面構圖不同**：桌機可能適合橫幅圖，手機可能適合裁切後的人物近照。

要記住一個大方向：

| 問題 | 主要工具 |
|---|---|
| 同一張圖，只是解析度不同 | `<img srcset="...">` 搭配 `1x` / `2x` |
| 同一張圖，有不同寬度版本 | `<img srcset="..." sizes="...">` 搭配 `480w` / `800w` |
| 不同圖片格式備援 | `<picture>` + `<source type>` |
| 不同螢幕要不同構圖 | `<picture>` + `<source media>` |
| 圖片只是普通內容圖 | 一般 `<img>` 就夠 |

---

## 30 秒複習入口

- `srcset`：提供多個候選圖片，讓瀏覽器選。
- `1x` / `2x`：描述圖片適合的像素密度，常用於 Logo、Icon、固定顯示尺寸圖片。
- `480w` / `800w`：描述圖片檔案本身的寬度，常用於文章圖、商品圖、Banner。
- `sizes`：告訴瀏覽器圖片在不同 viewport 下大約會顯示多寬。
- `<picture>`：適合處理格式備援或藝術方向切換。
- `<source>`：提供條件候選，自己不負責顯示圖片，也不寫 `alt`。
- `<img>`：永遠是最終顯示圖片與承載 `alt` 的核心元素。
- `alt`：不是永遠都要寫很多文字，而是永遠要判斷圖片的語意用途。

## 使用判斷流程

遇到圖片時，可以先照這個流程判斷：

```text
這張圖片是內容的一部分嗎？
├─ 不是，只是裝飾背景
│  └─ 可以考慮 CSS background
│
└─ 是，使用者需要理解這張圖片
   └─ 使用 <img> 或 <picture>

接著判斷：

只是普通圖片嗎？
├─ 是
│  └─ 使用一般 <img>
│
└─ 不是
   ├─ 同一張圖，只是高解析螢幕要更清楚？
   │  └─ 使用 srcset + 1x / 2x
   │
   ├─ 同一張圖，但不同裝置需要不同寬度版本？
   │  └─ 使用 srcset + 480w / 800w / 1200w + sizes
   │
   ├─ 想讓支援 AVIF / WebP 的瀏覽器優先載入新格式？
   │  └─ 使用 picture + source type
   │
   └─ 不同螢幕要不同裁切或不同構圖？
      └─ 使用 picture + source media
```

## 先分清楚三個寬度

學響應式圖片時，最容易混淆的是「寬度」到底指什麼。至少要分成三種：

| 名稱 | 說明 | 例子 |
|---|---|---|
| viewport 寬度 | 瀏覽器可視區域寬度 | 手機 viewport 可能是 `390px` |
| 顯示寬度 | 圖片實際在頁面上顯示多寬 | 卡片內圖片顯示 `300px` |
| 圖片檔案寬度 | 圖片資源本身的像素寬度 | `photo-800.jpg` 實際寬 `800px` |

再加上一個重要概念：**DPR**。

DPR 是 device pixel ratio，可以簡單理解成：

> 1 個 CSS px 需要多少個實體像素來顯示。

例如：

| 顯示寬度 | DPR | 理想圖片資源大約需要 |
|---|---:|---:|
| `400px` | `1` | `400px` 寬左右 |
| `400px` | `2` | `800px` 寬左右 |
| `400px` | `3` | `1200px` 寬左右 |

所以響應式圖片不是單純看螢幕寬度，而是會綜合考慮：

- 圖片在版面中會顯示多寬
- 裝置像素密度
- 可用候選圖片
- 瀏覽器策略、網路狀態、快取狀態

## 1. 一般圖片：普通 `<img>` 就夠

如果只有一張普通圖片，不需要多尺寸、不需要格式備援、不需要不同構圖，那就不用把事情複雜化。

```html
<img src="./images/article-cover.jpg" alt="HTML 教學文章封面">
```

這種寫法適合：

- 圖片尺寸不大
- 不需要針對不同螢幕提供不同版本
- 不需要 AVIF / WebP / JPG 備援
- 不需要手機與桌機顯示不同構圖

如果只是為了讓圖片不要超出容器，通常是 CSS 負責：

```css
img {
  max-width: 100%;
  height: auto;
}
```

這段 CSS 的作用是：**讓圖片在視覺上可以隨容器縮放**。  
但它不會改變瀏覽器實際下載的圖片檔案大小。

也就是說：

```text
CSS 控制圖片顯示多大。
srcset / sizes / picture 控制瀏覽器可以選哪些圖片資源。
```

---

## 2. 使用 `srcset` + `1x` / `2x`：處理像素密度

如果圖片在頁面上的顯示尺寸大致固定，例如 Logo、Icon、Avatar，就很適合用 `1x` / `2x`。

```html
<img
  src="./images/logo.png"
  srcset="./images/logo.png 1x, ./images/logo@2x.png 2x"
  alt="網站 Logo">
```

可以理解成：

| 候選圖片 | 描述符 | 意義 |
|---|---|---|
| `logo.png` | `1x` | 一般螢幕使用 |
| `logo@2x.png` | `2x` | 高解析度螢幕可使用 |

這種寫法的重點是：

- 圖片的顯示尺寸通常是固定的。
- `1x` / `2x` 描述的是「適合哪種像素密度」。
- 通常不需要搭配 `sizes`。

例如 Logo 在 CSS 中固定顯示為：

```css
.logo {
  width: 120px;
  height: auto;
}
```

那麼：

- 一般螢幕可能用 `120px` 寬左右的圖片。
- DPR 2 的螢幕可能用 `240px` 寬左右的圖片。

### `src` 在這裡的角色

當 `srcset` 使用 `1x` / `2x` 這類像素密度描述符時，`src` 可以作為基本來源，也可以被視為 `1x` 候選的一部分。

所以這種寫法常見且直覺：

```html
<img
  src="./images/avatar.png"
  srcset="./images/avatar.png 1x, ./images/avatar@2x.png 2x"
  alt="使用者頭像">
```

## 3. 使用 `srcset` + `w`：描述圖片本身的寬度

如果同一張圖片準備了多個寬度版本，例如：

```text
banner-480.jpg
banner-800.jpg
banner-1200.jpg
```

這時候適合使用 `w` 描述符：

```html
<img
  src="./images/banner-800.jpg"
  srcset="
    ./images/banner-480.jpg 480w,
    ./images/banner-800.jpg 800w,
    ./images/banner-1200.jpg 1200w
  "
  sizes="(max-width: 600px) 100vw, 800px"
  alt="前端課程封面">
```

這裡的 `480w`、`800w`、`1200w` 指的是：

> 圖片檔案本身的實際寬度。

不是 CSS 寬度，也不是 viewport 寬度。

例如：

```html
./images/banner-480.jpg 480w
```

意思是：

> `banner-480.jpg` 這個檔案本身大約是 480px 寬。

---

## 4. `sizes` 在做什麼？

> **`sizes` 本身不是直接挑圖片，而是先告訴瀏覽器：「這張圖片最後大約會顯示多寬」。**
> 然後瀏覽器再用這個寬度，搭配螢幕 DPR，去 `srcset` 裡挑一張最合適的圖片。

---

### 核心公式

假設你寫：

```html
<img
  src="./images/banner-800.jpg"
  srcset="
    ./images/banner-480.jpg 480w,
    ./images/banner-800.jpg 800w,
    ./images/banner-1200.jpg 1200w
  "
  sizes="(max-width: 600px) 100vw, 800px"
  alt="前端課程封面">
```

瀏覽器大概會這樣判斷：

```text
需要的圖片寬度 ≈ 圖片實際顯示寬度 × 螢幕 DPR
```

也就是：

```text
neededWidth = slotSize × DPR
```

其中：

* `slotSize`：瀏覽器從 `sizes` 算出來的圖片顯示寬度
* `DPR`：裝置像素比，例如一般螢幕是 `1`，Retina 螢幕可能是 `2` 或 `3`
* `srcset` 裡的 `480w`、`800w`、`1200w`：圖片檔案本身的實際寬度

---

### 第一步：瀏覽器先看 `sizes`

```html
sizes="(max-width: 600px) 100vw, 800px"
```

意思是：

```text
如果 viewport <= 600px：
    這張圖片大約顯示 100vw，也就是整個 viewport 寬度

否則：
    這張圖片大約顯示 800px
```

假設現在手機 viewport 是 `390px`。

因為：

```text
390px <= 600px
```

所以符合第一個條件：

```html
(max-width: 600px) 100vw
```

因此瀏覽器得到：

```text
slotSize = 100vw = 390px
```

這代表瀏覽器先認為：

```text
這張圖片在畫面上大約會顯示 390px 寬
```

---

### 第二步：再看螢幕 DPR

假設這台手機是 Retina 螢幕，DPR 是 `2`。

那瀏覽器會估算：

```text
需要的圖片寬度 = 390px × 2 = 780px
```

也就是說：

```text
畫面上顯示 390px 寬，
但因為螢幕像素密度是 2 倍，
所以最好拿接近 780px 寬的圖片。
```

---

### 第三步：去 `srcset` 裡挑圖片

你的 `srcset` 有三個候選：

```text
banner-480.jpg   480w
banner-800.jpg   800w
banner-1200.jpg  1200w
```

剛剛算出需要大約：

```text
780px
```

所以瀏覽器通常會選：

```text
banner-800.jpg 800w
```

因為 `800w` 最接近，而且足夠支援 `780px` 的需求。

---

### 用表格看會更清楚

假設：

```html
srcset="
  banner-480.jpg 480w,
  banner-800.jpg 800w,
  banner-1200.jpg 1200w
"
sizes="(max-width: 600px) 100vw, 800px"
```

| viewport | DPR | `sizes` 算出的顯示寬度 | 需要圖片寬度 | 可能選擇          |
| -------: | --: | --------------: | -----: | ------------- |
|    390px |   1 |           390px |  390px | 480w          |
|    390px |   2 |           390px |  780px | 800w          |
|    390px |   3 |           390px | 1170px | 1200w         |
|   1000px |   1 |           800px |  800px | 800w          |
|   1000px |   2 |           800px | 1600px | 1200w，因為沒有更大的 |

重點是：

```text
sizes 決定「圖片顯示寬度」
DPR 決定「需要幾倍清晰度」
srcset 提供「有哪些圖片可以選」
```

---

### 所以 `sizes` 的角色是什麼？

你可以把 `sizes` 想成瀏覽器選圖前的提示：

```text
這張圖片在這個版面條件下，大概會顯示多寬。
```

瀏覽器不是只看 viewport 大小，也不是只看 `srcset`，而是會把三件事放在一起看：

```text
sizes 算出的顯示寬度
×
裝置 DPR
=
需要的圖片資源寬度
```

然後再從 `srcset` 裡挑一張合適的。

## 5. 為什麼 `w` 描述符通常要搭配 `sizes`？

當你使用：

```html
srcset="photo-480.jpg 480w, photo-800.jpg 800w, photo-1200.jpg 1200w"
```

你只是告訴瀏覽器：

> 我有 480px、800px、1200px 寬的圖片可以選。

但瀏覽器還需要知道：

> 這張圖片在目前版面中到底會顯示多寬？

這就是 `sizes` 的用途。

### 沒有寫好 `sizes` 會怎樣？

如果 `sizes` 寫得不符合實際 CSS 版面，瀏覽器可能會選錯圖片。

常見結果有兩種：

1. **下載太大的圖片**：畫面沒比較好，但流量和載入時間增加。
2. **下載太小的圖片**：圖片可能變模糊。

所以使用 `w` 描述符時，實務上應該明確寫出 `sizes`，並讓它盡量對齊 CSS 版面。

---

## 6. `sizes` 要怎麼對齊 CSS 版面？

假設 CSS 是這樣：

```css
.article-cover {
  width: 100%;
  max-width: 800px;
}
```

意思是：

- 小螢幕時圖片會跟著容器變寬，接近 `100vw`。
- 大螢幕時圖片最多顯示 `800px`。

那 `sizes` 就可以寫成：

```html
<img
  class="article-cover"
  src="./images/cover-800.jpg"
  srcset="
    ./images/cover-480.jpg 480w,
    ./images/cover-800.jpg 800w,
    ./images/cover-1200.jpg 1200w
  "
  sizes="(max-width: 800px) 100vw, 800px"
  alt="HTML 響應式圖片教學封面">
```

可以理解成：

```text
CSS 決定實際顯示規則。
sizes 用文字描述這個顯示規則，讓瀏覽器提前選圖。
```

### `sizes` 的讀取規則

`sizes` 會從左到右判斷，找到第一個符合的條件就使用。

例如：

```html
sizes="(max-width: 600px) 100vw, (max-width: 1000px) 80vw, 800px"
```

意思是：

| 條件 | 使用的顯示寬度 |
|---|---|
| viewport ≤ `600px` | `100vw` |
| viewport ≤ `1000px` | `80vw` |
| 其他情況 | `800px` |

最後面的 `800px` 是 fallback，不能省略，因為它代表「前面條件都不符合時使用的大小」。

## 7. `x` 描述符和 `w` 描述符不要混用

同一個 `srcset` 裡，不建議、也不應該混用 `x` 和 `w` 描述符。

錯誤示範：

```html
<img
  src="./images/photo.jpg"
  srcset="./images/photo-480.jpg 480w, ./images/photo@2x.jpg 2x"
  alt="示例圖片">
```

這樣會讓瀏覽器難以正確判斷候選圖片的意義。

應該根據需求選一種：

### 情境一：固定顯示尺寸，處理高解析螢幕

```html
<img
  src="./images/logo.png"
  srcset="./images/logo.png 1x, ./images/logo@2x.png 2x"
  alt="網站 Logo">
```

### 情境二：不同版面寬度，處理不同圖片尺寸

```html
<img
  src="./images/photo-800.jpg"
  srcset="
    ./images/photo-480.jpg 480w,
    ./images/photo-800.jpg 800w,
    ./images/photo-1200.jpg 1200w
  "
  sizes="(max-width: 600px) 100vw, 800px"
  alt="文章封面圖片">
```

## 8. `src` 可以被 `srcset` 取代嗎？

不要把它理解成「取代」。

`src` 仍然很重要：

- 它是 `<img>` 的基本圖片來源。
- 它可以作為舊瀏覽器或不支援情境下的 fallback。
- 當 `srcset` 使用 `x` 描述符時，`src` 常可視為 `1x` 候選。

但要注意：

> 當 `srcset` 使用 `w` 描述符時，瀏覽器主要會根據 `srcset` 與 `sizes` 選圖，不要期待 `src` 也自動參與寬度候選選擇。

所以如果你希望某張圖參與選圖，應該把它放進 `srcset`：

```html
<img
  src="./images/banner-800.jpg"
  srcset="
    ./images/banner-480.jpg 480w,
    ./images/banner-800.jpg 800w,
    ./images/banner-1200.jpg 1200w
  "
  sizes="(max-width: 600px) 100vw, 800px"
  alt="網站首頁橫幅">
```

## 9. `<picture>` 在做什麼？

`<picture>` 可以提供多組 `<source>` 條件，最後搭配一個 `<img>` 顯示圖片。

它適合用在兩大情境：

1. **格式備援**：AVIF → WebP → JPG / PNG。
2. **藝術方向切換**：手機和桌機使用不同構圖。

基本結構是：

```html
<picture>
  <source srcset="./images/photo.avif" type="image/avif">
  <source srcset="./images/photo.webp" type="image/webp">
  <img src="./images/photo.jpg" alt="山上的日出">
</picture>
```

瀏覽器會大致照這個流程判斷：

```text
從上往下檢查 <source>
├─ 符合 type / media 條件？
│  └─ 符合就使用該 source 的 srcset
│
└─ 都不符合
   └─ 使用 <img src> 作為 fallback
```

重點：

- `<source>` 只是提供候選資源。
- `<source>` 自己不顯示圖片。
- `<source>` 不寫 `alt`。
- `<img>` 仍然是必要元素。
- `alt` 寫在 `<img>` 上。

---

## 10. 使用 `<picture>` 做格式備援

如果你希望支援新格式的瀏覽器優先載入 AVIF 或 WebP，可以這樣寫：

```html
<picture>
  <source srcset="./images/photo.avif" type="image/avif">
  <source srcset="./images/photo.webp" type="image/webp">
  <img src="./images/photo.jpg" alt="山上的日出">
</picture>
```

可以理解成：

| 順序 | 條件 | 結果 |
|---|---|---|
| 1 | 支援 AVIF | 使用 `photo.avif` |
| 2 | 不支援 AVIF，但支援 WebP | 使用 `photo.webp` |
| 3 | 都不支援 | 使用 `<img src>` 的 `photo.jpg` |

通常會把壓縮效率較好的新格式放前面，例如：

```text
AVIF → WebP → JPG / PNG
```

## 11. 使用 `<picture>` 做藝術方向切換

有時候不同螢幕不是只需要不同大小，而是需要不同構圖。

例如：

- 桌機：顯示橫幅大圖。
- 手機：顯示裁切過的人物近照。

這時候適合用 `<picture>`：

```html
<picture>
  <source
    media="(max-width: 600px)"
    srcset="./images/hero-mobile.jpg">

  <source
    media="(min-width: 601px)"
    srcset="./images/hero-desktop.jpg">

  <img src="./images/hero-desktop.jpg" alt="講師正在講解 HTML">
</picture>
```

這不是單純的「同一張圖不同尺寸」，而是「不同螢幕使用不同構圖」。

所以這種情況比起 `<img srcset>`，更適合 `<picture>`。

---

## 12. `<picture>` 也可以搭配 `srcset` 和 `sizes`

`<picture>` 不只能放單一圖片，也可以在 `<source>` 中繼續提供多個寬度候選。

例如：

```html
<picture>
  <source
    type="image/avif"
    srcset="
      ./images/banner-480.avif 480w,
      ./images/banner-800.avif 800w,
      ./images/banner-1200.avif 1200w
    "
    sizes="(max-width: 600px) 100vw, 800px">

  <source
    type="image/webp"
    srcset="
      ./images/banner-480.webp 480w,
      ./images/banner-800.webp 800w,
      ./images/banner-1200.webp 1200w
    "
    sizes="(max-width: 600px) 100vw, 800px">

  <img
    src="./images/banner-800.jpg"
    srcset="
      ./images/banner-480.jpg 480w,
      ./images/banner-800.jpg 800w,
      ./images/banner-1200.jpg 1200w
    "
    sizes="(max-width: 600px) 100vw, 800px"
    alt="前端課程首頁橫幅">
</picture>
```

這個例子同時處理：

- 格式備援：AVIF / WebP / JPG
- 寬度候選：480w / 800w / 1200w
- 顯示寬度提示：`sizes`

但這種寫法比較完整，也比較長。  
只有在圖片真的很重要、很大、流量成本明顯時，才值得這樣寫。

---

## 13. 為什麼 `<picture>` 裡一定要有 `<img>`？

因為 `<img>` 是最終顯示圖片的元素，也是提供替代文字的元素。

```html
<picture>
  <source srcset="./images/chart.webp" type="image/webp">
  <img src="./images/chart.png" alt="HTML 學習路線圖">
</picture>
```

如果沒有任何 `<source>` 符合，瀏覽器會使用 `<img src>`。

而且 `alt` 是寫在 `<img>` 上，不是寫在 `<source>` 上。

錯誤示範：

```html
<picture>
  <source srcset="./images/chart.webp" type="image/webp" alt="HTML 學習路線圖">
  <img src="./images/chart.png">
</picture>
```

正確寫法：

```html
<picture>
  <source srcset="./images/chart.webp" type="image/webp">
  <img src="./images/chart.png" alt="HTML 學習路線圖">
</picture>
```

---

## 14. `alt` 要怎麼判斷？

`alt` 不是永遠都要寫很多文字，而是要根據圖片用途判斷。

| 圖片用途 | `alt` 建議 |
|---|---|
| 有資訊意義的圖片 | 描述圖片傳達的資訊 |
| 裝飾圖片 | 使用 `alt=""` |
| 圖片是連結 | 描述點擊後的目的 |
| 圖片是按鈕 | 描述按鈕功能 |
| 圖表、流程圖 | 簡短說明重點，必要時搭配正文補充 |

範例一：文章配圖

```html
<img src="./images/sunrise.jpg" alt="山頂上的日出">
```

範例二：裝飾圖片

```html
<img src="./images/decor-line.png" alt="">
```

範例三：圖片連結

```html
<a href="/courses/html">
  <img src="./images/html-course.jpg" alt="前往 HTML 課程頁面">
</a>
```

不要寫成：

```html
<img src="./images/photo.jpg" alt="圖片">
```

因為「圖片」這兩個字沒有提供有效資訊。

---

## 15. 響應式圖片和 CSS 背景圖怎麼分？

判斷方式很簡單：

| 問題 | 適合做法 |
|---|---|
| 使用者需要理解這張圖傳達的內容 | 使用 `<img>` 或 `<picture>` |
| 圖片只是裝飾，不影響理解 | 可以考慮 CSS `background-image` |
| 圖片需要 `alt` | 使用 `<img>` 或 `<picture>` |
| 圖片只是背景紋理、裝飾線條 | 可以使用 CSS 背景圖 |

範例：商品照片應該用 HTML 圖片。

```html
<img src="./images/product.jpg" alt="黑色機械鍵盤">
```

範例：卡片背景紋理可以用 CSS。

```css
.card {
  background-image: url("./images/pattern.svg");
}
```

---

## 16. 實務上建議加上 `width` 和 `height`

響應式圖片常常會搭配 CSS 縮放：

```css
img {
  max-width: 100%;
  height: auto;
}
```

但 HTML 上仍然建議保留圖片原始比例資訊：

```html
<img
  src="./images/cover-800.jpg"
  width="800"
  height="450"
  alt="HTML 教學封面">
```

這樣瀏覽器可以在圖片載入前先預留空間，減少版面跳動。

如果搭配 `srcset`：

```html
<img
  src="./images/cover-800.jpg"
  srcset="
    ./images/cover-480.jpg 480w,
    ./images/cover-800.jpg 800w,
    ./images/cover-1200.jpg 1200w
  "
  sizes="(max-width: 800px) 100vw, 800px"
  width="800"
  height="450"
  alt="HTML 教學封面">
```

這裡的 `width="800"` 和 `height="450"` 不是說圖片永遠只能顯示 800×450，而是提供圖片比例，讓瀏覽器可以更穩定地排版。

---

## 17. 如何確認瀏覽器實際載入哪張圖片？

響應式圖片的實際選圖結果可能受很多因素影響：

- viewport 寬度
- DPR
- `srcset`
- `sizes`
- CSS 實際版面
- 快取
- 網路狀態
- 瀏覽器策略

所以不要只用肉眼猜，要用 DevTools 檢查。

以 Chrome DevTools 為例：

1. 開啟 DevTools。
2. 切到 Network 面板。
3. 勾選 Disable cache。
4. 重新整理頁面。
5. 篩選 Img。
6. 觀察實際被下載的是 `480w`、`800w`、`1200w`，還是 AVIF / WebP / JPG。
7. 如果要測試手機版，可以開啟裝置模擬模式，調整 viewport 與 DPR。

也可以在 Elements 面板查看 `<img>` 實際顯示尺寸，確認 `sizes` 是否和 CSS 版面相符。

## 18. 常見錯誤整理

### 錯誤一：以為 `sizes` 是 CSS

錯誤理解：

```text
sizes 會控制圖片顯示大小。
```

正確理解：

```text
CSS 控制圖片顯示大小。
sizes 只是告訴瀏覽器圖片大約會顯示多寬，方便提前選圖。
```

---

### 錯誤二：把 `480w` 當成 CSS 寬度

錯誤理解：

```html
srcset="photo-480.jpg 480w"
```

代表圖片會顯示成 480px。

正確理解：

```text
480w 表示 photo-480.jpg 這個圖片檔案本身寬度是 480px。
```

---

### 錯誤三：同一個 `srcset` 混用 `x` 和 `w`

錯誤示範：

```html
<img
  src="photo.jpg"
  srcset="photo-480.jpg 480w, photo@2x.jpg 2x"
  alt="示例圖片">
```

正確做法：根據需求選一種描述方式。

---

### 錯誤四：使用 `w` 描述符卻亂寫 `sizes`

錯誤示範：

```html
<img
  src="cover-800.jpg"
  srcset="cover-480.jpg 480w, cover-800.jpg 800w, cover-1200.jpg 1200w"
  sizes="100vw"
  alt="文章封面">
```

如果圖片在桌機實際只顯示 600px，但 `sizes` 卻一直寫 `100vw`，瀏覽器可能以為圖片很大，進而下載過大的圖片。

---

### 錯誤五：把 `<picture>` 當成永遠比較高級的寫法

`<picture>` 不是永遠比 `<img>` 好。

如果只是普通圖片，這樣就夠：

```html
<img src="photo.jpg" alt="山上的日出">
```

只有在需要格式備援、藝術方向切換或複雜條件選圖時，才需要 `<picture>`。

---

### 錯誤六：忘記 `<picture>` 裡面的 `<img>`

錯誤示範：

```html
<picture>
  <source srcset="photo.webp" type="image/webp">
</picture>
```

正確寫法：

```html
<picture>
  <source srcset="photo.webp" type="image/webp">
  <img src="photo.jpg" alt="山上的日出">
</picture>
```

---

## 19. 實務選擇建議

### 情境一：Logo / Icon

建議：`srcset` + `1x` / `2x`

```html
<img
  src="./images/logo.png"
  srcset="./images/logo.png 1x, ./images/logo@2x.png 2x"
  alt="網站 Logo">
```

---

### 情境二：文章封面圖

建議：`srcset` + `w` + `sizes`

```html
<img
  src="./images/cover-800.jpg"
  srcset="
    ./images/cover-480.jpg 480w,
    ./images/cover-800.jpg 800w,
    ./images/cover-1200.jpg 1200w
  "
  sizes="(max-width: 800px) 100vw, 800px"
  alt="HTML 教學文章封面">
```

---

### 情境三：首頁 Hero Banner，需要 AVIF / WebP / JPG 備援

建議：`<picture>` + `<source type>`

```html
<picture>
  <source srcset="./images/hero.avif" type="image/avif">
  <source srcset="./images/hero.webp" type="image/webp">
  <img src="./images/hero.jpg" alt="前端課程首頁橫幅">
</picture>
```

---

### 情境四：手機與桌機要不同構圖

建議：`<picture>` + `<source media>`

```html
<picture>
  <source media="(max-width: 600px)" srcset="./images/course-mobile.jpg">
  <source media="(min-width: 601px)" srcset="./images/course-desktop.jpg">
  <img src="./images/course-desktop.jpg" alt="講師正在講解前端課程">
</picture>
```

## 20. 最小記憶模型

如果只想先記最重要的，可以記這幾句：

```text
1. 普通圖片用 <img>。
2. 固定尺寸但要支援 Retina，用 srcset + 1x / 2x。
3. 不同寬度圖片版本，用 srcset + 480w / 800w + sizes。
4. AVIF / WebP / JPG 備援，用 picture + source type。
5. 手機和桌機要不同構圖，用 picture + source media。
6. source 只是候選，真正顯示圖片與寫 alt 的還是 img。
7. CSS 控制圖片顯示大小，sizes 告訴瀏覽器選圖時該假設多寬。
```

## 常見問法

### `srcset` 可以取代 `src` 嗎？

不建議這樣理解。

`src` 仍然是基本圖片來源與 fallback。  
`srcset` 是補充候選資源。

---

### 什麼時候需要 `sizes`？

當 `srcset` 使用 `480w`、`800w` 這類寬度描述符時，就應該搭配 `sizes`。

如果只使用 `1x`、`2x`，通常不需要 `sizes`。

---

### `<picture>` 是不是永遠比 `<img>` 好？

不是。

如果只是普通圖片，`<img>` 更簡單。  
只有在需要格式備援、不同構圖或複雜條件選圖時，才考慮 `<picture>`。

---

### `alt` 要寫在哪？

寫在 `<img>` 上。

`<source>` 只提供候選資源，不承擔替代文字。

---

### 使用了 `srcset`，圖片就一定會變小嗎？

不一定。

`srcset` 只是提供候選圖片。  
瀏覽器實際選哪一張，還會受到 `sizes`、DPR、CSS 版面、網路狀態與快取影響。

如果 `sizes` 寫錯，甚至可能下載比預期更大的圖片。

---

### 響應式圖片是不是可以不用 CSS？

不是。

響應式圖片和 CSS 解決的是不同問題：

| 技術 | 解決問題 |
|---|---|
| CSS `max-width: 100%; height: auto;` | 讓圖片在畫面上不要超出容器 |
| `srcset` / `sizes` / `<picture>` | 讓瀏覽器有機會下載更合適的圖片資源 |

3. `sizes` 不是 CSS，它是告訴瀏覽器圖片在不同 viewport 下大約顯示多寬。

10. 圖片只是裝飾背景、不影響內容理解時，可以考慮 CSS background。
