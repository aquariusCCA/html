響應式圖片 Demo

這個範例把筆記中的 `srcset`、`sizes`、`picture`、`source`、DPR、格式 fallback 與 art direction 放在同一頁。可直接用瀏覽器開啟，再搭配 DevTools 的 Network 與 Elements 面板觀察實際被選中的圖片候選。

觀察重點

  • `1x / 2x` 適合固定顯示尺寸的 Logo 或 Icon。
  • `480w / 800w / 1200w` 必須搭配 `sizes` 告訴瀏覽器圖片顯示槽大小。
  • `picture` 用在格式切換或不同版面裁切，不是拿來取代所有 `img`。
  • `picture` 內仍然需要最後的 `img src` 與 `alt`。
DPR calculator

`sizes`、DPR 與 `srcset` 候選計算

筆記中的核心公式是：瀏覽器先由 `sizes` 得到圖片顯示槽寬度，再乘上 DPR，最後從 `srcset` 挑一張足夠接近的圖片。

slot size 390px
needed width 780px
available candidates 480 / 800 / 1200w
browser would pick 800w
srcset x

固定尺寸圖片：`1x / 2x`

這種寫法用在顯示尺寸固定的 Logo、Icon、Avatar。瀏覽器依螢幕 DPR 選擇 `1x` 或 `2x`。 

<img
  class="logo"
  src="./images/logo-1x.svg"
  srcset="
    ./images/logo-1x.svg 1x,
    ./images/logo-2x.svg 2x
  "
  width="160"
  height="80"
  alt="響應式圖片課程 Logo">

srcset w

可變尺寸圖片：`w + sizes`

`480w` 不是 CSS 寬度，而是圖片檔本身的寬度描述。`sizes` 描述圖片在不同 viewport 下大約會佔多少 CSS 寬度。 

<img
  src="./images/banner-800.svg"
  srcset="
    ./images/banner-480.svg 480w,
    ./images/banner-800.svg 800w,
    ./images/banner-1200.svg 1200w
  "
  sizes="(max-width: 640px) 100vw,
         (max-width: 1040px) 80vw,
         800px"
  width="800"
  height="450"
  alt="山景示意圖">

picture type

格式候選：`source type`

真實專案常用 AVIF / WebP / JPG。此本地 demo 用 `image/svg+xml` 與 `img` fallback 示範同一個結構，避免需要額外轉檔工具。 

<picture>
  <source
    srcset="./images/format-modern.svg"
    type="image/svg+xml">
  <img
    src="./images/format-fallback.svg"
    alt="圖片格式 fallback 示意">
</picture>

picture media

版面裁切：`source media`

當手機與桌機不只是大小不同，而是構圖不同，應使用 `picture` 做 art direction。 

<picture>
  <source
    media="(max-width: 640px)"
    srcset="./images/hero-mobile.svg">
  <source
    media="(min-width: 641px)"
    srcset="./images/hero-desktop.svg">
  <img
    src="./images/hero-desktop.svg"
    alt="響應式圖片課程主視覺">
</picture>

fallback and alt

`picture` 仍然需要 `img`，`alt` 寫在 `img` 上

正確結構

<picture>
  <source srcset="./images/format-modern.svg" type="image/svg+xml">
  <img src="./images/format-fallback.svg" alt="格式 fallback 示意">
</picture>

`source` 只提供候選資源；最後的 `img` 才是實際圖片元素、fallback 與可存取性入口。

常見錯誤

<picture>
  <source
    srcset="./images/format-modern.svg"
    type="image/svg+xml"
    alt="錯誤：alt 不屬於 source">
</picture>

少了 `img`，舊環境沒有 fallback；把 `alt` 放在 `source` 上也不會提供圖片替代文字。

decision table

情境對照

需求 建議寫法 原因
固定尺寸 Logo / Icon `srcset` + `1x / 2x` 顯示槽固定，只需要依 DPR 提供更高密度圖片。
文章封面、Banner、商品圖 `srcset` + `w` + `sizes` 圖片顯示尺寸會跟版面變化，瀏覽器需要知道 slot size。
AVIF / WebP / JPG 格式降級 `picture` + `source type` + `img` 讓支援新格式的瀏覽器優先下載更小的檔案，不支援時回到 `img src`。
手機與桌機構圖不同 `picture` + `source media` 這是 art direction，不只是同一張圖換大小。
裝飾性背景 CSS `background-image` 或 `alt=""` 沒有內容意義的圖片不應干擾讀屏與主要內容。
