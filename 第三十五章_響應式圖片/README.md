# 第三十五章｜響應式圖片

> 章節定位：理解圖片如何根據螢幕大小、顯示寬度、像素密度與格式支援，選擇合適的圖片資源  
> 建議閱讀方式：先理解 `srcset` / `sizes`，再掌握瀏覽器選圖流程，最後看 `<picture>` 和 `<source>` 的進階用途

## 本章在學什麼

這一章是在第十八章 `<img>` 基礎上繼續往下補。

重點不是換一種寫圖片的方式，而是理解瀏覽器如何在不同裝置與版面條件下選擇合適的圖片資源。

你會學到：

- `srcset` 可以提供多個候選圖片。
- `1x` / `2x` 適合處理固定顯示尺寸下的不同螢幕像素密度。
- `480w` / `800w` / `1200w` 適合提供不同寬度版本的圖片。
- `sizes` 告訴瀏覽器圖片在不同條件下大約會顯示多寬。
- 瀏覽器會根據 `sizes` 算出的顯示寬度，再搭配 DPR，從 `srcset` 中挑選合適的圖片。
- `<picture>` 適合做格式備援或藝術方向切換。
- `<source>` 負責提供條件式候選圖片。
- `<img>` 仍然是最終顯示圖片與提供替代文字的核心元素。

## 本章與整體地圖的關係

- 前置知識：第十八章已學過 `<img>`、`src`、`alt` 與常見圖片格式。
- 這章的核心任務：讓圖片在不同裝置上載入更有效率、更合適、更清晰的版本。
- 讀完後通常接：圖文語意、媒體字幕、可存取性、效能優化等內容。

## 核心記憶句

### `srcset`

`srcset` 負責提供多個候選圖片，讓瀏覽器可以選擇。

### `sizes`

`sizes` 負責告訴瀏覽器：這張圖片在目前版面條件下大約會顯示多寬。

### `srcset + sizes`

當 `srcset` 使用 `480w`、`800w`、`1200w` 這種寬度描述符時，可以用下面這句話理解：

```text
需要的圖片寬度 ≈ sizes 算出的顯示寬度 × DPR
```

例如：

```text
圖片顯示寬度是 390px
螢幕 DPR 是 2
需要的圖片寬度約為 390px × 2 = 780px
```

如果 `srcset` 裡有 `480w`、`800w`、`1200w`，瀏覽器通常會選擇接近需求的 `800w` 圖片。

### `<picture>`

`<picture>` 適合在「不是只換圖片大小」的情況下使用，例如：

- 不同格式備援：AVIF / WebP / JPG
- 不同構圖切換：手機用直圖，桌機用橫幅圖
- 較複雜的條件式圖片選擇

## 易混淆主題

- `srcset` vs `<picture>`：前者偏候選資源選擇，後者偏格式備援、媒體條件或藝術方向切換。
- `x` 描述符 vs `w` 描述符：前者描述螢幕像素密度，後者描述圖片檔案本身的寬度。
- `sizes` vs CSS 寬度：CSS 決定圖片實際顯示多寬，`sizes` 是提前告訴瀏覽器「圖片大約會顯示多寬」，讓瀏覽器可以在下載前從 `srcset` 中選圖。
- `src` vs `srcset`：`src` 是基本圖片來源與 fallback，`srcset` 是補充候選資源。
- `<source>` vs `<img>`：`<source>` 提供候選來源，`<img>` 才是實際顯示圖片與承載 `alt` 的元素。
- 圖片內容 vs 裝飾圖片：有資訊意義的圖片應用 `<img>` / `<picture>`，純裝飾圖片可以考慮 CSS background 或 `alt=""`。
