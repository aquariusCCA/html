# 水平線標籤

> 所屬章節：第十五章｜水平線標籤  
> 關鍵字：水平線標籤、`<hr>`、horizontal rule、thematic break、主題分隔、內容切換、HTML 分隔線  
> 建議回查情境：想知道 `<hr>` 是做什麼的、想分清主題分隔和單純畫線的差別、想確認什麼情況適合用水平線標籤

## 本節導讀

這篇整理 HTML 的水平線標籤 `<hr>`。  
它的核心作用，不只是畫出一條線，更重要的是用來表示內容主題或段落脈絡發生切換。

初學時，很多教材會先從「畫一條水平線」來介紹 `<hr>`。  
這個理解不算錯，但如果只停在外觀層，就會忽略它更接近「主題分隔」的語意角色。

## 你會在這篇學到什麼

- `<hr>` 的基本用途
- `<hr>` 的語意重點與常見特點
- 什麼情況適合用水平線標籤
- `<hr>` 和單純視覺分隔線的差別

## 30 秒複習入口

- `<hr>` 用來表示內容主題或話題出現分隔與轉換。
- 它是空元素，常見寫法有 `<hr>` 和 `<hr />`。
- 瀏覽器通常會把它顯示成一條水平線，但外觀不是它唯一重點。
- 如果只是想做裝飾線條，仍要把語意和樣式分開理解。

## 速查區

### 核心概念

- `<hr>` 代表 thematic break，可以理解成內容主題的分隔點。
- 它常出現在兩段主題不同、脈絡切換的內容之間。

### 關鍵規則 / 判準

- 常見寫法是 `<hr>`；部分教材也會寫成 `<hr />`。
- `<hr>` 是空元素，本身不包內容，也不需要結束標籤。
- 如果兩塊內容之間出現明顯主題切換，可以考慮用 `<hr>`。
- 如果只是想做版面裝飾，不一定需要用 `<hr>` 來承擔語意。

### 常見使用場景

- 文章中不同主題段落之間的分隔
- 頁面中不同內容區塊之間的脈絡切換
- 同一頁內需要讓讀者明確感受到內容段落已經換題

### 常見混淆點

- `<hr>` 不只是畫線工具。
- 水平線外觀是預設呈現方式，不等於它的全部語意。
- 如果只是要控制線條樣式或裝飾效果，也可能改由 CSS 處理。

### 一句話對比

- `<hr>` 解決的是「內容主題是否切換」；單純線條樣式處理，通常交給 CSS。

## 正文筆記

### 這篇在解決什麼問題？

- 在一篇頁面內容裡，有時不只是換行或換段落，而是整個主題重點已經轉到另一塊內容。
- `<hr>` 就是在處理這種「內容分隔」的需求，讓讀者更容易感受到脈絡切換。

## 1. `<hr>` 的作用

> 標籤語意：主題的分割轉換。

- 這句話的方向是對的，意思是：`<hr>` 用來表示內容主題出現分隔或轉換。
- 它不是單純為了畫一條線，而是在標示「這裡的內容脈絡換了」。

## 2. `<hr>` 的特點

- `<hr>` 是單標籤，也就是空元素。
- 瀏覽器通常會把它顯示成一條水平線，所以初學時很容易先注意到它的外觀。
- 但更值得記住的是，它的用途偏向內容分隔，而不是純裝飾。

## 3. 什麼時候適合用 `<hr>`

- 當前後兩塊內容已經是不同主題，或閱讀上需要一個清楚的分隔點時。
- 例如文章從一個話題切到另一個話題、頁面從主要內容切到補充資訊時，就可能用到 `<hr>`。

### 什麼情況不適合？

- 如果只是想畫一條裝飾線，未必需要靠 `<hr>` 表達。
- 如果內容其實只是同一主題下的排版留白，也不應把 `<hr>` 當成單純版面工具濫用。

## 4. 基本示例

```html
<hr />
```

### 這個例子在說什麼？

- 這個標籤本身不包任何文字內容。
- 瀏覽器通常會把它呈現成一條水平線，讓讀者感受到內容之間有一道分隔。

## 5. 使用 `<hr>` 時的注意點

- 不要只把 `<hr>` 理解成「畫線標籤」。
- 更穩定的理解方式，是把它當成主題分隔標記。
- 如果只是想微調視覺效果，應優先考慮 CSS；如果是內容脈絡切換，再考慮是否用 `<hr>`。

## 常見問法

### `<hr>` 只是畫一條線嗎？

- 不只如此。
- 它常被顯示成水平線，但更核心的意義是表示內容主題的分隔。

### `<hr>` 和換行標籤 `<br>` 有什麼差別？

- `<br>` 是在同一段內容中強制換行。
- `<hr>` 則是在更高一層表示內容主題或段落脈絡的分隔。

### 如果只想做裝飾線條，還要用 `<hr>` 嗎？

- 不一定。
- 如果需求主要是視覺效果，而不是語意上的分隔，可以考慮交給 CSS 處理。
