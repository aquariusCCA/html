# 水平線標籤

> 所屬章節：第十五章｜水平線標籤
> 關鍵字：水平線標籤、`<hr>`、horizontal rule、thematic break、主題分隔、內容切換、HTML 分隔線
> 建議回查情境：想知道 `<hr>` 是做什麼的、想分清主題分隔和單純畫線的差別、想確認什麼情況適合用水平線標籤、想知道什麼情況應改用 CSS

## 本節導讀

這篇整理 HTML 的水平線標籤 `<hr>`。

很多初學者第一次接觸 `<hr>` 時，會先把它理解成「畫一條水平線的標籤」。
這個理解不算錯，因為瀏覽器通常確實會把它顯示成一條線。

但更準確的理解是：

**`<hr>` 不只是畫線，而是用來表示內容在主題或敘述脈絡上出現分隔。**

也就是說，`<hr>` 的核心不在外觀，而在語意。
它表達的是：

* 這裡的內容重點切換了
* 這裡和前面的內容已經是另一段脈絡
* 讀者在閱讀上應該感受到一個明顯的分隔點

* `<hr>` 的基本用途

## 30 秒複習入口

* `<hr>` 用來表示內容主題或敘述脈絡出現分隔與切換。
* 它是空元素，常見寫法有 `<hr>` 和 `<hr />`。
* 瀏覽器通常會把它顯示成一條水平線，但外觀不是它的核心。
* 如果只是想做裝飾線條，通常應優先交給 CSS。
* 如果前後內容在主題上有明顯切換，才比較適合考慮使用 `<hr>`。

## 速查區

### 核心概念

* `<hr>` 代表 thematic break，可以理解成內容脈絡中的主題分隔點。
* 它通常用在前後兩塊內容之間，表示閱讀脈絡已經出現明顯切換。

### 關鍵規則 / 判準

* 常見寫法是 `<hr>`；部分教材也會寫成 `<hr />`。
* `<hr>` 是空元素，本身不包內容，也不需要結束標籤。
* 如果前後內容之間有明顯主題切換，可以考慮使用 `<hr>`。
* 如果只是單純想畫一條線，不一定要讓 `<hr>` 承擔語意。

### 常見使用場景

* 文章中不同主題段落之間的分隔
* 一頁內容從主要說明切換到補充資訊
* 同一頁中話題、段落脈絡出現明顯轉折
* 需要讓讀者感受到「以下開始是另一塊內容」

### 常見混淆點

* `<hr>` 不只是畫線工具。
* 它的預設外觀是一條線，但這不等於它的全部意義。
* `<hr>` 不是拿來取代段落或標題的。
* 如果只是要做視覺裝飾線條，通常交給 CSS 更合適。

### 一句話對比

* `<hr>` 解決的是「內容主題是否切換」
* CSS 線條解決的是「畫面需不需要一條線」

## 正文筆記

## 1. 這篇在解決什麼問題？

在一篇頁面內容中，有時不只是換行，也不只是進入下一段，
而是整個主題、敘述方向、閱讀脈絡都已經轉到另一塊內容。

例如：

* 前面都在講基本概念，後面開始講實作注意事項
* 前面是主要內容，後面切到補充說明
* 前面是正文，後面切到附註、延伸資訊或結語

這種情況下，單靠換行或段落，有時還不夠明確。
這時就可能用到 `<hr>`，讓讀者感受到：

**這裡不是普通的排版分開，而是內容脈絡真的換了。**

---

## 2. `<hr>` 的作用

> 標籤語意：主題的分隔與轉換。

這句方向是對的，但可以再講得更完整：

`<hr>` 的作用，不是單純畫出一條水平線，
而是表示：

**前後內容在主題、話題或敘述脈絡上有明顯分隔。**

你可以把它理解成：

> 在文件中插入一個「主題分隔點」。

重點是：

* 它不是只是為了好看
* 它不是只是增加視覺層次
* 它更像是在告訴讀者：這裡之後是另一段內容脈絡

---

## 3. `<hr>` 的特點

* `<hr>` 是空元素，也就是單標籤。
* 它本身不包任何內容，也不需要結束標籤。
* 瀏覽器通常會把它呈現成一條水平線，因此初學者很容易先注意到它的外觀。
* 但更值得記住的是：它偏向**語意上的主題分隔**，而不是純裝飾用途。

### 常見寫法

```html
<hr>
```

有些教材也會寫成：

```html
<hr />
```

在一般 HTML 學習情境中，可以把它們視為同一個水平線標籤即可。

---

## 4. 什麼時候適合用 `<hr>`

當前後兩塊內容之間，已經不只是排版上的分開，
而是**主題、重點或閱讀脈絡有明顯切換**時，就比較適合考慮使用 `<hr>`。

### 典型情境 1：文章切換話題

```html
<p>前面介紹的是 HTML 的基本文字標籤。</p>
<p>接下來我們要開始整理表單相關元素。</p>

<hr>

<p>以下開始介紹 input、label 與 button 的基本用法。</p>
```

這裡的 `<hr>` 表示：
前面和後面雖然都屬於同一篇文章，但主題焦點已經切換。

---

### 典型情境 2：從正文切到補充資訊

```html
<p>以上是本章的核心整理。</p>

<hr>

<p>補充：如果你只是想畫裝飾線條，通常應優先考慮 CSS。</p>
```

這裡的分隔感，不只是版面上的空開，
而是讓讀者知道：下面這段比較像補充說明，而不是正文的直接延續。

---

### 最實用的判準

你可以直接這樣記：

* **前後內容只是自然往下延續：不一定需要 `<hr>`**
* **前後內容在主題或脈絡上明顯切換：可以考慮 `<hr>`**

---

## 5. 什麼情況不適合用 `<hr>`

如果你的需求只是以下這幾種，通常就不一定適合用 `<hr>`：

* 想單純畫一條好看的線
* 想讓版面更有層次
* 想撐出上下空白
* 想取代段落間距
* 想取代標題層級或內容分段結構

也就是說，`<hr>` 不應該被當成：

* 萬用裝飾工具
* 萬用版面分隔工具
* 段落標籤替代品

### 不太理想的想法

「這裡空一點看起來比較舒服，所以加一條 `<hr>`。」

如果你的目的只是視覺效果，
那通常更應該先想：

* 這是不是交給 CSS 就好？
* 這裡是不是其實應該用 margin、padding、border？
* 這裡是不是應該靠標題、section 或段落結構來分？

---

## 6. 基本示例

```html
<hr>
```

### 這個例子在說什麼？

* 這個標籤本身不包任何文字內容
* 瀏覽器通常會把它呈現成一條水平線
* 但它真正想表達的是：內容之間有一個分隔點

所以你可以把這個標籤的閱讀感受理解成：

**「從這裡開始，內容脈絡有明顯切換。」**

而不只是：

**「這裡多了一條線。」**

---

## 7. `<hr>` 和 `<br>` 有什麼差別？

這兩個標籤都可能讓畫面看起來有「分開」的感覺，
但它們處理的層級完全不同。

### `<br>`

`<br>` 是在**同一段內容中強制換行**。
它處理的是行與行之間的斷開。

例如：

```html
第一行內容<br>
第二行內容
```

這裡仍然是同一段內容，只是換到下一行。

---

### `<hr>`

`<hr>` 是在更高一層表示**內容主題或敘述脈絡的分隔**。
它不是在同一段裡換行，而是在告訴你：

**前後兩塊內容的閱讀脈絡已經不同。**

---

### 最簡單的記法

* `<br>`：同一段內容中的換行
* `<hr>`：不同內容脈絡之間的分隔

---

## 8. `<hr>` 和純 CSS 線條有什麼差別？

這也是實作時很常遇到的問題。

### 使用 `<hr>` 的情況

當你的需求是：

* 這裡真的有主題切換
* 這裡不是單純裝飾，而是內容語意上要被分開
* 你想表達這是一個閱讀脈絡的分隔點

這時就可以考慮使用 `<hr>`。

---

### 使用 CSS 線條的情況

如果你的需求只是：

* 想畫一條線讓畫面更好看
* 想做卡片分隔效果
* 想做視覺裝飾
* 想控制粗細、顏色、間距、長度

那很多時候更適合交給 CSS，例如：

* `border-top`
* 偽元素
* 區塊樣式設計

---

### 最實用的判準

可以直接記成一句：

* **有語意上的主題切換：考慮 `<hr>`**
* **只有視覺上的線條需求：交給 CSS**

---

## 9. 使用 `<hr>` 時的注意點

### 不要只把 `<hr>` 理解成畫線標籤

這是最常見的誤解。
雖然它預設會顯示成水平線，但更穩定的理解方式應該是：

**它是主題分隔標記，不是單純線條工具。**

---

### 不要拿 `<hr>` 取代段落或標題

如果內容只是一般地往下推進，
很多時候用：

* 標題層級
* 段落
* `section`
* 其他結構標籤

就已經足夠了。

只有在你想強調：

**這裡真的出現脈絡切換**

時，才比較適合使用 `<hr>`。

---

### 如果只是微調視覺效果，優先考慮 CSS

若需求主要是：

* 調整間距
* 增加空白
* 改變線條造型
* 做裝飾感

那應優先考慮 CSS，而不是先想到 `<hr>`。

因為：

* `<hr>` 處理的是語意分隔
* CSS 處理的是視覺呈現

兩者不要混成同一件事。

---

## 常見問法

### `<hr>` 只是畫一條線嗎？

不只如此。

它通常會被顯示成一條水平線，但更核心的意義是表示內容主題或閱讀脈絡的分隔。

---

### `<hr>` 和 `<br>` 有什麼差別？

* `<br>` 是同一段內容中的強制換行
* `<hr>` 是更高一層的內容主題分隔

兩者都可能讓畫面看起來有分開感，但語意層級不同。

---

### 如果只想做裝飾線條，還要用 `<hr>` 嗎？

不一定。

如果需求主要是視覺效果，而不是語意上的分隔，通常可以考慮交給 CSS 處理。

---

### `<hr>` 可以拿來取代段落分段嗎？

不建議。

因為 `<hr>` 表示的是主題分隔，不是一般段落結構。
如果只是內容自然分段，通常還是應該用段落、標題或其他結構方式來處理。
