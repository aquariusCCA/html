`<hr>` 的核心不是「畫出一條線」，而是表示前後內容在主題或閱讀脈絡上出現明顯分隔。它適合用在話題切換、正文轉補充、主要說明轉注意事項這類情境；如果你只是想要一條視覺上的裝飾線，通常應該先考慮 CSS。

`<hr>` 是主題分隔 它表示內容脈絡切換，而不只是讓畫面多出一條線。
適合內容轉場 例如正文切到補充、基礎概念切到實作提醒、文章切到結語。
不是 `<br>` `<br>` 是同段內換行，`<hr>` 是更高層的內容分隔。
不是萬用畫線工具 若需求只是裝飾線條或 spacing，應優先交給 CSS。

2. `<hr>` 和 `<br>` 的差異

        

          

            同段換行
            
`<br>` 處理的是同一段裡的斷行

            
客服信箱：service@example.com<br>
服務時間：週一到週五 09:00 - 18:00

            

              客服信箱：service@example.com

              服務時間：週一到週五 09:00 - 18:00
            

            

              這仍是同一組資訊，只是需要換到下一行。
            

          


          

            主題分隔
            
`<hr>` 處理的是內容脈絡切換

            
<p>以上整理的是基本觀念。</p>
<hr>
<p>下面開始進入實作時常見的錯誤。</p>

            

              
以上整理的是基本觀念。

              

              
下面開始進入實作時常見的錯誤。

            

            

              這裡不是單純換行，而是整個閱讀重點轉到另一塊內容。

語意分隔

            
有主題切換時，`<hr>` 比較合理

            

              如果線條代表閱讀脈絡的轉場，那它不只是外觀，這時用 `<hr>` 更貼切。
            

            

            

          

        

      


      

        
4. 不要把 `<hr>` 當萬用分隔工具

        

          

            

              
              
            

          


          

            
不建議

            
不是每個空隙都代表主題切換

            

              若只是想把內容切成一塊一塊，很多時候應先考慮標題、段落、section 或 CSS spacing。
            

            

            

            

              `<hr>` 用太多會讓內容看起來處處都在轉場，反而削弱真正重要的分隔點。
            

          

        

      


      

        
5. 實務提醒

        

          

            先判斷是不是主題切換
            如果前後只是自然延續，通常不必急著加 `<hr>`。
          

          

            線條外觀不是核心
            瀏覽器會把它顯示成線，但重點是語意，不是畫線本身。
          

          

            裝飾與 spacing 交給 CSS
            `border-top`、偽元素、`margin`、`padding` 才是視覺控制的正規工具。
