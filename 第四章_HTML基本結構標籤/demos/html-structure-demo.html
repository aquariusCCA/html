HTML 基本結構標籤 Demo

這頁把第四章最常見的文件開頭資訊拆成可切換示例：`DOCTYPE`、`lang`、`charset`、屬性、布林屬性與註解。 重點不是背寫法，而是看懂每一段到底在告訴瀏覽器什麼。

DOCTYPE 告訴瀏覽器這份文件要用哪種 HTML 規則解析。
lang 標示文件主要語言，不是翻譯功能。
charset 指定文字編碼方式，重點是避免亂碼。
屬性與註解 屬性補充元素設定；註解寫給開發者看，不參與畫面顯示。

這頁怎麼看

  • 先切換文件開頭的三種基礎設定，看它們分別處理哪件事。
  • 再看屬性區，對照一般屬性和布林屬性的差別。
  • 最後用註解區看「原始碼看得到，但頁面不會顯示」的差異。

1. 文件開頭那幾行在做什麼

文件類型

DOCTYPE：先告訴瀏覽器這是一份 HTML5 文件

這段不會顯示在畫面上，但它會先影響瀏覽器理解這份文件的方式。 

<!DOCTYPE html>
處理對象

文件類型與解析規則

常放位置

整份 HTML 文件最前面

常見誤解

它不是一般 HTML 元素，也不是頁面內容。

<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <title>我的第一個網頁</title>
  </head>
  <body>
    <h1>Hello HTML</h1>
  </body>
</html>
我的第一個網頁
你可以把這些設定想成「先交代文件背景」，再讓 body 內容顯示給使用者看。

Hello HTML

這是一份正常被瀏覽器理解的 HTML 文件。

2. 屬性是怎麼補充元素設定的

一般屬性

屬性寫在開始標籤內，用來補充元素資訊

例如 `href`、`src`、`placeholder` 這些，都屬於會帶值的常見屬性。 

<input type="text" placeholder="請輸入姓名" maxlength="20">
一般屬性

重點在屬性值，例如 placeholder 顯示什麼文字。

布林屬性

重點在有沒有出現這個屬性，而不是寫了什麼值。

屬性都寫在開始標籤內，標籤名和屬性之間要有空格，屬性之間也要分開。

3. 註解會出現在哪裡，不會出現在哪裡

<!-- 頁首區塊 -->
<header>
  <h1>我的網站</h1>
</header>
不顯示在畫面上

註解是寫給開發者看的說明

註解可以幫助閱讀結構，但它本身不是頁面內容，也不會被當成正文顯示。

畫面上真正會顯示的是：我的網站
這段代表註解內容，但你在正常頁面上不會看到它。
原始碼裡仍看得到註解，所以不要把帳號、密碼、金鑰等敏感資訊放進去。
