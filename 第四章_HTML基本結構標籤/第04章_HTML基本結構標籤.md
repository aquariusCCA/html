# HTML 基本結構標籤

> 所屬章節：第四章｜HTML 基本結構標籤  
> 關鍵字：`DOCTYPE`、文檔類型聲明、`lang`、語言設定、`charset`、字符編碼、`UTF-8`、HTML 屬性、布林屬性、註解、注釋  
> 建議回查情境：想確認 `<!DOCTYPE html>` 在做什麼、想知道 `lang` 和 `charset` 分別控制什麼、想快速回查 HTML 屬性怎麼寫、想複習 HTML 註解語法

## 本節導讀

這篇整理 HTML 文件剛開始最常看到的幾種基本結構資訊。  
重點不是背零碎名詞，而是先分清楚：哪些內容是在告訴瀏覽器「這份文件是什麼」、哪些是在描述標籤本身還可以附帶哪些資訊。

如果你剛開始讀 HTML，這篇可以當成進入 `html`、`head`、`meta`、標籤屬性與註解語法前的基本判讀頁。

## 你會在這篇學到什麼

- `<!DOCTYPE html>` 在頁面中的作用
- `lang` 屬性是在描述什麼
- `charset` 為什麼常寫成 `UTF-8`
- HTML 屬性的基本寫法與常見注意點
- HTML 註解的語法與用途

## 30 秒複習入口

- `<!DOCTYPE html>` 用來宣告這份文件採用 `HTML5` 的文檔類型。
- `lang` 用來標示目前文件的主要語言。
- `charset` 用來指定頁面使用的字符編碼，常見寫法是 `UTF-8`。
- 屬性是加在標籤上的附加資訊，通常寫在開始標籤內。
- HTML 註解以 `<!--` 開頭、`-->` 結尾，內容不會顯示在頁面上。

## 速查區

### 核心概念

- `DOCTYPE` 是在告訴瀏覽器目前文件採用哪一種 HTML 文檔類型。
- `lang` 與 `charset` 都屬於頁面層級的基礎設定，但處理的問題不同：前者是語言，後者是編碼。
- 屬性是附著在 HTML 標籤上的補充資訊，用來描述元素的行為、狀態或設定。
- 註解是給開發者看的輔助文字，不會被當成頁面內容顯示。

### 關鍵規則 / 判準

- `<!DOCTYPE html>` 要放在整份文件最前面，且位於 `<html>` 標籤之前。
- `lang` 通常寫在 `<html>` 標籤上，例如 `<html lang="en">`。
- `charset` 通常寫在 `<head>` 內的 `<meta charset="UTF-8">`。
- 屬性一般寫在開始標籤內，屬性之間用空格分隔。
- 註解內容只供閱讀與維護，不會直接顯示在頁面上。

### 常見混淆點

- `<!DOCTYPE>` 不是一般 HTML 標籤，而是文檔類型聲明。
- `lang` 不是在改變翻譯內容，而是在標示文件主要語言。
- `charset` 不是「顯示哪一國語言」，而是規定文字如何編碼與解碼。
- `disabled` 這類寫法不是「只有屬性值」，而是可以只寫屬性名的屬性形式。
- 註解是給人看的，不是讓瀏覽器執行的程式碼。

### 一句話對比

- `DOCTYPE` 管文檔類型，`lang` 管語言標示，`charset` 管字符編碼，屬性管標籤附加設定，註解管開發者閱讀說明。

## 正文筆記

### 這篇在解決什麼問題？

- 當我們開始寫 HTML 文件時，除了真正顯示在頁面上的內容，還需要先處理幾種基礎結構資訊。
- 這些資訊會影響瀏覽器如何理解頁面、如何解讀文字，以及如何判讀標籤上的附加設定。
- 這篇整理的重點，就是建立這些最常見基本結構標籤與語法的判讀框架。

## 1. 文檔類型聲明標籤：`<!DOCTYPE html>`

- 文檔類型聲明的作用，是告訴瀏覽器這份網頁應該採用哪種 HTML 文檔類型來解析。
- 在現代 HTML 中，最常見的寫法是 `<!DOCTYPE html>`，表示目前頁面採用的是 `HTML5`。

```html
<!-- 這句代碼的意思是：當前頁面採取的是 HTML5 版本來顯示網頁。 -->
<!DOCTYPE html>
```

### 注意點

- `<!DOCTYPE>` 聲明要位於整份文檔最前面，放在 `<html>` 標籤之前。
- `<!DOCTYPE>` 不是一個 HTML 標籤，它屬於文檔類型聲明。
- 它的重點不是顯示內容，而是幫助瀏覽器用較正確的方式解析頁面。

## 2. `lang`：標示當前文檔的語言

- `lang` 用來定義當前文檔的主要語言。
- 例如 `zh-CN` 常用來表示簡體中文語境，`en` 則表示英文語境。
- 這類設定通常會寫在 `<html>` 標籤上。

```html
<html lang="en">
  <!-- 省略其他內容 -->
</html>
```

### 這一段要怎麼理解？

- `lang` 的重點不是翻譯頁面，而是讓瀏覽器、搜尋系統或輔助工具知道這份文件主要使用哪種語言。
- 它屬於整份文件的基礎描述資訊，因此常放在最外層的 `<html>` 標籤上。

## 3. 字符集：`charset`

- `charset` 用來規定 HTML 文檔應該使用哪一種字符編碼。
- 最常見的寫法，是在 `<head>` 中加入 `<meta charset="UTF-8">`。

```html
<head>
  <meta charset="UTF-8">
</head>
```

> `charset` 常見的值有 `GB2312`、`BIG5`、`GBK` 和 `UTF-8`。其中 `UTF-8` 是目前最常見的通用編碼方式，能處理多數語言文字。

### 這一段要怎麼理解？

- `charset` 在處理的是「文字如何被編碼與解碼」。
- 如果編碼設定錯誤，就可能出現亂碼。
- 因為現代網頁常需要同時處理不同語言文字，所以 `UTF-8` 會是最常見的預設選擇。

## 4. 標籤屬性

> 屬性用來給標籤提供附加資訊。

![標籤屬性1.png](./images/image_1776059557269_2c6af9f1.png)

![標籤屬性2.png](./images/image_1776059557274_c55d9a66.png)

- 屬性可以寫在單標籤或雙標籤的開始標籤中，例如：

```html
<marquee loop="1" bgcolor="green">
  尚硅谷，讓天下沒有難學的技術!!!
</marquee>
```

- 有些特殊屬性可以只寫屬性名，不另外寫值，例如：

```html
<input disabled>
```

### 屬性注意點

- 不同標籤有不同屬性，也有一些屬於通用屬性，可以寫在多數標籤上。
- 屬性名與屬性值不能隨意自創，應依規範與標籤定義來寫。
- 屬性名與屬性值通常不區分大小寫，但實務上仍建議統一使用小寫。
- 雙引號、單引號通常都可以，但教材與實務中多半推薦雙引號。
- 標籤中不應重複出現同名屬性，例如：

```html
<!-- 同名屬性不應重複 -->
<input type="text" type="password">
```

- 標籤的屬性寫在開始標籤內部。
- 一個標籤可以同時有多個屬性。
- 屬性之間以空格隔開。
- 標籤名與第一個屬性之間必須以空格隔開。
- 多個屬性之間通常沒有固定順序。

## 5. 註解

> 註解是寫給開發者看的補充說明，也常叫作「注釋」。

```html
<!-- 註解語句 -->
```

- HTML 中的註解以 `<!--` 開頭，以 `-->` 結束。
- 註解裡的內容主要是給開發者閱讀，不會直接顯示在頁面上。
- 添加註解的目的是幫助自己或他人理解程式碼的功能與用途。
