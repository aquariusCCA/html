# details 與 summary：建立可展開的內容區塊

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `<details>` 與 `<summary>` 各自負責的語意。
- 使用原生 HTML 建立可展開與收合的內容區塊。
- 知道 `open` 屬性的作用。
- 避免把 `<details>` 當成所有互動元件的替代品。

## 使用情境

有些內容不需要一開始全部顯示出來，但又不能完全藏起來。例如常見問題、文章概要、補充說明、產品規格細節，或一段需要讀者自行展開查看的說明。

如果只是要做一個簡單的「點擊標題後展開內容」效果，HTML 已經提供了原生標籤：`<details>` 與 `<summary>`。這種情境不一定需要先寫 JavaScript。

## 一句話理解

`<details>` 表示一組可以展開或收合的詳細內容；`<summary>` 則是這組內容中永遠可見、可被點擊的摘要標題。

## 標籤語意

`<details>` 是 HTML5 新增的互動式標籤，用來包住一段可以被使用者開啟或關閉的內容。

`<summary>` 通常放在 `<details>` 裡面作為第一個子元素，用來定義這組可展開內容的標題。瀏覽器會讓 `<summary>` 顯示在畫面上，使用者點擊它時，`<details>` 裡的其他內容會展開或收合。

換句話說：

- `<summary>`：使用者一開始看得到、可以點擊的標題。
- `<details>` 裡的其他內容：預設可能被隱藏，展開後才顯示的詳細資訊。

## 基本語法

```html
<details>
  <summary>文章概要</summary>
  <div>這裡放文章的詳細概要內容。</div>
</details>
```

這段程式碼會產生一個可展開的區塊。畫面一開始通常只會看到「文章概要」，使用者點擊後才會看到 `<div>` 裡的詳細內容。

## 使用 open 預設展開

如果希望內容一開始就是展開狀態，可以在 `<details>` 上加入 `open` 屬性。

```html
<details open>
  <summary>文章概要</summary>
  <div>文章內容、重點整理或補充說明可以放在這裡。</div>
</details>
```

`open` 是布林屬性。只要出現在 `<details>` 上，就代表這個區塊預設展開；沒有 `open` 時，內容通常預設收合。

## 範例拆解

以上面的範例來看：

- `<details open>`：建立一個可展開與收合的詳細內容區塊，並讓它一開始就展開。
- `<summary>文章概要</summary>`：定義可見標題，使用者點擊這段文字時會切換展開狀態。
- `<div>...</div>`：放入真正的詳細內容。這裡也可以改成段落、列表、圖片或其他適合的 HTML 內容。

這個結構的重點不是「畫面會收合」，而是標籤本身已經表達了內容關係：這是一段詳細資訊，而它的摘要標題是 `<summary>`。

## 常見錯誤

### 只寫 details，不寫 summary

```html
<details>
  <p>這是一段補充說明。</p>
</details>
```

這樣瀏覽器仍然可能顯示一個預設標題，例如「Details」。問題是這個標題不一定符合你的內容，也不利於讀者理解。

比較好的寫法是明確提供 `<summary>`：

```html
<details>
  <summary>補充說明</summary>
  <p>這是一段補充說明。</p>
</details>
```

### 把 summary 放到 details 外面

```html
<summary>文章概要</summary>
<details>
  <p>文章詳細內容。</p>
</details>
```

`<summary>` 應該放在 `<details>` 裡面，並且通常作為第一個子元素。放在外面時，它就不再是該組可展開內容的摘要標題，結構語意也會變得不清楚。

### 用 details 取代所有互動元件

`<details>` 適合簡單的展開與收合內容，但不適合承擔所有互動需求。像是分頁切換、複雜選單、需要記錄狀態的操作面板，通常需要更完整的 HTML、CSS 與 JavaScript 配合。

## 實務判斷

可以優先考慮 `<details>` 的情境：

- 常見問題的答案展開。
- 文章中的補充說明。
- 表格或清單下方的詳細資訊。
- 不想一開始佔用太多版面的次要內容。

不適合只靠 `<details>` 的情境：

- 主要導覽選單。
- 多層級複雜互動。
- 需要細緻控制動畫、焦點管理或資料狀態的元件。

實務上可以先問自己：這塊內容是不是「一個標題加上一段可選擇查看的詳細資訊」？如果是，`<details>` 與 `<summary>` 通常就是合適的選擇。

## 重點整理

- `<details>` 用來建立可展開與收合的詳細內容區塊。
- `<summary>` 是 `<details>` 裡的摘要標題，通常放在第一個子元素位置。
- 詳細內容預設通常是隱藏的，使用者點擊 `<summary>` 後會顯示。
- 在 `<details>` 加上 `open` 屬性，可以讓內容預設展開。
- 單獨使用 `<details>` 時，瀏覽器可能使用預設標題；正式內容應該明確撰寫 `<summary>`。
