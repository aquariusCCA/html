# hr 水平線標籤與主題分隔

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `<hr>` 標籤的語意是「主題的分割轉換」。
- 知道 `<hr>` 是單標籤，不需要結尾標籤。
- 能在內容主題切換時正確使用 `<hr>`。
- 分辨「語意上的分隔」和「單純畫一條線」的差異。

## 問題情境

在一篇文章、說明頁或內容頁中，有時候前後兩段內容不是同一個主題。

例如前面在介紹產品特色，後面要切換到購買注意事項；或前面是一段文章正文，後面接著補充延伸說明。這時候讀者需要知道：內容已經進入下一個主題區塊。

HTML 的 `<hr>` 標籤就是用來表示這種「主題轉換」。

## 一句話理解

`<hr>` 是用來表示內容主題分隔的 HTML 標籤，瀏覽器預設通常會把它顯示成一條水平線。

## 標籤語意

`hr` 早期常被理解成 horizontal rule，也就是水平線。但在現代 HTML 中，更重要的是它的語意：表示前後內容之間有一個主題上的分隔或轉換。

所以使用 `<hr>` 時，判斷重點不是「我想在畫面上放一條線」，而是：

- 前後內容是否屬於不同主題？
- 這裡是否需要讓讀者知道內容已經切換？
- 這條分隔線是否代表文件結構上的段落轉換？

如果只是為了裝飾或排版，通常應該交給 CSS 處理，而不是用 `<hr>` 硬塞一條線。

## 基本語法

`<hr>` 是單標籤，也就是不需要寫 `</hr>`：

```html
<hr>
```

你也可能看到這種寫法：

```html
<hr />
```

在 HTML 中，`<hr>` 是最常見、最簡潔的寫法；`<hr />` 則常見於 XHTML 風格或部分格式化工具產生的 HTML。重點是它沒有內容，也不需要成對包住文字。

## 基本範例

以下是一篇內容頁中使用 `<hr>` 表示主題切換的例子：

```html
<article>
  <h1>前端學習路線</h1>

  <h2>第一階段：HTML</h2>
  <p>先理解 HTML 標籤語意，學會用正確的結構描述網頁內容。</p>

  <hr>

  <h2>第二階段：CSS</h2>
  <p>再學習 CSS，控制文字、間距、排版與響應式畫面。</p>
</article>
```

這個範例中：

- `<article>` 表示一篇完整內容。
- `<h1>` 是文章主標題。
- 兩個 `<h2>` 分別代表不同階段。
- `<hr>` 放在兩個主題中間，表示內容從 HTML 階段切換到 CSS 階段。

## 瀏覽器預設樣式

在瀏覽器的預設樣式下，`<hr>` 通常會顯示成一條水平線，並且前後會有一些間距。

這只是預設外觀。實務上可以用 CSS 調整它的寬度、顏色、粗細與間距：

```css
hr {
  border: 0;
  border-top: 1px solid #ccc;
  margin: 24px 0;
}
```

要注意的是，CSS 可以改變 `<hr>` 的樣子，但不會改變它在 HTML 裡代表「主題分隔」的語意。

## 常見錯誤

### 錯誤一：只為了裝飾而使用 hr

不建議：

```html
<h1>網站標題</h1>
<hr>
<p>歡迎來到我的網站。</p>
```

如果這條線只是設計上的裝飾，並不代表主題轉換，可以用 CSS 的 `border` 來處理：

```css
.site-title {
  border-bottom: 1px solid #ddd;
  padding-bottom: 12px;
}
```

HTML 負責語意，CSS 負責外觀。不要因為想畫線就直接使用 `<hr>`。

### 錯誤二：把 hr 寫成雙標籤

不建議：

```html
<hr></hr>
```

較好的寫法：

```html
<hr>
```

`<hr>` 是單標籤，不需要結尾標籤，也不應該拿來包住其他內容。

### 錯誤三：用大量 hr 取代內容結構

不建議：

```html
<h2>商品介紹</h2>
<p>商品特色說明。</p>
<hr>
<p>價格資訊。</p>
<hr>
<p>售後服務。</p>
```

如果內容其實是清楚的小節，應該優先用標題與段落建立結構：

```html
<h2>商品介紹</h2>
<p>商品特色說明。</p>

<h2>價格資訊</h2>
<p>價格與付款方式說明。</p>

<h2>售後服務</h2>
<p>保固與客服方式說明。</p>
```

`<hr>` 可以輔助表示主題轉換，但不應該取代標題、段落或其他語意標籤。

## 實務使用建議

使用 `<hr>` 前，可以先問自己：

- 前後內容是否真的從一個主題切換到另一個主題？
- 如果把這條線拿掉，讀者是否會比較難理解內容分段？
- 這裡需要的是 HTML 語意，還是純粹的視覺裝飾？

適合使用 `<hr>` 的情況：

- 文章正文與補充說明之間。
- 不同主題區塊之間。
- 表單或內容頁中，前後段落代表不同類型資訊時。

不太適合使用 `<hr>` 的情況：

- 單純想畫一條裝飾線。
- 只是想增加間距。
- 用來取代標題層級或區塊結構。

## 小練習

請判斷下列情境是否適合使用 `<hr>`：

1. 文章前半段是故事正文，後半段是作者補充說明。
2. 想讓網站標題底下多一條細線，看起來比較漂亮。
3. 表單上半部填寫個人資料，下半部填寫付款資訊。
4. 想讓兩段文字之間距離變大。

參考答案：

1. 適合，因為內容主題有明顯切換。
2. 不適合，這是視覺裝飾，應該使用 CSS。
3. 視情況可以使用，因為前後是不同資訊區塊；也可以搭配 `<fieldset>`、標題等更明確的結構。
4. 不適合，間距應該使用 CSS 控制。

## 重點整理

- `<hr>` 的語意是「主題的分割轉換」。
- `<hr>` 是單標籤，不需要結尾標籤。
- 瀏覽器預設通常會把 `<hr>` 顯示成水平線。
- 不要只因為想畫一條線就使用 `<hr>`；純外觀應該用 CSS。
- `<hr>` 適合放在前後內容主題明顯切換的位置。
- 文件結構仍應優先使用標題、段落、區塊等合適的 HTML 標籤建立。
