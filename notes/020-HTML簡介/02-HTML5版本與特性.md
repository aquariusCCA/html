# HTML5：現在學 HTML 為什麼是學它

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 HTML5 是什麼、它在 HTML 發展歷程中的定位。
- 列出 HTML5 新增特性的大致分類方向。
- 知道「狹義 HTML5」與「廣義 HTML5」的差別。
- 知道在實務上該如何看待瀏覽器兼容性問題。

## 問題情境

如果你在網路上搜尋 HTML 教學，可能會看到很多年代久遠的內容：用 `<table>` 排版整個頁面、用 `<font>` 標籤控制字體、嵌入 Flash 播放影片……這些是「以前」常見的做法。

但現在的教學（包括這個筆記系列）幾乎都直接以 HTML5 為基礎。為什麼？HTML5 到底改變了什麼，讓它成為現在的主流標準？

## 一句話理解

**HTML5 是目前主流的 HTML 標準，相較於舊版本新增了大量語意化標籤、表單功能與多媒體支援，讓網頁開發更貼近現代裝置與使用情境。**

## 概念拆解

### HTML5 的定位

HTML5 是新一代的 HTML 標準，於 2014 年 10 月由萬維網聯盟（W3C）完成標準制定。它的出現，主要是為了補足之前 HTML 版本的不足，新增了：

- 新的標籤
- 新的表單類型
- 新的表單屬性

讓 HTML5 擁有比以前更大的「技術集」，可以做出更多樣化、更強大的網站與應用程式。

### HTML5 新增特性的方向

HTML5 新增的特性可以大致分成幾個方向：

- **語義特性**：新增更多能表達內容角色的標籤（例如之後會學到的 `<header>`、`<nav>`、`<main>`、`<article>` 等），不再只能用 `<div>` 堆疊。
- **本地存儲特性**：讓瀏覽器可以在使用者裝置上儲存資料。
- **設備兼容特性**：讓網頁能更好地配合不同裝置（如手機、平板）。
- **連接特性**：提供更多與伺服器即時互動的能力。
- **網頁多媒體特性**：直接支援音訊、影片，不再依賴外部插件。
- **三維、圖形及特效特性**：支援繪圖、動畫等視覺效果。
- **性能與集成特性**：提升頁面執行效率、整合不同技術。
- **CSS3 特性**：搭配 CSS3 提供更豐富的樣式與動畫能力。

> 對初學者來說，**不需要一次背完這些分類**，重點是知道「HTML5 不只是換個版本號，而是補上了很多以前必須靠額外技巧或外部工具才能做到的功能」。

### 狹義 HTML5 與廣義 HTML5

- **狹義 HTML5**：單指這一代的 HTML 標準本身。
- **廣義 HTML5**：指 HTML5 加上 CSS3、JavaScript，也就是整個「前端」技術的組合，有時會被稱為「HTML5 和它的朋友們」，通常也直接簡稱為 HTML5。

之後當你看到文章或職缺寫「熟悉 HTML5」，多半指的是廣義 HTML5——也就是 HTML、CSS、JavaScript 這一整套前端基礎。

## 從舊做法看 HTML5 帶來的差異

舉兩個對照例子，幫助你感受 HTML5 的改變：

- **版面結構**：以前常見的是把整個頁面都用 `<div class="header">`、`<div class="nav">` 這類「沒有語意」的容器堆出來；HTML5 之後可以用 `<header>`、`<nav>` 等標籤，讓標籤名稱本身就說明這塊內容的角色。
- **多媒體**：以前要在網頁播放影片，常常需要嵌入 Flash 外掛；HTML5 之後可以直接使用 `<video>` 標籤，不需要額外插件。

這兩個例子之後會在對應章節詳細展開，這裡只需要先有「HTML5 讓很多以前要靠額外手段的事情變得內建」的整體印象。

## 實務上怎麼看兼容性

學新特性時，常見的疑問是：「這個功能所有瀏覽器都能用嗎？」實務上可以這樣判斷：

- HTML5 與相關 Web 平台特性的支援度，會因「功能本身」與「瀏覽器版本」而不同，沒有一個「全部都支援」或「全部都不支援」的簡單答案。
- Chrome、Safari、Firefox、Opera 等現代主流瀏覽器，對常用的 HTML5 功能大多有良好支援。
- 如果專案需要兼容比較舊的 IE 瀏覽器，應該針對要用到的功能逐項查詢支援度，必要時準備替代方案（fallback）。
- 如果專案不需要兼容舊瀏覽器，就可以更積極地使用目標瀏覽器已支援的新特性。

開發時的實務做法是：**先專注在常用的新特性上**，不需要一開始就追求用盡所有 HTML5 功能。

## HTML5 的優勢

把前面的內容收斂成幾個實際好處：

- 針對 JavaScript 提供了更多可操作的介面（API）。
- 新增了語意化標籤與一些全域屬性，讓結構更清楚。
- 新增多媒體標籤，能取代過去依賴 Flash 的做法。
- 更強調語意化，對 SEO（搜尋引擎優化）較友善。
- 可移植性好，能大量應用在行動裝置上。

## 常見錯誤與容易混淆處

- **把「HTML5」當成一個獨立、單一的技術**：忽略了實務上常說的 HTML5 其實是廣義 HTML5（HTML + CSS3 + JavaScript）。當你看到職缺或文章要求「HTML5 能力」，通常代表的是這整套前端基礎，而不只是 HTML 標籤本身。
- **不管專案需求，一律用上所有最新特性**：如果目標使用者中還有不少人使用較舊的瀏覽器，這樣做可能導致部分使用者看到的頁面功能缺失或樣式錯亂，卻沒有任何替代方案。
- **誤以為「HTML5 = 所有瀏覽器都支援所有功能」**：實際上支援度是「依功能、依瀏覽器版本」而定的，跳過查詢支援度這一步，等於把兼容性風險留到上線後才發現。

## 重點整理

- HTML5 是 2014 年由 W3C 完成標準制定的新一代 HTML 標準，新增了大量標籤、表單與屬性。
- 新增特性方向包含：語義、本地存儲、設備兼容、連接、多媒體、3D/圖形特效、性能與集成、CSS3。
- 狹義 HTML5 指標準本身；廣義 HTML5 指 HTML5 + CSS3 + JavaScript（前端整體）。
- 兼容性要依「功能 × 瀏覽器版本」判斷，舊瀏覽器需求要逐項查詢並準備 fallback。
- HTML5 優勢：更多 JS 操作介面、語意化標籤、內建多媒體、對 SEO 友善、適合行動裝置。
