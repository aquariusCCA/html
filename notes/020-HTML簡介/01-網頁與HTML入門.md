# 網頁與 HTML 入門：從打開瀏覽器開始

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明「網站」與「網頁」之間的關係。
- 用自己的話解釋 HTML 是什麼，以及「超文本」「標記語言」的意思。
- 知道一份 HTML 檔案是怎麼變成瀏覽器畫面的（解析與渲染的概念）。

## 問題情境

當你打開瀏覽器，輸入一個網址，按下 Enter 之後，畫面上就出現了文字、圖片、按鈕……這個過程中，你其實同時碰到了三個東西：「網站」「網頁」「HTML」。

很多人會把這三者混在一起講：「幫我看一下這個網站」「這個網頁怎麼跑版了」「這份 HTML 寫錯了」。在開始學寫程式碼之前，先把這幾個詞的關係搞清楚，之後溝通與學習才不會卡在一些基本問題上。

## 一句話理解

**網站是一組相關網頁的集合，網頁是用 HTML 描述內容、再由瀏覽器解析並渲染出來給你看的文件。**

## 概念拆解

### 網站與網頁的關係

- **網站（Website）**：在網際網路上，依照一定規則、使用 HTML 等技術做出來、用來呈現特定內容的「一組相關網頁的集合」。例如一個購物網站，裡面會有首頁、商品列表頁、商品詳細頁、購物車頁等很多頁面。
- **網頁（Web Page）**：網站中的「一頁」，通常是一份 HTML 格式的文件，需要透過瀏覽器才能閱讀。
- 網頁是構成網站的基本單位，內容通常由文字、圖片、音訊、影片、超連結等元素組成。
- 這類檔案常以 `.htm` 或 `.html` 結尾，所以也常被稱為「HTML 文件」。

換句話說：**網站是「整個店」，網頁是「店裡的某一個頁面」**。

### HTML：超文本標記語言

HTML 的全名是「超文本標記語言（HyperText Markup Language）」，是用來描述網頁內容的一種語言。這個名字拆開來看，其實藏了兩個關鍵概念：

- **超文本（HyperText）**：
  - 內容不只是純文字，還可以加入圖片、聲音、動畫、影片等多媒體。
  - 可以透過「超連結」從一份文件跳到另一份文件，把世界各地的文件互相連接起來——這正是「超」越單純文本的地方。
- **標記語言（Markup Language）**：
  - HTML **不是一種程式語言**，它不負責「計算」或「邏輯判斷」，而是一套用來「標記」內容的標籤系統。
  - 你可以把它想成：在一份純文字稿上，用各種標籤把段落、標題、圖片、連結「標」出來，讓瀏覽器知道這些內容分別是什麼角色。

### 網頁是怎麼「形成」的

一個網頁從寫程式碼到你看到畫面，大致會經過這三個階段：

1. **前端人員開發代碼**：用 HTML（加上之後會學到的 CSS、JavaScript）寫出網頁的內容與結構。
2. **瀏覽器讀取並處理代碼**：瀏覽器會把這些代碼進行「解析（parse）」與「渲染（render）」。
3. **生成你看到的 Web 頁面**：最終呈現在畫面上的，就是瀏覽器把代碼轉換出來的結果。

## 用「標記」的角度看一段內容

假設你有一段文字：「歡迎來到我的網站」。

如果只是一段純文字，瀏覽器並不知道這段文字在頁面中的角色是什麼——是標題？是一般段落？還是按鈕上的文字？

但如果寫成：

```html
<h1>歡迎來到我的網站</h1>
```

`<h1>` 這個標記就告訴瀏覽器：「這是一個最大層級的標題」。瀏覽器看到這個標記，就知道要用「標題」的樣式（通常是較大、較粗的字）來呈現這段文字，這就是「標記語言」最基本的作用——**用標籤描述內容的角色，而不是直接畫出畫面**。

## 對 HTML 學習的意義

理解這個全貌之後，之後學習各種標籤、屬性、結構時，你會更容易抓住重點：

- 每一個標籤，其實都是在「標記」某一段內容的角色或意義。
- 你寫的每一份 `.html` 檔案，最終都會經過瀏覽器解析、渲染，變成使用者看到的畫面。
- 學 HTML，本質上就是在學「如何用標籤正確描述內容，讓瀏覽器能正確呈現給使用者」。

## 常見錯誤與容易混淆處

- **把「網站」與「網頁」當成同義詞**：例如把「修改首頁的某個按鈕」說成「重做整個網站」，會讓需求範圍被誤解。記住：網站是集合，網頁是其中一頁。
- **以為 HTML 是程式語言，期待它能「計算」或「判斷」**：例如想用 HTML 寫「如果使用者點擊就跳出訊息」，這其實是 JavaScript 的工作。HTML 只負責描述內容與結構。
- **不理解「解析與渲染」，遇到畫面異常時無從下手**：實務上不需要一開始就深入瀏覽器引擎細節，但要建立一個基本認知——**畫面長什麼樣子，是瀏覽器依照你寫的 HTML（與 CSS）解析、渲染出來的結果**，所以排查問題時，第一步永遠是回頭檢查程式碼本身。

## 重點整理

- 網站＝一組相關網頁的集合；網頁＝網站中的一頁，通常是 HTML 文件。
- HTML＝超文本（多媒體＋超連結）＋標記語言（用標籤描述內容角色，不是程式語言）。
- 網頁形成流程：撰寫代碼 → 瀏覽器解析 → 瀏覽器渲染 → 呈現給使用者。

## 自我檢查

1. 如果同事說「幫我看一下這個網站」與「幫我看一下這個網頁」，兩者所指的範圍有什麼不同？
2. 為什麼 HTML 被稱為「標記語言」，而不是「程式語言」？
3. 當你寫好一個 `.html` 檔案並用瀏覽器打開後，畫面上的內容是怎麼出現的？請用自己的話描述這個過程。
