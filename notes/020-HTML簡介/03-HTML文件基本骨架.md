# HTML 文件的基本骨架

## 學習目標

讀完這篇筆記，你應該能夠：

- 寫出一份最基本的 HTML5 文件骨架。
- 說明 `<!DOCTYPE html>`、`<html>`、`<head>`、`<body>` 各自的角色與彼此的層級關係。
- 知道骨架寫錯或漏寫時，可能造成什麼問題。

## 整體結構：先看全貌

網頁其實有點像一篇文章：一篇文章有開頭、正文、落款；一份 HTML 文件也有自己固定的「骨架」結構。每一份 HTML 文件，幾乎都是從同一個骨架開始的：

```html
<!DOCTYPE html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8">
    <title>我的網頁</title>
  </head>
  <body>
    <h1>歡迎來到我的網頁</h1>
  </body>
</html>
```

這份骨架就是之後所有頁面內容書寫的「地基」——不論頁面內容多複雜，最外層的結構基本上都是這樣。

![HTML 文件骨架程式碼示例](../../origin/020-HTML簡介/assets/images/html-skeleton-img-001-8bc9f3.jpg)

## 骨架中的層級關係

把上面的骨架拆成層級關係來看：

```text
<!DOCTYPE html>      ← 文件類型宣告，不是標籤，寫在最前面
<html>                ← 整份文件的根容器
├── <head>           ← 「設定區」：放不會直接顯示在畫面上的資訊
│   ├── <meta>
│   └── <title>
└── <body>           ← 「內容區」：放會顯示在畫面上的內容
    └── <h1>
```

也就是說：`<head>` 和 `<body>` 都是 `<html>` 的子元素，彼此是兄弟關係；`<meta>`、`<title>` 是 `<head>` 的子元素；頁面真正會顯示出來的內容，則寫在 `<body>` 裡面。

## 逐段拆解

### `<!DOCTYPE html>`：告訴瀏覽器用哪種標準解析

這一行不是 HTML 標籤，而是「文件類型宣告」，放在整份文件的最開頭。它的作用是告訴瀏覽器：「請用 HTML5 標準來解析這份文件」。

### `<html>`：整份文件的根容器

`<html>` 是整份 HTML 文件的最外層容器，所有內容（包括 `<head>` 與 `<body>`）都會寫在這個標籤裡面。範例中的 `lang="zh-Hant"` 屬性用來告訴瀏覽器（以及搜尋引擎、螢幕報讀器）這份文件的主要語言是繁體中文。

### `<head>`：頁面的「設定區」

`<head>` 放的是「描述這份文件」的資訊，這些內容**不會直接顯示在頁面畫面上**，例如：

- `<meta charset="UTF-8">`：宣告文件使用的文字編碼，確保中文等字元能正確顯示，不會變成亂碼。
- `<title>我的網頁</title>`：設定瀏覽器分頁標題列上顯示的文字。

### `<body>`：頁面的「內容區」

`<body>` 放的是會實際顯示在畫面上的內容——標題、段落、圖片、連結、表單等，之後學到的大部分標籤都會寫在這裡。

## 常見錯誤

- **漏寫或寫錯 `<!DOCTYPE html>`**：如果文件開頭沒有正確的 `<!DOCTYPE html>` 宣告，瀏覽器可能會以「相容模式（quirks mode）」解析這份文件，導致樣式、版面出現預期之外的差異，而這種問題通常不容易從畫面上一眼看出原因。**避免方法**：每份 HTML5 文件的第一行，固定寫 `<!DOCTYPE html>`。
- **把內容寫到 `<head>` 裡，或寫在 `<html>` 標籤外面**：例如把一段文字直接寫在 `<head>` 裡，瀏覽器通常不會把它當作頁面內容顯示；寫在 `<html>` 標籤外面的內容，瀏覽器多數情況會自動「修正」位置，但這種寫法容易造成結構混亂，難以維護。**避免方法**：所有要顯示在畫面上的內容，一律寫在 `<body>` 內。
- **`<head>` 與 `<body>` 順序顛倒或巢狀錯誤**：例如把 `<head>` 放在 `<body>` 之後，雖然瀏覽器多半還是會嘗試解析，但這違反了文件結構的基本假設，也會讓其他開發者難以閱讀。**避免方法**：固定按照「`<head>` 在前、`<body>` 在後」的順序書寫。

## 實務上的意義

- 這份骨架幾乎是每一個 `.html` 檔案的固定起點，建議直接記下來，作為之後每次新建頁面的範本。
- 之後學到的標籤、屬性，絕大多數都是寫在 `<body>` 裡面，理解骨架可以幫助你快速判斷「這段程式碼應該放在哪裡」。
- 打開瀏覽器的開發者工具（DevTools）查看「Elements」面板時，看到的就是這個骨架被瀏覽器解析後的結構，理解骨架有助於之後讀懂這個面板。
- HTML 文件通常以 `.html` 或 `.htm` 為副檔名；在伺服器環境中，瀏覽器主要依回應的內容類型（例如 `text/html`）來判斷該如何處理這份檔案，但作為初學者，記住「`.html` 檔案會被瀏覽器當作網頁來顯示」即可。

## 重點整理

- 每份 HTML5 文件都以 `<!DOCTYPE html>` 開頭，告訴瀏覽器用 HTML5 標準解析。
- `<html>` 是根容器，內含 `<head>` 與 `<body>` 兩個子元素。
- `<head>` 放不會顯示在畫面上的設定資訊（如編碼、標題）；`<body>` 放會顯示的內容。
- 漏寫 `<!DOCTYPE html>` 或把內容放錯位置，都可能導致畫面顯示異常或結構混亂。

## 自我檢查

2. `<head>` 與 `<body>` 分別負責放什麼樣的內容？
3. 如果你打開一個網頁，發現瀏覽器分頁標題列顯示的文字跟你預期的不一樣，這可能跟哪個標籤有關？
