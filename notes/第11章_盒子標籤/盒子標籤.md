# HTML 盒子標籤

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `div` 與 `span` 的基本用途。
- 理解 `div` 與 `span` 都是沒有語意的容器元素。
- 分辨 `div` 預設獨佔一行、`span` 預設可與其他內容排在同一行的差異。
- 在需要包住內容、套用 CSS 或供 JavaScript 操作時，正確選擇 `div` 或 `span`。
- 避免在有更合適語意元素時，全部都使用 `div` 或 `span`。

## 2. 本章定位

本章接續「塊級元素與行內元素」以及「段落標籤」之後，專門說明 HTML 中最常見的兩個無語意容器：`div` 與 `span`。

`div` 與 `span` 常被初學者稱為「盒子標籤」，因為它們本身沒有特定內容語意，主要用來包住一段內容，方便後續用 CSS 排版、套樣式，或讓 JavaScript 找到並操作這些內容。

> 來源重點：原始資料指出，`div` 和 `span` 沒有語意，都是用來裝內容的盒子；`div` 通常一行只能放一個，`span` 一行可以放多個。

## 3. 核心概念

### 3.1 `div` 與 `span` 都是無語意容器

`div` 與 `span` 的共同特點是：它們本身不表示標題、段落、連結、圖片、清單或表單控制項等具體語意。

它們的主要用途是「包住內容」：

- 讓一組內容形成一個可被排版的區塊。
- 讓某一小段文字可以被單獨套用樣式。
- 讓 CSS 或 JavaScript 可以透過 `class`、`id` 等屬性選取特定範圍。

```html
<div class="card">
  <h2>HTML 基礎課程</h2>
  <p>適合剛開始學習網頁結構的初學者。</p>
</div>

<p>課程狀態：<span class="status">開放報名</span></p>
```

在這個例子中，`div` 包住一整組卡片內容；`span` 則只包住段落中的「開放報名」這一小段文字。

### 3.2 `div` 是區塊型容器

`div` 是 division 的縮寫，有分割、分區的意思。

在預設情況下，`div` 是區塊型元素。它通常會從新的一行開始，並且在一般排版流中佔滿父元素可用寬度。因此，多個 `div` 連續出現時，通常會由上到下排列。

```html
<div>我是一個 div 標籤，我通常會獨佔一行。</div>
<div>我也是一個 div 標籤，會排在下一行。</div>
```

`div` 適合用來包住一組較大的內容，例如頁面區塊、卡片、表單區域、導覽容器或版面中的某個區域。

> 注意：`div` 沒有語意。若內容其實是標題、段落、導覽、文章、頁首或頁尾，應優先考慮 `h1` 到 `h6`、`p`、`nav`、`article`、`header`、`footer` 等更有語意的元素。

### 3.3 `span` 是行內型容器

`span` 有跨度、跨距的意思。

在預設情況下，`span` 是行內元素。它不會主動換到新的一行，寬度通常由內容本身決定。因此，多個 `span` 連續出現時，通常可以排在同一行。

```html
<span>HTML</span>
<span>CSS</span>
<span>JavaScript</span>
```

`span` 適合用來包住一句話或一段文字中的小範圍內容，例如狀態文字、價格、關鍵字或需要特別標示的一小段文字。

## 4. 語法與基本用法

### 4.1 `div` 的基本語法

```html
<div>這是一個區塊容器。</div>
```

其中：

| 部分 | 說明 |
|---|---|
| `<div>` | `div` 開始標籤 |
| `這是一個區塊容器。` | 被 `div` 包住的內容 |
| `</div>` | `div` 結束標籤 |

常見用法是替 `div` 加上 `class`，讓 CSS 可以選取並設定樣式：

```html
<div class="product-card">
  <h2>無線滑鼠</h2>
  <p>適合日常辦公與學習使用。</p>
</div>
```

### 4.2 `span` 的基本語法

```html
<span>這是一小段行內內容。</span>
```

其中：

| 部分 | 說明 |
|---|---|
| `<span>` | `span` 開始標籤 |
| `這是一小段行內內容。` | 被 `span` 包住的內容 |
| `</span>` | `span` 結束標籤 |

常見用法是把段落中的某一小段內容包起來，再用 CSS 做局部樣式：

```html
<p>
  訂單狀態：
  <span class="status">已出貨</span>
</p>
```

### 4.3 `div` 與 `span` 的差異

| 元素 | 預設顯示方式 | 是否有語意 | 適合用途 |
|---|---|---|---|
| `div` | 區塊型，通常獨佔一行 | 無語意 | 包住一組區塊內容或建立通用版面容器 |
| `span` | 行內型，通常可與其他內容同一行 | 無語意 | 包住文字中的一小段內容 |

`div` 與 `span` 的差異主要在預設顯示方式與使用範圍，不在語意強弱。兩者都沒有明確語意，因此不應取代更精準的 HTML 元素。

## 5. 範例說明

### 範例一：比較 `div` 與 `span` 的預設排列

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>div 與 span 範例</title>
</head>
<body>
  <div>我是一個 div 標籤，通常會獨佔一行。</div>
  <div>我也是一個 div 標籤，通常會排在下一行。</div>

  <span>百度</span>
  <span>新浪</span>
  <span>搜狐</span>
</body>
</html>
```

這個範例中：

- 兩個 `div` 預設會由上到下排列。
- 三個 `span` 預設可以在同一行排列。
- `div` 與 `span` 的畫面差異來自瀏覽器預設樣式與元素顯示方式，不代表它們有不同的內容語意。

### 範例二：用 `div` 包住一組卡片內容

```html
<div class="course-card">
  <h2>HTML 入門</h2>
  <p>學習網頁內容結構與常見標籤用法。</p>
  <a href="/courses/html">查看課程</a>
</div>
```

這裡的 `div` 只是通用容器，用來包住一張課程卡片中的標題、描述與連結。真正有語意的是內部的 `h2`、`p` 與 `a`。

### 範例三：用 `span` 標記文字中的局部內容

```html
<p>
  本月優惠價：
  <span class="price">NT$ 990</span>
</p>
```

這裡的 `span` 只包住價格文字，方便 CSS 對價格設定顏色、粗細或其他樣式。整段文字仍然使用 `p` 表示段落。

## 6. 實務應用場景

`div` 常見於需要包住一組內容的情境：

- 建立卡片、列表項目、表單區塊等通用容器。
- 包住一段頁面區域，方便 CSS 做版面配置。
- 作為 JavaScript 操作某組內容的外層節點。
- 在沒有更合適語意元素時，作為中性的結構容器。

`span` 常見於需要標記一小段行內內容的情境：

- 標示狀態文字，例如「成功」、「失敗」、「處理中」。
- 標示價格、關鍵字、數字或單位。
- 在一段文字中套用局部樣式。
- 讓 JavaScript 只更新一句話中的某個小範圍。

實務上，`div` 和 `span` 很常搭配 `class` 使用：

```html
<div class="alert">
  <strong>提醒：</strong>
  <span class="message">請先完成必填欄位。</span>
</div>
```

CSS 可以選取 `.alert` 控制整個提示區塊，也可以選取 `.message` 控制訊息文字。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 所有內容都用 `div` 包起來 | `div` 沒有語意，會讓文件結構不清楚 | 有標題、段落、導覽、文章等語意時，優先使用對應元素 |
| 用 `span` 包住一大段區塊內容 | `span` 是行內容器，不適合承載整組區塊結構 | 改用 `div` 或更合適的語意元素 |
| 把 `div` 當成段落使用 | 段落文字應有段落語意 | 一般文字段落使用 `p` |
| 用 `div` 或 `span` 假裝按鈕 | 無語意元素缺少原生互動行為與可訪問性支援 | 使用 `button` 或適合的互動元素 |
| 只根據預設樣式選標籤 | HTML 應先描述內容語意，不是只追求畫面效果 | 先選語意正確的元素，再用 CSS 控制外觀 |

### 錯誤範例：用 `div` 取代標題與段落

```html
<div class="title">HTML 入門</div>
<div class="text">HTML 是用來描述網頁內容與結構的標記語言。</div>
```

問題：這段內容其實包含標題與段落，但全部使用 `div`，會讓 HTML 結構缺少語意。

### 正確範例：使用語意元素，再視需要搭配 `div`

```html
<div class="lesson-summary">
  <h2>HTML 入門</h2>
  <p>HTML 是用來描述網頁內容與結構的標記語言。</p>
</div>
```

修正後，外層 `div` 只負責包住這組內容；內部的標題與段落則使用更有語意的 `h2` 與 `p`。

### 錯誤範例：用 `span` 包住區塊結構

```html
<span class="card">
  <h2>會員方案</h2>
  <p>適合想要完整學習前端基礎的學生。</p>
</span>
```

問題：`span` 是行內容器，不適合包住標題與段落這類區塊內容。

### 正確範例：使用 `div` 作為通用區塊容器

```html
<div class="card">
  <h2>會員方案</h2>
  <p>適合想要完整學習前端基礎的學生。</p>
</div>
```

這裡改用 `div` 包住整組區塊內容，結構更合理。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| 塊級元素與行內元素 | `div` 預設是區塊型容器，`span` 預設是行內型容器 |
| 語意化 HTML | `div` 與 `span` 沒有語意，有更合適元素時應優先使用語意元素 |
| CSS | `div` 與 `span` 常搭配 `class` 或 `id`，作為 CSS 選擇器的目標 |
| DOM | 瀏覽器會把 `div` 與 `span` 解析成 DOM 節點，供 JavaScript 操作 |
| JavaScript | JavaScript 可選取 `div` 或 `span`，更新某個區塊或局部文字內容 |
| 可訪問性 | 濫用無語意元素可能讓輔助工具難以理解內容角色 |

## 9. 重點整理

- `div` 和 `span` 都是沒有語意的 HTML 容器元素。
- `div` 是 division 的縮寫，常用來包住一組區塊內容。
- `span` 表示一小段行內範圍，常用來包住文字中的局部內容。
- `div` 預設通常會獨佔一行，`span` 預設通常可以與其他行內內容排在同一行。
- `div` 和 `span` 常搭配 `class`、CSS 與 JavaScript 使用。
- 若有更合適的語意元素，應優先使用語意元素，而不是全部都用 `div` 或 `span`。

## 11. 延伸補充

> 補充說明：HTML 負責描述內容與結構，CSS 負責控制視覺呈現。`div` 和 `span` 常被拿來配合 CSS 排版或套樣式，但它們不應取代正確的語意元素。

瀏覽器會根據 HTML 建立 DOM Tree。當你寫下 `div` 或 `span` 時，它們會成為 DOM 中的元素節點，CSS 可以對這些節點套用樣式，JavaScript 也可以取得這些節點並修改內容。

例如：

```html
<p>目前人數：<span id="count">0</span></p>
```

```js
document.querySelector("#count").textContent = "10";
```

這個例子中，`span` 只包住需要被 JavaScript 更新的數字，因此比用 `div` 包整段文字更精準。
