# HTML 超鏈接標籤 a

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `<a>` 標籤在 HTML 中的用途與語意。
- 使用 `href` 建立外部連結、內部連結、檔案連結與錨點連結。
- 理解 `target="_self"` 與 `target="_blank"` 的差異。
- 使用 `tel:`、`mailto:` 等 URL scheme 喚起指定應用。
- 分辨空連結、回到頁首、JavaScript 連結與真正導航連結的差異。
- 避免連結文字不清楚、`target="_blank"` 安全性不足、濫用 `javascript:` 等常見問題。

## 2. 本章定位

本章介紹 HTML 中用來建立超鏈接的 `<a>` 元素。

連結是網頁最核心的能力之一。使用者可以透過連結從目前頁面跳到其他頁面、頁面中的特定位置、圖片、PDF、下載檔案，甚至喚起電話、郵件或特定應用程式。

`<a>` 不只是讓文字「可以點」。它代表一個可導航的目的地，因此連結文字、目標位置、開啟方式與可訪問性都會影響使用者體驗。

> 來源重點：原始資料指出 `<a>` 是 anchor 的縮寫，主要作用是從目前頁面進行跳轉，並整理了外部連結、內部連結、檔案連結、錨點連結與應用喚起等用法。

## 3. 核心概念

`<a>` 是 anchor 的縮寫，意思是「錨」。在 HTML 中，`<a>` 用來建立超鏈接，通常搭配 `href` 屬性指定連結目標。

基本語法如下：

```html
<a href="目標地址">連結文字</a>
```

其中：

| 部分 | 說明 |
|---|---|
| `<a>` | 建立超鏈接的元素 |
| `href` | 指定連結目標 |
| 連結文字 | 使用者看到並點擊的文字 |
| `</a>` | 結束標籤 |

例如：

```html
<a href="https://www.baidu.com/">百度</a>
```

這段程式碼會建立一個指向外部網站的連結，使用者點擊「百度」後，瀏覽器會前往 `href` 指定的網址。

> 補充說明：HTML 負責描述「這裡有一個連結」。連結的樣式，例如顏色、底線或 hover 效果，通常交給 CSS 控制。

## 4. 語法與基本用法

### 4.1 `href`：指定連結目標

`href` 是 hypertext reference 的縮寫，用來指定連結要前往的位置。

```html
<a href="https://example.com">前往 Example</a>
```

常見 `href` 類型如下：

| 類型 | 範例 | 說明 |
|---|---|---|
| 外部網址 | `https://example.com` | 前往其他網站 |
| 內部頁面 | `./about.html` | 前往同一網站中的其他頁面 |
| 檔案資源 | `./assets/pdfs/demo.pdf` | 開啟或下載檔案 |
| 頁面錨點 | `#section-2` | 跳到目前頁面的指定位置 |
| 其他頁面錨點 | `./page.html#section-2` | 前往其他頁面中的指定位置 |
| 電話 | `tel:10010` | 嘗試喚起撥號應用 |
| 電子郵件 | `mailto:service@example.com` | 嘗試喚起郵件應用 |

### 4.2 `target`：指定開啟方式

`target` 用來指定連結開啟的位置。

```html
<a href="https://example.com" target="_blank">在新分頁開啟</a>
```

常見值如下：

| `target` 值 | 說明 |
|---|---|
| `_self` | 在目前頁面開啟，這是預設值 |
| `_blank` | 在新分頁或新視窗開啟 |

如果沒有特別指定，連結會以 `_self` 的方式在目前頁面開啟。

> 注意：實務上使用 `target="_blank"` 開啟外部網站時，建議加上 `rel="noopener noreferrer"`，降低新頁面影響原頁面的風險。

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  前往 Example
</a>
```

### 4.3 `<a>` 可以包裹其他元素

原始資料提到：雖然 `<a>` 常被當成行內元素使用，但它也可以包裹其他元素。

```html
<a href="https://example.com">
  <div>這是一個包含在 a 標籤中的區塊元素</div>
</a>
```

這種寫法常見於整張卡片都可點擊的設計，例如商品卡、文章卡或功能入口卡。

> 注意：`<a>` 不應包住另一個互動元素，例如另一個 `<a>`、`button` 或表單控制項。互動元素互相巢狀會造成操作與可訪問性問題。

## 5. 範例說明

### 範例一：外部連結

```html
<h4>外部連結</h4>
<a href="https://www.qq.com" target="_self">騰訊</a>
<a href="https://www.itcast.cn" target="_blank" rel="noopener noreferrer">傳智播客</a>
```

這個範例中：

- 第一個連結會在目前頁面開啟。
- 第二個連結會在新分頁或新視窗開啟。
- `rel="noopener noreferrer"` 是 `target="_blank"` 的實務安全補充。

### 範例二：內部連結

```html
<body>
  <h4>內部連結</h4>
  <a href="./about.html" target="_blank">公司簡介</a>
</body>
```

內部連結通常指向同一網站中的其他頁面。這類連結常用相對路徑，例如 `./about.html`。

### 範例三：跳轉到檔案

```html
<!-- 瀏覽器能直接開啟的檔案 -->
<a href="./assets/images/photo.jpg">看圖片</a>
<a href="./assets/files/movie.mp4">看影片</a>
<a href="./assets/files/music.mp3">聽音樂</a>
<a href="./assets/pdfs/test.pdf">看 PDF 檔</a>

<!-- 瀏覽器不能直接開啟的檔案，通常會觸發下載 -->
<a href="./assets/files/demo.zip">下載範例壓縮檔</a>

<!-- 強制提示下載 -->
<a href="./assets/pdfs/test.pdf" download="一份PDF檔.pdf">下載 PDF 檔案</a>
```

瀏覽器能否直接開啟檔案，取決於檔案類型、瀏覽器能力與伺服器回應設定。圖片、影片、音訊與 PDF 通常可在瀏覽器中直接開啟；壓縮檔等格式通常會下載。

`download` 屬性可以提示瀏覽器將資源下載下來，而不是直接導航到該資源。

> 注意：`download` 是屬性名稱，原始資料中的 `dowload` 是拼字錯誤，正式寫法應為 `download`。

### 範例四：跳到本頁頂部或指定位置

```html
<div style="height: 2000px;">空盒子</div>

<!-- 跳到本頁的頂部 -->
<a href="#">回到頂部</a>
```

`href="#"` 通常會讓頁面跳到頂部，因為它代表目前頁面的片段識別位置。

更常見也更有語意的做法，是建立明確的錨點：

```html
<a href="#season-2">第二季</a>

<h3 id="season-2">第二季介紹</h3>
```

當使用者點擊「第二季」時，瀏覽器會尋找 `id="season-2"` 的元素，並將畫面捲動到該位置。

也可以跳到其他頁面的指定錨點：

```html
<a href="./test.html#intro">跳轉到測試頁面的 intro 區塊</a>
```

### 範例五：空連結

```html
<h4>空連結</h4>
<a href="">刷新本頁面</a>
```

空的 `href=""` 通常會指向目前頁面，可能造成頁面重新載入。

> 注意：若只是暫時沒有連結目標，不建議長期保留空連結。可以先不使用 `<a>`，或在互動需求明確時改用更適合的元素，例如 `<button>`。

### 範例六：執行 JavaScript

```html
<h4>執行 JS</h4>
<a href="javascript:alert('Hello, World!');">點擊我</a>
```

這種寫法可以透過 `javascript:` URL 執行 JavaScript，但實務上不建議作為一般互動寫法。

較好的做法是把連結與行為分開。如果動作不是「導航到某個地方」，通常應使用 `<button>`：

```html
<button type="button" id="show-message">點擊我</button>

<script>
  document.querySelector('#show-message').addEventListener('click', () => {
    alert('Hello, World!');
  });
</script>
```

這樣語意更清楚，也比較容易維護。

### 範例七：喚起指定應用

```html
<!-- 喚起設備撥號 -->
<a href="tel:10010">電話聯繫</a>

<!-- 喚起設備發送郵件 -->
<a href="mailto:10010@qq.com">郵件聯繫</a>

<!-- 喚起 LINE -->
<a href="line://msg/text/Hello%20World">使用 LINE 發送消息</a>
```

`tel:` 與 `mailto:` 是常見的 URL scheme，常用於手機撥號與郵件撰寫。

`line://` 這類應用程式專用 scheme 是否能正常運作，取決於使用者裝置是否安裝對應應用、瀏覽器是否允許喚起，以及應用本身的支援狀況。

### 範例八：網頁元素連結

```html
<h4>網頁元素的連結</h4>
<a href="https://www.baidu.com">
  <img src="./assets/images/example.jpg" alt="前往百度">
</a>
```

`<a>` 可以包裹圖片，讓圖片成為可點擊的連結。這種情境下，圖片的 `alt` 應描述連結目的或圖片意義，不能只寫檔名。

## 6. 實務應用場景

`<a>` 在實務開發中常用於：

- 網站導覽列，例如首頁、關於我們、產品介紹。
- 文章或文件中的延伸閱讀連結。
- 後台系統中的列表詳情頁入口。
- 下載 PDF、ZIP、CSV 或其他附件。
- 跳到頁面中的指定區塊，例如常見問題、章節目錄。
- 手機網頁中的撥號、寄信與開啟應用程式。
- 圖片、卡片、Logo 等可點擊元素。

設計連結時，應優先讓使用者在點擊前知道會前往哪裡。好的連結文字應描述目的，例如「查看公司簡介」比「點我」更清楚。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 連結文字只寫「點我」 | 使用者與輔助工具無法從文字判斷目的 | 改成「查看公司簡介」、「下載 PDF 講義」等具體文字 |
| `href` 路徑寫錯 | 瀏覽器找不到目標頁面或檔案 | 確認相對路徑、檔名與資料夾位置 |
| 使用 `target="_blank"` 但沒有 `rel` | 新開頁面可能影響原頁面，且來源資訊可能被帶出 | 加上 `rel="noopener noreferrer"` |
| 把非導航動作用 `<a>` 寫 | `<a>` 的語意是連結，不是一般按鈕 | 不是前往某處的動作，優先使用 `<button type="button">` |
| 長期保留 `href=""` | 可能重新載入目前頁面，造成使用者困惑 | 有明確目標再使用 `<a>`，暫時互動可改用 `button` |
| 濫用 `href="#"` | 點擊後可能跳到頁首，也可能干擾歷史紀錄或捲動位置 | 建立明確錨點，或使用適合的按鈕互動 |
| 錨點連結找不到對應 `id` | `href="#xxx"` 必須對應頁面中的 `id="xxx"` | 確認片段名稱一致且不重複 |
| `mailto:` 中加入多餘空白 | URL scheme 格式不正確，可能無法喚起郵件應用 | 使用 `mailto:user@example.com` |
| 把互動元素放進 `<a>` 裡 | 互動元素巢狀會造成操作與可訪問性問題 | 避免在 `<a>` 中放另一個 `<a>`、`button` 或表單控制項 |
| 用圖片當連結但沒有合適 `alt` | 螢幕閱讀器無法理解連結目的 | 讓 `alt` 描述連結目的或圖片內容 |

### 錯誤範例：用連結假裝按鈕

```html
<a href="#" onclick="submitForm()">送出表單</a>
```

問題：這個動作本質上不是前往某個 URL，而是觸發行為。使用 `<a>` 會讓語意不清楚，`href="#"` 也可能造成頁面跳動。

### 正確範例：使用按鈕處理動作

```html
<button type="button" onclick="submitForm()">送出表單</button>
```

修正點：`button` 才是執行動作的原生互動元素，語意與鍵盤操作都更符合預期。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| URL 與路徑 | `href` 需要指定正確 URL、相對路徑或片段識別符 |
| `id` 屬性 | 錨點連結透過 `href="#xxx"` 對應目標元素的 `id="xxx"` |
| `<img>` | 圖片可以放在 `<a>` 中，形成圖片連結 |
| `<button>` | `<a>` 用於導航，`button` 用於觸發動作，兩者語意不同 |
| CSS | CSS 可控制連結顏色、底線、hover 與 focus 狀態 |
| JavaScript | JavaScript 可攔截連結點擊或處理按鈕互動，但不應破壞原本語意 |
| 可訪問性 | 連結文字、focus 狀態與圖片 `alt` 會影響鍵盤與輔助工具使用者 |
| SEO | 清楚的連結文字與合理網站連結結構，有助於搜尋引擎理解頁面關係 |

## 9. 重點整理

- `<a>` 是 HTML 中用來建立超鏈接的元素。
- `href` 指定連結目標，可以是外部網址、內部頁面、檔案、錨點或應用 scheme。
- `target="_self"` 是預設開啟方式；`target="_blank"` 會嘗試在新分頁或新視窗開啟。
- 使用 `target="_blank"` 連到外部網站時，建議加上 `rel="noopener noreferrer"`。
- 錨點連結需要 `href="#名稱"` 與目標元素的 `id="名稱"` 對應。
- 如果互動不是導航，而是執行某個動作，通常應使用 `<button>`。
- 連結文字應清楚描述目的，不要只寫「點我」或「下載」。
- 圖片連結也需要合適的 `alt`，讓使用者理解圖片或連結目的。

## 11. 延伸補充

> 補充說明：在 HTML5 中，`<a>` 的內容模型比早期 HTML 更彈性，可以包裹許多 flow content，因此常見於整張卡片可點擊的設計。不過它仍然不應包住其他互動元素，否則會造成使用者操作、鍵盤焦點與輔助工具理解上的問題。

> 補充說明：`tel:`、`mailto:` 與 `line://` 這類連結不保證在所有環境都能成功喚起應用。實務上應考慮桌機、手機、不同瀏覽器與是否安裝應用程式的差異。
