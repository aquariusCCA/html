# 常見錯誤巢狀：標題、p 與 a 標籤

## 學習目標

讀完這篇筆記，你應該能夠：

- 避免把 `h1` 到 `h6` 標題元素互相包起來。
- 知道 `p` 標籤不能包含 `div` 這類區塊內容。
- 理解 `a` 標籤雖然彈性高，但不能包含另一個 `a` 或互動內容。
- 看懂瀏覽器自動修正錯誤巢狀後，DOM 為什麼可能和原始碼不同。

## 問題情境

很多 HTML 錯誤不是語法打錯，而是「包錯」。例如：

- 想做大標題和小標題，所以把 `h2` 放進 `h1`。
- 想讓段落裡有一整塊內容，所以把 `div` 放進 `p`。
- 想讓整張卡片可以點擊，所以用 `a` 包住很多內容，卻又在裡面放另一個連結或按鈕。

這些寫法可能看起來能顯示，但瀏覽器會依 HTML 解析規則自動修正結構。結果是：畫面可能還在，DOM 卻不一定是你原本以為的樣子。

## 一句話理解

HTML 巢狀不是看「能不能顯示」，而是看元素語意與內容模型是否允許這樣包。

## 錯誤一：標題元素不可互相嵌套

`h1` 到 `h6` 是標題元素，用來表示內容層級。標題之間應該是兄弟關係或位於不同章節中，而不是互相包住。

錯誤寫法：

```html
<h1>
  尚硅谷1
  <h2>尚硅谷2</h2>
</h1>
```

這段程式碼的問題是：`h1` 本身代表一個標題，`h2` 也代表另一個標題。把 `h2` 放進 `h1`，會讓標題層級變成錯誤的巢狀結構。

瀏覽器解析後可能會自動結束前面的標題元素，使實際 DOM 和你看到的原始碼不一致。

![h1 與 h2 巢狀的瀏覽器解析結果](../../origin/070-HTML塊級元素與行內元素/assets/images/html-block-inline-elements-img-001-85796c.png)

較合適的寫法是讓標題各自獨立：

```html
<h1>尚硅谷1</h1>
<h2>尚硅谷2</h2>
```

如果 `h2` 是 `h1` 底下某一節的標題，它們在文件結構上可以有層級關係，但在 HTML 原始碼中不應該互相嵌套。

## 錯誤二：`p` 標籤不可包含塊級元素

`p` 代表段落，適合放段落中的文字與 phrasing content。它不適合包住 `div` 這類區塊內容。

錯誤寫法：

```html
<p>
  <div>尚硅谷</div>
</p>
```

這段程式碼的問題是：`div` 是通用區塊容器，不屬於 `p` 應該包含的段落內內容。瀏覽器遇到這種結構時，通常會自動把 `p` 提前結束，造成實際 DOM 和原始碼不同。

![p 內放置 div 的瀏覽器解析結果](../../origin/070-HTML塊級元素與行內元素/assets/images/html-block-inline-elements-img-002-fda844.png)

較合適的寫法要看你的內容語意：

```html
<p>這是一段段落文字。</p>

<div>
  <p>這是一個區塊中的段落。</p>
</div>
```

如果你需要一整塊版面容器，用 `div` 包 `p`；不要反過來用 `p` 包 `div`。

## 錯誤三：`a` 標籤不能無限制巢狀

`a` 標籤比較特殊。它使用 transparent content model，所以在某些情況下可以包住文字，也可以包住一般區塊內容。

例如下面這種「整塊內容可點擊」的寫法，在符合限制時可以成立：

```html
<a href="https://example.com">
  <div>這是一個包含在 a 標籤中的區塊元素</div>
</a>
```

但 `a` 不是什麼都能包。它不能包含：

- 另一個 `a`
- `button` 等互動內容
- 表單控制項等互動元素
- 帶有 `tabindex` 的後代元素

錯誤寫法：

```html
<a href="https://example.com">
  <a href="https://example.com/about">這是一個超連結</a>
</a>
```

這種巢狀會讓使用者互動目標混亂，也不符合 HTML 規範。瀏覽器可能會修正結構，但結果不應該依賴瀏覽器猜測。

如果卡片內還需要放按鈕或另一個連結，通常應該重新設計結構，例如只讓標題或圖片成為連結，或把外層改成非連結容器。

## 三種錯誤的實務判斷表

| 情境 | 錯誤寫法 | 建議判斷 |
| --- | --- | --- |
| 大標題裡放小標題 | `h1` 包 `h2` | 標題應各自獨立，靠文件順序與層級表達關係 |
| 段落裡放區塊 | `p` 包 `div` | 用 `div` 包 `p`，或拆成多個段落與區塊 |
| 連結裡再放連結 | `a` 包 `a` | 不要巢狀互動元素，重新拆分可點擊區域 |

## 常見錯誤

### 錯誤一：以為畫面正常就代表 HTML 合法

瀏覽器非常容錯，會嘗試把錯誤 HTML 修成可顯示的 DOM。但這會讓你的原始碼和實際 DOM 不一致。

這種問題在初期可能只看到畫面有點怪，後面可能變成 CSS 選不到元素、JavaScript 事件綁錯位置、螢幕報讀器讀出奇怪結構。

### 錯誤二：把視覺層級寫成巢狀

`h2` 是 `h1` 底下的次級標題，這是內容層級，不代表 `h2` 要寫在 `h1` 裡面。

HTML 常常用「順序」和「語意標籤」表達結構，不是所有父子關係都要用標籤巢狀表示。

### 錯誤三：讓整張卡片可點擊時忽略互動元素限制

用 `a` 包住一整張卡片很常見，但如果卡片裡還有按鈕、表單欄位或另一個連結，就會形成互動元素巢狀問題。

這時要重新拆分互動區域，而不是把所有東西塞進同一個 `a`。

## 實務寫法建議

- 標題用 `h1` 到 `h6` 表示層級，但不要互相包住。
- 段落文字放在 `p` 裡；區塊容器放在 `p` 外。
- `a` 可以包一般文字或某些區塊內容，但不要包另一個 `a` 或互動內容。
- 遇到瀏覽器自動修正 DOM 的情況，要回頭修正原始 HTML，不要依賴瀏覽器容錯。

## 重點整理

- `h1` 到 `h6` 不應互相嵌套。
- `p` 不應包含 `div` 等區塊內容。
- `a` 有 transparent content model，但不能包含另一個 `a`、互動內容或帶 `tabindex` 的後代。
- 錯誤巢狀可能讓瀏覽器自動改寫 DOM。
- 合法 HTML 結構比「畫面剛好能顯示」更重要。
