# 塊級元素與行內元素的排版差異

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨塊級元素與行內元素在預設排版中的差異。
- 理解「獨佔一行」與「跟文字一起流動」代表什麼。
- 知道 `width`、`height`、`margin`、`padding` 對不同類型元素的影響不完全相同。
- 避免把 block、inline 當成絕對不變的死規則。

## 問題情境

當你在 HTML 裡寫下這些元素時：

```html
<h1>標題</h1>
<p>段落文字</p>
<div>區塊內容</div>

<input type="text">
<span>補充文字</span>
```

瀏覽器預設會讓 `h1`、`p`、`div` 一個接一個往下排；但 `input` 和 `span` 通常會留在同一行，直到空間不夠才換行。

這就是學習塊級元素與行內元素的第一個目的：你需要先知道元素在「正常文件流」中預設怎麼排列，後面學 CSS 版面、盒模型、巢狀規則時才不會混亂。

## 一句話理解

塊級元素偏向建立一整塊內容區域；行內元素偏向在一行文字中包住某個片段或插入某個小元件。

## 塊級元素：建立區塊

塊級元素在預設排版中通常會產生區塊盒。常見行為包括：

- 多數常見塊級元素會在正常文件流中另起一行顯示。
- 多個塊級元素預設會由上往下排列。
- 可以設定 `width`、`height`、`margin`、`padding` 等盒模型屬性。
- 如果沒有設定 `width`，常見的 block non-replaced 元素會用 `width: auto` 填滿可用水平空間。

常見塊級元素包括：

- `div`
- `p`
- `h1` 到 `h6`
- `ul`、`ol`、`li`
- `dl`、`dt`、`dd`
- `form`
- `table`
- `hr`
- `blockquote`
- `address`
- `menu`
- `pre`

要注意的是，`width: auto` 會填滿可用空間，但它不完全等於 `width: 100%`。例如元素有 margin、border、padding 時，兩者在盒模型計算上可能有差異。

## 行內元素：跟文字一起流動

行內元素在預設排版中通常會和文字一起排在同一行。常見行為包括：

- 不會預設獨佔一整行。
- 相鄰行內元素會排在同一行，直到空間不夠才換行。
- 一般 non-replaced inline 元素的寬度由內容決定。
- 一般 `span`、`em` 這類 inline 元素，不能直接用 `width`、`height` 改變盒子的主要尺寸。
- 左右 `margin`、`padding` 會影響水平排列。
- 上下 `padding`、`border` 會被繪製，但通常不會像 block 元素那樣撐高整個行盒。
- 上下 `margin` 對行盒高度沒有一般 block 元素那樣的效果。
- `line-height` 會影響行內盒參與行盒高度計算。

常見行內或 phrasing 類元素包括：

- `span`
- `em`
- `i`
- `b`
- `strong`
- `a`
- `img`
- `input`
- `br`
- `select`
- `textarea`
- `q`
- `bdo`
- `sub`
- `sup`

這裡要特別小心：`img`、部分表單控制項，以及被設定成 `inline-block` 的元素，可以有可控尺寸。不要把「行內元素」簡化成「一定不能設定寬高」。

## 範例對照

### 塊級元素會往下排列

```html
<body>
  <h1>尚硅谷</h1>
  <p>尚硅谷</p>
  <div>尚硅谷</div>
</body>
```

這段 HTML 中，`h1`、`p`、`div` 都是常見塊級元素。瀏覽器預設會把它們排成上下三塊，而不是擠在同一行。

### 行內元素會留在同一行

```html
<input type="text">
<span>尚硅谷</span>
```

`input` 和 `span` 在預設排版中會和行內內容一起流動。只要同一行還有空間，兩者通常會排在同一行。

## 實務判斷

寫 HTML 時，不需要把每個元素硬背成「塊級」或「行內」才敢使用。比較實用的判斷方式是：

| 使用情境 | 常見選擇 |
| --- | --- |
| 要建立文章段落、標題、清單、表單區塊 | 多半使用塊級元素 |
| 要包住一段文字中的某個片段 | 多半使用行內元素 |
| 要建立無語意的版面容器 | 常用 `div` |
| 要在文字中標示強調、連結或小段語意 | 常用 `strong`、`em`、`a`、`span` |
| 行內內容需要可控寬高 | 可考慮 replaced element、表單控制項，或用 CSS 改成 `inline-block` |

## 常見錯誤

### 錯誤一：把塊級元素理解成永遠等於 `width: 100%`

塊級元素預設常會填滿可用水平空間，但這通常來自 `width: auto`，不是直接等於 `width: 100%`。

如果你把兩者當成完全相同，之後遇到 margin、padding、border 或特殊盒模型時，可能會對元素實際寬度判斷錯誤。

### 錯誤二：以為所有行內元素都不能控制尺寸

一般 `span`、`em` 這類 non-replaced inline 元素不適合直接用 `width`、`height` 控制尺寸。但 `img`、`input` 等 replaced element 或表單控制項本來就有不同的尺寸行為。

如果你需要讓一段行內內容具有可控寬高，常見做法是用 CSS 把它設為 `inline-block` 或 `block`，而不是只靠 HTML 標籤本身。

### 錯誤三：只靠畫面換不換行判斷語意

塊級與行內描述的是預設排版行為，但 HTML 標籤本身還有語意。例如 `p` 代表段落，`h1` 代表最高層級標題，`span` 只是沒有特殊語意的行內容器。

實務上應該先選語意正確的標籤，再用 CSS 調整排版。

## 重點整理

- 塊級元素通常另起一行，形成一整塊內容區域。
- 行內元素通常與文字一起流動，不會預設獨佔一行。
- 一般 block 元素可用盒模型屬性控制尺寸與間距。
- 一般 non-replaced inline 元素的尺寸主要由內容決定，但 replaced element、表單控制項與 `inline-block` 是重要例外。
- 選標籤時不要只看排版，還要看語意與內容模型。

## 自我檢查

1. 為什麼 `div`、`p`、`h1` 通常會上下排列，而 `span` 通常不會？
2. `width: auto` 和 `width: 100%` 是否完全相同？為什麼不能直接畫上等號？
3. 如果你想在一段文字中標出某幾個字，通常會選 `div` 還是 `span`？理由是什麼？
4. 為什麼不能只用「會不會換行」來決定 HTML 標籤？
