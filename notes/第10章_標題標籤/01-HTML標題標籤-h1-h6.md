# HTML 標題標籤 h1-h6

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `<h1>` 到 `<h6>` 的標籤語意。
- 知道標題層級和文章結構的關係。
- 能寫出有清楚層級的 HTML 標題結構。
- 分辨「看起來像標題」和「語意上是標題」的差異。

## 問題情境

在新聞、文章、產品介紹、教學文件或部落格頁面中，內容通常不會只有一整段文字。讀者需要先看到主題，再依序看到章節、小節與更細的段落標題。

HTML 的標題標籤就是用來描述這種內容層級：

- 這篇頁面的主題是什麼？
- 這個區塊在講哪一件事？
- 這個小段落又隸屬於哪個上層標題？

如果只靠字體大小來做標題，畫面也許看得出差異，但 HTML 文件本身沒有清楚結構。標題標籤的重點不是「把字變大」，而是讓文件具備可理解的層級。

## 一句話理解

`<h1>` 到 `<h6>` 是用來表示標題層級的 HTML 標籤，數字越小，代表層級越高、重要性越大。

## 標籤語意

HTML 提供六個標題標籤：

```html
<h1>一級標題</h1>
<h2>二級標題</h2>
<h3>三級標題</h3>
<h4>四級標題</h4>
<h5>五級標題</h5>
<h6>六級標題</h6>
```

它們的語意是「作為標題使用，並且依據重要性遞減」。

一般可以這樣理解：

| 標籤 | 常見用途 |
| --- | --- |
| `<h1>` | 頁面或文章的主要標題 |
| `<h2>` | 主要段落或章節標題 |
| `<h3>` | `<h2>` 底下的小節標題 |
| `<h4>` | `<h3>` 底下更細的標題 |
| `<h5>` | 更深層的小標題 |
| `<h6>` | 最深層的小標題 |

標題層級應該反映內容結構，而不是只根據預設字體大小來選。

## 瀏覽器預設樣式

在瀏覽器的預設樣式下，標題標籤通常會有這些特點：

- 文字會加粗。
- 字號通常從 `<h1>` 到 `<h6>` 逐漸變小。
- 標題會獨佔一行，前後也常會有預設間距。

這些只是瀏覽器給的預設外觀。實務上可以用 CSS 改變字體大小、粗細、間距與顏色，但不應該因為想要某種大小，就亂選標題層級。

## 基本範例

以下是一篇文章頁面常見的標題結構：

```html
<article>
  <h1>HTML 標題標籤入門</h1>

  <h2>為什麼需要標題標籤？</h2>
  <p>標題可以幫助讀者理解內容結構。</p>

  <h2>標題標籤的層級</h2>
  <p>HTML 提供 h1 到 h6 六種標題層級。</p>

  <h3>h1 的用途</h3>
  <p>h1 通常用來表示頁面或文章最主要的標題。</p>

  <h3>h2 和 h3 的關係</h3>
  <p>h2 表示主要章節，h3 表示章節底下的小節。</p>
</article>
```

這個範例中：

- `<h1>` 是整篇文章的主題。
- `<h2>` 是文章中的主要段落。
- `<h3>` 是某個 `<h2>` 段落底下的細分主題。
- `<p>` 是標題底下的正文內容。

讀者可以只看標題就大致知道文章架構，搜尋引擎、輔助科技與開發者也能更容易理解文件的內容層級。

## 常見錯誤

### 錯誤一：用標題標籤控制字體大小

錯誤寫法：

```html
<h1>商品名稱</h1>
<h6>促銷活動</h6>
```

如果「促銷活動」其實是商品介紹中的重要小節，只因為想讓它看起來比較小就使用 `<h6>`，文件結構會變得不合理。

較好的寫法：

```html
<h1>商品名稱</h1>
<h2 class="campaign-title">促銷活動</h2>
```

如果需要調整視覺大小，交給 CSS 處理：

```css
.campaign-title {
  font-size: 18px;
}
```

HTML 負責語意和結構，CSS 負責外觀。

### 錯誤二：任意跳過標題層級

錯誤寫法：

```html
<h1>旅遊指南</h1>
<h4>交通方式</h4>
<h4>住宿建議</h4>
```

這樣會讓文件從一級標題直接跳到四級標題，中間的結構不清楚。

較好的寫法：

```html
<h1>旅遊指南</h1>
<h2>交通方式</h2>
<h2>住宿建議</h2>
```

除非內容真的有多層巢狀關係，否則不需要刻意使用到 `<h4>`、`<h5>` 或 `<h6>`。

### 錯誤三：把一般文字硬包成標題

錯誤寫法：

```html
<h2>本週限時優惠，全館滿千免運，活動只到週日晚上。</h2>
```

如果這段只是一般說明文，而不是一個區塊標題，就不適合使用標題標籤。

較好的寫法：

```html
<h2>本週優惠</h2>
<p>全館滿千免運，活動只到週日晚上。</p>
```

標題應該用來概括內容，正文才用來承載細節。

## 實務使用建議

撰寫標題標籤時，可以先把頁面想成一份有目錄的文件：

```text
頁面主題
  主要章節
    小節
    小節
  主要章節
    小節
```

再把它轉成 HTML：

```html
<h1>頁面主題</h1>

<h2>主要章節</h2>
<h3>小節</h3>
<h3>小節</h3>

<h2>主要章節</h2>
<h3>小節</h3>
```

實務上通常會讓一個頁面有一個清楚的 `<h1>`，用來表示頁面最主要的主題。接著依內容層級使用 `<h2>`、`<h3>`，較深的 `<h4>` 到 `<h6>` 只有在內容真的需要更細分時才使用。

## 重點整理

- `<h1>` 到 `<h6>` 是 HTML 的標題標籤。
- 數字越小，標題層級越高，重要性越大。
- 標題標籤的核心是語意和文件結構，不只是字體大小。
- 瀏覽器預設會讓標題加粗、獨佔一行，並讓字號由 `<h1>` 到 `<h6>` 逐漸變小。
- 不要為了視覺大小任意選擇標題層級；外觀應該交給 CSS。
- 一個清楚的標題結構能幫助讀者閱讀，也有助於維護、SEO 與無障礙。
