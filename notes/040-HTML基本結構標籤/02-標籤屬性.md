# 標籤屬性：替 HTML 標籤補充資訊

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 HTML 屬性是什麼，以及它和標籤的關係。
- 寫出包含多個屬性的 HTML 標籤。
- 分辨一般屬性與布林屬性的基本寫法。
- 避免重複屬性、缺少空格、屬性值寫法混亂等常見錯誤。

## 問題情境

同樣是 `<p>` 標籤，有時候我們希望其中一段文字有特定識別名稱，有時候希望套用某個樣式，有時候希望和 JavaScript 或 CSS 互動。單靠標籤名稱只能說明「這是一個段落」，無法提供更多細節。

這時候就需要屬性。屬性用來替標籤補充附加資訊，讓同一種標籤在不同情境下具備不同設定。

## 一句話理解

屬性就是寫在開始標籤或單標籤裡的補充設定，用來描述這個標籤的額外資訊。

## 屬性寫在哪裡

屬性寫在開始標籤或單標籤內部，通常由屬性名與屬性值組成。

![img 標籤屬性名與屬性值示意圖](../../origin/040-HTML基本結構標籤/assets/images/html-basic-structure-tags-img-001-4f91bb.png)

![標籤屬性位於開始標籤或單標籤內示意圖](../../origin/040-HTML基本結構標籤/assets/images/html-basic-structure-tags-img-002-b3d0e7.png)

基本形式如下：

```html
<p id="notice" class="highlight">
  尚硅谷，讓天下沒有難學的技術!!!
</p>
```

這段程式碼可以拆成：

- `<p>`：表示這是一個段落。
- `id="notice"`：替這個段落設定唯一識別名稱。
- `class="highlight"`：替這個段落設定分類名稱，常用於 CSS 選取。
- `尚硅谷，讓天下沒有難學的技術!!!`：實際顯示在頁面上的段落內容。

## 屬性名與屬性值

多數屬性會寫成 `屬性名="屬性值"`：

```html
<標籤名 屬性名="屬性值">內容</標籤名>
```

例如：

```html
<a href="https://example.com">前往網站</a>
<img src="logo.png" alt="網站標誌">
<input type="text" name="username">
```

不同標籤可以使用的屬性不同。例如 `href` 常用在 `<a>`，`src` 和 `alt` 常用在 `<img>`，`type` 常用在 `<input>`。也有一些通用屬性，例如 `id`、`class`，可以寫在很多標籤上。

## 布林屬性

有些屬性是布林屬性，表示某個狀態是否啟用。這類屬性可以只寫屬性名，不必寫屬性值。

```html
<input disabled>
```

這裡的 `disabled` 表示這個輸入框被停用。只要屬性存在，就代表狀態啟用。

初學時可以先記住：不是所有屬性都能省略屬性值。像 `id`、`class`、`href`、`src` 這類屬性仍然需要合理的屬性值。

## 多個屬性的寫法

一個標籤上可以同時存在多個屬性，屬性之間用空格隔開，屬性順序通常不影響 HTML 結構意義。

```html
<input type="text" name="username" id="username" class="form-control">
```

這段程式碼中：

- `type="text"`：設定輸入框類型。
- `name="username"`：設定表單送出時使用的欄位名稱。
- `id="username"`：設定頁面中的識別名稱。
- `class="form-control"`：設定分類名稱，通常給 CSS 或 JavaScript 使用。

## 實務寫法建議

屬性名通常不區分大小寫，但建議統一使用小寫，讓程式碼風格一致。

屬性值是否區分大小寫，取決於具體屬性，不能一概而論。初學階段更重要的是養成穩定的書寫習慣：屬性值一律使用雙引號包起來。

建議寫法：

```html
<p class="intro">這是一段介紹文字。</p>
```

不建議初學時寫成：

```html
<p CLASS=intro>這是一段介紹文字。</p>
```

後者在某些情況下可能仍能被解析，但可讀性與一致性較差，也容易在複雜屬性值中出錯。

## 常見錯誤

### 標籤名和屬性之間少了空格

錯誤寫法：

```html
<pclass="intro">這是一段文字。</p>
```

瀏覽器不會把這段理解成 `<p>` 標籤加上 `class` 屬性，因為標籤名和屬性之間少了必要空格。

正確寫法：

```html
<p class="intro">這是一段文字。</p>
```

### 屬性之間沒有空格

錯誤寫法：

```html
<input type="text"name="username">
```

屬性之間應該用空格隔開，否則會讓程式碼難以閱讀，也可能造成解析問題。

正確寫法：

```html
<input type="text" name="username">
```

### 同一個標籤出現同名屬性

錯誤寫法：

```html
<input type="text" type="password">
```

同一個標籤不要出現同名屬性。這類寫法會讓屬性設定互相衝突，後寫的屬性可能失效，程式碼也會變得難以判斷。

正確寫法應該先決定真正需要的類型：

```html
<input type="password">
```

### 亂省略屬性值的引號

HTML 在符合語法限制時可以省略屬性值引號，但初學階段不建議這樣做。

較穩定的寫法：

```html
<div class="card primary"></div>
```

如果省略引號，遇到包含空格的屬性值就會出問題：

```html
<div class=card primary></div>
```

這段並不等同於 `class="card primary"`。因此在學習與實務維護中，建議屬性值一律加上雙引號。

## 判斷準則

寫屬性時可以依序檢查：

1. 這個屬性是否適合寫在目前標籤上？
2. 屬性名與標籤名之間是否有空格？
3. 多個屬性之間是否有空格？
4. 屬性值是否需要引號？若不確定，就使用雙引號。
5. 同一個標籤是否不小心寫了重複屬性？

## 重點整理

- 屬性用來替標籤提供附加資訊。
- 屬性寫在開始標籤或單標籤內部。
- 多數屬性使用 `屬性名="屬性值"` 的形式。
- 布林屬性可以只寫屬性名，例如 `disabled`。
- 標籤名與屬性之間、不同屬性之間，都必須用空格隔開。
- 初學階段建議屬性名使用小寫，屬性值使用雙引號。

4. 如果要替一張圖片設定圖片來源與替代文字，可能會用到哪些屬性？
