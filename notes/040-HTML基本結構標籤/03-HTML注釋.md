# HTML 注釋：寫給開發者看的說明

## 學習目標

讀完這篇筆記，你應該能夠：

- 寫出正確的 HTML 注釋語法。
- 說明注釋和頁面顯示內容的差異。
- 判斷什麼情況適合加注釋，什麼情況不需要加。
- 避免在注釋中放入敏感資訊或無意義說明。

## 使用情境

寫 HTML 時，你可能會遇到這些情況：

- 一個頁面分成導覽列、主要內容、頁尾等多個區塊。
- 某段結構暫時保留，之後還會回來修改。
- 團隊成員需要理解某段標記的用途。
- 幾週後你自己回來看程式碼，希望快速想起當初為什麼這樣寫。

這時候可以使用 HTML 注釋，在程式碼中留下給開發者看的說明。

## 一句話理解

HTML 注釋是寫在原始碼中的說明文字，瀏覽器不會把它顯示成頁面內容。

## 基本語法

HTML 注釋以 `<!--` 開頭，以 `-->` 結束。

```html
<!-- 這是一段注釋 -->
```

注釋可以放在 HTML 結構中，用來說明附近程式碼的用途：

```html
<!-- 網站主導覽 -->
<nav>
  <a href="/">首頁</a>
  <a href="/about.html">關於我們</a>
</nav>
```

畫面上會顯示導覽連結，但不會顯示 `網站主導覽` 這段注釋文字。

## 注釋不會執行，也不會顯示在頁面中

注釋裡面的內容是給開發者看的。它不會變成頁面上的文字，也不會作為 HTML 標籤執行。

```html
<!-- <h1>這個標題不會顯示</h1> -->
```

上面這段 `<h1>` 被放在注釋裡，所以瀏覽器不會把它當成真正的標題顯示。

這個特性有時可用於暫時停用一小段 HTML，但不要長期把大量不用的程式碼留在注釋中，否則檔案會變得難以維護。

## 好注釋與壞注釋

好的注釋應該補充程式碼本身不容易看出的意圖，而不是重複明顯資訊。

較有幫助的注釋：

```html
<!-- 首頁主視覺區：包含標題、簡介與主要行動按鈕 -->
<section class="hero">
  <h1>前端學習筆記</h1>
  <p>整理 HTML、CSS 與 JavaScript 的基礎觀念。</p>
  <a href="/notes.html">開始閱讀</a>
</section>
```

這段注釋讓讀者快速知道整個 `<section>` 的角色。

較沒有幫助的注釋：

```html
<!-- 這是一個 h1 -->
<h1>前端學習筆記</h1>
```

`<h1>` 本身已經清楚表示這是標題，這種注釋只是重複程式碼，沒有提供額外價值。

## 常見錯誤

### 忘記結束注釋

錯誤寫法：

```html
<!-- 主要內容區
<main>
  <h1>文章標題</h1>
</main>
```

如果缺少結尾的 `-->`，後面的內容可能都被當成注釋，導致頁面顯示異常。

正確寫法：

```html
<!-- 主要內容區 -->
<main>
  <h1>文章標題</h1>
</main>
```

### 把敏感資訊寫進注釋

注釋不會顯示在頁面畫面上，但仍然可能存在於瀏覽器可查看的原始碼中。因此不要把密碼、金鑰、內部帳號、未公開資料等敏感資訊寫進 HTML 注釋。

錯誤示例：

```html
<!-- 測試帳號：admin / 密碼：123456 -->
```

這類資訊即使不出現在畫面上，也不代表安全。

### 用注釋取代清楚的 HTML 結構

如果頁面區塊本身可以用語意化標籤表達，就不要只依賴注釋說明。

較差寫法：

```html
<!-- 導覽列 -->
<div>
  <a href="/">首頁</a>
</div>
```

較好寫法：

```html
<nav>
  <a href="/">首頁</a>
</nav>
```

注釋可以輔助理解，但不能取代合適的 HTML 標籤。

## 實務使用建議

適合加注釋的情況：

- 標記大型區塊的開始或用途。
- 解釋短時間內不容易從程式碼看出的意圖。
- 提醒後續維護者某段結構與其他檔案或功能有關。

不太需要加注釋的情況：

- 注釋只是重複標籤名稱。
- 程式碼已經很短且語意清楚。
- 注釋內容會很快過期，卻沒有人維護。

## 小練習

請替下面這段 HTML 加上一段有意義的注釋：

```html
<footer>
  <p>Copyright 2026 Example Site</p>
  <a href="/privacy.html">隱私權政策</a>
</footer>
```

參考方向：不要寫「這是 footer」這種重複標籤名稱的注釋，而是說明這個區塊在頁面中的用途。

## 重點整理

- HTML 注釋以 `<!--` 開頭，以 `-->` 結束。
- 注釋內容是給開發者看的，不會顯示在頁面中，也不會被當成 HTML 執行。
- 好注釋應該說明意圖或區塊用途，不只是重複標籤名稱。
- 注釋仍可能出現在原始碼中，不要放入敏感資訊。
- 注釋可以輔助維護，但不能取代清楚的 HTML 結構。
