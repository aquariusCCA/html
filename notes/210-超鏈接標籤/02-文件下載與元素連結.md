# 文件下載與元素連結

## 學習目標

讀完這篇筆記，你應該能夠：

- 使用 `<a>` 連到圖片、音訊、影片、PDF 或壓縮檔。
- 說明瀏覽器直接開啟檔案與下載檔案的差異。
- 使用 `download` 屬性提示瀏覽器下載檔案。
- 讓圖片或非互動元素成為連結，同時避開不適合包進 `<a>` 的內容。

## 使用情境

超連結不只能連到網頁，也能連到檔案。

例如：

- 點擊「看圖片」開啟一張圖片。
- 點擊「看 PDF 檔」在瀏覽器中預覽 PDF。
- 點擊「內部資源」下載 zip 壓縮檔。
- 點擊商品圖片後前往商品詳情頁。

這些都可以用 `<a>` 完成，差別在於 `href` 指向的是 HTML 頁面、媒體檔案、文件檔案，還是其他資源。

## 一句話理解

只要目標可以用 URL 表示，`<a href="...">` 就可以連過去；至於瀏覽器是開啟、播放、預覽還是下載，取決於檔案類型、伺服器設定與瀏覽器支援。

## 連到可開啟的文件

有些檔案瀏覽器可以直接開啟或預覽，例如圖片、PDF、常見音訊與影片格式。

```html
<!-- 瀏覽器能直接打開文件 -->
<a href="./assets/images/hyperlink-tags-img-002-354aea.jpg">看圖片</a>
<a href="./assets/files/hyperlink-tags-file-002-9a18a4.mp4">看小電影</a>
<a href="./assets/files/hyperlink-tags-file-003-278402.mp3">聽音樂</a>
<a href="./assets/pdfs/hyperlink-tags-pdf-001-78621d.pdf">看 pdf 檔</a>
```

這些連結的寫法和一般網頁連結一樣，都是把目標放在 `href` 裡。使用者點擊後，瀏覽器會依照檔案類型決定怎麼處理。

## 連到會下載的文件

有些檔案瀏覽器通常不能直接顯示，例如 zip 壓縮檔。這類檔案常會觸發下載。

```html
<!-- 瀏覽器不能打開的文件，會自動觸發下載 -->
<a href="./assets/files/hyperlink-tags-file-004-28aaa1.zip">內部資源</a>
```

這不是 `<a>` 自己決定的，而是瀏覽器與伺服器共同決定的結果。`<a>` 只是告訴瀏覽器「請前往這個資源」。

## download 屬性

如果你希望提示瀏覽器把目標資源下載下來，可以在 `<a>` 上加 `download` 屬性。

```html
<!-- 提示瀏覽器下載 -->
<a href="./assets/pdfs/hyperlink-tags-pdf-001-78621d.pdf" download="一份PDF檔.pdf">下載 PDF 檔案</a>
```

`download` 的屬性值可以作為建議下載檔名。以上例子中，瀏覽器可能會把檔案存成 `一份PDF檔.pdf`。

但要注意：`download` 不是絕對命令。實際行為可能受到以下因素影響：

- 目標檔案是否同源。
- 伺服器回傳的 HTTP 標頭。
- 瀏覽器安全策略。
- 使用者的瀏覽器設定。

因此，`download` 可以理解為「提示瀏覽器下載」，不是保證所有環境都會照做。

## 讓圖片成為連結

`<a>` 可以包住圖片，讓使用者點圖片時前往指定目標。

```html
<h4>網頁元素的鏈接</h4>
<a href="http://www.baidu.com">
  <img src="./assets/images/hyperlink-tags-img-002-354aea.jpg" alt="前往百度" />
</a>
```

這段程式碼中：

- `<a href="http://www.baidu.com">` 決定點擊後前往哪裡。
- `<img src="..." alt="前往百度">` 決定畫面上顯示哪張圖片，並用 `alt` 說明這個圖片連結的目的。
- 圖片被放在 `<a>` 裡，所以整張圖片都會變成可點擊範圍。

這種寫法常用在：

- 商品卡片圖片連到商品詳情。
- Logo 圖片連回首頁。
- 文章縮圖連到文章頁。

## 哪些元素適合放進 a

適合放進 `<a>` 的內容通常是「非互動內容」：

- 文字。
- 圖片。
- 圖示。
- 卡片內容。
- 一般區塊元素。

不適合放進 `<a>` 的內容通常是「本身已經可以互動的元素」：

- `<button>`。
- `<input>`。
- `<select>`。
- 另一個 `<a>`。
- 帶有 `controls` 的音訊或影片。

原因是互動元素包互動元素，會讓瀏覽器、鍵盤操作與輔助技術難以判斷使用者到底要操作哪一層。

## 常見錯誤

### 錯誤一：以為加了 download 就一定會下載

```html
<a href="https://example.com/file.pdf" download="file.pdf">下載</a>
```

如果這個 PDF 來自不同來源，或伺服器設定不允許，瀏覽器可能仍然選擇開啟檔案，而不是下載。實務上要把 `download` 當成提示，並在必要時搭配後端檔案回應設定。

### 錯誤二：把按鈕放進連結

```html
<!-- 不建議 -->
<a href="product.html">
  <button>查看商品</button>
</a>
```

這會造成互動語意混亂。較清楚的寫法是直接讓 `<a>` 看起來像按鈕，或在需要操作時使用真正的 `<button>`。

```html
<!-- 前往頁面：使用 a -->
<a href="product.html" class="button-link">查看商品</a>

<!-- 觸發操作：使用 button -->
<button type="button">加入收藏</button>
```

### 錯誤三：圖片連結缺少替代文字

```html
<a href="index.html">
  <img src="logo.png" />
</a>
```

如果圖片無法載入，或使用者透過螢幕報讀器瀏覽，就不知道這個連結要去哪裡。應該補上能描述連結目的的 `alt`。

```html
<a href="index.html">
  <img src="logo.png" alt="回到首頁" />
</a>
```

## 實務判斷準則

- 連到檔案時，先判斷使用者是要「看」還是要「下載」。
- 可以被瀏覽器預覽的檔案，不一定會下載。
- `download` 可提示下載與建議檔名，但不能保證所有環境都生效。
- 圖片、卡片、文字可以放進 `<a>`，讓整塊內容成為連結。
- 不要把互動控制項放進 `<a>`。

## 重點整理

- `<a>` 的 `href` 可以指向網頁，也可以指向檔案。
- 檔案被開啟或下載，取決於檔案類型、伺服器與瀏覽器。
- `download` 是下載提示，不是絕對保證。
- 圖片可以包在 `<a>` 裡，讓圖片變成連結。
- `<a>` 裡應避免再放互動元素。
