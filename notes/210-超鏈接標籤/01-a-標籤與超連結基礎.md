# a 標籤與超連結基礎

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 `<a>` 標籤在 HTML 中的用途。
- 使用 `href` 建立可點擊、可導航的超連結。
- 分辨外部連結、內部連結與 `target` 開啟方式。
- 判斷什麼情境應該使用 `<a>`，什麼情境不適合。

## 使用情境

當你希望使用者點擊一段文字後，前往另一個網頁、網站內的另一個頁面，或開啟某個資源時，就會使用 `<a>` 標籤。

例如：

- 點擊「百度」前往 `https://www.baidu.com/`。
- 點擊「公司簡介」前往網站內部的介紹頁。
- 點擊導覽列項目切換到其他頁面。

超連結是網頁能彼此串接的基礎。沒有 `<a>`，使用者就只能停留在目前頁面，無法透過頁面內容自然移動到其他位置。

## 一句話理解

`<a>` 是用來建立「可導航連結」的 HTML 語意標籤；它的核心任務是把目前頁面連到另一個目標。

`a` 來自英文 `anchor`，意思是「錨」。在 HTML 中，它可以把一段文字、圖片或部分非互動內容變成連結入口。

## 基本語法

```html
<a href="目标地址">文字内容</a>
```

實際例子：

```html
<a href="https://www.baidu.com/">百度</a>
```

這段程式碼會在頁面上顯示「百度」兩個字。使用者點擊後，瀏覽器會依照 `href` 指定的網址前往目標頁面。

## 範例拆解

```html
<a href="https://www.baidu.com/">百度</a>
```

- `<a>`：建立超連結的標籤。
- `href="https://www.baidu.com/"`：指定連結目標。
- `百度`：使用者在畫面上看到並點擊的文字。
- `</a>`：結束這個連結範圍。

如果 `<a>` 沒有 `href`，它通常不會形成一般可導航的超連結。實務上，只要目的是讓使用者前往某個位置，就應該明確設定 `href`。

## href 屬性

`href` 用來指定連結目標。這個目標可以是：

- 外部網站網址，例如 `https://www.baidu.com/`。
- 站內頁面，例如 `about.html` 或其他相對路徑資源。
- 頁面中的某個位置，例如 `#section-2`。
- 特殊連結，例如 `mailto:` 或 `tel:`。

## target 屬性

`target` 用來指定連結目標的開啟方式。

常見值有兩個：

| target 值 | 開啟方式 | 使用情境 |
| --- | --- | --- |
| `_self` | 在目前頁籤開啟，這也是預設值 | 站內導覽、一般頁面切換 |
| `_blank` | 在新頁籤開啟 | 外部網站、文件預覽、希望保留原頁面的情境 |

例如：

```html
<a href="http://www.qq.com" target="_self">騰訊</a>
<a href="http://www.itcast.cn" target="_blank">傳智播客</a>
```

第一個連結會在目前頁籤開啟，第二個連結會在新頁籤開啟。

當 `_blank` 開啟外部或不完全可信任的頁面時，實務上常搭配 `rel="noopener"`，避免新頁面透過 `window.opener` 影響原本頁面。

```html
<a href="https://example.com" target="_blank" rel="noopener">外部網站</a>
```

## 外部連結與內部連結

### 外部連結

外部連結指向目前網站以外的網址，通常會寫完整網址。

```html
<h4>外部鏈接</h4>
<a href="http://www.qq.com" target="_self"> 騰訊</a>
<a href="http://www.itcast.cn" target="_blank">傳智播客</a>
```

如果連到別人的網站，常見做法是使用 `target="_blank"`，讓使用者保留目前網站頁面。不過是否開新頁籤仍要看產品需求，不能機械套用。

### 內部連結

內部連結指向同一個網站內的頁面或資源。通常可以使用相對路徑。

```html
<body>
  <h4>內部鏈接</h4>
  <a href="./assets/files/hyperlink-tags-file-001-b99c80.html" target="_blank">公司簡介</a>
</body>
```

這裡的 `href` 值代表從目前 HTML 檔案所在位置出發，找到網站內部的另一個 HTML 檔案。

## a 標籤可以包哪些內容

`<a>` 是行內元素，但 HTML5 之後，它可以包住多數非互動內容，例如文字、圖片或區塊元素。

```html
<a href="https://example.com">
    <div>這是一個包含在 a 標籤中的區塊元素</div>
</a>
```

這種寫法可以讓整個區塊都變成可點擊範圍，常見於卡片式列表、商品卡或文章卡。

但要注意：

- 不要在 `<a>` 裡面再放另一個 `<a>`。
- 避免把 `<button>`、`input` 等互動元素包進 `<a>`。
- 如果目的只是觸發動作，不是前往某個位置，通常應該使用 `<button>`。

## 常見錯誤

### 錯誤一：沒有 href 卻期待它像連結一樣運作

```html
<a>前往首頁</a>
```

這段文字看起來像是想做連結，但沒有 `href`，瀏覽器沒有可導航的目標。應改成：

```html
<a href="index.html">前往首頁</a>
```

### 錯誤二：把所有點擊行為都寫成 a

如果目的是「送出表單」「打開彈窗」「切換狀態」，這些不是導航行為，通常不應該用 `<a>` 假裝按鈕。

```html
<!-- 不建議：這是操作，不是導航 -->
<a href="javascript:saveData()">儲存</a>

<!-- 較合適：使用 button 表示操作 -->
<button type="button">儲存</button>
```

判斷原則很簡單：前往某個位置用 `<a>`，觸發某個操作用 `<button>`。

## 實務判斷準則

- 連到外部網站：使用 `<a href="https://...">`。
- 連到站內頁面：使用 `<a href="about.html">` 或相對路徑。
- 讓圖片或卡片可點擊前往詳情頁：可以用 `<a>` 包住非互動內容。
- 送出、取消、儲存、打開視窗：通常使用 `<button>`。
- 需要新頁籤開啟時才使用 `target="_blank"`，不要無條件套用。

## 重點整理

- `<a>` 的核心語意是建立可導航的超連結。
- `href` 指定連結目標，是超連結最重要的屬性。
- `target="_self"` 在目前頁籤開啟，`target="_blank"` 在新頁籤開啟。
- 外部連結通常使用完整網址，內部連結常使用相對路徑。
- `<a>` 可以包住多數非互動內容，但不應巢狀 `<a>` 或包住互動控制項。
