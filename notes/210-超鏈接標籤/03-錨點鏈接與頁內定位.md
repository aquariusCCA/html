# 錨點鏈接與頁內定位

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明錨點鏈接的用途。
- 使用 `href="#id"` 跳到同一頁中的指定位置。
- 使用 `頁面路徑#id` 跳到其他頁面的指定位置。
- 避免錨點名稱不一致、重複 `id` 等常見錯誤。

## 使用情境

當頁面內容很長時，使用者不一定想從頭慢慢捲到目標段落。這時可以使用錨點鏈接，讓使用者點一下就跳到指定位置。

常見情境包括：

- 文章目錄點擊後跳到某一節。
- FAQ 點擊問題後跳到答案。
- 商品頁點擊「規格」「評價」「運送方式」切換到對應段落。
- 文件頁點擊章節導覽快速定位內容。

錨點鏈接的重點不是前往另一個網站，而是在頁面內或指定頁面中精準定位。

## 一句話理解

錨點鏈接就是用 `<a href="#某個id">` 指向頁面中 `id="某個id"` 的元素。

## 基本寫法

錨點鏈接分成兩個部分：

1. 連結端：在 `<a>` 的 `href` 寫 `#名字`。
2. 目標端：在目標元素上設定相同的 `id`。

```html
<!-- 跳轉到本頁的錨點 -->
<a href="#two">第二季</a>
<h3 id="two">第二季介绍</h3>
```

點擊「第二季」時，瀏覽器會在目前頁面中尋找 `id="two"` 的元素，並捲動到那個位置。

## 範例拆解

```html
<a href="#two">第二季</a>
<h3 id="two">第二季介绍</h3>
```

- `href="#two"`：告訴瀏覽器要跳到名為 `two` 的位置。
- `#`：表示這是一個片段識別符，目標是頁面內的某個元素。
- `id="two"`：在目標元素上建立可以被連到的位置。
- `第二季`：使用者點擊的連結文字。

`href` 裡的 `#two` 和目標元素的 `id="two"` 必須對上。`#` 只寫在 `href` 裡，`id` 的值本身不要寫 `#`。

## 跳到其他頁面的錨點

錨點不只可以跳到本頁，也可以跳到其他頁面的指定位置。

```html
<!-- 跳轉到其他頁面的錨點 -->
<a href="./assets/files/hyperlink-tags-file-001-b99c80.html#test1">跳轉到測試頁面的 test1 錨點</a><br />
```

這段 `href` 可以拆成兩段理解：

- 前半段的 HTML 檔案路徑：要前往的頁面。
- `#test1`：進入該頁後要定位到的元素 `id`。

也就是說，瀏覽器會先開啟這個測試 HTML 檔案，再嘗試捲動到該頁中 `id="test1"` 的位置。目標頁面必須真的有對應的 `id`，錨點定位才會生效。

## 與一般連結的差異

| 類型 | href 寫法 | 用途 |
| --- | --- | --- |
| 一般外部連結 | `https://example.com` | 前往其他網站 |
| 一般內部連結 | `about.html` | 前往站內頁面 |
| 本頁錨點 | `#section` | 跳到目前頁面的某個元素 |
| 他頁錨點 | `about.html#team` | 前往其他頁面並定位到指定元素 |

錨點鏈接本質上仍然是 `<a>`，只是連結目標更精準。

## 常見錯誤

### 錯誤一：href 與 id 名稱不一致

```html
<!-- 錯誤：href 指向 #two，但目標 id 是 second -->
<a href="#two">第二季</a>
<h3 id="second">第二季介绍</h3>
```

這樣瀏覽器找不到 `id="two"` 的元素，就無法跳到預期位置。

正確寫法：

```html
<a href="#two">第二季</a>
<h3 id="two">第二季介绍</h3>
```

### 錯誤二：id 裡多寫了井號

```html
<!-- 錯誤 -->
<a href="#two">第二季</a>
<h3 id="#two">第二季介绍</h3>
```

`#` 是寫在 `href` 裡用來表示錨點的符號，`id` 的值應該只寫名稱。

### 錯誤三：同一頁有重複 id

```html
<h2 id="intro">第一段介紹</h2>
<h2 id="intro">第二段介紹</h2>
```

`id` 在同一頁中應該是唯一的。重複 `id` 會讓定位結果不可靠，也會影響 JavaScript 與無障礙工具辨識頁面。

### 錯誤四：誤以為兩個井號會回到頂部

`href="#"` 常被用來回到頁面頂部或作為臨時連結，但 `##` 不是「回到頂部」的正確寫法。兩個井號的點擊行為不應被當成可靠的頁面定位方式。

## 實務寫法建議

- `id` 命名要有意義，例如 `features`、`faq`、`contact`。
- 避免使用會隨內容變動而失去意義的名稱，例如 `a1`、`x2`。
- 頁面目錄的連結文字應該清楚描述目標段落。
- 如果頁面有固定頂部導覽列，錨點跳轉可能被導覽列遮住，實務上可搭配 CSS 的 `scroll-margin-top` 調整。

例如：

```css
h2 {
  scroll-margin-top: 80px;
}
```

這不是 HTML 錨點的必要語法，但在真實網站中很常見，能避免跳轉後標題被固定導覽列蓋住。

## 重點整理

- 錨點鏈接用來跳到頁面中的指定位置。
- `href="#name"` 要對應到目標元素的 `id="name"`。
- 跳到其他頁面的錨點可寫成 `page.html#name`。
- `id` 在同一頁中應保持唯一。
- `#` 寫在 `href` 裡，不能寫進 `id` 值。
