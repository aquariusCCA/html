# 特殊 href 用法補充

## 學習目標

讀完這篇筆記，你應該能夠：

- 認識 `href="#"`、`href=""`、`javascript:`、`tel:`、`mailto:` 等特殊寫法。
- 判斷這些寫法適合用在什麼情境。
- 避免把特殊 `href` 當成一般互動功能的主要寫法。
- 知道何時應改用 `<button>` 與 JavaScript 事件監聽。

## 為什麼這篇是補充

一般超連結的主線學習是：

```html
<a href="目標地址">連結文字</a>
```

也就是點擊後前往某個網址、頁面、檔案或錨點。

但實務上你也會看到一些特殊 `href` 寫法，例如 `#`、空字串、`javascript:`、`tel:`、`mailto:`。它們都和 `<a>` 有關，但不是每一種都適合當成正式網站的主要寫法，所以這篇作為補充整理。

## href="#"：跳到頁面頂部或臨時連結

```html
<div style="height: 2000px;">空盒子</div>
<!-- 跳到本頁的頂部 -->
<a href="#">回到頂部</a>
```

`href="#"` 會指向目前頁面的空片段。常見效果是點擊後回到頁面頂部。

要注意：

- `#` 可以作為簡單的回頂部示例。
- 兩個井號 `##` 不是可靠的回頂部寫法。
- 如果只是暫時占位，不建議長期保留 `href="#"`，因為點擊後仍可能改變網址片段或造成頁面跳動。

如果是正式的「回到頂部」功能，可以使用更明確的目標：

```html
<a href="#top">回到頂部</a>

<header id="top">
  ...
</header>
```

這樣讀者與維護者都能看懂目標位置。

## href=""：空連結

```html
<h4>空鏈接</h4>
<a href="">刷新本頁面</a>
```

空 `href` 通常會指向目前頁面。點擊後可能造成重新載入或回到目前 URL，具體行為仍會受瀏覽器與頁面狀態影響。

實務上不建議把空 `href` 當成真正功能，因為它的意圖不清楚：

- 是要重新整理頁面？
- 是還沒填連結？
- 是 placeholder？

如果目標是重新整理頁面，應該用更明確的 JavaScript 行為；如果只是還沒有連結，通常應先不要輸出可點擊的 `<a>`。

## javascript: URL

```html
<h4>執行 JS</h4>
<a href="javascript:alert('Hello, World!');">點擊我</a>
```

這種寫法會在點擊連結時執行 JavaScript。它可以幫助你理解 `href` 的目標不一定是一般網址，但正式頁面中通常應避免使用。

原因包括：

- HTML 結構與 JavaScript 行為混在一起，維護性較差。
- 不利於無障礙與語意判斷。
- 容易讓 `<a>` 被誤用成按鈕。
- 在安全與內容安全政策較嚴格的環境中可能受限制。

如果目的是觸發操作，建議改用 `<button>` 搭配事件監聽。

```html
<button type="button" id="helloBtn">點擊我</button>

<script>
  document.querySelector("#helloBtn").addEventListener("click", function () {
    alert("Hello, World!");
  });
</script>
```

這樣 HTML 負責語意，JavaScript 負責行為，兩者分工更清楚。

## tel: 電話連結

```html
<a href="tel:10010">電話聯繫</a>
```

`tel:` 可以嘗試喚起裝置的撥號功能。它常用於行動裝置上的客服電話、店家電話或聯絡資訊。

要注意：

- 桌面瀏覽器不一定有可用的撥號應用。
- 使用者裝置必須支援對應功能。
- 電話號碼格式應清楚，避免夾雜不必要文字。

## mailto: 郵件連結

```html
<a href="mailto:10010@qq.com">郵件聯繫</a>
```

`mailto:` 可以嘗試開啟使用者的郵件應用，並建立一封寄給指定地址的新郵件。

它適合用在簡單聯絡入口，但不適合取代正式表單。因為使用者不一定有設定郵件客戶端，也不一定希望離開目前瀏覽流程。

## app scheme：喚起指定應用

```html
<a href="line://msg/text/Hello%20World">使用 LINE 發送消息</a>
```

某些應用會提供自己的 URL scheme，讓瀏覽器嘗試開啟對應 app。這類寫法通常和裝置、瀏覽器、作業系統與應用是否安裝有關。

實務上要注意：

- 使用者沒有安裝對應 app 時可能沒有反應。
- 不同平台支援程度不同。
- 若是重要流程，應提供備用方案，例如一般網址或提示文字。

## 特殊 href 對照表

| href 寫法 | 常見用途 | 實務提醒 |
| --- | --- | --- |
| `#` | 回到頂部、空片段示例 | 不要把 `##` 當成回頂部；正式功能建議使用明確錨點 |
| `""` | 指向目前頁面 | 意圖不清楚，容易造成重新載入 |
| `javascript:...` | 點擊後執行 JS | 正式頁面通常避免，操作行為改用 `<button>` |
| `tel:10010` | 喚起撥號 | 依裝置與瀏覽器支援而定 |
| `mailto:name@example.com` | 開啟郵件應用 | 使用者需有可用郵件客戶端 |
| `line://...` | 嘗試喚起指定 app | 依 app、平台與安裝狀態而定 |

## 常見錯誤

### 錯誤一：用 javascript: 做主要互動

```html
<a href="javascript:submitForm()">送出</a>
```

這個動作是「送出」，不是「前往某個位置」。應改用 `<button>` 表示操作。

```html
<button type="submit">送出</button>
```

### 錯誤二：長期保留空連結

```html
<a href="">更多內容</a>
```

如果連結目標還沒決定，應先避免輸出可點擊連結，或在開發階段清楚標記待補。空連結在正式頁面中容易造成使用者誤點與頁面重新載入。

### 錯誤三：認為 app scheme 一定會成功

```html
<a href="line://msg/text/Hello%20World">使用 LINE 發送消息</a>
```

這只是在支援的環境中嘗試喚起 LINE，不代表所有使用者都能成功。重要流程應提供替代路徑。

## 實務判斷準則

- 前往頁面、檔案、錨點：使用一般 `<a href="...">`。
- 觸發操作：優先使用 `<button>` 和事件監聽。
- 電話與郵件入口：可以使用 `tel:`、`mailto:`，但要考慮使用者環境。
- 喚起 app：可以作為輔助功能，不應成為唯一完成任務的路徑。
- 空 `href` 與 `javascript:` 應避免成為正式網站的長期寫法。

## 重點整理

- 特殊 `href` 寫法可以解決特定情境，但不應混淆 `<a>` 的主要語意。
- `href="#"` 常見於回頂部或臨時連結，但正式頁面應盡量明確。
- `href=""` 意圖不清楚，容易造成目前頁面重新載入。
- `javascript:` 可以執行程式碼，但正式互動通常應改用 `<button>`。
- `tel:`、`mailto:`、app scheme 都受裝置與瀏覽器支援影響。
