# 水平線標籤：用 `<hr>` 表示主題分隔

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 `<hr>` 標籤在 HTML 裡的語意。
- 知道 `<hr>` 是單標籤，不需要包住文字內容。
- 判斷什麼時候適合用 `<hr>`，什麼時候應該改用 CSS 畫線。

## 使用情境

當一篇內容從一個主題切換到另一個主題時，可以用 `<hr>` 表示這裡有一個「主題分隔」。

例如一篇文章前半段在介紹產品特色，後半段改成列出注意事項；或一份教學筆記先講概念，再切換到補充說明。這種情況下，畫面上的水平線不只是裝飾，而是在提醒讀者：接下來的內容已經進入另一個段落或主題。

## 一句話理解

`<hr>` 是用來表示「主題轉換或內容分隔」的 HTML 標籤，瀏覽器通常會把它顯示成一條水平線。

## 標籤語意

`<hr>` 的重點不是「畫一條線」，而是「這裡有一個主題上的分隔」。

瀏覽器預設會把 `<hr>` 顯示為水平線，所以初學者常把它當成視覺裝飾工具。但從 HTML 語意來看，它應該用在內容結構真的需要分隔的地方。

## 基本語法

`<hr>` 是單標籤，不需要結束標籤，也不需要包住其他內容。

```html
<hr />
```

在 HTML 中也常看到以下寫法：

```html
<hr>
```

兩種寫法在 HTML 文件中都能運作。若專案慣例使用自閉合風格，可以寫成 `<hr />`；若專案使用一般 HTML5 風格，也可以寫成 `<hr>`。

## 範例拆解

下面的例子用 `<hr>` 分隔兩個不同主題的內容：

```html
<h2>課程介紹</h2>
<p>這門課會帶你認識 HTML 的常用標籤與文件結構。</p>

<hr />

<h2>上課前準備</h2>
<p>請先安裝瀏覽器與程式碼編輯器，並建立一個練習資料夾。</p>
```

這段 HTML 可以這樣理解：

- 第一組 `<h2>` 與 `<p>` 在說明課程內容。
- `<hr />` 表示前後內容進入不同主題。
- 第二組 `<h2>` 與 `<p>` 改成說明上課前要準備什麼。

如果拿掉 `<hr />`，內容仍然可以閱讀；但保留它可以讓主題切換更清楚，尤其是在長文章或教學筆記中。

## 常見錯誤

### 只為了畫線而使用 `<hr>`

如果只是想在區塊底部加一條裝飾線，不代表內容真的發生主題切換，這時不適合使用 `<hr>`。

```html
<!-- 不建議：只是為了視覺線條 -->
<div class="card">
  <h2>最新消息</h2>
  <hr />
  <p>今天新增三篇文章。</p>
</div>
```

這種情況通常應該交給 CSS 處理，例如用 `border-top` 或 `border-bottom` 做視覺分隔。HTML 負責表達內容結構，CSS 負責視覺樣式。

### 把 `<hr>` 寫成成對標籤

`<hr>` 是單標籤，不需要這樣寫：

```html
<!-- 不建議 -->
<hr></hr>
```

正確寫法是：

```html
<hr />
```

或：

```html
<hr>
```

## 實務判斷準則

使用 `<hr>` 前，可以先問自己一個問題：前後內容是否真的進入不同主題？

| 情境 | 是否適合 `<hr>` | 原因 |
| --- | --- | --- |
| 文章從正文切換到補充說明 | 適合 | 前後內容有明確主題分隔 |
| 教學筆記從概念說明切換到延伸閱讀 | 適合 | 可幫助讀者辨識段落轉換 |
| 卡片標題下方想加一條裝飾線 | 不適合 | 只是視覺樣式，應用 CSS |
| 表單欄位之間想增加排版間距 | 不適合 | 屬於版面設計，不是主題分隔 |

## 重點整理

- `<hr>` 用來表示主題轉換或內容分隔。
- `<hr>` 是單標籤，不需要包住文字，也不需要結束標籤。
- 瀏覽器通常會把 `<hr>` 顯示成水平線，但它的 HTML 語意不是單純裝飾。
- 如果只是為了畫線或排版，應該使用 CSS，而不是濫用 `<hr>`。
