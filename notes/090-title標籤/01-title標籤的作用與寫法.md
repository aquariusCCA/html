# title 標籤：設定網頁的文件標題

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 `<title>` 標籤在 HTML 文件中的作用。
- 知道 `<title>` 應該寫在 `<head>` 裡。
- 分辨 `<title>`、`<h1>` 與 `title` 屬性的差異。
- 為一個網頁寫出清楚、能反映主題的文件標題。

## 使用情境

當你打開一個網站時，瀏覽器分頁上通常會顯示該頁面的名稱。例如搜尋網站、購物網站或文章頁，都會用一段文字告訴使用者「這一頁是什麼」。

這段顯示在瀏覽器分頁、書籤，以及搜尋結果標題附近的文字，通常就是由 HTML 的 `<title>` 標籤設定的。

如果一個網頁沒有清楚的 `<title>`，使用者可能只看到籠統的分頁名稱；搜尋引擎也比較難判斷這個頁面的主題。

## 一句話理解

`<title>` 是用來設定「整份 HTML 文件標題」的標籤，它描述的是這一頁的主題。

## title 標籤的語意

`<title>` 不負責顯示頁面正文中的大標題，而是描述整個 HTML 文件本身。

它常見的影響位置包含：

- 瀏覽器分頁標題。
- 使用者加入書籤時的預設名稱。
- 搜尋結果中可能出現的頁面標題。
- 瀏覽器與搜尋引擎理解頁面主題時的參考資訊。

因此，`<title>` 不是裝飾文字，而是網頁文件中非常重要的描述資訊。

## 基本語法

`<title>` 必須放在 `<head>` 區塊內，文字寫在開始標籤與結束標籤之間。

```html
<head>
  <title>這是我的網站</title>
</head>
```

這裡的 `這是我的網站` 就是這份 HTML 文件的標題。

## 完整範例

```html
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>小明的個人網站</title>
</head>
<body>
  <h1>小明的個人網站</h1>
  <p>這裡介紹我的作品與學習紀錄。</p>
</body>
</html>
```

## 範例拆解

- `<!DOCTYPE html>`：告訴瀏覽器這是一份 HTML5 文件。
- `<html lang="zh-Hant">`：整份 HTML 文件的根元素，並標示主要語言。
- `<head>`：放置給瀏覽器、搜尋引擎讀取的文件資訊。
- `<meta charset="UTF-8">`：設定文件編碼，避免中文顯示異常。
- `<title>小明的個人網站</title>`：設定這份文件在分頁、書籤與搜尋結果中可能出現的標題。
- `<body>`：放置使用者會在頁面上看到的內容。
- `<h1>`：頁面正文中的主標題，會顯示在網頁畫面上。

注意：範例中的 `<title>` 與 `<h1>` 文字可以相同，也可以不同。`<title>` 是文件標題，`<h1>` 是頁面內容中的主標題。

## title 標籤的位置與內容

`<title>` 通常不需要額外屬性，最重要的是位置與內容：

- 位置：寫在 `<head>` 裡。
- 數量：一份 HTML 文件通常只需要一個 `<title>`。
- 內容：應該清楚描述目前頁面的主題。

例如：

```html
<title>HTML title 標籤教學</title>
```

比下面這種寫法更具體：

```html
<title>首頁</title>
```

如果網站有多個頁面，每個頁面的 `<title>` 應該反映各自的內容，而不是全部都叫「首頁」或「我的網站」。

## 常見錯誤

### 錯誤一：把 title 寫在 body 裡

```html
<body>
  <title>我的網站</title>
  <h1>我的網站</h1>
</body>
```

`<title>` 應該屬於文件資訊，所以要放在 `<head>` 裡。把它寫進 `<body>`，代表你把文件描述資訊放錯位置，瀏覽器可能會修正或忽略這種錯誤結構。

正確寫法：

```html
<head>
  <title>我的網站</title>
</head>
<body>
  <h1>我的網站</h1>
</body>
```

### 錯誤二：把 title 標籤和 h1 標籤混在一起

`<title>` 和 `<h1>` 都可能出現「頁面標題」文字，但它們的責任不同：

| 標籤 | 位置 | 作用 |
| --- | --- | --- |
| `<title>` | `<head>` 內 | 設定整份 HTML 文件的標題 |
| `<h1>` | `<body>` 內 | 顯示頁面正文中的主標題 |

如果想讓使用者在頁面上看到大標題，應該使用 `<h1>`，不是 `<title>`。

### 錯誤三：誤以為 title 屬性就是 title 標籤

HTML 中也有一個 `title` 屬性，例如：

```html
<abbr title="HyperText Markup Language">HTML</abbr>
```

這裡的 `title` 是屬性，不是 `<title>` 標籤。它用來補充某個元素的提示資訊，不能用來設定整份文件的標題。

### 錯誤四：標題太籠統或堆砌關鍵字

不建議這樣寫：

```html
<title>網站</title>
```

也不建議這樣寫：

```html
<title>HTML HTML教學 HTML入門 HTML標籤 HTML學習</title>
```

前者太籠統，使用者不知道頁面內容；後者像是在堆砌關鍵字，閱讀體驗差，也不利於維護。

較好的寫法：

```html
<title>HTML title 標籤的作用與寫法</title>
```

## 實務判斷準則

撰寫 `<title>` 時，可以用這幾個問題檢查：

- 這個標題是否能讓使用者知道目前頁面在講什麼？
- 如果使用者把這頁加入書籤，這個名稱是否容易辨識？
- 如果網站有很多頁，這一頁的標題是否和其他頁有所區分？
- 標題是否自然可讀，而不是只為了塞關鍵字？

對初學者來說，可以先採用這種格式：

```text
頁面主題 - 網站名稱
```

例如：

```html
<title>作品集 - 小明的個人網站</title>
```

## 重點整理

- `<title>` 用來設定整份 HTML 文件的標題。
- `<title>` 應該寫在 `<head>` 裡，不是 `<body>` 裡。
- `<title>` 常會影響瀏覽器分頁、書籤與搜尋結果中的標題顯示。
- `<title>` 和 `<h1>` 不同：前者是文件標題，後者是頁面正文主標題。
- 好的 `<title>` 應該清楚、具體，能反映目前頁面的主題。

2. 如果你想在網頁畫面上顯示一個大標題，應該使用 `<title>` 還是 `<h1>`？
3. `title` 屬性和 `<title>` 標籤有什麼不同？
4. 對一篇介紹 HTML 圖片標籤的教學頁，下面哪個 `<title>` 比較合適？為什麼？

```html
<title>HTML img 標籤：插入圖片與 alt 文字</title>
```
