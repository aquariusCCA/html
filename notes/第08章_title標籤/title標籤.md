# HTML title 標籤

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `<title>` 標籤的用途。
- 正確將 `<title>` 寫在 HTML 文件的 `<head>` 裡。
- 理解 `<title>` 與瀏覽器分頁標題、搜尋引擎結果之間的關係。
- 分辨 `<title>` 與頁面內標題，例如 `<h1>`，在用途上的差異。
- 避免空白、重複、過度堆疊關鍵字等常見錯誤。

## 2. 本章定位

`<title>` 是 HTML 文件中最基本的 metadata（中繼資料）之一，通常放在 `<head>` 區域，用來設定整份網頁文件的標題。

它不會像 `<h1>` 或 `<p>` 一樣出現在頁面主內容中，但會影響瀏覽器分頁顯示、書籤名稱，以及搜尋引擎對頁面主題的理解。

> 來源重點：原始資料說明 `<title>` 可以設定網站的專屬標題，並指出它有兩個主要作用：告訴訪客頁面或網站主題，以及提供搜尋引擎索引時理解主題的依據。

## 3. 核心概念

### 3.1 `<title>` 是什麼

`<title>` 用來定義 HTML 文件的標題。這個標題代表整個頁面的主題，而不是頁面內容中的某一段文字。

常見顯示位置包括：

| 位置 | 說明 |
|---|---|
| 瀏覽器分頁 | 多數瀏覽器會將 `<title>` 內容顯示在分頁上。 |
| 書籤或收藏 | 使用者收藏頁面時，瀏覽器通常會使用 `<title>` 作為預設名稱。 |
| 搜尋結果 | 搜尋引擎常會參考 `<title>` 作為結果標題，但不保證完全照原文顯示。 |

### 3.2 `<title>` 放在 `<head>` 裡

`<title>` 屬於描述文件本身的資訊，所以應放在 `<head>` 中，而不是放在 `<body>` 的可見內容區域。

基本結構如下：

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>這是我們的專屬網站</title>
</head>
<body>
  <h1>歡迎來到我的網站</h1>
</body>
</html>
```

在這個範例中：

- `<title>` 設定的是整份 HTML 文件的標題。
- `<h1>` 是頁面內容中使用者看得見的主要標題。
- 兩者可以相近，但用途不同，不應互相取代。

### 3.3 `<title>` 與頁面主題

好的 `<title>` 應該清楚描述頁面主題。它可以幫助使用者在多個分頁中辨認頁面，也能幫助搜尋引擎理解頁面內容。

例如：

```html
<title>HTML title 標籤教學</title>
```

這個標題比「首頁」或「我的網站」更清楚，因為它直接說明頁面內容是關於 HTML 的 `<title>` 標籤。

> 補充說明：搜尋引擎可能參考 `<title>` 顯示搜尋結果標題，但實際顯示內容可能會因搜尋關鍵字、頁面內容、網站品質或搜尋引擎判斷而被改寫。因此，`<title>` 很重要，但不是唯一的 SEO 因素。

## 4. 語法與基本用法

`<title>` 是雙標籤元素，需要開始標籤與結束標籤：

```html
<title>頁面標題</title>
```

它的內容應該是純文字，通常不會放其他 HTML 標籤。

完整範例：

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>我的第一個 HTML 網頁</title>
</head>
<body>
  <h1>我的第一個 HTML 網頁</h1>
  <p>這是一段頁面內容。</p>
</body>
</html>
```

撰寫 `<title>` 時可以遵守以下原則：

| 原則 | 說明 |
|---|---|
| 清楚描述頁面主題 | 讓使用者與搜尋引擎能快速理解此頁內容。 |
| 每個頁面使用不同標題 | 避免整個網站所有頁面都叫「首頁」或同一個名稱。 |
| 避免過度冗長 | 太長的標題在分頁或搜尋結果中可能被截斷。 |
| 避免堆疊關鍵字 | 標題應自然可讀，不應只塞滿搜尋關鍵字。 |

## 5. 範例說明

### 範例：網站首頁標題

```html
<title>小林前端筆記｜HTML、CSS 與 JavaScript 學習整理</title>
```

這個標題適合用於網站首頁，因為它同時包含網站名稱與網站主題。

### 範例：文章頁標題

```html
<title>HTML title 標籤的作用與寫法｜小林前端筆記</title>
```

這個標題適合用於單篇教學文章，因為它先說明文章主題，再補上網站名稱。

實務上，首頁與文章頁的標題寫法通常不同：

| 頁面類型 | 標題重點 |
|---|---|
| 首頁 | 網站名稱與網站定位。 |
| 文章頁 | 文章主題與網站名稱。 |
| 商品頁 | 商品名稱、品牌或分類。 |
| 後台頁面 | 功能名稱與系統名稱。 |

## 6. 實務應用場景

在實務開發中，`<title>` 常見於以下情境：

- 網站首頁設定品牌名稱與網站主題。
- 部落格或文件網站的每篇文章設定不同標題。
- 電商商品頁顯示商品名稱。
- 後台系統依目前功能切換分頁標題，例如「訂單管理 - 管理後台」。
- JavaScript 或前端框架在單頁應用程式中依路由更新 `document.title`。

例如，使用 JavaScript 動態更新分頁標題：

```js
document.title = "會員資料 - 管理後台";
```

> 補充說明：即使使用 Vue、React 或其他前端框架，最後仍會渲染成瀏覽器中的 DOM。頁面標題仍需要被正確設定，單頁應用程式也應在切換頁面狀態時更新標題。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 沒有寫 `<title>` | 使用者與搜尋引擎較難判斷頁面主題 | 在 `<head>` 中加入清楚的 `<title>` |
| 將 `<title>` 放在 `<body>` | `<title>` 是文件中繼資訊，不是頁面可見內容 | 將 `<title>` 放到 `<head>` 內 |
| 所有頁面都使用相同標題 | 使用者無法從分頁或搜尋結果分辨頁面 | 每個頁面依內容撰寫獨立標題 |
| 標題只寫「首頁」或「Untitled」 | 標題資訊不足，無法表達頁面主題 | 寫出網站名稱與頁面核心內容 |
| 為 SEO 堆疊大量關鍵字 | 文字不自然，也不利於使用者理解 | 使用自然、準確、可讀的標題 |
| 把 `<title>` 當成頁面主標題 | `<title>` 不會顯示在頁面主要內容中 | 頁面內主標題應使用 `<h1>` |

### 錯誤範例

```html
<body>
  <title>HTML title 標籤教學</title>
  <h1>HTML title 標籤教學</h1>
</body>
```

問題：`<title>` 不應放在 `<body>` 中。它描述的是整份文件，應放在 `<head>` 裡。

### 正確範例

```html
<head>
  <meta charset="UTF-8">
  <title>HTML title 標籤教學</title>
</head>
<body>
  <h1>HTML title 標籤教學</h1>
</body>
```

修正重點：`<title>` 放在 `<head>` 中，頁面可見主標題則用 `<h1>` 表達。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| `<head>` | `<title>` 應放在 `<head>` 中，作為文件 metadata。 |
| `<meta>` | `<meta>` 與 `<title>` 都屬於描述文件資訊的重要標籤。 |
| `<h1>` | `<h1>` 是頁面可見內容中的最高層級標題，不能取代 `<title>`。 |
| SEO | 搜尋引擎常參考 `<title>` 理解頁面主題，但 SEO 還包含內容品質、語意結構、效能與連結等因素。 |
| 瀏覽器分頁 | 瀏覽器通常會把 `<title>` 顯示在分頁或視窗標題上。 |
| JavaScript | 可以透過 `document.title` 讀取或更新頁面標題。 |

## 9. 重點整理

- `<title>` 用來設定整份 HTML 文件的標題。
- `<title>` 應寫在 `<head>` 裡，不應放在 `<body>` 中。
- `<title>` 常會顯示在瀏覽器分頁、書籤名稱與搜尋結果中。
- `<title>` 與 `<h1>` 不同，前者是文件標題，後者是頁面內容標題。
- 好的 `<title>` 應該清楚、具體、自然可讀，並符合頁面主題。
- SEO 不只取決於 `<title>`，還需要搭配內容品質、語意結構與整體網站品質。

## 11. 延伸補充

> 補充說明：瀏覽器會將 HTML 解析成 DOM。`<title>` 對應到文件的標題資訊，可以透過 JavaScript 的 `document.title` 讀取或修改。

```js
console.log(document.title);
document.title = "新的頁面標題";
```

在一般靜態頁面中，直接在 HTML 的 `<head>` 內寫好 `<title>` 即可。若是單頁應用程式，因為切換畫面時可能不會重新載入整份 HTML，通常需要在路由切換時同步更新 `document.title`，讓瀏覽器分頁與使用者目前所在頁面保持一致。
