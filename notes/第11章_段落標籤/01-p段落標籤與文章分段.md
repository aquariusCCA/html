# p 段落標籤與文章分段

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `<p>` 標籤用來表示文章或文字內容中的段落。
- 知道什麼情況適合使用 `<p>`。
- 能寫出多個段落組成的 HTML 內容。
- 分辨段落語意、換行與外觀間距的差異。

## 問題情境

在新聞、文章、商品介紹、部落格或教學文件中，文字通常不會全部擠在同一大段裡。讀者需要透過段落停頓來理解內容的層次與節奏。

HTML 的 `<p>` 標籤就是用來描述這種「一段完整文字」的語意。它不是單純讓文字中間多一點空白，而是告訴瀏覽器、搜尋引擎、輔助科技和維護程式碼的人：這裡是一個段落。

## 一句話理解

`<p>` 是用來表示段落的 HTML 標籤，適合包住文章正文、說明文字或一段完整的敘述內容。

## 標籤語意

`p` 是 paragraph 的縮寫，語意是「段落」。

當一段文字在內容上已經形成一個完整意思，就可以用一組 `<p>` 和 `</p>` 包起來：

```html
<p>這是一個段落。</p>
```

如果文章有多個段落，就使用多組 `<p>`：

```html
<p>第一段文字說明這篇文章的背景與主題。</p>

<p>第二段文字接著補充更細的內容，讓讀者逐步理解。</p>
```

這樣寫的重點是讓 HTML 結構反映內容本身，而不是只追求畫面看起來有分隔。

## 瀏覽器預設樣式

在瀏覽器的預設樣式下，`<p>` 通常會有這些特點：

- 每個段落通常會獨佔一個區塊。
- 段落上下通常會有預設外距，所以段落之間看起來有空隙。
- 段落文字會依照所在容器寬度自動換行。

這些只是預設外觀。實務上可以用 CSS 調整段落的字體、行高、間距與寬度，但 HTML 中仍然應該用 `<p>` 表示真正的段落語意。

## 基本範例

以下是一段文章內容的 HTML 寫法：

```html
<article>
  <h1>前端學習筆記</h1>

  <p>
    學習 HTML 時，第一步不是記住所有標籤，而是理解每個標籤用來描述什麼內容。
  </p>

  <p>
    當一段文字形成完整意思時，可以使用 p 標籤把它標記成段落，讓文件結構更清楚。
  </p>
</article>
```

這個範例中：

- `<article>` 表示一篇獨立文章內容。
- `<h1>` 表示文章主標題。
- 每一組 `<p>` 表示一個段落。
- 兩個段落各自包在不同的 `<p>` 中，而不是把整篇文章全部塞進同一個 `<p>`。

## 段落與換行的差異

初學者常把「段落」和「換行」混在一起。

段落代表內容上的分段；換行只代表文字顯示時換到下一行。一般文章正文中，如果是不同段落，應該使用多個 `<p>`；不要只靠空白或換行製造段落效果。

不建議：

```html
<p>
  第一段文字。
  第二段文字。
</p>
```

較好的寫法：

```html
<p>第一段文字。</p>
<p>第二段文字。</p>
```

HTML 原始碼中的換行和縮排主要是給開發者閱讀；瀏覽器會根據容器寬度自動排版段落文字。

## 常見錯誤

### 錯誤一：用 `<br>` 取代段落

不建議：

```html
第一段文字。<br><br>
第二段文字。
```

較好的寫法：

```html
<p>第一段文字。</p>
<p>第二段文字。</p>
```

`<br>` 適合用在地址、詩句、歌詞這類內容本身需要換行的位置；一般文章分段應該使用 `<p>`。

### 錯誤二：用 `<p>` 包住不屬於段落的結構

不建議：

```html
<p>
  文章重點：
  <ul>
    <li>理解標籤語意</li>
    <li>依內容選擇標籤</li>
  </ul>
</p>
```

較好的寫法：

```html
<p>文章重點：</p>
<ul>
  <li>理解標籤語意</li>
  <li>依內容選擇標籤</li>
</ul>
```

`<p>` 適合放文字型段落。清單、表格、區塊標題等結構應該使用對應的 HTML 標籤，而不是硬塞進段落裡。

### 錯誤三：為了控制間距而多包空段落

不建議：

```html
<p>第一段文字。</p>
<p></p>
<p>第二段文字。</p>
```

較好的寫法：

```html
<p>第一段文字。</p>
<p>第二段文字。</p>
```

如果想調整段落間距，應該使用 CSS：

```css
p {
  margin-bottom: 16px;
}
```

HTML 負責內容語意，CSS 負責外觀與排版。

## 實務使用建議

撰寫正文內容時，可以先用內容邏輯判斷段落：

- 這段文字是否表達一個相對完整的意思？
- 下一段是否切換到新的補充、例子或觀點？
- 如果只看 HTML 結構，是否能看出文字是如何分段的？

在文章頁面中，常見結構會像這樣：

```html
<article>
  <h1>文章標題</h1>

  <p>第一段：介紹主題。</p>
  <p>第二段：補充背景。</p>
  <p>第三段：提出結論或下一步。</p>
</article>
```

這樣的結構對讀者、開發者、搜尋引擎和輔助科技都比較清楚，也比較容易在之後用 CSS 統一調整段落樣式。

## 重點整理

- `<p>` 是 HTML 的段落標籤，用來標記一段完整文字。
- 新聞、文章、說明文字和教學內容都常使用 `<p>` 分段。
- 瀏覽器預設會讓段落之間有空隙，文字也會依容器寬度自動換行。
- 段落語意不等於單純換行；一般正文分段應使用多個 `<p>`。
- 不要用空 `<p>` 控制間距，外觀應該交給 CSS。
- `<p>` 適合文字段落，不應用來包住清單、表格或其他大型結構。
