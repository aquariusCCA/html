# pre 元素：保留文字原本的排版

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 `<pre>` 元素的用途與語意。
- 理解為什麼一般 HTML 文字中的多個空格與換行通常不會照原樣顯示。
- 使用 `<pre>` 保留純文字、CSS 或程式碼片段的縮排與換行。
- 知道展示 HTML 標籤本身時，必須先轉義 `<`、`>` 等字元。

## 使用情境

一般 HTML 內容會把連續空白、換行與縮排折疊成較簡單的顯示結果。這對文章段落很方便，因為作者不用擔心原始碼中的排版會影響畫面。

但有些內容的格式本身就是資訊的一部分，例如：

- 純文字格式的資料。
- 需要保留縮排的 CSS 片段。
- 要讓讀者看懂層級關係的程式碼。
- 需要照原樣呈現的範例文字。

這時候就適合使用 `<pre>`。

## 一句話理解

`<pre>` 是用來呈現「預先排好格式的文字」的 HTML 元素；它會保留內容中的空格、Tab 與換行。

## 標籤語意

`pre` 來自 preformatted text，意思是「預格式化文本」。它代表這段文字已經有自己的排版格式，瀏覽器應該盡量照原本的空白與換行呈現。

瀏覽器顯示 `<pre>` 內容時，通常會使用等寬字體。等寬字體的每個字元寬度一致，比較適合展示程式碼或需要對齊的文字。

## 基本語法

```html
<pre>
body {
  background: #fff;
  font: 12px/24px Arial, sans-serif;
}
</pre>
```

這段內容放在 `<pre>` 裡，縮排與換行會被保留下來。

## 範例：一般內容與 pre 的差異

```html
<p>
body {
  background: #fff;
  font: 12px/24px Arial, sans-serif;
}
</p>

<hr>

<pre>
body {
  background: #fff;
  font: 12px/24px Arial, sans-serif;
}
</pre>
```

## 範例拆解

- 第一段使用 `<p>`：它是一般段落元素，瀏覽器會把多餘空白與換行折疊，因此原本的程式碼排版不會完整保留。
- `<hr>`：用來在畫面上分隔兩個展示結果，方便比較。
- 第二段使用 `<pre>`：CSS 片段中的換行與縮排會照內容原樣顯示，比較適合閱讀。

如果你要讓讀者看一段排版好的文字或程式碼，`<pre>` 比一般段落更合適。

## 展示 HTML 標籤時要轉義

`<pre>` 會保留空白與換行，但它不會讓 HTML 標籤自動變成純文字。

如果你直接在 `<pre>` 裡寫：

```html
<pre>
<h1>標題</h1>
</pre>
```

瀏覽器仍然會把 `<h1>` 當成 HTML 標籤解析。若目標是「顯示這段 HTML 原始碼」，就要把特殊字元轉義：

```html
<pre>
&lt;h1&gt;標題&lt;/h1&gt;
</pre>
```

常見轉義如下：

| 原字元 | 轉義寫法 | 用途 |
| --- | --- | --- |
| `<` | `&lt;` | 避免被解析成標籤開頭 |
| `>` | `&gt;` | 避免被解析成標籤結尾 |
| `&` | `&amp;` | 避免被解析成字元實體開頭 |

## 常見錯誤

**錯誤一：用一般段落展示程式碼**

```html
<p>
body {
  background: #fff;
}
</p>
```

這樣寫不適合展示程式碼，因為段落元素不負責保留程式碼排版。讀者看到的縮排與換行可能和原始內容不同。

較合適的寫法是：

```html
<pre>
body {
  background: #fff;
}
</pre>
```

**錯誤二：在 `<pre>` 中直接寫要展示的 HTML 標籤**

```html
<pre>
<div class="card">內容</div>
</pre>
```

這段可能會被瀏覽器當成真正的 `<div>` 元素解析，而不是顯示字面上的 HTML 原始碼。要展示標籤文字時，應改成：

```html
<pre>
&lt;div class="card"&gt;內容&lt;/div&gt;
</pre>
```

## 實務判斷

當內容的「空白、縮排、換行」會影響閱讀或理解時，可以考慮使用 `<pre>`。

適合使用 `<pre>` 的情境：

- 展示一小段 CSS 或純文字程式碼。
- 呈現需要固定換行與縮排的文字。
- 比較格式保留前後的差異。

不適合只因為想讓文字換行就使用 `<pre>`。如果只是一般文章排版，應該使用段落、標題、清單等更符合內容語意的元素，再交給 CSS 控制視覺樣式。

## 重點整理

- `<pre>` 用來表示預格式化文本。
- `<pre>` 會保留內容中的空格、Tab 與換行。
- `<pre>` 內容通常以等寬字體顯示。
- 展示 CSS、純文字或程式碼片段時，`<pre>` 很適合用來保留排版。
- 如果要展示 HTML 標籤本身，必須把 `<`、`>` 等字元轉義。
