# 網頁與 HTML 入門

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨網站、網頁與 HTML 文件的關係。
- 說明 HTML 為什麼被稱為超文本標記語言。
- 理解瀏覽器如何把 HTML 轉換成使用者看到的網頁。

## 問題情境

當你打開一個網站時，看到的不是一個單純的圖片，而是一個由文字、圖片、影片、音訊、連結與互動功能組成的頁面。

這些內容需要被整理成瀏覽器能理解的結構。HTML 的任務，就是用標籤描述這些內容是什麼，以及它們在頁面中的關係。

## 一句話理解

網頁是網站中的一頁；HTML 是用標籤描述網頁內容與結構的標記語言。

## 網頁與網站是什麼

網站是一組互相關聯的網頁集合，通常用來展示某個主題、服務、產品或系統內容。

網頁則是網站中的一個頁面。它常見的形式是一份 HTML 文件，副檔名通常是 `.html` 或 `.htm`。例如：

```text
about.html
product.html
contact.html
```

一個網站可以只有一個網頁，也可以由許多網頁組成。例如公司網站可能包含首頁、產品介紹頁、聯絡我們頁與登入頁。

網頁中常見的內容包含：

- 文字。
- 圖片。
- 音訊與影片。
- 超連結。
- 表單與互動區塊。

這些內容不會憑空變成網頁畫面，而是需要透過 HTML 標籤描述，再交給瀏覽器解析。

## HTML 是什麼

HTML 是 HyperText Markup Language，常譯為超文本標記語言。

它有兩個重點：

| 概念 | 說明 |
| --- | --- |
| 超文本 | 不只描述普通文字，也可以連結圖片、音訊、影片、動畫或其他文件。 |
| 標記語言 | 使用一組標籤標記內容，讓瀏覽器知道每段內容的語意與結構。 |

例如：

```html
<h1>我的第一個網頁</h1>
<p>這是一段介紹文字。</p>
<a href="https://example.com">前往範例網站</a>
```

這段 HTML 告訴瀏覽器：

- `<h1>` 是頁面主要標題。
- `<p>` 是段落文字。
- `<a>` 是可以連到其他資源的超連結。

HTML 的核心不是讓畫面變漂亮，而是描述內容本身的意義。畫面樣式通常交給 CSS，互動行為通常交給 JavaScript。

## HTML 不是程式語言

HTML 常被初學者誤認為程式語言，但它主要不是用來寫邏輯判斷、資料計算或流程控制。

HTML 是標記語言。它負責標記內容，例如：

```html
<h2>章節標題</h2>
<p>這是一段說明。</p>
<img src="photo.jpg" alt="照片說明">
```

這些標籤會讓瀏覽器理解：

- 哪段內容是標題。
- 哪段內容是段落。
- 哪個位置要放圖片。

之後如果要控制顏色、排版或動畫，會使用 CSS；如果要處理點擊、資料更新或表單驗證，會使用 JavaScript。

## 瀏覽器如何形成網頁

前端開發者會撰寫 HTML、CSS、JavaScript 等檔案。使用者打開網頁時，瀏覽器會讀取這些資料，解析後再渲染成畫面。

可以先用簡化流程理解：

```text
開發者撰寫 HTML 等前端程式碼
  -> 瀏覽器讀取程式碼
  -> 瀏覽器解析 HTML 結構
  -> 瀏覽器渲染成可閱讀、可操作的網頁
```

瀏覽器在這裡扮演轉換器的角色。它把開發者寫的標籤、文字與資源路徑，轉換成使用者看得懂的頁面。

## 實務理解方式

學 HTML 時，可以把自己寫的內容分成兩層看：

| 層次 | 問題 | HTML 的任務 |
| --- | --- | --- |
| 內容 | 頁面裡有什麼？ | 標記標題、段落、圖片、連結等內容。 |
| 結構 | 內容之間是什麼關係？ | 建立標題層級、區塊順序、父子元素關係。 |

例如一篇文章頁面至少可能需要：

```html
<h1>文章標題</h1>
<p>文章摘要。</p>
<p>文章內文。</p>
```

這比單純把所有文字塞在一起更清楚，因為瀏覽器、搜尋引擎、螢幕閱讀器與維護這份程式碼的人，都能更容易理解頁面結構。

## 常見錯誤

### 把網站和網頁混為一談

網站是一組相關網頁的集合，網頁是其中的一頁。討論需求時，應先確認是在做整個網站，還是在做某一個頁面。

### 只把 HTML 當成畫面效果

HTML 的重點是語意與結構，不是顏色和排版。樣式應交給 CSS 處理。

### 以為瀏覽器只是顯示文字

瀏覽器會解析 HTML 標籤、資源路徑與文件結構，再把結果渲染成網頁。HTML 寫得清楚，瀏覽器與其他工具才更容易正確理解內容。

## 重點整理

- 網站是一組相關網頁的集合，網頁通常是一份 HTML 文件。
- HTML 是超文本標記語言，用標籤描述網頁內容與結構。
- HTML 不是程式語言，主要不負責邏輯運算。
- 瀏覽器會解析與渲染 HTML，最後形成使用者看到的網頁。
- 學 HTML 時，應先重視內容語意與結構關係。
