# HTML5 版本與兼容性

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 HTML5 在教材脈絡中的意義。
- 分辨狹義 HTML5 與廣義 HTML5 的差別。
- 說明學習 HTML5 時為什麼需要注意瀏覽器兼容性。

## 問題情境

你在舊教材、文件或面試題中，常會看到「HTML5 新特性」這個說法。

有些內容確實是 HTML 標準中的新標籤或語意能力；有些內容則把 CSS3、瀏覽器 API、本地儲存、多媒體、圖形技術等現代 Web 能力也一起放進來談。

因此，學 HTML5 時要先知道：對方說的 HTML5，可能是嚴格的 HTML 標準，也可能是較寬泛的現代前端技術集合。

## 一句話理解

HTML5 是現代 HTML 的重要版本脈絡；狹義上指 HTML 標準，廣義上常被用來泛稱一組現代 Web 技術。

## HTML5 是什麼

在這份教材的脈絡中，HTML5 指新一代 HTML 標準。它補足過去 HTML 在語意、表單、多媒體與瀏覽器能力上的不足，讓網頁能更清楚地描述內容，也能支援更完整的現代 Web 應用需求。

常見的 HTML5 相關改進包含：

- 新增語意化標籤。
- 新增表單類型與表單屬性。
- 新增多媒體標籤，減少對外掛技術的依賴。
- 更重視文件語意，對搜尋引擎與輔助技術更友好。
- 對行動裝置與跨平台 Web 應用更友好。

版本演進通常具有向下兼容的設計思路。也就是說，在 HTML5 的文件中，許多早期 HTML 標籤與功能仍可以使用。不過，能使用不代表都推薦使用；正式開發仍應優先選擇語意清楚、符合現代標準的寫法。

## 狹義 HTML5 與廣義 HTML5

閱讀資料時，可以用下面方式分辨：

| 說法 | 範圍 | 例子 |
| --- | --- | --- |
| 狹義 HTML5 | HTML 標準本身 | 語意化標籤、表單類型、`audio`、`video`。 |
| 廣義 HTML5 | 現代 Web 技術集合的非正式說法 | 本地儲存、設備能力、圖形、多媒體、CSS3、瀏覽器 API。 |

廣義說法在教學或市場語境中很常見，但它不等於標準意義上的 HTML 本身。

實務上要避免把所有現代瀏覽器能力都說成 HTML。例如排版樣式多半屬於 CSS，互動邏輯多半屬於 JavaScript，瀏覽器提供的能力則可能是 Web API。

## HTML5 的常見優勢

HTML5 讓現代網頁更容易表達清楚的結構與語意。

常見優勢包含：

- 語意化更清楚，讓頁面結構更容易被理解。
- 表單能力更完整，能支援更多輸入類型。
- 多媒體支援更好，可以直接使用 HTML 標籤嵌入音訊與影片。
- 對搜尋引擎、輔助技術與維護者更友好。
- 更適合行動裝置與跨平台 Web 頁面。

這些優勢不是要求你一次背下所有新特性，而是提醒你：學 HTML 時要重視語意、結構與實際支援情況。

## 兼容性為什麼重要

不同 HTML5 或現代 Web 特性的瀏覽器支援程度不一定相同。

例如：

- 某個新表單輸入類型在不同瀏覽器中可能呈現不同。
- 某個新的語意標籤雖然能被瀏覽器解析，但舊瀏覽器可能支援不完整。
- 某些 Web API 只在特定瀏覽器或版本中可用。

因此，不能只看到「HTML5 支援」就假設所有功能在所有瀏覽器中完全一致。正式開發時，應依照產品目標瀏覽器與使用者環境確認支援情況。

早期資料常把 Chrome、Safari、Opera、Firefox 與 IE 放在一起討論兼容性。閱讀這類資料時，要知道 IE 主要是歷史對照；現代開發不應把 IE 當成主要基準。

## 學習時的取捨

入門階段不需要一次掌握所有 HTML5 特性，可以先按實用程度學習：

```text
基礎文件結構
  -> 常用內容標籤
  -> 語意化結構標籤
  -> 表單與多媒體
  -> 需要時再查詢特定特性兼容性
```

這樣能避免一開始就被大量版本名詞和功能清單淹沒。

## 常見錯誤

### 把 HTML5 當成所有前端技術的總稱

HTML5 可以在廣義語境中泛稱現代 Web 技術，但正式學習時仍要分清楚 HTML、CSS、JavaScript 與 Web API 的責任。

### 只追求新特性

新特性不一定適合所有專案。實務上要同時考慮語意、兼容性、可維護性與團隊熟悉度。

### 忽略舊教材的時代背景

舊教材中對瀏覽器、標準與兼容性的描述，可能反映當時的環境。閱讀時應保留核心觀念，但不要把歷史瀏覽器當成現代主要基準。

## 重點整理

- HTML5 是現代 HTML 的重要版本脈絡。
- 狹義 HTML5 指 HTML 標準；廣義 HTML5 常泛稱現代 Web 技術集合。
- HTML5 強化語意化、表單、多媒體與現代 Web 能力。
- 不同功能的瀏覽器支援程度不同，實務開發需要確認兼容性。
- 入門學習應先掌握常用、穩定、語意清楚的 HTML 寫法。
