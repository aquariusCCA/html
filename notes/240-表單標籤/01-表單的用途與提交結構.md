# 表單的用途與提交結構

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明網頁為什麼需要表單。
- 分辨表單域、表單控件與提示資訊的角色。
- 看懂 `<form>` 的基本結構與常用提交屬性。
- 知道哪些表單資料會在提交時被送出。

## 為什麼需要表單

很多網頁不只是展示內容，也需要讓使用者輸入資料。例如登入帳號、註冊會員、搜尋商品、填寫聯絡資訊、送出留言，這些都需要表單。

表單的核心目的可以用一句話理解：

> 表單是網頁用來收集使用者資料，並把資料送到指定位置的 HTML 結構。

如果沒有表單，使用者雖然可以看網頁，但很難把自己的資訊交給網站處理。

## 表單由哪幾部分組成

一個完整表單通常包含三個部分：

- 表單域：用 `<form>` 包住要提交的表單內容。
- 表單控件：讓使用者輸入或選擇資料的元素，例如 `<input>`、`<select>`、`<textarea>`。
- 提示資訊：告訴使用者每個欄位要填什麼，例如文字標籤、說明文字或 placeholder。

![表單組成結構示意圖](../../origin/240-表單標籤/assets/images/form-introduction-img-001-a837a3.png)

表單不是只有一堆輸入框。若沒有清楚的表單域與提示資訊，使用者不知道怎麼填，瀏覽器與後端也不一定知道要怎麼處理資料。

## form 標籤的語意

`<form>` 用來定義一個表單域，也就是一個可以收集並提交資料的區域。

```html
<form action="url地址" method="提交方式" name="表單域的名稱" target="如何打開頁面">
  <!-- 表單控件放在這裡 -->
</form>
```

這段結構的重點不是畫面，而是提交行為。當使用者按下提交按鈕時，瀏覽器會依照 `<form>` 的屬性，把表單裡可提交的資料送出。

## 常用提交屬性

![form 表單域常用屬性表](../../origin/240-表單標籤/assets/images/form-tag-img-001-dedcdd.png)

常見屬性可以這樣理解：

- `action`：資料要送到哪個位址。
- `method`：資料用哪種方式送出，常見為 `get` 或 `post`。
- `name`：表單本身的名稱，方便識別。
- `target`：提交後回應頁面要在哪裡開啟。

初學時可以先抓住一個重點：`action` 決定送到哪裡，`method` 決定怎麼送。

## 哪些資料會被提交

表單提交時，瀏覽器不是把畫面上所有文字都送出，而是收集表單控件的資料。常見規則如下：

- 控件通常要放在 `<form>` 裡，才會跟著這個表單提交。
- 控件通常需要 `name`，後端才知道這筆資料的欄位名稱。
- 控件的值來自使用者輸入、選擇，或元素本身設定的 `value`。
- 被 `disabled` 禁用的控件通常不會被提交。

例如：

```html
<form action="/register" method="post">
  <label for="account">帳號</label>
  <input id="account" name="account" type="text">

  <button type="submit">註冊</button>
</form>
```

使用者在帳號欄位輸入 `amy` 後，提交時會形成類似 `account=amy` 的資料。`account` 來自 `name`，`amy` 來自使用者輸入。

## 常見錯誤

### 只放 input，沒有 form

```html
<input type="text" name="account">
<button>送出</button>
```

這樣畫面上有輸入框與按鈕，但沒有明確的表單域，瀏覽器不知道這一組資料應該如何提交。若需要原生表單提交，就應該使用 `<form>` 包起來。

### 忘記替欄位加上 name

```html
<form action="/login" method="post">
  <input type="text">
  <button type="submit">登入</button>
</form>
```

使用者看得到輸入框，也可以輸入文字，但提交資料缺少欄位名稱，後端很難接收這筆資料。應改成：

```html
<input type="text" name="username">
```

### 誤以為 disabled 欄位也會送出

被 `disabled` 的欄位是不可操作狀態，通常也不會包含在提交資料中。如果某筆資料需要送出但不想讓使用者修改，應重新判斷資料來源與後端處理方式，不要只靠前端禁用。

## 實務判斷準則

- 需要讓使用者輸入並送出資料時，優先考慮 `<form>`。
- 每個要提交的欄位都應該有清楚的 `name`。
- 表單中的按鈕要明確設定 `type`，避免意外提交。
- 表單的視覺排版交給 CSS，資料結構與提交語意交給 HTML。

## 重點整理

- 表單用來收集使用者資料。
- 完整表單通常包含表單域、表單控件與提示資訊。
- `<form>` 定義資料提交的範圍與方式。
- `action` 指定送出位置，`method` 指定提交方式。
- 可提交欄位通常需要 `name`，被 `disabled` 的欄位通常不會提交。
