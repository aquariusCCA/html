# 表單按鈕與提交行為

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨提交、重置與普通按鈕的行為差異。
- 正確使用 `<input>` 與 `<button>` 建立表單按鈕。
- 避免 `<button>` 未設定 `type` 造成意外提交。
- 判斷什麼情境適合使用哪一種按鈕。

## 使用情境

表單通常不只需要輸入欄位，也需要讓使用者執行動作。例如送出表單、清空表單、取得驗證碼、打開說明視窗。這些動作都可能使用按鈕，但按鈕的 `type` 不同，行為也不同。

一句話理解：

> 表單按鈕的 `type` 決定它是提交表單、重置表單，還是只作為一般操作按鈕。

## 提交按鈕 submit

提交按鈕會送出所在 `<form>` 的資料。

```html
<form action="/signup" method="post">
  <input type="text" name="account">
  <input type="submit" value="註冊">
</form>
```

也可以使用 `<button>`：

```html
<form action="/signup" method="post">
  <input type="text" name="account">
  <button type="submit">註冊</button>
</form>
```

兩者都可以提交表單。差別在於 `<input>` 使用 `value` 指定按鈕文字，而 `<button>` 把文字寫在開始與結束標籤之間。

## 重置按鈕 reset

重置按鈕會把表單控件恢復成初始狀態。

```html
<form action="/profile" method="post">
  <input type="text" name="nickname" value="小明">
  <input type="reset" value="重新填寫">
</form>
```

或：

```html
<button type="reset">重新填寫</button>
```

重置不是清空所有欄位，而是回到頁面載入時的初始值。若欄位原本有 `value="小明"`，按下重置後會恢復成 `小明`。

## 普通按鈕 button

普通按鈕不會自動提交或重置表單，通常用於搭配 JavaScript 執行操作。

```html
<button type="button">取得驗證碼</button>
```

也可以使用：

```html
<input type="button" value="取得驗證碼">
```

在現代 HTML 中，若按鈕需要包含圖示、文字或其他 HTML 內容，`<button>` 通常更彈性。

## button 標籤的 type

`<button>` 常見有三種 `type`：

```html
<button type="submit">提交按鈕</button>
<button type="reset">重置按鈕</button>
<button type="button">普通按鈕</button>
```

實務上建議總是明確寫出 `type`。尤其在表單裡，沒有設定 `type` 的 `<button>` 通常會被當成提交按鈕，可能造成意外送出表單。

## 錯誤寫法與正確寫法

### 普通操作按鈕忘記 type

```html
<form action="/login" method="post">
  <input type="text" name="account">
  <button>取得驗證碼</button>
</form>
```

這顆按鈕看起來只是取得驗證碼，但在表單中可能會提交表單。應改成：

```html
<button type="button">取得驗證碼</button>
```

### 把 button 文字寫在 value

```html
<button type="submit" value="送出"></button>
```

`<button>` 的可見文字應寫在標籤內容裡：

```html
<button type="submit">送出</button>
```

`value` 在 `<button>` 中可以作為提交值，但不是用來顯示按鈕文字的主要方式。

### 不必要地使用 reset

重置按鈕會清除使用者已經填寫的內容，容易造成誤觸與挫折。除非表單很短且確實需要重填，否則不一定要提供 reset。

## 實務判斷準則

- 要送出表單：使用 `type="submit"`。
- 要恢復表單初始狀態：使用 `type="reset"`，但謹慎使用。
- 要執行 JavaScript 操作：使用 `type="button"`。
- 在 `<form>` 裡的 `<button>` 一律明確寫 `type`。
- 按鈕文字使用清楚動詞，例如「登入」「送出留言」「取得驗證碼」。

## 重點整理

- `submit` 會提交表單。
- `reset` 會恢復表單初始狀態，不一定是清空。
- `button` 是普通按鈕，通常搭配 JavaScript。
- `<input>` 按鈕用 `value` 顯示文字，`<button>` 把文字放在標籤內容中。
- 表單中的 `<button>` 忘記寫 `type`，可能造成意外提交。
