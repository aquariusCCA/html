# 單選、複選與下拉選項

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨 radio、checkbox、select 的使用時機。
- 正確使用 `name`、`value`、`checked`、`selected`。
- 理解選項控件提交資料的方式。
- 避免把不同選擇情境混用。

## 使用情境

表單中常見的問題不是「讓使用者自由輸入」，而是「讓使用者從固定選項中選」。這時可以依選擇方式挑選控件：

- 只能選一個：使用 radio。
- 可以選多個：使用 checkbox。
- 選項很多且想節省空間：使用 select。

一句話理解：

> 選項型表單控件的核心，是把使用者選到的項目轉成可提交的欄位名稱與欄位值。

## 單選框 radio

當一組選項只能選一個時，使用 `type="radio"`。

```html
<input type="radio" id="gender-female" name="gender" value="female">
<label for="gender-female">女</label>

<input type="radio" id="gender-male" name="gender" value="male">
<label for="gender-male">男</label>
```

範例拆解：

- `type="radio"` 建立單選框。
- 同一組 radio 必須共用相同的 `name`，這樣才會形成多選一。
- `value` 是實際提交的資料值。
- `label` 讓使用者點文字也能選取選項。

如果兩個 radio 的 `name` 不同，瀏覽器會把它們視為不同組，就無法形成互斥選擇。

## 複選框 checkbox

當一組選項可以選多個時，使用 `type="checkbox"`。

```html
<input type="checkbox" id="hobby-reading" name="hobby" value="reading">
<label for="hobby-reading">閱讀</label>

<input type="checkbox" id="hobby-music" name="hobby" value="music">
<label for="hobby-music">音樂</label>

<input type="checkbox" id="hobby-sport" name="hobby" value="sport">
<label for="hobby-sport">運動</label>
```

多個 checkbox 常共用同一個 `name`，表示它們屬於同一類資料。使用者勾選多個項目時，提交結果會包含多個同名欄位值。

## 預設選中 checked

radio 與 checkbox 都可以使用 `checked` 設定預設選中。

```html
<input type="radio" name="plan" value="basic" checked>
<input type="radio" name="plan" value="pro">

<input type="checkbox" name="notice" value="email" checked>
```

預設選中應該用在真的有合理預設值的情境。例如訂閱偏好可以預設勾選，但涉及同意條款或重要授權時，不應草率預設勾選。

## 下拉列表 select 與 option

當選項很多，或希望節省畫面空間時，可以使用 `<select>`。

```html
<label for="province">地區</label>
<select id="province" name="province">
  <option value="heilongjiang">黑龍江</option>
  <option value="liaoning">遼寧</option>
  <option value="jilin">吉林</option>
  <option value="guangdong" selected>廣東</option>
</select>
```

範例拆解：

- `<select>` 定義下拉列表。
- `<option>` 定義每一個選項。
- `select` 的 `name` 是提交欄位名稱。
- `option` 的 `value` 是該選項被選中時提交的值。
- `selected` 表示預設選中的選項。

如果 `<option>` 沒有設定 `value`，瀏覽器通常會提交選項中間的文字。實務上建議明確設定 `value`，讓顯示文字與提交資料可以分開維護。

## 使用時機對照

| 情境 | 建議控件 | 原因 |
| --- | --- | --- |
| 性別、方案、付款方式等只能選一個 | radio | 選項少且需要直接看到所有選項 |
| 興趣、技能、通知方式等可選多個 | checkbox | 使用者可以勾選多個項目 |
| 地區、年份、分類等選項較多 | select | 節省畫面空間 |

## 常見錯誤

### radio 沒有共用 name

```html
<input type="radio" name="male" value="male">男
<input type="radio" name="female" value="female">女
```

這兩個 radio 不會互斥，因為它們不是同一組。應改成：

```html
<input type="radio" name="gender" value="male">男
<input type="radio" name="gender" value="female">女
```

### checkbox 缺少 value

```html
<input type="checkbox" name="hobby">閱讀
```

這樣後端難以判斷使用者勾選的是哪個興趣。應明確給值：

```html
<input type="checkbox" name="hobby" value="reading">閱讀
```

### 用 radio 呈現大量選項

如果有幾十個選項，全部用 radio 會讓畫面很長，使用者也難以掃描。這種情況通常更適合使用 `<select>` 或其他更進階的搜尋選擇介面。

## 重點整理

- radio 用於多選一，同一組要共用 `name`。
- checkbox 用於可選多個項目。
- select 適合選項多且需要節省空間的情境。
- `value` 是提交值，不一定等於畫面文字。
- `checked` 用於 radio/checkbox，`selected` 用於 option。
