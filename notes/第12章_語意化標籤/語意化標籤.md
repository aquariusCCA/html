# HTML 語意化標籤

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 semantic HTML（語意化 HTML）的用途。
- 分辨 `header`、`nav`、`main`、`article`、`section`、`aside`、`footer` 等常見語意化標籤的角色。
- 理解語意化標籤與 `div` 版面寫法的差異。
- 使用語意化標籤建立更清楚的頁面結構。
- 說明語意化 HTML 對閱讀、維護、SEO 與可訪問性的影響。
- 避免只為了畫面排版而濫用語意化標籤。

## 2. 本章定位

本章接續「盒子標籤」之後，說明 HTML5 常見的語意化結構標籤。

在早期或初學階段，頁面版面常大量使用 `div` 搭配 `class` 命名，例如 `div.header`、`div.nav`、`div.footer`。這樣雖然可以完成畫面排版，但 `div` 本身沒有語意，瀏覽器、搜尋引擎、輔助工具與後續維護者都必須依賴 class 名稱或上下文猜測這個區塊的用途。

語意化標籤的重點是讓 HTML 元素直接表達內容角色。也就是說，HTML 不只是在畫面上排出區塊，而是明確告訴閱讀者與機器：這裡是頁首、導覽、主要內容、文章、側邊補充或頁尾。

> 來源重點：原始資料指出，語意化標籤可以改善滿版 `div` 佈局，讓頁面結構更清楚，也有助於搜尋引擎、輔助工具、開發與維護。

## 3. 核心概念

### 3.1 什麼是語意化 HTML

語意化 HTML 是指根據內容的意義選擇合適的 HTML 元素，而不是只根據畫面外觀選擇標籤。

例如：

- 頁面或區塊的頭部內容使用 `header`。
- 主要導覽連結使用 `nav`。
- 可獨立閱讀的文章內容使用 `article`。
- 文件中的主題區塊使用 `section`。
- 與主要內容相關但不是主線的補充內容使用 `aside`。
- 頁面或區塊底部資訊使用 `footer`。

相較於全部使用 `div`，語意化標籤讓 HTML 原始碼本身更像一份有結構的文件。

### 3.2 語意不是預設樣式

語意化標籤的用途不是讓畫面自動變漂亮，也不是取代 CSS 排版。

HTML 負責描述內容與結構；CSS 負責控制視覺呈現。`header`、`nav`、`section`、`footer` 等元素在不同瀏覽器中可能有預設樣式，但選擇它們的主要原因應該是內容語意，而不是預設外觀。

```html
<header>網站頁首</header>
<nav>主要導覽</nav>
<main>主要內容</main>
<footer>頁尾資訊</footer>
```

這段 HTML 的價值在於結構清楚，而不是因為這些標籤本身會產生特殊版面。

### 3.3 語意化標籤可以重複使用

許多語意化標籤可以在同一頁中出現多次。常見情境包括：

- 整個網站可以有一個頁面 `header`。
- 每篇 `article` 內部也可以有自己的 `header` 與 `footer`。
- 不同內容區域可以各自使用 `section`。
- 頁面可以有主要導覽，也可以在側邊欄中有另一組補充導覽。

是否能重複使用，取決於它描述的是哪一個範圍的內容角色，而不是標籤名稱本身。

### 3.4 語意化標籤與 `div` 的差異

`div` 是無語意容器，適合在沒有更精準元素時包住一組內容。語意化標籤則直接表達內容角色。

| 寫法 | 語意清楚度 | 說明 |
|---|---|---|
| `<div class="nav">` | 較弱 | `div` 本身沒有語意，必須看 `class` 才能猜出用途 |
| `<nav>` | 較強 | 元素本身就表示這是一組導覽內容 |
| `<div class="footer">` | 較弱 | 只是一般容器 |
| `<footer>` | 較強 | 表示頁面或區塊的底部資訊 |

> 補充說明：如果有原生 HTML 元素可以表達語意，通常應優先使用原生元素；不要先用 `div` 再靠 `class` 或 ARIA 補語意。

## 4. 語法與基本用法

### 4.1 常見語意化標籤

| 標籤 | 主要用途 | 使用重點 |
|---|---|---|
| `header` | 頁面或區塊的頭部內容 | 常包含標題、Logo、簡介、搜尋或導覽 |
| `nav` | 導覽連結區 | 用於主要導覽或重要連結群組，不是所有連結都要放進 `nav` |
| `main` | 頁面的主要內容 | 一頁通常只應有一個主要 `main` |
| `article` | 可獨立閱讀或重複發布的內容 | 適合文章、新聞、部落格貼文、評論、卡片型內容 |
| `section` | 文件中的主題區塊 | 通常應有能說明主題的標題 |
| `aside` | 與主要內容相關的補充內容 | 適合側邊欄、延伸閱讀、相關連結、補充說明 |
| `footer` | 頁面或區塊的底部資訊 | 常包含版權、作者、聯絡資訊、相關連結 |

> 補充說明：原始資料列出 `header`、`nav`、`article`、`section`、`aside`、`footer`。本筆記額外補充 `main`，因為它是現代 HTML 中描述頁面主要內容的重要語意元素。

### 4.2 基本頁面結構

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>語意化標籤範例</title>
</head>
<body>
  <header>
    <h1>前端學習筆記</h1>
    <nav>
      <a href="/">首頁</a>
      <a href="/html">HTML</a>
      <a href="/css">CSS</a>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h2>HTML 語意化標籤</h2>
        <p>學習如何用正確元素描述頁面結構。</p>
      </header>

      <section>
        <h3>語意化的目的</h3>
        <p>語意化標籤可以讓頁面結構更容易被人與機器理解。</p>
      </section>
    </article>

    <aside>
      <h2>延伸閱讀</h2>
      <a href="/html/div-span">HTML 盒子標籤</a>
    </aside>
  </main>

  <footer>
    <p>&copy; 2026 Frontend Notes</p>
  </footer>
</body>
</html>
```

這個範例中：

- `header` 表示頁面上方區域，內含主標題與導覽。
- `nav` 表示主要導覽連結。
- `main` 包住頁面主要內容。
- `article` 表示一篇可以獨立閱讀的筆記。
- `section` 表示文章中的一個主題段落。
- `aside` 表示與主內容相關的延伸閱讀。
- `footer` 表示頁面底部資訊。

### 4.3 舊式 `div` 版面與語意化版面的比較

舊式寫法常用 `div` 搭配 class 命名：

```html
<body>
  <div class="header">頭部標籤</div>
  <div class="nav">導航欄標籤</div>
  <div class="section">某個區域</div>
</body>
```

這樣的 HTML 可以被 CSS 選取與排版，但 `div` 本身沒有語意。

改成語意化標籤後，結構會更清楚：

```html
<body>
  <header>頭部標籤</header>
  <nav>導航欄標籤</nav>
  <section>某個區域</section>
</body>
```

對應的 CSS 也可以直接選取語意化元素：

```css
header,
nav {
  width: 800px;
  height: 120px;
  margin: 15px auto;
  border-radius: 15px;
  background-color: pink;
}

section {
  width: 500px;
  height: 300px;
  background-color: skyblue;
}
```

> 注意：直接用元素選擇器適合小型範例。實務專案中，如果同一頁有多個 `header`、`section` 或 `footer`，通常仍會搭配 `class` 控制特定區塊樣式，避免影響範圍過大。

## 5. 範例說明

### 範例一：網站首頁結構

```html
<header class="site-header">
  <h1>城市旅遊指南</h1>
  <nav aria-label="主要導覽">
    <a href="/spots">景點</a>
    <a href="/food">美食</a>
    <a href="/traffic">交通</a>
  </nav>
</header>

<main>
  <section>
    <h2>熱門景點</h2>
    <article>
      <h3>河岸公園</h3>
      <p>適合散步、騎車與欣賞夜景。</p>
    </article>
  </section>

  <aside>
    <h2>旅遊提醒</h2>
    <p>週末人潮較多，建議提早出發。</p>
  </aside>
</main>

<footer>
  <p>聯絡信箱：service@example.com</p>
</footer>
```

這個結構讓不同角色的內容一眼可辨：

- 網站名稱與主要導覽集中在 `header`。
- 使用者最需要閱讀的內容放在 `main`。
- 景點資料是主題區塊，因此使用 `section`。
- 單一景點介紹可視為可獨立理解的內容，因此使用 `article`。
- 旅遊提醒是補充資訊，因此使用 `aside`。
- 聯絡方式放在 `footer`。

### 範例二：文章內部也可以使用 `header` 與 `footer`

```html
<article>
  <header>
    <h2>語意化 HTML 的基本觀念</h2>
    <p>作者：Alex，發布日期：2026-05-22</p>
  </header>

  <p>語意化 HTML 的核心是根據內容意義選擇合適元素。</p>

  <footer>
    <p>分類：HTML 基礎</p>
  </footer>
</article>
```

這裡的 `header` 與 `footer` 不代表整個頁面的頁首與頁尾，而是這篇 `article` 的頭部資訊與底部資訊。

## 6. 實務應用場景

語意化標籤在實務開發中常見於以下情境：

- 官網首頁：用 `header`、`nav`、`main`、`section`、`footer` 建立清楚頁面骨架。
- 文章頁：用 `article` 表示主要文章，用 `section` 組織文章段落。
- 新聞列表或部落格列表：每篇摘要可用 `article` 包住。
- 後台系統：用 `main` 表示目前頁面的主要工作區，用 `nav` 表示側邊選單。
- 內容網站：用合理標題與語意區塊幫助搜尋引擎理解內容主題。
- 無障礙頁面：讓螢幕閱讀器使用者可以透過 landmark 快速理解與跳轉頁面區域。

語意化 HTML 不只是給搜尋引擎看的。它同時影響：

- 開發者閱讀程式碼的效率。
- CSS 與 JavaScript 選取結構時的可維護性。
- 瀏覽器建立 DOM 後的內容結構。
- 輔助工具對頁面區域與內容角色的理解。
- 搜尋引擎對頁面上下文與重要內容的判讀。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 所有區塊都用 `div` | `div` 沒有語意，頁面結構不清楚 | 有明確內容角色時，優先使用語意化標籤 |
| 只因為想排版就使用 `section` | `section` 應表示主題區塊，不是通用容器 | 純排版容器可使用 `div`，主題區塊才使用 `section` |
| 每個連結群組都放進 `nav` | `nav` 代表主要或重要導覽，不是所有連結清單 | 只有主要導覽、頁內目錄等重要導航才使用 `nav` |
| 把 `article` 當成一般區塊 | `article` 應該能獨立閱讀或重複發布 | 一般主題區塊可用 `section`，通用容器可用 `div` |
| 一頁放多個主要 `main` | `main` 用來表示頁面主要內容 | 一個頁面通常只保留一個主要 `main` |
| 用語意化標籤後不寫標題 | 讀者與輔助工具較難理解區塊主題 | `section`、`article` 等區塊通常應搭配清楚標題 |
| 認為語意化只為了 SEO | SEO 是其中一個好處，但不是唯一目的 | 同時考慮可讀性、可訪問性、維護性與內容結構 |

### 錯誤範例：用 `section` 當純排版盒子

```html
<section class="card-wrapper">
  <div class="card">商品 A</div>
  <div class="card">商品 B</div>
</section>
```

問題：如果 `card-wrapper` 只是為了排版，沒有形成一個明確主題區塊，使用 `section` 會讓語意不夠準確。

### 正確範例：有主題時使用 `section`

```html
<section class="featured-products">
  <h2>精選商品</h2>

  <article class="product-card">
    <h3>商品 A</h3>
    <p>適合日常使用的入門款。</p>
  </article>

  <article class="product-card">
    <h3>商品 B</h3>
    <p>適合需要進階功能的使用者。</p>
  </article>
</section>
```

修正後，`section` 表示「精選商品」這個主題區塊，內部每個商品卡片則使用 `article` 表示可獨立理解的項目。

### 錯誤範例：用 `div` 取代導覽

```html
<div class="nav">
  <a href="/">首頁</a>
  <a href="/courses">課程</a>
  <a href="/contact">聯絡我們</a>
</div>
```

問題：這是一組主要導覽連結，但外層使用沒有語意的 `div`。

### 正確範例：使用 `nav`

```html
<nav aria-label="主要導覽">
  <a href="/">首頁</a>
  <a href="/courses">課程</a>
  <a href="/contact">聯絡我們</a>
</nav>
```

修正後，`nav` 直接表達這組連結的導覽角色；`aria-label` 則在有多組導覽時幫助輔助工具區分用途。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| `div` 與 `span` | `div`、`span` 是無語意容器；語意化標籤能更精準表達內容角色 |
| 標題標籤 | `section`、`article` 等區塊通常需要清楚標題來表示主題 |
| CSS | CSS 負責排版與視覺效果，語意化標籤仍可搭配 `class` 套用樣式 |
| DOM | 瀏覽器會把語意化標籤解析成 DOM 節點，供 CSS 與 JavaScript 操作 |
| JavaScript | JavaScript 可透過 DOM API 取得語意化元素，但不應因此忽略 HTML 結構本身 |
| 可訪問性 | 語意化標籤有助於輔助工具理解頁面區域與內容角色 |
| SEO | 合理語意與標題結構有助於搜尋引擎理解頁面，但 SEO 不只取決於標籤 |

## 9. 重點整理

- 語意化 HTML 是根據內容意義選擇合適元素。
- 語意化標籤的目的不是排版，而是讓頁面結構更清楚。
- `header` 表示頁面或區塊的頭部內容。
- `nav` 表示主要或重要的導覽連結。
- `main` 表示頁面主要內容，一頁通常只應有一個主要 `main`。
- `article` 適合可獨立閱讀或重複發布的內容。
- `section` 適合有明確主題的文件區塊，通常應搭配標題。
- `aside` 適合補充資訊或側邊欄內容。
- `footer` 表示頁面或區塊的底部資訊。
- 有合適語意元素時，應優先使用語意元素；沒有合適元素時才使用 `div`。
- 語意化 HTML 有助於閱讀、維護、可訪問性與搜尋引擎理解。

## 11. 延伸補充

### 11.1 舊版瀏覽器相容性

原始資料提到，在 IE9 中可能需要把這些新元素轉換為塊級元素。比較務實的寫法是：若專案需要支援舊版 IE 或特殊舊環境，應確認目標瀏覽器對 HTML5 語意化元素的支援狀況。

常見的基礎 CSS 處理如下：

```css
header,
nav,
main,
article,
section,
aside,
footer {
  display: block;
}
```

> 注意：現代瀏覽器通常不需要為這些元素額外設定 `display: block` 才能正常使用。只有在處理舊版瀏覽器相容性時，才需要特別確認這類設定。

### 11.2 語意化與行動端

原始資料提到行動端更常使用這些標籤，原因是行動端通常不需要處理早期 IE 之類的相容性問題。

更完整地說，語意化標籤不只適用於行動端，也適用於桌面端。只要是 HTML 文件，就應該盡量用正確元素描述內容結構。響應式排版則交給 CSS 處理。

### 11.3 語意化不等於不用 `class`

使用語意化標籤後，實務上仍然常搭配 `class`：

```html
<header class="site-header">
  <h1>網站名稱</h1>
</header>
```

`header` 負責表達語意，`site-header` 則提供 CSS 或 JavaScript 更精準的選取目標。兩者不是互斥關係。
