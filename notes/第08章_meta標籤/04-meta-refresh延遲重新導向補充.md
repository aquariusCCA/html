# meta refresh 延遲重新導向補充

## 學習目標

讀完這篇補充筆記，你應該能夠：

- 看懂 `http-equiv="refresh"` 的基本語法。
- 理解它可以用來重新整理頁面或延遲跳轉。
- 知道為什麼它通常不應作為主要導頁方案。
- 分辨什麼情境可以使用，什麼情境應改用伺服器重新導向或其他方式。

## 問題情境

有些頁面會在顯示幾秒訊息後，自動跳到另一個網址。例如：

- 舊頁面搬家後，提示使用者即將前往新頁面。
- 表單送出後，顯示短暫成功訊息再返回列表。
- 活動頁結束後，導向新的活動頁或首頁。

HTML 可以透過 `meta refresh` 做到這件事，但它是一種比較特殊的導頁方式，實務上要謹慎使用。

## 一句話理解

`meta refresh` 是放在 `<head>` 中的刷新或延遲重新導向設定，會讓瀏覽器在指定秒數後重新載入頁面或跳到另一個 URL。

## 基本語法

```html
<meta http-equiv="refresh" content="3;url=https://www.bilibili.com/">
```

這段程式碼表示：等待 3 秒後，導向 `https://www.bilibili.com/`。

## 語法拆解

| 片段 | 意義 |
| --- | --- |
| `http-equiv="refresh"` | 宣告這是一個刷新或重新導向設定。 |
| `content="3;url=..."` | `3` 代表等待秒數，`url` 代表目標網址。 |

如果只寫秒數，通常代表重新整理目前頁面：

```html
<meta http-equiv="refresh" content="10">
```

這表示每 10 秒重新整理目前頁面。

## 放置位置

`meta refresh` 應放在 `<head>` 中：

```html
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="refresh" content="3;url=https://example.com/new-page">
  <title>即將重新導向</title>
</head>
<body>
  <p>頁面即將前往新位置。</p>
</body>
</html>
```

如果頁面會自動跳轉，最好在 `body` 中提供可見提示，讓使用者知道即將發生什麼事。

## 適合使用的情境

`meta refresh` 可以用在一些簡單、低風險的情境：

- 靜態頁面提示幾秒後前往新網址。
- 沒有後端控制權時，臨時處理舊頁面導向。
- 教學範例中展示 HTML 可以控制瀏覽器刷新行為。

如果是正式網站的長期導頁，通常應優先考慮伺服器端重新導向，例如 HTTP 301 或 302。伺服器重新導向對瀏覽器、搜尋引擎、快取與維護流程通常更清楚。

## 常見問題與風險

### 使用者可能來不及閱讀

如果頁面 1 秒內自動跳走，使用者可能根本看不到提示文字。對需要閱讀、確認或操作的頁面，過快跳轉會造成不好的體驗。

### 輔助工具使用者可能被打斷

使用螢幕閱讀器或鍵盤操作的使用者，可能還沒理解頁面內容就被導走。若必須跳轉，應提供清楚文字與可點擊連結。

### 不適合取代正式導頁規則

`meta refresh` 是 HTML 文件內的設定。若網站頁面已經搬家，從伺服器層處理重新導向通常更可靠，也更方便維護。

## 常見錯誤

### 少寫目標網址

```html
<meta http-equiv="refresh" content="3">
```

這不是 3 秒後前往新頁面，而是 3 秒後重新整理目前頁面。

### 沒有提供手動連結

較不理想：

```html
<p>即將跳轉...</p>
```

建議至少提供可點擊連結：

```html
<p>
  頁面即將前往新位置。
  <a href="https://example.com/new-page">如果沒有自動跳轉，請點這裡</a>
</p>
```

這樣即使自動跳轉失敗，使用者仍然可以手動前往目標頁面。

## 實務使用原則

使用 `meta refresh` 前，可以先問三個問題：

- 使用者是否需要先閱讀或操作目前頁面？
- 這是否是正式網站的長期重新導向？
- 是否能用伺服器重新導向、正常連結或 JavaScript 流程更清楚地處理？

如果答案顯示它只是臨時、簡單、低風險的導向，`meta refresh` 可以使用。否則應選擇更明確的方案。

## 重點整理

- `meta refresh` 使用 `<meta http-equiv="refresh" content="秒數;url=目標網址">`。
- 只有秒數而沒有 `url` 時，通常是重新整理目前頁面。
- 它應放在 `<head>` 中，並搭配頁面可見提示。
- 正式網站長期導頁通常更適合使用伺服器重新導向。
- 自動跳轉可能影響使用者體驗與可及性，應謹慎使用。
