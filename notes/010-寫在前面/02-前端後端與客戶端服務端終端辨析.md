# 前端、後端、客戶端、服務端與終端：先把名詞釐清

## 學習目標

- 分辨「前端／後端」「客戶端／服務端」「終端」這三組詞各自描述的是什麼維度。
- 能用同一個情境（例如打開一個網站）說明這幾組詞分別對應哪個部分。
- 知道接下來要學的 HTML，主要落在哪一組詞的哪一邊。

## 問題情境

剛接觸 Web 開發時，很容易把「前端」「後端」「客戶端」「服務端」「終端」這幾個詞當成同一回事——反正都跟「電腦」「網路」「程式」有關。但其實這幾組詞分別在回答不同的問題：「這段程式碼跑在哪裡、負責什麼工作？」（前端／後端）、「這台機器在這次互動中扮演什麼角色？」（客戶端／服務端）、「你怎麼操作這台機器？」（終端）。先把這幾組詞分清楚，之後看技術文件、跟其他人溝通時才不會雞同鴨講。

## 一句話理解

「客戶端／服務端」說的是「誰發出請求、誰提供服務」的角色關係；「前端／後端」說的是「一個應用程式內部，哪一部分負責畫面展示、哪一部分負責提供資料」的分工；「終端」則是你用來操作一台機器（不管是你自己的電腦還是遠端主機）的操作介面。

## 概念拆解

### 前端與後端：負責的工作不同

最簡單的說法：**前端負責展示數據，後端負責提供數據**。

![前端與後端請求響應示意圖](../../origin/010-寫在前面/assets/images/web-frontend-backend-img-001-b8f042.png)

- 前端：使用者實際看到、操作的畫面與互動，也就是接下來要學的 HTML（結構）、CSS（樣式）、JavaScript（行為）。
- 後端：處理請求、執行邏輯、存取資料庫，再把結果交給前端去顯示。

補充兩個常一起出現的詞：

- **資料庫**：負責儲存資料的地方，後端通常會去資料庫查詢或寫入資料。
- **伺服器**：扮演中轉站的角色，幫忙保存資訊、傳遞訊息（下一篇會更詳細說明）。

### 客戶端與服務端：誰請求、誰服務

客戶端與服務端，本質上都只是「電腦」，差別只在於這次互動中扮演的**角色**。

可以用一個生活比喻理解：一個人在餐廳上班，因為他的工作是「為你提供服務」，所以被稱為「服務員」——但他本質上仍是一個和你一樣的人。電腦的世界也類似：當一台電腦（通常配置較高、運行較快）被用來「提供某種服務」時，我們就稱它是伺服器、是服務端；而發出請求、使用這項服務的那一端，就是客戶端（用戶端）。

兩個例子：

- **網站**：網站的網頁檔案存放在一台高性能電腦上，這台電腦叫網站伺服器（服務端），對外提供一個網址；用來打開這個網址的電腦，就是客戶端。
- **App**：手機 App 的檔案存放在一台電腦（app 伺服器，服務端）上；裝了這個 App 的手機，就是客戶端。

### 終端：操作機器的「窗口」

終端是另一個維度的概念——它比較像是「工廠裡一台機器的操作介面」，透過這個介面去掌控機器。

在電腦上，終端通常指的是命令列介面：Windows 上的 cmd 視窗、Mac 或 Linux 上的命令行視窗，都可以叫做「終端」。終端描述的是「你怎麼操作一台機器」，跟這台機器在某次網路互動中是客戶端還是服務端，是兩件不同的事——自己的電腦平時是客戶端，但依然可以打開它的終端來操作它。

## 範例拆解：打開一個網站時，這些詞分別對應什麼

假設在自己的電腦上打開瀏覽器，輸入某網站的網址：

- 自己的電腦在這次互動中是**客戶端**：發出請求、接收並顯示結果的一方。
- 對方的伺服器是**服務端**：儲存網站檔案、回應請求的一方。
- 服務端回應的內容裡，通常會包含**前端資源**（HTML/CSS/JS）；這些資源會在客戶端的瀏覽器中被解析、執行與呈現。**後端**則是在服務端處理請求、決定要回應什麼內容，可能還會查詢資料庫。
- 如果想直接在自己的電腦上執行一些指令（例如之後會用到的指令工具），就會打開**終端**來操作自己的電腦。

由此可以看出：客戶端與服務端描述「請求與提供服務」的角色，前端與後端描述「應用程式內部的分層」，這兩組概念常常相關（例如服務端通常會包含後端邏輯），但不能直接劃等號。

## 對照表

| 詞彙 | 描述的維度 | 例子 |
| --- | --- | --- |
| 前端 | 應用程式中「展示數據」的部分 | 瀏覽器看到的網頁畫面（HTML/CSS/JS） |
| 後端 | 應用程式中「提供數據」的部分 | 處理請求、查詢資料庫的伺服器端程式 |
| 客戶端 | 這次互動中「發出請求、使用服務」的電腦 | 用來上網的電腦或手機 |
| 服務端 | 這次互動中「提供服務」的電腦 | 存放網站檔案、對外提供網址的伺服器 |
| 終端 | 操作一台機器的介面 | Windows 的 cmd、Mac/Linux 的命令行視窗 |

## 與 HTML 學習的關係

接下來這本書要寫的 HTML（之後還會接觸 CSS、JavaScript），都屬於**前端**的範疇；當這些檔案被放上一台對外提供服務的電腦時，那台電腦就是**服務端**，而打開網頁的使用者所在的電腦就是**客戶端**。如果之後需要用命令列工具（例如執行某些指令來啟動本地伺服器），就會用到**終端**去操作自己的電腦。

## 常見錯誤

- **把「客戶端」當成「前端」**：客戶端是「角色」（這台機器在發請求），前端是「分工」（負責展示畫面的程式碼）。自己的電腦是客戶端，但電腦上執行的瀏覽器去顯示前端程式碼，是另一件事；一台機器同時是客戶端，也同時在執行前端程式碼，但這是兩個不同維度的描述。
- **把「終端」當成「客戶端」**：終端是操作介面，跟這台機器在某次網路互動中扮演客戶端或服務端無關——服務端的機器一樣可以有終端讓管理者操作它，並不會因此變成客戶端。

## 重點整理

- 前端／後端：分工——前端展示數據，後端提供數據。
- 客戶端／服務端：角色——客戶端發出請求，服務端提供服務，兩者本質都是電腦。
- 終端：操作介面——用來操作一台機器（不論它是客戶端或服務端）的命令列視窗。
- 三組詞描述不同維度，可以同時成立、互不衝突，但不能互相替換。
