# 瀏覽器與渲染引擎：HTML 最終是怎麼被畫出來的

## 學習目標

- 認識常見的現代瀏覽器有哪些。
- 理解「渲染引擎（瀏覽器內核）」是什麼，它在瀏覽器中負責什麼工作。
- 理解為什麼同一份 HTML，在不同瀏覽器中可能呈現得不太一樣。

## 問題情境

之後寫的每一行 HTML，最終都是要靠瀏覽器顯示出來的。但為什麼同樣的程式碼，在 Chrome 打開和在 Safari 打開，有時候畫面或效果會有些不同？答案跟瀏覽器內部負責「解析並畫出網頁」的那個部分有關。

## 一句話理解

瀏覽器是顯示與執行網頁的平台；瀏覽器內部負責「讀懂這份網頁、並把它畫出來」的部分，叫做渲染引擎，也常被稱為瀏覽器內核。

## 概念拆解

**常見的現代瀏覽器**

目前常見的現代瀏覽器有 Chrome、Edge、Firefox、Safari 和 Opera 等。

![五大主流瀏覽器與問世時間](../../origin/010-寫在前面/assets/images/browser-img-001-111cfe.png)

IE（Internet Explorer）曾經是重要的瀏覽器，但現在主要只存在於舊系統環境或相容模式的情境中，已經不適合再被視為現代主流瀏覽器——如果之後在舊文件或舊專案中看到針對 IE 的特殊處理，多半是為了相容過去的環境。

**瀏覽器內核（渲染引擎）是什麼**

當瀏覽一個網頁時，流程大致是：瀏覽器向伺服器請求資源（回想一下前面「Web 伺服器」那篇）；瀏覽器拿到這些資源（HTML、CSS、圖片等）後，需要對它們進行處理，這個處理的工作就是由**渲染引擎（瀏覽器內核）**負責的。

![瀏覽器內核處理網頁資源示意圖](../../origin/010-寫在前面/assets/images/browser-img-003-ccff67.png)

渲染引擎是瀏覽器中專門「解析並渲染代碼」的部分，它負責：

- 讀取網頁內容（HTML、CSS 等）。
- 整理這些內容傳遞的訊息。
- 計算網頁該如何顯示。
- 把最終結果顯示在畫面上。

內核處理完之後，才會看到一個完整的頁面。

**為什麼不同瀏覽器可能顯示不同**

不同瀏覽器可能使用不同的渲染引擎，也可能共用同一個渲染引擎；但即使使用相同的引擎，不同的瀏覽器版本、執行平台與功能支援程度，仍然可能讓相同的程式碼呈現出不同的結果。

![五大瀏覽器與內核對照圖](../../origin/010-寫在前面/assets/images/browser-img-002-b448ff.png)

## 範例：一份 HTML 從伺服器到畫面的旅程

把前面幾篇的概念串起來看：

1. 使用者在瀏覽器（客戶端）輸入網址。
2. 瀏覽器向 Web 伺服器（服務端）發出請求。
3. 伺服器回應 HTML、CSS、圖片等資源。
4. 瀏覽器的**渲染引擎**讀取這些資源、整理、計算顯示方式。
5. 使用者看到最終畫面。

同一份 HTML，在第 4 步可能因為不同瀏覽器的渲染引擎、版本或平台不同，而出現些微差異——這就是為什麼「跨瀏覽器測試」在前端開發中很重要。

## 對 HTML 學習的實務意義

寫 HTML 時，可以先建立一個習慣：**寫的程式碼，最終要經過某個瀏覽器的渲染引擎才會變成畫面**。如果之後發現某個效果在一個瀏覽器正常、在另一個瀏覽器跑掉，先別急著懷疑「程式碼是不是有問題」，也要考慮是不是渲染引擎、瀏覽器版本或平台支援度造成的差異。

## 常見錯誤

- **以為「在這個瀏覽器沒問題，所以在所有瀏覽器都會一樣」**：不同渲染引擎、版本、平台對同一段代碼的解析速度、效果可能不同，正確的程式碼仍可能在不同環境下呈現出差異。
- **只用「瀏覽器名稱」判斷相容性，忽略版本與平台**：例如「Chrome 沒問題」並不代表所有版本、所有作業系統上的 Chrome 都會一致；跨瀏覽器測試時，版本與平台同樣重要。

## 重點整理

- 常見現代瀏覽器：Chrome、Edge、Firefox、Safari、Opera；IE 已主要屬於舊環境或相容模式。
- 渲染引擎（瀏覽器內核）負責讀取、整理、計算並顯示網頁內容，是瀏覽器「把代碼變成畫面」的核心部分。
- 不同瀏覽器可能用不同或相同的渲染引擎；即使引擎相同，版本、平台、功能支援不同仍可能造成顯示差異。

## 自我檢查

1. Chrome 與 Edge 這兩個瀏覽器，可能使用相同還是不同的渲染引擎？這對寫的網頁有什麼影響？
2. 渲染引擎在「使用者看到網頁畫面」這個流程中，是負責哪一個步驟？
3. 為什麼測試一個網頁時，只在一個瀏覽器上測試可能不夠？
