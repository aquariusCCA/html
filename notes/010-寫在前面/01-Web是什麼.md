# Web 是什麼：在學 HTML 之前，先搞懂大局

## 學習目標

- 用自己的話解釋「Web」指的是什麼，以及它跟「網站」「瀏覽器」的差異。
- 知道 Web 上的資源有哪些類型，以及 HTML 在這些資源中扮演什麼角色。
- 理解「JavaWeb」這類名詞跟接下來要學的 HTML 之間的關係。

## 問題情境

學 HTML 之前，你可能已經聽過「網頁」「網站」「Web」「上網」這些詞混著用，也可能聽過「JavaWeb」「前端」「後端」這類詞，卻不確定它們跟「我要學 HTML」這件事有什麼關係。如果一開始就跳進 `<html>`、`<body>` 這些標籤，會少了一張地圖：你寫的這些標籤，最後是怎麼被「送到」使用者眼前的？這篇先把這張地圖畫出來。

## 一句話理解

Web（全球資訊網）是一個透過網路、用 HTTP/HTTPS 傳輸資源、用 HTML 描述內容的全球性資訊系統；而像「JavaWeb」這樣的技術，則是用來在這個系統的「伺服器端」實作出回應內容的方式之一。

## 概念拆解

**Web 是一個系統，不是一個網站**

Web 不是指某一個特定的網站，而是指由「超文本鏈接」串起來的整個資訊網路。可以把它想成一個巨大的資訊庫：裡面有網頁、圖像、音訊、影片等各種資源，散落在世界各地的電腦上，使用者透過網際網路和瀏覽器去存取它們。

**Web 資源是怎麼被傳輸與描述的**

- 資源要被傳輸，靠的是 HTTP / HTTPS 這類傳輸協定。
- 資源如果是一個網頁，它的結構與內容靠 HTML 描述。

這就是為什麼接下來整本書都在教 HTML：HTML 就是 Web 上「網頁」這種資源最基本的描述語言。

**JavaWeb：用 Java 在伺服器端實作 Web 應用**

JavaWeb 通常指使用 Java 技術開發、部署在 Web 或應用伺服器上、可透過瀏覽器或 HTTP 介面存取的應用。在這種應用裡：

- Java 程式通常在伺服器端執行，負責處理請求、產生回應或提供資料。
- 使用者不會看到這些 Java 程式碼，而是透過瀏覽器看到對應的頁面或結果。

對現在的你來說，重點不是去學 Java，而是先理解一件事：不管伺服器端用的是 Java、PHP、Node.js 或其他技術，最後「送到瀏覽器、被使用者看到」的東西，通常都會是一份 HTML（再加上 CSS、JavaScript）。HTML 是這條鏈路的最終出口。

## 範例：一次造訪網頁的簡化流程

1. 使用者在瀏覽器輸入網址，瀏覽器透過 HTTP/HTTPS 向某台伺服器發出請求。
2. 伺服器端的程式（可能是用 Java、也可能是其他技術寫的）處理這個請求，決定要回應什麼內容。
3. 伺服器把結果包成一份 HTML（通常還有 CSS、JavaScript、圖片等資源）回傳。
4. 瀏覽器收到這些資源，依照 HTML 的結構把畫面顯示出來。

接下來這本書要學的，就是第 3、4 步裡的「HTML」這一塊：怎麼用 HTML 描述出一份網頁該有的結構與內容。

## 對 HTML 學習的實務意義

- 不論未來接觸的後端是 JavaWeb、Node.js 還是其他技術，最終都需要產生 HTML 給瀏覽器，所以 HTML 是前後端工程師的共同語言之一。
- 理解「Web 是一個資源網路」這個大局，有助於之後學到「路徑」「連結」「資源引用」等概念時，知道它們在解決什麼問題。

## 常見錯誤

- **把「Web」當成某個特定網站或瀏覽器**：例如以為「Web」就是「我打開的這個網頁」。其實 Web 是整個由超文本鏈接構成的資訊網路，網站、瀏覽器都只是其中的一部分或存取方式。
- **把「JavaWeb」誤解成「用 Java 寫網頁畫面（前端）」**：JavaWeb 主要是伺服器端技術，負責處理請求與產生內容；真正顯示在瀏覽器上的畫面，仍是由 HTML/CSS/JavaScript 構成。這個區分會在下一篇「前端、後端、客戶端、服務端與終端」中進一步說明。

## 重點整理

- Web = 由超文本鏈接構成的全球資訊系統，透過 HTTP/HTTPS 傳輸，內容常以 HTML 描述。
- Web 上的資源不只有網頁，還包含圖像、音訊、影片等。
- JavaWeb 是一種在伺服器端實作 Web 應用的技術，最終仍會產生 HTML 給瀏覽器顯示。
- HTML 是「使用者最終看到的內容」的描述語言，是接下來學習的核心。

## 自我檢查

1. 「Web」與「某一個網站」有什麼不同？
2. 在 JavaWeb 應用中，Java 程式碼主要執行在哪裡？使用者是透過什麼方式看到結果的？
3. 不管後端使用什麼技術，最終呈現在瀏覽器上的內容通常會是什麼？
