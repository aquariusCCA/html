# Web 伺服器：讓你的網頁被別人看到

## 學習目標

- 理解「伺服器」在 Web 世界中扮演的角色。
- 知道 Web 伺服器具體提供什麼服務。
- 分辨本地伺服器與遠端伺服器的差異，以及各自適合的情境。

## 問題情境

上一篇提到「服務端」是負責提供服務的電腦。但具體來說，當你在自己電腦上寫好一個 HTML 檔案、用瀏覽器打開它時，為什麼只有你自己能看到，朋友卻打不開？這篇要解決的問題就是：一個網頁要怎麼從「只有我自己能看」變成「別人也能訪問」，伺服器在這中間做了什麼。

## 一句話理解

伺服器是一台專門「提供服務」的電腦；Web 伺服器負責回應瀏覽器（Web 客戶端）的請求，把網頁、圖片、檔案等資源送出去。

## 概念拆解

**伺服器＝主機，依服務類型有不同種類**

伺服器本質上也是一台計算機，只是它的角色是「提供計算服務」。依照提供的服務類型不同，伺服器可以再分為：文件伺服器、資料庫伺服器、應用程式伺服器、Web 伺服器等。伺服器有時也被稱為「主機」。

**Web 伺服器在做什麼**

Web 伺服器可以指：

- 一種提供 HTTP/HTTPS 服務的**軟體**；也可以指
- 承載這類服務的**主機（硬體）**。

它的工作是向瀏覽器這類「Web 客戶端」提供網頁文件、圖片、資料或下載資源——也就是回應客戶端的請求。

**本地伺服器與遠端伺服器**

接下來提到的「伺服器」，主要都是指 Web 伺服器。依照伺服器在網路中所在的位置，可以分為兩種：

- **本地伺服器**：把自己的電腦設定為本地伺服器，這樣同一個區域網路（例如同一個 Wi-Fi）下的其他人，就可以透過這台電腦訪問正在開發的網頁。
- **遠端伺服器**：通常是別家公司提供的一台主機。網站項目上傳後，還需要完成網路、DNS、權限或防火牆等設定，目標使用者才能透過網域名稱或 IP 訪問網站——例如 `www.baidu.com`。

## 範例：從「只有我能看」到「大家都能看」

1. 在自己電腦上寫好一個 `index.html`，用瀏覽器打開檔案——這時只有自己的電腦能看到，因為這份檔案還只存在於自己的電腦裡，沒有任何「服務」在對外提供它。
2. 如果把自己的電腦設定成**本地伺服器**，同一個 Wi-Fi 下的同學就可以透過這台電腦的 IP 位址，在他們的瀏覽器中看到這個網頁。
3. 如果想讓**全世界**的人都能訪問，就需要把網站項目放到一台**遠端伺服器**上，並完成網域名稱、DNS 等設定——就像 `www.baidu.com` 一樣，使用者輸入網址就能連到對方的伺服器。

## 對 HTML 學習的實務意義

接下來這本書主要會在自己的電腦上練習 HTML，這時這台電腦同時是「寫程式的地方」和「（如果需要的話）本地伺服器」。理解「伺服器」這個角色，能幫忙建立一個概念：**寫好的 HTML 檔案本身不會自動讓別人看到，必須有一台伺服器在「提供」這個檔案，瀏覽器才能透過請求拿到它**。之後如果接觸到「部署」「上線」這類詞，指的就是把網站放到遠端伺服器上的過程。

## 常見錯誤

- **以為「自己電腦能打開網頁」＝「網站已經上線」**：用瀏覽器直接打開本機檔案，跟透過伺服器提供網頁，是兩件不同的事；前者沒有經過任何「服務」，後者才是真正的 Web 伺服器在運作。
- **把伺服器單純想成「一台很貴、很厲害的電腦」**：伺服器的關鍵不是「貴不貴」，而是它**提供了什麼服務**——一台電腦只要對外提供 HTTP 服務，就可以算是 Web 伺服器，不一定要是高階設備。

## 重點整理

- 伺服器＝主機，是專門提供服務的電腦；依服務類型可分為文件、資料庫、應用程式、Web 伺服器等。
- Web 伺服器可以是軟體，也可以是承載軟體的主機，負責回應瀏覽器等 Web 客戶端的請求。
- 本地伺服器：自己的電腦對外提供服務，通常侷限在同一區域網路。
- 遠端伺服器：別人提供的主機，需透過網域名稱/IP、並完成相關設定才能讓外部使用者訪問。
