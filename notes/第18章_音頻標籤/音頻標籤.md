# HTML 音頻標籤 audio

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `<audio>` 標籤在 HTML 中的用途。
- 使用 `src`、`controls`、`autoplay`、`loop` 等常見屬性嵌入音訊。
- 理解為什麼需要搭配 `<source>` 提供多種音訊格式。
- 說明瀏覽器對音訊格式與自動播放行為可能存在差異。
- 避免音訊無法播放、缺少控制列、過度依賴自動播放等常見問題。

## 2. 本章定位

本章介紹 HTML 中用來嵌入音訊內容的 `<audio>` 元素。

網頁除了文字與圖片，也常需要播放多媒體內容，例如背景音樂、語音導覽、課程錄音、音效或播客片段。`<audio>` 的作用是讓瀏覽器直接在頁面中載入音訊資源，並提供播放、暫停、音量控制等能力。

學習 `<audio>` 時，不只要知道標籤怎麼寫，也要理解三件事：

- 音訊檔案必須能被瀏覽器找到。
- 不同瀏覽器對音訊格式的支援可能不同。
- 自動播放通常會受到瀏覽器政策限制，不能假設一定成功。

> 來源重點：原始資料指出 `<audio>` 可用來在頁面中插入音頻，常見格式包含 MP3、WAV、Ogg，並提醒不同瀏覽器的支援情況可能不同。

## 3. 核心概念

`<audio>` 是 HTML 的多媒體元素，用來在網頁中嵌入音訊。

最簡單的寫法是直接在 `<audio>` 上使用 `src` 指定音訊檔案：

```html
<audio src="./assets/files/music.mp3" controls></audio>
```

其中：

| 部分 | 說明 |
|---|---|
| `<audio>` | 建立音訊播放器 |
| `src` | 指定音訊檔案位置 |
| `controls` | 顯示瀏覽器內建播放控制列 |

如果沒有 `controls`，使用者通常看不到瀏覽器提供的播放按鈕、暫停按鈕與音量控制。除非你打算用 JavaScript 自行製作播放介面，否則初學階段應先加上 `controls`。

> 補充說明：`<audio>` 描述的是「頁面中有一段可播放的音訊內容」。實際播放器外觀由瀏覽器決定，也可以再透過 JavaScript 控制播放行為。

## 4. 語法與基本用法

### 4.1 基本語法

```html
<audio src="音訊檔案路徑" controls></audio>
```

常見屬性如下：

| 屬性 | 說明 | 常見使用情境 |
|---|---|---|
| `src` | 指定音訊檔案路徑 | 只有一個音訊來源時使用 |
| `controls` | 顯示瀏覽器內建控制列 | 讓使用者能播放、暫停、調整音量 |
| `autoplay` | 嘗試在頁面載入後自動播放 | 常受瀏覽器政策限制，不應依賴 |
| `loop` | 播放結束後自動重新播放 | 背景音、循環音效或展示頁面 |
| `preload` | 提示瀏覽器是否預先載入音訊 | 控制載入策略與效能 |

### 4.2 `controls`：顯示播放控制列

```html
<audio src="./assets/files/music.mp3" controls></audio>
```

`controls` 是布林屬性，只要出現在標籤上就代表啟用。啟用後，瀏覽器會顯示內建音訊控制介面。

### 4.3 `autoplay`：自動播放

```html
<audio src="./assets/files/music.mp3" controls autoplay></audio>
```

`autoplay` 表示希望音訊在頁面載入後自動播放。不過，許多瀏覽器會限制有聲音的自動播放，尤其是在使用者尚未與頁面互動之前。

> 注意：不要把 `autoplay` 當成一定會成功的功能。實務上更可靠的做法，是讓使用者主動點擊播放，或用清楚的播放按鈕觸發音訊。

### 4.4 `loop`：循環播放

```html
<audio src="./assets/files/music.mp3" controls loop></audio>
```

`loop` 會讓音訊播放結束後自動從頭播放。它適合循環背景音或展示用音效，但在一般內容頁中應謹慎使用，避免打擾使用者。

### 4.5 使用 `<source>` 提供多種格式

如果希望同一段音訊能在更多瀏覽器中播放，可以在 `<audio>` 裡放多個 `<source>`：

```html
<audio controls>
  <source src="./assets/files/music.mp3" type="audio/mpeg">
  <source src="./assets/files/music.ogg" type="audio/ogg">
  <source src="./assets/files/music.wav" type="audio/wav">
  您的瀏覽器不支援 audio 標籤。
</audio>
```

瀏覽器會依序檢查 `<source>`，選擇自己支援的格式播放。若都不支援，就會顯示 `<audio>` 標籤內的備用文字。

| 屬性 | 說明 |
|---|---|
| `src` | 指定這個音訊來源的檔案路徑 |
| `type` | 告訴瀏覽器檔案的 MIME type，幫助瀏覽器判斷是否支援 |

> 注意：原始資料強調 `<source>` 需要 `src` 與 `type`。實務上 `src` 是指定資源的關鍵；`type` 建議清楚填寫，讓瀏覽器更快判斷格式是否可用。

## 5. 範例說明

### 範例一：最基本的音訊播放器

```html
<audio src="./assets/files/music.mp3" controls></audio>
```

這個範例中：

- `src="./assets/files/music.mp3"` 指向音訊檔案。
- `controls` 顯示瀏覽器內建控制列。
- 使用者可以自行播放、暫停或調整音量。

### 範例二：提供多種音訊格式

```html
<audio controls>
  <source src="./assets/files/music.mp3" type="audio/mpeg">
  <source src="./assets/files/music.ogg" type="audio/ogg">
  您的瀏覽器不支援 audio 標籤。
</audio>
```

這個寫法的目的，是讓瀏覽器能從多個音訊來源中選擇可播放的版本。

如果瀏覽器支援 MP3，就可能播放第一個來源；如果不支援，會繼續檢查下一個 `<source>`。最後的文字是備用內容，通常給不支援 `<audio>` 的環境顯示。

### 範例三：循環播放音訊

```html
<audio src="./assets/files/background.mp3" controls loop></audio>
```

這個範例會在音訊結束後重新播放。若用於背景音樂，應讓使用者能清楚停止或關閉聲音。

### 錯誤範例：依賴自動播放

```html
<audio src="./assets/files/intro.mp3" autoplay></audio>
```

問題：瀏覽器可能阻擋有聲音的自動播放，而且沒有 `controls` 時，使用者可能無法操作音訊。

### 正確範例：提供控制列，讓使用者主動播放

```html
<audio src="./assets/files/intro.mp3" controls></audio>
```

修正點：使用 `controls` 讓播放行為交給使用者決定，體驗更穩定，也比較不容易違反瀏覽器的自動播放限制。

## 6. 實務應用場景

`<audio>` 常用在以下情境：

- 語言學習網站播放發音範例。
- 線上課程播放講解錄音。
- 文章頁嵌入播客或訪談片段。
- 遊戲或互動頁面播放音效。
- 產品展示頁提供語音說明。

在實務開發中，使用 `<audio>` 時通常還要注意：

- 音訊檔案路徑是否正確。
- 檔案大小是否會影響頁面載入速度。
- 是否需要提供多種格式提升相容性。
- 是否需要字幕、逐字稿或文字說明，讓無法聽音訊的使用者也能理解內容。
- 是否需要用 JavaScript 控制播放、暫停、進度或音量。

> 補充說明：如果音訊承載重要資訊，建議提供文字稿或摘要。這不只是使用體驗問題，也和可訪問性有關。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 音訊路徑寫錯 | 瀏覽器找不到檔案，播放器無法播放 | 確認 `src` 路徑與音訊檔案位置一致 |
| 忘記加 `controls` | 使用者看不到內建播放介面 | 沒有自訂播放器時，應加上 `controls` |
| 只提供單一格式 | 某些瀏覽器或環境可能不支援該格式 | 使用多個 `<source>` 提供替代格式 |
| 過度依賴 `autoplay` | 瀏覽器可能阻擋有聲音的自動播放 | 讓使用者主動播放，或設計明確播放按鈕 |
| 背景音無法停止 | 使用者缺少控制權，體驗不佳 | 提供 `controls` 或自訂停止按鈕 |
| 把音訊當成唯一資訊來源 | 聽障使用者或無法播放音訊的環境會失去資訊 | 提供文字稿、摘要或替代說明 |
| MIME type 寫錯 | 瀏覽器可能無法正確判斷音訊格式 | 依檔案格式填寫正確的 `type` |

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| `<video>` | 兩者都是 HTML 多媒體元素，常見屬性如 `controls`、`autoplay`、`loop` 類似 |
| `<source>` | 可放在 `<audio>` 中，提供多個候選音訊檔案 |
| JavaScript | 可透過 DOM API 控制播放、暫停、音量與進度 |
| CSS | 可調整播放器周邊版面，但瀏覽器內建控制列本身可客製程度有限 |
| 可訪問性 | 重要音訊內容應提供文字替代，例如逐字稿或摘要 |
| 檔案路徑 | `src` 必須指向正確資源，與圖片、影片等外部資源載入觀念相同 |

## 9. 重點整理

- `<audio>` 用來在 HTML 頁面中嵌入音訊內容。
- `src` 指定音訊檔案，`controls` 顯示瀏覽器內建播放控制列。
- 常見音訊格式包含 MP3、WAV、Ogg，但實際支援要看瀏覽器與檔案編碼。
- 使用多個 `<source>` 可以提供不同格式，增加播放成功的機會。
- `autoplay` 不保證成功，實務上應優先讓使用者主動播放。
- 若音訊包含重要資訊，應提供文字替代內容，避免只靠聲音傳達。

## 11. 延伸補充

### 11.1 音訊格式與瀏覽器支援

原始資料提到 MP3、WAV、Ogg 是常見音訊格式。正式開發時，不應只根據副檔名判斷能否播放，還要考慮瀏覽器支援、音訊編碼與伺服器回傳的 MIME type。

> 待確認：若教材需要列出各瀏覽器對 MP3、WAV、Ogg 的精確支援版本，應查證目標瀏覽器版本後再補充。

### 11.2 與 `<video>` 的相似處

`<audio>` 和 `<video>` 的使用方式很接近，兩者都可以使用 `controls`、`autoplay`、`loop` 與 `<source>`。

差別在於：

- `<audio>` 只處理聲音。
- `<video>` 同時處理影像與聲音。
- `<video>` 常需要額外考慮尺寸、封面圖、字幕與畫面比例。

因此，學會 `<audio>` 後，再學 `<video>` 會更容易理解。
