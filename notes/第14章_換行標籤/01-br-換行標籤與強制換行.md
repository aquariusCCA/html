# br 換行標籤與強制換行

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `<br>` 標籤用來表示內容中的強制換行。
- 知道 `<br>` 是不需要結束標籤的 void element。
- 分辨自動換行、強制換行和段落分段的差異。
- 避免用 `<br>` 取代段落或控制版面間距。

## 問題情境

在 HTML 中，一段文字會依照所在容器的寬度自動排列。當一行放不下時，瀏覽器會自己把文字換到下一行。

但有些內容的換行不是因為畫面寬度不夠，而是內容本身就需要在特定位置斷行。例如地址、詩句、歌詞或需要保留行列關係的短文字。這時候就可以使用 `<br>`，明確告訴瀏覽器：這裡要換到下一行。

## 一句話理解

`<br>` 是用來在文字內容中插入「強制換行」的 HTML 標籤，適合用在換行本身具有內容意義的位置。

## 標籤語意

`br` 是 `break` 的縮寫，意思是打斷、換行。

它的語意不是建立一個新段落，也不是增加上下間距，而是在同一段內容中插入一個換行點：

```html
第一行文本<br>第二行文本
```

瀏覽器顯示時會把 `第二行文本` 放到下一行。

## 基本語法

`<br>` 是 HTML 的 void element，也就是不包住其他內容、不需要結束標籤的元素。

HTML 中建議這樣寫：

```html
第一行文本<br>第二行文本
```

你也可能看過這種寫法：

```html
第一行文本<br />第二行文本
```

`<br />` 在 HTML 中也常見，但結尾的斜線不是 HTML 必要語法。初學 HTML 時，可以先使用簡潔的 `<br>`。

## 自動換行與強制換行

瀏覽器本來就會依照容器寬度自動換行：

```html
<p>
  這是一段比較長的文字，當畫面寬度不夠時，瀏覽器會自動把文字排到下一行。
</p>
```

這種換行是排版結果，不需要手動加入 `<br>`。

只有當你需要指定「這裡一定要換行」時，才使用 `<br>`：

```html
<p>
  台北市中正區<br>
  重慶南路一段 122 號
</p>
```

這個範例中，地址的兩行有內容上的分隔，所以 `<br>` 是合理的。

## `<br>` 與段落的差異

`<br>` 只是換到下一行；`<p>` 表示一個完整段落。

比較這兩種寫法：

```html
第一行文本<br>第二行文本
```

這表示兩段文字仍然屬於同一個文字內容，只是在指定位置換行。

如果內容其實是兩個不同段落，應該使用兩個 `<p>`：

```html
<p>第一段文字。</p>
<p>第二段文字。</p>
```

段落分段是內容結構；換行是同一段內容裡的斷行位置。不要只因為想讓文字看起來分開，就把所有段落都改成 `<br>`。

## 適合使用 `<br>` 的情境

`<br>` 適合用在換行本身有意義的地方，例如：

- 地址中的不同地址行。
- 詩句、歌詞或短句中的固定斷行。
- 簡短聯絡資訊中需要逐行顯示的內容。

例如：

```html
<p>
  客服信箱：service@example.com<br>
  服務時間：週一至週五 09:00-18:00
</p>
```

這裡兩行都屬於同一組聯絡資訊，只是需要分行呈現，因此可以使用 `<br>`。

## 常見錯誤

### 錯誤一：用 `<br>` 取代段落

不建議：

```html
第一段文字。<br><br>
第二段文字。
```

較好的寫法：

```html
<p>第一段文字。</p>
<p>第二段文字。</p>
```

如果內容是兩個段落，應該用 `<p>` 建立段落結構，而不是用多個 `<br>` 模擬段落間距。

### 錯誤二：用 `<br>` 控制版面空白

不建議：

```html
<h1>網站標題</h1>
<br><br><br>
<p>網站介紹文字。</p>
```

較好的方向是保留 HTML 語意，再用 CSS 控制間距：

```html
<h1>網站標題</h1>
<p>網站介紹文字。</p>
```

```css
h1 {
  margin-bottom: 24px;
}
```

HTML 負責內容語意，CSS 負責外觀與排版。用大量 `<br>` 撐開畫面會讓結構難以維護，也不容易在不同螢幕尺寸下調整。

### 錯誤三：為了讓原始碼換行而加入 `<br>`

不建議：

```html
<p>
  這是一段文字，<br>
  只是因為原始碼換行所以加入 br。
</p>
```

如果只是為了讓 HTML 原始碼好讀，可以直接在原始碼中換行與縮排，不需要加入 `<br>`：

```html
<p>
  這是一段文字，
  原始碼可以換行，但瀏覽器仍會依照內容與容器寬度排版。
</p>
```

原始碼中的換行主要是給開發者閱讀；`<br>` 則會影響瀏覽器顯示結果。

## 實務使用建議

使用 `<br>` 前，可以先問自己三個問題：

1. 這個換行是否是內容本身的一部分？
2. 如果畫面變寬或變窄，這個位置是否仍然應該固定換行？
3. 這裡是在同一段內容中換行，還是其實應該拆成不同段落或區塊？

如果答案是「內容本身需要固定斷行」，可以使用 `<br>`。如果只是為了排版、留白或分段，通常應該改用 `<p>`、其他語意標籤或 CSS。

## 小練習

請判斷下列情境適合使用 `<br>`、`<p>` 還是 CSS：

1. 一篇文章有兩個自然段。
2. 一段地址需要分成兩行顯示。
3. 標題和段落之間想多一點空白。
4. 一首短詩需要保留固定斷行。

參考答案：

1. 使用兩個 `<p>`。
2. 可以使用 `<br>`。
3. 使用 CSS 控制間距。
4. 可以使用 `<br>`，也可以依內容結構搭配更合適的標記。

## 重點整理

- `<br>` 是 HTML 的換行標籤，用來插入強制換行。
- `br` 是 `break` 的縮寫。
- `<br>` 是 void element，不需要結束標籤。
- 瀏覽器本來就會依容器寬度自動換行，不需要為一般文字手動加入 `<br>`。
- `<br>` 適合用在地址、詩句、歌詞等換行本身具有內容意義的地方。
- 不要用 `<br>` 取代 `<p>`，也不要用大量 `<br>` 控制版面間距。
