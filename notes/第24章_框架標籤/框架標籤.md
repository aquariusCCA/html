# HTML 框架標籤 iframe

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `<iframe>` 在 HTML 中的用途與基本概念。
- 使用 `src` 在頁面中嵌入其他網頁或檔案資源。
- 使用 `width`、`height` 控制 iframe 顯示尺寸。
- 理解 `name` 如何搭配 `<a>` 或 `<form>` 的 `target` 使用。
- 分辨 `frameborder` 的舊式寫法與現代 CSS 控制邊框的做法。
- 避免 iframe 缺少 `title`、嵌入不可信內容、尺寸控制不當等常見問題。

## 2. 本章定位

本章介紹 HTML 中的 `<iframe>` 元素。

`<iframe>` 可以在目前頁面中嵌入另一個頁面或資源，例如外部網站、地圖、廣告、影片播放器、PDF 文件或表單提交後的結果頁面。

學習 `<iframe>` 時，不能只把它理解成「頁面裡放另一個頁面」。它實際上會建立一個獨立的瀏覽上下文，裡面的文件有自己的載入流程、CSS、JavaScript 與瀏覽歷史。這也代表 iframe 會影響效能、安全性、可訪問性與頁面維護。

> 來源重點：原始資料說明了 iframe 的常見用途，包含在網頁中嵌入廣告，以及搭配超鏈接或表單的 `target` 屬性，在指定框架中展示不同內容。

## 3. 核心概念

`<iframe>` 是 inline frame 的意思，表示「行內框架」。它的作用是在目前 HTML 文件中建立一個框架，並在框架中載入另一份文件或資源。

基本語法如下：

```html
<iframe src="要嵌入的資源路徑"></iframe>
```

其中：

| 部分 | 說明 |
|---|---|
| `<iframe>` | 建立一個內嵌框架 |
| `src` | 指定框架中要載入的網頁或資源 |
| `</iframe>` | iframe 需要結束標籤 |

例如：

```html
<iframe src="https://example.com" title="Example 網站"></iframe>
```

這段程式碼會在目前頁面中嵌入 `https://example.com`。

> 補充說明：`<iframe>` 和圖片、影片不同。圖片或影片通常只是嵌入單一媒體資源；iframe 則是嵌入另一個瀏覽上下文，裡面可能還會載入自己的 HTML、CSS、JavaScript 與其他資源。

## 4. 語法與基本用法

### 4.1 基本語法

```html
<iframe
  src="https://example.com"
  title="Example 網站"
  width="900"
  height="300"
></iframe>
```

常見屬性如下：

| 屬性 | 說明 | 常見使用情境 |
|---|---|---|
| `src` | 指定 iframe 要載入的資源位置 | 嵌入網頁、PDF、地圖、播放器 |
| `title` | 描述 iframe 內容 | 幫助輔助工具辨識框架用途 |
| `name` | 設定 iframe 的名稱 | 搭配 `<a target>` 或 `<form target>` 指定顯示位置 |
| `width` | 設定 iframe 的顯示寬度 | 控制框架大小 |
| `height` | 設定 iframe 的顯示高度 | 控制框架大小 |
| `frameborder` | 舊式邊框控制屬性 | 舊教材常見，現代實務建議改用 CSS |

### 4.2 `src`：指定嵌入內容

`src` 是 source 的縮寫，用來指定 iframe 要載入的內容。

```html
<iframe
  src="https://www.bilibili.com/"
  title="Bilibili 首頁"
  width="900"
  height="300"
></iframe>
```

`src` 可以指向外部網址，也可以指向本地資源：

```html
<iframe
  src="./assets/pdfs/framework-tags-pdf-001-78621d.pdf"
  title="PDF 文件預覽"
  width="900"
  height="500"
></iframe>
```

> 注意：不是所有網站都允許被其他網站用 iframe 嵌入。若對方網站透過安全標頭限制嵌入，瀏覽器可能會阻擋顯示。

### 4.3 `width` 與 `height`：控制框架尺寸

`width` 與 `height` 可以直接設定 iframe 的顯示大小。

```html
<iframe
  src="https://example.com"
  title="Example 網站"
  width="900"
  height="300"
></iframe>
```

在正式專案中，也可以用 CSS 控制尺寸，讓 iframe 更容易配合響應式版面：

```html
<iframe
  class="preview-frame"
  src="https://example.com"
  title="Example 網站"
></iframe>
```

```css
.preview-frame {
  width: 100%;
  height: 360px;
  border: 0;
}
```

### 4.4 `frameborder`：舊式邊框控制

原始資料提到 `frameborder` 可用來控制是否顯示邊框，常見值是 `0` 或 `1`。

```html
<iframe
  src="https://example.com"
  width="900"
  height="300"
  frameborder="0"
></iframe>
```

這是舊式教材中常見的寫法。現代實務中，建議用 CSS 的 `border` 控制邊框：

```html
<iframe
  src="https://example.com"
  title="Example 網站"
  width="900"
  height="300"
  style="border: 0;"
></iframe>
```

> 注意：為了理解舊教材或舊專案，你需要看得懂 `frameborder="0"` 的意思；但在新程式碼中，優先使用 CSS 控制邊框。

### 4.5 `name` 搭配連結的 `target`

`iframe` 的 `name` 可以成為其他連結的 `target` 目標。當使用者點擊連結時，連結內容會載入到指定名稱的 iframe 中，而不是直接取代整個頁面。

```html
<a href="https://www.bilibili.com/" target="container">點我去 B 站</a>

<iframe
  name="container"
  title="連結內容顯示區"
  width="900"
  height="300"
></iframe>
```

這個範例中：

- `<iframe name="container">` 建立一個名為 `container` 的顯示區。
- `<a target="container">` 指定連結開啟位置。
- 點擊連結後，目標頁面會嘗試在這個 iframe 裡顯示。

### 4.6 `name` 搭配表單的 `target`

`form` 也有 `target` 屬性，可以指定表單送出後的回應頁面顯示在哪裡。

```html
<form action="https://so.toutiao.com/search" target="container2">
  <label for="keyword">搜尋關鍵字</label>
  <input type="text" id="keyword" name="keyword">
  <button type="submit">搜尋</button>
</form>

<iframe
  name="container2"
  title="搜尋結果顯示區"
  width="900"
  height="300"
></iframe>
```

這個範例中：

- `form target="container2"` 表示表單提交後的結果要顯示在名為 `container2` 的目標中。
- `iframe name="container2"` 提供了這個目標。
- `input name="keyword"` 是提交搜尋關鍵字時的欄位名稱。

> 補充說明：這種做法可以理解為「不離開目前頁面，而是在指定框架中顯示提交結果」。現代前端開發也常用 JavaScript 或框架狀態管理達成類似體驗，但理解 `target` 與 iframe 的搭配仍有助於讀懂傳統 HTML 寫法。

## 5. 範例說明

### 範例一：嵌入一般網頁

```html
<iframe
  src="https://www.bilibili.com/"
  title="Bilibili 首頁"
  width="900"
  height="300"
  style="border: 0;"
></iframe>
```

這個範例會在目前頁面中嵌入外部網站。`width` 與 `height` 控制框架尺寸，`style="border: 0;"` 移除預設邊框。

### 範例二：嵌入 PDF 文件

```html
<iframe
  src="./assets/pdfs/framework-tags-pdf-001-78621d.pdf"
  title="PDF 文件預覽"
  width="900"
  height="500"
></iframe>
```

這個範例會嘗試在 iframe 中顯示 PDF。瀏覽器是否能直接預覽 PDF，會受到瀏覽器能力、使用者設定與伺服器回應影響。

若 PDF 是正式下載資源，也可以另外提供一般連結：

```html
<a href="./assets/pdfs/framework-tags-pdf-001-78621d.pdf">開啟 PDF 文件</a>
```

### 範例三：讓連結內容顯示在 iframe 中

```html
<a href="https://www.bilibili.com/" target="container">點我去 B 站</a>

<iframe
  name="container"
  title="連結內容顯示區"
  width="900"
  height="300"
></iframe>
```

點擊連結後，瀏覽器會尋找名稱為 `container` 的瀏覽上下文。如果找到，就會把連結目標載入該 iframe。

### 範例四：讓表單結果顯示在 iframe 中

```html
<form action="https://so.toutiao.com/search" target="container2">
  <label for="keyword">搜尋關鍵字</label>
  <input type="text" id="keyword" name="keyword">
  <button type="submit">搜尋</button>
</form>

<iframe
  name="container2"
  title="搜尋結果顯示區"
  width="900"
  height="300"
></iframe>
```

這個範例保留了原始資料的教學目的：展示 `form target` 與 `iframe name` 如何互相配合。

## 6. 實務應用場景

`<iframe>` 在實務中常見於以下情境：

| 場景 | 說明 |
|---|---|
| 嵌入第三方內容 | 例如地圖、影片播放器、廣告、付款頁面、客服工具 |
| 嵌入文件預覽 | 例如 PDF 預覽、報表預覽、合約文件預覽 |
| 顯示表單提交結果 | 傳統 HTML 可用 `form target` 將結果顯示在指定 iframe |
| 隔離不完全受控的內容 | 讓某些內容在獨立瀏覽上下文中載入 |

使用 iframe 時要同時考慮：

- 來源網站是否允許被嵌入。
- iframe 內容是否可信。
- 載入第三方內容是否會拖慢頁面。
- 是否需要 `title` 讓輔助工具辨識內容。
- 是否需要 `sandbox` 限制 iframe 中內容的能力。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 缺少 `title` | 輔助工具使用者難以理解 iframe 內容用途 | 為 iframe 加上能描述內容的 `title` |
| 只用 `frameborder` 控制邊框 | `frameborder` 是舊式寫法，新程式碼較不建議使用 | 用 CSS 的 `border` 控制邊框 |
| iframe 尺寸寫死且不適合小螢幕 | 固定寬高可能造成手機版溢出 | 使用 CSS 設定 `width: 100%` 並搭配合適高度 |
| 嵌入不可信網站 | iframe 內容可能帶來安全、隱私或使用體驗風險 | 只嵌入可信來源，必要時使用 `sandbox` 限制能力 |
| 以為任何網站都能被 iframe 嵌入 | 目標網站可能禁止被其他網站嵌入 | 若被阻擋，改用一般連結或使用官方提供的嵌入方式 |
| `target` 名稱對不上 | `<a>` 或 `<form>` 找不到對應 iframe | 確認 `target` 的值與 iframe 的 `name` 完全一致 |

### 錯誤範例

```html
<iframe src="https://example.com"></iframe>
```

問題：缺少 `title`，輔助工具使用者不容易知道這個 iframe 的內容是什麼。

### 正確範例

```html
<iframe
  src="https://example.com"
  title="Example 網站內容"
></iframe>
```

修正重點：用 `title` 描述 iframe 的內容或用途。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| `<a>` | `<a target="iframeName">` 可以把連結內容載入指定 iframe |
| `<form>` | `<form target="iframeName">` 可以把提交結果顯示到指定 iframe |
| PDF 與檔案資源 | iframe 可用來預覽部分瀏覽器能顯示的文件資源 |
| CSS | iframe 的尺寸、邊框與響應式呈現通常應交給 CSS 控制 |
| JavaScript | 可以操作 iframe 元素本身；跨來源 iframe 內容會受到瀏覽器安全限制 |
| 可訪問性 | iframe 應提供 `title`，讓使用者知道框架內容用途 |
| 安全性 | 第三方嵌入內容可能需要 `sandbox`、權限限制與來源檢查 |

## 9. 重點整理

- `<iframe>` 用來在目前頁面中嵌入另一個頁面或資源。
- `src` 指定要嵌入的內容，`width` 與 `height` 控制顯示尺寸。
- `name` 可以搭配 `<a>` 或 `<form>` 的 `target`，把內容顯示到指定 iframe。
- `frameborder` 是舊教材常見屬性；新程式碼建議用 CSS 的 `border` 控制邊框。
- iframe 會建立獨立瀏覽上下文，因此要注意效能、安全性與可訪問性。
- 實務上應為 iframe 提供清楚的 `title`。

## 11. 延伸補充

> 補充說明：iframe 常用於第三方內容嵌入，但不是所有內容都適合用 iframe。若只是插入圖片、音訊或影片，通常應優先使用 `<img>`、`<audio>`、`<video>` 等對應元素；若是要嵌入另一個完整頁面或第三方服務，才比較適合使用 `<iframe>`。

> 補充說明：若 iframe 內容來自第三方網站，實務上常需要評估安全限制、權限設定、載入效能與隱私影響。正式專案中可以再研究 `sandbox`、`allow`、`referrerpolicy` 等屬性，但初學階段應先掌握 `src`、`name`、`target` 與尺寸控制。
