# HTML 布局標籤

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 HTML5 常見布局標籤的用途。
- 分辨 `header`、`nav`、`main`、`article`、`section`、`aside`、`footer` 的語意差異。
- 使用語意化標籤建立基本頁面結構。
- 判斷何時應使用 `article`，何時應使用 `section`。
- 理解 HTML 布局標籤與 CSS 排版的分工。
- 避免把布局標籤只當成視覺排版工具。

## 2. 本章定位

本章介紹 HTML5 常見的 semantic layout elements（語意化布局標籤）。

早期網頁常使用大量 `div` 搭配 `class` 做版面區塊，例如 `div.header`、`div.nav`、`div.footer`。這種寫法可以完成排版，但 `div` 本身沒有語意，閱讀程式碼的人、搜尋引擎與輔助工具都需要額外推測區塊用途。

HTML5 提供多個語意化布局標籤，讓文件結構更清楚。這些標籤不只是「換一個名字的 div」，而是用來描述內容在頁面中的角色。

> 來源重點：原始資料指出，HTML5 加入更多具有語意的標籤，能幫助前端人員更有效率地開發網頁。

## 3. 核心概念

### 3.1 布局標籤的重點是語意，不是外觀

HTML 布局標籤的主要任務是描述頁面區塊的意義。例如：

- `header` 表示頁首或某個內容區塊的頭部。
- `nav` 表示一組重要導覽連結。
- `main` 表示頁面主要內容。
- `article` 表示可獨立閱讀或重複發布的內容。
- `section` 表示文件中的主題區塊。
- `aside` 表示與主要內容相關的補充內容。
- `footer` 表示頁尾或某個內容區塊的底部資訊。

這些元素不負責完成複雜排版。版面位置、寬度、間距、顏色、左右欄等視覺效果，應交給 CSS 處理。

```html
<header>網站頁首</header>
<nav>主要導覽</nav>
<main>主要內容</main>
<footer>頁尾資訊</footer>
```

這段 HTML 的重點不是讓畫面自動變好看，而是讓頁面結構更容易理解。

### 3.2 常見頁面結構

一個常見網頁可以拆成頁首、導覽、主要內容、側邊欄與頁尾。

![布局介面](./assets/images/layout-interface.png)

在 HTML 中，可以用語意化布局標籤描述這些區域：

```html
<header>
  <h1>網站名稱</h1>
</header>

<nav>
  <a href="/">首頁</a>
  <a href="/products">商品</a>
  <a href="/contact">聯絡我們</a>
</nav>

<main>
  <article>
    <h2>文章標題</h2>
    <p>文章主要內容。</p>
  </article>

  <aside>
    <h2>相關連結</h2>
    <a href="/related">延伸閱讀</a>
  </aside>
</main>

<footer>
  <p>&copy; 2026 Example Site</p>
</footer>
```

### 3.3 `article` 與 `section` 的核心差異

`article` 和 `section` 都可以包住一組內容，但判斷標準不同。

| 元素 | 判斷重點 | 適合內容 |
|---|---|---|
| `article` | 內容是否可以獨立存在、獨立閱讀、獨立發布 | 部落格文章、新聞、商品評論、論壇貼文、卡片式文章 |
| `section` | 是否是文件中的一個主題區塊 | 文章中的小節、頁面中的功能區、章節段落 |

如果一塊內容本身相對完整，離開目前頁面仍能被理解，通常適合使用 `article`。如果只是把一篇文章或一個頁面分成幾個主題段落，通常適合使用 `section`。

```html
<article>
  <h2>HTML 布局標籤入門</h2>

  <section>
    <h3>為什麼需要布局標籤</h3>
    <p>布局標籤可以讓頁面結構更清楚。</p>
  </section>

  <section>
    <h3>`article` 與 `section` 的差異</h3>
    <p>`article` 強調獨立性，`section` 強調主題分段。</p>
  </section>
</article>
```

> 來源重點：原始資料強調，`article` 裡面可以有多個 `section`。`section` 強調分段或分塊，`article` 更強調內容的獨立性與完整性。

## 4. 語法與基本用法

### 4.1 常見布局標籤

| 標籤 | 語意 | 使用重點 |
|---|---|---|
| `header` | 頁面或區塊的頭部 | 可放標題、Logo、導覽、簡介，不一定只能出現一次 |
| `nav` | 導覽區 | 放主要導覽或重要連結群組，不是所有連結都要包進 `nav` |
| `main` | 頁面主要內容 | 一個頁面通常只應有一個主要 `main` |
| `article` | 可獨立閱讀的內容 | 適合文章、新聞、評論、貼文、商品卡片 |
| `section` | 主題區塊 | 通常應有標題，表示文件中的一個段落或主題 |
| `aside` | 補充內容 | 適合側邊欄、延伸閱讀、相關連結、廣告區 |
| `footer` | 頁面或區塊的底部 | 可放版權、作者資訊、相關連結、聯絡方式 |
| `hgroup` | 標題與副標題群組 | 較少用，需注意目前規格與舊教材說法差異 |

### 4.2 `header`

`header` 表示頁面或某個區塊的頭部內容。常見內容包括標題、Logo、簡介文字、搜尋框或導覽列。

```html
<header>
  <h1>前端學習筆記</h1>
  <p>整理 HTML、CSS 與 JavaScript 的基礎概念。</p>
</header>
```

`header` 可以用在整個頁面的頭部，也可以用在 `article` 或 `section` 內部，表示該內容區塊自己的頭部。

### 4.3 `nav`

`nav` 表示主要導覽區。它應用於頁面中重要的導覽連結群組。

```html
<nav aria-label="主要導覽">
  <a href="/">首頁</a>
  <a href="/html">HTML</a>
  <a href="/css">CSS</a>
  <a href="/javascript">JavaScript</a>
</nav>
```

不是每一組連結都需要放進 `nav`。例如文章內的一個普通參考連結，通常不需要使用 `nav`。

### 4.4 `main`

`main` 表示目前頁面的主要內容。它應排除網站共用的頁首、導覽、側邊補充與頁尾。

```html
<main>
  <h1>HTML 布局標籤</h1>
  <p>本頁主要介紹 HTML5 的語意化布局元素。</p>
</main>
```

> 注意：原始資料提到 `main` 幾乎不使用。從現代語意化 HTML 的角度，`main` 仍是描述頁面主要內容的重要元素。實務上應根據頁面結構合理使用，而不是完全忽略。

### 4.5 `article`

`article` 表示一段可以獨立存在的內容。

```html
<article>
  <h2>HTML 語意化布局的好處</h2>
  <p>語意化布局可以讓文件結構更清楚，也有助於維護與可訪問性。</p>
</article>
```

判斷方式可以問自己：這段內容如果被拿到另一個頁面、RSS、搜尋結果或卡片列表中，是否仍能獨立理解？如果可以，通常適合使用 `article`。

### 4.6 `section`

`section` 表示文件中的一個主題區塊。通常建議搭配標題使用，讓這個區塊的主題明確。

```html
<section>
  <h2>布局標籤的用途</h2>
  <p>布局標籤用來描述頁面中不同區域的語意。</p>
</section>
```

如果只是為了包住一組元素方便套 CSS，但內容沒有明確主題，通常使用 `div` 會比 `section` 更合適。

### 4.7 `aside`

`aside` 表示與主要內容相關，但不是主線內容的一部分。

```html
<aside>
  <h2>延伸閱讀</h2>
  <ul>
    <li><a href="/html/semantic">HTML 語意化標籤</a></li>
    <li><a href="/html/headings">HTML 標題標籤</a></li>
  </ul>
</aside>
```

常見用途包括側邊欄、相關文章、廣告、作者資訊、名詞補充或延伸閱讀。

### 4.8 `footer`

`footer` 表示頁面或區塊的底部資訊。

```html
<footer>
  <p>&copy; 2026 Frontend Notes</p>
  <a href="/privacy">隱私權政策</a>
</footer>
```

`footer` 不一定只能出現在整個頁面的最下方。`article` 裡也可以有自己的 `footer`，例如作者、發布日期或相關連結。

### 4.9 `hgroup`

`hgroup` 用於把一個標題與相關的副標題、替代標題或標語組成一組。

```html
<hgroup>
  <h1>前端學習筆記</h1>
  <p>從 HTML 基礎到實務開發</p>
</hgroup>
```

> 注意：`hgroup` 在 HTML 規格歷史中曾有變動，因此舊教材可能會說它已被移除。現代筆記中應把它視為較少用、需要查規格確認語意的元素；初學者通常先掌握 `header` 搭配標題與段落即可。

## 5. 範例說明

### 範例一：完整頁面布局

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>HTML 布局標籤範例</title>
</head>
<body>
  <header>
    <h1>尚品匯</h1>
  </header>

  <nav aria-label="主要導覽">
    <a href="/">首頁</a>
    <a href="/orders">訂單</a>
    <a href="/cart">購物車</a>
    <a href="/account">我的</a>
  </nav>

  <main>
    <aside>
      <h2>活動專區</h2>
      <nav aria-label="活動導覽">
        <ul>
          <li><a href="/sale">秒殺專區</a></li>
          <li><a href="/member">會員專區</a></li>
          <li><a href="/coupon">領取優惠券</a></li>
          <li><a href="/brands">品牌專區</a></li>
        </ul>
      </nav>
    </aside>

    <article>
      <header>
        <h2>如何學習前端開發</h2>
        <p>從 HTML 結構開始，逐步理解 CSS 與 JavaScript。</p>
      </header>

      <section>
        <h3>第一步：理解 HTML 結構</h3>
        <p>HTML 負責描述頁面內容與語意。</p>
      </section>

      <section>
        <h3>第二步：學習 CSS 排版</h3>
        <p>CSS 負責控制視覺呈現與版面布局。</p>
      </section>

      <section>
        <h3>第三步：加入 JavaScript 互動</h3>
        <p>JavaScript 通常透過 DOM 操作頁面互動。</p>
      </section>
    </article>
  </main>

  <footer>
    <nav aria-label="頁尾連結">
      <a href="/about">關於我們</a>
      <a href="/help">幫助中心</a>
      <a href="/contact">聯絡我們</a>
    </nav>
  </footer>
</body>
</html>
```

這個範例保留原始資料的頁首、導覽、側邊欄、文章內容與頁尾結構，並補上 `main` 與更清楚的標題層級。

範例中的重點是：

- `header` 放網站名稱。
- 第一個 `nav` 是主要導覽。
- `main` 包住本頁真正的主要內容。
- `aside` 放活動專區，屬於補充導覽。
- `article` 放一篇相對完整的內容。
- `article` 內用多個 `section` 分成幾個主題段落。
- `footer` 放頁尾連結。

### 範例二：`article` 裡包含多個 `section`

```html
<article>
  <h2>HTML 布局標籤介紹</h2>

  <section>
    <h3>頁首與導覽</h3>
    <p>`header` 與 `nav` 可以描述網站上方的結構。</p>
  </section>

  <section>
    <h3>主要內容與側邊欄</h3>
    <p>`main` 放主要內容，`aside` 放補充內容。</p>
  </section>

  <section>
    <h3>頁尾資訊</h3>
    <p>`footer` 通常放版權、聯絡方式或相關連結。</p>
  </section>
</article>
```

這種結構表示整篇 `article` 是一個完整主題，而每個 `section` 是文章內的一個分段。

## 6. 實務應用場景

在實務開發中，布局標籤常用於：

- 建立網站首頁、文章頁、商品頁或後台頁面的基本結構。
- 讓 CSS 根據清楚的 DOM 結構進行排版。
- 讓 JavaScript 更容易定位主要內容、導覽或側邊欄。
- 幫助搜尋引擎理解頁面中哪些是主要內容、導覽與補充資訊。
- 幫助螢幕閱讀器等輔助工具辨識頁面區域。
- 在 Vue、React 等元件化開發中，仍保持最後輸出的 DOM 具有正確語意。

> 補充說明：即使使用前端框架，最後瀏覽器接收到的仍是 HTML 與 DOM。元件名稱可以幫助開發者理解程式碼，但使用者、瀏覽器與輔助工具主要仍依賴實際輸出的 HTML 結構。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 把所有區塊都寫成 `div` | `div` 本身沒有語意，頁面結構不清楚 | 有明確語意時優先使用 `header`、`nav`、`main`、`article` 等元素 |
| 為了排版濫用 `section` | `section` 應表示主題區塊，不是單純排版容器 | 沒有明確主題時使用 `div` 或調整結構 |
| 混淆 `article` 與 `section` | 只看外觀分塊，沒有判斷內容是否可獨立 | 可獨立閱讀用 `article`，文章內主題分段用 `section` |
| 每一組連結都使用 `nav` | `nav` 應用於重要導覽，不是普通連結容器 | 只有主要導覽或重要連結群組才使用 `nav` |
| 忽略 `main` | 頁面主要內容缺少明確語意 | 使用 `main` 包住頁面核心內容 |
| 同一頁放多個主要 `main` | 主要內容範圍不清楚，可能影響可訪問性 | 一般頁面只保留一個主要 `main` |
| 把標籤寫成 `< header >` | HTML 標籤名稱與角括號之間不應有空白 | 在程式碼中寫成 `<header>`、`<nav>`、`<footer>` |

### 錯誤範例：用 `section` 當純排版容器

```html
<section class="card-wrapper">
  <div class="card">商品 A</div>
  <div class="card">商品 B</div>
</section>
```

問題：如果這個外層只是為了套版面樣式，沒有明確主題，`section` 的語意不夠準確。

### 較合適寫法

```html
<div class="card-wrapper">
  <article class="card">
    <h2>商品 A</h2>
    <p>商品 A 的簡介。</p>
  </article>

  <article class="card">
    <h2>商品 B</h2>
    <p>商品 B 的簡介。</p>
  </article>
</div>
```

這裡外層 `div` 負責排版容器，每張商品卡片若可獨立理解，則可以使用 `article`。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| CSS | HTML 布局標籤描述結構，CSS 負責排版、寬高、顏色、間距與響應式效果 |
| DOM | 瀏覽器會把 HTML 解析成 DOM Tree，布局標籤會成為 DOM 節點 |
| JavaScript | JavaScript 通常透過 DOM API 操作這些語意化節點 |
| 可訪問性 | `nav`、`main`、`aside` 等元素有助於輔助工具辨識頁面區域 |
| SEO | 清楚的語意結構有助於搜尋引擎理解頁面內容，但 SEO 不只取決於標籤 |
| Vue 或 React | 元件化不會取代 HTML 語意，元件最後仍應輸出合理的 HTML 結構 |

## 9. 重點整理

- HTML 布局標籤用來描述頁面區域的語意，不是用來直接完成視覺排版。
- `header`、`nav`、`main`、`article`、`section`、`aside`、`footer` 是常見語意化布局元素。
- `article` 強調內容可以獨立存在，`section` 強調文件中的主題分段。
- `section` 通常應有標題，否則可能只是錯用成排版容器。
- `main` 用來標示頁面主要內容，不應被忽略。
- CSS 控制畫面呈現，HTML 負責內容結構與語意。

## 11. 延伸補充

> 補充說明：HTML 負責描述內容與結構，DOM 是瀏覽器根據 HTML 建立出的物件模型。CSS 會根據 DOM 與選擇器套用樣式，JavaScript 也多半透過 DOM API 操作頁面。因此，清楚的 HTML 布局不只影響原始碼可讀性，也會影響後續樣式、互動、可訪問性與維護成本。

> 注意：`hgroup` 的規格歷史較容易造成混淆。舊教材可能會說 W3C 已將其刪除，但現行 HTML Living Standard 仍定義此元素。正式專案中若沒有明確需求，通常可先使用 `header` 搭配標題與說明文字，避免初學階段過度依賴較少見的元素。
