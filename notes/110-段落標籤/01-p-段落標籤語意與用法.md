# p 段落標籤：把文章文字分成清楚段落

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `<p>` 標籤代表「段落」的語意。
- 知道什麼時候應該用 `<p>` 包住文字內容。
- 分辨「段落」、「換行」與「排版間距」不是同一件事。
- 避免用 `<br>` 或空的 `<p>` 來製造段落效果。

## 使用情境

當網頁中有一段一段的文字內容時，例如新聞報導、文章內文、產品介紹、個人簡介或課程說明，就需要使用 `<p>` 標籤。

`<p>` 的重點不是「讓文字看起來隔開」，而是告訴瀏覽器與讀者：這裡是一個完整的文字段落。段落之間的距離可以用 CSS 調整，但段落本身的語意應該由 `<p>` 表達。

## 一句話理解

`<p>` 是用來表示一個文字段落的 HTML 標籤。

## 標籤語意

`p` 是 paragraph 的縮寫，意思是「段落」。

在 HTML 中，一個 `<p>` 通常包住一段完整的文字。當文章需要分成多個段落時，就使用多個 `<p>`：

```html
<p>這是第一個段落。</p>
<p>這是第二個段落。</p>
```

這樣寫不只是讓畫面分段，也讓 HTML 結構清楚表達文字內容的組織方式。

## 基本語法

```html
<p>段落文字</p>
```

`<p>` 有開始標籤與結束標籤：

- `<p>`：段落開始。
- `段落文字`：這個段落的內容。
- `</p>`：段落結束。

## 完整範例

```html
<p>
    在中國IT教育行業發展的軌跡中，能始終秉承著初心做教育的企業為數不多，而在李開復老師的心中，傳智播客就是其中之一。在11年的風雨歷程中，傳智播客在中國IT教育行業留下了“為莘莘學子改變命運而講課，為千萬學生少走彎路而著書”的獨特烙印。同時，傳智播客也憑藉著良好的口碑、更注重品質的教學及優秀的業績增長成功吸引了眾多知名投資者們的青睞，其中不乏創新工場、經緯、君度、藍圖、北城壹號等多個投資機構。
</p>

<p>
    不久前，傳智播客新加入了幾位知名股東，其中包括創投圈的大咖——創新工場的李開復老師。在中國，李開復老師至少影響了70和80後兩代人，然而翻開他的履歷，我們並沒有看到諸如馬雲、馬化騰那樣堪稱傳奇的成功經歷，我們看到的只是一位厚積薄發、深耕在世界IT互聯網行業多年的"導師"：
</p>
```

## 範例拆解

這段範例中有兩個 `<p>`，代表兩個不同的文字段落。

第一個 `<p>` 描述傳智播客在 IT 教育行業中的發展與投資背景。這些句子共同說明同一個主題，所以放在同一個段落裡。

第二個 `<p>` 開始談到新股東與李開復老師，主題已經轉到另一個敘述重點，因此使用新的 `<p>` 開始另一個段落。

這就是 `<p>` 最常見的使用方式：依照內容主題與閱讀節奏，把長文字切成多個段落。

## 瀏覽器預設效果

瀏覽器通常會替 `<p>` 加上預設的上下外距，所以多個段落之間看起來會有垂直間距。

另外，段落中的文字會依照父層容器的可用寬度自動換行。當瀏覽器視窗變窄，或父層容器寬度改變時，同一段文字的換行位置也會跟著改變。

這表示你不需要為了讓段落文字換行而手動插入很多 `<br>`。瀏覽器會根據版面寬度自動處理一般文字換行。

## 常用屬性

`<p>` 沒有必填的專屬屬性。實務上常見的是搭配全局屬性使用：

```html
<p class="lead">這是文章開頭的重點段落。</p>
<p id="summary">這是摘要段落。</p>
```

- `class`：方便用 CSS 選取並設定樣式。
- `id`：替特定段落建立唯一識別，必要時也可作為頁內連結目標。

屬性可以幫助樣式與互動，但不會改變 `<p>` 的核心語意：它仍然是一個段落。

## 常見錯誤

### 用 `<br>` 製造段落

錯誤寫法：

```html
第一段文字。<br><br>
第二段文字。
```

`<br>` 表示換行，不表示段落。連續使用 `<br>` 只是讓畫面看起來有空隙，卻沒有把內容標記成不同段落。

較好的寫法：

```html
<p>第一段文字。</p>
<p>第二段文字。</p>
```

如果內容本來就是兩個段落，應該用兩個 `<p>` 表達。

### 用空的 `<p>` 製造間距

錯誤寫法：

```html
<p>第一段文字。</p>
<p></p>
<p>第二段文字。</p>
```

空的 `<p>` 沒有實際內容，只是拿語意標籤做排版用途。這會讓 HTML 結構變得不乾淨，也可能影響輔助技術理解頁面。

較好的做法是保留正確的段落結構，再用 CSS 控制間距：

```html
<p>第一段文字。</p>
<p>第二段文字。</p>
```

```css
p {
  margin-bottom: 1rem;
}
```

### 把所有文字都塞進同一個 `<p>`

如果一篇文章有多個主題或多個自然段，卻全部放在同一個 `<p>` 裡，讀者會很難閱讀，HTML 結構也無法反映文章的層次。

當內容轉換到新的重點時，就應該開始新的 `<p>`。

## 實務判斷準則

可以用下面幾個問題判斷是否該使用 `<p>`：

1. 這是一段完整的文字敘述嗎？是的話，使用 `<p>`。
2. 這段文字和下一段文字是否有明顯的主題或語氣切換？有的話，拆成兩個 `<p>`。
3. 只是想讓畫面多一點空白嗎？不要新增空 `<p>`，改用 CSS。
4. 只是需要在同一段文字中強制換到下一行嗎？才考慮使用 `<br>`。

## 重點整理

- `<p>` 用來表示文字段落。
- 多段文章內容應該使用多個 `<p>`，而不是用連續 `<br>` 分隔。
- 瀏覽器會依容器寬度自動替段落文字換行。
- 段落間距是 CSS 的責任，不應該用空 `<p>` 製造。
- 寫 HTML 時要先考慮內容語意，再處理視覺樣式。
