# 文本格式化標籤：先判斷語意，再決定標籤

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨「讓文字看起來不同」與「文字本身有語意」的差異。
- 知道 `<strong>`、`<em>`、`<ins>`、`<del>` 的主要語意。
- 理解 `<b>`、`<i>`、`<u>`、`<s>` 不是單純的 CSS 替代品。
- 在實作時判斷什麼時候該用 HTML 標籤，什麼時候該交給 CSS。

## 問題情境

做網頁時，很常遇到這些需求：

- 想讓一句警告文字變粗。
- 想讓某個詞看起來像斜體。
- 想在文字下方加底線。
- 想把原價畫上刪除線。

如果只從畫面效果出發，很容易直接挑看起來剛好的標籤，例如想加粗就用 `<strong>`，想畫底線就用 `<ins>`。但 HTML 標籤的重點不是「預設長什麼樣子」，而是「這段內容代表什麼意思」。

## 一句話理解

文本格式化標籤要先看文字的語意；純粹的粗體、斜體、底線、刪除線外觀，通常應該交給 CSS。

## 先分清楚：語意與外觀

許多文本格式化標籤在瀏覽器裡會有預設樣式：

- `<strong>` 預設通常會變粗。
- `<em>` 預設通常會傾斜。
- `<ins>` 預設通常會加底線。
- `<del>` 預設通常會有刪除線。

但這些預設樣式不是使用它們的真正理由。使用標籤時要先問：

> 這段文字本身是否真的有這個語意？

如果答案是「只是想改外觀」，就應該用 CSS。  
如果答案是「文字本身確實有重要、強調、新增、刪除等意義」，才使用對應的 HTML 標籤。

## 常見標籤的語意差異

| 標籤 | 常見預設外觀 | 主要語意 | 適合情境 |
| --- | --- | --- | --- |
| `<strong>` | 加粗 | 重要、嚴重、需要優先注意 | 警告、重要提醒、必須先讀的內容 |
| `<em>` | 傾斜 | 語氣上的強調 | 想改變一句話的語氣重點 |
| `<ins>` | 底線 | 文件中新增的內容 | 顯示修訂後新增的文字 |
| `<del>` | 刪除線 | 文件中移除的內容 | 顯示已刪除或不再有效的內容 |
| `<b>` | 加粗 | 吸引注意但不表示更重要 | 關鍵字、產品名稱、摘要中的醒目文字 |
| `<i>` | 傾斜 | 特定語境中的不同聲音或術語 | 技術術語、外文詞、音譯、想法 |
| `<u>` | 底線 | 非連結的特殊標示 | 拼字錯誤標記、專有名稱標示 |
| `<s>` | 刪除線 | 內容已不準確或不再相關 | 原價、過期資訊 |

這些標籤不是單純成對替換。`<strong>` 和 `<b>` 可能都會變粗，但前者表示「重要」，後者只是讓文字被注意到；`<del>` 和 `<s>` 可能都有刪除線，但前者表示文件修訂中的刪除，後者多用於已不正確或不再適用的資訊。若要標示作品標題，通常應使用 `<cite>`，不要只因為作品名稱常被排成斜體就改用 `<i>`。

## 範例拆解

先看一組常見寫法：

```html
<b>關鍵字</b>
<strong>警告：請先存檔</strong>

<u>拼字錯誤標記</u>
<ins>新增內容</ins>

<i>技術術語</i>
<em>真的</em>

<s>原價 100 元</s>
<del>已刪除的內容</del>
```

逐段來看：

- `<b>關鍵字</b>`：只是讓「關鍵字」在文字中醒目，不代表它比其他內容更嚴重或更重要。
- `<strong>警告：請先存檔</strong>`：這是需要讀者優先注意的警告，使用 `<strong>` 比單純加粗更合適。
- `<u>拼字錯誤標記</u>`：底線用來標示特殊狀態，不代表文件新增內容。
- `<ins>新增內容</ins>`：表示這段文字是在文件修訂中被加入的內容。
- `<i>技術術語</i>`：可以用來標出特定術語或不同語境的文字。
- `<em>真的</em>`：表示語氣上的強調，會影響一句話的重點。
- `<s>原價 100 元</s>`：原價已不再適用，但不一定表示文件修訂時刪除了這段文字。
- `<del>已刪除的內容</del>`：表示這段內容在文件版本中被移除。

## 錯誤寫法與修正

### 只想加粗，卻誤用 `<strong>`

```html
<!-- 不建議：只是想讓商品名稱醒目 -->
<strong>新品上市</strong>
```

如果這段文字只是視覺上要醒目，不是警告、重要提醒或需要優先注意的內容，使用 `<strong>` 會讓語意變得過重。

可以改成：

```html
<span class="highlight">新品上市</span>
```

再用 CSS 控制外觀：

```css
.highlight {
  font-weight: 700;
}
```

### 只想畫底線，卻誤用 `<ins>`

```html
<!-- 不建議：這不是新增內容，只是想畫底線 -->
<ins>活動重點</ins>
```

`<ins>` 表示文件中新增的內容。如果只是想讓文字有底線，應該使用 CSS。

```html
<span class="underlined">活動重點</span>
```

```css
.underlined {
  text-decoration: underline;
}
```

### 把 `<del>` 當成一般刪除線樣式

```html
<!-- 不建議：這只是促銷原價，不是文件修訂刪除 -->
<del>原價 100 元</del>
```

如果是在商品價格中表示原價已不再適用，通常 `<s>` 更貼近語意。

```html
<s>原價 100 元</s>
<strong>特價 80 元</strong>
```

## 實務判斷準則

選擇文本格式化標籤時，可以照這個順序判斷：

1. 先問：這段文字是否有特殊語意？
2. 如果只是外觀需求，用 `<span>` 搭配 CSS。
3. 如果是重要或警告，用 `<strong>`。
4. 如果是語氣上的強調，用 `<em>`。
5. 如果是文件修訂新增，用 `<ins>`。
6. 如果是文件修訂刪除，用 `<del>`。
7. 如果只是醒目、術語、特殊標記、過期資訊，再考慮 `<b>`、`<i>`、`<u>`、`<s>` 是否符合語意。

不要因為某個標籤的預設外觀剛好符合需求，就直接把它當成樣式工具。HTML 負責內容結構與語意，CSS 才負責視覺呈現。

## 重點整理

- 文本格式化標籤會影響預設外觀，但選用理由應該是語意。
- `<strong>` 表示重要，`<em>` 表示語氣強調，`<ins>` 表示新增內容，`<del>` 表示刪除內容。
- `<b>`、`<i>`、`<u>`、`<s>` 也有各自語意，不是單純的樣式捷徑。
- 只想控制粗體、斜體、底線、刪除線時，通常使用 CSS。
- 實務上先判斷內容意義，再決定標籤；不要先看畫面效果再反推標籤。
