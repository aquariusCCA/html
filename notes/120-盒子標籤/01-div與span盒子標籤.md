# div 與 span：沒有語意的盒子標籤

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 `<div>` 和 `<span>` 都是沒有語意的容器標籤。
- 分辨 `<div>` 與 `<span>` 在預設排列方式上的差異。
- 判斷什麼時候適合用 `<div>`，什麼時候適合用 `<span>`。
- 避免把無語意容器誤當成所有 HTML 結構的唯一選擇。

## 使用情境

寫 HTML 時，常常會遇到一種需求：你只是想把幾個內容包起來，方便之後用 CSS 排版、套樣式，或用 JavaScript 操作，但這個外層容器本身沒有特別的內容意義。

例如：

- 把一整個區塊包起來，之後用 CSS 設定寬度、背景色或排版。
- 把一句話中的某幾個字包起來，之後單獨改顏色或加效果。

這種時候就會用到 `<div>` 和 `<span>`。它們的共同點是「沒有語意」，差別在於 `<div>` 偏向包住一整塊內容，`<span>` 偏向包住一小段行內文字。

## 一句話理解

`<div>` 是沒有語意的區塊容器，常被當成「大盒子」；`<span>` 是沒有語意的行內容器，常被當成「小盒子」。

## div 與 span 的基本差異

`div` 是 division 的縮寫，有分割、分區的意思。它本身不代表文章、標題、導覽或按鈕等具體語意，只是一個通用的區塊容器。

`span` 有跨度、跨距的意思。它同樣沒有語意，通常用來包住一小段文字或行內內容。

| 標籤 | 預設顯示特性 | 常見用途 |
| --- | --- | --- |
| `<div>` | 預設是區塊元素，通常會獨占一行 | 包住一塊內容、做版面分區、建立排版容器 |
| `<span>` | 預設是行內元素，不會強制前後換行 | 包住一小段文字、局部套樣式 |

要注意的是，「區塊」和「行內」是預設顯示方式，不是永遠不能改變。之後學 CSS 時，可以透過 `display` 改變它們的排列方式。

## 基本語法

```html
<div>我是一個 div 標籤，我通常會單獨占一行</div>
<div>我也是一個 div 標籤，我通常會單獨占一行</div>

<span>百度</span>
<span>新浪</span>
<span>搜狐</span>
```

這段程式碼中，兩個 `<div>` 預設會上下排列；三個 `<span>` 則會盡量出現在同一行。若畫面寬度不足，`<span>` 的內容仍可能因版面寬度而自然換行。

## 範例拆解

```html
<div>我是一個 div 標籤，我通常會單獨占一行</div>
```

這裡的 `<div>` 用來包住一整段內容。因為它預設是區塊元素，所以瀏覽器通常會讓它獨占一行。

```html
<span>百度</span>
<span>新浪</span>
<span>搜狐</span>
```

這裡的 `<span>` 分別包住三小段文字。因為它預設是行內元素，所以不會像 `<div>` 一樣強制每個標籤都換到新的一行。

## 實務判斷準則

如果你要包住一整塊內容，例如卡片、區域、版面外框，可以考慮使用 `<div>`。

```html
<div class="card">
  <h2>課程名稱</h2>
  <p>這是一門 HTML 入門課程。</p>
</div>
```

如果你只想包住一句話中的某幾個字，可以考慮使用 `<span>`。

```html
<p>這堂課的重點是 <span class="keyword">HTML 結構</span>。</p>
```

但如果內容本身有明確語意，應該優先使用有語意的標籤，而不是直接用 `<div>` 或 `<span>`。

例如，頁面標題應該使用 `<h1>` 到 `<h6>`，段落應該使用 `<p>`，導覽區之後可以使用 `<nav>`。`<div>` 和 `<span>` 適合用在「沒有更合適語意標籤」的情況。

## 常見錯誤

### 錯誤一：整個頁面都只用 div

```html
<div>網站標題</div>
<div>第一段內容</div>
<div>第二段內容</div>
```

這樣寫畫面可能看得出來，但 HTML 結構沒有清楚表達內容意義。瀏覽器、搜尋引擎、輔助工具和其他工程師都比較難判斷哪一段是標題、哪一段是正文。

比較好的做法是，有明確語意時使用對應標籤：

```html
<h1>網站標題</h1>
<p>第一段內容</p>
<p>第二段內容</p>
```

### 錯誤二：用 span 包住整個大區塊

```html
<span>
  <h2>最新消息</h2>
  <p>這裡是最新消息內容。</p>
</span>
```

`<span>` 適合包住行內的小片段，不適合拿來承擔整個區塊的結構。若要包住一塊內容，通常應該使用 `<div>` 或更有語意的區塊標籤。

### 錯誤三：以為 span 一定永遠不會換行

`<span>` 預設不會像區塊元素一樣強制前後換行，但如果行內空間不足，文字仍然可能自然換到下一行。它的重點是「不主動建立新區塊」，不是「永遠不能換行」。

## 重點整理

- `<div>` 和 `<span>` 都是沒有語意的容器標籤。
- `<div>` 預設是區塊容器，通常用來包住一整塊內容。
- `<span>` 預設是行內容器，通常用來包住一小段文字。
- 如果有更符合內容意義的 HTML 標籤，應該優先使用語意標籤。
- `<div>` 和 `<span>` 的預設排列方式之後可以用 CSS 的 `display` 調整。
