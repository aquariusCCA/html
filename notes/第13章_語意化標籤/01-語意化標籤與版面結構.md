# 語意化標籤與版面結構

早期切版常常會看到滿版的 `div`：

```html
<div class="header">頭部</div>
<div class="nav">導覽</div>
<div class="section">內容區塊</div>
<div class="footer">頁尾</div>
```

這樣的寫法可以做出畫面，但 HTML 本身沒有說清楚每一塊內容的用途。人類需要依賴 class 名稱猜測結構，搜尋引擎、螢幕閱讀器或其他工具也較難直接理解頁面內容。

HTML5 語意化標籤就是為了解決這類問題而出現的。它讓我們可以用更有意義的標籤描述頁面結構，例如 `header` 表示頁首，`nav` 表示導覽，`article` 表示一篇可獨立閱讀的內容。

## 什麼是語意化

語意化的重點不是「換一個比較新的標籤」，而是讓 HTML 標籤本身能說明內容的角色。

例如：

```html
<nav>
  <a href="/">首頁</a>
  <a href="/articles">文章</a>
  <a href="/about">關於我們</a>
</nav>
```

讀到這段 HTML 時，不需要額外解釋就能知道這是一組導覽連結。這就是語意化的價值：讓結構本身帶有意義。

語意化也和「上下文」有關。單看一段文字或一個元素，可能不容易判斷它在頁面中的角色；放進清楚的 HTML 結構後，前後關係會幫助人和工具理解內容。例如同樣是一串連結，放在 `nav` 裡通常代表主要導覽，放在 `footer` 裡可能代表頁尾資訊或次要連結。

## 常見的語意化版面標籤

下面這張圖展示常見語意化標籤在頁面版面中的位置：

![HTML5 語意化版面區塊示意圖](../../origin/第13章_語意化標籤/assets/images/semantic-tags-img-001-fe28a0.png)

常見標籤如下：

| 標籤 | 主要語意 | 常見使用情境 |
| --- | --- | --- |
| `<header>` | 頁面或區段的開頭 | 網站標題、Logo、前言、導覽容器 |
| `<nav>` | 主要導覽區域 | 主選單、頁面目錄、重要導覽連結 |
| `<main>` | 頁面主要內容 | 每個頁面最核心、最獨特的內容 |
| `<article>` | 可獨立成篇的內容 | 文章、卡片、新聞、部落格貼文 |
| `<section>` | 具有主題的一個區段 | 章節、功能區、內容分段 |
| `<aside>` | 補充或旁支內容 | 側欄、相關文章、廣告、補充說明 |
| `<footer>` | 頁面或區段的結尾 | 版權資訊、作者資訊、相關連結 |

其中 `main` 在原始筆記中沒有列出，但實務切版很常一起使用。通常一個頁面只會有一個可見的 `main`，用來包住本頁最重要、最獨特的內容。

`main` 不適合拿來包住整個頁面，也不應把頁首、主導覽或頁尾一起放進去。如果 `aside` 放在 `main` 裡，它通常應該是和本頁主要內容直接相關的補充資訊。

## 語意化標籤帶來的好處

語意化標籤的好處不只在於「比較符合 HTML5」，而是能讓同一份 HTML 被更多角色理解。

第一，對使用者比較友善。即使 CSS 載入失敗，語意清楚的 HTML 仍然比較容易呈現出合理的內容結構。

第二，有助於搜尋引擎理解頁面內容。SEO 是 Search Engine Optimization，也就是搜尋引擎最佳化。語意化標籤可以讓搜尋引擎更容易判斷頁面的結構與重點，但它不是保證排名變高的魔法；真正的 SEO 還包含內容品質、網站速度、連結結構、使用體驗等因素。

第三，對輔助技術更友善。螢幕閱讀器可以利用 `nav`、`main`、`article` 等語意結構，幫助使用者快速理解與跳轉頁面區塊。

第四，讓開發與維護更容易。當團隊看到 `header`、`nav`、`article`、`footer` 時，比看到一堆沒有語意的 `div` 更容易理解版面架構。

## 從 div 佈局改成語意化佈局

`div` 本身沒有語意，適合用在純粹包裝、排版或沒有明確內容角色的情境。如果一個區塊明明代表導覽、頁首、文章或頁尾，使用語意化標籤通常會更合適。

以下是單純用 `div` 的寫法：

```css
.header,
.nav {
  height: 120px;
  background-color: pink;
  border-radius: 15px;
  width: 800px;
  margin: 15px auto;
}

.section {
  width: 500px;
  height: 300px;
  background-color: skyblue;
}
```

```html
<body>
  <div class="header">頭部標籤</div>
  <div class="nav">導航欄標籤</div>
  <div class="section">某個區域</div>
</body>
```

這段程式碼可以運作，但 HTML 的標籤本身沒有說明這些區塊是什麼。

改成語意化標籤後，可以這樣寫：

```css
header,
nav {
  height: 120px;
  background-color: pink;
  border-radius: 15px;
  width: 800px;
  margin: 15px auto;
}

section {
  width: 500px;
  height: 300px;
  background-color: skyblue;
}
```

```html
<body>
  <header>頭部標籤</header>
  <nav>導航欄標籤</nav>
  <section>
    <h2>某個區域</h2>
  </section>
</body>
```

改寫後，HTML 結構變得更容易閱讀。`header` 表示頁首，`nav` 表示導覽，`section` 表示具有主題的一個區段。

注意 `section` 通常應該搭配標題，因為它表示一個有主題的內容區塊。如果只是為了排版包一層容器，使用 `div` 反而更自然。

## 一個更完整的頁面結構

實務上，語意化標籤常常會組合成下面這種頁面骨架：

```html
<body>
  <header>
    <h1>前端學習筆記</h1>
    <nav>
      <a href="/">首頁</a>
      <a href="/html">HTML</a>
      <a href="/css">CSS</a>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h2>語意化標籤入門</h2>
        <p>發布日期：2026-06-06</p>
      </header>

      <section>
        <h3>為什麼需要語意化</h3>
        <p>語意化標籤可以讓頁面結構更清楚。</p>
      </section>

      <section>
        <h3>常見標籤</h3>
        <p>常見標籤包含 header、nav、main、article、section、aside、footer。</p>
      </section>
    </article>

    <aside>
      <h2>相關文章</h2>
      <a href="/html/basic">HTML 基礎結構</a>
    </aside>
  </main>

  <footer>
    <p>&copy; 2026 Frontend Notes</p>
  </footer>
</body>
```

這個範例可以看到幾個層級：

- 最外層的 `header` 是整個網站或頁面的頁首。
- `main` 包住本頁主要內容。
- `article` 是一篇可獨立閱讀的文章。
- `article` 裡也可以有自己的 `header`，表示文章標題與發布資訊。
- `section` 用來切分文章中的主題段落。
- `aside` 放與主內容相關，但不是主線內容的補充資訊。
- `footer` 放頁尾資訊。

## 使用語意化標籤的判斷原則

不要只因為 HTML5 有這些標籤，就把所有 `div` 都換掉。選標籤時可以先問自己：

- 這個區塊在頁面中扮演什麼角色？
- 它是不是主要導覽？如果是，可以使用 `nav`。
- 它是不是一篇可獨立閱讀或轉載的內容？如果是，可以使用 `article`。
- 它是不是某個有標題、有主題的段落？如果是，可以使用 `section`。
- 它只是為了排版、對齊或套 CSS 嗎？如果是，`div` 可能更適合。

語意化不是消滅 `div`。真正的目標是：有語意的地方使用語意化標籤，沒有語意、只負責包裝的地方保留 `div`。

## 常見錯誤

### 把 section 當成萬用 div

錯誤示範：

```html
<section class="card-wrapper">
  <div class="card">商品卡片</div>
</section>
```

如果 `section` 只是為了包住卡片並套版面樣式，而且沒有自己的主題與標題，用 `div` 會更合理：

```html
<div class="card-wrapper">
  <div class="card">商品卡片</div>
</div>
```

### 把 nav 用在所有連結區塊

`nav` 應該用於主要導覽區域，不是任何有連結的地方都要使用。文章底部的一個「延伸閱讀」連結不一定需要包成 `nav`，除非它真的是一組重要導覽。

### 只為了 SEO 而語意化

語意化確實有助於搜尋引擎理解頁面，但它同時也服務於使用者、輔助技術與開發維護。不要把語意化理解成只針對搜尋引擎的技巧。

## 瀏覽器支援注意事項

現代瀏覽器普遍支援 HTML5 語意化標籤。若需要支援較舊的瀏覽器，可能要補上區塊顯示設定：

```css
header,
nav,
main,
article,
section,
aside,
footer {
  display: block;
}
```

非常舊的 IE 版本可能還需要額外相容處理。一般現代專案中，是否需要處理舊瀏覽器，應依專案的目標使用者與瀏覽器支援範圍決定。

## 小練習

請把下面的 `div` 結構改寫成更有語意的 HTML：

```html
<div class="page-header">
  <h1>城市旅遊指南</h1>
  <div class="menu">
    <a href="/">首頁</a>
    <a href="/spots">景點</a>
    <a href="/food">美食</a>
  </div>
</div>

<div class="content">
  <div class="post">
    <h2>台北一日遊</h2>
    <p>這篇文章介紹台北一日遊路線。</p>
  </div>
</div>

<div class="page-footer">
  <p>Copyright 2026</p>
</div>
```

可以先判斷每個區塊的角色，再選擇 `header`、`nav`、`main`、`article`、`footer` 等標籤。

參考改寫：

```html
<header class="page-header">
  <h1>城市旅遊指南</h1>
  <nav class="menu">
    <a href="/">首頁</a>
    <a href="/spots">景點</a>
    <a href="/food">美食</a>
  </nav>
</header>

<main class="content">
  <article class="post">
    <h2>台北一日遊</h2>
    <p>這篇文章介紹台北一日遊路線。</p>
  </article>
</main>

<footer class="page-footer">
  <p>Copyright 2026</p>
</footer>
```

這裡的 `header` 表示頁首，`nav` 表示主要導覽，`main` 包住本頁主要內容，`article` 表示可獨立閱讀的一篇內容，`footer` 表示頁尾資訊。

## 重點整理

- 語意化標籤讓 HTML 結構本身帶有意義。
- `div` 沒有錯，但它適合用在沒有明確語意、只負責包裝或排版的地方。
- `header`、`nav`、`main`、`article`、`section`、`aside`、`footer` 是常見的頁面結構標籤。
- 語意化有助於使用者閱讀、搜尋引擎理解、輔助技術解析與團隊維護。
- 選標籤時應先理解內容角色，不要只看畫面長相。
