# br 換行標籤：在文字中強制換行

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 `<br>` 標籤的語意與使用情境。
- 知道 `<br>` 為什麼不需要結束標籤。
- 分辨什麼時候該用 `<br>`，什麼時候該用 `<p>` 或 CSS 控制間距。
- 避免用連續 `<br>` 來做版面排版。

## 使用情境

在一般文字排版中，瀏覽器會根據容器寬度自動換行。例如一段文字太長時，它會自然折到下一行。

但有些內容不是「因為寬度不夠才換行」，而是換行本身就是內容的一部分。常見情境包括：

- 地址，例如公司地址分成縣市、路名與樓層。
- 詩句、歌詞或短句，每一行都有自己的節奏。
- 聯絡資訊，例如姓名、電話、Email 分行顯示。

這種需要在指定位置換行的文字，就可以使用 `<br>`。

## 一句話理解

`<br>` 是用來在文字中插入「強制換行」的 HTML 標籤。

`br` 來自英文 `break`，可以理解成把目前這一行打斷，讓後面的內容從新的一行繼續。

## 標籤語意

`<br>` 的語意是「換行」，不是「段落」、不是「空白」、也不是「版面間距」。

這點很重要，因為 HTML 標籤應該優先表達內容的意義：

- 如果內容只是同一段文字中的換行，用 `<br>`。
- 如果內容是兩個不同段落，用 `<p>`。
- 如果只是想讓畫面上下多一點空間，用 CSS 的 `margin` 或 `padding`。

## 基本語法

```html
第一行文本<br>
第二行文本
```

在瀏覽器中，`第二行文本` 會從下一行開始顯示。

`<br>` 是 HTML 的空元素，也就是 void element。它只需要開始標籤，不會包住任何內容，所以不需要寫成：

```html
<br></br>
```

在 HTML 中，以下寫法通常也能被瀏覽器解析：

```html
第一行文本<br />第二行文本
```

但這個斜線在 HTML 裡沒有額外作用。寫 HTML 時，慣用寫法是直接使用 `<br>`。

## 範例拆解

假設要顯示一段地址：

```html
台北市中正區<br>
重慶南路一段 122 號<br>
3 樓
```

這段 HTML 的意思是：

- `台北市中正區`：第一行地址資訊。
- `<br>`：在這裡強制換行。
- `重慶南路一段 122 號`：第二行地址資訊。
- 第二個 `<br>`：再換到下一行。
- `3 樓`：第三行地址資訊。

這裡使用 `<br>` 是合理的，因為地址的分行是內容呈現的一部分，不只是為了把畫面撐開。

## `<br>` 與 `<p>` 的差異

`<br>` 只讓後面的內容從新的一行開始，不代表新的段落。

`<p>` 則代表一個段落。瀏覽器預設通常會在段落上下加上外距，所以看起來會有比較明顯的垂直間距。

```html
<!-- 同一段內容中的換行 -->
地址：台北市中正區<br>
重慶南路一段 122 號

<!-- 兩個獨立段落 -->
<p>這是第一段。</p>
<p>這是第二段。</p>
```

如果兩段文字在語意上是不同段落，應該使用 `<p>`。如果只是同一段內容中必須分行，才使用 `<br>`。

## 常見錯誤

### 錯誤一：用很多 `<br>` 製造垂直間距

```html
<h1>文章標題</h1>
<br>
<br>
<p>文章內容...</p>
```

這種寫法看起來可以把內容往下推，但問題是它把「排版間距」寫進 HTML 結構裡。之後如果要調整畫面，就會很難維護，也會讓 HTML 失去清楚的語意。

比較好的做法是把間距交給 CSS：

```html
<h1>文章標題</h1>
<p>文章內容...</p>
```

```css
h1 {
  margin-bottom: 24px;
}
```

### 錯誤二：把段落分隔寫成 `<br><br>`

```html
這是第一段文字。<br><br>
這是第二段文字。
```

如果這兩段文字本來就是不同段落，應該使用 `<p>`：

```html
<p>這是第一段文字。</p>
<p>這是第二段文字。</p>
```

`<p>` 能清楚表達「這是段落」，也讓瀏覽器、搜尋引擎與輔助技術更容易理解內容結構。

### 錯誤三：以為 `<br>` 可以包住內容

```html
<br>這是一行文字</br>
```

`<br>` 是空元素，不包住內容，也不需要結束標籤。正確做法是把它放在需要換行的位置：

```html
第一行文字<br>
第二行文字
```

## 實務判斷準則

可以用 `<br>` 的情境：

- 地址、詩句、歌詞、聯絡資訊等分行有內容意義的文字。
- 同一段文字中，確實需要在固定位置換行。

不建議用 `<br>` 的情境：

- 想拉開段落之間的距離。
- 想控制版面區塊的位置。
- 想把多個段落硬塞在同一個文字節點裡。

判斷時可以先問自己一句話：這個換行是內容的一部分，還是只是想調整畫面？如果是內容的一部分，可以考慮 `<br>`；如果只是調整畫面，通常應該改用語意標籤或 CSS。

## 重點整理

- `<br>` 的語意是強制換行。
- `<br>` 是 void element，不需要結束標籤。
- `<br />` 通常能被解析，但在 HTML 中斜線沒有額外作用。
- `<br>` 適合用在地址、詩句、歌詞等換行有意義的文字。
- 不要連續使用 `<br>` 來做版面間距；段落用 `<p>`，間距用 CSS。
