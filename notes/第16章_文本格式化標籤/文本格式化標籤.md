# HTML 文本格式化標籤

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 HTML 文本格式化標籤的用途。
- 分辨 `<b>`、`<strong>`、`<u>`、`<ins>`、`<i>`、`<em>`、`<s>`、`<del>` 的差異。
- 理解「視覺效果」與「語意」不是同一件事。
- 在需要加粗、底線、斜體或刪除線時，選擇更符合內容語意的標籤。
- 避免只為了外觀而誤用 HTML 標籤。

## 2. 本章定位

本章介紹 HTML 中常見的文本格式化標籤。這些標籤會讓文字在瀏覽器中呈現不同外觀，例如加粗、底線、傾斜或刪除線。

不過，學習這些標籤時不能只記「畫面長怎樣」。HTML 的核心任務是描述內容與語意，外觀通常應交給 CSS 控制。因此，當兩個標籤看起來相似時，更重要的是理解它們背後代表的意思。

> 來源重點：原始資料列出四組常見對照：`<b>` 與 `<strong>`、`<u>` 與 `<ins>`、`<i>` 與 `<em>`、`<s>` 與 `<del>`，並建議優先使用後者，因為語意更清楚。

## 3. 核心概念

文本格式化標籤用來標示一段文字的特殊狀態或語氣。它們常見的畫面效果如下：

| 類型 | 常見視覺效果 | 常見標籤 |
|---|---|---|
| 加粗 | 文字變粗 | `<b>`、`<strong>` |
| 底線 | 文字下方有線 | `<u>`、`<ins>` |
| 傾斜 | 文字呈現斜體 | `<i>`、`<em>` |
| 刪除線 | 文字中間有線 | `<s>`、`<del>` |

這些標籤容易混淆，因為瀏覽器預設樣式可能很像。例如 `<b>` 和 `<strong>` 通常都會顯示為粗體，`<i>` 和 `<em>` 通常都會顯示為斜體。

真正的差異在於語意：

- `<strong>` 表示內容具有強烈重要性。
- `<em>` 表示語氣上的強調。
- `<ins>` 表示這段內容是新增或插入的內容。
- `<del>` 表示這段內容是被刪除的內容。

也就是說，選擇標籤時應先問：「這段文字在內容上代表什麼意思？」而不是只問：「我想讓它看起來怎樣？」

> 補充說明：若只是單純想改變文字外觀，例如讓某段文字變紅、變粗或加底線，通常應優先使用 CSS，而不是只靠 HTML 標籤控制樣式。

## 4. 語法與基本用法

### 4.1 加粗：`<b>` 與 `<strong>`

```html
<b>加粗文字</b>
<strong>重要文字</strong>
```

| 標籤 | 語意 | 適合情境 |
|---|---|---|
| `<b>` | 讓文字在視覺上被注意，但不表示更重要 | 關鍵字、產品名稱、摘要中的醒目字詞 |
| `<strong>` | 表示內容重要、需要特別注意 | 警告、重點提醒、重要結論 |

如果這段文字不只是要變粗，而是內容上真的很重要，應優先使用 `<strong>`。

### 4.2 底線：`<u>` 與 `<ins>`

```html
<u>帶底線的文字</u>
<ins>新增的文字</ins>
```

| 標籤 | 語意 | 適合情境 |
|---|---|---|
| `<u>` | 表示非文字本身語氣的標註，常見為拼字錯誤或特殊標記 | 標註文字、特殊註記 |
| `<ins>` | 表示新增、插入的內容 | 文件修訂、版本差異、補充新增內容 |

如果要表達「這段內容是後來新增的」，應使用 `<ins>`，而不是只用 `<u>` 做出底線效果。

### 4.3 傾斜：`<i>` 與 `<em>`

```html
<i>傾斜文字</i>
<em>強調語氣</em>
```

| 標籤 | 語意 | 適合情境 |
|---|---|---|
| `<i>` | 表示與一般文字不同的語氣、術語、外文詞或作品名等 | 技術詞、外文詞、另一種語氣 |
| `<em>` | 表示語氣上的強調 | 句子中特別要強調的字詞 |

`<em>` 的重點是強調語氣。不同位置的 `<em>` 可能會改變句子的重點。

### 4.4 刪除線：`<s>` 與 `<del>`

```html
<s>不再準確的內容</s>
<del>被刪除的內容</del>
```

| 標籤 | 語意 | 適合情境 |
|---|---|---|
| `<s>` | 表示內容已不再正確或不再相關，但不一定是正式刪除 | 原價、過期資訊、不再適用的說法 |
| `<del>` | 表示內容已被刪除 | 文件修訂、文章改稿、版本差異 |

如果要表達文件修訂中的刪除內容，應使用 `<del>`；如果只是表示某個資訊不再有效，例如商品原價，則可以使用 `<s>`。

## 5. 範例說明

### 範例一：原始資料中的四組標籤

```html
<b>加粗</b>
<strong>加粗</strong>

<u>下劃線</u>
<ins>下劃線</ins>

<i>傾斜</i>
<em>傾斜</em>

<s>刪除線</s>
<del>刪除線</del>
```

這段範例展示了四組外觀相近的標籤。它們的畫面效果可能接近，但語意不同：

- `<strong>` 比 `<b>` 更能表達「重要」。
- `<ins>` 比 `<u>` 更能表達「新增內容」。
- `<em>` 比 `<i>` 更能表達「語氣強調」。
- `<del>` 比 `<s>` 更能表達「被刪除的內容」。

因此，若你的目的不是單純做樣式，而是要讓 HTML 內容本身更有意義，通常應優先考慮語意更清楚的元素。

### 範例二：文章中的重點提醒

```html
<p>
  表單送出前，請務必確認
  <strong>必填欄位已完整填寫</strong>。
</p>
```

這裡使用 `<strong>`，因為這段文字不是單純要變粗，而是在內容上具有重要提醒的語意。

### 範例三：文件修訂內容

```html
<p>
  報名截止日期：
  <del>2026-06-01</del>
  <ins>2026-06-08</ins>
</p>
```

這個範例中：

- `<del>` 表示原本的日期已被刪除。
- `<ins>` 表示新的日期是後來插入的內容。

這比只用刪除線與底線更清楚，因為 HTML 本身就描述了修訂關係。

### 範例四：語氣強調

```html
<p>
  這個屬性不是可選的，而是<em>必須</em>提供。
</p>
```

`<em>` 表示語氣上的強調。閱讀時，重點會落在「必須」這個詞上。

## 6. 實務應用場景

文本格式化標籤常出現在以下情境：

- 教學文件中標示重要觀念。
- 商品頁顯示原價與折扣價。
- 文章或文件修訂時標示新增與刪除內容。
- 表單或操作流程中提醒使用者注意重要資訊。
- 技術文件中標示專有名詞、外文詞或特殊術語。

在實務開發中，常見做法是同時使用語意標籤與 CSS：

```html
<p class="notice">
  <strong>注意：</strong>刪除資料後將無法復原。
</p>
```

```css
.notice {
  color: #b42318;
}
```

這裡的 `<strong>` 負責表達重要性，CSS 則負責控制視覺呈現。兩者分工清楚，後續維護也更容易。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 只因為想要粗體就使用 `<strong>` | `<strong>` 表示重要性，不是單純的粗體工具 | 純視覺粗體使用 CSS；重要內容才使用 `<strong>` |
| 把 `<em>` 當成一般斜體工具 | `<em>` 表示語氣強調，不只是斜體 | 若只是術語、外文詞或作品名，可考慮 `<i>` 或 CSS |
| 用 `<u>` 表示新增內容 | `<u>` 不表示內容新增 | 文件修訂或新增內容使用 `<ins>` |
| 用 `<s>` 表示正式刪除 | `<s>` 表示不再準確或不再相關，不一定是修訂刪除 | 文件中被刪除的內容使用 `<del>` |
| 為了外觀大量堆疊格式化標籤 | HTML 結構會變得難讀，也可能造成語意混亂 | 先選正確語意，外觀交給 CSS 控制 |
| 只依賴顏色、底線或刪除線傳達重要資訊 | 使用者或輔助工具可能無法完整理解含義 | 搭配清楚文字與正確語意元素 |

### 錯誤範例：用 `<strong>` 當樣式工具

```html
<p>
  <strong>這段只是設計稿要求變粗，沒有特別重要。</strong>
</p>
```

問題：如果這段文字沒有重要性，只是視覺上要變粗，使用 `<strong>` 會讓語意不準確。

### 正確範例：使用 CSS 控制外觀

```html
<p class="lead-text">這段只是設計稿要求變粗，沒有特別重要。</p>
```

```css
.lead-text {
  font-weight: 700;
}
```

修正後，HTML 保持一般段落語意，粗體效果交給 CSS 控制。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| 語意化 HTML | 文本格式化標籤不只控制外觀，也應根據內容語意選擇 |
| CSS | CSS 負責文字顏色、粗細、底線、斜體等視覺樣式 |
| DOM | 瀏覽器會把這些標籤解析成 DOM 元素，CSS 與 JavaScript 可進一步選取或操作 |
| 可訪問性 | 正確語意有助於輔助工具理解哪些內容是強調、插入或刪除 |
| 文件修訂 | `<ins>` 與 `<del>` 適合表達新增與刪除內容 |
| SEO | 文本語意可以幫助機器理解內容結構，但單一標籤不應被誇大成 SEO 保證 |

## 9. 重點整理

- 文本格式化標籤常用來呈現加粗、底線、傾斜與刪除線。
- 外觀相似的標籤，語意可能不同。
- `<strong>` 表示重要內容；`<b>` 偏向視覺上的醒目。
- `<em>` 表示語氣強調；`<i>` 表示不同語氣、術語或外文詞等。
- `<ins>` 表示新增內容；`<del>` 表示刪除內容。
- `<s>` 表示內容已不再準確或不再相關。
- 單純視覺樣式應交給 CSS，HTML 應優先表達內容語意。

## 11. 延伸補充

HTML 的文本格式化標籤很適合用來學習「語意與樣式分離」這個觀念。

初學者常會先記住：「`<strong>` 會變粗、`<em>` 會變斜、`<del>` 會有刪除線。」這樣可以幫助入門，但正式開發時應再往前一步理解：這些元素真正重要的是內容語意。

> 補充說明：瀏覽器預設樣式可以被 CSS 改變，所以不能只靠預設外觀判斷標籤用途。選擇 HTML 元素時，應優先考慮它是否正確描述內容本身的意思。
