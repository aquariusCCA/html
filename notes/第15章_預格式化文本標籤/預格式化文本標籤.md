# HTML 預格式化文本標籤 pre

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `<pre>` 標籤的用途與基本行為。
- 理解 `<pre>` 如何保留文字中的空格、縮排與換行。
- 正確使用 `<pre>` 顯示需要固定格式的文字內容。
- 分辨 `<pre>` 與一般文字元素在空白處理上的差異。
- 避免在 `<pre>` 中直接放入未轉義的 HTML 程式碼。

## 2. 本章定位

本章介紹 HTML 中用來顯示預格式化文字的 `<pre>` 元素。

一般 HTML 文字會進行空白折疊：多個連續空格通常會被顯示成一個空格，原始碼中的換行也不一定會照原樣呈現在畫面上。`<pre>` 則適合用在「文字本身的排版格式就是內容的一部分」的情境，例如程式碼片段、命令列輸出、純文字表格或需要保留縮排的內容。

原始資料強調，`<pre>` 中的 tab、空格與換行會被保留下來，這正是學習本章的核心。

## 3. 核心概念

`<pre>` 是 preformatted text element，意思是「預格式化文本元素」。它會讓瀏覽器保留元素內容中的空白字元與換行，並通常以等寬字體顯示。

在一般元素中，HTML 會把許多空白視為同一個空白。例如下面這段文字即使在原始碼中分成多行，瀏覽器顯示時也可能被排成接近一般段落的樣子：

```html
<p>
  body {
    background: #fff;
  }
</p>
```

如果這段內容的縮排與換行本身很重要，就應該使用 `<pre>`：

```html
<pre>
body {
  background: #fff;
}
</pre>
```

`<pre>` 的重點不是讓文字「看起來像程式碼」，而是保留文字原本的格式。若內容是程式碼，通常會再搭配 `<code>` 表示程式碼語意。

> 補充說明：瀏覽器預設通常會以等寬字體顯示 `<pre>` 內容，但外觀仍可透過 CSS 調整。HTML 負責表達內容結構與語意，CSS 負責控制視覺樣式。

## 4. 語法與基本用法

基本語法如下：

```html
<pre>
要保留格式的文字
</pre>
```

`<pre>` 是一般容器元素，需要開始標籤與結束標籤。放在 `<pre>` 裡面的文字會保留原始碼中的空白、縮排與換行。

| 特性 | 說明 |
|---|---|
| 保留空格 | 多個連續空格不會像一般文字一樣被折疊成一個 |
| 保留換行 | 原始碼中的換行會依照內容呈現 |
| 保留縮排 | tab 或空格造成的縮排會顯示出來 |
| 常見預設字體 | 瀏覽器通常以等寬字體顯示 |

### 搭配 `<code>` 表示程式碼

如果 `<pre>` 的內容是一段程式碼，建議使用 `<pre><code>...</code></pre>`：

```html
<pre><code>body {
  background: #fff;
  font-size: 16px;
  line-height: 1.6;
}</code></pre>
```

這樣的寫法中：

- `<pre>` 負責保留換行與縮排。
- `<code>` 負責表示內容是一段程式碼。

## 5. 範例說明

### 範例：比較一般內容與 `<pre>` 的差異

```html
<body>
  <p>
    body {
      background: #fff;
      font-size: 16px;
    }
  </p>

  <hr>

  <pre>
body {
  background: #fff;
  font-size: 16px;
}
  </pre>
</body>
```

這個範例中：

- 第一段 CSS 片段放在 `<p>` 裡，瀏覽器會依照一般文字規則處理空白。
- `<hr>` 用來分隔兩段示範內容。
- 第二段 CSS 片段放在 `<pre>` 裡，換行、縮排與空格會被保留。

原始資料想表達的重點是：當你需要直接展示原始碼或固定格式文字時，`<pre>` 可以避免文字格式被瀏覽器自動折疊。

### 範例：顯示 HTML 程式碼

若要在網頁上顯示 HTML 程式碼，必須將 `<` 與 `>` 轉成 HTML entity，否則瀏覽器會把它們當成真正的標籤解析。

```html
<pre><code>&lt;h1&gt;Hello HTML&lt;/h1&gt;
&lt;p&gt;這是一段文字。&lt;/p&gt;</code></pre>
```

畫面上會顯示類似以下內容：

```html
<h1>Hello HTML</h1>
<p>這是一段文字。</p>
```

這是初學者很容易忽略的地方：`<pre>` 會保留文字格式，但不會自動把 HTML 標籤轉成可顯示的純文字。

## 6. 實務應用場景

在實務開發中，`<pre>` 常用於需要保留原始文字格式的內容，例如：

- 教學文章中的程式碼範例。
- API 文件中的請求或回應範例。
- 命令列輸出結果。
- 錯誤訊息、log 或 stack trace。
- 純文字表格、ASCII 圖或固定寬度格式資料。

如果只是想讓一段普通文字換行或增加間距，不應該直接使用 `<pre>`。普通段落應使用 `<p>`，視覺間距應由 CSS 控制。

### 用 CSS 改善 `<pre>` 的閱讀體驗

`<pre>` 內容通常可能很長，實務上常搭配 CSS 控制換行與水平捲動：

```html
<pre class="code-block"><code>body {
  background: #fff;
  font-size: 16px;
}</code></pre>
```

```css
.code-block {
  overflow-x: auto;
  padding: 16px;
  background: #f6f8fa;
}
```

這裡的 HTML 負責保留程式碼格式，CSS 負責讓程式碼區塊更容易閱讀。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 用 `<pre>` 製造一般段落換行 | `<pre>` 會保留所有空白，容易讓內容排版不自然 | 普通文字使用 `<p>`，換行需求視情況使用 `<br>` 或 CSS |
| 直接在 `<pre>` 中放 HTML 標籤 | 瀏覽器仍會解析 HTML 標籤，不一定會原樣顯示 | 將 `<` 寫成 `&lt;`，將 `>` 寫成 `&gt;` |
| 忽略原始碼縮排也會被顯示 | `<pre>` 會保留標籤內的空白，包含程式碼縮排 | 將真正要顯示的內容對齊，或用 CSS/工具處理格式 |
| 只用 `<pre>` 表示程式碼 | `<pre>` 表示預格式化文字，不一定代表程式碼語意 | 程式碼內容建議搭配 `<code>` |
| 用 `<pre>` 控制版面間距 | 版面間距屬於視覺呈現，不是內容語意 | 使用 CSS 的 `margin`、`padding` 或版面配置屬性 |

### 錯誤範例：直接顯示 HTML 程式碼

```html
<pre>
<h1>Hello HTML</h1>
</pre>
```

問題：`<h1>` 可能會被瀏覽器當成真正的 HTML 標籤解析，而不是單純顯示成文字。

### 正確範例：轉義 HTML 字元

```html
<pre><code>&lt;h1&gt;Hello HTML&lt;/h1&gt;</code></pre>
```

修正重點：如果要展示 HTML 原始碼，需要把特殊字元轉成 HTML entity。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| `<code>` | `<code>` 表示程式碼語意；`<pre>` 保留文字格式，兩者常一起使用 |
| `<p>` | `<p>` 表示一般段落；`<pre>` 表示需要保留原始格式的文字 |
| `<br>` | `<br>` 用於插入單一換行；`<pre>` 用於整段保留換行與空白 |
| CSS `white-space` | `<pre>` 的顯示效果與 CSS 的空白處理有關，可用 CSS 調整細節 |
| DOM | 瀏覽器會把 `<pre>` 解析成 DOM 元素節點，元素內的文字節點會保留空白內容 |
| 可訪問性 | 搭配 `<code>` 或清楚的前後文，有助於使用者理解這段預格式化文字的用途 |

## 9. 重點整理

- `<pre>` 用來表示預格式化文字。
- `<pre>` 會保留內容中的空格、縮排與換行。
- 瀏覽器通常會用等寬字體顯示 `<pre>` 內容。
- 顯示程式碼時，常見寫法是 `<pre><code>...</code></pre>`。
- 若要顯示 HTML 原始碼，需要轉義 `<`、`>` 等特殊字元。
- 不應使用 `<pre>` 取代一般段落、換行或版面間距控制。

## 11. 延伸補充

`<pre>` 很適合保存「文字原本長什麼樣子」這件事。它常出現在技術文件與教學文章中，因為程式碼、命令列輸出與錯誤訊息通常都依賴換行與縮排來表達結構。

> 補充說明：`<pre>` 解決的是 HTML 空白折疊造成的格式遺失問題；它不負責程式碼語法高亮，也不代表所有內容都一定是程式碼。若需要語法高亮，通常還需要額外的 CSS 或 JavaScript 工具配合。
