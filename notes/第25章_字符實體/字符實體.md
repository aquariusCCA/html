# HTML 字符實體

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 HTML 字符實體的用途。
- 理解為什麼某些符號不能直接寫在 HTML 原始碼中。
- 使用常見字符實體顯示空格、小於號、大於號、`&` 與版權符號。
- 分辨一般空白與 `&nbsp;` 的差異。
- 避免用大量字符實體處理版面排版問題。

## 2. 本章定位

本章介紹 HTML 中的字符實體。

HTML 是標記語言，某些符號在 HTML 原始碼中具有特殊意義。例如小於號 `<` 會被瀏覽器視為標籤開始，大於號 `>` 常用來結束標籤。如果想在頁面中顯示這些符號本身，就需要使用字符實體來表示。

> 來源重點：原始資料強調，初學階段應優先記住空格、小於號與大於號等常用字符實體；其他較少使用的符號，需要時再查表即可。

## 3. 核心概念

HTML 字符實體是用一段特殊文字表示某個字符的寫法。瀏覽器解析 HTML 時，會把字符實體轉換成對應的實際符號並顯示在頁面上。

例如，如果直接在 HTML 內容中寫 `<p>`，瀏覽器會把它理解成段落標籤；如果想讓頁面顯示字面上的 `<p>`，就要把小於號與大於號改寫成字符實體：

```html
&lt;p&gt;
```

瀏覽器畫面會顯示：

```text
<p>
```

### 3.1 字符實體的組成

原始資料提到，字符實體通常由三個部分組成：

| 部分 | 說明 | 範例 |
|---|---|---|
| `&` | 實體的開始符號 | `&copy;` 中的 `&` |
| 實體名稱 | 表示要輸出的字符 | `copy` |
| `;` | 實體結束符號 | `&copy;` 中的 `;` |

完整寫法如下：

```html
&copy;
```

瀏覽器會將它顯示為版權符號：

```text
©
```

> 注意：實務上建議字符實體都寫上結尾分號 `;`，可以避免瀏覽器解析時產生不必要的歧義。

### 3.2 為什麼需要字符實體

字符實體主要用來處理兩類情況：

| 情況 | 說明 |
|---|---|
| 字符本身具有 HTML 語法意義 | 例如 `<`、`>`、`&` |
| 字符不方便直接輸入或容易被一般空白規則影響 | 例如不換行空白 `&nbsp;`、版權符號 `&copy;` |

如果沒有字符實體，某些內容可能會被瀏覽器誤認為 HTML 語法，而不是普通文字。

## 4. 語法與基本用法

### 4.1 常見字符實體

初學 HTML 時，不需要背完整字符實體表。較常見、也最值得優先記住的有以下幾個：

| 顯示結果 | 字符實體 | 用途 |
|---|---|---|
| 空格 | `&nbsp;` | 顯示不換行空白 |
| `<` | `&lt;` | 顯示小於號，避免被解析成標籤開始 |
| `>` | `&gt;` | 顯示大於號 |
| `&` | `&amp;` | 顯示 ampersand，避免被解析成字符實體開頭 |
| `"` | `&quot;` | 顯示雙引號，常見於屬性值相關文字說明 |
| `'` | `&apos;` | 顯示單引號 |
| `©` | `&copy;` | 顯示版權符號 |

> 補充說明：HTML 支援大量字符實體，例如貨幣符號、數學符號、箭頭與特殊標點。正式開發時通常只需要記住常用項目，其他符號需要時查表即可。

### 4.2 顯示小於號與大於號

如果要在網頁中展示 HTML 程式碼片段，應把 `<` 與 `>` 寫成字符實體。

```html
<p>段落標籤寫法：&lt;p&gt;文字內容&lt;/p&gt;</p>
```

畫面會顯示：

```text
段落標籤寫法：<p>文字內容</p>
```

如果沒有轉義小於號，瀏覽器可能會把內容當成真正的 HTML 標籤解析。

### 4.3 顯示多個空格

HTML 中連續多個一般空格、換行或縮排，通常會被瀏覽器折疊成一個空格。

```html
hello     world<br>
```

雖然原始碼中 `hello` 和 `world` 之間有多個空格，瀏覽器畫面通常只會顯示成一個空格：

```text
hello world
```

如果想直接顯示多個空白，可以使用 `&nbsp;`：

```html
hello&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;world<br>
```

這樣畫面上會看到較大的空白距離。

> 注意：`&nbsp;` 是 non-breaking space（不換行空白），不完全等於一般空格。它會讓相鄰內容比較不容易在該位置自動換行，因此不應大量用來控制版面。

### 4.4 顯示版權符號

版權符號可以使用 `&copy;` 表示。

```html
<footer>
  &copy; 2026 HTML Knowledge Pack
</footer>
```

畫面會顯示：

```text
© 2026 HTML Knowledge Pack
```

## 5. 範例說明

### 範例一：展示 HTML 標籤文字

```html
<p>段落元素的寫法是：&lt;p&gt;內容&lt;/p&gt;</p>
```

這個範例中：

- `&lt;` 會顯示為 `<`。
- `&gt;` 會顯示為 `>`。
- 瀏覽器不會把 `&lt;p&gt;` 當成真正的段落標籤，而是把它顯示成文字。

### 範例二：顯示連續空白

```html
<p>hello&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;world</p>
```

這個範例使用多個 `&nbsp;` 製造可見空白。

> 注意：這種寫法適合示範字符實體，但不適合當成主要排版手段。如果只是想控制兩個文字或區塊之間的距離，應優先使用 CSS。

### 範例三：同時顯示特殊符號

```html
<p>HTML 中常見的特殊符號：&lt;、&gt;、&amp;</p>
<p>&copy; 2026</p>
```

畫面會顯示小於號、大於號、`&` 與版權符號。

## 6. 實務應用場景

字符實體常用在以下情境：

- 在教學文章或文件中顯示 HTML 標籤文字，例如 `&lt;div&gt;`。
- 在頁面中安全顯示小於號、大於號與 `&`。
- 顯示版權符號、商標符號、箭頭或其他不方便輸入的特殊符號。
- 在少量文字內容中保留不換行空白。
- 處理使用者輸入內容時，避免文字被錯誤當成 HTML 解析。

> 補充說明：若內容來自使用者輸入，實務上通常不會手動一個個替換字符，而是由後端模板引擎、前端框架或安全處理工具進行 escaping。這可以降低 XSS 等安全風險。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 直接寫 `<p>` 想顯示標籤文字 | 瀏覽器會把 `<p>` 當成 HTML 標籤解析 | 寫成 `&lt;p&gt;` |
| 忘記寫結尾分號 | 可能造成字符實體解析不穩定或與後方文字混淆 | 實體一律寫完整，例如 `&copy;` |
| 用大量 `&nbsp;` 排版 | `&nbsp;` 是不換行空白，不是版面工具 | 使用 CSS 的 `margin`、`padding`、`gap` 或排版模型 |
| 以為多個一般空格都會原樣顯示 | HTML 會折疊連續空白與換行 | 使用語意結構、CSS，或在必要時使用 `&nbsp;`、`<pre>` |
| 忘記把 `&` 寫成 `&amp;` | `&` 是字符實體的開始符號 | 顯示字面上的 `&` 時使用 `&amp;` |

### 錯誤範例：直接顯示 HTML 標籤

```html
<p>請使用 <strong> 標籤表示重要內容。</p>
```

問題：`<strong>` 可能會被瀏覽器當成真正的 HTML 標籤，而不是顯示成文字說明。

### 正確範例：使用字符實體顯示標籤

```html
<p>請使用 &lt;strong&gt; 標籤表示重要內容。</p>
```

修正重點：用 `&lt;` 與 `&gt;` 顯示小於號與大於號，讓標籤名稱成為頁面文字。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| HTML 解析 | 瀏覽器解析 HTML 時，會把字符實體轉成對應字符 |
| DOM | 字符實體解析後會成為 DOM 文字節點中的實際字符 |
| CSS | 視覺間距與排版應交給 CSS，不應依賴大量 `&nbsp;` |
| JavaScript | 使用 JavaScript 插入文字內容時，應理解文字插入與 HTML 插入的差異 |
| 安全性 | 顯示不可信內容時，需要正確 escaping，避免文字被當成 HTML 或腳本執行 |
| `<pre>` | 若要保留原始文字中的空白與換行，可視情境使用 `<pre>` 或 CSS 的 `white-space` |

## 9. 重點整理

- HTML 字符實體用來在 HTML 中表示特殊字符。
- 字符實體常見結構是 `&` 加上實體名稱，再加上分號 `;`。
- 小於號 `<` 應寫成 `&lt;`，大於號 `>` 應寫成 `&gt;`。
- 顯示字面上的 `&` 時，應寫成 `&amp;`。
- 多個一般空格與換行通常會被瀏覽器折疊成一個空格。
- `&nbsp;` 可以顯示不換行空白，但不應大量用來做版面排版。
- 常見字符實體記住即可，較少用的符號需要時再查表。

## 11. 延伸補充

### 11.1 命名實體與數字實體

> 補充說明：原始資料主要介紹命名實體，例如 `&copy;`。HTML 也可以使用數字實體表示字符。

同一個字符可能有不同寫法。例如小於號可以寫成：

```html
&lt;
&#60;
&#x3C;
```

其中：

| 寫法 | 說明 |
|---|---|
| `&lt;` | 命名實體，較容易閱讀 |
| `&#60;` | 十進位數字實體 |
| `&#x3C;` | 十六進位數字實體 |

初學與一般開發中，常見字符優先使用命名實體即可。

### 11.2 空白折疊與排版責任

HTML 中的連續空格、換行與縮排通常會被視為空白字元，並在一般文字排版中折疊顯示。這也是為什麼原始碼可以透過換行與縮排保持可讀性，而不會讓頁面出現大量空白。

如果需求是「內容本身需要保留空白」，可以考慮 `&nbsp;`、`<pre>` 或 CSS 的 `white-space`。如果需求是「畫面上需要間距」，應使用 CSS 控制。
