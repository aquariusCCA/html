# HTML 簡介

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明網頁、網站與 HTML 之間的關係。
- 理解 HTML 為什麼是標記語言，而不是程式語言。
- 說明瀏覽器如何將 HTML 文件顯示成網頁。
- 認識 HTML5、HTML4 與 XHTML 文件宣告的差異。
- 寫出一份基本 HTML 文件骨架。
- 分辨 HTML 標籤、元素、雙標籤、單標籤與元素之間的關係。
- 理解良好 HTML 結構對 CSS、JavaScript、SEO 與可訪問性的影響。

## 2. 本章定位

本章是 HTML 學習的入口，重點不是背很多標籤，而是先建立 HTML 的基本觀念：HTML 用來描述網頁的內容與結構，瀏覽器會解析 HTML 並將它呈現成使用者看到的頁面。

在學習 CSS、JavaScript 或 Vue 之前，必須先理解 HTML 文件的骨架、標籤的組成方式，以及元素之間的父子、兄弟與祖先後代關係。這些結構會直接影響樣式套用、DOM 操作、搜尋引擎理解頁面，以及輔助工具讀取內容。

## 3. 核心概念

### 3.1 網頁與網站

網頁是網站中的一個頁面，通常是一份 HTML 文件，常見副檔名是 `.html` 或 `.htm`。使用者透過瀏覽器開啟 HTML 文件後，瀏覽器會將文件內容解析並顯示成可以閱讀與操作的頁面。

網站則可以理解成多個相關網頁與資源組成的集合。例如一個教學網站可能包含首頁、課程列表頁、文章頁、關於我們頁等多個網頁。

一個網頁通常可以包含：

- 文字
- 圖片
- 音訊
- 影片
- 超連結
- 表單
- 其他可嵌入或互動的內容

### 3.2 HTML 是什麼

HTML 是 HyperText Markup Language，中文常翻成「超文本標記語言」。它是用來描述網頁內容與結構的語言。

「超文本」有兩層重要意義：

- 網頁不只可以放文字，也可以放圖片、聲音、影片、動畫等多媒體內容。
- 網頁可以透過超連結從一份文件跳到另一份文件，形成互相連接的資訊網路。

HTML 不是程式語言，因為它不主要負責流程控制、條件判斷或複雜運算。HTML 的核心任務是使用標籤標記內容，告訴瀏覽器「這段內容是標題」、「這段內容是段落」、「這裡是一張圖片」、「這裡是一個連結」。

### 3.3 HTML、瀏覽器與網頁形成

網頁的形成可以簡化成以下流程：

```text
前端人員撰寫 HTML / CSS / JavaScript
→ 瀏覽器讀取檔案
→ 瀏覽器解析 HTML 並建立 DOM
→ 套用 CSS 樣式與執行 JavaScript
→ 使用者看到最後的 Web 頁面
```

原始 HTML 文件本身只是文字檔。瀏覽器的工作是讀取這些文字，解析其中的標籤與內容，並將它們渲染成畫面。

> 補充說明：HTML 負責描述內容與結構，DOM 是瀏覽器根據 HTML 建立出的物件模型。CSS 與 JavaScript 多數情況下是基於 DOM 進行樣式套用與互動操作。

### 3.4 HTML5 的定位

HTML5 是新一代 HTML 標準。原始資料提到，W3C 於 2014 年 10 月完成 HTML5 標準制定。實務上，現代網頁開發通常都以 HTML5 的文件宣告與語法為基礎。

HTML5 狹義上指 HTML 標準本身；廣義上，有些教材會把 HTML5、CSS3、JavaScript 以及相關 Web API 合稱為「HTML5 技術集」。學習時要分清楚：HTML 本身負責文件結構，CSS 負責樣式，JavaScript 負責互動邏輯。

HTML5 相較於早期版本，常見重點包含：

- 更簡潔的文件宣告。
- 新增更多語意化元素。
- 新增或改善表單相關能力。
- 支援更多多媒體元素，例如音訊與影片。
- 提供更多可被 JavaScript 操作的 Web API。
- 更重視文件語意，對 SEO 與可訪問性有幫助。

> 注意：原始資料提到 HTML5 新特性在舊版 IE 中支援有限。現代開發多數情況可直接使用 HTML5，但若專案需要支援非常舊的瀏覽器，仍應針對實際目標環境確認相容性。

## 4. 語法與基本用法

### 4.1 基本 HTML5 文件骨架

一份基本 HTML5 文件通常會包含以下骨架：

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>我的第一個網頁</title>
</head>
<body>
  <h1>Hello HTML</h1>
  <p>這是一段網頁內容。</p>
</body>
</html>
```

各部分說明如下：

| 結構 | 說明 |
|---|---|
| `<!doctype html>` | 宣告這份文件使用 HTML5 解析方式。 |
| `<html>` | 整份 HTML 文件的根元素。 |
| `lang="zh-Hant"` | 宣告主要內容語言為繁體中文，有助於瀏覽器、搜尋引擎與輔助工具理解頁面。 |
| `<head>` | 放置文件資訊，例如編碼、標題、外部資源連結等，通常不直接顯示在頁面主體。 |
| `<meta charset="UTF-8">` | 設定文件字元編碼，避免中文或特殊字元顯示錯誤。 |
| `<title>` | 設定瀏覽器分頁標題，也常被搜尋結果參考。 |
| `<body>` | 放置使用者在頁面上主要看到的內容。 |

### 4.2 HTML4 與 XHTML 的文件宣告

早期 HTML4 的文件宣告較長，會指定 DTD：

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<html>
<head>
  <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
  <title>HTML4 文件</title>
</head>
<body>
  <p>這是 HTML4 文件範例。</p>
</body>
</html>
```

XHTML 則更接近 XML 的寫法，文件宣告與命名空間也更嚴格：

```html
<!DOCTYPE html
PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="zh">
<head>
  <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
  <title>XHTML 文件</title>
</head>
<body>
  <p>這是 XHTML 文件範例。</p>
</body>
</html>
```

現代初學與一般前端開發，通常優先使用 HTML5 的簡潔宣告：

```html
<!doctype html>
```

> 補充說明：HTML5 大多可以使用早期 HTML 中仍有效的元素與功能，但不代表所有舊寫法都值得繼續使用。若某些元素只是為了視覺效果或已不符合語意化實務，應改用更合適的 HTML 結構與 CSS。

### 4.3 HTML 標籤與元素

HTML 使用標籤來標記內容。標籤通常由 `<`、`>`、`/` 與標籤名稱組成。

一個常見 HTML 元素通常包含三部分：

```html
<p>這是一段文字。</p>
```

| 部分 | 說明 |
|---|---|
| `<p>` | 開始標籤，也稱起始標籤。 |
| `這是一段文字。` | 元素內容。 |
| `</p>` | 結束標籤。 |

嚴格來說，「標籤」是 `<p>` 或 `</p>` 這種標記本身；「元素」則包含開始標籤、內容與結束標籤形成的完整結構。

### 4.4 雙標籤與單標籤

大多數 HTML 元素是雙標籤，也就是有開始標籤與結束標籤：

```html
<h1>網頁標題</h1>
<p>這是一段段落文字。</p>
```

有些元素不需要包住文字內容，通常只用一個標籤表示。這類元素在許多教材中會稱為單標籤，HTML 標準中常稱為 void elements。

常見例子：

```html
<meta charset="UTF-8">
<img src="photo.jpg" alt="教室中的 HTML 課程投影片">
<br>
<input type="text" name="username">
```

> 注意：單標籤不是「可以隨便省略結束標籤」。只有特定元素屬於這類結構。一般雙標籤元素仍應寫出結束標籤，避免瀏覽器解析結果不如預期。

### 4.5 HTML 元素之間的關係

HTML 元素可以互相嵌套，也可以並列出現。這些關係會形成類似樹狀結構的文件階層。

```html
<body>
  <main>
    <h1>HTML 簡介</h1>
    <section>
      <h2>什麼是 HTML</h2>
      <p>HTML 是用來描述網頁內容與結構的標記語言。</p>
    </section>
  </main>
</body>
```

以這段程式碼為例：

| 關係 | 說明 |
|---|---|
| 父元素 | `main` 是 `h1` 與 `section` 的父元素。 |
| 子元素 | `h1` 與 `section` 是 `main` 的子元素。 |
| 祖先元素 | `body` 是 `p` 的祖先元素。 |
| 後代元素 | `h2` 與 `p` 是 `body` 的後代元素。 |
| 兄弟元素 | `h1` 與 `section` 同屬於 `main` 之下，是兄弟元素。 |
| 嵌套關係 | `section` 放在 `main` 裡面，`p` 又放在 `section` 裡面。 |
| 並列關係 | `h2` 與 `p` 在同一層，彼此並列。 |

理解這些關係很重要，因為 CSS 選擇器、JavaScript DOM 操作與網頁結構維護都會用到這些概念。

## 5. 範例說明

### 範例：一個簡單的文章頁面

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>HTML 入門文章</title>
</head>
<body>
  <header>
    <h1>HTML 入門文章</h1>
  </header>

  <main>
    <article>
      <h2>什麼是 HTML？</h2>
      <p>HTML 是用來描述網頁內容與結構的標記語言。</p>

      <h2>HTML 可以放哪些內容？</h2>
      <p>HTML 可以標記文字、圖片、連結、表單與多媒體內容。</p>

      <a href="https://developer.mozilla.org/">閱讀更多 HTML 文件</a>
    </article>
  </main>
</body>
</html>
```

這個範例展示了幾個基本觀念：

- `<!doctype html>` 使用 HTML5 文件宣告。
- `html` 是根元素，`head` 與 `body` 是它的子元素。
- `head` 放置文件資訊，`body` 放置頁面內容。
- `h1` 表示頁面最高層級標題，不是單純用來放大字體。
- `p` 表示段落。
- `a` 表示超連結，可以連到其他文件或網站。
- `header`、`main`、`article` 是語意化元素，可以讓頁面結構更清楚。

> 補充說明：即使目前只是學 HTML 簡介，也應開始習慣根據內容意義選擇元素。HTML 的重點不是「畫面看起來像什麼」，而是「這段內容在文件中代表什麼」。

## 6. 實務應用場景

### 6.1 建立所有網頁的基礎結構

不論是靜態網站、後端模板、Vue 元件或其他前端框架，最後都會產生瀏覽器可以解析的 HTML 與 DOM 結構。基本 HTML 骨架是所有網頁的起點。

### 6.2 讓 CSS 有清楚的套用目標

CSS 需要根據 HTML 元素、類別、屬性或階層關係選取目標。如果 HTML 結構混亂，CSS 很容易變得難維護。

### 6.3 讓 JavaScript 能操作正確節點

JavaScript 通常不是直接操作原始 HTML 字串，而是透過 DOM API 找到元素並改變內容、屬性或事件行為。理解父子、兄弟、祖先與後代關係，可以幫助你更準確地操作 DOM。

### 6.4 改善 SEO 與可訪問性

語意清楚的 HTML 可以幫助搜尋引擎理解頁面內容，也能讓螢幕閱讀器等輔助工具更正確地讀取頁面。像是合理使用 `title`、標題層級、連結文字與圖片 `alt`，都是 HTML 基礎的一部分。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 把 HTML 當成程式語言 | HTML 不負責主要邏輯控制，而是標記內容與結構 | 將 HTML 理解為標記語言，互動邏輯交給 JavaScript |
| 把網站與網頁混為一談 | 網站通常由多個網頁與資源組成，網頁是其中一個頁面 | 用「網站是集合，網頁是單頁」來區分 |
| 省略基本文件骨架 | 缺少 `doctype`、`html`、`head`、`body` 會讓文件結構不完整 | 使用 HTML5 基本骨架建立新頁面 |
| 沒有設定字元編碼 | 中文或特殊字元可能顯示錯誤 | 在 `head` 中加入 `<meta charset="UTF-8">` |
| 用標題元素控制字體大小 | `h1` 到 `h6` 表示文件標題層級，不是樣式工具 | 用正確標題層級表達結構，字體大小交給 CSS |
| 所有內容都用 `div` 包起來 | `div` 沒有語意，過度使用會降低可讀性與可訪問性 | 有適合語意元素時優先使用，例如 `main`、`article`、`nav` |
| 單標籤概念誤用 | 不是所有元素都可以省略結束標籤 | 只有特定 void elements 使用單標籤形式 |
| 巢狀結構混亂 | 父子與兄弟關係錯誤會影響 DOM、CSS 與 JavaScript | 保持縮排清楚，確認元素開始與結束位置一致 |

### 錯誤範例：把標題當成樣式工具

```html
<h1>文章標題</h1>
<h4>這其實是下一段主要小節標題</h4>
```

問題：如果中間沒有合理的 `h2`、`h3` 層級，標題結構會讓文件大綱不清楚。

### 正確範例：用標題表示文件層級

```html
<h1>文章標題</h1>
<h2>主要小節標題</h2>
```

修正重點：HTML 標題應先考慮文件結構，視覺大小再交給 CSS 控制。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| CSS | CSS 透過選擇器選取 HTML 或 DOM 中的元素並套用樣式。 |
| JavaScript | JavaScript 常透過 DOM API 操作 HTML 解析後的節點。 |
| DOM | DOM 是瀏覽器根據 HTML 建立出的樹狀物件模型。 |
| Vue | Vue template 最後仍會渲染成 DOM，因此 HTML 語意仍然重要。 |
| SEO | `title`、標題層級、語意化結構與圖片 `alt` 都會影響搜尋引擎理解頁面。 |
| 可訪問性 | 正確的 HTML 語意有助於螢幕閱讀器與鍵盤操作等輔助情境。 |
| 表單 | HTML 表單透過元素與 `name` / `value` 組成使用者輸入資料，是後續學習重點。 |

## 9. 重點整理

- 網頁通常是一份 HTML 文件，網站則是多個相關網頁與資源的集合。
- HTML 是超文本標記語言，用標籤描述網頁內容與結構。
- HTML 不是程式語言；互動邏輯通常由 JavaScript 處理。
- 瀏覽器會解析 HTML，建立 DOM，並將內容渲染成使用者看到的網頁。
- 現代 HTML 開發通常使用 HTML5 的簡潔文件宣告。
- 基本 HTML 文件包含 `doctype`、`html`、`head` 與 `body`。
- HTML 元素可以形成父子、兄弟、祖先與後代關係。
- 良好的 HTML 結構會影響 CSS、JavaScript、SEO、可訪問性與後續維護。

## 11. 延伸補充

### 11.1 HTML 文件與 HTML 元素不是同一件事

HTML 文件是整份 `.html` 檔案；HTML 元素則是文件中的一個結構單位。例如 `<p>這是一段文字。</p>` 是一個段落元素，而整份包含 `doctype`、`html`、`head`、`body` 的檔案才是 HTML 文件。

### 11.2 HTML5 新特性不等於全部都要立刻深入

HTML5 涵蓋的範圍很廣，包含語意化元素、表單能力、多媒體元素與許多 Web API。初學階段應先掌握文件結構、常用元素與語意化觀念，再逐步學習表單、多媒體、嵌入內容與更進階的 API。
