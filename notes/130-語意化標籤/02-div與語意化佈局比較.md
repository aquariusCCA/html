# div 與語意化佈局比較

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨 `<div>` 與語意化標籤在 HTML 結構上的差異。
- 看懂同一個版面用 `<div>` 與語意化標籤撰寫時，語意上有什麼不同。
- 判斷什麼時候該使用 `<header>`、`<nav>`、`<section>`，什麼時候仍該使用 `<div>`。
- 避免只把 class 名稱換成同名標籤的表面語意化。

## 問題情境

以前做網頁佈局時，很常使用 `<div>` 包出所有區塊：

```html
<div class="header">頭部標籤</div>
<div class="nav">導航欄標籤</div>
<div class="section">...</div>
```

這樣可以排出畫面，也方便加 CSS。但對搜尋引擎、輔助技術或剛接手專案的人來說，`<div>` 本身沒有語意；真正的角色藏在 class 名稱裡。

HTML5 提供語意化標籤後，我們可以在適合的地方直接用標籤表達內容角色。

## 一句話理解

`<div>` 是沒有語意的通用容器；語意化標籤是有內容角色的結構標籤。

## 使用 div 佈局

下面這段 CSS 先設定頁首與導覽列的外觀，再設定一個內容區塊：

```css
.header, .nav {
  height: 120px;
  background-color: pink;
  border-radius: 15px;
  width: 800px;
  margin: 15px auto;
}

.section {
  width: 500px;
  height: 300px;
  background-color: skyblue;
}
```

HTML 寫法如下：

```html
<body>
  <div class="header">頭部標籤</div>
  <div class="nav">導航欄標籤</div>
  <div class="section">
    <h2>最新消息</h2>
    <p>這是一段與最新消息相關的內容。</p>
  </div>
</body>
```

這個版本能正常顯示畫面，但它的語意主要依賴 class 名稱：

- `<div class="header">` 看起來像頁首，但標籤本身仍只是 `<div>`。
- `<div class="nav">` 看起來像導覽列，但標籤本身沒有導覽語意。
- `<div class="section">` 看起來像內容區塊，但標籤本身沒有說明這是一個有主題的區段。

也就是說，這份 HTML 的視覺結構清楚，但語意結構不足。

## 換成語意化標籤佈局

如果這些區塊本來就分別代表頁首、導覽列與主題區塊，可以改用語意化標籤：

```css
header, nav {
  height: 120px;
  background-color: pink;
  border-radius: 15px;
  width: 800px;
  margin: 15px auto;
}

section {
  width: 500px;
  height: 300px;
  background-color: skyblue;
}
```

HTML 寫法如下：

```html
<body>
  <header>頭部標籤</header>
  <nav>導航欄標籤</nav>
  <section>
    <h2>最新消息</h2>
    <p>這是一段與最新消息相關的內容。</p>
  </section>
</body>
```

這個版本的畫面可能與 `<div>` 版本相同，但 HTML 結構更清楚：

- `<header>` 明確表示這是頁面或區塊的開頭內容。
- `<nav>` 明確表示這是導覽區。
- `<section>` 表示這是一段有共同主題的內容，這裡的主題是「最新消息」。

語意化的差異不一定會直接反映在畫面上，但會反映在程式碼可讀性、輔助技術解析與搜尋引擎理解上。

## 比較表

| 面向 | `<div>` 佈局 | 語意化標籤佈局 |
| --- | --- | --- |
| 標籤本身是否有語意 | 沒有，主要靠 class 命名補充 | 有，標籤名稱直接表達內容角色 |
| 適合用途 | 排版容器、樣式包裝、沒有明確語意的分組 | 頁首、導覽、主要內容、文章、主題區塊、頁尾等 |
| 對維護者的幫助 | 需要看 class 與內容推測用途 | 看標籤就能先理解區塊角色 |
| 對輔助技術的幫助 | 較少 | 較容易提供導覽與內容結構 |
| 常見風險 | 整頁變成沒有語意的容器堆疊 | 誤把所有容器都換成語意標籤 |

## 實務判斷準則

選擇標籤時，不要只看畫面，也不要只看 class 名稱。可以依序問三個問題：

1. 這一塊內容是否有明確角色？
2. 這個角色是否有對應的語意化標籤？
3. 如果拿掉 CSS，這個 HTML 結構是否仍能讓人理解內容順序與區塊關係？

如果答案是肯定的，就優先考慮語意化標籤。

例如：

| 情境 | 建議標籤 | 原因 |
| --- | --- | --- |
| 網站主要導覽 | `<nav>` | 內容是導覽連結 |
| 頁面或文章開頭 | `<header>` | 內容是開頭資訊 |
| 一段有標題的主題內容 | `<section>` | 內容能形成一個主題區塊 |
| 單篇可獨立閱讀的文章 | `<article>` | 內容可獨立分發或重用 |
| 純粹為了套樣式的外層容器 | `<div>` | 沒有特別內容語意 |
| 包住一組按鈕方便排版 | `<div>` | 只是分組與排版需求 |

## 常見錯誤

### 錯誤一：只把 class 名稱換成同名標籤

不好的判斷方式是：

```html
<div class="nav">...</div>
```

看到 class 叫 `nav`，就直接改成：

```html
<nav>...</nav>
```

如果裡面真的放的是主要導覽連結，這樣改合理；但如果 class 只是樣式名稱，裡面其實不是導覽，就不該改成 `<nav>`。

語意化不是「名稱看起來像就換」，而是根據內容實際用途選標籤。

### 錯誤二：把所有 `<div>` 都改成 `<section>`

`<section>` 代表有共同主題的一段內容，通常應該能搭配標題。

不適合的寫法：

```html
<section class="button-group">
  <button>儲存</button>
  <button>取消</button>
</section>
```

這裡只是按鈕群組，沒有形成一個獨立主題，用 `<div>` 更合理：

```html
<div class="button-group">
  <button>儲存</button>
  <button>取消</button>
</div>
```

### 錯誤三：為了排版而誤用語意標籤

如果一個元素只是為了讓 CSS 好選取、讓內容置中、包住內層元素或製作版面外框，通常不需要語意化標籤。

這種情況使用 `<div>` 並不是退步，而是正確使用通用容器。

## 重點整理

- `<div>` 沒有內容語意，適合用於排版、分組與樣式容器。
- 語意化標籤能直接表達內容角色，例如 `<header>`、`<nav>`、`<section>`。
- 同一個畫面用 `<div>` 與語意化標籤都可能顯示正常，但 HTML 結構的可理解程度不同。
- 語意化不是把 class 名稱換成標籤名稱，而是根據內容含義選擇合適元素。
- 沒有明確語意需求時，繼續使用 `<div>` 是合理的。

## 自我檢查

1. `<div class="nav">` 一定應該改成 `<nav>` 嗎？你會檢查什麼？
2. 一個只用來包住兩個按鈕、方便 CSS 排版的容器，適合用 `<section>` 嗎？為什麼？
3. `<section>` 通常應該具備什麼特徵？
4. 如果語意化版本和 `<div>` 版本畫面完全一樣，語意化仍然有價值嗎？價值在哪裡？
