# HTML 列表標籤 ul、ol、dl

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 HTML 列表標籤的用途與常見使用情境。
- 分辨無序列表 `<ul>`、有序列表 `<ol>`、描述列表 `<dl>` 的語意差異。
- 正確使用 `<li>` 建立列表項目。
- 理解列表項目中可以放入其他 HTML 元素。
- 寫出基本的列表巢狀結構。
- 避免把列表標籤當成單純排版工具或寫出不合法的巢狀結構。

## 2. 本章定位

本章介紹 HTML 中用來表示一組相關項目的列表標籤。

列表常用於新聞列表、排行榜、功能選單、導覽項目、商品特色、帳單明細、術語解釋等場景。它的重點不是「預設畫面長得像有項目符號或數字」，而是用 HTML 表達內容之間的關係。

在實務開發中，列表常會搭配 CSS 移除預設樣式、調整排列方式，甚至做成橫向導覽列或卡片列表。但即使畫面被 CSS 改得不像傳統列表，只要內容本質是一組相關項目，仍然適合使用列表標籤。

> 來源重點：原始資料指出列表適合用來按照行展示關聯性內容，常見種類包含無序列表、有序列表與自定義列表，並說明列表項目可以容納其他元素。

## 3. 核心概念

HTML 列表用來描述「一組相關內容」。不同列表標籤代表不同語意：

| 標籤 | 名稱 | 語意 | 常見用途 |
|---|---|---|---|
| `<ul>` | unordered list，無序列表 | 一組沒有先後順序的項目 | 功能清單、選單、特色列表、新聞列表 |
| `<ol>` | ordered list，有序列表 | 一組有順序或排名意義的項目 | 步驟、排行榜、流程、排名 |
| `<dl>` | description list，描述列表 | 一組名詞與說明的配對 | 術語解釋、規格表、聯絡資訊 |
| `<li>` | list item，列表項目 | `<ul>` 或 `<ol>` 裡的每一個項目 | 單一列表內容 |
| `<dt>` | description term，描述項目名稱 | `<dl>` 中被解釋的詞或項目 | 名稱、術語、欄位名 |
| `<dd>` | description details，描述內容 | `<dl>` 中對 `<dt>` 的說明 | 解釋、內容、細節 |

### 3.1 列表是語意，不只是樣式

瀏覽器會替列表提供預設樣式，例如：

- `<ul>` 預設常見為項目符號。
- `<ol>` 預設常見為數字編號。
- `<dl>` 預設常見為術語與縮排說明。

這些只是瀏覽器的預設呈現方式。真正重要的是 HTML 結構表達出的語意：這些項目是一組清單、是一組有順序的項目，或是一組名詞與說明。

> 補充說明：HTML 負責描述內容結構，CSS 負責控制畫面樣式。因此不要為了移除項目符號就不用 `<ul>`，也不要為了讓文字有縮排就濫用 `<dl>`。

### 3.2 `<li>` 是列表項目的容器

在 `<ul>` 與 `<ol>` 中，每個項目都應使用 `<li>` 表示。

`<li>` 不只能放純文字，也可以包含段落、圖片、連結、按鈕，甚至另一個列表。這讓列表可以表達簡單項目，也可以表達較複雜的內容結構。

## 4. 語法與基本用法

### 4.1 無序列表 `<ul>`

`<ul>` 用來表示沒有順序差異的一組項目。

```html
<h4>您喜歡的食物？</h4>
<ul>
  <li>榴槤</li>
  <li>臭豆腐</li>
  <li>鯡魚罐頭</li>
</ul>
```

這個列表中的食物彼此是並列關係，沒有第一名、第二名或步驟順序，因此適合使用 `<ul>`。

使用 `<ul>` 時要注意：

- `<ul>` 的直接子元素應該是 `<li>`。
- 不應直接把文字、`<p>`、`<a>` 等元素放在 `<ul>` 的第一層。
- `<li>` 裡面可以再放其他元素。

### 4.2 有序列表 `<ol>`

`<ol>` 用來表示順序有意義的一組項目。

```html
<h4>粉絲排行榜</h4>
<ol>
  <li>劉德華 10000</li>
  <li>劉若英 1000</li>
  <li>pink 老師 1</li>
</ol>
```

這個範例有排名意義，項目順序會影響內容解讀，因此適合使用 `<ol>`。

`<ol>` 也常用於步驟：

```html
<ol>
  <li>打開編輯器。</li>
  <li>建立 HTML 檔案。</li>
  <li>撰寫基本文件結構。</li>
  <li>在瀏覽器中開啟檢查結果。</li>
</ol>
```

常見屬性如下：

| 屬性 | 可用於 | 說明 | 範例 |
|---|---|---|---|
| `start` | `<ol>` | 指定編號起始值 | `<ol start="3">` |
| `reversed` | `<ol>` | 讓編號倒序顯示 | `<ol reversed>` |
| `value` | `<li>` | 指定某個列表項目的編號值 | `<li value="10">` |

> 注意：編號樣式可以用 CSS 調整，但是否使用 `<ol>` 應根據內容是否有順序意義決定。

### 4.3 描述列表 `<dl>`

`<dl>` 用來表示一組「項目名稱」與「項目說明」。

```html
<dl>
  <dt>HTML</dt>
  <dd>用來描述網頁內容與結構的標記語言。</dd>

  <dt>CSS</dt>
  <dd>用來控制網頁外觀與排版的樣式語言。</dd>
</dl>
```

其中：

| 標籤 | 說明 |
|---|---|
| `<dl>` | description list，描述列表容器 |
| `<dt>` | description term，要被描述的項目或名稱 |
| `<dd>` | description details，對 `<dt>` 的說明內容 |

一個 `<dt>` 可以對應多個 `<dd>`，也可以多個 `<dt>` 共用一組說明，視內容語意而定。

```html
<dl>
  <dt>官方微博</dt>
  <dd>用於發布最新消息與活動資訊。</dd>

  <dt>官方微信</dt>
  <dd>用於提供客服、文章推播與會員通知。</dd>
</dl>
```

描述列表適合用於術語解釋、商品規格、聯絡方式、設定項說明等內容。如果只是一般項目清單，不需要硬改成 `<dl>`。

### 4.4 列表巢狀

列表巢狀是指某個列表項目中，又包含另一個列表。

正確做法是把子列表放在父層的 `<li>` 裡面：

```html
<h2>我想去的城市</h2>
<ul>
  <li>成都</li>
  <li>
    上海
    <ul>
      <li>外灘</li>
      <li>杜莎夫人蠟像館</li>
      <li>
        <a href="https://example.com">東方明珠</a>
      </li>
      <li>迪士尼樂園</li>
    </ul>
  </li>
  <li>西安</li>
  <li>武漢</li>
</ul>
```

這個範例中，「外灘、杜莎夫人蠟像館、東方明珠、迪士尼樂園」是「上海」底下的子項目，所以子 `<ul>` 應該放在「上海」這個 `<li>` 裡。

## 5. 範例說明

### 範例一：新聞列表

```html
<section>
  <h2>最新消息</h2>
  <ul>
    <li><a href="./news-1.html">HTML 課程更新公告</a></li>
    <li><a href="./news-2.html">前端讀書會報名開始</a></li>
    <li><a href="./news-3.html">期末專題繳交說明</a></li>
  </ul>
</section>
```

這裡使用 `<ul>`，因為三則消息是同一組相關內容，但不一定有嚴格排名或步驟順序。每個 `<li>` 裡放一個 `<a>`，表示每則消息都可以連到詳細頁。

### 範例二：教學步驟

```html
<h2>建立 HTML 頁面的基本步驟</h2>
<ol>
  <li>建立一個副檔名為 <code>.html</code> 的檔案。</li>
  <li>加入 <code>&lt;!doctype html&gt;</code> 與基本文件結構。</li>
  <li>在 <code>&lt;body&gt;</code> 中撰寫頁面內容。</li>
  <li>用瀏覽器開啟檔案並檢查結果。</li>
</ol>
```

這裡使用 `<ol>`，因為步驟順序會影響操作結果。若把步驟順序任意調換，學習者可能會理解錯誤。

### 範例三：名詞解釋

```html
<h2>HTML 常見名詞</h2>
<dl>
  <dt>元素</dt>
  <dd>由標籤與內容組成的 HTML 結構，例如 <code>&lt;p&gt;Hello&lt;/p&gt;</code>。</dd>

  <dt>屬性</dt>
  <dd>寫在開始標籤中，用來補充元素資訊，例如 <code>href</code>、<code>src</code>、<code>alt</code>。</dd>
</dl>
```

這裡使用 `<dl>`，因為內容是一組名詞與定義。若改用 `<ul>` 也能顯示，但語意不如 `<dl>` 清楚。

## 6. 實務應用場景

列表在實務開發中非常常見，例如：

- 網站導覽選單。
- 商品特色列表。
- 文章列表或新聞列表。
- 排行榜或比賽名次。
- 表單填寫步驟或操作流程。
- FAQ、術語表、商品規格說明。
- 卡片列表、側邊欄選單、頁尾連結群組。

常見的導覽列也可以用列表表示：

```html
<nav aria-label="主要導覽">
  <ul>
    <li><a href="./index.html">首頁</a></li>
    <li><a href="./courses.html">課程</a></li>
    <li><a href="./about.html">關於我們</a></li>
    <li><a href="./contact.html">聯絡</a></li>
  </ul>
</nav>
```

這段 HTML 表達的是「主要導覽中有一組連結」。至於它在畫面上要直排、橫排、是否顯示項目符號，都應交給 CSS 處理。

```css
nav ul {
  list-style: none;
  padding: 0;
  margin: 0;
  display: flex;
  gap: 16px;
}
```

> 補充說明：`list-style: none` 只是移除項目符號，不會改變 `<ul>` 與 `<li>` 的語意。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 在 `<ul>` 或 `<ol>` 裡直接放文字 | 列表容器的直接子元素應該是 `<li>` | 把每個項目包在 `<li>` 中 |
| 需要順序的步驟卻使用 `<ul>` | 無法表達步驟或排名的順序語意 | 改用 `<ol>` |
| 沒有順序意義的項目使用 `<ol>` | 會讓讀者誤以為項目有排名或先後 | 改用 `<ul>` |
| 把子列表放在父層 `<ul>` 外面 | 巢狀關係不清楚，DOM 結構也不正確 | 把子列表放進對應的父層 `<li>` |
| 為了縮排而濫用 `<dl>` | `<dl>` 的語意是描述列表，不是排版工具 | 若只是排版，使用適合的 HTML 結構並交給 CSS |
| 用表格排列表內容 | 表格應用於表格資料，不應用於一般排版 | 列表型內容用列表標籤，排版交給 CSS |

### 錯誤範例：直接在 `<ul>` 中放文字

```html
<ul>
  首頁
  關於我們
  聯絡我們
</ul>
```

問題：`<ul>` 中缺少 `<li>`，瀏覽器可能仍會嘗試修正 DOM，但這不是正確寫法，也不利於維護與可訪問性。

### 正確範例

```html
<ul>
  <li>首頁</li>
  <li>關於我們</li>
  <li>聯絡我們</li>
</ul>
```

每個項目都使用 `<li>` 包起來，結構清楚，也符合列表語意。

### 錯誤範例：子列表放錯位置

```html
<ul>
  <li>上海</li>
  <ul>
    <li>外灘</li>
    <li>迪士尼樂園</li>
  </ul>
</ul>
```

問題：子列表應該屬於「上海」這個項目，但它被放在父層 `<ul>` 的直接子層。

### 正確範例

```html
<ul>
  <li>
    上海
    <ul>
      <li>外灘</li>
      <li>迪士尼樂園</li>
    </ul>
  </li>
</ul>
```

子列表放在對應的父層 `<li>` 裡，才能正確表達階層關係。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| CSS | CSS 常用來移除列表預設項目符號、調整縮排、改成橫向排列或卡片排版 |
| JavaScript | JavaScript 可以透過 DOM API 新增、刪除或排序列表項目 |
| DOM | 瀏覽器會把列表解析成 DOM Tree，巢狀列表會形成父子節點關係 |
| 可訪問性 | 正確使用列表語意有助於輔助工具理解「這是一組項目」以及項目數量 |
| 語意化 HTML | 選擇 `<ul>`、`<ol>`、`<dl>` 應根據內容關係，而不是根據預設外觀 |
| 表格 `<table>` | 表格適合表格資料；一般項目集合、步驟或導覽不應用表格排版 |

## 9. 重點整理

- 列表標籤用來描述一組相關內容，不只是產生項目符號或數字。
- `<ul>` 適合沒有順序差異的項目。
- `<ol>` 適合順序、步驟或排名有意義的項目。
- `<dl>` 適合名詞與描述、欄位與說明、項目與細節的配對內容。
- `<ul>` 與 `<ol>` 的直接子元素應該是 `<li>`。
- `<li>` 可以包含文字、段落、連結、圖片、按鈕或巢狀列表。
- 列表的視覺樣式應交給 CSS，不應為了樣式而破壞 HTML 語意。

## 11. 延伸補充

> 補充說明：瀏覽器會根據 HTML 建立 DOM Tree。當列表結構寫得清楚時，CSS 比較容易選取與排版，JavaScript 也比較容易操作特定列表項目。

在大型專案中，列表也常出現在元件化開發中。例如 Vue 的 `v-for` 常會渲染出一組 `<li>`。即使是透過框架產生 DOM，最後呈現在瀏覽器中的仍然是 HTML 結構，因此仍應保留正確語意。

```html
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
```

如果這組內容未來要由 JavaScript 或框架動態產生，仍然應維持相同語意：一組相關項目使用列表，每個項目使用 `<li>`。
