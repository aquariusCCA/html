# HTML 塊級元素與行內元素

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明塊級元素與行內元素在預設排版行為上的差異。
- 分辨常見的塊級元素與行內元素。
- 理解 `width`、`height`、`margin`、`padding` 對不同類型元素的影響。
- 正確使用 `div` 與 `span` 作為無語意容器。
- 避免將不適合的元素互相巢狀，例如在 `p` 裡放 `div`、在標題裡再放標題、在 `a` 裡再放 `a`。
- 理解 HTML 元素的「預設顯示方式」不等於元素的「語意」。

## 2. 本章定位

塊級元素與行內元素是學習 HTML 結構與 CSS 排版前必須理解的基礎觀念。它們可以幫助你判斷元素在頁面中預設如何排列，也能幫助你理解為什麼有些元素會自動換行，有些元素會排在同一行。

本章的重點不是背誦所有元素分類，而是理解兩件事：

1. 不同元素有不同的預設顯示行為。
2. HTML 元素能不能互相包含，除了看畫面效果，也要看元素的語意與內容模型。

> 來源重點：原始資料主要說明塊級元素會獨佔一行、行內元素不會獨佔一行，並整理了 `h1` 到 `h6`、`p`、`a` 等元素的巢狀限制。

## 3. 核心概念

### 3.1 什麼是塊級元素

塊級元素通常會在頁面中形成一個獨立區塊。它的預設行為是從新的一行開始，並且在一般排版流中佔滿父元素可用寬度。

常見塊級元素包括：

| 元素 | 常見用途 |
|---|---|
| `div` | 無語意的區塊容器。 |
| `p` | 段落文字。 |
| `h1` 到 `h6` | 標題層級。 |
| `ul`、`ol`、`li` | 清單。 |
| `dl`、`dt`、`dd` | 描述清單。 |
| `form` | 表單區塊。 |
| `table` | 表格資料。 |
| `blockquote` | 區塊引用。 |
| `pre` | 保留空白與換行的預格式化文字。 |
| `hr` | 主題分隔線。 |

塊級元素常見特點：

- 預設會獨佔一行。
- 多個塊級元素連續出現時，通常會由上到下排列。
- 未設定寬度時，通常會填滿父元素可用寬度。
- 可以設定 `width`、`height`、`margin`、`padding`。
- 常被用來建立頁面結構或包住一組內容。

### 3.2 什麼是行內元素

行內元素通常會跟文字一起排在同一行。它不會主動另起一行，而是根據內容寬度自然排列；當同一行空間不足時，內容才會換到下一行。

常見行內元素包括：

| 元素 | 常見用途 |
|---|---|
| `span` | 無語意的行內容器。 |
| `a` | 超連結。 |
| `strong` | 重要內容。 |
| `em` | 強調內容。 |
| `b` | 視覺上加粗但不一定表示重要性。 |
| `i` | 視覺上斜體或特殊語氣內容。 |
| `q` | 行內短引用。 |
| `sub`、`sup` | 下標與上標。 |
| `br` | 換行。 |
| `img` | 圖片。 |
| `input`、`select`、`textarea` | 表單控制項。 |

行內元素常見特點：

- 預設不會獨佔一行。
- 寬度通常由內容決定。
- 一般文字型行內元素設定 `width` 與 `height` 不會產生預期效果。
- 左右方向的 `margin`、`padding` 通常會影響排列。
- 上下方向的 `margin` 通常不會推開其他行；上下 `padding` 可能影響視覺繪製，但不適合拿來控制段落間距。

> 補充說明：`img`、`input`、`textarea` 等元素雖然常出現在行內排版中，但它們屬於替換元素或表單控制項，瀏覽器會給它們特殊行為，因此可以設定尺寸。不要把它們和純文字型的 `span` 完全視為同一種盒模型行為。

### 3.3 預設顯示方式不等於語意

塊級與行內主要描述的是元素在瀏覽器中的預設排版行為。HTML 元素真正重要的是語意，也就是它表示什麼內容。

例如：

```html
<h1>課程介紹</h1>
<p>這是一段課程說明。</p>
```

`h1` 不只是「字比較大、會換行」，它代表頁面的最高層級標題。`p` 不只是「會獨佔一行」，它代表一段文字。即使 CSS 可以改變它們的顯示方式，也不應改變它們在 HTML 中的語意選擇。

```css
h1 {
  display: inline;
}
```

上面這段 CSS 可以讓 `h1` 在視覺上變成行內顯示，但它在 HTML 語意上仍然是標題。

## 4. 語法與基本用法

### 4.1 塊級元素預設獨佔一行

```html
<h1>HTML 教學</h1>
<p>這是一段介紹文字。</p>
<div>這是一個通用區塊。</div>
```

在預設情況下，`h1`、`p`、`div` 會由上到下排列。即使內容很短，每個元素仍會形成自己的區塊。

### 4.2 行內元素預設排在同一行

```html
<span>一般文字</span>
<strong>重要文字</strong>
<a href="https://example.com">前往網站</a>
```

`span`、`strong`、`a` 預設會跟相鄰的行內內容排在同一行。這類元素適合用來標記文字中的某一小段內容。

### 4.3 `div` 與 `span` 的差異

`div` 和 `span` 都是無語意容器，但使用情境不同。

| 元素 | 預設顯示方式 | 適合用途 |
|---|---|---|
| `div` | 塊級 | 包住一組區塊內容，建立通用結構。 |
| `span` | 行內 | 包住一句話中的部分文字，方便套樣式或操作。 |

正確範例：

```html
<div class="card">
  <h2>會員方案</h2>
  <p>適合想要系統學習 HTML 的初學者。</p>
</div>

<p>
  目前狀態：
  <span class="status">開放報名</span>
</p>
```

`div` 用來包住整張卡片，`span` 用來標記段落中的一小段文字。兩者都沒有額外語意，所以如果有更精準的語意元素，應優先使用語意元素。

### 4.4 塊級元素中通常可以放行內元素與其他區塊

一般容器型的塊級元素，例如 `div`，可以包含行內內容，也可以包含其他區塊內容：

```html
<div class="article-card">
  <h2>HTML 基礎</h2>
  <p>
    學習 HTML 時，請先理解
    <strong>結構</strong>
    與
    <strong>語意</strong>
    。
  </p>
</div>
```

這種結構很常見：外層 `div` 負責包住一組內容，內部再放標題、段落與行內強調文字。

> 注意：不是所有塊級元素都能隨意放入任何內容。能放什麼仍要看該元素本身的 HTML 規則，例如 `p` 不能包含 `div`。

### 4.5 行內元素通常用來包住行內內容

一般行內元素適合包住文字或其他行內內容：

```html
<p>
  請閱讀
  <a href="/terms.html"><strong>服務條款</strong></a>
  後再繼續。
</p>
```

在這個例子中，`a` 是連結，`strong` 表示連結文字中的重要內容。它們都位於段落文字中，因此適合使用行內結構。

## 5. 範例說明

### 範例一：比較塊級與行內元素

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>塊級與行內元素範例</title>
</head>
<body>
  <h1>課程清單</h1>
  <p>以下是目前開放的課程。</p>

  <div>
    <span>HTML</span>
    <span>CSS</span>
    <span>JavaScript</span>
  </div>
</body>
</html>
```

這個範例中：

- `h1` 是標題，預設會形成一個獨立區塊。
- `p` 是段落，預設會另起一行。
- `div` 是通用區塊容器，包住三個 `span`。
- 三個 `span` 是行內元素，預設會排在同一行。

### 範例二：`p` 中不要放 `div`

錯誤範例：

```html
<p>
  這是一段文字。
  <div>這是一個區塊。</div>
</p>
```

問題：`p` 元素代表段落，它不能包含 `div` 這類區塊內容。瀏覽器解析時可能會自動提早結束 `p`，導致 DOM 結構和你寫的原始碼不一致。

正確範例：

```html
<p>這是一段文字。</p>
<div>這是一個區塊。</div>
```

如果 `div` 是段落後面的獨立區塊，就把它放在 `p` 外面。

### 範例三：標題元素不要互相嵌套

錯誤範例：

```html
<h1>
  HTML 教學
  <h2>塊級元素與行內元素</h2>
</h1>
```

問題：`h1` 到 `h6` 是標題層級，不應互相包住。標題應該依文件結構平行或分層排列，而不是巢狀在另一個標題裡。

正確範例：

```html
<h1>HTML 教學</h1>
<h2>塊級元素與行內元素</h2>
```

這樣的結構表示「HTML 教學」是最高層級標題，「塊級元素與行內元素」是下一層標題。

### 範例四：`a` 可以包住多種內容，但不能包住另一個 `a`

HTML5 中，`a` 元素在符合內容規則的情況下，可以包住一段文字，也可以包住一組內容：

```html
<a href="/courses/html.html">
  <h2>HTML 基礎課程</h2>
  <p>從文件結構、標籤語意到表單基礎。</p>
</a>
```

但 `a` 不能包含另一個 `a`：

```html
<a href="/courses.html">
  課程列表
  <a href="/courses/html.html">HTML 課程</a>
</a>
```

問題：巢狀連結會讓瀏覽器、鍵盤操作與輔助工具難以判斷使用者到底要啟用哪一個連結。

正確範例：

```html
<a href="/courses.html">課程列表</a>
<a href="/courses/html.html">HTML 課程</a>
```

## 6. 實務應用場景

### 6.1 建立頁面結構

塊級元素常用於建立頁面的主要結構，例如標題、段落、清單、表單與內容區塊。

```html
<main>
  <h1>文章標題</h1>
  <p>文章導言。</p>

  <section>
    <h2>第一節</h2>
    <p>第一節內容。</p>
  </section>
</main>
```

在實務上，如果有語意元素可以使用，應優先選擇語意元素，例如 `main`、`section`、`article`、`nav`，而不是所有區塊都使用 `div`。

### 6.2 標記文字中的局部內容

行內元素常用來標記段落中的局部文字，例如強調、連結、縮寫、程式碼片段或狀態文字。

```html
<p>
  請使用
  <code>alt</code>
  屬性描述圖片內容，這對
  <strong>可訪問性</strong>
  很重要。
</p>
```

這種寫法可以保留段落的完整語意，同時標記出其中需要強調或特殊處理的部分。

### 6.3 搭配 CSS 控制顯示方式

HTML 負責內容與語意，CSS 負責畫面呈現。若需要改變元素排列方式，通常應透過 CSS 處理。

```html
<nav class="site-nav">
  <a href="/">首頁</a>
  <a href="/courses.html">課程</a>
  <a href="/about.html">關於我們</a>
</nav>
```

```css
.site-nav {
  display: flex;
  gap: 16px;
}
```

這裡的 `a` 仍然是連結，`nav` 仍然代表導覽區塊。CSS 只是改變它們在畫面上的排列。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 把 `h1` 到 `h6` 當成放大字體工具 | 標題元素代表文件層級，不是單純樣式 | 用正確標題層級表達結構，字體大小交給 CSS |
| 在 `p` 裡放 `div`、`h2`、`ul` 等區塊內容 | `p` 只能放適合段落內的內容，瀏覽器可能自動修正 DOM | 將區塊元素移到 `p` 外面 |
| 將 `h2` 放進 `h1` 裡 | 標題元素不應互相巢狀 | 依序排列標題，用層級表示文件結構 |
| 在 `a` 裡再放另一個 `a` | 巢狀連結會造成互動目標不清楚，也不符合 HTML 規則 | 將兩個連結拆成平行元素 |
| 對 `span` 設定 `width`、`height` 後期待它像區塊一樣排版 | 一般行內元素不會套用區塊尺寸行為 | 若真的需要尺寸控制，使用合適的區塊元素或搭配 CSS `display` |
| 所有區塊都用 `div` | `div` 沒有語意，過度使用會降低可讀性與可訪問性 | 優先使用 `header`、`main`、`section`、`article`、`nav` 等語意元素 |
| 用預設外觀判斷元素用途 | 預設外觀可以被 CSS 改變，但語意不會因此改變 | 根據內容意義選元素，再用 CSS 調整畫面 |
| 使用已過時的 `<marquee>` 做跑馬燈效果 | `<marquee>` 不適合現代 HTML 實務 | 若需要動畫效果，改用 CSS animation 或 JavaScript 控制 |

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| CSS `display` | 可以改變元素在畫面上的排版方式，例如 `block`、`inline`、`inline-block`、`flex`。 |
| 盒模型 | 塊級與行內元素對 `width`、`height`、`margin`、`padding` 的反應不同。 |
| DOM | 瀏覽器會依 HTML 規則解析文件，不合法的巢狀可能被自動修正成不同 DOM 結構。 |
| 語意化 HTML | 選擇元素時應先考慮內容語意，而不是只考慮預設換不換行。 |
| 可訪問性 | 正確的標題層級、連結結構與語意元素能幫助輔助工具理解頁面。 |
| JavaScript | JavaScript 操作的是瀏覽器解析後的 DOM；不合法 HTML 可能造成選取與操作結果和預期不同。 |
| Vue 或其他框架 | 即使使用 template 或元件，最後仍會渲染成 DOM，因此 HTML 結構與語意仍然重要。 |

## 9. 重點整理

- 塊級元素預設會獨佔一行，常用於建立頁面結構。
- 行內元素預設不會獨佔一行，常用於標記文字中的局部內容。
- `div` 是無語意區塊容器，`span` 是無語意行內容器。
- 不要只用預設畫面效果判斷元素用途，HTML 元素的語意更重要。
- 一般行內文字元素不適合直接用 `width`、`height` 控制尺寸。
- `p` 中不能放 `div` 這類區塊元素，標題元素也不應互相嵌套。
- `a` 可以包住多種內容，但不能包含另一個 `a`，也不應讓互動元素互相巢狀。
- 不合法的 HTML 巢狀可能會被瀏覽器自動修正，導致 DOM 結構與原始碼不同。

## 11. 延伸補充

> 補充說明：現代 HTML 更精確的判斷方式是看元素的內容模型，例如 flow content、phrasing content、interactive content 等。塊級與行內是非常實用的入門分類，但遇到元素能不能互相包含時，不能只看它們預設是不是換行，而要回到 HTML 規則與元素語意判斷。

瀏覽器解析 HTML 時，會把原始碼轉成 DOM Tree。如果原始碼中出現不合法的巢狀，例如在 `p` 裡放 `div`，瀏覽器可能會自動補上結束標籤或重排節點。這會讓 CSS 選擇器、JavaScript DOM 操作與輔助工具讀取結果和你原本想像的不一樣。

因此，學習塊級與行內元素時，應同時建立兩個觀念：

- 排版行為可以交給 CSS 調整。
- HTML 結構與語意要在原始碼中寫正確。
