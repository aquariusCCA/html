# HTML 圖片標籤 img

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `<img>` 標籤在 HTML 中的用途。
- 理解 `src`、`alt`、`title`、`width`、`height` 等常見屬性的作用。
- 正確撰寫基本圖片語法。
- 分辨常見圖片格式的特性與適合情境。
- 避免圖片路徑錯誤、缺少替代文字、圖片變形等常見問題。

## 2. 本章定位

本章介紹 HTML 中用來顯示圖片的 `<img>` 元素。

圖片是網頁內容中很常見的資源，例如產品照片、文章插圖、Logo、icon、教學截圖或活動海報。HTML 透過 `<img>` 把圖片資源嵌入頁面，瀏覽器會根據 `src` 指定的路徑載入圖片，並將圖片顯示在文件中的對應位置。

學習 `<img>` 時不能只記「圖片怎麼顯示」。還要理解圖片路徑、替代文字、尺寸控制、圖片格式與可訪問性的關係，這些都會影響網頁是否容易維護、是否能被搜尋引擎與輔助工具理解，以及圖片載入失敗時使用者是否仍能理解內容。

> 來源重點：原始資料指出 `<img>` 是單標籤，`src` 是必要屬性，用於指定圖像文件的路徑和文件名，並整理了常見圖片格式與 `width`、`height` 的注意事項。

## 3. 核心概念

`<img>` 是 image 的縮寫，表示「圖像」。它是 HTML 中用來在網頁中顯示圖片的元素。

`<img>` 有兩個重要特點：

- 它是單標籤，不需要結束標籤。
- 它必須透過屬性指定圖片來源與相關資訊。

最基本的圖片寫法如下：

```html
<img src="./images/product.jpg" alt="商品照片">
```

其中：

| 屬性 | 作用 |
|---|---|
| `src` | 指定圖片檔案的路徑，是顯示圖片最核心的屬性 |
| `alt` | 圖片無法顯示時的替代文字，也提供給輔助工具理解圖片內容 |

如果沒有 `src`，瀏覽器不知道要載入哪一張圖片，圖片就無法正常顯示。如果沒有適當的 `alt`，圖片載入失敗或使用螢幕閱讀器時，使用者可能無法理解圖片代表的內容。

> 補充說明：`<img>` 不只是視覺元素。對內容型圖片而言，`alt` 會影響可訪問性，也能幫助機器理解圖片內容。

## 4. 語法與基本用法

### 4.1 基本語法

```html
<img
  src="圖片路徑"
  alt="替代文字"
  title="提示文字"
  width="寬度"
  height="高度"
>
```

常見屬性如下：

| 屬性 | 是否常用 | 說明 |
|---|---|---|
| `src` | 必要 | 指定圖片檔案的位置，例如相對路徑、絕對網址或資料 URI |
| `alt` | 建議必寫 | 圖片替代文字，圖片失敗時顯示，也供輔助工具讀取 |
| `title` | 視情況使用 | 滑鼠停留時可能顯示提示文字，但不應取代 `alt` |
| `width` | 視情況使用 | 設定圖片顯示寬度 |
| `height` | 視情況使用 | 設定圖片顯示高度 |

### 4.2 `src`：圖片來源

`src` 是 source 的縮寫，用來指定圖片檔案的路徑。

```html
<img src="./images/avatar.jpg" alt="使用者頭像">
```

`src` 的值必須能讓瀏覽器找到圖片。常見來源包括：

| 寫法 | 範例 | 說明 |
|---|---|---|
| 相對路徑 | `./images/photo.jpg` | 指向目前 HTML 檔案附近的圖片 |
| 絕對網址 | `https://example.com/photo.jpg` | 指向網路上的圖片資源 |
| base64 資料 | `data:image/png;base64,...` | 把圖片資料直接寫在文字中 |

初學者最常遇到的問題是路徑寫錯。只要 `src` 指到不存在的位置，圖片就無法顯示。

### 4.3 `alt`：替代文字

`alt` 是 alternative text 的縮寫，表示圖片替代文字。

```html
<img src="./images/logo.png" alt="公司 Logo">
```

`alt` 的用途包括：

- 圖片載入失敗時，讓使用者知道原本應該是什麼圖片。
- 讓螢幕閱讀器等輔助工具能讀出圖片含義。
- 幫助搜尋引擎理解圖片內容。

`alt` 應描述圖片在目前頁面中的用途，而不是只描述檔名。

```html
<!-- 不建議 -->
<img src="./images/img001.png" alt="img001">

<!-- 較佳 -->
<img src="./images/login-error.png" alt="登入失敗錯誤提示畫面">
```

若圖片只是裝飾，不承載內容意義，可以使用空的 `alt`：

```html
<img src="./images/decorative-line.png" alt="">
```

### 4.4 `title`：提示文字

`title` 可以提供額外提示，部分瀏覽器在滑鼠停留時會顯示提示文字。

```html
<img
  src="./images/chart.png"
  alt="第二季銷售額統計圖"
  title="資料來源：2026 年第二季報表"
>
```

`title` 不應取代 `alt`。如果圖片本身有內容意義，仍然應該提供適當的 `alt`。

### 4.5 `width` 與 `height`：圖片尺寸

`width` 與 `height` 用來設定圖片顯示尺寸。

```html
<img src="./images/avatar.jpg" alt="使用者頭像" width="200">
```

原始資料提醒：

- 如果只設定 `width` 或 `height` 其中一個，另一個通常會依照圖片原比例自動縮放。
- 如果同時設定 `width` 與 `height`，但比例不符合原圖，圖片可能變形。

```html
<!-- 較不容易變形：只指定寬度 -->
<img src="./images/photo.jpg" alt="活動照片" width="300">

<!-- 可能變形：寬高比例與原圖不一致 -->
<img src="./images/photo.jpg" alt="活動照片" width="300" height="300">
```

> 補充說明：在實務專案中，常會搭配 CSS 控制圖片大小，例如 `max-width: 100%; height: auto;`，讓圖片能隨容器縮放並維持比例。

## 5. 範例說明

### 範例一：顯示基本圖片

```html
<body>
  <h1>產品介紹</h1>
  <img src="./images/product.jpg" alt="黑色無線耳機產品照片">
</body>
```

這個範例中：

- `src="./images/product.jpg"` 表示圖片位於目前頁面的 `images` 資料夾中。
- `alt="黑色無線耳機產品照片"` 說明圖片內容。
- `<img>` 不需要寫 `</img>`。

### 範例二：圖片載入失敗時的替代文字

```html
<img src="./images/not-found.jpg" alt="使用者個人頭像">
```

如果 `not-found.jpg` 不存在或無法載入，瀏覽器可能會顯示替代文字。即使圖片失敗，使用者仍能知道這裡原本是使用者頭像。

### 範例三：設定圖片尺寸

```html
<img src="./images/banner.jpg" alt="課程活動橫幅" width="600">
```

這個寫法只設定寬度，瀏覽器會依照圖片原始比例計算高度，通常可以避免圖片被拉伸變形。

若需要用 CSS 控制響應式圖片，可以這樣寫：

```html
<img class="responsive-image" src="./images/banner.jpg" alt="課程活動橫幅">
```

```css
.responsive-image {
  max-width: 100%;
  height: auto;
}
```

這樣圖片最多不超過容器寬度，並保留原始比例。

## 6. 常見圖片格式

不同圖片格式適合不同情境。選擇圖片格式時，應考慮圖片內容、透明需求、動畫需求、檔案大小與瀏覽器支援。

| 格式 | 特性 | 適合情境 | 注意事項 |
|---|---|---|---|
| `.jpg` / `.jpeg` | 有損壓縮，顏色豐富，檔案通常較小 | 照片、產品圖、宣傳圖 | 不支援透明背景與動畫 |
| `.png` | 無損壓縮，支援透明背景 | Logo、重要配圖、截圖、需要透明背景的圖片 | 檔案可能比 jpg 大 |
| `.bmp` | 幾乎不壓縮，保留細節多 | 特殊高細節需求 | 檔案通常很大，網頁中較少使用 |
| `.gif` | 支援簡單動畫，顏色數有限 | 小型動態圖片 | 色彩表現有限，不適合高品質照片 |
| `.webp` | 壓縮效率佳，可支援多種圖片需求 | 網頁圖片、縮圖、產品圖 | 專案仍應確認目標瀏覽器與環境支援 |
| base64 | 將圖片轉成文字資料 | 很小、需要和 HTML 或 CSS 一起載入的小圖 | 不適合大型圖片，會讓文字檔變大且較難維護 |

原始資料中特別提到 base64：它本質上是一種文字資料，瀏覽器可以把它解析成圖片。使用時可以直接放在 `img` 的 `src` 屬性中。

```html
<img src="data:image/png;base64,..." alt="內嵌的小圖示">
```

> 注意：base64 適合小型圖片或特殊情境，不應把大量或大型圖片都轉成 base64，否則 HTML 或 CSS 檔案會變得很大，也不利於維護。

## 7. 實務應用場景

`<img>` 常用在以下場景：

- 商品頁顯示產品照片。
- 文章或教學頁插入說明圖片。
- 網站頁首顯示 Logo。
- 使用者資料頁顯示頭像。
- 儀表板或報表中顯示截圖、圖表或狀態圖。

在實務開發中，圖片通常不只是單一 HTML 標籤的問題，還會牽涉到：

- 圖片檔案是否放在正確目錄。
- 檔案名稱是否容易維護。
- 圖片是否過大，影響頁面載入速度。
- `alt` 是否能準確描述圖片用途。
- 尺寸是否會導致版面跳動或圖片變形。
- 是否需要搭配 CSS 做響應式顯示。

## 8. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| `src` 路徑寫錯 | 瀏覽器找不到圖片檔案 | 確認 HTML 檔案與圖片檔案的相對位置 |
| 缺少 `alt` | 圖片失敗或輔助工具讀取時資訊不足 | 為有內容意義的圖片補上清楚替代文字 |
| 用 `title` 取代 `alt` | `title` 不是可靠的替代文字來源 | 圖片內容意義應放在 `alt` |
| 同時亂設 `width` 與 `height` | 寬高比例與原圖不同，圖片會變形 | 只設定其中一個，或用 CSS 保持比例 |
| 把大型圖片直接放進頁面 | 檔案太大會拖慢載入速度 | 壓縮圖片並選擇合適格式 |
| 裝飾圖片也寫冗長 `alt` | 輔助工具會讀出無意義內容 | 裝飾圖片可用 `alt=""` |
| 只用圖片呈現重要文字 | 圖片無法載入或無法被輔助工具理解時會遺失資訊 | 重要文字應使用 HTML 文字，圖片只作輔助 |

### 錯誤範例：缺少替代文字

```html
<img src="./images/error-message.png">
```

問題：如果圖片載入失敗，使用者不知道這張圖片代表什麼；使用輔助工具的使用者也難以理解圖片內容。

### 正確範例：提供清楚的 `alt`

```html
<img src="./images/error-message.png" alt="登入失敗，提示密碼輸入錯誤">
```

修正重點：`alt` 描述圖片在此處要傳達的資訊，而不是只寫檔名。

### 錯誤範例：圖片被壓成不正確比例

```html
<img src="./images/photo.jpg" alt="活動合照" width="300" height="100">
```

問題：如果原圖不是 3:1 比例，圖片會被拉伸或壓扁。

### 正確範例：保持圖片比例

```html
<img src="./images/photo.jpg" alt="活動合照" width="300">
```

或使用 CSS：

```css
img {
  max-width: 100%;
  height: auto;
}
```

## 9. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| HTML 路徑 | `src` 必須正確指向圖片檔案，路徑觀念會直接影響圖片是否顯示 |
| CSS | CSS 負責圖片尺寸、裁切、邊框、排版與響應式效果 |
| JavaScript | JavaScript 可透過 DOM 操作圖片屬性，例如切換 `src` |
| DOM | 瀏覽器會把 `<img>` 解析成 DOM 中的圖片元素節點 |
| 可訪問性 | `alt` 能幫助螢幕閱讀器理解圖片內容 |
| SEO | 適當的圖片替代文字與檔案管理有助於機器理解圖片，但不應誇大成排名保證 |
| 網頁效能 | 圖片通常是頁面中較大的資源，格式與尺寸會影響載入速度 |

## 10. 重點整理

- `<img>` 用來在 HTML 頁面中顯示圖片。
- `<img>` 是單標籤，不需要結束標籤。
- `src` 是圖片來源，是 `<img>` 的核心屬性。
- `alt` 是替代文字，與圖片載入失敗、可訪問性和內容理解有關。
- `title` 可提供提示，但不應取代 `alt`。
- 只設定 `width` 或 `height` 其中一個，通常可以讓另一邊依比例縮放。
- 同時設定錯誤的寬高比例，可能造成圖片變形。
- jpg、png、bmp、gif、webp、base64 各有不同適合場景。
- 圖片不是只關乎畫面，也會影響可訪問性、SEO、效能與維護性。

## 12. 延伸補充

`<img>` 是學習 HTML 資源引用的重要入口。它會同時牽涉到路徑、檔案格式、可訪問性、效能與 CSS 排版。

> 補充說明：HTML 負責把圖片資源放入文件結構中，CSS 負責控制圖片的視覺呈現，JavaScript 則可以在需要互動時動態切換圖片或修改圖片屬性。三者分工清楚，圖片相關程式碼才會比較容易維護。
