# mark 標籤：標示目前上下文中的重點文字

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `<mark>` 標籤的語意與使用情境。
- 寫出 `<mark>` 的基本語法。
- 分辨 `<mark>` 是「語意標記」，不是單純的黃底樣式工具。
- 知道如何用 CSS 調整 `<mark>` 的預設呈現。

## 使用情境

假設你正在做一篇教學文章，想讓讀者注意某個關鍵詞；或你正在做搜尋結果頁，想把符合搜尋字串的片段標示出來。這些文字不是整篇文章最重要的強調句，也不一定代表警告或強烈語氣，而是「在目前脈絡裡值得被標出來」。

這種情境就適合使用 `<mark>`。

## 一句話理解

`<mark>` 用來標示「在目前上下文中需要讀者注意或參考的文字片段」。

## 標籤語意

`<mark>` 是 HTML5 新增的語意標籤。它的重點不是「把文字變成黃色」，而是告訴瀏覽器與讀者：這段文字在目前內容脈絡中被標記出來。

常見使用情境包括：

- 標示文章中的關鍵詞。
- 標示搜尋結果中命中的文字。
- 標示一段說明裡目前正在討論的詞語。
- 標示需要讀者暫時注意的參考片段。

在大多數瀏覽器中，`<mark>` 內的文字預設會呈現黃色背景，但這只是瀏覽器給的預設樣式，不是 `<mark>` 的全部意義。

## 基本語法

```html
<p>學習 HTML 時，先理解 <mark>標籤語意</mark> 會比只記語法更重要。</p>
```

這段程式碼會把「標籤語意」標示出來，讓讀者知道這是目前句子中需要注意的關鍵片段。

## 範例拆解

```html
<h2>Example of Mark Tag</h2>
<p>You can easily learn designing a website with our <mark>Web designing tutorial.</mark></p>
```

逐段來看：

- `<h2>Example of Mark Tag</h2>`：建立範例標題。
- `<p>...</p>`：建立一段普通段落文字。
- `<mark>Web designing tutorial.</mark>`：把段落中的特定文字片段標示為目前上下文中的重點。

如果這段內容出現在教學或搜尋結果中，`<mark>` 可以讓讀者更快看到被標出的關鍵詞。

## 調整 mark 的樣式

瀏覽器通常會替 `<mark>` 加上黃色背景。若你的版面需要不同顏色，可以用 CSS 調整。

```html
<p>這堂課會先介紹 <mark>HTML5 新增標籤</mark> 的用途。</p>
```

```css
mark {
  background-color: #ffe08a;
  color: #1f2933;
}
```

這樣仍然保留 `<mark>` 的語意，只是改變它在畫面上的呈現方式。

## mark 與 strong 的差異

`<mark>` 和 `<strong>` 都可能讓某段文字看起來比較醒目，但語意不同。

| 情境 | 建議標籤 |
| --- | --- |
| 標示搜尋命中的文字 | `<mark>` |
| 標示目前句子中要讀者注意的關鍵詞 | `<mark>` |
| 表示內容本身非常重要或有強烈重要性 | `<strong>` |
| 只是想改變背景顏色，沒有語意上的標記需求 | CSS 樣式 |

例如「請務必填寫真實姓名」中的「務必」比較接近重要性強調，通常適合 `<strong>`。但搜尋「HTML」後，把結果摘要中的「HTML」標出來，則適合 `<mark>`。

## 常見錯誤

### 把 mark 當成黃底樣式工具

錯誤觀念是：只要想讓文字變黃，就使用 `<mark>`。

```html
<p>今天特價：<mark>全部商品 8 折</mark></p>
```

如果這段文字的目的只是做視覺促銷樣式，而不是標示目前上下文中被參照或需要注意的片段，使用 CSS class 會更清楚。

```html
<p>今天特價：<span class="sale-text">全部商品 8 折</span></p>
```

`<mark>` 應該先看語意是否成立，再看畫面效果。

### 用 mark 取代 strong

```html
<p><mark>請勿關閉電源。</mark></p>
```

如果這句話代表重要警告，重點是「內容很重要」，而不是「這段文字被目前上下文標記出來」，那麼 `<strong>` 通常更適合。

```html
<p><strong>請勿關閉電源。</strong></p>
```

### 標記過多文字

如果一整段文字幾乎都被 `<mark>` 包起來，讀者反而看不出真正需要注意的是哪裡。

```html
<p><mark>這一整段很長的文字全部都被標記，讀者很難判斷真正的重點在哪裡。</mark></p>
```

比較好的做法是只標示關鍵片段。

```html
<p>這一段主要說明 <mark>只標示關鍵片段</mark> 才能讓重點清楚。</p>
```

## 實務判斷準則

使用 `<mark>` 前，可以先問自己三個問題：

1. 這段文字是不是在目前上下文中被特別指出來？
2. 讀者看到標記後，是否能更快理解目前討論或搜尋命中的重點？
3. 如果拿掉背景顏色，這段標記在語意上是否仍然合理？

如果答案大多是「是」，`<mark>` 通常合適。反過來，如果只是想讓文字有顏色、背景或裝飾效果，應該使用 CSS；如果是要表示內容很重要，則應考慮 `<strong>`。

## 重點整理

- `<mark>` 用來標示目前上下文中值得注意或供參考的文字片段。
- `<mark>` 是 HTML5 新增的語意標籤。
- 大多數瀏覽器會把 `<mark>` 顯示成黃色背景，但可以用 CSS 調整。
- `<mark>` 不等於一般黃底樣式，也不等於 `<strong>`。
- 實務上常用於關鍵詞、搜尋命中片段與目前討論重點的標示。
