# HTML 路徑

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明路徑在 HTML 中的用途。
- 分辨絕對路徑與相對路徑。
- 使用相對路徑引用同層、下一層與上一層資料夾中的檔案。
- 理解為什麼本機絕對路徑不適合用在正式網頁中。
- 說明網路絕對路徑可能遇到的外部資源與防盜鏈問題。
- 避免因檔案位置調整造成圖片、CSS 或 JavaScript 載入失敗。

## 2. 本章定位

路徑是 HTML 引用外部資源時一定會遇到的基礎觀念。當頁面需要載入圖片、CSS、JavaScript、其他頁面或下載檔案時，都需要透過路徑告訴瀏覽器資源在哪裡。

本章以原始資料中的圖片引用情境為主，說明絕對路徑與相對路徑的差異。這個觀念會直接影響後續學習 `<img>`、`<a>`、`<link>`、`<script>` 等元素。

> 來源重點：原始資料主要說明頁面引用圖片時，需要透過路徑找到圖像檔案，並將路徑分為絕對路徑與相對路徑。

## 3. 核心概念

### 3.1 什麼是路徑

在 HTML 中，路徑可以理解為「檔案的位置」。如果 HTML 頁面要顯示一張圖片，瀏覽器必須知道圖片檔案在哪裡，才能送出請求並把圖片載入頁面。

例如：

```html
<img src="images/logo.png" alt="網站標誌">
```

這段程式中的 `src="images/logo.png"` 就是圖片檔案的路徑。它告訴瀏覽器：請到目前 HTML 文件所在位置底下的 `images` 資料夾中，尋找 `logo.png`。

路徑不只會出現在圖片，也常出現在以下元素：

| 元素或屬性 | 常見用途 |
|---|---|
| `<img src="...">` | 載入圖片。 |
| `<a href="...">` | 連到其他頁面、檔案或網站。 |
| `<link href="...">` | 載入 CSS 或其他外部資源。 |
| `<script src="...">` | 載入 JavaScript 檔案。 |
| `<video src="...">`、`<audio src="...">` | 載入影音資源。 |

### 3.2 路徑的兩大類型

HTML 常見路徑可以分成兩大類：

| 類型 | 說明 | 範例 |
|---|---|---|
| 絕對路徑 | 從固定起點直接描述完整位置。 | `https://example.com/images/logo.png` |
| 相對路徑 | 從目前引用檔案的位置出發，描述目標檔案的位置。 | `images/logo.png` |

初學 HTML 時，最常使用的是相對路徑，因為網站專案通常會把 HTML、圖片、CSS、JavaScript 放在同一個專案資料夾中。只要資料夾結構清楚，相對路徑就比較容易搬移與維護。

## 4. 語法與基本用法

### 4.1 絕對路徑

絕對路徑是從固定根位置出發，直接指出目標檔案的位置。原始資料提到兩種常見形式：

| 類型 | 範例 | 說明 |
|---|---|---|
| 本機絕對路徑 | `D:\day01\images\1.jpg` | 從電腦磁碟位置開始描述檔案位置。 |
| 網路絕對路徑 | `https://www.example.com/images/logo.png` | 從完整網址開始描述資源位置。 |

本機絕對路徑通常不適合寫進正式網頁：

```html
<!-- 不建議：這是某台 Windows 電腦上的本機路徑 -->
<img src="D:\day01\images\1.jpg" alt="範例圖片">
```

這種寫法的問題是，其他使用者的電腦上不一定有 `D:\day01\images\1.jpg` 這個檔案。即使在你自己的電腦可以顯示，換到另一台設備、部署到伺服器或交給同學開啟時，都很容易失敗。

網路絕對路徑則是完整 URL：

```html
<img src="https://example.com/images/logo.png" alt="Example 網站標誌">
```

這種寫法可以直接引用網路上的資源，但會依賴對方伺服器。如果對方移除檔案、改變網址、伺服器無法連線，或設定防盜鏈，圖片就可能無法正常顯示。

> 注意：不要未經允許直接引用別人網站的圖片。除了版權與授權問題，對方網站也可能透過防盜鏈阻止外部網站載入圖片。

### 4.2 相對路徑

相對路徑是從「目前引用這個資源的檔案」出發，去尋找目標檔案的位置。以 HTML 頁面引用圖片為例，相對路徑通常是從目前 HTML 檔案所在資料夾開始計算。

假設專案結構如下：

```text
site/
├── index.html
├── about.html
├── images/
│   └── logo.png
└── pages/
    └── contact.html
```

如果 `index.html` 要引用 `images/logo.png`：

```html
<img src="images/logo.png" alt="網站標誌">
```

因為 `images` 資料夾和 `index.html` 位在同一層，所以可以直接從 `images/` 開始寫。

### 4.3 同層、下一層與上一層

相對路徑最重要的是判斷目標檔案和目前檔案的位置關係。

| 位置關係 | 寫法 | 說明 |
|---|---|---|
| 同一層 | `logo.png` | 目標檔案和目前 HTML 檔案在同一個資料夾。 |
| 下一層 | `images/logo.png` 或 `./images/logo.png` | 目標檔案在目前資料夾底下的子資料夾中。 |
| 上一層 | `../logo.png` | 先回到上一層資料夾，再尋找目標檔案。 |
| 上兩層 | `../../logo.png` | 連續回到上兩層資料夾。 |

`./` 表示目前資料夾。原始資料提到，相對路徑中的 `./` 可以省略，因此：

```html
<img src="./images/logo.png" alt="網站標誌">
```

通常可以簡寫成：

```html
<img src="images/logo.png" alt="網站標誌">
```

`../` 表示上一層資料夾。以上面的專案結構為例，如果 `pages/contact.html` 要引用 `images/logo.png`，因為 `contact.html` 位在 `pages` 裡面，必須先回到上一層 `site/`，再進入 `images/`：

```html
<img src="../images/logo.png" alt="網站標誌">
```

## 5. 範例說明

### 範例：在 HTML 中引用圖片、CSS 與 JavaScript

假設專案結構如下：

```text
site/
├── index.html
├── css/
│   └── style.css
├── js/
│   └── main.js
└── images/
    └── banner.jpg
```

`index.html` 可以這樣引用外部資源：

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>路徑範例</title>
  <link rel="stylesheet" href="css/style.css">
</head>
<body>
  <h1>HTML 路徑範例</h1>
  <img src="images/banner.jpg" alt="網站橫幅">

  <script src="js/main.js"></script>
</body>
</html>
```

這個範例中：

- `href="css/style.css"` 表示到 `css` 資料夾中尋找 `style.css`。
- `src="images/banner.jpg"` 表示到 `images` 資料夾中尋找 `banner.jpg`。
- `src="js/main.js"` 表示到 `js` 資料夾中尋找 `main.js`。

### 範例：子資料夾中的頁面引用圖片

假設專案結構如下：

```text
site/
├── images/
│   └── avatar.png
└── pages/
    └── profile.html
```

`profile.html` 位於 `pages` 資料夾內，而圖片位於上一層的 `images` 資料夾內，所以要使用 `../`：

```html
<img src="../images/avatar.png" alt="使用者頭像">
```

這裡的路徑可以拆成兩段理解：

| 路徑片段 | 意義 |
|---|---|
| `../` | 從 `pages/` 回到上一層 `site/`。 |
| `images/avatar.png` | 再進入 `images/` 找到 `avatar.png`。 |

## 6. 實務應用場景

### 6.1 管理網站資源

當頁面中的圖片變多時，通常會建立專門資料夾管理資源，例如：

```text
project/
├── index.html
├── images/
├── css/
└── js/
```

這樣做可以讓專案結構更清楚，也能避免所有檔案都堆在同一層造成維護困難。

### 6.2 多頁網站連結

多頁網站常會有首頁、關於我們、產品頁、聯絡頁等檔案。這些頁面之間也會使用路徑互相連結：

```html
<a href="pages/about.html">關於我們</a>
<a href="pages/contact.html">聯絡我們</a>
```

如果目前檔案已經在 `pages/` 中，要回到首頁，常見寫法是：

```html
<a href="../index.html">回首頁</a>
```

### 6.3 部署與協作

相對路徑通常比本機絕對路徑更適合團隊協作。只要整個專案資料夾結構保持一致，其他人下載專案後，即使放在不同電腦位置，資源仍然可以正常被找到。

> 補充說明：在 HTML 文件中，相對路徑通常會根據目前文件的位置解析；如果頁面使用 `<base>` 元素，可能會改變相對 URL 的解析基準。初學階段若沒有明確需求，不建議隨意使用 `<base>`。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 使用 `D:\...` 這類本機絕對路徑 | 路徑只存在於自己的電腦，換設備或部署後容易失效。 | 把圖片放進專案資料夾，改用相對路徑。 |
| 把 `./` 和 `../` 混淆 | `./` 是目前資料夾，`../` 是上一層資料夾。 | 先畫出資料夾結構，再從目前 HTML 檔案位置開始找。 |
| HTML 檔案搬到其他資料夾後，圖片失效 | 相對路徑依賴目前檔案位置，檔案位置改變時路徑也可能需要調整。 | 搬移檔案後同步檢查 `src`、`href` 等路徑。 |
| 檔名大小寫不一致 | 有些伺服器會區分大小寫，`Logo.png` 和 `logo.png` 可能是不同檔案。 | 檔名與路徑大小寫保持完全一致。 |
| 直接引用別人網站圖片 | 外部資源可能被移除、禁止引用，或涉及授權問題。 | 使用自己有權使用的圖片，或使用合法授權的 CDN / 圖片來源。 |
| 路徑中包含空白或特殊字元 | 在網址中可能需要編碼，也容易造成引用錯誤。 | 檔名建議使用英文小寫、數字與連字號，例如 `main-banner.jpg`。 |

### 錯誤範例：使用本機絕對路徑

```html
<img src="D:\course\images\logo.png" alt="網站標誌">
```

問題：這張圖片只在特定電腦的特定磁碟位置中存在，其他人開啟網頁時很可能找不到。

### 正確範例：使用專案內的相對路徑

```html
<img src="images/logo.png" alt="網站標誌">
```

修正重點：把圖片放在專案資料夾中的 `images/`，讓 HTML 使用相對路徑引用。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| `<img>` | `src` 屬性需要路徑指定圖片位置。 |
| `<a>` | `href` 屬性可以使用路徑連到其他頁面、檔案或外部網站。 |
| `<link>` | 常用 `href` 載入 CSS 檔案。 |
| `<script>` | 常用 `src` 載入 JavaScript 檔案。 |
| 專案資料夾結構 | 路徑能否正確運作，取決於檔案實際位置與資料夾層級。 |
| URL | 網路絕對路徑本質上是完整 URL。 |
| CSS | CSS 檔案內的 `url(...)` 也會用到路徑，但解析基準通常是 CSS 檔案本身的位置。 |

## 9. 重點整理

- 路徑是用來告訴瀏覽器資源檔案位置的資訊。
- HTML 引用圖片、頁面、CSS、JavaScript 等資源時，都會使用路徑。
- 絕對路徑從固定位置出發，例如本機磁碟路徑或完整網址。
- 本機絕對路徑不適合正式網頁，因為換設備或部署後容易失效。
- 網路絕對路徑可以引用外部資源，但要注意授權、穩定性與防盜鏈問題。
- 相對路徑從目前引用檔案的位置出發，是初學 HTML 與專案開發中最常見的寫法。
- `./` 表示目前資料夾，通常可以省略；`../` 表示上一層資料夾。
- 若調整 HTML 或資源檔案的位置，必須同步檢查相關路徑。

## 11. 延伸補充

> 補充說明：瀏覽器看到 `<img src="images/logo.png">` 時，並不是只把文字顯示在畫面上，而是會根據目前頁面的 URL 解析出圖片的完整資源位置，再向對應位置請求圖片檔案。

在實務開發中，建議養成以下習慣：

- 建立清楚的資料夾結構，例如 `images/`、`css/`、`js/`。
- 檔名使用小寫英文、數字與連字號，避免空白與特殊符號。
- 在搬移 HTML 或資源檔案後，重新檢查頁面是否能載入圖片、CSS 與 JavaScript。
- 對外部圖片或資源先確認授權與穩定性，不要把別人網站當成自己的圖床。

> 待確認：原始資料未提供實際專案的最終資料夾規範；若本 Knowledge Pack 後續有統一目錄命名規則，可再補充到本章。
