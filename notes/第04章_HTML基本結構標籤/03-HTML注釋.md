# HTML 注釋

## 學習目標

讀完這篇筆記，你應該能夠：

- 看懂 HTML 注釋的語法。
- 知道注釋不會顯示在頁面內容中。
- 理解什麼時候適合使用注釋。
- 避免把重要內容只放在注釋裡。

## 問題情境

寫 HTML 時，程式碼可能會越來越長。當你隔一段時間再回來看，可能會忘記某段結構的用途；其他人接手你的檔案時，也需要快速理解頁面區塊。

注釋就是寫給開發者看的說明文字。它可以幫助閱讀與維護程式碼，但不會成為頁面上顯示的內容。

## 一句話理解

HTML 注釋是寫在程式碼中的備註，瀏覽器會忽略它，不會把它顯示成頁面內容。

## 基本語法

HTML 注釋以 `<!--` 開頭，以 `-->` 結束：

```html
<!-- 這是一段 HTML 注釋 -->
```

也可以用來標示頁面區塊：

```html
<!-- 主要內容開始 -->
<main>
  <h1>文章標題</h1>
  <p>文章內容。</p>
</main>
<!-- 主要內容結束 -->
```

注釋中的內容是給開發者看的，瀏覽器不會把它顯示在頁面中。

## 什麼時候使用注釋

適合使用注釋的情況：

- 標示較長 HTML 結構的開始與結束。
- 解釋某段結構存在的原因。
- 暫時保留一段尚未使用的 HTML 程式碼。
- 提醒自己或團隊某段程式碼需要後續調整。

例如：

```html
<!-- TODO: 之後補上文章分類導覽 -->
<nav>
  <a href="/articles">全部文章</a>
</nav>
```

## 常見錯誤

### 注釋結尾沒有寫完整

錯誤寫法：

```html
<!-- 主要內容開始
<main>
  <h1>文章標題</h1>
</main>
```

正確寫法：

```html
<!-- 主要內容開始 -->
<main>
  <h1>文章標題</h1>
</main>
```

注釋沒有正確結束時，後面的 HTML 可能會被瀏覽器當成注釋內容。

### 把要顯示給使用者看的內容放進注釋

錯誤寫法：

```html
<!-- 歡迎來到我的網站 -->
```

正確寫法：

```html
<p>歡迎來到我的網站</p>
```

注釋不會顯示在頁面上。如果內容是要給使用者看的，就應該使用合適的 HTML 標籤。

## 實務注意事項

注釋應該幫助理解程式碼，而不是重複描述每一行在做什麼。

不太有價值的注釋：

```html
<!-- 這是一個 h1 -->
<h1>網站標題</h1>
```

比較有價值的注釋：

```html
<!-- 首頁主標題：配合 SEO，每頁只保留一個 h1 -->
<h1>網站標題</h1>
```

此外，注釋雖然不會顯示在頁面畫面中，但仍可能出現在前端原始碼裡，因此不要把密碼、金鑰、私人資料或不該公開的資訊寫進 HTML 注釋。

## 重點整理

- HTML 注釋使用 `<!-- ... -->`。
- 注釋是給開發者看的，不會顯示成頁面內容。
- 注釋可以幫助標示結構、說明原因或留下待辦提醒。
- 注釋必須正確結束，否則可能影響後續 HTML 解析。
- 不要把敏感資訊寫在 HTML 注釋中。
