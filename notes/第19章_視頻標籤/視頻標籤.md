# HTML 視頻標籤 video

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `<video>` 標籤在 HTML 中的用途。
- 使用 `src`、`controls`、`autoplay`、`muted`、`loop`、`poster`、`preload`、`width`、`height` 等常見屬性。
- 理解為什麼需要搭配 `<source>` 提供多種影片格式。
- 說明瀏覽器對影片格式與自動播放行為可能存在差異。
- 避免影片無法播放、缺少控制列、尺寸比例錯誤與過度依賴自動播放等常見問題。

## 2. 本章定位

本章介紹 HTML 中用來嵌入影片內容的 `<video>` 元素。

在網頁中，影片常用於課程示範、產品展示、操作教學、短片播放或背景影片。`<video>` 的作用是讓瀏覽器直接在頁面中載入影片資源，並提供播放、暫停、音量、進度控制等能力。

學習 `<video>` 時，不只要知道標籤怎麼寫，也要理解三件事：

- 影片檔案必須能被瀏覽器找到。
- 不同瀏覽器與環境對影片格式、編碼與播放策略可能不同。
- 自動播放通常會受到瀏覽器政策限制，不能假設一定成功。

> 來源重點：原始資料指出 `<video>` 可用來在頁面中插入視頻，常見格式包含 `.mp4`、`.webm`、`.ogg`，並提醒不同瀏覽器的支援情況可能不同。

## 3. 核心概念

`<video>` 是 HTML 的多媒體元素，用來在網頁中嵌入可播放的影片。

最簡單的寫法是直接在 `<video>` 上使用 `src` 指定影片檔案：

```html
<video src="./assets/files/movie.mp4" controls></video>
```

其中：

| 部分 | 說明 |
|---|---|
| `<video>` | 建立影片播放器 |
| `src` | 指定影片檔案位置 |
| `controls` | 顯示瀏覽器內建播放控制列 |

如果沒有 `controls`，使用者通常看不到瀏覽器提供的播放、暫停、音量與進度控制。除非你打算用 JavaScript 自行製作播放器介面，否則初學階段應先加上 `controls`。

> 補充說明：`<video>` 描述的是「頁面中有一段可播放的影片內容」。實際控制列外觀由瀏覽器決定，也可以再透過 JavaScript 控制播放行為。

## 4. 語法與基本用法

### 4.1 基本語法

```html
<video src="影片檔案路徑" controls></video>
```

常見屬性如下：

| 屬性 | 說明 | 常見使用情境 |
|---|---|---|
| `src` | 指定影片檔案路徑 | 只有一個影片來源時使用 |
| `controls` | 顯示瀏覽器內建控制列 | 讓使用者能播放、暫停、調整音量與拖曳進度 |
| `poster` | 指定影片播放前顯示的封面圖片 | 影片尚未播放或尚未載入時顯示預覽圖 |
| `preload` | 提示瀏覽器是否預先載入影片資料 | 控制載入策略與效能 |
| `autoplay` | 嘗試在頁面載入後自動播放 | 常受瀏覽器政策限制，不應依賴 |
| `muted` | 預設靜音播放 | 常與 `autoplay` 搭配，降低被阻擋的機率 |
| `loop` | 播放結束後自動重新播放 | 背景影片、展示影片或循環短片 |
| `width` | 設定影片顯示寬度 | 控制影片在頁面中的呈現尺寸 |
| `height` | 設定影片顯示高度 | 控制影片在頁面中的呈現尺寸 |

### 4.2 `controls`：顯示播放控制列

```html
<video src="./assets/files/movie.mp4" controls></video>
```

`controls` 是布林屬性，只要出現在標籤上就代表啟用。啟用後，瀏覽器會顯示內建影片控制介面。

### 4.3 `poster`：影片封面

```html
<video src="./assets/files/movie.mp4" poster="./assets/images/movie-cover.jpg" controls></video>
```

`poster` 用來指定影片尚未播放前顯示的圖片。若沒有設定 `poster`，瀏覽器可能顯示影片第一幀、空白畫面或瀏覽器預設樣式。

### 4.4 `preload`：預載提示

```html
<video src="./assets/files/movie.mp4" controls preload="metadata"></video>
```

`preload` 是給瀏覽器的載入提示，常見值包含：

| 值 | 說明 |
|---|---|
| `none` | 不主動預載影片 |
| `metadata` | 預載影片長度、尺寸等中繼資訊 |
| `auto` | 允許瀏覽器自行判斷是否預載較多影片資料 |

> 注意：`preload` 是提示，不是強制命令。瀏覽器可能根據網路狀況、裝置設定或自身策略調整實際載入行為。

### 4.5 `autoplay`、`muted` 與 `loop`

```html
<video src="./assets/files/movie.mp4" autoplay muted loop></video>
```

`autoplay` 表示希望影片在頁面載入後自動播放。不過，許多瀏覽器會限制有聲音的自動播放，尤其是在使用者尚未與頁面互動之前。

實務上，如果影片需要自動播放，通常會搭配 `muted`：

```html
<video src="./assets/files/movie.mp4" autoplay muted loop></video>
```

這種寫法常用於背景影片或展示影片。`loop` 會讓影片播放結束後重新開始。

> 注意：即使加上 `muted`，自動播放仍可能受到瀏覽器、裝置、省電模式或使用者設定影響。不要把 `autoplay` 視為保證成功的功能。

### 4.6 使用 `<source>` 提供多種格式

如果希望同一段影片能在更多瀏覽器中播放，可以在 `<video>` 裡放多個 `<source>`：

```html
<video controls width="640" height="360">
  <source src="./assets/files/movie.mp4" type="video/mp4">
  <source src="./assets/files/movie.webm" type="video/webm">
  您的瀏覽器不支援 video 標籤。
</video>
```

瀏覽器會依序檢查 `<source>`，選擇自己支援的格式播放。若都不支援，就會顯示 `<video>` 標籤內的備用文字。

| 屬性 | 說明 |
|---|---|
| `src` | 指定這個影片來源的檔案路徑 |
| `type` | 告訴瀏覽器檔案的 MIME type，幫助瀏覽器判斷是否支援 |

> 注意：原始資料強調 `<source>` 需要 `src` 與 `type`。實務上 `src` 是指定資源的關鍵；`type` 建議清楚填寫，讓瀏覽器更快判斷格式是否可用。

## 5. 範例說明

### 範例一：最基本的影片播放器

```html
<video src="./assets/files/movie.mp4" controls></video>
```

這個範例中：

- `src="./assets/files/movie.mp4"` 指向影片檔案。
- `controls` 顯示瀏覽器內建控制列。
- 使用者可以自行播放、暫停、調整音量與控制進度。

### 範例二：提供多種影片格式

```html
<video controls width="640" height="360">
  <source src="./assets/files/movie.mp4" type="video/mp4">
  <source src="./assets/files/movie.webm" type="video/webm">
  您的瀏覽器不支援 video 標籤。
</video>
```

這個寫法的目的，是讓瀏覽器能從多個影片來源中選擇可播放的版本。

如果瀏覽器支援 MP4，就可能播放第一個來源；如果不支援，會繼續檢查下一個 `<source>`。最後的文字是備用內容，通常給不支援 `<video>` 的環境顯示。

### 範例三：自動播放的靜音循環影片

```html
<video src="./assets/files/banner.mp4" autoplay muted loop playsinline></video>
```

這個範例常見於首頁背景影片或展示區塊：

- `autoplay` 嘗試自動播放。
- `muted` 預設靜音，降低自動播放被阻擋的機率。
- `loop` 播放完後重新開始。
- `playsinline` 提示行動裝置可在頁面內播放，而不是強制進入全螢幕播放。

> 補充說明：`playsinline` 未出現在原始資料中，但在行動裝置背景影片或嵌入式影片場景中很常見，適合作為實務補充。

### 錯誤範例：依賴有聲音的自動播放

```html
<video src="./assets/files/intro.mp4" autoplay></video>
```

問題：瀏覽器可能阻擋有聲音的自動播放，而且沒有 `controls` 時，使用者可能無法操作影片。

### 正確範例：提供控制列，讓使用者主動播放

```html
<video src="./assets/files/intro.mp4" controls></video>
```

修正點：使用 `controls` 讓播放行為交給使用者決定，體驗更穩定，也比較不容易違反瀏覽器的自動播放限制。

## 6. 實務應用場景

`<video>` 常用在以下情境：

- 課程網站播放教學影片。
- 產品頁展示操作流程或商品特色。
- 文章頁嵌入示範短片。
- 首頁或活動頁使用靜音背景影片。
- 後台系統播放上傳後的影片預覽。

在實務開發中，使用 `<video>` 時通常還要注意：

- 影片檔案路徑是否正確。
- 檔案大小是否會影響頁面載入速度。
- 是否需要提供多種格式提升相容性。
- 是否需要封面圖，避免影片載入前畫面空白。
- 是否需要字幕、逐字稿或文字摘要，讓無法觀看或聽到影片的使用者也能理解內容。
- 是否需要用 JavaScript 控制播放、暫停、進度、音量或全螢幕。

> 補充說明：如果影片承載重要資訊，建議提供字幕、逐字稿或文字摘要。這不只是使用體驗問題，也和可訪問性有關。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 影片路徑寫錯 | 瀏覽器找不到檔案，播放器無法播放 | 確認 `src` 路徑與影片檔案位置一致 |
| 忘記加 `controls` | 使用者看不到內建播放介面 | 沒有自訂播放器時，應加上 `controls` |
| 只提供單一格式 | 某些瀏覽器或環境可能不支援該格式或編碼 | 使用多個 `<source>` 提供替代格式 |
| 過度依賴 `autoplay` | 瀏覽器可能阻擋有聲音的自動播放 | 讓使用者主動播放，或搭配 `muted` 設計靜音自動播放 |
| 背景影片無法停止 | 使用者缺少控制權，體驗不佳 | 提供控制方式，或避免把重要內容只放在背景影片 |
| 沒有設定 `poster` | 影片載入前可能顯示空白或不理想畫面 | 使用 `poster` 提供清楚封面圖 |
| `width` 與 `height` 比例不一致 | 影片畫面可能被拉伸或壓縮 | 使用符合影片比例的尺寸，或交給 CSS 維持比例 |
| MIME type 寫錯 | 瀏覽器可能無法正確判斷影片格式 | 依檔案格式填寫正確的 `type` |
| 把影片當成唯一資訊來源 | 無法播放、無法聽音訊或使用輔助工具的使用者會失去資訊 | 提供字幕、逐字稿、摘要或替代說明 |

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| `<audio>` | 兩者都是 HTML 多媒體元素，常見屬性如 `controls`、`autoplay`、`loop` 類似 |
| `<source>` | 可放在 `<video>` 中，提供多個候選影片檔案 |
| `<track>` | 可搭配 `<video>` 提供字幕、章節或其他文字軌道 |
| JavaScript | 可透過 DOM API 控制播放、暫停、音量、進度與全螢幕 |
| CSS | 可控制影片在版面中的尺寸、比例、定位與響應式呈現 |
| 可訪問性 | 重要影片內容應提供字幕、逐字稿或文字替代 |
| 檔案路徑 | `src` 必須指向正確資源，與圖片、音訊等外部資源載入觀念相同 |

## 9. 重點整理

- `<video>` 用來在 HTML 頁面中嵌入影片內容。
- `src` 指定影片檔案，`controls` 顯示瀏覽器內建播放控制列。
- 常見影片格式包含 MP4、WebM、Ogg，但實際支援要看瀏覽器、影片編碼與伺服器設定。
- 使用多個 `<source>` 可以提供不同格式，增加播放成功的機會。
- `poster` 可以提供影片播放前的封面畫面。
- `autoplay` 不保證成功，實務上應優先讓使用者主動播放；若需要自動播放，通常會搭配 `muted`。
- 若影片包含重要資訊，應提供字幕、逐字稿或摘要，避免只靠影音傳達。

## 11. 延伸補充

### 11.1 影片格式與瀏覽器支援

原始資料提到 MP4、WebM、Ogg 是常見影片格式。正式開發時，不應只根據副檔名判斷能否播放，還要考慮瀏覽器支援、影片編碼、音訊編碼與伺服器回傳的 MIME type。

> 待確認：若教材需要列出各瀏覽器對 MP4、WebM、Ogg 的精確支援版本，應查證目標瀏覽器版本後再補充。

### 11.2 與 `<audio>` 的相似處

`<audio>` 和 `<video>` 的使用方式很接近，兩者都可以使用 `controls`、`autoplay`、`loop` 與 `<source>`。

差別在於：

- `<audio>` 只處理聲音。
- `<video>` 同時處理影像與聲音。
- `<video>` 常需要額外考慮尺寸、封面圖、字幕、畫面比例與行動裝置播放方式。

因此，學會 `<audio>` 後，再學 `<video>` 會更容易理解。

### 11.3 HTML、DOM 與影片控制

瀏覽器會把 `<video>` 解析成 DOM 中的影片元素。JavaScript 可以透過 DOM API 操作它，例如呼叫 `play()`、`pause()`，或讀取目前播放進度。

這代表 `<video>` 不只是靜態標籤，也可以成為互動播放器的一部分。不過，正式筆記主線應先掌握 HTML 標籤與屬性，再進一步學 JavaScript 控制。
