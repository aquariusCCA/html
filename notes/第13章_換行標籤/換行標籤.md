# HTML 換行標籤 br

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `<br>` 標籤的用途與語意。
- 正確使用 `<br>` 讓文字在指定位置換行。
- 分辨 `<br>` 與段落元素 `<p>` 的差異。
- 避免把 `<br>` 當成排版或增加間距的工具。

## 2. 本章定位

本章介紹 HTML 中用來強制換行的 `<br>` 元素。

在一般文字內容中，瀏覽器會依照容器寬度自動換行；只有當換行本身是內容的一部分時，才適合使用 `<br>`。因此，學習 `<br>` 的重點不只是知道它會換行，也要理解它適合用在哪些情境，以及什麼情況應該改用段落或 CSS。

## 3. 核心概念

`<br>` 是 HTML 的換行元素，語意是 line break，也就是在目前文字流中插入一個換行位置。

原始資料提到，HTML 中一個段落的文字通常會從左到右排列，直到瀏覽器視窗或容器右側才自動換行。如果希望某段文字在指定位置強制換行，就可以使用 `<br>`。

`br` 來自英文單字 `break`，有「打斷、換行」的意思。

> 補充說明：在 HTML 中，`<br>` 是 void element（空元素），不需要結束標籤。常見的 `<br />` 寫法也能被 HTML 解析，但在 HTML 文件中通常寫成 `<br>` 即可。

## 4. 語法與基本用法

基本語法如下：

```html
第一行文字<br>
第二行文字
```

也常看到以下 XHTML 風格寫法：

```html
第一行文字 <br />第二行文字
```

這兩種寫法在一般 HTML 頁面中都會產生換行效果。重點是 `<br>` 只是在文字中開始新的一行，不會像 `<p>` 段落元素一樣自動產生段落間距。

| 寫法 | 說明 |
|---|---|
| `<br>` | HTML 中常見且簡潔的寫法 |
| `<br />` | XHTML 風格寫法，在 HTML 中通常也能被瀏覽器解析 |
| `</br>` | 不需要使用，因為 `<br>` 沒有結束標籤 |

## 5. 範例說明

### 範例：在文字中強制換行

```html
<p>
  第一行文本<br>
  第二行文本
</p>
```

這個範例中：

- `<p>` 表示這是一個段落。
- `<br>` 讓「第一行文本」後方立即換到下一行。
- 「第二行文本」仍然屬於同一個段落內容。

### 範例：地址或詩句中的換行

```html
<p>
  台北市中正區<br>
  重慶南路一段 122 號<br>
  3 樓
</p>
```

地址、詩句、歌詞這類內容，換行本身可能就是內容結構的一部分，這時使用 `<br>` 是合理的。

## 6. 實務應用場景

`<br>` 適合用在「同一段內容中需要保留換行語意」的場景，例如：

- 地址每一行需要分開顯示。
- 詩句、歌詞或短句需要保留原本的行分隔。
- 表單說明文字中有少量固定換行。

但如果目的是讓兩個區塊之間有距離，或只是想控制畫面排版，就不應該依賴 `<br>`。這類視覺間距應該交給 CSS，例如使用 `margin`、`padding` 或合適的版面配置方式。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 連續使用多個 `<br>` 製造垂直空白 | `<br>` 是內容換行，不是排版間距工具 | 使用 CSS 的 `margin` 或 `padding` 控制間距 |
| 用 `<br>` 分隔不同段落 | 段落應該使用 `<p>` 表示語意 | 每個段落使用獨立的 `<p>` |
| 寫成 `<br></br>` | `<br>` 是空元素，不需要結束標籤 | 寫成 `<br>` 或 `<br />` |
| 把所有文字排版都交給 `<br>` | 會讓 HTML 結構難維護，也不利於響應式排版 | 使用語意元素建立結構，再用 CSS 控制畫面 |

### 錯誤範例：用 `<br>` 製造段落間距

```html
這是第一段文字。<br><br>
這是第二段文字。
```

問題：這兩段文字其實是不同段落，但程式碼沒有用段落語意，只是用換行製造視覺效果。

### 正確範例：使用 `<p>` 表示段落

```html
<p>這是第一段文字。</p>
<p>這是第二段文字。</p>
```

修正重點：段落是內容結構，應該使用 `<p>`；段落之間的距離可以再用 CSS 調整。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| `<p>` | `<p>` 表示段落，適合包住一段完整文字；`<br>` 只是在同一段內容中換行 |
| CSS | 視覺間距與版面配置應使用 CSS 控制，不應大量依賴 `<br>` |
| DOM | 瀏覽器會把 `<br>` 解析成 DOM 中的換行元素節點 |
| 可訪問性 | 過度使用 `<br>` 可能讓輔助工具讀取內容時結構不清楚 |

## 9. 重點整理

- `<br>` 的語意是強制換行，來自英文 `break`。
- `<br>` 是空元素，不需要結束標籤。
- `<br>` 只會開始新的一行，不等於建立新段落。
- 段落應使用 `<p>`，版面間距應使用 CSS。
- 適合使用 `<br>` 的情境通常是地址、詩句、歌詞等換行本身有意義的內容。

## 11. 延伸補充

`<br>` 是一個簡單但容易被誤用的元素。它適合處理文字內容中的換行，不適合承擔整體排版責任。

> 補充說明：HTML 負責描述內容與結構，CSS 負責視覺呈現。當需求是「內容本身需要換行」時，可以考慮 `<br>`；當需求是「畫面看起來要有距離或排列效果」時，通常應該使用 CSS。
