# HTML 表單標籤

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 HTML 表單的用途，以及表單如何收集使用者資料。
- 使用 `<form>` 建立表單域，並理解 `action`、`method`、`name`、`target` 的作用。
- 使用 `<input>` 建立文字框、密碼框、單選框、複選框、隱藏欄位、檔案欄位與按鈕。
- 理解表單提交時 `name` 與 `value` 如何組成資料。
- 使用 `<label>` 關聯表單控制項，改善使用體驗與可訪問性。
- 使用 `<select>`、`<option>`、`<textarea>`、`<button>` 建立常見表單介面。
- 使用 `disabled`、`required`、`placeholder`、`autofocus`、`autocomplete` 等常見屬性。
- 使用 `<fieldset>` 與 `<legend>` 為表單控制項分組。
- 避免按鈕預設提交、表單欄位缺少 `name`、禁用表單方式錯誤等常見問題。

## 2. 本章定位

表單是 HTML 中用來收集使用者輸入的重要機制。登入、註冊、搜尋、留言、下單、上傳檔案、篩選資料等功能，通常都需要表單或表單控制項參與。

在 HTML 學習路線中，表單是從「展示內容」進入「與使用者互動」的重要章節。它不只涉及標籤外觀，也涉及資料如何被命名、保存、驗證與提交。

> 來源重點：原始資料將完整表單整理為三個部分：表單域、表單控制項與提示資訊。表單的主要目的，是收集使用者資料並提交給伺服器。

表單資料的基本流程可以理解為：

```text
使用者輸入資料 -> 表單控制項保存值 -> 透過 name/value 組成提交資料 -> 送到後端或交給 JavaScript 處理
```

## 3. 核心概念

HTML 表單由表單容器、表單控制項與提示文字共同組成。

| 組成部分 | 常見標籤 | 作用 |
|---|---|---|
| 表單域 | `<form>` | 定義表單範圍與提交設定 |
| 表單控制項 | `<input>`、`<select>`、`<textarea>`、`<button>` | 讓使用者輸入、選擇或觸發操作 |
| 提示資訊 | `<label>`、文字、說明、錯誤訊息 | 告訴使用者欄位用途與填寫要求 |

### 3.1 `<form>` 定義表單域

`<form>` 是表單的容器，用來包住一組要一起提交的表單控制項。

瀏覽器提交表單時，會收集 `<form>` 範圍內可提交控制項的資料，並依照 `action` 與 `method` 等屬性送出。

```html
<form action="/register" method="post">
  <label for="username">使用者名稱</label>
  <input type="text" id="username" name="username">

  <button type="submit">註冊</button>
</form>
```

### 3.2 表單控制項負責保存使用者輸入

表單控制項是使用者真正互動的元素。例如：

- `<input type="text">`：單行文字輸入。
- `<input type="password">`：密碼輸入。
- `<input type="radio">`：單選。
- `<input type="checkbox">`：複選。
- `<select>`：下拉選單。
- `<textarea>`：多行文字輸入。
- `<button>`：按鈕。

表單控制項不只是畫面上的輸入框，它們還會持有使用者輸入的值，並在提交時依照 `name` 與 `value` 形成資料。

### 3.3 `name` 與 `value` 是表單資料的核心

表單提交時，瀏覽器會把欄位整理成一組組 `name=value`。

```html
<input type="text" name="username" value="alex">
```

提交時可理解為：

```text
username=alex
```

因此：

| 屬性 | 意義 |
|---|---|
| `name` | 欄位名稱，後端或 JavaScript 會用它辨識這筆資料 |
| `value` | 欄位值，也就是實際提交的資料 |

> 注意：如果表單控制項沒有 `name`，通常不會形成有效提交欄位。初學者常以為只要畫面上有輸入框就會提交，但後端真正收到的是 `name/value` 資料。

### 3.4 提示文字應與表單控制項建立關聯

`<label>` 用來描述某個表單控制項。正確使用 label 可以讓使用者點擊文字時聚焦或選取對應欄位，也能幫助螢幕閱讀器理解欄位用途。

```html
<label for="email">電子信箱</label>
<input type="email" id="email" name="email">
```

`for` 的值應該對應控制項的 `id`。

## 4. 語法與基本用法

### 4.1 `<form>` 常用屬性

```html
<form action="/submit" method="post" name="registerForm" target="_self">
  <!-- 表單控制項 -->
</form>
```

| 屬性 | 說明 |
|---|---|
| `action` | 表單資料要提交到的 URL |
| `method` | 提交方式，常見為 `get` 或 `post` |
| `name` | 表單名稱，較常用於識別表單 |
| `target` | 指定提交後回應頁面的開啟位置 |

`method="get"` 通常會把資料放在 URL query string 中，適合搜尋或篩選這類可被分享、可被重新整理的請求。`method="post"` 通常把資料放在請求本文中，較常用於註冊、登入、下單、留言等會送出內容或改變伺服器資料的情境。

> 補充說明：無論前端是否使用 HTML 表單提交，後端仍必須檢查資料。HTML 表單驗證可以改善使用體驗，但不能取代後端驗證。

### 4.2 `<input>` 基本語法

`<input>` 是最常見的表單控制項，透過 `type` 屬性決定輸入型態。

```html
<input type="text" name="username" placeholder="請輸入使用者名稱">
```

常見 `input` 類型如下：

| `type` 值 | 用途 | 常見情境 |
|---|---|---|
| `text` | 單行文字輸入 | 使用者名稱、暱稱 |
| `password` | 密碼輸入，輸入內容會被遮蔽 | 登入密碼 |
| `radio` | 單選 | 性別、方案、單一選項 |
| `checkbox` | 複選 | 興趣、同意條款、多項篩選 |
| `hidden` | 隱藏欄位 | 攜帶固定資料或狀態值 |
| `file` | 檔案選擇 | 上傳圖片、文件 |
| `submit` | 提交按鈕 | 送出表單 |
| `reset` | 重置按鈕 | 還原表單初始值 |
| `button` | 普通按鈕 | 搭配 JavaScript 觸發操作 |

### 4.3 文字框與密碼框

文字框適合輸入單行文字；密碼框適合輸入密碼。

```html
<label for="username">使用者名稱</label>
<input
  type="text"
  id="username"
  name="username"
  placeholder="請輸入使用者名稱"
  maxlength="20"
>

<label for="password">密碼</label>
<input
  type="password"
  id="password"
  name="password"
  maxlength="30"
>
```

常用屬性：

| 屬性 | 說明 |
|---|---|
| `name` | 提交時的欄位名稱 |
| `value` | 欄位預設值或目前值 |
| `placeholder` | 輸入前顯示的提示文字 |
| `maxlength` | 可輸入的最大字元數 |

> 注意：`placeholder` 不是 label。它會在使用者輸入後消失，不應取代欄位名稱或必要說明。

### 4.4 單選框與複選框

單選框用於「多選一」，同一組 radio 必須使用相同的 `name`。

```html
<p>通知方式</p>

<input type="radio" id="notice-email" name="notice" value="email" checked>
<label for="notice-email">電子信箱</label>

<input type="radio" id="notice-sms" name="notice" value="sms">
<label for="notice-sms">簡訊</label>
```

複選框用於「可選多個」。

```html
<p>感興趣的主題</p>

<input type="checkbox" id="topic-html" name="topics" value="html">
<label for="topic-html">HTML</label>

<input type="checkbox" id="topic-css" name="topics" value="css" checked>
<label for="topic-css">CSS</label>

<input type="checkbox" id="topic-js" name="topics" value="javascript">
<label for="topic-js">JavaScript</label>
```

| 屬性 | 適用 | 說明 |
|---|---|---|
| `name` | radio、checkbox | 欄位名稱；同組 radio 要使用相同 `name` |
| `value` | radio、checkbox | 被選取時提交的值 |
| `checked` | radio、checkbox | 預設選取 |

> 注意：checkbox 若沒有設定 `value`，瀏覽器仍可能提交預設值，但實務上建議明確指定 `value`，讓後端資料更清楚。

### 4.5 隱藏欄位與檔案欄位

隱藏欄位不會顯示在畫面上，常用來攜帶固定資料。

```html
<input type="hidden" name="source" value="newsletter">
```

檔案欄位用於讓使用者選擇本機檔案。

```html
<label for="avatar">上傳頭像</label>
<input type="file" id="avatar" name="avatar">

<label for="photos">上傳多張照片</label>
<input type="file" id="photos" name="photos" multiple>
```

`multiple` 可以讓支援的檔案欄位選擇多個檔案。

如果使用瀏覽器的傳統表單提交檔案，`<form>` 通常需要設定 `enctype="multipart/form-data"`。

```html
<form action="/upload" method="post" enctype="multipart/form-data">
  <label for="document">上傳文件</label>
  <input type="file" id="document" name="document">
  <button type="submit">上傳</button>
</form>
```

> 注意：隱藏欄位只是畫面不可見，不代表安全。使用者仍可能透過開發者工具修改它，因此後端不能信任隱藏欄位中的敏感資料。

### 4.6 新增的常見輸入類型

HTML 提供多種更貼近資料型態的 `input` 類型。它們可以讓瀏覽器提供更合適的輸入介面，也能搭配原生驗證。

```html
<form action="/profile" method="post">
  <label for="email">電子信箱</label>
  <input type="email" id="email" name="email" required>

  <label for="website">個人網站</label>
  <input type="url" id="website" name="website">

  <label for="birth-date">生日</label>
  <input type="date" id="birth-date" name="birthDate">

  <label for="favorite-color">喜歡的顏色</label>
  <input type="color" id="favorite-color" name="favoriteColor">

  <button type="submit">送出</button>
</form>
```

常見類型如下：

| `type` 值 | 用途 |
|---|---|
| `email` | 電子信箱 |
| `url` | 網址 |
| `date` | 日期 |
| `time` | 時間 |
| `number` | 數字 |
| `tel` | 電話號碼 |
| `search` | 搜尋欄位 |
| `color` | 顏色選擇 |

> 注意：不同瀏覽器或裝置對這些輸入類型的介面呈現可能不同。例如手機可能會針對 `email`、`tel` 顯示不同鍵盤。不要只依賴前端輸入類型判斷資料一定正確。

### 4.7 `<label>` 的兩種關聯方式

方式一：使用 `for` 對應控制項 `id`。

```html
<input type="radio" id="gender-female" name="gender" value="female">
<label for="gender-female">女</label>
```

方式二：把控制項放在 `<label>` 裡面。

```html
<label>
  <input type="radio" name="gender" value="male">
  男
</label>
```

兩種方式都能建立 label 與控制項的關係。實務上，`for` 搭配 `id` 的方式在版面配置上更彈性，也常見於需要把 label 與控制項分開排版的情境。

### 4.8 `<select>` 與 `<option>`

`<select>` 用來建立下拉選單，適合選項較多且希望節省畫面空間的情境。

```html
<label for="city">城市</label>
<select id="city" name="city">
  <option value="">請選擇城市</option>
  <option value="taipei">台北</option>
  <option value="taichung">台中</option>
  <option value="kaohsiung" selected>高雄</option>
</select>
```

注意事項：

- `<select>` 中至少應包含一個 `<option>`。
- `<select>` 的 `name` 是提交資料的欄位名稱。
- `<option>` 的 `value` 是該選項被選中時提交的值。
- 如果 `<option>` 沒有 `value`，提交值通常會使用 option 文字內容。
- `selected` 可設定預設選取項目。

### 4.9 `<textarea>` 多行文字輸入

`<textarea>` 用於輸入較長文字，例如留言、評論、備註。

```html
<label for="message">留言內容</label>
<textarea id="message" name="message" rows="5" cols="40">預設文字</textarea>
```

| 屬性 | 說明 |
|---|---|
| `name` | 提交時的欄位名稱 |
| `rows` | 預設顯示行數 |
| `cols` | 預設顯示欄數 |

> 注意：`<textarea>` 不是 `<input>`，不能寫 `type` 屬性。它的預設文字要放在開始標籤與結束標籤之間。

實務上，`rows` 可用來提供基本高度，但寬高通常會交給 CSS 控制。

```css
textarea {
  width: 100%;
  min-height: 120px;
  resize: vertical;
}
```

### 4.10 `<button>` 按鈕標籤

`<button>` 可以建立提交、重置或普通按鈕。

```html
<button type="submit">提交表單</button>
<button type="reset">重置表單</button>
<button type="button">取得驗證碼</button>
```

| `type` 值 | 作用 |
|---|---|
| `submit` | 提交表單 |
| `reset` | 重置表單到初始狀態 |
| `button` | 普通按鈕，通常搭配 JavaScript |

> 注意：`<button>` 在表單內若沒有指定 `type`，預設行為是 `submit`。若只是一般操作按鈕，應明確寫 `type="button"`，避免誤送出表單。

`<button>` 與 `<input type="button">` 都可以建立按鈕，但 `<button>` 可以包含文字與其他行內內容，可讀性與彈性通常較好。

### 4.11 常見表單屬性

```html
<form action="/search" method="get">
  <label for="keyword">搜尋</label>
  <input
    type="search"
    id="keyword"
    name="keyword"
    required
    placeholder="請輸入關鍵字"
    autofocus
    autocomplete="off"
  >

  <button type="submit">搜尋</button>
</form>
```

| 屬性 | 說明 |
|---|---|
| `required` | 必填欄位，提交前瀏覽器會做基本檢查 |
| `placeholder` | 輸入前顯示的提示文字 |
| `autofocus` | 頁面載入後自動聚焦到該欄位 |
| `autocomplete` | 控制瀏覽器是否提供自動完成建議 |
| `maxlength` | 限制最大輸入字元數 |
| `multiple` | 允許多選，常見於檔案或 email 欄位 |
| `disabled` | 禁用控制項 |
| `readonly` | 讓文字類型欄位唯讀，但仍可聚焦且通常會提交 |

`placeholder` 的顏色可以透過 CSS 偽元素調整：

```css
input::placeholder {
  color: #777;
}
```

> 注意：原始資料中的 `autocommplete` 是拼字錯誤，正式屬性名稱應為 `autocomplete`。

### 4.12 禁用表單控制項

`disabled` 可以讓表單控制項不可操作。

```html
<input type="text" name="account" value="guest" disabled>
<button type="submit" disabled>送出中</button>
<select name="plan" disabled>
  <option value="basic">Basic</option>
  <option value="pro">Pro</option>
</select>
<textarea name="memo" disabled>不可編輯</textarea>
```

可設定 `disabled` 的常見元素包括：

- `<input>`
- `<textarea>`
- `<button>`
- `<select>`
- `<option>`
- `<fieldset>`

如果要一次禁用一組表單控制項，應使用 `<fieldset disabled>`。

```html
<fieldset disabled>
  <legend>帳號資訊</legend>

  <label for="account">帳號</label>
  <input type="text" id="account" name="account" value="demo">

  <button type="button">檢查帳號</button>
</fieldset>
```

> 注意：原始資料提到在 `<form>` 上設定 `disabled` 來禁用整個表單，但 `<form disabled>` 不是有效的 HTML 禁用方式。若要禁用一組控制項，應使用 `<fieldset disabled>`，或分別對控制項設定 `disabled`。

`disabled` 與 `readonly` 的差異：

| 屬性 | 可否操作 | 是否通常會提交 | 常見用途 |
|---|---|---|---|
| `disabled` | 不可操作 | 不會提交 | 暫時停用欄位或按鈕 |
| `readonly` | 不可編輯，但可聚焦 | 會提交 | 顯示不可修改但仍需提交的值 |

### 4.13 `<fieldset>` 與 `<legend>` 分組

`<fieldset>` 可以為一組相關表單控制項分組，`<legend>` 則是該分組的標題。

```html
<form action="/register" method="post">
  <fieldset>
    <legend>帳號資訊</legend>

    <label for="account">帳號</label>
    <input type="text" id="account" name="account" maxlength="20">

    <label for="password">密碼</label>
    <input type="password" id="password" name="password" maxlength="30">
  </fieldset>

  <fieldset>
    <legend>通知設定</legend>

    <input type="radio" id="notify-email" name="notify" value="email" checked>
    <label for="notify-email">電子信箱</label>

    <input type="radio" id="notify-sms" name="notify" value="sms">
    <label for="notify-sms">簡訊</label>
  </fieldset>

  <button type="submit">提交</button>
</form>
```

這種分組方式能讓表單結構更清楚，也有助於輔助工具告訴使用者目前欄位屬於哪個區塊。

## 5. 範例說明

### 範例一：註冊表單

```html
<form action="/register" method="post">
  <fieldset>
    <legend>基本資料</legend>

    <label for="username">使用者名稱</label>
    <input
      type="text"
      id="username"
      name="username"
      required
      maxlength="20"
      placeholder="請輸入使用者名稱"
    >

    <label for="email">電子信箱</label>
    <input
      type="email"
      id="email"
      name="email"
      required
      autocomplete="email"
    >

    <label for="password">密碼</label>
    <input
      type="password"
      id="password"
      name="password"
      required
      maxlength="30"
    >
  </fieldset>

  <fieldset>
    <legend>偏好設定</legend>

    <label for="city">所在城市</label>
    <select id="city" name="city">
      <option value="">請選擇</option>
      <option value="taipei">台北</option>
      <option value="taichung">台中</option>
      <option value="kaohsiung">高雄</option>
    </select>

    <p>感興趣的主題</p>
    <input type="checkbox" id="interest-html" name="interests" value="html">
    <label for="interest-html">HTML</label>

    <input type="checkbox" id="interest-css" name="interests" value="css">
    <label for="interest-css">CSS</label>
  </fieldset>

  <label for="bio">自我介紹</label>
  <textarea id="bio" name="bio" rows="5"></textarea>

  <input type="hidden" name="source" value="course-page">

  <button type="submit">註冊</button>
  <button type="reset">重新填寫</button>
</form>
```

這個範例中：

- `<form>` 定義要提交的資料範圍。
- `action="/register"` 指定提交目的地。
- `method="post"` 表示用 POST 方式提交資料。
- 每個需要提交的控制項都設定了 `name`。
- `<label>` 透過 `for` 與控制項的 `id` 建立關聯。
- `<fieldset>` 與 `<legend>` 將表單分成「基本資料」與「偏好設定」。
- `required` 讓瀏覽器在提交前檢查必填欄位。
- `type="email"` 讓瀏覽器對電子信箱格式做基本檢查。
- `type="hidden"` 用於提交畫面上不顯示的固定資料。
- `<button type="submit">` 明確表示提交表單。

### 範例二：搜尋表單

```html
<form action="/search" method="get">
  <label for="q">搜尋關鍵字</label>
  <input
    type="search"
    id="q"
    name="q"
    placeholder="搜尋文章"
    autocomplete="off"
  >

  <button type="submit">搜尋</button>
  <button type="button" id="clear-search">清除</button>
</form>
```

搜尋表單通常適合使用 `method="get"`，因為搜尋條件可以出現在網址中，方便分享或重新整理。

`清除` 按鈕若要透過 JavaScript 控制，應寫成 `type="button"`，避免它被當成提交按鈕。

## 6. 實務應用場景

HTML 表單常見於以下情境：

- 登入與註冊：輸入帳號、信箱、密碼。
- 搜尋與篩選：輸入關鍵字、選擇分類、勾選條件。
- 留言與評論：使用 `<textarea>` 輸入較長內容。
- 電商結帳：填寫收件人、地址、付款方式。
- 後台管理：新增、編輯、刪除資料時收集欄位值。
- 檔案上傳：使用 `<input type="file">` 選擇圖片或文件。

在實務開發中，即使使用 Vue、React 或其他框架，表單的本質仍然是處理使用者輸入。框架可能會接管提交流程，但 `name`、`value`、`label`、`type`、`disabled`、`required` 等 HTML 觀念仍然重要。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 表單控制項沒有 `name` | 提交時無法形成清楚的欄位資料 | 需要提交的欄位都設定有意義的 `name` |
| 用 `placeholder` 取代 `<label>` | placeholder 會消失，也不利於可訪問性 | 保留可見或可被輔助工具讀取的 label |
| 多個 radio 的 `name` 不一致 | 瀏覽器無法判斷它們屬於同一組 | 同一組 radio 使用相同 `name` |
| checkbox 沒有設定 `value` | 後端收到的資料不夠清楚 | 為每個 checkbox 明確設定 `value` |
| `<button>` 沒有寫 `type` | 表單內預設可能會提交表單 | 明確寫 `type="submit"`、`type="reset"` 或 `type="button"` |
| 用 `<form disabled>` 禁用表單 | `<form>` 沒有有效的 `disabled` 行為 | 使用 `<fieldset disabled>` 或個別控制項設定 `disabled` |
| 把前端驗證當成唯一驗證 | 使用者可以繞過前端限制 | 前端改善體驗，後端負責最終驗證 |
| 密碼欄位設定預設 `value` | 可能造成安全與體驗問題 | 密碼欄位通常不設定預設值 |
| 忘記 `disabled` 欄位不會提交 | 後端收不到被禁用欄位的值 | 若要不可編輯但仍提交，可考慮 `readonly` |

### 錯誤範例：普通按鈕沒有指定 `type`

```html
<form action="/send-code" method="post">
  <input type="tel" name="phone">
  <button>取得驗證碼</button>
</form>
```

問題：`button` 在表單中預設是 `submit`，點擊「取得驗證碼」可能直接提交表單。

### 正確範例

```html
<form action="/send-code" method="post">
  <input type="tel" name="phone">
  <button type="button">取得驗證碼</button>
  <button type="submit">送出</button>
</form>
```

修正點：普通操作按鈕明確使用 `type="button"`，真正送出表單的按鈕才使用 `type="submit"`。

### 錯誤範例：label 沒有關聯控制項

```html
<span>電子信箱</span>
<input type="email" name="email">
```

問題：文字只是普通內容，瀏覽器與輔助工具無法明確知道它描述哪個欄位。

### 正確範例

```html
<label for="email">電子信箱</label>
<input type="email" id="email" name="email">
```

修正點：使用 `label`，並讓 `for` 對應控制項的 `id`。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| CSS | CSS 負責控制表單外觀，例如間距、寬度、placeholder 顏色與錯誤狀態樣式 |
| JavaScript | JavaScript 可讀取表單值、攔截提交事件、動態啟用或禁用欄位 |
| DOM | 瀏覽器會把表單標籤解析成 DOM 節點，JavaScript 通常是操作這些節點 |
| 後端 | 表單提交後，後端依照欄位名稱接收資料並進行驗證、儲存或處理 |
| 可訪問性 | `<label>`、`fieldset`、`legend`、正確按鈕與輸入類型有助於輔助工具理解表單 |
| Vue | Vue 可使用 `v-model` 綁定表單值，但最終仍會渲染為 HTML 表單控制項 |

## 9. 重點整理

- 表單用來收集使用者資料，常見於登入、註冊、搜尋、留言與上傳。
- `<form>` 定義表單域，`action` 與 `method` 決定資料提交目的地與方式。
- 表單提交的核心是 `name/value`，沒有 `name` 的欄位通常不會成為有效提交資料。
- `<input>` 透過 `type` 決定控制項類型，例如 text、password、radio、checkbox、file。
- 同一組 radio 必須使用相同 `name`，才能形成單選效果。
- `<label>` 應與表單控制項建立關聯，改善點擊體驗與可訪問性。
- `<select>` 適合多選項下拉選單，`<textarea>` 適合多行文字輸入。
- `<button>` 在表單中預設是 submit，普通按鈕應明確寫 `type="button"`。
- `disabled` 控制項不可操作且不會提交；若要禁用一組控制項，可使用 `<fieldset disabled>`。
- HTML 表單驗證是前端體驗的一部分，後端仍必須做最終驗證。

## 11. 延伸補充

> 補充說明：HTML 表單可以直接由瀏覽器提交，也可以被 JavaScript 攔截後改用 `fetch` 或框架方法送出。無論哪種方式，良好的 HTML 結構、清楚的欄位名稱與正確的 label 關聯，都能降低維護成本並改善可訪問性。

在初學階段，可以先掌握三件事：

1. 使用正確的表單控制項表達資料型態。
2. 每個要提交的欄位都給清楚的 `name`。
3. 每個可見欄位都盡量搭配 `<label>`。

等熟悉基礎後，再進一步學習表單驗證、錯誤訊息顯示、檔案上傳、AJAX 提交，以及框架中的表單狀態管理。
