# 網頁與 HTML 入門

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明網頁、網站與 HTML 之間的關係。
- 理解 HTML 為什麼不是程式語言，而是標記語言。
- 說出「超文本」的兩個重要意思。
- 描述瀏覽器如何把 HTML 轉成使用者看到的網頁。

## 問題情境

打開瀏覽器時，你看到的可能是一篇文章、一個購物頁面、一段影片介紹，或是一個可以點擊跳轉的選單。

這些畫面不是憑空出現的。前端開發者會先寫出 HTML 等前端程式碼，瀏覽器再讀取這些檔案，把文字、圖片、音訊、影片與超連結組合成使用者看得懂、能操作的網頁。

所以學 HTML 的第一步，不是急著背標籤，而是先理解：網頁是什麼、HTML 負責什麼、瀏覽器又做了什麼。

## 一句話理解

HTML 是用來描述網頁內容與結構的標記語言；瀏覽器會解析 HTML，並把它呈現成使用者看到的網頁。

## 網頁與網站的關係

網頁是網站中的一頁，通常是一份 HTML 格式的文件，需要透過瀏覽器閱讀。

一個網站通常由多個網頁組成。例如一個學校網站可能包含：

- 首頁
- 最新消息頁
- 課程介紹頁
- 聯絡資訊頁

每一頁都是網站的一部分，而這些頁面彼此連結後，就形成完整的網站。

常見的網頁檔案副檔名是：

```text
.html
.htm
```

因此初學時常會把網頁檔案稱為 HTML 文件。

## 網頁裡通常有什麼

網頁不是只能放文字。常見的網頁元素包含：

| 元素 | 說明 |
| --- | --- |
| 文字 | 標題、段落、清單、說明文字 |
| 圖片 | 商品圖、插圖、頭像、示意圖 |
| 音訊 | 音樂、語音、課程錄音 |
| 影片 | 教學影片、產品展示影片 |
| 超連結 | 連到其他頁面、檔案或網站 |

HTML 的工作，就是用標籤描述這些內容的意義與結構。

## HTML 是什麼

HTML 是 HyperText Markup Language 的縮寫，中文通常翻成「超文本標記語言」。

它是用來描述網頁的一種語言，但它不是一般意義上的程式語言。

程式語言通常會處理運算、流程控制與邏輯，例如：

```text
如果使用者登入成功，就顯示會員頁面。
```

HTML 的重點則是標記內容，例如：

```html
<h1>網站標題</h1>
<p>這是一段網頁內容。</p>
<a href="about.html">關於我們</a>
```

這段 HTML 告訴瀏覽器：

| HTML | 意義 |
| --- | --- |
| `<h1>` | 這是最重要的標題 |
| `<p>` | 這是一段文字 |
| `<a>` | 這是一個連結 |

所以 HTML 比較像是在幫內容貼上「語意標籤」，讓瀏覽器知道每段內容代表什麼。

## 什麼是超文本

HTML 中的 HyperText 通常可以從兩個方向理解。

第一，超文本可以超越純文字限制。網頁不只顯示文字，也可以加入圖片、聲音、動畫、影片等多媒體內容。

第二，超文本可以透過超連結，把一份文件連到另一份文件。使用者點擊連結後，可以前往同一網站的其他頁面，也可以連到世界各地的網站或檔案。

這也是網頁和一般紙本文件最大的差異之一：網頁不是孤立的一頁，而是可以彼此連接的文件。

## HTML 是標記語言

HTML 是一種標記語言。標記語言的核心是一套標記標籤，用來描述內容的角色。

例如：

```html
<h1>HTML 入門</h1>
<p>HTML 用來描述網頁內容的結構。</p>
```

這裡的 `<h1>` 和 `<p>` 不是要讓電腦做複雜計算，而是告訴瀏覽器：

- `HTML 入門` 是主標題。
- `HTML 用來描述網頁內容的結構。` 是段落文字。

這就是標記語言的基本觀念。

## 網頁是怎麼形成的

網頁形成的流程可以簡化成三步：

```text
前端人員撰寫 HTML 等程式碼
→ 瀏覽器讀取並解析程式碼
→ 瀏覽器渲染成使用者看到的網頁
```

也就是說，開發者寫下的是程式碼，使用者看到的是瀏覽器處理後的畫面。

例如開發者寫：

```html
<h1>我的第一個網頁</h1>
<p>這是我用 HTML 寫出的第一段內容。</p>
```

瀏覽器會解析這些標籤，知道第一行是標題、第二行是段落，最後把它顯示成真正的網頁內容。

## 解析與渲染

瀏覽器把 HTML 轉成網頁畫面時，通常可以用兩個詞理解：

| 名詞 | 白話理解 |
| --- | --- |
| 解析 | 讀懂 HTML 標籤與內容之間的關係 |
| 渲染 | 把解析結果畫到瀏覽器畫面上 |

初學 HTML 時，可以先記住這個概念：瀏覽器不是直接「顯示程式碼」，而是先讀懂程式碼，再把結果呈現給使用者。

## 常見誤解

### 誤解一：HTML 是程式語言

HTML 不是用來寫邏輯運算的程式語言，而是描述網頁內容與結構的標記語言。

如果要處理互動邏輯，例如點擊按鈕後顯示訊息，通常會使用 JavaScript。

### 誤解二：網頁就是網站

網頁通常是網站中的單一頁面，網站則是由多個網頁、資源與連結組成的整體。

### 誤解三：瀏覽器只是打開檔案

瀏覽器不只是打開 HTML 檔案。它會解析 HTML，理解標籤結構，再把結果渲染成使用者看到的畫面。

## 重點整理

- 網頁通常是網站中的一頁，常見副檔名是 `.html` 或 `.htm`。
- 網站通常由多個網頁組成，網頁之間可以透過超連結彼此連接。
- HTML 是超文本標記語言，用來描述網頁內容與結構。
- HTML 不是程式語言，而是標記語言。
- 超文本代表網頁可以包含多媒體，也可以連到其他文件。
- 瀏覽器會解析 HTML，並把它渲染成使用者看到的網頁。
