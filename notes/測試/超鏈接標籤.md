# HTML 超連結標籤 `<a>`

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 `<a>` 標籤的語意與使用情境。
- 使用 `href` 建立外部連結、內部連結、檔案連結與錨點連結。
- 判斷什麼時候應該使用 `target="_blank"`。
- 理解空連結、JavaScript 連結與特殊協議連結的差異。
- 避免把「前往資源」和「觸發操作」混在一起。

## 問題情境

網頁不會只停留在單一頁面。使用者常常需要從目前頁面前往另一個網頁、下載檔案、跳到頁面中的某個段落，或開啟電話、Email、通訊軟體等應用。

這些「從目前位置連到另一個目標」的需求，通常都會使用 HTML 的 `<a>` 標籤完成。

## 一句話理解

`<a>` 是用來建立超連結的 HTML 語意標籤；它的重點是讓目前文件連到另一個網址、檔案、位置或指定協議。

`a` 來自英文 anchor，意思是「錨」。可以把它想成在頁面中放下一個可點擊的連接點，讓使用者跳到指定目標。

## 基本語法

```html
<a href="目標地址">連結文字</a>
```

例如：

```html
<a href="https://www.baidu.com/">前往百度</a>
```

這段程式碼可以拆成：

| 部分 | 說明 |
| --- | --- |
| `<a>` | 建立超連結的起始標籤 |
| `href` | 指定連結要前往的目標 |
| `https://www.baidu.com/` | 連結目標地址 |
| `前往百度` | 使用者在頁面上看到並點擊的文字 |
| `</a>` | 結束標籤 |

沒有 `href` 的 `<a>` 不算是一個完整可前往目標的超連結。

## 常用屬性

![超連結標籤 href 與 target 屬性表](../../origin/測試/assets/images/link-tags-img-001-910f05.png)

| 屬性 | 作用 | 常見寫法 |
| --- | --- | --- |
| `href` | 指定連結目標，是 `<a>` 最核心的屬性 | `href="https://example.com"` |
| `target` | 指定連結開啟方式 | `target="_self"`、`target="_blank"` |
| `download` | 提示瀏覽器下載連結資源 | `download="講義.pdf"` |

### `target` 的常見值

| 值 | 說明 | 使用時機 |
| --- | --- | --- |
| `_self` | 在目前頁籤開啟，這是預設值 | 網站內部頁面切換 |
| `_blank` | 在新頁籤開啟 | 前往外部網站或不希望使用者離開目前頁面 |

如果使用 `target="_blank"` 開啟外部網站，實務上通常會搭配 `rel="noopener noreferrer"`，降低新頁面取得原頁面控制權的風險。

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  前往外部網站
</a>
```

## 連結的常見類型

### 外部連結

外部連結指向其他網站，通常使用完整網址。

```html
<a href="http://www.qq.com" target="_self">騰訊</a>
<a href="http://www.itcast.cn" target="_blank" rel="noopener noreferrer">傳智播客</a>
```

`target="_self"` 會在目前頁籤開啟；`target="_blank"` 會開新頁籤。

### 內部連結

內部連結指向同一個網站中的其他頁面，常使用相對路徑。

```html
<a href="./test.html">公司簡介</a>
```

相對路徑會根據目前 HTML 檔案所在位置計算。如果檔案搬到不同資料夾，路徑也可能需要跟著調整。

### 檔案連結

`href` 不只能連到網頁，也可以連到圖片、影片、音訊、PDF 或壓縮檔。

### 頁面內錨點連結

錨點連結可以讓使用者點擊後跳到同一頁的指定位置。

```html
<a href="#season-2">跳到第二季介紹</a>

<h3 id="season-2">第二季介紹</h3>
```

寫法重點是：

1. 連結的 `href` 使用 `#id名稱`。
2. 目標元素要有對應的 `id`。
3. `href="#season-2"` 會找 `id="season-2"` 的元素。

也可以連到其他頁面的指定位置：

```html
<a href="./測試頁面.html#test1">跳到測試頁面的 test1 錨點</a>
```

### 回到頁面頂部

```html
<a href="#">回到頂部</a>
```

`href="#"` 通常會讓頁面回到頂部，但它也可能造成網址後方出現 `#`。如果頁面有明確的頂部區塊，更清楚的寫法是使用指定 `id`。

```html
<header id="top">網站標題</header>

<a href="#top">回到頂部</a>
```

注意：`href="##"` 不是「回到頂部」的標準寫法，初學時不要這樣寫。

### 空連結

```html
<a href="">刷新本頁面</a>
```

空的 `href` 可能會重新載入目前頁面。教學示範時可以看懂它，但實務上不建議把空連結當成暫時按鈕；如果還沒有目標，通常先不要放連結，或等目標確認後再補上。

### 特殊協議連結

`href` 也可以使用特殊協議，喚起裝置或應用程式。

```html
<a href="tel:10010">電話聯繫</a>
<a href="mailto:10010@qq.com">郵件聯繫</a>
<a href="line://msg/text/Hello%20World">使用 LINE 發送消息</a>
```

常見協議包含：

| 協議 | 用途 |
| --- | --- |
| `tel:` | 開啟撥號功能 |
| `mailto:` | 開啟郵件軟體 |
| `line:` | 嘗試喚起 LINE |
| `javascript:` | 執行 JavaScript 程式碼 |

`javascript:` 連結雖然能執行程式碼，但實務上通常不建議用它處理互動。若目的是觸發操作，通常應該使用 `<button>` 搭配 JavaScript。

```html
<!-- 不建議把操作寫成 javascript: 連結 -->
<a href="javascript:alert('Hello, World!');">點擊我</a>
```

## 讓圖片或區塊成為連結

`<a>` 不只能包住文字，也可以包住圖片或部分區塊內容，讓整個內容都可以點擊。

```html
<a href="https://www.baidu.com">
  <img src="../../origin/測試/assets/images/link-tags-img-002-354aea.jpg" alt="範例圖片">
</a>
```

在 HTML5 中，`<a>` 可以包住多種元素。不過要注意，不要在 `<a>` 裡再放另一個互動元素，例如另一個 `<a>` 或 `<button>`，否則使用者操作與無障礙閱讀都容易出問題。

```html
<a href="https://example.com">
  <div>
    <h2>文章標題</h2>
    <p>點擊整個區塊查看文章。</p>
  </div>
</a>
```

## 常見錯誤

### 錯誤一：只有 `<a>`，沒有 `href`

錯誤寫法：

```html
<a>查看 GitHub</a>
```

這樣雖然有 `<a>` 標籤，但沒有指定目標，使用者無法前往任何地方。

正確寫法：

```html
<a href="https://github.com">查看 GitHub</a>
```

### 錯誤二：用 `<a>` 做一般按鈕操作

錯誤寫法：

```html
<a href="#">送出表單</a>
```

如果目的是「送出表單」、「打開彈窗」、「切換狀態」這類操作，而不是前往另一個資源，通常不應該使用 `<a>`。

正確寫法：

```html
<button type="submit">送出表單</button>
```

判斷方式很簡單：

- 要前往某個網址、檔案或頁面位置：使用 `<a>`。
- 要執行某個操作：優先考慮 `<button>`。

### 錯誤三：錨點名稱對不起來

錯誤寫法：

```html
<a href="#section-two">跳到第二段</a>

<h2 id="section-2">第二段</h2>
```

`href="#section-two"` 會尋找 `id="section-two"`，但目標元素寫的是 `id="section-2"`，因此無法跳到預期位置。

正確寫法：

```html
<a href="#section-two">跳到第二段</a>

<h2 id="section-two">第二段</h2>
```

### 錯誤四：新分頁外部連結缺少安全屬性

不完整寫法：

```html
<a href="https://example.com" target="_blank">外部網站</a>
```

更建議的寫法：

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  外部網站
</a>
```

`rel="noopener noreferrer"` 可以降低新開頁面影響原頁面的風險。

## 實務使用建議

寫 `<a>` 標籤時，先問自己三個問題：

1. 使用者點擊後要去哪裡？
2. 這是前往資源，還是執行操作？
3. 這個連結文字是否能讓使用者知道點了會發生什麼事？

好的連結文字應該具體描述目標，不要只寫「點我」或「這裡」。

```html
<!-- 不清楚 -->
<a href="./intro.html">點我</a>

<!-- 較清楚 -->
<a href="./intro.html">查看公司簡介</a>
```

實務上也要避免濫用新分頁。不是每個連結都需要 `target="_blank"`；如果是網站內部導覽，使用目前頁籤通常比較自然。

## 重點整理

- `<a>` 用來建立超連結，語意是「前往某個目標」。
- `href` 是 `<a>` 最核心的屬性，用來指定連結目標。
- `target="_self"` 是預設開啟方式，`target="_blank"` 會開新頁籤。
- 連結目標可以是外部網址、內部頁面、檔案、錨點或特殊協議。
- `download` 可以提示瀏覽器下載資源。
- 頁面內錨點需要 `href="#目標id"` 搭配目標元素的 `id`。
- 如果目的是觸發操作，而不是前往資源，通常應該使用 `<button>`。
- 連結文字應該清楚說明目標，不要只寫「點我」或「這裡」。
