# Web 服務器與網站部署

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明為什麼網站需要 Web 服務器。
- 分辨服務器、主機、Web 服務器這幾個入門名詞。
- 理解本地服務器與遠程服務器的用途差異。
- 知道「把網站部署上線」大致代表什麼。

## 問題情境

當你在自己的電腦上寫了一個 HTML 檔案，通常只有你自己能直接打開它。如果希望同學、使用者或全世界的人都能透過網址訪問這個網站，就不能只把檔案留在自己的電腦裡。

這時需要一個可以長時間對外提供網站資源的環境，也就是 Web 服務器。

## 一句話理解

Web 服務器是負責保存網站資源並回應瀏覽器請求的服務；部署就是把網站放到能被目標使用者訪問的位置。

## 服務器是什麼

服務器是提供計算服務的設備、程式或系統。它本質上也可以是一台電腦，只是在網路環境中負責對外提供服務。

依照提供的服務不同，常見名稱包含：

| 類型 | 主要用途 |
|---|---|
| 文件服務器 | 保存與提供檔案 |
| 數據庫服務器 | 保存、查詢與管理資料 |
| 應用程序服務器 | 執行應用程式與業務邏輯 |
| Web 服務器 | 提供網站檔案或 Web 資源 |

服務器有時也會被稱為主機。入門時可以先把主機理解成「用來放網站或服務的電腦 / 環境」。

## Web 服務器的作用

Web 服務器一般指網站服務器。它可以接收瀏覽器等 Web 客戶端的請求，並回傳 HTML、CSS、JavaScript、圖片、檔案或其他資料。

![多位使用者連線至服務器示意圖](../../origin/第01章_寫在前面/assets/images/web-server-img-001-7d7f51.png)

一個基本訪問流程可以這樣看：

1. 使用者在瀏覽器輸入網址。
2. 瀏覽器向對應的 Web 服務器發出請求。
3. Web 服務器找到網站資源或交給後端程式處理。
4. Web 服務器把結果回傳給瀏覽器。
5. 瀏覽器解析並顯示頁面。

如果網站只放在自己的電腦上，其他人通常無法透過公開網址訪問。把網站放到可被訪問的服務器上，就是部署的重要一步。

## 本地服務器

本地服務器是指在自己的電腦上啟動服務器環境，通常用於開發與測試。

例如你寫了一個網站，可能會在終端執行某個開發工具，讓網站出現在：

```text
http://localhost:3000
```

這代表網站目前由你自己的電腦提供服務。瀏覽器訪問這個網址時，請求沒有跑到遠方的正式主機，而是回到你的本機。

本地服務器常用於：

- 預覽 HTML、CSS、JavaScript 的效果。
- 測試模組、路徑與資源載入。
- 模擬接近正式網站的訪問方式。
- 在上線前先修正錯誤。

如果網路、防火牆與服務綁定位址允許，同一個局域網內的其他裝置也可能訪問你的本地項目。但這通常仍屬於測試情境，不等於正式公開上線。

## 遠程服務器

遠程服務器通常是位於自己電腦之外、可透過網路連線的服務器。它可能由雲端平台、主機商、公司機房或其他服務提供者管理。

把網站部署到遠程服務器後，使用者可以透過域名或公開網址訪問網站。

例如：

```text
https://www.example.com
```

遠程服務器常用於：

- 讓網站對外公開。
- 提供穩定的網路與運行環境。
- 綁定正式域名。
- 長時間運行網站或後端服務。

## 本地與遠程的比較

| 項目 | 本地服務器 | 遠程服務器 |
|---|---|---|
| 位置 | 自己的電腦 | 網路上的主機或雲端環境 |
| 主要用途 | 開發、預覽、測試 | 正式提供服務 |
| 訪問對象 | 通常是自己或局域網內裝置 | 目標使用者或公開使用者 |
| 穩定性要求 | 較低，方便調整即可 | 較高，需要穩定、安全、可維護 |
| 常見網址 | `localhost`、本機 IP | 正式域名或公開 IP |

## 網站部署大致包含什麼

部署不是單純「把檔案丟到某個地方」。完整一點看，部署通常包含：

- 準備網站檔案或打包後的產物。
- 上傳或發佈到服務器、主機平台或靜態網站平台。
- 設定服務器如何回應請求。
- 確認圖片、CSS、JavaScript 等資源路徑正確。
- 綁定域名與 HTTPS。
- 測試不同裝置與瀏覽器是否能正常訪問。

對剛開始學 HTML 的階段，不需要一次學完所有部署細節，但要先建立觀念：本機能打開，不代表別人也能打開；正式訪問需要一個可被連線的服務環境。

## 常見錯誤與混淆

### 以為打開 HTML 檔案就是部署

直接雙擊 HTML 檔案，只是讓本機瀏覽器讀取本機檔案。這通常不是網站上線，也不代表其他人可以透過網址訪問。

避免方式：確認使用者是否能透過公開網址訪問，而不是只看自己電腦能否打開。

### 以為服務器一定是另一台昂貴電腦

服務器描述的是提供服務的角色。你的電腦啟動開發服務時，也可以暫時扮演本地服務器。

避免方式：先用「提供服務」理解服務器，再區分本地測試與正式部署。

### 混淆本地服務器與遠程服務器

本地服務器主要服務開發流程，遠程服務器主要服務真實使用者。兩者都可能用瀏覽器訪問，但目的與環境要求不同。

避免方式：看到網址時留意它是 `localhost`、局域網 IP，還是正式域名。

### 忽略資源路徑

網站部署後，HTML 可能能打開，但圖片或 CSS 載入失敗。常見原因是相對路徑、檔名大小寫或目錄結構和本機預期不同。

避免方式：部署後實際打開瀏覽器開發者工具，檢查 Network 面板是否有資源載入錯誤。

## 對 HTML 學習的意義

HTML 檔案不是只存在編輯器裡。它最後會被瀏覽器透過某種方式讀取：可能是本機檔案，也可能是本地服務器，也可能是遠程 Web 服務器。

理解服務器與部署後，你會更容易明白：

- 為什麼路徑寫法會影響圖片與連結。
- 為什麼專案常需要本地開發服務器。
- 為什麼網站上線後還要測試資源載入。
- 為什麼前端工作會和主機、域名、HTTPS 有關。

## 重點整理

- Web 服務器負責回應瀏覽器請求並提供網站資源。
- 本地服務器通常用於開發與測試。
- 遠程服務器通常用於正式部署與公開訪問。
- 部署是把網站放到目標使用者能訪問的環境中。
- 本機能打開網頁，不代表網站已經上線。
