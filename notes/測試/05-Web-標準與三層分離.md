# Web 標準與三層分離

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明為什麼需要 Web 標準。
- 理解 Web 標準中結構、表現、行為三層的基本分工。
- 知道 HTML、CSS、JavaScript 分別負責什麼。
- 理解三層分離對維護與前端開發的意義。

## 問題情境

如果同一份網頁在不同瀏覽器打開，結果排版、樣式或互動都不一樣，使用者體驗會很差，開發者也很難維護。

Web 標準的目的，就是讓瀏覽器與開發者盡量遵守共同規範，降低不同環境下的解析與呈現差異。

## 一句話理解

Web 標準讓網頁能更一致地被瀏覽器解析；三層分離則把 HTML、CSS、JavaScript 分別用來負責結構、表現與行為。

## 為什麼需要 Web 標準

不同瀏覽器可能使用不同渲染引擎，對相同程式碼的支援與解析細節也可能不同。如果沒有共同規範，開發者很難寫出能穩定運行的網頁。

![不同瀏覽器呈現不同網頁效果示意圖](../../origin/第01章_寫在前面/assets/images/web-standard-img-001-e119b2.png)

Web 標準的作用是提供共同規則，讓不同瀏覽器盡量用一致的方式理解 HTML、CSS、JavaScript 等 Web 技術。

![Web 標準統一不同瀏覽器顯示效果示意圖](../../origin/第01章_寫在前面/assets/images/web-standard-img-002-783e20.png)

標準不能保證所有瀏覽器每個細節完全相同，但它能大幅降低差異，讓開發與使用更可預期。

## Web 標準的三層構成

Web 前端常把頁面拆成三個層次：

| 層次 | 對應技術 | 負責內容 |
|---|---|---|
| 結構 | HTML | 頁面有哪些內容、內容之間是什麼關係 |
| 表現 | CSS | 內容如何呈現，例如顏色、大小、排版、間距 |
| 行為 | JavaScript | 使用者互動與動態效果，例如點擊、切換、資料請求 |

![Web 標準結構、表現與行為對照表](../../origin/第01章_寫在前面/assets/images/web-standard-img-003-2d03d3.png)

簡單理解：

- HTML 寫頁面結構。
- CSS 寫頁面樣式。
- JavaScript 寫頁面互動。

## 用一個按鈕區塊理解三層

假設頁面上有一個「加入購物車」按鈕。

HTML 負責描述它是什麼：

```html
<button type="button" class="cart-button">加入購物車</button>
```

CSS 負責決定它長什麼樣子：

```css
.cart-button {
  padding: 8px 16px;
  color: white;
  background: #0f766e;
}
```

JavaScript 負責處理使用者點擊後發生什麼事：

```js
const button = document.querySelector(".cart-button");

button.addEventListener("click", () => {
  console.log("加入購物車");
});
```

這三段程式碼各自關注不同責任。當它們分工清楚時，之後要改文案、改樣式、改互動，都比較容易找到位置。

## 為什麼要三層分離

三層分離不是為了形式漂亮，而是為了讓網頁更容易理解、維護與擴充。

### 可讀性更好

如果 HTML 裡同時塞滿樣式與互動程式，頁面結構會變得難讀。分離後，HTML 能更清楚呈現內容層級。

### 維護成本更低

當設計要改顏色或間距時，主要修改 CSS；當互動邏輯要調整時，主要修改 JavaScript；當內容結構改變時，主要修改 HTML。

### 團隊協作更清楚

在實務專案中，設計、前端、後端或不同前端成員可能分工合作。清楚的層次可以降低互相干擾。

### 更符合瀏覽器解析模型

瀏覽器本來就會解析 HTML 結構、計算 CSS 樣式、執行 JavaScript。三層分離能讓程式碼更貼近瀏覽器處理網頁的方式。

## 常見錯誤與混淆

### 把樣式直接寫進 HTML 結構

例如大量使用 `style` 屬性：

```html
<p style="color: red; font-size: 20px;">重要提醒</p>
```

這樣在小範例中可以運作，但在正式頁面中會讓樣式分散，難以統一調整。

較好的做法是把結構與樣式分開：

```html
<p class="notice">重要提醒</p>
```

```css
.notice {
  color: red;
  font-size: 20px;
}
```

### 用 HTML 標籤追求外觀，而不是語意

例如只因為文字想變大就亂用標題標籤。這會讓 HTML 結構失真，也可能影響搜尋、可及性與維護。

避免方式：HTML 先問「這段內容是什麼」，CSS 再處理「它看起來如何」。

### 以為三層分離代表永遠不能混在一起

在學習、小型示範或元件化開發中，有時會把結構、樣式或行為放在相近位置。但責任仍應清楚：HTML 描述結構，CSS 管樣式，JavaScript 管互動。

避免方式：不要只看檔案是否分開，要看責任是否混亂。

### 以為遵守標準就不用測試

Web 標準能降低差異，但不能取代測試。瀏覽器版本、裝置尺寸、使用者設定與新功能支援仍可能造成問題。

避免方式：重要頁面仍要在目標瀏覽器與裝置上檢查。

## 對 HTML 學習的意義

HTML 是三層中的結構層。學 HTML 時，重點不是把頁面做得漂亮，也不是先寫互動，而是先把內容和語意建立正確。

理解三層分離後，你會更容易判斷：

- 這段內容應該用哪個 HTML 標籤表示。
- 這個視覺效果應該交給 CSS。
- 這個點擊或資料變化應該交給 JavaScript。
- 為什麼語意正確的 HTML 比單純看起來像更重要。

## 重點整理

- Web 標準讓不同瀏覽器盡量依照共同規範解析與呈現網頁。
- Web 前端常分為結構、表現、行為三層。
- HTML 負責結構，CSS 負責表現，JavaScript 負責行為。
- 三層分離能提升可讀性、維護性與協作效率。
- 寫 HTML 時應優先考慮內容語意，而不是外觀。
