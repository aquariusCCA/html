# HTML5 版本與特性

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 HTML5 在 HTML 版本演進中的角色。
- 分辨狹義 HTML5 與廣義 HTML5 的差異。
- 認識 HTML5 常見的新特性方向。
- 理解使用 HTML5 時需要注意瀏覽器相容性。

## 問題情境

學 HTML 時，你很快會看到 `<!DOCTYPE html>`、語意化標籤、多媒體標籤、表單新屬性等內容。這些常常會被統稱為 HTML5。

但 HTML5 不只是「版本比較新」而已。它代表現代網頁標準的一次重要更新，讓網頁可以更清楚地描述語意，也能支援更多原本需要外掛或額外技術才能完成的功能。

## 一句話理解

HTML5 是新一代 HTML 標準；狹義上指 HTML 本身的新版標準，廣義上常被用來代表 HTML5、CSS3 與 JavaScript 組成的現代前端技術集合。

## HTML5 是什麼

HTML5 是新一代的 HTML 標準，原始資料提到它在 2014 年 10 月由 W3C 完成標準制定。

它的出現主要是為了補足舊版 HTML 在實務開發上的不足，例如：

- 新增更清楚的語意化標籤。
- 新增表單功能與表單屬性。
- 支援更好的多媒體能力。
- 提供更多能被 JavaScript 操作的介面。
- 更適合現代網站與行動裝置應用。

HTML5 也保留向下相容的特性。也就是說，在 HTML5 標準下，許多舊版本的標籤與功能仍然可以使用。

## 狹義與廣義的 HTML5

HTML5 這個詞在不同情境下可能有不同範圍。

| 說法 | 指的是什麼 | 適合情境 |
| --- | --- | --- |
| 狹義 HTML5 | HTML 本身的新一代標準 | 討論 HTML 標籤、文件結構、表單、多媒體 |
| 廣義 HTML5 | HTML5 + CSS3 + JavaScript 等前端技術集合 | 討論整體現代前端能力、網站與應用開發 |

初學時要特別注意：如果老師或文章說「HTML5 可以做很多互動效果」，有時候它其實是在用廣義 HTML5，也就是同時包含 CSS 與 JavaScript 的能力。

## HTML5 新特性的方向

HTML5 新增的能力很多，初學階段不用一次背完，可以先理解它們大致分成幾個方向。

| 特性方向 | 白話理解 |
| --- | --- |
| 語意化特性 | 使用更有意義的標籤描述頁面結構，例如頁首、導覽、文章、區塊 |
| 表單特性 | 提供更多輸入類型與表單屬性，讓表單更容易描述資料需求 |
| 多媒體特性 | 原生支援音訊與影片，減少對外掛的依賴 |
| 本地儲存特性 | 讓瀏覽器可以在使用者裝置中保存部分資料 |
| 設備與連接能力 | 支援更多與裝置、網路連線相關的應用情境 |
| 圖形與效果 | 支援更豐富的圖形、動畫與視覺效果 |
| 效能與整合 | 讓網頁應用更適合大型、互動性高的開發需求 |

這些特性不一定都屬於 HTML 標籤本身。有些需要 CSS 或 JavaScript 搭配，這也是「廣義 HTML5」這個說法會出現的原因。

## HTML5 的實務優勢

HTML5 對現代前端開發的重要性，主要體現在以下幾點：

- 新增語意化標籤，讓 HTML 結構更容易閱讀與維護。
- 對 SEO 更友好，因為文件結構與內容意義更清楚。
- 新增多媒體標籤，可以更自然地放入音訊與影片。
- 提供更多可由 JavaScript 操作的介面。
- 更適合行動裝置與跨平台網頁應用。

例如，以前網頁放影片常常需要依賴外掛；HTML5 之後，可以使用 `<video>` 這類原生標籤描述影片內容。

```html
<video controls src="intro.mp4">
  你的瀏覽器不支援影片播放。
</video>
```

這段程式碼不只是在畫面上放一個播放器，也是在 HTML 中清楚標記「這裡是一段影片內容」。

## 相容性注意事項

HTML5 新特性並不是所有舊瀏覽器都完整支援。

原始資料中特別提到：

- 主流瀏覽器如 Chrome、Safari、Opera、Firefox 對 HTML5 支援較好。
- IE 瀏覽器需要 IE9 以上才支援 HTML5，而且 IE9 也只支援部分新特性。
- 如果專案不需要考慮舊瀏覽器相容性，就可以更放心使用 HTML5 新特性。

實務開發時，不能只問「這是不是 HTML5 功能」，還要問：

- 目標使用者會用哪些瀏覽器？
- 專案是否需要支援舊版 IE？
- 這個新特性是否有替代方案？
- 不支援時，頁面是否仍然能閱讀或操作？

## 常見誤解

### 誤解一：HTML5 等於所有前端技術

嚴格來說，HTML5 是 HTML 的新版標準。但在廣義說法中，HTML5 常被用來代表 HTML5、CSS3、JavaScript 等現代前端技術集合。

學習時要看上下文，不要把所有視覺效果或互動功能都誤認為 HTML 本身能完成。

### 誤解二：HTML5 新特性可以無條件使用

新特性仍然可能有瀏覽器相容性問題。開發前要確認專案支援範圍，尤其是面向舊設備或舊企業環境時。

### 誤解三：HTML5 會讓舊 HTML 全部失效

HTML5 有向下相容特性，在 HTML5 標準下仍可使用許多舊版本標籤與功能。不過實務上應優先使用語意更清楚、符合現代標準的寫法。

## 重點整理

- HTML5 是新一代 HTML 標準，補足舊版 HTML 在現代網頁開發上的不足。
- 狹義 HTML5 指 HTML 本身的新標準；廣義 HTML5 常指 HTML5 + CSS3 + JavaScript 的前端技術集合。
- HTML5 強調語意化、多媒體、表單能力、行動裝置與現代網頁應用。
- HTML5 有向下相容特性，但新功能仍需要考慮瀏覽器支援。
- 初學時應先掌握常用特性，不需要一次記完所有 HTML5 能力。
