# 表單按鈕與提交重置

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨提交按鈕、重置按鈕與普通按鈕。
- 使用 `<input>` 或 `<button>` 建立表單按鈕。
- 理解 `<button>` 預設是提交按鈕。
- 避免普通按鈕誤觸表單提交。

## 問題情境

表單填完後，使用者通常需要按下按鈕。按鈕可能代表送出資料、還原表單，或只是觸發某個 JavaScript 操作。

這些按鈕看起來都像按鈕，但在表單中的行為不同。若沒有指定清楚，最常見的錯誤就是「原本只是想開彈窗，卻不小心把表單送出」。

## 一句話理解

表單按鈕的重點不是外觀看起來像按鈕，而是它的 `type` 會決定按下去後要提交、重置，還是只執行一般操作。

## 三種常見按鈕類型

| type | 作用 | 使用情境 |
| --- | --- | --- |
| `submit` | 提交表單 | 註冊、登入、送出留言 |
| `reset` | 還原表單到初始狀態 | 讓使用者重新填寫 |
| `button` | 普通按鈕 | 搭配 JavaScript 開彈窗、取得驗證碼、切換畫面 |

## 使用 input 建立按鈕

`<input>` 可以透過不同的 `type` 建立按鈕，按鈕文字由 `value` 決定。

```html
<form action="/register" method="post">
  <label for="username">帳號</label>
  <input type="text" id="username" name="username">

  <input type="submit" value="免費註冊">
  <input type="reset" value="重新填寫">
  <input type="button" value="取得簡訊驗證碼">
</form>
```

這段程式碼中：

- `type="submit"` 會提交表單。
- `type="reset"` 會把表單控件還原成初始預設狀態。
- `type="button"` 不會自動提交，通常搭配 JavaScript 使用。

## 使用 button 建立按鈕

`<button>` 的按鈕文字寫在開始標籤與結束標籤中間。

```html
<form action="/register" method="post">
  <button type="submit">提交按鈕</button>
  <button type="reset">重置按鈕</button>
  <button type="button">普通按鈕</button>
</form>
```

`<button>` 比 `<input type="button">` 更彈性，因為按鈕內容可以放文字，也可以放其他內嵌內容。初學階段先掌握 `type` 行為最重要。

## button 的預設 type

在表單裡，`<button>` 如果沒有指定 `type`，預設行為是 `submit`。

```html
<form action="/register" method="post">
  <button>未指定 type 的按鈕</button>
</form>
```

上面的按鈕會提交表單。若只是普通按鈕，應明確寫成：

```html
<button type="button">取得驗證碼</button>
```

## reset 是還原，不一定是清空

`type="reset"` 會把欄位恢復成頁面載入時的初始值，不是單純清空所有欄位。

```html
<form action="/profile" method="post">
  <label for="nickname">暱稱</label>
  <input type="text" id="nickname" name="nickname" value="小明">

  <button type="reset">重置</button>
</form>
```

如果使用者把暱稱改成其他文字，再按下重置，欄位會回到「小明」，而不是變成空白。

## 常見錯誤

### 普通 button 忘記指定 type

```html
<form action="/register" method="post">
  <button>取得驗證碼</button>
</form>
```

這個按鈕看起來像普通操作，但因為 `<button>` 預設是 `submit`，按下後會提交表單。

正確寫法：

```html
<form action="/register" method="post">
  <button type="button">取得驗證碼</button>
</form>
```

### 以為 reset 會清空所有欄位

```html
<input type="text" name="username" value="guest">
<button type="reset">清空</button>
```

按下 reset 會回到 `value="guest"`，不是清空。按鈕文字若寫成「清空」，容易誤導使用者。

## 實務判斷

- 要送出表單，使用 `type="submit"`。
- 要還原表單到初始狀態，使用 `type="reset"`。
- 要觸發 JavaScript 操作，使用 `type="button"`。
- 在 `<form>` 裡使用 `<button>` 時，養成明確寫 `type` 的習慣。

## 重點整理

- 表單按鈕主要有 `submit`、`reset`、`button` 三種。
- `<input>` 按鈕用 `value` 決定按鈕文字。
- `<button>` 按鈕文字寫在標籤內容中。
- `<button>` 未指定 `type` 時，預設是 `submit`。
- `reset` 是還原初始狀態，不一定是清空。
