# 表單按鈕與提交重置操作

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨 submit、reset、button 三種按鈕用途。
- 正確使用 `<input>` 與 `<button>` 建立按鈕。
- 避免 `<button>` 預設提交表單造成的錯誤。
- 判斷什麼時候要用提交按鈕，什麼時候只是一般操作按鈕。

## 問題情境

表單填完後，使用者通常需要按下某個按鈕完成操作。

但按鈕不只一種。有的會提交表單，有的會重置內容，有的只是等待 JavaScript 處理。

## 一句話理解

表單按鈕的行為主要由 `type` 決定。

## submit 提交按鈕

提交按鈕會把 `<form>` 裡的表單資料送出去。

```html
<form action="/register" method="post">
  <label for="username">帳號</label>
  <input type="text" id="username" name="username">

  <input type="submit" value="免費註冊">
</form>
```

也可以使用 `<button>`：

```html
<form action="/register" method="post">
  <label for="username">帳號</label>
  <input type="text" id="username" name="username">

  <button type="submit">免費註冊</button>
</form>
```

差異是：

- `<input type="submit">` 使用 `value` 決定按鈕文字。
- `<button type="submit">` 使用標籤中間的內容當按鈕文字。

## reset 重置按鈕

重置按鈕會把表單恢復成初始狀態。

```html
<form action="/profile" method="post">
  <label for="nickname">暱稱</label>
  <input type="text" id="nickname" name="nickname" value="小明">

  <input type="reset" value="重新填寫">
</form>
```

使用 `<button>` 的寫法：

```html
<button type="reset">重新填寫</button>
```

實務上要小心使用 reset，因為使用者可能不小心清掉已填好的資料。

## button 普通按鈕

普通按鈕不會自動提交表單，通常用來搭配 JavaScript，例如取得驗證碼、開啟彈窗、切換顯示狀態。

```html
<form action="/login" method="post">
  <label for="phone">手機號碼</label>
  <input type="text" id="phone" name="phone">

  <button type="button">取得簡訊驗證碼</button>
  <button type="submit">登入</button>
</form>
```

## 常見錯誤：忘記寫 button 的 type

```html
<form action="/login" method="post">
  <button>取得簡訊驗證碼</button>
</form>
```

在表單裡，`<button>` 如果沒有寫 `type`，預設是 `submit`。這可能導致使用者只是想取得驗證碼，卻意外提交表單。

正確寫法：

```html
<button type="button">取得簡訊驗證碼</button>
```

## 三種 type 對照

| type | 行為 | 使用時機 |
| --- | --- | --- |
| `submit` | 提交表單 | 註冊、登入、搜尋、送出留言 |
| `reset` | 重置表單到初始狀態 | 少數需要清空或還原表單的情境 |
| `button` | 不做預設表單行為 | 搭配 JavaScript 的一般操作 |

## 重點整理

- 提交表單用 `type="submit"`。
- 重置表單用 `type="reset"`，但實務上要謹慎使用。
- 一般操作按鈕用 `type="button"`。
- 表單中的 `<button>` 沒寫 `type` 時，預設是 `submit`。
