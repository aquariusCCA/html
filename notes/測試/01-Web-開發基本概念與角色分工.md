# Web 開發基本概念與角色分工

## 學習目標

讀完這篇筆記，你應該能夠：

- 用自己的話說明 Web 是什麼。
- 分辨前端、後端、客戶端、服務端這幾組常見名詞。
- 理解為什麼 HTML 學習會先從瀏覽器、網頁與服務器開始。
- 知道「終端」在入門開發環境中通常指什麼。

## 問題情境

剛開始學 HTML 時，很容易以為自己只是在寫一份檔案。可是只要進一步問：

- 為什麼瀏覽器能看到網頁？
- 網頁資料從哪裡來？
- 前端和後端到底差在哪裡？
- 教學常說的客戶端、服務端、終端又是什麼？

就會發現 HTML 並不是孤立存在的語法。它是 Web 世界裡負責描述網頁內容與結構的一部分。先理解這些角色，後面學 HTML、CSS、JavaScript、部署與除錯時，才不會只背名詞。

## 一句話理解

Web 是一套讓使用者透過瀏覽器訪問網路資源的系統；前端負責使用者看到與操作的部分，後端負責提供資料與服務。

## Web 是什麼

Web 是 World Wide Web 的簡稱，通常翻成全球資訊網。它可以理解成一個建立在網際網路上的資訊系統，使用者透過瀏覽器訪問網頁、圖片、音訊、影片與其他資源。

在 Web 裡，常見的幾個基礎角色是：

| 角色 | 白話理解 | 在 HTML 學習中的位置 |
|---|---|---|
| 瀏覽器 | 打開網頁、顯示網頁、執行網頁的工具 | 讀取並呈現 HTML |
| HTML | 描述網頁內容與結構的標記語言 | 告訴瀏覽器頁面有哪些內容 |
| HTTP/HTTPS | 瀏覽器與服務器交換資料時常用的通訊規則 | 讓瀏覽器能請求網頁資源 |
| 服務器 | 對外提供資料、檔案或服務的電腦或程式 | 保存並提供網站資源 |

Web 可以包含很多類型的資源，不只是一頁 HTML。圖片、CSS、JavaScript、字型、影片、API 回應，都可能是 Web 資源。

## JavaWeb 是什麼

JavaWeb 指的是使用 Java 語言開發，並且可以透過瀏覽器訪問的 Web 程式。

它不是另一種 HTML，也不是另一個瀏覽器，而是一種後端 Web 開發方向。Java 程式通常在應用服務器上運行，處理資料、邏輯、使用者請求，再把結果提供給瀏覽器。

對初學 HTML 來說，先知道這件事即可：

- HTML 關心網頁內容與結構。
- JavaWeb 關心使用 Java 建立可被瀏覽器訪問的 Web 應用。
- 瀏覽器看到的頁面，最終仍需要被轉成瀏覽器能理解的 HTML、CSS、JavaScript 等資源。

## 前端與後端的分工

前端是用來展示資料與接收使用者操作的部分，後端是用來提供資料、處理邏輯與管理資料的部分。

![前端與後端請求響應示意圖](../../origin/第01章_寫在前面/assets/images/web-frontend-backend-img-001-b8f042.png)

可以用一個登入頁面來理解：

| 使用者動作 | 前端負責 | 後端負責 |
|---|---|---|
| 打開登入頁 | 顯示帳號、密碼欄位與按鈕 | 提供頁面或相關資料 |
| 輸入帳號密碼 | 收集使用者輸入 | 驗證帳號密碼是否正確 |
| 按下登入 | 發送請求、顯示等待或錯誤訊息 | 查資料庫、建立登入狀態 |
| 登入成功 | 跳轉頁面、顯示使用者資料 | 回傳登入結果與必要資料 |

入門時可以先記：

- 前端：使用者看得到、點得到、輸入得到的部分。
- 後端：使用者通常看不到，但負責資料、規則、權限與服務的部分。
- 資料庫：保存資料的地方。
- 服務器：提供服務、傳遞資料、保存網站或後端程式的環境。

## 客戶端與服務端

客戶端和服務端描述的是「網路角色」，不是固定某一種設備。

客戶端是發出請求、使用服務的一方。服務端是接收請求、提供服務的一方。同一台電腦在不同情境下，可能扮演不同角色。

以網站為例：

- 使用者的電腦或手機打開瀏覽器，訪問某個網址，這一端可稱為客戶端。
- 保存網站檔案、接收請求、回傳網頁資源的機器或程式，可稱為服務端。

以 App 為例：

- 使用者手機上安裝的 App 是客戶端。
- 提供登入、資料、API、認證等功能的系統是服務端。

## 前端 / 後端不完全等於客戶端 / 服務端

初學時常會把這兩組概念混在一起，因為它們在很多 Web 情境中確實很接近：

| 概念組 | 描述角度 | 常見對應 |
|---|---|---|
| 前端 / 後端 | 應用功能分工 | 前端展示與互動，後端資料與邏輯 |
| 客戶端 / 服務端 | 網路通訊角色 | 客戶端請求服務，服務端提供服務 |

很多時候，瀏覽器裡的前端程式就是客戶端，後端程式就在服務端。但嚴格來說，它們不是同一組定義。

例如：

- 前端可以有複雜邏輯，不只是顯示畫面。
- 客戶端不一定是瀏覽器，也可能是手機 App、桌面軟體或命令列工具。
- 服務端不一定是一台實體機器，也可能是一組雲端服務或一個執行中的程式。

## 終端是什麼

終端這個詞在不同語境中可能有不同意思。對剛開始學前端的人來說，最常遇到的是命令列終端。

可以先這樣理解：終端是一個讓你用文字指令操作電腦或開發工具的界面。

常見例子：

- Windows 的 Command Prompt、PowerShell、Windows Terminal。
- macOS 的 Terminal。
- Linux 的 Terminal。
- 編輯器內建的終端，例如 VS Code Terminal。

後面你可能會在終端裡啟動本地服務器、安裝套件、執行開發工具。它不是 HTML 語法的一部分，但會是前端開發流程中常用的操作入口。

## 常見錯誤與混淆

### 以為 Web 等於 Internet

Internet 是底層網路基礎設施，Web 是建立在網際網路上的一種資訊服務。Web 常透過 HTTP/HTTPS 與瀏覽器運作，但網際網路上還有電子郵件、檔案傳輸、即時通訊等其他服務。

避免方式：說 Web 時，先想到「瀏覽器訪問網頁與資源」；說 Internet 時，先想到「讓不同設備互相連線的網路」。

### 以為前端只負責畫面

前端確實負責畫面，但不只畫面。現代前端也會處理互動、表單驗證、狀態管理、路由、資料請求與部分業務流程。

避免方式：把前端理解為「使用者操作入口」，而不是單純的美工或排版。

### 以為服務端一定是高階硬體

服務端的重點不是硬體等級，而是它在某個情境中提供服務。自己的電腦也可以啟動服務，暫時扮演服務端。

避免方式：用「角色」理解服務端，不要用「設備價格或性能」定義服務端。

### 把終端、客戶端、服務端混為一談

終端通常是操作界面，客戶端是請求服務的角色，服務端是提供服務的角色。三者可能出現在同一台電腦上，但概念不同。

避免方式：看到名詞時先問它是在描述「操作入口」、「請求方」，還是「服務提供方」。

## 對 HTML 學習的意義

HTML 是 Web 前端的基礎。你寫的 HTML 不是給人直接閱讀的純文字筆記，而是給瀏覽器解析的文件結構。

理解 Web 角色後，你會更容易掌握：

- 為什麼 HTML 要放在瀏覽器中看效果。
- 為什麼網頁常常還需要 CSS、JavaScript、圖片與其他資源。
- 為什麼前端會向後端請求資料。
- 為什麼本地開發與正式部署不是同一件事。
- 為什麼瀏覽器差異會影響頁面呈現。

## 重點整理

- Web 是讓使用者透過瀏覽器訪問網路資源的系統。
- HTML 負責描述網頁內容與結構，是前端學習的基礎。
- 前端負責使用者看見與操作的部分，後端負責資料、邏輯與服務。
- 客戶端 / 服務端描述網路角色，前端 / 後端描述應用分工。
- 終端在入門開發中通常指命令列操作界面。
