# 表單標籤、分組與禁用狀態

## 學習目標

讀完這篇筆記，你應該能夠：

- 使用 `<label>` 讓提示文字與表單控件建立關聯。
- 使用 `<fieldset>` 和 `<legend>` 替表單控件分組。
- 使用 `disabled` 禁用單個控件或一組控件。
- 理解這些結構如何改善表單可讀性與使用體驗。

## 問題情境

表單欄位變多後，如果只把輸入框一個接一個排下去，使用者很難理解每個欄位的用途，也不容易操作。

好的表單不只要能提交資料，還要讓使用者看得懂、點得到、知道哪些欄位目前不能使用。

## label：讓文字與控件建立關聯

`<label>` 可以把提示文字和表單控件綁在一起。關聯後，使用者點擊文字，也能聚焦或選取對應控件。

```html
<input type="radio" name="gender" id="male" value="male">
<label for="male">男</label>
```

這段程式碼中：

- `label` 的 `for="male"` 要對應到控件的 `id="male"`。
- 使用者點擊「男」這個文字，也會選取對應的 radio。

另一種寫法是把控件包在 `<label>` 裡：

```html
<label>
  <input type="radio" name="gender" value="female">
  女
</label>
```

## fieldset 與 legend：替表單分組

`<fieldset>` 可以把一組相關表單控件包起來，`<legend>` 是這組欄位的標題。

```html
<form action="/account" method="post">
  <fieldset>
    <legend>主要信息</legend>

    <label for="account">帳戶</label>
    <input type="text" id="account" name="account" maxlength="10">

    <label for="password">密碼</label>
    <input type="password" id="password" name="password" maxlength="6">
  </fieldset>

  <fieldset>
    <legend>勾選區域</legend>

    <label>
      <input type="radio" name="gender" value="woman">
      女
    </label>

    <label>
      <input type="radio" name="gender" value="man">
      男
    </label>
  </fieldset>

  <button type="submit">提交</button>
</form>
```

## disabled：禁用表單控件

替表單控件加上 `disabled`，可以讓它暫時不能被操作。

```html
<input type="text" value="無法輸入" disabled>
<button disabled>無法點擊的按鈕</button>
```

常見可禁用的控件包含：

- `input`
- `textarea`
- `button`
- `select`
- `option`

## 禁用一組控件

如果想一次禁用一組表單控件，可以使用 `<fieldset disabled>`。

```html
<form action="/settings" method="post">
  <fieldset disabled>
    <legend>暫停編輯的資料</legend>

    <label for="email">Email</label>
    <input type="email" id="email" name="email" value="user@example.com">

    <button type="button">取得驗證碼</button>
  </fieldset>
</form>
```

注意：被 `disabled` 的控件不會被包含在表單提交資料中。

## 常見錯誤

### label 的 for 和 id 對不起來

```html
<input type="text" id="username">
<label for="user">使用者名稱</label>
```

這樣點擊 label 文字時，不會正確聚焦到輸入框。

正確寫法：

```html
<input type="text" id="username">
<label for="username">使用者名稱</label>
```

### 想提交資料卻把欄位 disabled

```html
<input type="text" name="email" value="user@example.com" disabled>
```

如果欄位資料仍需要提交，就不應該使用 `disabled`。可以改成一般欄位，或依實際需求用其他方式呈現不可編輯狀態。

## 重點整理

- `<label>` 提升表單可點擊性與可讀性。
- `label for` 要對應控件的 `id`。
- `<fieldset>` 用來分組，`<legend>` 是分組標題。
- `disabled` 會禁用控件，而且被禁用的控件不會提交資料。
