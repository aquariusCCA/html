# HTML 標籤的組成和元素關係

## 學習目標

讀完這篇筆記，你應該能夠：

- 看懂一個 HTML 標籤由哪些符號與名稱組成。
- 分辨雙標籤與單標籤的差異。
- 判斷 HTML 元素之間的父子、祖先、後代與兄弟關係。
- 避免初學者常見的標籤巢狀錯誤。

## 問題情境

HTML 不是只把標籤一個一個寫出來而已。瀏覽器會根據標籤的開始、結束與巢狀位置，建立一棵文件結構。

如果標籤沒有正確成對、層級關係寫錯，瀏覽器雖然可能仍會嘗試修正，但畫面、樣式、JavaScript 選取元素，甚至無障礙閱讀順序都可能和你預期不同。

所以學 HTML 時，除了記標籤名稱，也要看懂「標籤怎麼組成」以及「元素彼此是什麼關係」。

## 一句話理解

HTML 標籤負責標記內容的意義；元素之間的巢狀與並列關係，決定了整份 HTML 文件的結構。

## 標籤是怎麼組成的

HTML 標籤通常由 `<`、`>`、`/` 與英文標籤名稱組成。

![HTML 雙標籤結構示意圖](../../origin/測試/assets/images/tag-composition-relations-img-001-979b79.png)

以段落標籤為例：

```html
<p>這是一段文字。</p>
```

這段 HTML 可以拆成：

| 部分 | 說明 |
| --- | --- |
| `<p>` | 起始標籤，表示段落內容從這裡開始 |
| `p` | 標籤名，代表 paragraph，也就是段落 |
| `這是一段文字。` | 元素內容 |
| `</p>` | 結束標籤，表示段落內容到這裡結束 |

在完整觀念上，`<p>這是一段文字。</p>` 這整體稱為一個 HTML 元素。

## 雙標籤與單標籤

常見 HTML 標籤大多是雙標籤，也就是有起始標籤與結束標籤。

```html
<h1>網頁標題</h1>
<p>這是一段說明文字。</p>
```

有些標籤本身不包住文字或其他元素，通常只需要單一標籤，例如圖片、換行、輸入欄位。

```html
<img src="photo.jpg" alt="個人照片">
<br>
<input type="text" name="username">
```

| 類型 | 結構 | 適合情境 | 常見例子 |
| --- | --- | --- | --- |
| 雙標籤 | `<標籤名>內容</標籤名>` | 需要包住文字或其他元素 | `<p>`、`<h1>`、`<a>`、`<div>` |
| 單標籤 | `<標籤名>` | 標籤本身就能表達完整功能 | `<img>`、`<br>`、`<input>` |

重點不是背「哪個看起來比較短」，而是判斷這個標籤需不需要包住內容。

## 元素之間的兩種基本關係

HTML 元素之間主要有兩種結構關係：

| 關係 | 白話理解 | 例子 |
| --- | --- | --- |
| 巢狀關係 | 一個元素包住另一個元素 | `<ul>` 裡面有多個 `<li>` |
| 並列關係 | 多個元素在同一層 | 兩個 `<li>` 彼此是同層項目 |

先看一段完整範例：

```html
<main>
  <section>
    <h1>HTML 基礎</h1>
    <p>HTML 用來描述網頁內容的結構。</p>
  </section>

  <section>
    <h2>前端學習順序</h2>
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </section>
</main>
```

這段程式碼中，`<main>` 包住兩個 `<section>`，每個 `<section>` 又包住自己的標題、段落或清單。這就是 HTML 的層級結構。

## 用親屬關係看懂 HTML 層級

HTML 元素之間常用親屬關係來描述，因為這樣比較容易說明誰包住誰、誰和誰同層。

### 父元素

父元素是「直接包住某個元素」的上一層元素。

![HTML 父元素關係示意圖](../../origin/測試/assets/images/tag-composition-relations-img-002-e7dad3.png)

以前面的範例來看，`<ul>` 是三個 `<li>` 的父元素，因為 `<li>` 直接放在 `<ul>` 裡面。

```html
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
```

### 子元素

子元素是「直接被某個元素包住」的下一層元素。

![HTML 子元素關係示意圖](../../origin/測試/assets/images/tag-composition-relations-img-003-1cbae7.png)

在這段 HTML 中，三個 `<li>` 都是 `<ul>` 的子元素。

### 祖先元素

祖先元素是不一定直接包住你，但位在你外層的所有元素。

![HTML 祖先元素關係示意圖](../../origin/測試/assets/images/tag-composition-relations-img-004-9cd56d.png)

以前面的完整範例來看，對 `<li>HTML</li>` 來說：

| 元素 | 關係 |
| --- | --- |
| `<ul>` | 父元素 |
| 第二個 `<section>` | 祖先元素 |
| `<main>` | 祖先元素 |

### 後代元素

後代元素是某個元素內部所有層級的元素，不只包含直接子元素。

![HTML 後代元素關係示意圖](../../origin/測試/assets/images/tag-composition-relations-img-005-eb1f13.png)

對 `<main>` 來說，裡面的 `<section>`、`<h1>`、`<p>`、`<h2>`、`<ul>`、`<li>` 都是它的後代元素。

### 兄弟元素

兄弟元素是擁有同一個父元素，並且位在同一層的元素。

![HTML 兄弟元素關係示意圖](../../origin/測試/assets/images/tag-composition-relations-img-006-e13b0d.png)

在這段 HTML 中，三個 `<li>` 彼此是兄弟元素：

```html
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
```

而完整範例中的兩個 `<section>` 也彼此是兄弟元素，因為它們同樣直接放在 `<main>` 裡面。

## 程式碼拆解：從一個元素往外看

假設我們觀察這個元素：

```html
<li>CSS</li>
```

放回完整結構後，它的關係可以這樣判斷：

```html
<main>
  <section>
    <h2>前端學習順序</h2>
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </section>
</main>
```

| 觀察對象 | 與 `<li>CSS</li>` 的關係 | 判斷方式 |
| --- | --- | --- |
| `<ul>` | 父元素 | 直接包住 `<li>CSS</li>` |
| `<section>` | 祖先元素 | 位在外層，但不是直接父元素 |
| `<main>` | 祖先元素 | 位在更外層 |
| `<li>HTML</li>` | 兄弟元素 | 同樣直接放在 `<ul>` 裡 |
| `<li>JavaScript</li>` | 兄弟元素 | 同樣直接放在 `<ul>` 裡 |

實務上看 HTML 結構時，可以先問三個問題：

1. 這個元素被誰直接包住？
2. 它裡面又包了哪些元素？
3. 哪些元素和它在同一層？

## 常見錯誤

### 錯誤一：結束標籤沒有對應

錯誤寫法：

```html
<p>這是一段文字。
```

問題在於 `<p>` 是雙標籤，應該有對應的 `</p>`。少了結束標籤時，瀏覽器可能會自動推測段落結束位置，但結果不一定符合你的預期。

正確寫法：

```html
<p>這是一段文字。</p>
```

### 錯誤二：巢狀順序交錯

錯誤寫法：

```html
<p>這是 <strong>重要文字</p></strong>
```

`<strong>` 在 `<p>` 裡面開始，就應該在 `</p>` 之前結束。標籤的開合順序要像括號一樣，後開始的要先結束。

正確寫法：

```html
<p>這是 <strong>重要文字</strong></p>
```

### 錯誤三：把單標籤當成可以包內容的雙標籤

錯誤寫法：

```html
<img src="photo.jpg">個人照片</img>
```

`<img>` 用來嵌入圖片，它本身不包住文字內容。圖片說明應該放在 `alt` 屬性，或使用其他元素搭配圖片。

正確寫法：

```html
<img src="photo.jpg" alt="個人照片">
```

如果需要圖片加上可見說明，可以另外使用 `<figure>` 與 `<figcaption>`：

```html
<figure>
  <img src="photo.jpg" alt="個人照片">
  <figcaption>我的個人照片</figcaption>
</figure>
```

## 實務使用建議

寫 HTML 結構時，先規劃「大區塊包小內容」，再往內放細節。這樣會比一邊想畫面、一邊隨意加標籤更穩定。

建議養成以下習慣：

- 使用縮排表示層級，讓父子關係一眼看得出來。
- 同一層的元素保持相同縮排，方便辨識兄弟元素。
- 寫完一組雙標籤後，立刻補上結束標籤。
- 不要只靠畫面判斷結構，應該回到 HTML 原始碼確認誰包住誰。
- 使用有語意的標籤描述內容，例如標題用 `<h1>` 到 `<h6>`，清單用 `<ul>`、`<ol>` 與 `<li>`。

## 重點整理

- 標籤名是 `<` 與 `>` 中間的英文名稱，例如 `<p>` 裡的 `p`。
- 雙標籤由起始標籤、內容、結束標籤組成，例如 `<p>文字</p>`。
- 單標籤通常不包住內容，例如 `<img>`、`<br>`、`<input>`。
- 巢狀關係表示一個元素包住另一個元素。
- 並列關係表示多個元素位在同一層。
- 父元素與子元素是直接上下層關係。
- 祖先元素與後代元素可以跨越多個層級。
- 兄弟元素擁有同一個父元素，並且位在同一層。

## 自我檢查

- 看到 `<p>文字</p>` 時，你能指出起始標籤、內容與結束標籤嗎？
- 你能說明雙標籤和單標籤的差異嗎？
- 在一段 HTML 中，你能找出某個元素的父元素與兄弟元素嗎？
- 你能判斷標籤巢狀順序是否正確嗎？
