# 瀏覽器與渲染引擎

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明瀏覽器在 Web 前端中的角色。
- 理解瀏覽器內核 / 渲染引擎負責什麼。
- 知道為什麼不同瀏覽器可能顯示出不同效果。
- 以正確方式看待早期教材中的「五大主流瀏覽器」說法。

## 問題情境

你寫了一段 HTML 和 CSS，在自己的瀏覽器看起來正常，但別人的電腦或手機上卻排版不同。這通常不是 HTML 檔案「變了」，而是瀏覽器、版本、渲染引擎或支援程度不同。

所以學前端不能只問「程式碼寫了什麼」，也要理解「瀏覽器如何解析與呈現它」。

## 一句話理解

瀏覽器是網頁顯示與運行的平台；渲染引擎是瀏覽器中負責解析網頁資源並把頁面畫出來的核心部分。

## 瀏覽器的角色

瀏覽器是使用者訪問 Web 的主要工具。它可以：

- 向服務器請求網頁資源。
- 讀取 HTML 文件。
- 載入 CSS、JavaScript、圖片、字型等資源。
- 解析結構、套用樣式、執行互動程式。
- 把最後結果呈現在螢幕上。

對 HTML 初學者來說，可以先記住：你寫的 HTML 最終是交給瀏覽器理解與顯示的。

## 主流瀏覽器

當代常見瀏覽器包含 Chrome、Edge、Firefox、Safari、Opera 等。早期教材常把 IE、Firefox、Chrome、Safari、Opera 稱為五大主流瀏覽器。

![五大主流瀏覽器與問世時間](../../origin/第01章_寫在前面/assets/images/browser-img-001-111cfe.png)

閱讀這類圖片時要注意：IE 屬於早期常見瀏覽器，現在已不屬於當代主流瀏覽器。它仍有歷史意義，因為很多前端相容性問題都曾與舊版 IE 有關。

入門階段的重點不是背每個瀏覽器年份，而是知道：

- 使用者可能使用不同瀏覽器。
- 不同瀏覽器可能有不同支援程度。
- 前端頁面需要考慮跨瀏覽器呈現。

## 瀏覽器內核與渲染引擎

瀏覽器內核在前端語境中常指渲染引擎。它負責讀取網頁內容、解析程式碼、計算顯示方式，並把頁面呈現出來。

![五大瀏覽器與內核對照圖](../../origin/第01章_寫在前面/assets/images/browser-img-002-b448ff.png)

瀏覽器拿到網頁資源後，大致會經過這樣的流程：

1. 讀取 HTML，建立頁面結構。
2. 讀取 CSS，計算每個元素的樣式。
3. 載入圖片、字型、腳本等其他資源。
4. 計算元素位置與大小。
5. 將結果繪製到螢幕。
6. 使用者互動時，根據需要重新計算或更新畫面。

![瀏覽器內核處理網頁資源示意圖](../../origin/第01章_寫在前面/assets/images/browser-img-003-ccff67.png)

這個流程不需要在入門階段背細節，但要知道：瀏覽器不是把 HTML 當圖片打開，而是一步步解析、計算、渲染。

## 為什麼會有渲染差異

不同瀏覽器可能使用不同渲染引擎；即使使用相近引擎，也可能因版本、平台或實作細節不同而產生差異。

常見差異來源包含：

| 差異來源 | 可能影響 |
|---|---|
| 瀏覽器引擎不同 | CSS 呈現、排版計算、功能支援 |
| 瀏覽器版本不同 | 新語法是否可用、舊錯誤是否修正 |
| 作業系統不同 | 字型、表單控制項、捲軸外觀 |
| 裝置尺寸不同 | 響應式排版、圖片顯示、觸控互動 |
| 預設樣式不同 | 標題、表單、按鈕、間距等預設呈現 |

因此，前端開發常會測試不同瀏覽器與裝置，並使用 Web 標準、重置樣式、漸進增強等方式降低差異。

## 實務觀念

### HTML 寫得正確，瀏覽器才容易正確解析

如果標籤巢狀錯誤、文件結構混亂、屬性使用不當，瀏覽器可能會自動修正，但不同瀏覽器的修正結果可能不同。

實務上應該盡量寫出清楚、符合標準的 HTML，而不是依賴瀏覽器猜測。

### 不要只在單一瀏覽器確認結果

如果網站要給真實使用者使用，只在自己的 Chrome 測試通常不夠。至少要考慮目標使用者常用的瀏覽器與裝置。

個人練習可以先以一個現代瀏覽器為主，但要逐步養成跨瀏覽器檢查習慣。

### 舊教材中的瀏覽器資訊要辨識時代背景

早期資料可能仍提到 IE 是主流，或用舊內核名稱整理瀏覽器。這些內容可以幫助理解歷史相容性問題，但不能直接當作最新市場現況。

## 常見錯誤與混淆

### 以為瀏覽器只是顯示圖片的工具

瀏覽器不是把網頁當成一張固定圖片顯示。它會解析 HTML、套用 CSS、執行 JavaScript，並根據視窗大小、裝置與使用者操作更新畫面。

避免方式：把網頁理解成瀏覽器動態解析出來的文件，而不是固定截圖。

### 以為不同瀏覽器一定顯示完全相同

Web 標準讓瀏覽器盡量一致，但不代表所有細節完全相同。尤其是新功能、舊版本、表單控制項與預設樣式，仍可能有差異。

避免方式：根據專案需求測試目標瀏覽器，不要只看單一環境。

### 以為「內核不同」是所有問題的唯一原因

渲染差異可能來自內核，也可能來自版本、CSS 寫法、預設樣式、裝置尺寸或作業系統。

避免方式：除錯時先用開發者工具檢查實際套用的 HTML、CSS 與資源載入，再判斷是否是瀏覽器支援問題。

## 對 HTML 學習的意義

HTML 是寫給瀏覽器解析的。理解瀏覽器與渲染引擎後，你會更重視：

- 文件結構是否清楚。
- 標籤語意是否正確。
- CSS 與 HTML 是否配合良好。
- 網頁是否能在不同瀏覽器中穩定呈現。
- 為什麼 Web 標準能降低瀏覽器差異。

## 重點整理

- 瀏覽器是網頁顯示與運行的平台。
- 渲染引擎負責解析網頁資源並呈現頁面。
- 不同瀏覽器、版本與平台可能造成顯示差異。
- 早期「五大主流瀏覽器」說法需要用歷史角度閱讀。
- 寫出符合標準的 HTML，有助於降低瀏覽器解析差異。

## 自我檢查

2. 渲染引擎大致做哪些事情？
3. 為什麼同一份網頁在不同瀏覽器可能看起來不同？
4. 為什麼舊教材提到 IE 時需要注意時代背景？
5. 學 HTML 時，為什麼不能完全依賴瀏覽器自動修正錯誤？
