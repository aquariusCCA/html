# HTML 字符實體基本概念

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 HTML 字符實體是什麼，以及為什麼需要它。
- 看懂常見的命名字符實體與數字字符引用。
- 正確使用 `&nbsp;`、`&lt;`、`&gt;`、`&amp;` 等常見字符實體。
- 避免把字符實體誤用成一般排版工具。

## 問題情境

假設你想在網頁上顯示這段文字：

```text
<p>Hello</p>
```

如果直接把 `<p>` 寫進 HTML，瀏覽器可能會把它當成真正的 HTML 標籤解析，而不是單純顯示成文字。

另一個常見情境是空白：

```html
hello     world
```

在一般 HTML 文字排版中，即使你在原始碼裡輸入多個連續空格，瀏覽器通常也只會顯示成一個空格。這些情況都需要用字符實體來處理。

## 一句話理解

字符實體就是「用一段特殊寫法，讓 HTML 正確顯示特殊符號或不方便直接輸入的字符」。

## 字符實體的基本形式

常見的命名字符實體由三部分組成：

```text
& + 實體名稱 + ;
```

例如：

```html
&copy;
```

這會在畫面上顯示版權符號：

```text
©
```

HTML 也支援數字字符引用，常見形式有兩種：

```html
&#169;
&#xA9;
```

它們同樣可以表示版權符號。初學時不需要背大量字符實體，先記住常用的幾個，其他需要時再查即可。

## 最常用的字符實體

| 想顯示的字符 | 字符實體 | 常見用途 |
| --- | --- | --- |
| 空格 | `&nbsp;` | 顯示不會被合併的空格 |
| `<` | `&lt;` | 在文字中顯示小於號或 HTML 開始標籤符號 |
| `>` | `&gt;` | 在文字中顯示大於號或 HTML 結束標籤符號 |
| `&` | `&amp;` | 在文字中顯示和號本身 |
| `©` | `&copy;` | 顯示版權符號 |

其中最應該熟悉的是 `&nbsp;`、`&lt;`、`&gt;`、`&amp;`。這幾個在寫 HTML 範例、說明文件或處理文字內容時很常見。

## 範例拆解

先看連續空格的例子：

```html
<!-- 原始碼中有 5 個空格，但畫面通常只顯示 1 個空格 -->
hello     world <br>

<!-- 使用 5 個 &nbsp;，畫面會保留 5 個空格效果 -->
hello&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;world <br>
```

第一行雖然在 `hello` 和 `world` 之間寫了多個空格，但瀏覽器在一般 HTML 文字排版中會把連續空白合併。

第二行使用 `&nbsp;`。`nbsp` 是 non-breaking space，意思是不換行空格。它不會像一般空格那樣被合併，因此可以用來顯示連續空格。

再看顯示 HTML 標籤文字的例子：

```html
&lt;p&gt;Hello&lt;/p&gt;
```

畫面會顯示：

```text
<p>Hello</p>
```

這裡的重點是：

- `&lt;` 顯示成 `<`。
- `&gt;` 顯示成 `>`。
- 瀏覽器看到的是字符實體，所以不會把它當成真正的 `<p>` 標籤執行。

## 實務使用原則

字符實體最常用在兩種情況：

1. 這個字符在 HTML 中有特殊意義，例如 `<`、`>`、`&`。
2. 這個字符不方便直接輸入，或直接輸入後顯示效果不符合預期，例如連續空格。

不過，字符實體不是排版系統。若只是想控制段落間距、元素位置、文字對齊，應該交給 CSS 處理，而不是堆很多個 `&nbsp;`。

## 常見錯誤

### 漏掉分號

錯誤寫法：

```html
&copy
```

正確寫法：

```html
&copy;
```

命名字符實體通常要以分號 `;` 結尾。漏掉分號可能導致瀏覽器解析結果不穩定，也會降低程式碼可讀性。

### 把 HTML 標籤直接寫成文字

如果你想教別人 `<h1>` 是標題標籤，這樣寫容易出問題：

```html
<h1> 是標題標籤
```

瀏覽器會把 `<h1>` 當成真正的標題開始標籤。應改成：

```html
&lt;h1&gt; 是標題標籤
```

這樣畫面才會顯示 `<h1>` 這段文字。

### 用大量 `&nbsp;` 做排版

不建議這樣寫：

```html
姓名：&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;小明
```

如果目的是控制表單、標籤或區塊之間的距離，應使用 CSS 的 `margin`、`padding`、`gap` 等排版方式。`&nbsp;` 適合處理文字中的特殊空格，不適合作為主要排版工具。

### 誤以為所有空白都會被保留

在一般 HTML 文字排版中，連續空格與換行通常會被合併顯示為一個空格。但這不是所有情況都相同，例如：

- `<pre>` 會保留文字中的空白與換行。
- `<textarea>` 會保留輸入內容的空白與換行。
- CSS 的 `white-space` 設定也可能改變空白顯示方式。

因此，看到空白顯示結果不同時，要先確認目前元素與 CSS 的空白處理規則。

## 重點整理

- 字符實體用來在 HTML 中表示特殊符號或不方便直接輸入的字符。
- 常見形式是 `&名稱;`，例如 `&copy;`。
- HTML 也支援數字字符引用，例如 `&#169;`、`&#xA9;`。
- 初學者優先記住 `&nbsp;`、`&lt;`、`&gt;`、`&amp;`。
- `&nbsp;` 可以保留特殊空格效果，但不應拿來取代 CSS 排版。
- 如果要在頁面上顯示 HTML 標籤文字，通常需要把 `<` 和 `>` 寫成 `&lt;` 和 `&gt;`。
