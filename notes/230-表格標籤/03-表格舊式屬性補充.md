# 表格舊式屬性補充：看懂 align、border、cellpadding 與 cellspacing

## 學習目標

讀完這篇筆記，你應該能夠：

- 看懂舊教材或舊專案中常見的表格外觀屬性。
- 說明 `align`、`border`、`cellpadding`、`cellspacing`、`width` 大致控制什麼。
- 知道這些屬性在現代實務中多半會改由 CSS 處理。
- 避免把 HTML 表格屬性誤當成精細樣式控制工具。

## 使用情境

你在讀早期 HTML 教材、維護舊網頁，或看見別人的範例程式時，可能會遇到這種寫法：

```html
<table align="center" border="1" cellpadding="20" cellspacing="0" width="500">
  <!-- 中間省略 ... -->
</table>
```

這些屬性確實可以影響表格外觀，但在現代前端開發中，表格的邊框、間距、寬度、對齊通常會交給 CSS 管理。這篇筆記的重點不是鼓勵你大量使用它們，而是讓你能讀懂它們，並知道它們的限制。

## 一句話理解

表格舊式屬性可以快速調整表格外觀，但它們只適合基礎展示或舊程式碼閱讀；正式樣式應優先使用 CSS。

## table 常見屬性

常見的 `<table>` 屬性包括：

- `align`：設定表格相對周圍元素的對齊方式，常見值有 `left`、`center`、`right`。
- `border`：設定表格邊框寬度，`border="0"` 或不寫通常表示無邊框。
- `cellpadding`：設定儲存格內容與儲存格邊界之間的距離。
- `cellspacing`：設定儲存格與儲存格之間的距離。
- `width`：設定表格寬度。

範例：

```html
<table align="center" border="1" cellpadding="20" cellspacing="0" width="500">
  <tr>
    <th>商品</th>
    <th>價格</th>
  </tr>
  <tr>
    <td>鍵盤</td>
    <td>1200</td>
  </tr>
</table>
```

## 範例拆解

`align="center"` 表示整張表格相對周圍內容置中。現代實務更常用 CSS 的 `margin` 或版面工具處理。

`border="1"` 表示表格顯示基本邊框。它適合教學或快速檢查表格結構，但不適合用來精細控制每條邊框的顏色、粗細與樣式。

`cellpadding="20"` 會讓儲存格內容與格線之間留出空間。現代 CSS 通常使用 `padding` 處理。

`cellspacing="0"` 會讓儲存格之間的空隙變成 0。現代 CSS 常見做法是使用 `border-collapse` 等表格樣式屬性。

`width="500"` 設定表格寬度。實務上更常透過 CSS 控制固定寬度、百分比寬度或響應式寬度。

學習這些屬性時，要把它們放在「讀懂舊程式碼」的位置，而不是把它們當成主要寫法。HTML 負責資料結構與語意，CSS 負責視覺表現，這個分工會讓後續維護更清楚。

## 舊式屬性與 CSS 的分工

| 目的 | 舊式 HTML 屬性 | 現代實務方向 |
| --- | --- | --- |
| 表格對齊 | `align` | CSS 版面與 `margin` |
| 邊框 | `border` | CSS `border`、`border-collapse` |
| 儲存格內距 | `cellpadding` | CSS `padding` |
| 儲存格間距 | `cellspacing` | CSS `border-spacing` 或 `border-collapse` |
| 表格寬度 | `width` | CSS `width`、`max-width`、響應式設定 |

這不代表你完全不能讀或用 HTML 屬性，而是要知道它們的角色有限。若是正式專案，外觀控制集中在 CSS，會比混在 HTML 屬性裡更容易維護。

## 實務注意事項

`border` 主要提供表格邊框的基礎顯示，不適合用來精細控制每個儲存格的框線。若要控制單元格邊框、框線合併、框線顏色或不同方向的邊線，應使用 CSS。

給某個 `<th>` 或 `<td>` 設定寬度後，通常會影響它所在欄的寬度。這是因為表格會根據欄位進行排版，而不是只把那一格孤立處理。

給某個 `<th>` 或 `<td>` 設定高度後，通常會影響同一列的高度。不過實際高度仍可能受到內容、字體、換行與表格排版規則影響，不能把它理解成永遠絕對固定。

## 常見錯誤

**把舊式屬性當成現代主要寫法**

初學者容易看到 `border`、`cellpadding` 很方便，就把所有表格樣式都寫在 HTML。短範例可以接受，但正式頁面會讓結構與樣式混在一起，後續修改成本高。

**以為 `border` 能控制所有框線細節**

`border="1"` 可以讓表格出現基本邊框，但如果要控制單元格框線、邊框顏色、雙線或合併邊框，應使用 CSS。不要期待 HTML 的 `border` 屬性完成精細設計。

**誤解寬高只影響單一儲存格**

表格是按列與欄一起排版的。你設定某個儲存格寬度，常會影響整欄；設定某個儲存格高度，常會影響整列。若不了解這點，很容易覺得表格樣式不受控制。

**為了調整距離而破壞表格語意**

如果只是想讓資料看起來有空隙，不應新增空白列或空白欄來撐版面。這會製造沒有資料意義的儲存格，應改用 CSS 間距處理。

## 重點整理

- `align`、`border`、`cellpadding`、`cellspacing`、`width` 常見於舊式表格寫法。
- 這些屬性可以讀懂，但現代實務多用 CSS 控制外觀。
- HTML 表格的核心仍是資料結構與語意，不是樣式控制。
- 表格寬高常會影響整欄或整列，不一定只影響單一儲存格。
