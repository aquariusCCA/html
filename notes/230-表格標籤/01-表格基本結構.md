# 表格基本結構：用 table 表達資料關係

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明表格適合用來呈現什麼類型的內容。
- 看懂 `<table>`、`<tr>`、`<th>`、`<td>` 之間的層級關係。
- 使用 `<caption>`、`<thead>`、`<tbody>`、`<tfoot>` 建立語意更清楚的表格。
- 避免把表格誤用成網頁排版工具。

## 使用情境

當資料本身有「列」與「欄」的對應關係時，就適合使用 HTML 表格。例如學生段考成績、後台會員清單、訂單明細、產品規格比較表，都需要讓讀者沿著欄位標題快速比對每一筆資料。

表格不是用來控制版面位置的工具。它的主要任務是呈現結構化資料，讓人、瀏覽器與輔助技術都能理解「這些資料彼此如何對應」。

## 一句話理解

`<table>` 建立資料表，`<tr>` 建立每一列，`<th>` 與 `<td>` 建立每一格資料，其中 `<th>` 表示標題儲存格，`<td>` 表示一般資料儲存格。

## 基本層級

一個最基本的表格通常有三層：

```text
table
  tr
    th 或 td
```

- `<table>`：整張表格的容器。
- `<tr>`：表格中的一列，必須放在表格結構內。
- `<th>`：表頭儲存格，通常用來表示欄位名稱或列標題。
- `<td>`：資料儲存格，放實際資料內容。

`td` 可以理解為 table data，也就是表格資料。初學時可以先記住：一列用 `<tr>` 包起來，一列裡面的每一格再用 `<th>` 或 `<td>` 表示。

## 基本範例

```html
<table>
  <tr>
    <th>姓名</th>
    <th>性別</th>
    <th>年齡</th>
  </tr>
  <tr>
    <td>劉德華</td>
    <td>男</td>
    <td>56</td>
  </tr>
  <tr>
    <td>張學友</td>
    <td>男</td>
    <td>58</td>
  </tr>
  <tr>
    <td>郭富城</td>
    <td>男</td>
    <td>51</td>
  </tr>
  <tr>
    <td>黎明</td>
    <td>男</td>
    <td>57</td>
  </tr>
</table>
```

## 範例拆解

第一列使用 `<th>`，因為「姓名」「性別」「年齡」不是普通資料，而是用來說明後面每一欄代表什麼。

後面的每一列使用 `<td>` 放入實際資料。例如「劉德華」「男」「56」是同一筆資料中的三個欄位，所以它們放在同一個 `<tr>` 裡。

如果你從橫向閱讀一列，可以看到一筆完整資料；如果你從縱向閱讀一欄，可以比較同一種欄位的不同資料。這就是表格適合呈現資料的原因。

## 更完整的表格結構

當表格不只一兩列，而是有明確的標題、表頭、主要資料與統計結果時，可以使用更完整的語意結構：

- `<caption>`：表格標題，用來說明這張表格在呈現什麼。
- `<thead>`：表格頭部，通常放欄位標題。
- `<tbody>`：表格主體，放主要資料列。
- `<tfoot>`：表格腳注，常放總計、平均或補充結果。

![表格結構與區塊標籤示意](../../origin/230-表格標籤/assets/images/html-table-tags-img-001-422a27.png)

## 完整結構範例

```html
<table border="1">
  <caption>段考成績</caption>
  <thead>
    <tr>
      <th>科目</th>
      <th>分數</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>語文</td>
      <td>99</td>
    </tr>
    <tr>
      <td>數學</td>
      <td>60</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>總分</td>
      <td>159</td>
    </tr>
  </tfoot>
</table>
```

## 完整結構拆解

`<caption>` 告訴讀者這張表格是「段考成績」。它不是普通的一列資料，而是整張表格的標題。

`<thead>` 放欄位標題。這裡的「科目」與「分數」會幫助讀者理解下方每一格資料的意義。

![thead 表格頭部與 th 單元格示意](../../origin/230-表格標籤/assets/images/html-table-tags-img-002-5f8921.png)

`<tbody>` 放主要資料，也就是各科成績。表格通常可以有很多資料列，這些列應放在表格主體中。

![tbody 表格主體與 tr 行示意](../../origin/230-表格標籤/assets/images/html-table-tags-img-003-99d970.png)

`<tfoot>` 放總計或腳注。即使它在 HTML 中寫在最後，瀏覽器與輔助技術仍能理解這一區是表格的總結性內容。

![tfoot 表格腳注與 td 單元格示意](../../origin/230-表格標籤/assets/images/html-table-tags-img-004-6abf79.png)

## 實務判斷

如果內容能自然放進「列」與「欄」的交叉關係中，就可以考慮表格。例如「商品名稱、價格、庫存」就是典型表格資料。

如果你只是想讓圖片在左、文字在右，或想把頁面切成幾個區塊，就不應該使用表格。這類版面安排應交給 CSS 處理。

建立表格時，先不要急著寫很多屬性。先問三件事：

1. 這張表格要呈現什麼資料？
2. 每一欄代表什麼？
3. 每一列是否是一筆完整資料？

這三件事清楚後，再決定是否需要 `<caption>`、`<thead>`、`<tbody>`、`<tfoot>`。

## 常見錯誤

**用表格做版面排版**

早期網頁常用表格排版，但這會讓 HTML 語意混亂。畫面可能看起來正確，實際上卻把「資料表」誤當成「版面容器」，輔助技術也可能把排版區塊讀成表格資料。

**把 `<tr>` 放在錯誤位置**

`<tr>` 是表格列，應該出現在 `<table>` 的表格結構中，或放在 `<thead>`、`<tbody>`、`<tfoot>` 之內。不要把 `<tr>` 當成一般區塊標籤拿到表格外使用。

**表頭全部用 `<td>`**

如果第一列是欄位名稱，應優先使用 `<th>`。只用 `<td>` 雖然畫面仍可能顯示，但語意上會失去「這格是標題」的資訊。

**只為了預設樣式使用 `<th>`**

瀏覽器通常會讓 `<th>` 文字加粗置中，但使用 `<th>` 的理由應該是語意，而不是樣式。如果只是想讓文字變粗，應該用 CSS 控制。

## 重點整理

- 表格用來呈現結構化資料，不用來做版面排版。
- `<table>` 是表格，`<tr>` 是列，`<th>` 是標題儲存格，`<td>` 是資料儲存格。
- `<caption>`、`<thead>`、`<tbody>`、`<tfoot>` 能讓表格結構更清楚。
- 先確認資料的列欄關係，再撰寫表格標籤。
