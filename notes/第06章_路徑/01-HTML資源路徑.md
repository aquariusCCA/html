# HTML 資源路徑：找到圖片與檔案的位置

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解為什麼 HTML 需要使用路徑找到圖片或其他檔案。
- 分辨絕對路徑與相對路徑的差異。
- 在 HTML 中用正確的相對路徑引用圖片。
- 避免使用容易失效的本機絕對路徑或外站圖片路徑。

## 問題情境

網頁中通常不會只有一個 HTML 檔案。實際專案常常會把圖片放在 `images/`，把 CSS 放在 `css/`，把 JavaScript 放在 `js/`，讓檔案比較好管理。

當圖片不在 HTML 檔案旁邊時，瀏覽器就需要依照你寫的路徑，找到那張圖片的位置。

這和生活中要去找某個地點很像：你需要知道從哪裡出發、要往哪裡走，才找得到目的地。

![交通路線規劃示意圖](../../origin/第06章_路徑/assets/images/path-img-001-d6a090.png)

在 HTML 中，路徑就是「告訴瀏覽器資源在哪裡」的寫法。

## 一句話理解

路徑是 HTML 檔案用來找到圖片、連結檔案或其他資源的位置描述。

常見路徑可以分成兩類：

- 絕對路徑：從固定的根位置或完整網址開始找。
- 相對路徑：從目前這份 HTML 檔案所在的位置開始找。

## 絕對路徑

絕對路徑指的是從一個固定起點出發，可以直接到達目標位置的路徑。

常見例子：

```text
D:\day01\images\1.jpg
https://www.itcast.cn/2018czgw/images/logo.gif
```

第一個是 Windows 電腦上的本機磁碟路徑；第二個是完整的網路圖片網址。

在 HTML 中，完整網址可以這樣使用：

```html
<img src="https://www.example.com/images/logo.png" alt="網站標誌">
```

這種寫法的優點是位置明確，只要網址有效，瀏覽器就能直接請求該資源。

不過，初學者要特別注意兩件事。

第一，不要在網頁專案中依賴自己電腦的本機絕對路徑：

```html
<img src="D:\day01\images\1.jpg" alt="本機圖片">
```

這種寫法只對你的電腦有意義。換一台電腦、換一個資料夾位置，或把網站部署到伺服器後，路徑就很容易失效。

第二，不要隨便把別人網站上的圖片網址拿來當自己的圖片來源。外站伺服器可能開啟防盜鏈，導致你在瀏覽器直接打開圖片網址時看得到，但放進自己的網頁後卻載入失敗。

## 相對路徑

相對路徑是從「目前這份 HTML 檔案所在的位置」出發，去尋找目標資源。

例如以下專案結構：

```text
project/
  index.html
  images/
    logo.png
  pages/
    about.html
```

如果現在正在編寫 `index.html`，而圖片在同一層的 `images/` 資料夾裡，可以這樣寫：

```html
<img src="images/logo.png" alt="網站標誌">
```

也可以明確寫出 `./`，表示「從目前目錄開始」：

```html
<img src="./images/logo.png" alt="網站標誌">
```

這兩種寫法在這個例子中代表同一件事。

如果現在正在編寫 `pages/about.html`，要引用上一層 `images/` 裡的 `logo.png`，就需要先回到上一層：

```html
<img src="../images/logo.png" alt="網站標誌">
```

相對路徑的判斷重點不是「圖片看起來在哪裡」，而是「目前這份 HTML 檔案從自己的位置出發，要怎麼走到圖片」。

![相對路徑分類與符號說明表](../../origin/第06章_路徑/assets/images/path-img-002-4c2e46.png)

## 常見相對路徑符號

| 寫法 | 意思 | 範例 |
| --- | --- | --- |
| `./` | 目前目錄 | `./images/logo.png` |
| 省略 `./` | 也是從目前目錄開始 | `images/logo.png` |
| `../` | 回到上一層目錄 | `../images/logo.png` |
| `資料夾名/檔名` | 進入下一層資料夾找檔案 | `images/logo.png` |

`./` 通常可以省略，所以 `./images/logo.png` 常簡寫成 `images/logo.png`。

但 `/images/logo.png` 不等於 `images/logo.png`。開頭的 `/` 通常表示從網站根目錄開始找，不是「下一層資料夾」。

## 實務判斷

初學 HTML 時，引用自己專案內的圖片，通常優先使用相對路徑：

```html
<img src="images/product.jpg" alt="商品照片">
```

這樣整個專案資料夾一起搬移時，只要內部檔案結構沒有改變，路徑仍然可以正常運作。

如果你把 HTML 檔案移到其他資料夾，相對路徑就要跟著修改。因為相對路徑的參考點是「目前這份 HTML 檔案的位置」，不是整個專案永遠固定的位置。

## 常見錯誤

### 錯誤一：使用自己電腦的本機路徑

```html
<img src="D:\day01\images\1.jpg" alt="圖片">
```

這種寫法換到其他環境很容易失效。

比較好的做法是把圖片放進專案資料夾，改用相對路徑：

```html
<img src="images/1.jpg" alt="圖片">
```

### 錯誤二：把 `/` 開頭誤以為是下一層

```html
<img src="/images/logo.png" alt="網站標誌">
```

這通常表示從網站根目錄開始找，不是從目前 HTML 檔案旁邊的 `images/` 開始找。

如果圖片資料夾就在目前 HTML 檔案旁邊，應該寫成：

```html
<img src="images/logo.png" alt="網站標誌">
```

### 錯誤三：搬動 HTML 檔案後沒有更新路徑

原本 `index.html` 和 `images/` 在同一層時：

```html
<img src="images/logo.png" alt="網站標誌">
```

如果把 HTML 移到 `pages/` 資料夾內，就要改成：

```html
<img src="../images/logo.png" alt="網站標誌">
```

## 重點整理

- 路徑用來告訴瀏覽器圖片或檔案在哪裡。
- 絕對路徑從固定位置或完整網址開始找。
- 本機絕對路徑不適合用在正式網頁專案中。
- 相對路徑從目前 HTML 檔案所在位置開始找。
- `./` 表示目前目錄，通常可以省略。
- `../` 表示上一層目錄。
- `/` 開頭通常是網站根目錄，不是下一層資料夾。
- 移動 HTML 檔案或資源檔案後，相對路徑也要重新檢查。
