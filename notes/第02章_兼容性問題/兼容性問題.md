# HTML 兼容性問題

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明為什麼不同瀏覽器可能會出現 HTML 與 CSS 顯示差異。
- 使用 `X-UA-Compatible` 讓舊版 Internet Explorer 優先採用較新的文件模式。
- 理解 `renderer` meta 標籤在部分雙核瀏覽器中的用途與限制。
- 說明 `html5shiv` 如何協助 IE6、IE7、IE8 識別 HTML5 語意化元素。
- 看懂 IE 條件註解的基本判斷語法。
- 分辨舊版瀏覽器相容處理與現代 HTML 開發慣例的差異。

## 2. 本章定位

本章說明 HTML 開發中常見的舊版瀏覽器兼容性處理，重點集中在 Internet Explorer 與部分雙核瀏覽器的渲染行為。

在現代 HTML 開發中，大多數專案會優先面向現代瀏覽器，因此不一定需要使用本章提到的所有相容寫法。不過，若專案仍需維護舊系統、企業內部系統，或需要支援特定舊版瀏覽器，理解這些寫法可以幫助你判斷舊程式碼的目的，也能避免誤刪與誤用。

> 來源重點：原始資料主要包含三個方向：透過 meta 控制瀏覽器渲染模式、透過 `renderer` 指定雙核瀏覽器優先使用 WebKit 內核，以及透過 `html5shiv` 讓低版本 IE 識別 HTML5 新元素。

## 3. 核心概念

### 3.1 什麼是瀏覽器兼容性問題

瀏覽器兼容性問題是指同一份 HTML、CSS 或 JavaScript，在不同瀏覽器、不同版本，甚至不同瀏覽模式下，呈現或執行結果不一致。

造成差異的原因可能包含：

- 瀏覽器支援的 HTML、CSS、JavaScript 特性不同。
- 舊版瀏覽器對標準的實作不完整或行為不同。
- Internet Explorer 曾提供不同的文件模式，例如標準模式與舊版相容模式。
- 部分瀏覽器可能內建多個渲染引擎，並依網站或設定切換。

HTML 本身負責描述內容與結構，但瀏覽器需要解析 HTML 並建立 DOM，再搭配 CSS 與 JavaScript 渲染頁面。如果瀏覽器對元素、屬性或樣式的理解不同，最後顯示結果就可能不同。

### 3.2 文件模式與 `X-UA-Compatible`

舊版 Internet Explorer 可能會根據文件宣告、相容性設定或網站環境，使用不同的文件模式渲染頁面。文件模式會影響 IE 如何解釋 HTML、CSS 與部分瀏覽器行為。

`X-UA-Compatible` 是一個面向 Internet Explorer 的相容性設定。常見寫法如下：

```html
<meta http-equiv="X-UA-Compatible" content="IE=edge">
```

這段設定的意思是：如果瀏覽器是支援此設定的 IE，請盡量使用該 IE 版本可用的最新文件模式來渲染頁面。

> 注意：`IE=edge` 不能讓舊版 IE 支援它原本不支援的所有 HTML5、CSS3 或 JavaScript 功能。它主要是避免 IE 落入較舊的文件模式，而不是萬能的相容性修復工具。

### 3.3 雙核瀏覽器與 `renderer`

原始資料提到的「雙核瀏覽器」，通常是指某些瀏覽器同時提供兩種渲染內核，例如：

- WebKit 或 Blink 類型的現代渲染內核。
- IE 的 Trident 內核。

部分瀏覽器會依網站、使用者設定或相容模式，自動切換不同內核。這可能導致同一個頁面在不同模式下呈現結果不同。

`renderer` meta 標籤是一種用來提示部分雙核瀏覽器優先使用指定內核的寫法：

```html
<meta name="renderer" content="webkit">
```

這段設定的目的，是提示支援此寫法的雙核瀏覽器優先使用 WebKit 內核渲染頁面。

> 注意：`renderer` 不是 HTML 標準中的通用能力，主要是特定瀏覽器環境中的相容處理。現代專案不應把它當成所有瀏覽器都會遵守的標準 meta 標籤。

### 3.4 `html5shiv` 與 HTML5 語意化元素

HTML5 新增了許多語意化元素，例如：

- `header`
- `nav`
- `main`
- `section`
- `article`
- `aside`
- `footer`

現代瀏覽器可以正確解析這些元素。但在 IE6、IE7、IE8 這類舊版瀏覽器中，這些新元素可能不會被正確識別，導致 CSS 樣式套用或 DOM 行為出現問題。

`html5shiv` 是一段 JavaScript，主要用來讓 IE6、IE7、IE8 能夠建立並識別 HTML5 新元素，使這些元素比較能被樣式化。

> 注意：`html5shiv` 不是讓舊版 IE 完整支援 HTML5 的工具。它主要處理舊版 IE 對部分 HTML5 元素識別不足的問題，不能補齊所有新 API、表單功能或 CSS 能力。

### 3.5 IE 條件註解

IE 條件註解是 Internet Explorer 曾經支援的一種特殊註解語法。對非 IE 瀏覽器來說，它看起來只是一般 HTML 註解；對支援條件註解的 IE 來說，符合條件時會執行其中的 HTML 內容。

常見用途是只針對舊版 IE 載入相容性腳本，例如 `html5shiv`：

```html
<!--[if lt IE 9]>
<script src="./js/html5shiv.min.js"></script>
<![endif]-->
```

這段程式碼表示：如果瀏覽器是低於 IE9 的 IE，才載入 `html5shiv.min.js`。

> 注意：原始資料提到 IE10 不支援條件註解，因此需要將 IE10 視為不會執行這類條件註解的瀏覽器。

## 4. 語法與基本用法

### 4.1 建議放置位置

這類與瀏覽器解析、渲染模式有關的 meta 標籤，通常會放在 `<head>` 前段，讓瀏覽器盡早讀取。

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="renderer" content="webkit">
  <title>舊版瀏覽器相容處理範例</title>
</head>
<body>
  <h1>HTML 兼容性問題</h1>
</body>
</html>
```

各設定說明如下：

| 寫法 | 主要用途 | 適用範圍 |
|---|---|---|
| `<meta http-equiv="X-UA-Compatible" content="IE=edge">` | 提示 IE 使用最新可用文件模式 | 舊版 IE 相容處理 |
| `<meta name="renderer" content="webkit">` | 提示部分雙核瀏覽器優先使用 WebKit 內核 | 特定雙核瀏覽器環境 |
| `html5shiv` | 讓 IE6 至 IE8 較能識別 HTML5 新元素 | 低版本 IE |

### 4.2 使用 `html5shiv`

若專案需要支援低於 IE9 的瀏覽器，也就是 IE8 與更舊版本，且頁面使用 HTML5 語意化元素，可以透過條件註解只讓舊版 IE 載入 `html5shiv`。

```html
<!--[if lt IE 9]>
<script src="./js/html5shiv.min.js"></script>
<![endif]-->
```

這樣做的好處是：

- 現代瀏覽器不會載入這段舊版相容腳本。
- 只有符合條件的舊版 IE 會執行。
- 可以降低對其他瀏覽器的無意義負擔。

### 4.3 IE 條件註解的比較符號

IE 條件註解可以使用一些比較符號判斷版本。

| 條件 | 意義 |
|---|---|
| `lt` | 小於 |
| `lte` | 小於等於 |
| `gt` | 大於 |
| `gte` | 大於等於 |
| `!` | 邏輯非 |

常見範例如下：

```html
<!--[if IE 8]>僅 IE8 可見<![endif]-->
<!--[if gt IE 8]>高於 IE8 的 IE 可見<![endif]-->
<!--[if lt IE 8]>低於 IE8 的 IE 可見<![endif]-->
<!--[if gte IE 8]>IE8 或更新的 IE 可見<![endif]-->
<!--[if lte IE 8]>IE8 及以下可見<![endif]-->
<!--[if !IE 8]>非 IE8 的 IE 可見<![endif]-->
```

> 注意：這些語法屬於歷史相容處理，主要用於閱讀與維護舊專案。新專案通常不應以 IE 條件註解作為主要相容策略。

## 5. 範例說明

### 範例：舊版 IE 與雙核瀏覽器相容設定

```html
<!doctype html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="X-UA-Compatible" content="IE=edge">
  <meta name="renderer" content="webkit">
  <title>兼容性設定範例</title>

  <!--[if lt IE 9]>
  <script src="./js/html5shiv.min.js"></script>
  <![endif]-->
</head>
<body>
  <header>
    <h1>網站標題</h1>
  </header>

  <main>
    <article>
      <h2>文章標題</h2>
      <p>這是一段使用 HTML5 語意化元素撰寫的內容。</p>
    </article>
  </main>

  <footer>
    <p>頁尾資訊</p>
  </footer>
</body>
</html>
```

這個範例中：

- `<!doctype html>` 讓瀏覽器以 HTML5 標準模式解析文件。
- `X-UA-Compatible` 用於提示 IE 使用最新可用文件模式。
- `renderer` 用於提示部分雙核瀏覽器優先使用 WebKit 內核。
- `html5shiv` 只會在低於 IE9 的 IE 中被載入。
- `header`、`main`、`article`、`footer` 是 HTML5 語意化元素，用來描述頁面區塊的意義。

> 補充說明：`main` 是 HTML5 語意化元素之一，但非常舊的瀏覽器可能不認識它。若專案需要支援這些環境，除了使用 `html5shiv`，也應搭配實際測試確認 CSS 與互動行為是否符合需求。

## 6. 實務應用場景

### 6.1 維護舊系統

在企業內部系統、舊版後台或早期網站中，可能仍會看到 `X-UA-Compatible`、IE 條件註解與 `html5shiv`。這些寫法通常不是為了現代瀏覽器，而是為了避免舊版 IE 進入不合預期的渲染模式，或讓舊版 IE 能處理部分 HTML5 標籤。

### 6.2 判斷能否移除舊相容程式碼

如果專案已經不需要支援 IE6、IE7、IE8，通常可以評估移除 `html5shiv`。但移除前應確認：

- 專案支援的瀏覽器版本範圍。
- 使用者或客戶是否仍有舊版 IE 使用情境。
- 自動化測試或手動測試是否涵蓋目標瀏覽器。
- CSS 與 JavaScript 是否依賴舊相容腳本產生的行為。

### 6.3 調查畫面差異

若同一頁面在不同瀏覽器看起來不同，應先確認：

- 是否使用正確的 `<!doctype html>`。
- 是否進入了舊版 IE 的相容模式。
- 是否有瀏覽器不支援的 HTML、CSS 或 JavaScript 特性。
- 是否有特定瀏覽器內核切換造成差異。
- 是否需要針對支援範圍加入 polyfill、fallback 或替代寫法。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 以為 `IE=edge` 可以解決所有兼容性問題 | 它主要影響 IE 文件模式，不能補齊所有標準支援 | 搭配功能支援檢查、fallback、polyfill 與實際測試 |
| 把 `renderer` 當成所有瀏覽器都支援的標準 | `renderer` 主要針對特定雙核瀏覽器，並非通用標準 | 只在確有目標瀏覽器需求時使用，並註明目的 |
| 在所有瀏覽器都載入 `html5shiv` | `html5shiv` 對非舊版 IE 通常沒有意義 | 使用 IE 條件註解只讓低於 IE9 的瀏覽器載入 |
| 以為 `html5shiv` 等於完整支援 HTML5 | 它主要處理 HTML5 元素識別問題，不包含所有 HTML5 API | 根據需要另外處理 CSS、JavaScript 或 API 相容性 |
| 沒有寫 `<!doctype html>` | 瀏覽器可能進入怪異模式或非預期解析模式 | 在 HTML 文件最前面加入 `<!doctype html>` |
| 新專案照抄舊版 IE 相容程式碼 | 會增加無用程式碼與維護成本 | 根據專案瀏覽器支援範圍決定是否保留 |

### 錯誤範例

```html
<head>
  <meta charset="UTF-8">
  <title>新專案</title>
  <script src="./js/html5shiv.min.js"></script>
</head>
```

問題：如果專案不支援低於 IE9 的瀏覽器，直接載入 `html5shiv` 沒有實質必要；如果真的需要支援舊版 IE，也不應讓所有瀏覽器都載入這個腳本。

### 正確範例

```html
<head>
  <meta charset="UTF-8">
  <title>需要支援舊版 IE 的專案</title>

  <!--[if lt IE 9]>
  <script src="./js/html5shiv.min.js"></script>
  <![endif]-->
</head>
```

修正重點：使用 IE 條件註解，讓相容腳本只在需要的舊版 IE 環境中載入。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| HTML 語意化 | `html5shiv` 主要與 HTML5 語意化元素在舊版 IE 的識別問題有關。 |
| DOM | 瀏覽器解析 HTML 後會建立 DOM；舊版瀏覽器若不認識新元素，DOM 與樣式套用可能受影響。 |
| CSS | 兼容性問題常表現在樣式差異，例如舊版瀏覽器不支援某些選擇器或屬性。 |
| JavaScript | 舊瀏覽器可能缺少現代 JavaScript API，不能只靠 HTML meta 解決。 |
| Polyfill | `html5shiv` 可視為一種面向舊版 IE 的相容補丁，但它只處理特定問題。 |
| 瀏覽器支援範圍 | 是否使用這些相容寫法，應由專案要支援的瀏覽器與版本決定。 |

## 9. 重點整理

- 瀏覽器兼容性問題來自不同瀏覽器與不同版本對標準支援和渲染行為的差異。
- `X-UA-Compatible` 常用於提示舊版 IE 使用最新可用文件模式。
- `renderer` 是特定雙核瀏覽器環境中的內核提示，不是通用 HTML 標準。
- `html5shiv` 主要用於讓 IE6、IE7、IE8 識別 HTML5 語意化元素。
- IE 條件註解可以讓特定版本的 IE 載入專用程式碼，但 IE10 不支援這類條件註解。
- 兼容性處理應根據專案支援範圍決定，不應在新專案中無條件照抄舊寫法。

## 11. 延伸補充

> 補充說明：兼容性處理不應只依賴固定程式碼片段。更完整的做法是先定義專案的瀏覽器支援範圍，再根據實際需求選擇 `doctype`、meta 設定、polyfill、fallback、CSS 前綴或替代實作，最後透過目標瀏覽器測試確認結果。

> 待確認：原始資料未提供目標專案實際需要支援的瀏覽器版本，因此本筆記只能將這些寫法整理為舊版瀏覽器相容處理，是否應在特定專案中使用仍需依支援範圍判斷。
