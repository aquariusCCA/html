# progress 標籤：顯示任務完成進度

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `<progress>` 標籤在 HTML 中代表的語意。
- 使用 `value` 與 `max` 屬性建立可讀的進度條。
- 分辨 `<progress>` 適合用在什麼情境，避免把它當成一般裝飾用長條。

## 使用情境

當網頁需要讓使用者知道「某件事已經完成多少」時，就可以使用 `<progress>`。

常見情境包括：

- 檔案上傳到一半，顯示目前已上傳 50%。
- 檔案下載時，顯示已下載多少。
- 表單送出或資料處理時，顯示目前處理進度。
- 載入大量資料時，顯示已完成的比例。

這類情境的共同點是：任務有一個「目前進度」與「目標總量」。`<progress>` 正是用來表達這種進度狀態的 HTML 標籤。

## 一句話理解

`<progress>` 是用來表示「一項任務目前完成到哪裡」的 HTML5 語意標籤。

## 基本語法

```html
<span>當前進度: </span>
<progress value="50" max="100"></progress>
```

這段程式碼會建立一個進度條，表示目前進度是 `50`，目標值是 `100`，也就是完成了一半。

## 範例拆解

```html
<span>當前進度: </span>
<progress value="50" max="100"></progress>
```

- `<span>當前進度: </span>`：提供文字說明，讓使用者知道這個進度條代表什麼。
- `<progress>`：建立進度條。
- `value="50"`：目前完成的數值。
- `max="100"`：任務完成時的目標值。

瀏覽器會根據 `value / max` 的比例顯示進度。以上例來說，`50 / 100` 代表完成 50%。

## 常用屬性

### max

`max` 用來指定進度完成時的目標值。

```html
<progress value="3" max="10"></progress>
```

這表示目前完成 `3`，總目標是 `10`。

### value

`value` 用來指定目前進度。

```html
<progress value="75" max="100"></progress>
```

這表示目前完成 75%。

如果省略 `value`，進度條會變成「不確定進度」的狀態，適合用在只知道正在處理、但不知道完成比例的情境。

```html
<progress max="100"></progress>
```

## 正確使用方式

如果你知道任務的目前值與總目標，可以明確寫出 `value` 與 `max`。

```html
<label for="upload-progress">上傳進度</label>
<progress id="upload-progress" value="40" max="100"></progress>
```

這個寫法比只放一條進度條更清楚，因為文字標籤說明了進度條的用途。

## 常見錯誤

### 把 progress 當成純裝飾線條

錯誤寫法：

```html
<progress value="80" max="100"></progress>
```

如果畫面上完全沒有文字說明，使用者可能不知道這條進度條代表上傳、下載還是其他任務。實務上應該搭配文字、`label` 或附近的說明文字。

較好的寫法：

```html
<label for="download-progress">下載進度</label>
<progress id="download-progress" value="80" max="100"></progress>
```

### value 大於 max

錯誤寫法：

```html
<progress value="120" max="100"></progress>
```

`value` 代表目前進度，`max` 代表完成目標。當目前值大於目標值時，語意會變得不合理，也會讓後續維護者難以判斷真實狀態。

較好的寫法：

```html
<progress value="100" max="100"></progress>
```

### 用 progress 表示一般數值

`<progress>` 適合表示任務進度，不適合用來表示評分、溫度、庫存量這類一般數值。

如果你要表示「某個範圍內的測量值」，通常應該考慮其他更符合語意的做法，而不是直接把所有長條狀畫面都寫成 `<progress>`。

## 實務判斷準則

使用 `<progress>` 前，可以先問自己兩個問題：

1. 這個畫面是在表示一項任務的完成進度嗎？
2. 是否有目前值與目標值，或至少有「正在進行但比例未知」的狀態？

如果答案是肯定的，`<progress>` 通常是合適的選擇。

如果只是要做一條視覺上的分隔線、裝飾線、評分條或普通數值條，就不應該只因為外觀看起來像進度條而使用 `<progress>`。

## 重點整理

- `<progress>` 是 HTML5 的進度條標籤，用來表示任務完成進度。
- `value` 表示目前值，`max` 表示目標值。
- `value / max` 會形成進度比例。
- 已知進度時使用 `value` 與 `max`，未知進度時可以省略 `value`。
- 實務上應搭配文字或標籤說明進度條代表的任務。

## 自我檢查

1. 如果目前上傳了 30%，而總進度以 100 為滿值，`value` 與 `max` 應該怎麼寫？
2. 只知道資料正在載入，但不知道完成比例時，可以省略哪一個屬性？
3. 為什麼不應該把所有看起來像長條的 UI 都寫成 `<progress>`？
