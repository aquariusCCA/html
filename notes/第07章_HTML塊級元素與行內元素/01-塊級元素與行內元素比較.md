# 塊級元素與行內元素比較

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨塊級元素與行內元素在版面排列上的差異。
- 理解 `width`、`height`、`margin`、`padding` 對兩類元素的影響。
- 知道「塊級 / 行內」主要描述預設排版行為，不能單獨決定元素能不能互相巢狀。

## 問題情境

剛開始寫 HTML 時，你可能會發現有些標籤會自動換到下一行，例如 `<h1>`、`<p>`、`<div>`；有些標籤則會跟其他內容排在同一行，例如 `<span>`、`<a>`、`<em>`。

這不是瀏覽器隨機決定的，而是不同元素預設有不同的顯示行為。理解塊級元素與行內元素，可以幫助你判斷 HTML 結構為什麼會這樣排列，也能避免把排版問題誤以為是標籤壞掉。

## 一句話理解

塊級元素通常會形成一個獨立區塊並佔據一行；行內元素通常跟文字一起流動，不會主動換行。

## 塊級元素是什麼

塊級元素可以想成頁面中的「段落或區塊」。它通常會從新的一行開始，並在正常排版流中佔據父元素可用的整行寬度。

常見塊級元素包括：

- `<div>`
- `<p>`
- `<h1>` 到 `<h6>`
- `<ul>`、`<ol>`、`<li>`
- `<dl>`、`<dt>`、`<dd>`
- `<form>`
- `<table>`
- `<hr>`
- `<blockquote>`
- `<address>`
- `<pre>`

塊級元素常見特性：

- 預設會獨佔一行。
- 可以設定 `width`、`height`。
- `margin`、`padding` 在上下左右方向通常都會影響版面。
- 如果沒有設定寬度，通常會佔滿父元素中的可用寬度。
- 多個塊級元素連續出現時，通常會由上往下排列。

```html
<body>
  <h1>尚硅谷</h1>
  <p>尚硅谷</p>
  <div>尚硅谷</div>
</body>
```

這段程式中，`<h1>`、`<p>`、`<div>` 都會各自形成一個區塊，所以畫面上通常會分成三行呈現。

## 行內元素是什麼

行內元素可以想成文字流中的一小段內容。它不會主動佔據整行，而是會跟前後文字或其他行內元素排在同一行，直到空間不夠才換行。

常見行內元素包括：

- `<span>`
- `<em>`、`<i>`
- `<strong>`、`<b>`
- `<a>`
- `<img>`
- `<input>`
- `<br>`
- `<select>`
- `<textarea>`
- `<q>`
- `<sub>`、`<sup>`

行內元素常見特性：

- 不會預設獨佔一行。
- 寬度通常由內容長度決定。
- 一般非置換行內元素設定 `width`、`height` 通常不會產生預期效果。
- 左右方向的 `margin`、`padding` 會影響排列；上下方向對行盒與版面的影響較有限。
- `line-height` 會影響文字所在行的高度。

```html
<input type="text">
<span>尚硅谷</span>
```

這段程式中，`<input>` 和 `<span>` 預設會盡量排在同一行。

## 特性對照

| 比較項目 | 塊級元素 | 行內元素 |
| --- | --- | --- |
| 預設排列 | 通常獨佔一行 | 通常與其他行內內容同列 |
| 寬度 | 未設定時通常佔滿父元素可用寬度 | 通常由內容決定 |
| `width` / `height` | 通常有效 | 一般非置換行內元素通常無效 |
| `margin` / `padding` | 上下左右通常都影響版面 | 左右較直覺，上下效果較有限 |
| 常見用途 | 建立段落、區塊、容器、列表、表格 | 標記文字片段、連結、強調、小型控制項 |

## 重要例外

不要把「行內元素」理解成永遠不能設定尺寸。像 `<img>`、`input` 這類元素屬於置換元素，雖然常被放在行內排版脈絡中，但它們本身可以有可見寬高。

`<br>` 也是常見例外：它屬於文字流中的元素，但用途是明確插入換行，不應拿來取代 CSS 做版面排版。

另外，CSS 可以改變元素的 `display`，例如把 `<span>` 設成 `display: block`，或把 `<div>` 設成 `display: inline`。不過這只是在改變排版行為，不代表元素原本的 HTML 語意也跟著改變。

```html
<span style="display: block;">這個 span 會像區塊一樣換行</span>
```

## 常見誤解

### 誤解一：塊級元素一定可以放任何內容

不是所有看起來像區塊的元素都可以放任何內容。例如 `<p>` 是段落，不能包含 `<div>` 這類區塊內容。

### 誤解二：行內元素一定不能放其他元素

也不完全正確。行內元素能放什麼，要看它自己的內容模型。例如 `<span>` 可以包含其他 phrasing content，但 `<img>`、`<input>`、`<br>` 這類 void elements 不能有子內容。

### 誤解三：塊級 / 行內可以決定所有巢狀規則

塊級與行內主要描述「排版行為」。元素能不能互相包含，還要回到 HTML 的語意與內容模型判斷。下一篇筆記會專門整理巢狀規則。

## 實務判斷

寫 HTML 時可以先用這個方式思考：

- 這段內容是不是一個完整區塊、段落、容器或列表？通常會使用塊級元素。
- 這段內容是不是文字中的一小段標記、連結、強調或小型輸入控制？通常會使用行內元素。
- 如果只是想改變排列方式，優先用 CSS 調整 `display`，不要為了外觀亂換沒有語意的標籤。

## 重點整理

- 塊級元素通常獨佔一行，常用來建立頁面的主要結構。
- 行內元素通常跟文字一起排列，常用來標記文字片段或小型互動內容。
- 一般非置換行內元素設定 `width`、`height` 通常不會產生預期效果。
- `<img>`、`input` 等置換元素是常見例外。
- 巢狀規則不能只靠「塊級 / 行內」判斷，還要看元素的 HTML 內容模型。
