# HTML 元素巢狀規則與特殊限制

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 HTML 巢狀結構中的父子關係。
- 分辨「看起來能放」和「語意上允許放」的差異。
- 避免標題、段落與連結常見的錯誤巢狀。
- 看懂瀏覽器為什麼會自動修正不合法的 HTML 結構。

## 問題情境

HTML 不是只要標籤有開始和結束就一定正確。某些標籤不能放在某些標籤裡，即使瀏覽器仍然能把畫面渲染出來，也可能已經把你的 DOM 結構自動修正成另一種結果。

這會造成幾個問題：

- CSS 選擇器選不到你以為存在的結構。
- JavaScript 取得到的 DOM 跟原始碼不同。
- 無障礙工具或搜尋引擎讀到錯誤的文件語意。
- 後續維護者難以判斷內容真正的層級。

## 一句話理解

HTML 巢狀規則不是只看元素是塊級或行內，而是要看該元素允許哪些子內容。

## 先理解父子關係

當一個元素包住另一個元素時，外層元素是父元素，內層元素是子元素。

```html
<div>
  <span>尚硅谷1</span>
  <input type="text">
  <div>尚硅谷2</div>
</div>
```

在這段結構中，最外層的 `<div>` 是父元素；`<span>`、`<input>` 和內層 `<div>` 都是它的子元素。

像 `<div>` 這類通用容器通常可以放入多種內容，所以很常用來組織頁面結構。不過這不代表所有塊級元素都能放任何標籤。每個 HTML 元素都有自己的內容模型。

## 基本巢狀判斷

初學時可以先用這組判斷方式：

| 判斷問題 | 說明 |
| --- | --- |
| 外層元素的語意是什麼？ | 例如段落、標題、連結、通用容器，語意不同就會有不同限制。 |
| 外層元素允許哪些內容？ | 有些元素只能包含 phrasing content，有些可以包含更完整的 flow content。 |
| 內層元素是否本身可以有子內容？ | `<img>`、`<input>`、`<br>` 這類 void elements 不能再包子元素。 |
| 是否造成互動內容互相包住？ | 例如 `<a>` 裡再放 `<a>` 或其他互動控制，通常是不合法或不適合的。 |

## 行內元素的巢狀不是一律相同

`<span>` 可以包含其他 phrasing content，所以這樣的結構是合理的：

```html
<span>
  <span>尚硅谷1</span>
  <input type="text">
  <span>尚硅谷2</span>
</span>
```

但不能把「行內元素可以包行內元素」當成絕對規則。像 `<img>`、`<input>`、`<br>` 是 void elements，本身不能有子內容。

```html
<!-- 錯誤：input 不能包內容 -->
<input type="text">
  <span>提示文字</span>
</input>
```

如果需要提示文字，應該把文字放在 `<label>` 或其他合適元素中。

```html
<label>
  姓名
  <input type="text" name="username">
</label>
```

## 特殊限制一：h1 到 h6 不能相互嵌套

標題元素 `<h1>` 到 `<h6>` 用來表示章節標題層級。標題本身應該是「這一段內容的標題」，不是拿來包住下一層標題的容器。

錯誤寫法：

```html
<h1>
  尚硅谷1
  <h2>尚硅谷2</h2>
</h1>
```

正確寫法：

```html
<h1>尚硅谷1</h1>
<h2>尚硅谷2</h2>
```

這樣 `<h1>` 與 `<h2>` 才是兩個獨立標題，文件大綱也更清楚。

瀏覽器遇到不合法的標題巢狀時，可能會自動關閉或重組標籤，最後產生的 DOM 不一定等於你寫下的原始碼。

## 特殊限制二：p 標籤不能包含塊級內容

`<p>` 表示一個段落。段落裡可以放文字、連結、強調等 phrasing content，但不應該把 `<div>` 這類區塊內容放進段落裡。

錯誤寫法：

```html
<p>
  <div>尚硅谷</div>
</p>
```

正確寫法：

```html
<p>這是一段段落文字。</p>
<div>這是一個獨立區塊。</div>
```

如果只是想在段落中的某段文字加上樣式，可以使用 `<span>`。

```html
<p>
  這是一段文字，<span>這裡是需要特別標記的片段</span>。
</p>
```

### 瀏覽器解析後的效果

![p 內放置 div 的瀏覽器解析結果](../../origin/第07章_HTML塊級元素與行內元素/assets/images/html-block-inline-elements-img-002-fda844.png)

瀏覽器通常會在 `<div>` 出現前自動結束 `<p>`，所以最後的 DOM 結構會跟原始碼不同。這也是為什麼不要把 `<p>` 當成通用容器。

## 特殊限制三：a 可以包內容，但不能包另一個 a

`<a>` 用來建立超連結。現代 HTML 中，`<a>` 的內容模型比較彈性：在所在位置允許的情況下，它可以包住文字，也可以包住一整塊卡片式內容。

```html
<article>
  <a href="https://example.com">
    <div>這是一個包含在 a 標籤中的區塊元素</div>
  </a>
</article>
```

這種寫法常見於「整張卡片可以點擊」的設計，但仍要放在允許 flow content 的位置中；如果外層是 `<p>` 這類只適合放 phrasing content 的元素，裡面就不應該再塞入 `<div>`。另外，`<a>` 不能包含另一個 `<a>`，也不應該包含其他互動內容，例如 `<button>`、可操作的表單控制項等。

錯誤寫法：

```html
<a href="https://example.com">
  <a href="https://example.com/profile">這是一個超連結</a>
</a>
```

正確寫法：

```html
<a href="https://example.com">前往首頁</a>
<a href="https://example.com/profile">查看個人頁</a>
```

如果卡片裡有多個可點擊目標，通常應該把連結拆開，而不是讓一個大連結包住其他連結。

## 實務使用準則

- 需要通用容器時，優先考慮 `<div>`，再依內容語意選擇更精確的標籤。
- 寫段落時使用 `<p>`，但不要把 `<div>`、列表、表格等區塊內容塞進 `<p>`。
- 寫標題時讓 `<h1>` 到 `<h6>` 彼此平行出現，用層級表示章節關係，不要互相包住。
- 寫連結時可以讓 `<a>` 包住所在位置允許的內容，但不要讓互動內容互相巢狀。
- 不確定能不能巢狀時，不要只看畫面是否正常，應該檢查 DOM 結構與元素語意。

## 常見錯誤整理

| 錯誤寫法 | 問題 | 建議做法 |
| --- | --- | --- |
| `<h1><h2>...</h2></h1>` | 標題不能互相巢狀 | 讓標題元素平行排列 |
| `<p><div>...</div></p>` | 段落不能包含區塊內容 | 把 `<p>` 和 `<div>` 拆成同層元素 |
| `<a><a>...</a></a>` | 連結不能包含另一個連結 | 拆成兩個獨立連結 |
| 把 `<p>` 當成版面容器 | 段落有語意限制 | 使用 `<div>` 或更合適的結構元素 |

## 重點整理

- HTML 巢狀規則要看內容模型，不只看塊級或行內。
- `<div>` 是通用容器，但其他塊級元素不一定能放任何內容。
- `<h1>` 到 `<h6>` 不能互相巢狀，應用層級順序表達文件結構。
- `<p>` 只能放適合段落中的內容，不能包 `<div>` 這類區塊內容。
- `<a>` 可以包所在位置允許的內容，但不能包另一個 `<a>` 或其他互動內容。
- 瀏覽器會嘗試修正錯誤 HTML，因此畫面能顯示不代表結構正確。

- 我是否能說明為什麼 `<a>` 裡不能再放另一個 `<a>`？
- 我是否會在結構怪異時檢查瀏覽器解析後的 DOM？
