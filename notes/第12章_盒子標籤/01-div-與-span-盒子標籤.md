# div 與 span：沒有語意的盒子標籤

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `<div>` 與 `<span>` 都是沒有語意的容器標籤。
- 分辨什麼時候適合使用 `<div>`，什麼時候適合使用 `<span>`。
- 知道 `<div>` 預設是區塊顯示，`<span>` 預設是行內顯示。
- 避免把 `<div>`、`<span>` 當成所有版面與文字標記的唯一選擇。

## 問題情境

寫 HTML 時，常常會遇到一種情況：你需要把一群內容包起來，方便之後用 CSS 排版、上色或用 JavaScript 操作，但這一群內容本身沒有更明確的 HTML 語意。

例如：

- 把一整塊卡片內容包起來，之後用 CSS 做成區塊。
- 把一句話中的幾個字包起來，之後改顏色或加效果。
- 把多個元素分組，讓版面結構比較容易管理。

這時就會用到 `<div>` 和 `<span>`。

## 一句話理解

`<div>` 和 `<span>` 都是「沒有語意的盒子」：`<div>` 常用來包一大塊內容，`<span>` 常用來包一小段行內內容。

## 核心概念

`<div>` 是 `division` 的縮寫，意思接近「分割、分區」。它通常用來建立一個區塊容器。

`<span>` 的意思接近「跨度、跨距」。它通常用來包住一小段行內內容，例如一句話中的某幾個字。

它們的共同點是：本身不告訴瀏覽器或讀者「這段內容是什麼」。換句話說，`<div>` 不代表文章、導覽列、標題或段落；`<span>` 也不代表強調、連結或按鈕。

所以在實務上要先判斷：

1. 如果有更符合語意的標籤，優先使用語意標籤。
2. 如果只是需要一個容器來分組、排版或操作，再使用 `<div>` 或 `<span>`。

## 基本語法

```html
<div>我是一個 div 標籤，我通常會單獨占一行。</div>
<div>我也是一個 div 標籤，我也通常會單獨占一行。</div>

<span>百度</span>
<span>新浪</span>
<span>搜狐</span>
```

這段範例中：

- 前兩個 `<div>` 會預設以區塊方式顯示，因此通常會各自另起一行。
- 後三個 `<span>` 會預設以行內方式顯示，因此可以在同一行排列。

## div：適合包住一塊內容

`<div>` 常被當作「大盒子」。它適合用來包住一組相關內容，例如卡片、區塊、欄位群組或頁面中的某個版面區域。

```html
<div class="profile-card">
  <h2>王小明</h2>
  <p>前端學習者，正在練習 HTML 與 CSS。</p>
</div>
```

在這個範例中，`<div class="profile-card">` 的作用是把標題與段落包成同一組內容，之後可以透過 CSS 把整組內容設計成個人簡介卡片。

如果這塊內容有明確語意，實務上可以考慮更適合的語意標籤。例如文章可以用 `<article>`，導覽區可以用 `<nav>`，頁尾可以用 `<footer>`。如果只是一般分組，`<div>` 就很合適。

## span：適合包住一小段行內內容

`<span>` 常被當作「小盒子」。它適合用來包住文字中的一小段內容，而且不想打斷整個段落的行內流動。

```html
<p>今天的課程重點是 <span class="keyword">HTML 盒子標籤</span>。</p>
```

在這個範例中，`<span class="keyword">` 只包住「HTML 盒子標籤」這幾個字。之後可以用 CSS 讓這幾個字變色、加粗或套用其他樣式。

如果文字本身具有語意，例如「需要強調的重要內容」，通常應該考慮 `<strong>` 或 `<em>`，而不是只用 `<span>` 裝起來。

## div 與 span 的比較

| 比較項目 | `<div>` | `<span>` |
| --- | --- | --- |
| 標籤語意 | 沒有語意 | 沒有語意 |
| 常見用途 | 包住一大塊內容 | 包住一小段行內內容 |
| 預設顯示 | 區塊顯示，通常會另起一行 | 行內顯示，可以和文字在同一行 |
| 常見角色 | 大盒子 | 小盒子 |
| 是否能用 CSS 改變顯示方式 | 可以 | 可以 |

要注意的是，「大盒子」與「小盒子」只是初學時的理解方式，不是 HTML 規定的永久行為。實際顯示效果可以透過 CSS 的 `display` 屬性改變。

## 常見錯誤

### 錯誤一：只為了文字換行就濫用 div

```html
<div>這是一段文字。</div>
<div>這是另一段文字。</div>
```

如果內容本質上是段落，應該優先使用 `<p>`：

```html
<p>這是一段文字。</p>
<p>這是另一段文字。</p>
```

`<div>` 可以造成視覺上的分行，但它不代表段落語意。

### 錯誤二：用 span 包住一整塊結構

```html
<span class="card">
  <h2>商品名稱</h2>
  <p>商品介紹文字</p>
</span>
```

`<span>` 適合行內短內容，不適合拿來包住標題、段落這類區塊結構。這不只是版面選擇問題，也不符合 `<span>` 通常只能包行內內容的 HTML 結構限制。這種情況應該改用 `<div>` 或更合適的語意標籤：

```html
<div class="card">
  <h2>商品名稱</h2>
  <p>商品介紹文字</p>
</div>
```

### 錯誤三：有語意標籤時仍全部使用 div

```html
<div class="nav">
  <a href="/">首頁</a>
  <a href="/about">關於我們</a>
</div>
```

如果這是一組主要導覽連結，使用 `<nav>` 會更能表達內容用途：

```html
<nav>
  <a href="/">首頁</a>
  <a href="/about">關於我們</a>
</nav>
```

這不只是程式碼看起來不同，也會影響文件結構的可讀性、維護性與輔助工具理解頁面的能力。

## 實務判斷

使用 `<div>` 或 `<span>` 前，可以先問自己三個問題：

1. 這段內容有沒有更明確的 HTML 語意標籤可以使用？
2. 我要包的是一整塊內容，還是文字中的一小段內容？
3. 我使用這個容器，是為了分組、排版、套樣式，還是為了表達內容意義？

如果答案是「只是需要一個容器」，再使用 `<div>` 或 `<span>`。

一般判斷方式如下：

- 包一整個區塊：通常使用 `<div>`。
- 包文字中的幾個字：通常使用 `<span>`。
- 表達段落：使用 `<p>`。
- 表達標題：使用 `<h1>` 到 `<h6>`。
- 表達導覽區：使用 `<nav>`。
- 表達重要內容：視情況使用 `<strong>` 或其他語意標籤。

## 小練習

請判斷下列情境適合使用哪個標籤：

1. 想把一張商品卡片的圖片、標題、價格包成一組。
2. 想把一句話中的「限時優惠」四個字標成紅色。
3. 想寫兩段文章內容。
4. 想建立網站上方的主要導覽連結。

參考答案：

1. 可以使用 `<div>`，如果商品卡片本身是一篇獨立內容，也可以思考是否適合 `<article>`。
2. 可以使用 `<span>`；如果是語意上的重要提醒，也可以思考是否需要更具語意的標籤。
3. 使用 `<p>`。
4. 使用 `<nav>`。

## 重點整理

- `<div>` 和 `<span>` 都是沒有語意的容器標籤。
- `<div>` 預設是區塊顯示，常用來包住一大塊內容。
- `<span>` 預設是行內顯示，常用來包住一小段文字或行內內容。
- 預設顯示效果可以被 CSS 改變，所以不要只用畫面效果理解 HTML 標籤。
- 實務上要優先使用有語意的 HTML 標籤；沒有更合適的語意標籤時，再使用 `<div>` 或 `<span>`。
