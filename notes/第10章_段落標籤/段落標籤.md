# HTML 段落標籤

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `p` 元素的用途與語意。
- 使用 `p` 標籤建立文章或新聞中的文字段落。
- 理解段落預設會獨立成段，並會依照容器寬度自動換行。
- 分辨段落文字、標題文字與一般無語意容器的差異。
- 避免把 `p` 標籤用在不適合的內容結構中。

## 2. 本章定位

段落標籤是 HTML 文件內容結構中的基礎元素，用來表示一段完整的文字內容。

在學習標題標籤後，接著理解段落標籤很重要，因為大多數文章、新聞、介紹頁、說明文件與商品描述，都是由標題與段落共同組成。標題負責指出內容層級，段落則承載主要文字說明。

## 3. 核心概念

`<p>` 是 paragraph 的縮寫，表示「段落」。

在 HTML 中，段落不是單純讓文字換行的工具，而是用來表達一段文字在內容結構上的完整單位。例如文章中的一段敘述、新聞中的一段報導、商品介紹中的一段說明，都適合使用 `p` 元素。

```html
<p>HTML 是用來描述網頁內容與結構的標記語言。</p>
```

> 來源重點：原始資料指出，段落標籤的語意是「可以把 HTML 文檔分割為若干個段落」，常見於新聞與文章頁面中的分段顯示。

### 3.1 段落元素的語意

`p` 元素代表一段文字內容。它告訴瀏覽器、搜尋引擎與輔助工具：這裡是一個文字段落，而不是標題、清單項目、按鈕或單純的排版容器。

範例：

```html
<h1>網站公告</h1>
<p>本網站將於週末進行系統維護，部分功能可能暫時無法使用。</p>
<p>維護完成後，所有服務將恢復正常，感謝你的理解。</p>
```

這裡的 `h1` 是公告標題，兩個 `p` 則分別表示兩段公告內容。

### 3.2 段落的預設呈現

瀏覽器通常會替 `p` 元素套用預設樣式，因此段落在畫面上常見以下特徵：

- 段落通常會獨立顯示成一個區塊。
- 段落與段落之間通常會有預設間距。
- 段落中的文字會依照瀏覽器視窗或父容器寬度自動換行。

> 補充說明：段落之間的空隙主要來自瀏覽器預設樣式中的 margin。實務上若需要調整段落間距，應使用 CSS 控制，而不是加入多個空白或濫用 `<br>`。

## 4. 語法與基本用法

`p` 元素由開始標籤、段落內容與結束標籤組成：

```html
<p>這是一段文字。</p>
```

其中：

| 部分 | 說明 |
|---|---|
| `<p>` | 段落開始標籤 |
| `這是一段文字。` | 段落內容 |
| `</p>` | 段落結束標籤 |

多個段落應使用多個 `p` 元素表示：

```html
<p>第一段文字，說明第一個重點。</p>
<p>第二段文字，說明另一個重點。</p>
```

不建議把多個獨立段落全部塞進同一個 `p` 元素，再用換行或空白硬做出分段效果。HTML 應該先描述正確的內容結構，再由 CSS 控制視覺呈現。

## 5. 範例說明

### 範例：文章中的段落

```html
<article>
  <h1>學習 HTML 的第一步</h1>

  <p>HTML 是建立網頁內容結構的基礎。透過不同的標籤，我們可以描述標題、段落、連結、圖片與表單等內容。</p>

  <p>在撰寫文章或說明文件時，應該使用段落標籤把文字切分成容易閱讀的段落，而不是把所有文字放在同一大段中。</p>
</article>
```

這個範例中：

- `article` 表示一篇可獨立閱讀的文章內容。
- `h1` 表示文章主標題。
- 第一個 `p` 說明 HTML 的基本用途。
- 第二個 `p` 說明撰寫文章時分段的重要性。

### 範例：段落會依照容器寬度自動換行

```html
<p>
  這是一段較長的文字。當瀏覽器視窗或父容器寬度不足時，文字會在同一個段落內自動換行，而不需要手動在每一行加入換行標籤。
</p>
```

段落中的文字會在可用寬度內自然排列。只要內容仍屬於同一段意思，就應該留在同一個 `p` 元素中；只有進入另一段意思時，才建立新的 `p`。

## 6. 實務應用場景

`p` 元素常見於以下場景：

- 新聞頁：用多個段落呈現報導內容。
- 文章頁：搭配標題標籤建立文章結構。
- 商品介紹頁：描述商品特色、規格說明或注意事項。
- 關於我們頁面：呈現公司、團隊或品牌介紹。
- 說明文件：用段落解釋概念、步驟或背景。

在實務開發中，段落結構會影響：

- 使用者閱讀長文字內容的節奏。
- CSS 對段落間距、行高與文字樣式的控制。
- JavaScript 透過 DOM 查找或操作段落節點。
- 搜尋引擎與輔助工具對文字內容結構的理解。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 把整篇文章放在同一個 `p` 裡 | 內容缺少清楚分段，閱讀與維護都較困難 | 每個獨立段落使用一個 `p` |
| 用多個 `<br>` 製造段落間距 | `<br>` 表示換行，不是段落結構或間距工具 | 使用多個 `p` 表達段落，使用 CSS 控制間距 |
| 用 `div` 包一般文章段落 | `div` 沒有段落語意，會降低內容結構清晰度 | 一般文字段落優先使用 `p` |
| 用 `p` 包標題 | 標題與段落語意不同 | 標題使用 `h1` 到 `h6`，段落內容使用 `p` |
| 為了改變間距而亂用空白字元 | HTML 不是用空白控制版面的主要工具 | 使用 CSS 的 `margin`、`line-height` 等屬性控制視覺效果 |

### 錯誤範例

```html
<p>
  HTML 是用來描述網頁內容與結構的標記語言。<br><br>
  CSS 則用來控制畫面樣式。
</p>
```

問題：這裡其實是兩個不同段落，卻用多個 `<br>` 製造段落間隔，HTML 結構不夠清楚。

### 正確範例

```html
<p>HTML 是用來描述網頁內容與結構的標記語言。</p>
<p>CSS 則用來控制畫面樣式。</p>
```

修正後，每段文字都有自己的 `p` 元素，段落語意更清楚，後續也更容易用 CSS 控制段落間距。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| 標題標籤 | 標題標籤用來建立內容層級，`p` 用來承載標題底下的段落文字 |
| 語意化 HTML | `p` 是語意化文字內容的基礎元素，表示一段完整文字 |
| CSS | CSS 可控制段落的字體、行高、顏色、寬度與段落間距 |
| DOM | 瀏覽器會把 `p` 解析成 DOM 節點，供 CSS 與 JavaScript 使用 |
| JavaScript | JavaScript 可以透過 DOM API 取得、修改或新增段落內容 |
| 可訪問性 | 正確使用段落結構有助於輔助工具理解文字內容的分段 |
| SEO | 清楚的文章結構有助於搜尋引擎理解頁面內容，但 SEO 不只取決於 `p` 標籤 |

## 9. 重點整理

- `p` 元素表示 paragraph，也就是段落。
- 段落標籤用來把 HTML 文件中的文字內容分割成多個段落。
- 新聞、文章、介紹頁與說明文件中，常會使用多個 `p` 元素呈現正文。
- 段落之間通常會有瀏覽器預設間距，文字也會依照容器寬度自動換行。
- `p` 的重點是文字段落語意，不是單純製造換行或空白。
- 視覺間距應交給 CSS 控制，內容結構應交給正確的 HTML 元素表達。

## 11. 延伸補充

> 補充說明：HTML 負責描述內容與結構，CSS 負責控制視覺呈現。`p` 元素表達的是「這是一段文字」，而段落的字體大小、行高、顏色與間距，通常應交給 CSS 設定。

在實務專案中，若設計稿要求調整段落間距，不應為了畫面效果加入多個空白、空的段落或連續 `<br>`。較好的做法是保留清楚的 `p` 結構，再使用 CSS 控制：

```css
p {
  line-height: 1.7;
  margin-block: 0 1em;
}
```

這樣可以讓 HTML 維持語意清楚，也讓樣式更容易統一管理。
