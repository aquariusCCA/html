# HTML 水平線標籤 hr

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `<hr>` 標籤的用途與語意。
- 正確使用 `<hr>` 分隔不同主題或段落群組。
- 理解 `<hr>` 是空元素，不需要結束標籤。
- 分辨「語意上的主題分隔」與「純視覺裝飾線」的差異。
- 避免把 `<hr>` 當成排版間距工具。

## 2. 本章定位

本章介紹 HTML 中用來表示主題分隔的 `<hr>` 元素。

原始資料提到，`<hr>` 的標籤語意是「主題的分割轉換」，常見畫面效果是在頁面上顯示一條水平線。學習 `<hr>` 時，重點不只是知道它會畫出一條線，而是要理解：這條線背後代表內容上的分隔。

在 HTML 中，元素應優先根據內容語意選擇，而不是只根據預設畫面效果選擇。因此，當內容真的從一個主題切換到另一個主題時，`<hr>` 才是合適的選擇；如果只是想畫一條裝飾線，通常應該交給 CSS。

## 3. 核心概念

`<hr>` 是 HTML 的 thematic break element，用來表示段落層級的主題轉換或內容分隔。

在瀏覽器預設樣式中，`<hr>` 通常會被顯示為一條水平線，所以中文教材常稱它為「水平線標籤」。但它的語意不是「幫畫面加一條線」，而是「這裡的內容主題發生了切換」。

常見適合使用 `<hr>` 的情境包括：

- 文章中從一個主題轉到另一個主題。
- 長篇內容中分隔不同段落群組。
- 故事、文章或說明文件中表示場景或語氣轉換。
- 表單或頁面內容中，分隔兩組語意上不同的資訊。

> 補充說明：`<hr>` 是 void element（空元素），不需要結束標籤。HTML 中通常寫成 `<hr>`；`<hr />` 是 XHTML 風格寫法，在一般 HTML 中也常被瀏覽器接受。

## 4. 語法與基本用法

基本語法如下：

```html
<hr>
```

也常看到以下寫法：

```html
<hr />
```

這兩種寫法在一般 HTML 頁面中通常都能被解析。不過在 HTML 文件中，較簡潔常見的寫法是 `<hr>`。

| 寫法 | 說明 |
|---|---|
| `<hr>` | HTML 中常見寫法，表示主題分隔 |
| `<hr />` | XHTML 風格寫法，在 HTML 中通常也能被解析 |
| `<hr></hr>` | 不建議使用，因為 `<hr>` 是空元素，不需要結束標籤 |

`<hr>` 不應該放入文字內容，因為它沒有內部內容。它通常放在兩段內容或兩個區塊之間。

## 5. 範例說明

### 範例：分隔文章中的兩個主題

```html
<article>
  <h1>前端學習筆記</h1>

  <section>
    <h2>HTML 的角色</h2>
    <p>HTML 負責描述網頁內容與結構。</p>
  </section>

  <hr>

  <section>
    <h2>CSS 的角色</h2>
    <p>CSS 負責控制網頁的視覺樣式與版面配置。</p>
  </section>
</article>
```

這個範例中：

- 第一個 `section` 說明 HTML。
- 第二個 `section` 說明 CSS。
- `<hr>` 放在兩個主題之間，表示內容從 HTML 主題切換到 CSS 主題。

這裡的 `<hr>` 不只是畫線，而是表達「前後內容有語意上的分隔」。

### 範例：用 CSS 調整水平線外觀

```html
<hr class="topic-divider">
```

```css
.topic-divider {
  border: 0;
  border-top: 1px solid #999;
  margin: 24px 0;
}
```

`<hr>` 提供語意，CSS 負責視覺呈現。若希望水平線顏色、粗細或上下間距不同，應使用 CSS 調整，而不是依賴舊式 HTML 屬性。

## 6. 實務應用場景

在實務開發中，`<hr>` 適合用於有明確內容分隔的地方，例如：

- 教學文章中分隔不同知識段落。
- 商品頁中分隔商品說明與注意事項。
- 表單頁中分隔不同資訊群組，例如基本資料與付款資訊。
- 文件頁中分隔不同主題區塊。

如果需求只是「畫面上想要一條線」，但前後內容沒有主題轉換，通常可以改用 CSS，例如 `border-top`、`border-bottom` 或偽元素。

### 純裝飾線的替代做法

```html
<div class="card">
  <h2>帳戶資訊</h2>
  <p>這裡是帳戶內容。</p>
</div>
```

```css
.card {
  border-top: 1px solid #ddd;
  padding-top: 16px;
}
```

這個例子中的線只是卡片視覺樣式的一部分，不代表內容主題轉換，因此使用 CSS 會比使用 `<hr>` 更合適。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 把 `<hr>` 當成純排版裝飾線 | `<hr>` 有主題分隔語意，不只是視覺線條 | 若只是裝飾，使用 CSS 的 `border` 或偽元素 |
| 用多個 `<hr>` 製造大段空白 | 空白與間距屬於視覺排版，不是內容語意 | 使用 CSS 的 `margin` 或 `padding` |
| 寫成 `<hr></hr>` | `<hr>` 是空元素，不需要結束標籤 | 寫成 `<hr>` 或 `<hr />` |
| 用舊式屬性控制外觀 | 外觀應交給 CSS 維護 | 使用 CSS 控制顏色、寬度、粗細與間距 |
| 在不相關位置大量插入 `<hr>` | 會讓文件結構變得雜亂，也可能干擾輔助工具理解 | 只在內容真的需要主題分隔時使用 |

### 錯誤範例：用 `<hr>` 當排版間距

```html
<h2>最新消息</h2>
<hr>
<hr>
<p>今天發布了新版本。</p>
```

問題：這裡連續使用 `<hr>` 只是為了製造距離，沒有表達新的主題分隔。

### 正確範例：用 CSS 控制間距

```html
<section class="news">
  <h2>最新消息</h2>
  <p>今天發布了新版本。</p>
</section>
```

```css
.news h2 {
  margin-bottom: 24px;
}
```

修正重點：內容結構由 HTML 表達，視覺距離由 CSS 控制。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| 語意化 HTML | `<hr>` 應用於主題分隔，而不是只看它的預設樣式 |
| CSS | `<hr>` 的線條樣式、顏色、粗細與間距應由 CSS 控制 |
| `<section>` | `<section>` 用來表示有主題的章節，`<hr>` 可用來表示段落層級的主題轉換 |
| DOM | 瀏覽器會把 `<hr>` 解析成 DOM 中的元素節點 |
| 可訪問性 | `<hr>` 可能被輔助工具視為分隔符，濫用會讓內容結構變得零碎 |

## 9. 重點整理

- `<hr>` 的核心語意是主題分隔或內容轉換。
- `<hr>` 在瀏覽器中通常顯示為一條水平線，但它不是單純的畫線工具。
- `<hr>` 是空元素，不需要結束標籤。
- 若只是要控制視覺線條或間距，應使用 CSS。
- 正確使用 `<hr>` 可以讓文件結構更清楚，也有助於維護語意化 HTML。

## 11. 延伸補充

`<hr>` 是一個容易被低估的語意元素。它的預設畫面很簡單，但在 HTML 結構中代表內容節奏或主題切換。

> 補充說明：HTML 負責描述內容與語意，CSS 負責視覺呈現。當需求是「前後內容有主題分隔」時，可以考慮 `<hr>`；當需求只是「畫面上需要一條線」時，通常應該使用 CSS。
