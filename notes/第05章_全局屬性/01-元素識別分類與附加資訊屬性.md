# 元素識別、分類與附加資訊屬性

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `id`、`class`、`data-*` 與 `style` 各自解決什麼問題。
- 分辨「唯一識別」、「分類共用」、「自訂資料」與「行內樣式」的使用時機。
- 寫出可被 CSS 與 JavaScript 穩定選取的 HTML。
- 避免把所有需求都塞進 `id` 或行內 `style`。

## 問題情境

HTML 標籤描述的是內容本身，例如「這是一段文字」、「這是一個按鈕」。但在實際開發中，我們還需要回答更多問題：

- 這個元素是否有唯一身分？
- 這批元素是否屬於同一類，可以套用相同樣式？
- JavaScript 是否需要讀取某些自訂資料？
- 是否需要直接在這個元素上加一段 CSS？

`id`、`class`、`data-*` 與 `style` 都是全局屬性，可以放在多數 HTML 元素上。它們的共同點是替元素補充資訊，但用途並不相同。

## 一句話理解

`id` 用來標記唯一元素，`class` 用來分類多個元素，`data-*` 用來放自訂資料，`style` 用來直接寫行內 CSS。

## `id`：元素的唯一識別

`id` 屬性規定 HTML 元素的唯一識別名稱。同一份 HTML 文件中，某個 `id` 值應該只出現一次。

```html
<p id="intro">這是網站介紹段落。</p>
```

`id` 常見用途：

- 讓 CSS 精準選取某一個元素。
- 讓 JavaScript 用 `getElementById()` 取得元素。
- 作為頁面內錨點連結的目標。

例如：

```html
<h2 id="pricing">方案價格</h2>
<a href="#pricing">查看方案價格</a>
```

這裡的 `href="#pricing"` 會連到同一頁中 `id="pricing"` 的位置。

## `class`：元素的分類名稱

`class` 屬性定義元素的類名，常用來讓 CSS 或 JavaScript 選取一組具有相同特徵的元素。

```html
<p class="important">這是一段重要提醒。</p>
```

搭配 CSS：

```css
.important {
  color: green;
}
```

同一個 `class` 可以重複用在多個元素上：

```html
<p class="notice">系統維護中。</p>
<p class="notice">請先儲存你的資料。</p>
```

一個元素也可以有多個 class，使用空格分隔：

```html
<h1 class="intro important">Header 1</h1>
```

這代表這個 `<h1>` 同時屬於 `intro` 與 `important` 兩個分類。

## `data-*`：元素上的自訂資料

`data-*` 屬性用來在 HTML 元素上存放頁面或應用程式自己的私有資料。

```html
<li class="product" data-product-id="A102" data-stock-status="in-stock">
  無線鍵盤
</li>
```

這些資料不會讓瀏覽器自動改變畫面，也不會自動產生互動效果。它們的價值在於：JavaScript 可以從 DOM 中讀取這些資訊。

```html
<ul>
  <li class="animal" data-animal-type="bird">喜鵲</li>
  <li class="animal" data-animal-type="fish">金槍魚</li>
  <li class="animal" data-animal-type="spider">蠅虎</li>
</ul>

<script>
  const animals = document.querySelectorAll('.animal');

  for (let i = 0; i < animals.length; i++) {
    console.log(animals[i].dataset.animalType);
  }
</script>
```

在 JavaScript 的 `dataset` 中，`data-animal-type` 會變成 `animalType`。

`data-*` 的命名要注意：

- 屬性名稱必須以 `data-` 開頭。
- `data-` 後面至少要有一個字元。
- 屬性名稱不應包含大寫字母。
- 屬性值可以是任意字串。

如果要用 JavaScript 設定，也可以使用 `setAttribute()`：

```html
<p id="apple">蘋果</p>

<script>
  const apple = document.getElementById('apple');
  apple.setAttribute('data-food-type', 'fruit');
</script>
```

## `style`：直接寫在元素上的 CSS

`style` 屬性規定元素的行內樣式。

```html
<h1 style="color: blue; text-align: center;">This is a header</h1>
<p style="color: red;">This is a paragraph.</p>
```

行內樣式會直接作用在該元素上。一般情況下，它會優先於 `<style>` 標籤或外部樣式表中的普通作者樣式，但實際結果仍會受到 CSS cascade、`!important`、使用者樣式與瀏覽器規則影響。

實務上，`style` 不適合作為主要樣式管理方式。若多個元素需要共用樣式，通常應該使用 `class` 搭配 CSS：

```html
<p class="warning">請先完成付款。</p>
<p class="warning">此操作無法復原。</p>
```

```css
.warning {
  color: red;
  font-weight: bold;
}
```

這樣比在每個元素上重複寫 `style` 更容易維護。

## 四個屬性的實務比較

| 屬性 | 主要用途 | 是否適合重複使用同一值 | 常見搭配 |
| --- | --- | --- | --- |
| `id` | 唯一識別元素 | 不適合 | 錨點、JavaScript |
| `class` | 分類與共用樣式 | 適合 | CSS、JavaScript |
| `data-*` | 存放自訂資料 | 視資料而定 | JavaScript、互動邏輯 |
| `style` | 寫行內 CSS | 不適合大量重複 | 一次性樣式、動態樣式 |

## 常見錯誤

### 把多個元素設成同一個 `id`

錯誤寫法：

```html
<p id="notice">第一則提醒</p>
<p id="notice">第二則提醒</p>
```

如果它們是一組提醒，應該使用 `class`：

```html
<p class="notice">第一則提醒</p>
<p class="notice">第二則提醒</p>
```

### 把資料塞進 class 名稱

錯誤寫法：

```html
<button class="product-A102 in-stock">加入購物車</button>
```

如果這些資料要給 JavaScript 使用，應該使用 `data-*`：

```html
<button class="add-to-cart" data-product-id="A102" data-stock-status="in-stock">
  加入購物車
</button>
```

### 大量使用行內樣式

不建議：

```html
<p style="color: red; font-weight: bold;">錯誤訊息</p>
<p style="color: red; font-weight: bold;">請重新輸入</p>
```

較好的做法：

```html
<p class="error-message">錯誤訊息</p>
<p class="error-message">請重新輸入</p>
```

## 重點整理

- `id` 是元素的唯一識別，同一份文件中不要重複。
- `class` 適合把多個元素歸成同一類，常用於共用樣式。
- 一個元素可以有多個 class，使用空格分隔。
- `data-*` 適合存放給 JavaScript 使用的自訂資料。
- `style` 是行內 CSS，適合少量特殊情況，不適合作為主要樣式管理方式。
- 選擇屬性時，先想清楚你要的是「識別」、「分類」、「資料」還是「樣式」。
