# 顯示狀態與補充提示屬性

## 學習目標

讀完這篇筆記，你應該能夠：

- 使用 `hidden` 隱藏暫時不應顯示的元素。
- 使用 JavaScript 移除 `hidden`，讓元素重新顯示。
- 使用 `title` 補充元素的額外資訊。
- 避免把重要內容只放在 `title` 提示文字裡。

## 問題情境

網頁常常需要根據狀態決定哪些內容要出現。例如使用者勾選某個選項後，才顯示進一步設定；或在某個縮寫上提供額外說明。

`hidden` 與 `title` 都和使用者是否能看到額外資訊有關，但它們處理的是不同問題：

- `hidden`：這個元素目前是否應該顯示。
- `title`：這個元素是否有額外補充說明。

## 一句話理解

`hidden` 用來表示元素目前不應顯示，`title` 用來提供元素的補充提示資訊。

## `hidden`：隱藏元素

`hidden` 屬性用來表示元素目前不應被呈現。最常見的寫法是直接寫上 `hidden`，讓元素不顯示。

```html
<p>這是一個沒有隱藏的段落。</p>
<p hidden>這是一個隱藏的段落。</p>
```

`hidden` 在現行 HTML 中是枚舉屬性。入門最常遇到的是省略屬性值、直接寫 `hidden` 的簡寫方式，表示一般隱藏狀態；它也可以使用 `hidden="until-found"`，讓內容平時隱藏，但仍可被頁面搜尋或片段連結揭示。

## 用 JavaScript 顯示 hidden 元素

`hidden` 常用於「符合某些條件後才顯示」的情境。例如使用者勾選核取方塊後，顯示額外欄位：

```html
<label>
  <input type="checkbox" id="needInvoice">
  需要統一編號
</label>

<p id="invoiceField" hidden>
  <label for="taxId">統一編號</label>
  <input type="text" id="taxId" name="taxId">
</p>

<script>
  const checkbox = document.getElementById('needInvoice');
  const invoiceField = document.getElementById('invoiceField');

  checkbox.addEventListener('change', function () {
    invoiceField.hidden = !checkbox.checked;
  });
</script>
```

這裡的判斷是：

- 沒勾選時，`invoiceField.hidden` 是 `true`，欄位隱藏。
- 勾選後，`invoiceField.hidden` 變成 `false`，欄位顯示。

## `title`：提供額外提示資訊

`title` 屬性規定關於元素的額外資訊。這些資訊通常會在滑鼠移到元素上時顯示為工具提示文字。

```html
<abbr title="People's Republic of China">PRC</abbr> was founded in 1949.
<p title="Free Web tutorials">W3School.com.cn</p>
```

第一個例子中，`title` 補充了縮寫 `PRC` 的完整名稱。

## `title` 的實務注意事項

`title` 看起來很方便，但不適合承載重要資訊。

原因包括：

- 觸控裝置上不一定有滑鼠懸停行為。
- 鍵盤使用者不一定能穩定看到提示。
- 輔助科技對 `title` 的處理方式可能不同。
- 使用者可能不知道某個元素有提示文字。

如果資訊很重要，應該直接顯示在畫面上，或使用更明確的說明文字。

## 常見錯誤

### 把必要說明只放在 `title`

不建議：

```html
<button title="刪除後無法復原">刪除</button>
```

如果這是重要警告，應該讓使用者直接看見：

```html
<button type="button">刪除</button>
<p>刪除後無法復原。</p>
```

### 用 `hidden` 只做視覺裝飾

`hidden` 表示內容目前不應被呈現。若只是為了切換視覺樣式，通常應考慮 CSS class；若內容狀態真的不應顯示，再使用 `hidden`。

## 實務判斷

使用 `hidden` 前先問：

- 這個元素目前是否真的不應出現在頁面中？
- 使用者滿足條件後，是否需要透過 JavaScript 顯示它？

使用 `title` 前先問：

- 這段資訊是否只是補充？
- 如果使用者完全看不到 `title`，是否仍能完成任務？

如果答案是否定的，就不要把資訊只放在 `title`。

## 重點整理

- `hidden` 是枚舉屬性；常見 `<p hidden>` 寫法代表元素處於一般隱藏狀態。
- 可以透過 JavaScript 移除或設定 `hidden` 來控制顯示狀態。
- `title` 提供元素的額外提示資訊，常見於滑鼠懸停提示。
- 不要把重要資訊只放在 `title`。
- `hidden` 處理內容是否顯示，`title` 處理額外提示，兩者用途不同。
