# 可編輯內容與拼寫檢查屬性

## 學習目標

讀完這篇筆記，你應該能夠：

- 使用 `contenteditable` 讓元素內容可被使用者編輯。
- 使用 `spellcheck` 控制瀏覽器是否進行拼寫與語法檢查。
- 理解這兩個屬性常一起出現在文字輸入或簡易編輯情境中。
- 分辨瀏覽器提供的編輯能力與真正資料儲存之間的差異。

## 問題情境

一般情況下，使用者只能在 `<input>` 或 `<textarea>` 中輸入文字。如果你希望某段普通 HTML 內容也可以直接被編輯，例如備註區、簡易富文字區、待辦事項名稱，就會遇到 `contenteditable`。

當使用者正在輸入文字時，瀏覽器也可能提供拼寫或語法檢查。這時可以透過 `spellcheck` 提示瀏覽器是否要檢查該區域。

## 一句話理解

`contenteditable` 控制元素能不能直接被編輯，`spellcheck` 控制可編輯文字是否要接受瀏覽器拼寫檢查。

## `contenteditable`：讓元素變成可編輯

`contenteditable` 是 HTML5 新增的全局屬性，目的是讓使用者能修改頁面上的元素內容。

```html
<p contenteditable="true">I love China!</p>
<p contenteditable="false">I love China!</p>
```

在這個例子中：

- `contenteditable="true"`：這段文字可以被使用者直接編輯。
- `contenteditable="false"`：這段文字不可被直接編輯。

`contenteditable` 是枚舉屬性，常見值包含：

| 值 | 意思 |
| --- | --- |
| `true` | 內容可以被編輯 |
| `false` | 內容不可被編輯 |
| `plaintext-only` | 內容可以被編輯，但貼上內容會保留純文字、移除格式 |

更完整的頁面範例：

```html
<body>
  <h1>個人簡介</h1>
  <p contenteditable="true">
    點擊這段文字後，可以直接修改內容。
  </p>
</body>
```

## 可編輯不等於自動儲存

`contenteditable` 只負責讓畫面上的 DOM 內容可以被編輯。它不會自動把修改後的內容送到伺服器，也不會自動存進資料庫。

如果要真正保存內容，仍然需要 JavaScript 讀取內容，再送到後端或存到本地儲存空間。

```html
<p id="note" contenteditable="true">今天要完成 HTML 筆記。</p>
<button type="button" id="save">儲存</button>

<script>
  const note = document.getElementById('note');
  const save = document.getElementById('save');

  save.addEventListener('click', function () {
    console.log(note.textContent);
  });
</script>
```

這段程式只是示範如何讀取編輯後的文字；真正專案中還要處理驗證、儲存與安全問題。

## `spellcheck`：控制拼寫與語法檢查

`spellcheck` 屬性規定是否對元素進行拼寫和語法檢查。

```html
<input type="text" spellcheck="false">
<textarea spellcheck="true"></textarea>
<div contenteditable="true" spellcheck="true">
  這是一個支援拼寫檢查的可編輯內容。
</div>
```

可以被拼寫檢查的內容通常包含：

- `input` 元素中的文字值，但不包含密碼欄位。
- `<textarea>` 中的文字。
- 可編輯元素中的文字。

拼寫檢查的實際行為由瀏覽器決定。不同瀏覽器、系統語言與使用者設定，可能會有不同結果。

## 搭配使用範例

```html
<form>
  <label for="username">使用者名稱:</label>
  <input type="text" id="username" name="username" spellcheck="false">

  <label for="message">留言:</label>
  <textarea id="message" name="message" spellcheck="true"></textarea>

  <div contenteditable="true" spellcheck="true">
    這是一個支援拼寫檢查的可編輯內容。
  </div>

  <input type="submit" value="提交">
</form>
```

拆開來看：

- 使用者名稱通常是帳號或專有字串，停用拼寫檢查較合理。
- 留言欄位通常是自然語言文字，啟用拼寫檢查較合理。
- 可編輯的 `<div>` 同時透過 `contenteditable` 開放編輯，並透過 `spellcheck` 提示瀏覽器檢查文字。

## 常見錯誤

### 以為 `contenteditable` 會自動送出表單

```html
<form>
  <div contenteditable="true">請輸入備註</div>
  <button type="submit">提交</button>
</form>
```

`contenteditable` 元素不是表單控制項，表單送出時不會像 `<input>` 或 `<textarea>` 一樣自動帶出欄位值。

如果內容需要被表單提交，通常應使用 `<textarea>`，或用 JavaScript 將內容同步到表單欄位。

### 以為 `spellcheck` 一定會生效

`spellcheck="true"` 是提供給瀏覽器的提示，不保證每個環境都會用相同方式檢查。正式功能需求不能只依賴瀏覽器拼寫檢查。

## 實務使用場景

`contenteditable` 適合用在：

- 簡單筆記或備註編輯。
- 可直接點擊修改的標題或文字。
- 需要自訂編輯體驗的介面。

如果只是一般文字輸入，優先使用 `<input>` 或 `<textarea>`。它們有較完整的表單語意、鍵盤操作與瀏覽器支援。

## 重點整理

- `contenteditable="true"` 可以讓普通 HTML 元素變成可編輯。
- `contenteditable` 常見值包含 `true`、`false` 與 `plaintext-only`。
- `contenteditable` 不會自動儲存，也不會自動隨表單提交。
- `spellcheck` 控制瀏覽器是否對文字進行拼寫與語法檢查。
- 拼寫檢查行為由瀏覽器決定，不同環境可能不同。
- 編輯內容如果要正式儲存，仍然需要 JavaScript 與資料處理流程。
