# 鍵盤操作與焦點順序屬性

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 `accesskey` 如何替元素設定鍵盤快捷鍵。
- 理解 `tabindex` 如何影響鍵盤焦點順序。
- 知道為什麼實務上要謹慎使用快捷鍵與正整數 `tabindex`。
- 從可及性角度判斷鍵盤操作是否自然。

## 問題情境

不是所有使用者都用滑鼠操作網頁。有人習慣用鍵盤快速跳到連結、按鈕或表單欄位，也有人依賴鍵盤操作才能順利使用網站。

HTML 提供了兩個和鍵盤操作密切相關的全局屬性：

- `accesskey`：提供快捷鍵。
- `tabindex`：調整 Tab 鍵焦點順序。

它們都能影響操作效率，但也很容易破壞使用者習慣，所以要小心使用。

## 一句話理解

`accesskey` 是快速啟用元素的快捷鍵，`tabindex` 是元素在鍵盤 Tab 導航中的焦點規則。

## `accesskey`：設定訪問快捷鍵

`accesskey` 可以為元素設定一個或多個鍵盤快捷方式，讓熟悉網站的使用者快速訪問常用元素。

```html
<a href="https://www.baidu.com" accesskey="a">百度</a>
<a href="https://www.sina.com" accesskey="g">新浪</a>
```

基本語法：

```html
<element accesskey="character">
```

不過，`accesskey` 的啟用方式會依作業系統與瀏覽器而異。例如 Windows 上常見組合包含 `Alt + key` 或 `Alt + Shift + key`，但不同瀏覽器可能不同。

在 HTML5 中，`accesskey` 可用於任何 HTML 元素。這不代表每個元素加上快捷鍵都有實務價值；它只是語法上可以使用。

## `accesskey` 的使用風險

快捷鍵可能和瀏覽器、作業系統、輔助科技或網站自身的快捷鍵衝突。使用者也不一定知道頁面上有哪些 `accesskey`。

因此實務上應該只在非常確定有價值時使用，例如：

- 管理後台中高頻使用的功能。
- 內部系統中經過清楚訓練的快捷鍵。
- 搭配頁面說明，明確告知使用者快捷鍵。

一般內容頁不需要為每個連結都加 `accesskey`。

## `tabindex`：控制 Tab 焦點順序

`tabindex` 屬性規定使用 Tab 鍵導航時，元素如何進入焦點順序。

```html
<a href="https://www.google.com/" tabindex="1">Google</a>
<a href="https://www.w3school.com.cn/" tabindex="2">W3School</a>
<a href="https://www.microsoft.com/" tabindex="3">Microsoft</a>
```

這種寫法會用正整數重新指定焦點順序。不過現代實務通常不建議這樣做，因為它會讓鍵盤順序脫離 DOM 內容順序，維護成本高，也容易讓使用者迷失。

## 常見 `tabindex` 值

| 寫法 | 意思 | 實務建議 |
| --- | --- | --- |
| 不寫 `tabindex` | 依元素本身與 DOM 順序決定是否可聚焦 | 優先選擇 |
| `tabindex="0"` | 加入自然 Tab 順序 | 可用於需要聚焦的自訂互動元素 |
| `tabindex="-1"` | 不進入 Tab 順序，但可用程式聚焦 | 常用於跳轉焦點或錯誤提示 |
| `tabindex` 大於 `0` | 自訂優先焦點順序 | 通常應避免 |

最穩定的做法是：讓 HTML 的 DOM 順序本身就符合使用者操作順序。

## 較好的焦點順序寫法

如果操作順序應該是「姓名、電子郵件、送出」，就讓 HTML 也照這個順序排列：

```html
<form>
  <label for="name">姓名</label>
  <input id="name" name="name" type="text">

  <label for="email">電子郵件</label>
  <input id="email" name="email" type="email">

  <button type="submit">送出</button>
</form>
```

不要用正整數 `tabindex` 去修補混亂的 HTML 順序。

## 常見錯誤

### 用正整數 `tabindex` 重排整個頁面

不建議：

```html
<button tabindex="3">送出</button>
<input tabindex="1" type="text">
<input tabindex="2" type="email">
```

較好的做法是調整 HTML 順序：

```html
<input type="text">
<input type="email">
<button>送出</button>
```

### 為太多元素設定 `accesskey`

快捷鍵太多會讓網站難以記憶，也容易和瀏覽器快捷鍵衝突。除非使用者真的需要，否則不要把 `accesskey` 當成一般裝飾屬性。

## 重點整理

- `accesskey` 可以替元素設定快捷鍵，但啟用方式依作業系統與瀏覽器而異。
- `accesskey` 可能造成快捷鍵衝突，應謹慎使用。
- `tabindex` 控制元素在鍵盤 Tab 導航中的焦點行為。
- 通常應避免使用大於 `0` 的 `tabindex`。
- 優先讓 DOM 順序本身符合操作順序，再視需要使用 `tabindex="0"` 或 `tabindex="-1"`。
