# 語言、文字方向與翻譯屬性

## 學習目標

讀完這篇筆記，你應該能夠：

- 使用 `lang` 標示內容語言。
- 使用 `dir` 控制文字方向。
- 使用 `translate` 指定內容是否應被翻譯。
- 理解這些屬性對可及性、翻譯與國際化內容的意義。

## 問題情境

同一個網頁可能包含中文、英文、阿拉伯文、品牌名稱、書名或不應被翻譯的專有名詞。如果 HTML 沒有清楚標示語言與文字方向，瀏覽器、翻譯工具、搜尋引擎與輔助科技就只能猜測。

這一組屬性不是為了改變外觀而已，而是讓機器更正確地理解文字內容。

## 一句話理解

`lang` 說明內容使用哪種語言，`dir` 說明文字往哪個方向排列，`translate` 說明內容是否應被翻譯。

## `lang`：標示元素內容語言

`lang` 屬性規定元素內容的語言。它可以放在 `<html>` 上，標示整份文件的主要語言：

```html
<html lang="zh-TW">
```

也可以放在局部元素上，標示某一段內容使用不同語言：

```html
<p>這是一段中文內容，包含一個英文詞 <span lang="en">accessibility</span>。</p>
```

`lang` 的屬性值應使用有效的 BCP 47 語言標籤，例如：

- `en`
- `en-US`
- `zh-TW`

![lang 屬性常用語言代碼列表](../../origin/第05章_全局屬性/assets/images/lang-attribute-img-001-38e880.png)

在 HTML5 中，`lang` 可用於任何 HTML 元素。實務上最常見的是先在 `<html>` 上設定整份文件語言，再針對局部不同語言的片段另外設定。

## `dir`：控制文字方向

`dir` 屬性規定元素內容的文字方向。

![dir 屬性值與文字方向對照表](../../origin/第05章_全局屬性/assets/images/dir-attribute-img-001-5f8241.png)

常見值包含：

| 值 | 意思 |
| --- | --- |
| `ltr` | left to right，從左到右 |
| `rtl` | right to left，從右到左 |
| `auto` | 由瀏覽器根據內容判斷方向 |

例如：

```html
<p dir="rtl">文本方向從右到左!</p>
```

如果網站主要服務從右到左書寫的語言，可以在較上層的元素設定方向；如果只是某一小段文字需要不同方向，就只在該段元素上設定。

## `translate`：控制內容是否被翻譯

`translate` 屬性規定元素內容是否應該被翻譯。

當內容是品牌名稱、作品名稱、程式碼、專有名詞或不應被機器翻譯的文字，可以使用 `translate="no"`：

```html
<p>
  The question in the
  <cite translate="no">How Far Can You Go?</cite>
  applies to this chapter.
</p>
```

如果整份文件都不希望被翻譯，可以放在 `<html>` 上：

```html
<!DOCTYPE html>
<html translate="no">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>
  <p>
    The question in the
    <cite>How Far Can You Go?</cite>
    applies to this chapter.
  </p>
</body>
</html>
```

實務上，通常不會隨意禁止全站翻譯。更常見的做法是只對不該翻譯的局部文字加上 `translate="no"`。

## 三者如何一起使用

以下範例同時處理整份文件語言、局部外語內容、文字方向與禁止翻譯：

```html
<!DOCTYPE html>
<html lang="zh-TW">
<head>
  <meta charset="UTF-8">
  <title>語言屬性範例</title>
</head>
<body>
  <p>
    這篇文章會提到
    <cite lang="en" translate="no">How Far Can You Go?</cite>
    這本書。
  </p>

  <p lang="ar" dir="rtl">هذا نص عربي.</p>
</body>
</html>
```

拆開來看：

- `<html lang="zh-TW">`：整份文件主要是繁體中文。
- `lang="en"`：書名是英文。
- `translate="no"`：書名不希望被翻譯。
- `lang="ar" dir="rtl"`：這段是阿拉伯文，文字方向從右到左。

## 常見錯誤

### 完全不設定文件語言

不建議：

```html
<html>
```

較好的做法：

```html
<html lang="zh-TW">
```

文件語言可以幫助瀏覽器、搜尋引擎、翻譯工具與輔助科技更正確地處理內容。

### 用 CSS 取代語意屬性

如果問題是文字方向，不要只用 CSS 做表面排列，應該讓 HTML 也表達文字方向：

```html
<p lang="ar" dir="rtl">هذا نص عربي.</p>
```

## 重點整理

- `lang` 用來標示內容語言，建議至少在 `<html>` 上設定。
- `lang` 值應使用有效語言標籤，例如 `en`、`en-US`、`zh-TW`。
- `dir` 用來標示文字方向，常見值有 `ltr`、`rtl`、`auto`。
- `translate="no"` 適合用在不應被翻譯的專有名詞、品牌、作品名稱或程式碼片段。
- 語言、方向與翻譯屬性都會影響機器如何理解內容，不只是畫面呈現。
