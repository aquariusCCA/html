# HTML 全局屬性

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明全局屬性的用途，以及為什麼它們可以套用在多數 HTML 元素上。
- 正確使用 `id`、`class`、`style`、`title`、`hidden` 與 `data-*`。
- 理解 `lang`、`dir`、`translate` 與 `spellcheck` 對語言、文字方向與輸入體驗的影響。
- 使用 `contenteditable`、`draggable`、`tabindex`、`accesskey` 等屬性時知道它們的限制。
- 分辨哪些屬性適合實務使用，哪些屬性因支援度或維護性問題不建議依賴。
- 說明全局屬性與 CSS、JavaScript、DOM、可訪問性的關係。

## 2. 本章定位

全局屬性是 HTML 屬性中很重要的一類。它們不像 `href` 主要屬於 `<a>`，也不像 `src` 常見於 `<img>` 或 `<script>`；全局屬性通常可以寫在多數 HTML 元素上，用來補充元素的識別、分類、樣式、語言、互動狀態或自訂資料。

本章的原始資料分散介紹多個全局屬性。單一屬性的內容都不算大，合併成一份「HTML 全局屬性」總覽筆記比較適合初學者建立整體觀念。若後續要製作練習題或深入範例，可以再把 `data-*`、拖放、可訪問性焦點控制、國際化屬性拆成獨立補充。

> 來源重點：原始資料主要整理了 `accesskey`、`class`、`contenteditable`、`contextmenu`、`data-*`、`dir`、`draggable`、`dropzone`、`hidden`、`id`、`lang`、`spellcheck`、`style`、`tabindex`、`title` 與 `translate` 等全局屬性。

## 3. 核心概念

### 3.1 什麼是全局屬性

全局屬性是可以用在大多數 HTML 元素上的屬性。它們不一定直接改變元素的語意，但會提供元素額外資訊，讓 CSS、JavaScript、瀏覽器或輔助工具能更好地處理這個元素。

例如：

```html
<p id="intro" class="lead" lang="zh-Hant" data-section="hero">
  歡迎學習 HTML。
</p>
```

這個 `<p>` 元素除了本身代表段落之外，還透過屬性補充了以下資訊：

| 屬性 | 補充資訊 |
|---|---|
| `id="intro"` | 這個元素在文件中的唯一識別名稱。 |
| `class="lead"` | 這個元素屬於 `lead` 這個分類，可供 CSS 或 JavaScript 選取。 |
| `lang="zh-Hant"` | 這段內容主要使用繁體中文。 |
| `data-section="hero"` | 存放開發者自訂資料，可供 JavaScript 讀取。 |

### 3.2 可以使用不代表一定有用

原始資料多次提到：在 HTML5 中，許多全局屬性可以用於任何 HTML 元素，但不一定在每個元素上都有實際價值。

例如：

```html
<br class="line-break">
```

這樣寫通常不會造成語法錯誤，但對學習與維護未必有幫助。真正重要的是判斷這個屬性是否能清楚表達元素的用途、狀態或操作需求。

> 注意：不要因為全局屬性「可以寫」就隨意添加。屬性越多，DOM 結構越難閱讀，也可能增加 CSS、JavaScript 與可訪問性維護成本。

### 3.3 全局屬性的常見分類

本章可以把全局屬性分成幾個常見用途：

| 類型 | 常見屬性 | 主要用途 |
|---|---|---|
| 識別與分類 | `id`、`class` | 讓元素可被 CSS、JavaScript 或頁內連結識別。 |
| 自訂資料 | `data-*` | 在 HTML 元素上存放與頁面互動相關的自訂資料。 |
| 樣式與提示 | `style`、`title` | 設定行內樣式或提供額外提示文字。 |
| 語言與文字方向 | `lang`、`dir`、`translate` | 表示內容語言、文字方向與是否翻譯。 |
| 編輯與輸入輔助 | `contenteditable`、`spellcheck` | 讓內容可編輯，或控制拼字檢查。 |
| 顯示與互動 | `hidden`、`draggable` | 控制元素是否隱藏，或是否可拖曳。 |
| 鍵盤操作 | `tabindex`、`accesskey` | 影響鍵盤焦點順序或快捷鍵。 |
| 低支援或不建議依賴 | `contextmenu`、`dropzone` | 原始資料標示主流瀏覽器支援不佳，實務上應謹慎使用。 |

## 4. 語法與基本用法

### 4.1 全局屬性的基本語法

屬性寫在開始標籤中，通常使用 `屬性名="屬性值"` 的形式：

```html
<element attribute="value">內容</element>
```

實際範例：

```html
<button type="button" id="save-button" class="primary-action" data-action="save">
  儲存
</button>
```

其中：

| 部分 | 說明 |
|---|---|
| `id="save-button"` | 讓元素有唯一識別名稱。 |
| `class="primary-action"` | 讓元素可被 CSS 或 JavaScript 依分類選取。 |
| `data-action="save"` | 存放自訂資料，表示這個按鈕的操作目的。 |

### 4.2 常用全局屬性速查

| 屬性 | 常見寫法 | 主要用途 | 實務注意事項 |
|---|---|---|---|
| `id` | `id="main-title"` | 定義文件中唯一的元素識別名稱。 | 同一份 HTML 文件中不應重複。 |
| `class` | `class="card active"` | 定義元素分類，常用於 CSS 與 JavaScript。 | 多個 class 用空格分隔，不要重複寫多個 `class` 屬性。 |
| `style` | `style="color: red"` | 設定行內樣式。 | 會提高樣式覆蓋優先權，實務上不宜大量使用。 |
| `title` | `title="補充說明"` | 提供元素額外資訊，常見為滑鼠停留提示。 | 不應把重要資訊只放在 `title`，鍵盤與觸控使用者可能不容易取得。 |
| `hidden` | `<p hidden>` | 隱藏元素。 | 若要顯示元素，可用 JavaScript 移除或切換此屬性。 |
| `data-*` | `data-user-id="42"` | 存放頁面或應用程式需要的自訂資料。 | 屬性名應以 `data-` 開頭，後面至少有一個字元，且建議使用小寫與連字號。 |
| `lang` | `lang="en"` | 指定元素內容語言。 | 有助於搜尋引擎、瀏覽器與螢幕閱讀器理解語言。 |
| `dir` | `dir="rtl"` | 指定文字方向。 | 常見值有 `ltr`、`rtl`、`auto`。 |
| `translate` | `translate="no"` | 指定內容是否應被翻譯。 | 常用於品牌名、書名、程式碼或專有名詞。 |
| `contenteditable` | `contenteditable="true"` | 讓元素內容可被使用者編輯。 | 只讓畫面可編輯，不等於自動保存資料。 |
| `spellcheck` | `spellcheck="true"` | 控制是否進行拼字或語法檢查。 | 實際檢查方式由瀏覽器與使用者環境決定。 |
| `draggable` | `draggable="true"` | 指定元素是否可拖曳。 | 通常需要搭配 JavaScript Drag and Drop API。 |
| `tabindex` | `tabindex="0"` | 控制元素是否可被 Tab 鍵聚焦，以及焦點順序。 | 通常優先使用自然 DOM 順序，避免濫用正整數。 |
| `accesskey` | `accesskey="s"` | 為元素設定鍵盤快捷鍵。 | 不同作業系統與瀏覽器啟用方式不同，可能與既有快捷鍵衝突。 |
| `contextmenu` | `contextmenu="menu-id"` | 指定元素右鍵選單。 | 原始資料標示主流瀏覽器支援不佳，不建議依賴。 |
| `dropzone` | `dropzone="copy"` | 指定拖放資料進入元素時的處理意圖。 | 原始資料標示主流瀏覽器不支援，實務上不建議使用。 |

### 4.3 `id` 與 `class`

`id` 用來標記文件中的唯一元素，常用於頁內連結、JavaScript 選取或特定區塊定位。

```html
<h2 id="features">產品特色</h2>
<a href="#features">跳到產品特色</a>
```

`class` 用來表示元素分類。多個元素可以共用同一個 class，同一個元素也可以有多個 class：

```html
<p class="intro important">這是一段重要的介紹文字。</p>
```

CSS 可以用 class 或 id 選取元素：

```css
.intro {
  color: blue;
}

#features {
  scroll-margin-top: 80px;
}
```

> 注意：`id` 適合唯一識別，`class` 適合分類與重複套用。不要為了套樣式而替每個元素都建立不同 `id`。

### 4.4 `data-*`

`data-*` 屬性用來在 HTML 元素上存放自訂資料，常見於前端互動。它適合保存與元素相關、但不適合用語意化 HTML 屬性表達的資訊。

```html
<li class="animal" data-animal-type="bird">喜鵲</li>
<li class="animal" data-animal-type="fish">金槍魚</li>
<li class="animal" data-animal-type="spider">蠅虎</li>
```

JavaScript 可以透過 `dataset` 讀取：

```js
const animals = document.querySelectorAll(".animal");

for (const animal of animals) {
  console.log(animal.dataset.animalType);
}
```

`data-animal-type` 在 `dataset` 中會變成 `animalType`。這是因為連字號後的字母會轉成 camelCase 形式。

也可以透過 JavaScript 設定：

```js
const apple = document.querySelector("#apple");

apple.setAttribute("data-food-type", "fruit");
console.log(apple.dataset.foodType);
```

> 注意：`data-*` 內容會出現在 HTML 或 DOM 中，不適合存放密碼、token、個人隱私或其他敏感資料。

### 4.5 `hidden`

`hidden` 屬性表示元素目前不應顯示。常見情境是某段內容需要等使用者完成操作後才顯示。

```html
<p>這是一段可見內容。</p>
<p hidden>這是一段暫時隱藏的內容。</p>
```

JavaScript 可以切換 `hidden` 狀態：

```html
<button type="button" id="toggle-help">顯示或隱藏說明</button>
<p id="help-text" hidden>請輸入有效的電子郵件地址。</p>

<script>
  const button = document.querySelector("#toggle-help");
  const helpText = document.querySelector("#help-text");

  button.addEventListener("click", () => {
    helpText.hidden = !helpText.hidden;
  });
</script>
```

> 補充說明：若內容只是要「視覺上隱藏，但仍提供給輔助工具」，通常不應直接使用 `hidden`，而要使用專門的 visually hidden CSS 寫法。

### 4.6 語言、文字方向與翻譯控制

`lang` 用來標示內容語言：

```html
<p lang="en">This paragraph is written in English.</p>
```

`dir` 用來控制文字方向：

```html
<p dir="rtl">這段文字會以從右到左的方向呈現。</p>
```

`translate` 用來指定內容是否適合被自動翻譯：

```html
<p>
  書名
  <cite translate="no">How Far Can You Go?</cite>
  不應被翻譯成其他語言。
</p>
```

如果整站內容都不希望被自動翻譯，可以放在 `<html>` 上：

```html
<html lang="en" translate="no">
  <!-- page content -->
</html>
```

> 注意：是否翻譯仍可能受到瀏覽器、翻譯工具或使用者設定影響，`translate` 應視為給工具的提示，不應拿來保護機密內容。

### 4.7 `contenteditable` 與 `spellcheck`

`contenteditable` 可以讓元素內容變成可編輯：

```html
<p contenteditable="true">這段文字可以被使用者編輯。</p>
<p contenteditable="false">這段文字不允許被編輯。</p>
```

`spellcheck` 可以提示瀏覽器是否對文字進行拼字或語法檢查，常用於文字輸入、`textarea` 或可編輯元素：

```html
<label for="message">留言</label>
<textarea id="message" name="message" spellcheck="true"></textarea>

<div contenteditable="true" spellcheck="true">
  這是一段可編輯並可進行拼字檢查的內容。
</div>
```

> 注意：`contenteditable` 只處理瀏覽器中的編輯互動。若要保存結果，需要額外用 JavaScript 讀取內容並送到後端或儲存位置。

### 4.8 `draggable`、`contextmenu` 與 `dropzone`

`draggable` 指定元素是否可拖曳。原始資料提醒：連結與圖片通常預設可拖動，而 `draggable` 常與 JavaScript 拖放操作一起使用。

```html
<div id="drag-item" draggable="true">拖曳我</div>
<div id="drop-target">放到這裡</div>
```

實務上，只有 HTML 屬性通常不夠，還需要處理拖放事件：

```js
const item = document.querySelector("#drag-item");
const target = document.querySelector("#drop-target");

item.addEventListener("dragstart", (event) => {
  event.dataTransfer.setData("text/plain", item.id);
});

target.addEventListener("dragover", (event) => {
  event.preventDefault();
});

target.addEventListener("drop", (event) => {
  event.preventDefault();
  const id = event.dataTransfer.getData("text/plain");
  target.append(document.getElementById(id));
});
```

`contextmenu` 原始用途是把元素與自訂右鍵選單關聯：

```html
<p contextmenu="my-menu">右鍵點擊這段文字。</p>
```

`dropzone` 原始用途是描述拖放資料進入元素時要複製、移動或建立連結：

```html
<div dropzone="copy"></div>
```

> 注意：原始資料標示 `contextmenu` 只有 Firefox 支援，`dropzone` 則標示所有主流瀏覽器都不支援。實務開發不建議依賴這兩個屬性；若要做拖放或自訂選單，通常改用 JavaScript 事件與可訪問性設計來處理。

### 4.9 `tabindex` 與 `accesskey`

`tabindex` 會影響使用者按 Tab 鍵時的焦點行為：

```html
<button type="button" tabindex="0">可以依自然順序聚焦</button>
<div tabindex="0">這個區塊也可以被鍵盤聚焦</div>
```

常見觀念：

| 寫法 | 意義 |
|---|---|
| `tabindex="0"` | 讓元素可被 Tab 聚焦，順序依照 DOM 自然順序。 |
| `tabindex="-1"` | 元素不能被 Tab 自然聚焦，但可用 JavaScript 聚焦。 |
| `tabindex="1"` 或更大 | 強制指定焦點順序，通常不建議濫用。 |

`accesskey` 可以替元素設定快捷鍵：

```html
<a href="/search" accesskey="s">搜尋</a>
```

但不同作業系統與瀏覽器使用快捷鍵的方式不一樣。例如原始資料提到 Windows 下常見方式是 `Alt + accesskey` 的屬性值，但實際組合可能因瀏覽器而異。

> 注意：鍵盤操作直接影響可訪問性。通常應優先使用正確的 HTML 結構與自然焦點順序，而不是大量使用 `tabindex` 或 `accesskey` 強行改變使用者操作方式。

## 5. 範例說明

### 範例一：用 `id`、`class` 與 `data-*` 連接 HTML、CSS、JavaScript

```html
<article id="product-42" class="product-card featured" data-product-id="42">
  <h2 class="product-card__title">經典馬克杯</h2>
  <p class="product-card__price" data-currency="TWD">NT$ 450</p>
  <button type="button" class="product-card__action" data-action="add-to-cart">
    加入購物車
  </button>
</article>
```

這個範例中：

- `id="product-42"` 可以唯一識別這張商品卡。
- `class="product-card featured"` 讓 CSS 可以套用卡片樣式，也可以標記它是精選商品。
- `data-product-id="42"` 存放商品 ID，供 JavaScript 處理加入購物車等互動。
- `data-action="add-to-cart"` 表示按鈕的操作語意，方便事件委派時判斷行為。

搭配 JavaScript：

```js
const product = document.querySelector("#product-42");
const button = product.querySelector("[data-action='add-to-cart']");

button.addEventListener("click", () => {
  console.log(product.dataset.productId);
});
```

### 範例二：使用語言與翻譯屬性保留專有名詞

```html
<p lang="zh-Hant">
  本章會提到
  <span lang="en" translate="no">HTML Global Attributes</span>
  這個主題。
</p>
```

這個範例中：

- 外層段落使用 `lang="zh-Hant"` 表示主要內容是繁體中文。
- 內層英文術語使用 `lang="en"`，讓工具知道該片段是英文。
- `translate="no"` 表示這個英文術語不建議被自動翻譯。

### 範例三：可編輯內容與拼字檢查

```html
<section>
  <h2>個人簡介</h2>
  <p id="bio" contenteditable="true" spellcheck="true">
    請在這裡輸入你的個人簡介。
  </p>
  <button type="button" id="save-bio">儲存</button>
</section>
```

這個範例中，`contenteditable="true"` 讓段落可以直接在頁面上編輯，`spellcheck="true"` 則提示瀏覽器可以進行拼字檢查。

> 注意：這個範例只示範前端可編輯狀態。真正儲存時，仍需要 JavaScript 讀取 `bio` 的內容，並根據專案需求送到後端或本地儲存。

## 6. 實務應用場景

全局屬性常出現在以下場景：

- CSS 樣式管理：用 `class` 表示元件狀態，例如 `is-active`、`is-disabled`。
- JavaScript DOM 操作：用 `id`、`class`、`data-*` 選取元素並讀取資料。
- 元件化開發：在 Vue 或其他框架中，`class`、`style`、`data-*` 仍會反映到最後的 DOM。
- 國際化內容：用 `lang`、`dir`、`translate` 幫助瀏覽器與翻譯工具理解內容。
- 可訪問性：用合理的焦點順序、語言標示與隱藏狀態，讓鍵盤與輔助工具使用者更容易操作。
- 拖放與互動：用 `draggable` 標示可拖曳元素，再搭配 JavaScript 處理事件。
- 使用者輸入體驗：用 `spellcheck` 改善文字輸入或編輯體驗。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 在同一份文件中重複使用相同 `id` | `id` 應該唯一，重複會讓 CSS、JavaScript 或頁內連結行為變得難以預測。 | 改用唯一 `id`，重複分類需求使用 `class`。 |
| 為同一元素寫多個 `class` 屬性 | HTML 不應重複同名屬性。 | 把多個 class 放在同一個屬性值中，用空格分隔。 |
| 大量使用 `style` 寫樣式 | 行內樣式不易維護，也容易覆蓋外部 CSS。 | 優先使用 class 搭配 CSS 檔案管理樣式。 |
| 把重要說明只放在 `title` | `title` 不一定能被觸控、鍵盤或輔助工具使用者穩定取得。 | 重要資訊應放在可見文字、label 或適當的輔助文字中。 |
| 用正整數 `tabindex` 強制調整焦點順序 | 會破壞 DOM 自然順序，頁面變大後很難維護。 | 優先調整 HTML 結構，必要時使用 `tabindex="0"` 或 `-1`。 |
| 依賴 `accesskey` 作為主要操作方式 | 快捷鍵組合因系統與瀏覽器不同，且可能衝突。 | 把它當輔助功能，不要取代清楚的按鈕、連結與表單操作。 |
| 把敏感資料放進 `data-*` | `data-*` 會暴露在 DOM 中，使用者可透過開發者工具看到。 | 敏感資料應留在後端或透過安全流程取得。 |
| 使用 `contenteditable` 但沒有保存與清理資料 | 使用者修改的內容只存在瀏覽器畫面中，也可能輸入不安全 HTML。 | 明確設計儲存流程，送出前進行必要驗證與清理。 |
| 依賴 `contextmenu` 或 `dropzone` | 原始資料標示這些屬性支援不佳。 | 改用 JavaScript 事件與可訪問性友善的互動設計。 |

### 錯誤範例：重複寫 `class`

```html
<p class="intro" class="important">這是一段文字。</p>
```

問題：同一個元素不應重複寫同名屬性，瀏覽器解析結果不應依賴。

### 正確範例：多個 class 用空格分隔

```html
<p class="intro important">這是一段文字。</p>
```

這樣同一個元素同時屬於 `intro` 與 `important` 兩個分類。

### 錯誤範例：用 `style` 管理大量樣式

```html
<h1 style="color: blue; text-align: center; margin-top: 40px;">標題</h1>
```

問題：當頁面變大後，樣式會散落在 HTML 中，不利於統一調整。

### 正確範例：用 class 管理樣式

```html
<h1 class="page-title">標題</h1>
```

```css
.page-title {
  color: blue;
  margin-top: 40px;
  text-align: center;
}
```

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| CSS | `class`、`id`、`style` 會直接影響樣式選取與覆蓋順序。 |
| JavaScript | JavaScript 可以透過 DOM API 讀寫 `id`、`class`、`data-*`、`hidden` 等屬性。 |
| DOM | HTML 會被瀏覽器解析成 DOM，全局屬性會成為 DOM 節點上的屬性或 attribute。 |
| 可訪問性 | `lang`、`hidden`、`tabindex`、`accesskey` 等會影響輔助工具與鍵盤操作體驗。 |
| SEO | `lang`、合理的元素識別與可讀結構有助於搜尋引擎理解內容，但 SEO 不只靠屬性。 |
| 國際化 | `lang`、`dir`、`translate` 可協助多語系內容被瀏覽器、翻譯工具與輔助工具正確處理。 |
| Vue 或其他框架 | template 最終仍會渲染成 DOM，因此全局屬性的語意與行為仍然重要。 |

## 9. 重點整理

- 全局屬性通常可以用在多數 HTML 元素上，但「可以用」不等於「應該用」。
- `id` 適合唯一識別，`class` 適合分類與重複套用。
- `data-*` 適合存放前端互動需要的自訂資料，但不可存放敏感資訊。
- `style` 是行內樣式，實務上應避免大量使用，優先用 CSS class 管理樣式。
- `lang`、`dir`、`translate` 有助於語言、文字方向與翻譯控制。
- `hidden` 可隱藏元素，JavaScript 可以切換它的狀態。
- `contenteditable` 讓內容可編輯，但不會自動保存，也需要注意輸入資料處理。
- `tabindex` 與 `accesskey` 會影響鍵盤操作，使用時要特別考慮可訪問性。
- `contextmenu` 與 `dropzone` 原始資料標示支援不佳，不建議在正式專案中依賴。
