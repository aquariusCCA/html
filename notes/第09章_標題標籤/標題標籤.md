# HTML 標題標籤

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 `h1` 到 `h6` 的用途與語意。
- 分辨標題層級與文字大小之間的差異。
- 正確安排網頁中的標題結構。
- 避免把標題標籤當成單純的字體放大工具。

## 2. 本章定位

標題標籤是 HTML 文件結構中的基礎元素，用來描述頁面內容的層級。

在新聞、文章、文件、商品頁、後台系統頁面中，標題通常用來指出頁面主題、段落主題或區塊主題。良好的標題結構可以幫助使用者快速理解內容，也能讓瀏覽器、搜尋引擎與輔助工具更容易判斷頁面架構。

## 3. 核心概念

HTML 提供六個標題元素：

```html
<h1>一級標題</h1>
<h2>二級標題</h2>
<h3>三級標題</h3>
<h4>四級標題</h4>
<h5>五級標題</h5>
<h6>六級標題</h6>
```

`h1` 到 `h6` 表示標題層級，數字越小，層級越高；數字越大，層級越低。

| 標籤 | 語意 |
|---|---|
| `h1` | 頁面或主要內容的最高層級標題 |
| `h2` | `h1` 底下的主要段落標題 |
| `h3` | `h2` 底下的子段落標題 |
| `h4` 到 `h6` | 更細的內容層級 |

> 來源重點：原始資料指出標題標籤的語意是「作為標題使用，並且依據重要性遞減」。

### 3.1 標題標籤的重點是語意

標題標籤不是單純用來讓文字變大或變粗，而是用來表達內容在文件中的層級。

例如一篇文章可以有一個文章主題，底下再分成多個段落主題：

```html
<h1>HTML 入門教學</h1>

<h2>HTML 是什麼</h2>
<p>HTML 是用來描述網頁內容與結構的標記語言。</p>

<h2>HTML 標籤的基本結構</h2>
<p>大多數 HTML 元素由開始標籤、內容與結束標籤組成。</p>
```

這裡的 `h1` 表示整篇內容的主題，`h2` 表示主題底下的主要章節。

### 3.2 預設樣式不等於真正用途

瀏覽器通常會替 `h1` 到 `h6` 套用預設樣式，例如：

- 文字通常會加粗。
- `h1` 預設字體通常最大，往 `h6` 逐漸變小。
- 標題元素通常會獨佔一行，前後也可能有預設間距。

但這些只是瀏覽器的預設呈現方式，不是選擇標題標籤的主要理由。是否使用 `h1` 到 `h6`，應該先看內容層級，而不是看字體大小。

> 補充說明：如果只是想改變文字大小、粗細或間距，應該使用 CSS 控制樣式，而不是為了視覺效果亂用標題層級。

## 4. 語法與基本用法

標題元素由開始標籤、標題文字與結束標籤組成：

```html
<h1>頁面主標題</h1>
```

常見用法如下：

```html
<h1>網站最新消息</h1>

<h2>產品更新</h2>
<p>本段介紹產品的新功能。</p>

<h2>活動公告</h2>
<p>本段介紹近期活動。</p>

<h3>報名方式</h3>
<p>本段說明活動報名流程。</p>
```

標題層級應該反映內容結構。若 `h3` 是某個 `h2` 的子內容，它就應該出現在對應的 `h2` 底下。

## 5. 範例說明

### 範例：文章頁標題結構

```html
<article>
  <h1>如何開始學習 HTML</h1>

  <section>
    <h2>先理解 HTML 的用途</h2>
    <p>HTML 用來描述網頁內容與結構。</p>
  </section>

  <section>
    <h2>再學習常見標籤</h2>
    <p>例如標題、段落、連結、圖片與表單元素。</p>

    <h3>標題標籤</h3>
    <p>標題標籤用來建立內容層級。</p>
  </section>
</article>
```

這個範例中：

- `article` 表示一篇可獨立閱讀的文章內容。
- `h1` 是文章主標題。
- `h2` 是文章中的主要段落標題。
- `h3` 是某個 `h2` 底下更細的子段落標題。

### 範例：用 CSS 控制標題外觀

```html
<h2 class="section-title">課程特色</h2>
```

```css
.section-title {
  font-size: 32px;
  font-weight: 700;
  margin-block: 24px 12px;
}
```

這個例子中，`h2` 負責表達「這是某個章節標題」的語意，CSS 則負責控制字體大小、粗細與間距。

## 6. 實務應用場景

標題標籤常見於以下場景：

- 文章頁：用 `h1` 表示文章標題，用 `h2`、`h3` 組織段落。
- 新聞頁：用標題標籤凸顯新聞主題與內容分段。
- 商品頁：用 `h1` 表示商品名稱，用 `h2` 表示商品規格、介紹、評價等區塊。
- 後台系統：用標題標籤標示頁面名稱、表單區塊或資料區塊。
- 文件頁：用清楚的標題層級建立可掃描的內容結構。

在實務開發中，標題結構會影響：

- 使用者掃讀頁面的效率。
- CSS 選擇器與樣式維護。
- JavaScript 透過 DOM 查找或操作標題節點。
- 搜尋引擎對頁面主題與段落結構的理解。
- 螢幕閱讀器使用者透過標題快速跳轉內容。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 用 `h1` 只是為了讓字變大 | `h1` 表示最高層級標題，不是字體大小工具 | 使用符合內容層級的標題，再用 CSS 調整外觀 |
| 標題層級亂跳，例如 `h1` 後直接接 `h4` | 內容結構不清楚，輔助工具與讀者較難理解層級 | 依照內容關係安排 `h1`、`h2`、`h3` |
| 整頁沒有清楚主標題 | 使用者與搜尋引擎較難判斷頁面主要內容 | 為主要內容提供清楚的 `h1` 或等價主標題 |
| 把普通段落文字包成標題 | 會讓不重要的內容被誤認為章節標題 | 一般文字使用 `p`，只有章節名稱才使用標題標籤 |
| 為了取消粗體而不用標題標籤 | 混淆語意與樣式責任 | 保留正確標題元素，用 CSS 調整 `font-weight` |

### 錯誤範例

```html
<h1>商品介紹</h1>
<h4>特色</h4>
<p>這裡介紹商品特色。</p>
```

問題：`h1` 後直接跳到 `h4`，中間缺少合理的層級脈絡。

### 正確範例

```html
<h1>商品介紹</h1>
<h2>特色</h2>
<p>這裡介紹商品特色。</p>
```

修正後，`h2` 清楚表示它是 `h1` 底下的主要段落標題。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| 語意化 HTML | 標題標籤用來表達文件層級，是語意化 HTML 的重要基礎 |
| CSS | CSS 負責控制標題的字體大小、粗細、顏色與間距 |
| DOM | 瀏覽器會把標題元素解析成 DOM 節點，供 CSS 與 JavaScript 使用 |
| JavaScript | JavaScript 可以透過 DOM API 取得或操作標題元素 |
| SEO | 合理的標題層級有助於搜尋引擎理解頁面內容結構，但 SEO 不只取決於標題標籤 |
| 可訪問性 | 螢幕閱讀器使用者常透過標題列表快速理解與跳轉頁面 |

## 9. 重點整理

- `h1` 到 `h6` 是標題元素，用來表示內容層級。
- `h1` 層級最高，`h6` 層級最低，重要性依序遞減。
- 標題元素常見預設樣式是加粗、字體大小不同、獨佔一行。
- 預設樣式不是標題標籤的真正用途；真正用途是表達文件結構。
- 視覺樣式應交給 CSS，HTML 標題標籤應根據內容語意選擇。
- 良好的標題結構有助於閱讀、維護、SEO 與可訪問性。

## 11. 延伸補充

> 補充說明：HTML 負責描述內容與結構，DOM 是瀏覽器根據 HTML 建立出的物件模型。標題元素在 DOM 中也是節點，因此 CSS 可以選取它們套用樣式，JavaScript 也可以查找或操作它們。

在專案中，通常建議每個頁面或主要內容區域有清楚的主標題。這不只是為了畫面好看，而是為了讓內容主題更容易被使用者、輔助工具與搜尋引擎理解。
