# 常見圖片格式與 Base64 編碼

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨 JPG、PNG、BMP、GIF、WebP 的主要特性。
- 根據圖片用途選擇合適的圖片格式。
- 理解 Base64 不是圖片格式，而是一種編碼方式。
- 知道什麼情境適合使用 Base64，什麼情境不適合。

## 問題情境

網頁中的圖片可能是商品照片、Logo、透明圖示、動態圖，也可能是小型內嵌圖像。不同圖片格式的壓縮方式、檔案大小、透明支援、動畫支援都不一樣。

如果格式選錯，可能會出現這些問題：

- 商品照片檔案太大，頁面載入很慢。
- Logo 背景不能透明。
- 動態圖片無法播放。
- 為了省請求把大圖轉成 Base64，反而讓 HTML 或 CSS 檔案變得很大。

因此學圖片標籤時，也需要知道常見圖片格式的取捨。

## 一句話理解

圖片格式要依內容特性選擇；照片常用 JPG，透明圖常用 PNG 或 WebP，動畫可用 GIF 或 WebP，Base64 則是把圖片資料轉成文字的編碼方式。

## 格式總覽

| 類型 | 主要特性 | 透明背景 | 動態圖 | 常見用途 |
| --- | --- | --- | --- | --- |
| JPG / JPEG | 有損壓縮，適合豐富顏色 | 不支援 | 不支援 | 照片、產品宣傳圖 |
| PNG | 無損壓縮，品質較高 | 支援 | 不支援 | Logo、透明圖、重要配圖 |
| BMP | 通常檔案很大 | 不支援 | 不支援 | 網頁中極少使用 |
| GIF | 顏色少，可做簡單動畫 | 支援簡單透明 | 支援 | 簡單動態圖片 |
| WebP | 現代 Web 圖片格式，壓縮能力佳 | 支援 | 支援 | 網頁圖片、透明圖、動畫 |
| Base64 | 編碼方式，不是圖片格式 | 取決於原圖片 | 取決於原圖片 | 小圖或需要內嵌的圖片資料 |

## JPG / JPEG

JPG 的副檔名通常是 `.jpg` 或 `.jpeg`。它是一種有損壓縮格式，會捨棄肉眼不容易察覺的細節，藉此減少檔案大小。

主要特點：

- 支援豐富顏色。
- 檔案通常比未壓縮圖片小。
- 不支援透明背景。
- 不支援動態圖。

適合用在照片或產品宣傳圖，例如：

```html
<img src="product-photo.jpg" alt="商品情境照片">
```

如果圖片是照片，而且不需要透明背景，JPG 通常是很常見的選擇。

## PNG

PNG 的副檔名是 `.png`。它是一種無損壓縮格式，能以較高品質保存圖片。

主要特點：

- 支援豐富顏色。
- 支援透明背景。
- 不支援動態圖。
- 檔案通常比同類型 JPG 大。

PNG 適合用在需要透明背景或品質要求較高的圖片，例如公司 Logo、圖示、重要配圖。

```html
<img src="logo.png" alt="公司標誌">
```

如果你需要讓圖片背景透明，PNG 通常比 JPG 合適。

## BMP

BMP 的副檔名是 `.bmp`。常見 BMP 多為未壓縮或壓縮效率不佳，因此檔案通常很大。

主要特點：

- 可保留很多圖片細節。
- 檔案體積極大。
- 不支援透明背景。
- 不支援動態圖。
- 網頁中極少使用。

BMP 更常出現在特定軟體或原始圖片資料中，不適合作為一般網頁圖片格式。正式網頁通常會先轉成 JPG、PNG 或 WebP。

## GIF

GIF 的副檔名是 `.gif`。它只支援 256 種顏色，所以色彩呈現能力有限，但可以支援簡單動畫。

主要特點：

- 支援顏色較少。
- 支援簡單透明背景。
- 支援動態圖。
- 適合短小、簡單的動畫。

GIF 常用在簡單動態圖片，例如小型動畫圖示。

```html
<img src="loading.gif" alt="載入中">
```

如果動畫顏色很多或檔案很大，實務上也可以考慮使用影片格式或 WebP 動畫。

## WebP

WebP 的副檔名是 `.webp`，是 Google 推出的現代 Web 圖片格式，專門用來在網頁中呈現圖片。

主要特點：

- 支援有損壓縮。
- 支援無損壓縮。
- 支援透明背景。
- 支援動態圖。
- 通常能在相近品質下產生較小檔案。

WebP 適合用在多種網頁圖片場景，例如照片、透明圖與動畫。

```html
<img src="banner.webp" alt="活動主視覺">
```

主流現代瀏覽器已支援 WebP。不過如果專案需要兼容舊瀏覽器，就應該準備 fallback。常見作法是使用 `<picture>` 提供不同格式：

```html
<picture>
  <source srcset="banner.webp" type="image/webp">
  <img src="banner.jpg" alt="活動主視覺">
</picture>
```

瀏覽器支援 WebP 時會使用 `banner.webp`；不支援時，仍可使用 `banner.jpg`。

## Base64 編碼

Base64 不是圖片格式，而是一種把二進位資料轉成文字的編碼方式。

圖片經過 Base64 編碼後，會變成一串很長的文字，常見用法是 Data URL：

```html
<img src="data:image/png;base64,..." alt="內嵌小圖示">
```

這種寫法的意思是：圖片資料直接寫在 HTML 或 CSS 裡，不需要再另外透過路徑載入圖片檔案。

Base64 的特點：

- 不受圖片檔案位置影響。
- 可以把小圖片直接內嵌在 HTML 或 CSS 中。
- 文字長度通常會比原始二進位資料更大。
- 不適合大量或大型圖片。

適合情境：

- 很小的圖示。
- 需要和 HTML 或 CSS 一起打包的小型圖片。
- 特定情境下想減少額外請求。

不適合情境：

- 商品照片。
- 大型背景圖。
- 很多張圖片。
- 需要被瀏覽器獨立快取的圖片資源。

## 實務選擇準則

選圖片格式時，可以先照這個順序判斷：

1. 是照片或顏色豐富的圖片，且不需要透明背景：優先考慮 JPG 或 WebP。
2. 需要透明背景或高品質圖示：考慮 PNG 或 WebP。
3. 需要簡單動畫：考慮 GIF 或 WebP。
4. 是網頁正式資源：通常避免直接使用 BMP。
5. 是非常小、需要內嵌的圖片資料：可以考慮 Base64。
6. 需要兼容舊瀏覽器：使用 WebP 時準備 JPG 或 PNG fallback。

## 常見錯誤

### 把 Base64 當成圖片格式

```text
base64 格式
```

這種說法不精確。Base64 是編碼方式，不是像 JPG、PNG 那樣的圖片格式。正確理解是：可以把 JPG、PNG、GIF 等圖片資料轉成 Base64 文字。

### 所有圖片都用 PNG

PNG 支援透明且品質好，但不代表所有圖片都適合 PNG。大量照片如果都用 PNG，檔案可能過大，影響載入速度。

照片類圖片通常應比較 JPG、WebP 等格式。

### 所有圖片都轉成 Base64

Base64 可以減少額外圖片請求，但會讓 HTML 或 CSS 變大，也不利於大型圖片管理。大型圖片通常仍應作為獨立圖片檔案引用。

### 使用 WebP 卻沒有考慮兼容需求

如果專案必須支援舊瀏覽器，直接只提供 WebP 可能會讓部分使用者看不到圖片。這時可以使用 `<picture>` 提供備用格式。

## 重點整理

- JPG / JPEG 適合照片和顏色豐富的圖片，但不支援透明背景。
- PNG 支援透明背景，適合 Logo、圖示和重要配圖。
- BMP 檔案通常很大，網頁中極少使用。
- GIF 支援簡單動畫，但顏色數有限。
- WebP 是現代 Web 圖片格式，支援有損、無損、透明與動畫。
- Base64 是編碼方式，不是圖片格式。
- 大型圖片通常不要轉成 Base64，應使用獨立圖片檔案。
