# 列表嵌套

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解列表嵌套代表「某個列表項目裡還有一組子列表」。
- 正確把內層列表放在外層的 `<li>` 裡。
- 看懂巢狀列表中的父子層級關係。
- 避免只靠縮排寫出看似正確、結構錯誤的列表。

## 使用情境

有些列表項目本身還能繼續拆分成子項目，例如：

- 城市列表中，某個城市底下有多個景點。
- 課程章節中，某一章底下有多個小節。
- 導覽選單中，某個選單項目底下還有子選單。
- 分類列表中，某個分類底下還有更細的分類。

這時候就會使用列表嵌套，也就是在某個 `<li>` 裡再放一組 `<ul>` 或 `<ol>`。

## 一句話理解

列表嵌套就是「列表項目裡面還有列表」；內層列表必須屬於某一個外層 `<li>`。

## 先看整體結構

列表嵌套的關鍵不是縮排，而是父子關係。

```html
<ul>
  <li>外層項目 A</li>
  <li>
    外層項目 B
    <ul>
      <li>內層項目 B-1</li>
      <li>內層項目 B-2</li>
    </ul>
  </li>
  <li>外層項目 C</li>
</ul>
```

這裡的內層 `<ul>` 放在「外層項目 B」的 `<li>` 裡，表示 B 底下還有 B-1、B-2 兩個子項目。

## 完整範例

```html
<h2>我想去幾個城市</h2>
<ul>
  <li>成都</li>
  <li>
    <span>上海</span>
    <ul>
      <li>外灘</li>
      <li>杜莎夫人蠟像館</li>
      <li>
        <a href="">東方明珠</a>
      </li>
      <li>迪士尼樂園</li>
    </ul>
  </li>
  <li>西安</li>
  <li>武漢</li>
</ul>
```

這段程式碼表示：

- 外層列表列出想去的城市：成都、上海、西安、武漢。
- 「上海」這個 `<li>` 裡還有一組內層 `<ul>`。
- 內層 `<ul>` 列出上海底下的景點：外灘、杜莎夫人蠟像館、東方明珠、迪士尼樂園。
- 「東方明珠」這個列表項目裡放了 `<a href="">`，代表列表項目內也可以包含連結。

## 父子關係拆解

巢狀列表可以用這樣的層級理解：

```text
我想去幾個城市
- 成都
- 上海
  - 外灘
  - 杜莎夫人蠟像館
  - 東方明珠
  - 迪士尼樂園
- 西安
- 武漢
```

HTML 中真正建立這個層級的是標籤位置：

- 外層 `<ul>` 負責城市清單。
- 每個城市都應該是一個外層 `<li>`。
- 上海底下的景點清單放在上海那個 `<li>` 裡。
- 景點清單中的每個景點又是內層 `<ul>` 的 `<li>`。

所以，內層列表不是外層列表的另一個同級項目，而是某個外層項目的子內容。

## 常見錯誤

### 錯誤一：把內層 ul 直接放在外層 ul 裡

```html
<ul>
  <li>上海</li>
  <ul>
    <li>外灘</li>
    <li>迪士尼樂園</li>
  </ul>
</ul>
```

這段看起來像是把景點縮排到上海底下，但實際上內層 `<ul>` 沒有放在上海的 `<li>` 裡。外層 `<ul>` 的直接子元素應該是 `<li>`，所以這不是良好的列表結構。

應改成：

```html
<ul>
  <li>
    上海
    <ul>
      <li>外灘</li>
      <li>迪士尼樂園</li>
    </ul>
  </li>
</ul>
```

### 錯誤二：只相信縮排，不檢查標籤閉合位置

```html
<ul>
  <li>上海</li>
    <ul>
      <li>外灘</li>
    </ul>
</ul>
```

這段縮排看起來像內層列表屬於上海，但 `</li>` 已經在內層 `<ul>` 之前結束了。因此內層列表並不在上海的 `<li>` 裡。

寫巢狀列表時，要特別注意 `</li>` 的位置。父層 `<li>` 應該包住它的文字內容與子列表。

### 錯誤三：層級過深

列表可以嵌套，但不代表應該無限制嵌套。層級太深會讓讀者難以理解，也會讓 CSS、鍵盤操作與螢幕報讀器的閱讀體驗變差。

如果列表超過兩到三層，通常應該重新思考資訊架構，例如拆成多個區塊、改用標題分組，或使用更清楚的頁面結構。

## 實務寫法

撰寫列表嵌套時，可以照這個順序檢查：

1. 先確認外層列表的每個項目都是 `<li>`。
2. 如果某個項目有子列表，把子列表放在該項目的 `<li>` 裡。
3. 確認父層 `<li>` 的結束標籤在子列表之後。
4. 使用縮排讓層級容易閱讀，但不要只靠縮排判斷結構。
5. 如果層級太深，考慮拆成更清楚的章節或區塊。

## 重點整理

- 列表嵌套表示某個列表項目裡還有子列表。
- 內層 `<ul>` 或 `<ol>` 應該放在外層某個 `<li>` 裡。
- 外層 `<ul>`、`<ol>` 的直接子元素仍然應該是 `<li>`。
- 縮排只是幫助閱讀，真正的結構由標籤位置決定。
- 巢狀層級不要過深，否則會降低可讀性與可維護性。

## 自我檢查

1. 內層 `<ul>` 應該直接放在外層 `<ul>` 裡，還是放在某個外層 `<li>` 裡？
2. 為什麼只看縮排不能保證列表嵌套是正確的？
3. 如果「上海」底下有多個景點，景點列表應該放在哪個標籤裡？
4. 當列表層級超過三層時，你可以考慮哪些替代做法？
