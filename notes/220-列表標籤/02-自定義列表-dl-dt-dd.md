# 自定義列表 dl、dt、dd

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 `<dl>`、`<dt>`、`<dd>` 各自的語意。
- 判斷什麼內容適合用描述列表，而不是 `<ul>` 或 `<ol>`。
- 寫出一組名稱與描述對應清楚的描述列表。
- 避免把 `<li>` 或普通段落誤放進 `<dl>` 結構。

## 使用情境

有些資料不是單純的一串項目，而是「名稱」與「描述」的關係，例如：

- 術語與定義。
- 商品規格名稱與規格內容。
- 聯絡方式名稱與對應資訊。
- 常見問題的問題與答案。
- 頁尾中的分類名稱與多個連結。

這類內容如果只用 `<ul>`，可以列出來，但「哪個是名稱、哪個是說明」的語意不夠清楚。描述列表就是用來表達這種對應關係。

## 一句話理解

`<dl>` 是描述列表；`<dt>` 放被描述的名稱或項目，`<dd>` 放對應的描述內容。

## 基本語法

```html
<dl>
  <dt>HTML</dt>
  <dd>用來描述網頁內容結構的標記語言。</dd>

  <dt>CSS</dt>
  <dd>用來控制網頁外觀與版面呈現的樣式語言。</dd>
</dl>
```

這段程式碼中：

- `<dl>` 包住整組描述列表。
- `<dt>` 表示要被說明的項目名稱。
- `<dd>` 表示對該項目的描述。

它不像 `<ul>` 預設有項目符號，也不像 `<ol>` 預設有數字，因為它的重點不是「列出項目」，而是呈現名稱與描述的關係。

## 範例拆解

```html
<dl>
  <dt>关注我们</dt>
  <dd>新浪微博</dd>
  <dd>官方微信</dd>
  <dd>联系我们</dd>

  <dt>关注我们</dt>
  <dd>新浪微博</dd>
  <dd>官方微信</dd>
  <dd>联系我们</dd>
</dl>
```

這個範例可以拆成三層來看：

- `<dl>`：建立整個描述列表。
- `<dt>关注我们</dt>`：建立一個項目名稱。
- 多個 `<dd>`：描述或列出這個名稱底下的相關內容。

一個 `<dt>` 可以對應一個 `<dd>`，也可以對應多個 `<dd>`。因此，描述列表不只能寫「名詞：解釋」，也能表達「一個分類名稱底下有多個描述項」。

## dl、dt、dd 的關係

| 標籤 | 語意 | 常見內容 |
| --- | --- | --- |
| `<dl>` | description list，描述列表 | 包住整組 `dt` 與 `dd` |
| `<dt>` | description term，被描述的名稱或項目 | 名詞、分類名稱、問題、規格名稱 |
| `<dd>` | description details，描述內容 | 定義、說明、答案、規格值、連結 |

`<dl>` 裡主要放 `<dt>` 與 `<dd>`。`<dt>` 與 `<dd>` 裡可以放文字與多數常見元素，但仍要符合 HTML 內容模型。

## 什麼時候不用 dl

描述列表不是「比較少見的列表樣式」，也不是 `<ul>` 的替代外觀。判斷時要看資料關係。

適合用 `<dl>`：

```html
<dl>
  <dt>作者</dt>
  <dd>王小明</dd>
  <dt>出版日期</dt>
  <dd>2026-06-17</dd>
</dl>
```

這裡每個項目都是「名稱 + 值」的描述關係。

較適合用 `<ul>`：

```html
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
```

這裡只是三個並列項目，沒有明確的名稱與描述關係。

較適合用 `<ol>`：

```html
<ol>
  <li>打開編輯器</li>
  <li>建立 HTML 檔案</li>
  <li>在瀏覽器中開啟</li>
</ol>
```

這裡是有先後順序的操作步驟。

## 常見錯誤

### 錯誤一：在 dl 裡放 li

```html
<dl>
  <li>HTML</li>
  <li>CSS</li>
</dl>
```

`<li>` 是 `<ul>` 與 `<ol>` 的列表項目，不是描述列表的項目。若內容只是並列項目，應使用 `<ul>`；若內容是名稱與描述，應改用 `<dt>` 與 `<dd>`。

### 錯誤二：只用 p 模擬名稱與描述

```html
<p>HTML</p>
<p>用來描述網頁內容結構的標記語言。</p>
```

畫面可以顯示，但 HTML 沒有表達出這兩段文字之間的對應關係。若內容本身就是「名稱 + 描述」，用 `<dl>` 更清楚。

### 錯誤三：把 dl 當成沒有符號的 ul

如果只是因為不想要項目符號，就把 `<ul>` 改成 `<dl>`，語意會變得不正確。是否使用 `<dl>`，應該取決於資料是否有描述關係，不是取決於視覺樣式。

## 實務判斷準則

可以用下面幾個問題判斷是否適合使用 `<dl>`：

1. 這組內容是否可以看成「名稱 + 描述」？
2. 某個名稱是否可能對應多個描述？
3. 使用 `<ul>` 會不會讓名稱與說明的關係變模糊？
4. 這組資料是否像術語解釋、規格表、問題答案或分類描述？

如果答案多半是肯定的，描述列表通常是合適選擇。

## 重點整理

- `<dl>` 用來建立描述列表。
- `<dt>` 表示被描述的名稱、術語或項目。
- `<dd>` 表示對應的描述內容。
- 一個 `<dt>` 可以對應多個 `<dd>`。
- 描述列表適合名稱與說明的資料，不適合單純並列項目。
- 不要在 `<dl>` 裡放 `<li>`。

## 自我檢查

1. 「HTML：超文本標記語言」這種內容適合用 `<dl>` 嗎？為什麼？
2. 如果只是列出三個喜歡的食物，應該用 `<dl>` 還是 `<ul>`？
3. `<dt>` 與 `<dd>` 各自代表什麼？
4. 為什麼不應該只為了去掉項目符號而改用 `<dl>`？
