# 列表標籤入門與 ul、ol

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明列表在 HTML 中適合表達什麼內容。
- 分辨無序列表 `<ul>` 與有序列表 `<ol>` 的使用時機。
- 正確使用 `<li>` 表示列表中的每一個項目。
- 避免把文字或其他元素直接放在 `<ul>`、`<ol>` 裡。

## 使用情境

網頁上常會出現一組彼此相關的項目，例如新聞標題、商品特色、排行榜、待辦事項、帳單明細、步驟說明。這些內容不是單獨存在的段落，而是一組「同類型的項目」。

列表標籤就是用來表達這種成組內容的 HTML 結構。

![無序、有序與自定義列表範例](../../origin/220-列表標籤/assets/images/list-elements-img-001-43864e.jpg)

## 一句話理解

列表用來表示一組相關項目；如果項目沒有順序關係，用 `<ul>`，如果項目有排名、步驟或先後順序，用 `<ol>`。

列表的外觀，例如項目符號、數字、縮排和間距，是瀏覽器預設樣式與 CSS 控制的結果；HTML 標籤主要負責表達內容結構。

## 三種常見列表

HTML 常見列表可分成三類：

![列表標籤定義與說明表](../../origin/220-列表標籤/assets/images/list-elements-img-002-a5d743.png)

| 類型 | 主要標籤 | 適合內容 |
| --- | --- | --- |
| 無序列表 | `<ul>`、`<li>` | 並列項目，例如功能特色、食物清單、新聞列表 |
| 有序列表 | `<ol>`、`<li>` | 有順序的項目，例如排行榜、操作步驟、流程 |
| 描述列表 | `<dl>`、`<dt>`、`<dd>` | 名詞與說明，例如術語解釋、規格描述 |

這篇先處理最常用的 `<ul>` 與 `<ol>`。描述列表會在下一篇單獨說明。

## 無序列表 ul

`<ul>` 表示 unordered list，也就是「無序列表」。它適合用來呈現沒有排名、步驟或先後差異的並列項目。

```html
<h4>您喜歡的食物?</h4>
<ul>
  <li>榴槤</li>
  <li>臭豆腐</li>
  <li>鲱鱼罐头</li>
  <li>
    <p>123</p>
  </li>
</ul>
```

這段程式碼中：

- `<ul>` 建立一組無序列表。
- 每一個 `<li>` 代表一個列表項目。
- 「榴槤」「臭豆腐」「鲱鱼罐头」彼此是並列關係，沒有第一名、第二名或步驟順序。
- 最後一個 `<li>` 裡放了 `<p>`，代表 `<li>` 可以作為項目容器，包住文字與多數常見元素。

瀏覽器通常會替 `<ul>` 加上項目符號、縮排與上下間距。實務上，這些外觀會交給 CSS 調整。

## 有序列表 ol

`<ol>` 表示 ordered list，也就是「有序列表」。它適合用來呈現順序有意義的項目，例如排行、步驟、流程。

```html
<h4>粉丝排行榜</h4>
<ol>
  <li>刘德华 10000</li>
  <li>刘若英 1000</li>
  <li>pink老师 1</li>
</ol>
```

這段程式碼中：

- `<ol>` 建立一組有序列表。
- 每個 `<li>` 仍然代表一個列表項目。
- 項目的先後位置有意義，第一個項目會被理解為排序中的第一項。
- 瀏覽器預設通常會用數字標記項目，也可以透過屬性或 CSS 調整標記樣式。

如果把排行榜寫成 `<ul>`，畫面仍然可能看起來像清單，但語意上就少了「順序很重要」這層資訊。

## ul 與 ol 的選擇

選擇 `<ul>` 或 `<ol>` 時，不要只看你想要圓點還是數字，而要看內容本身是否有順序意義。

| 情境 | 建議標籤 | 原因 |
| --- | --- | --- |
| 商品特色清單 | `<ul>` | 每個特色並列，沒有固定順序 |
| 新聞標題列表 | `<ul>` | 通常只是同類項目集合 |
| 排行榜 | `<ol>` | 名次順序是內容的一部分 |
| 安裝步驟 | `<ol>` | 步驟先後不能任意交換 |
| 帳單明細 | 視情況使用 `<ul>` 或表格 | 若只是項目集合可用列表；若有多欄資料，表格更適合 |

## li 是列表項目的容器

`<ul>` 與 `<ol>` 的直接子元素應該是 `<li>`。也就是說，列表中的每個項目都要放進 `<li>` 裡。

正確寫法：

```html
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
```

不建議寫法：

```html
<ul>
  HTML
  <p>CSS</p>
  <li>JavaScript</li>
</ul>
```

問題在於 `HTML` 文字與 `<p>` 直接出現在 `<ul>` 裡，沒有被 `<li>` 包住。這會讓列表結構不清楚，也不符合列表的內容模型。

如果某個列表項目裡需要放段落、連結、圖片或其他內容，應該把它們放進 `<li>` 內：

```html
<ul>
  <li>
    <p>HTML 是網頁內容的骨架。</p>
    <a href="intro.html">查看介紹</a>
  </li>
</ul>
```

## 常見錯誤

### 錯誤一：用外觀決定標籤

有些人會因為想要數字就用 `<ol>`，想要圓點就用 `<ul>`。但標籤選擇應先看語意。

例如操作步驟即使最後用 CSS 拿掉數字，仍然適合用 `<ol>`，因為步驟順序是內容的一部分。

### 錯誤二：把內容直接放在 ul 或 ol 裡

```html
<ol>
  第一步
  <li>第二步</li>
</ol>
```

`第一步` 沒有放進 `<li>`，這不是良好的列表結構。應改成：

```html
<ol>
  <li>第一步</li>
  <li>第二步</li>
</ol>
```

### 錯誤三：把列表當成排版工具

列表適合表達一組相關項目，不是用來單純製作縮排或符號效果。若只是想讓文字縮排或排列，應該用 CSS 控制；若內容不是一組項目，就不應硬套列表。

## 實務判斷準則

- 內容是一組相關項目時，優先考慮列表。
- 項目順序不重要，用 `<ul>`。
- 項目順序、排名或步驟重要，用 `<ol>`。
- `<ul>` 與 `<ol>` 的直接子元素應該是 `<li>`。
- `<li>` 可以包含文字與多數常見元素，但仍需符合 HTML 內容模型。
- 外觀交給 CSS，語意交給 HTML 標籤。

## 重點整理

- 列表用來表示一組相關項目。
- `<ul>` 表示無序列表，適合並列項目。
- `<ol>` 表示有序列表，適合順序有意義的項目。
- `<li>` 表示列表中的單一項目。
- 不要把文字或其他元素直接放在 `<ul>`、`<ol>` 裡，應放進 `<li>`。
