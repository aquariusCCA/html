# Web 與 JavaWeb 基本概念

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 Web 是什麼。
- 理解瀏覽器、網路協定、HTML 與 Web 資源之間的關係。
- 分辨 Web 與 JavaWeb 的範圍差異。

## 問題情境

學 HTML 之前，先要知道自己正在學的是哪一種開發環境。

你平常打開瀏覽器，輸入網址，看到新聞、影片、購物頁、後台管理系統，這些都屬於 Web 世界的一部分。HTML 則是 Web 裡最基礎的文件結構語言，用來描述頁面上有哪些內容，以及這些內容彼此的關係。

## 一句話理解

Web 是一個透過網際網路連接大量資源的資訊系統；HTML 是瀏覽器理解網頁內容的重要語言之一。

## Web 是什麼

Web 是 World Wide Web 的簡稱，常譯為全球資訊網。它可以理解成一個由許多資源彼此連結而成的巨大資訊系統。

這些資源可能是：

- 一份 HTML 網頁。
- 一張圖片。
- 一段影片或音訊。
- 一個可下載檔案。
- 一個由後端程式產生的資料結果。

使用者通常透過瀏覽器訪問這些資源。瀏覽器會依照網址向遠端服務請求資料，再把拿到的內容解析成可以閱讀、操作的頁面。

## Web 如何把資源連起來

Web 能運作，通常需要幾個角色配合：

| 角色 | 作用 |
| --- | --- |
| URL | 描述資源的位置，也就是你在瀏覽器輸入的網址。 |
| HTTP / HTTPS | 瀏覽器與服務器之間交換資料時常用的通訊規則。 |
| HTML | 描述網頁內容與結構。 |
| 瀏覽器 | 發送請求、接收資源、解析 HTML，最後呈現頁面。 |
| 服務器 | 保存或產生資源，並回應瀏覽器的請求。 |

入門階段可以先把流程理解成：

```text
使用者輸入網址
  -> 瀏覽器向服務器請求資源
  -> 服務器回傳 HTML、圖片、CSS、JavaScript 等資料
  -> 瀏覽器解析資料並顯示頁面
```

## HTML 在 Web 中的位置

HTML 不是程式邏輯的全部，也不是一個網站的全部。

HTML 的核心任務是描述文件結構，例如：

```html
<h1>網站標題</h1>
<p>這是一段介紹文字。</p>
<a href="https://example.com">前往範例網站</a>
```

這段 HTML 告訴瀏覽器：

- `h1` 是頁面的主要標題。
- `p` 是一段文字。
- `a` 是一個可以前往其他資源的連結。

瀏覽器之所以能把這些內容顯示成網頁，是因為 Web 有一套共同規則，讓 HTML、CSS、JavaScript、圖片與其他資源可以一起被解析與呈現。

## JavaWeb 是什麼

JavaWeb 通常指使用 Java 技術開發、部署在 Web 環境中，並可以透過瀏覽器訪問的 Web 應用。

它的重點不是「Web 只能使用 Java」，而是：

- Web 是更大的網頁與網路應用平台。
- JavaWeb 是使用 Java 技術實作 Web 應用的一種方向。
- 使用者最後仍然常透過瀏覽器看到頁面或操作系統。

例如一個 JavaWeb 系統可能會：

- 使用 Java 程式處理登入、查詢、訂單等後端邏輯。
- 從資料庫取得資料。
- 把資料整理成 HTML 或 API 回傳給瀏覽器。
- 讓使用者在瀏覽器中操作頁面。

## 常見混淆

### Web 不等於網際網路

網際網路是更底層的網路基礎設施；Web 是建立在網際網路之上的一種資訊服務。

可以簡化理解為：

```text
網際網路提供連線能力。
Web 提供透過網址、瀏覽器和超連結訪問資源的方式。
```

### Web 不等於 HTML

HTML 是 Web 的重要組成，但一個完整網站還可能包含 CSS、JavaScript、圖片、服務器、資料庫、部署環境與安全設定。

學 HTML 時，先把重點放在「頁面內容與結構」會比較穩。

### JavaWeb 不等於所有 Web 開發

JavaWeb 是 Web 開發的一種技術路線。Web 後端也可以用其他語言或平台完成，例如 JavaScript、Python、PHP、Go、C# 等。

## 重點整理

- Web 是透過網際網路連接各種資源的資訊系統。
- 瀏覽器是使用者訪問 Web 的主要入口。
- HTML 負責描述網頁內容與結構，是前端學習的基礎。
- JavaWeb 是使用 Java 技術開發 Web 應用的一種方向，不代表 Web 只能使用 Java。
