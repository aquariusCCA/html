# C/S 與 B/S 架構比較

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 C/S 架構與 B/S 架構的基本差異。
- 理解 B/S 架構為什麼和 Web 前端學習密切相關。
- 看懂表現層、邏輯層、數據層的基本分工。
- 從安裝、更新、互動能力與維護成本角度比較兩種架構。

## 問題情境

有些軟件需要先下載安裝，例如桌面版 Office 或某些企業內部系統；有些系統只要打開瀏覽器就能使用，例如大多數網站後台、線上文件或購物網站。

這兩種使用方式背後，常可以對應到 C/S 與 B/S 兩種架構思路。

## 一句話理解

C/S 偏向「專用客戶端 + 服務器」；B/S 偏向「瀏覽器 + 服務器」，使用者通常不需要額外安裝專用客戶端。

## C/S 架構

C/S 是 Client/Server，也就是客戶端 / 服務器架構。

在 C/S 架構中，使用者通常需要先安裝一個專用客戶端。這個客戶端不只顯示畫面，也可能承擔部分運算、業務邏輯、檔案處理或本機設備整合。

![C/S 架構客戶機與服務器雙向請求示意圖](../../origin/第01章_寫在前面/assets/images/cs-bs-architecture-img-001-0f94cf.png)

常見例子包含：

- 桌面聊天軟件。
- 辦公軟件。
- 部分企業內部管理系統。
- 需要離線能力、硬體整合或特殊權限控制的專用軟件。

## C/S 架構的優點

- 客戶端操作界面可以做得很豐富，互動能力通常較強。
- 專用客戶端可以更直接整合本機能力或特定硬體。
- 在受控環境中，設備與使用者管控可以做得較細。
- 某些場景下可以把部分運算放到客戶端，降低服務端壓力。

安全性不是 C/S 架構天然保證的結果。實際安全仍取決於認證、授權、加密、更新機制與整體系統設計。

## C/S 架構的缺點

- 使用前通常需要下載與安裝。
- 升級時可能需要更新每一個客戶端。
- 不適合面向大量未知使用者的快速訪問場景。
- 多平台支援成本可能較高，例如 Windows、macOS、Linux、行動裝置都要各自適配。

## B/S 架構

B/S 是 Browser/Server，也就是瀏覽器 / 服務器架構。

在 B/S 架構中，使用者通常只需要瀏覽器即可使用系統。前端頁面在瀏覽器中顯示，主要業務邏輯和資料處理多由服務器端完成。

這也是 Web 前端開發最常面對的架構。

## B/S 架構的三層結構

B/S 架構常被拆成三層：

![B/S 架構界面、邏輯與數據層示意圖](../../origin/第01章_寫在前面/assets/images/cs-bs-architecture-img-002-fc4d53.png)

| 層級 | 主要角色 | 說明 |
| --- | --- | --- |
| 表現層 | 瀏覽器中的前端頁面 | 負責使用者介面、資料呈現與基本互動。 |
| 邏輯層 | 後端服務 | 負責業務規則、請求處理、權限驗證等。 |
| 數據層 | 資料庫或資料服務 | 負責資料儲存、查詢與管理。 |

這裡的「表現層」是系統架構中的前端層，代表瀏覽器端的呈現與互動；它不等於 Web 標準三層中的「表現」，後者通常特指 CSS 負責的視覺樣式。

在理想情況下，瀏覽器不會直接操作資料庫，而是透過後端邏輯層間接存取資料。

## B/S 架構的優點

- 使用者通常不用安裝專用客戶端，有瀏覽器即可使用。
- 更新主要集中在服務器端，使用者重新打開或重新整理頁面就能取得新版本。
- 適合放在廣域網上，讓不同地點的使用者訪問。
- 對大量使用者發布與維護的成本通常較低。

## B/S 架構的缺點

- 需要考慮不同瀏覽器與不同版本的呈現差異。
- 若要達到接近專用客戶端的複雜互動，需要更多前端設計與工程投入。
- 效能與安全性需要完整系統設計，例如快取、權限、加密、輸入驗證、後端保護等。

## C/S 與 B/S 對照

| 比較項目 | C/S 架構 | B/S 架構 |
| --- | --- | --- |
| 使用入口 | 專用客戶端 | Web 瀏覽器 |
| 是否安裝 | 通常需要安裝 | 通常不需要額外安裝 |
| 更新方式 | 可能要更新每個客戶端 | 多數更新集中在服務器 |
| 前端載體 | 桌面或行動客戶端 | 瀏覽器頁面 |
| 適合場景 | 專用系統、離線能力、本機能力整合 | 網站、後台系統、跨裝置訪問 |
| 維護成本 | 客戶端版本管理成本較高 | 客戶端版本管理成本較低；主要成本在瀏覽器兼容、服務端與安全設計 |

## 對 HTML 學習的意義

HTML 主要出現在 B/S 架構中的表現層，也就是瀏覽器端負責呈現與互動的前端層。

當你寫 HTML 時，實際上是在定義瀏覽器要呈現的文件結構。這份結構會與 CSS、JavaScript 一起構成使用者看得到、操作得到的前端頁面。

所以學 HTML 不只是學幾個標籤，也是在理解 B/S 架構中「表現層」的基礎。

## 常見錯誤

### 以為 B/S 一定比 C/S 好

兩者是不同架構選擇，沒有絕對優劣。要看使用場景、更新方式、使用者範圍、本機能力需求與維護成本。

### 以為瀏覽器不能做複雜事情

現代瀏覽器可以承擔大量互動與運算，但 B/S 架構仍然需要清楚劃分前端、後端與資料層責任。

### 讓瀏覽器直接接觸資料庫

在 Web 系統中，前端通常不應直接操作資料庫。資料存取應經過後端服務，才能處理權限、安全與業務規則。

## 重點整理

- C/S 是客戶端 / 服務器架構，通常需要安裝專用客戶端。
- B/S 是瀏覽器 / 服務器架構，通常透過瀏覽器訪問。
- B/S 常分成表現層、邏輯層、數據層。
- HTML 屬於 B/S 架構中表現層的核心基礎。
- 架構選擇要看場景，不是簡單判斷誰一定更好。
