# 瀏覽器與渲染引擎

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明瀏覽器在 Web 中扮演的角色。
- 理解瀏覽器為什麼能把 HTML 顯示成網頁。
- 說明渲染引擎的基本作用。
- 知道不同瀏覽器可能造成呈現差異。

## 問題情境

同一份 HTML 檔案，為什麼放進瀏覽器就能變成有標題、段落、圖片與連結的頁面？

原因是瀏覽器不只是「開網頁的工具」。它會向服務器請求資源、解析 HTML、套用 CSS、執行 JavaScript，最後把結果繪製到畫面上。

## 一句話理解

瀏覽器是 Web 頁面的運行平台；渲染引擎是瀏覽器中負責解析與呈現網頁內容的核心部分。

## 瀏覽器是什麼

瀏覽器是使用者訪問 Web 的主要工具，也是網頁顯示與運行的平台。

常見瀏覽器包含 Chrome、Edge、Firefox、Safari、Opera 等。早期教材常把 IE、Firefox、Chrome、Safari、Opera 稱為五大主流瀏覽器；現在閱讀這類資料時，要把 IE 視為歷史對照，而不是現代 Web 開發的主要目標。

![五大主流瀏覽器與問世時間](../../origin/第01章_寫在前面/assets/images/browser-img-001-111cfe.png)

在 Web 開發中，瀏覽器通常會負責：

- 發送請求取得 HTML、CSS、JavaScript、圖片等資源。
- 解析 HTML，建立頁面結構。
- 解析 CSS，計算元素樣式。
- 執行 JavaScript，處理互動邏輯。
- 把結果繪製成使用者看到的畫面。

## 渲染引擎是什麼

渲染引擎也常被稱為瀏覽器內核。它是瀏覽器中負責解析與呈現網頁的核心部分。

它會讀取網頁資源，整理資訊，計算頁面應該如何顯示，最後把內容呈現在畫面上。

![五大瀏覽器與內核對照圖](../../origin/第01章_寫在前面/assets/images/browser-img-002-b448ff.png)

這張圖保留早期教材脈絡，IE 主要作為歷史對照；現代 Edge、Chrome、Opera 多基於 Chromium / Blink，實際開發仍應依目標瀏覽器與版本確認支援情況。

不同瀏覽器可能使用不同渲染引擎；也有多個瀏覽器使用相同或相近的引擎。因此，不能只用瀏覽器名稱判斷所有技術細節，還要理解背後的解析與渲染實作可能不同。

## 從請求到畫面的簡化流程

當你打開一個網頁時，可以先理解成以下流程：

```text
瀏覽器向服務器請求資源
  -> 取得 HTML、CSS、JavaScript、圖片等檔案
  -> 渲染引擎解析 HTML 與 CSS
  -> 瀏覽器建立頁面結構與樣式計算結果
  -> JavaScript 可能改變內容或互動狀態
  -> 瀏覽器把頁面繪製到畫面
```

![瀏覽器內核處理網頁資源示意圖](../../origin/第01章_寫在前面/assets/images/browser-img-003-ccff67.png)

這也是為什麼 HTML 寫法會影響頁面呈現。瀏覽器不是隨意顯示文字，而是依照規則解析標籤、屬性、巢狀關係與文件結構。

## 為什麼不同瀏覽器會有差異

不同瀏覽器在版本、渲染引擎、預設樣式、功能支援與實作細節上可能存在差異。

差異可能表現在：

- 同一段 CSS 的呈現略有不同。
- 某些新的 HTML、CSS 或 JavaScript 功能支援程度不同。
- 預設表單樣式不完全一致。
- 特定效能表現不同。

這也是 Web 標準重要的原因。標準越清楚，瀏覽器越能依照共同規則實作，開發者越能寫出穩定的頁面。

## 對 HTML 學習的意義

學 HTML 時，要把瀏覽器視為你的第一個讀者。

你寫下：

```html
<h1>文章標題</h1>
<p>這是一段文章內容。</p>
```

瀏覽器會根據 HTML 語意理解：

- `h1` 是主要標題。
- `p` 是段落。
- 這兩個元素在文件中有先後順序。

如果 HTML 結構混亂，瀏覽器仍可能盡量顯示畫面，但可維護性、可讀性、搜尋引擎理解與無障礙體驗都會受影響。

## 常見錯誤

### 只在一個瀏覽器中測試

初學時可以先用一個瀏覽器練習，但正式頁面應該至少考慮主要目標瀏覽器。特別是樣式、表單、互動功能，需要確認實際呈現。

### 把瀏覽器顯示正常當成 HTML 結構正確

瀏覽器有容錯能力，會嘗試修正錯誤標記。但「看起來有顯示」不代表 HTML 語意與結構都是合理的。

### 忽略瀏覽器預設樣式

標題、段落、列表、表單元素都有瀏覽器預設樣式。之後學 CSS 時，會進一步控制這些樣式。

## 重點整理

- 瀏覽器是 Web 頁面的顯示與運行平台。
- 渲染引擎負責解析 HTML、CSS 等資源並呈現頁面。
- 不同瀏覽器與版本可能造成呈現或功能支援差異。
- HTML 結構會影響瀏覽器、搜尋引擎與輔助技術如何理解頁面。
