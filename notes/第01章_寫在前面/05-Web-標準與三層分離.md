# Web 標準與三層分離

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明為什麼需要 Web 標準。
- 理解 Web 標準中的結構、表現與行為。
- 知道為什麼實務上會建議分離 HTML、CSS 與 JavaScript。

## 問題情境

如果同一份網頁程式碼在不同瀏覽器中顯示出完全不同的畫面，使用者體驗會很差，開發者也很難維護。

Web 標準就是為了讓不同瀏覽器盡量依照共同規範解析與呈現內容，降低畫面與行為差異。

![不同瀏覽器呈現不同網頁效果示意圖](../../origin/第01章_寫在前面/assets/images/web-standard-img-001-e119b2.png)

## 為什麼需要 Web 標準

不同瀏覽器的渲染引擎、版本與實作細節可能不同。如果每個瀏覽器都用自己的方式理解 HTML、CSS 和 JavaScript，同一份網頁就可能在不同環境中出現不同結果。

Web 標準提供共同規範，讓開發者能用一致的方式撰寫網頁，也讓瀏覽器有共同依據去解析與呈現。

![Web 標準統一不同瀏覽器顯示效果示意圖](../../origin/第01章_寫在前面/assets/images/web-standard-img-002-783e20.png)

## Web 標準的三層構成

Web 標準通常可以用三層來理解：

![Web 標準結構、表現與行為對照表](../../origin/第01章_寫在前面/assets/images/web-standard-img-003-2d03d3.png)

| 層級 | 對應技術 | 負責內容 |
| --- | --- | --- |
| 結構 | HTML | 描述頁面有哪些內容與元素。 |
| 表現 | CSS | 描述頁面外觀、排版、顏色與視覺樣式。 |
| 行為 | JavaScript | 描述互動、動態效果與使用者操作後的反應。 |

簡單理解：

```text
HTML 管結構，CSS 管表現，JavaScript 管行為。
```

## 三層分離是什麼

三層分離是指把頁面結構、視覺樣式與互動行為分別放在適合的位置，而不是全部混在一起。

例如一個按鈕可以這樣分工：

```html
<button class="primary-button">送出</button>
```

```css
.primary-button {
  background-color: #0f766e;
  color: white;
}
```

```js
document.querySelector(".primary-button").addEventListener("click", function () {
  console.log("送出表單");
});
```

在這個例子中：

- HTML 負責說明這是一個按鈕，文字是「送出」。
- CSS 負責按鈕的顏色與外觀。
- JavaScript 負責點擊後要發生什麼事。

## 為什麼要分離

三層分離的目的不是為了形式好看，而是為了降低維護成本。

| 做法 | 影響 |
| --- | --- |
| 結構、表現、行為混在一起 | 修改畫面或互動時容易牽動很多地方。 |
| 結構、表現、行為分開管理 | 更容易閱讀、維護、重用與除錯。 |

當 HTML 專注描述內容結構時，瀏覽器、搜尋引擎、輔助工具與其他開發者都比較容易理解頁面。

## 實務注意

### HTML 不應只為了畫面效果而寫

HTML 的核心是結構與語意。不要只因為某個標籤有預設樣式就使用它，應該先判斷它是否符合內容語意。

### CSS 不應承擔內容結構

CSS 可以改變外觀，但不應用來彌補混亂的 HTML 結構。結構清楚，樣式才容易維護。

### JavaScript 不應取代基本 HTML 語意

如果 HTML 本身已有合適標籤，應優先使用正確標籤，再用 JavaScript 增加互動。這樣對可維護性與無障礙都比較友善。

## 常見誤解

### 誤解一：Web 標準只是瀏覽器廠商的事情

開發者也需要遵守標準。寫出語意清楚、結構合理的 HTML，是讓瀏覽器正確解析的基礎。

### 誤解二：只要畫面看起來對，就代表寫法正確

畫面正確不一定代表結構正確。錯誤或混亂的結構可能在日後維護、SEO、無障礙或跨瀏覽器顯示時造成問題。

## 重點整理

- Web 標準是為了降低不同瀏覽器解析與呈現的差異。
- Web 標準可用結構、表現、行為三層理解。
- HTML 負責結構，CSS 負責樣式，JavaScript 負責互動。
- 三層分離能提升可讀性、可維護性與實務開發品質。
