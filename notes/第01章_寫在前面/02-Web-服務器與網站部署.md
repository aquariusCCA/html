# Web 服務器與網站部署

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 Web 服務器在網站中的角色。
- 理解為什麼網站需要部署到服務器上。
- 分辨本地服務器與遠程服務器的差異。

## 問題情境

你在自己的電腦上寫好 HTML、CSS 和 JavaScript 後，通常只有自己能直接打開這些檔案。

如果希望其他人也能透過網址訪問網站，就需要把網站放到能對外提供服務的環境中。這個環境通常就是 Web 服務器。

## 服務器是什麼

服務器是提供計算服務的設備、程式或系統。它本質上也可以是一台計算機，只是在網路中負責對外提供某種服務。

根據提供的服務不同，服務器可以分為：

| 類型 | 主要用途 |
| --- | --- |
| 文件服務器 | 提供檔案儲存與下載。 |
| 資料庫服務器 | 提供資料儲存、查詢與管理。 |
| 應用程序服務器 | 執行應用程式與業務邏輯。 |
| Web 服務器 | 提供網頁文件、圖片、資料或其他 Web 資源。 |

服務器有時也會被稱為主機。

## Web 服務器是什麼

Web 服務器一般指網站服務器。它可以向瀏覽器等 Web 客戶端提供文件與資源，例如 HTML、CSS、JavaScript、圖片或可下載檔案。

![多位使用者連線至服務器示意圖](../../origin/第01章_寫在前面/assets/images/web-server-img-001-7d7f51.png)

當使用者在瀏覽器輸入網址時，大致會發生以下流程：

1. 瀏覽器根據網址向服務器發出請求。
2. Web 服務器接收請求。
3. 服務器找到對應的網站文件或資料。
4. 服務器把結果回傳給瀏覽器。
5. 瀏覽器解析並顯示頁面。

這也是為什麼只把網站放在自己電腦的資料夾中，通常不能讓全世界的人直接訪問。

## 網站部署是什麼

部署可以先理解成「把網站放到可以被訪問的服務器環境中」。

開發時，檔案可能只在你的電腦上。部署後，網站檔案會放到服務器上，其他使用者就可以透過網址訪問。

入門階段不需要立刻掌握完整部署流程，但要先建立這個觀念：

```text
本機開發不等於公開上線。
```

你在本機能打開，不代表其他人也能透過網路訪問。

## 本地服務器

本地服務器是指在自己的電腦上啟動的服務器環境。它通常用於開發與測試。

例如你在本機啟動一個開發服務器後，可能會看到類似這樣的網址：

```text
http://localhost:3000
```

這表示瀏覽器正在訪問你自己電腦上的服務。若網路、防火牆與服務綁定位址設定允許，同一個局域網中的其他裝置也可能訪問你的本地項目。

## 遠程服務器

遠程服務器通常是由公司、雲端平台或主機商提供的電腦或服務環境。

當你把網站項目上傳到遠程服務器後，其他人就可以透過域名或公開網址訪問你的網站。例如：

```text
www.baidu.com
```

遠程服務器的重點是它不在你的本機上，並且通常具備對外提供服務的網路環境。

## 本地服務器與遠程服務器比較

| 比較項目 | 本地服務器 | 遠程服務器 |
| --- | --- | --- |
| 位置 | 自己的電腦或開發機器 | 雲端主機、公司主機或第三方服務 |
| 主要用途 | 開發、測試、預覽 | 對外提供網站或服務 |
| 訪問對象 | 通常是自己，或局域網內裝置 | 網際網路上的使用者 |
| 維護重點 | 開發效率與測試方便 | 穩定性、安全性、網路與部署流程 |

## 常見誤解

### 誤解一：只要有 HTML 檔案，就等於網站已上線

HTML 檔案可以在本機打開，但如果沒有放到可被訪問的服務器環境中，其他人通常無法透過網址訪問它。

### 誤解二：服務器一定只能是遠端主機

本機也可以啟動服務器。差別在於它是否用於本機開發，還是對外公開提供服務。

## 重點整理

- 服務器是對外提供服務的設備、程式或系統。
- Web 服務器負責向瀏覽器提供網頁文件與 Web 資源。
- 部署是把網站放到可被訪問的服務器環境中。
- 本地服務器多用於開發測試，遠程服務器多用於公開訪問。
