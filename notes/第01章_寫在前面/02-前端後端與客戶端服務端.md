# 前端、後端、客戶端與服務端

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明前端與後端的基本分工。
- 理解客戶端與服務端是網路互動中的角色。
- 分辨前端 / 後端與客戶端 / 服務端不是完全相同的概念。
- 知道終端這個詞在入門語境中的常見意思。

## 問題情境

當你在瀏覽器打開一個網站，畫面上看到的按鈕、圖片、表單、文字，不是憑空出現的。

通常會有一段互動流程：

```text
使用者操作頁面
  -> 前端把請求送出去
  -> 後端處理資料或規則
  -> 後端回傳結果
  -> 前端把結果顯示給使用者
```

所以學 HTML 時，雖然你主要在學前端基礎，但也要先知道前端不是孤立存在的。

## 一句話理解

前端主要負責使用者看得到、操作得到的部分；後端主要負責資料、規則、儲存與服務回應。

## 前端負責什麼

前端面向使用者，是瀏覽器中呈現與操作的部分。

常見工作包含：

- 使用 HTML 建立內容結構。
- 使用 CSS 控制畫面樣式。
- 使用 JavaScript 處理互動行為。
- 向後端請求資料。
- 把資料整理成使用者能理解的畫面。

例如一個商品頁中，商品名稱、價格、圖片、加入購物車按鈕，都是前端需要呈現與組織的內容。

## 後端負責什麼

後端面向資料與規則，是使用者通常看不到、但系統必須依賴的部分。

常見工作包含：

- 接收前端請求。
- 驗證使用者身分。
- 處理業務規則。
- 讀寫資料庫。
- 回傳資料或頁面給前端。

例如使用者按下「登入」時，前端負責送出帳號密碼；後端負責檢查帳密是否正確，並決定是否允許登入。

## 資料庫與服務器

入門時也常聽到兩個相關詞：

| 名詞 | 白話理解 |
| --- | --- |
| 資料庫 | 儲存資料的地方，例如使用者資料、商品資料、訂單資料。 |
| 服務器 | 對外提供服務的機器、程式或系統，可以保存資訊、處理請求、傳回結果。 |

服務器不是只代表一台硬體很強的電腦。更重要的是它在某個情境中「提供服務」。

## 客戶端與服務端

客戶端和服務端描述的是網路互動中的角色。

| 角色 | 說明 |
| --- | --- |
| 客戶端 | 發出請求、使用服務的一方。 |
| 服務端 | 接收請求、提供服務的一方。 |

在網站情境中：

- 使用者的瀏覽器通常是客戶端。
- 存放網站、處理請求的系統通常是服務端。

在 App 情境中：

- 安裝在手機上的 App 通常是客戶端。
- 提供登入、資料、API、推播等能力的系統通常是服務端。

同一台電腦在不同情境下可以扮演不同角色。判斷重點不是設備本身，而是它此刻是在請求服務，還是在提供服務。

## 前端 / 後端與客戶端 / 服務端的差異

這兩組詞常被一起使用，但角度不同。

| 分類方式 | 關心重點 | 常見例子 |
| --- | --- | --- |
| 前端 / 後端 | 應用系統的分工 | 畫面呈現、互動、資料處理、業務規則 |
| 客戶端 / 服務端 | 網路互動中的角色 | 瀏覽器向 Web 服務器請求頁面 |

入門時可以簡化理解：

```text
前端常運行在客戶端。
後端常運行在服務端。
```

但不能完全畫等號。原因是：

- 客戶端也可能包含複雜邏輯。
- 服務端也可能產生 HTML 頁面。
- 前端 / 後端是工程分工，客戶端 / 服務端是互動角色。

## 終端是什麼

終端這個詞在不同語境中意思可能不同，初學者先掌握常見用法即可。

在日常開發中，終端通常指可以輸入命令、操作系統或開發工具的命令列介面。例如：

- Windows 中的 cmd、PowerShell 或 Windows Terminal。
- macOS、Linux 中的 Terminal。

它更像是一個操作入口。你可以在終端中執行指令、啟動本地服務器、安裝套件、查看檔案或執行專案。

終端不等於客戶端，也不等於服務端。它只是你操作電腦或開發環境的一種介面。

## 常見錯誤

### 把前端直接等同於客戶端

很多時候前端確實在瀏覽器中執行，但前端更強調應用分工。若後端產生 HTML 頁面，這份 HTML 仍然屬於前端頁面的呈現結果。

### 把服務器理解成只是一台硬體

服務器可以是硬體主機，也可以指在主機上運行並提供服務的程式或系統。學 Web 時，要更重視「它提供什麼服務」。

### 把終端當成網站的組成部分

終端是開發者操作環境的工具，不是使用者訪問網站時必然看到的畫面。

## 重點整理

- 前端負責呈現與互動，後端負責資料與服務邏輯。
- 客戶端是發出請求的一方，服務端是提供服務的一方。
- 前端 / 後端是應用分工；客戶端 / 服務端是網路角色。
- 終端通常指命令列操作介面，是開發過程中常用的工具。
