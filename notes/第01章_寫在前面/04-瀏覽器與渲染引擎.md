# 瀏覽器與渲染引擎

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明瀏覽器在 Web 開發中的角色。
- 理解瀏覽器渲染引擎負責做什麼。
- 知道為什麼不同瀏覽器可能呈現出不同效果。

## 問題情境

你寫了一份 HTML 文件後，使用者不是直接「看見 HTML 原始碼」，而是透過瀏覽器看到解析後的頁面。

因此，學 HTML 不只是背標籤，也要知道瀏覽器會讀取、解析、整理並顯示這些內容。

## 瀏覽器是什麼

瀏覽器是網頁顯示與運行的平台。當使用者輸入網址或點擊連結時，瀏覽器會向服務器請求資源，再把取得的 HTML、CSS、JavaScript、圖片等內容處理成畫面。

當代常見瀏覽器包含：

- Chrome
- Edge
- Firefox
- Safari
- Opera

早期教材常把 IE、Firefox、Chrome、Safari、Opera 稱為五大主流瀏覽器。這可以作為歷史對照，但 IE 已不屬於當代主流瀏覽器。

![五大主流瀏覽器與問世時間](../../origin/第01章_寫在前面/assets/images/browser-img-001-111cfe.png)

## 瀏覽器做了什麼

瀏覽器不是只把檔案下載下來而已。它至少會做幾件重要的事：

1. 根據網址或請求取得資源。
2. 解析 HTML，建立網頁內容結構。
3. 解析 CSS，計算畫面樣式。
4. 執行必要的 JavaScript。
5. 把結果繪製成使用者看到的頁面。

對初學者來說，先記住一件事就夠：

```text
HTML 描述內容結構，瀏覽器負責解析並顯示它。
```

## 瀏覽器內核與渲染引擎

瀏覽器內核在這裡主要指渲染引擎。它是瀏覽器中負責解析與渲染網頁內容的重要部分。

![五大瀏覽器與內核對照圖](../../origin/第01章_寫在前面/assets/images/browser-img-002-b448ff.png)

渲染引擎會讀取網頁內容、整理訊息、計算顯示方式，最後讓頁面出現在瀏覽器視窗中。

當我們看一個網頁時，流程可以簡化成：

1. 瀏覽器向服務器請求資源。
2. 服務器回傳 HTML、CSS、JavaScript、圖片等內容。
3. 瀏覽器內核解析這些資源。
4. 內核計算並渲染畫面。
5. 使用者看到完整頁面。

![瀏覽器內核處理網頁資源示意圖](../../origin/第01章_寫在前面/assets/images/browser-img-003-ccff67.png)

## 為什麼不同瀏覽器可能有差異

不同瀏覽器可能使用不同的渲染引擎；即使使用相同或相近的引擎，也可能因版本、設定或實作細節不同，而在解析速度、效能或顯示效果上有差異。

這也是前端開發需要關心瀏覽器相容性的原因。

## 實務注意

### 不要只在一個瀏覽器中測試

入門練習可以先用一個瀏覽器，但實際開發時，至少要在目標使用者常用的瀏覽器中確認畫面與互動。

### 不要依賴過時瀏覽器的行為

如果教材或圖片提到 IE，要把它當成歷史背景理解，不要把它視為現代 Web 開發的主要目標。

### 語法正確能降低差異

瀏覽器差異無法完全消失，但清楚、標準、語意正確的 HTML 結構，能降低瀏覽器誤解或容錯造成的問題。

## 重點整理

- 瀏覽器是顯示與運行網頁的平台。
- HTML 不會自己變成畫面，需要瀏覽器解析與渲染。
- 渲染引擎負責處理網頁資源並計算顯示結果。
- 不同瀏覽器或不同版本可能造成呈現差異，因此需要理解 Web 標準與相容性。
