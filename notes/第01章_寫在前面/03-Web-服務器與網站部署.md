# Web 服務器與網站部署

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 Web 服務器的作用。
- 理解為什麼只放在自己電腦上的網頁通常只能自己訪問。
- 分辨本地服務器與遠程服務器。
- 理解部署網站的基本意義。

## 問題情境

你在自己的電腦上寫了一個 HTML 檔案，雙擊打開後能看到頁面。這代表你自己能查看它，但不代表全世界的人都能訪問它。

如果希望其他人透過網址看到你的網站，就需要把網站放到能被網路訪問的服務器環境中。

![多位使用者連線至服務器示意圖](../../origin/第01章_寫在前面/assets/images/web-server-img-001-7d7f51.png)

## 一句話理解

Web 服務器負責接收瀏覽器請求，並把網頁文件、圖片、資料或其他資源回傳給使用者。

## 服務器是什麼

服務器是對外提供計算或網路服務的設備、程式或系統。

依照提供的服務不同，常見類型包含：

| 類型 | 主要作用 |
| --- | --- |
| 文件服務器 | 提供檔案存取。 |
| 資料庫服務器 | 儲存與查詢資料。 |
| 應用程序服務器 | 執行應用邏輯。 |
| Web 服務器 | 向瀏覽器等 Web 客戶端提供網頁與資源。 |

在 Web 入門階段，服務器也常被稱為主機。重點不是名稱，而是它能接收請求並回傳資源。

## Web 服務器做什麼

Web 服務器一般指網站服務器。它可以：

- 存放 HTML、CSS、JavaScript、圖片等網站文件。
- 接收瀏覽器發出的 HTTP / HTTPS 請求。
- 回傳瀏覽器需要的文件或資料。
- 配合後端程式產生動態內容。
- 提供下載檔案或 API 資料。

一個簡化流程如下：

```text
瀏覽器請求 https://example.com/index.html
  -> Web 服務器收到請求
  -> Web 服務器找到對應的 HTML 文件
  -> Web 服務器把文件回傳
  -> 瀏覽器解析並顯示頁面
```

## 為什麼本機檔案不能直接讓所有人訪問

如果你的網站只是放在自己的電腦桌面上，其他人通常無法直接透過網際網路存取它，原因包含：

- 你的電腦不一定對外開放連線。
- 網路、防火牆、路由器可能阻擋外部請求。
- 你的檔案沒有穩定的公開網址。
- 你的電腦不一定長時間開機並提供服務。

所以正式網站通常會部署到遠程服務器、雲端主機、靜態網站平台或其他可公開訪問的環境。

## 本地服務器

本地服務器是指在自己的電腦上啟動的服務器環境。

它常用於開發與測試，例如：

- 預覽 HTML、CSS、JavaScript 頁面。
- 測試本機 API。
- 模擬正式網站的請求流程。
- 避免直接用檔案路徑打開頁面造成某些功能失效。

本地服務器通常優先供自己的電腦訪問。如果網路、防火牆與服務綁定位址允許，同一個局域網內的其他裝置也可能訪問。

## 遠程服務器

遠程服務器通常是部署在其他機房、雲端平台或主機服務商上的服務器。

當網站放到遠程服務器後，使用者可以透過域名或公開網址訪問。例如：

```text
www.baidu.com
```

遠程服務器的價值在於：

- 能長時間提供服務。
- 能被網路上的使用者訪問。
- 可以綁定域名。
- 可以配合安全憑證、資料庫、後端服務等完整環境。

## 網站部署是什麼

部署就是把本地開發完成的網站或應用，放到可以提供服務的環境中。

對靜態 HTML 網站來說，部署可能只是上傳這些檔案：

```text
index.html
style.css
main.js
images/
```

對動態網站來說，部署還可能包含：

- 啟動後端服務。
- 設定資料庫。
- 設定域名與 HTTPS。
- 設定環境變數。
- 設定反向代理或 Web 服務器。

入門階段不用一次掌握所有部署細節，但要先理解：網站要讓別人訪問，必須被放到能對外提供服務的位置。

## 常見錯誤

### 以為能在自己電腦打開就代表已經上線

本機能開，只代表你的電腦能讀到檔案。其他人是否能訪問，取決於是否有公開服務地址與可連線的服務器環境。

### 把服務器只理解成遠程主機

本地也可以啟動服務器。差別在於用途、可訪問範圍與部署環境。

### 忽略瀏覽器與服務器的請求關係

瀏覽器不是直接「看到服務器硬碟」。它是透過請求取得資源，再由瀏覽器解析與呈現。

## 重點整理

- Web 服務器負責向瀏覽器提供網頁文件、資料或其他資源。
- 本地檔案通常只能自己訪問，不等於網站已正式部署。
- 本地服務器主要用於開發與測試。
- 遠程服務器可以讓使用者透過公開網址訪問網站。
- 部署是把網站放到能提供服務的環境中。
