# Web 標準與三層分離

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明為什麼需要 Web 標準。
- 理解結構、表現、行為三層分工。
- 知道 HTML、CSS、JavaScript 在頁面中的主要責任。
- 理解三層分離如何提升可讀性與維護性。

## 問題情境

如果每個瀏覽器都用自己的方式解析網頁，同一份程式碼在不同瀏覽器中顯示完全不同，使用者體驗會很差，開發者也很難維護。

Web 標準就是為了讓不同瀏覽器盡量遵循共同規範，降低解析與呈現差異。

![不同瀏覽器呈現不同網頁效果示意圖](../../origin/第01章_寫在前面/assets/images/web-standard-img-001-e119b2.png)

## 一句話理解

Web 標準讓瀏覽器、開發者與工具依照共同規則理解網頁；三層分離則讓 HTML、CSS、JavaScript 各自負責清楚的任務。

## 為什麼需要 Web 標準

瀏覽器需要解析 HTML、CSS、JavaScript，並把它們轉換成使用者看到與操作的頁面。

如果沒有共同標準，會出現幾個問題：

- 同一份 HTML 在不同瀏覽器中結構理解不同。
- 同一段 CSS 在不同瀏覽器中樣式差異很大。
- 同一段 JavaScript 在不同環境中行為不穩定。
- 開發者需要為每個瀏覽器寫大量特殊版本。
- 使用者在不同裝置上得到不一致的體驗。

Web 標準的目的，就是讓不同瀏覽器盡量依照共同規範解析與呈現內容。

![Web 標準統一不同瀏覽器顯示效果示意圖](../../origin/第01章_寫在前面/assets/images/web-standard-img-002-783e20.png)

## Web 標準的三層構成

Web 標準常從三個層面理解：

![Web 標準結構、表現與行為對照表](../../origin/第01章_寫在前面/assets/images/web-standard-img-003-2d03d3.png)

| 層次 | 主要技術 | 負責內容 |
| --- | --- | --- |
| 結構 | HTML | 描述頁面有哪些內容，以及內容的語意與層級。 |
| 表現 | CSS | 控制頁面外觀，例如顏色、排版、字體、間距。 |
| 行為 | JavaScript | 控制互動與動態效果，例如點擊、資料更新、表單驗證。 |

簡單理解：

```text
HTML 寫結構。
CSS 寫樣式。
JavaScript 寫互動。
```

## 三層分離是什麼

三層分離不是說三者完全沒有關係，而是讓不同責任盡量放在合適的位置。

例如同樣做一個按鈕區塊：

```html
<button class="primary-button" type="button">送出</button>
```

HTML 負責說明這是一個按鈕，文字是「送出」。

```css
.primary-button {
  color: white;
  background-color: #2563eb;
}
```

CSS 負責按鈕長什麼樣子。

```javascript
document.querySelector(".primary-button").addEventListener("click", function () {
  console.log("使用者點擊送出");
});
```

JavaScript 負責使用者點擊後發生什麼事。

這樣分工後，當你只想改顏色，主要改 CSS；只想改文字結構，主要改 HTML；只想改互動流程，主要改 JavaScript。

## 為什麼 HTML 要重視語意

在三層分離中，HTML 的責任不是「把畫面弄漂亮」，而是建立正確的內容結構。

例如：

```html
<h1>文章標題</h1>
<p>文章內容。</p>
```

比起只用沒有語意的元素硬排畫面，語意化 HTML 更有利於：

- 瀏覽器正確解析文件結構。
- 搜尋引擎理解頁面內容。
- 螢幕閱讀器等輔助技術讀取頁面。
- 工程師日後維護與修改。

CSS 可以改變外觀，但不應取代 HTML 的語意責任。

## 常見錯誤

### 把樣式直接塞進 HTML 結構

不建議把大量樣式混在 HTML 中，否則結構會變得難讀，也不利於重複使用。

較不理想：

```html
<p style="color: red; font-size: 20px;">重要提醒</p>
```

較易維護：

```html
<p class="notice">重要提醒</p>
```

```css
.notice {
  color: red;
  font-size: 20px;
}
```

### 用 JavaScript 補救不合理的 HTML 結構

如果一開始 HTML 結構混亂，後續用 JavaScript 操作時也會變得複雜。應先讓 HTML 結構清楚，再增加樣式與互動。

### 只追求畫面像，不在意結構

初學者常用任何能顯示畫面的方式完成頁面。但正式開發要重視語意、層級、可讀性與可維護性。

## 實務使用場景

寫一個頁面時，可以先按這個順序思考：

```text
1. 這個頁面有哪些內容？
2. 這些內容的語意與層級是什麼？
3. 用哪些 HTML 標籤表達最合適？
4. 需要哪些 CSS 讓畫面符合設計？
5. 需要哪些 JavaScript 增加互動？
```

這個順序能避免一開始就陷入顏色、動畫或細節效果，先把頁面骨架建立好。

## 重點整理

- Web 標準讓瀏覽器與開發者依照共同規則理解網頁。
- 結構、表現、行為分別對應 HTML、CSS、JavaScript。
- HTML 的核心責任是描述內容結構與語意。
- 三層分離能提升可讀性、可維護性與團隊協作效率。
- 學 HTML 時，應先重視結構正確，再處理樣式與互動。
