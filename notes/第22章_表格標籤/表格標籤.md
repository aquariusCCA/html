# HTML 表格標籤 table

## 1. 本章學習目標

讀完本章後，你應該能夠：

- 說明 HTML 表格的用途，以及為什麼表格不應用來做版面布局。
- 使用 `<table>`、`<tr>`、`<th>`、`<td>` 建立基本表格。
- 理解 `<caption>`、`<thead>`、`<tbody>`、`<tfoot>` 在完整表格中的角色。
- 分辨表頭儲存格 `<th>` 與一般資料儲存格 `<td>` 的語意差異。
- 使用 `rowspan` 與 `colspan` 合併儲存格。
- 認識表格常見舊屬性，並理解實務上為什麼多數樣式應交給 CSS。
- 避免表格巢狀錯誤、語意不清與無障礙支援不足等常見問題。

## 2. 本章定位

本章介紹 HTML 中用來呈現表格資料的相關標籤。

表格適合用來展示具有「行與列關係」的資料，例如成績表、報表、商品規格、訂單明細、後台資料列表、統計數據等。表格的價值不只是讓資料排得整齊，而是用 HTML 結構明確表達資料之間的對應關係。

> 來源重點：原始資料強調表格不是用來布局的，而是用來展示數據；表格能讓資料顯示得規整，尤其常見於後台資料展示。

在現代前端開發中，頁面排版主要交給 CSS，例如 Flexbox、Grid 或一般區塊排版。`<table>` 應該保留給真正的表格資料，避免只為了排版而使用表格。

## 3. 核心概念

HTML 表格由一組行與儲存格組成。最基本的結構是：

- `<table>`：表格容器。
- `<tr>`：table row，表格中的一列。
- `<th>`：table header，表頭儲存格，用來表示欄位或列的標題。
- `<td>`：table data，資料儲存格，用來放實際資料。

### 3.1 表格描述的是資料關係

表格適合呈現「一筆資料有多個欄位」或「多筆資料之間需要橫向、縱向比較」的內容。

例如一份成績表中，每一列代表一個學生，每一欄代表姓名、科目或分數。使用表格可以讓瀏覽器、搜尋引擎與輔助工具理解這些資料不是普通段落，而是有欄列關係的資料。

### 3.2 表格不是排版工具

早期網頁常用 `<table>` 切版，但這不是現代 HTML 的建議做法。

不建議用表格排版的原因包括：

- 表格語意會讓輔助工具誤以為內容是資料表。
- 表格結構通常較重，維護成本高。
- 響應式排版較不彈性。
- CSS 已提供更適合版面控制的方式。

> 補充說明：HTML 負責內容與語意，CSS 負責畫面樣式。若內容不是表格資料，就不應為了對齊畫面而使用 `<table>`。

### 3.3 完整表格可以分成多個區域

較完整的表格常包含：

| 標籤 | 語意 | 用途 |
|---|---|---|
| `<caption>` | 表格標題 | 說明整張表格的主題 |
| `<thead>` | 表格頭部 | 放欄位標題列 |
| `<tbody>` | 表格主體 | 放主要資料列 |
| `<tfoot>` | 表格腳注 | 放總計、備註或彙總資料 |

這些標籤能讓表格結構更清楚，也有助於 CSS 選取、JavaScript 操作與輔助工具理解內容。

## 4. 語法與基本用法

### 4.1 基本表格結構

最基本的表格會使用 `<table>` 包住多個 `<tr>`，再由 `<th>` 或 `<td>` 組成每一列中的儲存格。

```html
<table>
  <tr>
    <th>姓名</th>
    <th>性別</th>
    <th>年齡</th>
  </tr>
  <tr>
    <td>劉德華</td>
    <td>男</td>
    <td>56</td>
  </tr>
  <tr>
    <td>張學友</td>
    <td>男</td>
    <td>58</td>
  </tr>
</table>
```

這個表格中：

| 標籤 | 說明 |
|---|---|
| `<table>` | 定義整張表格 |
| `<tr>` | 定義表格中的一列 |
| `<th>` | 定義表頭儲存格，瀏覽器預設通常會加粗並置中 |
| `<td>` | 定義一般資料儲存格 |

> 注意：`<tr>` 必須放在表格相關容器中，常見是 `<table>`、`<thead>`、`<tbody>` 或 `<tfoot>` 內；`<td>` 與 `<th>` 則應放在 `<tr>` 內。

### 4.2 加上表格標題與區域分組

對較完整的表格，建議使用 `<caption>`、`<thead>`、`<tbody>`、`<tfoot>` 組織內容。

```html
<table>
  <caption>段考成績</caption>
  <thead>
    <tr>
      <th>科目</th>
      <th>分數</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>語文</td>
      <td>99</td>
    </tr>
    <tr>
      <td>數學</td>
      <td>60</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>總分</td>
      <td>159</td>
    </tr>
  </tfoot>
</table>
```

這個結構比單純把所有 `<tr>` 都放在 `<table>` 下面更清楚：

- `<caption>` 告訴讀者這張表的主題是「段考成績」。
- `<thead>` 放欄位名稱。
- `<tbody>` 放主要分數資料。
- `<tfoot>` 放總分。

### 4.3 常見表格屬性

原始資料列出了一些常見表格屬性，例如 `align`、`border`、`cellpadding`、`cellspacing`、`width`。

```html
<table align="center" border="1" cellpadding="20" cellspacing="0" width="500">
  <tr>
    <th>商品</th>
    <th>價格</th>
  </tr>
  <tr>
    <td>鍵盤</td>
    <td>1200</td>
  </tr>
</table>
```

這些屬性在早期 HTML 中常用來控制表格外觀：

| 屬性 | 用途 | 實務提醒 |
|---|---|---|
| `align` | 設定表格相對周圍元素的對齊方式 | 現代開發通常改用 CSS |
| `border` | 設定表格邊框寬度 | 可用於初學展示，但正式樣式建議用 CSS |
| `cellpadding` | 設定儲存格內容與邊框之間的距離 | 現代開發通常用 CSS `padding` |
| `cellspacing` | 設定儲存格之間的距離 | 現代開發通常用 CSS `border-spacing` 或 `border-collapse` |
| `width` | 設定表格寬度 | 現代開發通常用 CSS `width` |

> 注意：原始資料提到這些屬性在實際開發中不常用，後續多半會透過 CSS 設定。正式專案中，建議將結構寫在 HTML，視覺樣式交給 CSS。

### 4.4 使用 CSS 控制表格樣式

較推薦的做法是讓 HTML 保持語意結構，再用 CSS 控制外觀。

```html
<table class="score-table">
  <caption>段考成績</caption>
  <thead>
    <tr>
      <th>科目</th>
      <th>分數</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>語文</td>
      <td>99</td>
    </tr>
    <tr>
      <td>數學</td>
      <td>60</td>
    </tr>
  </tbody>
</table>
```

```css
.score-table {
  width: 100%;
  border-collapse: collapse;
}

.score-table th,
.score-table td {
  border: 1px solid #999;
  padding: 8px 12px;
  text-align: left;
}
```

這樣可以讓 HTML 專注描述資料結構，CSS 專注控制表格寬度、邊框、間距與對齊。

### 4.5 合併儲存格

當資料需要跨多列或多欄呈現時，可以使用 `rowspan` 與 `colspan`。

| 屬性 | 放置位置 | 作用 |
|---|---|---|
| `rowspan` | `<th>` 或 `<td>` | 讓一個儲存格跨越多列 |
| `colspan` | `<th>` 或 `<td>` | 讓一個儲存格跨越多欄 |

```html
<table>
  <tr>
    <td></td>
    <td colspan="2">跨兩欄</td>
  </tr>
  <tr>
    <td rowspan="2">跨兩列</td>
    <td>資料 A</td>
    <td>資料 B</td>
  </tr>
  <tr>
    <td>資料 C</td>
    <td>資料 D</td>
  </tr>
</table>
```

合併儲存格時可以依照三個步驟思考：

1. 先判斷要跨列還是跨欄。
2. 找到要保留的目標儲存格，加入 `rowspan` 或 `colspan`。
3. 刪除被合併後多餘的儲存格，避免表格欄列數錯亂。

## 5. 範例說明

### 範例一：後台資料表

```html
<table>
  <caption>會員清單</caption>
  <thead>
    <tr>
      <th>姓名</th>
      <th>Email</th>
      <th>會員狀態</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>王小明</td>
      <td>ming@example.com</td>
      <td>啟用</td>
    </tr>
    <tr>
      <td>林小華</td>
      <td>hua@example.com</td>
      <td>停用</td>
    </tr>
  </tbody>
</table>
```

這個範例適合使用表格，因為每一列都是一個會員，每一欄都是該會員的一個資料欄位。這種欄列關係正是 `<table>` 的使用場景。

### 範例二：含總計的成績表

```html
<table>
  <caption>期末成績</caption>
  <thead>
    <tr>
      <th>科目</th>
      <th>分數</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>國文</td>
      <td>90</td>
    </tr>
    <tr>
      <td>英文</td>
      <td>85</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>平均</th>
      <td>87.5</td>
    </tr>
  </tfoot>
</table>
```

這裡使用 `<tfoot>` 放平均分數，讓彙總資訊與主要資料區分開。`平均` 是該列的標題，因此使用 `<th>` 會比 `<td>` 更有語意。

### 範例三：課表中的合併儲存格

```html
<table>
  <caption>週一課表</caption>
  <thead>
    <tr>
      <th>時間</th>
      <th>課程</th>
      <th>教室</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>09:00 - 10:00</td>
      <td rowspan="2">HTML 基礎</td>
      <td rowspan="2">A101</td>
    </tr>
    <tr>
      <td>10:00 - 11:00</td>
    </tr>
  </tbody>
</table>
```

這個範例中，「HTML 基礎」與「A101」跨越兩個時間段，因此使用 `rowspan="2"`。第二列不再重複寫課程與教室欄位，因為它們已被上一列的儲存格合併。

## 6. 實務應用場景

表格常見於需要比較、檢索或管理資料的畫面，例如：

- 後台會員列表、訂單列表、商品列表。
- 成績表、課表、班級名單。
- 價格比較表、方案比較表。
- 報表、統計數據、財務資料。
- 商品規格表、技術參數表。

在實務開發中，表格通常還會搭配：

| 技術 | 常見用途 |
|---|---|
| CSS | 控制邊框、間距、對齊、斑馬紋、響應式呈現 |
| JavaScript | 排序、篩選、分頁、勾選列、批次操作 |
| 後端 API | 取得表格資料，例如會員、訂單或報表資料 |
| Vue 等框架 | 將資料陣列渲染成多列 `<tr>` |

即使使用框架產生表格，最後瀏覽器中仍然會形成 HTML 與 DOM 結構。因此表格的語意、欄列結構與可訪問性仍然重要。

## 7. 常見錯誤與修正

| 常見錯誤 | 問題原因 | 修正方式 |
|---|---|---|
| 用 `<table>` 做整頁排版 | 表格語意是資料表，不是版面布局 | 使用 CSS Flexbox、Grid 或一般區塊排版 |
| 直接把 `<td>` 放在 `<table>` 裡 | `<td>` 應該放在 `<tr>` 中 | 先建立 `<tr>`，再放入 `<td>` 或 `<th>` |
| 所有儲存格都用 `<td>` | 欄位標題缺少表頭語意 | 欄位名稱或列標題應使用 `<th>` |
| 合併儲存格後沒有刪除多餘儲存格 | 欄列數會錯亂，畫面與語意都可能出問題 | 使用 `rowspan` 或 `colspan` 後，移除被合併位置上的多餘 `<td>` |
| 使用舊屬性控制所有樣式 | HTML 與樣式混在一起，後續不好維護 | 用 HTML 寫結構，用 CSS 控制外觀 |
| 表格沒有標題或上下文 | 使用者不容易知道表格用途 | 使用 `<caption>` 或在表格前提供清楚標題 |
| 把 `border` 當成控制所有儲存格高度的工具 | `border` 主要控制邊框，不負責資料列高度設計 | 用 CSS 的 `padding`、`line-height`、`height` 等控制視覺效果 |

### 錯誤範例：用表格做版面布局

```html
<table>
  <tr>
    <td>左側選單</td>
    <td>主要內容</td>
  </tr>
</table>
```

問題：這不是表格資料，而是頁面版面。使用 `<table>` 會造成語意錯誤。

### 正確範例：用語意元素與 CSS 做版面

```html
<main class="layout">
  <nav>左側選單</nav>
  <section>主要內容</section>
</main>
```

```css
.layout {
  display: grid;
  grid-template-columns: 240px 1fr;
  gap: 24px;
}
```

這樣 HTML 表達的是頁面區域語意，CSS 負責控制版面排列。

## 8. 與其他概念的關係

| 相關概念 | 關係 |
|---|---|
| CSS | 表格的邊框、間距、寬度、對齊、斑馬紋與響應式效果通常交給 CSS 控制 |
| JavaScript | 常用於表格排序、篩選、分頁、動態新增列或刪除列 |
| DOM | 瀏覽器會把表格解析成 DOM 節點，JavaScript 操作的是這些節點 |
| 可訪問性 | 正確使用 `<caption>`、`<th>` 與清楚的表格結構，有助於輔助工具理解資料 |
| 後台系統 | 表格常用於資料管理介面，例如會員、訂單、庫存與報表 |
| Vue | Vue 可以用資料陣列渲染 `<tr>`，但仍應輸出正確的表格語意 |

## 9. 重點整理

- `<table>` 用來呈現表格資料，不應用來做頁面布局。
- `<tr>` 表示表格中的一列，`<th>` 表示表頭儲存格，`<td>` 表示一般資料儲存格。
- 完整表格可以使用 `<caption>`、`<thead>`、`<tbody>`、`<tfoot>` 提升結構清晰度。
- `rowspan` 用於跨列合併，`colspan` 用於跨欄合併。
- 合併儲存格時，要刪除被合併掉的多餘儲存格，避免欄列錯亂。
- 表格舊屬性可以幫助初學理解效果，但正式專案中多數樣式應用 CSS 控制。
- 良好的表格語意會影響可讀性、可維護性、DOM 操作與可訪問性。

## 11. 延伸補充

> 補充說明：瀏覽器會將 HTML 表格解析成 DOM Tree。CSS 可以根據 `<table>`、`<thead>`、`<tbody>`、`<tr>`、`<th>`、`<td>` 等節點套用樣式；JavaScript 則可以基於 DOM 進行排序、篩選或動態更新資料。

> 補充說明：複雜表格在可訪問性上需要更仔細設計，例如讓表頭與資料儲存格的關係更清楚。初學階段至少應先做到：使用 `<caption>` 說明表格主題、使用 `<th>` 表示欄位或列標題、保持欄列結構一致。
