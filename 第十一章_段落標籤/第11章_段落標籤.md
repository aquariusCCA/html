# 段落標籤

> 所屬章節：第十一章｜段落標籤

> 建議回查情境：想知道 `<p>` 是做什麼的、想確認段落標籤有哪些預設特點、想分清「新段落」和「單純換行」的差別、想知道什麼時候該開新的 `<p>`

## 本節導讀

這篇整理 HTML 的段落標籤 `<p>`。

`<p>` 的核心作用，是把一段連續文字標示成**一個完整段落**。
它不只是讓畫面出現空隙，也不只是讓文字看起來分成幾塊，而是在描述：

* 這裡是一段完整的文字內容
* 這一段和下一段是不同的敘述單位
* 文件在結構上已經進入新的段落

初學 HTML 時，很多人會先注意到兩件事：

* 段落和段落之間好像有空隙
* 文字會隨畫面寬度自動換行

但真正更重要的是理解：

> `<p>` 代表的是「段落」，不是單純的空白，也不是單純的換行。

* `<p>` 用來表示 HTML 文件中的一個段落。
* 一個 `<p>` 通常包住一段意思相對完整的文字。
* 不同段落之間，瀏覽器通常會顯示出間距。

## 速查區

### 核心概念

* `<p>` 是 HTML 中最基本的文字段落標籤。
* 它表示的是一個完整的段落單位，而不是單純插入空白。
* 重點不只是畫面怎麼顯示，而是內容在結構上怎麼分段。

### 關鍵判準

當你在判斷要不要開新的 `<p>` 時，可以先問自己：

1. 這段文字是不是一個新的敘述單位？
2. 內容是不是已經換到新的意思或新的重點？
3. 我現在是在分段，還是只是想讓畫面空一點？

如果只是想調整外觀間距，通常應該交給 CSS；
如果是內容真的進入新段落，才適合開新的 `<p>`。

### 常見特點

* 瀏覽器預設通常會讓段落之間保有空隙
* `<p>` 通常會以區塊方式呈現
* 段落中的文字會隨視窗寬度自動換行

### 常見混淆點

* 段落是內容結構，不只是視覺留白
* 自動換行不代表產生新段落
* 新段落和同一段中的換行不是同一件事
* `<p>` 處理的是段落，`<br>` 處理的通常是同段落中的換行

### 一句話對比

* `<p>` 解決的是「這是不是一個新的段落」
* `<br>` 解決的是「同一段內容要不要在這裡換行」

## 正文筆記

## 1. `<p>` 的作用是什麼？

`<p>` 是 HTML 用來表示**段落**的標籤。

所謂段落，不只是畫面上看起來分成幾塊，而是內容本身已經形成一個相對完整的敘述單位。
因此，當文字進入新的意思、新的重點、新的敘述方向時，通常就適合開新的 `<p>`。

你可以把它理解成：

* 一個 `<p>` = 一段相對完整的文字內容
* 多個 `<p>` = 文件被分成多個不同段落

---

## 2. 為什麼需要段落？

在文章、新聞、教學說明、產品介紹裡，文字通常不會全部擠成一整塊。

如果所有內容都塞在同一大段裡：

* 讀者不容易理解結構
* 讀起來會很累
* 很難快速抓到每段重點

而 `<p>` 的作用，就是把內容切成一段一段更容易閱讀的單位。
所以段落不只是排版方便，而是和閱讀理解直接有關。

---

## 3. 段落標籤的常見預設特點

瀏覽器通常會讓 `<p>` 有幾個常見表現：

* 段落和段落之間會有空隙
* 每個段落通常各自佔據一個區塊
* 段落內文字會依畫面寬度自動換行

### 這些特點怎麼理解？

這些都是**瀏覽器預設呈現效果**，目的是讓段落更容易辨認。
但要注意：

> 真正代表「這是新段落」的，不是空隙，也不是文字換行，而是你有沒有使用新的 `<p>`。

也就是說：

* 畫面變窄，文字自然換行
  → 仍然可能還是同一個段落
* 使用另一個新的 `<p>`
  → 才表示內容進入新的段落

---

## 4. 基本寫法

```html
<p>今天我學習 HTML 的段落標籤。</p>
<p>段落標籤可以讓文章內容更容易閱讀。</p>
```

### 這個例子在表達什麼？

* 第一個 `<p>` 是第一段
* 第二個 `<p>` 是第二段
* 兩段文字彼此分開，表示它們是兩個不同的段落單位

這種短例子比長文章更容易看出重點：

> 一個 `<p>` 包住的是一整段，而不是只表示空白效果。

---

## 5. 用長一點的例子理解「分段」

```html
<p>前端開發不只是把畫面做出來，也包含如何讓內容結構清楚、容易閱讀。</p>

<p>在 HTML 中，段落標籤可以把長文字拆成較容易理解的閱讀單位，避免所有內容擠成一整塊。</p>
```

### 這段例子重點在哪？

* 這裡用了兩個 `<p>`
* 每個 `<p>` 都包住一個相對完整的意思
* 第二段不是第一段的單純續行，而是另一個新的敘述重點

所以「新段落」不是看畫面是不是換到下一行，而是看**內容是不是進入另一個新的段落單位**。

---

## 6. 自動換行和新段落有什麼不同？

這是初學者最容易混淆的地方。

### 自動換行

當瀏覽器可用寬度變窄時，同一段文字可能會自動換到下一行。
這只是顯示上的自然排版，不代表你建立了新的段落。

例如：

```html
<p>這是一段很長的文字，當畫面變窄時，它可能會自動換到下一行，但它仍然是同一個段落。</p>
```

上面不管換成幾行，只要還包在同一個 `<p>` 裡，它本質上都還是**同一段**。

### 新段落

只有當你開了新的 `<p>`，才代表內容在結構上進入了另一段。

```html
<p>這是第一段。</p>
<p>這是第二段。</p>
```

所以要記住：

* 換行是顯示現象
* 分段是內容結構

---

## 7. `<p>` 和 `<br>` 的差別

這一段很重要，因為很多人剛學 HTML 時會混在一起。

### `<p>`

`<p>` 用來表示**新的段落**。

```html
<p>這是第一段。</p>
<p>這是第二段。</p>
```

這表示文件中有兩個不同段落。

### `<br>`

`<br>` 通常用來表示**同一段內容中的換行**。

```html
<p>台北市中正區<br>忠孝西路一段 100 號</p>
```

這裡仍然是同一個段落，只是內容在中間換了一行。

### 一句話記法

* `<p>`：換段
* `<br>`：換行

---

## 8. 錯誤示範 vs 正確示範

### 情境一：只是想讓畫面空一點

#### 不好的想法

因為想讓畫面多幾個空隙，所以亂切很多 `<p>`。

```html
<p>今天學 HTML。</p>
<p>我覺得很有趣。</p>
<p>真的很有趣。</p>
```

如果這三句其實都還在同一個敘述脈絡中，硬拆太多段，閱讀節奏反而可能變得零碎。

#### 更好的判斷

應先看內容是不是已經進入新的意思。
如果還是同一段敘述，就未必需要拆成多個 `<p>`。

---

### 情境二：把整篇長文塞進同一個 `<p>`

#### 不好的寫法

```html
<p>這裡放了一整篇超長文章，從開頭講到結尾，中間完全沒有分段……</p>
```

這樣做雖然語法上不一定錯，但閱讀性通常很差。

#### 更好的寫法

```html
<p>第一段先介紹主題背景。</p>
<p>第二段開始補充主要內容。</p>
<p>第三段再做進一步說明。</p>
```

當內容已經換到新的重點時，就應該分段。

---

### 情境三：把 `<p>` 當成單純外觀工具

#### 錯誤觀念

> 我想要多一點空白，所以我就多開幾個 `<p>`

這種想法把內容結構和畫面樣式混在一起了。

#### 正確觀念

* 內容分段 → 用 `<p>`
* 外觀間距 → 用 CSS

例如：

```html
<p class="intro">這是一段介紹文字。</p>
```

```css
.intro {
  margin-bottom: 24px;
}
```

這樣才是把「結構」和「樣式」分開。

---

## 9. 什麼時候應該開新的 `<p>`？

通常遇到下面幾種情況，就很適合開新的 `<p>`：

* 已經進入新的意思
* 已經換了新的敘述重點
* 閱讀上明顯需要切段
* 這段內容本身就可以獨立成一段

反過來說，如果只是：

* 同一個意思的延續
* 只是因為畫面太窄而換行
* 只是想製造空白效果

那就不一定要開新的 `<p>`。

---

## 10. 寫 `<p>` 時可以怎麼自我檢查？

寫完後可以快速問自己：

1. 這段文字是不是一個完整段落？
2. 我現在開新 `<p>`，是因為內容換了，還是只是因為想留白？
3. 這裡應該是新段落，還是其實只是同段中的換行？
4. 如果讀者只看段落切分，能不能感受到內容結構？

如果這幾題都能回答得清楚，通常你的段落就不會切錯。

## 常見問法

### 為什麼同一個 `<p>` 裡的文字會自己換行？

因為瀏覽器會依目前可用寬度自動排版文字。
這只是顯示上的自然換行，不代表建立了新段落。

---

### 什麼時候應該開新的 `<p>`？

當內容已經進入新的意思、新的敘述重點，或閱讀上需要明顯分段時，就適合開新的 `<p>`。

---

### 段落和空白行是一樣的事嗎？

不一樣。

* 空白行偏向視覺效果
* 段落是在描述內容本身的結構

---

### `<p>` 和 `<br>` 有什麼差別？

* `<p>`：表示新段落
* `<br>`：表示同一段中的換行
