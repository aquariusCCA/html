路徑 Demo

這頁把第六章的核心問題做成可切換示例：路徑到底是從哪裡開始算、什麼是同層 / 下一層 / 上一層、 以及絕對路徑和相對路徑各自常見的優缺點。

絕對路徑 不依賴目前檔案位置，直接指向固定位置或完整網址。
相對路徑 一定要先看「目前這個 HTML 檔案在哪裡」。
./ 與 ../ ./ 是目前層級，常可省略；../ 代表上一層。
常見錯誤 搞錯基準位置、直接寫本地磁碟路徑、移動檔案後忘記更新相對路徑。

這頁怎麼看

  • 先看第一區，把「目前檔案」和「目標檔案」的相對位置走一次。
  • 再切換同層、下一層、上一層，對照實際寫法。
  • 最後看絕對路徑與常見錯誤，建立判讀框架。

1. 相對路徑到底是以誰為基準

同層尋找

同層：目前檔案和目標檔案在同一個資料夾

這時候通常直接寫檔名就夠了，不需要先進資料夾，也不需要退上一層。 

<img src="logo.png" alt="logo">
Step 1 先看目前 HTML 檔案在哪裡。
Step 2 再看目標檔案是不是在同一層。
Step 3 如果是，就直接寫檔名。
相對路徑不是先看專案根目錄，而是先看「你正在寫這條路徑的那個檔案」在哪裡。
index.html
logo.png
目前與目標在同一層，所以直接寫 `logo.png`。

2. ./、省略寫法、../ 的差別

目前層級

./：表示從目前層級開始找

在很多情況下，`./images/logo.png` 和 `images/logo.png` 的效果是一樣的。 

<img src="./images/logo.png" alt="logo">
常見理解

`./` 代表目前資料夾，寫出來會更明確。

實務提醒

很多專案會直接省略 `./`，因為效果通常一樣。

如果要回上一層才找得到目標，就不能省略成同層寫法，必須明確寫 `../`。

3. 絕對路徑和相對路徑怎麼分

固定位置

絕對路徑：不依賴目前 HTML 檔案位置

它可以是本地磁碟的完整位置，也可以是完整網址；共同點是不用先看目前檔案在哪。 

D:\day01\images\1.jpg
https://www.example.com/images/logo.png
參考基準 固定位置，不依賴目前 HTML 檔案所在資料夾。
常見優點 位置明確，一看就知道要去哪裡找。
常見限制 本地磁碟路徑換環境容易壞；外部網址則可能碰到權限或防盜鏈。

4. 常見錯誤判讀

環境依賴

本地絕對路徑通常不適合直接寫進網頁

這種寫法可能只在你的電腦有效，一旦換機器、換目錄或部署上伺服器，路徑就失效。 

<img src="D:\project\images\a.png" alt="">
路徑能不能成立，不是只看字串長相，而是要看目前檔案位置、目標位置與執行環境。
