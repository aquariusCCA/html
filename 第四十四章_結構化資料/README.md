# 第四十四章｜結構化資料

> 章節定位：理解網頁如何用機器可讀資料描述頁面內容，讓搜尋引擎與工具更容易理解  
> 建議閱讀方式：先理解 JSON-LD 放在哪裡與描述什麼，再看它和一般可見 HTML 的關係

## 本章在學什麼

這一章是選修進階章，整理 HTML 頁面中常見的結構化資料觀念。

重點不是保證 SEO 成效，而是理解：

- 結構化資料是給機器讀的補充描述。
- JSON-LD 常透過 `<script type="application/ld+json">` 放進頁面。
- 結構化資料應該和頁面可見內容一致。
- 它不能取代語意化 HTML，也不能保證搜尋結果一定有特殊樣式。

## 適合快速回查的主題

- 結構化資料是什麼
- JSON-LD 怎麼放進 HTML
- `type="application/ld+json"` 是什麼
- Schema.org 扮演什麼角色
- 結構化資料和 `meta`、Open Graph 差在哪
- 為什麼結構化資料不能亂寫不存在的內容

## 易混淆主題

- 結構化資料 vs 可見內容：前者給機器讀，後者給使用者讀，兩者應一致。
- JSON-LD vs JavaScript 程式：JSON-LD 是資料，不是用來執行頁面邏輯的腳本。
- SEO 保證 vs 搜尋理解：結構化資料有助於理解，不保證排名或特殊展示。
