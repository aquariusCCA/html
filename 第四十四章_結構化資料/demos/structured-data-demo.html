結構化資料 Demo

這個範例把第 44 章筆記中的 JSON-LD、Schema.org、可見內容一致性、語意化 HTML、meta description 與 Open Graph 分工放在同一頁觀察。右側的 <script type="application/ld+json"> 是資料，不是頁面互動程式。

JSON-LD 用機器可讀格式描述頁面上的實體與屬性。
@context 指定詞彙表，常見為 https://schema.org。
@type 指定這段資料描述的是文章、商品、課程、活動或 FAQ。
一致性 JSON-LD 應描述頁面上真實可見的內容。
不是保證 有助理解，不保證排名或 rich result。

先有內容

先把頁面內容寫完整、真實、可見，再用 JSON-LD 補充機器可讀描述。

不是 JS 程式

application/ld+json 裡放的是資料，不是操作 DOM 的 JavaScript。

不能偽造

頁面沒有評分、價格、評論或活動時間，就不要只寫在 JSON-LD 裡。

5. 常見錯誤修正

        

          

            
尾逗號

            
JSON 不允許最後一個欄位後面多逗號。先確保格式合法，再談 Schema 是否合理。

          

          

            
內容不一致

            
頁面沒有評分就不要寫 aggregateRating，沒有價格就不要寫商品價格。

          

          

            
類型不符合

            
商品頁不要誤用 Article；課程頁應考慮 Course。

          

        

      


      

        
6. 實務檢查清單

        
    
          
  • 頁面是否真的有文章、商品、課程、活動或 FAQ 等明確內容？
    • 
          
  • JSON-LD 裡的標題、作者、日期、價格、地點或問答是否能在頁面上看到？
    • 
          
  • @context 是否使用合適詞彙表，例如 https://schema.org？
    • 
          
  • @type 是否符合頁面內容？
    • 
          
  • JSON 格式是否有效，沒有尾逗號或錯誤引號？
    • 
          
  • 是否已先做好語意化 HTML，而不是只依賴 JSON-LD？
    • 
          
  • 是否用官方測試工具或 DevTools 確認 JSON-LD 真的輸出到 HTML？
