# 引用與註記文本標籤

> - 所屬章節：第三十九章｜引用與註記文本標籤  
> - 關鍵字：`blockquote`、`q`、`cite`、`abbr`、`dfn`、引用、縮寫、定義、作品名、來源標題、術語  
> - 建議回查情境：想標示引用文字、作品來源、縮寫全稱、正在定義的術語，或想分清 `<blockquote>` / `<q>` / `<cite>` 時

## 本節導讀

文章、教材和技術文件裡，經常會出現以下幾種文字：

- 從其他地方引用來的一段話。
- 某本書、某篇文章、某份規格文件的名稱。
- 像 HTML、CSS、API 這類縮寫。
- 第一次被解釋的術語或專有名詞。

這些內容如果只用普通 `<span>`、`<div>` 或純文字，也能正常顯示，但語意不夠清楚。

HTML 提供了一組文字語意標籤，讓瀏覽器、搜尋工具、輔助技術和維護者更容易理解這些文字的角色。

本節要掌握的核心是：

> 不要只問「畫面看起來像不像」，而要問「這段文字在內容中扮演什麼角色」。

## 整理說明

- 原始學習重點：補齊文本格式化章節以外的語意文本標籤。
- 本次整理方向：把引用文字、來源標題、縮寫、術語定義分開整理。
- 補強內容：加入選擇流程、常見誤用、可存取性提醒，以及 `cite` 屬性和 `<cite>` 元素的差異。
- 使用原則：先用正確 HTML 表達語意，再用 CSS 處理外觀。
- 待查證：本章整理 HTML 語意用法，具體引用格式仍可依寫作規範、公司規範或文件規範調整。

## 你會在這篇學到什麼

- `<blockquote>` 和 `<q>` 的差別。
- `cite` 屬性和 `<cite>` 元素的差別。
- `<cite>` 通常表示什麼，以及為什麼不建議拿來包普通作者姓名。
- `<abbr>` 如何搭配 `title` 補充縮寫全稱。
- `<dfn>` 如何標示正在被定義的術語。
- 這些標籤和可存取性的關係。
- 什麼情況不該使用這些標籤。

## 30 秒複習入口

- `<blockquote>`：表示一整段或多段區塊引用。
- `<q>`：表示句子中的行內短引用。
- `cite` 屬性：通常放引用來源 URL，不一定會顯示在畫面上。
- `<cite>` 元素：通常標示作品、文章、書籍、文件、節目、規格等來源標題。
- `<abbr>`：表示縮寫，可用 `title` 補充全稱。
- `<dfn>`：表示這裡正在定義一個術語。
- 這些標籤的核心是語意，不是外觀。

## 速查區

### 1. 標籤對照

| 標籤 | 核心語意 | 適合情境 | 不適合情境 |
|---|---|---|---|
| `<blockquote>` | 區塊引用 | 引用一整段文章、文件、規格內容 | 只是在句子中引用幾個字 |
| `<q>` | 行內短引用 | 句子裡引用一句短文字 | 包整段長引用 |
| `<cite>` | 作品或來源標題 | 書名、文章名、規格名、文件名 | 普通作者姓名、單純斜體樣式 |
| `<abbr>` | 縮寫 | HTML、CSS、API、HTTP | 不是縮寫的普通詞語 |
| `<dfn>` | 正在定義的術語 | 第一次定義概念或專有名詞 | 只是普通提到關鍵字 |

### 2. 選擇流程

```text
引用的是一整段或多段文字？
→ 用 <blockquote>

引用的是句子中的短文字？
→ 用 <q>

標示的是作品名稱、文章名稱、書名、規格名稱、文件名稱？
→ 用 <cite>

標示的是縮寫，而且需要補充全稱？
→ 用 <abbr>

這裡正在定義一個術語或概念？
→ 用 <dfn>

只是想改樣式，例如斜體、縮排、顏色？
→ 不要用這些語意標籤，改用 CSS
```

### 3. 容易混淆

| 寫法 | 注意 |
|---|---|
| `<blockquote cite="...">` | `cite` 屬性是來源 URL，不會直接顯示給讀者看 |
| `<cite>HTML Living Standard</cite>` | `<cite>` 是頁面上可見的作品或來源標題 |
| `<abbr title="HyperText Markup Language">HTML</abbr>` | `title` 可放縮寫全稱，但不要完全依賴它傳達重要資訊 |
| `<dfn>語意化 HTML</dfn>` | 表示目前正在定義「語意化 HTML」這個詞 |

## 正文筆記

### 1. 為什麼不用 `<span>` 就好？

從畫面來看，下面兩種寫法可能都能顯示文字：

```html
<span>HTML Living Standard</span>
```

```html
<cite>HTML Living Standard</cite>
```

但語意不同。

`<span>` 沒有告訴瀏覽器這段文字是什麼，只是一般行內容器。  
`<cite>` 則表示這段文字是作品或來源標題。

所以這一章的重點不是讓畫面變漂亮，而是讓 HTML 結構更精準。

可以把判斷方式記成一句話：

> 如果這段文字有明確的內容角色，就優先考慮是否有對應的語意標籤；如果只是改外觀，就交給 CSS。

---

### 2. `<blockquote>`：區塊引用

如果引用的是一整段或多段文字，通常使用 `<blockquote>`。

```html
<blockquote cite="https://html.spec.whatwg.org/">
  <p>HTML elements represent semantics, or meaning.</p>
</blockquote>
```

這裡有兩個重點：

- `<blockquote>` 表示裡面的內容是區塊引用。
- `cite` 屬性可以放來源 URL。

但要注意：`cite` 屬性通常不會直接顯示在畫面上。  
如果希望讀者看到來源，應該另外把來源寫出來。

比較完整的寫法：

```html
<figure>
  <blockquote cite="https://html.spec.whatwg.org/">
    <p>HTML elements represent semantics, or meaning.</p>
  </blockquote>

  <figcaption>
    來源：<cite>HTML Living Standard</cite>
  </figcaption>
</figure>
```

這種寫法表達得更完整：

- `<blockquote>`：引用內容。
- `cite="..."`：引用來源 URL。
- `<figcaption>`：引用說明。
- `<cite>`：來源作品名稱。

#### 常見誤用

不要只因為想讓文字縮排，就使用 `<blockquote>`。

不建議：

```html
<blockquote>
  這段只是想做縮排，並不是引用。
</blockquote>
```

較合適：

```html
<p class="indent">
  這段只是想做縮排，並不是引用。
</p>
```

如果只是視覺縮排，應該交給 CSS，而不是假裝成引用。

---

### 3. `<q>`：行內短引用

如果引用很短，而且放在句子中，可以使用 `<q>`。

```html
<p>
  老師說：<q>標籤的重點不是外觀，而是語意。</q>
</p>
```

`<q>` 的語意是：這一小段文字是行內引用。

瀏覽器通常會自動替 `<q>` 加上引號，但具體顯示可能依語言、瀏覽器和 CSS 設定不同。

#### `<q>` 和直接打引號差在哪？

直接打引號：

```html
<p>老師說：「標籤的重點不是外觀，而是語意。」</p>
```

這只是普通文字。

使用 `<q>`：

```html
<p>
  老師說：<q>標籤的重點不是外觀，而是語意。</q>
</p>
```

這會額外告訴 HTML：這段文字是引用。

#### 使用時機

適合：

```html
<p>
  文件中提到 <q>elements represent semantics</q>，這說明 HTML 標籤本身帶有意義。
</p>
```

不適合：

```html
<p>
  <q>
    這是一整段很長的引用文字，可能包含好幾句話，甚至多個段落。
    如果引用內容已經形成獨立區塊，就不適合用 q。
  </q>
</p>
```

長引用通常改用 `<blockquote>`。

---

### 4. `cite` 屬性和 `<cite>` 元素不同

這是本章最容易混淆的地方。

它們名字都叫 cite，但用途不同。

#### 4.1 `cite` 屬性：放來源 URL

```html
<blockquote cite="https://example.com/article">
  <p>這是一段被引用的文字。</p>
</blockquote>
```

這裡的 `cite` 是屬性，通常放來源 URL。

它的特點是：

- 它是資料，不一定顯示在畫面上。
- 它通常用在 `<blockquote>` 或 `<q>` 上。
- 它用來補充引用內容的來源位置。

#### 4.2 `<cite>` 元素：顯示作品或來源標題

```html
<p>
  這個觀點可參考 <cite>HTML Living Standard</cite>。
</p>
```

這裡的 `<cite>` 是元素，會顯示在畫面上。

它通常用來標示：

- 書名
- 文章標題
- 規格名稱
- 文件名稱
- 電影名稱
- 音樂作品
- 研究報告
- 節目名稱

#### 4.3 對照記憶

| 寫法 | 類型 | 主要用途 | 是否通常顯示 |
|---|---|---|---|
| `cite="https://example.com"` | 屬性 | 放來源 URL | 通常不顯示 |
| `<cite>HTML Living Standard</cite>` | 元素 | 標示作品或來源標題 | 會顯示 |

可以這樣記：

> `cite` 屬性是「來源位置」；`<cite>` 元素是「來源標題」。

---

### 5. `<cite>`：作品或來源標題

`<cite>` 通常用來標示作品名稱或來源標題。

例如：

```html
<p>
  這段說明整理自 <cite>HTML Living Standard</cite>。
</p>
```

這裡的 `<cite>` 表示 `HTML Living Standard` 是一份文件或規格名稱。

#### 適合使用 `<cite>` 的例子

```html
<p>
  我最近在讀 <cite>Clean Code</cite>。
</p>
```

```html
<p>
  這個用法可以參考 <cite>MDN Web Docs</cite> 的說明。
</p>
```

```html
<p>
  這段引用來自 <cite>HTML Living Standard</cite>。
</p>
```

#### `<cite>` 通常不是作者姓名

如果只是作者姓名，通常不一定要使用 `<cite>`。

不建議把普通作者姓名當成作品名：

```html
<p>
  作者：<cite>小明</cite>
</p>
```

較合適：

```html
<p>作者：小明</p>
```

更完整的引用說明可以寫成：

```html
<p>
  來源：小明，<cite>HTML 語意化入門</cite>
</p>
```

這裡：

- `小明` 是作者。
- `<cite>HTML 語意化入門</cite>` 是作品標題。

---

### 6. `<abbr>`：縮寫

`<abbr>` 用來標示縮寫。

```html
<p>
  <abbr title="HyperText Markup Language">HTML</abbr> 是用來描述網頁結構的標記語言。
</p>
```

這裡同時表達兩件事：

- `HTML` 是縮寫。
- `title` 補充它的完整名稱：`HyperText Markup Language`。

#### 使用時機

適合：

```html
<p>
  <abbr title="Application Programming Interface">API</abbr> 是系統之間溝通的介面。
</p>
```

不一定需要：

```html
<p>
  HTML 是用來描述網頁結構的標記語言，HTML 的全稱是 HyperText Markup Language。
</p>
```

如果正文已經把全稱說清楚，就不一定每次都要加 `title`。

#### 不要過度使用 `<abbr>`

不建議每一次出現縮寫都包 `<abbr>`：

```html
<p>
  <abbr title="HyperText Markup Language">HTML</abbr> 是標記語言。
  <abbr title="HyperText Markup Language">HTML</abbr> 負責描述結構。
  <abbr title="HyperText Markup Language">HTML</abbr> 不是程式語言。
</p>
```

通常第一次出現，或讀者可能不熟悉時標示即可。

#### 可存取性提醒

`title` 可以補充縮寫全稱，但不要完全依賴 `title` 傳達重要資訊。

原因是：

- 手機使用者不一定能像滑鼠一樣 hover。
- 鍵盤操作不一定容易觸發提示。
- 不同瀏覽器和輔助技術對 `title` 的支援體驗不完全一致。

因此，如果全稱很重要，建議在正文中也說明一次。

例如：

```html
<p>
  <abbr title="HyperText Markup Language">HTML</abbr>
  ，也就是 HyperText Markup Language，是用來描述網頁結構的標記語言。
</p>
```

---

### 7. `<dfn>`：正在定義術語

`<dfn>` 表示這裡正在定義一個術語。

```html
<p>
  <dfn>語意化 HTML</dfn> 是指使用符合內容意義的 HTML 標籤來描述頁面結構。
</p>
```

這裡的語意是：

> 「語意化 HTML」這個詞正在這裡被定義。

#### 適合使用 `<dfn>` 的情境

適合用在：

- 教材第一次定義概念。
- 技術文件第一次解釋術語。
- 字彙表、名詞解釋頁面。
- 規格或文件中正式定義某個詞。

例如：

```html
<p>
  <dfn>閉包</dfn> 是指函式可以記住並存取其詞法作用域的能力。
</p>
```

#### 不是每次看到專有名詞都要用 `<dfn>`

不建議：

```html
<p>
  我今天複習了 <dfn>HTML</dfn>、<dfn>CSS</dfn> 和 <dfn>JavaScript</dfn>。
</p>
```

這裡只是普通提到這些詞，並不是正在定義它們。

較合適：

```html
<p>
  我今天複習了 HTML、CSS 和 JavaScript。
</p>
```

---

### 8. `<abbr>` 和 `<dfn>` 可以搭配嗎？

可以，但要有明確語意，不要為了形式而過度包標籤。

例如第一次定義 HTML：

```html
<p>
  <dfn><abbr title="HyperText Markup Language">HTML</abbr></dfn>
  是一種用來描述網頁結構的標記語言。
</p>
```

這裡同時表達：

- `<abbr>`：`HTML` 是縮寫。
- `title`：補充完整名稱。
- `<dfn>`：這裡正在定義 `HTML` 這個術語。

但如果只是普通提到 HTML，就不需要這麼包：

```html
<p>
  HTML 常和 CSS、JavaScript 一起使用。
</p>
```

判斷標準是：

> 標籤不是越多越好，而是語意越準越好。

---

### 9. 這些標籤和 ARIA 的關係

`<blockquote>`、`<q>`、`<cite>`、`<abbr>`、`<dfn>` 都是 HTML 原生語意標籤。

ARIA 主要用來補充互動元件、複雜 UI 或無法用原生 HTML 表達的可存取性語意。

所以在這一章的情境中，通常不需要先想到 ARIA。

比較好的原則是：

```text
能用正確 HTML 表達 → 優先用 HTML
HTML 語意不足以表達複雜互動 → 再考慮 ARIA
```

例如，要表示一段區塊引用，應該直接用：

```html
<blockquote>
  <p>這是一段引用。</p>
</blockquote>
```

而不是用普通 `<div>` 再硬加 ARIA：

```html
<div role="blockquote">
  <p>這是一段引用。</p>
</div>
```

上面這種寫法不建議，因為原生 HTML 已經有更適合的 `<blockquote>`。

可以記成一句話：

> 先寫對 HTML，再思考是否需要 ARIA。

---

### 10. 什麼時候不用這些標籤？

如果只是想改樣式，不需要使用這些語意標籤。

例如只是讓某段文字斜體：

```html
<span class="italic">這段文字只是樣式效果</span>
```

如果只是想加引號，也不一定代表它是引用：

```html
<p>這個按鈕名稱叫做「送出」。</p>
```

這裡的「送出」是 UI 文字名稱，不一定是引用內容，所以不一定要使用 `<q>`。

可以用下面的表格判斷：

| 想做的事 | 是否適合使用本章標籤 |
|---|---|
| 引用一整段文字 | 適合 `<blockquote>` |
| 在句子中引用一句短話 | 適合 `<q>` |
| 標示作品或來源標題 | 適合 `<cite>` |
| 標示縮寫 | 適合 `<abbr>` |
| 定義術語 | 適合 `<dfn>` |
| 只是斜體 | 不適合，改用 CSS |
| 只是縮排 | 不適合，改用 CSS |
| 只是加顏色 | 不適合，改用 CSS |
| 只是普通提到一個詞 | 不一定需要語意標籤 |

## 常見問法

### `<blockquote>` 會自動顯示來源嗎？

通常不會。

`cite` 屬性可以放來源 URL，但如果要讓讀者看到來源，應該另外用文字、連結或 `<cite>` 顯示。

---

### `<q>` 和直接打引號有什麼不同？

直接打引號只是文字。  
`<q>` 則表示這段內容是行內引用，語意更清楚。

如果只是 UI 名稱、按鈕名稱或強調某個字，不一定要用 `<q>`。

---

### `cite` 屬性和 `<cite>` 元素差在哪？

`cite` 屬性通常放來源 URL，而且通常不會直接顯示。

```html
<blockquote cite="https://example.com/article">
  <p>引用內容</p>
</blockquote>
```

`<cite>` 元素則是頁面上可見的來源標題或作品名稱。

```html
<p>來源：<cite>某篇文章標題</cite></p>
```

---

### `<cite>` 可以包作者姓名嗎？

通常不建議把普通作者姓名當作 `<cite>` 的主要用途。

`<cite>` 更常用來表示作品或來源標題，例如書名、文章名、規格名、文件名。

如果同時有作者和作品，可以這樣寫：

```html
<p>
  來源：小明，<cite>HTML 語意化入門</cite>
</p>
```

---

### `<abbr>` 一定要寫 `title` 嗎？

不一定。

常見寫法會搭配 `title` 補全稱：

```html
<abbr title="Application Programming Interface">API</abbr>
```

但如果上下文已經說明全稱，或頁面有其他清楚的解釋方式，也可以視情況處理。

需要注意的是，不要把重要資訊只放在 `title` 裡，因為不是所有使用情境都容易取得 `title` 提示。

---

### `<dfn>` 可以包所有關鍵字嗎？

不建議。

`<dfn>` 的重點是「這裡正在定義這個術語」。

如果只是普通提到某個詞，不需要使用 `<dfn>`。

---

### 這些標籤會自動改善畫面嗎？

有些瀏覽器可能會給預設樣式，例如縮排、引號或斜體，但這不是使用它們的主要原因。

這些標籤的主要價值是語意。  
外觀應該交給 CSS 控制。

## 最終記憶版

可以把本章濃縮成下面幾句：

```text
<blockquote>：整段引用。
<q>：行內短引用。
cite 屬性：來源 URL，通常不顯示。
<cite>：作品或來源標題，會顯示。
<abbr>：縮寫，可用 title 補全稱。
<dfn>：這裡正在定義術語。

不要為了外觀使用語意標籤。
先寫對 HTML，再用 CSS 控制樣式。
能用原生 HTML 表達，就不要先用 ARIA 硬補。
```
