# 引用與註記文本標籤

> - 所屬章節：第三十九章｜引用與註記文本標籤  
> - 關鍵字：`blockquote`、`q`、`cite`、`abbr`、`dfn`、引用、縮寫、定義、作品名、術語  
> - 建議回查情境：想標示引用文字、作品來源、縮寫全稱、正在定義的術語，或想分清 `<blockquote>` / `<q>` / `<cite>` 時

## 本節導讀

文章、教材和技術文件裡經常會出現引用、來源、縮寫和術語定義。

這些內容如果只用普通 `<span>` 或純文字，也能顯示出來，但語意不夠清楚。  
HTML 提供了一組文字語意標籤，讓瀏覽器、搜尋工具、輔助技術和維護者更容易理解這些文字的角色。

## 整理說明

- 原始學習重點：補齊文本格式化章節以外的語意文本。
- 本次整理方向：把引用、來源名稱、縮寫和定義術語分開整理。
- 補強內容：加入 `<blockquote cite>`、`<q>`、`<cite>` 的常見誤解。
- 待查證：本章整理 HTML 語意用法，具體引用格式仍可依寫作規範調整。

## 你會在這篇學到什麼

- `<blockquote>` 和 `<q>` 的差別
- `cite` 屬性和 `<cite>` 元素的差別
- `<cite>` 通常表示什麼
- `<abbr>` 如何搭配 `title`
- `<dfn>` 如何標示被定義的術語
- 什麼情況不用這些標籤

## 30 秒複習入口

- `<blockquote>` 表示區塊引用。
- `<q>` 表示行內短引用。
- `cite` 屬性可以放來源 URL。
- `<cite>` 通常標示作品、文章、書籍、文件、節目等來源標題。
- `<abbr>` 表示縮寫，可用 `title` 補全稱。
- `<dfn>` 表示這裡正在定義一個術語。
- 這些標籤的核心是語意，不是外觀。

## 速查區

### 標籤對照

| 標籤 | 核心語意 | 例子 |
|---|---|---|
| `<blockquote>` | 區塊引用 | 引用一段文章或規格內容 |
| `<q>` | 行內短引用 | 句子裡引用一句話 |
| `<cite>` | 作品或來源標題 | 書名、文章名、規格名 |
| `<abbr>` | 縮寫 | HTML、CSS、API |
| `<dfn>` | 被定義的術語 | 第一次定義某個概念 |

### 容易混淆

| 寫法 | 注意 |
|---|---|
| `<blockquote cite="...">` | `cite` 屬性是來源 URL，不會直接顯示 |
| `<cite>HTML Standard</cite>` | `<cite>` 是可見的來源標題 |
| `<abbr title="HyperText Markup Language">HTML</abbr>` | `title` 放全稱 |
| `<dfn>語意化 HTML</dfn>` | 表示目前正在定義這個詞 |

## 正文筆記

### 1. `<blockquote>`：區塊引用

如果引用的是一整段或多段文字，通常用 `<blockquote>`。

```html
<blockquote cite="https://html.spec.whatwg.org/">
  <p>HTML elements represent semantics, or meaning.</p>
</blockquote>
```

這裡的 `cite` 屬性可以放來源 URL。  
但它通常不會直接顯示在畫面上，所以如果希望讀者看到來源，還要額外寫出來。

```html
<figure>
  <blockquote cite="https://html.spec.whatwg.org/">
    <p>HTML elements represent semantics, or meaning.</p>
  </blockquote>
  <figcaption>
    來源：<cite>HTML Living Standard</cite>
  </figcaption>
</figure>
```

這種寫法讓引用文字和來源說明形成一組，也可以搭配前一章的 `<figure>`。

### 2. `<q>`：行內短引用

如果引用很短，並且放在句子中，可以用 `<q>`。

```html
<p>
  老師說：<q>標籤的重點不是外觀，而是語意。</q>
</p>
```

瀏覽器通常會自動替 `<q>` 加上引號，但具體顯示可能依語言和瀏覽器不同。

`<q>` 適合短引用，不適合包整段長引用。  
長引用通常使用 `<blockquote>`。

### 3. `cite` 屬性和 `<cite>` 元素不同

這是很容易混淆的一組。

`cite` 屬性：

```html
<blockquote cite="https://example.com/article">
  <p>這是一段被引用的文字。</p>
</blockquote>
```

它是資料，不一定顯示。

`<cite>` 元素：

```html
<p>這個觀點可參考 <cite>HTML Living Standard</cite>。</p>
```

它是頁面中的可見文字，通常表示作品或來源標題。

### 4. `<cite>` 通常不是作者姓名

`<cite>` 常用來標示作品名稱，例如：

- 書名
- 文章標題
- 規格名稱
- 電影名稱
- 音樂作品
- 研究報告

例如：

```html
<p>
  這段說明整理自 <cite>HTML Living Standard</cite>。
</p>
```

如果只是作者姓名，通常不一定需要 `<cite>`：

```html
<p>作者：小明</p>
```

### 5. `<abbr>`：縮寫

`<abbr>` 用來標示縮寫。

```html
<p>
  <abbr title="HyperText Markup Language">HTML</abbr> 是用來描述網頁結構的標記語言。
</p>
```

`title` 裡可以放縮寫全稱。  
使用者滑鼠停留時，某些瀏覽器可能顯示提示。

注意：不要為每一次出現的縮寫都塞大量 `title`。  
通常第一次出現或需要說明時標示即可。

### 6. `<dfn>`：正在定義術語

`<dfn>` 表示這裡正在定義一個術語。

```html
<p>
  <dfn>語意化 HTML</dfn> 是指使用符合內容意義的 HTML 標籤來描述頁面結構。
</p>
```

它適合用在教材、字彙表、技術文件第一次定義某個詞的地方。

如果只是普通提到一個詞，不一定要用 `<dfn>`。

### 7. `<abbr>` 和 `<dfn>` 可以搭配嗎？

可以。

例如第一次定義 HTML：

```html
<p>
  <dfn><abbr title="HyperText Markup Language">HTML</abbr></dfn>
  是一種用來描述網頁結構的標記語言。
</p>
```

這裡同時表達：

- `HTML` 是縮寫。
- 這裡正在定義 `HTML` 這個術語。

但不要為了形式而過度包標籤。  
語意清楚比標籤越多越重要。

### 8. 什麼時候不用這些標籤？

如果只是想改樣式，不需要用這些語意標籤。

例如只是讓某段文字斜體：

```html
<span class="italic">這段文字只是樣式效果</span>
```

如果不是引用，就不要用 `<q>` 或 `<blockquote>`。  
如果不是作品名稱，就不要用 `<cite>`。  
如果不是縮寫，就不要用 `<abbr>`。

## 常見問法

### `<blockquote>` 會自動顯示來源嗎？

通常不會。

`cite` 屬性可以放來源 URL，但如果要讓讀者看到來源，應該另外用文字、連結或 `<cite>` 顯示。

### `<q>` 和直接打引號有什麼不同？

直接打引號只是文字。  
`<q>` 表示這段內容是行內引用，語意更清楚。

### `<cite>` 可以包作者姓名嗎？

通常不建議把普通作者姓名當作 `<cite>` 的主要用途。  
它更常用來表示作品或來源標題。

### `<abbr>` 一定要寫 `title` 嗎？

不一定，但常見用法會搭配 `title` 補全稱。  
如果上下文已經清楚，或頁面有其他解釋方式，也可以視情況處理。
