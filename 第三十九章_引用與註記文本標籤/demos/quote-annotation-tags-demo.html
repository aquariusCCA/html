HTML Notes / 引用與註記文本標籤
<blockquote> <q> <cite> <abbr> <dfn>

引用與註記文本標籤 Demo

這個範例把第 39 章筆記中的區塊引用、行內短引用、來源標題、縮寫與術語定義放在同一頁觀察。重點不是預設樣式，而是每段文字在內容中扮演的角色。

<blockquote> 整段或多段引用內容。
<q> 句子中的短引用。
<cite> 作品、文章、書籍、文件或規格名稱。
<abbr> 縮寫，通常搭配 title 補全稱。
<dfn> 正在被定義的術語。

先判斷角色

這段文字是引用、來源名稱、縮寫，還是正在定義的術語？先回答這個問題，再選標籤。

來源分兩種

cite 屬性通常放來源 URL；<cite> 元素放畫面上可見的作品或文件標題。

樣式交給 CSS

如果目的只是變斜體、縮排、加引號或改顏色，應使用 CSS，而不是濫用語意標籤。

1. 快速選擇器

這個選擇器對應筆記中的「快速選擇流程」：用語意標籤表達文字角色，用 CSS 處理外觀。

2. 區塊引用：<blockquote>

<figure>
  <blockquote cite="https://html.spec.whatwg.org/">
    <p>HTML elements represent semantics, or meaning.</p>
  </blockquote>
  <figcaption>
    來源：<cite>HTML Living Standard</cite>
  </figcaption>
</figure>

不要拿來做縮排

如果一段文字不是引用，只是想讓它縮排，應使用一般段落加 CSS。

<p class="indent">
  這段是自己的說明，不是外部引用。
</p>

3. 行內短引用：<q>

適合短引用

筆記提醒我們：不要只問畫面像不像，而要問 這段文字在內容中扮演什麼角色。 

<p>
  不要只問畫面像不像，而要問
  <q>這段文字在內容中扮演什麼角色</q>。
</p>

不適合整段長引用

如果引用內容已經形成一段或多段文字，改用 <blockquote>，讓文件結構更清楚。

<blockquote>
  <p>這裡是一整段被引用的文字。</p>
</blockquote>

4. cite 屬性 vs <cite> 元素

寫法 用途 是否通常顯示在畫面上
cite="https://example.com/article" 放引用來源 URL，常見於 <blockquote> 或 <q>。 不會自動顯示。
<cite>HTML Living Standard</cite> 標示作品、文章、書籍、文件、規格等可見來源標題。 會顯示為頁面內容。

5. 縮寫與術語：<abbr>、<dfn>

縮寫補全稱

HTML 是 HyperText Markup Language 的縮寫，用來描述網頁內容的結構與語意。 

<abbr title="HyperText Markup Language">HTML</abbr>
是 HyperText Markup Language 的縮寫。

正在定義術語

語意化 HTML 是指使用符合內容角色的 HTML 標籤，讓文件更容易被瀏覽器、搜尋工具與輔助技術理解。 

<dfn>語意化 HTML</dfn>
是指使用符合內容角色的 HTML 標籤。

6. <abbr> 與 <dfn> 可以一起用

第一次定義縮寫

API 是應用程式之間交換資料或功能的介面。 

<dfn>
  <abbr title="Application Programming Interface">API</abbr>
</dfn>
是應用程式之間交換資料或功能的介面。

注意 title 的限制

重要全稱不要只藏在 title 裡。滑鼠提示、觸控裝置與部分輔助技術情境不一定穩定，正文中也可以直接補全稱。

7. 常見誤用修正

作者姓名不是普通 <cite> 情境

<p>作者：Ada Lovelace</p>

作品或文件標題才適合用 <cite>。

不要用 <abbr> 包普通詞

<p>這是一份重要文件。</p>

<abbr> 適合 HTML、CSS、API 這類縮寫。

不要用 ARIA 取代原生語意

<blockquote>
  <p>引用內容</p>
</blockquote>

原生 HTML 已能表達引用時，通常不需要先寫 div role。

8. 回查清單

  • 整段引用用 <blockquote>，句中短引用用 <q>。
  • cite 屬性放來源 URL；<cite> 元素放可見的作品或來源標題。
  • <abbr> 可補縮寫全稱，但不要完全依賴 title 傳達重要資訊。
  • <dfn> 用在正在定義術語的位置，不是每次提到關鍵字都要包。
  • 只是想改外觀時，使用 CSS。
