# 第三十九章｜引用與註記文本標籤

> 章節定位：理解引用、作品來源、縮寫、定義術語等文字如何用更精確的 HTML 語意表達  
> 建議閱讀方式：先分清「引用內容」和「來源名稱」，再理解縮寫、定義術語與可存取性注意事項

## 本章在學什麼

這一章整理偏「文字語意補充」的 HTML 標籤：

- `<blockquote>`：區塊引用。
- `<q>`：行內短引用。
- `<cite>`：作品或來源標題。
- `<abbr>`：縮寫。
- `<dfn>`：正在被定義的術語。

這些標籤的重點不是把文字變成某種樣式，而是讓 HTML 更清楚地表達「這段文字的角色」。

也就是說：

- 這段文字是引用內容嗎？
- 這段文字是作品名稱嗎？
- 這段文字是縮寫嗎？
- 這段文字是在定義一個術語嗎？

如果只是想改變外觀，應該使用 CSS，而不是濫用語意標籤。

## 快速選擇流程

當你不知道該用哪個標籤時，可以先照這個流程判斷：

```text
如果是整段或多段引用
→ 使用 <blockquote>

如果是句子中的短引用
→ 使用 <q>

如果是作品名稱、文章名稱、書名、規格名稱、電影名稱、研究報告名稱
→ 使用 <cite>

如果是縮寫，例如 HTML、CSS、API
→ 使用 <abbr>

如果這裡正在定義一個術語
→ 使用 <dfn>

如果只是想讓文字變斜體、加引號、改顏色
→ 不要使用這些語意標籤，改用 CSS
```

## 適合快速回查的主題

- `<blockquote>` 和 `<q>` 差在哪
- `cite` 屬性和 `<cite>` 元素差在哪
- `<cite>` 是作品名、來源名，還是作者名
- `<abbr title>` 怎麼表示縮寫全稱
- `<abbr title>` 有哪些可存取性限制
- `<dfn>` 如何標示正在定義的術語
- `<abbr>` 和 `<dfn>` 能不能一起使用
- ARIA 是否可以取代這些原生語意標籤
- 引用來源 URL 該放在哪裡
- 只是想改樣式時，為什麼不該使用這些標籤

## 易混淆主題

| 主題 | 容易誤解 | 正確理解 |
|---|---|---|
| `<blockquote>` vs `<q>` | 都是引用，所以可以混用 | `<blockquote>` 是區塊引用，`<q>` 是行內短引用 |
| `cite` 屬性 vs `<cite>` 元素 | 名字一樣，所以用途一樣 | `cite` 屬性放來源 URL；`<cite>` 元素標示可見的作品或來源標題 |
| `<cite>` vs 作者姓名 | 作者姓名應該用 `<cite>` | `<cite>` 通常標示作品名稱，不是普通作者姓名 |
| `<abbr>` vs `title` | 只要加 `title` 就完整支援所有使用者 | `title` 可補全稱，但不能完全依賴，行動裝置與輔助技術情境可能不穩定 |
| `<dfn>` vs 普通術語 | 只要是專有名詞都要用 `<dfn>` | `<dfn>` 適合用在「正在定義」某個術語的位置 |
| 原生語意 vs ARIA | 可以用 ARIA 取代所有語意標籤 | 原生 HTML 語意優先；ARIA 通常用來補充，不應濫用來取代原生標籤 |

## 本章核心觀念

### 1. 語意優先，不是樣式優先

這些標籤不是為了讓文字長得不一樣，而是為了讓文件結構更清楚。

例如：

```html
<cite>HTML Living Standard</cite>
```

這不是單純讓文字變斜體，而是表示這段文字是作品或文件名稱。

### 2. 看得見的來源和看不見的來源不同

`cite` 屬性可以放來源 URL，但通常不會直接顯示在畫面上。

```html
<blockquote cite="https://html.spec.whatwg.org/">
  <p>HTML elements represent semantics, or meaning.</p>
</blockquote>
```

如果希望讀者看到來源，應該另外用文字、連結或 `<cite>` 顯示。

### 3. 不要過度依賴 `title`

`<abbr title="...">` 可以補充縮寫全稱，但 `title` 的提示效果不一定適合所有裝置與操作方式。

因此重要資訊不要只放在 `title` 裡，必要時也可以在正文中直接寫出全稱。

### 4. ARIA 不應取代原生 HTML 語意

如果 HTML 已經有合適的原生標籤，通常優先使用原生標籤。

例如引用文字應先考慮 `<blockquote>` 或 `<q>`，而不是先用 `<span>` 再補 ARIA。

## 最終記憶版

- 整段引用：用 `<blockquote>`。
- 句中短引用：用 `<q>`。
- 作品或來源標題：用 `<cite>`。
- 縮寫：用 `<abbr>`。
- 正在定義的術語：用 `<dfn>`。
- `cite` 屬性是來源 URL，不是畫面上可見的標題。
- `<cite>` 是可見的作品或來源名稱。
- `title` 可以補充資訊，但不要完全依賴它。
- 原生 HTML 語意優先，ARIA 用來補充，不是拿來取代原生標籤。
- 如果只是想改樣式，請用 CSS。
