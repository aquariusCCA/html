# 禁用表單控件

> 所屬章節：第二十四章｜表單標籤  
> 關鍵字：disabled、`disabled`、禁用表單控件、不可編輯、不可點擊、fieldset、`fieldset disabled`、表單提交  
> 建議回查情境：想知道怎麼讓表單控件不可操作、想分清禁用單一控件與禁用一整組控件的差別、想確認 `disabled` 對表單提交有什麼影響

## 本節導讀

這篇整理表單控件的禁用方式，也就是 `disabled` 屬性。  
當某個欄位暫時不該輸入、某個按鈕暫時不該點擊，或一整組表單控件需要鎖住時，就會用到這個屬性。

這個主題最容易出錯的地方，是誤以為可以直接在 `<form>` 上加 `disabled` 來禁用整個表單。  
更穩定的理解是：`disabled` 可直接加在多種表單控件上；若要一次禁用一整組控件，常見做法是用 `<fieldset disabled>` 包住它們。

## 你會在這篇學到什麼

- `disabled` 在表單裡代表什麼
- 哪些表單控件可以直接使用 `disabled`
- 如何禁用單一控件
- 如何更穩定地禁用一整組表單控件
- 為什麼被禁用的控件不會進入提交資料

## 30 秒複習入口

- `disabled` 會讓表單控件變成不可操作狀態。
- `input`、`textarea`、`button`、`select`、`option` 等都可使用 `disabled`。
- 被禁用的控件通常不能輸入、不能點擊、不能選擇。
- 被禁用的控件不會包含在表單提交資料中。
- 若要一次禁用一整組控件，常見做法是用 `<fieldset disabled>`，不是直接對 `<form>` 寫 `disabled`。

## 速查區

### 核心概念

- `disabled` 是用來讓表單控件暫時失效的屬性。
- 控件被禁用後，使用者通常無法與它互動。
- 這不只是視覺上的灰化，而是行為上也會失效。

### 關鍵規則 / 判準

- 可直接套用在多種表單控件上，例如 `input`、`textarea`、`button`、`select`、`option`。
- 被禁用的控件通常不會參與表單提交。
- 若要禁用一整組控件，常見做法是用 `<fieldset disabled>` 包住它們。
- 不要把「禁用整組控件」誤寫成「在 `<form>` 上加 `disabled`」。

### 常見使用場景

- 欄位尚未開放編輯
- 條件未達成前禁止送出
- 依照其他選項動態鎖定欄位
- 暫時關閉某些操作
- 分步驟表單中的未啟用區塊

### 常見混淆點

- `disabled` 不是單純外觀灰掉，而是真的不能互動。
- 被禁用的欄位通常不會一起提交。
- `<form disabled>` 不是常見且可靠的整表單禁用寫法。
- 若只是不能改但仍想提交資料，思路通常不等於直接用 `disabled`。

### 一句話抓核心

- `disabled` 用來讓表單控件暫時失效；要禁用整組控件時，常見做法是用 `<fieldset disabled>`。

## 正文筆記

- 給表單控件加上 `disabled`，即可禁用該控件。
- `input`、`textarea`、`button`、`select`、`option` 等都可以使用 `disabled`。

### 1. 禁用單個表單控件

**輸入框（`input`）：**

```html
<input type="text" value="無法輸入" disabled />
```

**按鈕（`button`）：**

```html
<button disabled>無法點擊的按鈕</button>
```

**下拉選單（`select`）：**

```html
<select disabled>
  <option>選項 1</option>
  <option>選項 2</option>
</select>
```

**多行文本框（`textarea`）：**

```html
<textarea disabled>無法編輯</textarea>
```

**單選按鈕（`radio`）與複選框（`checkbox`）：**

```html
<input type="radio" disabled /> 單選按鈕
<input type="checkbox" disabled /> 複選框
```

### 2. 禁用一整組表單控件

若想一次禁用一整組相關控件，常見做法是使用 `<fieldset disabled>`：

```html
<form>
  <fieldset disabled>
    <input type="text" value="這裡無法輸入" />
    <button>這裡無法點擊</button>
    <textarea>這裡無法編輯</textarea>
  </fieldset>
</form>
```

這樣可以把一組表單元素一起設成不可操作，比逐個加 `disabled` 更方便。

## 注意事項

- 被禁用的控件不會包含在表單提交資料中。
- 瀏覽器通常會以灰化等方式提示該控件目前不可用。
- 若需求是「不能改，但資料仍要送出」，就要特別注意不能只憑 `disabled` 來理解。

## 常見問法

### `disabled` 會帶來什麼效果？

- 控件通常會變成不可輸入、不可點擊或不可選擇。
- 同時也通常不會被包含在提交資料中。

### 哪些表單元素可以寫 `disabled`？

- 常見的有 `input`、`textarea`、`button`、`select`、`option` 等。

### 可以直接用 `<form disabled>` 禁用整個表單嗎？

- 不建議把這件事當成穩定規則來記。
- 若要禁用一整組控件，常見且更可靠的做法是使用 `<fieldset disabled>`。

### 被禁用的欄位會一起提交嗎？

- 一般不會。
- 這是 `disabled` 很重要的影響之一。
