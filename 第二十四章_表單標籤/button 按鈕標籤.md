# button 按鈕標籤

> - 所屬章節：第二十四章｜表單標籤  
> - 關鍵字：button、`button`、submit、reset、button type、提交按鈕、重置按鈕、普通按鈕、預設型別  
> - 建議回查情境：想知道 `<button>` 可以有哪些類型、想分清 `submit`、`reset`、`button` 的差別、想確認為什麼不寫 `type` 可能會意外送出表單

## 本節導讀

前面已經認識了各種表單控件，也知道有些元素是拿來輸入資料，有些是拿來選擇資料。  
而 `<button>` 則是表單裡非常常見的「操作型控件」。

但 `<button>` 這個主題最容易讓初學者搞混的地方，不是它長得像按鈕，而是：

**同樣都是按鈕，行為卻會因為 `type` 不同而完全不一樣。**

你可能只是想放一顆普通互動按鈕，  
結果它卻把表單送出了。  
也可能以為重置按鈕是把所有內容清空，  
但它其實是把表單恢復到初始狀態。

所以學這篇時，先抓住四件事就好：

1. `<button>` 常見有哪三種 `type`
2. `submit`、`reset`、`button` 各自在做什麼
3. 為什麼不寫 `type` 容易出錯
4. 什麼時候應該明確寫出 `type="button"`

只要這幾個觀念先站穩，後面寫表單時，就不會再把「普通按鈕」和「預設提交按鈕」混成同一件事。

## 你會在這篇學到什麼

## 30 秒複習入口

- `<button>` 常見有 `submit`、`reset`、`button` 三種型別。
- `submit` 用來提交表單。
- `reset` 用來把表單恢復成初始狀態。
- `button` 是普通按鈕，本身不會自動提交表單。
- 在表單裡若省略 `type`，`<button>` 預設通常是 `submit`。
- 只要不是要提交表單，通常都建議把 `type` 寫清楚。

## 速查區

### 核心概念

- `<button>` 是可點擊的按鈕元素。
- 它的行為主要由 `type` 決定。
- 同樣都是按鈕，提交、重置、一般互動按鈕的用途並不一樣。

### 關鍵規則 / 判準

- `type="submit"`：提交表單。
- `type="reset"`：把表單恢復到初始狀態。
- `type="button"`：一般按鈕，不自動提交表單。
- 在表單中若不寫 `type`，`<button>` 預設通常會被當成 `submit`。

### 常見使用場景

- 表單送出
- 表單重填
- 彈出視窗
- 執行 JavaScript 互動
- 驗證碼、展開收合、切換內容

### 常見混淆點

- `button` 是一種型別名稱，不是 `<button>` 的預設安全行為。
- `reset` 不是單純把欄位清空，而是恢復到初始值。
- 在表單裡省略 `type`，常會讓本來只是想做互動的按鈕誤送出表單。
- `<button>` 看起來只是按鈕，但它在表單裡其實可能直接參與提交行為。

### 一句話抓核心

- **`<button>` 的重點不是有沒有按鈕，而是 `type` 決定它到底是提交、重置，還是單純互動按鈕。**

## 正文筆記

## 1. `<button>` 在做什麼？

`<button>` 是 HTML 中的按鈕元素。

它可以出現在很多互動情境裡，例如：

- 送出表單
- 重置表單
- 開啟彈窗
- 觸發 JavaScript
- 切換畫面內容

所以 `<button>` 不是只有一種固定用途。  
真正決定它在做什麼的關鍵，是 `type`。

也就是說，學 `<button>` 時不要只看它是不是按鈕，  
而要先問自己：

**這顆按鈕是要提交？重置？還是只是一般互動？**

---

## 2. `<button>` 常見有哪三種 `type`？

`<button>` 最常見的三種型別是：

- `submit`
- `reset`
- `button`

```html
<button type="submit">提交按鈕</button>
<button type="reset">重置按鈕</button>
<button type="button">普通按鈕</button>
<button>未指定 type 的按鈕</button>
```

這四行範例裡，前三顆是明確指定型別的按鈕，
最後一顆則是故意不寫 `type`，拿來觀察預設行為。

真正要記住的不是英文名稱本身，
而是這三種型別分別對應三種不同用途。

---

## 3. `submit`：提交按鈕

```html
<button type="submit">送出表單</button>
```

`submit` 的作用是：**提交表單資料。**

當按鈕放在表單裡時，點擊後通常會觸發表單送出。
所以如果這顆按鈕的目的，就是把使用者填好的資料送出去，通常就會使用 `submit`。

### 怎麼理解？

* 它是「送出」用途的按鈕
* 常見於登入、註冊、查詢、提交表單等情境
* 它不是一般互動按鈕，而是直接參與表單提交流程的按鈕

---

## 4. `reset`：重置按鈕

```html
<button type="reset">重新填寫</button>
```

`reset` 的作用是：**把表單恢復成初始狀態。**

這裡最容易誤會的地方是，很多人會把它記成「清空表單」。
但更精確的理解不是這樣。

它真正做的是：

* 讓表單欄位回到頁面一開始載入時的值
* 而不是保證把所有欄位都變成空白

所以如果欄位原本就有預設值，
按下 `reset` 後，通常會回到那個預設值，而不是全部清空。

### 怎麼理解？

* 它是在做「恢復初始狀態」
* 不是單純做「一鍵清空」
* 這是學 `reset` 時最重要的一個觀念

---

## 5. `button`：普通按鈕

```html
<button type="button">取得驗證碼</button>
```

`button` 是一般互動按鈕。

它本身不會自動提交表單，
通常是搭配 JavaScript 來做其他互動行為，例如：

* 取得驗證碼
* 展開 / 收合內容
* 開啟提示視窗
* 切換區塊
* 執行某段腳本

### 怎麼理解？

* 它是「互動用途」的按鈕
* 它不是提交按鈕
* 它的價值在於：按下去可以做其他操作，但不會自動送表單

---

## 6. 為什麼要特別注意「預設型別」？

這是 `<button>` 最重要、也最容易踩雷的地方。

如果 `<button>` 放在表單裡，但你**沒有寫 `type`**，
它的預設行為通常會被當成 `submit`。

例如：

```html
<form action="">
  <input type="text" name="username" />
  <button>只是想切換內容</button>
</form>
```

很多初學者看到這段時，會直覺以為這只是一顆普通按鈕。
但實際上，它在表單中通常會被當成提交按鈕。

也就是說，
你原本可能只是想做：

* 切換畫面
* 跳提示
* 執行腳本
* 開彈窗

結果卻因為沒寫 `type="button"`，
按下去直接把表單送出了。

這就是為什麼 `<button>` 的重點不是只有按鈕本身，
而是一定要注意它的型別。

---

## 7. 什麼時候要明確寫 `type="button"`？

只要這顆按鈕**不是拿來送出表單**，通常就建議明確寫上：

```html
<button type="button">開啟說明</button>
```

這樣做的目的，是避免瀏覽器把它當成 `submit`，造成誤提交。

所以可以直接記一個很實用的判斷方式：

* 要送表單 → `type="submit"`
* 要重置表單 → `type="reset"`
* 只是做一般互動 → `type="button"`

這樣在寫表單時，會穩很多。

---

## 8. `<button>` 和 `<input type="button">` 要先怎麼理解？

在前面〈表單控件〉中，你已經看過：

* `<input type="submit">`
* `<input type="reset">`
* `<input type="button">`

而這一篇則是把 `<button>` 單獨拉出來講。
初學時先知道：

* 它們都可以表現出按鈕用途
* 但 `<button>` 本身也有自己的 `type`
* 在表單裡，`<button>` 若不寫 `type`，預設通常會是 `submit`

所以這篇最重要的，不是去比較哪個一定比較好，
而是先把 `<button>` 自己的型別規則搞清楚。

## 常見問法

### `<button>` 不寫 `type` 會怎樣？

* 在表單裡，預設通常會被當成 `submit`。
* 所以它可能會直接觸發表單提交。

### `button` 和 `submit` 差在哪？

* `submit` 會送出表單。
* `button` 只是一般按鈕，本身不會自動送出表單。

### `reset` 是不是把所有欄位清空？

* 不一定。
* 更精確的說法是：它會把表單恢復成初始狀態。

### 什麼時候要明確寫 `type="button"`？

* 只要這顆按鈕不是拿來送出表單，就通常建議明確寫上。
* 這樣可以避免因為預設行為造成誤提交。

### 為什麼 `<button>` 明明只是按鈕，還會送出表單？

因為在表單裡，`<button>` 若未指定 `type`，
預設通常會被當成 `submit`。
所以它不一定只是一般互動按鈕，而可能直接參與表單提交。
