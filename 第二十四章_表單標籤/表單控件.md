## 本節導讀

上一篇認識了表單域，也就是 `<form>` 這個外層容器。  
接下來要看的，就是放在表單裡、讓使用者真正操作的那些元素，也就是：**表單控件。**

不過要先分清一件事：

**表單控件不只 `<input>`。**

像後面會學到的 `<textarea>`、`<select>`、`<button>`，也都屬於表單控件。  
只是對初學者來說，最常接觸、變化也最多的，通常會先是 `<input>`。

所以這篇的重點，不是把所有表單元素一次背完，  
而是先抓住一個核心：

**同一個 `<input>`，會因為 `type` 不同，變成不同用途的表單控件。**

只要先把這件事搞懂，後面看到文字框、密碼框、單選框、複選框、按鈕、檔案上傳等元素時，就不會覺得它們是很多零散知識，而會知道它們其實都和 `<input>` 有關。

## 速查區

### 核心概念

- 表單控件是使用者在表單中實際會操作的元素。
- `<input>` 是其中最常見的一種，而且可以透過 `type` 變成不同用途。
- 學這篇的重點，不是死背標籤，而是理解同一個 `<input>` 為什麼能變成很多不同控件。

### 關鍵規則 / 判準

- `type` 決定控件類型。
- `name` 通常和提交資料時的欄位名稱有關。
- `value` 可能代表預設值、提交值，或按鈕文字，要看控件類型。
- `checked` 主要用於單選框與複選框的預設選中。
- `maxlength` 常用於限制文字輸入長度。
- `<input>` 是單標籤。

### 常見使用場景

- 帳號密碼輸入
- 性別或單選題
- 興趣或多選題
- 提交、重置與一般互動按鈕
- 檔案上傳
- 手機、日期、信箱等格式化輸入

### 常見混淆點

- 表單控件不等於只有 `<input>`，但 `<input>` 確實是最常見的一類。
- `name` 不是所有單選框和複選框都必須完全同一種寫法，而是要看是否屬於同一組資料。
- 單選框的互斥效果，關鍵在同組 `radio` 是否共用相同 `name`。
- 重置按鈕不是把表單硬清空，而是恢復成頁面初始載入時的狀態。
- `<button>` 如果沒寫 `type`，在表單裡預設通常會是 `submit`。

### 一句話抓核心

- **`<input>` 的關鍵不是背標籤本身，而是理解 `type` 如何把它變成不同表單控件。**

## 正文筆記

## 1. 什麼是表單控件？

表單控件，指的是表單中那些讓使用者可以實際操作的元素。

例如：

- 輸入文字
- 輸入密碼
- 勾選選項
- 點擊按鈕
- 上傳檔案

這些都屬於表單控件。

如果和上一篇的表單域一起看，可以這樣理解：

- **表單域**：把一整份表單包起來
- **表單控件**：讓使用者實際輸入、選擇、點擊

也就是說，表單域是外層結構，表單控件是裡面真正被操作的內容。

---

## 2. 為什麼 `<input>` 這麼重要？

因為 `<input>` 是表單中最常見、變化也最多的控件之一。

它本身不是固定只代表某一種輸入框，  
而是會因為 `type` 屬性不同，呈現不同功能。

例如：

- `type="text"`：文字框
- `type="password"`：密碼框
- `type="radio"`：單選框
- `type="checkbox"`：複選框
- `type="submit"`：提交按鈕

所以學 `<input>` 時，真正的重點不是只記住 `<input>` 這個標籤，  
而是要理解：**`type` 決定它要扮演哪一種控件角色。**

---

## 3. `<input>` 的核心屬性有哪些？

### （1）`type`

`type` 是 `<input>` 最核心的屬性。  
它決定這個 `<input>` 會以什麼形式出現在畫面上，也決定它的用途。

你可以把它理解成：

**同一個 `<input>`，透過不同的 `type`，切換成不同控件。**

---

### （2）`name`

`name` 通常和表單提交時的欄位名稱有關。

你可以先把它理解成：

**這筆資料送出去時，系統要用什麼名字辨識它。**

例如帳號欄位、密碼欄位、性別欄位，都會有各自的資料名稱。

---

### （3）`value`

`value` 的意思，會隨著控件類型不同而改變。

常見理解方式如下：

- 在文字框中：常代表預設內容
- 在單選框或複選框中：常代表被提交的資料值
- 在按鈕中：常代表按鈕上顯示的文字

所以學 `value` 時，不要死背成只有一種意思，  
而要看它是套用在哪一種控件上。

---

### （4）`checked`

`checked` 主要用在：

- 單選框 `radio`
- 複選框 `checkbox`

它的作用，是讓選項在頁面一開始載入時就先被選中。

---

### （5）`maxlength`

`maxlength` 常用於文字輸入類型，  
作用是限制使用者最多可以輸入多少字元。

例如：

```html
<input type="text" maxlength="10" />
```

表示最多輸入 10 個字元。

---

## 4. 先看一個整體範例

```html
<form action="xxx.php" method="get">
  使用者名稱：
  <input type="text" name="username" maxlength="12" />
  <br />

  密碼：
  <input type="password" name="pwd" maxlength="20" />
  <br />

  性別：
  <input type="radio" name="gender" value="male" /> 男
  <input type="radio" name="gender" value="female" checked /> 女
  <input type="radio" name="gender" value="other" /> 其他
  <br />

  興趣：
  <input type="checkbox" name="hobby" value="reading" /> 閱讀
  <input type="checkbox" name="hobby" value="music" /> 音樂
  <input type="checkbox" name="hobby" value="sports" checked /> 運動
  <br />

  頭像：
  <input type="file" name="avatar" />
  <br />

  <input type="submit" value="提交資料" />
  <input type="reset" value="恢復初始狀態" />
  <input type="button" value="取得驗證碼" />
</form>
```

這個例子真正要看的，不是語法有多長，
而是同一個 `<input>` 因為 `type` 不同，分別變成了：

* 文字框
* 密碼框
* 單選框
* 複選框
* 檔案上傳
* 提交按鈕
* 重置按鈕
* 普通按鈕

---

## 5. 常見控件類型怎麼分？

## 5-1. 文字框 `text`

```html
<input type="text" />
```

文字框用來輸入單行文字，
常見於姓名、帳號、標題、搜尋關鍵字這類短內容欄位。

### 常用屬性

* `name`：資料名稱
* `value`：預設內容
* `maxlength`：最大輸入長度

### 怎麼理解？

* 需要單行、較短的文字輸入時，通常先想到 `text`
* 如果是較長、多行內容，通常就不會用 `text`，而會改用 `<textarea>`

---

## 5-2. 密碼框 `password`

```html
<input type="password" />
```

密碼框和文字框很像，
差別在於輸入內容通常會以遮罩形式顯示，不直接顯示原文。

### 常用屬性

* `name`：資料名稱
* `maxlength`：最大輸入長度

### 怎麼理解？

* 它本質上也是輸入欄位
* 只是顯示方式和一般文字框不同，主要用在密碼等敏感資訊

---

## 5-3. 單選框 `radio`

```html
<input type="radio" name="gender" value="male" /> 男
<input type="radio" name="gender" value="female" /> 女
```

單選框適合「多個選項中只能選一個」的情境。

### 常用屬性

* `name`：同一組單選框若要形成互斥選擇，`name` 必須一致
* `value`：提交的資料值
* `checked`：預設選中

### 重點怎麼記？

* 單選框是「多選一」
* 同組 `radio` 之所以會互斥，關鍵就在於它們是否共用相同 `name`

---

## 5-4. 複選框 `checkbox`

```html
<input type="checkbox" name="hobby" value="reading" /> 閱讀
<input type="checkbox" name="hobby" value="music" /> 音樂
<input type="checkbox" name="hobby" value="sports" /> 運動
```

複選框適合「多個選項中可以同時選多個」的情境。

### 常用屬性

* `name`：資料名稱；若多個複選框代表同一組選項，常會共用同一個 `name`
* `value`：提交的資料值
* `checked`：預設選中

### 重點怎麼記？

* 複選框是「多選多」
* 是否共用同一個 `name`，要看你是否想把它們視為同一組資料提交

---

## 5-5. 隱藏域 `hidden`

```html
<input type="hidden" name="token" value="abc123" />
```

隱藏域是使用者看不到的輸入區域，
常見用途是在提交表單時，一起夾帶某些固定資料。

### 常用屬性

* `name`：資料名稱
* `value`：真正提交的資料值

### 怎麼理解？

* 它不是給使用者操作的
* 但仍然屬於表單資料的一部分

---

## 5-6. 檔案上傳 `file`

```html
<input type="file" name="avatar" />
<input type="file" name="photos" multiple />
```

檔案上傳控件用來讓使用者選擇本機檔案。

### 常見用法

* 上傳單一檔案
* 搭配 `multiple` 上傳多個檔案

### 怎麼理解？

* 它不是輸入文字，而是讓使用者挑選檔案
* 常見於頭像、附件、證明文件、照片上傳等情境

---

## 6. 提交、重置與普通按鈕

## 6-1. 提交按鈕 `submit`

```html
<input type="submit" value="提交表單" />
<button>提交表單</button>
```

提交按鈕的作用，是把表單資料送出。

### 先記住兩個重點

* `<input type="submit">` 會提交表單
* `<button>` 如果放在表單裡又沒寫 `type`，預設通常也會是提交按鈕

這是初學者很容易忽略的地方。

---

## 6-2. 重置按鈕 `reset`

```html
<input type="reset" value="重置表單" />
<button type="reset">重置表單</button>
```

重置按鈕的作用，不是單純把所有欄位清空，
而是讓表單恢復成頁面一開始載入時的初始狀態。

這是理解 `reset` 時最重要的一點。

---

## 6-3. 普通按鈕 `button`

```html
<input type="button" value="普通按鈕" />
<button type="button">普通按鈕</button>
```

普通按鈕本身不會自動提交表單，
通常是搭配 JavaScript 來做其他互動，例如：

* 取得驗證碼
* 開啟彈窗
* 切換內容
* 展開收合區塊

### 重點怎麼記？

* `button` 是一般互動按鈕
* 只要不是要提交表單，通常就應明確寫出 `type="button"`

---

## 7. HTML5 新增的常見輸入類型

```html
<form action="">
  <ul>
    <li>信箱：<input type="email" /></li>
    <li>網址：<input type="url" /></li>
    <li>日期：<input type="date" /></li>
    <li>時間：<input type="time" /></li>
    <li>數量：<input type="number" /></li>
    <li>手機號碼：<input type="tel" /></li>
    <li>搜尋：<input type="search" /></li>
    <li>顏色：<input type="color" /></li>
    <li><input type="submit" value="提交" /></li>
  </ul>
</form>
```

### 先怎麼理解這些新增類型？

它們是 HTML5 增加的輸入形式。
初學時不用急著把所有細節一次背完，先知道它們各自對應的輸入情境即可。

* `email`：信箱輸入
* `url`：網址輸入
* `date`：日期選擇
* `time`：時間選擇
* `number`：數字輸入
* `tel`：電話輸入
* `search`：搜尋欄位
* `color`：顏色選擇

重點不是背完全部規則，
而是先建立一個概念：**不同的 `type`，會讓 `<input>` 更貼近不同的輸入情境。**

## 常見問法

### 表單控件是不是只有 `<input>`？

不是。

表單控件還包含其他元素，例如 `<textarea>`、`<select>`、`<button>`。
只是這一篇先把重點放在最常見、變化也最多的 `<input>`。

### 為什麼同一個 `<input>` 能表示很多不同控件？

因為 `<input>` 的用途會隨著 `type` 不同而改變。
所以真正決定它功能的關鍵，是 `type`。

### `name` 是不是每個單選框和複選框都一定要完全一樣？

不是這樣背。

* 單選框若要形成同一組互斥選項，`name` 必須一致
* 複選框是否共用同一個 `name`，要看你是否想把它們當成同一組資料提交

### `value` 在不同控件裡意思都一樣嗎？

不完全一樣。

* 在文字框中，它常表示預設內容
* 在單選框或複選框中，它常表示被提交的值
* 在按鈕中，它常表示按鈕文字

### 重置按鈕是不是把所有欄位直接清空？

不一定。

更精確的理解是：
**它會把表單恢復成初始載入時的狀態。**

### `<button>` 和 `<input type="submit">` 有什麼要先注意的？

在表單裡，`<button>` 沒寫 `type` 時，常會直接變成提交按鈕。
所以只要不是要提交表單，通常都建議把 `type` 寫清楚。
