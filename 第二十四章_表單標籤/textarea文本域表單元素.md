## 本節導讀

前面已經認識了表單控件，也知道 `<input>` 可以做很多不同類型的輸入。  
但當欄位內容不再只是短短一行，而是可能要輸入留言、評論、備註、回饋或較長描述時，一般單行文字框就不太夠用了。

這時就會用到：`<textarea>`。

`<textarea>` 的重點，不是「另一種文字框」而已，  
而是它專門用來處理**多行長文本輸入**。

所以學這篇時，先抓住四件事就好：

1. 什麼情況適合用 `<textarea>`
2. 它和一般單行輸入框差在哪
3. `rows` 和 `cols` 分別在控制什麼
4. 為什麼 `<textarea>` 不寫 `type`

只要這幾個觀念先站穩，後面看到留言區、備註欄、意見回饋這類表單設計時，就會很容易判斷什麼時候該用文本域，而不是一般文字輸入框。

## 30 秒複習入口

- `<textarea>` 用來輸入多行文字。
- 它適合留言、評論、備註、回饋這類內容較長的欄位。

## 速查區

### 核心概念

- `<textarea>` 是表單中的多行文字輸入控件。
- 它讓使用者可以一次輸入較長、可換行的內容。
- 和單行輸入框相比，它更適合承載長文本。

### 關鍵規則 / 判準

- 需要輸入多行內容時，通常用 `<textarea>`。
- `rows` 指預設顯示的行數，會影響高度。
- `cols` 指每行預設顯示的字元數，會影響寬度。
- `<textarea>` 不使用 `type` 屬性。
- 它不是單標籤，中間可以放預設文字內容。
- 實務上尺寸常交給 CSS 控制。

### 常見使用場景

- 留言板
- 評論區
- 問題回報
- 備註欄位
- 意見回饋
- 商品描述補充
- 後台說明文字欄位

### 常見混淆點

- `<textarea>` 不是 `<input type="text">` 的另一種寫法，而是不同的表單元素。
- 它不是單標籤，開頭與結尾標籤之間可以放預設文字內容。
- `rows` 和 `cols` 是顯示尺寸，不是限制總輸入字數的屬性。
- 它適合長文本，但不代表所有文字輸入都應改用 `<textarea>`。

### 一句話抓核心

- **`<textarea>` 是專門給多行長文本輸入使用的表單元素。**

## 正文筆記

## 1. 什麼是 `<textarea>`？

`<textarea>` 是表單中的多行文字輸入控件。

你可以把它理解成：  
當一般文字框只適合輸入短短一行內容時，`<textarea>` 則是拿來承接較長、可換行內容的輸入區域。

所以它不是單純把文字框放大而已，  
而是專門處理「多行輸入」這種使用情境的表單元素。

---

## 2. 什麼時候該用 `<textarea>`？

當欄位內容預期會超過一行時，通常就可以優先考慮 `<textarea>`。

例如：

- 留言
- 評論
- 使用者回饋
- 備註說明
- 問題描述
- 申請原因或補充資料

相對地，如果只是這類短內容：

- 姓名
- 帳號
- 標題
- 搜尋關鍵字
- 電話

通常單行文字框就已經足夠，不需要特別用到 `<textarea>`。

所以最簡單的判斷方式是：

- **短、單行內容** → 常用 `<input type="text">`
- **長、多行內容** → 常用 `<textarea>`

---

## 3. `<textarea>` 和一般文字輸入框差在哪？

兩者都能讓使用者輸入文字，  
但它們適合的情境不一樣。

### 一般文字輸入框

- 通常用來輸入單行內容
- 適合短文字
- 常見於姓名、帳號、標題、搜尋欄位

### `<textarea>`

- 適合輸入多行內容
- 更適合長文字
- 常見於留言、評論、備註、回饋欄位

所以真正的差別，不只是外觀看起來大小不同，  
而是它們原本就是為了不同輸入情境而設計的。

---

## 4. 基本寫法長什麼樣子？

```html
<textarea name="message" rows="5" cols="30">預設文字內容</textarea>
```

這個例子可以先看三個重點：

* `name`：表單提交時的欄位名稱
* `rows`：預設顯示的行數
* `cols`：預設顯示的寬度

另外還要注意：
`<textarea>` 不是單標籤，所以開始標籤和結束標籤之間，可以放預設文字內容。

---

## 5. `rows` 和 `cols` 怎麼理解？

這是初學者最常問的地方。

### `rows`

`rows` 表示文本域預設顯示幾行，
它主要影響的是高度。

例如：

```html
<textarea rows="4"></textarea>
```

代表一開始大約顯示 4 行的高度。

---

### `cols`

`cols` 表示每行預設顯示多少字元寬度，
它主要影響的是寬度。

例如：

```html
<textarea cols="30"></textarea>
```

代表文本域一開始會以大約 30 個字元寬度來顯示。

---

### 要特別注意什麼？

`rows` 和 `cols` 的重點是：

**它們控制的是預設顯示尺寸，不是輸入長度限制。**

也就是說：

* `rows` 不是限制只能輸入幾行
* `cols` 不是限制只能輸入幾個字
* 它們只是影響一開始看起來有多高、多寬

這點一定要和 `maxlength` 這種限制輸入字數的概念分開。

---

## 6. 為什麼 `<textarea>` 不寫 `type`？

因為 `<textarea>` 本身就是獨立的表單元素，
不是靠 `type` 來切換型態的 `<input>`。

你可以這樣對照理解：

* `<input>`：同一個標籤，透過 `type` 切換成不同控件
* `<textarea>`：本身就是固定用途的獨立元素

所以它不像 `<input>` 那樣需要透過 `type="text"`、`type="password"` 這種方式來改變型別。

---

## 7. 為什麼 `<textarea>` 不是單標籤？

因為它允許你在開始標籤和結束標籤之間放內容。

例如：

```html
<textarea name="remark">這裡是預設文字</textarea>
```

這段中間的文字，就是文本域一開始顯示的內容。

這也是它和很多 `<input>` 類型不同的地方之一。
很多 `<input>` 是靠 `value` 放預設值，而 `<textarea>` 則是把預設內容直接寫在標籤中間。

---

## 8. 實務上怎麼控制大小？

教學時，常會用 `rows` 和 `cols` 幫助理解文本域的基本尺寸概念。
但在實際開發中，很多情況會改用 CSS 控制寬高，因為這樣通常更容易統一版面樣式。

例如：

```html
<textarea class="feedback-box" name="feedback"></textarea>
```

```css 
.feedback-box {
  width: 400px;
  height: 160px;
}
```

所以可以把它理解成：

* `rows` / `cols`：幫助理解文本域預設尺寸
* CSS：更常用來做實務上的版面控制

## 常見問法

### `<textarea>` 和文字輸入框最大的差別是什麼？

* 文字輸入框通常是單行輸入。
* `<textarea>` 則適合多行、較長內容的輸入。

### `<textarea>` 是單標籤嗎？

不是。

它有開始標籤和結束標籤，中間還可以放預設文字內容。

### `rows` 和 `cols` 是不是拿來限制輸入長度？

不是。

它們主要影響的是預設顯示尺寸，不是總輸入字數限制。

### 為什麼 `<textarea>` 不需要 `type` 屬性？

因為它本身就是獨立的表單元素，不是靠 `type` 切換型態的 `<input>`。

### 實際開發還會常用 `rows` 和 `cols` 嗎？

可以用，但很多情況會改用 CSS 控制尺寸。
這樣通常更容易統一版面樣式。
