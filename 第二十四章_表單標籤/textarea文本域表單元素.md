## 本節導讀

這篇整理 `<textarea>` 文本域表單元素。  
當表單需要讓使用者輸入較長內容，例如留言、評論、備註或回饋時，一般單行文字輸入框就不夠用了，這時通常會改用文本域。

重點是先理解：`<textarea>` 是多行文字輸入控件，不是單標籤；`rows` 和 `cols` 會影響預設顯示尺寸，但實際開發常會改用 CSS 控制外觀。

## 30 秒複習入口

- `<textarea>` 用來輸入多行文字。

## 速查區

### 核心概念

- `<textarea>` 是表單中的多行文字輸入控件。
- 它讓使用者可以一次輸入多段或較長內容。
- 和單行輸入框相比，它更適合承載長文本。

### 關鍵規則 / 判準

- 需要輸入多行內容時，通常用 `<textarea>`。
- `rows` 指預設顯示的行數，會影響高度。
- `cols` 指每行預設顯示的字元數，會影響寬度。
- `<textarea>` 不使用 `type` 屬性。
- 實務上元件尺寸常交給 CSS 處理。

### 常見使用場景

- 留言板
- 評論區
- 問題回報
- 備註欄位
- 意見回饋

### 常見混淆點

- `<textarea>` 不是 `<input type="text">` 的另一種寫法，而是不同的表單元素。
- 它不是單標籤，開頭與結尾標籤之間可以放預設文字內容。
- `rows` 和 `cols` 是顯示尺寸，不是限制可輸入總字數的屬性。
- 是否換行顯示，和單行輸入框的使用情境不同。

### 一句話抓核心

- `<textarea>` 是專門給多行長文本輸入使用的表單元素。

## 正文筆記

> 在表單元素中，`<textarea>` 標籤用於定義多行文本輸入控件。

- 當使用者需要輸入較多內容時，就不適合再用一般單行文字框。
- 這時可以改用 `<textarea>`，讓使用者輸入多行文字。
- 常見情境包括留言板、評論區、回饋欄位與備註內容。

```html
<textarea name="msg" rows="22" cols="3">文本內容</textarea>
```

### 1. 什麼時候該用 `<textarea>`？

- 當欄位內容預期會超過一行時，可以優先考慮文本域。
- 如果只是姓名、帳號、標題這類短內容，通常單行文字框就足夠。
- 如果是意見、留言、心得、備註等較長內容，文本域會更適合。

### 2. `rows` 和 `cols` 怎麼理解？

- `rows`：指定預設顯示的行數，會影響文本域高度。
- `cols`：指定每行預設顯示的字元數，會影響文本域寬度。

這兩個屬性主要是控制一開始看到的尺寸，不是在限制最終只能輸入多少內容。

### 3. 為什麼 `<textarea>` 不寫 `type`？

- 因為 `<textarea>` 本身就是獨立的表單元素，不是靠 `type` 變化型態的 `<input>`。
- 所以它不像 `<input>` 那樣需要用 `type="text"`、`type="password"` 這種方式切換類型。

### 4. 實務上怎麼控制大小？

- 教學中常用 `rows` 和 `cols` 幫助理解文本域的基本尺寸概念。
- 但在實際開發裡，常會改用 CSS 控制寬高，畫面會更好調整。

## 常見問法

### `<textarea>` 和文字輸入框最大的差別是什麼？

- 文字輸入框通常是單行輸入。
- `<textarea>` 則適合多行、較長內容的輸入。

### `<textarea>` 是單標籤嗎？

- 不是。
- 它有開始標籤和結束標籤，中間還可以放預設文字內容。

### `rows` 和 `cols` 是不是拿來限制輸入長度？

- 不是。
- 它們主要影響的是預設顯示尺寸，不是總輸入字數限制。

### 實際開發還會常用 `rows` 和 `cols` 嗎？

- 可以用，但很多情況會改用 CSS 控制尺寸。
- 這樣通常更容易統一版面樣式。
