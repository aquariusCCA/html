## 本節導讀

這篇整理表單中常見的幾個屬性：`required`、`placeholder`、`autofocus`、`autocomplete`。  
它們不會改變表單控件的種類，但會直接影響欄位是否必填、畫面提示、初始焦點位置，以及瀏覽器是否協助自動填寫。

原始內容只有一段範例碼，且 `autocomplete` 拼字有誤，容易讓人只看到寫法卻不知道每個屬性在解決什麼問題。  
這裡改成先講用途，再看範例，最後補上最容易混淆的地方。

## 你會在這篇學到什麼

- `required` 會帶來什麼效果
- `placeholder` 和實際值有什麼差別
- `autofocus` 什麼時候適合使用
- `autocomplete` 怎麼控制瀏覽器自動填寫
- 如何修改 `placeholder` 提示文字的樣式

## 30 秒複習入口

- `required` 讓欄位變成必填，未填時通常不能送出表單。
- `placeholder` 是輸入前的提示文字，不是欄位真正的值。
- `autofocus` 會讓頁面載入後游標自動落在指定欄位。
- `autocomplete` 用來控制瀏覽器是否提供自動填寫建議。
- `::placeholder` 可用來調整提示文字樣式，例如字色。

## 速查區

### 核心概念

- 這些屬性主要是在補強表單的互動體驗。
- 它們常搭配 `<input>` 一起使用，也可能出現在其他表單元素上。
- 重點不是死背屬性名，而是知道每個屬性在處理哪一種使用者體驗問題。

### 關鍵規則 / 判準

- `required`：欄位必填。
- `placeholder`：顯示提示文字，輸入後提示通常會消失。
- `autofocus`：頁面載入時自動聚焦到該欄位。
- `autocomplete`：控制瀏覽器是否顯示自動填寫或歷史輸入建議。
- `::placeholder`：可用 CSS 修改提示文字樣式。

### 常見使用場景

- 搜尋框或登入欄位提示輸入內容
- 註冊表單要求必填欄位
- 首個重要欄位載入後自動聚焦
- 不希望共用裝置記住輸入內容的欄位

### 常見混淆點

- `placeholder` 只是提示，不會等於欄位真正提交的值。
- `required` 不是提示使用者而已，而是會影響表單是否能提交。
- `autofocus` 適合少量使用；通常不會在多個欄位同時強調它。
- `autocomplete="off"` 是在要求瀏覽器不要自動填寫，不是清空欄位內容。

### 一句話抓核心

- 這些表單屬性是在補強欄位的提示、驗證、焦點與填寫體驗。

## 正文筆記

```html
<form action="">
  <input
    type="search"
    name="search"
    required
    placeholder="請輸入關鍵字"
    autofocus
    autocomplete="off"
  />
  <input type="submit" value="提交" />
</form>
```

### 1. `required`：把欄位設成必填

- `required` 表示這個欄位不能留空就直接送出。
- 如果使用者沒有填寫，瀏覽器通常會先阻止提交，並提示該欄位需要輸入內容。
- 它常用在帳號、密碼、信箱、搜尋條件等必要欄位。

### 2. `placeholder`：顯示輸入提示文字

- `placeholder` 會在欄位內顯示提示文字，幫助使用者理解應該輸入什麼。
- 這段文字主要是提示，不是欄位預設值。
- 使用者開始輸入後，提示文字通常就會消失。

```html
<input type="text" placeholder="請輸入使用者名稱" />
```

### 3. `autofocus`：頁面載入後自動聚焦

- `autofocus` 會讓頁面載入完成後，游標直接落在指定欄位上。
- 這適合搜尋框、登入頁第一個欄位等希望使用者立刻開始輸入的情境。
- 但不需要對很多欄位同時使用，否則重點反而會變得不清楚。

### 4. `autocomplete`：控制瀏覽器自動填寫

- `autocomplete` 用來控制瀏覽器是否根據歷史紀錄或已儲存資訊提供建議。
- 常見值包括 `on` 與 `off`。
- 寫成 `autocomplete="off"` 時，表示希望瀏覽器不要自動填寫或提示歷史內容。

```html
<input type="search" autocomplete="off" />
```

### 5. 怎麼修改 `placeholder` 的樣式？

- 若想改變提示文字顏色，可以使用 `::placeholder` 偽元素。
- 這是在改提示文字的外觀，不是在改欄位真正輸入內容的樣式。

```css
input::placeholder {
  color: pink;
}
```

## 常見問法

### `placeholder` 和 `value` 差在哪？

- `placeholder` 是提示文字，幫助使用者理解要輸入什麼。
- `value` 則是欄位真正的內容或預設值，可能會被提交出去。

### `required` 會讓欄位變成什麼狀態？

- 它會讓欄位變成必填。
- 若未填寫，瀏覽器通常會先阻止表單提交。

### `autofocus` 是不是讓欄位自動填值？

- 不是。
- 它只是在頁面載入後，讓游標先落到該欄位。

### `autocomplete="off"` 是不是把輸入內容清掉？

- 不是。
- 它是在控制瀏覽器是否提供自動填寫或歷史輸入建議。
