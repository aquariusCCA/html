# 新增表單屬性

> - 所屬章節：第二十四章｜表單標籤  
> - 關鍵字：新增表單屬性、`required`、`placeholder`、`autofocus`、`autocomplete`、必填欄位、提示文字、自動聚焦、自動完成  
> - 建議回查情境：想知道怎麼把欄位設成必填、想分清提示文字和實際輸入值的差別、想確認 `autofocus` 與 `autocomplete` 分別在做什麼

## 本節導讀

前面已經認識了表單域、表單控件，以及像 `<input>`、`<textarea>`、`<select>`、`<button>` 這些常見元素。  
但光知道「這是什麼元素」還不夠，因為同一個表單欄位，還可能需要進一步補強：

- 這個欄位是不是必填？
- 欄位裡要不要先顯示提示文字？
- 頁面載入後游標要不要直接落在這裡？
- 瀏覽器要不要幫忙提供歷史輸入或自動填寫？

這些問題，通常不是靠換標籤解決，  
而是靠表單屬性來控制。

所以這篇的重點，不是再認識新的表單元素，  
而是理解幾個很常用、也很實用的屬性：

- `required`
- `placeholder`
- `autofocus`
- `autocomplete`

它們不會改變控件種類，  
但會直接影響欄位的提示方式、驗證方式、焦點位置與填寫體驗。

## 你會在這篇學到什麼

- `required` 會帶來什麼效果
- `placeholder` 和實際值有什麼差別
- `autofocus` 什麼時候適合使用
- `autocomplete` 怎麼控制瀏覽器自動填寫
- 如何修改 `placeholder` 提示文字的樣式
- 這些屬性和表單互動體驗有什麼關係

## 30 秒複習入口

- `required` 讓欄位變成必填，未填時通常不能送出表單。
- `placeholder` 是輸入前的提示文字，不是欄位真正的值。
- `autofocus` 會讓頁面載入後游標自動落在指定欄位。
- `autocomplete` 用來控制瀏覽器是否提供自動填寫建議。
- `::placeholder` 可用來調整提示文字樣式，例如字色。
- 這些屬性主要是在補強欄位的提示、驗證、焦點與填寫體驗。

## 速查區

### 核心概念

- 這些屬性主要是在補強表單的互動體驗。
- 它們常搭配 `<input>` 一起使用，也可能出現在其他表單元素上。
- 重點不是死背屬性名，而是知道每個屬性在處理哪一種使用者體驗問題。

### 關鍵規則 / 判準

- `required`：欄位必填。
- `placeholder`：顯示提示文字，輸入後提示通常會消失。
- `autofocus`：頁面載入時自動聚焦到該欄位。
- `autocomplete`：控制瀏覽器是否顯示自動填寫或歷史輸入建議。
- `::placeholder`：可用 CSS 修改提示文字樣式。

### 常見使用場景

- 搜尋框或登入欄位提示輸入內容
- 註冊表單要求必填欄位
- 首個重要欄位載入後自動聚焦
- 不希望共用裝置記住輸入內容的欄位
- 需要讓使用者更快理解欄位用途的輸入框

### 常見混淆點

- `placeholder` 只是提示，不會等於欄位真正提交的值。
- `required` 不是單純提醒使用者，而是會影響表單是否能提交。
- `autofocus` 適合少量使用；通常不會在很多欄位同時強調它。
- `autocomplete="off"` 是要求瀏覽器不要提供自動填寫建議，不是清空欄位內容。
- 這些屬性是在補強既有控件，不是取代原本的表單元素。

### 一句話抓核心

- **這些表單屬性是在補強欄位的提示、驗證、焦點與填寫體驗。**

## 正文筆記

## 1. 先看一個整體範例

```html
<form action="">
  <input
    type="search"
    name="search"
    required
    placeholder="請輸入關鍵字"
    autofocus
    autocomplete="off"
  />
  <input type="submit" value="提交" />
</form>
```

這段範例想讓你先看到的是：

* `required`：這個欄位不能空著直接送出
* `placeholder`：欄位中先顯示提示文字
* `autofocus`：頁面載入後，游標先落在這裡
* `autocomplete="off"`：希望瀏覽器不要提供歷史輸入建議

也就是說，
同一個輸入框除了 `type` 之外，還可以再透過這些屬性，進一步調整使用體驗。

---

## 2. `required`：把欄位設成必填

`required` 的作用，是讓某個欄位變成必填。

```html
<input type="email" name="email" required />
```

當使用者沒有填寫這個欄位就嘗試送出表單時，
瀏覽器通常會先阻止提交，並提示這個欄位需要輸入內容。

### 怎麼理解？

* 它不是單純顯示「這欄很重要」而已
* 它會實際影響表單能不能送出
* 常見於帳號、密碼、信箱、搜尋條件等必要欄位

所以 `required` 的重點不是提示語氣，
而是：**讓欄位正式進入必填規則。**

---

## 3. `placeholder`：顯示輸入提示文字

`placeholder` 會在欄位內先顯示一段提示文字，
幫助使用者理解這裡應該輸入什麼。

```html
<input type="text" placeholder="請輸入使用者名稱" />
```

當使用者開始輸入後，這段提示文字通常就會消失。

### 怎麼理解？

* 它是提示
* 不是預設值
* 不是表單真正提交的內容

這是最常被混淆的地方。
很多初學者會把 `placeholder` 和 `value` 混在一起，但它們其實在做不同的事。

---

## 4. `placeholder` 和 `value` 差在哪？

這一點一定要分清楚。

### `placeholder`

* 是提示文字
* 主要作用是告訴使用者「這裡該填什麼」
* 通常在使用者開始輸入後就會消失

### `value`

* 是欄位真正的內容或預設值
* 可能會被表單提交出去
* 不是單純提示

你可以這樣記：

* `placeholder`：給人看的提示
* `value`：給欄位本身的內容

所以 `placeholder` 不能直接當成欄位值來理解。

---

## 5. `autofocus`：頁面載入後自動聚焦

`autofocus` 會讓頁面載入完成後，
游標直接落在指定欄位上。

```html
<input type="search" autofocus />
```

這個屬性常見於：

* 搜尋框
* 登入頁第一個欄位
* 希望使用者一進頁面就立刻開始輸入的情境

### 怎麼理解？

* 它是在控制「焦點先放哪裡」
* 不是幫你自動填值
* 也不是幫你送出表單

所以 `autofocus` 處理的是輸入起點，而不是資料內容。

### 使用時要注意什麼？

雖然它很好用，
但通常不需要在很多欄位同時強調它。
因為頁面載入時，真正能先獲得焦點的通常只會是一個主要欄位。

---

## 6. `autocomplete`：控制瀏覽器自動填寫

`autocomplete` 用來控制瀏覽器是否根據歷史紀錄或已儲存資訊，提供自動填寫建議。

```html
<input type="search" autocomplete="off" />
```

常見值包括：

* `on`
* `off`

### 怎麼理解？

* `autocomplete="on"`：允許瀏覽器提供自動填寫或歷史輸入建議
* `autocomplete="off"`：希望瀏覽器不要提供這些建議

這通常會出現在：

* 共用裝置不想保留輸入紀錄的欄位
* 不希望跳出舊資料干擾輸入的搜尋框
* 某些敏感或一次性輸入欄位

### 最容易誤會的地方

`autocomplete="off"` 不是：

* 把欄位內容清掉
* 把已輸入資料刪除
* 讓欄位失效

它只是在控制瀏覽器要不要主動提供自動填寫或歷史建議。

---

## 7. 怎麼修改 `placeholder` 的樣式？

如果你想改變提示文字的外觀，例如字色、字體風格，
可以使用 `::placeholder` 偽元素。

```css
input::placeholder {
  color: pink;
}
```

### 怎麼理解？

* 你改的是提示文字的樣式
* 不是輸入內容本身的樣式
* 也不是在修改欄位的值

所以 `::placeholder` 處理的是「提示文字的外觀」，
不是欄位真正輸入進去的資料。

---

## 8. 這幾個屬性和表單元素的關係是什麼？

這篇最重要的一個總結是：

這些屬性不是新的表單元素，
而是加在既有欄位上的補強工具。

也就是說，它們處理的是：

* 欄位是不是必填
* 欄位要不要先顯示提示
* 頁面一開始焦點落在哪
* 瀏覽器要不要幫忙補資料

所以學這篇時，不要把它們當成零散規則。
更好的理解方式是：

**它們是在優化既有表單控件的互動體驗。**

## 常見問法

### `placeholder` 和 `value` 差在哪？

* `placeholder` 是提示文字，幫助使用者理解要輸入什麼。
* `value` 則是欄位真正的內容或預設值，可能會被提交出去。

### `required` 會讓欄位變成什麼狀態？

* 它會讓欄位變成必填。
* 若未填寫，瀏覽器通常會先阻止表單提交。

### `autofocus` 是不是讓欄位自動填值？

* 不是。
* 它只是在頁面載入後，讓游標先落到該欄位。

### `autocomplete="off"` 是不是把輸入內容清掉？

* 不是。
* 它是在控制瀏覽器是否提供自動填寫或歷史輸入建議。

### `::placeholder` 是改什麼？

* 它是用來改提示文字樣式的。
* 不是在改欄位真正輸入的內容樣式。
