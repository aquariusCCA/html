這個範例把第 41 章筆記中的非模態、模態、開關 API、確認框、表單關閉、返回值、背景遮罩與焦點回復放在同一頁觀察。重點不是自製彈窗外觀，而是使用瀏覽器原生對話框能力。

<dialog> 具有原生語意與開關狀態的互動容器。
show() 開啟非模態對話框，背景通常仍可操作。
showModal() 開啟模態對話框，背景互動會被阻擋。
close(value) 關閉並可設定 returnValue。
::backdrop 模態 dialog 背後的背景遮罩。

非模態

使用 show()。適合輔助說明、浮動提示或不需要阻擋背景的面板。

模態

使用 showModal()。適合確認刪除、重要表單或必須先回應的流程。

不要只靠 open

open 是狀態，不等於完整模態流程；開啟互動請優先用 API。

3. 刪除確認框重點

form method="dialog"

按下表單按鈕時，dialog 會關閉，按鈕的 value 會成為 returnValue。

close 事件

在 close 事件中讀取 returnValue，再決定是取消還是執行刪除。

<dialog id="deleteDialog" aria-labelledby="deleteTitle" aria-describedby="deleteDesc">
  <form method="dialog">
    <h2 id="deleteTitle">確認刪除筆記？</h2>
    <p id="deleteDesc">刪除後無法復原。</p>
    <button value="cancel">取消</button>
    <button value="delete">確認刪除</button>
  </form>
</dialog>

4. API 對照

方法 / 屬性 作用 重點提醒
dialog.show() 開啟非模態對話框。 背景通常仍可互動。
dialog.showModal() 開啟模態對話框。 背景會被阻擋，可使用 ::backdrop。
dialog.close(value) 關閉 dialog，並可設定返回值。 返回值可在 close 事件中讀取。
dialog.open 表示目前是否開啟。 它是狀態，不是推薦的模態開啟方式。

5. 可存取性回查

  • dialog 應有清楚標題，可用 aria-labelledby 指向標題。
  • 重要說明可用 aria-describedby 和對應段落建立關聯。
  • 提供明確的取消、關閉或返回按鈕，不要只依賴 Esc。
  • 重新開啟前可清空 returnValue，避免讀到上一次結果。
  • 關閉後把焦點回到開啟 dialog 的按鈕，鍵盤使用者比較不會迷失。
  • 大量內容、複雜多步驟流程或長時間工作區通常不適合塞進 dialog。
