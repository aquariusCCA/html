# 第四十一章｜`dialog` 標籤

> 章節定位：理解 HTML 原生對話框元素如何表示彈窗、確認框與模態互動  
> 建議閱讀方式：先分清非模態與模態，再理解 `show()`、`showModal()`、`close()` 和表單關閉方式

## 本章在學什麼

這一章整理 HTML 的 `<dialog>` 元素。

`<dialog>` 可以表示一個對話框，例如提示、確認、表單彈窗或模態互動。  
重點不是「做出彈窗外觀」，而是理解原生對話框的語意、開關方式、焦點處理與可存取性注意點。

## 易混淆主題

- `open` vs `showModal()`：前者是狀態屬性，後者是以模態方式開啟的 API。
- 模態 vs 非模態：模態會阻擋背景互動，非模態不一定。
- 彈窗外觀 vs 對話框語意：樣式可以用 CSS，但互動語意要正確。
