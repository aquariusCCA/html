# 第二十二章｜列表標籤

> 章節定位：學會用 HTML 表達一組相關項目的並列、排序與術語說明  
> 建議閱讀方式：先分清三種列表的用途，再看結構規則與列表嵌套

## 本章在學什麼

這一章整理 HTML 的列表標籤。  
重點不是把內容排成好幾行，而是分清楚列表是在表達什麼關係，以及不同列表類型應該用在什麼情境。

## 本章與整體地圖的關係

- 前置知識：前面學過的基本標籤、盒子標籤與超鏈接標籤，能幫助理解列表項目裡還能再放哪些內容。
- 這章的核心任務：分清 `ul`、`ol`、`dl` 的用途，並理解 `li`、`dt`、`dd` 的結構角色。
- 讀完後通常接：之後在做導覽列、功能清單、步驟流程、FAQ 與術語說明時，都能回來快速查這一章。

## 易混淆主題

- 列表 vs 布局：列表是在表達內容關係，不是專門的排版工具。
- `ul` vs `ol`：差別在順序語意，不只是圓點和數字外觀。
- 描述列表 vs 一般項目列表：前者偏向名稱與說明，後者偏向並列或排序項目。
