# 第22章｜列表標籤

> - 關鍵字：列表、list、`ul`、`ol`、`li`、`dl`、`dt`、`dd`、無序列表、有序列表、描述列表、列表嵌套
> - 建議回查情境：想分清三種列表的用途、想確認各自該搭配哪些標籤、想知道列表能不能巢狀、想避免把列表誤當成版面布局工具

## 本節導讀

HTML 的列表標籤，不是單純把內容排成一條一條而已。
它真正的作用是：**把一組彼此相關的項目，用正確的語意結構表達出來**。

學列表時，最容易犯的錯誤有兩個：

* 把列表當成頁面布局工具
* 只看外觀，不看內容關係

所以這一章的重點，不是記住哪個標籤會出現圓點或數字，而是分清楚：

* 哪些內容只是並列項目
* 哪些內容有先後順序
* 哪些內容是「名稱 + 說明」的對應關係

只要這三種情況分得清楚，列表就不容易用錯。

---

## 你會在這篇學到什麼

* 列表在 HTML 中真正的用途
* `ul`、`ol`、`dl` 三種列表怎麼分
* `li`、`dt`、`dd` 各自在做什麼
* 列表中能不能放其他元素
* 什麼是列表嵌套，應該怎麼寫
* 為什麼列表不是拿來做整頁布局的

---

## 30 秒複習入口

* 列表是用來表達**一組相關項目**，不是用來做整個頁面布局。
* `ul`：沒有先後順序的列表。
* `ol`：有先後順序的列表。
* `dl`：名稱與說明的列表。
* `ul`、`ol` 的主要列表項目都是 `li`。
* `dl` 通常搭配 `dt` 和 `dd`。
* 子列表必須寫在某個列表項目裡，而不是直接並排塞進外層列表。

---

## 速查區

### 核心概念

* 列表的本質是**分組與表達關係**。
* 重點不是「排成一列一列」，而是讓瀏覽器與開發者都知道：這些內容屬於同一組，而且彼此之間有特定關係。

### 三種列表的判準

* **沒有順序，只是列出項目** → `ul`
* **有順序、有步驟、有名次** → `ol`
* **名稱對應說明** → `dl`

### 關鍵規則

* `ul`、`ol` 的列表項目使用 `li`
* `li` 裡可以放文字、段落、圖片、連結，甚至另一個列表
* `dl` 用來包整組描述列表
* `dt` 表示名稱 / 術語 / 項目名
* `dd` 表示對該名稱的說明內容
* 子列表要放在某個 `li` 裡，才算結構正確

### 一句話抓核心

* 列表標籤的任務，是**正確表達一組項目的語意關係**，不是硬拿來做版面。

# 正文筆記

## 1. 列表標籤是什麼？

當你要在網頁中呈現一組彼此相關的內容時，就很適合用列表。

例如：

* 導航選單
* 商品分類
* 新聞清單
* 操作步驟
* 排行榜
* 名詞與解釋

這些內容雖然看起來都像「一條一條列出來」，但它們彼此的關係不一定一樣。
有些只是並列，有些有順序，有些是名稱加說明。

因此，HTML 把列表分成三種：

* 無序列表 `ul`
* 有序列表 `ol`
* 描述列表 `dl`

---

## 2. 無序列表 `ul`

當一組項目之間**沒有明確先後順序**時，用無序列表。

```html
<h4>您喜歡的食物？</h4>
<ul>
  <li>榴槤</li>
  <li>臭豆腐</li>
  <li>鯡魚罐頭</li>
  <li>
    <p>其他特殊口味</p>
  </li>
</ul>
```

### 怎麼理解？

* `<ul>` 表示這是一組沒有順序要求的內容
* 每一個項目都用 `<li>` 表示
* 項目之間可以交換順序，但通常不影響核心意思

### 重點不是圓點

瀏覽器常把 `ul` 顯示成帶圓點的清單，但那只是預設樣式。
`ul` 真正重要的是：

**這些項目屬於同一組，但沒有先後順序。**

### 初學者要先記住

* `ul` 裡的列表項目主要就是 `li`
* 不要直接把純文字丟進 `ul` 裡
* 外觀上的圓點、縮排、間距，通常交給 CSS 控制

---

## 3. 有序列表 `ol`

當一組項目之間**有先後順序**時，用有序列表。

例如：

* 排行榜
* 操作流程
* 教學步驟
* 按順序執行的事項

```html
<h4>粉絲排行榜</h4>
<ol>
  <li>劉德華 10000</li>
  <li>劉若英 1000</li>
  <li>pink 老師 1</li>
</ol>
```

### 怎麼理解？

* `<ol>` 表示這是一組有順序的內容
* 每一個項目仍然用 `<li>` 表示
* 如果把順序換掉，通常會影響原本意義

### 它和 `ul` 的真正差別

不是差在外觀。
不是一個有圓點、一個有數字而已。
真正差別是：

* `ul`：順序不重要
* `ol`：順序重要

所以即使你用 CSS 把 `ol` 的數字拿掉，它在語意上仍然是有序列表。

### 初學者要先記住

* `ol` 的項目也是 `li`
* 不要直接把純文字丟進 `ol` 裡
* 只要內容有步驟、先後、名次，就優先考慮 `ol`

---

## 4. 描述列表 `dl`

當內容的重點是**名稱與說明的對應關係**時，用描述列表。

這種列表常見於：

* 術語解釋
* 產品規格
* 問答說明
* 聯絡資訊
* 名稱對應描述的資料

```html
<dl>
  <dt>HTML</dt>
  <dd>負責網頁的結構</dd>

  <dt>CSS</dt>
  <dd>負責網頁的樣式</dd>

  <dt>JavaScript</dt>
  <dd>負責網頁的互動行為</dd>
</dl>
```

這個例子比單純「一個分類下面列幾個項目」更能代表 `dl` 的本質。

### 三個標籤各自在做什麼？

* `<dl>`：整組描述列表
* `<dt>`：description term，名稱 / 術語 / 項目名
* `<dd>`：description details，對該名稱的說明內容

### 怎麼理解它和前兩種列表的差異？

* `ul` / `ol` 是在列一組項目
* `dl` 是在表達「某個名稱對應什麼說明」

所以 `dl` 的閱讀方式，通常不是看「第 1 項、第 2 項」，而是看「這個詞是什麼意思」。

### 初學者要先記住

* `dl` 通常會搭配 `dt` 和 `dd`
* 一個 `dt` 可以對應一個或多個 `dd`
* `dt` 和 `dd` 裡也可以再放常見內容元素

例如：

```html
<dl>
  <dt>前端技術</dt>
  <dd>HTML</dd>
  <dd>CSS</dd>
  <dd>JavaScript</dd>
</dl>
```

這種寫法也成立，因為它仍然是「一個名稱，對應多個說明內容」。

---

## 5. `li` 裡面可以放什麼？

`li` 不只是放一段文字而已。
它可以放很多常見內容，例如：

* 文字
* 段落
* 圖片
* 連結
* 其他區塊元素
* 另一個列表

例如：

```html
<ul>
  <li>
    <a href="">首頁</a>
  </li>
  <li>
    <p>最新公告</p>
  </li>
  <li>
    <img src="demo.jpg" alt="示意圖" />
  </li>
</ul>
```

### 要抓住的重點

`li` 的意思不是「只能放一行字」，而是：

**這是一個列表項目。**

至於這個項目裡面裝的是什麼內容，可以很單純，也可以比較豐富。

---

## 6. 列表嵌套

如果某一個列表項目底下，還有更細的子項目，就可以在那個 `li` 裡面再放一個列表。
這就叫做列表嵌套。

```html
<h2>我想去幾個城市</h2>
<ul>
  <li>成都</li>
  <li>
    上海
    <ul>
      <li>外灘</li>
      <li>杜莎夫人蠟像館</li>
      <li>東方明珠</li>
      <li>迪士尼樂園</li>
    </ul>
  </li>
  <li>西安</li>
  <li>武漢</li>
</ul>
```

### 怎麼判讀這段？

* 外層 `ul` 在列城市
* 「上海」是一個外層列表項目
* 但「上海」底下又有更細的景點，所以在它的 `li` 內再放一個 `ul`

### 最重要的規則

子列表一定要寫在某個 `li` 裡。
不要把外層列表和子列表直接並排丟在一起。

正確理解是：

* 子列表屬於某個父項目
* 所以它必須放進那個父項目的 `li` 裡

---

## 7. 三種列表怎麼選？

| 情況              | 應該用什麼 |
| --------------- | ----- |
| 只是列出一組相關項目，沒有順序 | `ul`  |
| 項目有步驟、流程、名次、先後  | `ol`  |
| 要呈現名稱與說明的對應關係   | `dl`  |

你可以用這個判準快速判斷：

* **順序不重要** → `ul`
* **順序重要** → `ol`
* **重點在解釋** → `dl`

---

## 8. 列表不是布局工具

這句話很重要。

列表確實常常會被做成：

* 導航列
* 側邊選單
* 商品清單
* 卡片區塊

所以很多初學者會誤以為：

「列表就是拿來排版的。」

但更精確的說法是：

* 列表負責提供**語意結構**
* CSS 負責控制**視覺布局**

也就是說，列表可以參與布局，但它本身的核心任務不是布局，而是表達「這些內容屬於同一組」。

### 簡單區分

* 想表達內容關係：先想 HTML 語意
* 想改位置、橫排、間距、對齊：交給 CSS

---

# 常見問法

## 列表可以拿來做頁面布局嗎？

不要把它當成主要用途來記。
列表的核心是表達一組相關項目，版面安排應該主要交給 CSS。

---

## `ul` 和 `ol` 最重要的差別是什麼？

不是外觀差別，而是內容有沒有順序。

* 沒有順序：`ul`
* 有順序：`ol`

---

## `li` 裡面只能放文字嗎？

不是。
`li` 可以放文字，也可以放段落、圖片、連結，甚至再放另一個列表。

---

## `dl` 為什麼和前兩種列表不一樣？

因為它不是在列「一組並列項目」，而是在表達「名稱和說明」的對應關係。

---

## 如果我把 `ol` 的數字樣式拿掉，它還算有序列表嗎？

算。
因為列表的語意是由 HTML 標籤決定，不是由畫面外觀決定。

# 一句話總結

HTML 的列表標籤，不是在幫你「把內容排成一列一列」，而是在幫你**正確表達一組項目之間的關係**：是並列、是排序，還是名稱對應說明。
