# 列表標籤

> 所屬章節：第二十二章｜列表標籤  
> 關鍵字：列表、list、`ul`、`ol`、`li`、`dl`、`dt`、`dd`、無序列表、有序列表、描述列表、自定義列表、列表嵌套  
> 建議回查情境：想分清三種列表的用途、想確認各自該搭配哪些標籤、想知道列表能不能巢狀、想避免把列表誤當成版面布局工具

## 本節導讀

這篇整理 HTML 的列表標籤。  
核心不是「把內容排成一行一行」而已，而是分清楚什麼情況該用無序列表、有序列表或描述列表，並理解它們各自的結構規則。

原始內容有兩個需要先修正的地方：章次誤寫成第 23 章，以及把列表說成「用來布局」，這會讓初學者把內容語意和版面設計混在一起。  
這裡改成較穩定的學習順序：先理解列表在解決什麼問題，再分別看三種列表與列表嵌套。

## 你會在這篇學到什麼

- 列表在 HTML 中的真正用途
- 無序列表、有序列表、描述列表怎麼分
- `ul`、`ol`、`li`、`dl`、`dt`、`dd` 各自在做什麼
- 列表中能不能再放其他元素
- 什麼是列表嵌套，實際上怎麼寫

## 30 秒複習入口

- 列表是用來表達一組彼此相關的項目，不是用來做整個頁面的版面布局。
- `ul` 表示無序列表，適合沒有先後順序的項目。
- `ol` 表示有序列表，適合有名次、步驟或順序的內容。
- `dl` 表示描述列表，常用來呈現術語與說明。
- 列表可以嵌套；外層列表項目裡面可以再放一個列表。

## 速查區

### 核心概念

- 列表的核心任務是把同類型內容整理成一組可讀的項目。
- 如果重點在語意分組、順序或條目關係，列表就是合適的 HTML 結構。
- 如果重點只是排版位置，通常應該交給 CSS 處理，而不是把列表當成布局工具。

### 關鍵規則 / 判準

- `ul` 的直接子元素初學時先記成 `li`。
- `ol` 的直接子元素初學時先記成 `li`。
- `li` 裡面可以再放文字、段落、圖片、連結，甚至另一個列表。
- `dl` 通常搭配 `dt` 和 `dd` 一起使用。
- 描述列表常見型態是一個 `dt` 對應一個或多個 `dd`。

### 常見使用場景

- 新聞標題清單
- 排行榜或步驟流程
- 產品規格、術語解釋、聯絡資訊說明
- 導航列或側邊選單的條目結構
- 某個項目底下再細分子項目的巢狀內容

### 常見混淆點

- 列表能把內容一列一列排出來，但它不是專門的頁面布局工具。
- `ul` 和 `ol` 的差別不在外觀，而在內容是否有先後順序。
- 教材常把 `dl` 稱為「自定義列表」，更穩定的名稱是「描述列表」。
- `li` 可以包很多內容，但 `ul` / `ol` 不應直接塞沒有包在 `li` 裡的純文字。

### 一句話抓核心

- 列表標籤的重點是正確表達一組相關項目的關係，而不是拿來硬做版面。

## 正文筆記

### 這篇在解決什麼問題？

- 當你要在頁面中呈現一組彼此相關的內容時，常常不只是把字換行而已，而是要表達「這些東西屬於同一組」。
- HTML 的列表標籤就是用來處理這種情況，並進一步區分它們有沒有順序、是不是在做名詞解釋。

## 1. 列表標籤是什麼？

- 列表適合拿來呈現一組相關項目，例如新聞清單、排行榜、待辦事項、步驟說明或名詞解釋。
- 它的重點是內容分組與語意表達，不是整個頁面的視覺布局。
- HTML 常見的列表有三種：無序列表、有序列表、描述列表。

## 2. 無序列表 `ul`

- 當一組項目之間沒有明確先後順序時，通常使用無序列表。
- 瀏覽器預設常會顯示項目符號，但那只是預設樣式，不是它最重要的價值。

```html
<h4>您喜歡的食物？</h4>
<ul>
  <li>榴槤</li>
  <li>臭豆腐</li>
  <li>鯡魚罐頭</li>
  <li>
    <p>123</p>
  </li>
</ul>
```

### 怎麼理解這段結構？

- `<ul>` 表示一組沒有排序要求的項目。
- 每個項目用 `<li>` 包起來。
- `<li>` 不只可以放純文字，也可以再放段落、圖片、連結或其他元素。

### 初學時先記的規則

- `ul` 的直接子元素先理解成 `li`。
- 不要把純文字或其他無關標籤直接塞在 `ul` 裡。
- 樣式上的圓點、縮排等外觀，多半會交給 CSS 控制。

## 3. 有序列表 `ol`

- 當項目之間有順序時，例如排名、流程、步驟，就適合用有序列表。
- 瀏覽器預設常用數字顯示順序，但同樣只是預設表現。

```html
<h4>粉絲排行榜</h4>
<ol>
  <li>劉德華 10000</li>
  <li>劉若英 1000</li>
  <li>pink 老師 1</li>
</ol>
```

### 它和無序列表差在哪？

- 結構上兩者都會搭配 `li`。
- 真正差別在於內容是否有先後次序。
- 如果順序換掉會影響意義，就應優先考慮 `ol`。

### 初學時先記的規則

- `ol` 的直接子元素先理解成 `li`。
- 不要直接把純文字放進 `ol` 裡。
- 名次、步驟、流程這類內容，通常比 `ul` 更適合用 `ol`。

## 4. 描述列表 `dl`

- 這種列表常用來呈現「名詞與說明」或「項目與描述」。
- 有些教材會叫它「自定義列表」，但更穩定的名稱是描述列表。

```html
<dl>
  <dt>關注我們</dt>
  <dd>新浪微博</dd>
  <dd>官方微信</dd>
  <dd>聯絡我們</dd>

  <dt>商務合作</dt>
  <dd>品牌合作信箱</dd>
  <dd>合作窗口電話</dd>
</dl>
```

### 這三個標籤各自在做什麼？

- `<dl>`：表示整組描述列表。
- `<dt>`：description term，表示項目名稱或術語。
- `<dd>`：description details，表示對該項目的說明內容。

### 初學時先記的規則

- `dl` 通常搭配 `dt` 和 `dd` 一起出現。
- 一個 `dt` 可以對應一個或多個 `dd`。
- `dt` 和 `dd` 裡面也可以再放其他常見內容元素。

## 5. 列表嵌套

- 如果某個列表項目底下還有更細的子項目，可以在該 `li` 裡再放一個列表。
- 這就是列表嵌套。

```html
<h2>我想去幾個城市</h2>
<ul>
  <li>成都</li>
  <li>
    <span>上海</span>
    <ul>
      <li>外灘</li>
      <li>杜莎夫人蠟像館</li>
      <li>
        <a href="">東方明珠</a>
      </li>
      <li>迪士尼樂園</li>
    </ul>
  </li>
  <li>西安</li>
  <li>武漢</li>
</ul>
```

### 怎麼判讀？

- 外層列表在列出城市。
- 「上海」這個項目裡又細分出景點，所以在它的 `li` 內再放一個 `ul`。
- 巢狀列表的關鍵不是多縮排幾格，而是「子列表必須放在某個列表項目裡面」。

## 6. 三種列表怎麼選？

| 情況 | 適合的列表 |
| --- | --- |
| 只是列出一組相關項目，沒有先後順序 | `ul` |
| 項目有名次、流程或步驟順序 | `ol` |
| 要呈現術語、標題與說明 | `dl` |

## 常見問法

### 列表可以拿來做頁面布局嗎？

- 初學時不要這樣理解。
- 列表可以形成一列一列的畫面效果，但它的核心用途仍是表達一組相關項目。
- 頁面布局應優先交給 CSS。

### `ul` 和 `ol` 最重要的差別是什麼？

- 不在於圓點或數字外觀。
- 真正差別在於內容有沒有順序。

### `li` 裡面可以再放其他標籤嗎？

- 可以。
- `li` 可以放文字，也可以放段落、圖片、連結，甚至另一個列表。

### `dl` 為什麼和前兩種列表不一樣？

- 因為它不是在列「並列項目」或「排序項目」。
- 它更像是在處理「名稱和說明」的對應關係。
