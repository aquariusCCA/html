HTML 列表標籤 Demo

這個範例把第二十二章的重點做成可操作頁面：列表不是拿來把內容排成一行一行，而是用正確語意表達一組項目之間的關係。先判斷內容是並列、排序，還是名稱對應說明，再選 `ul`、`ol` 或 `dl`。

`ul` 表達無序並列 項目之間沒有明確先後順序，交換位置通常不影響核心意思。
`ol` 表達有序流程 內容有步驟、名次、先後關係，順序改變會影響意義。
`dl` 表達名稱與說明 用 `dt` 放名稱或術語，用 `dd` 放對應說明。
列表不是版面工具 HTML 負責內容關係，橫排、間距、卡片樣式交給 CSS。

本頁會示範

  • `ul`、`ol`、`dl` 的選擇判準
  • `li`、`dt`、`dd` 的結構角色
  • `li` 裡放段落、圖片、連結與子列表
  • 列表語意與 CSS 版面控制的分工

1. 三種列表怎麼選

Unordered List

沒有先後順序的一組項目

`ul` 的重點不是圓點，而是項目屬於同一組，但順序不是核心。 


          


          

        

      


      

        
2. `li` 裡可以放豐富內容

        

          

            Rich List Item
            
`li` 的本質是列表項目，不是只能放一行文字

            

              `li` 裡可以放文字、段落、圖片、連結、區塊元素，甚至另一個列表。重點是這些內容仍然屬於某個列表項目。

4. 列表語意與 CSS 版面分工

        

          

            

              
              
            

          


          

            語意先行
            
導覽列仍然可以是列表

            

              導覽列是一組相關導航項目，用 `ul` 表達分組，再用 CSS 橫排。
            

            

            

          

        

      


      

        
5. 三種列表的快速判斷

        

          

            `ul`
            
只是列出一組相關項目

            
例如分類、導覽項目、標籤、特色清單。順序換掉通常不影響意思。

          


          

            `ol`
            
項目有步驟、流程、名次

            
例如操作步驟、排行榜、安裝流程。順序換掉會改變意義。

          


          

            `dl`
            
名稱對應說明

            
例如術語解釋、FAQ、產品規格、聯絡資訊。

          


          

            不要只看外觀
            
圓點、數字都可以用 CSS 改

            
列表語意由 HTML 標籤決定，不是由畫面上的圓點或數字決定。

          

        

      


      

        
6. 實務提醒

        

          

            不要把純文字直接丟進 `ul` / `ol`
            主要列表項目應該放在 `li` 裡，這樣結構才清楚。
          

          

            `ol` 拿掉數字後仍是有序列表
            CSS 改掉外觀，不會改變 HTML 的語意。
          

          

            `dl` 不是一般項目清單
            它重點是名稱與說明的對應，不是單純並列或排序。
