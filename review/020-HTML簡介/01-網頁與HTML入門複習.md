# 網頁與 HTML 入門複習

> 回憶檢索複習。概念與用法說明見 notes/020-HTML簡介/01-網頁與HTML入門.md。

## 重點摘要

- **網站 vs 網頁**：網站＝一組相關網頁的集合（整個店）；網頁＝網站中的一頁，通常是 HTML 文件（店裡的某一個頁面）。
- **HTML 全名**：超文本標記語言（HyperText Markup Language）。
  - 超文本（HyperText）：能加入多媒體＋能透過超連結跳到其他文件。
  - 標記語言（Markup Language）：不計算、不判斷邏輯，只用標籤標記內容的角色。
- **HTML 非程式語言**：不負責計算或邏輯判斷；那是 JavaScript 的工作。
- **網頁形成三步驟**：①撰寫代碼 → ②瀏覽器解析（parse）→ ③瀏覽器渲染（render）→ 呈現畫面。

---

### 問答 1 · 基礎 · {#020-intro-q1}

網站與網頁的關係是什麼？

<details>
<summary>答案</summary>

- **網站（Website）**：在網際網路上，依照一定規則、由 HTML 等技術做出的一組相關網頁的集合。
- **網頁（Web Page）**：網站中的「一頁」，通常是一份 HTML 格式的文件，需要透過瀏覽器才能閱讀。
- 口訣：網站是「整個店」，網頁是「店裡的某一個頁面」。

</details>

---

### 問答 2 · 基礎 · {#020-intro-q2}

為什麼說 HTML 是「標記語言」而不是「程式語言」？

<details>
<summary>答案</summary>

HTML 不負責計算或邏輯判斷，而是一套用來「標記」內容的標籤系統——用標籤描述段落、標題、圖片、連結的角色，讓瀏覽器知道這些內容分別是什麼，而不是直接畫出畫面。想讓使用者點擊後跳出訊息這類「判斷」，那是 JavaScript 的工作。

</details>

---

### 問答 3 · 基礎 · {#020-intro-q3}

一個網頁從寫好到使用者看到畫面，經歷哪三個階段？

<details>
<summary>答案</summary>

1. **撰寫代碼**：前端人員用 HTML（加上 CSS、JavaScript）寫出網頁的內容與結構。
2. **瀏覽器解析（parse）**：瀏覽器讀取並處理這些代碼。
3. **瀏覽器渲染（render）**：把解析結果轉換成使用者在畫面上看到的 Web 頁面。

</details>

### 填空 1 · 基礎 · {#020-intro-f1}

HTML 的全名是「____標記語言（HyperText ____ Language）」。

<details>
<summary>答案</summary>

**超文本**標記語言（HyperText **Markup** Language）

</details>

---

### 填空 2 · 基礎 · {#020-intro-f2}

「超文本（HyperText）」之所以「超」越純文字，是因為它能加入____，以及可透過____從一份文件跳到另一份文件。

<details>
<summary>答案</summary>

- 加入：**圖片、聲音、動畫、影片等多媒體**
- 透過：**超連結**

</details>

### 卡片 1 · 基礎 · {#020-intro-c1}

**網站（Website）**

<details>
<summary>定義</summary>

在網際網路上，依照一定規則、使用 HTML 等技術做出來、用來呈現特定內容的**一組相關網頁的集合**。

</details>

---

### 卡片 2 · 基礎 · {#020-intro-c2}

**瀏覽器「渲染（render）」**

<details>
<summary>含義</summary>

把瀏覽器解析後的代碼**轉換成使用者在畫面上看到的結果**（Web 頁面）。這是網頁形成三步驟的最後一步。

</details>
