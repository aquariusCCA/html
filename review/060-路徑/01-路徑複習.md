# 路徑複習

> 回憶檢索複習。概念與用法說明見 notes/060-路徑/01-路徑：讓 HTML 找到圖片與資源.md。

## 重點摘要

- **路徑**：告訴瀏覽器去哪裡找圖片、CSS、JS 等外部資源的位置說明；路徑寫錯時 HTML 語法無誤，資源仍會載入失敗。
- **絕對路徑**：從固定根位置或完整網址出發，不依賴目前 HTML 所在位置。
  - 本機磁碟路徑（`D:\day01\images\1.jpg`）：只在本機有效，部署後通常失效。
  - 完整 URL（`https://…`）：外站資源可能被刪除、改名或防盜鏈拒絕。
  - `/` 開頭網站根目錄路徑：需伺服器環境，直接用檔案總管開啟可能不符預期。
- **相對路徑**：從目前 HTML 檔案所在位置出發。
  - `./`（可省略）→ 目前資料夾；`../` → 上一層資料夾
  - 同層：`logo.png` 或 `./logo.png`
  - 下一層：`images/logo.png`
  - 上一層：`../images/logo.png`
- **實務選用**：自己專案的資源優先用相對路徑；外部 CDN／函式庫用完整 URL；避免依賴外站圖片作為穩定資源。
- **移動 HTML 檔案後**：相對路徑的參考點隨之改變，需同步更新路徑。

## 問答

### 問答 1 · 基礎 · {#060-path-q1}

什麼是 HTML 路徑？它在 HTML 中的用途是什麼？

<details>
<summary>答案</summary>

路徑是告訴瀏覽器去哪裡找圖片、CSS、JavaScript 等外部資源的位置說明。路徑寫錯時，HTML 標籤本身可能沒有語法錯誤，但資源仍然會載入失敗。

</details>

---

### 問答 2 · 基礎 · {#060-path-q2}

絕對路徑與相對路徑最根本的差別是什麼？

<details>
<summary>答案</summary>

兩者的**參考點不同**：
- 絕對路徑：從固定根位置或完整網址出發，不依賴目前 HTML 檔案放在哪裡。
- 相對路徑：從目前 HTML 檔案所在位置出發，隨 HTML 位置不同而指向不同目標。

</details>

---

### 問答 3 · 基礎 · {#060-path-q3}

為什麼本機磁碟絕對路徑（如 `D:\day01\images\1.jpg`）不適合放在正式網頁中？

<details>
<summary>答案</summary>

這種路徑只對擁有相同磁碟路徑的電腦有意義。換到別人的電腦，對方可能沒有 D 槽，也可能沒有相同資料夾；部署到伺服器後路徑同樣通常失效。正式專案應把圖片放進專案資料夾，改用相對路徑。

</details>

---

### 問答 4 · 進階 · {#060-path-q4}

什麼是「防盜鏈」？它對直接引用外站圖片有什麼影響？

<details>
<summary>答案</summary>

防盜鏈是對方伺服器拒絕外部網站引用其圖片的機制。直接用瀏覽器打開圖片網址可能看得到，但把那張圖片的網址放到自己的網站中引用時，對方伺服器會拒絕顯示，使頁面出現破圖。

</details>

## 填空

`____` 代表目前資料夾（通常可省略）；`____` 代表上一層資料夾。

<details>
<summary>答案</summary>

`./`、`../`

</details>

相對路徑的參考點是「____」所在的位置。

<details>
<summary>答案</summary>

目前 HTML 檔案

</details>

如果 `about.html` 在 `pages` 資料夾，而圖片 `logo.png` 在上一層的 `images` 資料夾，`src` 應寫 `____/logo.png`。

<details>
<summary>反面</summary>

絕對路徑

<details>

### 卡片 4 · 進階 · {#060-path-c4}

搬動 HTML 檔案後，相對路徑為什麼可能失效？

<details>
<summary>反面</summary>

相對路徑以目前 HTML 檔案所在位置為參考點；HTML 移到新資料夾後，參考點改變，原有路徑就可能指向錯誤位置，需同步更新路徑。

</details>
