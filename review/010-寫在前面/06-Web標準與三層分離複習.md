# Web 標準與三層分離：回憶複習

> 回憶檢索複習。概念與用法說明見 notes/010-寫在前面/06-Web標準與三層分離.md。

## 重點摘要

- **Web 標準**把網頁拆成三層：**結構**（HTML）、**表現**（CSS）、**行為**（JavaScript）。
- Web 標準的目的：讓不同瀏覽器依共同規範實作，提升互通性與顯示一致性（但仍可能受版本、平台影響）。
- **三層分離的核心是職責分離**：HTML 管「有哪些內容、結構關係」；CSS 管「看起來怎樣」；JS 管「互動後發生什麼」。
- 把表現層（顏色、間距）直接寫進 HTML 標籤屬性 = 違反三層分離，長期難以維護。

---

### 問答 1 · 基礎 · {#010-webstd-q1}

Web 標準把網頁分成哪三層？各對應什麼技術？

<details>
<summary>答案</summary>

- **結構**：HTML，描述頁面的元素與內容結構（這裡有什麼、它們的層級關係）。
- **表現**：CSS，描述頁面的樣式（顏色、版面、間距等）。
- **行為**：JavaScript，描述頁面的互動與動態效果（點擊後發生什麼）。

</details>

---

### 問答 2 · 基礎 · {#010-webstd-q2}

Web 標準存在的目的是什麼？

<details>
<summary>答案</summary>

讓**不同瀏覽器依照共同的規範來實作**，藉此提升互通性與顯示結果的一致性——若沒有 Web 標準，每個瀏覽器各自定義一套規則，開發者就要為每個瀏覽器寫不同的程式碼，使用者體驗也會因瀏覽器而異。（但實際效果仍可能受瀏覽器版本、平台與功能支援程度影響。）

</details>

---

### 問答 3 · 進階 · {#010-webstd-q3}

「三層分離」的核心精神是什麼？只是把程式碼放進三個檔案就算做到了嗎？

<details>
<summary>答案</summary>

三層分離的核心是**職責分離**，不只是把程式碼分成三個檔案。

重點在於每一層只做自己該做的事：HTML 只描述「內容結構」，不去管顏色或點擊行為；CSS 只管「樣式」；JavaScript 只管「互動行為」。即使技術上把所有程式碼寫在同一個檔案裡，也應該保持職責上的分離。違反職責分離（例如把顏色直接寫進 HTML 標籤屬性）雖然短期能運作，但會讓維護變得困難。

</details>

### 填空 1 · 基礎 · {#010-webstd-f1}

Web 標準的三層：____ 負責結構、____ 負責表現、____ 負責行為。

<details>
<summary>答案</summary>

HTML；CSS；JavaScript

</details>

### 卡片 1 · 基礎 · {#010-webstd-c1}

**正面**：結構層

<details>
<summary>反面</summary>

對應 **HTML**，描述頁面的元素與內容結構——「這裡有什麼內容、它們之間的層級關係是什麼」，不管顏色或互動。

</details>

---

### 卡片 2 · 基礎 · {#010-webstd-c2}

**正面**：表現層

<details>
<summary>反面</summary>

對應 **CSS**，描述頁面的樣式——顏色、版面、間距、字體大小等一切「看起來怎樣」的設定。

</details>

---

### 卡片 3 · 基礎 · {#010-webstd-c3}

**正面**：行為層

<details>
<summary>反面</summary>

對應 **JavaScript**，描述頁面的互動與動態效果——「使用者點擊後發生什麼」「資料送出後怎麼反應」等。

</details>
