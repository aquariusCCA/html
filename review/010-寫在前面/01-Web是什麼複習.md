# Web 是什麼：回憶複習

> 回憶檢索複習。概念與用法說明見 notes/010-寫在前面/01-Web是什麼.md。

## 重點摘要

- **Web**（全球資訊網）= 由超文本鏈接構成的全球資訊系統，透過 HTTP/HTTPS 傳輸資源，網頁內容以 HTML 描述。
- Web 資源不只有網頁，還包含圖像、音訊、影片等。
- **JavaWeb** = 用 Java 在伺服器端實作 Web 應用的技術，最終仍會產生 HTML 給瀏覽器顯示。
- HTML 是使用者最終看到的內容的描述語言，是前後端的共同語言之一。

---

### 問答 1 · 基礎 · {#010-web-q1}

用自己的話說明「Web」是什麼？它與「某一個網站」有什麼不同？

<details>
<summary>答案</summary>

Web（全球資訊網）是由超文本鏈接串起來的整個全球資訊網路，透過 HTTP/HTTPS 傳輸資源，網頁內容以 HTML 描述。它不是某一個特定的網站，而是由世界各地電腦上散落的各種資源（網頁、圖像、音訊、影片等）構成的整個系統；網站只是 Web 上的一個節點，瀏覽器則是訪問它的工具。

</details>

---

### 問答 2 · 基礎 · {#010-web-q2}

在 JavaWeb 應用中，Java 程式碼主要執行在哪裡？使用者是透過什麼方式看到結果的？

<details>
<summary>答案</summary>

Java 程式碼主要執行在**伺服器端**，負責處理請求、產生回應或提供資料。使用者不會看到 Java 程式碼，而是透過**瀏覽器**看到伺服器回傳的 HTML（再加上 CSS、JavaScript）所呈現的頁面。

</details>

### 填空 1 · 基礎 · {#010-web-f1}

Web 上的資源不只有網頁，還包含 ____、____ 等多種類型。

<details>
<summary>答案</summary>

圖像、音訊（也可填「影片」）

</details>

### 卡片 1 · 基礎 · {#010-web-c1}

**正面**：HTTP / HTTPS

<details>
<summary>反面（用途）</summary>

用於在 Web 上傳輸資源的協定（傳輸協定）。瀏覽器透過 HTTP/HTTPS 向伺服器發出請求、取得網頁等資源。

</details>

---

### 卡片 2 · 基礎 · {#010-web-c2}

**正面**：HTML 在 Web 中的角色

<details>
<summary>反面</summary>

HTML 是描述 Web 網頁結構與內容的語言——當瀏覽器請求到一個「網頁」資源時，這個資源就是一份 HTML，瀏覽器依此把畫面顯示出來。

</details>
