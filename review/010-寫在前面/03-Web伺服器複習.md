# Web 伺服器：回憶複習

> 回憶檢索複習。概念與用法說明見 notes/010-寫在前面/03-Web服務器與本地遠程伺服器.md。

## 重點摘要

- **伺服器**（主機）= 提供計算服務的電腦；依服務類型可分為文件、資料庫、應用程式、Web 伺服器等。
- **Web 伺服器** = 提供 HTTP/HTTPS 服務的軟體，或承載該軟體的主機，負責回應瀏覽器等 Web 客戶端的請求。
- **本地伺服器**：把自己電腦設定成伺服器，通常只有同一區域網路（如同一 Wi-Fi）的裝置可訪問。
- **遠端伺服器**：別人提供的主機，需完成網域名稱、DNS、防火牆等設定，外部使用者才能透過網址訪問。
- 在自己電腦上打開 `index.html` ≠ 網站上線；必須有伺服器在「提供」這份檔案，別人才能訪問。

---

### 問答 1 · 基礎 · {#010-server-q1}

Web 伺服器的具體工作是什麼？

<details>
<summary>答案</summary>

Web 伺服器（可以是軟體，也可以指承載軟體的主機）負責**回應瀏覽器等 Web 客戶端的請求**，把網頁文件、圖片、下載資源等送出去。它提供的是 HTTP/HTTPS 服務。

</details>

---

### 問答 2 · 基礎 · {#010-server-q2}

本地伺服器與遠端伺服器的差異是什麼？各自適合什麼情境？

<details>
<summary>答案</summary>

- **本地伺服器**：把自己的電腦設定為伺服器，通常只有同一區域網路（例如同一 Wi-Fi）下的裝置可以訪問，適合開發階段讓同事或同學預覽。
- **遠端伺服器**：別家公司提供的主機，上傳網站後需完成網域名稱、DNS、權限或防火牆等設定，才能讓全世界的使用者透過網址（如 `www.example.com`）訪問，適合正式上線部署。

</details>

### 卡片 1 · 基礎 · {#010-server-c1}

**正面**：本地伺服器

<details>
<summary>反面</summary>

把自己的電腦設定成 Web 伺服器，對外提供服務的範圍通常侷限在同一區域網路（如同一 Wi-Fi）。同一網路下的其他裝置可透過這台電腦的 IP 訪問正在開發的網頁。

</details>
