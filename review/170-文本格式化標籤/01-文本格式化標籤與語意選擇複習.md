# 文本格式化標籤與語意選擇複習

> 來源筆記：notes/170-文本格式化標籤/01-文本格式化標籤與語意選擇.md

## 重點摘要

- 文本格式化標籤不只是在改變外觀，選用時應先判斷文字是否有語意上的重要性、強調、新增或刪除。
- `<strong>` 表示重要或需要特別注意的內容；`<em>` 表示語氣上的強調。
- `<ins>` 表示新增的內容；`<del>` 表示刪除或已移除的內容。
- `<b>`、`<i>`、`<u>`、`<s>` 也有各自語意，常見預設外觀相似，但不應只為了外觀而使用。
- 如果只是想做粗體、斜體、底線或刪除線等外觀效果，應優先使用 CSS。
- 選擇標籤時可以依序判斷：是否有語意、是哪一種語意、是否只是外觀需求。

## 問答題

### 問答 1（基礎） {#170-format-q1}

使用文本格式化標籤時，為什麼不能只看它在瀏覽器中的外觀？

<details>
<summary>答案</summary>

因為這些標籤常常帶有語意。瀏覽器可能把不同標籤顯示成類似的外觀，例如粗體或斜體，但 HTML 標籤本身可能是在表達重要、強調、新增或刪除等不同意思。
</details>

### 問答 2（基礎） {#170-format-q2}

`<strong>` 和 `<b>` 的主要差異是什麼？

<details>
<summary>答案</summary>

`<strong>` 表示內容具有重要性或需要特別注意；`<b>` 可用於吸引注意但不表示更重要的文字，例如關鍵字或產品名稱。若內容真的重要，應使用 `<strong>`；若只是外觀粗體，應考慮用 CSS。
</details>

### 問答 3（基礎） {#170-format-q3}

`<em>` 和 `<i>` 的主要差異是什麼？

<details>
<summary>答案</summary>

`<em>` 表示語氣上的強調，會影響句子的重點；`<i>` 偏向視覺上的斜體呈現，適合沒有強調語意、只是需要不同文字樣貌的情境。若只是外觀斜體，應考慮用 CSS。
</details>

### 問答 4（基礎） {#170-format-q4}

`<ins>` 和 `<del>` 分別適合表達什麼語意？

<details>
<summary>答案</summary>

`<ins>` 適合表示新增或插入的內容；`<del>` 適合表示已刪除、移除或不再有效的內容。它們不是單純的底線與刪除線外觀，而是用來標示內容變更的語意。
</details>

### 問答 5（進階） {#170-format-q5}

如果只是想讓一段文字看起來像重點、加底線或畫刪除線，但沒有語意上的重要、新增或刪除，應該怎麼做？

<details>
<summary>答案</summary>

應使用 CSS 控制外觀，例如用 class 搭配 `font-weight` 或 `text-decoration`。這樣可以避免用錯語意標籤，讓 HTML 保持描述內容結構與語意，CSS 負責視覺樣式。
</details>

## 填空題

### 填空 1（基礎） {#170-format-f1}

`<strong>` 表示內容具有 ____；`<em>` 表示語氣上的 ____。

<details>
<summary>答案</summary>

重要性；強調

</details>

### 填空 2（基礎） {#170-format-f2}

`<ins>` 表示 ____ 的內容；`<del>` 表示 ____ 的內容。

<details>
<summary>答案</summary>

新增或插入；刪除、移除或不再有效

</details>

### 填空 3（進階） {#170-format-f3}

當需求只是改變粗體、斜體、底線或刪除線外觀，而不是表達語意時，應優先使用 ____。

<details>
<summary>答案</summary>

CSS

</details>

## 翻面卡片

### 卡片 1（基礎） {#170-format-c1}

正面：語意型文本格式化標籤四件組

<details>
<summary>背面</summary>

- `<strong>`：重要內容
- `<em>`：語氣強調
- `<ins>`：新增或插入內容
- `<del>`：刪除、移除或不再有效的內容
</details>

### 卡片 2（基礎） {#170-format-c2}

正面：有各自語意但常見外觀相似的文本格式化標籤四件組

<details>
<summary>背面</summary>

- `<b>`：吸引注意但不表示更重要
- `<i>`：特定語境中的不同聲音或術語
- `<u>`：非連結的特殊標示
- `<s>`：內容已不準確或不再相關
</details>

### 卡片 3（進階） {#170-format-c3}

正面：選擇文本格式化標籤的判斷順序

<details>
<summary>背面</summary>

先判斷文字是否有語意；如果有，再選擇對應的語意標籤，例如重要用 `<strong>`、強調用 `<em>`、新增用 `<ins>`、刪除用 `<del>`。如果只是外觀需求，應使用 CSS 控制樣式。
</details>
