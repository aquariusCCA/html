# div 與語意化佈局比較複習

> 回憶檢索複習。概念與用法說明見 notes/130-語意化標籤/02-div與語意化佈局比較.md。

## 重點摘要

- `<div>` 是沒有語意的通用容器；語意化標籤是有內容角色的結構標籤。
- 同一個畫面用 `<div>` 與語意化標籤都可能顯示正常，但 HTML 結構的可理解程度不同。
- 使用 `<div>` 時語意主要依賴 class 名稱補充；語意化標籤的標籤名稱本身就說明用途。
- 判斷是否使用語意化標籤的三步準則：內容有無明確角色？有無對應語意標籤？拿掉 CSS 後結構仍可理解？
- `<section>` 代表有共同主題的一段內容，通常應能搭配標題；不應只為排版而用。
- 語意化不是把 class 名稱換成同名標籤，而是根據內容實際用途選標籤。
- 沒有明確語意需求時，繼續使用 `<div>` 是合理的。

## 問答題

### 問答 1 · 基礎 · {#130-div-q1}

`<div>` 與語意化標籤在 HTML 結構上的根本差異是什麼？

<details>
<summary>答案</summary>

`<div>` 是沒有語意的通用容器，內容角色主要靠 class 名稱補充；語意化標籤是有內容角色的結構標籤，標籤名稱本身就直接表達區塊角色。

</details>

### 問答 2 · 基礎 · {#130-div-q2}

選擇標籤時，可以依序問哪三個問題來判斷是否使用語意化標籤？

<details>
<summary>答案</summary>

一、這一塊內容是否有明確角色？二、這個角色是否有對應的語意化標籤？三、如果拿掉 CSS，這個 HTML 結構是否仍能讓人理解內容順序與區塊關係？三問皆肯定，就優先使用語意化標籤。

</details>

### 問答 3 · 進階 · {#130-div-q3}

`<section>` 適合使用與不適合使用的情況各是什麼？

<details>
<summary>答案</summary>

適合：一段具有共同主題的內容，且通常能搭配標題。不適合：只是包住一組按鈕方便排版，或沒有獨立主題的容器——這些情況使用 `<div>` 更合理。

</details>

## 填空題

### 填空 1 · 基礎 · {#130-div-f1}

`<div>` 是沒有 ____ 的通用容器；語意化標籤是有 ____ 的結構標籤。

<details>
<summary>答案</summary>

語意（兩空相同）

</details>

### 填空 2 · 基礎 · {#130-div-f2}

如果一個元素只是為了讓 CSS 好選取、讓內容置中，或製作版面外框，沒有特別語意需求，應使用 `<____>` 而非語意化標籤。

<details>
<summary>答案</summary>

`div`

</details>

## 翻面卡片

### 卡片 1 · 基礎 · {#130-div-c1}

正面：情境——單篇可獨立閱讀的文章

<details>
<summary>答案</summary>

建議使用 `<article>`，因為內容可獨立分發或重用。

</details>

### 卡片 2 · 基礎 · {#130-div-c2}

正面：情境——包住一組按鈕方便排版

<details>
<summary>答案</summary>

建議使用 `<div>`，因為只是分組與排版需求，沒有特別內容語意。

</details>

### 卡片 3 · 進階 · {#130-div-c3}

正面：「只把 class 名稱換成同名標籤」，例如看到 `<div class="nav">` 就直接改成 `<nav>`——這樣做有什麼問題？

<details>
<summary>答案</summary>

class 名稱只是樣式名稱，不等同內容語意。若裡面其實不是導覽連結，換成 `<nav>` 反而語意錯誤。應根據內容實際用途選標籤，而非依名稱外觀對換。

</details>
