# 語意化標籤入門複習

> 回憶檢索複習。概念與用法說明見 notes/130-語意化標籤/01-語意化標籤入門.md。

## 重點摘要

- 語意化就是使用能表達內容意義的 HTML 標籤，而不是只用通用容器來包版面。
- HTML 的任務是描述內容結構；畫面樣式交給 CSS，內容角色應盡量由 HTML 本身表達。
- 常見語意化標籤包含 `<header>`、`<nav>`、`<main>`、`<article>`、`<section>`、`<aside>`、`<footer>`。
- 語意化能改善程式碼可讀性與維護性，也有助於搜尋引擎與輔助技術理解頁面。
- SEO 是語意化的受益場景之一，但語意化不是只為了 SEO。
- 語意化不是禁止 `<div>`；沒有特別內容語意、只是排版或分組時，使用 `<div>` 仍合理。
- 選標籤時要根據內容意義，不是根據畫面長相。

## 問答題

### 問答 1 · 基礎 · {#130-semantic-intro-q1}

在 HTML 中，「語意化」是什麼意思？

<details>
<summary>答案</summary>

語意化就是使用能表達內容意義的 HTML 標籤，而不是只用通用容器來包版面。

</details>

### 問答 2 · 基礎 · {#130-semantic-intro-q2}

語意化標籤要解決什麼問題？

<details>
<summary>答案</summary>

語意化標籤讓 HTML 不只排得出畫面，也能說得出內容的角色，讓瀏覽器、搜尋引擎、螢幕閱讀器與工程師更容易理解頁面結構。

</details>

### 問答 3 · 基礎 · {#130-semantic-intro-q3}

語意化 HTML 主要帶來哪些好處？

<details>
<summary>答案</summary>

語意化讓程式碼更容易閱讀與維護；樣式遺失時仍保留清楚結構；有助於搜尋引擎理解頁面；也讓螢幕閱讀器等輔助技術能更好地解析頁面。

</details>

### 問答 4 · 進階 · {#130-semantic-intro-q4}

語意化是否代表完全不能使用 `<div>`？為什麼？

<details>
<summary>答案</summary>

不是。語意化不是禁止 `<div>`。當某個元素只是為了排版、分組或套樣式，沒有特別的內容語意時，使用 `<div>` 仍然合理。

</details>
