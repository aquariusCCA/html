# 水平線標籤複習

> 回憶檢索複習。概念與用法說明見 notes/150-水平線標籤/01-水平線標籤.md。

## 重點摘要

- `<hr>` 用來表示主題轉換或內容分隔，瀏覽器通常會把它顯示成一條水平線。
- `<hr>` 的重點不是「畫一條線」，而是「這裡有一個主題上的分隔」。
- 當一篇內容從一個主題切換到另一個主題時，可以使用 `<hr>` 幫助讀者辨識轉換。
- `<hr>` 是單標籤，不需要結束標籤，也不需要包住文字或其他內容。
- 在 HTML 文件中，`<hr />` 與 `<hr>` 都能運作；可依專案慣例選擇寫法。
- 如果只是想在區塊底部加裝飾線，或增加排版分隔，應交給 CSS 處理。
- HTML 負責表達內容結構，CSS 負責視覺樣式。

## 問答題

### 問答 1 · 基礎 · {#150-hr-q1}

`<hr>` 在 HTML 中代表什麼語意？

<details>
<summary>答案</summary>

`<hr>` 表示主題轉換或內容分隔；它提醒讀者接下來的內容已經進入另一個段落或主題。

</details>

### 問答 2 · 基礎 · {#150-hr-q2}

為什麼 `<hr>` 不是單純用來畫一條裝飾線？

<details>
<summary>答案</summary>

從 HTML 語意來看，`<hr>` 的重點是內容結構中的主題分隔。瀏覽器通常會把它顯示成水平線，但它不應只是視覺裝飾工具。

</details>

### 問答 3 · 基礎 · {#150-hr-q3}

什麼情境適合使用 `<hr>`？

<details>
<summary>答案</summary>

當前後內容真的進入不同主題時適合使用 `<hr>`，例如文章從正文切換到補充說明，或教學筆記從概念說明切換到延伸閱讀。

</details>

### 問答 4 · 進階 · {#150-hr-q4}

如果只是想在標題下方或卡片底部加一條線，為什麼應該用 CSS 而不是 `<hr>`？

<details>
<summary>答案</summary>

因為這只是視覺樣式，不代表內容真的發生主題切換。這種情況通常應該交給 CSS，例如用 `border-top` 或 `border-bottom` 做視覺分隔。

</details>

<details>

單標籤

</details>

### 填空 2 · 基礎 · {#150-hr-f2}

如果只是做視覺分隔，CSS 可以用 ____ 或 ____ 處理。

<details>
<summary>答案</summary>

`border-top`、`border-bottom`

</details>
