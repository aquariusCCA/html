# pre 元素與預格式化文本複習

> 回憶檢索複習。概念與用法說明見 notes/160-預格式化文本標籤/01-pre-元素與預格式化文本.md。

## 重點摘要

- `<pre>` 用來呈現「預先排好格式的文字」，代表這段文字已經有自己的排版格式。
- `<pre>` 會保留內容中的空格、Tab 與換行。
- 一般 HTML 內容會把連續空白、換行與縮排折疊成較簡單的顯示結果。
- 瀏覽器顯示 `<pre>` 內容時，通常會使用等寬字體，適合展示程式碼或需要對齊的文字。
- `<pre>` 適合用來展示 CSS、純文字程式碼、需要固定換行與縮排的文字。
- `<pre>` 會保留空白與換行，但不會讓 HTML 標籤自動變成純文字。
- 如果要展示 HTML 標籤本身，必須把 `<`、`>`、`&` 等特殊字元轉義。
- 不適合只因為想讓一般文章換行就使用 `<pre>`；一般文章應使用更符合語意的元素，再交給 CSS 控制樣式。

## 問答題

### 問答 1 · 基礎 · {#160-pre-q1}

`<pre>` 的用途與語意是什麼？

<details>
<summary>答案</summary>

`<pre>` 用來呈現預先排好格式的文字，代表這段文字已經有自己的排版格式，瀏覽器應該盡量照原本的空白與換行呈現。

</details>

### 問答 2 · 基礎 · {#160-pre-q2}

`<pre>` 會保留內容中的哪些格式特徵？

<details>
<summary>答案</summary>

`<pre>` 會保留內容中的空格、Tab 與換行。

</details>

### 問答 3 · 基礎 · {#160-pre-q3}

為什麼用一般 `<p>` 展示程式碼不合適？

<details>
<summary>答案</summary>

一般段落元素不負責保留程式碼排版，瀏覽器會把多餘空白與換行折疊，因此讀者看到的縮排與換行可能和原始內容不同。

</details>

### 問答 4 · 進階 · {#160-pre-q4}

如果想在頁面上顯示 HTML 原始碼，為什麼不能直接把標籤寫在 `<pre>` 裡？

<details>
<summary>答案</summary>

`<pre>` 會保留空白與換行，但不會讓 HTML 標籤自動變成純文字。直接寫在 `<pre>` 裡的標籤仍可能被瀏覽器當成真正的 HTML 元素解析，因此要先轉義特殊字元。

</details>

### 填空 2 · 基礎 · {#160-pre-f2}

<details>

</details>
