# 塊級元素與行內元素排版差異複習

> 回憶檢索複習。概念與用法說明見 notes/070-HTML塊級元素與行內元素/01-塊級元素與行內元素的排版差異.md。

## 重點摘要

- 塊級元素偏向建立一整塊內容區域，常見預設行為是另起一行，並由上往下排列。
- 行內元素偏向在一行文字中包住片段或插入小元件，通常和文字一起流動，空間不足才換行。
- 常見 block non-replaced 元素沒有設定 `width` 時，會用 `width: auto` 填滿可用水平空間，但這不完全等於 `width: 100%`。
- 一般 non-replaced inline 元素的寬度由內容決定，`span`、`em` 這類元素不能直接靠 `width`、`height` 改變主要尺寸。
- `img`、部分表單控制項，以及 `inline-block` 是行內內容可控尺寸的重要例外。
- 選標籤時要先看語意與內容模型，再用 CSS 調整排版，不能只靠畫面是否換行判斷。

### 問答 1 · 基礎 · {#070-block-inline-q1}

塊級元素與行內元素在預設排版上最核心的差異是什麼？

<details>
<summary>答案</summary>

塊級元素通常會另起一行，形成一整塊內容區域，並由上往下排列；行內元素通常和文字一起排在同一行，直到空間不夠才換行。

</details>

### 問答 2 · 進階 · {#070-block-inline-q2}

為什麼不能把塊級元素的預設寬度簡化成永遠等於 `width: 100%`？

<details>
<summary>答案</summary>

常見 block non-replaced 元素預設填滿可用水平空間通常來自 `width: auto`，不是直接等於 `width: 100%`。當元素有 margin、border、padding 時，兩者在盒模型計算上可能不同。

</details>

### 問答 3 · 基礎 · {#070-block-inline-q3}

為什麼不能只用「會不會換行」來決定 HTML 標籤？

<details>
<summary>答案</summary>

因為 block 與 inline 描述的是預設排版行為，但 HTML 標籤本身還有語意。實務上應先選語意正確的標籤，再用 CSS 調整排版。

</details>

學習塊級元素與行內元素時，要先理解元素在 ____ 中預設怎麼排列。

<details>
<summary>答案</summary>

</details>

### 填空 2 · 基礎 · {#070-block-inline-f2}

一般 `span`、`em` 這類 ____ 元素，不能直接用 `width`、`height` 改變盒子的主要尺寸。

<details>
<summary>答案</summary>

non-replaced inline

</details>

### 卡片 1 · 基礎 · {#070-block-inline-c1}

正面：`div`

<details>
<summary>答案</summary>

常用來建立無語意的版面容器，屬於常見塊級元素。

</details>

### 卡片 2 · 基礎 · {#070-block-inline-c2}

正面：`span`

<details>
<summary>答案</summary>

常用來包住一段文字中的某個片段，是沒有特殊語意的行內容器。

</details>

### 卡片 3 · 進階 · {#070-block-inline-c3}

正面：`img`、部分表單控制項、`inline-block`

<details>
<summary>答案</summary>

它們是行內內容可以有可控尺寸的重要例外；不要把「行內元素」簡化成「一定不能設定寬高」。

</details>
