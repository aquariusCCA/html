# 內容模型與基本巢狀規則複習

> 回憶檢索複習。概念與用法說明見 notes/070-HTML塊級元素與行內元素/02-HTML元素的內容模型與基本巢狀規則.md。

## 重點摘要

- HTML 巢狀是否合法，要看元素允許的內容模型；block、inline 只能幫助理解排版，不能取代合法性判斷。
- `div` 是沒有特定語意的通用區塊容器，可包含多數 flow content。
- `span` 是沒有特定語意的行內容器，通常只應包含 phrasing content。
- 不是所有塊級元素都能放任何內容，也不是所有行內元素都有完全相同的限制。
- `p`、`h1` 到 `h6`、`a` 都有特定限制，不能只用 block/inline 規則概括。
- 瀏覽器能顯示錯誤 HTML，不代表原始結構正確；錯誤巢狀可能讓 DOM、CSS、JavaScript 或輔助技術理解出問題。

### 問答 1 · 基礎 · {#070-content-model-q1}

為什麼不能只用「塊級元素」與「行內元素」判斷所有 HTML 巢狀規則？

<details>
<summary>答案</summary>

因為 block、inline 主要是排版直覺，真正決定元素能放什麼內容的是 HTML 規範中的 content model。

</details>

### 問答 2 · 進階 · {#070-content-model-q2}

`div` 可以包住另一個 `div`，是否代表 `p` 也可以包住 `div`？為什麼？

<details>
<summary>答案</summary>

不代表。`div` 是通用 flow 容器，可包含多數 flow content；但 `p` 代表段落，有自己的內容限制，不適合包含 `div` 這類區塊內容。

</details>

### 問答 3 · 基礎 · {#070-content-model-q3}

瀏覽器會修正錯誤 HTML，為什麼仍然不能把「能顯示」當成「合法」？

<details>
<summary>答案</summary>

瀏覽器會盡量修正錯誤 HTML 讓頁面可顯示，但這不代表原始寫法正確。修正後的 DOM 可能和想像不同，造成 CSS 選擇器、JavaScript 節點結構、輔助技術或搜尋引擎理解錯誤。

</details>

### 填空 1 · 基礎 · {#070-content-model-f1}

HTML 巢狀規則要看元素允許的 ____；block、inline 不能取代合法性判斷。

<details>
<summary>答案</summary>

內容模型

</details>

### 填空 2 · 基礎 · {#070-content-model-f2}

`span` 這類行內容器通常只應包含 ____，例如文字、`em`、`strong`、`a`、`input` 等。

<details>
<summary>答案</summary>

phrasing content

</details>

### 卡片 1 · 基礎 · {#070-content-model-c1}

正面：`div`

<details>
<summary>答案</summary>

沒有特定語意的通用區塊容器，常用來分組、排版或套用 CSS，可包含多數 flow content。

</details>

### 卡片 2 · 基礎 · {#070-content-model-c2}

正面：`span`

<details>
<summary>答案</summary>

沒有特定語意的行內容器，常用來包住文字中的某一小段內容，通常只應包含 phrasing content。

</details>
