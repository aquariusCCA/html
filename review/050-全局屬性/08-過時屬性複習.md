# 過時屬性複習：contextmenu 與 dropzone

> 回憶檢索複習。概念與用法說明見 notes/050-全局屬性/08-過時屬性-contextmenu-dropzone.md。

---

## 重點摘要

| 舊屬性 | 曾經用途 | 現狀 | 現代替代方向 |
|---|---|---|---|
| `contextmenu` | 指定元素的上下文選單 | **已過時** | 監聽 JavaScript 的 `contextmenu` 事件 |
| `dropzone` | 指定拖放資料的處理方式（copy/move/link） | **已過時** | 使用 Drag and Drop API 拖放事件（dragenter、dragover、drop） |

- 遇到舊教材或舊程式碼中出現這兩個屬性，應視為舊用法，不要依賴它們建立現代功能。

---

## 問答 1 · 基礎 · {#050-old-q1}

`contextmenu` 作為 HTML 屬性，是否仍適合現代開發使用？現代替代方向是什麼？

<details>
<summary>答案</summary>

**不適合**。`contextmenu` 屬性已過時，現代 HTML 不應再依賴它。

若需要處理右鍵選單，應改用 **JavaScript 監聽 `contextmenu` 事件**（注意：`contextmenu` 在這裡是事件名稱，不是 HTML 屬性）。

</details>

---

## 問答 2 · 基礎 · {#050-old-q2}

`dropzone` 屬性已過時，現代拖放行為應改用什麼？

<details>
<summary>答案</summary>

改用 **Drag and Drop API 事件**，包含：
- `dragenter`：拖曳元素進入目標區域
- `dragover`：拖曳元素停留在目標上方
- `drop`：在目標上放下元素

</details>
