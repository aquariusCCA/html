# div 與 span 盒子標籤複習

> 回憶檢索複習。概念與用法說明見 notes/120-盒子標籤/01-div與span盒子標籤.md。

## 重點摘要

- `<div>` 和 `<span>` 都是沒有語意的容器標籤，用來包住內容，方便 CSS 排版、套樣式或 JavaScript 操作。
- `<div>` 是沒有語意的區塊容器，常被當成「大盒子」，適合包住一整塊內容。
- `<span>` 是沒有語意的行內容器，常被當成「小盒子」，適合包住一小段文字或行內內容。
- `<div>` 預設是區塊元素，通常會獨占一行；`<span>` 預設是行內元素，不會強制前後換行。
- 「區塊」和「行內」是預設顯示方式，之後可以用 CSS 的 `display` 改變排列方式。
- 如果內容本身有明確語意，應優先使用對應的語意標籤，而不是直接用 `<div>` 或 `<span>`。
- 常見誤用包括整頁都只用 `div`、用 `span` 包住大區塊、以為 `span` 永遠不會換行。

## 問答題

### 問答 1 · 基礎 · {#120-div-span-q1}

`<div>` 和 `<span>` 的共同點是什麼？

<details>
<summary>答案</summary>

它們都是沒有語意的容器標籤，常用來把內容包起來，方便之後用 CSS 排版、套樣式，或用 JavaScript 操作。

</details>

### 問答 2 · 基礎 · {#120-div-span-q2}

`<div>` 和 `<span>` 在預設排列方式與常見用途上有什麼差異？

<details>
<summary>答案</summary>

`<div>` 預設是區塊元素，通常會獨占一行，常用來包住一塊內容、做版面分區、建立排版容器；`<span>` 預設是行內元素，不會強制前後換行，常用來包住一小段文字、局部套樣式。

</details>

### 問答 3 · 基礎 · {#120-div-span-q3}

什麼情況適合使用 `<div>`？什麼情況適合使用 `<span>`？

<details>
<summary>答案</summary>

要包住一整塊內容，例如卡片、區域、版面外框，可以考慮使用 `<div>`；只想包住一句話中的某幾個字，可以考慮使用 `<span>`。

</details>

### 問答 4 · 進階 · {#120-div-span-q4}

為什麼不應該把整個頁面都只用 `div` 來寫？

<details>
<summary>答案</summary>

整個頁面都只用 `div` 時，畫面可能看得出來，但 HTML 結構沒有清楚表達內容意義。瀏覽器、搜尋引擎、輔助工具和其他工程師都比較難判斷哪一段是標題、哪一段是正文。有明確語意時應使用對應標籤。

</details>

<details>

</details>
