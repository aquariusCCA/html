# HTML 注釋複習

> 回憶檢索複習。概念與用法說明見 notes/040-HTML基本結構標籤/03-HTML注釋.md。

## 重點摘要

- HTML 注釋是寫在原始碼中的說明文字，瀏覽器不會把它顯示成頁面內容。
- HTML 注釋以 `<!--` 開頭，以 `-->` 結束。
- 注釋裡面的內容不會變成頁面文字，也不會作為 HTML 標籤執行。
- 注釋可用於說明區塊用途、補充程式碼不容易看出的意圖、提醒後續維護者相關背景。
- 好注釋應補充意圖或區塊用途；壞注釋只是重複明顯資訊。
- 注釋仍可能存在於瀏覽器可查看的原始碼中，不應放入密碼、金鑰、內部帳號或未公開資料。
- 注釋可以輔助維護，但不能取代合適的 HTML 標籤。

### 問答 1 · 基礎 · {#040-comments-q1}

HTML 注釋的基本語法是什麼？它的用途是什麼？

<details>
<summary>答案</summary>

HTML 注釋以 `<!--` 開頭，以 `-->` 結束。它是寫在原始碼中的說明文字，用來留給開發者閱讀，瀏覽器不會把它顯示成頁面內容。

</details>

### 問答 2 · 基礎 · {#040-comments-q2}

如果把 `<h1>` 放在 HTML 注釋裡，瀏覽器會怎麼處理？

<details>
<summary>答案</summary>

瀏覽器不會把它當成真正的標題顯示，也不會把注釋中的內容作為 HTML 標籤執行。

</details>

### 問答 3 · 基礎 · {#040-comments-q3}

為什麼 HTML 注釋裡不應該放密碼、金鑰或內部帳號？

<details>
<summary>答案</summary>

注釋不會顯示在頁面畫面上，但仍然可能存在於瀏覽器可查看的原始碼中，所以不代表安全。

</details>

### 填空 1 · 基礎 · {#040-comments-f1}

HTML 注釋以 ____ 開頭，以 ____ 結束。

<details>
<summary>答案</summary>

`<!--`；`-->`

</details>

### 填空 2 · 基礎 · {#040-comments-f2}

如果忘記寫注釋結尾的 ____，後面的內容可能都被當成注釋，導致頁面顯示異常。

<details>
<summary>答案</summary>

`-->`

</details>

### 卡片 1 · 基礎 · {#040-comments-c1}

正面：好注釋

<details>
<summary>反面</summary>

補充程式碼本身不容易看出的意圖，或說明附近區塊的用途。

</details>

### 卡片 2 · 基礎 · {#040-comments-c2}

正面：壞注釋

<details>
<summary>反面</summary>

只是重複明顯資訊，例如在 `<h1>` 前寫「這是一個 h1」，沒有提供額外價值。

</details>
