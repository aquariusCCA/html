# HTML 字符實體基本概念複習

> 回憶檢索複習。概念與用法說明見 notes/260-字符實體/01-字符實體基本概念.md。

## 重點摘要

- 字符實體是用一段特殊寫法，讓 HTML 正確顯示特殊符號或不方便直接輸入的字符。
- 常見命名字符實體由 `&`、實體名稱、`;` 三部分組成，例如 `&copy;`。
- HTML 也支援數字字符引用，例如 `&#169;` 與 `&#xA9;` 都可以表示版權符號。
- 初學者優先熟悉 `&nbsp;`、`&lt;`、`&gt;`、`&amp;`。
- `&nbsp;` 是不換行空格，不會像一般空格那樣被合併，可用來顯示特殊空格效果。
- 要在頁面上顯示 HTML 標籤文字，通常需要把 `<` 和 `>` 寫成 `&lt;` 和 `&gt;`。
- 命名字符實體通常要以分號 `;` 結尾，漏掉分號可能導致解析結果不穩定，也會降低可讀性。
- 字符實體不是排版系統；段落間距、元素位置、文字對齊應交給 CSS 處理。
- 一般 HTML 文字排版中，連續空格與換行通常會被合併，但 `<pre>`、`<textarea>` 與 CSS `white-space` 可能改變空白顯示方式。

## 問答題

### 問答 1 · 基礎 · {#260-entities-q1}

HTML 字符實體是什麼？

<details>
<summary>答案</summary>

字符實體是用一段特殊寫法，讓 HTML 正確顯示特殊符號或不方便直接輸入的字符。

</details>

### 問答 2 · 基礎 · {#260-entities-q2}

常見的命名字符實體由哪三部分組成？

<details>
<summary>答案</summary>

常見命名字符實體由 `&`、實體名稱、`;` 三部分組成，例如 `&copy;`。

</details>

### 問答 3 · 基礎 · {#260-entities-q3}

如果要在頁面文字中顯示 `<p>Hello</p>`，應該如何避免瀏覽器把它當成真正的 HTML 標籤解析？

<details>
<summary>答案</summary>

應把 `<` 和 `>` 寫成字符實體，例如 `&lt;p&gt;Hello&lt;/p&gt;`。瀏覽器看到的是字符實體，就不會把它當成真正的 `<p>` 標籤執行。

</details>

### 問答 4 · 進階 · {#260-entities-q4}

為什麼不建議用大量 `&nbsp;` 來做排版？

<details>
<summary>答案</summary>

字符實體不是排版系統。若只是想控制段落間距、元素位置或文字對齊，應該使用 CSS 的 `margin`、`padding`、`gap` 等排版方式，而不是堆很多個 `&nbsp;`。

</details>

### 問答 5 · 進階 · {#260-entities-q5}

在 HTML 中，連續空白是否所有情況都會被合併顯示？請說明 notes 提到的例外方向。

<details>
<summary>答案</summary>

不是所有情況都相同。一般 HTML 文字排版中，連續空格與換行通常會被合併顯示為一個空格；但 `<pre>` 會保留文字中的空白與換行，`<textarea>` 會保留輸入內容的空白與換行，CSS 的 `white-space` 設定也可能改變空白顯示方式。

</details>

## 翻面卡片

### 卡片 1 · 基礎 · {#260-entities-c1}

正面：`&nbsp;`

<details>
<summary>答案</summary>

顯示不會被合併的空格，也就是 non-breaking space（不換行空格）。

</details>

### 卡片 2 · 基礎 · {#260-entities-c2}

正面：`&lt;` / `&gt;`

<details>
<summary>答案</summary>

分別顯示 `<` 和 `>`，可用來在文字中顯示 HTML 標籤符號，避免被瀏覽器當成真正的標籤解析。

</details>

### 卡片 3 · 基礎 · {#260-entities-c3}

正面：`&#169;` / `&#xA9;`

<details>
<summary>答案</summary>

這兩種是數字字符引用，和 `&copy;` 一樣可以表示版權符號 `©`。

</details>
