# HTML 簡介 - 複習材料

## 1. 章節總覽

### 本章主題

本章是 HTML 學習的入口，重點在於理解 HTML 如何描述網頁內容與結構、瀏覽器如何把 HTML 轉成可閱讀的網頁，以及 HTML 文件骨架、標籤、元素與元素關係如何影響後續的 CSS、JavaScript、SEO 與可訪問性。

### 學完本章後應該能夠

- 說明網頁、網站與 HTML 之間的關係。
- 解釋 HTML 為什麼是標記語言，而不是程式語言。
- 描述瀏覽器將 HTML 文件顯示成網頁的基本流程。
- 辨識 HTML5、HTML4 與 XHTML 文件宣告的差異。
- 寫出基本 HTML5 文件骨架。
- 分辨標籤、元素、雙標籤、單標籤與元素之間的關係。
- 說明良好 HTML 結構對 CSS、JavaScript、SEO 與可訪問性的影響。

---

## 2. 核心重點摘要

- 網頁通常是一份 HTML 文件；網站則是多個相關網頁與資源組成的集合。
- HTML 是 HyperText Markup Language，中文常稱為「超文本標記語言」。
- HTML 的任務是用標籤標記內容，描述網頁的內容與結構。
- HTML 不是程式語言，因為它不主要負責流程控制、條件判斷或複雜運算。
- 瀏覽器會讀取 HTML，解析標籤與內容，建立 DOM，再套用 CSS、執行 JavaScript，最後渲染成使用者看到的頁面。
- 現代網頁開發通常使用 HTML5 的簡潔文件宣告：`<!doctype html>`。
- 基本 HTML5 文件骨架包含 `doctype`、`html`、`head` 與 `body`。
- `head` 放文件資訊，`body` 放使用者主要看到的頁面內容。
- HTML 元素可形成父子、兄弟、祖先、後代、嵌套與並列關係。
- 良好 HTML 結構能讓 CSS 套用更清楚、JavaScript 操作更準確，也有助於 SEO 與可訪問性。

## 4. 必背觀念

| 觀念 | 複習重點 |
| -- | ---- |
| 網頁 | 網站中的一個頁面，通常是一份 HTML 文件，常見副檔名是 `.html` 或 `.htm`。 |
| 網站 | 多個相關網頁與資源組成的集合。 |
| HTML | 用來描述網頁內容與結構的超文本標記語言。 |
| 超文本 | 不只放文字，也能包含圖片、音訊、影片、動畫等多媒體；也能透過超連結連到其他文件。 |
| HTML 不是程式語言 | HTML 不主要負責流程控制、條件判斷或複雜運算。 |
| 瀏覽器角色 | 讀取 HTML 文字檔，解析標籤與內容，建立 DOM 並渲染成頁面。 |
| DOM | 瀏覽器根據 HTML 建立出的物件模型，CSS 與 JavaScript 多數情況下基於 DOM 運作。 |
| HTML5 宣告 | 現代 HTML 開發通常使用 `<!doctype html>`。 |
| `head` | 放文件資訊，例如編碼、標題、外部資源連結等，通常不直接顯示在頁面主體。 |
| `body` | 放使用者在頁面上主要看到的內容。 |
| 標籤 | `<p>` 或 `</p>` 這類標記本身。 |
| 元素 | 由開始標籤、內容與結束標籤形成的完整結構。 |
| 雙標籤 | 有開始標籤與結束標籤，例如 `h1`、`p`。 |
| 單標籤 | 特定不需要包住文字內容的元素，例如 `meta`、`img`、`br`、`input`。 |
| 語意化 | 根據內容意義選擇元素，而不是只看畫面效果。 |

## 5. 易混淆觀念比較

### 5.1 網頁與網站

| 比較項目 | 網頁 | 網站 |
| ---- | ---- | ---- |
| 定義 | 網站中的一個頁面，通常是一份 HTML 文件。 | 多個相關網頁與資源組成的集合。 |
| 使用時機 | 指單一頁面，例如首頁、文章頁、關於我們頁。 | 指整個教學網站或一組互相關聯的頁面與資源。 |
| 注意事項 | 一個網頁可包含文字、圖片、影片、連結、表單等內容。 | 網站通常包含多個網頁，不是單一文件。 |
| 常見錯誤 | 把單一 HTML 頁面稱成整個網站。 | 把網站理解成只有一個網頁。 |

### 5.2 HTML、CSS 與 JavaScript

| 比較項目 | HTML | CSS | JavaScript |
| ---- | ---- | ---- | ---- |
| 定義 | 描述網頁內容與結構。 | 套用網頁樣式。 | 處理互動邏輯與 DOM 操作。 |
| 使用時機 | 標記標題、段落、圖片、連結、表單等內容。 | 根據 HTML 元素、類別、屬性或階層關係設定外觀。 | 透過 DOM API 改變內容、屬性或事件行為。 |
| 注意事項 | HTML 本身不主要負責流程控制或複雜運算。 | CSS 需要清楚的 HTML 結構作為套用目標。 | JavaScript 通常操作 HTML 解析後形成的 DOM 節點。 |
| 常見錯誤 | 把 HTML 當成程式語言。 | 用 HTML 標題元素控制視覺大小。 | 不理解元素關係就操作節點，容易選錯目標。 |

### 5.3 標籤與元素

| 比較項目 | 標籤 | 元素 |
| ---- | ---- | ---- |
| 定義 | `<p>` 或 `</p>` 這種標記本身。 | 開始標籤、內容與結束標籤組成的完整結構。 |
| 使用時機 | 討論 HTML 標記的符號。 | 討論頁面中的完整結構單位。 |
| 注意事項 | 標籤通常由 `<`、`>`、`/` 與標籤名稱組成。 | 元素可以彼此嵌套或並列。 |
| 常見錯誤 | 把單一開始標籤當成完整元素。 | 忘記元素包含內容與結束標籤。 |

### 5.4 雙標籤與單標籤

| 比較項目 | 雙標籤 | 單標籤 |
| ---- | ---- | ---- |
| 定義 | 有開始標籤與結束標籤。 | 特定不需要包住文字內容的元素。 |
| 使用時機 | 用於包住內容，例如標題或段落。 | 用於特定元素，例如 `meta`、`img`、`br`、`input`。 |
| 注意事項 | 一般雙標籤元素仍應寫出結束標籤。 | 單標籤不是任意省略結束標籤。 |
| 常見錯誤 | 遺漏結束標籤造成解析不如預期。 | 以為所有元素都可以改寫成單標籤。 |

### 5.5 `head` 與 `body`

| 比較項目 | `head` | `body` |
| ---- | ---- | ---- |
| 定義 | 放置文件資訊。 | 放置頁面主體內容。 |
| 使用時機 | 放編碼、標題、外部資源連結等。 | 放使用者主要看到的文字、標題、段落、連結等內容。 |
| 注意事項 | 通常不直接顯示在頁面主體。 | 內容會構成使用者閱讀與操作的頁面。 |
| 常見錯誤 | 把應顯示的頁面內容放進 `head`。 | 忘記將可見內容放在 `body` 中。 |

## 6. 常見錯誤與注意事項

| 錯誤或注意事項 | 說明 |
| ------- | -- |
| 把 HTML 當成程式語言 | HTML 是標記語言，主要描述內容與結構；互動邏輯通常由 JavaScript 處理。 |
| 把網站與網頁混為一談 | 網站是集合，網頁是單頁。 |
| 省略基本文件骨架 | 缺少 `doctype`、`html`、`head`、`body` 會讓文件結構不完整。 |
| 沒有設定字元編碼 | 中文或特殊字元可能顯示錯誤，應在 `head` 中設定 `<meta charset="UTF-8">`。 |
| 用標題元素控制字體大小 | `h1` 到 `h6` 表示文件標題層級，不是單純的樣式工具。 |
| 所有內容都用 `div` 包起來 | `div` 沒有語意；過度使用會降低可讀性與可訪問性。 |
| 誤用單標籤 | 只有特定 void elements 使用單標籤形式，不是所有元素都能省略結束標籤。 |
| 巢狀結構混亂 | 父子與兄弟關係錯誤會影響 DOM、CSS 與 JavaScript。 |
| 忽略 `lang` | `lang="zh-Hant"` 有助於瀏覽器、搜尋引擎與輔助工具理解頁面語言。 |
| 忽略語意化 | HTML 重點是內容在文件中的意義，不是畫面看起來像什麼。 |

## 9. 判斷題

1. （ ）HTML 主要負責流程控制、條件判斷與複雜運算。
2. （ ）網頁通常可以理解成網站中的一個頁面。
3. （ ）HTML 原始文件本身只是文字檔，瀏覽器會解析後渲染成畫面。
4. （ ）現代初學與一般前端開發通常優先使用 HTML5 的簡潔宣告。
5. （ ）`head` 主要放使用者在頁面中看到的主體內容。
6. （ ）`<title>` 可以設定瀏覽器分頁標題，也常被搜尋結果參考。
7. （ ）所有 HTML 元素都可以省略結束標籤。
8. （ ）`<p>` 是標籤，而 `<p>這是一段文字。</p>` 是元素。
9. （ ）父子、兄弟、祖先與後代關係會影響 CSS 與 JavaScript 的操作。
10. （ ）只要畫面看起來對，標題元素就可以任意拿來控制字體大小。

### 答案與解析

1. 錯。HTML 不主要負責流程控制、條件判斷或複雜運算；它的核心任務是標記內容與結構。
2. 對。網站通常由多個相關網頁與資源組成，網頁是其中一個頁面。
3. 對。瀏覽器會讀取 HTML 文字、解析標籤與內容，再渲染成畫面。
4. 對。本章指出現代初學與一般前端開發通常優先使用 `<!doctype html>`。
5. 錯。`head` 放文件資訊；`body` 才放使用者主要看到的內容。
6. 對。`title` 設定瀏覽器分頁標題，也常被搜尋結果參考。
7. 錯。只有特定單標籤元素屬於這類結構，一般雙標籤元素仍應寫出結束標籤。
8. 對。標籤是標記本身，元素是包含開始標籤、內容與結束標籤的完整結構。
9. 對。這些關係會影響 CSS 選擇器、JavaScript DOM 操作與結構維護。
10. 錯。`h1` 到 `h6` 表示文件標題層級，視覺大小應交給 CSS 控制。

## 10. 簡答題

1. 請說明網頁與網站的差異。
2. 請說明為什麼 HTML 是標記語言，而不是程式語言。
3. 請描述瀏覽器如何將 HTML 文件變成使用者看到的網頁。
4. 請寫出基本 HTML5 文件骨架中應出現的主要結構。
5. 請說明 `head` 與 `body` 的差異。
6. 請說明標籤與元素的差異。
7. 請說明雙標籤與單標籤的差異，並各舉本章出現過的例子。
8. 請說明父元素、子元素、祖先元素、後代元素與兄弟元素如何區分。
9. 為什麼不應該所有內容都只用 `div` 包起來？
10. 為什麼良好的 HTML 結構會影響 CSS、JavaScript、SEO 與可訪問性？

### 參考答案

1. 網頁是網站中的單一頁面，通常是一份 HTML 文件；網站則是多個相關網頁與資源組成的集合。
2. HTML 用標籤描述內容與結構，不主要負責流程控制、條件判斷或複雜運算，所以它是標記語言，不是程式語言。
3. 前端人員撰寫 HTML、CSS、JavaScript 後，瀏覽器讀取檔案，解析 HTML 並建立 DOM，再套用 CSS、執行 JavaScript，最後渲染成使用者看到的 Web 頁面。
4. 基本 HTML5 文件骨架包含 `<!doctype html>`、`<html lang="zh-Hant">`、`<head>`、`<meta charset="UTF-8">`、`<title>` 與 `<body>`。
5. `head` 放文件資訊，例如編碼、標題、外部資源連結，通常不直接顯示在頁面主體；`body` 放使用者主要看到的頁面內容。
6. 標籤是 `<p>` 或 `</p>` 這類標記本身；元素是開始標籤、內容與結束標籤形成的完整結構。
7. 雙標籤有開始標籤與結束標籤，例如 `h1`、`p`；單標籤是特定不需要包住文字內容的元素，例如 `meta`、`img`、`br`、`input`。
8. 父元素是直接包住其他元素的元素；子元素是直接被包住的元素；祖先元素是上層更高階的元素；後代元素是某元素內部各層包含的元素；兄弟元素是同屬一個父元素下的同層元素。
9. 因為 `div` 沒有語意，過度使用會讓結構不清楚，降低可讀性與可訪問性；有合適語意元素時應優先使用。
10. CSS 需要清楚結構作為套用目標，JavaScript 常透過 DOM API 操作節點；語意清楚的 HTML 也能幫助搜尋引擎與輔助工具理解頁面。

## 12. 考前快速複習版

### 必看重點

* 網頁是單一 HTML 頁面；網站是多個網頁與資源的集合。
* HTML 用標籤描述內容與結構，是標記語言，不是程式語言。
* 瀏覽器會讀取 HTML、建立 DOM、套用 CSS、執行 JavaScript，最後渲染頁面。
* 現代 HTML5 宣告是 `<!doctype html>`。
* 基本骨架：`doctype`、`html`、`head`、`meta charset`、`title`、`body`。
* `head` 放文件資訊；`body` 放頁面主要可見內容。
* 標籤是標記本身；元素是完整結構。
* 雙標籤有開始與結束標籤；單標籤只適用於特定元素。
* 元素關係包含父子、兄弟、祖先、後代、嵌套、並列。
* 良好 HTML 結構會影響 CSS、JavaScript、SEO 與可訪問性。

### 必記關鍵字

* HTML
* HyperText Markup Language
* 超文本標記語言
* `<!doctype html>`
* `html`
* `head`
* `meta charset`
* `title`
* `body`
* 標籤
* 元素
* 雙標籤
* 單標籤
* DOM
* 語意化
* SEO
* 可訪問性

### 最容易錯的地方

* 把 HTML 當成程式語言。
* 把網站與網頁混為一談。
* 省略基本 HTML 文件骨架。
* 忘記設定 `<meta charset="UTF-8">`。
* 把 `head` 與 `body` 的用途搞反。
* 把標籤與元素當成完全相同。
* 以為所有元素都可以省略結束標籤。
* 用 `h1` 到 `h6` 只是為了控制字體大小。
* 所有內容都用 `div`，忽略語意化元素。
* 巢狀結構混亂，導致 CSS 與 JavaScript 操作變困難。
