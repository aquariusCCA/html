<svg> 與 <canvas> 圖形 Demo

這個範例把第 42 章筆記中的選型流程、SVG 結構化向量圖、Canvas 像素畫布、可存取名稱與同步 HTML 資料放在同一頁觀察。重點是分清「保留圖形結構」和「用程式繪製像素」。

<img> 顯示既有照片、商品圖、文章配圖或固定圖片。
<svg> 保留圖形元素結構，適合圖示、Logo、流程圖與少量互動圖表。
viewBox 定義 SVG 內部座標系，讓圖形更容易縮放。
<canvas> 用 JavaScript 在畫布上繪製像素，適合高頻重繪。
a11y 重要資訊不要只畫在 Canvas，應提供文字或資料表。

<img>

最適合普通圖片或固定圖檔。若是 SVG 檔但不需操作內部元素，也可以用 <img src="icon.svg">。

SVG

圖形元素仍在 DOM / SVG 節點中，可被 CSS 和 JavaScript 選取、改色、綁事件。

Canvas

畫完後主要是像素結果。適合遊戲、簽名板、粒子動畫、影像處理或大量動態繪製。

Canvas 繪圖控制

按鈕會重新繪製同一塊畫布。Canvas 裡的柱狀圖是像素，不能像 SVG 的 <rect> 那樣被 CSS 逐一選到。

const canvas = document.querySelector("#salesCanvas");
const ctx = canvas.getContext("2d");

ctx.fillStyle = "#2f6f56";
ctx.fillRect(x, y, width, height);

5. 常見選擇對照

需求 建議 原因
文章配圖、照片 <img> 語意直接，搭配 alt 即可描述圖片。
可縮放圖示、Logo、線條圖 SVG 向量清晰，圖形結構可保留。
需要點擊單一圖形區塊 內嵌 SVG 可以選到 <circle>、<path> 等元素。
大量高頻更新的畫面 Canvas 適合用程式反覆重繪像素畫面。
重要文字或資料 HTML / SVG，或 Canvas 搭配同步 HTML Canvas 文字是像素，不等於 DOM 文字。

6. 回查清單

  • 普通圖片優先用 <img>。
  • 圖示、Logo、流程圖、少量互動圖形優先考慮 SVG。
  • viewBox 定義 SVG 內部座標系，和頁面顯示尺寸不是同一件事。
  • Canvas 適合大量、高頻、像素導向的繪製。
  • SVG 有資訊時提供 <title>、<desc> 或 ARIA 名稱；純裝飾 SVG 可隱藏。
  • Canvas 承載重要資料時，提供 fallback、說明文字或同步 HTML 表格。
