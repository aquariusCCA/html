# 預格式化文本標籤

> 所屬章節：第十六章｜預格式化文本標籤
> 關鍵字：預格式化文本標籤、`<pre>`、preformatted text、保留空格、保留換行、等寬字體、程式碼展示、HTML 文本格式
> 建議回查情境：想知道 `<pre>` 是做什麼的、想分清一般文字和預格式化文字的差別、想確認什麼時候適合用 `<pre>` 顯示內容、想理解 `<pre>` 和 `<code>` 的關係

## 本節導讀

這篇整理 HTML 的預格式化文本標籤 `<pre>`。

很多初學者第一次接觸 `<pre>` 時，只記得它的文字常常會變成等寬字體。
但更重要的理解是：

**`<pre>` 的核心不是改變字體，而是保留文字原本的排版格式。**

也就是說，在 `<pre>` 裡，文字原本寫下來時的：

* 空格
* 縮排
* 換行

通常都會被保留下來。

所以 `<pre>` 適合處理的，不是一般文章段落，
而是那種**格式本身就有意義**的內容，例如：

* 程式碼
* 指令輸出
* 設定檔片段
* 詩句
* 固定排版的文本

* `<pre>` 的基本用途
* `<pre>` 會保留哪些格式特徵
* 一般文字和 `<pre>` 的差別

## 30 秒複習入口

* `<pre>` 用來顯示預格式化文本。
* 在 `<pre>` 中，空格、縮排和換行通常會被保留下來。
* 瀏覽器通常會把 `<pre>` 內容顯示成等寬字體，但這不是它唯一重點。
* `<pre>` 很適合展示程式碼、指令輸出、固定格式文字等格式敏感內容。
* 如果只是一般段落敘述，通常應優先用 `<p>` 或其他一般文字標籤。

## 速查區

### 核心概念

* `<pre>` 是 preformatted text 的縮寫，可以理解成「預先排好格式的文字」。
* 它和一般文字顯示最大的差別，在於它會盡量保留原本輸入時的排版痕跡。

### 關鍵規則 / 判準

* 如果文字中的空格、縮排和換行本身很重要，可以考慮使用 `<pre>`。
* `<pre>` 中的內容通常會以等寬字體顯示。
* 如果只是一般閱讀段落，不需要保留原始格式，通常不必用 `<pre>`。
* 展示程式碼時，常會看到 `<pre>` 和 `<code>` 搭配使用。

### 常見使用場景

* 展示程式碼
* 顯示命令列輸出
* 顯示設定檔片段
* 呈現詩句、歌詞或固定排版內容
* 顯示需要保留縮排的文字

### 常見混淆點

* `<pre>` 不是單純為了讓字體變成等寬。
* 它的核心是保留文本格式，不只是改變外觀。
* `<pre>` 不是所有文字的通用容器。
* 如果只是想讓畫面看起來整齊，不代表就該用 `<pre>`。

### 一句話對比

* `<pre>` 解決的是「文字格式要不要照原樣保留」
* 一般段落重點在內容閱讀，不會特別保證保留原始排版

## 正文筆記

## 1. 這篇在解決什麼問題？

在一般 HTML 文字中，瀏覽器通常不會完全照你輸入的格式顯示內容。

例如：

* 連續空格常會被折疊
* 換行不一定照原樣保留
* 縮排也不一定維持

所以一般文字在畫面上看到的結果，常常不是你原始輸入時的樣子。

但有些內容本身就依賴：

* 空格
* 縮排
* 換行
* 對齊方式

來表達結構。
這時就需要 `<pre>`。

你可以把它理解成：

> **當文字格式本身就是內容的一部分時，`<pre>` 就會很重要。**

---

## 2. `<pre>` 的作用

> `pre` 是 preformatted text，也就是預格式化文本。

`<pre>` 最重要的用途，是讓文字在顯示時，盡量保留原本輸入時的排版格式。

也就是說，它會特別保留像這些東西：

* 空格
* 縮排
* 換行

所以 `<pre>` 的核心不是：

* 讓字體變得比較像程式碼
* 讓內容看起來比較整齊

而是：

**保留原始文本格式。**

等寬字體只是瀏覽器常見的呈現方式，
不是 `<pre>` 最根本的意義。

---

## 3. 一般文字和 `<pre>` 有什麼差別？

這是理解 `<pre>` 最重要的一步。

### 一般文字

在一般 HTML 文字中，瀏覽器通常會幫你做整理：

* 多個空格可能被視為一個空格
* 手動換行不一定照原樣顯示
* 縮排通常不會完整保留

也就是說，瀏覽器會把它當成一般閱讀文字來處理。

---

### `<pre>` 文字

放進 `<pre>` 裡後，內容通常會更接近你原本輸入時的樣子。

例如你怎麼排：

* 換行
* 縮排
* 空格
* 對齊

畫面上就比較可能照那個格式呈現。

---

### 最簡單的記法

可以直接記成一句：

* **一般文字：瀏覽器會幫你整理格式**
* **`<pre>`：盡量保留你原本寫下的格式**

---

## 4. 什麼時候適合用 `<pre>`

當內容的格式本身有意義時，就很適合用 `<pre>`。

### 常見情境 1：展示程式碼

```html
<pre>
body {
  background: #fff;
  font: 12px/24px 1.66;
}
</pre>
```

這種情況下，縮排、換行和對齊都很重要。
如果不用 `<pre>`，程式碼的結構感通常會差很多。

---

### 常見情境 2：顯示命令列輸出

```html
<pre>
$ npm run dev
Server running at:
- Local: http://localhost:5173/
- Network: http://192.168.1.10:5173/
</pre>
```

這類內容常需要保留原本的輸出格式，
所以也很適合使用 `<pre>`。

---

### 常見情境 3：固定格式文本

```html
<pre>
姓名：王小明
部門：前端開發部
電話：02-1234-5678
</pre>
```

如果你的重點是保留這種固定排版，
那 `<pre>` 也很常見。

---

### 常見情境 4：詩句或歌詞

```html
<pre>
床前明月光
疑是地上霜
舉頭望明月
低頭思故鄉
</pre>
```

這種內容每一行的斷點都有意義，
因此也適合用 `<pre>` 來保留格式。

---

## 5. 基本對照示例

下面這個例子，可以更直觀看出一般文字和 `<pre>` 的差別：

```html
<p>
body { background: #fff; font: 12px/24px 1.66; }
</p>

<pre>
body {
  background: #fff;
  font: 12px/24px 1.66;
}
</pre>
```

### 這個例子在說什麼？

* 上半部是一般段落顯示方式
* 下半部放進 `<pre>` 後，換行和縮排會更接近原本輸入格式
* 這說明 `<pre>` 的核心價值在於：**保留格式**

也就是說，`<pre>` 不是讓內容「看起來比較專業」，
而是讓格式敏感的文字能正確呈現。

---

## 6. `<pre>` 和 `<code>` 的關係

這也是很常被問到的地方。

### `<pre>` 的重點

`<pre>` 的重點是：

**保留文字格式。**

---

### `<code>` 的重點

`<code>` 的重點是：

**標示這段內容是程式碼。**

---

### 為什麼常一起出現？

因為展示程式碼時，通常會同時有兩個需求：

1. 要保留縮排、換行、空格
2. 要表達這是一段程式碼

所以常見寫法會像這樣：

```html
<pre><code>
const message = 'Hello';
console.log(message);
</code></pre>
```

你可以這樣記：

* `<pre>`：保留格式
* `<code>`：標示程式碼語意

兩者不是互相取代，而是常常搭配使用。

---

## 7. 什麼情況不適合用 `<pre>`

如果你的內容只是一般文章敘述，
通常就不需要用 `<pre>`。

例如：

* 一般說明文字
* 教學段落
* 文章正文
* 普通列表說明

這些內容的重點在閱讀流暢，不在保留原始輸入格式。

這時更合適的通常是：

* `<p>`
* 標題標籤
* 列表標籤
* 其他一般語意化結構

---

### 常見誤解

有些人會想：

> 我只是想讓文字排得整齊一點，能不能直接用 `<pre>`？

這通常不是最好的想法。

因為 `<pre>` 適合的是：

**格式本身有內容意義**

不是：

**只是想讓畫面比較整齊**

如果需求只是視覺排版，很多時候應該優先考慮 CSS，而不是先想到 `<pre>`。

---

## 8. 使用 `<pre>` 時的注意點

### 不要把 `<pre>` 當成所有文字的預設容器

`<pre>` 不是一般文字容器。
只有在你真的需要保留原始格式時，才值得使用它。

---

### 不要只記得等寬字體

很多人看到 `<pre>` 就只記「它會變成等寬字體」。
這個印象不算錯，但不夠完整。

更穩定的理解應該是：

**等寬字體只是常見外觀，保留格式才是核心。**

---

### 展示程式碼時，常和 `<code>` 一起使用

如果你只是單純保留文本格式，用 `<pre>` 就可以。
但如果你還想清楚表達「這段內容是程式碼」，那常常會搭配 `<code>`。

---

### 一般文章敘述仍應優先用普通文字結構

如果內容只是自然閱讀的段落，
通常應優先使用：

* `<p>`
* 標題
* 列表
* 其他一般結構標籤

不要為了外觀整齊，就把所有東西都塞進 `<pre>`。

---

## 常見問法

### 為什麼 `<pre>` 裡的文字看起來和一般段落不一樣？

因為瀏覽器通常會保留它的空格、縮排和換行，
而且常以等寬字體顯示，所以它更像在呈現「原始排版後的文字」。

---

### `<pre>` 最常用來做什麼？

最常見的是展示：

* 程式碼
* 指令輸出
* 設定片段
* 其他格式敏感內容

只要內容需要保留縮排與換行，`<pre>` 都很常見。

---

### `<pre>` 和 `<p>` 有什麼差別？

* `<p>` 重點在一般段落閱讀
* `<pre>` 重點在保留原始文本格式

兩者處理目的不同，不能只看畫面像不像。

---

### `<pre>` 和 `<code>` 可以互相取代嗎？

不完全一樣。

* `<pre>` 是保留格式
* `<code>` 是標示程式碼

展示程式碼時，它們常常一起用，而不是互相取代。

---

### 什麼情況下不該用 `<pre>`？

如果內容只是一般文章、說明文字或普通段落，
而不是格式敏感內容，通常就不需要用 `<pre>`。
