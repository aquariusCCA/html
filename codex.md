# 路徑

所屬章節：[第六章｜路徑](./README.md)

## 本節導讀

在 HTML 裡，圖片、樣式表、JavaScript、音訊、影片等外部資源，都需要靠「路徑」才能被頁面正確找到。
所以，**路徑的核心任務，就是告訴瀏覽器：目標檔案在哪裡。**

這篇的重點只有兩件事：

1. 搞懂 **絕對路徑** 和 **相對路徑** 的差別
2. 搞懂 **路徑到底是以誰為參考基準**

很多人不是不會寫路徑，而是常常在這幾個地方搞混：

* 不知道路徑是從哪裡開始算
* 不確定 `./`、`../`、`/` 的意思
* 圖片明明存在，卻還是顯示不出來
* 換資料夾位置後，原本的路徑突然失效

這篇就是專門處理這些問題的。

## 30 秒複習入口

* 路徑的作用，是讓瀏覽器找到目標檔案。
* 絕對路徑：從固定位置直接指向目標。
* 相對路徑：從**目前這個檔案所在位置**出發去找目標。
* 相對路徑最常用，但一旦移動檔案位置，原本路徑可能就要一起改。
* `./` 表示目前層級，通常可以省略。
* `../` 表示上一層資料夾。

---

## 速查區

### 核心定義

* **路徑**：用來指定目標檔案位置的寫法。
* **絕對路徑**：從固定起點直接寫到目標位置，不依賴目前檔案在哪裡。
* **相對路徑**：從目前程式碼檔案的位置出發，去找目標檔案。

---

### 關鍵規則 / 判準

#### 1. 判斷是不是相對路徑，先問一件事

> 這個路徑是不是要看「我現在這個 HTML 檔案放在哪裡」才知道怎麼走？

* 如果 **要看目前檔案位置**，那就是 **相對路徑**
* 如果 **不需要看目前檔案位置**，直接就能找到目標，那通常就是 **絕對路徑**

---

#### 2. 相對路徑是以「當前檔案」為基準，不是以網站任意位置為基準

這是最容易搞錯的點。

例如目前檔案是：

```html
/pages/home/index.html
```

圖片在：

```html
/pages/home/images/logo.png
```

那麼在 `index.html` 裡要寫：

```html
<img src="images/logo.png" alt="logo">
```

因為路徑是從 `index.html` 所在位置出發，不是從整個專案根目錄隨便算。

---

#### 3. `./` 表示目前層級，通常可以省略

下面兩種寫法效果一樣：

```html
<img src="./images/logo.png" alt="logo">
<img src="images/logo.png" alt="logo">
```

所以在實務上，`./` 很常省略。

---

#### 4. `../` 表示回到上一層

例如目前檔案是：

```html
/pages/news/detail.html
```

圖片在：

```html
/pages/images/banner.png
```

那麼要先從 `news` 回到 `pages`，再進入 `images`：

```html
<img src="../images/banner.png" alt="banner">
```

如果要回上兩層，就寫兩次：

```html
../../images/banner.png
```

---

### 一句話對比

* **絕對路徑**：直接寫出固定位置，獨立、不看目前檔案在哪
* **相對路徑**：從目前檔案位置開始走到目標

## 正文

## 1. 路徑是什麼？

當頁面要顯示圖片、載入 CSS、引入 JavaScript 時，瀏覽器必須知道這些檔案在哪裡。
而「路徑」，就是用來描述檔案位置的方式。

你可以把它想成：

* 瀏覽器 = 找東西的人
* 路徑 = 地址
* 圖片 / CSS / JS = 目標物件

沒有正確地址，瀏覽器就找不到檔案。

常見使用情境：

```html
<img src="images/pic.png" alt="圖片">
<link rel="stylesheet" href="css/style.css">
<script src="js/main.js"></script>
```

上面的 `src`、`href` 裡面寫的，就是路徑。

---

## 2. 路徑分成哪兩種？

HTML 中最常見的路徑有兩種：

* 絕對路徑
* 相對路徑

---

## 3. 絕對路徑

### 定義

絕對路徑是指：
**從一個固定起點開始，直接完整指向目標檔案的位置。**

它不需要依賴目前 HTML 檔案放在哪裡。

---

### 常見形式

#### （1）本地電腦的絕對路徑

```html
D:\day01\images\1.jpg
```

這種寫法是從磁碟位置直接寫到檔案。

---

#### （2）完整網址

```html
https://www.itcast.cn/2018czgw/images/logo.gif
```

這種寫法是從完整網路位址直接找到資源。

---

### 特點

* 位置明確
* 不用看目前檔案在哪裡
* 可以直接指到目標

---

### 注意事項

#### 1. 本地絕對路徑通常不適合直接寫在網頁中

例如：

```html
<img src="D:\day01\images\1.jpg" alt="圖片">
```

這樣的路徑只在**你的電腦**可能有效。
一旦換電腦、換資料夾、部署到伺服器，這條路徑通常就失效。

所以，本地絕對路徑在實際開發中很少直接用在網頁上。

---

#### 2. 使用外部網址雖然方便，但可能有防盜鏈問題

例如：

```html
<img src="https://xxx.com/images/a.jpg" alt="圖片">
```

有些網站雖然圖片網址可以直接打開，但不代表你可以在自己的網站中任意引用。
對方伺服器如果有做防盜鏈限制，可能會出現：

* 圖片載入失敗
* 顯示禁止引用
* 顯示替代圖

也就是說：

> 用瀏覽器單獨開得出來，不代表你放進自己網站就一定能正常顯示。

---

## 4. 相對路徑

### 定義

相對路徑是指：
**從目前這個程式碼檔案所在的位置出發，去找到目標檔案。**

它最大的特點是：

> 路徑怎麼寫，取決於你現在這個檔案放在哪裡。

---

### 為什麼相對路徑最常用？

因為在同一個專案內，HTML、CSS、JS、圖片通常都放在彼此有關聯的資料夾中。
使用相對路徑：

* 搬到別台電腦仍然可用
* 專案一起移動時比較方便
* 比本地絕對路徑更適合團隊開發與部署

---

### 相對路徑的三種常見寫法

#### （1）同一層

目前檔案與目標檔案在同一個資料夾：

```html
index.html
logo.png
```

寫法：

```html
<img src="logo.png" alt="logo">
```

---

#### （2）下一層

目前檔案：

```html
index.html
```

圖片在 `images` 資料夾裡：

```html
images/logo.png
```

寫法：

```html
<img src="images/logo.png" alt="logo">
```

也可以寫成：

```html
<img src="./images/logo.png" alt="logo">
```

---

#### （3）上一層

目前檔案在 `pages` 裡，圖片在外層的 `images`：

```html
/pages/detail.html
/images/logo.png
```

寫法：

```html
<img src="../images/logo.png" alt="logo">
```

意思是：

1. 先從 `pages` 回到上一層
2. 再進入 `images`
3. 找到 `logo.png`

---

## 5. 怎麼快速判斷路徑該怎麼寫？

可以照下面順序想：

### 第一步：先確認「目前檔案」在哪裡

不是看圖片在哪裡先，而是先看**你正在寫路徑的那個檔案在哪裡**。

### 第二步：再確認「目標檔案」在哪裡

例如圖片、CSS、JS 的實際位置。

### 第三步：從目前檔案走到目標檔案

這時再決定要不要：

* 直接同層找
* 進入下一層
* 回到上一層

---

## 6. 常見錯誤

### 錯誤一：把本地絕對路徑直接寫進網頁

```html
<img src="D:\project\images\a.png" alt="">
```

問題：只能在特定電腦環境下使用，換環境就容易壞。

---

### 錯誤二：搞錯相對路徑的基準

很多人會以為路徑是從「專案根目錄」開始算。
但一般相對路徑不是這樣。

它是從：

> **目前正在寫這條路徑的檔案位置**

開始算。

---

### 錯誤三：忘記 `../` 的作用

圖片不在同層時，很多人只會一直往下寫：

```html
images/logo.png
```

但如果你的檔案其實要先退一層，這樣當然找不到。
這時就要用：

```html
../images/logo.png
```

---

### 錯誤四：移動了 HTML 或資源資料夾，卻沒改路徑

相對路徑非常依賴檔案彼此的位置關係。
所以只要你移動了：

* HTML 檔案
* 圖片資料夾
* CSS / JS 檔案

原本能用的相對路徑就可能失效。

---

### 錯誤五：外部網址能打開，就以為一定能引用

這不一定。
有些外部資源有防盜鏈或權限限制，實際放進 `<img>`、`<video>`、`<audio>` 裡可能仍然失敗。

---

## 7. 實務記憶法

你可以用這個最簡單的口訣記：

> **相對路徑先看自己在哪，再看目標在哪。**

再進一步記成：

* 同層：直接寫檔名
* 下一層：寫資料夾 / 檔名
* 上一層：先寫 `../`

---

## 8. 本節總結

這篇最重要的不是背名詞，而是建立正確判斷方式：

### 你要真的記住的核心只有三句

1. **路徑是拿來讓瀏覽器找到檔案的**
2. **絕對路徑不依賴目前檔案位置**
3. **相對路徑一定要以目前檔案位置為基準**

只要這三句有真正理解，之後你在寫：

* 圖片
* CSS
* JavaScript
* 其他靜態資源

時，路徑就不容易再亂掉。
