# HTML 基本結構與基礎設定

> 所屬章節：第四章｜HTML 基本結構標籤  
> 關鍵字：`DOCTYPE`、文檔類型聲明、`lang`、語言設定、`charset`、字符編碼、`UTF-8`、HTML 屬性、布林屬性、註解  
> 建議回查情境：想知道 HTML 文件一開始那些東西各自是做什麼的、想搞懂 `lang` 和 `charset` 的差別、想快速複習屬性與註解怎麼寫

---

## 本節導讀

剛開始學 HTML 時，最容易混淆的不是內容怎麼寫，而是：

- 頁面最前面那幾行到底是什麼？
- `<!DOCTYPE html>`、`lang`、`charset` 分別在管什麼？
- 屬性和註解又是什麼？

這篇的目標不是背名詞，而是先建立一個清楚的判讀框架：

1. 哪些是「告訴瀏覽器怎麼理解這份文件」
2. 哪些是「補充標籤本身的設定」
3. 哪些只是「寫給開發者看的說明」

只要這個框架清楚，之後看到 HTML 文件開頭的內容，就不容易混亂。

---

## 你會在這篇學到什麼

- `<!DOCTYPE html>` 的作用是什麼
- `lang` 為什麼通常寫在 `<html>` 上
- `charset` 為什麼常見寫法是 `UTF-8`
- HTML 屬性怎麼寫、怎麼讀
- 布林屬性是什麼
- HTML 註解怎麼寫、拿來做什麼

---

## 先看一個完整最小範例

```html
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>我的第一個網頁</title>
</head>
<body>
  <h1>Hello HTML</h1>
</body>
</html>
```

### 這段裡面各自負責什麼？

* `<!DOCTYPE html>`：告訴瀏覽器這是一份 HTML5 文件
* `<html lang="zh-Hant">`：標示這份文件主要語言是繁體中文
* `<meta charset="UTF-8">`：指定文字編碼使用 UTF-8
* `<head>`：放頁面設定資訊
* `<body>`：放實際顯示在頁面上的內容

先把這個骨架記住，後面每個概念就比較容易放回正確位置。

## 速查區

### 一句話總整理

* `DOCTYPE` 管「文件類型」
* `lang` 管「語言標示」
* `charset` 管「文字編碼」
* 屬性管「元素的附加設定」
* 註解管「給開發者看的說明」

### 常見混淆點

* `<!DOCTYPE html>` **不是一般 HTML 標籤**，而是文檔類型聲明
* `lang` **不是翻譯功能**
* `charset` **不是指定哪一國語言**，而是指定文字如何編碼與解碼
* 屬性不是寫在標籤外面，而是寫在**開始標籤內**
* 註解不會顯示在頁面上，但原始碼裡看得到，所以**不要把敏感資訊寫進註解**

---

## 1. 文檔類型聲明：`<!DOCTYPE html>`

### 它是做什麼的？

`<!DOCTYPE html>` 用來告訴瀏覽器：

> 這份文件要按照 HTML5 的規則來解析。

```html
<!DOCTYPE html>
```

### 為什麼它重要？

如果瀏覽器沒有正確知道這份文件的類型，就可能用不理想的方式解析頁面，造成顯示或行為不一致。

### 注意點

* 要放在**整份 HTML 文件最前面**
* 它要寫在 `<html>` 標籤之前
* 它不是頁面內容的一部分
* 它不是一般的 HTML 標籤

### 你可以這樣記

`DOCTYPE` 的工作不是「顯示東西」，而是「先告訴瀏覽器這份文件是什麼」。

---

## 2. 語言設定：`lang`

### 它是做什麼的？

`lang` 用來標示這份 HTML 文件的主要語言。

最常見的寫法是在 `<html>` 標籤上：

```html
<html lang="zh-Hant">
```

或：

```html
<html lang="en">
```

### 它的作用是什麼？

`lang` 的重點不是把頁面翻譯成某種語言，而是讓瀏覽器、搜尋引擎、螢幕閱讀器等工具知道：

> 這份文件主要是用哪種語言寫的。

### 常見例子

* `zh-Hant`：繁體中文
* `zh-CN`：簡體中文語境
* `en`：英文
* `ja`：日文

### 注意點

* `lang` 通常寫在最外層的 `<html>` 上
* 它描述的是「整份文件的主要語言」
* 它不會自動幫你把內容翻譯掉

### 你可以這樣記

`lang` 管的是「這份文件主要在說哪種語言」。

---

## 3. 字符編碼：`charset`

### 它是做什麼的？

`charset` 用來指定這份 HTML 文件採用哪一種文字編碼。

最常見的寫法是：

```html
<meta charset="UTF-8">
```

通常會放在 `<head>` 裡：

```html
<head>
  <meta charset="UTF-8">
</head>
```

### 為什麼它重要？

因為電腦看到的不是「字」，而是「編碼後的資料」。
如果編碼方式不一致，就可能出現亂碼。

### 為什麼常用 UTF-8？

因為 UTF-8 可以處理多數語言的文字，是現代網頁最常見、最通用的編碼方式。

### 注意點

* `charset` 處理的是「文字怎麼被正確解讀」
* 它不等於語言設定
* `lang` 和 `charset` 常一起出現，但兩者負責的事情不同
* `meta charset="UTF-8"` 通常會放在 `<head>` 的前面位置

### 你可以這樣記

* `lang`：這份文件主要用什麼語言
* `charset`：這些文字要用什麼編碼方式去解讀

---

## 4. HTML 屬性

### 什麼是屬性？

屬性是加在 HTML 標籤上的補充資訊，用來描述元素的設定、狀態或行為。

屬性通常寫在**開始標籤內**。

例如：

```html
<a href="https://example.com">前往網站</a>
```

這裡的：

* `href` 是屬性名
* `"https://example.com"` 是屬性值

---

### 屬性的基本寫法

```html
<標籤名 屬性名="屬性值">
```

例如：

```html
<img src="cat.png" alt="一隻貓">
```

這裡的屬性有兩個：

* `src`：圖片來源
* `alt`：圖片替代文字

---

### 一個標籤可以有多個屬性

```html
<input type="text" placeholder="請輸入姓名" maxlength="20">
```

注意幾個規則：

* 屬性寫在開始標籤內
* 屬性之間用空格隔開
* 標籤名和第一個屬性之間也要有空格

---

### 布林屬性是什麼？

有些屬性屬於**布林屬性**。
它的重點不是屬性值，而是「有沒有出現」。

例如：

```html
<input disabled>
```

這表示此輸入框被停用。

常見理解方式是：

* 有寫這個屬性：代表啟用這個狀態
* 沒寫這個屬性：代表沒有這個狀態

常見布林屬性例子：

* `disabled`
* `checked`
* `readonly`
* `required`

---

### 屬性注意點

* 不同標籤可用的屬性不同
* 有些屬性是通用屬性，可以用在很多標籤上
* 屬性名通常建議使用小寫
* 屬性值通常建議加雙引號
* 不要重複寫同名屬性

錯誤示例：

```html
<input type="text" type="password">
```

同一個標籤不應重複出現同名屬性。

## 5. HTML 註解

### 什麼是註解？

註解是寫給開發者看的說明文字，不會顯示在頁面上。

語法如下：

```html
<!-- 這是一段註解 -->
```

### 註解有什麼用途？

* 提醒自己這段程式碼在做什麼
* 幫助別人理解你的結構
* 暫時標記某一段區塊的用途

例如：

```html
<!-- 頁首區塊 -->
<header>
  <h1>我的網站</h1>
</header>
```

### 注意點

* 註解不會變成頁面內容
* 註解是給人看的，不是給瀏覽器執行的
* 不要把帳號、密碼、金鑰等敏感資訊寫進註解

### 你可以這樣記

註解的作用是：
**幫助閱讀與維護，不是參與畫面顯示。**

## 最後整理：這五個概念各自在做什麼？

| 概念                | 作用            |
| ----------------- | ------------- |
| `<!DOCTYPE html>` | 宣告這是 HTML5 文件 |
| `lang`            | 標示文件主要語言      |
| `charset`         | 指定文字編碼方式      |
| 屬性                | 補充元素的設定與狀態    |
| 註解                | 提供給開發者閱讀的說明   |

## 自測問題

1. `<!DOCTYPE html>` 為什麼要放在文件最前面？
2. `lang` 和 `charset` 的差別是什麼？
3. HTML 屬性通常寫在哪裡？
4. 為什麼 `disabled` 可以只寫屬性名？
5. HTML 註解會顯示在頁面上嗎？
6. 為什麼不應該把敏感資訊寫進註解？

---

## 自測答案

1. 因為它要先告訴瀏覽器這份文件要用 HTML5 規則解析。
2. `lang` 管語言標示；`charset` 管文字編碼。
3. 寫在開始標籤內。
4. 因為它是布林屬性，重點在「是否存在」。
5. 不會。
6. 因為雖然不顯示在頁面上，但原始碼中仍然看得到。
