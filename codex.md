> 建議回查情境：想知道什麼是語意化標籤、想分清 `section` 和 `article`、想知道語意化和 `div` 的差別、想快速回查常見語意化標籤用途、想知道語意化對 SEO 與可存取性有什麼幫助

## 本節導讀

這篇整理 HTML 的語意化標籤。

所謂語意化，不是把標籤名字換得比較「高級」，也不是單純為了 SEO。
它的核心價值是：**讓 HTML 本身就能表達內容在頁面中的角色與結構。**

也就是說，當你看到一段標記時，不需要完全依賴 class 名稱去猜用途，而是能從標籤本身大致判斷：

* 這是頁首
* 這是導覽
* 這是主要內容
* 這是一篇可獨立閱讀的內容
* 這是補充資訊
* 這是頁尾

語意化做得好，受益的不只是搜尋引擎，也包含：

* 開發者自己與團隊成員
* 瀏覽器與其他工具
* 螢幕閱讀器等輔助技術
* 後續維護與擴充的人

## 30 秒複習入口

* 語意化標籤是**能描述內容角色與結構**的 HTML 標籤。
* 常見例子有 `header`、`nav`、`main`、`article`、`section`、`aside`、`footer`。
* 語意化的重點不只在 SEO，也包含可讀性、可維護性與可存取性。
* `div` 不是錯，但如果內容本身有更明確的語意角色，通常應優先使用對應標籤。
* `article` 和 `section` 最容易混淆：

  * `article` 偏向**可獨立成立的內容**
  * `section` 偏向**文件中的主題區段**

## 速查區

### 核心概念

* 語意化標籤是在描述「這段內容在頁面中扮演什麼角色」。
* 它的目的不是讓畫面長得不一樣，而是讓**結構更容易被理解**。
* 語意標籤看的是**內容角色**，不是只看版型外觀。

### 常見語意化標籤

* `header`：頁首，或某個區塊的標頭
* `nav`：導覽區
* `main`：頁面的主要內容區
* `article`：可獨立閱讀、脫離上下文後仍有意義的內容
* `section`：文件中的某個主題區段
* `aside`：補充資訊、延伸資訊、側欄
* `footer`：頁尾，或某個區塊的尾部資訊

### 一句話對比

* 語意化標籤：在描述內容角色
* `div`：通用容器，本身不說明內容是什麼

### 常見優點

* 結構更清楚
* 樣式拿掉後仍較容易閱讀
* 有助於搜尋引擎理解頁面結構
* 對螢幕閱讀器等輔助技術更友善
* 對團隊協作與維護更有利

### 常見混淆點

* 語意化不等於只服務 SEO
* 語意化標籤不是每頁只能出現一次
* 語意化標籤不是用來取代所有 `div`
* `section` 不是「看到一塊就拿來包」的通用盒子
* `article` 與 `section` 的差別不在外觀，而在內容是否可獨立成立

## 正文筆記

## 1. 這篇在解決什麼問題？

以前做頁面時，很多區塊常常全部都用 `div` 包起來，例如：

* `div class="header"`
* `div class="nav"`
* `div class="content"`
* `div class="footer"`

這樣不是不能寫，但有一個問題：

**只看標籤本身時，你無法立刻知道這段內容的角色。**

你往往得靠：

* class 名稱
* CSS 命名
* 團隊習慣
* 上下文推測

才能知道這一塊到底是在做什麼。

語意化標籤就是在改善這個問題。
它讓結構資訊直接存在 HTML 本身，而不是全部依賴 class 名稱補充。

---

## 2. 什麼是語意化標籤？

所謂語意化標籤，指的是：

**這個標籤本身就能表達內容在頁面中的角色與意義。**

例如：

* `nav` 代表導覽
* `main` 代表主要內容
* `article` 代表一篇可獨立閱讀的內容
* `footer` 代表頁尾或尾部資訊

所以語意化不是「名字比較好看」，而是：

**HTML 本身就在說明這段內容是什麼。**

這樣的好處是：

* 人比較容易讀懂
* 瀏覽器與工具較容易理解結構
* 搜尋引擎較容易判讀內容區域
* 輔助技術較容易建立清楚的導覽與閱讀脈絡

---

## 3. 常見語意化標籤

![語意化標籤.png](./images/image_1776136001360_fa3a622b.png)

### `header`

表示頁首，或某個區塊的標頭部分。

常見情境：

* 整個網站的頁首
* 某篇文章的標題區
* 某個區塊的開頭資訊

> 注意：`header` 不一定只能代表整頁最上方。
> 一篇文章也可以有自己的 `header`。

---

### `nav`

表示導覽區，通常用來放主要導航連結。

常見情境：

* 主選單
* 側邊導覽
* 文章目錄導覽

> `nav` 的重點不是「放連結就算」，而是這些連結主要用於導覽。

---

### `main`

表示頁面的主要內容區。

它通常對應的是：
**這一頁真正的核心內容是什麼。**

例如：

* 文章頁的文章內容
* 商品頁的商品資訊
* 首頁的主要展示區塊

> 一個頁面通常會有一個主要的 `main` 區域，用來表示本頁核心內容。

---

### `article`

表示一段**可獨立成立、可單獨閱讀**的內容。

常見情境：

* 部落格文章
* 新聞內容
* 論壇貼文
* 留言項目
* 商品評論

判斷方式可以想成：

> **把這段內容單獨抽出來看，它還是不是一個完整、有意義的單位？**

如果答案是「是」，那它通常很適合用 `article`。

---

### `section`

表示文件中的某個**主題區段**。

它通常是在整體內容中，依照主題切分出來的一部分。

常見情境：

* 文章中的某個章節
* 首頁中的「最新消息區」
* 商品頁中的「規格說明區」
* 文件中的一個段落主題

判斷方式可以想成：

> **這是一個主題分段，但它不一定能脫離上下文獨立成立。**

---

### `aside`

表示與主內容**間接相關**的補充資訊。

常見情境：

* 側欄
* 延伸閱讀
* 相關文章
* 廣告區
* 作者簡介
* 額外說明

重點不是它一定在畫面側邊，
而是它的內容通常不是主線核心，而是補充性的。

---

### `footer`

表示頁尾，或某個區塊的尾部資訊。

常見情境：

* 整站頁尾
* 文章作者資訊
* 版權資訊
* 聯絡方式
* 區塊結尾補充說明

> 跟 `header` 一樣，`footer` 也不一定只會出現在整頁最底部。
> 某篇文章本身也可以有自己的 `footer`。

---

## 4. `article` 和 `section` 怎麼分？

這是最常見的混淆點之一。

### `article` 的重點

`article` 是一段**可獨立成立的內容**。

也就是說，就算把它從頁面中抽出來，單獨放到別的地方閱讀，通常仍然有意義。

例如：

* 一篇新聞
* 一篇部落格文章
* 一則論壇貼文
* 一則留言

---

### `section` 的重點

`section` 是一個**主題區段**。

它通常是整體文件中的某個部分，重點是「分段整理主題」，不一定能單獨成立。

例如一篇文章裡的：

* 前言
* 使用方式
* 常見錯誤
* 結論

這些通常比較像 `section`。

---

### 最實用的判斷方式

可以直接這樣記：

* **能單獨拿出去閱讀，通常偏 `article`**
* **只是整份內容中的一個主題區塊，通常偏 `section`**

再簡化成一句：

* `article`：獨立內容
* `section`：主題分段

---

## 5. 為什麼要使用語意化標籤？

## 對閱讀與維護有幫助

當 HTML 本身就表達角色時，程式碼會更容易閱讀。

例如你看到：

```html
<header></header>
<nav></nav>
<main></main>
<footer></footer>
```

通常比看到：

```html
<div class="header"></div>
<div class="nav"></div>
<div class="main"></div>
<div class="footer"></div>
```

更容易立即理解整體結構。

這種差異在以下情境特別明顯：

* 自己隔一段時間回頭看程式碼
* 團隊協作交接
* 元件數量變多
* 頁面規模變大

---

## 對搜尋引擎理解有幫助

語意化標籤能幫助搜尋引擎更容易理解頁面的區域分工與內容結構。

但要注意：

**這不代表只要改成語意化標籤，排名就一定上升。**

SEO 是多因素問題，還包含：

* 內容品質
* 結構層次
* 載入速度
* 連結關係
* 使用者體驗
* 行動裝置適配等

所以更準確的說法是：

> 語意化對 SEO 可能有幫助，
> 但它不是保證排名提升的魔法。

---

## 對可存取性較友善

語意化標籤不只讓人比較容易讀，也能讓螢幕閱讀器等輔助技術更容易理解頁面區塊與內容順序。

例如：

* 哪裡是主要內容
* 哪裡是導覽
* 哪裡是補充資訊
* 哪裡是一篇獨立內容

所以語意化除了和 SEO 有關，也和**可存取性**密切相關。

## 6. 原本全用 `div` 的寫法

```css
.header, .nav {
  height: 120px;
  background-color: pink;
  border-radius: 15px;
  width: 800px;
  margin: 15px auto;
}

.section {
  width: 500px;
  height: 300px;
  background-color: skyblue;
}
```

```html
<body>
  <div class="header">頭部標籤</div>
  <div class="nav">導航欄標籤</div>
  <div class="section">某個區域</div>
</body>
```

### 這段在說什麼？

這種寫法不是錯，畫面也可以正常做出來。
問題在於：

**標籤本身沒有直接表達內容角色。**

你必須靠：

* `class="header"`
* `class="nav"`
* `class="section"`

這些命名，才知道每個區塊的用途。

也就是說，結構資訊是靠命名補上的，不是 HTML 標籤本身就有。

---

## 7. 換成語意化標籤的寫法

```css
header, nav {
  height: 120px;
  background-color: pink;
  border-radius: 15px;
  width: 800px;
  margin: 15px auto;
}

section {
  width: 500px;
  height: 300px;
  background-color: skyblue;
}
```

```html
<body>
  <header>頭部標籤</header>
  <nav>導航欄標籤</nav>
  <section>某個區域</section>
</body>
```

### 這樣改有什麼差別？

這樣改完後，只看 HTML，你就比較容易知道：

* 這是頁首
* 這是導覽
* 這是一個區段

這就是語意化的核心價值：

**讓結構資訊直接存在標記本身，而不是完全依靠 class 名稱猜用途。**

## 8. 更貼近實務的語意化骨架範例

下面這個例子，比前面的簡化版更接近實際頁面：

```html
<body>
  <header>
    <h1>我的網站</h1>
  </header>

  <nav>
    <a href="#">首頁</a>
    <a href="#">文章</a>
    <a href="#">關於我</a>
  </nav>

  <main>
    <article>
      <header>
        <h2>語意化標籤入門</h2>
        <p>發佈日期：2026-04-17</p>
      </header>

      <section>
        <h3>什麼是語意化？</h3>
        <p>語意化是讓標籤本身能描述內容角色。</p>
      </section>

      <section>
        <h3>為什麼要使用？</h3>
        <p>因為它有助於結構理解、維護與可存取性。</p>
      </section>

      <footer>
        <p>作者：王小明</p>
      </footer>
    </article>

    <aside>
      <h3>延伸閱讀</h3>
      <ul>
        <li><a href="#">HTML 基礎</a></li>
        <li><a href="#">可存取性入門</a></li>
      </ul>
    </aside>
  </main>

  <footer>
    <p>Copyright © 2026</p>
  </footer>
</body>
```

### 這個例子要看懂什麼？

* 整頁可以有自己的 `header` 與 `footer`
* `main` 代表頁面的主要內容
* `article` 代表一篇可獨立閱讀的文章
* 文章內部還可以再用 `section` 分主題
* `article` 自己也可以有 `header` 和 `footer`
* `aside` 是補充資訊，不是主線核心內容

這也說明了一件很重要的事：

> **語意化標籤可以巢狀使用。**
> 重點不是「每頁只能出現一次」，而是它的角色是否合理。

## 9. 使用語意化標籤時的注意點

### 不要把語意化理解成只對搜尋引擎有用

語意化確實可能幫助搜尋引擎理解頁面結構，
但它更大的價值在於：

* 可讀性
* 可維護性
* 文件結構清晰
* 可存取性提升

所以不要把它縮小理解成「SEO 小技巧」。

---

### 不要把語意標籤當成版型名稱

很多初學者會誤以為：

* 最上面一定是 `header`
* 左右欄一定是 `aside`
* 任何一塊內容都可以叫 `section`

這種理解不夠準。

語意標籤看的是：

**這段內容在結構中的角色是什麼。**

不是只看它在畫面中的位置或長相。

---

### `section` 不要拿來當通用容器亂包

`section` 常被誤用成「比 `div` 更高級的盒子」。

但它其實不是普通容器，而是**有主題意義的區段**。

如果那塊只是為了排版、包樣式、做 flex 或 grid 容器，
又沒有明確主題，那很多時候用 `div` 反而更合適。

---

### 沒有合適語意時，`div` 完全可以用

語意化標籤不是要取代所有 `div`。

如果某個區塊：

* 沒有明確語意角色
* 只是為了布局
* 只是單純容器

那麼使用 `div` 是合理且正常的。

真正的原則不是「硬套語意化」，而是：

> **有合適語意時優先使用；沒有合適語意時就用 `div`。**

---

### 舊版瀏覽器相容性屬於歷史背景知識

有些舊教材會提到早期瀏覽器對語意化標籤支援較弱。
這可以當作歷史背景知道即可，不是現在學習的主要重點。

現代前端學習更重要的是理解：

* 為什麼要語意化
* 什麼情況該用哪個標籤
* 怎麼寫出結構清楚的 HTML

## 常見問法

### 語意化標籤是不是主要為了 SEO？

不是只為了 SEO。

更準確的說法是：
語意化有助於搜尋引擎理解頁面結構，但它更大的價值在於讓 HTML 更清楚、更容易維護，並提升可存取性。

---

### `header`、`footer` 可以重複使用嗎？

可以，只要結構上合理。

例如：

* 整頁可以有自己的 `header`
* 某篇文章也可以有自己的 `header`
* 整頁可以有自己的 `footer`
* 某個區塊也可以有自己的 `footer`

重點不是「只能出現一次」，而是它所代表的角色是否合理。

---

### `article` 和 `section` 到底差在哪？

最簡單的記法是：

* `article`：可獨立成立的內容
* `section`：同一份文件中的主題分段

如果把這段內容單獨抽出去看，仍然完整有意義，通常較偏向 `article`。

---

### 既然有語意化標籤，還需要 `div` 嗎？

需要。

`div` 仍然是非常重要的通用容器。
語意化標籤不是拿來消滅 `div`，而是讓有明確角色的內容能被更準確地表達。

---

### `section` 是不是只要一個區塊就能用？

不是。

`section` 通常應該表示一個有主題意義的區段。
如果只是單純為了布局或包樣式，常常用 `div` 更合適。
