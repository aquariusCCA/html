HTML 塊級元素與行內元素 Demo

這個示例頁把第七章的核心觀念整理成可直接觀察的畫面：誰會獨占一行、誰只佔內容寬度、哪些巢狀是合法的、哪些寫法會被瀏覽器自動修正。

塊級元素 通常會從新的一行開始，預設傾向佔滿可用寬度，適合段落、區塊、標題。
行內元素 通常和文字排在同一行，寬度主要跟內容走，適合局部語意或文字修飾。
巢狀規則 `p` 不能包 `div`，標題不能互相巢狀，`a` 不能再包另一個 `a`。
例外情況 `img`、`input` 雖常視為行內，但在尺寸控制上和純文字型行內元素不同。

本頁會看到什麼

  • 塊級元素和行內元素的實際排版差異
  • `div`、`p`、`span`、`a` 的典型使用位置
  • 非法巢狀在瀏覽器中的實際結果
  • 為什麼不能只靠「塊級 / 行內」決定所有布局

`p` 不能包塊級容器

`p` 只適合承載段落文字內容。當你把 `div` 這類區塊元素塞進去時，瀏覽器會重組 DOM。 


            

              
            

            

          

        

      


      

        
4. 用語意挑標籤，不只看換不換行

        

          

            `p` vs `div`
            `p` 是段落；`div` 是泛用區塊容器。不是只因為兩者都常換行就能互相替代。
          

          

            `span` vs `a`
            `span` 沒有超連結語意；`a` 是可導航的互動元素，語意和行為都不同。
          

          

            CSS 才是布局主角
            塊級 / 行內是理解 HTML 行為的起點，但真正的布局仍要靠 `display`、`flex`、`grid` 等 CSS。
