# HTML 塊級元素與行內元素

所屬章節：[第七章｜HTML 塊級元素與行內元素](./README.md)

## 本節導讀

這篇最重要的，不是背一堆標籤名稱，而是先分清楚下面兩件事：

1. **元素預設怎麼排在頁面上**
2. **元素之間能不能合法嵌套**

這兩件事很常一起出現，但**不是同一套規則**。

很多初學者會把它們混在一起，最後變成：

* 以為塊級元素一定能包任何東西
* 以為行內元素一定不能包塊級元素
* 看到某些特殊標籤時，規則突然全部失效

其實問題不在你記性差，而是你把**排版分類**和**內容模型 / 嵌套規則**混成同一件事了。

這篇的目標，就是幫你把這兩條線徹底拆開。

---

## 關鍵字

* 主題：HTML 塊級元素、HTML 行內元素
* 別名：區塊元素、內聯元素
* 英文：block-level element、inline element
* 常見搜尋：

  * 塊級元素和行內元素差別
  * `p` 能不能包 `div`
  * `a` 能不能包 `a`
  * `a` 能不能包 `div`
  * 行內元素能不能設寬高
* 易混淆：

  * 排版方式 vs 合法嵌套
  * 塊級 / 行內 vs 元素自己的內容模型
  * 一般行內元素 vs `a` 這種特殊元素

---

## 先記住核心觀念

### 先記這三句就夠了

1. **塊級 / 行內，主要是在描述元素的預設排版表現**
2. **能不能包別的元素，不能只靠塊級 / 行內來判斷**
3. **遇到特殊元素時，要看元素本身的規則**

也就是說：

> **塊級 / 行內，回答的是「怎麼排」；內容模型，回答的是「能不能這樣包」。**

這一句是整章核心。

---

## 30 秒複習入口

* 塊級元素：預設常從新的一行開始排列
* 行內元素：預設常跟文字排在同一行
* 但「塊級 / 行內」不等於「能不能合法嵌套」
* 判斷嵌套是否合法，要看元素自己的規則
* `p`、`h1~h6`、`a` 是最值得另外記的三組例外 / 特殊情況

## 速查區

### 核心定義

* **塊級元素**：預設通常會從新的一行開始排列，常拿來表示一整塊內容
* **行內元素**：預設通常會留在文字流中，和文字或其他行內內容排在同一行
* **合法嵌套**：某個元素裡面能不能放另一個元素，這件事要看該元素自己的內容模型與規則

---

### 一句話對比

* **塊級 / 行內**：在說「預設怎麼排」
* **合法嵌套**：在說「能不能這樣包」

---

### 最容易背錯的地方

下面這兩句話，只能當入門口訣，不能當完整法律：

* 塊級元素可以包任何元素
* 行內元素不能包塊級元素

這樣記太粗了。
真正遇到實際標籤時，還是要看該元素本身的規則。

---

## 1. 什麼是塊級元素？

塊級元素可以先理解成：

> **預設通常會從新的一行開始，像一整塊內容一樣出現在頁面上。**

常見特徵：

* 預設常從新的一行開始
* 常拿來表示較大的區塊內容
* 常用來做段落、區塊、清單、區段結構

常見例子：

* `div`
* `p`
* `h1` ~ `h6`
* `ul`
* `ol`
* `li`

範例：

```html
<h1>標題</h1>
<p>這是一段文字</p>
<div>這是一個區塊</div>
```

在預設情況下，這些元素通常都會各自從新的一行開始出現。

---

## 2. 什麼是行內元素？

行內元素可以先理解成：

> **預設通常留在文字流裡，會和文字或其他行內內容排在同一行。**

常見特徵：

* 預設通常不會自己獨佔一整行
* 常和文字一起出現
* 常拿來標記文字的一部分、語氣、強調、連結

常見例子：

* `span`
* `em`
* `strong`
* `a`

範例：

```html
<span>這是 span</span>
<em>這是強調</em>
<strong>這是加強語氣</strong>
<a href="https://example.com">這是連結</a>
```

這些元素預設通常會排在同一行，而不是一個元素就自動換一行。

---

## 3. 為什麼不能只靠塊級 / 行內判斷嵌套？

這是整章最重要的地方。

很多人一開始會把「塊級元素」和「行內元素」當成一套完整規則，於是得到下面這種簡化版印象：

* 塊級元素可以包任何元素
* 行內元素不能包塊級元素

這種說法有一點入門幫助，但**不夠精確**。

因為：

* 塊級 / 行內，主要是在幫你理解「預設怎麼排版」
* 某個元素能不能包另一個元素，真正要看的是**該元素自己的內容模型與規則**

所以遇到嵌套問題時，正確思路應該是：

### 判斷流程

1. **先看它預設怎麼排**
   這一步用塊級 / 行內來理解

2. **再看這個元素本身能包什麼**
   這一步看元素自己的規則

3. **不要只靠塊級 / 行內直接下結論**

---

## 4. `p` 標籤的規則

`p` 代表段落。
它適合放一段文字內容，不適合直接拿來包一般區塊元素，例如 `div`。

例如下面這種寫法不合適：

```html
<p>
  <div>這是一個 div</div>
</p>
```

### 為什麼不合適？

因為 `p` 是段落元素，它的內容有自己的限制。
當瀏覽器在 `p` 裡面遇到像 `div` 這種不適合放進段落內容的元素時，通常會：

> **先自動把前面的 `p` 結束掉，再去解析後面的 `div`**

也就是說，你原本以為自己寫的是：

```html
<p>
  <div>這是一個 div</div>
</p>
```

但瀏覽器實際解析時，常常會變成接近下面這種效果：

```html
<p></p>
<div>這是一個 div</div>
<p></p>
```

重點不是你背「`p` 不能包 `div`」而已，
而是你要知道：

> **瀏覽器不會照你表面寫的巢狀結構原封不動理解。**

所以 `p` 比 `div` 更嚴格，它不是一般萬用容器。

---

## 5. `h1` ~ `h6` 的規則

`h1` 到 `h6` 是標題元素。
它們用來表示內容層級中的標題。

例如：

```html
<h1>網站主標題</h1>
<h2>章節標題</h2>
<h3>小節標題</h3>
```

### 要注意的點

標題元素裡面，不應再直接嵌套另一個標題元素。

例如下面這種寫法不合適：

```html
<h1>
  主標題
  <h2>副標題</h2>
</h1>
```

原因不是因為「它們都是塊級元素」，
而是因為：

> **標題元素本身的內容規則，不適合再放另一個標題元素。**

所以你要再一次記住：

* 這不是單純的塊級 / 行內問題
* 這是元素自己的嵌套規則問題

如果你要表達主標題和副標題，應該寫成平行結構，而不是互相包住：

```html
<h1>主標題</h1>
<h2>副標題</h2>
```

---

## 6. `a` 標籤的規則

`a` 是這一章最容易讓人誤會的標籤之一。

因為很多人會先背：

> 行內元素不能包塊級元素

然後看到 `a` 可以包較大範圍內容時，就覺得規則壞掉了。

其實不是規則壞掉，而是：

> **`a` 本身就是一個要特別記的特殊元素。**

---

### `a` 的重點要這樣記

* `a` 預設常被當作行內元素理解
* 但它不只是單純的文字容器
* 在 HTML5 的使用情境中，`a` 可以包比純文字更多的內容
* **但 `a` 不能再包另一個 `a`**

例如：

```html
<a href="https://example.com">
  <div>整塊內容都可以點擊</div>
</a>
```

這種寫法在現代 HTML 中可以用來表達「整塊區域都是連結」。

但下面這種寫法不可以：

```html
<a href="https://example.com">
  <a href="https://example.org">另一個連結</a>
</a>
```

原因很簡單：

> **一個超連結裡面不能再嵌套另一個超連結。**

所以 `a` 這個例子剛好最能說明：

* 「行內元素不能包塊級元素」不是完整規則
* 真正的判斷方式，是看元素自己允不允許這樣做

---

## 7. 關於寬度和高度，要怎麼記才不容易背錯？

很多入門資料常寫：

* 塊級元素可以設定寬高
* 行內元素不能設定寬高

這種說法**可以拿來建立第一印象**，但不要把它背成絕對規則。

因為實際上更安全的理解是：

### 比較穩的記法

* 很多常見的文字型行內元素，例如 `span`、`em`、`strong`，通常不是拿來控制區塊尺寸的
* 所以你平常不會靠它們來設定明確的 `width`、`height`
* 但某些元素，例如 `img`、某些 `input`，不能直接用「行內元素都不能碰寬高」來一刀切

因此，這一段最不容易背錯的版本是：

> **很多常見行內元素不是拿來做區塊尺寸控制的；真的要精確判斷，還是要回到該元素本身和 CSS 呈現方式來看。**

---

## 8. 常見混淆點

### 混淆一：塊級 / 行內 = 合法嵌套規則

錯。
塊級 / 行內主要是在幫你理解預設排版，不足以單獨判定所有嵌套是否合法。

---

### 混淆二：`p` 和 `div` 只是兩種不同的塊級元素

不只這樣。
雖然它們都常被當作塊級元素理解，但 `p` 是段落元素，限制比 `div` 更嚴格。
`div` 更像一般用途容器。

---

### 混淆三：行內元素都不能包塊元素

這句只能當很粗的入門印象。
像 `a` 就是很典型的反例之一，不能直接一刀切。

---

### 混淆四：`a` 可以包區塊元素，所以它什麼都能包

也不對。
`a` 雖然能包比純文字更多的內容，但它不能再包另一個 `a`，而且也不是完全沒限制的萬用容器。

---

### 混淆五：寬高規則可以只用塊級 / 行內來判斷

不夠準。
實際顯示效果，還要看元素本身和 CSS。

---

## 9. 一個最好用的判斷口訣

你可以把這章的重點記成下面這句：

> **先看它怎麼排，再看它能包什麼。**

拆開來就是：

* **怎麼排** → 用塊級 / 行內理解
* **能包什麼** → 看元素本身規則

這樣你之後遇到新標籤時，比較不會只靠死背分類亂猜。

---

## 10. 本節總結

這一章最重要的，不是背出哪些標籤屬於塊級、哪些屬於行內。
真正要抓住的是下面這個結構：

### 第一層：預設排版

* 塊級元素：常從新的一行開始
* 行內元素：常跟文字排在同一行

### 第二層：合法嵌套

* 不能只靠塊級 / 行內判斷
* 要看元素自己的內容模型與規則

### 第三層：特殊元素要另外記

* `p`：不能拿來當一般區塊容器
* `h1~h6`：標題元素之間不應互相嵌套
* `a`：可以包較大內容區塊，但不能包另一個 `a`

所以這章最終你要真的記住的核心只有一句：

> **排版分類是一回事，嵌套合法性是另一回事。**
