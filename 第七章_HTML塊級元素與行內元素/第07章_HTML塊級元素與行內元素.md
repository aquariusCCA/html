# HTML 塊級元素與行內元素

所屬章節：[第七章｜HTML 塊級元素與行內元素](./README.md)

## 本節導讀

這篇要先幫你分清兩件事：

1. 元素預設怎麼排在頁面上
2. 元素能不能合法地包住其他元素

這兩件事常常一起出現，但不是同一套規則。  
先把這個觀念抓穩，再看 `p`、`h1` ~ `h6`、`a` 這幾個特殊例子，就不容易混亂。

## 關鍵字

- 主題：HTML 塊級元素、HTML 行內元素
- 別名：區塊元素、內聯元素
- 英文：block-level element、inline element
- 常見搜尋：塊級元素和行內元素差別、`p` 能不能包 `div`、`a` 能不能包 `a`

## 先記住核心觀念

- 塊級元素常見的預設行為，是從新的一行開始排列。
- 行內元素常見的預設行為，是跟文字排在同一行。
- 但「能不能包別的元素」不能只看塊級或行內，還要看該元素自己的規則。

你可以先把「塊級 / 行內」當成入門分類。  
它很適合拿來理解版面排列，但不適合單獨拿來判定所有嵌套是否合法。

## 1. 什麼是塊級元素

塊級元素可以先理解成：  
預設通常會從新的一行開始，常拿來表示一整塊內容的元素。

常見特徵：

- 預設常從新的一行開始
- 常作為較大的內容區塊
- 常見例子有 `div`、`p`、`h1` ~ `h6`、`ul`、`ol`、`li`

範例：

```html
<h1>尚硅谷</h1>
<p>尚硅谷</p>
<div>尚硅谷</div>
```

這些元素在預設情況下，通常都會各自從新的一行開始。

## 2. 什麼是行內元素

行內元素可以先理解成：  
預設通常會留在文字流裡，和其他文字或元素排在同一行的元素。

常見特徵：

- 預設通常不會自己獨佔一整行
- 常和文字一起出現
- 常見例子有 `span`、`em`、`strong`、`a`

範例：

```html
<span>尚硅谷</span>
<em>前端</em>
<a href="https://example.com">超連結</a>
```

這些元素預設通常會排在同一行，而不是一個元素就換一行。

## 3. 為什麼不能只靠塊級 / 行內判斷能不能包

初學時很容易把下面兩句話背成完整規則：

- 塊級元素可以包任何元素
- 行內元素不能包塊級元素

這樣記太粗。

比較實用的理解方式是：

- 塊級 / 行內，主要是在幫你理解「預設怎麼排」
- 真正要判斷能不能包，還要看元素本身的內容模型

所以遇到嵌套問題時，請優先看：

1. 這是不是一般的文字型行內元素
2. 這個元素本身有沒有特殊限制

下面三個元素最值得另外記：`p`、`h1` ~ `h6`、`a`

## 4. `p` 標籤的規則

`p` 代表一個段落。  
它適合放段落中的文字內容，不適合直接拿來包 `div` 這類區塊元素。

例如下面這種寫法不合適：

```html
<p>
  <div>尚硅谷</div>
</p>
```

當瀏覽器解析到 `div` 時，通常會先把前面的 `p` 結束掉。  
所以最後畫面結構常常不會等於你原本寫下去的巢狀結構。

## 5. `h1` ~ `h6` 的規則

`h1` 到 `h6` 是標題元素。  
標題元素裡面不應該再嵌套另一個標題元素。

例如：

```html
<h1>
  尚硅谷1
  <h2>尚硅谷2</h2>
</h1>
```

這種寫法不合法，因為標題裡面不應再塞另一個標題。

## 6. `a` 標籤的規則

`a` 是這一章最容易讓人搞混的例子。

你可以先記住：

- `a` 不只是一般的文字容器
- 它可以包比純文字更多的內容
- 但它不能包另一個 `a`

例如下面這種寫法可以用來理解「`a` 可以包比較大的內容區塊」：

```html
<a href="https://example.com">
  <div>這是一個包含在 a 標籤中的區塊元素</div>
</a>
```

但下面這種寫法不可以：

```html
<a href="https://example.com">
  <a>這是一個超連結</a>
</a>
```

原因很簡單：  
一個超連結裡面不能再包另一個超連結。

## 7. 關於寬度和高度，要怎麼記比較不會出錯

很多入門資料會說：

- 塊級元素可以設定寬高
- 行內元素不能設定寬高

這種說法可以幫你快速建立印象，但不要把它當成百分之百不變的法律。

比較安全的記法是：

- 很多常見的純文字型行內元素，例如 `span`、`em`，通常不是拿來靠 `width`、`height` 控制區塊大小的
- 但像 `img`、部分 `input` 這類元素，不能直接用「行內元素都不能碰寬高」一刀切

所以這一條建議這樣記：

> 行內元素通常不是拿來做區塊尺寸控制的；如果要精確控制呈現，還是要回到實際元素特性和 CSS 來看。

## 常見混淆點

- 塊級元素 vs 行內元素：這組分類主要在幫你理解預設排版。
- 預設排版 vs 合法嵌套：前者在說「怎麼排」，後者在說「能不能這樣包」。
- `p` vs `div`：`p` 是段落元素，限制比 `div` 更嚴格；`div` 更常被當成一般容器。
- 行內元素不能包塊元素 vs `a` 能包區塊元素：前者只能當入門近似規則，碰到 `a` 這類特殊元素時，要看元素自己的規則。

## 一句話抓核心

學這一章最重要的，不是死背哪些標籤是塊級、哪些是行內，而是分清楚：預設排版是一回事，元素能不能合法嵌套是另一回事。
