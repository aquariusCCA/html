# 第21章｜超鏈接標籤

> - 關鍵字：超鏈接標籤、`a`、anchor、`href`、`target`、外部連結、內部連結、檔案連結、錨點連結、`download`、`mailto`、`tel`、`javascript:`
> - 建議回查情境：想知道 `<a>` 怎麼寫、想分清不同連結類型、想知道 `href="#"` / `href=""` 差在哪、想做錨點跳轉、下載連結或喚起應用

## 本節導讀

`<a>` 是 HTML 中最重要的連結標籤之一。
它的核心任務不是單純「點一下跳頁」，而是讓使用者能從目前位置**導向另一個資源、頁面位置，或外部應用**。

學這個標籤時，不能只背一行：

```html
<a href="...">文字</a>
```

真正要搞懂的是：

* `href` 到底在指定什麼
* `target` 會怎麼影響開啟方式
* 外部連結、內部連結、檔案連結、錨點連結差在哪
* `href="#"`、`href=""`、`javascript:` 各自代表什麼
* 哪些寫法只是教材常見示範，哪些才比較符合實務習慣

* `href="#"` 和 `href=""` 的差異

## 30 秒複習入口

* `<a>` 用來建立超鏈接，讓使用者跳到另一個目標。
* `href` 決定連結要去哪裡，`target` 決定怎麼開。
* `href="#id"` 會跳到頁面中對應 `id` 的位置。
* `download` 可提示瀏覽器下載檔案。
* `mailto:` 可開啟郵件應用，`tel:` 可喚起撥號功能。
* `javascript:` 雖然能用，但通常不是現代實務最建議的寫法。
* **導向用 `<a>`，操作用 `<button>`。**

---

## 速查區

### 核心概念

* `<a>` 的本質是**導航（navigation）**。
* 它可以連到：

  * 其他網站
  * 站內頁面
  * 檔案
  * 同頁或其他頁的某個位置
  * 郵件、電話、特定應用

### 關鍵規則

* `href` 用來指定目標位置。
* 沒有 `href` 的 `<a>` 可以存在，但通常不應把它當成真正的超鏈接。
* `target="_self"` 是預設值，表示在目前頁面開啟。
* `target="_blank"` 通常表示在新頁籤或新視窗開啟。
* `href="#id"` 表示跳到頁面中對應 `id` 的位置。
* `href="#"` 常見效果是回到頁面頂部。
* `href=""` 常會被解讀為目前文件本身，效果常接近重新載入當前頁面。
* `download` 表示「希望瀏覽器把資源當下載處理」。

### 一句話抓核心

* `<a>` 是拿來**導向目標**的，不是拿來做一般互動操作的。

# 正文筆記

## 1. `<a>` 標籤是什麼？

`<a>` 是 **anchor** 的縮寫，通常翻成**超鏈接標籤**或**錨點標籤**。
它的作用是建立一個可點擊的連結，讓使用者從目前內容跳到另一個目標。

基本語法：

```html
<a href="目標地址">連結內容</a>
```

例子：

```html
<a href="https://www.baidu.com/">百度</a>
```

### 重點理解

你可以把 `<a>` 想成一個「可導航的入口」。
只要點下去，瀏覽器就會根據 `href` 指向的內容進行跳轉、定位、下載，或喚起其他應用。

---

## 2. `<a>` 可以包住什麼？

`<a>` 不一定只能包文字。
它也可以包圖片，甚至包住一整塊內容，讓整個區域都可點擊。

```html
<a href="https://example.com">
  <div>這是一整塊可點擊區域</div>
</a>
```

### 但要注意

雖然 `<a>` 可以包住較多內容，但實務上仍要避免把太多互動元素混在裡面，例如：

* 連結裡再包按鈕
* 連結裡再包另一個連結
* 一個區塊同時承擔多種點擊行為

這樣容易讓結構混亂，也會影響使用者操作體驗。

---

## 3. `<a>` 最重要的屬性

## 3.1 `href`

`href` 是 **hypertext reference**，用來指定連結目標。
它是 `<a>` 最核心的屬性。

```html
<a href="https://example.com">前往網站</a>
```

### 你要這樣理解它

`href` 就是在回答：

**「點了之後，要去哪裡？」**

它可以指向：

* 完整網址
* 相對路徑
* 檔案
* `#id`
* `mailto:`
* `tel:`
* `javascript:`

### 沒有 `href` 會怎樣？

沒有 `href` 的 `<a>` 可以存在，但通常不再是標準意義上的可導航超鏈接。

```html
<a>這不是一般的可導航連結</a>
```

所以如果你的需求本質是「點了要導向某個目標」，通常就應該寫 `href`。

---

## 3.2 `target`

`target` 用來指定連結的開啟方式。

### `_self`

預設值。
表示在目前頁面或目前瀏覽上下文開啟。

```html
<a href="https://example.com" target="_self">目前頁面開啟</a>
```

### `_blank`

通常表示在新頁籤或新視窗開啟。

```html
<a href="https://example.com" target="_blank">新頁籤開啟</a>
```

### 初學者先記這個判準就夠了

* `_self`：留在原本這一頁
* `_blank`：另外開新頁

---

## 4. 常見連結類型

## 4.1 外部連結

連到其他網站或不同網域的頁面。

```html
<a href="http://www.qq.com" target="_self">騰訊</a>
<a href="http://www.itcast.cn" target="_blank">傳智播客</a>
```

### 判斷方式

只要目標不是你目前網站裡的頁面，通常就可以理解成外部連結。

---

## 4.2 內部連結

連到自己網站中的其他頁面。
通常會搭配**相對路徑**使用。

```html
<a href="./test.html">公司簡介</a>
```

### 判斷方式

如果連結目標仍然屬於同一個網站內部頁面，通常就是內部連結。

---

## 4.3 檔案連結

`<a>` 不只能連網頁，也可以連到檔案，例如：

* 圖片
* 音樂
* 影片
* PDF
* ZIP 壓縮檔

```html
<a href="./images/demo.jpg">看圖片</a>
<a href="./media/movie.mp4">看影片</a>
<a href="./media/music.mp3">聽音樂</a>
<a href="./media/test.pdf">看 PDF</a>
<a href="./test.zip">下載壓縮檔</a>
```

### 為什麼有些會開啟，有些會下載？

這取決於：

* 瀏覽器能不能直接處理該檔案
* 該資源本身的類型
* 是否有搭配 `download`

簡單記：

* **能預覽的資源**，瀏覽器常直接開啟
* **不能直接處理的資源**，瀏覽器常改成下載

---

## 4.4 錨點連結

錨點連結的作用是：**快速跳到頁面中的某個位置**。

```html
<a href="#two">第二季</a>
<h3 id="two">第二季介紹</h3>
```

### 原理

* 連結用 `href="#two"`
* 目標元素用 `id="two"`

點擊後，畫面就會跳到對應位置。

---

## 5. `href="#..."`、`href="#"`、`href=""` 差在哪？

這一段很容易搞混，必須分清楚。

## 5.1 `href="#某個id"`

這是**標準錨點連結**。
意思是跳到頁面中 `id="某個id"` 的元素。

```html
<a href="#two">跳到第二季</a>
<h3 id="two">第二季介紹</h3>
```

---

## 5.2 `href="#"`

這不是指向某個具名錨點，而是指向目前頁面最上方附近的位置。
常見效果是回到頁面頂部。

```html
<div style="height: 2000px;"></div>
<a href="#">回到頂部</a>
```

### 重點

`href="#"` 不是「什麼都不做」。
它通常仍會產生跳轉效果，只是跳回目前頁面的頂端附近。

---

## 5.3 `href=""`

這通常會被解讀為：**目標就是目前這份文件本身**。
在很多情況下，效果常接近重新載入當前頁面。

```html
<a href="">刷新本頁面</a>
```

### 最重要的結論

* `href="#id"`：跳到指定錨點
* `href="#"`：常見效果是回到頁面頂部
* `href=""`：常接近重新載入本頁

---

## 6. 錨點連結的完整做法

### 同頁跳轉

```html
<a href="#intro">前往介紹區</a>

<h2 id="intro">介紹區</h2>
```

### 跨頁跳轉

也可以跳到**其他頁面中的某個錨點**。

```html
<a href="./test.html#part2">前往 test.html 的 part2 區塊</a>
```

對應頁面裡要有：

```html
<h2 id="part2">第二部分</h2>
```

### 核心規則

錨點能不能跳成功，關鍵在於：

* `href` 裡的 `#名字`
* 目標元素上的 `id`
* 兩者必須一致

---

## 7. `download` 是什麼？

`download` 用來提示瀏覽器：
**這個連結比較希望以下載方式處理，而不是直接開啟。**

```html
<a href="./media/test.pdf" download>下載 PDF</a>
```

也可以指定下載後的檔名：

```html
<a href="./media/test.pdf" download="一份PDF檔.pdf">下載 PDF 檔案</a>
```

### 你要這樣記

* 沒有 `download`：可能開啟，也可能下載
* 有 `download`：更明確表達「這是下載用途」

但要注意，實際結果仍可能受到瀏覽器或資源來源影響。

---

## 8. 特殊用途的 `href`

## 8.1 `mailto:`

用來喚起郵件應用。

```html
<a href="mailto:10010@qq.com">郵件聯繫</a>
```

點下去後，系統可能會打開預設信箱程式。

---

## 8.2 `tel:`

用來喚起電話撥號功能。

```html
<a href="tel:10010">電話聯繫</a>
```

這在手機裝置上尤其常見。

---

## 8.3 喚起其他應用

某些應用會支援自己的連結協議，例如：

```html
<a href="line://msg/text/Hello%20World">使用 LINE 發送消息</a>
```

這類寫法的本質也是一種特殊連結：
不是導向一般網頁，而是嘗試喚起特定應用。

---

## 8.4 `javascript:`

你可以把 JavaScript 程式直接寫進 `href`：

```html
<a href="javascript:alert('Hello, World!');">點擊我</a>
```

### 但實務上不建議把它當成一般互動方案

原因很簡單：

* 行為直接寫在 `href` 裡，可讀性差
* 結構與行為耦合在一起，不好維護
* 需求本質若是「執行操作」，通常更適合用按鈕加事件處理

所以初學時知道它能做什麼就好，但不要把它當成日常首選。

---

## 9. `<a>` 和 `<button>` 怎麼分？

這是實務上很重要的判準。

### 用 `<a>` 的情況

當需求是：

* 前往其他頁面
* 前往某個網址
* 跳到頁面某個位置
* 下載檔案
* 開啟郵件或撥號功能

本質上都屬於**導向 / 導航**，所以適合用 `<a>`。

### 用 `<button>` 的情況

當需求是：

* 送出表單
* 開啟彈窗
* 切換選單
* 展開收合內容
* 執行某段互動邏輯

本質上都屬於**操作 / 行為**，所以更適合用 `<button>`。

### 一句話記憶

* **要去某個地方，用 `<a>`**
* **要做某件事，用 `<button>`**

---

## 10. 圖片也可以做成連結

可點擊的不一定只能是文字。
圖片也常會被包在 `<a>` 裡，形成可點擊圖片。

```html
<a href="http://www.baidu.com">
  <img src="./images/demo.jpg" alt="示意圖片" />
</a>
```

這在商品列表、Banner、文章封面都很常見。

# 常見問法

## `href` 一定要寫嗎？

如果你要的是**真正可導航的超鏈接**，通常就要寫。
沒有 `href` 的 `<a>` 可以存在，但不應把它直接理解成一般連結。

---

## `href="#"` 和 `href=""` 差在哪？

* `href="#"`：通常回到頁面頂部
* `href=""`：通常把目前文件當目標，效果常接近重新載入本頁

---

## `download` 一定會強制下載嗎？

不一定。
它是很明確的下載意圖提示，但實際行為仍可能受到瀏覽器與資源條件影響。

---

## `javascript:` 能不能用？

可以，但通常不建議當成一般互動方案。
若需求本質是操作，通常改用 `<button>` 搭配事件處理會更合適。

---

## `<a>` 可不可以只拿來當按鈕用？

不建議。
因為 `<a>` 的語意重點是**導向**，不是**操作**。
如果只是要執行功能，應優先考慮 `<button>`。

# 一句話總結

`<a>` 的本質不是「藍色可點文字」，而是 **HTML 中負責導航與導向的標籤**；只要需求是「帶使用者去某個地方」，就應優先想到它。
