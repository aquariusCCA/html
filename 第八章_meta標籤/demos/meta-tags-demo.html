HTML meta 標籤 Demo

`meta` 大多不直接顯示在頁面上，但它會影響瀏覽器如何理解文件、搜尋引擎怎麼產生摘要，以及社群平台分享時要抓哪些資訊。這個 demo 會把這些差異視覺化。

文件層級 `charset`、`viewport` 主要影響瀏覽器怎麼解析內容、怎麼在裝置上顯示頁面。
SEO 層級 `description`、`robots`、`keywords` 主要服務搜尋結果與索引控制。
分享層級 `og:title`、`og:description`、`og:image` 用來決定社群分享卡長什麼樣。
寫法差異 常見屬性分三類：`name`、`http-equiv`、`property`。

本頁涵蓋內容

  • `viewport` 對手機版顯示的差異模擬
  • SEO 摘要卡與 Open Graph 分享卡對照
  • `name` / `http-equiv` / `property` 的角色區分

4. `name` / `http-equiv` / `property` 的差異

        

          

            

              
              
              
            

          


          

            
name

            
一般頁面資訊最常用的寫法

            

              `name` 類型最常見，因為它負責描述頁面的普通 metadata，而不是模擬 HTTP 行為或社群協定。
            

            

            

              典型例子：`description`、`viewport`、`robots`、`author`。
            

          

        

      


      

        
5. 常見實務提醒

        

          

            `charset` 優先放前面
            先告訴瀏覽器編碼，能降低中文亂碼或解析錯誤的風險。
          

          

            `keywords` 重要性較低
            相比之下，`description` 對理解頁面內容通常更有實際價值。
          

          

            SEO 與 OG 不要混為一談
            `description` 是搜尋摘要導向；`og:description` 是社群分享導向。
