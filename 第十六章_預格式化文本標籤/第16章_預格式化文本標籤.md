# 預格式化文本標籤

> 所屬章節：第十六章｜預格式化文本標籤  
> 關鍵字：預格式化文本標籤、`<pre>`、preformatted text、保留空格、保留換行、等寬字體、程式碼展示、HTML 文本格式  
> 建議回查情境：想知道 `<pre>` 是做什麼的、想分清一般文字和預格式化文字的差別、想確認什麼時候適合用 `<pre>` 顯示內容

## 本節導讀

這篇整理 HTML 的預格式化文本標籤 `<pre>`。  
它的核心作用，是讓文字在畫面上盡量保留原本輸入時的空格、縮排和換行。

初學時，很多人只記得 `<pre>` 會顯示成等寬字體。  
但更重要的理解是：它適合處理「格式本身就有意義」的文本，例如程式碼、詩句、固定排版內容或需要保留縮排的文字。

## 你會在這篇學到什麼

- `<pre>` 的基本用途
- `<pre>` 會保留哪些格式特徵
- 什麼情況適合用預格式化文本標籤
- `<pre>` 和一般文字顯示方式有什麼差別

## 30 秒複習入口

- `<pre>` 用來顯示預格式化文本。
- 在 `<pre>` 中，空格、縮排和換行通常會被保留下來。
- 瀏覽器通常會把 `<pre>` 內容顯示成等寬字體。
- 它很適合展示程式碼、固定格式文字或需要保留排版的內容。

## 速查區

### 核心概念

- `<pre>` 是 preformatted text 的縮寫，可以理解成「預先排好格式的文字」。
- 它和一般文字顯示最大的差別，是會保留原本輸入的排版痕跡。

### 關鍵規則 / 判準

- 如果文字中的空格、縮排和換行本身很重要，可以考慮用 `<pre>`。
- `<pre>` 中的內容通常會以等寬字體顯示。
- 如果只是一般段落敘述，不需要保留輸入格式，通常不必用 `<pre>`。
- 展示程式碼時，常會看到 `<pre>` 和 `<code>` 搭配使用。

### 常見使用場景

- 展示程式碼
- 顯示保留縮排的文本
- 呈現詩句、歌詞或固定排版內容
- 顯示指令輸出、設定片段或格式敏感的文字

### 常見混淆點

- `<pre>` 不是單純為了字體變成等寬。
- 它的重點是保留文本格式，不只是改變外觀。
- 如果只是要做一般段落排版，仍應優先使用段落或其他語意標籤。

### 一句話對比

- `<pre>` 解決的是「文字格式要不要照原樣保留」；一般段落則重點在內容閱讀，不保證保留輸入排版。

## 正文筆記

### 這篇在解決什麼問題？

- 在一般 HTML 文字中，連續空格和換行通常會被瀏覽器自動折疊。
- 但有些內容本身就依賴縮排、空格與換行來表達結構，這時就需要 `<pre>`。

## 1. `<pre>` 的作用

> 作用：HTML 裡的 `pre` 元素，可定義預格式化的文本。在 `pre` 元素中的文本會保留空格和換行符。文本通常顯現為等寬字體。

- 這段話的核心是對的。
- `pre` 最重要的用途，是讓文字盡量保留原本寫下來時的排版格式。
- 瀏覽器通常也會把它顯示成等寬字體，讓縮排和字元對齊更容易閱讀。

## 2. 什麼時候適合用 `<pre>`

- 當內容的空格、tab、換行和縮排本身就有意義時。
- 最常見的例子是展示程式碼、命令列輸出、設定檔片段，或需要保留排版的文字內容。

### 這個場景怎麼理解？

- 如果把同一段程式碼放進一般標籤裡，換行與多餘空格常會被折疊，閱讀起來就不容易維持原本格式。
- 放進 `<pre>` 後，原本的排版通常就能更完整地保留下來。

## 3. 基本示例

```html
<body>
  body {
    background: #fff;
    font: 12px/24px 1.66;
  }

  <hr>

  <pre>
body {
  background: #fff;
  font: 12px/24px 1.66;
}
  </pre>
</body>
```

### 這個例子在說什麼？

- 上半部那段 CSS 文字沒有放在 `<pre>` 裡，所以瀏覽器會把它當成一般文字處理，空格與換行不一定照原樣呈現。
- 下半部同樣內容放進 `<pre>` 後，縮排、空格與換行就比較能保留下來。
- 中間的 `<hr>` 只是用來把兩段顯示結果分開，方便比較。

## 4. 使用 `<pre>` 時的注意點

- 不要把 `<pre>` 當成所有文字的預設容器。
- 只有在文字格式本身有意義時，才值得使用 `<pre>`。
- 如果你要展示的是程式碼語意本身，實務上也常搭配 `<code>` 一起使用。
- 如果內容只是一般文章敘述，用 `<p>` 或其他普通文字結構通常更合適。

## 常見問法

### 為什麼 `<pre>` 裡的文字看起來和一般段落不一樣？

- 因為瀏覽器通常會保留它的空格與換行，並以等寬字體顯示。
- 所以它更像在顯示「原始排版後的文字」，而不是重新幫你做一般段落排版。

### `<pre>` 最常用來做什麼？

- 最常見的是展示程式碼或其他格式敏感的內容。
- 只要內容需要保留縮排和換行，`<pre>` 都是常見選擇。

### `<pre>` 和一般段落 `<p>` 有什麼差別？

- `<p>` 重點在段落語意與一般閱讀。
- `<pre>` 重點在保留原始文字格式，兩者處理目的不同。
