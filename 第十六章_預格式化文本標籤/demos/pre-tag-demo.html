`<pre>` 的核心不是讓文字看起來像程式碼，而是保留原本輸入時的空格、縮排與換行。它適合展示程式碼、命令輸出、設定片段、詩句這類格式本身就有意義的內容；如果只是一般段落閱讀，通常應優先用 `<p>` 或其他普通文字結構。

`<pre>` 保留格式 它盡量維持原始空格、縮排、換行，而不是幫你自動整理成一般段落。
等寬字體不是核心 瀏覽器常用等寬字體呈現，但那只是常見外觀，不是主要語意。
適合格式敏感內容 程式碼、命令列輸出、設定檔、詩句與固定排版文字都很常見。
不是所有文字都該放進去 一般敘述若只求閱讀流暢，通常不應用 `<pre>`。

2. 一般文字和 `<pre>` 的差異

        

          

            一般段落
            
瀏覽器會幫你整理格式

            
<p>
body {
  background: #fff;
  font: 12px/24px 1.66;
}
</p>

            

              body { background: #fff; font: 12px/24px 1.66; }
            

            

              多個空格、縮排和原始換行，不會完整照原樣保留下來。
            

          


          

            預格式化文本
            
`<pre>` 會盡量保留原始排版

            
<pre>
body {
  background: #fff;
  font: 12px/24px 1.66;
}
</pre>

            
body {
  background: #fff;
  font: 12px/24px 1.66;
}

            

              這裡的格式感本身就是內容的一部分，所以保留原樣更合理。
            

          

        

      


      

        
3. `<pre>` 和 `<code>` 的關係

        

          

            

              
              
            

          


          

            
保留格式

            
只有格式重要時，用 `<pre>` 就可以

            

              `<pre>` 關心的是排版格式；若內容本身還需要明確標示為程式碼，再加上 `<code>` 會更完整。
            

            

            

          

        

      


      

        
4. 不要把 `<pre>` 當萬用整齊工具

        

          

            

              
              
            

          


          

            
不建議

            
不是所有整齊排版都代表要保留原樣

            

              如果內容的重點是自然閱讀，而不是忠實保留每個空格與換行，那通常不該優先選 `<pre>`。
            

            

            

            

              `<pre>` 用錯地方時，反而會讓一般說明文字變得僵硬，閱讀感也會變差。
            

          

        

      


      

        
5. 實務提醒

        

          

            先判斷格式是不是內容的一部分
            如果空格、縮排和換行本身有意義，再考慮用 `<pre>`。
          

          

            不要只記等寬字體
            真正要記的是「保留格式」，不是外觀像不像程式碼區塊。
          

          

            一般排版問題優先想 CSS
            若只是想讓畫面整齊或留白一致，應先考慮樣式，而不是把內容塞進 `<pre>`。
