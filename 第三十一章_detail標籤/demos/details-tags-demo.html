第三十一章：details 標籤 Demo

這個示例展示原生 <details> / <summary> 如何建立展開收合區塊，並對照 open、open="false"、沒有 summary、FAQ 與 JavaScript 的 toggle 事件。

一句話核心

<details> 是原生收合區塊，<summary> 是可點擊標題，open 決定一開始是否展開。

details

外層容器，負責管理展開與收合狀態。

summary

使用者看到並點擊的摘要、標題或開關文字。

open

布林屬性；只要出現就代表預設展開。

1. 基本展開 / 收合

什麼是 HTML？

HTML 是用來描述網頁內容結構的標記語言。

這一題預設展開

因為外層 <details> 寫了 open 屬性。

<details>
  <summary>什麼是 HTML？</summary>
  <p>HTML 是用來描述網頁內容結構的標記語言。</p>
</details>

<details open>
  <summary>這一題預設展開</summary>
  <p>因為 details 寫了 open 屬性。</p>
</details>

2. open 是布林屬性

open="false" 仍是展開

看似 false，但仍預設展開

HTML 布林屬性只看有沒有出現，不看字串值是不是 false。

<details open="false">
  ...
</details>

要收合就不要寫 open

沒有 open，所以預設收合

預設收合時，直接省略 open 屬性。

<details>
  ...
</details>

3. FAQ 很適合用 details

忘記密碼怎麼辦？

請點選登入頁面的「忘記密碼」，並依照系統指示重設密碼。

可以修改會員資料嗎？

可以，請登入後進入會員中心修改個人資料。

HTML 可以控制網頁樣式嗎？

HTML 主要負責結構，樣式通常由 CSS 控制。

為什麼適合

FAQ 的閱讀模式通常是先看問題，再決定是否展開答案。這正好符合 summary 作為問題、詳細內容作為答案的結構。

4. 表單旁的補充說明

查看帳號名稱規則
  • 長度建議 4 到 20 個字元。
  • 可使用英文字母、數字與連字號。
  • 不要使用密碼、生日或身分證字號。

適合的原因

欄位規則不是每次都需要完整顯示，但使用者需要時可以展開查看。這種補充資訊適合用 <details>。

5. 沒有 summary 會怎樣

可運作，但不建議

這個 details 沒有明確的 summary，瀏覽器通常會提供預設標題文字。

預設文字可能不符合你的頁面語境。

建議明確寫出標題

查看詳細說明

明確的 summary 讓使用者知道展開後會看到什麼。

6. 用 JavaScript 讀取與切換狀態

點擊後會觸發 toggle 事件

這個範例會讀取 details.open，並把目前狀態顯示在右側。

details.addEventListener('toggle', () => {
  console.log(details.open ? '目前展開' : '目前收合');
});

7. 角色與注意事項速查

元素 / 屬性 角色 提醒
details 可展開 / 收合的外層容器。 不是普通排版容器，重點是資訊是否適合收合。
summary 摘要、標題或開關文字。 實務上建議放在 details 裡最前面。
open 控制初始是否展開。 布林屬性，有寫就代表展開。
toggle 展開 / 收合狀態改變時觸發的事件。 搭配 JavaScript 讀取 details.open。

9. 不適合硬用

  • 重要警告或錯誤訊息
  • 交易風險或法律聲明
  • 複雜分頁切換介面
  • 需要大量狀態控制的 accordion
  • 只是為了排版包一層容器

10. 常見錯誤

把 open 當事件

open 是布林屬性；事件是 toggle。

以為 open="false" 會收合

布林屬性只看有沒有出現。要收合就不要寫 open。

把重要內容藏起來

使用者必須立刻看到的資訊，不應只放在收合區裡。
