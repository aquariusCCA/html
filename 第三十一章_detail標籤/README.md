# 第三十一章｜`details` 標籤

> 章節定位：理解 `<details>` 如何建立原生的展開收合區塊  
> 建議閱讀方式：先掌握 `<summary>` 與 `open` 的角色，再回看常見應用情境

## 本章在學什麼

這一章整理 HTML5 的 `<details>` 標籤。  
重點是建立一個穩定觀念：它不是單純的容器，而是原生的展開 / 收合互動元件。

## 本章與整體地圖的關係

- 前置知識：前面已學過多種語意化與互動標籤，這一章補的是「可收合資訊區塊」的原生寫法。
- 這章的核心任務：理解 `<details>`、`<summary>`、`open` 之間的關係。
- 讀完後通常接：之後在做 FAQ、補充說明、摺疊式內容區塊時，都可以回查這一章。

## 易混淆主題

- `<details>` vs 一般 `div`：前者有原生收合互動，後者沒有。
- `<summary>` vs 詳細內容：前者是標題 / 開關，後者才是展開後內容。
- 有沒有 `open`：會決定初始是展開還是收合。
