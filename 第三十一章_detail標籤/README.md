# 第三十一章｜`details` 標籤

> 章節定位：理解 `<details>` 如何建立原生的展開收合區塊  
> 建議閱讀方式：先掌握 `<details>`、`<summary>`、`open` 的基本關係，再補看常見誤區、適用情境與進階用法

## 本章在學什麼

這一章整理 HTML5 的 `<details>` 標籤。

重點不是只記住語法，而是理解：

- `<details>` 是原生的展開 / 收合互動元件
- `<summary>` 是使用者點擊的摘要或標題
- `open` 是控制初始展開狀態的布林屬性
- 什麼情境適合使用 `<details>`
- 什麼情境不適合硬用 `<details>`

## 易混淆主題

- `<details>` vs 一般 `div`：前者有原生展開 / 收合互動，後者只是一般容器。
- `<summary>` vs 詳細內容：前者是標題 / 開關，後者才是展開後看到的內容。
- `open` vs JavaScript 事件：`open` 是布林屬性，不是事件。
- `open` vs `open="false"`：布林屬性只看有沒有出現，不是看字串值是不是 `false`。
- 沒有 `<summary>` vs 不能使用：沒有 `<summary>` 仍可能顯示預設標題，但實務上不建議。
- `<details>` vs 自製 Accordion：簡單收合可用 `<details>`，複雜互動通常需要額外 JavaScript。
