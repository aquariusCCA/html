同樣都是粗體、斜體、底線或刪除線，背後代表的意思可能完全不同。這章的重點不是只看畫面，而是分清楚哪些標籤偏外觀、哪些標籤同時在表達重要性、語氣強調、修訂痕跡或內容狀態。

外觀相似，不代表語意相同 `b` 和 `strong` 常都會變粗，但前者偏突顯，後者偏重要性。
語氣和樣式是兩回事 `i` 不一定是在強調，`em` 才更接近「這個詞要重讀」。
修訂痕跡有專門語意 `ins` 表示新增內容，`del` 表示刪除內容，和單純底線 / 刪除線不同。
純外觀很多時候交給 CSS 如果只是想讓字看起來不一樣，通常應先想樣式，而不是硬塞語意標籤。

本頁會示範

  • `b / strong`、`i / em`、`u / ins`、`s / del` 的差異
  • 哪些標籤偏外觀、哪些偏語意
  • 何時該交給 CSS 處理純視覺需求
  • 實務上如何挑選更貼切的標籤

2. 八個常見標籤的角色分工

        

          

            
<b> vs <strong>

            
`b` 偏向文字突顯；`strong` 偏向重要性。

            
本頁重點詞：HTML。
注意：送出後資料將無法復原。

          


          

            
<i> vs <em>

            
`i` 偏向不同語氣或術語；`em` 偏向語氣上的重讀。

            
單字 status 常出現在介面中。
我說的是 現在，不是明天。

          


          

            
<u> vs <ins>

            
`u` 偏向標記呈現；`ins` 偏向新增、插入內容。

            
請留意 專有名詞拼寫。
本次更新已加入：支援匯出 PDF 功能

          


          

            
<s> vs <del>

            
`s` 偏向不再適用；`del` 偏向被刪除的內容。

3. 只是想改外觀時，很多時候交給 CSS

        

          

            

              
              
            

          


          

            
語意優先

            
內容本身有意思，就別只把它當樣式

            

              當文字背後真的有重要性、重讀感或修訂含義時，HTML 標籤比單純的樣式更貼切。
            

            

            

          

        

      


      

        
4. 最容易混淆的選擇題

        

          

            較貼切
            
內容真的有語意時，選對應標籤

            
<p><strong>注意：</strong>表單送出後將無法復原。</p>
<p>我說的是 <em>現在</em>，不是明天。</p>

            

              注意：表單送出後將無法復原。

              我說的是 現在，不是明天。
            

            

              這裡的粗體和斜體不是為了好看，而是有內容上的重要性與語氣差異。
            

          


          

            較弱
            
只因為效果像，就隨便互換

            
<p><b>注意：</b>表單送出後將無法復原。</p>
<p>我說的是 <i>現在</i>，不是明天。</p>

            

              注意：表單送出後將無法復原。

              我說的是 現在，不是明天。
            

            

              畫面接近，但少了「重要性」和「重讀強調」的語意判讀。
            

          

        

      


      

        
5. 實務提醒

        

          

            先問自己要表達什麼
            你要的是重要性、語氣、修訂痕跡，還是單純外觀？先分清楚再選標籤。
          

          

            外觀像，不代表可以互換
            `strong` 不等於 `b`，`em` 不等於 `i`，重點在語意而不是畫面相似度。
          

          

            純裝飾需求優先交給 CSS
            HTML 更適合描述內容本身；若只是視覺微調，樣式表通常更穩定。
