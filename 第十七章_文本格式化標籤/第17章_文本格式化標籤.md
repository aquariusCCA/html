# 文本格式化標籤

> 所屬章節：第十七章｜文本格式化標籤
> 關鍵字：文本格式化標籤、`<b>`、`<strong>`、`<i>`、`<em>`、`<u>`、`<ins>`、`<s>`、`<del>`、文字強調、語意標籤、文字樣式
> 建議回查情境：想分清楚「只是改外觀」和「同時表達語意」的差別、想知道加粗／斜體／底線／刪除線該用哪個標籤、想知道實務上該怎麼選

## 本節導讀

HTML 的文本格式化標籤，不只是讓文字看起來變粗、變斜、加底線或刪除線。
更重要的是：**有些標籤只是在改變呈現方式，有些標籤則同時在表達內容的語意。**

初學時很容易把這些標籤全部當成「外觀工具」來記。
但更穩定的理解方式是：

* 有些標籤偏向**視覺呈現**
* 有些標籤偏向**語意表達**
* 實務上若只是想改外觀，通常也會優先考慮交給 **CSS**

所以這一節真正要學的，不是「哪個標籤會變粗」，而是：
**同樣的畫面效果，背後代表的意思可能完全不同。**

---

## 你會在這篇學到什麼

* 常見文本格式化標籤有哪些
* `b / strong`、`i / em`、`u / ins`、`s / del` 的差別
* 哪些標籤偏向外觀，哪些標籤帶有語意
* 實務上什麼情況該用語意標籤，什麼情況交給 CSS 更合適

---

## 30 秒複習入口

* `b` 和 `strong` 都可能讓文字變粗，但 `strong` 強調的是**重要性**。
* `i` 和 `em` 都可能讓文字變斜，但 `em` 強調的是**語氣上的重讀或強調**。
* `u` 和 `ins` 都可能看起來有底線，但 `ins` 表示的是**新增、插入的內容**。
* `s` 和 `del` 都可能看起來有刪除線，但 `del` 表示的是**被刪除的內容**。
* 單純想改樣式時，很多情況更適合使用 CSS，而不是直接靠 HTML 標籤處理。

---

## 速查區

### 核心概念

文本格式化標籤可以粗分成兩類：

1. **偏視覺效果的標籤**
   主要影響文字外觀，例如粗體、斜體、底線、刪除線。

2. **偏語意表達的標籤**
   除了可能改變畫面效果，也會傳達內容的重要性、強調、修訂或狀態變更。

---

### 常見對照表

| 分類    | 偏外觀   | 偏語意        |
| ----- | ----- | ---------- |
| 粗體相關  | `<b>` | `<strong>` |
| 斜體相關  | `<i>` | `<em>`     |
| 底線相關  | `<u>` | `<ins>`    |
| 刪除線相關 | `<s>` | `<del>`    |

---

### 快速判準

* 想表達「**這段內容很重要**」：優先考慮 `strong`
* 想表達「**這個詞需要語氣強調**」：優先考慮 `em`
* 想表達「**這是新增內容**」：優先考慮 `ins`
* 想表達「**這是被刪除的內容**」：優先考慮 `del`
* 如果只是想讓字看起來不一樣，通常可交給 CSS 處理

## 正文筆記

## 1. 這些標籤在解決什麼問題？

在 HTML 裡，文字不一定只是普通顯示。
有時你會希望某段內容：

* 更重要
* 被特別強調
* 是後來新增的
* 已經被刪除
* 或只是外觀上想和一般文字不同

這時就會用到文本格式化標籤。

但要注意：
**「看起來一樣」不代表「意思一樣」。**

例如：

* 兩段文字都可能看起來是粗體
* 但其中一段可能只是外觀效果
* 另一段可能是在告訴瀏覽器與讀者：**這段內容很重要**

這就是文本格式化標籤最容易混淆、也最值得學的地方。

---

## 2. 常見標籤總覽

```html
<b>加粗</b>
<strong>重要內容</strong>

<i>斜體</i>
<em>需要強調的語氣</em>

<u>底線</u>
<ins>新增內容</ins>

<s>不再適用的內容</s>
<del>已刪除的內容</del>
```

這組例子不是要你只看外觀，而是要你理解：

* 有些標籤偏向「怎麼顯示」
* 有些標籤偏向「這段內容代表什麼」

---

## 3. 每一組標籤怎麼理解？

### `b` 和 `strong`

#### `b`

`b` 比較偏向**文字上的突顯**，重點在呈現效果，而不是強調內容的重要性。
它可以讓文字看起來更醒目，但通常**不代表這段內容在語意上特別重要**。

#### `strong`

`strong` 表示這段內容具有**較強的重要性**。
瀏覽器常會把它顯示成粗體，但它的核心不是「變粗」，而是「重要」。

#### 怎麼選？

* 只是想讓某個詞更醒目：`b`
* 想表達警告、提醒、關鍵重點：`strong`

#### 例子

```html
<p>本頁重點詞：<b>HTML</b></p>
<p><strong>注意：</strong>表單送出後將無法復原。</p>
```

---

### `i` 和 `em`

#### `i`

`i` 比較偏向讓文字以不同語氣或風格呈現，常見於：

* 外來語
* 術語
* 心裡想法
* 分類名稱
* 與一般敘述不同語氣的片段

它不一定是在做「強調」。

#### `em`

`em` 表示這段內容需要**語氣上的強調**。
重點不是外觀，而是朗讀時會自然重讀的那種強調感。

#### 怎麼選？

* 只是想表現不同語氣、術語、外來語：`i`
* 想表達「這個字我要特別重讀」：`em`

#### 例子

```html
<p>單字 <i>status</i> 常出現在介面中。</p>
<p>我說的是 <em>現在</em>，不是明天。</p>
```

---

### `u` 和 `ins`

#### `u`

`u` 現在不建議單純理解成「幫文字加底線」。
更精確地說，它偏向一種**標記、註記式的呈現**。
雖然畫面上常顯示為底線，但語意上不等於「新增內容」。

另外，實務上也要小心：
**底線很容易讓使用者誤以為這是連結。**

#### `ins`

`ins` 表示這段內容是**後來插入、補上的內容**。
它的重點在「內容有被新增」這件事，而不是底線本身。

#### 怎麼選？

* 單純只是想加底線：通常先考慮 CSS，而不是直接用 `u`
* 想表達這是修訂後新增的內容：`ins`

#### 例子

```html
<p>請閱讀 <u>重要名詞</u> 的拼寫。</p>
<p>最新版本已補充：<ins>支援匯出 PDF 功能</ins></p>
```

---

### `s` 和 `del`

#### `s`

`s` 表示這段內容**已不再正確、已不再適用，或失去現實有效性**。
它常顯示為刪除線，但不一定表示真的從文件中刪除了。

#### `del`

`del` 表示這段內容是**從文件中被刪除的內容**。
這比 `s` 更明確，因為它直接對應到「修訂、移除」的語意。

#### 怎麼選？

* 表示資訊過時、不再適用：`s`
* 表示文件修訂時被刪除的內容：`del`

#### 例子

```html
<p><s>原價 1200 元</s>，現在特價 899 元。</p>
<p>修訂紀錄：<del>需附紙本申請書</del></p>
```

---

## 4. 為什麼教材常建議優先理解語意標籤？

因為 HTML 不只是拿來「把畫面做出來」，也是在描述內容本身的結構與意義。

所以：

* `strong` 不只是粗體，而是重要性
* `em` 不只是斜體，而是語氣強調
* `ins` 不只是底線，而是新增內容
* `del` 不只是刪除線，而是刪除內容

也就是說，
**語意標籤不是在和樣式競爭，而是在補充「這段文字為什麼要這樣呈現」。**

這種寫法通常比單純追求畫面效果更有價值，因為它讓 HTML 更接近「描述內容」而不只是「畫畫面」。

---

## 5. 實務上怎麼選？

可以用下面這個判斷方式：

### 情況一：只是想改外觀

例如只是想把字變粗、變斜、加底線。
這種情況很多時候更適合用 CSS。

```html
<span class="highlight">重點文字</span>
```

```css
.highlight {
  font-weight: bold;
}
```

---

### 情況二：內容本身有語意

例如：

* 這段內容很重要
* 這個字需要重讀
* 這是後來新增的內容
* 這是被刪除的內容

這時就應該優先考慮對應的語意標籤。

---

## 6. 這一章最容易搞混的地方

### 混淆一：畫面一樣，就以為可以互換

不行。
畫面效果接近，不代表語意相同。

---

### 混淆二：後者一定比前者高級

也不能這樣理解。
重點不是「哪個比較高級」，而是「哪個更符合內容本身的意思」。

---

### 混淆三：只要看到底線就用 `u`

不建議這樣記。
因為 `u` 不等於「所有底線需求」，而且底線也容易和超連結混淆。
單純裝飾性的底線，多半先交給 CSS。

---

## 一句話總結

**文本格式化標籤不只是讓文字變樣子，更是在告訴別人：這段文字為什麼要這樣呈現。**

---

## 常見問法

### `b` 和 `strong` 都會變粗，差在哪？

差在語意。
`b` 偏向呈現上的突顯；`strong` 偏向重要性強調。

---

### `i` 和 `em` 看起來都會變斜，差在哪？

`i` 比較像不同語氣或文字類型的呈現；`em` 比較像說話時會特別重讀的強調。

---

### `u` 和 `ins` 有什麼不同？

`u` 偏向標記式的呈現；`ins` 明確表示這是新增、插入的內容。
兩者可能都出現底線，但意思不同。

---

### `s` 和 `del` 有什麼不同？

`s` 表示內容不再適用；`del` 表示內容已被刪除。
前者偏狀態，後者偏修訂。

---

### 為什麼很多情況會建議交給 CSS？

因為 CSS 更適合處理純外觀需求；
HTML 更適合描述內容本身的結構與語意。
