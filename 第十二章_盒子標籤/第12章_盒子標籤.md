# 盒子標籤

> 所屬章節：第十二章｜盒子標籤  
> 關鍵字：盒子標籤、`<div>`、`<span>`、container、generic container、無語意標籤、區塊容器、行內容器、block、inline  
> 建議回查情境：想知道 `div` 和 `span` 是做什麼的、想分清兩者的差別、想確認它們是語意標籤還是通用容器、想回查預設顯示方式和 CSS 的關係

## 本節導讀

這篇整理 HTML 裡常見的兩個通用容器標籤：`<div>` 和 `<span>`。  
它們最常用來包住內容，方便分組、套樣式、掛上 class 或 id，或配合 JavaScript 操作。

初學時，很多教材會把它們直接叫成「大盒子」和「小盒子」。  
這個說法能幫助入門，但如果只停在這裡，容易把「沒有語意」「預設顯示方式」和「實際用途」混在一起，所以這篇會把三者分清楚。

## 你會在這篇學到什麼

- `div` 和 `span` 各自在做什麼
- 為什麼它們常被說成沒有語意的容器標籤
- `div` 和 `span` 的預設顯示差異
- 為什麼不能把預設顯示方式當成永遠不變的硬規則

## 30 秒複習入口

- `<div>` 和 `<span>` 都是通用容器，用來包住內容。
- 它們本身不強調特定語意，重點是分組與承載內容。
- `<div>` 預設通常像區塊元素，常從新的一行開始。
- `<span>` 預設通常像行內元素，常可和同一行其他內容並排。

## 速查區

### 核心概念

- `div` 和 `span` 都可以理解成「容器」。
- 它們不是拿來描述特定內容種類的語意標籤，而是較通用的包裝元素。

### 關鍵規則 / 判準

- `div` 常用來包住較大塊、較完整的區域或區塊內容。
- `span` 常用來包住行內的小範圍文字或局部內容。
- `div` 與 `span` 的差別，最常見的是預設顯示方式不同，而不是誰比較大、誰比較高級。
- 預設顯示方式可以被 CSS 改變，因此不該把「一行只能放一個 div」當成絕對規則。

### 常見特點

- `<div>` 預設通常會各自佔據一個區塊。
- `<span>` 預設通常會留在文字流中，和其他行內內容排在一起。
- 兩者都很常搭配 `class`、`id` 與 CSS 使用。

### 常見混淆點

- 沒有語意，不等於沒有用途。
- `div` / `span` 常被拿來做版面分組，但不代表它們只負責布局。
- 「預設是 block / inline」和「永遠只能這樣顯示」不是同一件事。

### 一句話對比

- `div` 比較像通用的區塊容器，`span` 比較像通用的行內容器，但兩者的顯示方式都可以由 CSS 調整。

## 正文筆記

### 這篇在解決什麼問題？

- 當頁面內容需要分區、分組，或只想包住某一小段文字方便套樣式時，就會遇到 `div` 和 `span`。
- 這篇的目標，是分清它們的定位、差別，以及初學時最容易誤會的地方。

## 1. `div` 和 `span` 是什麼

> `<div>` 和 `<span>` 是沒有語意的，它們就是一個盒子，用來裝內容。

- 這句話的核心意思可以保留：它們都可以當成容器來包住內容。
- 所謂「沒有語意」，是指它們不像 `<h1>`、`<p>`、`<nav>` 那樣，會直接告訴你這段內容是標題、段落或導覽區。
- 但這不代表它們沒用；相反地，它們在實務上非常常見，因為很多時候你就是需要一個通用容器。

## 2. 名稱怎麼理解

### `div`

- `div` 是 `division` 的縮寫，可以理解成分割、分區。
- 所以它很常被用來把頁面切成不同區塊。

### `span`

- `span` 有跨度、範圍的意思。
- 它更常用在文字中的小範圍內容，像是某幾個字、某一段行內內容。

## 3. `div` 和 `span` 的預設差別

### `<div>`

- `<div>` 預設通常表現得像區塊元素。
- 在很多常見情況下，它會從新的一行開始，並把一個區域包成獨立區塊。

### `<span>`

- `<span>` 預設通常表現得像行內元素。
- 它常會留在原本那一行裡，和其他文字或行內內容一起排列。

### 這裡最容易誤會的是什麼？

- 很多入門教材會說：`div` 一行只能放一個，`span` 一行可以放很多個。
- 比較穩定的理解方式是：`div` 預設通常會形成獨立區塊，`span` 預設通常不會。
- 這是預設顯示結果，不是 HTML 規格在說「一行只能有一個 div」。
- 如果用 CSS 改變顯示方式，畫面表現也可能跟預設不同。

## 4. 基本示例

```html
<div>我是一個 div 標籤，我通常會形成獨立區塊</div>
<div>我是一個 div 標籤，我通常會形成獨立區塊</div>
<span>百度</span>
<span>新浪</span>
<span>搜狐</span>
```

### 這個例子在說什麼？

- 兩個 `<div>` 都各自包住一整塊內容，所以畫面上常會看到它們分成上下兩塊。
- 三個 `<span>` 則各自包住小段文字，因此常會排在同一行文字流裡。
- 這個例子主要是在展示兩者的預設顯示差異，而不是說它們只能這樣使用。

## 5. 什麼時候用 `div`，什麼時候用 `span`

### 適合用 `div` 的情況

- 你要包住一大塊區域，例如卡片、區塊、欄位、段落群組或模組容器。
- 你想把一整塊內容當成一個群組來套樣式或操作。

### 適合用 `span` 的情況

- 你只想包住一句話中的幾個字、某個詞、某個數字或一小段行內內容。
- 你想局部改變文字顏色、字重，或加上行內標記。

## 6. 使用時的注意點

- 如果內容本身有更明確的語意標籤可以用，通常應優先用語意標籤，而不是一律用 `div`、`span` 代替。
- 不要把「沒有語意」誤解成「可以隨便亂包、不用管結構」。
- 也不要把預設顯示方式誤解成絕對規則；真正控制畫面呈現的，仍然還有 CSS。

## 常見問法

### `div` 和 `span` 誰比較常用？

- 兩者都很常用，但用途不同。
- `div` 常用來包大區塊，`span` 常用來包小範圍行內內容。

### `div` 是不是一定獨佔一行？

- 比較準確的說法是：它預設通常會形成獨立區塊。
- 但這是預設顯示結果，不是永遠不能改。

### 既然它們沒有語意，為什麼還要學？

- 因為實務上很常需要一個通用容器來分組內容、套樣式或掛程式邏輯。
- 沒有語意不代表沒價值，只代表它們不像標題、段落那樣直接說明內容類型。
