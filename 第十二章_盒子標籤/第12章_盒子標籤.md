# 盒子標籤

> 所屬章節：第十二章｜盒子標籤
> 關鍵字：盒子標籤、`<div>`、`<span>`、container、generic container、無語意標籤、區塊容器、行內容器、block、inline
> 建議回查情境：想知道 `div` 和 `span` 是做什麼的、想分清兩者差別、想確認它們是不是語意標籤、想理解它們和 CSS 的關係、想知道實務上什麼時候該用哪一個

## 本節導讀

這篇整理 HTML 裡最常見的兩個**通用容器標籤**：`<div>` 和 `<span>`。

它們最常見的用途，是把內容包起來，方便：

* 分組內容
* 套用 CSS 樣式
* 加上 `class` / `id`
* 配合 JavaScript 操作

很多初學教材會把它們叫成「大盒子」和「小盒子」。
這種說法可以幫助快速入門，但如果只記到這裡，很容易把下面幾件事混在一起：

* 沒有語意
* 預設顯示方式不同
* 實務上的使用情境不同

所以這篇的重點，是把這三件事拆開來理解。

## 30 秒複習入口

* `<div>` 和 `<span>` 都是通用容器，用來包住內容。
* 它們本身不強調特定內容語意，重點是分組與承載內容。
* `<div>` 預設通常像區塊元素，常會形成獨立區塊。
* `<span>` 預設通常像行內元素，常會留在同一行文字流裡。

* 有更適合的語意標籤時，通常應優先使用語意標籤。

## 速查區

### 核心概念

* `div` 和 `span` 都可以理解成「通用容器」。
* 它們不是用來直接描述內容種類的語意標籤，而是比較通用的包裝元素。
* 沒有語意，不等於沒有用途；反而因為夠通用，所以在實務上非常常見。

### 關鍵判準

當你在判斷要用 `div` 還是 `span` 時，可以先問自己：

1. 我現在要包的是**一整塊區域**，還是**一句話中的局部內容**？
2. 這裡有沒有更適合的語意標籤可以用？
3. 我現在在處理的是內容結構，還是只是想方便套樣式或操作？

### 常見特點

* `<div>` 預設通常會形成獨立區塊
* `<span>` 預設通常會留在文字流中
* 兩者都很常搭配 `class`、`id`、CSS、JavaScript 使用

### 常見混淆點

* 沒有語意，不代表沒有功能
* `div` / `span` 不只是拿來做版面，也常拿來做分組與局部包裝
* 預設是 block / inline，不代表永遠只能這樣顯示
* `div` / `span` 的差別不只是「大或小」，而是**包住的範圍與使用情境**不同

### 一句話對比

* `div` 比較像通用的**區域容器**
* `span` 比較像通用的**局部行內容器**

## 正文筆記

## 1. `div` 和 `span` 是什麼？

`<div>` 和 `<span>` 都是 HTML 的**通用容器標籤**。

它們的共同點是：

* 都能把內容包起來
* 都很常用來分組、套樣式、掛識別名稱
* 都不直接表達明確內容語意

這裡所謂的「沒有語意」，是指它們不像下面這些標籤那樣，會直接說明內容是什麼：

* `<h1>`：標題
* `<p>`：段落
* `<nav>`：導覽區
* `<header>`：頁首
* `<section>`：內容區段

而 `div` / `span` 比較像是在說：

> 我先把這一塊內容包起來，但它本身不主動說明這塊內容是哪一種類型。

所以，「沒有語意」的真正意思不是「沒用」，而是「它是通用容器，不負責直接定義內容類型」。

---

## 2. 名稱怎麼理解？

### `div`

`div` 是 `division` 的縮寫，可以理解成：

* 分區
* 區域
* 區塊切分

所以它很常被用來包住一整塊較完整的區域。

---

### `span`

`span` 有「跨度、範圍」的意思。

它更常被拿來包住：

* 某幾個字
* 一小段文字
* 某個數字
* 某個局部行內內容

所以和 `div` 相比，`span` 通常比較偏向**小範圍、局部、行內**的包裝。

---

## 3. `div` 和 `span` 的預設差別

### `<div>`

`<div>` 預設通常表現得像**區塊元素**。

常見結果是：

* 從新的一行開始
* 形成一個相對獨立的區塊
* 前後內容通常會分開顯示

---

### `<span>`

`<span>` 預設通常表現得像**行內元素**。

常見結果是：

* 留在目前文字流中
* 和同一行其他文字一起排列
* 通常不會自己切成上下兩塊

---

### 這裡最容易誤會的是什麼？

很多入門教材會說：

* `div` 一行只能放一個
* `span` 一行可以放很多個

這種說法不夠精確。

更穩定的理解方式是：

* `div` 預設通常會形成獨立區塊
* `span` 預設通常不會脫離原本的文字流

所以差別的重點在於**預設顯示方式**，而不是「HTML 規則規定一行只能出現一個 div」。

---

## 4. 為什麼不能把 block / inline 當成硬規則？

因為這只是**預設顯示方式**，而不是永遠不能改變的規則。

例如：

```html
<div class="inline-box">第一個區塊</div>
<div class="inline-box">第二個區塊</div>
```

```css
.inline-box {
  display: inline;
}
```

上面這兩個 `div` 雖然預設通常是區塊元素，但在 CSS 改成 `display: inline` 之後，就可能和文字一樣排在同一行。

所以更精準的理解應該是：

* `div` / `span` 有預設顯示方式
* 但真正控制畫面呈現的，還有 CSS

---

## 5. 基本示例

```html
<div>這是一個區塊容器</div>
<div>這是另一個區塊容器</div>

<span>紅色</span>
<span>藍色</span>
<span>綠色</span>
```

### 這個例子在說什麼？

* 兩個 `<div>` 常會形成上下分開的兩塊
* 三個 `<span>` 常會留在同一行文字流中
* 這個例子是在展示**預設顯示差異**
* 它不是在說 `div` / `span` 只能這樣使用

---

## 6. 更貼近實戰的例子

### 用 `div` 包一整塊模組

```html
<div class="card">
  <h2>商品名稱</h2>
  <p>這是一張商品卡片的內容。</p>
</div>
```

這裡的 `div` 適合用來包住整張卡片，因為它是一整塊區域。

---

### 用 `span` 包局部內容

```html
<p>目前價格：<span class="price">999 元</span></p>
```

這裡的 `span` 適合用來包住「999 元」這一小段文字，方便做局部樣式處理。

例如：

```css
.price {
  color: red;
  font-weight: bold;
}
```

這就是 `span` 很典型的使用方式：
**不改變整段結構，只處理局部內容。**

---

## 7. 什麼時候用 `div`？

通常在下面這些情況，比較適合用 `div`：

* 要包住一整塊區域
* 要建立模組、卡片、欄位、容器、版面區塊
* 想把多個元素視為同一組來操作
* 需要一個通用的區域包裝，但又沒有更適合的語意標籤

例如：

* 卡片容器
* 表單區塊
* 側邊欄內容群組
* 某一塊功能模組

你可以把 `div` 想成：

> 我現在不是只包某幾個字，而是要包一整塊內容。

---

## 8. 什麼時候用 `span`？

通常在下面這些情況，比較適合用 `span`：

* 只想包住一句話中的局部內容
* 想局部改變字色、字重、背景色
* 想替某個詞、某個數字、某段小範圍內容加樣式或標記
* 不希望它脫離原本的文字流

例如：

* 強調某幾個字
* 包住價格
* 包住狀態文字
* 包住某個標記用字串

你可以把 `span` 想成：

> 我不是在包一整塊區域，而是在包一小段局部內容。

---

## 9. `div` 和 `span` 的真正差別，不只是 block / inline

這點很重要。

很多人學到最後，只記住：

* `div = block`
* `span = inline`

這樣記不能說錯，但太淺了。

更完整的理解應該是：

* `div` 常拿來包**區域**
* `span` 常拿來包**局部內容**
* block / inline 只是它們的**預設顯示方式**
* 實務上選哪一個，還要看你包住的內容範圍與用途

所以，`div` 和 `span` 真正的差別可以分成兩層：

### 第一層：預設顯示方式不同

* `div` 預設通常像 block
* `span` 預設通常像 inline

### 第二層：常見使用範圍不同

* `div` 常包整塊內容
* `span` 常包局部行內內容

第二層才是更接近實務判斷的重點。

---

## 10. 使用時的注意點

### 有更適合的語意標籤時，應優先使用語意標籤

例如：

* 頁首內容：`<header>`
* 導覽區：`<nav>`
* 段落文字：`<p>`
* 主標題：`<h1>`
* 區段內容：`<section>`

不要因為 `div` 很萬用，就把所有東西都寫成 `div`。

---

### 不要把「沒有語意」理解成「可以隨便亂包」

`div` / `span` 雖然是通用容器，但仍然應該有清楚目的，例如：

* 分組
* 套樣式
* 操作 DOM
* 包局部內容

如果只是無意義地一層包一層，結構就會變得混亂。

---

### 不要只靠外觀決定要用哪個標籤

不是看到它排成一整塊就一定用 `div`，也不是看到字小小一段就一定用 `span`。
更好的判斷方式是：

* 這是區域還是局部內容？
* 這裡有沒有更適合的語意標籤？
* 我現在是為了結構、分組，還是局部處理？

---

## 11. 錯誤示範 vs 較好的寫法

### 情境一：明明是段落，卻只用 `div`

#### 不夠好的寫法

```html
<div>這是一段文章內容。</div>
```

如果它本質上是段落文字，通常 `<p>` 會更合適。

#### 較好的寫法

```html
<p>這是一段文章內容。</p>
```

---

### 情境二：想改局部文字樣式，卻包成整塊 `div`

#### 不夠好的寫法

```html
<div>特價商品</div>
```

如果只是想標記句子中的局部內容，`div` 可能太大了。

#### 較好的寫法

```html
<p>今天的主打是 <span class="highlight">特價商品</span></p>
```

---

### 情境三：所有東西都用 `div`

#### 不夠好的想法

> 反正 `div` 很通用，所以全部都用 `div`

這樣做會讓 HTML 結構失去很多可讀性。

#### 更好的想法

> 有語意標籤時用語意標籤；沒有更適合的時候，再用 `div` / `span`

---

## 12. 寫 `div` / `span` 時可以怎麼自我檢查？

寫完後可以快速問自己：

1. 這裡是不是有更適合的語意標籤？
2. 我包的是一整塊區域，還是一小段局部內容？
3. 我用這個容器的目的是什麼？分組、樣式、還是操作？
4. 我是不是只是因為習慣，什麼都用 `div`？
5. 如果拿掉這個容器，會不會其實完全不影響結構與功能？

如果這幾題都能回答清楚，通常你就不是在亂用容器標籤。

## 常見問法

### `div` 和 `span` 誰比較常用？

兩者都很常用，但用途不同。

* `div` 常用來包大區塊
* `span` 常用來包局部行內內容

---

### `div` 是不是一定獨佔一行？

更精確的說法是：
它**預設通常**會形成獨立區塊。

但這是預設顯示結果，不是永遠不能改。

---

### `span` 是不是只能包文字？

不是只能包文字。
更精確地說，它是通用的**行內容器**，常見情況下最常拿來包文字中的局部內容。

---

### 既然它們沒有語意，為什麼還要學？

因為實務上很常需要通用容器來：

* 分組內容
* 套樣式
* 加上識別名稱
* 配合程式操作

它們的價值就在於**夠通用、夠靈活**。

## 一句話總結

`<div>` 和 `<span>` 的本質，都是用來包住內容的通用容器；
前者比較常包**整塊區域**，後者比較常包**局部行內內容**，而它們的預設顯示方式都可以由 CSS 調整。
