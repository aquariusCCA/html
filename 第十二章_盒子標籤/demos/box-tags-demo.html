`div` 和 `span` 是 HTML 裡最常見的泛用容器。它們本身幾乎沒有內容語意，但非常適合拿來包裝區塊、綁 class、套樣式與配合 JavaScript 操作。

`div` 常作為區塊級泛用容器，適合包整塊區域、卡片、版面區段。
`span` 常作為行內泛用容器，適合包一小段文字、價格、關鍵詞、狀態。
本質是容器 它們的重點是「包起來」，而不是自帶強烈語意。
樣式靠 CSS `div` / `span` 的外觀和排列方式，真正多半是由 `display` 與其他 CSS 決定。

本頁會示範

  • `div` 與 `span` 的典型使用情境
  • block / inline 的預設差異
  • CSS `display` 改變外觀但不改變本質
  • 何時應優先用更有語意的標籤

1. `div` 和 `span` 的典型差異

Block Container

`div` 適合包整塊內容

`div` 常拿來包一張卡片、一整個區域、或一組排版單位。它通常和區塊內容一起工作。

5. 常見實務提醒

        

          

            `div` 管整塊
            當你要包卡片、區域、版面模組、群組內容時，`div` 很常是合理選擇。
          

          

            `span` 管局部
            當你只想標記一句話裡的一小段文字，`span` 通常比 `div` 更自然。
          

          

            不要把語意元素都換成 `div`
            若有更具體的 `
`、`
`、`
`、`
`，通常應優先使用它們。
