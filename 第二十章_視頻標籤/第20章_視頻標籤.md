# 視頻標籤

> 所屬章節：第二十章｜視頻標籤

> 建議回查情境：想知道 HTML 視頻怎麼插入、想分清 `controls` / `autoplay` / `muted` / `poster` / `preload` 的差別、想知道 `<source>` 為什麼要寫多組、想理解視頻自動播放限制

## 本節導讀

這篇整理 HTML 的視頻標籤 `<video>`。  
重點不只是把影片放進頁面，而是理解影片來源怎麼指定、播放控制屬性在做什麼、封面與預載入怎麼用，以及為什麼實務上常搭配多個 `<source>` 提供不同格式。

原始內容能看出核心方向，但有幾個明顯問題：章次寫錯、把音頻章節的說法混進來、`<source>` 說明不夠準，且總結偏像口語講稿。  
這裡改成較穩定的學習順序：先理解 `<video>` 基本用途，再看常見屬性、`<source>` 備援寫法，以及自動播放限制。

## 你會在這篇學到什麼

- `<video>` 是什麼、基本語法怎麼寫
- `controls`、`autoplay`、`muted`、`loop`、`poster`、`preload` 各自在做什麼
- `width`、`height` 對影片顯示有什麼影響
- 為什麼 `<source>` 常用來提供多種影片格式
- 視頻標籤和音頻標籤哪些地方像、哪些地方不同

## 30 秒複習入口

- `<video>` 用來在網頁中嵌入影片。
- `controls` 會顯示播放控制介面；`poster` 是未播放前顯示的封面圖。
- `autoplay` 不代表一定會自動播；現代瀏覽器常限制未經使用者互動的自動播放。
- `muted` 常和 `autoplay` 一起出現，因為靜音自動播放比較容易被允許。
- `<source>` 常用來提供多個影片格式，讓瀏覽器挑自己支援的版本。

## 速查區

### 核心概念

- `<video>` 的核心任務是把影片資源放進頁面。
- 真正要一起學的是：來源、播放控制、封面、尺寸、格式支援與播放限制。

### 關鍵規則 / 判準

- `src` 指向影片資源位置。
- `controls` 決定要不要顯示播放器介面。
- `autoplay` 只是提出自動播放要求，實際是否生效仍要看瀏覽器政策。
- `muted` 可讓影片初始為靜音，常用於自動播放情境。
- `poster` 是影片尚未播放前顯示的封面。
- `<source>` 常用於同一支影片提供多個格式版本。

### 常見使用場景

- 教學影片、產品示範、短片展示
- 首屏 Banner 影片
- 需要自訂播放流程的網站影片區塊
- 需要針對不同瀏覽器提供不同影片格式的情境

### 常見混淆點

- `autoplay` 不是寫了就一定自動播放。
- `<source>` 不是只為了「低版本瀏覽器」，而是為了多格式備援。
- `poster` 是封面圖，不是影片第一幀的保證替代品。
- 視頻標籤和音頻標籤很像，但視頻還要處理畫面與尺寸。

### 一句話抓核心

- 視頻標籤的重點不只是把影片放進頁面，而是同時處理來源、播放控制、封面、尺寸、格式備援與播放限制。

## 正文筆記

### 這篇在解決什麼問題？

- 在 HTML 裡，如果你要把影片放進頁面，最直接的方式就是用 `<video>`。
- 但只會寫 `<video src="...">` 還不夠；你還需要分清控制屬性怎麼影響播放、封面和尺寸怎麼處理，以及不同格式為什麼要準備多個來源。

## 1. `<video>` 標籤在做什麼？

- `<video>` 用來在網頁中嵌入影片內容。
- 它是 HTML 的媒體元素之一，除了播放控制外，還會牽涉到畫面呈現、尺寸與封面圖。

```html
<video
  src="影片地址"
  controls
  autoplay
  muted
  loop
  poster="./images/poster.jpg"
  width="640"
  height="360"
></video>
```

## 2. 常見屬性怎麼理解？

### `src`

- 指定影片檔案的位置。
- 可以寫相對路徑或其他可被瀏覽器讀取的 URL。

### `controls`

- 讓瀏覽器顯示播放、暫停、進度列、音量、全螢幕等控制介面。
- 初學時通常會先保留，因為不寫它時，使用者可能沒有明確的操作入口。

### `autoplay`

- 表示頁面載入後希望自動播放影片。
- 但現代瀏覽器通常會限制未經使用者互動的有聲自動播放，所以不能把它當成一定成功的規則。

### `muted`

- 表示影片初始為靜音。
- 在很多瀏覽器中，若你希望影片自動播放，常會搭配 `muted` 一起使用。

### `loop`

- 影片播完後會重新開始。
- 常見於短循環背景影片或展示型動畫片段。

### `poster`

- 指定影片在未播放前顯示的封面圖。
- 對於教學頁、產品頁或 Banner 區塊來說，這通常很有用。

### `preload`

- 用來提示瀏覽器要不要先載入影片資料。
- 常見值可以先記成 `none`、`metadata`、`auto`。
- 初學時先知道它在處理「預載入程度」就夠，不必把它理解成絕對保證。

### `width` / `height`

- 控制影片在頁面上的顯示尺寸。
- 和圖片類似，如果同時指定兩者但比例和原影片不一致，畫面可能變形。

## 3. `<source>` 為什麼常和 `<video>` 一起用？

- 不同瀏覽器對影片格式的支援可能不同。
- 因此實務上常把多個 `<source>` 寫在 `<video>` 裡，讓瀏覽器挑選自己支援的格式。
- 這個做法更準確的理解是「多格式備援」，不只是「照顧低版本瀏覽器」。

```html
<video controls poster="./images/poster.jpg">
  <source src="./media/movie.mp4" type="video/mp4" />
  <source src="./media/movie.webm" type="video/webm" />
  <source src="./media/movie.ogg" type="video/ogg" />
  您的瀏覽器不支持 video 標籤。
</video>
```

### 這段例子在展示什麼？

- 同一支影片可以準備多個檔案格式。
- 瀏覽器會依序嘗試能播放的來源。
- 最後那段文字是 fallback：如果瀏覽器完全不支援 `<video>`，就至少顯示提示訊息。

## 4. 常見影片格式怎麼理解？

### `MP4`

- 最常見的網頁影片格式之一。
- 相容性通常不錯，因此是很常見的預設選項。

### `WebM`

- 常見的網頁影片格式之一。
- 有些情境會拿來作為 `MP4` 之外的備援格式。

### `Ogg`

- 也可能出現在影片格式備援中。
- 初學時先把它理解成備援格式之一即可。

## 5. 自動播放要注意什麼？

- 原文提到「谷歌瀏覽器需要配合 muted 靜音播放」，方向接近實務，但更穩定的理解是：現代瀏覽器普遍會對自動播放做限制。
- 特別是帶聲音的媒體，常需要使用者互動後才允許播放。
- 所以實務上不要把 `autoplay` 學成「一定會自動播放」。

## 6. 視頻標籤和音頻標籤哪些地方像，哪些地方不同？

### 相似處

- 都屬於 HTML 媒體元素。
- 都可使用 `controls`、`autoplay`、`loop`、`muted`。
- 都可以搭配 `<source>` 提供多個來源。

### 不同處

- `<video>` 還要處理畫面呈現，因此常會碰到 `poster`、尺寸與版面配置。
- `<audio>` 主要處理聲音，不需要處理影片畫面。

## 常見問法

### `controls` 一定要寫嗎？

- 不一定。
- 但如果你沒有自己做播放器介面，通常會先保留 `controls`，讓使用者能直接操作。

### 為什麼寫了 `autoplay` 還是不會自動播？

- 因為瀏覽器通常會限制未經互動的自動播放。
- 這不是標籤失效，而是瀏覽器播放政策在介入。

### `muted` 為什麼常和 `autoplay` 一起出現？

- 因為靜音影片的自動播放，通常比有聲影片更容易被瀏覽器允許。

### `<source>` 是不是只在老瀏覽器才需要？

- 不是。
- 更穩定的理解是：當你要提供多格式備援時，就很適合使用 `<source>`。
