# 第二十八章｜度量衡標籤

> - 章節定位：理解 `<meter>` 如何表示「已知範圍內的測量值」  
> - 建議閱讀方式：先分清 `<meter>` 和 `<progress>` 的語意差異，再回看 `value`、`min`、`max`、`low`、`high`、`optimum` 的屬性關係

## 本章在學什麼

這一章整理 HTML 的度量衡標籤 `<meter>`。  
重點是建立一個穩定觀念：`<meter>` 不是拿來表示任務進度，而是拿來表示某個數值在「已知範圍」中的位置。

例如：

- 手機電量剩多少
- 磁碟空間用了多少
- 考試分數落在哪個區間
- 庫存比例是否偏低
- 某個測量值是否接近理想範圍

本章也會特別釐清 `<meter>` 的屬性規則：  
`value` 在撰寫 HTML 時應明確指定；但瀏覽器解析時，會對缺漏、格式錯誤或超出範圍的值進行容錯處理。  
`min`、`max`、`low`、`high`、`optimum` 則共同決定整體範圍、低區間、高區間與理想區間。

## 易混淆主題

- `<meter>` vs `<progress>`：前者是狀態值或測量值，後者是任務進度。
- `value` vs `optimum`：前者是目前值，後者是理想值或理想區間的參考點。
- `value` 是否可以省略：撰寫 HTML 時應明確指定；但瀏覽器解析時會對缺漏或錯誤值容錯。
- `min` / `max`：負責定義整體範圍；若沒寫，預設是 `0` 到 `1`。
- `low` / `high` / `optimum`：不是單純改顏色，而是在描述低區間、高區間與理想區間。
- 寫了條狀 UI vs 用對語意：重點不在外觀像不像進度條，而在你描述的是「狀態值」還是「任務進度」。

## 本章記憶口訣

> `<meter>` 看的是「目前狀態落在哪裡」；  
> `<progress>` 看的是「任務完成到哪裡」。

寫 `<meter>` 時，先確認三件事：

1. 這是狀態值或測量值，不是任務進度嗎？
2. 這個數值是否有明確的 `min` 和 `max`？
3. 是否需要用 `low`、`high`、`optimum` 補充區間判斷？
