# 第二十八章｜度量衡標籤

> 章節定位：理解 `<meter>` 如何表示已知範圍內的測量值  
> 建議閱讀方式：先分清 `<meter>` 和 `<progress>` 的差別，再回看各屬性與電量範例

## 本章在學什麼

這一章整理 HTML 的度量衡標籤 `<meter>`。  
重點是建立一個穩定觀念：它不是拿來表示任務進度，而是拿來表示某個數值在已知範圍中的位置。

## 本章與整體地圖的關係

- 前置知識：前面已學過語意標籤、表單與布局等基礎內容，這一章補的是帶有數值語意的 UI 元素。
- 這章的核心任務：分清 `<meter>` 的用途，並掌握 `value`、`min`、`max`、`low`、`high`、`optimum` 的關係。
- 讀完後通常接：之後在做狀態顯示、儀表式資訊或比較 `meter` / `progress` 時，都可以回查這一章。

## 易混淆主題

- `<meter>` vs `<progress>`：前者是狀態值，後者是任務進度。
- `value` vs `optimum`：前者是目前值，後者是理想值。
- 寫了條狀 UI vs 用對語意：重點不在外觀像條，而在你描述的是哪種資訊。
