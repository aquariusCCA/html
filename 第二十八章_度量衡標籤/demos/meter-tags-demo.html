第二十八章：度量衡標籤 Demo

這個示例用原生 <meter> 展示「已知範圍內的測量值」，並和 <progress> 的任務進度語意做對照。

記憶口訣

<meter> 看的是「目前狀態落在哪裡」；<progress> 看的是「任務完成到哪裡」。

狀態值

電量、磁碟用量、庫存比例，都是已知範圍中的狀態。

有上下限

能明確說出 min 和 max，才適合用 gauge 表達。

不是任務進度

上傳、下載、安裝、送出流程應該使用 <progress>。

1. 常見測量值範例

手機電量 70 / 100
70%

電量是狀態值，範圍是 0 到 100，且高區間通常比較理想。

磁碟已用空間 420GB / 500GB
已使用 420GB / 500GB

已用空間不是越高越好，所以理想值放在低區間。

考試分數 85 / 100
85 分

分數越高通常越理想，optimum 可放在高區間。

室內溫度 26°C / 0-40°C
26°C

溫度不是越高越好，理想值通常落在中間區間。

2. <meter> 和 <progress> 差在哪

電量 70%：用 meter

目前電量 70%
70% 
<meter min="0" max="100" value="70" low="20" high="80" optimum="90">
  70%
</meter>

這是在描述一個狀態值落在已知範圍中的位置。

上傳 70%：用 progress

上傳進度 70%
70% 
<progress max="100" value="70">
  70%
</progress>

這是在描述任務完成程度，不是測量值。

3. 屬性關係速查

屬性 角色 提醒
value 目前測量值。 實務上應明確指定，否則語意不完整。
min / max 整體範圍上下限。 沒寫時預設是 0 到 1。
low / high 低區間上界與高區間下界。 應符合 min ≤ low ≤ high ≤ max。
optimum 理想值或理想區間參考點。 它不是目前值；目前值是 value。

4. 互動觀察：調整目前值與理想值

互動 meter value=65, low=20, high=80, optimum=90
65 / 100 
<meter min="0" max="100" low="20" high="80" optimum="90" value="65">
  65 / 100
</meter>

區間判讀

  • 整體範圍：0 到 100
  • 目前值落在中間區間。
  • 理想值落在高區間，通常表示越高越理想。

注意

不同瀏覽器可能用不同方式呈現 <meter> 的顏色。請把 low、high、optimum 理解成語意與區間資料，不要只背顏色。

5. 預設範圍陷阱

若沒有寫 min 和 max，<meter> 預設範圍是 0 到 1。

容易誤會

<meter value="70">70%</meter>
70%

70 不是 70%，而是超過預設最大值 1。

百分比應寫清楚

<meter min="0" max="100" value="70">70%</meter>
70%

這樣才是 70 / 100。

6. 不適合使用 <meter> 的例子

任務進度

「下載完成 80%」是流程進度，應該用 <progress>。

沒有已知範圍的單純數字

<meter value="12">12cm</meter> 不清楚 12 在什麼範圍裡，通常直接寫文字即可。

只想畫條狀 UI

HTML 標籤先看語意。資料不是已知範圍內的測量值，就不要硬用 <meter>。

7. 使用前判斷流程

1. 這是狀態值嗎？

電量、分數、使用量、溫度是；上傳或下載進度不是。

2. 範圍清楚嗎？

先定義 min / max，再填入目前 value。

3. 需要區間判準嗎？

有偏低、偏高、理想區間時，再補 low、high、optimum。
