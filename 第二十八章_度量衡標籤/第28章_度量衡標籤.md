# 度量衡標籤（`<meter>`）

> 所屬章節：第二十八章｜度量衡標籤  
> 關鍵字：度量衡標籤、`meter`、gauge、標量測量、範圍值、電量條、磁碟用量、`progress`  
> 建議回查情境：想知道 `<meter>` 是做什麼的、想分清 `<meter>` 和 `<progress>` 的差別、想確認 `value` / `min` / `max` / `low` / `high` / `optimum` 怎麼搭配時

## 本節導讀

這篇整理 HTML 的 `<meter>` 標籤，也就是常說的度量衡標籤。  
它的用途不是表示「任務做了多少進度」，而是表示某個數值在已知範圍內的位置，例如電量、磁碟使用量、考試分數、溫度區間等。

原始內容方向是對的，但還不足以直接當成穩定筆記：`value` 被寫成必填、屬性之間的關係沒有講清楚、`meter` 和 `progress` 的差別只被簡短提醒，讀者容易只記住零碎屬性名稱。  
這裡採「輕度修復」重構，保留原本用手機電量示範的方向，但先穩住概念，再整理屬性規則與例子。

## 你會在這篇學到什麼

- `<meter>` 在解決什麼問題
- 什麼情況適合用 `<meter>`，什麼情況要改用 `<progress>`
- `value`、`min`、`max`、`low`、`high`、`optimum` 分別在做什麼
- 為什麼同樣是條狀顯示，`meter` 和 `progress` 不能混用

## 30 秒複習入口

- `<meter>` 用來表示已知範圍內的測量值，不是任務進度。
- 常見場景有電量、磁碟用量、分數、溫度。
- `value` 是目前數值，但不是必填；沒寫時會有預設值。
- `min` / `max` 定義整體範圍，`low` / `high` 定義偏低與偏高區間，`optimum` 定義理想值或理想區域。
- 如果你要表示「上傳到 70%」、「下載完成 3/10」，那應該用 `<progress>`。

## 速查區

### 核心概念

- `<meter>` 表示某個數值位於已知範圍中的哪裡。
- 它適合描述狀態或測量值，不適合描述工作完成進度。
- 它常被視為 gauge，也就是儀表或刻度概念。

### 什麼時候用 `<meter>`

- 電量還剩多少
- 磁碟已用空間比例
- 考試分數落在哪個區間
- 某個數值是否接近理想範圍

### 什麼時候不要用 `<meter>`

- 上傳進度
- 下載進度
- 安裝進度
- 任務完成百分比

這類情況應該改用 `<progress>`。

### 重要屬性

- `value`：目前數值。
- `min`：範圍最小值，預設為 `0`。
- `max`：範圍最大值，預設為 `1`。
- `low`：偏低區間的上界。
- `high`：偏高區間的下界。
- `optimum`：理想值或理想區域所在位置。

### 常見錯誤

- 以為 `value` 一定要寫。
- 把 `<meter>` 當成進度條使用。
- 只記屬性名字，卻沒理解 `low` / `high` / `optimum` 是在描述區間判準。
- 沒先想清楚最小值和最大值，導致數值範圍不合理。

### 一句話抓核心

- `<meter>` 是拿來顯示「某個測量值落在已知範圍中的哪裡」，不是拿來顯示「任務做完了多少」。

## 正文筆記

### 1. `<meter>` 在解決什麼問題？

有些數值不是單純想顯示一個數字，而是想讓使用者一眼看出它在整體範圍中的位置。  
例如手機電量剩多少、磁碟空間用了多少、分數目前高不高，這些都屬於「已知範圍內的測量值」。

`<meter>` 就是為這類情況設計的。  
它不是表示流程進度，而是表示數值狀態。

### 2. `<meter>` 和 `<progress>` 差在哪？

這兩個標籤外觀都可能像條狀圖，所以很容易混淆，但語意不同：

- `<meter>`：表示測量值或狀態值，範圍已知。
- `<progress>`：表示任務完成進度。

可以這樣記：

- 「電量剩 70%」適合用 `<meter>`
- 「檔案上傳到 70%」適合用 `<progress>`

差別不在畫面長得像不像，而在你表達的是「狀態」還是「進度」。

### 3. 重要屬性怎麼理解？

#### `value`

- 表示目前的數值。
- 它不是必填；若未指定或格式錯誤，會有預設值。
- 實務上通常還是會寫，因為沒有目前值就很難表達測量結果。

#### `min`

- 表示範圍最小值。
- 若未指定，預設為 `0`。

#### `max`

- 表示範圍最大值。
- 若未指定，預設為 `1`。

#### `low`

- 表示偏低區間的上界。
- 用來告訴瀏覽器：低於哪個位置，可視為較低或較不理想。

#### `high`

- 表示偏高區間的下界。
- 用來告訴瀏覽器：高於哪個位置，可視為較高。

#### `optimum`

- 表示理想值，或理想區域所在位置。
- 它會配合 `low` 和 `high`，讓瀏覽器理解哪一段範圍比較理想。

### 4. 屬性之間的關係怎麼看？

先有整體範圍，再談區間判準：

1. `min` 和 `max` 先定義總範圍
2. `value` 表示目前數值落在哪
3. `low` 和 `high` 補充「偏低」與「偏高」的界線
4. `optimum` 補充「最理想」大致在哪一段

因此真正的閱讀順序是：

- 先看範圍有多大
- 再看目前值在哪
- 最後看哪些區間被視為低、高、最佳

### 5. 範例怎麼理解？

#### 範例 1：基本電量顯示

![範例1示意圖.png](./images/image_1776155061644_97034888.png)

```html
<span>手機電量：</span>
<meter value="50" min="0" max="100" low="20" high="80"></meter><br>
```

這段的意思是：

- 電量範圍從 `0` 到 `100`
- 目前值是 `50`
- `20` 以下可視為偏低
- `80` 以上可視為偏高

它在表達的是「目前電量落在整體區間中的哪裡」。

#### 範例 2：加入理想值

![範例2示意圖.png](./images/image_1776155061649_e9d1d1b6.png)

```html
<span>手機電量：</span>
<meter value="70" min="0" max="100" low="20" high="80" optimum="90"></meter><br>
```

這段和上一個例子相比，多了 `optimum="90"`。  
它的意思是：目前值是 `70`，而系統認為更理想的狀態接近 `90`。

也就是說，`optimum` 不是目前值，而是「理想值參考點」。

### 6. 使用 `<meter>` 時該先想什麼？

在寫 `<meter>` 之前，先回答這三個問題：

1. 這是狀態值，還是任務進度？
2. 我的最小值和最大值是什麼？
3. 需不需要定義偏低、偏高或理想值？

如果第一題回答是「任務進度」，那通常就不該用 `<meter>`。

## 常見問法

### `value` 一定要寫嗎？

不一定。  
依 MDN，若沒有指定 `value`，會有預設值；但實務上通常還是會明確寫出目前值，讓語意更完整。

### `meter` 和 `progress` 為什麼不能混用？

因為它們表達的不是同一件事。  
`meter` 表達的是測量值在範圍中的位置；`progress` 表達的是任務完成了多少。

### `optimum` 是不是等於目前值？

不是。  
`optimum` 是理想值，`value` 才是目前值。

### 只寫 `value` 可以嗎？

可以，但前提是你接受預設範圍。  
若沒有寫 `min` 和 `max`，預設範圍會是 `0` 到 `1`，所以大多數百分比情境通常還是會一起把 `max` 寫清楚。

## 速查區

### 核心概念

`<meter>` 表示某個數值位於已知範圍中的位置。

它適合描述狀態或測量值，例如：

- 電量剩多少
- 磁碟空間用了多少
- 考試分數落在哪個區間
- 庫存比例是否偏低
- 某個數值是否接近理想範圍

### 什麼時候用 `<meter>`

當你可以明確回答下面三件事時，就適合考慮 `<meter>`：

1. 這是一個測量值或狀態值嗎？
2. 這個數值是否有已知的最小值與最大值？
3. 使用者是否需要知道它在整體範圍中的位置？

例如：

```html
<p>
  手機電量：
  <meter min="0" max="100" value="70">70%</meter>
</p>
```

這裡的 `70` 不是任務做到 70%，而是「目前電量這個狀態值是 70/100」。

### 什麼時候不要用 `<meter>`

以下情境應該使用 `<progress>`，不要使用 `<meter>`：

- 上傳進度
- 下載進度
- 安裝進度
- 表單送出進度
- 任務完成百分比

例如：

```html
<label for="upload">上傳進度：</label>
<progress id="upload" value="70" max="100">70%</progress>
```

這裡的 `70` 表示任務已經完成了 70%，所以語意上應該用 `<progress>`。

## `<meter>` 和 `<progress>` 的差別

兩者外觀都可能像條狀圖，但語意完全不同。

| 標籤 | 核心語意 | 適合情境 | 範例 |
| --- | --- | --- | --- |
| `<meter>` | 已知範圍內的測量值 | 狀態、比例、儀表值 | 電量 70%、磁碟使用 80%、分數 85 分 |
| `<progress>` | 任務完成進度 | 任務、流程、處理進度 | 上傳 70%、下載 3/10、安裝中 |

一句話記法：

> `<meter>` 看的是「目前狀態落在哪裡」；`<progress>` 看的是「任務完成到哪裡」。

## 重要屬性

### `value`

`value` 表示目前測量值。

```html
<meter value="50" min="0" max="100">50%</meter>
```

上面代表目前值是 `50`，整體範圍是 `0` 到 `100`。

要注意兩個層次：

- 從 HTML 作者撰寫規則來看，`<meter>` 應明確指定 `value`。
- 從瀏覽器解析行為來看，如果 `value` 沒有指定或格式錯誤，瀏覽器會先以 `0` 作為候選值，再依照 `min` / `max` 把實際值夾在合法範圍內。

因此，實務上請把 `value` 當成「應該要寫」的屬性。  
沒有目前值，`<meter>` 的語意就不完整。

### `min`

`min` 表示整體範圍的最小值。

```html
<meter min="0" max="100" value="40">40%</meter>
```

若沒有指定 `min`，預設為 `0`。

### `max`

`max` 表示整體範圍的最大值。

```html
<meter min="0" max="100" value="40">40%</meter>
```

若沒有指定 `max`，預設為 `1`。

所以如果你寫：

```html
<meter value="40">40%</meter>
```

它不是代表 `40 / 100`。  
因為預設範圍是 `0` 到 `1`，`40` 會超出範圍，瀏覽器最後會把它視為最大值，畫面看起來會像滿格。

百分比情境通常應該明確寫出：

```html
<meter min="0" max="100" value="40">40%</meter>
```

### `low`

`low` 表示低區間的上界。

```html
<meter min="0" max="100" low="20" value="10">10%</meter>
```

可以理解成：

- `0` 到 `20` 是低區間
- 目前值 `10` 落在低區間

若沒有指定 `low`，瀏覽器會把低區間邊界視為 `min`。

### `high`

`high` 表示高區間的下界。

```html
<meter min="0" max="100" low="20" high="80" value="90">90%</meter>
```

可以理解成：

- `80` 到 `100` 是高區間
- 目前值 `90` 落在高區間

若沒有指定 `high`，瀏覽器會把高區間邊界視為 `max`。

### `optimum`

`optimum` 表示理想值所在的位置，並且會影響瀏覽器判斷哪一段範圍比較理想。

```html
<meter min="0" max="100" low="20" high="80" optimum="90" value="70">70%</meter>
```

這段代表：

- 整體範圍是 `0` 到 `100`
- 低區間是 `0` 到 `20`
- 高區間是 `80` 到 `100`
- `optimum="90"` 落在高區間
- 因此瀏覽器會理解成「數值越高越理想」
- 目前值是 `70`，尚未進入最理想區間

`optimum` 不是目前值。  
目前值是 `value`，理想值或理想區域參考點才是 `optimum`。

## 屬性之間的合法關係

寫 `<meter>` 時，請先記住這個順序：

1. 先用 `min` / `max` 定義整體範圍
2. 再用 `value` 表示目前測量值
3. 再視需要用 `low` / `high` 定義低區間與高區間
4. 最後用 `optimum` 表示哪個位置或哪段範圍比較理想

合法關係可以簡化記成：

```txt
min ≤ value ≤ max
min ≤ low ≤ high ≤ max
min ≤ optimum ≤ max
```

比較完整地看：

- `value` 必須落在 `min` 和 `max` 之間。
- `low` 如果有寫，必須落在 `min` 和 `max` 之間。
- `high` 如果有寫，必須落在 `min` 和 `max` 之間。
- `optimum` 如果有寫，必須落在 `min` 和 `max` 之間。
- 如果 `low` 和 `high` 都有寫，應符合 `low ≤ high`。

## `low` / `high` / `optimum` 怎麼共同判斷理想區間？

`low` 和 `high` 會把整體範圍切成三段：

```txt
min ── low ── high ── max
 低區間   中間區間   高區間
```

`optimum` 則用來指出哪一段比較理想。

### 情況 1：`optimum` 落在高區間

```html
<meter min="0" max="100" low="20" high="80" optimum="90" value="85">85%</meter>
```

因為 `optimum="90"` 高於 `high="80"`，所以代表高區間比較理想。  
這種情境可以用在「電量越高越好」、「考試分數越高越好」。

### 情況 2：`optimum` 落在低區間

```html
<meter min="0" max="100" low="20" high="80" optimum="10" value="15">15%</meter>
```

因為 `optimum="10"` 低於 `low="20"`，所以代表低區間比較理想。  
這種情境可以用在「延遲越低越好」、「錯誤率越低越好」、「耗電量越低越好」。

### 情況 3：`optimum` 落在中間區間

```html
<meter min="0" max="100" low="20" high="80" optimum="50" value="55">55%</meter>
```

因為 `optimum="50"` 落在 `low` 和 `high` 之間，所以代表中間區間比較理想。  
這種情境可以用在「溫度維持在中間區間最好」、「濕度不要太高也不要太低」。

### 注意：顏色不應作為唯一判斷依據

不同瀏覽器可能用不同方式呈現 `<meter>` 的顏色與樣式。  
`low` / `high` / `optimum` 的核心價值是語意與區間判斷，不是保證某個瀏覽器一定顯示成特定顏色。

因此不要寫成：

> `low` 一定會變紅，`high` 一定會變綠。

應該理解成：

> 這些屬性幫助瀏覽器知道目前值和低、中、高、理想區間之間的關係；實際顏色由瀏覽器與 CSS 決定。

## 範例理解

### 範例 1：基本電量顯示

```html
<label for="battery">手機電量：</label>
<meter id="battery" value="50" min="0" max="100">50%</meter>
```

這段的意思是：

- 電量範圍從 `0` 到 `100`
- 目前值是 `50`
- 文字內容 `50%` 可作為備援內容，也讓語意更清楚

這裡表達的是「目前電量落在整體區間中的哪裡」。

### 範例 2：加入低、高區間

```html
<label for="battery2">手機電量：</label>
<meter id="battery2" value="15" min="0" max="100" low="20" high="80">15%</meter>
```

這段的意思是：

- 電量範圍是 `0` 到 `100`
- 目前電量是 `15`
- `20` 以下屬於低區間
- 目前值落在低區間

### 範例 3：加入理想值

```html
<label for="battery3">手機電量：</label>
<meter id="battery3" value="70" min="0" max="100" low="20" high="80" optimum="90">70%</meter>
```

這段的意思是：

- 目前值是 `70`
- 低區間是 `0` 到 `20`
- 高區間是 `80` 到 `100`
- `optimum="90"` 落在高區間
- 因此這個 gauge 的語意是「越接近高區間越理想」

### 範例 4：磁碟使用量

```html
<label for="disk">磁碟使用量：</label>
<meter id="disk" min="0" max="500" value="420" low="100" high="400" optimum="100">
  已使用 420GB / 500GB
</meter>
```

這個例子要特別注意：磁碟使用量不是越高越好。  
所以 `optimum="100"` 放在低區間，代表使用量低一點比較理想。

## 常見錯誤

### 錯誤 1：把 `<meter>` 當成進度條

```html
<!-- 不建議：這是在表示上傳任務進度 -->
<meter value="70" min="0" max="100">70%</meter>
```

應改成：

```html
<label for="upload2">上傳進度：</label>
<progress id="upload2" value="70" max="100">70%</progress>
```

### 錯誤 2：忘記寫 `max`

```html
<!-- 不建議 -->
<meter value="70">70%</meter>
```

因為 `<meter>` 預設範圍是 `0` 到 `1`，所以 `70` 不是 70%。  
百分比情境應改成：

```html
<meter min="0" max="100" value="70">70%</meter>
```

### 錯誤 3：把 `optimum` 當成目前值

```html
<!-- 不建議：如果你想表示目前值是 70，應該寫 value，不是 optimum -->
<meter min="0" max="100" optimum="70">70%</meter>
```

應改成：

```html
<meter min="0" max="100" value="70" optimum="90">70%</meter>
```

`value` 是目前值；`optimum` 是理想值或理想區間的參考點。

### 錯誤 4：沒有已知範圍卻使用 `<meter>`

```html
<!-- 不建議：沒有上下文範圍，12cm 本身不是 gauge -->
<p>半徑：<meter value="12">12cm</meter></p>
```

如果只是單純顯示數字，直接寫文字即可：

```html
<p>半徑：12cm</p>
```

如果真的要表達它在某個範圍中的位置，才需要補上完整範圍：

```html
<p>半徑：<meter min="0" max="20" value="12">12cm / 20cm</meter></p>
```

## 使用 `<meter>` 時的思考流程

寫 `<meter>` 之前，先問自己：

1. 這是狀態值，還是任務進度？
2. 這個值有明確的最小值與最大值嗎？
3. `value` 目前值是多少？
4. 是否需要用 `low` / `high` 區分低區間與高區間？
5. 哪一段才是理想區間？需要寫 `optimum` 嗎？
6. 是否有提供文字內容或標籤，讓使用者知道這個 gauge 代表什麼？

如果第一題答案是「任務進度」，通常就不該用 `<meter>`。

## 推薦寫法模板

### 基本模板

```html
<label for="score">考試分數：</label>
<meter id="score" min="0" max="100" value="85">85 分</meter>
```

### 含低、高、理想區間的模板

```html
<label for="temperature">室內溫度：</label>
<meter
  id="temperature"
  min="0"
  max="40"
  low="18"
  high="28"
  optimum="24"
  value="26"
>
  26°C
</meter>
```

這個例子代表：

- 溫度範圍是 `0°C` 到 `40°C`
- `18°C` 以下偏低
- `28°C` 以上偏高
- `24°C` 附近較理想
- 目前溫度是 `26°C`

## 常見問法

### `value` 一定要寫嗎？

實務上應該寫。

更精準地說：HTML 作者撰寫 `<meter>` 時，應明確指定 `value`。  
雖然瀏覽器在解析時可以對缺漏或格式錯誤的 `value` 做容錯，但沒有目前值會讓 `<meter>` 的語意不完整。

### 沒寫 `min` 和 `max` 會怎樣？

預設範圍會是 `0` 到 `1`。

所以：

```html
<meter value="0.7">70%</meter>
```

可以理解為 `0.7 / 1`。

但如果你想用百分比數字：

```html
<meter value="70" max="100">70%</meter>
```

就應該明確寫 `max="100"`。

### `meter` 和 `progress` 為什麼不能混用？

因為語意不同。

`meter` 表達的是測量值在已知範圍中的位置；`progress` 表達的是任務完成了多少。

- 電量 70%：狀態值，適合 `<meter>`
- 上傳 70%：任務進度，適合 `<progress>`

### `optimum` 是不是等於目前值？

不是。

- `value`：目前值
- `optimum`：理想值或理想區間的參考點

### `low` 和 `high` 是不是代表一定不好？

不一定。

`low` 和 `high` 只是切出低區間與高區間。  
哪一段比較理想，要看 `optimum` 落在哪裡。

例如：

- 電量：高區間通常比較理想
- 錯誤率：低區間通常比較理想
- 溫度：中間區間可能比較理想

### `<meter>` 可以單純拿來美化條狀 UI 嗎？

不建議。

如果只是想畫一個自訂樣式的條狀 UI，但資料本身不是「已知範圍內的測量值」，就不應該使用 `<meter>`。  
HTML 標籤要先看語意，再看外觀。

## 自測問題

1. `<meter>` 最適合表達哪一類資訊？
2. `<meter>` 和 `<progress>` 的最大差別是什麼？
3. 為什麼 `value="70"` 不一定代表 70%？
4. `min` 和 `max` 沒寫時，預設範圍是多少？
5. `low` 和 `high` 分別代表什麼邊界？
6. `optimum` 落在高區間時，通常代表什麼？
7. `optimum` 落在低區間時，通常代表什麼？
8. 為什麼 `<meter value="12">12cm</meter>` 這種寫法通常不合適？
9. 顯示「下載完成 80%」應該用 `<meter>` 還是 `<progress>`？
10. 顯示「磁碟已使用 420GB / 500GB」適合用 `<meter>` 嗎？為什麼？

## 一句話總結

`<meter>` 是用來表示「某個測量值落在已知範圍中的哪裡」；它描述的是狀態，不是任務進度。
