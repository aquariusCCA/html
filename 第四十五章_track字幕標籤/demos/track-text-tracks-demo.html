<track> 字幕標籤 Demo

這個範例把第 45 章筆記中的 <source> / <track> 分工、字幕與聽障字幕差異、WebVTT 格式、多語字幕、章節、metadata 與排查流程放在同一頁觀察。

<track> 放在 <audio> 或 <video> 裡，載入時間軸文字軌。
subtitles 翻譯字幕，通常假設使用者聽得到聲音。
captions 聽障字幕，包含對白、說話者、音效或音樂提示。
WebVTT 常見文字軌格式，檔案通常以 WEBVTT 開頭。
metadata 給 JavaScript 或播放器邏輯讀取的時間軸資料。

<source>

提供媒體檔來源，例如 MP4、WebM 或 MP3。

<track>

提供與媒體時間同步的文字資料，例如字幕、章節或 metadata。

default

布林屬性，通常一個媒體元素只讓一條文字軌預設啟用。

4. WebVTT 基本格式


          

            
格式重點

            
    
              
  • 檔案第一行通常是 WEBVTT。
    • 
              
  • 每段 cue 需要時間範圍，例如 00:00:00.000 --> 00:00:03.000。
    • 
              
  • 時間範圍下面放要顯示或讀取的文字。
    • 
              
  • 段落之間用空行分隔。
    • 
              
  • 原生 <track> 通常使用 WebVTT，不是 SRT。

5. kind 對照表

        

          
            

              kind
              用途
              適合情境
            

          
          
            

              subtitles
              翻譯字幕
              使用者聽得到聲音，但不懂影片語言。
            

            

              captions
              聽障字幕
              需要包含對白、說話者、音效、音樂提示。
            

            

              descriptions
              音訊描述
              描述重要視覺資訊，協助看不到畫面的使用者。
            

            

              chapters
              章節導航
              讓影片可依段落跳轉，依瀏覽器播放器 UI 支援顯示。
            

            

              metadata
              程式使用的時間軸資料
              用於自訂播放器、投影片同步、互動提示或觀看進度。
            

          
        

      


      

        
6. 字幕沒有顯示時的排查清單

        
    
          
  • <track> 是否放在 <video> 或 <audio> 裡。
    • 
          
  • 多個 <source> 時，<track> 是否放在 <source> 後面。
    • 
          
  • src 是否指向可讀取的 .vtt 檔案。
    • 
          
  • WebVTT 是否以 WEBVTT 開頭，時間格式是否正確。
    • 
          
  • kind="subtitles" 時是否提供 srclang。
    • 
          
  • label 是否是使用者看得懂的名稱。
    • 
          
  • 是否只有一條軌使用 default。
    • 
          
  • 正式伺服器是否用合適 MIME type，例如 text/vtt，並處理 CORS。
