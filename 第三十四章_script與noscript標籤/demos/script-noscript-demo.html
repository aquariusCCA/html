Script 與 Noscript Demo

這個範例示範 HTML 如何接入 JavaScript：內嵌腳本、外部腳本、`defer`、`async`、`type="module"`、`nomodule` 與 `<noscript>`。重點不是 JavaScript 語法，而是腳本下載、執行時機和退路設計。

核心觀念

下載腳本和執行腳本是兩件事

`defer`、`async`、`module` 的差異，主要就在於瀏覽器何時下載、何時執行，以及是否保留多個腳本的順序。

`defer`

不阻塞 HTML 解析地下載，等 HTML 解析完成後依照文件順序執行。

`async`

不阻塞下載，但下載完成後盡快執行，不保證多個腳本的先後順序。

`noscript`

只處理 JavaScript 停用或不支援，不會處理程式 bug、API 失敗或事件綁定錯誤。

1. 常見 `<script>` 寫法速查

inline `<script>...</script>`：少量初始化或範例測試可以使用，但不適合放大量程式碼。
src `<script src="./js/main.js"></script>`：把 JavaScript 放到外部檔案，利於快取、共用與維護。
defer `<script src="./js/main.js" defer></script>`：常用於頁面主程式和有依賴順序的腳本。
async `<script src="analytics.js" async></script>`：常用於統計、廣告、第三方獨立腳本。
module `<script type="module" src="./js/main.js"></script>`：現代 JavaScript 模組入口，預設有類似 `defer` 的特性。
nomodule `<script nomodule src="./legacy.js"></script>`：給不支援 module 的舊瀏覽器使用。

      4. `defer` 適合有依賴順序的腳本

      本頁實測

      `defer-library.js` 會先於 `defer-main.js` 執行

      即使下載速度不同，多個 `defer` 腳本仍會依 HTML 中出現的順序執行。這也是主程式依賴工具庫時常用 `defer` 的原因。

      <script src="./js/defer-library.js" defer></script>
<script src="./js/defer-main.js" defer></script>

      5. `async` 適合獨立腳本

      不保證順序

      不要用 `async` 載入彼此依賴的程式

      統計、廣告、追蹤工具通常不依賴頁面主程式，也不被主程式依賴，因此適合 `async`。如果 `main.js` 依賴 `library.js`，應改用 `defer`。

      <!-- 適合：獨立第三方腳本 -->
<script src="https://example.com/analytics.js" async></script>

<!-- 不適合：有依賴順序 -->
<script src="./js/library.js" async></script>
<script src="./js/main.js" async></script>

      6. `type="module"` 的基本特性

      模組作用域

      模組頂層變數不會自動變成全域變數

      等待 module 腳本回報。

      傳統想像

      以為頂層變數會出現在 `window` 上。

      模組實際行為

      模組有自己的作用域，跨檔案應透過 `import` / `export`。

      <script type="module">
  const moduleLocalName = "module scoped value";
  // moduleLocalName 不會自動變成 window.moduleLocalName
</script>

      7. `noscript` 和漸進增強

      Progressive Enhancement Form
      邊界

      `noscript` 不是錯誤處理工具

      如果使用者停用 JavaScript，頁面上方的 `<noscript>` 提示會顯示。但 JavaScript 語法錯誤、API 失敗、套件載入失敗，都需要程式本身處理。

      <noscript>
  <p>請啟用 JavaScript 以使用完整功能。</p>
</noscript>

      9. 實務選擇表

      情境 建議寫法 原因
      一般頁面主程式 `defer` 不阻塞解析，能等 DOM 解析完成，且保留多個腳本順序。
      第三方統計或廣告 `async` 通常獨立運作，不需要依賴頁面主程式。
      現代前端入口 `type="module"` 支援 `import` / `export`，預設類似 `defer`。
      舊瀏覽器備援 `nomodule` 讓不支援 module 的瀏覽器載入 legacy script。
      腳本停用提示 `<noscript>` 提供最低限度的替代內容或引導。

      `script` 接入 JavaScript

      可以內嵌，也可以用 `src` 載入外部檔案。實務專案通常優先外部檔案。

      `defer` 和 `async` 都要看執行時機

      `defer` 保序且延後，`async` 下載完成就盡快執行。

      `noscript` 只處理腳本不可用

      它不是 JavaScript 錯誤提示，也不能取代漸進增強。
