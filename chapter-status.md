## 流程步驟說明

| 流程步驟 | 輸入來源 | 輸出位置 | 完成標準 |
| --- | --- | --- | --- |
| origin整理 | 原始教材 / 手動筆記 | `origin/<章節>/*.md` | 原始 Markdown 已整理到章節根目錄，assets 放在 `origin/<章節>/assets/` |
| 資產命名 | `origin/<章節>/*.md` + assets | `origin/<章節>/assets/*` | 本地資產符合標準命名，引用路徑同步更新 |
| alt與連結文字 | `origin/<章節>/*.md` | `origin/<章節>/*.md` | 圖片 alt 與附件連結文字清楚且未猜測內容 |
| atomic切分提案 | `origin/<章節>/*.md` | 切分提案 | 已確認哪些內容要拆分、合併、移動或保留 |
| atomic產生 | `origin/<章節>/*.md` | `atomic/<章節>/*.md` | 每篇 atomic note 聚焦一個完整主題概念 |
| atomic內容審查 | `atomic/<章節>/*.md` + 必要時回查 origin | 審查報告 / 修正後 atomic | S1/S2 技術錯誤已修正或標記待確認 |
| notes生成 | `atomic/<章節>/*.md` | `notes/<章節>/*.md` | 正式教學筆記符合學習順序與教學品質 |
| notes完成檢查 | `notes/<章節>/*.md` | 檢查結果 | 標題、範例、錯誤對比、資產路徑、重點整理皆通過 |
| demos生成 | `notes/<章節>/<筆記名>.md` | `demos/<章節>/<筆記名>/` | 每個 demo 可直接開啟且聚焦單一觀念 |
| practice生成 | `notes/<章節>/` 或單篇 notes | `practice/<章節>/` | 題目分級清楚，能對應來源筆記 |
| review生成 | `notes/<章節>/` | `review/<章節>/` | 含題庫、間隔複習、錯題回流與檢查點 |
| appendix索引 | `notes/<章節>/<筆記名>.md` | `appendix/<章節>/<筆記名>/` | 索引項目能連到具體段落 |
| 最終驗收 | notes / demos / practice / review / appendix | 章節完成狀態 | 下游產物與 notes 一致，章節狀態可標示完成 |

## 使用方式

1. 將本檔放在 `html/chapter-status.md`。
2. 依你的實際章節替換「章節」欄位。
3. 每跑完一個 workflow，就更新對應欄位狀態。
4. 如果 `notes/` 有修改，請回頭檢查 `demos/`、`practice/`、`review/`、`appendix/` 是否需要重生成。
5. 只有當整章的正式筆記與下游材料一致時，才把「最終驗收」標記為「已完成」。
